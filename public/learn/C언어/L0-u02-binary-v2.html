0과 1의 세계

컴퓨터가 0과 1만 사용하는 이유를 이해하고, 10진수와 2진수를 자유롭게 변환하며, 비트/바이트/ASCII의 개념을 설명할 수 있습니다.

이 유닛에서 배우는 것

  • 컴퓨터가 0과 1만 사용하는 이유를 전기 스위치 비유로 설명할 수 있습니다.
  • 10진수를 2진수로, 2진수를 10진수로 변환할 수 있습니다.
  • 비트(bit)와 바이트(byte)의 관계와 표현 범위를 말할 수 있습니다.
  • ASCII 코드의 개념을 이해하고 대표적인 문자의 코드 값을 말할 수 있습니다.

이야기 — 컴퓨터는 왜 0과 1만 이해하는가

컴퓨터 내부는 수십억 개의 아주 작은 전기 스위치(트랜지스터)로 이루어져 있습니다. 각 스위치는 딱 두 가지 상태만 가질 수 있습니다: 전류가 흐르거나(ON), 흐르지 않거나(OFF). 이것이 컴퓨터가 0과 1만 이해하는 근본적인 이유입니다.

방의 전등 스위치를 떠올려 보십시오. 스위치를 올리면 불이 켜지고(1), 내리면 불이 꺼집니다(0). 컴퓨터의 트랜지스터도 이와 똑같습니다. 전압이 높으면 1, 낮으면 0입니다. 이 단순한 원리 위에 문자, 사진, 음악, 동영상 등 모든 정보가 표현됩니다.

핵심 원리 전기 스위치 = 0 또는 1
- OFF (전류 없음) = 0
- ON (전류 있음) = 1

스위치 1개 = 2가지 상태 표현 가능
스위치 2개 = 4가지 상태 (00, 01, 10, 11)
스위치 8개 = 256가지 상태

스위치를 많이 모을수록 더 많은 정보를 표현할 수 있습니다.

핵심 개념

0과 1, 두 개의 숫자만 사용하는 수 체계를 이진수(binary number)라고 합니다. 우리가 일상에서 사용하는 수 체계는 0부터 9까지 10개의 숫자를 사용하므로 십진수(decimal number)라고 부릅니다.

이진수의 원리는 전구 비유로 쉽게 이해할 수 있습니다. 전구 4개가 일렬로 놓여 있다고 상상해 보십시오. 각 전구는 켜짐(1) 또는 꺼짐(0) 두 가지 상태만 가집니다.

10진수를 2진수로 변환하는 방법

10진수를 2진수로 바꾸는 방법은 간단합니다. 숫자를 2로 계속 나누고, 나머지를 아래에서 위로 읽으면 됩니다. 이것을 "나누기 2 반복법"이라고 합니다.

1
1단계: 13 / 2 = 6 ... 나머지 1
13을 2로 나누면 몫은 6, 나머지는 1입니다.
2
2단계: 6 / 2 = 3 ... 나머지 0
몫 6을 다시 2로 나누면 몫은 3, 나머지는 0입니다.
3
3단계: 3 / 2 = 1 ... 나머지 1
몫 3을 다시 2로 나누면 몫은 1, 나머지는 1입니다.
4
4단계: 1 / 2 = 0 ... 나머지 1
몫 1을 다시 2로 나누면 몫은 0, 나머지는 1입니다. 몫이 0이 되면 중단합니다.
5
5단계: 나머지를 아래에서 위로 읽기
나머지를 역순으로 읽으면 1101입니다. 따라서 13(10) = 1101(2)입니다.
나눗셈몫나머지읽는 순서
13 / 2614번째 (맨 아래)
6 / 2303번째
3 / 2112번째
1 / 2011번째 (맨 위)
결과 나머지를 아래에서 위로 읽으면: 1 1 0 1
따라서 13(10진수) = 1101(2진수)

2진수를 10진수로 변환하는 방법

2진수를 10진수로 바꾸려면 각 자릿값을 계산하여 더합니다. 2진수의 자릿값은 오른쪽부터 1, 2, 4, 8, 16, 32 ... 순서입니다. 즉, 2의 0제곱, 2의 1제곱, 2의 2제곱 ... 입니다.

자릿수4번째3번째2번째1번째
자릿값8 (2^3)4 (2^2)2 (2^1)1 (2^0)
이진수 11011101
계산1 x 8 = 81 x 4 = 40 x 2 = 01 x 1 = 1
자릿값 합산 1101(2) = 8 + 4 + 0 + 1 = 13(10)

각 자리의 숫자(0 또는 1)에 해당 자릿값(2의 거듭제곱)을 곱한 뒤, 모두 더하면 10진수가 됩니다.

비트(bit)와 바이트(byte)

컴퓨터가 다루는 정보의 가장 작은 단위를 비트(bit)라고 합니다. bit는 binary digit(이진 숫자)의 줄임말입니다. 1비트는 0 또는 1, 딱 하나의 이진수 자리를 뜻합니다.

비트 8개를 묶은 것을 바이트(byte)라고 합니다. 1바이트는 8비트이며, 이것이 컴퓨터에서 데이터를 처리하는 기본 단위입니다.

비트로 표현할 수 있는 범위 n비트로 표현 가능한 경우의 수 = 2^n

- 1비트: 2^1 = 2가지 (0, 1)
- 2비트: 2^2 = 4가지 (00, 01, 10, 11)
- 3비트: 2^3 = 8가지 (000 ~ 111)
- 4비트: 2^4 = 16가지 (0000 ~ 1111)
- 8비트(1바이트): 2^8 = 256가지 (00000000 ~ 11111111)
- 16비트: 2^16 = 65,536가지
- 32비트: 2^32 = 약 42억 가지
단위비트 수표현 범위 (부호 없는 정수)일상 비유
1 비트10 ~ 1 (2가지)스위치 1개
1 니블40 ~ 15 (16가지)16진수 1자리
1 바이트80 ~ 255 (256가지)영문 글자 1개
2 바이트160 ~ 65,535유니코드 기본 문자
4 바이트320 ~ 약 42억C언어 int 자료형
8 바이트640 ~ 약 1,844경C언어 long long 자료형
데이터 크기 단위 1 KB (킬로바이트) = 1,024 바이트
1 MB (메가바이트) = 1,024 KB = 1,048,576 바이트
1 GB (기가바이트) = 1,024 MB
1 TB (테라바이트) = 1,024 GB

1,024 = 2^10이므로 컴퓨터에서는 1,000 대신 1,024를 기준으로 합니다.

문자도 숫자 — ASCII 코드

컴퓨터는 숫자만 이해합니다. 그렇다면 문자(A, B, C ...)는 어떻게 저장할까요? 답은 간단합니다. 각 문자에 고유한 번호를 부여하면 됩니다. 이 약속 체계를 ASCII(아스키) 코드라고 합니다.

ASCII는 American Standard Code for Information Interchange의 약자입니다. 1963년에 만들어졌으며, 7비트를 사용하여 총 128개의 문자를 정의합니다.

ASCII 규칙 3가지 1. 대문자 A~Z: 65~90 (연속 26개)
2. 소문자 a~z: 97~122 (연속 26개)
3. 대문자와 소문자의 차이: 항상 32 (A=65, a=97 → 97-65=32)

이 규칙은 C 프로그래밍에서 대소문자 변환에 직접 활용됩니다.
숫자 문자와 숫자 값은 다르다 문자 '0'의 ASCII 코드는 48입니다. 숫자 값 0과는 다릅니다.
문자 '1'의 ASCII 코드는 49입니다. 숫자 값 1과는 다릅니다.

문자 '0'~'9'의 ASCII 코드는 48~57입니다.
따라서 문자 '5'의 숫자 값을 구하려면: '5' - '0' = 53 - 48 = 5

4

5

L0-U02 핵심 정리
1. 컴퓨터는 전기 스위치(트랜지스터)로 이루어져 있으므로 0과 1만 이해합니다
