Chapter 03

입력 -> 처리 -> 출력 -- 모든 프로그램의 기본 구조

세상의 모든 프로그램은 입력을 받아 처리한 뒤 결과를 출력하는 구조로 이루어져 있음을 이해합니다.

이 유닛에서 배우는 것

  • 모든 프로그램이 입력, 처리, 출력의 세 단계로 구성됨을 설명할 수 있습니다.
  • 일상 속 사물(자판기, 계산기 등)에서 입력/처리/출력을 구분할 수 있습니다.
  • 하드웨어 관점에서 입력 장치, 처리 장치, 출력 장치를 대응시킬 수 있습니다.
  • 간단한 문제를 입력/처리/출력으로 분해하여 의사코드를 작성할 수 있습니다.

아직 코드가 아닙니다 위 세 줄은 프로그래밍 언어가 아닙니다. 사람의 말로 적은 것이지만, 이것만으로도 프로그램이 하는 일을 정확히 파악할 수 있습니다. 이처럼 사람의 말로 프로그램의 동작을 적은 것을 의사코드(pseudocode)라고 합니다.

이야기 -- 모든 것은 입력에서 시작된다

여러분이 아침에 일어나서 하는 일을 생각해 보십시오. 알람 소리가 들리면(입력), 뇌가 '일어나야 한다'고 판단하고(처리), 몸을 일으킵니다(출력). 배가 고프다는 느낌이 오면(입력), 냉장고에 뭐가 있는지 생각하고(처리), 음식을 꺼내 먹습니다(출력).

우리가 하루에 하는 거의 모든 행동이 이 세 단계를 따릅니다. 컴퓨터 프로그램도 마찬가지입니다. 세상에 존재하는 모든 프로그램은, 아무리 복잡해 보여도, 결국 '무언가를 받아서 / 처리한 뒤 / 결과를 내놓는' 구조입니다. 이것을 IPO(Input-Process-Output) 모델이라고 합니다.

이 개념을 확실히 이해하면, 앞으로 어떤 프로그래밍 문제를 만나든 '입력이 무엇인가 / 처리는 무엇인가 / 출력은 무엇인가'를 나누어 생각할 수 있게 됩니다. 이것이 프로그래밍적 사고의 출발점입니다.

핵심 개념 -- 입력, 처리, 출력

프로그램의 기본 구조를 세 단계로 정리하겠습니다.

1
입력 (Input)
프로그램이 외부로부터 데이터를 받는 단계입니다. 키보드 입력, 파일 읽기, 센서 데이터 수신, 네트워크 요청 수신 등이 해당합니다.
2
처리 (Process)
받은 데이터를 규칙에 따라 변환, 계산, 판단하는 단계입니다. 프로그램의 핵심 로직이 여기에 해당합니다.
3
출력 (Output)
처리 결과를 외부로 내보내는 단계입니다. 화면 표시, 파일 저장, 소리 출력, 네트워크 전송 등이 해당합니다.
IPO 모델의 핵심 원리 모든 프로그램 = 입력 + 처리 + 출력

아무리 복잡한 프로그램이라도 이 세 가지를 벗어나지 않습니다.
- 카카오톡: 메시지 입력 -> 서버 전송 처리 -> 상대방 화면에 출력
- 유튜브: 검색어 입력 -> 관련 영상 탐색 처리 -> 영상 목록 출력
- 내비게이션: 목적지 입력 -> 최단 경로 계산 처리 -> 경로 안내 출력

일상 속 입력/처리/출력 -- 자판기

가장 쉬운 예시부터 살펴보겠습니다. 자판기는 입력/처리/출력 구조를 가장 명확하게 보여주는 기계입니다.

1
입력
동전(또는 지폐)을 넣고, 원하는 음료의 버튼을 누릅니다.
2
처리
투입 금액이 음료 가격 이상인지 확인합니다. 충분하면 해당 음료를 준비합니다. 부족하면 '금액 부족' 신호를 보냅니다.
3
출력
음료가 배출구로 나오고, 거스름돈이 반환됩니다.

자판기의 '처리' 단계에 주목하십시오. 단순히 음료를 내보내는 것이 아니라, '금액이 충분한가?'를 판단합니다. 이 판단이 바로 프로그래밍에서 말하는 조건 분기입니다. 프로그램의 처리 단계에서는 항상 이런 판단과 계산이 일어납니다.

일상 속 입력/처리/출력 -- 계산기와 검색엔진

계산기와 검색엔진도 같은 구조를 따릅니다.

사물입력처리출력

계산기는 처리가 단순합니다(사칙연산). 검색엔진은 처리가 매우 복잡합니다(수십억 페이지 분석). 하지만 둘 다 입력/처리/출력이라는 동일한 구조를 따릅니다. 프로그램마다 다른 것은 '처리'의 내용뿐이고, 전체 구조는 항상 같습니다.

하드웨어 관점 -- 입력 장치, 처리 장치, 출력 장치

하드웨어와 IPO의 대응 입력 장치: 키보드, 마우스, 마이크, 카메라, 터치스크린, 센서

메모리(RAM)의 역할 -- 처리 중 데이터 임시 저장

RAM -- 작업 책상 CPU가 처리를 수행하려면 데이터를 어딘가에 올려놓아야 합니다. 그 장소가 바로 RAM(메모리)입니다.

비유: 책상 위에서 공부하는 상황을 생각하십시오.
- 책장(하드디스크/SSD) = 데이터가 영구 보관되는 곳
- 책상(RAM) = 지금 작업 중인 데이터를 올려놓는 곳
- 머리(CPU) = 실제로 읽고 계산하는 곳

책장에서 책을 꺼내 책상에 올려놓고(입력), 읽으면서 문제를 풀고(처리), 답안지를 작성합니다(출력). 컴퓨터도 정확히 같은 방식으로 동작합니다.

중요: RAM은 전원이 꺼지면 내용이 사라집니다. 그래서 결과를 영구 보존하려면 하드디스크/SSD에 저장(출력)해야 합니다.

정리하면, 컴퓨터의 데이터 흐름은 다음과 같습니다: 입력 장치 -> RAM -> CPU(처리) -> RAM -> 출력 장치. RAM은 입력과 처리 사이, 처리와 출력 사이에서 데이터를 임시로 보관하는 중간 저장소 역할을 합니다.

프로그램으로 보는 입력/처리/출력 흐름

"두 수의 합 구하기" 프로그램을 입력/처리/출력으로 분해해 보겠습니다. 아직 C 코드가 아니라 의사코드(pseudocode)로 작성합니다.

1
[입력] 숫자 두 개를 받는다
사용자가 키보드로 숫자 두 개를 입력합니다. 예: 첫 번째 숫자 = 3, 두 번째 숫자 = 5
2
[처리] 두 숫자를 더한다
CPU가 3 + 5를 계산합니다. 계산 결과 8이 RAM에 저장됩니다.
3
[출력] 결과를 화면에 표시한다
RAM에 저장된 8을 모니터 화면에 출력합니다. 예: "3 + 5 = 8"

축 10: 하드웨어 연결 [선택]
키보드(입력 장치) -> CPU(처리 장치) -> 모니터(출력 장치). RAM은 처리 중 데이터를 임시 저장합니다.
