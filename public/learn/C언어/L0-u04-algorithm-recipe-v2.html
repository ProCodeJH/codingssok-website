Chapter 04

알고리즘은 요리 레시피

알고리즘이 무엇인지 이해하고, 요리 레시피 비유를 통해 입력-처리-출력 구조를 파악하며, 좋은 알고리즘의 조건을 학습합니다.

이 유닛에서 배우는 것

  • 알고리즘의 정의를 자신의 말로 설명할 수 있습니다.
  • 요리 레시피 비유를 통해 입력-처리-출력 구조를 구분할 수 있습니다.
  • 좋은 알고리즘의 조건 3가지(명확성, 유한성, 효율성)를 나열할 수 있습니다.
  • 일상생활에서 알고리즘의 예시를 3가지 이상 찾아 설명할 수 있습니다.

핵심 개념 1 — 알고리즘의 정의

undefined

1
입력 (Input)
문제를 풀기 위해 주어지는 데이터입니다. 예: 정렬할 숫자들, 검색할 단어
2
처리 (Process)
입력을 원하는 결과로 바꾸는 단계별 절차입니다. 예: 비교, 교환, 계산
3
출력 (Output)
처리 과정을 거쳐 얻어진 최종 결과입니다. 예: 정렬된 숫자들, 검색 결과
알고리즘의 핵심 구조 입력 --> 단계별 절차(처리) --> 출력

모든 알고리즘은 이 세 가지 요소를 반드시 갖추어야 합니다. 입력이 없는 알고리즘, 출력이 없는 알고리즘은 의미가 없습니다.

핵심 개념 3 — 좋은 알고리즘의 조건

undefined

1
명확성 (Definiteness)
각 단계가 모호하지 않고 정확해야 합니다. "적당히 섞는다"는 알고리즘이 될 수 없습니다. "3번 저어서 섞는다"처럼 구체적이어야 합니다.
2
유한성 (Finiteness)
반드시 유한한 단계 안에 끝나야 합니다. 영원히 끝나지 않는 절차는 알고리즘이 아닙니다.
3
효율성 (Effectiveness)
각 단계가 실행 가능해야 합니다. "내일의 날씨를 정확히 맞혀라"처럼 실행 불가능한 단계가 포함되면 안 됩니다.

알고리즘의 표현 방법

알고리즘을 표현하는 3가지 방법 1. 자연어 (Natural Language)
일상 언어(한국어, 영어)로 각 단계를 설명합니다.
장점: 누구나 읽을 수 있습니다.
단점: 모호해질 수 있습니다.

2. 순서도 (Flowchart)
도형과 화살표로 절차를 시각적으로 표현합니다.
장점: 흐름이 한눈에 보입니다.
단점: 복잡한 알고리즘은 그리기 어렵습니다.

3. 의사코드 (Pseudocode)
프로그래밍 언어와 비슷하지만 엄격한 문법 없이 작성합니다.
장점: 코드로 변환하기 쉽습니다.
단점: 프로그래밍 기초 지식이 필요합니다.

이번 유닛에서는 주로 자연어를 사용했습니다. 다음 유닛에서 순서도를 본격적으로 학습합니다.

오류 1: 순서 오류 알고리즘: 계란 프라이 만들기 (잘못된 버전)

1단계: 계란을 프라이팬에 올린다.
2단계: 불을 켠다.
3단계: 기름을 두른다.

문제: 불을 켜기 전에 계란을 올렸습니다. 기름도 없는 차가운 프라이팬에 계란을 올리면 실패합니다.
수정: 불 켜기 -> 기름 두르기 -> 계란 올리기 순서가 올바릅니다.
오류 2: 종료 조건 없음 알고리즘: 물 끓이기 (잘못된 버전)

1단계: 냄비에 물을 넣는다.
2단계: 불을 켠다.
3단계: 기다린다.
4단계: 3단계로 돌아간다.

문제: 영원히 기다리기만 합니다. 유한성 조건을 위반합니다.
수정: 3단계를 "물이 끓으면 4단계로, 아니면 기다린다"로 바꾸어야 합니다.
오류 3: 모호한 단계 알고리즘: 가장 큰 수 찾기 (잘못된 버전)

1단계: 숫자들을 본다.
2단계: 가장 큰 수를 찾는다.
3단계: 출력한다.

문제: "가장 큰 수를 찾는다"가 어떻게 찾는지 설명하지 않습니다. 명확성 조건을 위반합니다.
수정: 하나씩 비교하며 최대값을 갱신하는 구체적인 절차가 필요합니다.

용어설명
알고리즘 (Algorithm)문제를 해결하기 위한 단계별 절차입니다. 입력을 받아 처리하여 출력을 만들어 냅니다.
입력 (Input)알고리즘에 주어지는 데이터입니다. 문제를 풀기 위해 필요한 정보입니다.
출력 (Output)알고리즘의 실행 결과입니다. 문제의 답에 해당합니다.
명확성 (Definiteness)알고리즘의 각 단계가 모호하지 않고 정확하게 정의되어 있는 성질입니다.
유한성 (Finiteness)알고리즘이 반드시 유한한 단계 안에 종료되는 성질입니다.
효율성 (Effectiveness)알고리즘의 각 단계가 실제로 실행 가능한 성질입니다.
순서도 (Flowchart)도형과 화살표를 사용하여 알고리즘의 흐름을 시각적으로 표현하는 방법입니다.
의사코드 (Pseudocode)프로그래밍 언어와 유사하지만 엄격한 문법 없이 알고리즘을 표현하는 방법입니다.
트레이싱 (Tracing)알고리즘을 한 단계씩 따라가며 변수의 값 변화를 추적하는 것입니다.
조건 분기조건에 따라 서로 다른 경로로 진행하는 알고리즘 구조입니다.
