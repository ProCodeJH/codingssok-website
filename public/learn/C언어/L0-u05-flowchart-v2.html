순서도 그리기 규칙

올바른 순서도를 위한 7가지 규칙 1. 위에서 아래로, 왼쪽에서 오른쪽으로 그립니다.
2. 시작 타원은 반드시 하나, 끝 타원도 하나입니다.
3. 화살표는 한 방향으로만 그립니다 (양방향 화살표 금지).
4. 마름모(조건)에서는 반드시 "예"와 "아니오" 두 갈래로 나뉩니다.
5. 각 기호에서 나가는 화살표는 하나뿐입니다 (마름모만 예외로 두 개).
6. 흐름선이 교차하지 않도록 배치합니다.
7. 기호 안의 문장은 간결하고 명확하게 적습니다.

에러 체험

잘못된 순서도 1: 끝이 없는 순서도 [시작] → [숫자 입력] → [숫자 + 1 계산] → [결과 출력]

문제: 끝 타원이 없습니다. 프로그램이 어디서 종료되는지 알 수 없습니다.
해결: 마지막에 반드시 [끝] 타원을 추가하십시오.
잘못된 순서도 2: 조건에 한쪽 경로만 있음 [시작] → /숫자 입력/ → <숫자 > 0?> —예→ /"양수" 출력/ → [끝]

문제: 아니오 경로가 없습니다. 숫자가 0 이하이면 어디로 가야 할지 모릅니다.
해결: 마름모에서는 반드시 예/아니오 두 경로를 모두 그리십시오.
잘못된 순서도 3: 반복에서 변화 누락 [시작] → [i = 1] → —예→ /i 출력/ → (되돌아감)

문제: i = i + 1 처리가 없으므로 i가 영원히 1입니다. 무한 반복에 빠집니다.
해결: 반복 내부에 반드시 조건을 변화시키는 처리를 포함하십시오.

요약

핵심 정리
1. 순서도: 알고리즘의 흐름을 기호와 화살표로 시각화한 그림
2. 5가지 기호: 타원(시작/끝), 직사각형(처리), 마름모(조건), 평행사변형(입출력), 화살표(흐름)
3. 3가지 구조: 순차(위→아래), 선택(마름모 분기), 반복(화살표 되돌아감)
4. 그리기 규칙: 위→아래, 시작/끝 각 1개, 마름모는 예/아니오 2갈래
5. 순서도 → 코드: 순서도를 먼저 그리면 코드 작성이 쉬워집니다

다음 유닛으로

선택 구조조건에 따라 서로 다른 경로를 실행하는 구조입니다.
반복 구조조건이 참인 동안 같은 처리를 되풀이하는 구조입니다.
