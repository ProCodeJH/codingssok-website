의사코드란 무엇인가

의사코드(Pseudo Code)는 프로그래밍 언어와 자연어(한국어, 영어)의 중간에 위치하는 표현 방식입니다. "의사(pseudo)"는 "가짜"라는 뜻이므로, 의사코드는 "가짜 코드" 즉, 실제 프로그래밍 언어는 아니지만 코드처럼 보이는 글입니다.

의사코드는 컴퓨터가 실행할 수 없습니다. 컴파일러도, 인터프리터도 의사코드를 이해하지 못합니다. 그렇다면 왜 의사코드를 쓸까요? 프로그래밍 언어의 문법에 얽매이지 않고 알고리즘의 논리(로직)에만 집중할 수 있기 때문입니다.

의사코드 = 사람이 읽을 수 있는 프로그램 설계도 목적: 코드를 작성하기 전에 알고리즘의 논리를 정리하는 도구
대상: 컴퓨터가 아닌, 사람(자신 포함)이 읽기 위한 것
특징: 특정 프로그래밍 언어의 문법을 따르지 않음
비유: 건물을 짓기 전의 설계도, 요리를 시작하기 전의 레시피

전문 프로그래머도 복잡한 문제를 풀 때 바로 코드를 작성하지 않습니다. 먼저 의사코드로 논리를 정리한 뒤, 그것을 프로그래밍 언어로 변환합니다. 의사코드는 초보자만 쓰는 것이 아니라, 모든 수준의 프로그래머가 사용하는 핵심 도구입니다.

의사코드 작성 규칙

의사코드에는 엄격한 문법 규칙이 없습니다. 하지만 누구나 읽을 수 있으려면 최소한의 약속이 필요합니다. 이 교재에서는 다음 규칙을 따릅니다.

1
규칙 1: 한 줄에 한 동작
각 줄에는 하나의 명령만 적습니다. '변수 A에 10을 넣고 B에 20을 넣는다'가 아니라, 두 줄로 나누어 적습니다.
2
규칙 2: 들여쓰기로 구조 표현
조건문이나 반복문 안에 속하는 동작은 한 단계 들여씁니다. 어떤 동작이 어디에 속하는지 한눈에 파악할 수 있습니다.
3
규칙 3: 시작과 끝 표시
알고리즘의 시작은 '시작', 끝은 '끝'으로 명시합니다.
4
규칙 4: 구체적으로 작성
'적절히 처리한다'와 같은 모호한 표현을 쓰지 않습니다. 컴퓨터가 따라할 수 있을 만큼 구체적으로 적습니다.
5
규칙 5: 일관된 표현 사용
'입력받는다', '읽는다'를 섞어 쓰지 않습니다. 한 가지 표현을 정하면 끝까지 같은 표현을 사용합니다.

기본 구조 1 - 순차 구조

순차(Sequential) 구조는 가장 단순한 형태입니다. 명령을 위에서 아래로 차례대로 실행합니다. 모든 알고리즘의 기본입니다.

Csequential.txt
1시작
2 이름을 입력받는다
3 "안녕하세요, " + 이름 + "님!" 을 출력한다
4끝

위 의사코드에서 2번 줄이 먼저 실행되고, 3번 줄이 그다음에 실행됩니다. 순서를 바꾸면 이름을 입력받기 전에 출력하게 되므로 논리가 틀어집니다. 순서가 중요합니다.

기본 구조 2 - 조건 구조

조건(Selection) 구조는 특정 조건에 따라 다른 동작을 수행합니다. "만약 ~이면"이라는 표현을 사용합니다.

Ccondition.txt
1시작
2 숫자 변수 점수를 입력받는다
3 만약 점수 >= 60 이면:
4 "합격" 을 출력한다
5 아니면:
6 "불합격" 을 출력한다
7끝

들여쓰기에 주목하십시오. '"합격"을 출력한다'는 '만약 점수 >= 60 이면' 아래에 들여쓰기 되어 있습니다. 이것은 이 동작이 조건이 참일 때만 실행된다는 뜻입니다. 들여쓰기가 없으면 어떤 동작이 조건 안에 속하는지 알 수 없습니다.

기본 구조 3 - 반복 구조

반복(Iteration) 구조는 특정 조건이 만족되는 동안 같은 동작을 되풀이합니다. "~동안 반복" 또는 "~번 반복"이라는 표현을 사용합니다.

Cloop.txt
1시작
2 숫자 변수 i = 1
3 i가 10 이하인 동안 반복:
4 i를 출력한다
5 i = i + 1
6끝

위 의사코드는 1부터 10까지 숫자를 출력합니다. 반복할 동작(출력과 증가)이 반복문 아래에 들여쓰기 되어 있습니다. i가 11이 되면 조건 'i가 10 이하'가 거짓이 되어 반복이 끝납니다.

세 가지 기본 구조 정리 순차: 위에서 아래로 차례대로 실행
조건: 만약 ~이면 / 아니면 (분기)
반복: ~동안 반복 (되풀이)

이 세 가지 구조만으로 모든 알고리즘을 표현할 수 있습니다. 이것을 구조적 프로그래밍(Structured Programming)이라고 합니다.

자연어 vs 의사코드 vs 프로그래밍 언어

같은 알고리즘을 자연어, 의사코드, 프로그래밍 언어로 각각 표현하면 어떻게 다른지 비교해 봅시다. "두 수 중 큰 수를 출력하는" 알고리즘입니다.

예제 2 - 거스름돈 계산

물건 가격과 지불 금액을 입력받아, 거스름돈을 500원, 100원, 50원, 10원 동전으로 나누어 주는 알고리즘입니다. 순차 구조와 나눗셈(몫, 나머지) 연산을 활용합니다.

예제 3 - 구구단 출력

특정 단의 구구단을 출력하는 알고리즘입니다. 반복 구조를 사용합니다.

Cmultiplication.txt
1시작
2 숫자 변수 단을 입력받는다
3 숫자 변수 i = 1
4 i가 9 이하인 동안 반복:
5 결과 = 단 * i
6 단 + " x " + i + " = " + 결과 를 출력한다
7 i = i + 1
8끝
반복 횟수단i결과출력
1회3133 x 1 = 3
2회3263 x 2 = 6
3회3393 x 3 = 9
...3.........
9회39273 x 9 = 27

반복문에서 i는 1부터 시작하여 매 반복마다 1씩 증가합니다. i가 10이 되면 '9 이하'라는 조건이 거짓이 되어 반복이 멈춥니다. 총 9번 반복하므로 구구단 한 단(1~9)이 모두 출력됩니다.

반복문의 세 가지 요소 모든 반복문에는 세 가지 요소가 있습니다.

1. 초기화: i = 1 (반복 변수의 시작값 설정)
2. 조건: i가 9 이하인 동안 (언제까지 반복할지)
3. 갱신: i = i + 1 (반복 변수를 변화시킴)

셋 중 하나라도 빠지면 반복이 의도대로 작동하지 않습니다. 특히 갱신을 빠뜨리면 무한 반복에 빠집니다.

의사코드에서 C코드로 - 미리보기

아직 C언어 문법을 배우지 않았지만, 의사코드가 실제 프로그래밍 언어로 어떻게 변환되는지 맛보기로 살펴봅니다. 자세한 문법은 다음 Layer에서 학습합니다.

의사코드 -> C코드 변환 미리보기 의사코드의 각 표현이 C언어에서 어떤 형태가 되는지 대응 관계를 보여드립니다.

의사코드: 숫자 변수 A를 입력받는다
C코드: scanf("%d", &a);

의사코드: 만약 A > B 이면:
C코드: if (a > b) {

의사코드: i가 10 이하인 동안 반복:
C코드: while (i <= 10) {

의사코드: A를 출력한다
C코드: printf("%d", a);

의사코드를 정확히 작성할 수 있다면, C코드로 변환하는 것은 단순히 문법을 맞추는 작업입니다. 논리는 이미 완성되어 있기 때문입니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 연결: 의사코드는 C코드 작성의 전 단계입니다
깊이: LAYER 0 (PRE-CODING) - 아직 C코드를 작성하지 않으며, 논리 설계에 집중
변환 미리보기: '만약 ~이면' -> if, '~동안 반복' -> while, '입력받는다' -> scanf, '출력한다' -> printf
다음 단계: Layer 1에서 의사코드를 실제 C코드로 변환하는 연습을 합니다
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 알고리즘적 사고(Algorithmic Thinking)
의사코드 작성은 알고리즘적 사고의 핵심 훈련입니다:
- 문제를 분석하고 -> 해결 절차를 설계하고 -> 그 절차를 명확한 언어로 표현
- 순차, 조건, 반복 세 구조의 조합으로 모든 문제를 해결할 수 있습니다

이 세 구조를 자유자재로 사용하는 것이 프로그래밍 능력의 핵심입니다.
축 3: 디버깅 [필수] 이 유닛의 디버깅 포인트:
1. 반복문에서 갱신(i = i + 1)을 빠뜨리면 무한 반복
2. 조건문에서 '만약'을 연달아 쓰면 중복 실행 발생 -> '아니고 만약' 사용
3. 변수를 사용하기 전에 선언/입력하지 않는 실수

의사코드 단계에서 논리 오류를 잡으면, 코드 작성 후의 디버깅 시간을 크게 줄일 수 있습니다.
축 4: KOI - 정보올림피아드 [필수] 난이도: TIER 0 (준비 단계)
정보올림피아드 문제 풀이 과정:
1. 문제 읽기 -> 2. 입출력 분석 -> 3. 의사코드로 풀이 설계 -> 4. 코드 작성 -> 5. 테스트

3단계(의사코드 설계)를 건너뛰고 바로 코드를 작성하는 학생이 많지만, 어려운 문제일수록 의사코드를 먼저 작성하는 습관이 결정적인 차이를 만듭니다.
축 5: 코딩 습관 [선택] PRE-CODING 습관:
- 코드를 작성하기 전에 반드시 의사코드 또는 순서도로 설계하기
- 추적표를 그려서 의사코드의 논리를 검증하기
- 한 줄에 한 동작, 들여쓰기로 구조 표현하기

이 습관은 프로그래밍 언어를 배운 뒤에도 그대로 적용됩니다.
축 6: 실생활 연결 [선택] 의사코드는 프로그래밍에만 쓰이는 것이 아닙니다.

- 요리 레시피: 순서대로 적힌 조리법이 곧 순차 구조의 의사코드
- 응급 처치 매뉴얼: '의식이 있으면 ~, 없으면 ~' -> 조건 구조
- 운동 루틴: '3세트 반복' -> 반복 구조
- 업무 프로세스: 회사의 업무 절차서도 의사코드와 같은 구조
