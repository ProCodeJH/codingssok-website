패턴 인식 -- 반복되는 것 찾기

숫자, 도형, 실생활에서 반복되는 규칙(패턴)을 발견하고, 이를 프로그래밍의 반복문으로 연결하는 사고력을 기릅니다.

이야기 -- 패턴을 발견한 사람들

인류의 위대한 발견 대부분은 패턴 인식에서 시작되었습니다. 고대 바빌로니아인은 별의 움직임에서 패턴을 발견하여 계절을 예측했고, 뉴턴은 사과가 떨어지는 현상에서 중력의 패턴을 발견했습니다.

독일의 수학자 가우스(Carl Friedrich Gauss)는 10살 때 선생님이 낸 문제 "1부터 100까지 모두 더하시오"를 순식간에 풀었습니다. 다른 아이들이 1+2+3+...을 하나씩 더하는 동안, 가우스는 패턴을 발견했습니다. 1+100=101, 2+99=101, 3+98=101... 이렇게 101이 50쌍이므로 답은 101 x 50 = 5050입니다.

이것이 바로 패턴 인식의 힘입니다. 같은 문제라도 패턴을 발견하면 훨씬 빠르고 정확하게 풀 수 있습니다. 프로그래밍에서도 마찬가지입니다. 패턴을 발견하면 수백 줄의 코드를 몇 줄로 줄일 수 있습니다.

핵심 개념 -- 패턴 인식이란

패턴 인식(Pattern Recognition)이란, 데이터나 현상 속에서 반복되거나 규칙적인 것을 발견하는 능력입니다. 컴퓨팅 사고력(Computational Thinking)의 4가지 핵심 요소 중 하나이며, 프로그래밍의 기초가 되는 사고 방식입니다.

패턴 인식의 정의 패턴 인식이란: 반복되거나 규칙적인 것을 발견하는 능력

핵심 질문:
- 이것은 이전에 본 것과 비슷한가?
- 어떤 규칙이 반복되고 있는가?
- 이 규칙을 일반화할 수 있는가?

컴퓨팅 사고력 4요소: 분해 → 패턴 인식 → 추상화 → 알고리즘 설계

프로그래밍에서 패턴 인식의 역할

패턴을 발견하면 반복문(loop)으로 자동화할 수 있습니다. 반복문이란 같은 작업을 조건이 만족되는 동안 반복하는 프로그래밍 구조입니다. 패턴을 모르면 모든 것을 일일이 작성해야 하고, 패턴을 알면 규칙만 작성하면 됩니다.

위 비교에서 왼쪽은 패턴을 인식하지 못한 경우입니다. 줄이 10개면 10번 작성하고, 100개면 100번 작성해야 합니다. 오른쪽은 "n번째 줄에 별 n개"라는 패턴을 발견한 경우입니다. 숫자만 바꾸면 어떤 크기든 대응할 수 있습니다.

패턴 인식의 3단계

1
1단계: 관찰
데이터를 나열하고, 변하는 것과 변하지 않는 것을 구분합니다. 예: 별 찍기에서 줄 번호는 변하고, 별 모양(*)은 변하지 않습니다.
2
2단계: 규칙 발견
변하는 것 사이의 관계를 식으로 표현합니다. 예: n번째 줄의 별 개수 = n개.
3
3단계: 일반화
발견한 규칙을 모든 경우에 적용할 수 있는 형태로 정리합니다. 예: 1부터 N까지 반복하며 각 줄에 별 n개 출력.
패턴 인식에서 반복문으로 패턴 인식 → 반복문 → 효율적 프로그램

1. 데이터에서 규칙(패턴)을 발견합니다.
2. 규칙을 수식으로 표현합니다.
3. 수식을 반복문의 조건과 본문에 넣습니다.
4. 컴퓨터가 반복 실행합니다.

패턴 인식은 "사람이 규칙을 발견"하는 단계이고, 반복문은 "컴퓨터가 규칙을 실행"하는 단계입니다.

실생활 속의 패턴

패턴은 수학과 프로그래밍뿐 아니라 일상생활 곳곳에 존재합니다. 패턴을 인식하면 예측이 가능해지고, 예측이 가능해지면 계획을 세울 수 있습니다.

영역패턴 예시규칙활용
달력7일마다 같은 요일 반복요일 = 일수 % 7100일 뒤 요일 계산
시간표매주 같은 요일에 같은 수업주기 = 7일시간 관리, 일정 계획
구구단2x1=2, 2x2=4, 2x3=6...2 x n (n=1~9)곱셈 암기, 반복문 기초
음악같은 멜로디가 반복 (후렴구)주기적 반복작곡, 음악 구조 분석
계절봄-여름-가을-겨울 반복12개월 주기농업, 의류 산업 계획
버스배차 간격 15분도착 시각 = 첫차 + 15n분도착 시간 예측
구구단과 반복문 구구단은 패턴 인식과 반복문의 관계를 가장 잘 보여주는 예입니다.

2단: 2x1=2, 2x2=4, 2x3=6, ..., 2x9=18
3단: 3x1=3, 3x2=6, 3x3=9, ..., 3x9=27

패턴: a x b (a는 단, b는 1부터 9까지)

이 패턴을 발견하면 9x9=81줄을 일일이 적는 대신, 반복문 2개만으로 구구단 전체를 출력할 수 있습니다.
