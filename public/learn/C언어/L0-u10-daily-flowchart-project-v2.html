Chapter 10

나의 하루를 순서도+의사코드로

순서도, 의사코드, 분해, 패턴, 추상화를 종합하여 '나의 하루'를 체계적으로 설계합니다. 3단계 버전업을 통해 순차에서 조건, 반복, 추상화까지 확장합니다.

이 유닛에서 배우는 것

  • 순서도와 의사코드를 사용하여 일상 문제를 절차적으로 표현할 수 있습니다.
  • 조건 분기와 반복 구조를 순서도에 자연스럽게 적용할 수 있습니다.
  • 분해, 패턴 인식, 추상화를 하나의 프로젝트에 통합할 수 있습니다.
  • Layer 0에서 배운 컴퓨팅 사고 도구를 종합하여 실생활 문제를 해결할 수 있습니다.

예측

이야기

이번 유닛은 Layer 0의 마무리 프로젝트입니다. 지금까지 배운 순서도, 의사코드, 분해, 패턴 인식, 추상화를 모두 활용하여 '나의 하루'를 체계적으로 설계합니다. 프로젝트는 3단계로 진행됩니다. v1에서 가장 단순한 순차 구조로 시작하고, v2에서 조건 분기를 추가하고, v3에서 반복과 추상화까지 확장합니다.

핵심 개념

3-Version 프로젝트 구조 v1 기본: 아침 등교 준비 (순차 구조만)
v2 확장: 하루 일과 (조건 분기 포함: 비가 오면? 시험이 있으면?)
v3 완성: 일주일 시간표 관리 (반복 + 조건 + 분해 + 패턴 + 추상화 모두 활용)

각 버전에서 순서도와 의사코드를 모두 작성합니다.

v1은 가장 단순한 형태입니다. 아침에 일어나서 학교에 도착하기까지의 과정을 순차 구조만으로 표현합니다.

v3는 Layer 0에서 배운 모든 도구를 총동원합니다. 하루가 아니라 일주일을 관리해야 하므로 반복이 필요합니다. 요일마다 시간표가 다르므로 조건 분기가 필요합니다. 전체를 한 번에 다루면 복잡하므로 분해가 필요합니다.

v3 완성 — 분해, 패턴, 추상화 적용 분해: 일주일을 7개 하위 절차로 나눕니다.
1. 아침_루틴() - 매일 동일
2. 등교_루틴() - 날씨 조건
3. 오전_수업(요일) - 요일별 과목
4. 오후_수업(요일) - 시험/동아리 분기
5. 하교_루틴(요일) - 학원 여부
6. 저녁_루틴() - 매일 동일
7. 주말_루틴(요일) - 토/일 별도

패턴 인식:
- 매일 반복: 아침 루틴(알람-기상-세수-식사), 저녁 루틴(저녁-숙제-취침)
- 요일별 변화: 오전/오후 과목, 방과후 활동

추상화:
- 추상화 전: 월~일 7일을 각각 따로 작성 (7배 분량)
- 추상화 후: '하루_진행(요일)' 하나의 틀로 통합

프로그램: 일주일_시간표_관리

절차 아침_루틴():
  알람이 울린다

절차 오전_수업(요일):
  만약 요일 = 월요일: 국어, 수학, 영어
  만약 요일 = 화요일: 과학, 체육, 음악
  만약 요일 = 수요일: 수학, 사회, 미술
  만약 요일 = 목요일: 영어, 국어, 과학
  만약 요일 = 금요일: 체육, 수학, 사회

절차 오후_수업(요일):
  만약 오늘 시험이 있으면:
    시험을 본다
  아니면:
    만약 요일 = 수요일: 동아리 활동
    아니면: 일반 수업

절차 하교_루틴(요일):
  만약 요일 = 화요일 또는 목요일:
    학원에 간다
  아니면:
    집에 간다

절차 저녁_루틴():
  저녁을 먹는다
  숙제를 한다
  내일 준비물을 확인한다
  잠을 잔다

  늦잠을 잔다
  만약 요일 = 토요일: 취미 활동
  만약 요일 = 일요일: 밀린 숙제, 다음 주 준비

Unit 10 핵심 정리
1. v1(기본): 순차 구조만으로 아침 등교 준비를 표현합니다 - 빠짐없이 순서대로 나열하는 것이 기본
2. v2(확장): 조건 분기를 추가하여 현실적인 하루 일과를 표현합니다 - 마름모로 예/아니오 경로 분리
3. v3(완성): 반복 + 분해 + 패턴 + 추상화를 모두 활용하여 일주일 시간표를 관리합니다
4. 분해는 큰 문제를 작은 절차로 나누고, 패턴은 반복 구조를 찾고, 추상화는 이를 일반화합니다
5. 순서도와 의사코드는 프로그래밍 전에 논리를 설계하는 핵심 도구입니다
6. 3-Version 패턴: 단순 -> 확장 -> 완성으로 점진적 확장이 올바른 설계 방법입니다

다음 유닛으로

Layer 0 완주 Layer 0을 완주했습니다.

순서도, 의사코드, 분해, 패턴 인식, 추상화 - 프로그래밍의 기초 체력을 모두 갖추었습니다.

다음 Layer 1에서 진짜 C 프로그래밍을 시작합니다! Layer 0에서 설계한 순서도와 의사코드가 어떻게 실제 C 코드로 변환되는지 직접 경험하게 됩니다.

Layer 1 첫 번째 유닛: Hello, World! - 첫 번째 C 프로그램을 작성하고 실행합니다.

UNIT 10 3과목 연결

10축 매핑

축 1: C언어 / 축 2: 컴퓨팅 사고력 [필수] C 문법: 없음 (Layer 0 마무리)
컴퓨팅 사고: 분해, 패턴 인식, 추상화, 순서도, 의사코드를 모두 종합

v1(순차) -> v2(조건) -> v3(반복+분해+패턴+추상화)로 점진 확장했습니다. Layer 1에서 이 설계를 실제 C 코드로 변환합니다.
축 3: 디버깅 / 축 4: KOI [필수] 디버깅: 단계 누락, 경로 누락, 추상화 부재 등 설계 단계의 오류를 경험
KOI: 코드 작성 전 의사코드로 풀이 전략을 설계하는 습관이 대회 실력의 기본입니다. 백준 기초 문제를 의사코드로 먼저 설계하면 구현이 쉬워집니다.
축 6: 실생활 / 축 7: 프로젝트 [선택] 실생활: 매일 하는 행동에 순차(아침 루틴), 조건(비-우산), 반복(일주일)이 모두 포함되어 있습니다.
프로젝트: v1에서 v3로 발전시키는 과정은 소프트웨어 개발의 '반복적 개선(iterative refinement)'과 같습니다. 단순한 것에서 시작하여 점진적으로 확장하십시오.
