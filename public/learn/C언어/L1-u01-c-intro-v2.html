C언어의 탄생

1972년, 미국 AT&T 벨 연구소(Bell Labs)의 Dennis Ritchie가 C언어를 만들었습니다. 당시 벨 연구소에서는 Unix라는 운영체제를 개발하고 있었는데, 기존의 어셈블리어로는 개발 속도가 너무 느렸습니다.

어셈블리어는 컴퓨터가 직접 이해하는 기계어와 거의 1:1 대응되는 저수준 언어입니다. 강력하지만 코드가 길고, CPU마다 문법이 다르며, 읽기 어렵다는 단점이 있었습니다. Dennis Ritchie는 어셈블리어처럼 하드웨어를 직접 제어하면서도, 사람이 읽기 쉬운 언어를 원했습니다.

C언어가 해결한 문제 기존 문제: 어셈블리어로 Unix를 만들자니 코드가 수만 줄, CPU가 바뀌면 처음부터 다시 작성
해결: C언어로 작성하면 코드를 다른 컴퓨터로 쉽게 옮길 수 있음 (이식성)
결과: Unix의 90% 이상이 C로 재작성됨 → C와 Unix가 함께 전 세계에 퍼짐

C언어의 역사

이 교재에서는 C99 표준을 기준으로 합니다. 대부분의 컴파일러가 C99를 완벽히 지원하며, 온라인 저지(백준, 코드업)에서도 C99를 사용할 수 있습니다.

C언어는 어디에 쓰이는가

50년이 넘은 언어가 왜 아직도 사용될까요? C언어는 하드웨어를 직접 제어할 수 있고, 실행 속도가 매우 빠르기 때문입니다. 성능이 중요한 곳에서는 여전히 C가 최선의 선택입니다.

분야사용 예시왜 C인가
운영체제Windows, Linux, macOS 커널하드웨어 직접 제어 필요
임베디드 시스템자동차 ECU, 가전제품 펌웨어, IoT 센서메모리가 적은 환경에서 효율적
게임 엔진Unreal Engine(C++), Unity 내부(C/C++)C의 확장인 C++과 함께 사용. 초당 60프레임 이상의 빠른 연산 필요
데이터베이스MySQL, PostgreSQL, SQLite대용량 데이터 처리 속도
컴파일러/인터프리터Python, PHP, Ruby의 구현체다른 언어 자체가 C로 만들어짐
네트워크 장비라우터, 스위치 펌웨어실시간 패킷 처리 성능
Python도 C로 만들었다 우리가 흔히 사용하는 Python의 공식 구현체(CPython)는 C언어로 작성되어 있습니다. Java의 가상 머신(JVM)도 C/C++로 만들어졌습니다. C를 이해하면 다른 언어의 내부 동작 원리까지 파악할 수 있습니다.

C언어의 5가지 특징

C언어가 반세기 넘게 살아남은 이유를 5가지 핵심 특징으로 정리합니다.

1
① 절차적 언어
코드가 위에서 아래로, 순서대로 실행됩니다. 함수를 사용하여 코드를 구조화합니다.
2
② 저수준 접근
메모리 주소를 직접 다루는 포인터 기능이 있습니다. 하드웨어에 가까운 프로그래밍이 가능합니다.
3
③ 빠른 실행 속도
컴파일 언어이므로 실행 전에 기계어로 변환됩니다. 인터프리터 언어(Python)보다 10~100배 빠릅니다.
4
④ 이식성
표준을 따르면 Windows, Linux, macOS, 임베디드 등 다양한 환경에서 같은 코드를 사용할 수 있습니다.
5
⑤ 작은 언어
키워드가 32개뿐입니다(C89 기준). 문법이 단순하여 배울 내용이 적지만, 그 조합으로 무엇이든 만들 수 있습니다.

C 프로그램의 기본 구조

모든 C 프로그램은 동일한 뼈대를 가지고 있습니다. 아직 직접 작성하지는 않지만, 구조를 눈으로 익혀 두십시오.

Cskeleton.c
1#include <stdio.h> /* ① 헤더 파일 포함 */
2
3int main(void) { /* ② 프로그램 시작점 */
4 /* ③ 여기에 실행할 코드 작성 */
5 return 0; /* ④ 프로그램 종료 */
6}
줄코드역할
1#include <stdio.h>표준 입출력 기능을 사용하겠다는 선언입니다. stdio는 Standard Input/Output의 약자입니다.
3int main(void)프로그램의 시작점입니다. 운영체제는 프로그램을 실행할 때 항상 main 함수를 먼저 찾습니다.
5return 0;프로그램이 정상적으로 끝났다는 신호를 운영체제에 보냅니다. 0은 "문제 없음"을 뜻합니다.

소스 코드에서 실행까지 — 컴파일 과정

C언어는 소스 코드(.c 파일)를 바로 실행할 수 없습니다. 기계어로 변환하는 과정이 필요합니다. 이 과정은 4단계로 이루어집니다.

1
① 전처리(Preprocessing)
#include, #define 등 #으로 시작하는 지시자를 처리합니다. stdio.h의 내용이 소스 코드에 삽입됩니다.
2
② 컴파일(Compilation)
전처리된 코드를 어셈블리어로 변환합니다. 문법 오류가 있으면 이 단계에서 에러가 발생합니다.
3
③ 어셈블(Assembly)
어셈블리어를 기계어(0과 1)로 변환합니다. 결과물은 오브젝트 파일(.o)입니다.
4
④ 링크(Linking)
오브젝트 파일과 라이브러리를 결합하여 최종 실행 파일(.exe)을 생성합니다.
주의: 컴파일 ≠ 전체 과정 일상에서 "컴파일한다"라고 하면 보통 전처리~링크까지 전체 과정을 뜻합니다. 하지만 정확히는 ②번 단계만 컴파일입니다. 시험이나 대회에서 "컴파일 과정을 설명하시오"라는 문제가 나오면 4단계 모두 적어야 합니다.

2

C와 다른 언어 비교

같은 동작을 하는 프로그램을 C, Python, Java로 작성하면 어떻게 다른지 비교해 봅시다. "Hello, World!"를 출력하는 코드입니다.

언어코드특징
C#include int main(void) { printf("Hello, World!\n"); return 0; }5줄. 헤더 포함, main 함수, 세미콜론 필수
Pythonprint("Hello, World!")1줄. 간결하지만 실행 속도 느림
Javapublic class Main { public static void main(String[] args) { System.out.println("Hello, World!"); } }5줄. 클래스 필수, 문법이 가장 장황함
어떤 언어가 가장 좋은가? "최고의 언어"는 없습니다. 목적에 맞는 언어가 있을 뿐입니다.
- 속도가 중요하면 → C
- 빠르게 프로토타입을 만들려면 → Python
- 대규모 기업용 시스템 → Java

C를 먼저 배우면 다른 언어를 훨씬 쉽게 익힐 수 있습니다. C의 문법이 Java, C++, C#, JavaScript 등에 그대로 영향을 주었기 때문입니다.

에러 체험 — 코드에서 에러 찾기

아직 코드를 직접 작성하지는 않지만, 에러를 눈으로 찾는 연습을 해 봅시다. 아래 코드에서 잘못된 부분을 찾아보십시오.

DEBUG LAB — 에러를 찾으시오
C
1#include <stdio.h>
2
3int main(void) {
4 printf("Hello, World!\n")
5 return 0;
6}

에러: error: expected ';' before 'return'

수정: printf 뒤에 세미콜론(;) 추가

C에서 모든 문장은 세미콜론(;)으로 끝나야 합니다. 세미콜론이 없으면 컴파일러는 다음 줄까지 한 문장으로 인식하여 에러를 발생시킵니다.

DEBUG LAB — 에러를 찾으시오
C
1#include <stdio.h>
2
3int Main(void) {
4 printf("Hello\n");
5 return 0;
6}

에러: undefined reference to 'main'

수정: Main → main (소문자)

C언어는 대소문자를 구분합니다. Main과 main은 완전히 다른 이름입니다. 프로그램의 시작점은 반드시 소문자 main이어야 합니다.

C언어는 대소문자를 구분한다 printf와 Printf는 다른 이름입니다.
int와 Int는 다른 이름입니다.
main과 Main은 다른 이름입니다.

키워드와 함수 이름은 모두 소문자로 작성하는 것이 규칙입니다.

용어 정리

용어설명
C언어1972년 Dennis Ritchie가 벨 연구소에서 개발한 범용 프로그래밍 언어입니다.
컴파일(compile)소스 코드를 기계어로 변환하는 과정입니다. 넓은 의미로는 전처리~링크까지 전체를, 좁은 의미로는 어셈블리어 변환 단계만 뜻합니다.
컴파일러(compiler)컴파일을 수행하는 프로그램입니다. C의 대표적인 컴파일러로 gcc, clang이 있습니다.
소스 코드프로그래머가 작성한 텍스트 파일입니다. C언어 소스 코드의 확장자는 .c입니다.
기계어CPU가 직접 실행할 수 있는 0과 1로 이루어진 명령어입니다.
어셈블리어기계어와 1:1 대응되는 저수준 언어입니다. 사람이 읽을 수 있는 형태의 기계어 표현입니다.
키워드프로그래밍 언어에서 미리 의미가 정해진 단어입니다. 변수 이름 등으로 사용할 수 없습니다.
이식성(portability)같은 코드를 서로 다른 환경(OS, CPU)에서 실행할 수 있는 성질입니다.
TIOBE 지수전 세계 프로그래밍 언어의 인기도를 측정하는 지표입니다. 검색 엔진 데이터를 기반으로 매월 발표됩니다.
Unix1969년 벨 연구소에서 개발된 운영체제입니다. Linux, macOS의 기반이 되었습니다.

축 1: C언어 [필수] 이 유닛의 C 문법: #include, main, printf, return 0 (구조 파악 수준)
깊이: LAYER 1 시작 — 아직 코드를 직접 작성하지 않고, C의 역사와 특징을 학습하는 단계
컴파일 과정: 소스코드(.c) → 전처리(#include 확장) → 컴파일(어셈블리어) → 어셈블(기계어 .o) → 링크 → 실행파일(.exe)
키워드: C89 기준 32개. 이 교재에서 모두 다룹니다.
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 추상화(Abstraction)
C언어의 역사를 통해 추상화의 핵심을 배웠습니다:
- 기계어 → 어셈블리어 → C언어: 단계마다 불필요한 세부사항을 숨기고 핵심만 남김
- 기계어로 10줄 걸리는 작업을 C로는 1줄로 표현 가능

이것이 추상화입니다. 복잡한 것을 단순하게 표현하는 능력이 프로그래밍의 본질입니다.
축 3: 디버깅 [필수] 이 유닛의 디버깅 포인트:
1. 세미콜론(;) 누락 → error: expected ';'
2. 대소문자 오류(Main vs main) → undefined reference to 'main'

디버깅 원칙: C는 대소문자를 구분합니다. 키워드와 함수 이름은 모두 소문자입니다. 에러 메시지의 줄 번호를 먼저 확인하십시오.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 0 (준비 단계)
아직 코드를 작성하지 않았으므로 실전 문제는 풀 수 없습니다. 하지만 대회 문제의 구조를 미리 파악했습니다:
- 문제를 읽고 → 입력/출력 형식을 파악하고 → 코드를 작성하고 → 제출
- 출력이 한 글자라도 다르면 오답

Unit 03을 마치면 이번 유닛의 대회 문제 10개를 모두 풀 수 있습니다.
축 5: 코딩 습관 [선택] LAYER 0 — 학습 습관:
- 코드를 눈으로 읽는 연습 (이번 유닛에서 3개 코드를 읽었습니다)
- 출력 결과를 먼저 예측한 뒤 정답을 확인하는 습관
- 에러 메시지를 두려워하지 않고 읽는 자세

이 세 가지가 코딩 실력의 기초 체력입니다.
축 6: 실생활 연결 [선택] C언어가 만든 것들은 우리 일상에 깊이 들어와 있습니다.

- 스마트폰: Android는 Linux 커널(C) 위에서 동작
- 인터넷: 웹 서버(nginx, Apache), DNS 서버 대부분 C로 작성
- 게임: 게임 엔진의 핵심 루프(렌더링, 물리 연산)는 C/C++
- 가전제품: 냉장고, 세탁기, 전자레인지의 마이크로컨트롤러 펌웨어
- 자동차: ECU(엔진 제어 장치) 소프트웨어
축 9: AI 연결 [선택] AI와 C언어의 관계:
- Python이 AI/ML의 대표 언어이지만, 내부 연산은 C/C++로 처리
- NumPy, TensorFlow, PyTorch의 핵심 연산 엔진은 C/C++
- AI 모델이 임베디드 기기(자율주행차, 드론)에서 실행될 때는 C로 변환됨

AI 시대에도 C는 "뒤에서 실제로 일하는 언어"입니다.
축 10: 하드웨어 연결 [선택] C언어가 하드웨어에 가까운 이유:
- 포인터로 메모리 주소를 직접 조작할 수 있음
- 비트 연산으로 하드웨어 레지스터를 제어할 수 있음
- 운영체제 없이도 동작하는 펌웨어를 작성할 수 있음 (베어메탈 프로그래밍)

컴파일 과정과 하드웨어:
소스코드 → (CPU가 이해하는) 기계어로 변환 → CPU가 명령어를 하나씩 실행 → RAM에서 데이터를 읽고 씀
