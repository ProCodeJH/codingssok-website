예측

왜 개발 환경이 필요한가

C 프로그램을 만들려면 두 가지 도구가 반드시 필요합니다. 첫째, 코드를 작성할 텍스트 편집기. 둘째, 코드를 기계어로 변환할 컴파일러입니다.

메모장으로도 C 코드를 작성할 수 있습니다. 하지만 메모장에는 구문 강조(키워드에 색상 입히기), 자동 완성, 에러 표시 같은 기능이 없습니다. 전문 편집기를 사용하면 코딩 속도와 정확도가 크게 올라갑니다.

DEBUG LAB — 에러를 찾으시오
C
1[메모장]
2- 모든 글자가 같은 색
3- 오타를 찾기 어려움
4- 자동 완성 없음
5- 줄 번호 없음

에러: 코딩에 불편

수정: 코딩에 최적화

1
1. 텍스트 편집기 (VS Code)
코드를 작성하고 편집하는 도구입니다. 구문 강조, 자동 완성, 에러 표시 기능이 있습니다.
2
2. 컴파일러 (gcc)
작성한 코드(.c)를 실행 파일(.exe)로 변환합니다. MinGW를 통해 Windows에서 gcc를 사용합니다.
3
3. 터미널 (명령 프롬프트)
컴파일 명령을 입력하고 프로그램을 실행하는 곳입니다. VS Code에 내장되어 있습니다.

VS Code 설치

Visual Studio Code(VS Code)는 Microsoft가 만든 무료 텍스트 편집기입니다. 가볍고 빠르며, C/C++을 포함한 거의 모든 프로그래밍 언어를 지원합니다.

주의: Visual Studio와 다릅니다 Visual Studio Code와 Visual Studio는 다른 프로그램입니다.
- Visual Studio Code: 가벼운 텍스트 편집기 (이것을 설치합니다)
- Visual Studio: 대형 통합 개발 환경 (용량이 크고 복잡합니다)

검색할 때 반드시 "Visual Studio Code"로 검색하십시오.
1
1단계: 다운로드
code.visualstudio.com에 접속합니다. 화면 중앙의 파란색 Download 버튼을 클릭합니다. Windows용이 자동으로 선택됩니다.
2
2단계: 설치
다운로드된 VSCodeUserSetup-x64-*.exe 파일을 실행합니다. 설치 중 "PATH에 추가" 옵션에 반드시 체크하십시오.
3
3단계: 실행 확인
바탕화면 또는 시작 메뉴에서 Visual Studio Code를 실행합니다. 파란색 창이 나타나면 설치 성공입니다.
4
4단계: 한국어 설정
왼쪽 사이드바에서 확장(Extensions) 아이콘을 클릭하고 "Korean Language Pack"을 검색하여 설치합니다. 재시작하면 한국어로 전환됩니다.
5
5단계: C/C++ 확장 설치
확장 탭에서 "C/C++"을 검색하고 Microsoft의 공식 확장을 설치합니다. 구문 강조와 IntelliSense(자동 완성) 기능이 활성화됩니다.

gcc 설치 (MinGW-w64)

gcc(GNU Compiler Collection)는 세계에서 가장 널리 사용되는 C 컴파일러입니다. Windows에서는 MinGW-w64를 통해 gcc를 사용할 수 있습니다.

gcc란? gcc = GNU Compiler Collection
1987년 Richard Stallman이 시작한 오픈소스 컴파일러입니다. Linux에는 기본 내장, Windows에서는 MinGW를 통해 설치합니다. 백준, 코드업 등 온라인 저지에서도 gcc를 사용합니다.
1
1단계: MSYS2 다운로드
msys2.org에 접속하여 설치 파일을 다운로드합니다. MSYS2는 Windows에서 Unix 도구를 사용할 수 있게 해주는 환경입니다.
2
2단계: MSYS2 설치
다운로드된 파일을 실행하고 기본 경로(C:\msys64)에 설치합니다. 설치 완료 후 MSYS2 터미널이 자동으로 열립니다.
3
3단계: gcc 설치
MSYS2 터미널에 아래 명령어를 입력합니다. pacman -S mingw-w64-ucrt-x86_64-gcc 설치 확인 질문에 Y를 입력합니다.
4
4단계: PATH 설정
Windows 검색에서 "환경 변수"를 검색합니다. 시스템 변수의 Path에 C:\msys64\ucrt64\bin을 추가합니다.
5
5단계: 설치 확인
새 명령 프롬프트(cmd)를 열고 gcc --version을 입력합니다. 버전 정보가 출력되면 설치 성공입니다.
Ccmd
1C:\Users\사용자> gcc --version
2gcc(Rev3, Built by MSYS2 project) 14.2.0
3Copyright(C) 2024 Free Software Foundation, Inc.
설치 확인 실패 시 'gcc'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다.

이 메시지가 나오면 PATH 설정이 잘못된 것입니다. 4단계를 다시 확인하고, 반드시 새 명령 프롬프트를 열어서 테스트하십시오. 기존 창에서는 변경된 PATH가 반영되지 않습니다.

작업 폴더 만들기

코드를 작성하기 전에 파일을 정리할 폴더를 만듭니다. 바탕화면이나 문서 폴더에 C_Study라는 폴더를 만드십시오.

Ccmd
1C:\Users\사용자> mkdir C:\C_Study
2C:\Users\사용자> cd C:\C_Study
3C:\C_Study>
TIP: 폴더 이름 규칙 폴더 이름에 한글, 공백, 특수문자를 넣지 마십시오. 일부 컴파일러에서 경로 인식 오류가 발생할 수 있습니다.

좋은 예: C_Study, c_lang, coding
나쁜 예: C 공부, 코딩쏙 자료, my code!

첫 번째 C 파일 만들기

VS Code를 열고 File > Open Folder를 선택하여 C_Study 폴더를 엽니다. 그런 다음 새 파일을 만들어 아래 코드를 직접 입력하십시오.

반드시 직접 타이핑하십시오 코드를 복사-붙여넣기 하면 문법이 손에 익지 않습니다. 한 글자씩 직접 입력하십시오. 오타가 나도 괜찮습니다. 오타를 찾고 고치는 과정 자체가 학습입니다.
Chello.c
1#include <stdio.h>
2
3int main(void) {
4 printf("Hello, World!\n");
5 return 0;
6}

파일 이름을 hello.c로 저장합니다. 확장자가 반드시 .c여야 합니다. .txt로 저장하면 컴파일러가 C 소스 코드로 인식하지 못합니다.

컴파일하기

VS Code 하단의 터미널(Terminal > New Terminal)을 열고 아래 명령어를 입력합니다.

Ccmd
1C:\C_Study> gcc hello.c -o hello
2C:\C_Study>
줄코드역할
1gcc컴파일러를 실행합니다.
1hello.c컴파일할 소스 파일입니다.
1-o hello출력 파일 이름을 hello로 지정합니다. Windows에서는 hello.exe가 생성됩니다.

아무 메시지 없이 다음 줄로 넘어가면 컴파일 성공입니다. 에러가 있으면 에러 메시지가 출력됩니다.

-o 옵션을 안 쓰면? gcc hello.c만 입력하면 기본 이름인 a.exe(Windows) 또는 a.out(Linux)이 생성됩니다. -o를 사용하면 원하는 이름을 지정할 수 있습니다.

실행하기

Ccmd
1C:\C_Study> .\hello
2Hello, World!

.\hello는 "현재 폴더에 있는 hello.exe를 실행하라"는 뜻입니다. 화면에 Hello, World!가 출력되면 성공입니다.

TIP: 전체 과정 요약 1. 작성: VS Code에서 hello.c 파일에 코드 입력
2. 저장: Ctrl+S
3. 컴파일: 터미널에서 gcc hello.c -o hello
4. 실행: 터미널에서 .\hello

이 4단계를 앞으로 수백 번 반복하게 됩니다. 손에 익을 때까지 연습하십시오.

에러 체험 — 저장 안 하고 컴파일

초보자가 가장 많이 하는 실수는 코드를 수정한 뒤 저장하지 않고 컴파일하는 것입니다.

1
1. hello.c에서 "Hello"를 "Hi"로 수정
printf("Hi, World!\n"); 으로 변경합니다.
2
2. 저장하지 않고 바로 컴파일
gcc hello.c -o hello 실행
3
3. 실행하면?
.\hello 결과: Hello, World! — 수정 전 내용이 출력됩니다.
4
4. 원인
Ctrl+S를 누르지 않아서 디스크의 파일은 그대로이기 때문입니다.
핵심 코드를 수정했으면 반드시 Ctrl+S → 컴파일 → 실행 순서를 지키십시오. VS Code 탭에 흰색 점(●)이 보이면 아직 저장되지 않은 것입니다.

에러 체험 — 파일 이름 오타

Ccmd
1C:\C_Study> gcc helo.c -o hello
2gcc: error: helo.c: No such file or directory
3gcc: fatal error: no input files

파일 이름을 helo.c로 잘못 입력했습니다. gcc는 해당 파일을 찾을 수 없다는 에러를 출력합니다. 파일 이름의 철자를 정확히 입력하십시오.

에러 체험 — 잘못된 폴더에서 컴파일

Ccmd
1C:\Users\사용자> gcc hello.c -o hello
2gcc: error: hello.c: No such file or directory

현재 폴더에 hello.c 파일이 없으면 같은 에러가 발생합니다. cd C:\C_Study로 이동한 뒤 컴파일하십시오. dir 명령으로 현재 폴더의 파일 목록을 확인할 수 있습니다.

Ccmd
1C:\Users\사용자> cd C:\C_Study
2C:\C_Study> dir
3hello.c
4C:\C_Study> gcc hello.c -o hello

에러 체험 — 코드에 오타가 있을 때

Ctypo.c
1#include <stdio.h>
2
3int main(void) {
4 print("Hello\n");
5 return 0;
6}
Ccmd
1C:\C_Study> gcc typo.c -o typo
2typo.c:4:5: warning: implicit declaration of function 'print'
3undefined reference to 'print'
4collect2.exe: error: ld returned 1 exit status

printf를 print로 잘못 입력했습니다. 컴파일러는 print라는 함수를 찾을 수 없다는 에러를 출력합니다.

에러 메시지 읽는 법 typo.c:4:5는 "typo.c 파일의 4번째 줄, 5번째 열"이라는 뜻입니다. 에러 메시지에서 파일명:줄번호를 먼저 찾고, 해당 줄을 확인하십시오.

에러 체험 — 실행 파일을 찾을 수 없을 때

Ccmd
1C:\C_Study> hello
2'hello'은(는) 내부 또는 외부 명령...이 아닙니다.
3
4C:\C_Study> .\hello
5Hello, World!

Windows에서는 현재 폴더의 실행 파일을 실행할 때 .\을 앞에 붙여야 합니다. .\은 "현재 폴더"를 뜻합니다.

용어 정리

용어설명
VS CodeMicrosoft가 만든 무료 텍스트 편집기입니다. 구문 강조, 자동 완성, 내장 터미널을 지원합니다.
gccGNU Compiler Collection. 세계에서 가장 널리 사용되는 오픈소스 C/C++ 컴파일러입니다.
MinGW-w64Windows에서 gcc를 사용할 수 있게 해주는 도구 모음입니다. MSYS2를 통해 설치합니다.
PATH운영체제가 실행 파일을 찾는 폴더 목록입니다. gcc를 어디서든 실행하려면 PATH에 등록해야 합니다.
터미널텍스트 명령어를 입력하여 컴퓨터를 조작하는 도구입니다. 명령 프롬프트(cmd), PowerShell 등이 있습니다.
-o 옵션gcc에서 출력 파일의 이름을 지정하는 옵션입니다. 예: gcc hello.c -o hello
구문 강조텍스트 편집기가 키워드, 문자열, 숫자 등을 다른 색상으로 표시하는 기능입니다.
.\현재 폴더를 뜻하는 경로 표기입니다. Windows에서 현재 폴더의 실행 파일을 실행할 때 사용합니다.

3. 잘못된 폴더 → No such file or directory
4. 코드 오타(prntf) → implicit declaration + undefined reference
5. .\없이 실행 → 명령을 찾을 수 없음

- 작업 폴더를 정리하여 사용 (C_Study/unit02/ 등)
- 파일 이름은 영문 소문자 + 밑줄 (hello_world.c)

- 에러가 나면 당황하지 않고 메시지를 읽음
축 6: 실생활 연결 [선택] 개발 환경은 목수의 연장통과 같습니다.

- gcc = 톱, 대패 (원재료를 제품으로 가공하는 도구)
- 터미널 = 손 (도구를 직접 조작하는 인터페이스)

좋은 도구를 갖추는 것이 모든 작업의 첫 단계입니다.
축 10: 하드웨어 연결 [선택] gcc가 만드는 실행 파일의 정체:
- .exe 파일은 CPU가 직접 실행할 수 있는 기계어 명령어의 모음입니다
- x86-64 CPU용으로 컴파일된 .exe는 ARM CPU에서 실행할 수 없습니다
- 이것이 "컴파일러가 플랫폼별로 다른 이유"입니다

MinGW의 gcc는 Windows x86-64용 기계어를 생성합니다.
