Hello, World!의 전통

1978년, Brian Kernighan과 Dennis Ritchie가 쓴 The C Programming Language라는 책에서 첫 예제로 hello, world를 출력했습니다. 이후 거의 모든 프로그래밍 입문서가 Hello World로 시작하는 전통이 생겼습니다.

Hello World는 단순한 예제가 아닙니다. 이 한 줄의 출력을 성공시키면 개발 환경이 정상적으로 동작한다는 증거입니다. 편집기, 컴파일러, 운영체제, 화면 출력까지 전체 파이프라인이 작동해야만 Hello World가 출력됩니다.

Hello, World! 완전 분석

Chello.c
1#include <stdio.h>
2
3int main(void) {
4 printf("Hello, World!\n");
5 return 0;
6}
줄코드역할
1#include <stdio.h>표준 입출력 헤더 파일을 포함합니다. printf 함수를 사용하려면 반드시 필요합니다. stdio = Standard Input/Output.
3int main(void)프로그램의 시작점입니다. 운영체제는 프로그램을 실행할 때 main 함수를 찾아서 호출합니다. int는 정수를 반환한다는 뜻, void는 매개변수가 없다는 뜻입니다.
4printf("Hello, World!\n");큰따옴표 안의 문자열을 화면에 출력합니다. \n은 줄바꿈입니다. 끝에 세미콜론(;)이 반드시 필요합니다.
5return 0;프로그램이 정상 종료되었음을 운영체제에 알립니다. 0은 "문제 없음"을 뜻합니다. 0이 아닌 값은 에러를 의미합니다.
실행 결과

printf — 화면에 출력하는 함수

printf는 C에서 가장 먼저 배우고, 끝까지 가장 많이 사용하는 함수입니다. 이름의 f는 formatted(서식)의 약자입니다. 큰따옴표 안에 넣은 문자열을 화면에 그대로 출력합니다.

Cprintf_basic.c
1#include <stdio.h>
2
3int main(void) {
4 printf("Hello");
5 printf(" World");
6 printf("!\n");
7 return 0;
8}
실행 결과
Hello World!

printf를 3번 호출했지만 출력은 한 줄입니다. printf는 기본적으로 줄을 바꾸지 않고 이어서 출력합니다. 줄바꿈은 \n을 넣었을 때만 발생합니다.

핵심 규칙 printf는 줄바꿈을 자동으로 하지 않습니다.
- printf("A"); printf("B"); → 출력: AB
- printf("A\n"); printf("B\n"); → 출력: A (줄바꿈) B

\n — 줄바꿈

Cnewline.c

2

\n을 만나면 출력 커서가 다음 줄의 맨 앞으로 이동합니다. 각 printf 끝에 \n을 넣으면 한 줄씩 출력됩니다.

이스케이프 시퀀스

큰따옴표 안에서 \(역슬래시)로 시작하는 특수 문자를 이스케이프 시퀀스라고 합니다. 화면에 직접 입력할 수 없는 문자를 표현할 때 사용합니다.

시퀀스의미설명
\n줄바꿈출력 커서를 다음 줄로 이동
\t탭다음 탭 스톱까지 이동 (보통 8칸 간격)
\\역슬래시 출력\를 화면에 출력
\"큰따옴표 출력"를 화면에 출력
\0널 문자문자열의 끝을 표시 (나중에 학습)
주의 \n과 \t는 두 글자처럼 보이지만 실제로는 한 개의 문자입니다. 역슬래시(\)가 뒤의 문자와 합쳐져 하나의 특수 문자를 만듭니다.

\t — 탭

Ctab.c
1#include <stdio.h>
2
3int main(void) {
4 printf("이름\t나이\t학교\n");
5 printf("김철수\t13\t코딩중\n");
6 printf("이영희\t14\t알고중\n");
7 return 0;
8}
실행 결과
이름 나이 학교 김철수 13 코딩중 이영희 14 알고중

\t은 다음 탭 스톱 위치까지 커서를 이동시킵니다. 표 형태로 데이터를 정렬할 때 유용합니다.

\\ 와 \" — 특수문자 출력

Cescape.c
1#include <stdio.h>
2
3int main(void) {
4 printf("경로: C:\\Users\\hello\n");
5 printf("그가 말했다: \"안녕\"\n");
6 return 0;
7}
실행 결과
경로: C:\Users\hello 그가 말했다: "안녕"

역슬래시(\)를 출력하려면 \\, 큰따옴표(")를 출력하려면 \"를 사용합니다. \는 이스케이프 시퀀스의 시작이고, "는 문자열의 경계이므로, 각각 두 번 써야 문자 자체를 출력합니다.

에러 체험 — 세미콜론 누락

DEBUG LAB — 에러를 찾으시오
C
1#include <stdio.h>
2
3int main(void) {
4 printf("Hello")
5 return 0;
6}

에러: error: expected ';' before 'return'

수정: printf 뒤에 세미콜론(;) 추가

C의 모든 문장은 세미콜론으로 끝나야 합니다. 세미콜론이 없으면 컴파일러는 다음 줄까지 한 문장으로 인식합니다.

에러 체험 — 따옴표 빠짐

DEBUG LAB — 에러를 찾으시오
C
1printf(Hello);

에러: error: 'Hello' undeclared

수정: 문자열은 반드시 큰따옴표로 감쌈

큰따옴표가 없으면 컴파일러는 Hello를 변수 이름으로 인식합니다. 선언된 적 없는 변수이므로 에러가 발생합니다.

에러 체험 — #include 누락

DEBUG LAB — 에러를 찾으시오
C
1int main(void) {
2 printf("test");
3 return 0;
4}

에러: warning: implicit declaration of function 'printf'

수정: 첫 줄에 #include 추가

#include <stdio.h>가 없으면 컴파일러는 printf가 어떤 함수인지 모릅니다. 경고 또는 에러가 발생합니다.

에러는 발생하지 않지만 두 단어가 붙어서 출력됩니다. 의도한 출력이 아니라면 \n이나 공백을 넣어야 합니다. 온라인 저지에서는 이런 차이로 오답 처리됩니다.

에러 체험 — 역슬래시 하나만 쓰면?

Cbad_escape.c
1#include <stdio.h>
2
3int main(void) {
4 printf("C:\new\test\n");
5 return 0;
6}
실행 결과
C: ew est

\n이 줄바꿈으로, \t가 탭으로 해석됩니다. 경로를 출력하려면 C:\\new\\test로 역슬래시를 두 번 써야 합니다.

역슬래시 규칙 문자열 안에서 \ 뒤에 오는 문자에 따라 동작이 달라집니다.
- \n → 줄바꿈 (의도적)
- \t → 탭 (의도적)
- \a → 경고음 (의도하지 않은 경우 많음)

역슬래시 자체를 출력하려면 항상 \\로 써야 합니다.

UNIT 03 3과목 연결

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: printf, 이스케이프 시퀀스(\n, \t, \\, \"), #include, main, return 0
깊이: LAYER 1 (출력) — printf로 화면에 텍스트를 자유자재로 출력하는 단계
printf의 동작 원리: 큰따옴표 안의 문자열을 stdout(표준 출력)에 전달 → 운영체제가 터미널에 표시
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 패턴 인식(Pattern Recognition)
이번 유닛에서 발견한 패턴:
- 모든 C 프로그램은 같은 뼈대를 가짐 (#include → main → return)
- 모든 문장은 세미콜론으로 끝남
- \로 시작하는 문자는 항상 특수 동작

패턴을 인식하면 새로운 코드도 빠르게 이해할 수 있습니다.
축 3: 디버깅 [필수] 이 유닛의 에러 5가지:
1. 세미콜론 누락 → expected ';'
2. 따옴표 빠짐 → undeclared
3. #include 누락 → implicit declaration
4. \n 빠짐 → 출력이 붙어 나옴 (논리 에러)
5. 역슬래시 하나 → 의도치 않은 이스케이프

디버깅 체크리스트: 세미콜론 → 따옴표 → #include → \n 순서로 확인
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1 (출력 정확성)
출력 문제에서 주의할 점:
- 공백 하나, 줄바꿈 하나가 달라도 오답
- 마지막 줄 뒤에 불필요한 \n이 있으면 오답이 될 수 있음
- 특수문자(\, ") 출력 시 이스케이프 시퀀스 정확히 사용

이 유닛의 대회 문제 10개를 모두 "맞았습니다!!"로 통과하십시오.
축 5: 코딩 습관 [선택] LAYER 1 — 출력 습관:
- printf 끝에 \n을 넣는 습관 (마지막 줄 포함)
- 코드를 복사-붙여넣기 하지 않고 직접 타이핑
- 에러가 나면 에러 메시지의 줄 번호부터 확인
- 한 줄에 한 문장만 작성하여 가독성 유지
축 6: 실생활 연결 [선택] printf는 사용자와 대화하는 가장 기본적인 방법입니다.

실제 printf가 쓰이는 곳:
- ATM 화면의 "비밀번호를 입력하세요" 메시지
- 엘리베이터의 층수 표시
- 게임 로딩 화면의 "Loading..." 텍스트
- 의료기기의 측정값 출력

화면에 무언가를 표시하는 모든 프로그램이 printf 또는 유사 함수를 사용합니다.
축 10: 하드웨어 연결 [선택] printf가 화면에 글자를 띄우는 과정:
1. printf 호출 → C 라이브러리가 문자열을 버퍼에 저장
2. 버퍼가 꽉 차거나 \n을 만나면 운영체제에 전달 (시스템 콜)
3. 운영체제가 콘솔 드라이버에 전달
4. CPU가 문자를 프레임버퍼에 기록
5. 디스플레이 컨트롤러가 프레임버퍼를 읽어 모니터에 표시

printf("A") 한 줄이 실행되기까지 이 모든 과정이 일어납니다.
