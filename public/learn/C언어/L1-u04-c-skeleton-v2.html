C 프로그램의 세 기둥

모든 C 프로그램은 세 가지 요소로 구성됩니다. 이 세 가지를 완전히 이해하면 어떤 C 코드든 구조를 파악할 수 있습니다.

1
1. #include — 도구 불러오기
필요한 함수가 선언된 헤더 파일을 포함합니다. printf를 쓰려면 stdio.h가 필요합니다.
2
2. main — 프로그램 시작점
운영체제가 프로그램을 실행할 때 가장 먼저 호출하는 함수입니다.
3
3. return — 종료 보고
프로그램이 끝났음을 운영체제에 알립니다. 0은 정상 종료를 뜻합니다.

#include — 전처리 지시자

#include <stdio.h>는 컴파일 전에 처리됩니다. 컴파일러가 코드를 읽기 전에 전처리기(preprocessor)가 먼저 실행되어 stdio.h 파일의 내용을 소스 코드에 삽입합니다.

Cinclude_example.c
1#include <stdio.h> /* stdio.h의 내용이 여기에 삽입됨 */
2
3int main(void) {
4 printf("Hello\n"); /* printf는 stdio.h에 선언되어 있음 */
5 return 0;
6}
#include가 하는 일 #include <stdio.h>는 말 그대로 "stdio.h 파일을 여기에 포함시켜라"입니다.

stdio.h 안에는 printf, scanf, puts, getchar 등의 함수 선언이 들어있습니다. 컴파일러는 함수가 선언되어 있어야만 사용할 수 있습니다.

C 표준 라이브러리에는 stdio.h 외에도 다양한 헤더 파일이 있습니다.

헤더 파일용도주요 함수
stdio.h표준 입출력printf, scanf, puts, getchar, fopen
stdlib.h일반 유틸리티malloc, free, atoi, rand, exit
string.h문자열 처리strlen, strcpy, strcmp, strcat
math.h수학 함수sqrt, pow, sin, cos, fabs
time.h시간/날짜time, clock, difftime
TIP: < > vs " " #include <stdio.h> — 시스템 헤더 파일 (C 표준 라이브러리)
#include "myfile.h" — 사용자가 직접 만든 헤더 파일

지금은 < >만 사용합니다. " "는 나중에 여러 파일로 프로그램을 나눌 때 배웁니다.

main — 프로그램의 시작점

Cmain_detail.c
1int main(void) {
2 /* 이 안에 실행할 코드를 작성 */
3 return 0;
4}
줄코드역할
1intmain 함수가 정수(integer)를 반환한다는 뜻입니다. 이 정수가 바로 return 뒤의 0입니다.
1main함수 이름입니다. 운영체제는 반드시 이 이름을 찾습니다. Main, MAIN 등은 인식하지 못합니다.
1(void)매개변수가 없다는 뜻입니다. main이 외부에서 값을 받지 않습니다.
1{ }함수의 시작과 끝을 표시합니다. { 와 } 사이의 코드가 순서대로 실행됩니다.

중괄호 { } — 코드 블록

중괄호는 코드를 하나의 블록으로 묶습니다. main 함수의 { }는 "이 안의 코드가 main 함수에 속한다"는 의미입니다.

DEBUG LAB — 에러를 찾으시오
C
1int main(void)
2 printf("Hello\n");
3 return 0;

에러: error: expected '{' before 'printf'

수정: 중괄호로 코드 블록을 감쌈

중괄호 짝 맞추기 여는 중괄호 {와 닫는 중괄호 }는 반드시 짝이 맞아야 합니다. VS Code에서 {를 입력하면 자동으로 }가 생성됩니다. 짝이 안 맞으면 에러가 발생합니다.

return 0 — 종료 상태 코드

return 0;은 운영체제에 "프로그램이 정상적으로 끝났습니다"라고 보고하는 문장입니다.

반환값의미사용 예
0정상 종료프로그램이 문제 없이 끝남
1 이상에러 종료프로그램이 비정상적으로 끝남
Creturn_demo.c
1#include <stdio.h>
2
3int main(void) {
4 printf("프로그램 시작\n");
5 printf("프로그램 끝\n");
6 return 0; /* 여기서 main 함수가 종료됨 */
7}
TIP: return 0; 생략 C99 표준부터 main 함수에서 return 0;을 생략할 수 있습니다. 생략하면 자동으로 0을 반환합니다. 하지만 명시적으로 쓰는 것이 좋은 습관입니다.

#include 없이 컴파일하면?

예측 코드의 결과를 확인합니다.

Cno_include.c
1int main(void) {
2 printf("test\n");
3 return 0;
4}
Ccmd
1C:\C_Study> gcc no_include.c -o no_include
2no_include.c:2:5: warning: implicit declaration of function 'printf'
3no_include.c:2:5: warning: incompatible implicit declaration of built-in function 'printf'

경고(warning)가 발생하지만 컴파일은 됩니다. gcc가 printf를 알아서 찾아주기 때문입니다. 하지만 경고가 있는 코드는 불완전한 코드입니다. 항상 #include를 작성하십시오.

경고(warning) vs 에러(error) 경고: 컴파일은 되지만 문제가 있을 수 있음. 무시하면 안 됩니다.
에러: 컴파일이 중단됨. 반드시 수정해야 합니다.

온라인 저지에서는 경고가 있어도 채점되지만, 좋은 코드는 경고가 0개입니다.

용어 정리

용어설명
전처리(preprocessing)컴파일 전에 #include, #define 등 #으로 시작하는 지시자를 처리하는 단계입니다.
헤더 파일(.h)함수 선언, 매크로 정의 등이 들어있는 파일입니다. #include로 포함합니다.
함수 선언함수의 이름, 매개변수 타입, 반환 타입을 컴파일러에 알리는 것입니다. 실제 코드(정의)와 구분됩니다.
코드 블록{ }로 감싼 코드 영역입니다. 함수, 조건문, 반복문 등에서 사용됩니다.
종료 상태 코드프로그램이 운영체제에 반환하는 정수입니다. 0이면 성공, 그 외는 에러를 의미합니다.
경고(warning)컴파일은 되지만 잠재적 문제가 있음을 알리는 메시지입니다.
에러(error)컴파일이 중단되는 심각한 문제입니다. 반드시 수정해야 합니다.

UNIT 04 3과목 연결

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: #include, 헤더 파일(.h), int main(void), { }, return 0
깊이: LAYER 1 — 프로그램 뼈대의 각 요소를 전처리 수준까지 이해
새 명령어: gcc -E (전처리 결과 확인)
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 분해(Decomposition)
C 프로그램을 3개의 독립된 역할로 분해했습니다:
1. #include — 준비 (도구 불러오기)
2. main { } — 실행 (할 일 수행)
3. return — 종료 (결과 보고)

복잡한 시스템을 독립된 역할로 나누는 것이 분해의 핵심입니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. Main → main (대소문자) → undefined reference
2. 닫는 } 누락 → expected '}' at end of input
3. return 0 세미콜론 누락 → expected ';'
4. #include 누락 → implicit declaration (경고)
5. # 빠진 include → 에러

새 구분: 경고(warning)는 컴파일됨, 에러(error)는 컴파일 중단
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1 (출력 정확성)
대회 코드 작성 시 뼈대:
#include <stdio.h>
int main(void) {
    // 풀이 코드
    return 0;
}

이 뼈대를 손이 기억할 때까지 반복하십시오.
축 5: 코딩 습관 [선택] LAYER 1 — 구조 습관:
- 새 파일을 만들면 먼저 뼈대부터 작성 (#include → main → return)
- 여는 { 를 쓰면 즉시 닫는 }를 쓴 뒤, 안에 코드를 채움
- 경고를 무시하지 않음. gcc -Wall 옵션으로 모든 경고를 확인
축 10: 하드웨어 연결 [선택] return 0;이 운영체제에 전달되는 과정:
1. main 함수에서 return 0; 실행
2. 0이 CPU의 레지스터(EAX)에 저장됨
3. 프로그램이 종료되면 운영체제가 EAX 값을 읽음
4. 셸(cmd)에서 echo %errorlevel%로 확인 가능

return 값은 배치 스크립트에서 프로그램 성공/실패를 판단하는 데 사용됩니다.
