프로젝트란?

프로젝트 유닛은 지금까지 배운 내용을 종합하여 하나의 완성된 프로그램을 만드는 유닛입니다. 새로운 문법을 배우는 것이 아니라, 이미 배운 도구를 조합하는 연습입니다.

이번 프로젝트의 도구 사용할 도구 (모두 이전 유닛에서 학습 완료):

1. #include <stdio.h> — 표준 입출력 헤더
2. int main(void) { ... return 0; } — 프로그램 뼈대
3. printf("..."); — 화면 출력
4. \n 줄바꿈, \t 탭, \\ 역슬래시, \" 큰따옴표

프로젝트 기획

프로그래밍의 첫 단계는 코드 작성이 아니라 기획입니다. 무엇을 만들지 먼저 정해야 합니다.

1
1단계: 출력 내용 결정
자기소개에 포함할 항목을 정합니다. 이름, 나이, 학교, 취미, 좋아하는 것, 한마디 등.
2
2단계: 출력 형태 설계
구분선, 탭 정렬, 빈 줄 등을 사용하여 보기 좋은 형태를 종이에 그려 봅니다.
3
3단계: 코드 작성
뼈대를 먼저 쓰고, printf를 하나씩 추가합니다.
4
4단계: 컴파일 및 테스트
gcc로 컴파일하고 실행합니다. 출력이 설계와 다르면 수정합니다.
5
5단계: 개선
더 보기 좋게, 더 깔끔하게 다듬습니다.

TIP: 탭 정렬의 한계 \t은 보통 8칸 단위로 정렬됩니다. 항목 이름의 길이에 따라 탭 개수를 조절해야 합니다.

"좋아하는 과목:" (7글자) → \t 1개

완벽한 정렬이 어렵다면 공백 문자를 직접 사용하는 것도 방법입니다.

용어 정리

용어설명
프로젝트(project)여러 개념을 종합하여 하나의 완성된 프로그램을 만드는 활동입니다.
기획(planning)프로그램을 작성하기 전에 무엇을 만들지, 어떤 형태로 출력할지 미리 설계하는 단계입니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: printf 다중 호출, 이스케이프 시퀀스 조합, 프로그램 구조 종합
깊이: LAYER 1 — 기존 문법을 종합하여 프로젝트 완성
핵심: 새 문법 없이 기존 도구만으로 완성된 프로그램을 만들 수 있습니다
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 설계(Design)
코드를 바로 작성하지 않고, 출력 형태를 먼저 설계했습니다. 이것이 컴퓨팅 사고력의 핵심인 '문제를 정의하고 해결 절차를 설계하는 능력'입니다.

적용한 단계: 기획 → 설계 → 구현 → 테스트 → 개선
축 3: 디버깅 [필수] 이 유닛의 에러:
1. \" 누락 → expected ')' (문자열 안 큰따옴표)
2. 세미콜론 누락 → expected ';'
3. 닫는 " 누락 → missing terminating "
4. 닫는 ) 누락 → expected ')'
5. 닫는 } 누락 → expected '}'

디버깅 습관: 한 줄씩 추가하며 컴파일하면 에러 위치를 즉시 파악할 수 있습니다.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1 (출력 정확성)
대회에서 중요한 것:
- 출력이 한 글자라도 다르면 오답 처리됩니다
- 공백, 줄바꿈, 특수문자를 정확히 맞추는 습관이 필수입니다
- 이번 유닛의 ASCII 아트 문제가 바로 이 정확성 훈련입니다
축 6: 실생활 연결 [선택] 자기소개 프로그램 → 명함 프로그램:
실제로 텍스트 기반 명함, 이메일 서명, 터미널 배너 등은 printf와 같은 방식으로 만듭니다.

서버에 SSH로 접속할 때 나타나는 환영 메시지(MOTD)도 printf로 출력한 ASCII 아트입니다.
축 7: 프로젝트 [선택] 이 유닛 자체가 프로젝트입니다.
기획 → 설계 → 구현 → 테스트 → 개선의 5단계를 경험했습니다.
