세미콜론 — 문장의 끝

한국어에서 마침표(.)가 문장의 끝을 표시하듯, C에서는 세미콜론(;)이 문장의 끝을 표시합니다. C 컴파일러는 줄바꿈을 문장의 끝으로 인식하지 않습니다. 오직 세미콜론만 문장의 끝입니다.

핵심 규칙 C에서 문장(statement)이란?
컴퓨터에게 하나의 동작을 지시하는 단위입니다.

printf("Hello\n"); — 출력 문장
return 0; — 반환 문장
int a = 5; — 선언 문장 (나중에 학습)

모든 문장은 세미콜론(;)으로 끝납니다. 예외 없습니다.

세미콜론이 필요한 곳

규칙은 단순합니다. 컴퓨터에게 동작을 지시하는 모든 문장 뒤에 세미콜론을 붙입니다.

Csemicolon_yes.c
1#include <stdio.h>
2
3int main(void) {
4 printf("첫 번째\n"); /* 출력 문장 → ; 필요 */
5 printf("두 번째\n"); /* 출력 문장 → ; 필요 */
6 return 0; /* 반환 문장 → ; 필요 */
7}

세미콜론이 불필요한 곳

전처리 지시자와 함수 정의의 중괄호 뒤에는 세미콜론을 붙이지 않습니다.

코드세미콜론이유
#include 붙이지 않음전처리 지시자 — 컴파일러가 아닌 전처리기가 처리
int main(void) {붙이지 않음함수 정의의 시작 — 문장이 아님
}붙이지 않음블록의 끝 — 문장이 아님
printf("Hi\n");붙임출력 문장
return 0;붙임반환 문장
TIP: 간단한 판별법 세미콜론을 붙이는가?

- #으로 시작하면 → 붙이지 않음 (전처리 지시자)
- { 또는 }이면 → 붙이지 않음 (블록 경계)
- 그 외 문장 → 붙임

지금 단계에서는 이 규칙만 기억하면 충분합니다. 예외는 나중에 배웁니다.

줄바꿈과 세미콜론은 다릅니다

Python에서는 줄바꿈이 문장의 끝을 의미하지만, C에서는 줄바꿈이 아무 의미가 없습니다. 세미콜론만이 문장의 끝입니다.

Coneline.c
1#include <stdio.h>
2int main(void) { printf("A\n"); printf("B\n"); return 0; }

위 코드는 한 줄이지만 정상적으로 컴파일됩니다. 세미콜론이 문장을 구분하기 때문입니다. 다만 가독성이 나쁘므로 이렇게 작성하면 안 됩니다.

Cmultiline.c
1#include <stdio.h>
2
3int main(void) {
4 printf(
5 "Hello\n"
6 );
7 return 0;
8}

위 코드는 printf가 세 줄에 걸쳐 있지만 세미콜론이 6번째 줄에 있으므로 하나의 문장입니다. C는 세미콜론을 만날 때까지 문장이 계속된다고 판단합니다.

중괄호 — 코드 블록

중괄호 { }는 여러 문장을 하나의 블록(block)으로 묶습니다. main 함수에서 { 와 } 사이의 모든 코드가 main 함수에 속합니다.

Cbraces.c
1#include <stdio.h>
2
3int main(void) { /* 블록 시작 */
4 printf("A\n");
5 printf("B\n");
6 printf("C\n");
7 return 0;
8} /* 블록 끝 */
중괄호의 규칙 1. 여는 중괄호 {와 닫는 중괄호 }는 반드시 짝이 맞아야 합니다.
2. 여는 { 를 쓰면 즉시 닫는 }를 쓴 뒤, 안에 코드를 채우십시오.
3. { } 안의 코드는 들여쓰기(4칸 공백)합니다.
4. 닫는 }는 여는 {와 같은 열(column)에 위치합니다.

중괄호와 들여쓰기

들여쓰기는 중괄호 안의 코드가 블록에 속한다는 것을 시각적으로 보여줍니다. 컴파일러는 들여쓰기를 무시하지만, 사람은 들여쓰기로 코드 구조를 파악합니다.

DEBUG LAB — 에러를 찾으시오
C

2int main(void) {
3printf("A\n");
4printf("B\n");
5return 0;
6}

에러: 들여쓰기 없음 — 블록 구조가 보이지 않음

수정: 4칸 들여쓰기 — 블록 구조가 명확함

TIP: VS Code 자동 들여쓰기 VS Code에서 { 를 입력하고 Enter를 누르면 자동으로 들여쓰기됩니다.

코드 전체를 자동 정렬하려면: Shift + Alt + F (Windows)

문장(statement) vs 블록(block)

문장과 블록은 C 프로그램의 두 가지 기본 구성 요소입니다.

구분문장(statement)블록(block)
정의하나의 동작을 지시여러 문장을 묶는 단위
끝 표시세미콜론 ;닫는 중괄호 }
예시printf("Hi\n");{ printf("A\n"); printf("B\n"); }
중첩불가블록 안에 블록 가능 (나중에 학습)

빈 문장과 의미 없는 세미콜론

세미콜론만 단독으로 쓰면 빈 문장(null statement)이 됩니다. 아무 동작도 하지 않습니다.

Cnull_stmt.c
1#include <stdio.h>
2
3int main(void) {
4 ; /* 빈 문장 — 아무 일도 안 함 */
5 ;; /* 빈 문장 2개 — 역시 아무 일도 안 함 */
6 printf("OK\n");
7 return 0;
8}

위 코드는 경고 없이 컴파일되고, "OK"를 출력합니다. 빈 문장은 에러가 아니지만 불필요한 코드입니다. 실수로 세미콜론을 이중으로 쓰면 빈 문장이 생길 수 있습니다.

용어 정리

용어설명
문장(statement)컴퓨터에게 하나의 동작을 지시하는 코드 단위입니다. 세미콜론(;)으로 끝납니다.
블록(block)중괄호 { }로 감싼 코드 영역입니다. 여러 문장을 하나로 묶습니다.
세미콜론(;)C에서 문장의 끝을 표시하는 기호입니다. 마침표(.)와 같은 역할입니다.
빈 문장(null statement)세미콜론만 단독으로 쓴 문장입니다. 아무 동작도 하지 않습니다.
들여쓰기(indentation)코드 앞에 공백을 넣어 블록 구조를 시각적으로 표현하는 것입니다. 보통 4칸을 사용합니다.
전처리 지시자(preprocessor directive)#으로 시작하는 명령입니다. 컴파일 전에 처리되며 세미콜론을 붙이지 않습니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: 세미콜론(;), 중괄호({ }), 빈 문장(null statement)
깊이: LAYER 1 — 세미콜론 규칙, 문장과 블록의 구분
핵심 규칙: 실행 문장 뒤에 ;, 전처리 지시자와 블록 경계 뒤에는 ; 없음
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 패턴 인식(Pattern Recognition)
세미콜론이 필요한 곳과 불필요한 곳의 규칙(패턴)을 발견했습니다:
- 동작 지시(문장) → ; 필요
- 구조 표시(블록, 전처리) → ; 불필요

이 패턴을 기억하면 새로운 C 구문을 만나도 세미콜론 여부를 판단할 수 있습니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. printf 뒤 ; 누락 → expected ';' before 'printf'
2. return 뒤 ; 누락 → expected ';' before '}'
3. 닫는 } 누락 → expected '}' at end of input
4. 여는 { 누락 → expected '{' before 'printf'
5. #include 뒤 ; → expected identifier or '('

핵심 기법: 에러가 가리키는 줄의 위쪽 줄도 확인하십시오.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1 (출력 정확성)
대회 코드 작성 시 체크리스트:
1. #include 뒤에 ; 없는지 확인
2. 모든 printf, return 뒤에 ; 있는지 확인
3. { } 짝이 맞는지 확인
4. 컴파일 에러 0개, 경고 0개 확인
축 5: 코딩 습관 [선택] LAYER 1 — 세미콜론/중괄호 습관:
- { 를 입력하면 즉시 }를 쓰고, 사이에 코드를 작성
- 한 문장을 쓸 때마다 세미콜론을 확인
- VS Code의 자동 완성({} 쌍 자동 생성)을 활용
- 에러 발생 시 해당 줄과 윗줄을 함께 확인
축 8: 타과목 연결 [선택] 국어 문법과의 비교:
- 세미콜론(;) = 마침표(.): 문장의 끝
- 중괄호({ }) = 문단: 관련 문장을 묶음
- 들여쓰기 = 들여쓰기: 하위 내용을 표시

프로그래밍 언어도 자연어처럼 문법 규칙이 있습니다. 다만 컴퓨터는 규칙을 하나라도 어기면 이해하지 못합니다.
