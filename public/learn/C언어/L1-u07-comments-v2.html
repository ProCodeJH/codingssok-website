주석이란?

주석(comment)은 코드에 다는 메모입니다. 컴파일러는 주석을 완전히 무시합니다. 주석은 사람이 코드를 이해하기 위해 존재합니다.

주석의 역할 1. 코드 설명: 이 코드가 무엇을 하는지, 왜 이렇게 작성했는지 기록합니다.
2. 임시 비활성화: 코드를 삭제하지 않고 잠시 실행되지 않게 합니다.
3. 정보 기록: 작성자, 날짜, 버전 등 부가 정보를 남깁니다.

블록 주석 — /* */

/*로 시작하고 */로 끝나는 주석입니다. 여러 줄에 걸쳐 쓸 수 있습니다. C89 표준부터 사용 가능합니다.

Cblock_comment.c
1#include <stdio.h>
2
3/* 이 프로그램은 인사말을 출력합니다.
4 작성일: 2026-03-12
5 작성자: 김코딩 */
6
7int main(void) {
8 printf("Hello\n"); /* 인사말 출력 */
9 return 0;
10}
줄코드역할
3/*블록 주석의 시작입니다. 이 지점부터 주석이 시작됩니다.
5*/블록 주석의 끝입니다. 이 지점에서 주석이 끝납니다.
8/* ... */한 줄 안에서 를 사용할 수도 있습니다. 코드 뒤에 짧은 설명을 달 때 유용합니다.

한 줄 주석 — //

//부터 그 줄의 끝까지가 주석입니다. C99 표준부터 공식 지원합니다. 대부분의 최신 컴파일러에서 사용 가능합니다.

Cline_comment.c
1#include <stdio.h>
2
3int main(void) {
4 // 첫 번째 출력
5 printf("Hello\n");
6
7 printf("World\n"); // 두 번째 출력
8 return 0; // 프로그램 종료
9}
줄코드역할
4//줄의 시작 부분에 를 쓰면 그 줄 전체가 주석입니다.
7//코드 뒤에 를 쓰면 그 지점부터 줄 끝까지만 주석입니다. 앞의 코드는 정상 실행됩니다.

주석은 컴파일에서 제거됩니다

주석은 전처리 단계에서 공백 하나로 대체됩니다. 컴파일러는 주석을 절대 보지 못합니다.

1
원본 코드
printf("Hi\n"); /* 인사 */
2
전처리 후
printf("Hi\n"); (주석이 공백으로 대체됨)
3
컴파일
주석이 없는 코드를 컴파일합니다
TIP: gcc -E로 확인 gcc -E hello.c 명령으로 전처리 결과를 확인하면 주석이 모두 사라진 것을 볼 수 있습니다. Unit 04에서 배운 전처리 과정에서 주석도 함께 처리됩니다.

주석 처리 — 디버깅 기법

코드의 일부를 주석으로 감싸면 삭제하지 않고 실행을 막을 수 있습니다. 이것을 "주석 처리(comment out)"라고 합니다. 버그를 찾을 때 유용합니다.

Ccomment_out.c
1#include <stdio.h>
2
3int main(void) {
4 printf("Step 1\n");
5 /* printf("Step 2\n"); */ /* 이 줄을 잠시 비활성화 */
6 printf("Step 3\n");
7 return 0;
8}
실행 결과

Step 2가 출력되지 않습니다. 주석 처리된 코드는 존재하지만 실행되지 않습니다. 문제를 찾으면 주석을 제거하여 다시 활성화합니다.

TIP: VS Code 단축키 주석 처리 단축키: Ctrl + /

줄을 선택하고 Ctrl + /를 누르면 주석 처리됩니다. 다시 누르면 주석이 해제됩니다. 여러 줄을 선택하고 누르면 한꺼번에 주석 처리됩니다.

좋은 주석 vs 나쁜 주석

주석은 코드가 "왜" 이렇게 작성되었는지를 설명해야 합니다. 코드를 그대로 번역한 주석은 불필요합니다.

DEBUG LAB — 에러를 찾으시오
C
1/* i를 1 증가시킨다 */
2i = i + 1;

에러: 코드를 그대로 번역 — 불필요한 주석

수정: "왜" 증가시키는지 설명 — 유용한 주석

좋은 주석나쁜 주석
왜 이렇게 작성했는지 설명코드를 그대로 번역
복잡한 로직의 의도 설명누구나 읽으면 아는 내용 반복
주의사항이나 제한사항 기록모든 줄에 주석 달기
TODO: 나중에 수정할 부분 표시오래되어 실제 코드와 맞지 않는 주석
주의: 거짓 주석 코드를 수정한 뒤 주석을 업데이트하지 않으면 주석과 코드가 불일치합니다. 잘못된 주석은 주석이 없는 것보다 해롭습니다.

/* 나이를 출력 */
printf("이름: 김코딩\n");

위 주석은 거짓입니다. 이름을 출력하지만 주석은 나이라고 합니다.

용어 정리

용어설명
주석(comment)코드에 다는 메모입니다. 컴파일러는 주석을 무시합니다.
블록 주석(block comment)/* 로 시작하고 */로 끝나는 주석입니다. 여러 줄에 걸쳐 사용할 수 있습니다.
한 줄 주석(line comment)//부터 줄 끝까지의 주석입니다. C99 표준부터 공식 지원합니다.
주석 처리(comment out)코드를 주석으로 감싸서 실행되지 않게 하는 디버깅 기법입니다.
파일 헤더(file header)파일 맨 위에 작성하는 프로그램 설명, 작성자, 날짜 등의 정보입니다.
중첩(nesting)하나의 구조 안에 같은 종류의 구조를 넣는 것입니다. /* */ 주석은 중첩할 수 없습니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: /* */ 블록 주석, // 한 줄 주석
깊이: LAYER 1 — 주석의 두 형태, 전처리에서의 처리 과정
핵심: 주석은 전처리 단계에서 공백으로 대체됩니다
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 추상화(Abstraction)
주석은 코드의 세부 구현을 숨기고 '의도'만 전달합니다. 이것이 추상화의 기본 개념입니다.

복잡한 코드를 한 줄 주석으로 요약하는 것은 추상화 연습입니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. /* 닫지 않음 → unterminated comment
2. /* */ 중첩 → 예상치 못한 위치에서 주석 종료
3. 코드 중간에 주석 삽입 → 식별자 분리
4. // 뒤 코드 실행 안 됨

새 기법: 주석 처리(comment out)로 버그 원인 범위를 좁힙니다
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1 (출력 정확성)
대회에서 주석 활용:
- 풀이 전략을 주석으로 먼저 정리
- 디버깅 시 코드 일부를 주석 처리하여 원인 파악
- 제출 시 불필요한 주석 제거 (채점에는 영향 없음)
축 5: 코딩 습관 [선택] LAYER 1 — 주석 습관:
- 파일 상단에 헤더 주석 작성 (파일명, 설명, 날짜)
- '왜'를 설명하는 주석만 작성
- 코드를 수정하면 관련 주석도 함께 수정
- VS Code 단축키 Ctrl + / 활용
축 9: AI 연결 [선택] AI와 주석의 관계:
AI 코드 생성 도구(GitHub Copilot 등)는 주석을 읽고 코드를 작성합니다. 주석으로 의도를 정확히 표현하면 AI가 더 정확한 코드를 생성합니다.

예: /* 1부터 100까지의 합을 구한다 */라고 주석을 쓰면 AI가 반복문 코드를 자동 생성할 수 있습니다.
