왜 코드 스타일이 중요한가?

C 컴파일러는 공백, 줄바꿈, 들여쓰기를 무시합니다. 문법만 맞으면 한 줄에 모든 코드를 써도 컴파일됩니다. 하지만 코드는 컴퓨터만 읽는 것이 아닙니다.

코드 스타일의 목적 코드는 사람이 읽는 것입니다.

1. 자기 자신: 일주일 뒤에 자신이 쓴 코드를 다시 읽어야 합니다.
2. 다른 사람: 팀원, 선생님, 대회 검토위원이 코드를 읽습니다.
3. 디버깅: 읽기 좋은 코드에서 버그를 더 빨리 찾습니다.

코드 스타일은 선택이 아니라 프로그래머의 기본 소양입니다.

들여쓰기 — 블록 구조 표현

들여쓰기(indentation)는 중괄호 { } 안의 코드를 오른쪽으로 밀어 블록 구조를 시각적으로 표현하는 것입니다.

DEBUG LAB — 에러를 찾으시오
C
1#include <stdio.h>
2
3int main(void) {
4printf("A\n");
5printf("B\n");
6return 0;
7}

에러: 들여쓰기 없음 — 어디서 블록이 시작/끝나는지 불명확

수정: 4칸 들여쓰기 — 블록 구조가 한눈에 보임

탭 vs 공백

들여쓰기에 탭(Tab 키)을 사용할 수도 있고, 공백(Space 키)을 사용할 수도 있습니다.

구분탭(Tab)공백(Space)
입력Tab 키 1번Space 키 4번
문자 수1문자4문자
표시 폭편집기 설정에 따라 다름항상 동일
장점적은 키 입력모든 환경에서 동일하게 보임
단점환경마다 폭이 다를 수 있음키 입력이 많음
TIP: VS Code 설정 VS Code는 Tab 키를 누르면 자동으로 공백 4칸을 삽입합니다(기본 설정).

하단 상태 표시줄에 "Spaces: 4"라고 표시됩니다. 이 설정을 유지하십시오.

탭과 공백을 섞어 쓰면 코드가 환경에 따라 다르게 보일 수 있으므로, 하나만 일관되게 사용합니다.

들여쓰기 깊이

중괄호가 중첩될 때마다 들여쓰기를 한 단계 더 합니다. 지금은 main 함수 하나만 사용하므로 1단계 들여쓰기만 필요합니다. 나중에 조건문, 반복문을 배우면 2단계, 3단계 들여쓰기를 사용하게 됩니다.

Cindent_levels.c
1#include <stdio.h> /* 0단계: 최상위 */
2
3int main(void) { /* 0단계 */
4 printf("Hello\n"); /* 1단계: main 블록 안 */
5 return 0; /* 1단계 */
6} /* 0단계 */

중괄호 배치 스타일

여는 중괄호 {의 위치에 따라 코드 스타일이 나뉩니다. 대표적으로 두 가지가 있습니다.

K&R 스타일

여는 중괄호를 같은 줄에 씁니다. K&R은 C 언어의 창시자인 Kernighan과 Ritchie의 이름에서 따왔습니다. Linux 커널, Google 스타일 가이드 등에서 사용합니다.

Cknr_style.c
1#include <stdio.h>
2
3int main(void) { /* { 가 같은 줄 */
4 printf("K&R 스타일\n");
5 return 0;
6}

Allman 스타일

여는 중괄호를 다음 줄에 씁니다. BSD 스타일이라고도 합니다. Microsoft, Visual Studio 기본 설정 등에서 사용합니다.

Callman_style.c
1#include <stdio.h>
2
3int main(void)
4{ /* { 가 다음 줄 */
5 printf("Allman 스타일\n");
6 return 0;
7}

어떤 스타일을 선택하든 일관성이 가장 중요합니다. 하나의 프로젝트 안에서 스타일을 섞지 마십시오.

공백 규칙

코드의 가독성을 높이는 공백 규칙입니다.

위치규칙예시
함수 이름과 괄호 사이공백 없음printf("Hi"); (O) / printf ("Hi"); (X)
{ 앞공백 1칸main(void) { (O) / main(void){ (X)
콤마 뒤공백 1칸printf("%d %d", a, b);

빈 줄논리적 구분#include와 main 사이에 빈 줄 1개

VS Code 자동 정렬

VS Code는 코드를 자동으로 정렬하는 기능을 제공합니다.

1
전체 자동 정렬
Shift + Alt + F — 파일 전체를 자동 정렬합니다.
2
선택 영역 정렬
코드를 선택한 뒤 Shift + Alt + F — 선택한 부분만 정렬합니다.
3
저장 시 자동 정렬
Settings → Format On Save 체크 → 저장할 때마다 자동 정렬됩니다.
TIP: Format On Save 설정 VS Code 설정(Ctrl + ,)에서 "Format On Save"를 검색하고 체크하면, Ctrl + S로 저장할 때마다 코드가 자동 정렬됩니다.

이 기능을 켜면 들여쓰기를 신경 쓰지 않아도 저장만 하면 코드가 정리됩니다.

에러 체험 — 들여쓰기가 논리를 가리는 경우

C에서 들여쓰기는 컴파일러에 영향을 주지 않습니다. 따라서 들여쓰기가 잘못되면 코드의 실제 동작과 눈에 보이는 구조가 달라질 수 있습니다.

Cmisleading.c
1#include <stdio.h>
2
3int main(void) {
4 printf("main 시작\n");
5 return 0;
6 printf("이 줄은 실행될까?\n");
7}

6번 줄은 들여쓰기가 깊지만 return 0; 이후에 있으므로 실행되지 않습니다. 들여쓰기가 실행 순서를 바꾸지 않습니다.
