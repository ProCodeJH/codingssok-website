컴파일 에러

자주 발생하는 컴파일 에러의 유형을 분류하고, 에러 메시지를 읽고 원인을 찾아 수정하는 체계적 방법을 학습합니다.

이 유닛에서 배우는 것

  • 에러 메시지의 구조(파일명:줄번호:설명)를 읽을 수 있습니다.
  • 자주 발생하는 컴파일 에러 10가지의 원인과 해결법을 알고 있습니다.
  • 에러(error)와 경고(warning)의 차이를 설명할 수 있습니다.
  • 첫 번째 에러부터 순서대로 수정하는 디버깅 전략을 적용할 수 있습니다.

에러 메시지의 구조

gcc 에러 메시지는 일정한 형식을 따릅니다. 이 형식을 알면 에러의 위치와 원인을 빠르게 파악할 수 있습니다.

Ccmd
1hello.c:4:5: error: expected ';' before 'return'
2 4 | printf("Hello\n")
3 | ^~~~~~
줄코드역할
1hello.c에러가 발생한 파일 이름입니다.
14에러가 발생한 줄 번호입니다.
15에러가 발생한 열(column) 번호입니다.
1error:에러 종류입니다. error 또는 warning입니다.
1expected ';' before 'return'에러 설명입니다. return 앞에 세미콜론이 필요하다는 뜻입니다.
에러 메시지 읽기 공식 파일명 : 줄번호 : 열번호 : 종류 : 설명

이 공식만 기억하면 어떤 에러 메시지든 읽을 수 있습니다. 가장 중요한 정보는 줄 번호와 설명입니다.

error vs warning

구분에러(error)경고(warning)
컴파일중단됨 — 실행 파일 생성 안 됨계속됨 — 실행 파일 생성됨
심각도반드시 수정해야 함수정하지 않아도 실행 가능
의미문법 규칙을 위반함잠재적 문제가 있음
예시세미콜론 누락, 괄호 짝 안 맞음#include 누락, 사용하지 않는 변수
목표에러 0개에러 0개 + 경고 0개
경고를 무시하지 마십시오 경고는 컴파일이 되지만 프로그램이 잘못 동작할 수 있다는 신호입니다.

gcc -Wall hello.c -o hello

-Wall 옵션으로 모든 경고를 확인하십시오. 좋은 코드는 -Wall에서도 경고가 0개입니다.

첫 번째 에러부터 수정하기

에러가 여러 개 표시되더라도 실제 원인은 하나일 수 있습니다. 하나의 실수가 연쇄적으로 여러 에러를 발생시키기 때문입니다.

Cchain_error.c
1#include <stdio.h>
2
3int main(void) {
4 printf("A\n")
5 printf("B\n");
6 printf("C\n");
7 return 0;
8}
Ccmd
1chain.c:5:5: error: expected ';' before 'printf'
2chain.c:5:5: error: expected expression before 'printf'
3chain.c:5:5: error: 'printf' undeclared

에러가 3개 표시되지만 실제 원인은 4번 줄의 세미콜론 누락 하나입니다. 첫 번째 에러만 수정하면 나머지도 사라집니다.

디버깅 전략 항상 첫 번째 에러부터 수정하십시오.

1. 첫 번째 에러 메시지를 읽습니다.
2. 해당 줄과 윗줄을 확인합니다.
3. 원인을 수정합니다.
4. 다시 컴파일합니다.
5. 남은 에러가 있으면 1번부터 반복합니다.

여러 에러를 한꺼번에 수정하려 하지 마십시오.

에러 유형 1 — 세미콜론 누락

가장 흔한 에러입니다. 문장 끝의 세미콜론을 빠뜨리면 발생합니다.

DEBUG LAB — 에러를 찾으시오
C
1printf("Hello\n")
2return 0;

에러: error: expected ';' before 'return'

수정: 세미콜론 추가

TIP: 에러 줄 번호의 함정 세미콜론 누락 에러는 다음 줄을 가리킵니다. "expected ';' before 'return'"이라면 return이 있는 줄이 아니라 그 윗줄에 세미콜론이 빠진 것입니다.

실전 디버깅 연습

에러가 있는 코드를 보고 원인을 찾아 수정하는 연습입니다.

Cdebug_practice.c
1#include <stdio.h>;
2
3int main(void) {
4 printf("Hello\n")
5 printf("World\n)
6 rturn 0;
7}

이 코드에는 에러가 4개 있습니다.

1
에러 1: 1번 줄
#include ; → 전처리 지시자 뒤에 세미콜론 제거
2
에러 2: 4번 줄
printf("Hello\n") → 세미콜론 누락. 끝에 ; 추가
3
에러 3: 5번 줄
printf("World\n) → 닫는 큰따옴표 누락. \n"); 로 수정
4
에러 4: 6번 줄
rturn 0; → return 철자 오류. return으로 수정

에러 체험 — 연쇄 에러

하나의 실수가 여러 에러를 발생시키는 연쇄 에러를 경험합니다.

Ccascade.c
1#include <stdio.h>
2
3int main(void) {
4 printf("A\n"
5 printf("B\n");
6 return 0;
7}
Ccmd
1cascade.c:5:5: error: expected ')' before 'printf'
2cascade.c:5:5: error: too few arguments to function 'printf'
3cascade.c:7:1: error: expected ';' before '}'

에러가 3개 표시되지만 실제 원인은 4번 줄의 닫는 괄호 누락 하나입니다. printf("A\n")에서 )와 ;가 빠졌습니다.

용어 정리

용어설명
컴파일 에러(compile error)C 문법 규칙을 위반하여 컴파일이 중단되는 오류입니다. 실행 파일이 생성되지 않습니다.
경고(warning)컴파일은 되지만 잠재적 문제가 있음을 알리는 메시지입니다.
연쇄 에러(cascade error)하나의 실수가 여러 에러 메시지를 발생시키는 현상입니다.
-Wall 옵션gcc의 옵션으로, 모든 경고 메시지를 활성화합니다. Warning All의 약자입니다.
implicit declaration함수가 선언되지 않은 상태에서 사용되었다는 경고입니다. #include 누락이나 함수 이름 오타가 원인입니다.
undefined reference링크 단계에서 함수의 실제 코드를 찾지 못했다는 에러입니다. main 이름 오류 등에서 발생합니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 지식: gcc 에러 메시지 형식, -Wall 옵션, 10가지 에러 유형
깊이: LAYER 1 — 컴파일 에러의 체계적 분류와 해결 전략
핵심: 에러 메시지는 적이 아니라 도우미입니다. 읽는 법을 익히면 디버깅이 빨라집니다.
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 체계적 디버깅(Systematic Debugging)
에러를 무작정 고치지 않고, 메시지를 읽고 → 위치를 찾고 → 원인을 분석하고 → 수정하는 체계적 절차를 적용했습니다.

이 절차는 프로그래밍뿐 아니라 모든 문제 해결에 적용됩니다.
축 3: 디버깅 [필수] 이 유닛의 디버깅 기법:
1. 에러 메시지 읽기 (파일:줄:열:종류:설명)
2. 첫 번째 에러부터 수정
3. 에러 줄의 윗줄도 확인
4. 연쇄 에러 인식
5. gcc -Wall로 경고까지 확인

이 다섯 가지 기법은 앞으로 모든 유닛에서 사용합니다.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1
대회에서 에러 대처법:
- 에러 메시지를 읽는 습관이 시간을 절약합니다
- 컴파일 에러는 제출 전에 반드시 해결 (CE는 0점)
- -Wall로 경고까지 확인하면 런타임 에러를 예방할 수 있습니다
축 5: 코딩 습관 [선택] LAYER 1 — 에러 대응 습관:
- 에러 메시지를 끝까지 읽기 (첫 줄만이라도)
- 패닉하지 않기 — 에러는 자연스러운 과정
- 한 줄 추가할 때마다 컴파일하기 (에러 위치 즉시 파악)
- gcc -Wall을 기본으로 사용하기
축 8: 타과목 연결 [선택] 영어와 에러 메시지:
gcc 에러 메시지는 영어입니다. 자주 나오는 단어를 알면 메시지를 빠르게 해석할 수 있습니다.

expected = 예상된, before = 앞에, missing = 빠진, undeclared = 선언되지 않은, implicit = 암묵적인, terminating = 끝맺는

에러 메시지를 읽는 것만으로도 영어 독해 연습이 됩니다.
