에러의 세 가지 종류

프로그래밍에서 발생하는 에러는 크게 세 가지로 분류됩니다.

종류발생 시점컴파일예시
컴파일 에러컴파일할 때실패 — 실행 파일 생성 안 됨세미콜론 누락, 괄호 짝 안 맞음
런타임 에러실행 중성공 — 실행 중 비정상 종료0으로 나누기, 잘못된 메모리 접근
논리 에러실행 중성공 — 정상 종료되지만 결과가 틀림잘못된 계산, 출력 순서 오류
핵심 구분 컴파일 에러: 컴파일러가 잡아줍니다. 에러 메시지가 나옵니다.
런타임 에러: 실행해 봐야 알 수 있습니다. 프로그램이 비정상 종료됩니다.
논리 에러: 실행은 되지만 결과가 틀립니다. 가장 찾기 어렵습니다.

난이도 순서: 컴파일 에러(쉬움) < 런타임 에러 < 논리 에러(어려움)

런타임 에러란?

런타임(runtime)은 프로그램이 실행되는 시간을 뜻합니다. 런타임 에러는 프로그램이 실행 중에 만나는 오류로, 프로그램이 비정상적으로 종료됩니다.

1
1. 소스 코드 작성
hello.c 파일을 작성합니다.
2
2. 컴파일 성공
gcc hello.c -o hello — 에러 없이 실행 파일 생성
3
3. 실행
./hello 실행 시작
4
4. 런타임 에러 발생
실행 도중 문제 발생 → 프로그램 비정상 종료

런타임 에러 1 — 0으로 나누기

수학에서 0으로 나누는 것은 정의되지 않습니다. 컴퓨터도 마찬가지입니다.

Cdiv_zero.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 10;
5 int b = 0;
6 printf("%d\n", a / b); /* 0으로 나누기 */
7 return 0;
8}
Ccmd
1C:\C_Study> gcc div_zero.c -o div_zero
2C:\C_Study> ./div_zero
3Floating point exception

컴파일은 정상적으로 완료됩니다. 하지만 실행하면 "Floating point exception"이라는 메시지와 함께 프로그램이 종료됩니다.

왜 컴파일러가 잡지 못하는가? int b = 0;은 문법적으로 완벽한 코드입니다. a / b도 문법적으로 올바릅니다. 컴파일러는 b의 값이 0인지 아닌지 실행해 보기 전에는 알 수 없습니다.

나중에 변수를 사용자 입력으로 받으면, 어떤 값이 들어올지 컴파일 시점에 알 수 없으므로 런타임에서만 확인 가능합니다.

런타임 에러 2 — 무한 출력

지금은 반복문을 배우지 않았으므로 직접 경험할 수 없지만, 개념을 미리 알아둡니다.

무한 루프란? 프로그램이 같은 동작을 끝없이 반복하는 것입니다. 화면에 글자가 끝없이 출력되거나, 프로그램이 멈추지 않습니다.

나중에 반복문(for, while)을 배울 때 자주 만나게 됩니다. 종료 조건을 잘못 설정하면 발생합니다.

해결: Ctrl + C로 프로그램을 강제 종료합니다.

런타임 에러 3 — 스택 오버플로

함수가 자기 자신을 끝없이 호출하면 메모리가 가득 차서 프로그램이 종료됩니다. 이것을 스택 오버플로(stack overflow)라고 합니다.

Cstack_overflow.c
1#include <stdio.h>
2
3void infinite(void) {
4 printf("calling...\n");
5 infinite(); /* 자기 자신을 다시 호출 — 끝나지 않음 */
6}
7
8int main(void) {
9 infinite();
10 return 0;
11}

이 코드는 컴파일됩니다. 하지만 실행하면 infinite 함수가 끝없이 호출되다가 메모리가 부족해져서 "Segmentation fault" 또는 "Stack overflow"로 종료됩니다.

TIP: 지금은 이해만 함수 정의, 재귀 호출은 나중에 배웁니다. 지금은 "함수가 자기 자신을 호출하면 메모리가 고갈될 수 있다"는 개념만 이해하면 충분합니다.

논리 에러란?

논리 에러는 프로그램이 정상적으로 실행되고 종료되지만, 결과가 기대와 다른 경우입니다. 에러 메시지가 전혀 나오지 않으므로 세 가지 에러 중 가장 찾기 어렵습니다.

Clogic_error.c
1#include <stdio.h>
2
3int main(void) {
4 /* 의도: "Hello World" 출력 */
5 printf("Hello Wrold\n"); /* World가 아닌 Wrold */
6 return 0;
7}
실행 결과

컴파일 에러도 없고 런타임 에러도 없습니다. 프로그램은 정상 종료됩니다. 하지만 출력이 "Hello World"가 아니라 "Hello Wrold"입니다. 이것이 논리 에러입니다.

논리 에러 예시 — 출력 순서

Clogic_order.c
1#include <stdio.h>
2
3int main(void) {
4 /* 의도: A B C 순서로 출력 */
5 printf("A\n");
6 printf("C\n"); /* B와 C가 뒤바뀜 */
7 printf("B\n");
8 return 0;
9}
실행 결과

컴파일러는 printf의 순서가 올바른지 판단하지 않습니다. 프로그래머가 의도한 순서와 실제 코드의 순서가 다르면 논리 에러입니다.

논리 에러 예시 — 이스케이프 시퀀스 실수

Clogic_escape.c
1#include <stdio.h>
2
3int main(void) {
4 /* 의도: 두 줄로 출력 */
5 printf("첫 번째 줄");
6 printf("두 번째 줄\n");
7 return 0;
8}
실행 결과

5번 줄에 \n이 없어서 두 문장이 한 줄에 붙어 출력됩니다. 에러 메시지는 없지만 의도와 다른 결과입니다.

온라인 저지에서의 에러

백준, 코드업 등 온라인 저지에서는 에러 유형에 따라 다른 판정을 내립니다.

판정의미원인
맞았습니다!!정답출력이 정확히 일치
컴파일 에러(CE)컴파일 실패문법 오류
런타임 에러(RE)실행 중 에러0으로 나누기, 메모리 접근 오류 등
틀렸습니다(WA)출력 불일치논리 에러 — 출력이 정답과 다름
시간 초과(TLE)시간 내 미완료무한 루프 또는 비효율적 알고리즘
메모리 초과(MLE)메모리 한도 초과과도한 메모리 사용
판정별 대응 CE(컴파일 에러): 에러 메시지를 읽고 문법 수정
RE(런타임 에러): 0으로 나누기, 배열 범위 초과 등 확인
WA(틀렸습니다): 출력 형식, 계산 로직, 예외 케이스 확인
TLE(시간 초과): 무한 루프 확인, 알고리즘 효율 개선

지금 단계에서는 CE와 WA를 주로 만납니다.

Ctrl + C — 프로그램 강제 종료

프로그램이 무한히 실행되거나 멈추지 않을 때 사용하는 단축키입니다.

Ctrl + C 터미널에서 Ctrl + C를 누르면 현재 실행 중인 프로그램을 강제로 종료합니다.

프로그램이 멈추지 않거나, 입력을 기다리는 상태에서 빠져나올 때 사용합니다.

이 단축키는 앞으로 자주 사용하게 됩니다.

에러 체험 — return 값과 종료 상태

Creturn_value.c
1#include <stdio.h>
2
3int main(void) {
4 printf("Hello\n");
5 return 1; /* 0이 아닌 값 반환 */
6}

이 코드는 정상적으로 실행되고 Hello를 출력합니다. 하지만 return 1;은 운영체제에 "에러가 있었다"고 보고합니다. 일부 시스템에서는 이것을 런타임 에러로 판정할 수 있습니다.

에러 체험 — printf 포맷과 값 불일치

Cformat_mismatch.c
1#include <stdio.h>
2
3int main(void) {
4 printf("%d\n"); /* %d가 있지만 값이 없음 */
5 return 0;
6}

이 코드는 gcc -Wall에서 경고가 발생합니다. 실행하면 쓰레기 값(예측할 수 없는 값)이 출력됩니다. 컴파일 에러는 아니지만 런타임에 잘못된 결과를 냅니다.

용어 정리

용어설명
런타임(runtime)프로그램이 실행되는 시간입니다. 컴파일 타임(compile time)의 반대 개념입니다.
런타임 에러(runtime error)컴파일은 성공하지만 실행 중에 프로그램이 비정상 종료되는 에러입니다.
논리 에러(logic error)프로그램이 정상 실행되지만 결과가 기대와 다른 에러입니다. 가장 찾기 어렵습니다.
0으로 나누기(division by zero)정수를 0으로 나누는 연산입니다. 대부분의 시스템에서 런타임 에러를 발생시킵니다.
스택 오버플로(stack overflow)함수 호출이 과도하게 쌓여 메모리가 부족해지는 현상입니다.
Segmentation fault프로그램이 허용되지 않은 메모리 영역에 접근할 때 발생하는 에러입니다.
WA(Wrong Answer)온라인 저지에서 출력이 정답과 일치하지 않을 때의 판정입니다.

핵심 요약

Unit 10 핵심 정리
1. 에러는 세 종류: 컴파일 에러(문법) → 런타임 에러(실행 중 종료) → 논리 에러(결과 틀림)
2. 런타임 에러: 컴파일은 되지만 실행 중에 비정상 종료됩니다
3. 논리 에러: 에러 메시지 없이 정상 종료되지만 결과가 틀립니다 — 가장 찾기 어렵습니다
4. 0으로 나누기, 무한 루프, 스택 오버플로가 대표적 런타임 에러입니다
5. 온라인 저지 판정: CE(컴파일), RE(런타임), WA(논리), TLE(시간 초과)
6. Ctrl + C로 멈추지 않는 프로그램을 강제 종료할 수 있습니다

축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 분류(Classification)
에러를 세 가지 종류로 분류했습니다. 분류는 복잡한 문제를 체계적으로 다루는 기본 기법입니다.

에러의 종류를 먼저 판별하면 해결 방향을 빠르게 결정할 수 있습니다.
축 3: 디버깅 [필수] 이 유닛의 디버깅 기법:
1. 에러 종류 판별: 컴파일 에러 → 런타임 에러 → 논리 에러
2. 런타임 에러: 에러 메시지 + 마지막 출력으로 위치 추정
3. 논리 에러: 예상 출력과 실제 출력을 한 줄씩 비교
4. 강제 종료: Ctrl + C

에러 종류에 따라 디버깅 전략이 달라집니다.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1
대회 판정 이해:
- CE → 문법 수정
- RE → 0으로 나누기, 범위 초과 확인
- WA → 출력 정확히 비교 (공백, 줄바꿈, 대소문자)
- TLE → 무한 루프 확인

판정 유형을 이해하면 수정 방향을 빠르게 결정할 수 있습니다.
축 5: 코딩 습관 [선택] LAYER 1 — 런타임 에러 예방 습관:
- 나누기 연산 전에 0인지 확인하는 습관 (나중에 if문으로)
- 출력 결과를 예상 결과와 한 줄씩 비교
- Ctrl + C 사용법 숙지
- 온라인 저지 제출 전 출력 형식 재확인
축 10: 하드웨어 연결 [선택] 런타임 에러와 하드웨어:
0으로 나누기는 CPU가 감지합니다. CPU의 나눗셈 회로가 0을 만나면 예외 신호를 발생시키고, 운영체제가 이를 받아 프로그램을 종료합니다.

스택 오버플로는 메모리(RAM) 부족입니다. 함수 호출마다 스택 메모리를 사용하는데, 무한 호출하면 할당된 메모리를 모두 소진합니다.
