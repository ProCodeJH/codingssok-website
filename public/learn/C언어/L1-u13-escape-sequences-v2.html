이스케이프 시퀀스란?

이스케이프 시퀀스(escape sequence)는 역슬래시(\)로 시작하는 특수 문자 표현입니다. 키보드로 직접 입력할 수 없는 문자(줄바꿈, 탭 등)나, 문자열 안에서 특별한 의미를 가진 문자(큰따옴표, 역슬래시)를 표현합니다.

왜 'escape'인가? 문자열 안에서 역슬래시(\)를 만나면 컴파일러는 "다음 문자를 일반 문자가 아닌 특수 문자로 해석하라"고 판단합니다. 일반적인 해석에서 '탈출(escape)'한다는 뜻입니다.

"Hello\n"에서 n은 알파벳이 아니라 줄바꿈으로 해석됩니다.

\n — 줄바꿈 (복습)

가장 자주 사용하는 이스케이프 시퀀스입니다. 커서를 다음 줄의 시작으로 이동합니다.

1#include <stdio.h>
2
3int main(void) {
4 printf("First\nSecond\nThird\n");
5 return 0;
6}

\a — 경고음

\a는 시스템 경고음(비프음)을 발생시킵니다. 터미널에 따라 소리가 나거나 화면이 깜빡입니다.

Calert.c
1#include <stdio.h>
2
3int main(void) {
4 printf("Error!\a\n");
5 return 0;
6}

"Error!"가 출력되고 시스템 경고음이 재생됩니다. 소리가 들리지 않는 환경도 있습니다.

참고 \a는 ASCII 코드 7번(BEL, bell)입니다. 초기 컴퓨터에는 실제 종(bell)이 달려 있었고, 이 문자를 출력하면 종이 울렸습니다.

\b — 백스페이스

\b는 커서를 한 칸 뒤로 이동합니다. 이전 문자를 지우는 것이 아니라 커서만 이동합니다. 뒤에 새 문자를 쓰면 이전 문자를 덮어씁니다.

Cbackspace.c
1#include <stdio.h>
2
3int main(void) {
4 printf("ABCDE\b\bXY\n");
5 return 0;
6}

동작 과정을 추적합니다.

1
ABCDE 출력
화면: ABCDE 커서 위치: E 뒤
2
\b — 커서 한 칸 뒤로
화면: ABCDE 커서 위치: E 위
3
\b — 커서 한 칸 뒤로
화면: ABCDE 커서 위치: D 위
4
X 출력 — D를 덮어씀
화면: ABCXE 커서 위치: E 위
5
Y 출력 — E를 덮어씀
화면: ABCXY 커서 위치: Y 뒤
6
\n — 줄바꿈
최종 출력: ABCXY

\r — 캐리지 리턴

\r은 커서를 현재 줄의 처음으로 이동합니다. 줄바꿈(\n)과 달리 줄을 바꾸지 않습니다.

Ccarriage.c
1#include <stdio.h>
2
3int main(void) {
4 printf("12345\rHi\n");
5 return 0;
6}
1
12345 출력
화면: 12345 커서 위치: 5 뒤
2
\r — 커서를 줄 처음으로
화면: 12345 커서 위치: 1 위
3
Hi 출력 — 1과 2를 덮어씀
화면: Hi345 커서 위치: 3 위
4
\n — 줄바꿈
최종 출력: Hi345
실행 결과
캐리지 리턴의 유래 타자기에서 종이를 고정한 캐리지(carriage)를 줄 시작 위치로 되돌리는(return) 동작에서 유래했습니다. 줄바꿈(\n)은 종이를 한 줄 올리는 라인 피드(line feed)입니다.

Windows의 줄바꿈은 \r\n(캐리지 리턴 + 라인 피드)이고, Linux/Mac은 \n만 사용합니다.

\0 — 널 문자

\0은 문자열의 끝을 나타내는 특수 문자입니다. C에서 모든 문자열은 \0으로 끝납니다.

Cnull_char.c
1#include <stdio.h>
2
3int main(void) {
4 printf("Hello"); /* 실제 메모리: H e l l o \0 */
5 return 0;
6}

"Hello"는 5글자이지만 메모리에는 6바이트가 저장됩니다. 마지막에 \0(널 문자)이 자동으로 추가됩니다. printf는 \0을 만나면 출력을 멈춥니다.

TIP: \0은 나중에 깊이 학습 \0(널 문자)은 문자열과 배열을 배울 때 매우 중요해집니다. 지금은 "C의 문자열은 \0으로 끝난다"는 사실만 기억하면 충분합니다.

이스케이프 시퀀스와 ASCII

모든 이스케이프 시퀀스는 ASCII 코드표의 제어 문자(control character)에 대응합니다. 제어 문자는 화면에 보이지 않지만 특정 동작을 수행합니다.

이스케이프ASCII 이름코드동작
\0NUL0문자열 끝 표시
\aBEL7경고음
\bBS8백스페이스
\tHT9수평 탭
\nLF10줄바꿈
\rCR13캐리지 리턴

ASCII 코드표에서 0~31번은 모두 제어 문자입니다. 이 중 C에서 이스케이프 시퀀스로 제공하는 것이 위 표의 문자들입니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: 이스케이프 시퀀스 전체 목록 (\n \t \\ \" \' \a \b \r \0 \f \v)
깊이: LAYER 1 — 자주 쓰는 4개 완전 숙달 + 나머지 이해
핵심: 이스케이프 시퀀스는 ASCII 제어 문자를 코드에서 표현하는 방법입니다
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 추적(Tracing)
\b와 \r의 동작을 이해하려면 커서의 위치를 한 문자씩 추적해야 합니다. 이것은 프로그램의 상태 변화를 추적하는 사고력의 기초입니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. \ 하나만 사용 → \n, \t 등으로 잘못 해석
2. 정의되지 않은 이스케이프 → unknown escape sequence 경고
3. 작은따옴표로 문자열 → character constant too long
4. \r로 출력 덮어쓰기 → 예상치 못한 결과
5. \b 후 새 문자 없음 → 효과 없음
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1 (출력 정확성)
대회에서 이스케이프 활용:
- 역슬래시 출력 문제 (고양이, 개, 새싹) → \\ 필수
- 큰따옴표 출력 → \" 사용
- 파일 경로 출력 → \\ 사용
- 정확한 줄바꿈 → \n 확인
축 8: 타과목 연결 [선택] 역사와 이스케이프 시퀀스:
\r(캐리지 리턴)과 \n(라인 피드)은 1800년대 타자기에서 유래했습니다. 타자기의 캐리지를 줄 시작으로 되돌리고(\r), 종이를 한 줄 올리는(\n) 두 동작이 별개였습니다.

Windows(\r\n)는 타자기 방식을 유지하고, Unix/Linux(\n)는 한 문자로 단순화했습니다.
축 10: 하드웨어 연결 [선택] ASCII와 하드웨어:
ASCII의 제어 문자(0~31번)는 원래 하드웨어 장치를 제어하기 위해 만들어졌습니다.

BEL(7) → 종 울림, BS(8) → 프린터 헤드 이동, HT(9) → 탭 위치 이동, LF(10) → 종이 올림, CR(13) → 캐리지 복귀

소프트웨어가 발전하면서 이 제어 문자들이 터미널 에뮬레이터에서 시뮬레이션됩니다.
