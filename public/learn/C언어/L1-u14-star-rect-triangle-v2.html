왜 별 그리기인가?

별 그리기는 단순해 보이지만 프로그래밍의 핵심 사고력을 기르는 훈련입니다. 도형의 패턴을 분석하고, 각 줄의 공백과 별 개수 규칙을 찾아내는 과정이 알고리즘적 사고의 기초입니다.

별 그리기의 목표 1. 패턴 분석: 각 줄의 공백과 별 개수를 규칙으로 표현합니다.
2. 정확한 출력: 공백 하나, 별 하나도 틀리지 않게 출력합니다.
3. 반복문 준비: 나중에 for문을 배우면 이 패턴을 자동화합니다.

지금은 printf로 직접 작성합니다. 패턴을 이해하는 것이 목표입니다.

에러 체험 — 공백과 별의 합 불일치

Cmisaligned.c
1printf(" *\n"); /* 공백4 + 별1 = 5 */
2printf(" **\n"); /* 공백3 + 별2 = 5 */
3printf(" ***\n"); /* 공백1 + 별3 = 4 (!) */
4printf(" ****\n"); /* 공백1 + 별4 = 5 */
5printf("*****\n"); /* 공백0 + 별5 = 5 */

3번째 줄에서 공백이 2개가 아니라 1개입니다. 합이 4로 다른 줄(5)과 다릅니다. 오른쪽 정렬 도형에서는 공백+별의 합이 모든 줄에서 동일해야 합니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 활용: printf로 공백과 별을 조합한 도형 출력
깊이: LAYER 1 — 새 문법 없이 printf만으로 복잡한 패턴 생성
핵심: 출력의 정밀 제어 (공백 1칸 단위 정렬)
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 패턴 인식(Pattern Recognition)
도형의 각 줄에서 공백과 별의 개수 규칙을 찾아냈습니다. 이것이 패턴 인식의 핵심입니다.

i번째 줄의 규칙을 수식으로 표현하는 것은 일반화(generalization)의 기초입니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. 공백 개수 실수 → 도형 어긋남
2. 탭과 공백 혼용 → 정렬 깨짐
3. 줄 끝 불필요한 공백 → WA
4. 마지막 \n 누락
5. 공백+별 합 불일치 → 오른쪽 정렬 실패

검증: 공백+별=총너비가 모든 줄에서 동일한지 확인
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 1-2 (출력 패턴)
대회에서 별 그리기:
- 별 그리기 문제는 반복문(for)으로 풀지만, 패턴을 먼저 분석해야 합니다
- 각 줄의 공백/별 규칙을 표로 정리한 뒤 코드를 작성하는 습관
- 줄 끝 공백, 마지막 줄바꿈 등 출력 형식 주의
축 5: 코딩 습관 [선택] LAYER 1 — 별 그리기 습관:
- 코드 작성 전에 패턴 분석표를 먼저 만듭니다
- 공백은 공백( )만 사용, 탭(\t) 금지
- 줄 끝에 불필요한 공백 넣지 않기
- 출력과 기대 결과를 한 줄씩 비교
축 7: 프로젝트 [선택] 별 그리기 → 텍스트 그래픽:
별 그리기는 텍스트 기반 그래픽의 기초입니다. 초기 컴퓨터 게임과 UI는 모두 문자로 화면을 구성했습니다.

공백과 별의 조합으로 도형을 만드는 원리는 픽셀 단위로 화면을 구성하는 그래픽 프로그래밍과 동일합니다.
