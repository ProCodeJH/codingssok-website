Chapter 03

calloc, realloc

calloc으로 0 초기화된 메모리를 할당하고, realloc으로 이미 할당된 메모리의 크기를 변경하는 방법을 배웁니다.

이 유닛에서 배우는 것

  • calloc의 사용법과 malloc과의 차이를 이해합니다.
  • realloc으로 동적 배열의 크기를 변경할 수 있습니다.
  • realloc의 반환값을 안전하게 처리할 수 있습니다.
  • 세 함수(malloc, calloc, realloc)를 상황에 맞게 선택할 수 있습니다.

calloc / realloc 문법
#include

void *calloc(size_t count, size_t size);
→ count개 × size바이트 할당, 0으로 초기화
→ calloc(5, sizeof(int)) = 20바이트, 모두 0

void *realloc(void *ptr, size_t newSize);
→ ptr의 크기를 newSize로 변경
→ 기존 데이터 유지, 늘어난 부분은 초기화되지 않음
→ 실패 시 NULL (원본은 그대로 유지)

대회 기출

요약

핵심 정리
1. calloc(count, size): 0으로 초기화된 메모리 할당
2. realloc(ptr, newSize): 기존 메모리 크기 변경
3. realloc 안전 패턴: temp 변수로 NULL 검사 후 대입
4. realloc(NULL, size) == malloc(size)
5. 2배 확장 전략: cap *= 2 (ArrayList 패턴)
6. 선택 기준: 0 초기화 → calloc, 크기 변경 → realloc, 기본 → malloc

다음 유닛으로

다음: 크기가 늘어나는 동적 배열 (프로젝트)
malloc, calloc, realloc을 모두 배웠습니다. 다음 프로젝트에서는 이 세 함수를 활용하여 크기가 자동으로 늘어나는 동적 배열(ArrayList)을 구현합니다.
