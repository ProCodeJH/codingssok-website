프로젝트 개요

설정 헤더 분리의 장점
1. 설정 변경 시 config.h 하나만 수정
2. 소스 코드(main.c)를 건드리지 않음
3. 빌드 모드 전환이 간편 (DEBUG_MODE 주석 토글)
4. 여러 .c 파일에서 같은 설정을 공유
5. gcc -DDEBUG_MODE로 명령줄에서도 전환 가능

대회 기출

요약

핵심 정리
1. 상수 설정: #define MAX_SIZE 100 (한 곳에서 수정)
2. 기능 토글: #define FEATURE_X 1 + #if FEATURE_X
3. 디버그 모드: #ifdef DEBUG_MODE + LOG 매크로
4. 설정 분리: config.h에 모든 #define 집중
5. 빌드 전환: gcc -DDEBUG_MODE (소스 수정 불필요)
6. 원칙: 설정(config.h) + 로직(main.c) 분리
