auto, register, volatile 키워드의 의미와 동작을 이해하고, extern/static과 함께 C의 기억 부류 지정자를 종합 정리합니다.

  • auto 키워드가 지역 변수의 기본 기억 부류임을 이해합니다.
  • register 키워드의 의미와 주소 연산자 사용 불가 제약을 이해합니다.
  • volatile 키워드가 컴파일러 최적화를 방지하는 이유를 설명할 수 있습니다.
  • 네 가지 기억 부류 지정자(auto, register, static, extern)를 비교 정리할 수 있습니다.

에러:

수정:

기억 부류 지정자 요약 규칙
1. auto: 쓸 필요 없습니다. 지역 변수는 자동으로 auto입니다.
2. register: 현대 코드에서는 거의 사용하지 않습니다. 컴파일러가 더 잘 최적화합니다.
3. volatile: 외부 요인(하드웨어, 시그널, 다른 스레드)이 변수를 바꿀 때 반드시 사용합니다.
4. static: 값 유지(지역) 또는 가시성 제한(전역/함수)에 사용합니다.
5. extern: 다른 파일의 전역 변수를 참조할 때 사용합니다.

하나의 변수에는 하나의 기억 부류 지정자만 사용할 수 있습니다.
(예: static register int x; 는 허용되지 않습니다.)

대회 기출

요약

핵심 정리
1. auto: 지역 변수의 기본값, 생략 가능
2. register: CPU 레지스터 저장 힌트, & 사용 불가
3. volatile: 컴파일러 최적화 방지, 매번 메모리에서 읽기
4. static: 값 유지(지역) 또는 파일 내부 제한(전역/함수)
5. extern: 다른 파일의 전역 변수 참조 (선언만)
6. 기억 부류 지정자는 하나만 사용 가능 (volatile은 별도 한정자)
