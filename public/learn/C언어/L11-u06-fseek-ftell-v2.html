Chapter 06

파일 위치

파일 내 읽기/쓰기 위치를 자유롭게 이동하는 fseek, ftell, rewind를 배웁니다.

이 유닛에서 배우는 것

  • ftell로 현재 파일 위치를 알 수 있습니다.
  • fseek로 파일의 원하는 위치로 이동할 수 있습니다.
  • rewind로 파일 처음으로 돌아갈 수 있습니다.
  • SEEK_SET, SEEK_CUR, SEEK_END의 차이를 이해합니다.

핵심 개념

fseek 함수
int fseek(FILE *fp, long offset, int origin);

fp — 파일 포인터
offset — 이동할 바이트 수 (양수: 앞으로, 음수: 뒤로)
origin — 기준 위치

SEEK_SET (0) — 파일 처음부터
SEEK_CUR (1) — 현재 위치부터
SEEK_END (2) — 파일 끝부터

성공 시 0, 실패 시 0이 아닌 값을 반환합니다.
ftell 함수
long ftell(FILE *fp);

현재 파일 위치를 바이트 단위로 반환합니다.
파일 시작이 0이고, 읽기/쓰기를 수행할수록 위치가 증가합니다.
실패 시 -1L을 반환합니다.
rewind 함수
void rewind(FILE *fp);

파일 위치를 처음(0)으로 되돌립니다.
fseek(fp, 0, SEEK_SET)과 동일한 효과이지만,
rewind는 에러 표시자도 초기화합니다.
C
1/* 파일 크기 측정 패턴 */
2FILE *fp = fopen("data.dat", "rb");
3if (fp == NULL) return 1;
4
5fseek(fp, 0, SEEK_END); /* 파일 끝으로 이동 */
6long size = ftell(fp); /* 현재 위치 = 파일 크기 */
7printf("파일 크기: %ld 바이트\n", size);
8
9rewind(fp); /* 다시 처음으로 */
10/* 이제 처음부터 읽기 가능 */
11fclose(fp);
C
1/* N번째 레코드 직접 접근 */
2typedef struct {
3 int id;
4 char name[20];
5 int score;
6} Record;
7
8FILE *fp = fopen("records.dat", "rb");
9int n = 5; /* 6번째 레코드 (0부터 시작) */
10
11fseek(fp, n * sizeof(Record), SEEK_SET);
12
13Record r;
14fread(&r, sizeof(Record), 1, fp);
15printf("ID: %d, 이름: %s, 점수: %d\n", r.id, r.name, r.score);
16fclose(fp);

1/* 레코드 수정 패턴: 읽기 → 수정 → 되돌아가기 → 쓰기 */
2FILE *fp = fopen("records.dat", "r+b");
3int n = 3; /* 4번째 레코드 수정 */
4
5fseek(fp, n * sizeof(Record), SEEK_SET);
6
7Record r;
8fread(&r, sizeof(Record), 1, fp); /* 읽기 (위치가 다음 레코드로 이동) */
9
10r.score = 100; /* 점수 수정 */
11
12/* fread 후 위치가 이동했으므로 다시 해당 레코드 위치로 되돌아감 */
13fseek(fp, n * sizeof(Record), SEEK_SET);
14fwrite(&r, sizeof(Record), 1, fp); /* 수정된 레코드 쓰기 */
15
16fclose(fp);

C
1// 에러 1: 텍스트 파일에서 fseek 사용 → 예측 불가능한 위치 (개행 변환)
2FILE *fp = fopen("text.txt", "r"); // 텍스트 모드!
3fseek(fp, 50, SEEK_SET);
4// Windows에서 \r\n이 \n으로 변환되므로 실제 위치가 달라질 수 있음
5// 수정: 바이너리 파일에서 fseek를 사용하거나, "rb" 모드로 열기
6fclose(fp);
7
8// 에러 2: fseek offset 계산 실수
9typedef struct { int id; char name[20]; } Rec;
10FILE *fp2 = fopen("data.dat", "rb");
11fseek(fp2, 3 * sizeof(int), SEEK_SET); // 잘못된 크기!
12// sizeof(int)가 아니라 sizeof(Rec)를 사용해야 올바른 레코드 위치로 이동
13// 수정: fseek(fp2, 3 * sizeof(Rec), SEEK_SET);
14fclose(fp2);
15
16// 에러 3: ftell 반환값이 long임을 무시
17FILE *fp3 = fopen("big.dat", "rb");
18fseek(fp3, 0, SEEK_END);
19int size = ftell(fp3); // int로 받으면 2GB 이상에서 오버플로!
20// 수정: long size = ftell(fp3);
21fclose(fp3);

대회 기출

요약

핵심 정리
1. fseek(fp, offset, origin) — 파일 위치를 원하는 곳으로 이동
2. ftell(fp) — 현재 파일 위치(바이트)를 반환
3. rewind(fp) — 파일 처음으로 이동 (에러 표시자도 초기화)
4. SEEK_SET: 파일 처음, SEEK_CUR: 현재 위치, SEEK_END: 파일 끝
5. 파일 크기 측정: fseek(fp, 0, SEEK_END); long size = ftell(fp);
6. N번째 레코드: fseek(fp, n * sizeof(Record), SEEK_SET);
7. 바이너리 파일에서 fseek가 텍스트 파일보다 안전함 (개행 변환 없음)

다음 유닛으로

다음: 파일 에러 처리 — feof, ferror
fseek, ftell, rewind로 파일 위치를 자유롭게 제어하는 방법을 배웠습니다. 다음 유닛에서는 feof와 ferror를 사용하여 파일 읽기 끝과 에러를 안전하게 처리하는 방법을 배웁니다.
