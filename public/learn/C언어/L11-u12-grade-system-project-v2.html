프로젝트 개요

완전한 성적 관리 시스템 (Grade Management System)
파일 입출력의 모든 기법을 종합하여 학생 성적을 관리하는 완전한 시스템을 구현합니다.

버전 1: 기본 -- 기본 CRUD + 텍스트 파일 저장

버전 3: 완성 -- 자동 저장 + CSV 내보내기 + 백업

실수 체험

실수 1: 삭제 시 파일 재구성 패턴
배열에서 요소를 삭제한 후 파일에 저장하지 않으면 데이터가 불일치합니다.

/* 잘못된 패턴: 삭제 후 파일 저장을 잊음 */
void deleteStudent(void) {
/* 배열에서는 삭제했지만... */
for (int i = idx; i < count - 1; i++) {
students[i] = students[i + 1];
}
count--;
/* 파일에는 여전히 이전 데이터가 남아 있음! */
}

/* 올바른 패턴: v3에서는 종료 시 자동 저장으로 해결 */
/* 또는 삭제 직후 즉시 파일에 다시 저장 */
void deleteAndSave(void) {
for (int i = idx; i < count - 1; i++) {
students[i] = students[i + 1];
}
count--;
saveBinary(); /* 즉시 파일에 반영 */
}

배열 기반 삭제는 뒤의 요소를 한 칸씩 앞으로 이동시킵니다. 삭제 후에는 반드시 파일에도 변경 사항을 반영해야 합니다.
실수 2: 파일 손상 시 복구
바이너리 파일이 손상되면 fread가 예상과 다른 값을 반환합니다.

/* 손상 감지 없이 읽기 */
fread(&count, sizeof(int), 1, fp);
fread(students, sizeof(Student), count, fp); /* count가 음수이면? */

/* 올바른 패턴: 유효성 검사 + 백업 복원 */
if (fread(&count, sizeof(int), 1, fp) != 1) {
/* 읽기 실패 -> 백업에서 복원 */
return restoreFromBackup();
}
if (count < 0 || count > MAX_STUDENTS) {
/* 비정상적인 값 -> 백업에서 복원 */
return restoreFromBackup();
}

중요한 데이터를 다루는 프로그램에서는 반드시 백업 전략을 갖추어야 합니다. 저장 전에 기존 파일을 .bak으로 복사하고, 손상 감지 시 백업에서 복원하는 것이 안전한 패턴입니다.
