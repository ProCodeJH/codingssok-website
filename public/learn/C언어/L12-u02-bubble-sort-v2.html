버블 정렬

인접한 두 원소를 비교하여 교환하는 버블 정렬을 구현하고, 시간복잡도 O(n²)를 분석합니다.

이 유닛에서 배우는 것

  • 버블 정렬의 동작 원리를 이해합니다.
  • 버블 정렬을 C로 구현할 수 있습니다.
  • 비교/교환 횟수로 시간복잡도 O(n²)를 이해합니다.
  • 조기 종료 최적화를 적용할 수 있습니다.

비교/교환 횟수 카운팅

C
1#include <stdio.h>
2int main(void) {
3 int arr[] = {5, 3, 8, 1, 2};
4 int n = 5;
5 int comp = 0, swap = 0;
6
7 for (int i = 0; i < n - 1; i++) {
8 for (int j = 0; j < n - 1 - i; j++) {
9 comp++; // 비교 횟수
10 if (arr[j] > arr[j + 1]) {
11 int temp = arr[j];
12 arr[j] = arr[j + 1];
13 arr[j + 1] = temp;
14 swap++; // 교환 횟수
15 }
16 }
17 }
18
19 printf("비교: %d회, 교환: %d회\n", comp, swap);
20 return 0;
21}

비교: 10회, 교환: 5회

1#include <stdio.h>
2int main(void) {
3 int arr[] = {1, 2, 3, 5, 8};
4 int n = 5;
5
6 for (int i = 0; i < n - 1; i++) {
7 int swapped = 0; // 매 패스 시작 시 초기화
8 for (int j = 0; j < n - 1 - i; j++) {
9 if (arr[j] > arr[j + 1]) {
10 int temp = arr[j];
11 arr[j] = arr[j + 1];
12 arr[j + 1] = temp;
13 swapped = 1;
14 }
15 }
16 if (!swapped) break; // 교환 없으면 정렬 완료
17 }
18
19 for (int i = 0; i < n; i++)
20 printf("%d ", arr[i]);
21 printf("\n");
22 return 0;
23}

시간복잡도 분석
최악(역순 배열): 비교 n(n-1)/2 = O(n²), 교환 n(n-1)/2 = O(n²)
최선(이미 정렬, 최적화 적용 시): 비교 n-1 = O(n), 교환 0
평균: O(n²)
공간복잡도: O(1) — 추가 배열 없이 제자리 정렬

패스별 배열 상태 변화

패스배열 상태확정 위치
초기[5, 3, 8, 1, 2]없음
1패스[3, 5, 1, 2, 8]arr[4] = 8
2패스[3, 1, 2, 5, 8]arr[3] = 5
3패스[1, 2, 3, 5, 8]arr[2] = 3
4패스[1, 2, 3, 5, 8]arr[1] = 2 (교환 없음, 종료)

에러 체험

에러 1: 이중 루프 범위 실수
for (int j = 0; j < n - 1; j++) // i를 빼지 않음

→ 이미 확정된 뒤쪽 원소까지 불필요하게 비교합니다.
→ 더 심각한 경우: j < n으로 쓰면 arr[j+1]이 배열 범위를 초과합니다.
→ for (int j = 0; j < n - 1 - i; j++)가 올바릅니다.
에러 2: 교환 로직 실수
arr[j] = arr[j + 1]; // temp 없이 바로 대입
arr[j + 1] = arr[j]; // 이미 덮어써진 값을 대입!

→ 두 값이 같아집니다. 반드시 임시 변수 temp를 사용하십시오.
int temp = arr[j]; arr[j] = arr[j+1]; arr[j+1] = temp;
에러 3: 조기 종료 플래그 초기화 위치 실수
int swapped = 0; // 바깥 루프 밖에서 한 번만 초기화
for (int i = 0; i < n - 1; i++) {
for (int j = 0; j < n - 1 - i; j++) { ... }
if (!swapped) break;
}

→ 첫 패스에서 교환이 없으면 바로 종료하지만,
첫 패스에서 교환 후 swapped가 1로 고정되어 이후 조기 종료가 불가능합니다.
→ swapped = 0은 바깥 for 루프 안, 안쪽 for 루프 전에 위치해야 합니다.

용어설명
버블 정렬(Bubble Sort)인접한 두 원소를 비교하여 순서가 잘못되면 교환하는 정렬. 가장 큰 값이 거품처럼 뒤로 이동합니다.
패스(Pass)배열 전체를 한 번 훑는 과정. 한 패스마다 가장 큰 미정렬 원소가 올바른 위치에 확정됩니다.
시간복잡도(Time Complexity)입력 크기 n에 대해 알고리즘이 수행하는 연산 횟수의 증가 속도. 버블 정렬은 O(n²)입니다.
조기 종료(Early Termination)한 패스에서 교환이 없으면 정렬이 완료된 것이므로 남은 패스를 건너뛰는 최적화 기법입니다.
역전 수(Inversion Count)i < j이면서 arr[i] > arr[j]인 쌍의 수. 버블 정렬의 교환 횟수와 같습니다.
