정렬 속도 경주 시각화 -- 프로젝트

버블, 선택, 삽입 정렬의 비교/교환 횟수를 측정하고 시각적으로 비교하는 프로그램을 구현합니다.

이 유닛에서 배우는 것

  • 여러 정렬 알고리즘의 성능을 실측할 수 있습니다.
  • 비교/교환 횟수를 카운팅하는 코드를 작성할 수 있습니다.
  • 결과를 표 형태로 시각화할 수 있습니다.
  • 입력 조건(정렬됨, 역순, 랜덤)에 따른 성능 차이를 확인할 수 있습니다.

프로젝트 개요

정렬 속도 경주 시각화 (Sort Race Visualizer)
버블 정렬, 선택 정렬, 삽입 정렬의 성능을 실측하여 비교하는 프로그램을 구현합니다.

버전 1: 기본 -- 버블 vs 선택 비교 (비교/교환 횟수 표 출력)
버전 2: 확장 -- 3가지 정렬 + 3가지 입력 조건 (3x3 결과 표 + 실행 시간)
버전 3: 완성 -- 크기별 비교 + ASCII 막대그래프 + CSV 저장

bubbleSort(arr, n); /* 정렬 완료 */
selectionSort(arr, n); /* 이미 정렬된 배열을 정렬! */

/* 올바른 패턴: 원본 배열을 복사하여 각각 정렬 */
copyArray(working, original, n);
bubbleSort(working, n);

copyArray(working, original, n); /* 원본에서 다시 복사 */
selectionSort(working, n);

정렬 알고리즘을 비교할 때는 반드시 같은 원본 배열을 복사하여 사용해야 합니다. 이미 정렬된 배열에서는 버블 정렬의 교환이 0회가 되어 공정한 비교가 불가능합니다.
실수 2: 카운터 위치를 잘못 배치
/* 잘못된 패턴: if 안에서만 카운팅 */
for (int j = 0; j < n - 1 - i; j++) {
if (arr[j] > arr[j + 1]) {
r.compares++; /* 비교가 참일 때만 세는 실수! */
/* swap */
}
}

/* 올바른 패턴: if 밖에서 카운팅 */
for (int j = 0; j < n - 1 - i; j++) {
r.compares++; /* 모든 비교를 셈 */
if (arr[j] > arr[j + 1]) {
/* swap */
r.swaps++; /* 교환이 발생할 때만 셈 */
}
}

비교 횟수는 if 조건을 평가하는 모든 경우를 세야 합니다. if 블록 안에 넣으면 조건이 참인 경우만 세게 되어 실제 비교 횟수보다 적게 측정됩니다.
정렬 알고리즘 성능 비교 정리
비교 횟수 교환 횟수 정렬된 배열
버블 n(n-1)/2 최대 n(n-1)/2 비교 n(n-1)/2, 교환 0
선택 n(n-1)/2 최대 n-1 비교 n(n-1)/2, 교환 0
삽입 최대 n(n-1)/2 최대 n(n-1)/2 비교 n-1, 교환 0

공통점: 세 알고리즘 모두 최악의 경우 O(n^2)입니다.
차이점: 삽입 정렬은 이미 정렬된 배열에서 O(n)으로 가장 빠릅니다.
선택 정렬은 입력 조건에 관계없이 비교 횟수가 일정합니다.
선택 정렬은 교환 횟수가 항상 최소(최대 n-1회)입니다.
