병합 정렬

배열을 반으로 나누어 각각 정렬한 뒤 합치는 병합 정렬을 배웁니다.

이 유닛에서 배우는 것

  • 병합(merge) 과정을 이해합니다.
  • 병합 정렬을 재귀로 구현합니다.
  • 병합 정렬이 항상 O(n log n)을 보장하는 이유를 이해합니다.
  • 퀵 정렬과 병합 정렬의 차이를 비교합니다.

병합 과정 시각화
두 정렬된 배열을 하나로 합치는 과정:

배열 A: [1, 3, 5] 배열 B: [2, 4, 6]
↑ i=0 ↑ j=0

단계 1: A[0]=1 < B[0]=2 → 결과에 1 추가 → [1]
단계 2: A[1]=3 > B[0]=2 → 결과에 2 추가 → [1, 2]
단계 3: A[1]=3 < B[1]=4 → 결과에 3 추가 → [1, 2, 3]
단계 4: A[2]=5 > B[1]=4 → 결과에 4 추가 → [1, 2, 3, 4]
단계 5: A[2]=5 < B[2]=6 → 결과에 5 추가 → [1, 2, 3, 4, 5]
단계 6: B[2]=6 남음 → 결과에 6 추가 → [1, 2, 3, 4, 5, 6]

1/* 정렬된 두 부분 배열을 합치는 함수 */
2void merge(int arr[], int left, int mid, int right) {
3 int n = right - left + 1;
4 int *temp = (int *)malloc(n * sizeof(int));
5 int i = left; /* 왼쪽 부분 배열의 인덱스 */
6 int j = mid + 1; /* 오른쪽 부분 배열의 인덱스 */
7 int k = 0; /* 임시 배열의 인덱스 */
8
9 /* 두 부분 배열을 비교하며 합침 */
10 while (i <= mid && j <= right) {
11 if (arr[i] <= arr[j])
12 temp[k++] = arr[i++];
13 else
14 temp[k++] = arr[j++];
15 }
16
17 /* 왼쪽 부분 배열의 남은 원소 복사 */
18 while (i <= mid)
19 temp[k++] = arr[i++];
20
21 /* 오른쪽 부분 배열의 남은 원소 복사 */
22 while (j <= right)
23 temp[k++] = arr[j++];
24
25 /* 임시 배열의 결과를 원래 배열에 복사 */
26 for (int m = 0; m < n; m++)
27 arr[left + m] = temp[m];
28
29 free(temp);
30}

6/* 병합 정렬: 배열을 반으로 나누어 재귀 정렬 후 합침 */
7void mergeSort(int arr[], int left, int right) {
8 if (left >= right) return; /* 원소가 1개 이하면 종료 */
9
10 int mid = (left + right) / 2;
11 mergeSort(arr, left, mid); /* 왼쪽 반 정렬 */
12 mergeSort(arr, mid + 1, right); /* 오른쪽 반 정렬 */
13 merge(arr, left, mid, right); /* 합치기 */
14}

시간복잡도와 공간복잡도
시간복잡도: 항상 O(n log n)
— 최선, 평균, 최악 모두 동일합니다.
— 배열을 log n 단계로 나누고, 각 단계에서 n번 비교하므로 n × log n입니다.

공간복잡도: O(n)
— 합칠 때 임시 배열이 필요하므로, 원래 배열 크기만큼의 추가 메모리가 필요합니다.
— 이것이 병합 정렬의 단점입니다. 퀵 정렬은 추가 메모리가 O(log n)만 필요합니다.

요약

핵심 정리
1. 병합 정렬: 배열을 반으로 나누고, 정렬된 두 부분을 합치면서 정렬
2. merge 함수: 정렬된 두 부분 배열을 임시 배열에 합친 뒤 원래 배열에 복사
3. mergeSort 함수: left >= right이면 종료, 아니면 반으로 나누어 재귀 호출 후 merge
4. 시간복잡도: 항상 O(n log n) — 최선, 평균, 최악 모두 동일
5. 공간복잡도: O(n) — 임시 배열이 필요하므로 퀵 정렬보다 메모리를 더 사용
6. 안정 정렬: 같은 값의 원래 순서가 유지됨 (퀵 정렬은 불안정)
