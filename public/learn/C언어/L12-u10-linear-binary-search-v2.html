Chapter 10

선형 탐색 vs 이진 탐색

순차적으로 찾는 선형 탐색과 반씩 줄여가는 이진 탐색을 비교하고 구현합니다.

이 유닛에서 배우는 것

  • 선형 탐색의 원리를 이해하고 O(n)으로 구현합니다.
  • 이진 탐색의 원리를 이해하고 O(log n)으로 구현합니다.
  • 이진 탐색의 전제조건(정렬된 배열)을 이해합니다.
  • 선형 탐색과 이진 탐색의 비교 횟수를 실측하여 성능 차이를 확인합니다.

선형 탐색 (Linear Search)
배열의 처음부터 끝까지 순서대로 하나씩 비교하여 원하는 값을 찾는 알고리즘입니다.
시간 복잡도: O(n) — 최악의 경우 배열 전체를 탐색합니다.
장점: 정렬되지 않은 배열에서도 사용할 수 있습니다.
단점: 데이터가 많으면 탐색 시간이 급격히 증가합니다.
C
1#include <stdio.h>
2
3/* 선형 탐색: 배열에서 target을 찾아 인덱스 반환, 없으면 -1 */
4int linear_search(int arr[], int n, int target) {
5 for (int i = 0; i < n; i++) {
6 if (arr[i] == target) {
7 return i; /* 찾으면 인덱스 반환 */
8 }
9 }
10 return -1; /* 못 찾으면 -1 반환 */
11}
12
13int main(void) {
14 int data[] = {4, 2, 7, 1, 9, 3, 8, 5};
15 int n = sizeof(data) / sizeof(data[0]);
16 int target = 9;
17
18 int idx = linear_search(data, n, target);
19 if (idx != -1) {
20 printf("%d은(는) 인덱스 %d에 있습니다.\n", target, idx);
21 } else {
22 printf("%d을(를) 찾을 수 없습니다.\n", target);
23 }
24 return 0;
25}
이진 탐색 (Binary Search)
정렬된 배열에서 중간값과 비교하여 탐색 범위를 절반씩 줄여가는 알고리즘입니다.
시간 복잡도: O(log n) — 매 단계마다 탐색 범위가 절반으로 줄어듭니다.
전제조건: 배열이 반드시 정렬되어 있어야 합니다.
n=1,000,000일 때 최대 약 20번의 비교로 값을 찾을 수 있습니다.
C
1#include <stdio.h>
2
3/* 이진 탐색 (while 루프 버전): 정렬된 배열에서 target 검색 */
4int binary_search(int arr[], int n, int target) {
5 int left = 0;
6 int right = n - 1;
7
8 while (left <= right) {
9 int mid = left + (right - left) / 2; /* 오버플로 방지 */
10
11 if (arr[mid] == target) {
12 return mid; /* 찾음 */
13 } else if (arr[mid] < target) {
14 left = mid + 1; /* 오른쪽 절반 탐색 */
15 } else {
16 right = mid - 1; /* 왼쪽 절반 탐색 */
17 }
18 }
19 return -1; /* 못 찾음 */
20}
21
22int main(void) {
23 int data[] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};
24 int n = sizeof(data) / sizeof(data[0]);
25 int target = 23;
26
27 int idx = binary_search(data, n, target);
28 if (idx != -1) {
29 printf("%d은(는) 인덱스 %d에 있습니다.\n", target, idx);
30 } else {
31 printf("%d을(를) 찾을 수 없습니다.\n", target);
32 }
33 return 0;
34}
이진 탐색 동작 추적 (Trace Table)
배열: [2, 5, 8, 12, 16, 23, 38, 56, 72, 91] target = 23

단계 | left | right | mid | arr[mid] | 비교 결과
-----|------|-------|-----|----------|----------
1 | 0 | 9 | 4 | 16 | 16 < 23 → left = 5
2 | 5 | 9 | 7 | 56 | 56 > 23 → right = 6
3 | 5 | 6 | 5 | 23 | 23 == 23 → 찾음!

3번의 비교만으로 10개 원소에서 값을 찾았습니다.
선형 탐색이었다면 6번 비교가 필요했습니다.

bsearch() 표준 라이브러리 함수
C 표준 라이브러리 에는 이진 탐색 함수 bsearch()가 포함되어 있습니다.

원형: void *bsearch(const void *key, const void *base, size_t nmemb, size_t size, int (*compar)(const void *, const void *));

매개변수:
key — 찾을 값의 포인터
base — 정렬된 배열의 시작 주소
nmemb — 배열의 원소 개수
size — 각 원소의 크기 (바이트)
compar — 비교 함수 (qsort와 동일한 형식)

반환값: 찾은 원소의 포인터 (못 찾으면 NULL)
C
1#include <stdio.h>
2#include <stdlib.h>
3
4int compare(const void *a, const void *b) {
5 return (*(int *)a - *(int *)b);
6}
7
8int main(void) {
9 int data[] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};
10 int n = sizeof(data) / sizeof(data[0]);
11 int key = 23;
12
13 int *result = (int *)bsearch(&key, data, n, sizeof(int), compare);
14
15 if (result != NULL) {
16 printf("%d을(를) 찾았습니다. (인덱스: %ld)\n", key, result - data);
17 } else {
18 printf("%d을(를) 찾을 수 없습니다.\n", key);
19 }
20 return 0;
21}
lower_bound / upper_bound 개념
이진 탐색을 확장하면 '값이 처음 나타나는 위치'와 '값보다 큰 첫 위치'를 찾을 수 있습니다.

lower_bound: target 이상인 첫 번째 위치
— 정렬된 배열에 target을 삽입할 수 있는 가장 왼쪽 위치

upper_bound: target 초과인 첫 번째 위치
— 정렬된 배열에 target을 삽입할 수 있는 가장 오른쪽 위치

예시: 배열 [1, 2, 2, 2, 3, 5], target = 2
lower_bound = 인덱스 1 (첫 번째 2의 위치)
upper_bound = 인덱스 4 (2보다 큰 첫 원소 3의 위치)
2의 개수 = upper_bound - lower_bound = 4 - 1 = 3개
C
1#include <stdio.h>
2
3/* target 이상인 첫 번째 인덱스 반환 */
4int lower_bound(int arr[], int n, int target) {
5 int left = 0, right = n;
6 while (left < right) {
7 int mid = left + (right - left) / 2;
8 if (arr[mid] < target) {
9 left = mid + 1;
10 } else {
11 right = mid;
12 }
13 }
14 return left;
15}
16
17/* target 초과인 첫 번째 인덱스 반환 */
18int upper_bound(int arr[], int n, int target) {
19 int left = 0, right = n;
20 while (left < right) {
21 int mid = left + (right - left) / 2;
22 if (arr[mid] <= target) {
23 left = mid + 1;
24 } else {
25 right = mid;
26 }
27 }
28 return left;
29}
30
31int main(void) {
32 int data[] = {1, 2, 2, 2, 3, 5};
33 int n = sizeof(data) / sizeof(data[0]);
34 int target = 2;
35
36 int lb = lower_bound(data, n, target);
37 int ub = upper_bound(data, n, target);
38
39 printf("lower_bound(%d) = %d\n", target, lb);
40 printf("upper_bound(%d) = %d\n", target, ub);
41 printf("%d의 개수 = %d\n", target, ub - lb);
42 return 0;
43}

대회 기출

요약

핵심 정리
1. 선형 탐색: 처음부터 끝까지 순서대로 비교 — O(n), 정렬 불필요
2. 이진 탐색: 중간값 비교로 범위를 절반씩 축소 — O(log n), 정렬 필수
3. mid 계산: mid = left + (right - left) / 2 (오버플로 방지)
4. 이진 탐색 종료 조건: left <= right (등호 포함 필수)
5. left/right 업데이트: left = mid + 1, right = mid - 1 (무한 루프 방지)
6. lower_bound: target 이상인 첫 위치 / upper_bound: target 초과인 첫 위치
7. bsearch(): C 표준 라이브러리의 이진 탐색 함수 (stdlib.h)
8. n=1,000,000일 때 선형 1,000,000번 vs 이진 약 20번
