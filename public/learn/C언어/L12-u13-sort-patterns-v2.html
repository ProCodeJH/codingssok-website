상황별 알고리즘 선택 가이드
n < 50 → 삽입 정렬: 오버헤드가 적고 코드가 간단합니다.
n < 10,000 → qsort: 평균 O(n log n)으로 실전에서 가장 빠릅니다.
값의 범위가 작음 (k < 10,000) → 계수 정렬: O(n+k)로 비교 기반보다 빠릅니다.
안정 정렬 필요 → 병합 정렬: 같은 키의 상대 순서를 보존합니다.
거의 정렬된 데이터 → 삽입 정렬: 최선 O(n)으로 이동이 거의 없습니다.
교환 비용이 큰 경우 → 선택 정렬: 교환 횟수가 최대 n-1번입니다.
메모리 제한 엄격 → 버블/선택/삽입/퀵 정렬: O(1) 추가 공간만 사용합니다.

2// n = 100,000인데 버블 정렬 사용 → O(n²) = 10,000,000,000 연산
3// 1초 제한에서 시간 초과 발생!
4// 수정: qsort 사용 → O(n log n) ≈ 1,700,000 연산
5
6void bubbleSort(int arr[], int n) { // n=100000이면 TLE!
7 for (int i = 0; i < n - 1; i++)
8 for (int j = 0; j < n - 1 - i; j++)
9 if (arr[j] > arr[j+1]) {
10 int t = arr[j]; arr[j] = arr[j+1]; arr[j+1] = t;
11 }
12}
13// 수정: qsort(arr, n, sizeof(int), cmpInt); 로 변경
14
15// 에러 2: 안정 정렬이 필요한데 qsort 사용
16// 1차 이름순 정렬 후 2차 점수순 정렬 시
17// qsort는 불안정 → 같은 점수 내 이름순 깨짐!
18// 수정: 비교 함수에서 점수 같으면 이름으로 2차 비교
19int cmpStudent(const void *a, const void *b) {
20 Student *sa = (Student *)a, *sb = (Student *)b;
21 if (sa->score != sb->score)
22 return sa->score - sb->score;
23 return strcmp(sa->name, sb->name); // 2차 기준 추가!
24}
25
26// 에러 3: qsort 비교 함수에서 int 오버플로
27int cmpBad(const void *a, const void *b) {
28 return *(int *)a - *(int *)b; // a=2000000000, b=-2000000000이면 오버플로!
29}
30// 수정: 조건 분기 사용
31int cmpSafe(const void *a, const void *b) {
32 int va = *(int *)a, vb = *(int *)b;
33 if (va < vb) return -1;
34 if (va > vb) return 1;
35 return 0;
36}

대회 기출

요약

핵심 정리
1. 6대 정렬: 버블/선택/삽입은 O(n²), 퀵/병합은 O(n log n), 계수는 O(n+k)입니다
2. 안정 정렬: 버블, 삽입, 병합, 계수 — 같은 키의 원래 순서를 보존합니다
3. 불안정 정렬: 선택, 퀵 — 같은 키의 순서가 바뀔 수 있습니다
4. 실전 선택: n 작으면 삽입, 일반적이면 qsort, 값 범위 작으면 계수, 안정 필요 시 병합입니다
5. 탐색: 정렬 안 됨 → 선형 O(n), 정렬됨 → 이진/bsearch O(log n)입니다
6. KOI 패턴: 정렬+탐색 조합, 좌표 압축, 비교 함수 커스텀이 핵심입니다
7. 비교 함수 주의: 뺄셈 방식은 int 오버플로 위험이 있으므로 조건 분기를 권장합니다
