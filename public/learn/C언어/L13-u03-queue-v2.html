선형 큐의 문제점
선형 큐에서 dequeue를 하면 front가 앞으로 이동합니다.
앞쪽에 빈 공간이 생기지만, rear가 SIZE에 도달하면
더 이상 삽입할 수 없습니다.

┌───┬───┬───┬───┬───┐
│빈 │빈 │ 30│ 40│ 50│ ← rear==5, front==2
└───┴───┴───┴───┴───┘
앞 2칸이 비었는데 "가득 참" 판정!

해결책: 원형 큐 (Circular Queue)

C
1#include <stdio.h>
2#define SIZE 5 /* 실제 저장 가능: SIZE-1 = 4개 (한 칸 비움) */
3
4int queue[SIZE];
5int front = 0;
6int rear = 0;
7
8int isEmpty(void) {
9 return front == rear;
10}
11
12int isFull(void) {
13 return (rear + 1) % SIZE == front;
14}
15
16void enqueue(int x) {
17 if (isFull()) {
18 printf("원형 큐가 가득 찼습니다!\n");
19 return;
20 }
21 queue[rear] = x;
22 rear = (rear + 1) % SIZE;
23 printf("enqueue: %d (rear=%d)\n", x, rear);
24}
25
26int dequeue(void) {
27 if (isEmpty()) {
28 printf("원형 큐가 비었습니다!\n");
29 return -1;
30 }
31 int val = queue[front];
32 front = (front + 1) % SIZE;
33 printf("dequeue: %d (front=%d)\n", val, front);
34 return val;
35}
36
37int main(void) {
38 enqueue(10); /* rear=1 */
39 enqueue(20); /* rear=2 */
40 enqueue(30); /* rear=3 */
41 enqueue(40); /* rear=4 */
42 /* (4+1)%5==0==front → 가득 참 */
43
44 dequeue(); /* 10, front=1 */
45 dequeue(); /* 20, front=2 */
46
47 enqueue(50); /* rear=0 (순환!) */
48 enqueue(60); /* rear=1 */
49 /* (1+1)%5==2==front → 가득 참 */
50
51 printf("\n=== 남은 데이터 ===\n");
52 while (!isEmpty()) {
53 dequeue();
54 }
55 return 0;
56}

원형 큐 인덱스 순환 원리
배열 크기 SIZE = 5일 때:

인덱스: 0 → 1 → 2 → 3 → 4 → 0 → 1 → ...
계산: (현재 + 1) % 5

예시:
(3 + 1) % 5 = 4
(4 + 1) % 5 = 0 ← 끝에서 처음으로 순환!
(0 + 1) % 5 = 1

큐 활용 사례
1. 프린터 대기열
문서를 보낸 순서대로 인쇄합니다.

2. BFS (너비 우선 탐색)
그래프/트리에서 가까운 노드부터 탐색할 때 큐를 사용합니다.

3. 작업 스케줄링
운영체제가 프로세스를 순서대로 실행할 때 사용합니다.

4. 버퍼(Buffer)
데이터 전송 속도 차이를 큐로 완충합니다.
(키보드 입력 버퍼, 네트워크 패킷 버퍼 등)

대회 기출

요약

핵심 정리
1. 큐: FIFO (First In First Out) — 먼저 넣은 데이터가 먼저 나옴
2. 선형 큐: front++로 삭제, rear++로 삽입 — 앞 공간 재사용 불가
3. 원형 큐: (index + 1) % SIZE로 인덱스 순환 — 공간 재사용
4. full 판별: (rear + 1) % SIZE == front (한 칸 비우기 방식)
5. empty 판별: front == rear
6. 활용: 프린터 대기열, BFS, 작업 스케줄링, 버퍼
