앞 삽입 시 포인터 연결 순서
head가 [20] → [30]일 때 [10]을 앞에 삽입:

단계 1: newNode->next = head [10]→[20]→[30]
단계 2: head->prev = newNode [10]⇄[20]→[30]

[10] ⇄ [20] ⇄ [30]에서 [20]을 삭제:

삭제 전: [10] ⇄ [20] ⇄ [30]
단계 1: [10].next = [20].next → [10]→[30] (앞 노드의 next)
단계 2: [30].prev = [20].prev → [10]←[30] (뒷 노드의 prev)
삭제 후: [10] ⇄ [30] free([20])

head 삭제: NULL ← [20] ⇄ [30] → head = [20]
tail 삭제: [10] ⇄ [20] → NULL → tail = [20]

요약

핵심 정리
1. 이중 연결 리스트 노드: struct DNode { int data; struct DNode *prev, *next; };
2. 양방향 탐색: head에서 next로 정방향, tail에서 prev로 역방향 순회 가능
3. 삽입 시 포인터 4개 연결: newNode의 prev/next 먼저 설정, 그 후 주변 노드 연결
4. 삭제 시 앞 노드의 next와 뒷 노드의 prev를 서로 연결 후 free
5. head/tail 경계 처리: 삭제/삽입 시 head나 tail이 변경되는 경우 반드시 갱신
6. 단일 대비 장점: 역방향 순회 O(n), 특정 노드 삭제 O(1) (노드 주소를 알 때)
