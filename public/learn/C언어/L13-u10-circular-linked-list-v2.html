tail 포인터의 장점
head만 유지하는 경우:
앞 삽입 O(1), 뒤 삽입 O(n) — tail을 찾으려면 순회 필요

tail을 유지하는 경우:
앞 삽입 O(1): tail->next가 head이므로 바로 접근
뒤 삽입 O(1): tail 뒤에 삽입 후 tail 갱신

→ tail 포인터 하나로 head와 tail 모두 O(1) 접근 가능

1void insertFront(Node **tail, int data) {
2 Node *newNode = (Node *)malloc(sizeof(Node));

4
5 if (*tail == NULL) {

앞에서 삭제 (deleteFront)

C
1int deleteFront(Node **tail) {
2 if (*tail == NULL) {
3 printf("리스트가 비어 있습니다.\n");
4 return -1;
5 }
6
7 Node *head = (*tail)->next;
8 int data = head->data;
9
10 if (head == *tail) {
11 /* 노드가 1개: 삭제 후 빈 리스트 */
12 *tail = NULL;
13 } else {
14 /* tail->next를 head 다음으로 연결 */
15 (*tail)->next = head->next;
16 }
17
18 free(head);
19 return data;
20}

특정 노드 삭제 (deleteNode)

C
1int deleteNode(Node **tail, int target) {
2 if (*tail == NULL) return -1;
3
4 Node *head = (*tail)->next;
5 Node *prev = *tail;
6 Node *cur = head;
7
8 /* 삭제할 노드 탐색 */
9 do {
10 if (cur->data == target) {
11 if (cur == prev) {
12 /* 노드가 1개뿐 */
13 *tail = NULL;
14 } else {
15 prev->next = cur->next;
16 if (cur == *tail) *tail = prev;
17 }
18 int data = cur->data;
19 free(cur);
20 return data;
21 }
22 prev = cur;
23 cur = cur->next;
24 } while (cur != head);
25
26 return -1; /* 못 찾음 */
27}

9void josephus(int n, int k) {

12 head->data = 1;
13 head->next = head;
14

에러 체험

에러 1: 무한 루프 (순회 종료 조건 실수)
Node *cur = head;
while (cur != NULL) { /* 원형 리스트에 NULL이 없음! */
printf("%d ", cur->data);
cur = cur->next;
}
// → 무한 루프 발생

해결: while (cur != NULL) 대신 do-while (cur != head) 사용
에러 2: 빈 리스트에서 tail->next 접근
Node *tail = NULL;
Node *head = tail->next; /* NULL 역참조! → 세그폴트 */

해결: if (tail == NULL) return; 으로 빈 리스트 검사
에러 3: 노드 1개일 때 삭제 후 tail 미갱신
if (head == tail) {
free(head);
/* tail을 NULL로 설정하지 않음! */
/* → tail이 해제된 메모리를 가리킴 (댕글링 포인터) */
}

해결: free(head); *tail = NULL; 순서로 처리

용어 정리

용어설명
원형 연결 리스트(Circular Linked List)마지막 노드의 next가 첫 번째 노드를 가리켜 순환 구조를 이루는 연결 리스트입니다.
tail 포인터원형 연결 리스트에서 마지막 노드를 가리키는 포인터입니다. tail->next로 head에 접근합니다.
순환 순회(Circular Traversal)시작 노드에서 출발하여 다시 시작 노드로 돌아올 때까지 순회하는 방법입니다.
요세푸스 문제(Josephus Problem)N명이 원형으로 앉아 K번째 사람을 반복 제거할 때 최후의 생존자를 구하는 문제입니다.
댕글링 포인터(Dangling Pointer)이미 해제된 메모리를 가리키는 포인터입니다. 접근하면 정의되지 않은 동작이 발생합니다.
라운드 로빈(Round Robin)순환 방식으로 자원을 배분하는 스케줄링 알고리즘입니다. 원형 연결 리스트로 구현할 수 있습니다.

핵심 요약

핵심 정리
1. 원형 연결 리스트: 마지막 노드의 next가 head를 가리키는 순환 구조
2. tail 포인터로 관리하면 head와 tail 모두 O(1) 접근 가능
3. 순회 종료 조건: do-while (cur != head) 사용 필수
4. 빈 리스트(tail == NULL)와 노드 1개(자기 자신 가리킴)는 특수 처리 필요
5. 요세푸스 문제: 원형 연결 리스트의 대표 활용 사례
6. 단일/이중/원형 각각 장단점이 있으며 용도에 맞게 선택

축 1: C언어 [필수]
이 유닛의 C 지식: 구조체 포인터, 동적 메모리 할당, 자기 참조 구조체

핵심: tail->next = head 한 줄이 단일 연결 리스트를 원형으로 바꿉니다
축 2: 컴퓨팅 사고력 [필수]
사고 패턴: 순환(Cycle) 인식
원형 구조는 끝이 없습니다. 종료 조건을 명시적으로 설정해야 합니다.
순환 구조의 시작과 끝을 정의하는 것이 핵심 사고력입니다.
축 3: 디버깅 [필수]
이 유닛의 디버깅 기법:
1. 무한 루프: while (cur != NULL) 대신 do-while (cur != head)
2. NULL 역참조: 빈 리스트에서 tail->next 접근 전 검사
3. 댕글링 포인터: free 후 반드시 tail = NULL 설정
4. 노드 1개 삭제: 자기 자신을 가리키는 특수 케이스 처리
축 4: KOI — 정보올림피아드 [필수]
난이도: TIER 4
대회 활용:
- 요세푸스 문제 (백준 1158): 원형 연결 리스트의 직접 출제
- 라운드 로빈 스케줄링 시뮬레이션
- 순환 구조 탐지 문제의 기초

원형 연결 리스트는 시뮬레이션 문제에서 자주 등장합니다.
축 5: 코딩 습관 [선택]
LAYER 3 — 방어적 프로그래밍:
- 빈 리스트 검사를 모든 함수의 첫 줄에 배치
- 노드 1개 케이스를 별도 분기로 처리
- free 직후 포인터를 NULL로 초기화
- 함수 이름으로 동작을 명확히 표현 (insertFront, deleteFront)
축 6: 수학 연결 [선택]
요세푸스 문제의 수학적 풀이:
J(n, k) = (J(n-1, k) + k) % n (0-indexed)
J(1, k) = 0

재귀 공식으로 O(n)에 해결 가능하지만, 원형 연결 리스트를 사용하면 제거 순서까지 출력할 수 있습니다.
축 7: 실생활 연결 [선택]
원형 연결 리스트의 실생활 활용:
- 음악 플레이어의 반복 재생: 마지막 곡 → 첫 곡으로 돌아감
- 운영체제의 라운드 로빈 CPU 스케줄링
- 멀티플레이어 게임의 턴 관리
- 회전목마, 원형 버퍼(Ring Buffer)
축 10: 하드웨어 연결 [선택]
원형 버퍼(Ring Buffer)와 하드웨어:
네트워크 카드, 오디오 장치 등의 드라이버는 원형 버퍼를 사용합니다. 고정 크기 배열에서 head와 tail 인덱스가 끝에 도달하면 처음으로 돌아갑니다. 이것이 하드웨어 레벨의 원형 연결 리스트입니다.
