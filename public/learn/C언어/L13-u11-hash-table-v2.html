해시 테이블 기초

해시 함수와 체이닝으로 O(1) 평균 탐색이 가능한 해시 테이블을 구현합니다.

이 유닛에서 배우는 것

  • 해시 함수의 개념(키를 인덱스로 변환)을 이해할 수 있습니다.
  • 충돌(collision)의 원인을 파악하고 체이닝으로 해결할 수 있습니다.
  • 해시 테이블의 insert/search/delete 연산을 직접 구현할 수 있습니다.
  • O(1) 평균 탐색 시간의 원리와 적재율의 관계를 설명할 수 있습니다.

핵심 개념

해시 함수 (Hash Function)
키(key)를 배열 인덱스로 변환하는 함수입니다.

기본 형태: index = key % TABLE_SIZE

예) TABLE_SIZE = 7일 때
hash(15) = 15 % 7 = 1
hash(29) = 29 % 7 = 1 (충돌 발생!)
hash(10) = 10 % 7 = 3
충돌(Collision)과 체이닝(Chaining)
서로 다른 키가 같은 인덱스를 반환하는 현상을 충돌이라 합니다.

체이닝: 각 인덱스에 연결 리스트를 두어 충돌된 데이터를 이어 붙이는 방식입니다.

[0] -> NULL
[1] -> [15] -> [29] -> NULL
[2] -> NULL
[3] -> [10] -> NULL
...

같은 인덱스의 데이터는 연결 리스트를 순회하여 탐색합니다.

해시 테이블 구조체 정의

C
1#include <stdio.h>
2#include <stdlib.h>
3#include <string.h>
4
5#define TABLE_SIZE 13
6
7typedef struct Node {
8 int key;
9 int value;
10 struct Node *next;
11} Node;
12
13typedef struct {
14 Node *table[TABLE_SIZE];
15} HashTable;
16
17int hash(int key) {
18 return ((key % TABLE_SIZE) + TABLE_SIZE) % TABLE_SIZE;
19 // key가 음수여도 양수 인덱스를 보장합니다.
20}
21
22void initTable(HashTable *ht) {
23 for (int i = 0; i < TABLE_SIZE; i++)
24 ht->table[i] = NULL;
25}

insert 함수

C
1void insert(HashTable *ht, int key, int value) {
2 int idx = hash(key);
3
4 // 기존 키가 있으면 값 갱신
5 Node *cur = ht->table[idx];
6 while (cur != NULL) {
7 if (cur->key == key) {
8 cur->value = value;
9 return;
10 }
11 cur = cur->next;
12 }
13
14 // 새 노드를 연결 리스트 앞에 삽입
15 Node *newNode = (Node *)malloc(sizeof(Node));
16 newNode->key = key;
17 newNode->value = value;
18 newNode->next = ht->table[idx];
19 ht->table[idx] = newNode;
20}

search 함수

C
1Node *search(HashTable *ht, int key) {
2 int idx = hash(key);
3 Node *cur = ht->table[idx];
4 while (cur != NULL) {
5 if (cur->key == key)
6 return cur;
7 cur = cur->next;
8 }
9 return NULL; // 키를 찾지 못함
10}

delete 함수

C
1void delete(HashTable *ht, int key) {
2 int idx = hash(key);
3 Node *cur = ht->table[idx];
4 Node *prev = NULL;
5
6 while (cur != NULL) {
7 if (cur->key == key) {
8 if (prev == NULL)
9 ht->table[idx] = cur->next;
10 else
11 prev->next = cur->next;
12 free(cur);
13 return;
14 }
15 prev = cur;
16 cur = cur->next;
17 }
18}

체이닝 vs 개방 주소법 비교

충돌 해결 방법 비교표
항목 | 체이닝(Chaining) | 개방 주소법(Open Addressing)
-----------------+------------------------+-----------------------------
구조 | 배열 + 연결 리스트 | 배열만 사용
충돌 처리 | 같은 인덱스에 노드 추가| 다른 빈 칸을 탐색
메모리 | 동적 할당 필요 | 추가 할당 불필요
삭제 | 간단 (노드 제거) | 복잡 (삭제 표시 필요)
적재율 > 1 | 가능 | 불가능
캐시 성능 | 낮음 (포인터 추적) | 높음 (연속 메모리)
구현 난이도 | 보통 | 높음

탐색 시간 비교

자료구조별 탐색 시간 비교표
자료구조 | 평균 탐색 | 최악 탐색 | 삽입 평균
-----------------+--------------+--------------+----------
배열 (순차) | O(N) | O(N) | O(1)
정렬 배열 (이진) | O(log N) | O(log N) | O(N)
연결 리스트 | O(N) | O(N) | O(1)
해시 테이블 | O(1) | O(N) | O(1)

해시 테이블의 최악 O(N)은 모든 키가 같은 인덱스에 몰릴 때 발생합니다.
좋은 해시 함수와 적절한 테이블 크기로 이를 방지합니다.

적재율(Load Factor)

적재율(Load Factor)
적재율 = 저장된 원소 수 / 테이블 크기

alpha = n / TABLE_SIZE

적재율이 높을수록 충돌 확률이 증가합니다.
일반적으로 적재율 0.7~0.8 이하를 유지하는 것이 좋습니다.
적재율이 임계값을 넘으면 테이블 크기를 2배로 늘리고
모든 원소를 재해싱(rehashing)합니다.

TABLE_SIZE는 소수(prime number)로 선택하면
해시 값의 분포가 고르게 됩니다. (예: 13, 31, 97, 997)

에러 체험

에러 1: 해시 함수가 음수를 반환
int hash(int key) {
return key % TABLE_SIZE; // key가 음수이면 음수 인덱스!
}

hash(-15) = -15 % 13 = -2 (배열 범위 초과!)

해결: return ((key % TABLE_SIZE) + TABLE_SIZE) % TABLE_SIZE;

C언어에서 음수 % 양수는 음수가 될 수 있으므로
TABLE_SIZE를 더한 뒤 다시 나머지를 구합니다.
에러 2: 체이닝에서 메모리 해제 누락
void destroyTable(HashTable *ht) {
for (int i = 0; i < TABLE_SIZE; i++) {
// ht->table[i] = NULL; // 노드를 free하지 않고 NULL만 대입!
}
}

해결: 각 연결 리스트의 모든 노드를 순회하며 free해야 합니다.

void destroyTable(HashTable *ht) {
for (int i = 0; i < TABLE_SIZE; i++) {
Node *cur = ht->table[i];
while (cur != NULL) {
Node *temp = cur;
cur = cur->next;
free(temp);
}
ht->table[i] = NULL;
}
}
에러 3: 테이블 크기를 2의 거듭제곱으로 선택
#define TABLE_SIZE 16 // 2^4

key % 16은 하위 4비트만 사용하므로 충돌이 집중됩니다.

해결: TABLE_SIZE를 소수로 선택합니다.
#define TABLE_SIZE 13 // 소수
#define TABLE_SIZE 97 // 소수

소수 크기는 해시 값의 분포를 고르게 만들어
충돌 확률을 낮춥니다.
