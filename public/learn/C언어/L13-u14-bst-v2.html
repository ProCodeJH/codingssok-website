이진 탐색 트리 (BST)

왼쪽<루트<오른쪽 규칙으로 O(log n) 탐색이 가능한 BST를 구현합니다.

이 유닛에서 배우는 것

  • BST 속성(왼쪽 < 루트 < 오른쪽)을 이해하고 설명할 수 있습니다.
  • 재귀를 이용하여 BST에 노드를 삽입하는 함수를 구현할 수 있습니다.
  • BST에서 값을 탐색하는 재귀/반복 함수를 구현할 수 있습니다.
  • 리프 노드, 자식 1개, 자식 2개인 세 가지 경우의 삭제를 구현할 수 있습니다.
  • 중위 순회가 오름차순 정렬 결과를 생성함을 이해하고 활용할 수 있습니다.
  • BST에서 최솟값과 최댓값을 찾는 함수를 구현할 수 있습니다.

핵심 개념

BST 속성 (Binary Search Tree Property)
모든 노드 N에 대해:
- N의 왼쪽 서브트리의 모든 값 < N의 값
- N의 오른쪽 서브트리의 모든 값 > N의 값
- 왼쪽, 오른쪽 서브트리도 각각 BST

예시:
8
/ \
3 10
/ \ \
1 6 14

중위 순회 결과: 1, 3, 6, 8, 10, 14 (오름차순 정렬)

BST vs 배열 이진탐색 비교

시간복잡도 정리

BST 시간복잡도
평균 (균형 BST): O(log n)
- 삽입, 탐색, 삭제 모두 트리의 높이에 비례
- 균형 BST의 높이 = O(log n)

최악 (편향 BST): O(n)
- 1, 2, 3, 4, 5 순서로 삽입하면 한쪽으로만 자라는 편향 트리 생성
- 사실상 연결 리스트와 동일한 성능

에러 체험

에러 1: 삭제 시 후계자 처리 누락
// 자식이 2개인 노드 삭제 시 값만 복사하고 후계자를 삭제하지 않음
root->data = successor->data;
// root->right = deleteNode(root->right, successor->data); <-- 빠짐!

-> 후계자 노드가 트리에 중복으로 남아 BST 속성이 깨집니다.
-> 반드시 후계자의 원래 위치에서 해당 노드를 삭제해야 합니다.
에러 2: 편향 트리로 성능 저하
// 정렬된 순서로 삽입하면 편향 트리 생성
for (int i = 1; i <= 10000; i++)
root = insert(root, i); // O(n) 삽입이 n번 -> O(n^2)

-> 1, 2, 3, ... 순서로 삽입하면 오른쪽으로만 자라는 편향 트리
-> 탐색, 삽입, 삭제 모두 O(n)으로 퇴화
-> 데이터를 무작위 순서로 삽입하거나 균형 BST를 사용해야 합니다.
에러 3: 삽입 함수의 반환값을 무시
insert(root, 5); // 반환값 무시!
root = insert(root, 5); // 올바른 사용

-> insert는 새로운 루트를 반환합니다.
-> root가 NULL일 때 반환값을 받지 않으면 트리가 연결되지 않습니다.
-> 항상 root = insert(root, data) 패턴을 사용하십시오.

핵심 정리
1. BST 속성: 모든 노드에서 왼쪽 < 루트 < 오른쪽
2. 삽입: 재귀적으로 위치를 찾아 NULL 자리에 새 노드 생성

4. 삭제: 리프(직접 삭제), 자식1개(자식으로 대체), 자식2개(후계자로 교체)
5. 중위 순회: BST를 중위 순회하면 오름차순 정렬 결과
6. 최솟값: 가장 왼쪽 노드 / 최댓값: 가장 오른쪽 노드
7. 시간복잡도: 균형 O(log n), 편향 O(n)

용어설명
이진 탐색 트리(BST)모든 노드가 '왼쪽 < 루트 < 오른쪽' 속성을 만족하는 이진 트리입니다.
중위 후계자(Inorder Successor)중위 순회에서 현재 노드 다음에 방문되는 노드입니다. 오른쪽 서브트리의 최솟값과 같습니다.

균형 트리(Balanced Tree)왼쪽과 오른쪽 서브트리의 높이 차이가 일정 범위 이내인 트리입니다. AVL, 레드-블랙 트리가 대표적입니다.
중위 순회(Inorder Traversal)왼쪽 -> 루트 -> 오른쪽 순서로 방문하는 트리 순회 방법입니다. BST에서 오름차순 정렬을 생성합니다.
