힙(Heap)과 우선순위 큐

배열로 최대/최소 힙을 구현하고, 우선순위 큐의 원리를 배웁니다.

이 유닛에서 배우는 것

  • 힙 속성(부모>=자식 또는 부모<=자식)을 이해할 수 있습니다.
  • 배열 기반 힙을 구현하고 인덱스 관계(부모=i/2, 왼쪽=2i, 오른쪽=2i+1)를 활용할 수 있습니다.
  • insert(heapUp)와 delete(heapDown) 연산을 구현할 수 있습니다.
  • 우선순위 큐의 원리와 활용 사례를 설명할 수 있습니다.

핵심 개념

힙(Heap) 구조
힙은 완전 이진 트리(Complete Binary Tree)이며, 다음 조건을 만족합니다.

최대 힙(Max Heap): 부모 >= 자식 (루트가 최댓값)
최소 힙(Min Heap): 부모 <= 자식 (루트가 최솟값)

배열 인덱스 관계 (1-based):
부모 = i / 2
왼쪽 자식 = 2 * i
오른쪽 자식 = 2 * i + 1

[50] 인덱스 1
/ \
[30] [40] 인덱스 2, 3
/ \
[10] [20] 인덱스 4, 5

배열: [-, 50, 30, 40, 10, 20] (인덱스 0은 사용하지 않음)

삭제 -- heapDown (siftDown)

undefined

C
1void heapDown(MaxHeap *h, int idx) {
2 int largest = idx;
3 int left = 2 * idx;
4 int right = 2 * idx + 1;
5
6 if (left <= h->size && h->data[left] > h->data[largest])
7 largest = left;
8 if (right <= h->size && h->data[right] > h->data[largest])
9 largest = right;
10
11 if (largest != idx) {
12 swap(&h->data[idx], &h->data[largest]);
13 heapDown(h, largest);
14 }
15}
16
17int deleteMax(MaxHeap *h) {
18 if (isEmpty(h)) {
19 printf("힙이 비어 있습니다!\n");
20 return -1;
21 }
22 int maxVal = h->data[1];
23 h->data[1] = h->data[h->size];
24 h->size--;
25 heapDown(h, 1);
26 return maxVal;
27}

최대 힙 vs 최소 힙

우선순위 큐 활용 사례

우선순위 큐(Priority Queue)가 사용되는 곳
1. 작업 스케줄링 -- 운영체제에서 우선순위가 높은 프로세스를 먼저 실행합니다.
2. 다익스트라 최단 경로 -- 현재 거리가 가장 짧은 정점을 먼저 처리합니다 (최소 힙 사용).
3. 허프만 코딩 -- 빈도가 가장 낮은 두 노드를 먼저 합칩니다 (최소 힙 사용).
4. 이벤트 시뮬레이션 -- 가장 빠른 시각의 이벤트를 먼저 처리합니다.
5. K번째 큰/작은 값 찾기 -- 크기 K의 힙을 유지하여 효율적으로 구합니다.

에러 체험

에러 1: 인덱스 0 시작과 1 시작 혼동

힙은 1-based 인덱스를 사용합니다. 인덱스 0에 값을 저장하면
부모/자식 인덱스 공식(i/2, 2i, 2i+1)이 깨집니다.

-> h->size++;
h->data[h->size] = value; // 먼저 증가 후 저장
에러 2: heapUp 조건을 반대로 작성

최대 힙은 자식이 부모보다 '클 때' 교환해야 합니다.

-> while (idx > 1 && h->data[idx] > h->data[idx / 2])
에러 3: 빈 힙에서 deleteMax 호출

init(&h);

-> 반드시 isEmpty() 검사 후 삭제해야 합니다.
if (!isEmpty(&h)) val = deleteMax(&h);

대회 기출

요약

핵심 정리
1. 힙은 완전 이진 트리이며, 배열로 구현합니다 (1-based: 부모=i/2, 자식=2i, 2i+1).
2. 최대 힙: 부모 >= 자식 / 최소 힙: 부모 <= 자식.
3. 삽입(insert): 배열 끝에 추가 후 heapUp으로 위로 교환 -- O(log n).
4. 삭제(deleteMax): 루트를 꺼내고 마지막 원소를 루트로 옮긴 후 heapDown -- O(log n).
5. 우선순위 큐: 힙 기반으로 우선순위가 높은 원소를 O(log n)에 삽입/삭제합니다.

용어설명
힙(Heap)완전 이진 트리 형태의 자료구조로, 부모-자식 간 대소 관계(힙 속성)를 유지합니다.
최대 힙(Max Heap)부모 노드의 값이 자식 노드의 값보다 항상 크거나 같은 힙입니다. 루트가 최댓값입니다.
최소 힙(Min Heap)부모 노드의 값이 자식 노드의 값보다 항상 작거나 같은 힙입니다. 루트가 최솟값입니다.
heapUp (siftUp)삽입된 원소를 부모와 비교하며 위로 올리는 연산입니다.
heapDown (siftDown)루트에 놓인 원소를 자식과 비교하며 아래로 내리는 연산입니다.
우선순위 큐(Priority Queue)우선순위가 가장 높은 원소를 먼저 꺼내는 추상 자료형이며, 힙으로 효율적으로 구현합니다.
완전 이진 트리(Complete Binary Tree)마지막 레벨을 제외한 모든 레벨이 꽉 차 있고, 마지막 레벨은 왼쪽부터 채워진 이진 트리입니다.
힙 정렬(Heap Sort)힙의 삭제 연산을 반복하여 정렬하는 O(n log n) 알고리즘입니다.
