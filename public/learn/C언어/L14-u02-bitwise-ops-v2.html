비트 연산①

비트 단위 논리 연산(&, |, ^, ~)의 동작 원리와 활용법을 배웁니다.

이 유닛에서 배우는 것

  • AND(&), OR(|), XOR(^), NOT(~) 네 가지 비트 연산자의 동작 원리를 이해할 수 있습니다.
  • 비트 단위 진리표를 작성하고 연산 결과를 2진수로 추적할 수 있습니다.
  • n & 1을 이용한 홀짝 판별 패턴을 활용할 수 있습니다.
  • 비트 플래그를 사용하여 설정/해제/토글 패턴을 구현할 수 있습니다.

비트 연산 vs 논리 연산 비교

비트 연산자와 논리 연산자의 차이
비트 연산자 논리 연산자 차이
───────── ───────── ─────────────────────────────
& (AND) && (AND) &는 각 비트에, &&는 참/거짓에 적용
| (OR) || (OR) |는 각 비트에, ||는 참/거짓에 적용
^ (XOR) (없음) 논리 연산에는 XOR이 없습니다
~ (NOT) ! (NOT) ~는 모든 비트 반전, !는 0↔1만

예시:
5 & 3 = 1 (0101 & 0011 = 0001)
5 && 3 = 1 (참 && 참 = 참)
5 | 3 = 7 (0101 | 0011 = 0111)
5 || 3 = 1 (참 || 참 = 참)
~5 = -6 (모든 비트 반전)
!5 = 0 (참의 반대 = 거짓)

에러 체험

에러 1: & vs && 혼동
if (a & b) // 비트 AND → a와 b의 비트를 AND한 값이 0이 아니면 참
if (a && b) // 논리 AND → a와 b 모두 0이 아니면 참

→ 조건 판단에는 &&, 비트 조작에는 &를 사용하십시오.
→ if (flag & MASK)처럼 비트 플래그 검사는 &가 맞습니다.
에러 2: ~연산자와 부호 비트
unsigned char a = 5; // 00000101
int result = ~a; // int로 확장 → 11111111...11111010 = -6

→ ~연산은 int로 승격된 후 수행됩니다.
→ unsigned에서 기대한 결과(250)를 얻으려면:
unsigned char result = (unsigned char)~a;
에러 3: 연산자 우선순위 — 비교 연산이 비트 연산보다 먼저
if (n & 1 == 0) // 1 == 0이 먼저 → n & 0 → 항상 0!
if ((n & 1) == 0) // 올바른 방법: 괄호로 우선순위 지정

→ 비트 연산자는 비교 연산자(==, !=)보다 우선순위가 낮습니다.
→ 비트 연산에는 반드시 괄호를 사용하십시오.

핵심 정리
1. AND(&): 두 비트 모두 1이면 1 — 마스킹, 비트 추출에 사용
2. OR(|): 하나라도 1이면 1 — 비트 설정(켜기)에 사용
3. XOR(^): 서로 다르면 1 — 토글, 값 교환에 사용
4. NOT(~): 모든 비트 반전 — 비트 해제(&~)에 활용
5. 홀짝 판별: n & 1이 1이면 홀수, 0이면 짝수
6. 비트 플래그: OR로 설정, AND+NOT으로 해제, XOR로 토글
