오버플로와 언더플로

정수와 실수의 표현 범위 한계, 오버플로/언더플로 현상과 대처법을 배웁니다.

이 유닛에서 배우는 것

  • 정수 오버플로 현상을 이해하고 발생 조건을 설명할 수 있습니다.
  • INT_MAX, INT_MIN, UINT_MAX 등 limits.h 상수를 사용할 수 있습니다.
  • unsigned 정수의 랩어라운드(wrap-around) 동작을 이해합니다.
  • KOI 대회 문제에서 오버플로를 방지하는 전략을 적용할 수 있습니다.

핵심 개념 5 — 오버플로 방지 패턴

오버플로 방지 3대 패턴
패턴 1: long long 사용
int a = 100000, b = 100000;
long long result = (long long)a * b; // int끼리 곱하면 오버플로

패턴 2: 덧셈 전 검사
if (a > INT_MAX - b) { /* 오버플로 발생 */ }

패턴 3: 곱셈 시 주의
// int × int는 int로 계산됨 → 오버플로 위험
// 한쪽을 long long으로 캐스팅하면 안전
C
1#include <stdio.h>
2int main(void) {
3 int a = 100000;
4 int b = 100000;
5
6 // 위험: int * int = int (오버플로!)
7 int bad = a * b;
8 printf("int 결과 (오류): %d\n", bad);
9
10 // 안전: (long long) * int = long long
11 long long good = (long long)a * b;
12 printf("long long 결과 (정상): %lld\n", good);
13
14 return 0;
15}

핵심 개념 6 — KOI 오버플로 방지 전략

KOI/백준 대회에서 오버플로 방지 전략
1. 문제의 입력 범위를 반드시 확인하십시오.
- N이 100,000 이하이고 N*N을 계산한다면 long long이 필요합니다.
- 합계를 구할 때도 int 범위를 넘을 수 있습니다.

2. 중간 계산 결과에 주의하십시오.
- 최종 결과가 int 범위여도 중간 계산이 넘칠 수 있습니다.
- 예: 평균 = 합계 / 개수에서 합계가 int를 넘을 수 있습니다.

3. 의심스러우면 long long을 사용하십시오.
- long long은 약 920경까지 표현 가능합니다.
- 서식 지정자는 %lld입니다.

4. unsigned는 대회에서 거의 사용하지 않습니다.
- 음수 처리 실수를 방지하기 위해 signed를 선호합니다.

에러 체험

에러 1: int 곱셈 오버플로
int a = 50000, b = 50000;
int result = a * b;
printf("%d\n", result); // -1794967296 (엉뚱한 값)

원인: 50000 * 50000 = 2,500,000,000 → int 최댓값(2,147,483,647) 초과
해결: long long result = (long long)a * b;
에러 2: 음수를 unsigned에 대입
unsigned int u = -1;
printf("%u\n", u); // 4294967295 (UINT_MAX)

원인: 음수 -1이 unsigned로 변환되면 가장 큰 값이 됩니다.
해결: unsigned 변수에 음수를 대입하지 마십시오.
에러 3: sizeof와 범위 혼동
// sizeof(int) = 4 → 4바이트 = 32비트
// 하지만 int의 최댓값은 2^32가 아니라 2^31 - 1입니다!
// 부호 비트 1개를 빼야 합니다.

혼동: "4바이트니까 2^32 = 42억까지 저장되겠지"
정답: 부호 있는 int는 2^31 - 1 = 약 21억까지입니다.

설명하기

설명하기

signed int의 오버플로가 미정의 동작(undefined behavior)이고, unsigned int의 랩어라운드는 정의된 동작(defined behavior)인 이유를 설명하십시오. 힌트: C 표준이 unsigned에 대해서만 모듈러 연산을 보장합니다.
