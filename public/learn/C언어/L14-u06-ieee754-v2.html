Chapter 06

실수의 비밀

부동소수점의 표현 방식(부호, 지수, 가수)과 정밀도 한계를 배웁니다.

이 유닛에서 배우는 것

  • IEEE 754 구조(부호 1비트, 지수 8비트, 가수 23비트)를 이해할 수 있습니다.
  • 실수 비교에서 == 연산자의 위험성을 설명할 수 있습니다.
  • 오차 누적 문제를 인식하고 원인을 분석할 수 있습니다.
  • epsilon 비교 패턴(fabs(a - b) < 1e-9)을 적용하여 안전하게 실수를 비교할 수 있습니다.

핵심 개념

IEEE 754 구조 (32비트 float)
[부호 1비트][지수 8비트][가수 23비트]

부호(Sign): 0이면 양수, 1이면 음수
지수(Exponent): 2의 거듭제곱을 나타냄 (바이어스 127)
가수(Mantissa): 소수점 이하 유효 숫자

float: 32비트 = 1 + 8 + 23 (유효숫자 약 7자리)
double: 64비트 = 1 + 11 + 52 (유효숫자 약 15자리)

float vs double 비교

float와 double 비교표
항목 | float | double
------------|---------------|----------------
크기 | 4바이트(32비트) | 8바이트(64비트)
부호 비트 | 1비트 | 1비트
지수 비트 | 8비트 | 11비트
가수 비트 | 23비트 | 52비트
유효숫자 | 약 7자리 | 약 15자리
범위 | ~3.4 x 10^38 | ~1.8 x 10^308
서식지정자 | %f | %lf (scanf) / %f (printf)

IEEE 754 특수 값 정리
Infinity : 지수 비트 전부 1, 가수 비트 전부 0 → 양/음 무한대
NaN : 지수 비트 전부 1, 가수 비트 0이 아닌 값 → Not a Number
-0.0 : 부호 비트만 1, 나머지 0 → 음의 영(0.0과 비교 시 같음)

NaN은 자기 자신과도 같지 않은 유일한 값입니다. (NaN != NaN)

실수 비교 안전 함수

C
1#include <stdio.h>
2#include <math.h>
3
4#define EPSILON 1e-9
5
6int double_equal(double a, double b) {
7 return fabs(a - b) < EPSILON;
8}
9
10int double_less(double a, double b) {
11 return (b - a) > EPSILON;
12}
13
14int double_greater(double a, double b) {
15 return (a - b) > EPSILON;
16}
17
18int double_leq(double a, double b) {
19 return !double_greater(a, b);
20}
21
22int double_geq(double a, double b) {
23 return !double_less(a, b);
24}
25
26int main(void) {
27 double x = 0.1 + 0.2;
28 double y = 0.3;
29
30 printf("x == y (unsafe): %s\n", (x == y) ? "true" : "false");
31 printf("x == y (safe) : %s\n", double_equal(x, y) ? "true" : "false");
32 printf("x < y (safe) : %s\n", double_less(x, y) ? "true" : "false");
33 printf("x <= y (safe) : %s\n", double_leq(x, y) ? "true" : "false");
34
35 return 0;
36}
실행 결과
x == y (unsafe): false x == y (safe) : true x < y (safe) : false x <= y (safe) : true

에러 체험

에러 1: == 로 실수 비교
if (0.1 + 0.2 == 0.3) // 거의 항상 false!
printf("같다\n");

→ 실수는 == 로 비교하면 안 됩니다.
→ fabs(a - b) < EPSILON 패턴을 사용하십시오.
에러 2: float 정밀도 부족
float total = 0.0f;
for (int i = 0; i < 1000000; i++)
total += 0.01f; // 심각한 오차 누적!

→ 정밀도가 필요한 계산에는 반드시 double을 사용하십시오.
→ float는 유효숫자 약 7자리, double은 약 15자리입니다.
에러 3: 큰 수 + 작은 수 정밀도 손실
double big = 1e16;
double small = 1.0;
printf("%.1f\n", big + small); // 10000000000000000.0 (1이 사라짐)

→ 크기 차이가 큰 실수의 덧셈에서 작은 수가 흡수됩니다.
→ 가능하면 비슷한 크기의 수끼리 먼저 더하십시오.

용어설명
IEEE 754부동소수점 수의 표현 방식을 정의한 국제 표준. 부호, 지수, 가수 세 부분으로 실수를 저장합니다.
부동소수점(Floating Point)소수점의 위치가 고정되지 않고 떠다니는 방식의 실수 표현. 넓은 범위의 수를 표현할 수 있지만 정밀도에 한계가 있습니다.
가수(Mantissa)부동소수점에서 유효숫자를 저장하는 부분. float는 23비트, double은 52비트입니다.
지수(Exponent)부동소수점에서 2의 거듭제곱 값을 저장하는 부분. 수의 크기(규모)를 결정합니다.
epsilon두 실수를 비교할 때 허용하는 최소 오차 범위. 일반적으로 1e-9 정도의 매우 작은 값을 사용합니다.
NaNNot a Number. 0/0 같은 정의되지 않는 연산의 결과. 자기 자신과도 같지 않은 유일한 값입니다.
InfinityIEEE 754에서 무한대를 표현하는 특수 값. 1.0/0.0의 결과입니다.
정밀도 손실유효숫자 비트 수의 한계로 인해 정확한 값을 저장하지 못하고 근사값이 저장되는 현상입니다.
