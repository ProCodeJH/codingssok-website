함수의 주소를 저장하는 함수 포인터의 선언과 사용법을 배웁니다.

이 유닛에서 배우는 것

  • 함수 포인터의 선언 문법을 이해하고 올바르게 작성할 수 있습니다.
  • 함수 포인터를 사용하여 함수를 호출할 수 있습니다.
  • 함수 포인터 배열을 만들어 여러 함수를 선택적으로 호출할 수 있습니다.

핵심 개념

함수 포인터 선언
반환형 (*포인터이름)(매개변수 목록);

int (*fp)(int, int) = &add;

- 괄호 (*fp)가 반드시 필요합니다.
- &add와 add는 동일합니다 (함수 이름 자체가 주소).
함수 포인터로 호출
fp(3, 4); // 간접 호출 (간략 표기)
(*fp)(3, 4); // 간접 호출 (명시적 역참조)

두 방식 모두 동일한 결과를 냅니다.
비교: 일반 호출 vs 함수 포인터 호출
일반 호출: int result = add(3, 4);
함수 포인터 호출: int (*fp)(int, int) = add;
int result = fp(3, 4);

일반 호출은 컴파일 시점에 함수가 결정됩니다.
함수 포인터 호출은 실행 시점에 함수를 바꿀 수 있습니다.

에러 체험

에러 1: 선언에서 괄호 누락
int *fp(int, int); // 함수 포인터가 아니라 int*를 반환하는 함수 선언!
int (*fp)(int, int); // 올바른 함수 포인터 선언

(*fp) 괄호를 반드시 써야 합니다. 괄호가 없으면 전혀 다른 의미가 됩니다.
에러 2: 함수 포인터 vs 함수 호출 혼동
int (*fp)(int, int) = add(3, 4); // 오류! add(3,4)는 int 값 7
int (*fp)(int, int) = add; // 올바름! add는 함수의 주소

함수 이름 뒤에 ()를 붙이면 호출이 되어 반환값이 됩니다.
주소를 저장하려면 함수 이름만 씁니다.
에러 3: 시그니처 불일치
double avg(double a, double b) { return (a + b) / 2; }
int (*fp)(int, int) = avg; // 경고 또는 오류!

함수 포인터의 반환형과 매개변수 타입이 대상 함수와 정확히 일치해야 합니다.
