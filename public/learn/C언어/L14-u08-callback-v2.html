호출자가 콜백으로 '동작'을 주입합니다.

하드코딩 vs 콜백 패턴 비교

하드코딩 vs 콜백 패턴 비교표
항목 | 하드코딩 방식 | 콜백 패턴
-----------------+------------------------------+----------------------------
동작 변경 | 함수 내부 코드를 수정해야 함 | 콜백 함수만 교체하면 됨
함수 개수 | 동작마다 별도 함수 필요 | 하나의 범용 함수로 처리
재사용성 | 낮음 (특정 동작에 종속) | 높음 (동작을 외부에서 주입)
유연성 | 낮음 | 높음
코드량 | 동작이 늘수록 함수도 증가 | 콜백만 추가하면 됨
실전 예 | sortAsc(), sortDesc() 각각 | sort(arr, n, compare)
대표 사례 | -- | qsort, bsearch, signal

에러 체험

에러 1: 콜백 시그니처 불일치
void forEach(int arr[], int n, void (*action)(int)) { ... }

int addTen(int x) { return x + 10; } // 반환형이 int!

forEach(data, 5, addTen); // 경고: 시그니처 불일치

forEach가 기대하는 콜백은 void (*)(int)이지만
addTen은 int (*)(int)입니다.
컴파일러가 경고를 발생시키며, 예기치 않은 동작이 발생할 수 있습니다.

해결: 콜백의 반환형과 매개변수가 정확히 일치하는지 확인하십시오.
void addTenAndPrint(int x) { printf("%d ", x + 10); }
에러 2: NULL 콜백 호출
void process(int x, void (*callback)(int)) {
callback(x); // callback이 NULL이면 세그멘테이션 오류!
}

process(5, NULL); // 런타임 에러 발생

해결: 콜백을 호출하기 전에 반드시 NULL 검사를 수행하십시오.

void process(int x, void (*callback)(int)) {
if (callback != NULL) {
callback(x);
}
}
에러 3: 콜백에서 외부 변수 접근 제한
int total = 0;

void addToTotal(int x) {
total += x; // 전역 변수에 의존 -- 함수의 독립성이 깨짐
}

C언어의 함수 포인터 콜백은 외부 상태(클로저)를 캡처할 수 없습니다.
콜백이 외부 데이터에 접근해야 할 경우,
전역 변수를 사용하거나 void* 매개변수로 컨텍스트를 전달해야 합니다.

개선:
void forEach2(int arr[], int n, void (*action)(int, void*), void *ctx) {
for (int i = 0; i < n; i++) action(arr[i], ctx);
}
void addToSum(int x, void *ctx) {
*(int *)ctx += x;
}
