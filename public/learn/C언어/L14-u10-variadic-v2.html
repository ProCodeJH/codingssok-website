가변 인자 함수 -- stdarg.h

va_list, va_start, va_arg, va_end로 인자 개수가 변하는 함수를 만듭니다.

이 유닛에서 배우는 것

  • stdarg.h가 제공하는 매크로 4종(va_list, va_start, va_arg, va_end)의 역할을 설명할 수 있습니다.
  • 인자 개수를 전달하는 방식으로 가변 인자 sum, max 함수를 구현할 수 있습니다.
  • 센티넬 값 패턴을 이용하여 종료 조건을 지정하는 가변 인자 함수를 작성할 수 있습니다.
  • 서식 문자열을 파싱하는 간단한 my_printf를 구현할 수 있습니다.
  • 가변 인자 함수의 타입 안전성 한계를 이해하고 주의사항을 설명할 수 있습니다.

핵심 개념

stdarg.h 매크로 4종
┌──────────────────┬──────────────────────────────────────────────────┐
│ 매크로 │ 역할 │
├──────────────────┼──────────────────────────────────────────────────┤
│ va_list │ 가변 인자 목록을 순회하기 위한 타입 │
│ va_start(ap, p) │ ap를 초기화. p는 ...앞의 마지막 고정 매개변수 │
│ va_arg(ap, type) │ 다음 인자를 type 타입으로 꺼내고 포인터를 이동 │
│ va_end(ap) │ ap 정리. 반드시 호출해야 합니다 │
└──────────────────┴──────────────────────────────────────────────────┘

사용 순서: va_list 선언 -> va_start -> va_arg 반복 -> va_end

가변 인자의 한계

가변 인자 함수의 한계와 주의사항
1. 타입 검사 없음
- va_arg(ap, double)로 꺼냈는데 실제로 int가 전달되면 미정의 동작 발생
- 컴파일러가 타입 불일치를 경고하지 않습니다

2. 인자 개수를 모름
- ... 부분에 몇 개의 인자가 왔는지 알 방법이 없습니다
- 개수 전달, 서식 문자열, 센티넬 중 하나를 직접 구현해야 합니다

3. 기본 타입 승격 (Default Argument Promotion)
- char, short -> int로 승격
- float -> double로 승격
- va_arg(ap, char)는 오류! va_arg(ap, int)로 읽어야 합니다
- va_arg(ap, float)는 오류! va_arg(ap, double)로 읽어야 합니다

4. va_end 필수
- va_end를 호출하지 않으면 메모리 누수나 스택 손상이 발생할 수 있습니다

기본 타입 승격 비교표

va_arg에서 올바른 타입 지정
┌────────────────────┬────────────────────┬────────────────────┐
│ 전달한 타입 │ va_arg에 쓸 타입 │ 이유 │
├────────────────────┼────────────────────┼────────────────────┤
│ char │ int │ int로 승격됨 │
│ short │ int │ int로 승격됨 │
│ int │ int │ 그대로 │
│ long │ long │ 그대로 │
│ float │ double │ double로 승격됨 │
│ double │ double │ 그대로 │
│ char * │ char * │ 포인터는 그대로 │
│ int * │ int * │ 포인터는 그대로 │
└────────────────────┴────────────────────┴────────────────────┘

에러 체험

에러 1: va_end 누락
int sum(int count, ...) {
va_list ap;
va_start(ap, count);
int total = 0;
for (int i = 0; i < count; i++)
total += va_arg(ap, int);
// va_end(ap); <-- 누락!
return total;
}

-> va_end를 호출하지 않으면 스택이 제대로 정리되지 않습니다.
-> 일부 플랫폼에서는 메모리 누수나 비정상 동작이 발생합니다.
-> 반드시 va_start와 va_end를 쌍으로 사용하십시오.
에러 2: 잘못된 타입으로 va_arg 호출
double avg(int count, ...) {
va_list ap;
va_start(ap, count);
double total = 0;
for (int i = 0; i < count; i++)
total += va_arg(ap, int); // 실제로 double이 전달됨!
va_end(ap);
return total / count;
}

avg(3, 1.5, 2.5, 3.5); // 미정의 동작!

-> 호출부에서 double을 전달했는데 va_arg(ap, int)로 읽으면
스택에서 잘못된 바이트를 해석하여 엉뚱한 값이 나옵니다.
-> va_arg(ap, double)로 읽어야 합니다.
에러 3: 인자 개수 불일치
printf("%d %d %d\n", 10, 20);
// 서식 지정자 3개, 인자 2개 -> 세 번째 %d는 쓰레기 값 출력

sum(5, 1, 2, 3); // count=5인데 실제 인자 3개
// 네 번째, 다섯 번째 va_arg는 쓰레기 값을 읽음

-> 컴파일러가 경고하지 않으므로 프로그래머가 직접 확인해야 합니다.
-> 개수와 실제 인자 수를 반드시 일치시키십시오.

용어설명
가변 인자 함수 (Variadic Function)인자의 개수가 고정되지 않은 함수. 선언부에 ...을 사용하며, printf, scanf 등이 대표적입니다.
va_list가변 인자 목록을 순회하기 위한 타입. stdarg.h에 정의되어 있습니다.
va_start(ap, param)va_list를 초기화하는 매크로. param은 ... 앞의 마지막 고정 매개변수입니다.
va_arg(ap, type)va_list에서 다음 인자를 type 타입으로 꺼내는 매크로. 호출할 때마다 다음 인자로 이동합니다.
va_end(ap)va_list를 정리하는 매크로. va_start와 반드시 쌍으로 호출해야 합니다.
센티넬 (Sentinel)가변 인자의 끝을 표시하는 특별한 값. -1, 0, NULL 등을 사용합니다.
기본 타입 승격 (Default Argument Promotion)가변 인자로 전달될 때 char/short는 int로, float는 double로 자동 변환되는 규칙입니다.
vprintfva_list를 인자로 받는 printf 변형 함수. 가변 인자 함수 내부에서 서식 출력을 위임할 때 사용합니다.
