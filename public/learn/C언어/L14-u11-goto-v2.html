goto문

goto문의 동작 원리와 유일하게 허용되는 사용 패턴(에러 처리 cleanup)을 배웁니다.

이 유닛에서 배우는 것

  • goto와 레이블(label) 문법을 이해하고 작성할 수 있습니다.
  • goto가 위험한 이유(스파게티 코드)를 설명할 수 있습니다.
  • 유일한 합법적 사용 패턴인 에러 처리 cleanup 패턴을 작성할 수 있습니다.
  • 구조적 프로그래밍 원칙을 이해하고 goto 없이 코드를 작성할 수 있습니다.

왜 goto를 쓰지 말아야 하는가

다익스트라의 주장 (1968)
"go to 문이 포함된 프로그램의 품질은 go to 문의 개수에 반비례한다."

[문제점 1] 코드가 위아래로 자유롭게 점프하면 실행 순서를 추적할 수 없습니다.
[문제점 2] 반복, 조건 등의 논리 구조가 파괴됩니다.
[문제점 3] 디버깅이 극도로 어려워집니다.
[문제점 4] 다른 개발자가 코드를 이해할 수 없습니다.

해결책: if, for, while, 함수 등 구조적 제어문으로 모든 흐름을 표현합니다.

스파게티 코드 예시 (나쁜 예)

C
1/* 절대 이렇게 작성하지 마십시오! */
2#include <stdio.h>
3int main(void) {
4 int i = 1;
5start:
6 if (i > 10) goto done;
7 if (i % 2 == 0) goto even;

10even:

12next:

Linux 커널에서의 goto cleanup 사용 사례

실무 사례: Linux 커널 코드 스타일
Linux 커널 코딩 스타일 문서(Documentation/process/coding-style.rst)에서 goto의 사용을 명시적으로 허용합니다.

"goto 문이 유용한 경우는 함수가 여러 종료 지점에서 공통 cleanup 작업을 수행할 때이다."

실제로 Linux 커널 소스 코드에는 수십만 개의 goto cleanup 패턴이 존재합니다.
이는 C 언어에 try-catch-finally 같은 예외 처리 구문이 없기 때문입니다.
goto cleanup은 C에서 finally 블록을 모방하는 유일한 관용적 패턴입니다.

예시 레이블 이름: err_free, err_close, out_unlock, fail_cleanup 등

에러 체험

에러 1: goto로 변수 선언 건너뛰기
#include
int main(void) {
goto skip;
int x = 10; // 이 선언이 건너뛰어짐
skip:
printf("%d\n", x); // x가 초기화되지 않음!
return 0;
}

-> C에서는 컴파일이 될 수 있지만, x가 초기화되지 않은 상태로 사용됩니다.
-> C++에서는 변수 선언을 건너뛰는 goto가 컴파일 에러입니다.
-> 해결: goto 점프 경로에 변수 선언을 두지 마십시오.
에러 2: goto로 무한 루프 생성
#include
int main(void) {
int count = 0;
start:
printf("%d\n", count);
goto start; // 조건 없이 항상 되돌아감 -> 무한 루프!
return 0;
}

-> goto로 위쪽 레이블로 점프하면 무한 루프가 됩니다.
-> 종료 조건이 없으므로 프로그램이 멈추지 않습니다.
-> 해결: 반복이 필요하면 for나 while을 사용하십시오.
에러 3: 가독성 파괴 - 여러 goto의 교차 점프
goto A;
B: printf("B");
goto C;
A: printf("A");
goto B;
C: printf("C");

-> 실행 순서: A -> B -> C. 하지만 코드 순서와 실행 순서가 완전히 다릅니다.
-> 3개만으로도 추적이 어렵습니다. 10개라면 불가능합니다.
-> 해결: goto 대신 함수 호출로 실행 순서를 표현하십시오.
