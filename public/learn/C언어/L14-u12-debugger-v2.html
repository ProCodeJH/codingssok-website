디버거 사용법 -- GDB/VS 디버거

GDB와 Visual Studio 디버거로 프로그램을 단계 실행하고 변수를 검사하는 방법을 배웁니다.

이 유닛에서 배우는 것

  • gcc -g 옵션으로 디버그 정보를 포함하여 컴파일할 수 있습니다.
  • GDB 기본 명령어(break, run, next, step, print, watch)를 사용할 수 있습니다.
  • 중단점을 설정하고 단계 실행으로 프로그램 흐름을 추적할 수 있습니다.
  • VS Code/Visual Studio 디버거를 설정하고 사용할 수 있습니다.

핵심 개념

디버거란?
디버거(debugger)는 프로그램의 실행을 제어하고 내부 상태를 관찰하는 도구입니다.

┌──────────────┬─────────────────────┬─────────────────────┐
│ 항목 │ printf 디버깅 │ 디버거 │
├──────────────┼─────────────────────┼─────────────────────┤
│ 준비 작업 │ printf 문 삽입/삭제 │ 중단점 설정 │
│ 실행 제어 │ 불가능 │ 한 줄씩 실행 가능 │
│ 변수 검사 │ 미리 출력문 필요 │ 실시간 검사 가능 │
│ 코드 수정 │ 필요 (나중에 삭제) │ 불필요 │
│ 호출 스택 │ 확인 어려움 │ backtrace로 즉시 확인│
│ 조건부 중단 │ if + printf 필요 │ 조건부 중단점 설정 │
│ 학습 비용 │ 낮음 │ 초기 학습 필요 │
└──────────────┴─────────────────────┴─────────────────────┘

printf 디버깅은 간단한 버그에 효과적이지만, 복잡한 버그에는 디버거가 훨씬 효율적입니다.

중요: -O2와 -g를 함께 쓰지 마십시오

최적화 옵션(-O2)을 켜면 컴파일러가 코드를 재배치하거나 변수를 제거합니다.
디버거에서 변수가 'optimized out'으로 표시되거나, 줄 번호가 뒤섞여 보입니다.
디버깅할 때는 최적화 옵션을 끄십시오: gcc -g main.c -o main

디버깅 전략

효과적인 디버깅 전략
1. 이분 탐색 디버깅
- 코드 중간에 중단점을 설정하여 버그가 앞쪽인지 뒤쪽인지 판별합니다.
- 범위를 절반씩 좁혀가며 버그 위치를 찾습니다.
- 100줄 코드에서 최대 7번(log2(100))의 검사로 버그를 찾을 수 있습니다.

2. 가설 검증 디버깅
- "이 변수가 이 시점에 이 값일 것이다"라는 가설을 세웁니다.
- 중단점을 설정하고 print로 실제 값을 확인합니다.
- 가설과 다르면 그 지점이 버그 발생 위치입니다.

3. 역추적 디버깅
- 잘못된 출력이 나오는 지점에서 시작합니다.
- backtrace로 호출 경로를 확인합니다.
- 어떤 함수에서 잘못된 값이 전달되었는지 역으로 추적합니다.

에러 체험

에러 1: -g 옵션 없이 디버깅
$ gcc main.c -o main
$ gdb ./main
(gdb) break main
Breakpoint 1 at 0x401136
(gdb) run
(gdb) print x
No symbol "x" in current context.

-g 없이 컴파일하면 디버그 정보가 없어 변수 이름을 찾을 수 없습니다.
줄 번호도 표시되지 않아 어디를 실행 중인지 알 수 없습니다.
반드시 gcc -g main.c -o main으로 컴파일하십시오.
에러 2: 최적화(-O2)와 디버거 충돌
$ gcc -O2 -g main.c -o main
$ gdb ./main
(gdb) break 10
(gdb) run
(gdb) print counter
$1 =

-O2 최적화가 변수를 레지스터에만 저장하거나 제거하여 디버거가 값을 읽을 수 없습니다.
next로 한 줄씩 실행할 때 줄 순서가 뒤바뀌어 혼란스럽습니다.
디버깅 시에는 최적화를 끄십시오: gcc -g -O0 main.c -o main
에러 3: 중단점 위치 실수
(gdb) break 1
Breakpoint 1 at 0x...: file main.c, line 1.
(gdb) run

1번 줄(#include )에 중단점을 설정하면 의미가 없습니다.
실행 가능한 코드가 있는 줄에 중단점을 설정해야 합니다.

올바른 예:
(gdb) break main // main 함수 시작
(gdb) break 12 // 12번 줄 (실행문이 있는 줄)
(gdb) break sum_array // 특정 함수의 시작

용어설명
