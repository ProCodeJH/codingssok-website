핵심 개념

브루트포스(Brute Force)란?
가능한 모든 경우의 수를 빠짐없이 검사하여 정답을 찾는 방법입니다.

적용 조건:
1. 경우의 수가 충분히 적을 것 (보통 1억 이하)
2. 각 경우를 검사하는 비용이 낮을 것
3. 정답이 반드시 존재할 것 (또는 존재 여부를 판별)

브루트포스의 시간복잡도 패턴
| 탐색 유형 | 경우의 수 | 예시 |
|---------------|-------------|------------------|
| 모든 쌍 | O(n^2) | 두 수의 합 찾기 |

기준: n <= 20이면 O(2^n) 가능, n <= 10이면 O(n!) 가능, n <= 10,000이면 O(n^2) 가능

에러 체험

에러 1: 같은 원소를 두 번 선택
for (int i = 0; i < n; i++)
for (int j = 0; j < n; j++) // j = 0부터 시작!
if (arr[i] + arr[j] == target) ...

문제: (i, i) 쌍도 검사하여 자기 자신과 더하는 경우가 포함됩니다.
해결: for (int j = i + 1; j < n; j++)로 변경합니다.
에러 2: 부분집합에서 빈 집합 포함
for (int mask = 0; mask < (1 << n); mask++)
// mask = 0은 공집합!

문제: 문제에 따라 공집합을 제외해야 할 수 있습니다.
해결: for (int mask = 1; ...)로 시작하여 공집합을 제외합니다.
에러 3: 순열에서 used 배열 복원 누락
used[i] = 1;
perm[depth] = arr[i];
generate(depth + 1);
// used[i] = 0; 누락!

문제: 백트래킹 시 상태를 복원하지 않으면 일부 순열이 생성되지 않습니다.
해결: 재귀 호출 후 반드시 used[i] = 0;을 추가합니다.
에러 4: 비트마스크 연산 우선순위
if (mask & (1 << i) != 0) // 잘못된 우선순위!

문제: != 가 & 보다 우선순위가 높아 (1 << i) != 0이 먼저 평가됩니다.
해결: if ((mask & (1 << i)) != 0) 또는 if (mask & (1 << i))

요약

핵심 정리
1. 브루트포스: 모든 경우의 수를 빠짐없이 검사하여 정답을 찾는 방법
2. 모든 쌍 탐색: 이중 for문, O(n^2) — j = i + 1부터 시작하여 중복 방지
3. 부분집합 열거: 비트마스크 0~2^n-1, O(2^n) — n <= 20일 때 적용 가능
4. 순열 생성: 재귀 + used 배열, O(n!) — n <= 10일 때 적용 가능
5. 적용 판단: 경우의 수를 계산하여 1초(약 1억 연산) 내 가능한지 확인

다음 유닛으로

다음 유닛: 백트래킹
브루트포스는 모든 경우를 검사하지만, 불필요한 경우를 미리 차단할 수 있다면 어떨까요? 다음 유닛에서는 탐색 도중 조건에 맞지 않는 가지를 잘라내는 백트래킹(Backtracking) 기법을 학습합니다. 브루트포스의 정확성을 유지하면서 탐색 범위를 대폭 줄이는 전략입니다.
용어설명
브루트포스(Brute Force)가능한 모든 경우의 수를 빠짐없이 검사하여 정답을 찾는 알고리즘 전략입니다.
완전 탐색(Exhaustive Search)브루트포스와 같은 의미로, 해 공간 전체를 탐색하는 방법입니다.
부분집합(Subset)집합의 원소 일부(또는 전부, 또는 없음)로 구성된 집합입니다. n개 원소의 부분집합은 2^n개입니다.
순열(Permutation)n개의 원소를 순서를 고려하여 나열하는 방법입니다. 경우의 수는 n!입니다.
비트마스크(Bitmask)정수의 각 비트를 이용하여 집합의 포함 여부를 표현하는 기법입니다.
백트래킹(Backtracking)탐색 도중 조건에 맞지 않으면 되돌아가는 기법으로, 브루트포스를 최적화합니다.
