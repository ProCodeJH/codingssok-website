핵심 개념

백트래킹(Backtracking)이란?
해를 구하기 위해 후보를 하나씩 선택하다가, 현재 선택이 해가 될 수 없다고 판단되면 즉시 이전 단계로 되돌아가(backtrack) 다른 후보를 시도하는 기법입니다.

핵심 3단계:
1. 선택(Choose): 현재 단계에서 후보를 하나 선택합니다.
2. 검증(Check): 유망한지 확인합니다.
3. 되돌림(Unchoose): 유망하지 않으면 선택을 취소하고 다음 후보로 넘어갑니다.
상태 공간 트리(State Space Tree)
문제의 모든 가능한 해를 트리 형태로 나타낸 것입니다.

- 루트: 아무것도 선택하지 않은 초기 상태
- 각 레벨: 하나의 결정을 내리는 단계
- 리프: 모든 결정이 완료된 최종 상태
- 경로: 루트에서 리프까지의 선택 순서가 하나의 후보 해

브루트포스는 이 트리의 모든 리프를 방문합니다.
백트래킹은 유망하지 않은 가지를 잘라내어 리프 방문을 줄입니다.
가지치기(Pruning)
상태 공간 트리에서 유망하지 않은(promising하지 않은) 노드의 하위 트리를 탐색하지 않고 건너뛰는 것입니다.

가지치기 조건의 예:
- N-Queen: 같은 열 또는 같은 대각선에 이미 퀸이 있으면 가지치기
- 부분집합 합: 현재까지의 합이 이미 목표를 초과하면 가지치기
- 순열: 이미 사용한 숫자를 다시 선택하려 하면 가지치기

가지치기가 강력할수록 탐색 속도가 빨라집니다.

N-Queen 충돌 조건
queen[i] = 행 i에 놓인 퀸의 열 번호

두 퀸 (i, queen[i])와 (j, queen[j])가 충돌하는 조건:
1. 같은 열: queen[i] == queen[j]
2. 같은 대각선: |i - j| == |queen[i] - queen[j]|

행은 i마다 하나씩 배치하므로 행 충돌은 발생하지 않습니다.

N-Queen 풀이 코드

C
1#include <stdio.h>
2#include <stdlib.h>
3#define MAX 20
4
5int n;
6int queen[MAX]; /* queen[i] = i행에 놓인 퀸의 열 번호 */
7int count = 0; /* 해의 개수 */
8
9/* row행까지 놓인 퀸이 유망한지 검사 */
10int isPromising(int row) {
11 for (int i = 0; i < row; i++) {
12 /* 같은 열 검사 */
13 if (queen[i] == queen[row])
14 return 0;
15 /* 대각선 검사: 행 차이 == 열 차이 */
16 if (abs(i - row) == abs(queen[i] - queen[row]))
17 return 0;
18 }
19 return 1;
20}
21
22void solve(int row) {
23 if (row == n) {
24 /* 모든 행에 퀸을 성공적으로 배치 */
25 count++;
26 printf("해 %d: ", count);
27 for (int i = 0; i < n; i++)
28 printf("%d ", queen[i] + 1);
29 printf("\n");
30 return;
31 }
32 for (int col = 0; col < n; col++) {
33 queen[row] = col;
34 if (isPromising(row)) {
35 solve(row + 1); /* 다음 행으로 진행 */
36 }
37 /* 유망하지 않으면 자동으로 다음 col 시도 (되돌림) */
38 }
39}
40
41int main(void) {
42 printf("N 입력: ");
43 scanf("%d", &n);
44 solve(0);
45 printf("총 해의 개수: %d\n", count);
46 return 0;
47}

8void subsetSum(int depth, int currentSum) {

12 if (selected[i]) printf("%d ", arr[i]);

에러 체험

에러 1: visited 배열을 되돌리지 않음
visited[i] = 1;
perm[depth] = i;
generatePerm(depth + 1);
/* visited[i] = 0; 빠짐! */

→ 한 번 사용한 숫자가 영구적으로 잠겨 순열 1개만 생성됩니다.
→ 재귀 호출 후 반드시 visited[i] = 0으로 되돌려야 합니다.
에러 2: 대각선 검사에서 절댓값 누락
if (i - row == queen[i] - queen[row]) /* 한쪽 대각선만 검사! */

→ 오른쪽 아래 대각선(\)만 검사하고 왼쪽 아래 대각선(/)을 놓칩니다.
→ abs(i - row) == abs(queen[i] - queen[row])로 양쪽 모두 검사해야 합니다.
에러 3: isPromising에서 자기 자신과 비교
for (int i = 0; i <= row; i++) /* i == row일 때 자기 자신과 비교 */

→ 자기 자신은 항상 같은 열, 같은 위치이므로 항상 충돌로 판정됩니다.
→ for (int i = 0; i < row; i++)로 자기 자신을 제외해야 합니다.

용어설명
백트래킹(Backtracking)해를 구하기 위해 후보를 선택하다가 유망하지 않으면 이전 단계로 되돌아가 다른 후보를 시도하는 알고리즘 기법입니다.

유망 함수(Promising Function)현재 상태가 해로 이어질 가능성이 있는지 판별하는 함수입니다. isPromising()이 대표적입니다.
