핵심 개념

그리디 알고리즘이란?
그리디(Greedy, 탐욕) 알고리즘은 매 단계에서 현재 상태에서의 최선의 선택을 반복하여 최종 해를 구성하는 전략입니다.

핵심 아이디어:
1. 현재 시점에서 가장 좋아 보이는 것을 선택합니다
2. 한 번 선택하면 되돌리지 않습니다
3. 선택을 반복하여 전체 해를 완성합니다

그리디가 최적해를 보장하려면 두 가지 조건이 필요합니다:
- 탐욕 선택 속성(Greedy Choice Property)
- 최적 부분 구조(Optimal Substructure)

문제 1: 동전 거스름돈
금액 N원을 거슬러 줄 때, 사용하는 동전 개수를 최소화하는 문제입니다.

그리디 전략: 가장 큰 단위의 동전부터 최대한 많이 사용합니다.

예) 1260원, 동전 = {500, 100, 50, 10}
500원: 1260 / 500 = 2개, 나머지 260원
100원: 260 / 100 = 2개, 나머지 60원
50원: 60 / 50 = 1개, 나머지 10원
10원: 10 / 10 = 1개, 나머지 0원
합계: 6개 (최적해)
C
1/* 동전 거스름돈 -- 그리디 O(k), k = 동전 종류 수 */
2#include <stdio.h>
3int main(void) {
4 int coins[] = {500, 100, 50, 10};
5 int k = 4;
6 int n;
7 scanf("%d", &n);
8
9 int count = 0;
10 for (int i = 0; i < k; i++) {
11 count += n / coins[i]; /* 현재 동전을 최대한 사용 */
12 n %= coins[i]; /* 나머지 금액 갱신 */
13 }
14 printf("%d\n", count);
15 return 0;
16}
거스름돈 그리디가 실패하는 경우
동전 단위가 {400, 300, 100}이고 금액이 600원인 경우:

그리디: 400원 1개 + 100원 2개 = 3개
최적해: 300원 2개 = 2개

그리디가 실패합니다. 큰 동전이 작은 동전의 배수가 아니면 탐욕 선택 속성이 성립하지 않기 때문입니다.

한국 화폐(10, 50, 100, 500)는 큰 단위가 작은 단위의 배수이므로 그리디가 항상 최적해를 보장합니다.

문제 2: 활동 선택 (Activity Selection)
여러 활동의 시작 시간과 종료 시간이 주어질 때, 서로 겹치지 않으면서 최대한 많은 활동을 선택하는 문제입니다.

그리디 전략: 종료 시간이 가장 빠른 활동부터 선택합니다.

예) 활동: (1,3), (2,5), (4,7), (6,9), (8,10)
(1,3) 선택 -> 종료 3
(2,5) 시작 2 < 3 -> 불가
(4,7) 시작 4 >= 3 -> 선택, 종료 7
(6,9) 시작 6 < 7 -> 불가
(8,10) 시작 8 >= 7 -> 선택, 종료 10
결과: 3개 선택 (최적해)

종료 시간이 빠른 활동을 선택하면 남는 시간이 최대화되어 더 많은 활동을 넣을 수 있습니다. 이것이 탐욕 선택 속성입니다.
C
1/* 활동 선택 문제 -- 그리디 O(n log n) */
2#include <stdio.h>
3#include <stdlib.h>
4
5typedef struct {
6 int start;
7 int end;
8} Activity;
9
10int compare(const void *a, const void *b) {
11 return ((Activity*)a)->end - ((Activity*)b)->end;
12}
13
14int main(void) {
15 int n;
16 scanf("%d", &n);
17 Activity act[100];
18 for (int i = 0; i < n; i++)
19 scanf("%d %d", &act[i].start, &act[i].end);
20
21 /* 종료 시간 기준 오름차순 정렬 */
22 qsort(act, n, sizeof(Activity), compare);
23
24 int count = 1; /* 첫 활동은 무조건 선택 */
25 int lastEnd = act[0].end; /* 마지막 선택 활동의 종료 시간 */
26
27 for (int i = 1; i < n; i++) {
28 if (act[i].start >= lastEnd) { /* 겹치지 않으면 선택 */
29 count++;
30 lastEnd = act[i].end;
31 }
32 }
33 printf("%d\n", count);
34 return 0;
35}

문제 3: 분할 배낭 (Fractional Knapsack)
배낭 용량 W가 주어지고, 각 물건의 무게와 가치가 주어질 때 배낭에 담을 수 있는 최대 가치를 구하는 문제입니다.

핵심: 물건을 쪼개서 넣을 수 있습니다 (분할 가능).

그리디 전략: 단위 무게당 가치(가치/무게)가 높은 물건부터 넣습니다.

예) 배낭 용량 = 50kg
물건 A: 10kg, 60만원 (kg당 6만원)
물건 B: 20kg, 100만원 (kg당 5만원)
물건 C: 30kg, 120만원 (kg당 4만원)

A 전부(10kg) + B 전부(20kg) + C의 20/30 = 232만원

참고: 물건을 쪼갤 수 없는 경우(0-1 배낭)에는 그리디가 최적해를 보장하지 않습니다. 이 경우 DP를 사용해야 합니다.
C
1/* 분할 배낭 문제 -- 그리디 O(n log n) */
2#include <stdio.h>
3#include <stdlib.h>
4
5typedef struct {
6 double weight;
7 double value;
8 double ratio; /* 단위 무게당 가치 */
9} Item;
10
11int compare(const void *a, const void *b) {
12 double diff = ((Item*)b)->ratio - ((Item*)a)->ratio;
13 return (diff > 0) ? 1 : (diff < 0) ? -1 : 0;
14}
15
16int main(void) {
17 int n;
18 double capacity;
19 scanf("%d %lf", &n, &capacity);
20
21 Item items[100];
22 for (int i = 0; i < n; i++) {
23 scanf("%lf %lf", &items[i].weight, &items[i].value);
24 items[i].ratio = items[i].value / items[i].weight;
25 }
26
27 /* 단위 무게당 가치 기준 내림차순 정렬 */
28 qsort(items, n, sizeof(Item), compare);
29
30 double totalValue = 0.0;
31 for (int i = 0; i < n; i++) {
32 if (capacity >= items[i].weight) {
33 /* 물건 전체를 넣을 수 있으면 전부 넣기 */
34 totalValue += items[i].value;
35 capacity -= items[i].weight;
36 } else {
37 /* 일부만 넣기 (분할) */
38 totalValue += items[i].ratio * capacity;
39 break; /* 배낭이 가득 참 */
40 }
41 }
42 printf("%.2f\n", totalValue);
43 return 0;
44}

그리디 알고리즘 설계 절차
1. 문제를 작은 선택 단위로 분해합니다
2. 각 단계에서 '최선의 기준'을 정합니다

- 활동 선택: 가장 빨리 끝나는 활동
- 분할 배낭: kg당 가치가 가장 높은 물건
3. 그리디 선택을 반복하여 해를 구성합니다
4. 최적성을 검증합니다
- 탐욕 선택 속성이 성립하는지 확인합니다
- 반례가 있으면 그리디를 포기하고 DP를 고려합니다
