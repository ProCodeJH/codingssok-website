DP(1) 기초 - 메모이제이션

동적 프로그래밍(DP)의 기본 원리를 이해하고, 메모이제이션과 타뷸레이션 기법으로 중복 계산을 제거하는 방법을 학습합니다.

이 유닛에서 배우는 것

  • 동적 프로그래밍의 두 가지 조건(최적 부분 구조, 중복 부분 문제)을 설명할 수 있습니다.
  • 메모이제이션(Top-Down) 기법으로 재귀 함수의 중복 호출을 제거할 수 있습니다.
  • 타뷸레이션(Bottom-Up) 기법으로 반복문 기반 DP를 구현할 수 있습니다.
  • 간단한 DP 점화식을 세우고 base case를 정의할 수 있습니다.

핵심 개념

DP의 두 가지 조건
1. 최적 부분 구조 (Optimal Substructure)
큰 문제의 최적 해가 작은 부분 문제의 최적 해로 구성됩니다.
예: fib(n) = fib(n-1) + fib(n-2)

2. 중복 부분 문제 (Overlapping Subproblems)
같은 부분 문제가 여러 번 반복적으로 등장합니다.
예: fib(5) 계산 시 fib(2)가 3번 호출됩니다.

두 조건이 모두 만족되면 DP를 적용할 수 있습니다.

타뷸레이션 (Bottom-Up)
메모이제이션은 재귀(Top-Down) 방식이지만, 타뷸레이션은 반복문(Bottom-Up) 방식입니다.

- 작은 문제부터 차례로 해결하며 테이블(배열)을 채워 나갑니다.
- 재귀 호출 오버헤드가 없어 일반적으로 더 빠릅니다.
- 스택 오버플로 위험이 없습니다.
- 대부분의 DP 문제는 타뷸레이션으로 구현합니다.

피보나치 - 메모이제이션 (Top-Down)

C
1#include <stdio.h>
2
3#define MAX 101
4int memo[MAX]; /* 메모 배열: 0으로 초기화 */
5
6/* 메모이제이션: O(n) */
7int fib(int n) {
8 if (n <= 1) return n;
9 if (memo[n] != 0) return memo[n]; /* 이미 계산했으면 반환 */

11 return memo[n];
12}
13
14int main(void) {
15 for (int i = 0; i <= 10; i++) {
16 printf("fib(%d) = %d\n", i, fib(i));
17 }
18 printf("fib(40) = %d\n", fib(40)); /* 즉시 출력 */
19 return 0;
20}
실행 결과
fib(0) = 0 fib(1) = 1 ... fib(10) = 55 fib(40) = 102334155

피보나치 - 타뷸레이션 (Bottom-Up)

C
1#include <stdio.h>
2
3int main(void) {
4 int n = 40;
5 int dp[101]; /* 테이블 배열 */
6
7 dp[0] = 0; /* base case 1 */
8 dp[1] = 1; /* base case 2 */
9
10 for (int i = 2; i <= n; i++) {
11 dp[i] = dp[i - 1] + dp[i - 2]; /* 점화식 */
12 }
13
14 for (int i = 0; i <= 10; i++) {
15 printf("fib(%d) = %d\n", i, dp[i]);
16 }
17 printf("fib(%d) = %d\n", n, dp[n]);
18 return 0;
19}
실행 결과
fib(0) = 0 fib(1) = 1 ... fib(10) = 55 fib(40) = 102334155

C
1#include <stdio.h>
2
3int main(void) {
4 int n;
5 printf("정수 입력: ");

15 }

18 return 0;
19}
실행 결과
정수 입력: 4 4를 1,2,3의 합으로 나타내는 방법: 7가지
Top-Down vs Bottom-Up 선택 기준
Top-Down (메모이제이션)
- 재귀적 사고가 자연스러운 문제에 적합합니다.
- 필요한 부분 문제만 계산합니다 (lazy evaluation).
- 스택 깊이 제한에 주의해야 합니다.

Bottom-Up (타뷸레이션)
- 대부분의 대회 문제에서 선호됩니다.
- 반복문으로 구현하여 스택 오버플로가 없습니다.
- 모든 부분 문제를 순서대로 계산합니다.

에러 체험

에러 1: Base Case 누락
int fib(int n) {
if (memo[n] != 0) return memo[n];
memo[n] = fib(n-1) + fib(n-2); /* n=0, n=1에서 무한 재귀! */
return memo[n];
}

원인: if (n <= 1) return n; 을 빠뜨렸습니다.
해결: 반드시 base case를 먼저 처리한 뒤 점화식을 적용하십시오.
에러 2: 배열 크기 부족
int dp[40];
dp[40] = dp[39] + dp[38]; /* 배열 범위 초과! */

원인: dp[40]은 크기 40인 배열의 범위(0~39) 밖입니다.
해결: int dp[41]; 또는 넉넉하게 int dp[MAX]; (MAX = n+1 이상)로 선언하십시오.
에러 3: 메모 배열 초기화 실수
int memo[MAX];
memo[0] = 0; /* fib(0)=0과 '아직 계산 안 함'을 구분 못함 */
if (memo[n] != 0) return memo[n];

원인: fib(0)=0인데 초기값도 0이므로 매번 다시 계산합니다.
해결 1: memo 배열을 -1로 초기화하고 if (memo[n] != -1)로 검사합니다.
해결 2: fib(0)은 base case로 직접 처리하여 memo[0]을 사용하지 않습니다.

핵심 정리
1. DP 조건: 최적 부분 구조 + 중복 부분 문제가 모두 만족되어야 합니다
2. 메모이제이션(Top-Down): 재귀 + 메모 배열로 중복 호출을 제거합니다
3. 타뷸레이션(Bottom-Up): 반복문으로 작은 문제부터 테이블을 채웁니다
4. DP 풀이 4단계: 점화식 → base case → 메모 배열 → 순서대로 채우기
5. 피보나치 기준 O(2^n) → O(n)으로 시간 복잡도가 극적으로 개선됩니다

용어설명
동적 프로그래밍(Dynamic Programming)중복되는 부분 문제의 결과를 저장하여 같은 계산을 반복하지 않는 알고리즘 설계 기법입니다.
메모이제이션(Memoization)재귀 함수에서 이미 계산한 결과를 배열에 저장해두고, 같은 인자로 호출되면 저장된 값을 반환하는 Top-Down 기법입니다.
타뷸레이션(Tabulation)작은 부분 문제부터 순서대로 테이블(배열)을 채워 나가는 Bottom-Up 방식의 DP 구현입니다.
점화식(Recurrence Relation)큰 문제의 해를 작은 문제의 해로 표현하는 관계식입니다. 예: dp[n] = dp[n-1] + dp[n-2]
Base Case점화식에서 더 이상 분해할 수 없는 가장 작은 경우의 값입니다. 예: dp[0] = 0, dp[1] = 1
최적 부분 구조(Optimal Substructure)큰 문제의 최적 해가 작은 부분 문제들의 최적 해로 구성되는 성질입니다.
중복 부분 문제(Overlapping Subproblems)문제를 분해했을 때 같은 부분 문제가 여러 번 반복적으로 등장하는 성질입니다.
