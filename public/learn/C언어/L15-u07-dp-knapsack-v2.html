핵심 개념

0/1 배낭 문제란?
주어진 조건:
- 배낭의 최대 무게 용량: W
- 물건 n개, 각각 무게 w[i]와 가치 v[i]를 가짐

목표:
- 배낭에 담은 물건의 총 무게가 W 이하이면서
- 총 가치가 최대가 되도록 물건을 선택

제약:
- 각 물건은 넣거나(1) 안 넣거나(0) 둘 중 하나 -- 그래서 0/1 배낭
완전 탐색의 한계
물건 n개에 대해 넣기/안넣기 두 가지 선택이 있으므로
경우의 수 = 2^n

n = 20이면 약 100만 가지
n = 30이면 약 10억 가지
n = 40이면 약 1조 가지

완전 탐색은 n이 커지면 사실상 불가능합니다.
DP를 사용하면 O(n * W) 시간에 해결할 수 있습니다.

2D DP 테이블 설계

DP 테이블 정의
dp[i][w] = 처음 i개의 물건만 고려하고,
배낭 용량이 w일 때의 최대 가치

- i: 0 ~ n (물건 번호, 0은 아무 물건도 없는 상태)
- w: 0 ~ W (현재 배낭 용량)

초기값:
- dp[0][w] = 0 (물건이 없으면 가치도 0)
- dp[i][0] = 0 (용량이 0이면 아무것도 못 넣음)
점화식
i번째 물건(무게 wi, 가치 vi)에 대해:

경우 1: i번째 물건을 넣지 않는다
dp[i][w] = dp[i-1][w]

경우 2: i번째 물건을 넣는다 (wi <= w일 때만 가능)
dp[i][w] = dp[i-1][w - wi] + vi

최종:
dp[i][w] = max(dp[i-1][w], dp[i-1][w - wi] + vi)

만약 wi > w이면 (물건이 너무 무거우면):
dp[i][w] = dp[i-1][w] (넣을 수 없으므로)

1D 최적화 점화식
dp[j] = max(dp[j], dp[j - w[i]] + v[i])

순회 방향:
- 0/1 배낭: j를 W부터 w[i]까지 역순 (각 물건 최대 1번)

에러 체험

에러 1: 1D 배열에서 정순 순회
/* 잘못된 코드 -- 같은 물건을 여러 번 넣게 됩니다 */
for (int j = w; j <= W; j++) { /* 정순 순회! */
dp[j] = max(dp[j], dp[j - w] + v);
}

0/1 배낭에서는 반드시 역순(j = W부터 w까지)으로 순회해야 합니다.
정순으로 순회하면 이미 갱신된 dp[j-w]를 다시 참조하여
같은 물건을 중복으로 넣는 결과가 됩니다.
에러 2: 배열 인덱스 0부터 시작 시 초기화 누락
/* 잘못된 코드 -- dp[0][j] 초기화를 빠뜨림 */
for (int i = 0; i < n; i++) {
for (int j = 0; j <= W; j++) {
dp[i][j] = max(dp[i-1][j], ...); /* i=0일 때 dp[-1][j] 접근! */
}
}

물건 인덱스를 1부터 사용하고, dp[0][j] = 0으로 초기화해야 합니다.
또는 전역 배열로 선언하면 자동으로 0 초기화됩니다.
에러 3: 무게 초과 검사 누락
/* 잘못된 코드 */
dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i]);
/* w[i] > j이면 j - w[i]가 음수 -> 배열 범위 초과! */

/* 올바른 코드 */
if (w[i] > j)
dp[i][j] = dp[i-1][j];
else
dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i]);

요약

핵심 정리
1. 0/1 배낭: 각 물건을 넣거나(1) 안 넣거나(0) 선택하는 최적화 문제

6. 시간 O(nW), 공간 O(W) -- 1D 최적화 시
