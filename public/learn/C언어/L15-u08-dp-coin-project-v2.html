최소 동전 문제 (DP)

동적 프로그래밍(Bottom-Up)으로 최소 동전 개수를 구하고, 역추적과 거스름돈 불가능 처리까지 확장하는 프로젝트입니다.

이 유닛에서 배우는 것

  • DP Bottom-Up 방식으로 목표 금액을 만드는 최소 동전 개수를 구할 수 있습니다.
  • 어떤 동전을 사용했는지 역추적(backtracking) 배열로 복원하고, 모든 방법의 수를 구할 수 있습니다.
  • 여러 화폐 단위를 지원하며, 거스름돈이 불가능한 경우를 올바르게 처리할 수 있습니다.
  • 최적해에 사용된 동전 조합을 출력하고, 다양한 금액에 대해 결과를 검증할 수 있습니다.

10/* 최소 동전 개수 구하기 (Bottom-Up DP) */
11int minCoins(int coins[], int numCoins, int amount) {
12 /* 초기화: 금액 0은 동전 0개, 나머지는 무한대 */
13 dp[0] = 0;
14 for (int i = 1; i <= amount; i++) {
15 dp[i] = INF;
16 }
17
18 /* Bottom-Up: 금액 1부터 amount까지 순차 계산 */
19 for (int i = 1; i <= amount; i++) {
20 for (int j = 0; j < numCoins; j++) {
21 if (coins[j] <= i && dp[i - coins[j]] + 1 < dp[i]) {
22 dp[i] = dp[i - coins[j]] + 1;
23 }
24 }
25 }
26
27 return (dp[amount] >= INF) ? -1 : dp[amount];
28}

164/* 탐욕법과 DP 비교 */
165void compareGreedyDP(int coins[], int numCoins, int amount) {
166 /* 탐욕법: 큰 동전부터 사용 */
167 int greedyCount = 0;
168 int greedyRemain = amount;
169 int greedyUsed[MAX_COINS] = {0};
170 int greedyPossible = 1;
171
172 /* 동전을 내림차순으로 정렬된 순서로 사용한다고 가정 */
173 for (int j = numCoins - 1; j >= 0; j--) {
174 if (coins[j] <= greedyRemain) {
175 greedyUsed[j] = greedyRemain / coins[j];
176 greedyRemain -= coins[j] * greedyUsed[j];
177 greedyCount += greedyUsed[j];
178 }
179 }
180 if (greedyRemain != 0) greedyPossible = 0;
181
182 /* DP */
183 int dpResult = solve(coins, numCoins, amount);
184

188 printf("---------------------------------------------\n");
189
190 if (greedyPossible) {

192 "탐욕법", greedyCount, "가능");
193 } else {
194 printf("%-15s %-15s %-15s\n",

요약

핵심 정리
1. 최소 동전 문제: 목표 금액을 만드는 최소 동전 개수를 구하는 DP 문제
2. 상태 정의: dp[i] = 금액 i를 만드는 최소 동전 개수
3. 초기값: dp[0] = 0, 나머지 dp[i] = INF (무한대)
4. 점화식: dp[i] = min(dp[i], dp[i - coin] + 1)
5. 역추적: used[i]에 마지막 사용 동전을 기록하여 최적해 복원
6. 방법의 수(조합): 동전을 외부, 금액을 내부로 순회하여 중복 방지
7. 방법의 수(순열): 금액을 외부, 동전을 내부로 순회하면 순서가 다른 것을 다르게 셈
8. 불가능 판별: dp[amount] >= INF이면 거스름돈 불가능
9. 탐욕법 한계: 동전 단위가 배수 관계가 아니면 최적해를 보장하지 못함
10. 시간 복잡도: O(amount x 동전 종류 수), 공간 복잡도: O(amount)
