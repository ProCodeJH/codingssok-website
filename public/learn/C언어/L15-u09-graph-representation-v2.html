핵심 개념

그래프 용어 정리
정점(Vertex, Node): 그래프의 각 점. V로 표기합니다.
간선(Edge): 정점과 정점을 연결하는 선. E로 표기합니다.
차수(Degree): 한 정점에 연결된 간선의 수입니다.
경로(Path): 정점 A에서 정점 B까지 간선을 따라 이동하는 순서입니다.
가중치(Weight): 간선에 부여된 비용 또는 거리입니다.
방향 그래프(Directed Graph): 간선에 방향이 있습니다. A->B와 B->A는 다른 간선입니다.
무방향 그래프(Undirected Graph): 간선에 방향이 없습니다. A-B이면 양방향 이동 가능합니다.

예시: 정점 4개, 간선 3개인 무방향 그래프
0 --- 1
| |
2 3
V = {0, 1, 2, 3}, E = {(0,1), (0,2), (1,3)}

방향 그래프와 무방향 그래프 비교

방향 그래프의 인접 행렬/리스트 차이
무방향 그래프: 간선 (u,v) 추가 시 adj[u][v]와 adj[v][u] 모두 설정합니다.
방향 그래프: 간선 u->v 추가 시 adj[u][v]만 설정합니다.

방향 그래프에서의 차수:
진입 차수(in-degree): 해당 정점으로 들어오는 간선 수 (열 합)
진출 차수(out-degree): 해당 정점에서 나가는 간선 수 (행 합)

예: 방향 그래프 0->1, 0->2, 1->3
0 1 2 3
0 [ 0 1 1 0 ] 진출 차수: 2
1 [ 0 0 0 1 ] 진출 차수: 1
2 [ 0 0 0 0 ] 진출 차수: 0
3 [ 0 0 0 0 ] 진출 차수: 0
C
1/* 방향 그래프 — 인접 행렬 */
2void addDirectedEdge(int from, int to) {
3 adj[from][to] = 1; /* 한 방향만 설정 */
4}
5
6/* 진입 차수와 진출 차수 계산 */
7void calcDegrees(int v) {
8 int inDeg = 0, outDeg = 0;
9 for (int i = 0; i < V; i++) {
10 outDeg += adj[v][i]; /* v에서 나가는 간선 */
11 inDeg += adj[i][v]; /* v로 들어오는 간선 */
12 }
13 printf("정점 %d: 진입=%d, 진출=%d\n", v, inDeg, outDeg);
14}

에러 체험

에러 1: 무방향 그래프에서 한쪽만 설정
adj[u][v] = 1;
/* adj[v][u] = 1; 을 빠뜨림 */

무방향 그래프는 양방향이므로 adj[u][v]와 adj[v][u]를 모두 1로 설정해야 합니다.
한쪽만 설정하면 방향 그래프가 되어 탐색 결과가 달라집니다.
에러 2: 인접 리스트 메모리 해제 누락
/* 프로그램 종료 전 해제하지 않음 */

addEdge()에서 malloc()으로 할당한 노드를 free()하지 않으면
메모리 누수가 발생합니다.
반드시 freeGraph() 함수로 모든 노드를 해제하십시오.
에러 3: 정점 번호 범위 초과
int adj[100][100];
addEdge(0, 100); /* 배열 범위 초과! */

정점 번호가 0부터 V-1까지인지 확인하십시오.
문제에서 정점 번호가 1부터 시작하면 배열 크기를 V+1로 선언합니다.
에러 4: 자기 자신으로의 간선(자기 루프) 미처리
adj[u][u] = 1; /* 자기 루프 */

문제 조건에 따라 자기 루프가 허용되는지 확인해야 합니다.
차수 계산 시 자기 루프는 보통 2로 셉니다(한 간선이 같은 정점에 두 번 연결).

대회 기출

요약

핵심 정리
1. 그래프 = 정점(Vertex) + 간선(Edge), 관계를 표현하는 비선형 자료구조
2. 인접 행렬: V x V 2차원 배열, 간선 확인 O(1), 메모리 O(V^2), 밀집 그래프에 적합
3. 인접 리스트: V개의 연결 리스트, 이웃 순회 O(차수), 메모리 O(V+E), 희소 그래프에 적합
4. 무방향 그래프: 양쪽 모두 설정 (adj[u][v] = adj[v][u] = 1)
5. 방향 그래프: 한쪽만 설정 (adj[from][to] = 1)
6. 가중 그래프: 0/1 대신 가중치 값을 저장하거나 노드에 weight 필드 추가

그래프를 표현할 수 있게 되었으니, 이제 그래프 위를 탐색하는 방법을 배울 차례입니다. 다음 유닛에서는 DFS(Depth-First Search, 깊이 우선 탐색)를 학습합니다. 한 방향으로 끝까지 깊이 들어간 뒤 되돌아오는 방식으로, 재귀 호출 또는 스택을 이용하여 구현합니다. DFS는 경로 탐색, 연결 요소 판별, 사이클 검출 등 수많은 그래프 문제의 기반이 됩니다.
용어설명
그래프(Graph)정점(vertex)과 간선(edge)으로 구성된 비선형 자료구조입니다. 대상 간의 관계를 표현합니다.
정점(Vertex)그래프의 각 점입니다. 노드(Node)라고도 합니다.
간선(Edge)두 정점을 연결하는 선입니다. 방향이 있을 수도, 없을 수도 있습니다.
차수(Degree)한 정점에 연결된 간선의 수입니다. 방향 그래프에서는 진입 차수와 진출 차수로 구분합니다.
인접 행렬(Adjacency Matrix)V x V 크기의 2차원 배열로 그래프를 표현합니다. adj[i][j] = 1이면 간선이 존재합니다.
인접 리스트(Adjacency List)각 정점마다 연결된 정점들의 목록을 연결 리스트로 관리하는 표현 방식입니다.
가중 그래프(Weighted Graph)간선에 비용 또는 거리 등의 가중치가 부여된 그래프입니다.
희소 그래프(Sparse Graph)간선 수가 정점 수에 비해 적은 그래프입니다. 인접 리스트가 효율적입니다.
밀집 그래프(Dense Graph)간선 수가 정점 수의 제곱에 가까운 그래프입니다. 인접 행렬이 효율적입니다.
