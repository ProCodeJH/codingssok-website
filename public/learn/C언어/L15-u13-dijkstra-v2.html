다익스트라 -- 최단 거리

다익스트라 알고리즘으로 가중 그래프에서 단일 출발점 최단 거리를 구하는 방법을 배웁니다.

이 유닛에서 배우는 것

  • 다익스트라 알고리즘의 원리(그리디 + BFS 확장)를 이해할 수 있습니다.
  • 인접 행렬 기반 O(V^2) 다익스트라를 구현할 수 있습니다.
  • 우선순위 큐(힙)를 활용한 O(E log V) 다익스트라를 구현할 수 있습니다.
  • 음수 가중치에서 다익스트라가 실패하는 이유를 설명할 수 있습니다.

핵심 개념

다익스트라 알고리즘 원리
다익스트라 알고리즘은 그리디(Greedy) 전략과 BFS의 확장입니다.

핵심 아이디어:
'아직 방문하지 않은 정점 중 거리가 가장 짧은 정점'을 선택하여
그 정점의 인접 정점 거리를 갱신(relaxation)합니다.

전제 조건:
모든 간선의 가중치가 0 이상이어야 합니다 (음수 가중치 불가).

시간복잡도:
인접 행렬 + 선형 탐색: O(V^2)
인접 리스트 + 우선순위 큐(최소 힙): O(E log V)

결과:
시작 정점에서 모든 정점까지의 최단 거리 배열 dist[]를 구합니다.

15void dijkstra(int start) {
16 int dist[V];
17 int visited[V] = {0};
18 int parent[V];
19
20 // 1단계: 초기화
21 for (int i = 0; i < V; i++) {
22 dist[i] = INF;
23 parent[i] = -1;
24 }
25 dist[start] = 0;
26
27 // V번 반복
28 for (int count = 0; count < V; count++) {
29 // 2단계: 미방문 정점 중 최소 거리 정점 선택
30 int u = -1;
31 int minDist = INF;
32 for (int i = 0; i < V; i++) {
33 if (!visited[i] && dist[i] < minDist) {
34 minDist = dist[i];
35 u = i;
36 }
37 }
38
39 if (u == -1) break; // 연결되지 않은 정점
40 visited[u] = 1;
41
42 // 3단계: 인접 정점 거리 갱신 (relaxation)
43 for (int v = 0; v < V; v++) {
44 if (!visited[v] && graph[u][v] != INF &&
45 dist[u] + graph[u][v] < dist[v]) {
46 dist[v] = dist[u] + graph[u][v];
47 parent[v] = u;
48 }
49 }
50 }
51
52 // 결과 출력
53 printf("정점\t최단거리\t경로\n");
54 for (int i = 0; i < V; i++) {
55 printf("%d\t%d\t\t", i, dist[i]);
56 // 경로 역추적
57 int path[V], len = 0;
58 int cur = i;
59 while (cur != -1) {
60 path[len++] = cur;
61 cur = parent[cur];
62 }
63 for (int j = len - 1; j >= 0; j--) {
64 printf("%d", path[j]);
65 if (j > 0) printf(" -> ");
66 }
67 printf("\n");
68 }
69}

C
1#include <stdio.h>
2#include <limits.h>
3
4#define MAX_V 1000
5#define MAX_E 10000
6#define INF INT_MAX
7
8/* --- 인접 리스트 --- */
9typedef struct {
10 int to, weight, next;
11} Edge;
12
13Edge edges[MAX_E];
14int head[MAX_V]; // head[u]: u에서 나가는 첫 간선 인덱스
15int edgeCnt = 0;
16
17void initGraph(int n) {
18 for (int i = 0; i < n; i++) head[i] = -1;
19 edgeCnt = 0;
20}
21
22void addEdge(int from, int to, int weight) {
23 edges[edgeCnt].to = to;
24 edges[edgeCnt].weight = weight;
25 edges[edgeCnt].next = head[from];
26 head[from] = edgeCnt++;
27}
28
29/* --- 최소 힙 (우선순위 큐) --- */
30typedef struct {
31 int vertex, dist;
32} HeapNode;
33
34typedef struct {
35 HeapNode data[MAX_E + 1];
36 int size;
37} MinHeap;
38
39void heapInit(MinHeap *h) { h->size = 0; }
40
41void heapSwap(HeapNode *a, HeapNode *b) {
42 HeapNode t = *a; *a = *b; *b = t;
43}
44
45void heapPush(MinHeap *h, int v, int d) {
46 int i = ++h->size;
47 h->data[i].vertex = v;
48 h->data[i].dist = d;
49 while (i > 1 && h->data[i].dist < h->data[i/2].dist) {
50 heapSwap(&h->data[i], &h->data[i/2]);
51 i /= 2;
52 }
53}
54
55HeapNode heapPop(MinHeap *h) {
56 HeapNode top = h->data[1];
57 h->data[1] = h->data[h->size--];
58 int i = 1;
59 while (1) {
60 int smallest = i;
61 int left = 2 * i, right = 2 * i + 1;
62 if (left <= h->size && h->data[left].dist < h->data[smallest].dist)
63 smallest = left;
64 if (right <= h->size && h->data[right].dist < h->data[smallest].dist)
65 smallest = right;
66 if (smallest == i) break;
67 heapSwap(&h->data[i], &h->data[smallest]);
68 i = smallest;
69 }
70 return top;
71}

73/* --- 다익스트라 (힙 버전) --- */
74void dijkstraHeap(int n, int start) {
75 int dist[MAX_V];
76 int visited[MAX_V] = {0};
77
78 for (int i = 0; i < n; i++) dist[i] = INF;
79 dist[start] = 0;
80
81 MinHeap heap;
82 heapInit(&heap);
83 heapPush(&heap, start, 0);
84
85 while (heap.size > 0) {
86 HeapNode cur = heapPop(&heap);
87 int u = cur.vertex;
88
89 if (visited[u]) continue; // 이미 확정된 정점은 건너뜁니다
90 visited[u] = 1;
91
92 // u의 모든 인접 정점 탐색
93 for (int e = head[u]; e != -1; e = edges[e].next) {
94 int v = edges[e].to;
95 int w = edges[e].weight;
96 if (!visited[v] && dist[u] + w < dist[v]) {
97 dist[v] = dist[u] + w;
98 heapPush(&heap, v, dist[v]);
99 }
100 }
101 }
102
103 printf("시작점 %d에서의 최단 거리:\n", start);
104 for (int i = 0; i < n; i++) {
105 printf(" -> 정점 %d: ", i);
106 if (dist[i] == INF) printf("도달 불가\n");
107 else printf("%d\n", dist[i]);
108 }
109}

두 구현의 시간복잡도 비교

음수 가중치에서 다익스트라가 작동하지 않는 이유
다익스트라는 '한 번 확정된 정점의 최단 거리는 변하지 않는다'는 가정에 기반합니다.

반례:
A --(1)--> B --(−3)--> C
A --(2)--> C

다익스트라 진행:
1. dist[A]=0, dist[B]=1, dist[C]=2
2. B 선택: dist[C] = min(2, 1+(−3)) = −2 이지만,
이미 C가 dist=2로 확정되었다면 갱신하지 않습니다.
3. 올바른 답은 −2이지만, 다익스트라는 2를 반환합니다.

음수 가중치가 있으면 확정된 거리가 나중에 더 짧아질 수 있으므로
그리디 전략이 실패합니다. 이 경우 벨만-포드(Bellman-Ford) 알고리즘을 사용해야 합니다.

에러 1: 음수 가중치를 가진 그래프에 다익스트라 적용
addEdge(0, 1, 3);

다익스트라는 음수 가중치를 처리하지 못합니다.
정점 2의 올바른 최단 거리는 3 + (−5) = −2이지만,
다익스트라는 1을 반환할 수 있습니다.

-> 음수 가중치가 있으면 벨만-포드(Bellman-Ford) 알고리즘을 사용하십시오.
에러 2: 초기 거리 배열을 INF로 설정하지 않음
int dist[V];
// dist 초기화를 빼먹음!
dist[start] = 0;

초기화하지 않으면 dist 배열에 쓰레기 값이 들어 있어
relaxation 비교(dist[u] + w < dist[v])가 올바르게 작동하지 않습니다.

-> 반드시 모든 dist를 INF로 초기화한 후 시작점만 0으로 설정하십시오.
for (int i = 0; i < V; i++) dist[i] = INF;
dist[start] = 0;
에러 3: visited 검사 누락 (힙 버전)
HeapNode cur = heapPop(&heap);
int u = cur.vertex;
// if (visited[u]) continue; // 이 줄을 빼먹으면!
visited[u] = 1;

힙에는 같은 정점이 여러 번 들어갈 수 있습니다.
visited 검사를 하지 않으면 이미 확정된 정점을 다시 처리하여
불필요한 연산이 발생하고, 최악의 경우 무한 루프에 빠질 수 있습니다.

-> heapPop 직후 반드시 visited[u] 검사를 수행하십시오.

대회 기출 문제

대회 기출

요약

핵심 정리
1. 다익스트라 알고리즘은 단일 출발점에서 모든 정점까지의 최단 거리를 구합니다.
2. 핵심 전략: 미방문 정점 중 거리가 가장 짧은 정점을 선택(그리디)하고 인접 정점을 갱신(relaxation)합니다.
3. 인접 행렬 O(V^2) 버전은 구현이 단순하고, 힙 O(E log V) 버전은 희소 그래프에서 빠릅니다.
4. 음수 가중치가 있으면 다익스트라는 올바른 답을 보장하지 못합니다 (벨만-포드 사용).
5. dist 배열은 반드시 INF로 초기화하고, 힙 버전에서는 visited 검사를 빠뜨리지 않아야 합니다.

다음 유닛으로

다음: 플로이드-워셜
다익스트라는 하나의 시작점에서 모든 정점까지의 최단 거리를 구합니다. 만약 모든 정점 쌍 사이의 최단 거리를 한꺼번에 구해야 한다면 어떻게 해야 할까요? 다음 유닛에서는 3중 for문만으로 모든 쌍 최단 거리를 구하는 플로이드-워셜(Floyd-Warshall) 알고리즘을 배웁니다. 음수 가중치도 처리할 수 있어 다익스트라와 상호 보완적입니다.
용어설명
다익스트라 알고리즘(Dijkstra's Algorithm)단일 출발점에서 모든 정점까지의 최단 거리를 구하는 그리디 알고리즘입니다. 음수 가중치가 없어야 합니다.
Relaxation (갱신/완화)현재 알려진 최단 거리보다 더 짧은 경로를 발견하면 거리를 갱신하는 연산입니다. if (dist[u]+w < dist[v]) dist[v] = dist[u]+w;
INF (무한대)아직 경로가 발견되지 않은 정점의 초기 거리 값입니다. 보통 INT_MAX를 사용합니다.
최단 경로 트리(Shortest Path Tree)시작점에서 모든 정점까지의 최단 경로를 모아 만든 트리 구조입니다. parent 배열로 표현합니다.
희소 그래프(Sparse Graph)간선 수가 정점 수에 비해 적은 그래프입니다. E가 V^2보다 훨씬 작으며, 인접 리스트와 힙 사용이 유리합니다.
밀집 그래프(Dense Graph)간선 수가 정점 수의 제곱에 가까운 그래프입니다. 인접 행렬 사용이 적합합니다.
플로이드-워셜(Floyd-Warshall)모든 정점 쌍 사이의 최단 거리를 O(V^3)에 구하는 알고리즘으로, 음수 가중치도 처리 가능합니다.
벨만-포드(Bellman-Ford)음수 가중치가 있는 그래프에서 단일 출발점 최단 거리를 O(VE)에 구하는 알고리즘입니다.
