핵심 개념

플로이드-워셜 핵심 아이디어
질문: i에서 j로 갈 때, 경유지 k를 거쳐가면 더 짧은가?

현재까지 알려진 최단 거리: dist[i][j]
경유지 k를 거치는 거리: dist[i][k] + dist[k][j]

만약 dist[i][k] + dist[k][j] < dist[i][j]이면
dist[i][j]를 더 짧은 값으로 갱신합니다.

이 질문을 모든 경유지 k(0, 1, 2, ..., V-1)에 대해,
모든 출발지 i와 도착지 j에 대해 반복하면
결과적으로 모든 정점 쌍의 최단 거리가 구해집니다.

점화식

플로이드-워셜 점화식
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

- dist[i][j]: 정점 i에서 j까지의 최단 거리
- k: 경유 가능한 정점 (0, 1, 2, ..., V-1)
- 경유지 k를 0부터 V-1까지 순서대로 추가하며 갱신

초기값:
- dist[i][i] = 0 (자기 자신까지의 거리)
- dist[i][j] = w(i,j) (간선이 있으면 가중치)
- dist[i][j] = INF (간선이 없으면 무한대)

다익스트라 vs 플로이드-워셜 비교

음수 가중치 처리
플로이드-워셜은 음수 가중치 간선이 있어도 올바르게 동작합니다.
단, 음수 사이클(순환하면 거리가 계속 줄어드는 사이클)이 있으면
최단 거리가 정의되지 않습니다.

음수 사이클 검출 방법:
플로이드-워셜 수행 후 dist[i][i] < 0인 정점 i가 있으면
해당 정점은 음수 사이클에 포함되어 있습니다.

for (int i = 0; i < V; i++) {
if (dist[i][i] < 0)
printf("정점 %d: 음수 사이클 존재\n", i);
}

에러 체험

에러 1: k 루프의 순서가 잘못된 경우
// 잘못된 코드: k가 안쪽 루프
for (int i = 0; i < V; i++)
for (int j = 0; j < V; j++)
for (int k = 0; k < V; k++) // k가 가장 안쪽!
if (dist[i][k] + dist[k][j] < dist[i][j])
dist[i][j] = dist[i][k] + dist[k][j];

-> 경유지를 순서대로 추가하는 동적 프로그래밍 원리가 깨집니다.
-> k가 가장 바깥 루프여야 "경유지 {0..k}만 사용했을 때의 최단 거리"가
올바르게 갱신됩니다.
-> k를 안쪽에 두면 아직 갱신되지 않은 dist 값을 참조하여 오답이 나옵니다.

올바른 순서: k(바깥) -> i(중간) -> j(안쪽)
에러 2: INF + INF 오버플로
// 잘못된 코드: 오버플로 검사 없음
if (dist[i][k] + dist[k][j] < dist[i][j])
dist[i][j] = dist[i][k] + dist[k][j];

-> dist[i][k]와 dist[k][j]가 모두 INF(예: INT_MAX)이면
두 값의 합이 정수 오버플로를 일으킵니다.
-> INT_MAX + INT_MAX = 음수 (정수 오버플로)
-> 이 음수가 dist[i][j]보다 작다고 판단되어 잘못된 갱신 발생

해결: INF 검사를 먼저 수행하거나, INF 값을 INT_MAX/2 등으로 설정

// 올바른 코드
if (dist[i][k] != INF && dist[k][j] != INF) {
if (dist[i][k] + dist[k][j] < dist[i][j])
dist[i][j] = dist[i][k] + dist[k][j];
}
에러 3: 자기 자신으로의 거리 초기화 누락
// 잘못된 코드: 대각선을 0으로 초기화하지 않음
for (int i = 0; i < V; i++)
for (int j = 0; j < V; j++)
dist[i][j] = graph[i][j]; // graph[i][i]가 0이 아닐 수 있음

-> dist[i][i]가 0이 아니면 자기 자신을 경유하는 계산이 모두 오류
-> 반드시 dist[i][i] = 0으로 초기화해야 합니다.

// 올바른 코드
for (int i = 0; i < V; i++) {
for (int j = 0; j < V; j++)
dist[i][j] = graph[i][j];
dist[i][i] = 0; // 명시적으로 0 설정
}

요약

핵심 정리
1. 플로이드-워셜: 모든 정점 쌍의 최단 거리를 O(V^3)에 계산
2. 점화식: dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
3. 핵심: k(경유지) 루프가 반드시 가장 바깥에 위치
4. 경로 역추적: next[i][j] 배열로 실제 경로 복원 가능
5. 음수 가중치 처리 가능, 음수 사이클은 dist[i][i] < 0으로 검출
6. 다익스트라(단일 출발) vs 플로이드-워셜(모든 쌍): 상황에 맞게 선택
7. 정점 수가 적을 때(수백 이하) 구현이 간단하여 매우 유용
