최소 신장 트리 -- 크루스칼, 프림

모든 정점을 최소 비용으로 연결하는 최소 신장 트리(MST)의 개념과 크루스칼, 프림 알고리즘을 학습합니다.

이 유닛에서 배우는 것

  • 신장 트리의 정의와 성질(모든 정점 연결, 사이클 없음, 간선 V-1개)을 설명할 수 있습니다.
  • 크루스칼 알고리즘의 원리를 이해하고 Union-Find를 활용하여 구현할 수 있습니다.
  • 프림 알고리즘의 원리를 이해하고 인접 행렬 방식으로 구현할 수 있습니다.
  • 두 알고리즘의 차이를 비교하고 상황에 맞는 알고리즘을 선택할 수 있습니다.

핵심 개념

신장 트리(Spanning Tree)
신장 트리란 그래프의 모든 정점을 포함하면서 사이클이 없는 부분 그래프입니다.

성질 1: 정점이 V개이면 간선은 정확히 V-1개입니다.
성질 2: 임의의 두 정점 사이에 경로가 정확히 하나 존재합니다.
성질 3: 간선을 하나 추가하면 반드시 사이클이 생깁니다.
성질 4: 간선을 하나 제거하면 그래프가 분리됩니다.

최소 신장 트리(MST)는 가능한 모든 신장 트리 중 간선 가중치의 합이 최소인 것입니다.

크루스칼 알고리즘

크루스칼 알고리즘 단계
1단계: 모든 간선을 가중치 기준으로 오름차순 정렬합니다.
2단계: 정렬된 간선을 순서대로 확인합니다.
3단계: 해당 간선의 두 정점이 같은 집합에 속하면(사이클 발생) 건너뜁니다.
4단계: 다른 집합에 속하면 간선을 MST에 추가하고 두 집합을 합칩니다.
5단계: V-1개의 간선을 선택하면 종료합니다.

시간복잡도: O(E log E) -- 간선 정렬이 지배적입니다.

Union-Find (Disjoint Set) 자료구조

Union-Find 핵심 연산
find(x): x가 속한 집합의 대표(루트)를 반환합니다.
union(a, b): a가 속한 집합과 b가 속한 집합을 하나로 합칩니다.

경로 압축(Path Compression): find 과정에서 만나는 모든 노드를 루트에 직접 연결하여 이후 find를 O(1)에 가깝게 만듭니다.
rank 기반 합치기: 높이가 낮은 트리를 높은 트리 아래에 붙여 트리의 높이 증가를 억제합니다.

두 최적화를 적용하면 연산당 거의 O(1) (정확히는 O(alpha(N)), alpha는 역 아커만 함수)입니다.

1#include <stdio.h>
2
3#define MAX 1001
4int parent[MAX];
5int rank_arr[MAX]; /* rank는 C++ 키워드와 충돌 방지 */
6
7void init(int n) {
8 for (int i = 0; i < n; i++) {
9 parent[i] = i;
10 rank_arr[i] = 0;
11 }
12}
13
14int find(int x) {
15 if (parent[x] != x)
16 parent[x] = find(parent[x]); /* 경로 압축 */
17 return parent[x];
18}
19
20void unite(int a, int b) {
21 int ra = find(a);
22 int rb = find(b);
23 if (ra == rb) return; /* 이미 같은 집합 */
24 if (rank_arr[ra] < rank_arr[rb]) {
25 parent[ra] = rb;
26 } else if (rank_arr[ra] > rank_arr[rb]) {
27 parent[rb] = ra;
28 } else {
29 parent[rb] = ra;
30 rank_arr[ra]++;
31 }
32}

프림 알고리즘

undefined

프림 알고리즘 단계
1단계: 시작 정점을 MST 집합에 넣고, 나머지 정점의 key 값을 무한대로 초기화합니다.
2단계: MST에 포함되지 않은 정점 중 key 값이 가장 작은 정점 u를 선택합니다.
3단계: u를 MST에 추가합니다.
4단계: u와 인접한 정점 v에 대해, 간선 (u,v)의 가중치가 v의 현재 key보다 작으면 key[v]를 갱신합니다.
5단계: 모든 정점이 MST에 포함될 때까지 2~4단계를 반복합니다.

시간복잡도: O(V^2) (인접 행렬), O(E log V) (우선순위 큐 사용 시)

프림 구현

C
1#include <stdio.h>
2#include <limits.h>
3
4#define MAX 1001
5#define INF INT_MAX
6
7int graph[MAX][MAX]; /* 인접 행렬 (0이면 간선 없음) */
8int key[MAX]; /* MST에 연결하는 최소 비용 */
9int inMST[MAX]; /* MST 포함 여부 */
10int parent_node[MAX]; /* MST에서의 부모 정점 */
11
12void prim(int V) {
13 for (int i = 0; i < V; i++) {
14 key[i] = INF;
15 inMST[i] = 0;
16 parent_node[i] = -1;
17 }
18 key[0] = 0; /* 시작 정점 */
19
20 for (int count = 0; count < V; count++) {
21 /* key가 최소인 정점 선택 */
22 int u = -1, minKey = INF;
23 for (int v = 0; v < V; v++) {
24 if (!inMST[v] && key[v] < minKey) {
25 minKey = key[v];
26 u = v;
27 }
28 }
29 inMST[u] = 1;
30
31 /* u와 인접한 정점의 key 갱신 */
32 for (int v = 0; v < V; v++) {
33 if (graph[u][v] && !inMST[v] && graph[u][v] < key[v]) {
34 key[v] = graph[u][v];
35 parent_node[v] = u;
36 }
37 }
38 }
39}
40
41int main(void) {
42 int V, E;
43 scanf("%d %d", &V, &E);
44
45 for (int i = 0; i < E; i++) {
46 int u, v, w;
47 scanf("%d %d %d", &u, &v, &w);
48 graph[u][v] = w;
49 graph[v][u] = w; /* 무방향 그래프 */
50 }
51
52 prim(V);
53
54 int total = 0;
55 printf("MST 간선:\n");
56 for (int i = 1; i < V; i++) {
57 printf("%d -- %d (가중치 %d)\n", parent_node[i], i, key[i]);
58 total += key[i];
59 }
60 printf("MST 총 비용: %d\n", total);
61 return 0;
62}

알고리즘 선택 기준
간선이 적은 희소 그래프(E << V^2): 크루스칼이 유리합니다.
간선이 많은 밀집 그래프(E 근처 V^2): 프림이 유리합니다.
간선 리스트로 입력이 주어지는 경우: 크루스칼이 편리합니다.
인접 행렬로 입력이 주어지는 경우: 프림이 편리합니다.

대회에서는 크루스칼이 더 자주 사용됩니다. 구현이 간결하고 간선 리스트 입력에 바로 적용할 수 있기 때문입니다.

에러 체험

에러 1: Union-Find 초기화 누락
/* parent 배열을 초기화하지 않으면 */
int parent[MAX]; /* 쓰레기 값! */
find(3); /* 무한 루프 또는 잘못된 결과 */

-> 반드시 parent[i] = i로 초기화해야 합니다.
에러 2: 사이클 검사 없이 간선 추가
/* find 비교 없이 모든 간선을 추가하면 */
for (int i = 0; i < E; i++)
mst_weight += edges[i].w; /* 사이클 포함! */

-> find(u) != find(v)일 때만 간선을 추가해야 합니다.
에러 3: 프림에서 이미 MST에 포함된 정점을 다시 선택
/* inMST 체크 누락 */
if (graph[u][v] < key[v]) /* inMST[v] 확인 안 함 */
key[v] = graph[u][v];

-> !inMST[v] 조건을 반드시 포함해야 합니다.
이미 MST에 포함된 정점의 key를 갱신하면 잘못된 결과가 나옵니다.
에러 4: 간선 가중치 비교 함수의 오버플로
int cmp(const void *a, const void *b) {
return ((Edge *)a)->w - ((Edge *)b)->w; /* 오버플로 가능! */
}

-> 가중치가 매우 크면 뺄셈 오버플로가 발생합니다.
안전한 방법: if/else로 -1, 0, 1을 반환합니다.

요약

핵심 정리
1. 신장 트리는 모든 정점을 연결하고 사이클이 없으며 간선이 V-1개인 부분 그래프입니다
2. 최소 신장 트리(MST)는 간선 가중치 합이 최소인 신장 트리입니다
3. 크루스칼: 간선을 가중치 순으로 정렬하고, Union-Find로 사이클 없이 V-1개 선택합니다
4. Union-Find: find(경로 압축)와 union(rank 기반)으로 거의 O(1) 집합 연산을 수행합니다
5. 프림: 시작 정점에서 출발하여 매번 최소 key 정점을 MST에 추가합니다
6. 크루스칼은 희소 그래프에, 프림은 밀집 그래프에 유리합니다
