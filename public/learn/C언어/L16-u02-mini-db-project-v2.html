미니 DB -- CRUD 프로그램

구조체, 동적 메모리, 파일 입출력을 종합하여 데이터베이스의 기본 CRUD(생성/조회/수정/삭제) 기능을 갖춘 프로그램을 단계적으로 완성합니다.

이 유닛에서 배우는 것

  • 구조체 배열을 활용하여 CRUD(Create/Read/Update/Delete) 기능이 있는 메뉴 프로그램을 구현할 수 있습니다.
  • CSV 파일 저장/불러오기와 이름/ID 기반 검색, 정렬 기능을 추가할 수 있습니다.
  • 동적 메모리(realloc)로 가변 크기 레코드를 관리하고 이진탐색 인덱스를 구현할 수 있습니다.
  • 트랜잭션 개념(백업/복원)을 적용하여 데이터 무결성을 보장하는 프로그램을 작성할 수 있습니다.

요약

핵심 정리
1. CRUD: Create(추가), Read(조회), Update(수정), Delete(삭제)의 네 가지 기본 데이터 조작
2. 구조체 배열: 같은 형태의 레코드를 저장하는 가장 기본적인 테이블 구조
3. 배열 삭제: 뒤쪽 원소를 앞으로 이동, O(n) 시간 복잡도
4. CSV 파일: 쉼표로 구분된 텍스트 형식, fprintf로 저장하고 fgets + sscanf로 불러오기
5. 검색: 선형탐색 O(n) vs 이진탐색 O(log n), 이진탐색은 정렬 전제 필요
6. 동적 메모리: realloc으로 배열 크기를 2배씩 확장, 반드시 임시 포인터로 결과 확인
7. 다중 테이블: 학생/과목 테이블 분리, studentId로 관계 연결 (외래키)
8. 트랜잭션: 작업 전 상태를 백업(begin), 확정(commit), 취소 시 복원(rollback)
9. qsort: ID 기준 정렬 유지로 이진탐색 가능하게 함
10. 메모리 해제: malloc/realloc으로 할당한 메모리는 반드시 free로 해제
