들여쓰기 규칙
1. 탭(Tab) 또는 스페이스 4칸 중 하나를 선택하고 섞지 않습니다.
2. 중괄호 안의 모든 문장은 한 단계 들여씁니다.
3. if/for/while의 본문이 한 줄이라도 중괄호를 사용하는 것이 안전합니다.

나쁨:
if (x > 0)
printf("양수\n");
printf("처리 완료\n"); // 들여쓰기만으로는 if에 속하지 않음!

좋음:
if (x > 0) {
printf("양수\n");
printf("처리 완료\n");
}

const vs #define
#define: 전처리기가 텍스트 치환. 타입 검사 없음. 스코프 제한 불가.
const: 컴파일러가 처리. 타입 검사 가능. 스코프 제한 가능.

#define MAX_SIZE 100 // 전처리기 상수
const int MAX_SIZE = 100; // 컴파일러 상수 (권장)

배열 크기 지정 등 컴파일 시점 상수가 필요하면 #define을,
일반적인 상수에는 const를 사용합니다.

코드 작성 후 점검 체크리스트
[ ] 변수명과 함수명이 역할을 설명하는가?
[ ] 하나의 이름 규칙(snake_case 또는 camelCase)을 일관되게 사용했는가?
[ ] 들여쓰기가 일관적인가? (탭/스페이스 혼용 없음)
[ ] 모든 if/for/while에 중괄호를 사용했는가?
[ ] 매직 넘버를 상수(#define 또는 const)로 치환했는가?
[ ] 함수가 단일 책임 원칙을 지키는가? (20줄 이내)
[ ] 주석이 '왜(Why)'를 설명하는가? (코드 반복 주석 제거)
[ ] 전역 변수를 최소화했는가?
[ ] 논리 블록 사이에 빈 줄을 삽입했는가?
[ ] 컴파일 경고(-Wall) 없이 빌드되는가?

에러 체험

실수 1: 의미 없는 변수명
int a = 10, b = 20, c;
c = a + b;
printf("%d\n", c);

--> a, b, c가 무엇을 의미하는지 알 수 없습니다.
--> int width = 10, height = 20;
int area = width + height; 처럼 수정합니다.
(더 나아가 width + height가 맞는 공식인지도 이름 덕분에 의심할 수 있습니다.)
실수 2: 중괄호 생략으로 인한 논리 오류
if (temperature > 30)
printf("더움\n");
turn_on_ac(); // 항상 실행됨!

--> 들여쓰기만으로는 블록이 만들어지지 않습니다.
--> 반드시 중괄호를 사용합니다:
if (temperature > 30) {
printf("더움\n");
turn_on_ac();
}
실수 3: 거짓 주석
/* 배열을 오름차순으로 정렬 */
for (int i = 0; i < n - 1; i++)
for (int j = 0; j < n - 1 - i; j++)
if (arr[j] < arr[j+1]) // 실제로는 내림차순!
swap(&arr[j], &arr[j+1]);

--> 코드를 수정하면 주석도 반드시 함께 수정합니다.
--> 주석이 거짓이면 없는 것보다 해롭습니다.
실수 4: 매직 넘버 남용
if (retry > 3) {
sleep(5);
return -1;
}

--> 3은 최대 재시도 횟수, 5는 대기 시간, -1은 오류 코드입니다.
--> #define MAX_RETRY 3
#define RETRY_DELAY 5
#define ERROR_TIMEOUT -1 로 치환합니다.

좋은 코딩 스타일을 갖추었다면, 이제 C 표준 라이브러리가 제공하는 도구들을 총정리할 차례입니다. 다음 유닛에서는 stdio.h, stdlib.h, string.h, math.h, ctype.h, time.h 등 주요 헤더 파일의 핵심 함수들을 한눈에 정리하고, 실전에서 자주 사용하는 패턴을 학습합니다.
용어설명
코딩 컨벤션(Coding Convention)팀이나 프로젝트에서 합의한 코드 작성 규칙. 이름 짓기, 들여쓰기, 주석 등을 포함합니다.
camelCase첫 단어는 소문자, 이후 단어의 첫 글자를 대문자로 쓰는 방식. 예: studentCount, maxScore.
snake_case모든 단어를 소문자로 쓰고 밑줄(_)로 연결하는 방식. 예: student_count, max_score. C 언어 표준 스타일입니다.
K&R 스타일Kernighan & Ritchie가 사용한 중괄호 스타일. 여는 중괄호를 같은 줄에 배치합니다.
Allman 스타일BSD 스타일이라고도 합니다. 여는 중괄호를 새 줄에 배치하여 블록 경계를 명확히 합니다.
매직 넘버(Magic Number)코드에 의미 설명 없이 직접 사용된 숫자 리터럴. #define이나 const로 이름을 부여하여 제거합니다.
단일 책임 원칙(SRP)하나의 함수는 하나의 책임(역할)만 가져야 한다는 설계 원칙입니다.
리팩토링(Refactoring)동작을 바꾸지 않으면서 코드의 구조와 가독성을 개선하는 작업입니다.
문서 주석함수의 목적, 매개변수, 반환값 등을 기술하는 주석. 코드 이해와 유지보수에 필수적입니다.
