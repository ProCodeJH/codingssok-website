C11/C17 새 기능 맛보기

C 언어 표준의 진화를 살펴보고, C11과 C17에서 추가된 주요 기능(_Generic, _Static_assert, _Alignas 등)을 실습합니다.

이 유닛에서 배우는 것

  • C 표준의 역사(C89, C99, C11, C17, C23)를 설명할 수 있습니다.
  • _Generic 매크로를 사용하여 타입별 함수를 선택 호출할 수 있습니다.
  • _Static_assert로 컴파일 시점에 조건을 검증할 수 있습니다.
  • 의 정확한 크기 정수형을 활용할 수 있습니다.

_Generic - 제네릭 선택 (Type-Generic Selection)
_Generic(expr, type1: val1, type2: val2, default: val3)

컴파일 시점에 expr의 타입을 검사하여 대응하는 값을 선택합니다.
C++의 함수 오버로딩과 유사한 효과를 매크로로 구현할 수 있습니다.
C_Generic 매크로로 타입별 함수 호출
1#include <stdio.h>
2#include <math.h>
3
4// 타입에 따라 적절한 abs 함수를 선택하는 매크로
5#define my_abs(x) _Generic((x), \
6 int: abs, \
7 long: labs, \
8 float: fabsf, \
9 double: fabs \
10)(x)
11
12// 타입 이름을 문자열로 반환하는 매크로
13#define type_name(x) _Generic((x), \
14 int: "int", \
15 float: "float", \
16 double: "double", \
17 char: "char", \
18 default: "other" \
19)
20
21int main(void) {
22 int a = -5;
23 double b = -3.14;
24 float c = -2.7f;
25
26 printf("abs(%d) = %d\n", a, my_abs(a));
27 printf("abs(%.2f) = %.2f\n", b, my_abs(b));
28 printf("abs(%.1f) = %.1f\n", c, my_abs(c));
29
30 printf("a의 타입: %s\n", type_name(a));
31 printf("b의 타입: %s\n", type_name(b));
32 printf("c의 타입: %s\n", type_name(c));
33 return 0;
34}
실행 결과
abs(-5) = 5 abs(-3.14) = 3.14 abs(-2.7) = 2.7 a의 타입: int b의 타입: double c의 타입: float
_Static_assert - 컴파일 시점 단언
_Static_assert(조건식, "에러 메시지");

조건식이 거짓이면 컴파일 에러가 발생합니다.
런타임 assert와 달리 프로그램 실행 전에 오류를 잡아냅니다.
C23부터는 메시지 생략이 가능합니다.
C_Static_assert 사용 예
1#include <stdio.h>
2#include <limits.h>
3
4// int가 최소 4바이트인지 컴파일 시점에 검증
5_Static_assert(sizeof(int) >= 4,
6 "이 프로그램은 int가 최소 4바이트인 환경이 필요합니다.");
7
8// char가 1바이트인지 검증
9_Static_assert(sizeof(char) == 1,
10 "C 표준에 의해 sizeof(char)는 항상 1이어야 합니다.");
11
12// 구조체 크기 검증 (네트워크 프로토콜 등에서 유용)
13typedef struct {
14 unsigned char id;
15 unsigned char type;
16 unsigned short length;
17} PacketHeader;
18
19_Static_assert(sizeof(PacketHeader) == 4,
20 "PacketHeader는 정확히 4바이트여야 합니다.");
21
22int main(void) {
23 printf("sizeof(int) = %zu\n", sizeof(int));
24 printf("sizeof(PacketHeader) = %zu\n", sizeof(PacketHeader));

sizeof(int) = 4 sizeof(PacketHeader) = 4 모든 정적 단언을 통과했습니다.
_Alignas / _Alignof - 메모리 정렬
_Alignas(N) 변수선언; // 변수를 N바이트 경계에 정렬
_Alignof(타입) // 타입의 정렬 요구 사항을 반환

를 포함하면 alignas, alignof로 짧게 쓸 수 있습니다.
캐시 라인 최적화나 SIMD 연산에서 중요합니다.
_Noreturn - 반환하지 않는 함수
#include

noreturn void fatal_error(const char *msg) {
fprintf(stderr, "치명적 오류: %s\n", msg);
exit(1);
}

함수가 절대 반환하지 않음을 컴파일러에게 알립니다.
exit(), abort() 등을 호출하는 함수에 사용합니다.
C23에서는 [[noreturn]] 속성으로 대체되었습니다.
- 멀티스레드 지원 (소개)

thrd_join() - 스레드 종료 대기

mtx_unlock() - 뮤텍스 해제
_Atomic 타입 - 원자적 연산 보장

주의: MSVC는 를 지원하지 않으므로
Windows에서는 Windows API나 POSIX 스레드를 사용합니다.
익명 구조체와 익명 공용체
struct Vector3 {
union {
struct { float x, y, z; }; // 익명 구조체

struct Vector3 v = {1.0f, 2.0f, 3.0f};

printf("%.1f\n", v.data[0]); // 1.0 (배열 접근)

에러 체험

에러 1: _Generic에서 타입 불일치
#define print_val(x) _Generic((x), \
int: printf("%d", x), \
double: printf("%f", x)) \

print_val("hello"); // 컴파일 에러!

"hello"는 char*이므로 일치하는 타입이 없습니다.
default: 절을 추가하면 해결됩니다.

수정:
#define print_val(x) _Generic((x), \
int: printf("%d", x), \
double: printf("%f", x), \
default: printf("%p", (void*)(x)))
에러 2: _Static_assert를 함수 안에서만 사용 가능하다고 착각
// 파일 범위에서도 사용 가능합니다!
_Static_assert(sizeof(int) == 4, "int는 4바이트여야 합니다.");

int main(void) {
// 함수 안에서도 사용 가능합니다.
_Static_assert(sizeof(long) >= 4, "long은 최소 4바이트");
return 0;
}
에러 3: -std 옵션 없이 C11 기능 사용
// C11 기능을 사용했지만 C99 모드로 컴파일
// gcc -std=c99 source.c

_Static_assert(1, "test"); // 에러: '_Static_assert' 미선언

해결: gcc -std=c11 source.c 또는 gcc -std=c17 source.c
