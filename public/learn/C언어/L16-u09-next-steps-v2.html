대회 준비 로드맵

알고리즘 대회 단계별 학습 로드맵
[1단계] 기초 다지기 (1~3개월)
- 입출력 처리, 시간복잡도 분석 능력 확보
- 기본 자료구조: 배열, 스택, 큐, 연결 리스트
- 기본 알고리즘: 정렬, 이진 탐색, 브루트포스
- 백준 브론즈~실버 문제 100문제 이상 풀기

[2단계] 핵심 알고리즘 (3~6개월)
- 그래프: DFS, BFS, 다익스트라, 플로이드-워셜
- 동적 프로그래밍: 기초 DP, 배낭 문제, LIS
- 그리디 알고리즘, 분할 정복
- 문자열: KMP, 트라이
- 백준 실버~골드 문제 200문제 이상 풀기

[3단계] 고급 알고리즘 (6~12개월)
- 세그먼트 트리, 펜윅 트리
- 최소 스패닝 트리, 네트워크 플로우
- 고급 DP (비트마스크 DP, 트리 DP)
- 기하 알고리즘, 수학 (정수론, 조합론)
- 백준 골드~플래티넘 문제 도전

[4단계] 대회 실전 (지속)
- 모의 대회 참가 (Codeforces, AtCoder 라운드)
- KOI/USACO/IOI 기출 문제 풀기
- 팀 연습 (ICPC 준비 시)

시스템 프로그래밍에서 C가 사용되는 이유
1. 하드웨어 직접 제어: 포인터로 메모리 주소에 직접 접근할 수 있습니다.
2. 최소한의 런타임 오버헤드: 가비지 컬렉터가 없어 예측 가능한 성능을 보장합니다.
3. 어셈블리와의 친화성: 인라인 어셈블리를 통해 CPU 명령어를 직접 사용할 수 있습니다.
4. 이식성: 거의 모든 하드웨어 플랫폼에 C 컴파일러가 존재합니다.
5. 역사적 유산: 주요 운영체제와 시스템 라이브러리가 이미 C로 작성되어 있습니다.

C 마스터가 된 것을 축하합니다
여러분은 이 교재를 통해 다음을 달성했습니다.

변수, 자료형, 연산자 -- 데이터를 저장하고 계산하는 기초를 익혔습니다.
조건문, 반복문 -- 프로그램의 흐름을 제어하는 방법을 배웠습니다.
배열, 문자열 -- 데이터를 묶어서 다루는 능력을 갖추었습니다.
함수 -- 코드를 모듈화하고 재사용하는 설계력을 키웠습니다.
포인터, 동적 메모리 -- C 언어의 가장 강력하고 어려운 개념을 정복했습니다.
구조체, 파일 입출력 -- 실세계 데이터를 프로그램으로 다루는 능력을 얻었습니다.
자료구조 -- 연결 리스트, 스택, 큐, 트리, 해시를 직접 구현했습니다.
알고리즘 -- 정렬, 탐색, 그래프, DP, 문자열 알고리즘을 학습했습니다.

문제: C++에서 scanf와 cout을 혼용하면 입출력 버퍼 동기화 문제가 발생할 수 있습니다.
출력 순서가 뒤바뀌거나 입력이 씹히는 현상이 나타납니다.
해결: ios_base::sync_with_stdio(false)를 사용하거나,
cin/cout 또는 scanf/printf 중 하나만 일관되게 사용합니다.
에러 2: C 코드에서 C++ 문법 사용
// C 파일(.c)에서 C++ 문법을 사용한 경우
#include

int main(void) {
// bool 타입 (C에서는 stdbool.h 필요)
bool flag = true; // 컴파일 에러!

해결: 파일 확장자를 .cpp로 변경하거나, C99/C11 표준을 사용합니다.
C99에서 bool을 쓰려면 #include 가 필요합니다.

int n;

문제: 시간복잡도를 고려하지 않으면 시간 초과(TLE)가 발생합니다.
일반적으로 1초에 약 1억 번의 연산이 가능합니다.
해결: n = 100,000이면 O(n log n) 이하의 알고리즘을 선택해야 합니다.
문제의 입력 범위를 확인하고 시간복잡도를 먼저 계산합니다.
