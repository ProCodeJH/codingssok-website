콜백 지옥(Callback Hell)
콜백 안에서 또 다른 비동기 작업을 시작하고, 그 콜백 안에서 또 다른 작업을 시작하면 코드가 깊게 중첩됩니다.

read_file("a.txt", function_a);
→ function_a 내부에서 parse_data(data, function_b);
→ function_b 내부에서 save_result(result, function_c);
→ function_c 내부에서 send_response(...);

이를 '콜백 지옥'이라 합니다. 이벤트 루프나 상태 머신 패턴으로 해결할 수 있습니다.

에러 1: 콜백에서 dangling pointer
void start_task(void) {
int local_data = 42;
register_callback(&local_data); /* 지역 변수 주소 전달! */
} /* 함수 종료 시 local_data 소멸 */
/* 나중에 콜백 호출 시 → dangling pointer → 정의되지 않은 동작 */

→ 콜백에 전달하는 데이터는 힙(malloc) 또는 전역/정적 변수로 관리하십시오.
에러 2: 이벤트 루프 무한 대기
while (1) {
if (!dequeue_event(&evt)) continue; /* 이벤트 없으면 CPU 100% 사용 */
}

→ 실무에서는 poll(), select(), epoll() 등으로 이벤트 발생까지 대기(block)합니다.
→ 임베디드에서는 sleep이나 인터럽트 대기를 삽입합니다.
에러 3: 스레드 경쟁 조건(Race Condition)
int counter = 0;
/* 스레드 1 */ counter++; /* 읽기 → 증가 → 쓰기 */
/* 스레드 2 */ counter++; /* 동시에 읽기 → 같은 값 증가 → 쓰기 */
/* 결과: 2가 아닌 1이 될 수 있음! */

→ mtx_lock/mtx_unlock으로 임계 영역을 보호하십시오.
→ 또는 atomic_int와 atomic_fetch_add를 사용하십시오.

ASYNC_END;
}

→ AWAIT를 넘어서 유지해야 하는 변수는 반드시 구조체 멤버로 선언하십시오.
→ 지역 변수는 switch-case 점프 시 초기화 코드를 건너뜁니다.

설명하기

설명하기

마지막으로

C의 유연함
C 표준에는 async/await이 없지만, 매크로와 패턴으로 비동기 프로그래밍이 가능합니다. 이것이 C의 유연함입니다. Python의 async/await, JavaScript의 Promise, Go의 goroutine은 모두 언어 차원에서 비동기를 지원하지만, C는 프로그래머가 직접 메커니즘을 설계합니다. 4개의 매크로만으로 코루틴을 만들 수 있다는 사실은 C가 얼마나 강력한 도구인지를 보여줍니다. 여러분은 이제 비동기 프로그래밍의 본질을 이해했습니다.
용어설명
동기(Synchronous)작업이 완료될 때까지 호출자가 대기하는 방식. blocking이라고도 합니다.
비동기(Asynchronous)작업 요청 후 완료를 기다리지 않고 다음 코드를 실행하는 방식. non-blocking이라고도 합니다.
콜백(Callback)다른 함수에 인자로 전달되어 특정 시점에 호출되는 함수. C에서는 함수 포인터로 구현합니다.
이벤트 루프(Event Loop)이벤트 큐에서 이벤트를 꺼내 등록된 핸들러를 호출하는 무한 루프 구조입니다.
상태 머신(State Machine)유한한 상태 집합과 상태 간 전이 규칙으로 동작을 정의하는 모델입니다.
코루틴(Coroutine)실행을 중단(yield)했다가 재개(resume)할 수 있는 함수. C에서는 매크로로 흉내냅니다.
ProtothreadsAdam Dunkels가 만든 매크로 기반 경량 코루틴 라이브러리. 임베디드 OS인 Contiki에서 사용됩니다.
Duff's Deviceswitch-case의 fall-through를 이용한 루프 최적화 기법. 매크로 코루틴의 이론적 기반입니다.
경쟁 조건(Race Condition)두 이상의 스레드가 공유 자원에 동시 접근하여 결과가 실행 순서에 따라 달라지는 버그입니다.
뮤텍스(Mutex)상호 배제(Mutual Exclusion)를 보장하는 동기화 도구. 한 번에 하나의 스레드만 임계 영역에 진입합니다.
