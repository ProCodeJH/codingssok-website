변수 이름 규칙 4가지

C언어에서 변수 이름을 지을 때 반드시 지켜야 하는 규칙이 4가지 있습니다. 이 규칙을 어기면 컴파일 에러가 발생합니다.

1
규칙 1: 영문자, 숫자, 밑줄(_)만 사용
알파벳 대소문자(a-z, A-Z), 숫자(0-9), 밑줄(_)만 허용됩니다. 공백, 하이픈(-), 특수문자(@, #, $ 등)는 사용할 수 없습니다.
2
규칙 2: 첫 글자는 숫자 불가
영문자 또는 밑줄로 시작해야 합니다. 숫자로 시작할 수 없습니다. score1은 가능하지만 1score는 에러입니다.
3
규칙 3: 예약어 사용 불가
int, return, if, while 등 C언어가 이미 사용하는 단어(예약어)는 변수 이름으로 쓸 수 없습니다.
4
규칙 4: 대소문자 구분
Score, score, SCORE는 모두 다른 변수입니다. C는 대소문자를 엄격하게 구분합니다.

규칙 1: 사용 가능한 문자

이름가능 여부이유
score가능영문자만 사용
my_score가능밑줄(_) 허용
score1가능숫자가 첫 글자가 아님
_count가능밑줄로 시작 가능
my-score불가하이픈(-) 사용 불가
my score불가공백 사용 불가
score@1불가특수문자(@) 사용 불가
점수불가한글 사용 불가
하이픈(-)은 빼기 연산자입니다 my-score를 쓰면 컴파일러는 "my 빼기 score"로 해석합니다.
두 단어를 연결하려면 밑줄을 사용하십시오: my_score

규칙 2: 숫자로 시작 불가

Crule2.c
1int score1 = 90; /* 가능 — 숫자가 뒤에 */
2int score_2 = 85; /* 가능 — 숫자가 뒤에 */
3int 1score = 80; /* 에러 — 숫자로 시작 */
4int 2nd = 75; /* 에러 — 숫자로 시작 */

컴파일러가 숫자로 시작하는 이름을 금지하는 이유가 있습니다. 123을 만나면 컴파일러는 이것이 숫자인지 변수 이름인지 구별할 수 없기 때문입니다. 첫 글자가 영문자 또는 밑줄이면 변수 이름, 숫자이면 숫자 값으로 바로 판단할 수 있습니다.

규칙 3: 예약어(keyword)

C언어에는 미리 정해진 의미를 가진 단어들이 있습니다. 이것을 예약어(keyword) 또는 키워드라고 합니다. 예약어는 변수 이름으로 사용할 수 없습니다.

분류예약어
자료형int, char, float, double, void, short, long, unsigned, signed
제어문if, else, switch, case, default
반복문for, while, do, break, continue
기타return, sizeof, typedef, struct, enum, union, const, static, extern, register, volatile, auto, goto

C89 표준에서 예약어는 총 32개입니다. 이 단어들은 모두 소문자입니다.

Ckeyword_err.c
1int int = 5; /* 에러 — int는 예약어 */
2int return = 10; /* 에러 — return은 예약어 */
3int Int = 5; /* 가능 — 대문자 I (규칙 4) */
예약어를 외울 필요는 없습니다 모든 예약어를 외울 필요는 없습니다. 에디터(VS Code)에서 예약어는 자동으로 색이 바뀝니다. 변수 이름을 입력했는데 색이 바뀌면 예약어이므로 다른 이름을 사용하십시오.

규칙 4: 대소문자 구분

Ccase_sensitive.c
1#include <stdio.h>
2
3int main(void) {
4 int score = 90;
5 int Score = 80;
6 int SCORE = 70;
7
8 printf("score = %d\n", score);
9 printf("Score = %d\n", Score);
10 printf("SCORE = %d\n", SCORE);
11 return 0;
12}
실행 결과
score = 90 Score = 80 SCORE = 70

score, Score, SCORE는 완전히 다른 변수입니다. 같은 이름의 변형을 여러 개 사용하면 혼란스럽기 때문에 실제 코드에서는 이렇게 사용하지 않습니다.

주의: printf의 f도 소문자입니다 Printf, PRINTF로 쓰면 에러가 발생합니다. C의 표준 함수(printf, scanf, main)는 모두 소문자입니다.

좋은 이름 vs 나쁜 이름

규칙만 지키면 컴파일은 됩니다. 하지만 좋은 이름을 지어야 코드를 이해하기 쉽습니다.

DEBUG LAB — 에러를 찾으시오
C
1int a = 1500;
2int b = 3;
3int c = a * b;
4printf("%d\n", c);

에러: a, b, c가 무엇인지 알 수 없습니다

수정: 이름만 보면 의미를 알 수 있습니다

이름 짓기 관례(Convention)

C 프로그래머들이 관례적으로 사용하는 이름 짓기 방식이 있습니다.

방식예시설명
snake_casemy_score, birth_year단어 사이를 밑줄로 연결. C에서 가장 일반적
camelCasemyScore, birthYear두 번째 단어부터 첫 글자를 대문자로. Java, JavaScript에서 주로 사용
UPPER_CASEMAX_SIZE, PI상수(const, #define)에 사용. 모두 대문자 + 밑줄

이 교재에서는 C의 전통적인 방식인 snake_case를 사용합니다. 어떤 방식이든 하나를 골라서 일관되게 사용하는 것이 중요합니다.

좋은 이름의 조건 1. 의미가 명확합니다 — 이름만 보면 역할을 알 수 있습니다
2. 적절한 길이입니다 — 너무 짧지도(x), 너무 길지도(the_total_score_of_student) 않습니다
3. 일관된 규칙을 따릅니다 — snake_case 또는 camelCase 중 하나만 사용합니다
4. 약어를 남용하지 않습니다 — cnt보다 count, num보다 number가 명확합니다

변수 이름 길이

C99 표준에서는 변수 이름의 처음 63글자까지 구별합니다. 실용적으로는 3~20글자 사이가 적당합니다.

유형추천 길이예시
반복 카운터1글자i, j, k, n
일반 변수5~15글자score, total_price, birth_year
상수5~20글자MAX_SIZE, BUFFER_LENGTH

반복문의 카운터(i, j, k)나 수학 공식의 좌표(x, y)처럼 관례적으로 한 글자를 쓰는 경우도 있습니다. 이런 경우를 제외하면 의미 있는 이름을 사용하십시오.

용어 정리

용어설명
식별자(identifier)변수, 함수 등의 이름을 통칭하는 용어입니다. 변수 이름 규칙은 모든 식별자에 동일하게 적용됩니다.
예약어(keyword)C언어가 미리 정해놓은 의미를 가진 단어입니다. int, return, if 등 32개(C89 기준)가 있으며 변수 이름으로 사용할 수 없습니다.
snake_case단어 사이를 밑줄(_)로 연결하는 이름 짓기 방식입니다. my_score, birth_year 등. C에서 가장 일반적입니다.
camelCase두 번째 단어부터 첫 글자를 대문자로 쓰는 방식입니다. myScore, birthYear 등. Java, JavaScript에서 주로 사용합니다.
이름 짓기 관례(naming convention)코드의 가독성을 위해 팀이나 언어 커뮤니티가 합의한 이름 짓기 방식입니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: 식별자 규칙, 예약어 32개(C89), 대소문자 구분
깊이: LAYER 2 — 변수 이름 규칙과 관례
핵심 규칙: 영문자/숫자/밑줄만 사용, 숫자 시작 불가, 예약어 불가, 대소문자 구분
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 추상화(Abstraction) — 이름 짓기
좋은 이름은 코드를 읽는 사람이 의미를 바로 파악할 수 있게 합니다.
a = b * c; → 무엇을 계산하는지 알 수 없음
total = price * quantity; → 가격 × 수량 = 합계

이름을 잘 짓는 것은 추상화의 핵심 기술입니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. 하이픈(-) → 빼기로 해석
2. 숫자 시작 → invalid suffix
3. 예약어 사용 → expected identifier
4. 대소문자 불일치 → undeclared

디버깅 팁: 'undeclared' 에러가 나면 대소문자를 먼저 확인하십시오.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 2 (변수 이름 규칙)
대회에서의 이름 짓기:
- 간결하게: n, m, x, y, ans, res, cnt
- 배열: a[], b[], arr[]
- 반복 변수: i, j, k

대회에서는 시간 제한이 있으므로 짧은 이름을 사용하는 것이 일반적입니다. 하지만 학습 단계에서는 의미 있는 이름을 연습하십시오.
축 5: 코딩 습관 [선택] LAYER 2 — 이름 짓기 습관:
- 영어 단어를 사용합니다 (score, count, price)
- snake_case를 일관되게 사용합니다
- 한 글자 변수는 반복 카운터(i, j)에만 사용합니다
- 비슷한 변수에는 접미사로 구분합니다 (score_1, score_2 또는 korean_score, math_score)
축 8: 타과목 연결 [선택] 수학과의 연결:
수학에서는 미지수를 x, y, z로 표현합니다. 프로그래밍에서도 변수를 사용하지만, 더 구체적인 이름을 붙일 수 있습니다.

수학: S = a × b (넓이 = 가로 × 세로)
C: int area = width * height;

프로그래밍은 수학의 변수 개념을 확장한 것입니다.
