int는 몇 바이트인가

컴퓨터는 모든 데이터를 0과 1(비트, bit)로 저장합니다. 8비트가 모이면 1바이트(byte)입니다.

int는 대부분의 현대 컴퓨터에서 4바이트(32비트)를 사용합니다. 4바이트로 표현할 수 있는 정수의 범위가 정해져 있습니다.

int의 저장 범위 크기: 4바이트 (32비트)
범위: -2,147,483,648 ~ 2,147,483,647
약: -21억 ~ +21억

이 범위를 넘는 값을 저장하면 오버플로(overflow)가 발생합니다.

약 21억이라는 숫자를 기억하십시오. 대부분의 프로그래밍 문제에서 이 범위 안에 있는지 확인해야 합니다.

Csizeof_int.c
1#include <stdio.h>
2
3int main(void) {
4 printf("int의 크기: %d바이트\n", (int)sizeof(int));
5 printf("최대값: %d\n", 2147483647);
6 return 0;
7}
실행 결과
int의 크기: 4바이트 최대값: 2147483647
sizeof 연산자 sizeof(자료형)은 해당 자료형의 크기(바이트 단위)를 알려주는 연산자입니다. sizeof(int)는 4를 반환합니다.

printf("%d %% %d = %d", a, b, a % b); → 17 % 5 = 2

정수 나눗셈 — 소수점은 버려진다

정수끼리 나누면 결과도 정수입니다. 소수점 이하는 버려집니다(내림이 아닌 절삭).

Cint_div.c
1#include <stdio.h>
2
3int main(void) {
4 printf("7 / 2 = %d\n", 7 / 2); /* 3 (3.5가 아님) */
5 printf("10 / 3 = %d\n", 10 / 3); /* 3 (3.33...이 아님) */
6 printf("1 / 2 = %d\n", 1 / 2); /* 0 (0.5가 아님) */
7 printf("5 / 5 = %d\n", 5 / 5); /* 1 */
8 return 0;
9}
실행 결과
7 / 2 = 3 10 / 3 = 3 1 / 2 = 0 5 / 5 = 1
정수 나눗셈 규칙 int / int = int (소수점 이하 절삭)

7 / 2 → 3 (3.5의 소수점 버림)
1 / 2 → 0 (0.5의 소수점 버림)
-7 / 2 → -3 (0을 향해 절삭)

소수점을 유지하려면 float 또는 double 자료형을 사용해야 합니다 (이후 유닛에서 학습).

나머지 연산자 %

% 연산자는 나눗셈의 나머지를 구합니다. 생각보다 자주 사용되는 연산자입니다.

Cmodulo.c
1#include <stdio.h>
2
3int main(void) {
4 printf("10 %% 3 = %d\n", 10 % 3); /* 1 */
5 printf("15 %% 5 = %d\n", 15 % 5); /* 0 */
6 printf("7 %% 10 = %d\n", 7 % 10); /* 7 */
7 return 0;
8}
실행 결과
10 % 3 = 1 15 % 5 = 0 7 % 10 = 7
나머지 연산의 활용 짝수/홀수 판별: n % 2 == 0이면 짝수, 1이면 홀수
배수 판별: n % 3 == 0이면 3의 배수
자릿수 추출: 123 % 10 → 3 (일의 자리)
순환: 시계의 시간 = 총 시간 % 12

나눗셈과 나머지의 관계

나눗셈(/)과 나머지(%)는 항상 다음 관계를 만족합니다.

공식 피제수 = (피제수 / 제수) * 제수 + (피제수 % 제수)

17 = (17 / 5) * 5 + (17 % 5)
17 = 3 * 5 + 2
17 = 15 + 2 ✓
Cdiv_mod_relation.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 17;
5 int b = 5;
6 int quotient = a / b; /* 몫: 3 */
7 int remainder = a % b; /* 나머지: 2 */
8
9 printf("%d / %d = 몫 %d, 나머지 %d\n", a, b, quotient, remainder);
10 printf("검증: %d * %d + %d = %d\n", quotient, b, remainder,
11 quotient * b + remainder);
12 return 0;
13}
실행 결과
17 / 5 = 몫 3, 나머지 2 검증: 3 * 5 + 2 = 17

오버플로 — 범위를 넘으면

int가 저장할 수 있는 최대값은 2,147,483,647(약 21억)입니다. 이 값에 1을 더하면 어떻게 될까요?

Coverflow.c
1#include <stdio.h>
2
3int main(void) {
4 int max = 2147483647;
5 printf("max = %d\n", max);
6 printf("max + 1 = %d\n", max + 1);
7 return 0;
8}
실행 결과
max = 2147483647 max + 1 = -2147483648

2,147,483,647에 1을 더했더니 -2,147,483,648이 되었습니다. 이것을 오버플로(overflow)라고 합니다. 자동차 계기판이 99999에서 00000으로 돌아가는 것과 비슷합니다.

오버플로는 에러 메시지가 없습니다 오버플로가 발생해도 컴파일러는 경고를 주지 않고, 프로그램도 멈추지 않습니다. 그저 잘못된 값이 저장될 뿐입니다.

큰 수를 다루는 문제에서는 int의 범위(±21억)를 항상 확인해야 합니다. 범위가 부족하면 long long(±약 922경)을 사용합니다.

0으로 나누기

정수를 0으로 나누면 런타임 에러가 발생합니다.

Cdiv_zero.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 10;
5 int b = 0;
6 printf("%d\n", a / b); /* 런타임 에러! */
7 return 0;
8}
0으로 나누기 = 프로그램 비정상 종료 컴파일은 정상적으로 완료되지만, 실행 시 프로그램이 즉시 종료됩니다.
Windows: "프로그램이 작동을 멈췄습니다"
Linux: Floating point exception

나누기 전에 제수(나누는 수)가 0이 아닌지 확인하는 것이 좋습니다.
