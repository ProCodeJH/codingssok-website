scanf로 키보드 입력 받기

scanf 함수로 사용자의 키보드 입력을 받아 변수에 저장하는 방법을 학습합니다.

이 유닛에서 배우는 것

  • scanf로 정수, 실수, 문자를 입력받을 수 있습니다.
  • & 연산자가 필요한 이유를 설명할 수 있습니다.
  • scanf의 서식 지정자(%d, %f, %c, %lf)를 올바르게 사용할 수 있습니다.
  • 여러 값을 한 번에 입력받는 방법을 이해합니다.

아래 코드에서 scanf가 하는 역할을 추측하십시오. 사용자가 25를 입력하면 어떤 출력이 나올까요?

printf("나이를 입력하세요: ");

scanf란

printf가 화면에 출력하는 함수라면, scanf는 키보드에서 입력을 받는 함수입니다.

scanf의 기본 형태 scanf("서식 지정자", &변수);

scanf("%d", &age); → 키보드에서 정수를 입력받아 age에 저장

printf와 비슷하지만 두 가지가 다릅니다:
1. 변수 앞에 &를 붙입니다
2. 출력이 아니라 입력입니다

& 연산자 — 왜 필요한가

scanf에서 변수 앞에 &를 붙여야 합니다. &는 "주소 연산자"로, 변수가 저장된 메모리 주소를 의미합니다.

택배를 보내려면 수신 주소가 필요합니다. scanf도 마찬가지로 "어디에 값을 넣을지" 주소를 알아야 합니다. &age는 "age 변수의 주소"를 scanf에게 알려주는 것입니다.

& 를 빠뜨리면 프로그램이 비정상 종료됩니다 scanf("%d", age); → &가 없으면 잘못된 주소에 값을 쓰려고 합니다
scanf("%d", &age); → 올바른 사용

&를 빠뜨리면 컴파일은 되지만 실행 시 세그멘테이션 오류(segmentation fault)가 발생할 수 있습니다. scanf에서 가장 흔한 실수입니다.

여러 값 입력

scanf 한 번으로 여러 값을 입력받을 수 있습니다.

Cscanf_multi.c
1#include <stdio.h>
2
3int main(void) {
4 int a, b;
5
6 printf("두 정수를 입력하세요: ");
7 scanf("%d %d", &a, &b);
8
9 printf("%d + %d = %d\n", a, b, a + b);
10 printf("%d - %d = %d\n", a, b, a - b);
11 printf("%d * %d = %d\n", a, b, a * b);
12 return 0;
13}

사용자는 두 수를 공백(스페이스) 또는 Enter로 구분하여 입력합니다. 예: 10 20 또는 10(Enter)20(Enter)

한 줄에 여러 변수 선언 int a, b;는 int a; int b;와 같습니다. 같은 자료형의 변수를 쉼표로 구분하여 한 줄에 선언할 수 있습니다.

int x, y, z; → int형 변수 3개 선언

용어 정리

용어설명
scanf키보드에서 입력을 받아 변수에 저장하는 C 표준 함수입니다. scan formatted의 약자입니다.
& (주소 연산자)변수의 메모리 주소를 구하는 연산자입니다. scanf에서 값을 저장할 위치를 알려주는 데 사용합니다.
버퍼(buffer)키보드 입력이 임시로 저장되는 공간입니다. Enter를 누르면 버퍼의 내용이 scanf에 전달됩니다.
표준 입력(stdin)프로그램이 입력을 받는 기본 채널입니다. 보통 키보드이지만, 온라인 저지에서는 파일이 연결됩니다.
표준 출력(stdout)프로그램이 출력을 보내는 기본 채널입니다. 보통 화면(터미널)입니다.

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: scanf, & 연산자, 서식 지정자(%d, %lf, %c, %s), 표준 입력(stdin)
깊이: LAYER 2 — 키보드 입력과 변수 저장
핵심 패턴: scanf("%d", &변수); — & 필수, double은 %lf
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 입력-처리-출력(IPO)
모든 프로그램의 기본 구조입니다:
- Input: scanf로 데이터 입력
- Process: 연산으로 처리
- Output: printf로 결과 출력

L2-U05의 프로젝트에서 고정값을 사용했던 것이, 이제 사용자 입력으로 바뀌었습니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. & 누락 → segmentation fault 또는 쓰레기 값
2. double에 %f 사용 → 잘못된 값 저장
3. 잘못된 형식 입력 → 변환 실패
4. Enter가 %c에 들어감 → 공백으로 해결

디버깅 원칙: scanf 에러는 &와 서식 지정자를 먼저 확인합니다.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 3 (입력+연산+출력)
대회 입출력 패턴:
1. 안내 메시지 없이 scanf만 사용
2. 문제에서 지정한 형식대로만 출력
3. 줄 끝에 불필요한 공백 주의
4. 큰 수는 long long + %lld 사용
축 5: 코딩 습관 [선택] LAYER 2 — scanf 습관:
- scanf 후 반드시 & 확인
- double은 항상 %lf로 입력
- 연습용 코드에는 안내 메시지(printf) 포함, 대회 제출 코드에는 제외
- 여러 값 입력 시 scanf 하나로 처리 가능: scanf("%d %d", &a, &b)
축 7: 실생활 연결 [선택] 입력-처리-출력은 모든 시스템의 기본입니다:
- ATM: 카드 삽입(입력) → 잔액 확인(처리) → 화면 표시(출력)
- 검색 엔진: 검색어 입력 → 검색(처리) → 결과 출력
- 계산기: 숫자·연산자 입력 → 계산(처리) → 결과 출력

scanf는 이 패턴의 첫 단계(입력)를 담당합니다.
