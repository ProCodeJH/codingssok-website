연산자 우선순위

수학에서 곱셈·나눗셈이 덧셈·뺄셈보다 먼저 계산되는 것처럼, C에서도 같은 규칙이 적용됩니다.

우선순위연산자예시계산 순서
1 (높음)( )(2+3)*4괄호 안을 먼저
2*, /, %2+3*4곱셈 먼저: 2+12=14
3 (낮음)+, -2+3*4덧셈은 나중에
4 (가장 낮음)=x = 2+3모든 계산 후 대입
Cpriority.c
1#include <stdio.h>
2
3int main(void) {
4 printf("%d\n", 2 + 3 * 4); /* 14 (3*4=12, 2+12=14) */
5 printf("%d\n", (2 + 3) * 4); /* 20 (2+3=5, 5*4=20) */
6 printf("%d\n", 10 - 6 / 2); /* 7 (6/2=3, 10-3=7) */
7 printf("%d\n", (10 - 6) / 2); /* 2 (10-6=4, 4/2=2) */
8 return 0;
9}
실행 결과
14 20 7 2
우선순위 규칙 정리 괄호 > 곱셈·나눗셈·나머지 > 덧셈·뺄셈 > 대입

같은 우선순위면 왼쪽에서 오른쪽으로 계산합니다 (좌결합):
10 - 3 + 2 → (10-3)+2 = 9
20 / 4 * 3 → (20/4)*3 = 15

헷갈리면 괄호를 쓰십시오. 괄호는 항상 가장 먼저 계산됩니다.

같은 우선순위 — 좌결합

같은 우선순위의 연산자가 연속되면 왼쪽에서 오른쪽으로 계산합니다.

Cleft_assoc.c
1#include <stdio.h>
2
3int main(void) {
4 printf("%d\n", 100 / 10 / 2); /* (100/10)/2 = 5 */
5 printf("%d\n", 100 - 30 - 20); /* (100-30)-20 = 50 */
6 printf("%d\n", 100 / 10 * 3); /* (100/10)*3 = 30 */
7 return 0;
8}
실행 결과
5 50 30
대입 연산자는 우결합 = 연산자만 오른쪽에서 왼쪽으로 결합합니다:

a = b = c = 5; → a = (b = (c = 5)); → c, b, a 모두 5

지금은 이 패턴을 자주 사용하지 않으므로, 왼쪽에서 오른쪽이 기본이라는 점만 기억하면 됩니다.

괄호로 순서 제어

우선순위를 바꾸고 싶을 때는 괄호를 사용합니다. 가독성을 위해 불필요한 괄호를 추가하는 것도 좋은 습관입니다.

Cparentheses.c
1#include <stdio.h>
2
3int main(void) {
4 int a, b, c;
5 scanf("%d %d %d", &a, &b, &c);
6
7 /* 평균 계산 — 괄호 필수 */
8 int avg = (a + b + c) / 3;
9
10 /* 괄호가 없으면: a + b + (c / 3) 이 됨 */
11 int wrong = a + b + c / 3;

괄호가 필요한 대표적 상황 1. 평균: (a + b + c) / 3
2. 온도 변환: (fahrenheit - 32) * 5 / 9
3. 총 금액: price * (100 - discount) / 100

괄호 없이 쓰면 나눗셈/곱셈이 먼저 계산되어 의도와 다른 결과가 나옵니다.

정수 나눗셈 활용 패턴

정수 나눗셈의 절삭 특성을 활용하면 다양한 문제를 풀 수 있습니다.

패턴수식예시
일의 자리 제거n / 101234 / 10 = 123
일의 자리 추출n % 101234 % 10 = 4
십의 자리 추출n / 10 % 101234 / 10 % 10 = 3
백의 자리 추출n / 100 % 101234 / 100 % 10 = 2
올림 나눗셈(a + b - 1) / b7을 3으로 올림: (7+2)/3 = 3
N의 배수로 반올림(n + N/2) / N * N57을 10 단위: (57+5)/10*10 = 60

7 / 3 = 2 (내림)
(7 + 3 - 1) / 3 = 9 / 3 = 3 (올림)

9 / 3 = 3 (딱 맞으면 그대로)
(9 + 3 - 1) / 3 = 11 / 3 = 3 (올림도 동일)

곱셈 먼저 원칙 (정밀도)

1 / 3 * 100 = 0 * 100 = 0 ✗

용어 정리

용어설명
연산자 우선순위(operator precedence)여러 연산자가 있을 때 어떤 것을 먼저 계산할지 정하는 규칙입니다. 곱셈·나눗셈이 덧셈·뺄셈보다 우선합니다.
좌결합(left associativity)같은 우선순위의 연산자가 연속될 때 왼쪽에서 오른쪽으로 계산하는 규칙입니다.
올림 나눗셈(ceiling division)정수 나눗셈에서 나머지가 있으면 몫을 1 올리는 계산입니다. (a+b-1)/b 공식을 사용합니다.
절삭(truncation)정수 나눗셈에서 소수점 이하를 버리는 것입니다. 0을 향해 절삭합니다.
트레이스(trace)수식이나 프로그램의 실행 과정을 한 단계씩 추적하는 것입니다.
