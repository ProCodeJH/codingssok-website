% 연산자 복습

% 연산자는 나눗셈의 나머지를 구합니다. L2-U03에서 기초를 배웠고, 이번 유닛에서는 활용 패턴을 깊이 학습합니다.

수식계산 과정결과
10 % 310 ÷ 3 = 몫 3, 나머지 11
15 % 515 ÷ 5 = 몫 3, 나머지 00
7 % 107 ÷ 10 = 몫 0, 나머지 77
0 % 50 ÷ 5 = 몫 0, 나머지 00
3 % 33 ÷ 3 = 몫 1, 나머지 00
% 연산의 핵심 성질 1. a % b의 결과는 항상 0 이상, b 미만입니다 (a ≥ 0, b > 0일 때)
2. a % b == 0이면 a는 b의 배수입니다
3. a < b이면 a % b == a입니다 (7 % 10 = 7)

활용 1: 짝수/홀수 판별

정수를 2로 나눈 나머지가 0이면 짝수, 1이면 홀수입니다.

Ceven_odd.c
1#include <stdio.h>
2
3int main(void) {
4 int n;
5 scanf("%d", &n);
6
7 int remainder = n % 2;
8 printf("%d %% 2 = %d\n", n, remainder);
9
10 /* 0이면 짝수, 1이면 홀수 */
11 printf("%d은(는) ", n);
12 if (remainder == 0) {
13 printf("짝수입니다.\n");
14 } else {
15 printf("홀수입니다.\n");
16 }
17 return 0;
18}
if문은 아직 배우지 않았습니다 if문은 Layer 3에서 학습합니다. 지금은 "n % 2 == 0이면 짝수"라는 판별 원리만 이해하면 됩니다.

활용 2: 배수 판별

n % k == 0이면 n은 k의 배수입니다. 이 원리로 어떤 수의 배수 여부를 판별할 수 있습니다.

Cmultiple.c
1#include <stdio.h>
2
3int main(void) {
4 int n;
5 scanf("%d", &n);
6
7 printf("%d %% 2 = %d\n", n, n % 2); /* 0이면 2의 배수 */
8 printf("%d %% 3 = %d\n", n, n % 3); /* 0이면 3의 배수 */
9 printf("%d %% 5 = %d\n", n, n % 5); /* 0이면 5의 배수 */
10 printf("%d %% 7 = %d\n", n, n % 7); /* 0이면 7의 배수 */
11 return 0;
12}
배수 판별 공식 n % k == 0 → n은 k의 배수

12 % 3 == 0 → 12는 3의 배수 ✓
12 % 5 == 2 → 12는 5의 배수 ✗
15 % 5 == 0 → 15는 5의 배수 ✓
15 % 3 == 0 → 15는 3의 배수 ✓ (3과 5의 공배수)

활용 3: 순환 (시계, 요일)

% 연산은 값을 특정 범위 안에서 순환시킵니다. 시계가 12시 다음에 1시로 돌아가는 것과 같은 원리입니다.

Cclock.c
1#include <stdio.h>
2
3int main(void) {
4 /* 시계: 0~23 범위 순환 */
5 int hour = 22;
6 printf("%d시의 3시간 후: %d시\n", hour, (hour + 3) % 24);
7 printf("%d시의 5시간 후: %d시\n", hour, (hour + 5) % 24);
8
9 /* 요일: 0(일)~6(토) 순환 */
10 /* 오늘이 수요일(3)이면 10일 후는? */
11 int today = 3; /* 수요일 */
12 int after_10 = (today + 10) % 7;
13 printf("수요일의 10일 후: %d (0=일,1=월,...6=토)\n", after_10);
14 return 0;
15}
실행 결과
22시의 3시간 후: 1시 22시의 5시간 후: 3시 수요일의 10일 후: 6 (0=일,1=월,...6=토)
상황수식결과
22시 + 3시간(22+3) % 24 = 25 % 241시
22시 + 5시간(22+5) % 24 = 27 % 243시
수요일(3) + 10일(3+10) % 7 = 13 % 76 (토요일)
순환 공식 (현재값 + 증가량) % 전체범위

시계: (hour + n) % 24
분: (minute + n) % 60
요일: (day + n) % 7
월: (month + n - 1) % 12 + 1

활용 4: 자릿수 분리

% 10으로 마지막 자리를, / 10으로 마지막 자리를 제거합니다. 이 패턴을 반복하면 모든 자릿수를 분리할 수 있습니다.

Cdigits.c
1#include <stdio.h>
2
3int main(void) {
4 int n;
5 scanf("%d", &n);
6
7 int d0 = n % 10; /* 일의 자리 */
8 int d1 = n / 10 % 10; /* 십의 자리 */
9 int d2 = n / 100 % 10; /* 백의 자리 */
10 int d3 = n / 1000 % 10; /* 천의 자리 */
11
12 printf("천:%d 백:%d 십:%d 일:%d\n", d3, d2, d1, d0);
13 printf("자릿수 합: %d\n", d3 + d2 + d1 + d0);
14 printf("역순: %d%d%d%d\n", d0, d1, d2, d3);
15 return 0;
16}
자릿수 분리 패턴 n = 5678일 때:

n % 10 = 8 (일의 자리)
n / 10 % 10 = 7 (십의 자리)
n / 100 % 10 = 6 (백의 자리)
n / 1000 % 10 = 5 (천의 자리)
n / 1000 = 5 (천의 자리, 맨 앞자리라면 % 불필요)

물건을 묶음으로 판매할 때, 필요한 묶음 수와 낱개 수를 구하는 문제입니다.

에러 1: 0으로 나머지 연산 나눗셈(/)과 마찬가지로 나머지(%) 연산에서도 0으로 나누면 런타임 에러가 발생합니다.

% 연산 전에 제수가 0이 아닌지 확인해야 합니다.
Cerr2.c
1#include <stdio.h>
2
3int main(void) {
4 printf("%d\n", -7 % 3); /* 결과는? */
5 printf("%d\n", 7 % -3); /* 결과는? */
6 return 0;
7}
에러 2: 음수의 나머지 C99에서 나머지의 부호는 피제수(왼쪽)의 부호를 따릅니다:

-7 % 3 = -1 (피제수 -7이 음수이므로 결과도 음수)
7 % -3 = 1 (피제수 7이 양수이므로 결과도 양수)

대회 문제에서는 보통 양수만 다루므로 크게 걱정할 필요는 없지만, 음수가 나올 수 있는 경우 주의하십시오.
Cerr3.c
1#include <stdio.h>
2
3int main(void) {
4 /* 실수에는 % 사용 불가 */
5 printf("%f\n", 7.5 % 2.5); /* 에러! */
6 return 0;
7}
에러 3: 실수에 % 사용 에러: error: invalid operands to binary %
원인: % 연산자는 정수에만 사용할 수 있습니다. float, double에는 사용할 수 없습니다.
대안: 실수의 나머지가 필요하면 fmod() 함수를 사용합니다 (math.h).

용어 정리

용어설명
나머지 연산(modulo operation)% 연산자로 나눗셈의 나머지를 구하는 연산입니다. 정수에만 사용할 수 있습니다.
배수(multiple)어떤 수로 나누어떨어지는 수입니다. n % k == 0이면 n은 k의 배수입니다.
순환(cycle)값이 일정 범위를 넘으면 처음으로 돌아가는 패턴입니다. % 연산으로 구현합니다.
자릿수 분리(digit extraction)숫자에서 특정 자릿수를 추출하는 것입니다. % 10과 / 10을 조합합니다.

Unit 08 핵심 정리

3. 실수에 % → 컴파일 에러
4. 자릿수 추출 순서 혼동 → 트레이스로 확인

- 곱셈 과정: 2588 — % 10으로 자릿수 추출

축 5: 코딩 습관 [선택] LAYER 2 — 나머지 연산 습관:

- 자릿수 추출 시 변수에 저장하여 재사용합니다
- 음수가 나올 수 있는 경우 (result + N) % N 패턴을 사용합니다
축 9: 수학 연결 [선택] 수학의 합동(congruence):
a ≡ b (mod n)은 "a를 n으로 나눈 나머지가 b와 같다"는 뜻입니다.

17 ≡ 2 (mod 5) — 17 % 5 = 2
25 ≡ 1 (mod 24) — 25 % 24 = 1

나머지 연산의 성질:
(a+b) % n = ((a%n) + (b%n)) % n
(a×b) % n = ((a%n) × (b%n)) % n

백준 10430이 바로 이 성질을 확인하는 문제입니다.
