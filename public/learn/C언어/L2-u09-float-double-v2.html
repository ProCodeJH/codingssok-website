왜 실수형이 필요한가

int로는 소수점이 있는 값을 저장할 수 없습니다. 원주율(3.14159), 온도(36.5), 환율(1,328.50) 등을 다루려면 실수형이 필요합니다.

Cwhy_float.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 3.14; /* 3만 저장됨! 소수점 버려짐 */
5 double b = 3.14; /* 3.14 정확히 저장 */
6
7 printf("int: %d\n", a); /* 3 */
8 printf("double: %f\n", b); /* 3.140000 */
9 return 0;
10}
실행 결과
int: 3 double: 3.140000

float vs double

항목floatdouble
크기4바이트 (32비트)8바이트 (64비트)
유효 자릿수약 7자리약 15자리
범위약 ±3.4 × 10^38약 ±1.7 × 10^308
scanf 서식%f%lf
printf 서식%f%f
접미사3.14f3.14 (기본)
double을 사용하십시오 C에서 소수점 숫자(3.14)는 기본적으로 double로 취급됩니다.

float은 정밀도가 낮아(7자리) 계산 오차가 쉽게 발생합니다. 특별한 이유가 없으면 항상 double을 사용하십시오.

대부분의 대회 문제와 실무 코드에서도 double이 표준입니다.

double 선언과 입출력

Cdouble_io.c
1#include <stdio.h>

6
7 printf("반지름을 입력하세요: ");

scanf와 printf의 서식 차이 (복습) scanf: double → %lf, float → %f
printf: double, float 모두 → %f

scanf에서 double에 %f를 쓰면 값이 제대로 저장되지 않습니다. 이것은 가장 흔한 실수입니다.

실수 나눗셈 — 소수점 유지

정수끼리 나누면 소수점이 버려지지만, 한쪽이라도 실수이면 실수 나눗셈이 됩니다.

Creal_div.c
1#include <stdio.h>
2
3int main(void) {
4 printf("int / int: %d\n", 7 / 2); /* 3 */
5 printf("double / double: %f\n", 7.0 / 2.0); /* 3.500000 */
6 printf("double / int: %f\n", 7.0 / 2); /* 3.500000 */
7 printf("int / double: %f\n", 7 / 2.0); /* 3.500000 */
8 return 0;
9}
실행 결과
int / int: 3 double / double: 3.500000 double / int: 3.500000 int / double: 3.500000
혼합 연산의 규칙 정수와 실수가 섞인 연산에서는 정수가 자동으로 실수로 변환됩니다.

int ◯ double → double

7 / 2 → int / int = 3 (정수)
7.0 / 2 → double / int = 3.5 (실수)
7 / 2.0 → int / double = 3.5 (실수)

한쪽만 .0을 붙여도 실수 나눗셈이 됩니다.

정수를 실수로 나누기 — 형변환

두 int 변수를 나눌 때 실수 결과가 필요하면 형변환(casting)을 사용합니다.

Ccasting.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 7, b = 2;
5
6 /* 방법 1: 변수 하나를 double로 캐스팅 */
7 printf("%.1f\n", (double)a / b); /* 3.5 */
8
9 /* 방법 2: 1.0을 곱하기 */
10 printf("%.1f\n", a * 1.0 / b); /* 3.5 */
11
12 /* 주의: 이미 나눈 결과를 캐스팅하면 늦음 */
13 printf("%.1f\n", (double)(a / b)); /* 3.0! */
14 return 0;
15}
실행 결과
3.5 3.5 3.0
캐스팅 위치에 주의 (double)a / b → a를 double로 변환한 뒤 나눗셈 → 3.5 ✓
(double)(a / b) → 먼저 int 나눗셈(3) 후 double로 변환 → 3.0 ✗

캐스팅은 나눗셈 전에 해야 합니다.

실수의 정밀도 문제

컴퓨터는 실수를 이진수(0과 1)로 저장합니다. 0.1처럼 10진수로 깔끔한 수도 이진수로는 무한소수가 됩니다. 이 때문에 미세한 오차가 발생합니다.

Cprecision.c
1#include <stdio.h>
2
3int main(void) {
4 double a = 0.1 + 0.2;
5 printf("0.1 + 0.2 = %.20f\n", a);
6 printf("0.3과 같은가? %d\n", a == 0.3); /* 0! */
7 return 0;
8}
실행 결과
0.1 + 0.2 = 0.30000000000000004441 0.3과 같은가? 0

0.1 + 0.2가 정확히 0.3이 아닙니다. 소수점 17번째 자리에서 미세한 오차가 있습니다.

실수는 == 로 비교하지 마십시오 부동소수점 오차 때문에 실수를 == 로 비교하면 예상과 다른 결과가 나올 수 있습니다.

잘못된 방법: if (a == 0.3)
올바른 방법: if (fabs(a - 0.3) < 0.000001)

fabs()는 절대값 함수(math.h)입니다. 두 값의 차이가 충분히 작으면 같다고 판단합니다. (if문은 Layer 3에서 학습)
대회에서의 실수 비교 대회 문제에서 "오차 10^-6 이내"라는 조건이 자주 나옵니다. 이것은 소수점 6자리까지 맞으면 정답으로 인정한다는 의미입니다.

printf("%.6f", answer); 로 출력하면 대부분 통과합니다.

용어 정리

용어설명
부동소수점(floating point)소수점의 위치가 고정되지 않고 떠다니는(floating) 방식으로 실수를 표현하는 방법입니다.
형변환(type casting)자료형을 다른 자료형으로 변환하는 것입니다. (double)n처럼 괄호 안에 자료형을 씁니다.
암시적 형변환(implicit conversion)프로그래머가 명시하지 않아도 컴파일러가 자동으로 수행하는 형변환입니다. int + double → double이 대표적입니다.
명시적 형변환(explicit conversion)(double)a처럼 프로그래머가 직접 지정하는 형변환입니다. 캐스팅(casting)이라고도 합니다.
유효 자릿수(significant digits)정밀하게 표현할 수 있는 자릿수입니다. float은 약 7자리, double은 약 15자리입니다.
