단위 변환기

실수 연산을 활용하여 길이, 온도, 무게, 속도 등 다양한 단위 변환 프로그램을 만듭니다.

이 유닛에서 배우는 것

  • 실수 연산으로 단위 변환 공식을 구현할 수 있습니다.
  • 여러 단위 변환을 하나의 프로그램에 통합할 수 있습니다.
  • scanf와 printf를 조합하여 사용자 친화적인 출력을 할 수 있습니다.
  • 점진적 개발로 프로그램을 확장하는 경험을 쌓습니다.

변환공식예시

9.0 / 5.0 vs 9 / 5 celsius * 9 / 5 + 32 → 9/5 = 1 (정수 나눗셈!) → 틀린 결과
celsius * 9.0 / 5.0 + 32 → 9.0/5.0 = 1.8 (실수 나눗셈) → 올바른 결과

변수를 아껴 쓰는 것이 항상 좋은 것은 아닙니다. 여러 값을 동시에 비교해야 할 때는 각각 다른 변수에 저장해야 합니다.

에러 체험

단위 변환에서 자주 발생하는 실수를 경험합니다.

Cerr1.c
1#include <stdio.h>
2
3int main(void) {
4 double celsius = 100;
5 double fahrenheit = celsius * 9 / 5 + 32; /* 9/5=1! */
6 printf("%.2f\n", fahrenheit); /* 132.00 (틀림!) */
7 return 0;
8}
에러 1: 정수 나눗셈 혼입 celsius * 9 / 5에서 9/5 = 1 (정수 나눗셈)이 먼저 계산될 수 있습니다.

실제로 C에서는 왼쪽부터 계산하므로 celsius*9=900.0(double), 900.0/5=180.0이 되어 결과적으로 맞지만, 9/5를 먼저 쓰면 틀립니다:
celsius * (9/5) = 100 * 1 = 100

안전한 방법: 항상 9.0 / 5.0으로 쓰십시오.
Cerr2.c
1#include <stdio.h>
2
3int main(void) {
4 int height_cm = 175;
5 double height_m = height_cm / 100; /* 정수 나눗셈! */
6 printf("%.2f m\n", height_m); /* 1.00 (틀림!) */
7 return 0;
8}
에러 2: cm→m 변환 시 정수 나눗셈 175 / 100 = 1 (정수 나눗셈) → double에 저장해도 1.00

수정: height_cm / 100.0 또는 (double)height_cm / 100

용어 정리

용어설명
단위 변환(unit conversion)같은 양을 다른 단위로 표현하는 것입니다. 예: 170cm = 66.93inch.
변환 계수(conversion factor)단위를 변환할 때 곱하거나 나누는 상수입니다. 1inch = 2.54cm에서 2.54가 변환 계수입니다.
점진적 개발(incremental development)간단한 버전에서 시작하여 기능을 단계적으로 추가하는 개발 방법입니다.
