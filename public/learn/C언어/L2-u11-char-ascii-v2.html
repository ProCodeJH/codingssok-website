char 자료형

char는 문자(character) 하나를 저장하는 자료형입니다. 크기는 1바이트(8비트)입니다.

Cchar_basic.c
1#include <stdio.h>
2
3int main(void) {
4 char grade = 'A';
5 char initial = 'K';
6 char digit = '7';
7 char symbol = '#';
8
9 printf("학점: %c\n", grade);
10 printf("이니셜: %c\n", initial);
11 printf("숫자 문자: %c\n", digit);
12 printf("기호: %c\n", symbol);
13 return 0;
14}
실행 결과
학점: A 이니셜: K 숫자 문자: 7 기호: #
char 사용 규칙 1. 작은따옴표로 감쌉니다: 'A', 'z', '3', '!'
2. 문자 하나만 저장합니다: 'AB'는 불가
3. 서식 지정자: printf → %c, scanf → %c (또는 " %c")
4. 크기: 1바이트 (8비트)

ASCII 코드 — 문자는 숫자다

컴퓨터는 문자를 직접 저장하지 않습니다. 대신 각 문자에 번호를 매겨 그 번호(정수)를 저장합니다. 이 번호 체계가 ASCII(American Standard Code for Information Interchange)입니다.

Cascii_demo.c
1#include <stdio.h>
2
3int main(void) {
4 char ch = 'A';
5
6 printf("문자: %c\n", ch); /* A */
7 printf("ASCII: %d\n", ch); /* 65 */
8
9 printf("문자: %c\n", 66); /* B */
10 printf("ASCII: %d\n", 'B'); /* 66 */
11 return 0;
12}
실행 결과
문자: A ASCII: 65 문자: B ASCII: 66

'A'를 %c로 출력하면 문자 A, %d로 출력하면 숫자 65가 나옵니다. 저장된 값은 같지만 해석 방식이 다릅니다.

주요 ASCII 코드

문자코드문자코드문자코드
'0'48'A'65'a'97
'1'49'B'66'b'98
'2'50'C'67'c'99
..................
'9'57'Z'90'z'122
' '(공백)32'!'33'\n'10
반드시 기억할 3가지 '0' = 48 — 숫자 문자의 시작
'A' = 65 — 대문자의 시작
'a' = 97 — 소문자의 시작

대문자와 소문자의 차이: 'a' - 'A' = 32
숫자 문자와 실제 숫자의 차이: '0' = 48 (즉, '5' - '0' = 5)

문자 연산

char는 내부적으로 정수이므로 산술 연산이 가능합니다.

Cchar_math.c
1#include <stdio.h>
2
3int main(void) {
4 char ch = 'A';
5
6 printf("%c\n", ch); /* A */
7 printf("%c\n", ch + 1); /* B (65+1=66) */
8 printf("%c\n", ch + 2); /* C (65+2=67) */
9 printf("%c\n", ch + 25); /* Z (65+25=90) */
10
11 /* 대문자 → 소문자 */
12 printf("%c\n", ch + 32); /* a (65+32=97) */
13
14 /* 소문자 → 대문자 */
15 char lower = 'g';
16 printf("%c\n", lower - 32); /* G (103-32=71) */
17 return 0;
18}
실행 결과
A B C Z a G
문자 연산 패턴 다음 문자: ch + 1 ('A'+1 = 'B')
대→소: ch + 32 ('A'+32 = 'a')
소→대: ch - 32 ('a'-32 = 'A')
숫자문자→정수: ch - '0' ('5'-'0' = 5)
정수→숫자문자: n + '0' (5+'0' = '5')

숫자 문자와 정수의 변환

문자 '5'와 정수 5는 다릅니다. '5'의 ASCII 코드는 53입니다.

Cdigit_convert.c
1#include <stdio.h>
2
3int main(void) {
4 char digit_char = '7';
5 int digit_num = digit_char - '0'; /* '7'-'0' = 55-48 = 7 */
6
7 printf("문자: %c\n", digit_char); /* 7 */
8 printf("ASCII: %d\n", digit_char); /* 55 */
9 printf("숫자: %d\n", digit_num); /* 7 */
10
11 /* 역방향: 정수 → 숫자 문자 */
12 int num = 3;
13 char ch = num + '0'; /* 3+48=51 → '3' */
14 printf("문자: %c\n", ch); /* 3 */
15 return 0;
16}
실행 결과
문자: 7 ASCII: 55 숫자: 7 문자: 3
'5'는 5가 아닙니다 '5' — 문자. ASCII 코드 53. %c로 출력하면 5, %d로 출력하면 53
5 — 정수. 숫자 그 자체. %d로 출력하면 5

변환: 숫자값 = 문자 - '0'
'5' - '0' = 53 - 48 = 5

char 입력 — scanf와 getchar

Cchar_input.c
1#include <stdio.h>
2
3int main(void) {
4 char ch;
5
6 printf("문자를 입력하세요: ");
7 scanf(" %c", &ch); /* 공백+%c: Enter 건너뛰기 */
8
9 printf("입력한 문자: %c\n", ch);
10 printf("ASCII 코드: %d\n", ch);
11
12 /* 대문자면 소문자로, 소문자면 대문자로 */
13 if (ch >= 'A' && ch <= 'Z') {
14 printf("소문자: %c\n", ch + 32);
15 } else if (ch >= 'a' && ch <= 'z') {
16 printf("대문자: %c\n", ch - 32);
17 }
18 return 0;
19}
scanf(" %c") — 공백 주의 scanf(" %c", &ch); — %c 앞에 공백 한 칸을 넣으면 이전 입력에서 남은 Enter(\n)를 건너뜁니다.

공백 없이 scanf("%c", &ch);를 쓰면 이전 Enter가 문자로 읽힐 수 있습니다.

char와 int의 관계

char는 사실 1바이트 크기의 정수형입니다. 문자를 저장하는 용도로 사용하지만, 작은 정수를 저장하는 데도 쓸 수 있습니다.

자료형크기범위용도
char1바이트-128 ~ 127문자, 작은 정수
int4바이트약 ±21억일반 정수
short2바이트-32768 ~ 32767작은 정수
long4~8바이트±21억 이상큰 정수
long long8바이트약 ±922경매우 큰 정수
Cchar_int.c
1#include <stdio.h>
2
3int main(void) {
4 char a = 65; /* 숫자로 저장해도 */
5 int b = 'A'; /* 문자로 저장해도 */
6
7 printf("%c %c\n", a, b); /* A A — 같은 값 */
8 printf("%d %d\n", a, b); /* 65 65 — 같은 값 */

10}

A A 65 65

에러 체험

char 사용 시 자주 발생하는 실수를 경험합니다.

Cerr1.c
1#include <stdio.h>
2
3int main(void) {

에러 1: 큰따옴표 사용 경고: warning: initialization of 'char' from 'char *'
원인: "A"는 문자열(char *)이고, 'A'가 문자(char)입니다.
수정: char ch = 'A';
Cerr2.c
1#include <stdio.h>
2

4 char ch = 'AB'; /* 문자 2개! */
5 printf("%c\n", ch);
6 return 0;
7}
에러 2: 여러 문자 저장 시도 경고: warning: multi-character character constant
원인: char는 문자 하나만 저장합니다. 'AB'는 불가합니다.
수정: 문자열을 저장하려면 char 배열을 사용합니다 (이후 학습).
Cerr3.c
1#include <stdio.h>
2
3int main(void) {
4 char ch = '5';
5 int num = ch; /* 5가 아닌 53! */
6 printf("%d\n", num);

에러 3: 숫자 문자를 정수로 착각 '5'의 ASCII 코드는 53이므로 num에 53이 저장됩니다.

수정: int num = ch - '0'; → 53 - 48 = 5

10축 매핑

이번 유닛에서 배운 내용을 10개의 학습 축으로 연결합니다.

축 1: C언어 [필수] 이 유닛의 C 문법: char(1바이트), ASCII 코드, 문자 연산, %c 서식, 문자↔정수 변환
깊이: LAYER 2 — 문자형과 ASCII 코드 체계
핵심: char는 1바이트 정수형. 'A'=65, 대소문자 차이=32, 숫자 문자 차이='0'
축 2: 컴퓨팅 사고력 [필수] 사고 패턴: 추상화(Abstraction) — 인코딩
문자를 숫자로 표현하는 것은 인코딩(encoding)의 가장 기본적인 형태입니다.

인코딩: 'A' → 65 (문자를 숫자로)
디코딩: 65 → 'A' (숫자를 문자로)

모든 디지털 데이터(이미지, 소리, 영상)도 결국 숫자로 인코딩됩니다.
축 3: 디버깅 [필수] 이 유닛의 에러:
1. 큰따옴표("A") vs 작은따옴표('A') 혼동
2. 여러 문자('AB') 저장 시도
3. '5'를 정수 5로 착각 (실제 53)
4. scanf에서 Enter가 %c에 들어감

디버깅 팁: 문자 변수를 %c와 %d 모두로 출력하여 확인하십시오.
축 4: KOI — 정보올림피아드 [필수] 난이도: TIER 3 (문자 처리)
대회 필수 패턴:
- 백준 11654(ASCII) — 기본 문자→숫자 변환
- 대소문자 변환: ±32
- 문자→숫자: -'0'
- 시저 암호: (ch - 'A' + shift) % 26 + 'A'
축 5: 코딩 습관 [선택] LAYER 2 — char 사용 습관:
- 문자는 작은따옴표: 'A', 'z', '0'
- 매직 넘버(32, 48, 65) 대신 'a'-'A', '0' 등 문자 표현 사용
- scanf(" %c")로 Enter 문제 방지
축 8: 타과목 연결 [선택] 사회/역사와의 연결:
ASCII는 1963년 미국에서 만들어진 표준입니다. 영어 문자만 포함하므로 한글, 한자 등은 표현할 수 없습니다.

이 한계를 해결하기 위해:
- EUC-KR: 한글 전용 인코딩
- Unicode(UTF-8): 전 세계 모든 문자를 표현

현재는 UTF-8이 국제 표준으로 사용됩니다.
