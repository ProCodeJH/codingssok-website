형변환(Type Casting)

서로 다른 자료형 간의 자동 변환(암시적)과 프로그래머가 직접 지시하는 강제 변환(명시적)을 학습합니다.

이 유닛에서 배우는 것

  • 암시적 형변환(자동 승격)이 일어나는 규칙을 설명할 수 있습니다.
  • 명시적 형변환(캐스팅)의 문법과 용도를 알고 사용할 수 있습니다.
  • 정수 나눗셈에서 소수점 결과를 얻기 위한 캐스팅을 적용할 수 있습니다.
  • 형변환 시 데이터 손실이 발생하는 경우를 식별할 수 있습니다.

핵심 개념

암시적 형변환 (자동 승격)
작은 자료형 → 큰 자료형으로 자동 변환됩니다.
char → short → int → long → long long → float → double

데이터 손실이 없으므로 컴파일러가 자동으로 처리합니다.
C
1#include <stdio.h>
2int main(void) {
3 int a = 10;
4 double b = 3.5;
5
6 // a가 자동으로 double(10.0)로 변환된 후 계산
7 double result = a + b; // 10.0 + 3.5 = 13.5
8 printf("%f\n", result); // 13.500000
9
10 return 0;
11}
실행 결과
13.500000

undefined

명시적 형변환 (캐스팅)
(변환할_자료형)값

프로그래머가 직접 자료형 변환을 지시합니다.
예: (double)a → a를 double로 변환
(int)3.7 → 3.7을 int로 변환 (소수점 버림 → 3)
C
1#include <stdio.h>
2int main(void) {
3 int a = 7, b = 2;
4
5 // 방법 1: 캐스팅 없이 (정수 나눗셈)
6 printf("정수 나눗셈: %d\n", a / b); // 3
7
8 // 방법 2: 피연산자 하나를 double로 캐스팅
9 printf("캐스팅 후: %f\n", (double)a / b); // 3.500000
10
11 // 방법 3: 결과를 캐스팅 (이미 3이 된 후라 소용없음)
12 printf("늦은 캐스팅: %f\n", (double)(a / b)); // 3.000000
13
14 return 0;
15}
실행 결과
정수 나눗셈: 3 캐스팅 후: 3.500000 늦은 캐스팅: 3.000000
캐스팅 위치가 중요합니다
(double)a / b → 7.0 / 2 → 3.5 ✓ (나누기 전에 변환)
(double)(a / b) → (double)3 → 3.0 ✗ (나눈 후에 변환, 이미 소수점 손실)

데이터 손실에 주의

undefined

C
1#include <stdio.h>
2int main(void) {
3 double pi = 3.141592;
4 int truncated = (int)pi; // 소수점 버림
5
6 printf("원래 값: %f\n", pi); // 3.141592
7 printf("변환 후: %d\n", truncated); // 3
8
9 int big = 100000;
10 short small = (short)big; // short 범위 초과
11 printf("int: %d\n", big); // 100000
12 printf("short: %hd\n", small); // -31072 (오버플로)
13
14 return 0;
15}
실행 결과
원래 값: 3.141592 변환 후: 3 int: 100000 short: -31072
데이터 손실 정리
double → int: 소수점 이하가 버려집니다 (반올림 아님, 버림).
int → short: 범위를 초과하면 오버플로가 발생합니다.
int → char: 0~127 범위만 안전합니다.

실습 — 평균 계산

C
1#include <stdio.h>
2int main(void) {
3 int kor = 85, eng = 92, math = 78;
4 int sum = kor + eng + math;
5
6 // 잘못된 방법: 정수 나눗셈
7 int avg_wrong = sum / 3;
8 printf("잘못된 평균: %d\n", avg_wrong); // 85
9
10 // 올바른 방법: 캐스팅
11 double avg_right = (double)sum / 3;
12 printf("올바른 평균: %.1f\n", avg_right); // 85.0
13
14 // 더 정확한 결과
15 printf("정확한 평균: %.2f\n", (double)sum / 3); // 85.00
16
17 return 0;
18}
실행 결과
잘못된 평균: 85 올바른 평균: 85.0 정확한 평균: 85.00

에러 체험

에러 1: 캐스팅 위치 오류
double avg = (double)(sum / 3);

→ sum / 3이 먼저 정수 나눗셈으로 계산된 후 double로 변환됩니다. 소수점 이하가 이미 사라진 후입니다.
→ 올바른 방법: (double)sum / 3
에러 2: 대입 시 자동 절삭
int x = 3.999;

→ 3.999가 int에 대입되면서 소수점이 버려져 3이 저장됩니다. 반올림이 아닙니다. 컴파일러가 경고를 표시할 수 있습니다.
에러 3: printf 서식과 캐스팅 혼동
int a = 7, b = 2;
printf("%f\n", a / b); // ✗ 정수 나눗셈 결과를 %f로 출력

→ a / b는 정수 3입니다. 정수를 %f로 출력하면 엉뚱한 값이 나옵니다.
→ printf("%f\n", (double)a / b); 로 캐스팅해야 합니다.

요약

핵심 정리
1. 암시적 형변환: 작은 자료형 → 큰 자료형 (자동, 안전)
2. 명시적 형변환: (자료형)값 — 프로그래머가 직접 지시
3. 정수 나눗셈 해결: (double)a / b (나누기 전에 캐스팅)
4. 큰 → 작은 변환: 데이터 손실 주의 (소수점 버림, 오버플로)
5. (double)(a/b)는 이미 늦음 — 캐스팅 위치가 핵심
