실습 — 괄호의 힘

C
1#include <stdio.h>
2int main(void) {
3 int a = 10, b = 3, c = 2;
4
5 printf("a + b * c = %d\n", a + b * c); // 16
6 printf("(a + b) * c = %d\n", (a + b) * c); // 26
7 printf("a * b + c = %d\n", a * b + c); // 32
8 printf("a * (b + c) = %d\n", a * (b + c)); // 50
9 printf("a / b * c = %d\n", a / b * c); // 6
10 printf("a / (b * c) = %d\n", a / (b * c)); // 1
11
12 return 0;
13}
실행 결과
a + b * c = 16 (a + b) * c = 26 a * b + c = 32 a * (b + c) = 50 a / b * c = 6 a / (b * c) = 1

에러 체험

에러 1: 우선순위 착각
int avg = a + b + c / 3;

→ c/3이 먼저 계산됩니다. a + b + (c/3)가 됩니다.
→ 올바른 평균: (a + b + c) / 3
에러 2: 좌→우 결합 착각
int x = 100 / 10 / 2;
// (100/10)/2 = 10/2 = 5 ✓
// 100/(10/2) = 100/5 = 20 ✗ (이렇게 계산되지 않음)

→ 같은 우선순위는 왼쪽부터 계산합니다.
에러 3: 불필요한 복잡성
int y = a + b * c - d / e + f % g;

→ 이렇게 복잡한 수식은 괄호를 사용하거나 중간 변수로 분리하는 것이 좋습니다.
→ 읽기 어려운 코드는 버그를 유발합니다.

요약

핵심 정리
1. 우선순위: () > 단항(++,--,(자료형)) > */% > +- > 대입(=,+=)
2. 같은 우선순위: 좌→우 결합 (단, 대입은 우→좌)
3. 괄호를 쓰면 우선순위를 외울 필요 없음
4. 복잡한 수식은 중간 변수로 분리하는 것이 좋은 습관
5. 캐스팅(자료형)은 단항 연산자 — /보다 먼저 적용
