비교 연산자 6종 정리
| 연산자 | 의미 | 예 | 결과 |
|--------|------|-----|------|
| == | 같다 | 5 == 5 | 1 (참) |
| != | 같지 않다 | 5 != 3 | 1 (참) |
| > | 크다 | 5 > 3 | 1 (참) |
| < | 작다 | 3 < 5 | 1 (참) |
| >= | 크거나 같다 | 5 >= 5 | 1 (참) |
| <= | 작거나 같다 | 5 <= 3 | 0 (거짓) |
C
1#include <stdio.h>
2int main(void) {
3 int a = 10, b = 20;
4
5 // 비교 결과를 변수에 저장할 수 있음
6 int is_equal = (a == b); // 0
7 int is_greater = (a > b); // 0
8 int is_less = (a < b); // 1
9
10 printf("a == b: %d\n", is_equal);
11 printf("a > b: %d\n", is_greater);
12 printf("a < b: %d\n", is_less);
13
14 // 비교 결과를 계산에 사용
15 printf("참의 개수: %d\n", is_equal + is_greater + is_less);
16
17 return 0;
18}
실행 결과
a == b: 0 a > b: 0 a < b: 1 참의 개수: 1

위험: = 와 == 혼동

C
1#include <stdio.h>
2int main(void) {
3 int x = 5;
4
5 // 올바른 비교
6 if (x == 10) {
7 printf("x는 10입니다.\n");
8 }
9
10 // 잘못된 코드 (대입!)
11 if (x = 10) { // x에 10을 대입하고, 10은 참
12 printf("항상 실행됩니다.\n");
13 }
14 // 이 시점에서 x는 10으로 바뀌어 있음!
15 printf("x = %d\n", x); // 10
16
17 return 0;
18}
실행 결과
항상 실행됩니다. x = 10
= vs == 방지법
1. 상수를 왼쪽에 쓰기: if (10 == x) — 실수로 =를 쓰면 컴파일 에러
2. 컴파일러 경고 활용: -Wall 옵션을 켜면 if (x = 10) 경고
3. 습관화: == 를 쓸 때 항상 두 번 확인

실습 — 비교 연산자 활용

C
1#include <stdio.h>
2int main(void) {
3 int a, b;
4 printf("두 정수: ");
5 scanf("%d %d", &a, &b);
6
7 printf("%d == %d → %d\n", a, b, a == b);
8 printf("%d != %d → %d\n", a, b, a != b);
9 printf("%d > %d → %d\n", a, b, a > b);
10 printf("%d < %d → %d\n", a, b, a < b);
11 printf("%d >= %d → %d\n", a, b, a >= b);
12 printf("%d <= %d → %d\n", a, b, a <= b);
13
14 return 0;
15}
실행 결과
두 정수: 7 7 7 == 7 → 1 7 != 7 → 0 7 > 7 → 0 7 < 7 → 0 7 >= 7 → 1 7 <= 7 → 1

에러 체험

에러 1: =를 ==로 쓰기
if (x = 0) { // x에 0 대입, 0은 거짓 → 절대 실행 안 됨
printf("0입니다\n");
}

→ 의도: x가 0인지 확인 / 실제: x에 0을 대입
→ if (x == 0)이 올바릅니다.
에러 2: 수학식 그대로 쓰기
if (1 < x < 10) { // ✗ 수학에서는 맞지만 C에서는 오류
printf("1~10 사이\n");
}

→ C에서 1 < x는 0 또는 1을 반환합니다.
→ 그 다음 (0 또는 1) < 10이 되어 항상 참입니다.
→ if (x > 1 && x < 10)이 올바릅니다.
에러 3: => 또는 =< 사용
if (x => 10) { // ✗ 컴파일 에러
printf("10 이상\n");
}

→ C에서는 >= 와 <= 만 유효합니다. =>와 =<는 없습니다.
