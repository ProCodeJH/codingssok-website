| A | B | A && B |
|---|---|--------|

| 참 | 거짓 | 거짓 |
| 거짓 | 참 | 거짓 |
| 거짓 | 거짓 | 거짓 |
진리표 — OR (||)
| A | B | A || B |
|---|---|--------|
| 참 | 참 | 참 |

| 거짓 | 거짓 | 거짓 |

| A | !A |
|---|----|
| 참 | 거짓 |

단축 평가

undefined

단축 평가 규칙
&&: 왼쪽이 거짓이면 오른쪽을 검사하지 않음 (결과 확정: 거짓)
||: 왼쪽이 참이면 오른쪽을 검사하지 않음 (결과 확정: 참)

활용: if (b != 0 && a / b > 5) — b가 0이면 나눗셈 실행 안 함 (안전)

드모르간의 법칙

드모르간의 법칙
!(A && B) == !A || !B
!(A || B) == !A && !B

예: !(x > 0 && x < 10) == x <= 0 || x >= 10

에러 체험

에러 1: &와 && 혼동
if (a > 0 & b > 0) { // ✗ &는 비트 AND
printf("둘 다 양수\n");
}

→ &는 비트 연산자이고 &&는 논리 연산자입니다.
→ 조건식에서는 반드시 &&를 사용하십시오.
에러 2: |와 || 혼동
if (a == 0 | b == 0) { // ✗ |는 비트 OR
printf("하나가 0\n");
}

→ |는 비트 연산자, ||는 논리 연산자입니다.
→ 비트 연산자는 단축 평가를 하지 않습니다.
에러 3: OR 조건의 오해
if (x == 1 || 2 || 3) { // ✗ 항상 참!
printf("1, 2, 3 중 하나\n");
}

→ 2와 3은 0이 아니므로 항상 참입니다.
→ if (x == 1 || x == 2 || x == 3)이 올바릅니다.
