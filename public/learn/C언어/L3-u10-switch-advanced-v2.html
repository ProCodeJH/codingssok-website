switch vs if-else if 비교

DEBUG LAB — 에러를 찾으시오

에러:

수정:

undefined

에러 체험

에러 1: 실수로 break 누락
case 1:
printf("메뉴 1\n");
// break를 깜빡함
case 2:
printf("메뉴 2\n");
break;

→ 1을 선택하면 "메뉴 1"과 "메뉴 2"가 모두 출력됩니다.
→ 의도적 fall-through가 아니라면 반드시 break를 작성하십시오.
에러 2: case에 변수 사용
int x = 3;
switch (n) {
case x: // 컴파일 에러!
printf("three\n");
break;
}

→ case 값은 반드시 상수(리터럴 또는 const)여야 합니다.
→ case 3: 으로 작성하십시오.
에러 3: 실수형 switch
double d = 1.5;
switch (d) { // 컴파일 에러!
case 1.5: ...
}

→ switch는 정수형과 문자형만 사용할 수 있습니다.
→ 실수 비교는 if-else if를 사용하십시오.

요약

핵심 정리
1. fall-through: break 없으면 아래 case로 계속 실행
2. 의도적 fall-through: 여러 case를 묶어 같은 코드 공유
3. 누적 패턴: 상위→하위 순서로 break 없이 나열
4. case 제약: 상수만 가능, 정수/문자형만 가능
5. switch vs if: 정확한 값 → switch, 범위 → if-else if
