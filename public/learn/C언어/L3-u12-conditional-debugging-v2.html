디버깅 팁
if 조건에 = 가 하나만 있으면 대입입니다.
습관: 조건식을 쓸 때 == 를 먼저 확인하십시오.

실수로 = 를 쓰면 컴파일 에러가 나도록 합니다.

디버깅 전략 — printf로 값 확인

C
1#include <stdio.h>
2int main(void) {
3 int a = 10, b = 20;
4 int result = a + b;
5
6 // 디버깅용 출력: 조건에 사용되는 값을 확인
7 printf("[DEBUG] result = %d\n", result);
8
9 if (result > 50) {
10 printf("50 초과\n");
11 } else {
12 printf("50 이하\n");
13 }
14 return 0;
15}
실행 결과
[DEBUG] result = 30 50 이하

undefined

에러 체험

에러 1: 논리 연산자 혼동
if (score >= 0 && <= 100) // 컴파일 에러!

→ 각 비교는 완전한 식이어야 합니다.
→ if (score >= 0 && score <= 100)
에러 2: switch에서 break 누락
switch (menu) {
case 1: printf("저장\n");
case 2: printf("불러오기\n");
}
// 1 입력 시 "저장"과 "불러오기" 모두 출력

→ 의도하지 않은 fall-through.
→ 각 case 끝에 break; 추가
에러 3: 0으로 비교하지 않는 실수
int found = 0;
// ... 검색 로직 ...
if (found) // found가 0이 아니면 참
if (!found) // found가 0이면 참
if (found == 1) // 위험: found가 2일 수도 있음

→ 불리언 값은 == 1 대신 if (found)를 사용하십시오.
