세 가지 반복문 비교

언제 어떤 반복문을 사용하는가
while: 횟수 불확실, 0번 실행 가능한 경우
for: 횟수가 정해진 경우 (가장 많이 사용)
do-while: 최소 1번 실행이 필요한 경우 (입력 검증, 메뉴)

에러 체험

에러 1: 세미콜론 누락
do {
printf("Hello\n");
} while (i < 5) // ← 세미콜론 없음! 컴파일 에러

→ do-while문은 반드시 while (조건); 으로 끝나야 합니다.
→ while문과 달리 세미콜론이 필수입니다.
에러 2: 무한 루프 (변화 없음)
int n = 1;
do {
printf("%d\n", n);
// n++ 빠짐!
} while (n <= 5);

→ n이 항상 1이므로 무한 루프.
→ 반복 변수를 반드시 변화시키십시오.
에러 3: do-while을 while처럼 사용
// 불필요한 do-while 사용
int sum = 0;
do {
sum += i;
i++;
} while (i <= n);
// n이 0이면 i=1을 더해버림!

→ 0번 실행 가능한 경우에는 while이나 for를 사용하십시오.
→ do-while은 '최소 1번 실행'이 자연스러운 경우에만 사용하십시오.
