중첩 반복문에서 break

C
1#include <stdio.h>
2int main(void) {
3 for (int i = 1; i <= 3; i++) {
4 for (int j = 1; j <= 3; j++) {
5 if (j == 2) break; // 안쪽 for만 종료

8 printf("\n");
9 }
10 return 0;
11}

(1,1) (2,1) (3,1)

undefined

에러 체험

에러 1: while에서 continue 후 변화 누락
int i = 0;
while (i < 10) {
if (i == 5) continue; // i++에 도달 못함 → 무한 루프!
printf("%d\n", i);
i++;
}

→ while에서 continue를 쓸 때는 변화식이 continue 위에 있어야 합니다.
→ for문에서는 변화식이 자동 실행되므로 이 문제가 없습니다.
에러 2: break가 if만 종료한다는 착각
for (int i = 0; i < 10; i++) {
if (i == 5) break; // for문을 종료 (if를 종료하는 것이 아님)
}

→ break는 반복문이나 switch를 종료합니다.
→ if문을 종료하는 것이 아닙니다.
에러 3: 불필요한 break/continue
for (int i = 0; i < 10; i++) {
if (i % 2 == 0) {
printf("%d\n", i);
} else {
continue; // 불필요! else 블록이 끝나면 자동으로 다음 반복
}
}

→ continue 없이도 같은 동작. 불필요한 continue는 코드를 복잡하게 만듭니다.

대회 기출

요약

핵심 정리
1. break: 반복문 즉시 종료, 다음 코드로 이동
2. continue: 현재 회차만 건너뛰고, 다음 반복으로
3. while(1) + break: 종료 조건이 복잡할 때 유용
4. 중첩에서 break: 가장 가까운 반복문 하나만 종료
5. while + continue 주의: 변화식이 continue 아래면 무한 루프
