off-by-one 오류

C
1// 5번 반복해야 하는데...
2for (int i = 1; i < 5; i++) { ... } // 4번 (1,2,3,4)
3for (int i = 1; i <= 5; i++) { ... } // 5번 (1,2,3,4,5)
4for (int i = 0; i < 5; i++) { ... } // 5번 (0,1,2,3,4)
5for (int i = 0; i <= 5; i++) { ... } // 6번 (0,1,2,3,4,5)
off-by-one 방지법
1. 0부터 시작하면 i < N (N번 반복)
2. 1부터 시작하면 i <= N (N번 반복)
3. 확신이 없으면 작은 수(3~5)로 테스트하여 횟수 확인

디버깅 체크리스트
1. 반복 변수가 올바르게 초기화되었는가?
2. 조건식이 올바른 비교를 하는가? (=, ==, <, <=)
3. 변화식이 조건을 거짓으로 만드는 방향인가?
4. continue가 변화식을 건너뛰지 않는가?
5. while 뒤에 불필요한 세미콜론이 없는가?

에러 체험

에러 1: for문에서 조건과 변화 뒤바꿈
for (int i = 0; i++; i < 10) // 조건과 변화 위치 바뀜!

→ for (초기화; 조건; 변화) 순서를 지키십시오.
→ for (int i = 0; i < 10; i++)
에러 2: 부호 없는 정수의 무한 루프
unsigned int i = 10;
while (i >= 0) { // unsigned는 항상 >= 0!
printf("%u\n", i);
i--;
}
// i가 0에서 1 빼면 4294967295 (언더플로)

→ unsigned 변수의 >= 0 조건은 항상 참입니다.
→ int를 사용하거나 조건을 i > 0으로 변경하십시오.
에러 3: 실수형 반복 변수
for (double d = 0.0; d != 1.0; d += 0.1) {
printf("%.1f\n", d);
}
// 부동소수점 오차로 d가 정확히 1.0이 되지 않음 → 무한 루프

→ 반복 변수는 정수형을 사용하십시오.
→ 실수 비교는 d < 1.0처럼 범위로 하십시오.
