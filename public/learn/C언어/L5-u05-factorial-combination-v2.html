팩토리얼과 조합(nCr)

반복문으로 팩토리얼을 계산하고, 조합 공식 nCr을 구현하며 오버플로 문제를 다룹니다.

이 유닛에서 배우는 것

  • 팩토리얼의 정의를 이해하고 반복문으로 구현할 수 있습니다.
  • 순열(nPr)과 조합(nCr)의 차이를 설명할 수 있습니다.
  • nCr = n! / (r! × (n-r)!)을 프로그램으로 작성할 수 있습니다.

핵심 개념

팩토리얼(Factorial)
n! = 1 × 2 × 3 × ... × n
0! = 1 (정의)
1! = 1, 2! = 2, 3! = 6, 4! = 24, 5! = 120
10! = 3,628,800 / 13! = 6,227,020,800 (int 범위 초과!)
순열과 조합
순열(nPr): n개에서 r개를 순서 있게 뽑기 = n! / (n-r)!
조합(nCr): n개에서 r개를 순서 없이 뽑기 = n! / (r! × (n-r)!)

에러 체험

에러 1: int로 큰 팩토리얼 계산
int fact = 1;
for (int i = 1; i <= 15; i++) fact *= i;
// 15! = 1,307,674,368,000 → int 범위 초과!

→ long long을 사용하십시오.
→ long long fact = 1;
에러 2: nCr에서 나눗셈 순서 오류
result = result / (i + 1) * (n - i); // 정수 나눗셈 먼저!

→ 곱셈을 먼저 수행해야 나눗셈이 정확합니다.
→ result = result * (n - i) / (i + 1);
에러 3: 0! 처리 누락
long long fact = 1;
for (int i = 1; i <= n; i++) fact *= i;
// n = 0이면 반복 안 함 → fact = 1 (우연히 맞음)

→ 동작하지만 0!을 명시적으로 처리하는 것이 안전합니다.

설명하기

설명하기

nCr을 계산할 때 result *= (n-i); result /= (i+1); 순서로 곱하며 나누면 항상 정수가 되는 이유를 설명하십시오. 연속된 k개의 정수의 곱이 k!로 나누어떨어지는 원리를 서술하십시오.

핵심 정리
1. 팩토리얼: n! = 1 × 2 × ... × n, 0! = 1
2. int는 12!까지, long long은 20!까지 저장 가능
3. 순열 nPr: n부터 r개를 곱하기 (n × (n-1) × ... × (n-r+1))
4. 조합 nCr: 곱하며 나누기 패턴으로 오버플로 방지
5. 파스칼의 삼각형: nCr = (n-1)C(r-1) + (n-1)Cr

용어설명
팩토리얼n! = 1 × 2 × ... × n. 0! = 1로 정의합니다. 경우의 수 계산의 기본 연산입니다.
순열(Permutation)n개에서 r개를 순서 있게 뽑는 경우의 수. nPr = n! / (n-r)!.
