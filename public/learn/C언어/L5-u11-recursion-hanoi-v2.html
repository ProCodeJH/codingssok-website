핵심 개념

하노이의 탑 규칙
1. 한 번에 맨 위의 원반 한 개만 이동
2. 큰 원반 위에 작은 원반만 올릴 수 있음
3. 기둥 3개: 출발(A), 보조(B), 목표(C)
재귀적 분해 전략
N개의 원반을 A에서 C로 옮기려면:
1단계: 위의 N-1개를 A에서 B로 옮긴다 (C를 보조로)
2단계: 가장 큰 원반을 A에서 C로 옮긴다
3단계: B의 N-1개를 B에서 C로 옮긴다 (A를 보조로)

3void hanoi(int n, char from, char to, char via) {

이동 횟수 계산

C
1#include <stdio.h>
2
3int count = 0;
4
5void hanoi(int n, char from, char to, char via) {
6 if (n == 1) {
7 count++;
8 return;
9 }
10 hanoi(n - 1, from, via, to);
11 count++;
12 hanoi(n - 1, via, to, from);
13}
14
15int main(void) {
16 for (int n = 1; n <= 10; n++) {
17 count = 0;
18 hanoi(n, 'A', 'C', 'B');
19
20 // 2^n - 1 검증
21 int expected = 1;
22 for (int i = 0; i < n; i++) expected *= 2;
23 expected--;
24
25 printf("원반 %2d개: %5d번 이동 (2^%d-1 = %d) %s\n",
26 n, count, n, expected,
27 (count == expected) ? "OK" : "ERROR");
28 }
29 return 0;
30}
실행 결과
원반 1개: 1번 이동 (2^1-1 = 1) OK 원반 2개: 3번 이동 (2^2-1 = 3) OK 원반 3개: 7번 이동 (2^3-1 = 7) OK 원반 4개: 15번 이동 (2^4-1 = 15) OK 원반 5개: 31번 이동 (2^5-1 = 31) OK ... 원반 10개: 1023번 이동 (2^10-1 = 1023) OK
이동 횟수 점화식
T(1) = 1
T(n) = 2 × T(n-1) + 1

풀면: T(n) = 2^n - 1
n = 20이면 1,048,575번
n = 64이면 약 1.8 × 10^19번 (우주의 나이로도 부족)

에러 체험

에러 1: 매개변수 순서 혼동
hanoi(n-1, from, to, via); // 보조와 목표가 뒤바뀜!

→ hanoi(n-1, from, via, to);
→ 매개변수 이름을 from/to/via로 명확히 지으십시오.
에러 2: 기저 조건에서 return 누락
if (n == 1) {
printf("이동\n");
// return 없음! 아래 재귀 호출이 실행됨
}
hanoi(n-1, from, via, to);

→ 기저 조건에서 반드시 return하십시오.
에러 3: n == 0 기저 조건과 n == 1 기저 조건 혼동
// n == 0으로 하면 n=1일 때 불필요한 호출 발생
if (n == 0) return;
hanoi(n-1, from, via, to);
printf("원반 %d: %c→%c\n", n, from, to);
hanoi(n-1, via, to, from);
// 동작은 하지만 n=1에서 두 번의 빈 호출 추가

→ 두 방식 모두 가능하나 n == 1이 더 효율적입니다.

핵심 정리
1. 하노이의 탑: 3기둥, N원반, 한 번에 1개, 큰 위에 작은 것만
2. 재귀 전략: N-1개를 보조로 → 큰 원반 이동 → N-1개를 목표로
3. 이동 횟수: T(n) = 2^n - 1
4. 매개변수: from(출발), to(목표), via(보조)
5. 재귀의 위력: 반복문으로는 구현이 매우 어려운 문제를 간결하게 해결
