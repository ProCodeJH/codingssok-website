Chapter 06

지역변수 vs 전역변수

변수의 유효 범위(스코프)를 이해하고, 지역변수와 전역변수의 차이와 올바른 사용법을 학습합니다.

이 유닛에서 배우는 것

  • 지역변수와 전역변수의 차이를 구분할 수 있습니다.
  • 변수의 스코프(유효 범위)와 수명을 설명할 수 있습니다.
  • 같은 이름의 지역변수와 전역변수가 있을 때 우선순위를 판단할 수 있습니다.
  • 전역변수의 남용이 위험한 이유를 이해하고, 지역변수를 우선 사용할 수 있습니다.

전역변수를 피해야 하는 이유
1. 어떤 함수가 값을 바꿨는지 추적이 어렵습니다.
2. 함수의 독립성이 깨져 재사용이 어렵습니다.
3. 이름 충돌 위험이 높아집니다.
4. 디버깅이 매우 어려워집니다.
원칙: 가능하면 지역변수 + 매개변수/반환값으로 데이터를 주고받으십시오.

대회 기출

요약

핵심 정리
1. 지역변수: 함수/블록 안에서만 유효, 끝나면 소멸
2. 전역변수: 프로그램 전체에서 유효, 자동 0 초기화
3. Shadowing: 같은 이름이면 지역변수가 전역을 가림
4. 블록 스코프: {} 안에서 선언한 변수는 그 블록에서만 유효
5. 원칙: 전역변수 최소화, 매개변수/반환값으로 데이터 전달
