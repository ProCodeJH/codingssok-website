메모리 주소란?

변수가 메모리에 저장되는 원리와 주소 연산자(&)로 주소를 확인하는 방법을 배웁니다.

이 유닛에서 배우는 것

  • 변수가 메모리의 특정 주소에 저장된다는 것을 이해합니다.
  • & 연산자로 변수의 메모리 주소를 확인할 수 있습니다.
  • %p 서식 지정자로 주소를 출력할 수 있습니다.
  • 자료형에 따라 메모리 크기가 다르다는 것을 확인합니다.

scanf와 & — 다시 보기
scanf("%d", &a);에서 &a는 '변수 a의 주소'입니다.
scanf는 이 주소로 찾아가서 값을 저장합니다.

scanf("%s", str);에서 str은 배열 이름이므로 이미 주소입니다.
따라서 &를 붙이지 않습니다.

&를 빼먹으면 scanf는 엉뚱한 곳에 값을 저장하여 프로그램이 비정상 종료됩니다.

에러 체험

C
1// 에러 1: scanf에서 & 누락
2int x;
3scanf("%d", x); // x의 '값'을 주소로 해석 → 비정상 종료
4// 수정: scanf("%d", &x);
5
6// 에러 2: 주소를 %d로 출력
7printf("%d\n", &x); // 주소를 정수로 해석 → 경고, 잘못된 값
8// 수정: printf("%p\n", &x);
9
10// 에러 3: 상수에 & 사용
11printf("%p\n", &10); // 상수는 메모리 주소가 없음 → 컴파일 에러
12// & 연산자는 메모리에 저장된 변수에만 사용 가능

요약

핵심 정리
1. 메모리: 바이트 단위, 각 바이트에 고유 주소
2. & 연산자: 변수의 메모리 주소를 반환
3. %p: 주소 출력 서식 지정자 (16진수)
4. sizeof: 자료형이 차지하는 바이트 수
5. 배열 이름 = 첫 번째 원소의 주소 (&arr[0])
6. scanf(&a): 주소를 넘겨야 값을 저장할 수 있음
