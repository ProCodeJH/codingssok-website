Chapter 07

포인터 연산

포인터에 정수를 더하거나 뺄 때 자료형 크기만큼 이동하는 원리와, 포인터끼리의 뺄셈을 배웁니다.

이 유닛에서 배우는 것

  • 포인터 + n이 n×sizeof(자료형) 바이트만큼 이동함을 이해합니다.
  • 자료형에 따라 포인터 이동 크기가 달라지는 것을 확인합니다.
  • 포인터끼리 뺄셈으로 원소 간 거리를 구할 수 있습니다.
  • 포인터 연산을 활용하여 배열을 순회할 수 있습니다.

포인터 연산의 핵심 규칙
포인터 + n = 현재 주소 + n × sizeof(자료형)

int *p에서 p + 1: 주소 + 4바이트
int *p에서 p + 3: 주소 + 12바이트
double *p에서 p + 1: 주소 + 8바이트
char *p에서 p + 1: 주소 + 1바이트

이것이 arr[i] == *(arr + i)가 성립하는 이유입니다.

포인터 연산 규칙 정리
허용되는 연산:
포인터 + 정수 → 포인터 (p + 3)
포인터 - 정수 → 포인터 (p - 2)
포인터 - 포인터 → 정수 (p2 - p1, 원소 간 거리)
포인터 비교 → 참/거짓 (p1 < p2)

허용되지 않는 연산:
포인터 + 포인터 (의미 없음)
포인터 × 정수 (의미 없음)
포인터 / 정수 (의미 없음)

핵심 정리
1. p + n → 주소 + n × sizeof(자료형) 바이트
2. int*: +1 = 4바이트, double*: +1 = 8바이트
3. p++ / p-- : 다음/이전 원소로 이동
4. p2 - p1 → 원소 간 거리 (바이트가 아님)
5. 허용: 포인터±정수, 포인터-포인터, 포인터 비교
6. 금지: 포인터+포인터, 포인터×정수
