핵심 개념

void 포인터 핵심 규칙
1. void*는 모든 자료형의 주소를 저장할 수 있습니다.
void *vp = &a; (int, double, char 등 모두 가능)

2. void*는 직접 역참조할 수 없습니다.
*vp → 컴파일 에러 (몇 바이트를 읽을지 모름)

3. 형변환 후 역참조해야 합니다.
*(int*)vp → int로 해석하여 4바이트 읽기
*(double*)vp → double로 해석하여 8바이트 읽기

4. void*에 포인터 연산(+, -)은 불가능합니다.
vp + 1 → 컴파일 에러 (이동 크기를 모름)

5void genericSwap(void *a, void *b, int size) {
6 char temp[256]; /* 임시 버퍼 */

10}

void*를 사용할 때 주의사항
1. 형변환을 잘못하면 값이 깨집니다.
int a = 10; void *vp = &a;
*(double*)vp → int 4바이트를 double 8바이트로 읽어 쓰레기 값

2. 포인터 연산 불가 → char*로 변환 후 연산
(char*)vp + sizeof(int) → 다음 int 위치로 이동

3. void*에서 자료형 정보가 사라지므로, 프로그래머가 책임지고 올바른 자료형으로 변환해야 합니다.

에러 체험

C

4// 수정: printf("%d\n", *(int*)vp);
5
6// 에러 2: 잘못된 형변환

핵심 정리
1. void*: 모든 자료형의 주소를 저장하는 만능 포인터
