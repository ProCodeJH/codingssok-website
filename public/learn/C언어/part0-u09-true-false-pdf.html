참일까? 거짓일까?

안녕! 오늘은 참과 거짓을 배울 거야. 컴퓨터는 모든 것을 '참 아니면 거짓'으로 판단해. 이게 프로그래밍의 시작이야!

우리가 매일 쓰는 말 중에는 맞다/틀리다를 확실히 말할 수 있는 것이 있어요. 예를 들어볼까요?

[예시] 맞다/틀리다를 말할 수 있는 문장
"서울은 한국의 수도다" → 참 (맞아요!)
"1 + 1 = 3" → 거짓 (틀려요! 1+1=2니까)
"물은 100°C에서 끓는다" → 참
"고양이는 새다" → 거짓

이렇게 확실하게 참인지 거짓인지 판단할 수 있는 문장을 명제라고 해요.

[핵심] 명제(Proposition)의 정의
명제란 참(True)인지 거짓(False)인지 확실하게 판단할 수 있는 문장이에요.

중요한 조건: 누가 판단해도 같은 결과가 나와야 해요!

명제가 아닌 것

그런데 모든 문장이 명제인 건 아니에요. 사람마다 다르게 생각할 수 있는 문장은 명제가 아니에요.

[주의] 이런 건 명제가 아니에요!
"오늘 날씨 좋다" → 사람마다 다르게 느껴요
"피자가 치킨보다 맛있다" → 취향이에요
"수학은 재미있다" → 사람마다 달라요
"빨리 와!" → 명령이지 판단할 수 있는 문장이 아니에요
"지금 몇 시야?" → 질문이에요
명제 = 참/거짓을 확실히 판단할 수 있는 문장
쉽게 기억하는 방법! '이 문장은 참이야? 거짓이야?'라고 물었을 때 누구나 같은 대답을 할 수 있으면 명제야!

[꿀팁] 명제 구별 3초 규칙

컴퓨터는 참/거짓으로 생각해요

컴퓨터는 0과 1만 이해한다고 했죠? (U0-03에서 배웠어요!) 참과 거짓도 마찬가지예요. 컴퓨터에서는 참 = 1, 거짓 = 0이에요.

[핵심] 컴퓨터의 참/거짓

거짓 (False) = 0 = 전기가 안 흐른다 = 스위치 OFF

컴퓨터는 모든 판단을 이 두 가지로 해요!
참 = ON = 1, 거짓 = OFF = 0

[주의] C 언어의 독특한 참/거짓 규칙

[심화] 변수가 있는 문장 — 열린 문장

"x + 3 = 7"은 명제일까요? x가 4이면 참이고, x가 5이면 거짓이에요. 이렇게 변수의 값에 따라 참/거짓이 달라지는 문장을 열린 문장(open sentence)이라고 해요.

[심화] 명제 vs 열린 문장
명제: 참/거짓이 확정됨 — "3 + 4 = 7" (참)
열린 문장: 변수 값에 따라 달라짐 — "x + 4 = 7"

열린 문장은 변수에 값을 대입하면 명제가 돼요!
x = 3 → "3 + 4 = 7" → 참인 명제
x = 5 → "5 + 4 = 7" → 거짓인 명제
프로그래밍에서 if (x > 10) 같은 조건도 일종의 열린 문장이야. x에 실제 값이 들어가는 순간 참/거짓이 결정돼!

[심화] 디버깅과 참/거짓

프로그래밍에서 버그의 많은 부분은 참/거짓 판단이 잘못되어서 생겨요. 조건을 잘못 쓰면 프로그램이 예상과 다르게 동작해요.

[심화] 역사 — 불리언의 아버지, 조지 불

[역사] 조지 불 (George Boole, 1815-1864)
영국의 수학자 조지 불은 1854년에 《사고의 법칙》이라는 책을 썼어요. 이 책에서 그는 참/거짓으로 수학적 계산을 하는 체계를 만들었어요.

이것이 바로 불리언 대수(Boolean Algebra)예요. 당시에는 '쓸모없는 이론'이라는 평가를 받았지만, 약 80년 후 클로드 섀넌이 전기 회로 설계에 응용하면서 컴퓨터 시대의 기반이 되었어요!

조지 불의 이름에서 Boolean이라는 단어가 나왔어요.
[역사] 클로드 섀넌 — 디지털 시대를 연 사람
1937년, MIT 대학원생 클로드 섀넌(Claude Shannon)은 석사 논문에서 불리언 대수를 전기 스위치 회로에 적용했어요.

스위치 ON = 참(1), OFF = 거짓(0)

이 아이디어가 현대 디지털 컴퓨터의 기초가 되었어요. 이 논문은 '20세기 가장 중요한 석사 논문'으로 불려요!

[심화] 하드웨어 — 트랜지스터와 참/거짓

컴퓨터 내부에서 참/거짓은 전기 신호로 구현돼요. 트랜지스터라는 아주 작은 스위치가 전기를 흘려보내거나(참) 차단하거나(거짓) 해요.

[HW 연결] 트랜지스터 = 참/거짓 스위치
현대 CPU 안에는 트랜지스터가 수십억 개 들어있어요.

각 트랜지스터는 전기가 흐르면 1(참), 안 흐르면 0(거짓).

[심화 내용] 한정사 맛보기
∀x P(x) — '모든 x에 대해 P(x)가 참' (전칭 한정사)

예: ∃x (x² = 4) — 'x²=4인 x가 존재한다' (x=2 또는 x=-2)

[심화 내용] 진리값과 불리언 대수

불리언 대수에서는 참/거짓에 대해 수학적 연산 규칙을 정의해요. 이것이 디지털 회로 설계와 프로그래밍의 수학적 기반이에요.

[심화 내용] 불리언 대수의 기본 법칙
항등 법칙: A AND 1 = A, A OR 0 = A
영 법칙: A AND 0 = 0, A OR 1 = 1
보수 법칙: A AND NOT A = 0, A OR NOT A = 1
멱등 법칙: A AND A = A, A OR A = A
이중 부정: NOT (NOT A) = A

이 법칙들은 다음 유닛(AND, OR, NOT)에서 더 자세히 배워요!

[심화 내용] CPU 플래그 레지스터
CPU에는 플래그 레지스터(FLAGS)가 있어요. 비교(CMP) 명령 실행 후:

• ZF (Zero Flag): 결과가 0이면 1 → 두 값이 같다
• SF (Sign Flag): 결과가 음수면 1
• CF (Carry Flag): 자리올림 발생 시 1

조건 분기 명령(JE, JNE, JG, JL…)은 이 플래그를 보고 점프 여부를 결정해요.

[심화] 면접에서 나오는 참/거짓 질문

[면접 대비] 자주 나오는 질문들
Q1: C 언어에서 NULL은 참인가요 거짓인가요?
→ 거짓. NULL은 보통 0으로 정의되어 있어요.

Q2: if (ptr)는 무슨 뜻인가요?
→ ptr이 NULL이 아닌지(유효한 포인터인지) 확인하는 관용적 표현이에요.

Q3: 불리언 값을 정수로 변환하면?
→ true = 1, false = 0. 연산에 활용 가능: count += (x > 0); 처럼 쓸 수 있어요.

OX 퀴즈

'5 > 3'은 명제이다. (O/X)

정답: ⭕ O
5는 3보다 크므로 참인 명제입니다.
OX 퀴즈

'이 영화 재미있다'는 명제이다. (O/X)

정답: ❌ X
재미있는지는 사람마다 다르게 느끼므로 명제가 아닙니다.
객관식

다음 중 명제인 것을 모두 고르세요.

    정답: (1), (3), (5)
    (2)는 사람마다 다르고, (4)는 명령문이므로 명제가 아닙니다.
    OX 퀴즈

    컴퓨터에서 참(True)은 숫자 0으로 표현한다. (O/X)

    정답: ❌ X
    참(True)은 1, 거짓(False)이 0입니다.
    객관식

    다음 비교 연산의 결과가 참(1)인 것은?

      정답: (2)
      10 == 10은 같으므로 참(1)입니다. 나머지는 모두 거짓(0)입니다.
      단답형

      C 언어에서 다음 중 '거짓'으로 판단되는 값은?
      (1) 1
      (2) 0
      (3) -5
      (4) 100

      정답: (2) 0
      C 언어에서는 0만 거짓이고, 0이 아닌 모든 수(1, -5, 100 등)는 참입니다.
      단답형

      다음 코드의 출력 결과를 쓰세요.
      printf("%d\n", 10 > 5);
      printf("%d\n", 3 == 4);

      정답: 1
      0
      10 > 5는 참이므로 1, 3 == 4는 거짓이므로 0이 출력됩니다.
      단답형

      다음 코드에서 버그를 찾고 수정하세요.
      int x = 7;
      if (x = 10) { printf("같다"); }

      정답: if (x == 10) 으로 수정해야 합니다.
      = 는 대입 연산자, == 가 비교 연산자입니다. x = 10은 x에 10을 대입한 후 10(참)을 반환하므로 항상 실행됩니다.
      객관식

      불리언(Boolean)이라는 이름은 누구에게서 유래했나요?

        정답: (2) 조지 불
        영국 수학자 George Boole(1815-1864)이 참/거짓으로 수학적 계산을 하는 불리언 대수를 만들었어요.
        단답형

        실생활에서 참/거짓 판단이 사용되는 예시를 3가지 쓰세요.

        정답: 예시: (1) 신호등 — 빨간불이면 멈춤 (2) 비밀번호 확인 — 맞으면 로그인 (3) ATM — 잔액 충분하면 출금
        우리 주변의 거의 모든 자동 판단 시스템은 참/거짓 로직으로 작동합니다.
        단답형

        [심화] 'x + 2 = 8'은 명제인가요? 이유를 설명하세요.

        정답: 명제가 아닙니다. x의 값에 따라 참/거짓이 달라지는 '열린 문장'입니다.
        x = 6이면 참이지만 x = 3이면 거짓이므로, 변수에 값이 확정되기 전에는 명제가 아닙니다. 이를 열린 문장(open sentence)이라고 합니다.
        단답형

        [심화] C 언어에서 if (ptr)의 의미를 설명하세요. ptr은 포인터 변수입니다.

        정답: ptr이 NULL이 아닌지(유효한 포인터인지) 확인하는 표현입니다.
        NULL은 보통 0으로 정의되므로 거짓입니다. ptr이 NULL이 아니면(0이 아니면) 참이 되어 if 블록이 실행됩니다. if (ptr != NULL)과 같은 의미입니다.
        단답형

        [심화] Python에서 다음 중 거짓(falsy)으로 판단되는 것을 모두 고르세요.
        (1) 0
        (2) ''(빈 문자열)
        (3) [](빈 리스트)
        (4) 'hello'
        (5) None

        정답: (1), (2), (3), (5)
        Python에서 0, 빈 문자열, 빈 리스트, None은 모두 falsy 값입니다. 'hello'는 비어있지 않은 문자열이므로 truthy입니다.
        단답형

        [심화 내용] '∀x (x > 0 → x + 1 > 0)'을 한국어로 해석하고 참/거짓을 판단하세요.

        정답: '모든 x에 대해, x가 0보다 크면 x+1도 0보다 크다' — 참입니다.
        양수에 1을 더하면 반드시 양수이므로 참입니다. ∀는 전칭 한정사(모든), →는 '이면'(함의)을 뜻합니다.
        단답형

        [심화 내용] x86 CPU에서 CMP 명령 실행 후 ZF(Zero Flag)가 1이 되었다면, 비교한 두 값의 관계를 설명하세요.

        정답: 두 값이 같습니다(equal).
        CMP는 내부적으로 뺄셈(A - B)을 수행합니다. 결과가 0이면 ZF=1이 됩니다. 즉, 두 피연산자가 같다는 의미입니다. JE(Jump if Equal) 명령이 이 플래그를 검사합니다.

        용어 정리

        명제 (Proposition)
        참인지 거짓인지 확실히 판단할 수 있는 문장.
        참 (True)
        맞다는 뜻. 컴퓨터에서는 1로 표현해요.
        거짓 (False)
        틀리다는 뜻. 컴퓨터에서는 0으로 표현해요.
        불리언 (Boolean)
        참과 거짓, 단 2가지 값만 가지는 자료형. 영국 수학자 조지 불(George Boole)의 이름에서 왔어요.
        조건 (Condition)
        참인지 거짓인지 판단할 수 있는 식. if문에서 사용해요.
        if문
        C 언어에서 조건이 참일 때만 코드를 실행하는 명령어.
        비교 연산자
        두 값을 비교해서 참/거짓을 반환하는 연산자. ==, !=, >, <, >=, <=
        열린 문장 (Open Sentence)
        변수가 포함되어 값에 따라 참/거짓이 달라지는 문장. 변수에 값을 대입하면 명제가 돼요.
        플래그 (Flag)
        프로그래밍에서 어떤 상태를 참/거짓으로 기억하는 불리언 변수.
        불리언 대수 (Boolean Algebra)
        참/거짓에 대한 수학적 연산 체계. 조지 불이 창시하고, 클로드 섀넌이 디지털 회로에 응용했어요.
        트랜지스터 (Transistor)
        전기 신호를 켜고 끄는 아주 작은 스위치. CPU 안에 수십억 개가 들어있어요.
        명제 논리 (Propositional Logic)
        문장 단위로 참/거짓을 다루는 논리 체계.
        술어 논리 (Predicate Logic)
        주어와 술어로 분해하고 한정사(∀, ∃)를 사용하는 더 강력한 논리 체계.
        falsy
        프로그래밍에서 거짓으로 취급되는 값들. 언어마다 다르지만 보통 0, 빈 값, null 등.
