비교는 왜 중요할까?

컴퓨터는 숫자를 비교해서 판단을 내려. "점수가 90점 이상이면 A등급"처럼 말이야. 비교 없이는 아무것도 판단할 수 없어!

우리는 매일 비교를 해요. "이게 더 비싸다", "키가 같다", "시간이 부족하다"... 컴퓨터도 마찬가지예요. 두 값을 비교해서 참 또는 거짓을 내놓아요.

6가지 비교 연산

[핵심] 비교 연산자 6종류
같다 (==): 5 == 5 → 참, 3 == 7 → 거짓
다르다 (!=): 5 != 3 → 참, 5 != 5 → 거짓
크다 (>): 7 > 3 → 참, 3 > 7 → 거짓
작다 (<): 3 < 7 → 참, 7 < 3 → 거짓
크거나 같다 (>=): 5 >= 5 → 참, 5 >= 3 → 참
작거나 같다 (<=): 3 <= 5 → 참, 3 <= 3 → 참
비교 연산자를 수직선으로 이해해 봐요
'같다'는 등호를 두 개(==) 쓰는 거야! 하나(=)만 쓰면 C에서는 '값을 넣겠다'는 뜻이 돼서 완전 다른 일을 해. 초보자가 가장 많이 하는 실수야!
비교 연산자 진리표 (a=5, b=3, c=5)
표현식결과설명
a == b거짓5와 3은 다름
a == c참5와 5는 같음
a != b참5와 3은 다름
a > b참5는 3보다 큼
a < b거짓5는 3보다 작지 않음
a >= c참5는 5 이상
a <= b거짓5는 3 이하가 아님

이상/이하 vs 초과/미만

수학에서 자주 쓰는 용어인데, 혼동하기 쉬워요. 핵심은 그 수를 포함하느냐 마느냐예요.

[핵심] 이상/이하 vs 초과/미만
5 이상: 5를 포함해서 5, 6, 7, 8, ... (>= 5)
5 초과: 5를 빼고 6, 7, 8, ... (> 5)
5 이하: 5를 포함해서 ..., 3, 4, 5 (<= 5)
5 미만: 5를 빼고 ..., 3, 4 (< 5)

이상/이하 = 그 수 포함
초과/미만 = 그 수 불포함
채워진 점(●)은 포함, 빈 점(○)은 불포함!
[예시] 놀이공원 키 제한
"키 120cm 이상만 탈 수 있어요"
→ 120cm인 사람: 탈 수 있다 (이상이니까 포함!)

"키 120cm 초과만 탈 수 있어요"
→ 120cm인 사람: 탈 수 없다 (초과니까 불포함!)
→ 120.1cm인 사람: 탈 수 있다

두 수 비교하기

두 수를 비교하는 건 간단해요. 비교 연산자를 사이에 넣으면 돼요.

세 수 비교하기

세 수를 비교하려면 비교를 두 번 해서 AND로 연결해요.

[예시] 성적 등급 매기기
점수가 80점 이상이고 90점 미만이면 B등급:

수학에서: 80 ≤ 점수 < 90
논리식으로: 점수 >= 80 AND 점수 < 90

점수 = 85일 때: 85 >= 80 → 참, 85 < 90 → 참, 참 AND 참 = 참 → B등급!
수학에서는 80 ≤ x < 90 이렇게 한 줄로 쓰지만, C에서는 꼭 두 번 비교해서 &&로 연결해야 해. 80 <= x && x < 90 이렇게!

C 언어 연결: 비교 연산자

compare_demo.c
1#include <stdio.h>
2
3int main(void) {
4 int score = 85;
5
6 // 비교 연산자의 결과는 0(거짓) 또는 1(참)
7 printf("%d == 85: %d\n", score, score == 85); // 1
8 printf("%d > 90: %d\n", score, score > 90); // 0
9 printf("%d != 0: %d\n", score, score != 0); // 1
10
11 // 범위 비교
12 if (score >= 90) {
13 printf("A등급\n");
14 } else if (score >= 80 && score < 90) {
15 printf("B등급\n"); // 이게 출력됨!
16 } else if (score >= 70) {
17 printf("C등급\n");
18 }
19
20 return 0;
21}
실행 결과
85 == 85: 1 85 > 90: 0 85 != 0: 1 B등급
[디버깅] 가장 흔한 실수: = vs ==
if (score = 100) ← score에 100을 넣어버림! 항상 참!
if (score == 100) ← score가 100인지 비교 ✅

방어 코딩 팁: 상수를 왼쪽에 쓰면 실수를 방지할 수 있어요.
if (100 == score) — 실수로 =를 쓰면 컴파일 에러!
이걸 요다 조건문(Yoda Condition)이라고 불러요.

[심화] 부동소수점 비교의 함정

실수(float, double)를 ==로 비교하면 예상과 다른 결과가 나올 수 있어요!

float_compare.c
1#include <stdio.h>
2#include <math.h>
3
4int main(void) {
5 double a = 0.1 + 0.2;
6 double b = 0.3;
7
8 printf("0.1 + 0.2 = %.17f\n", a); // 0.30000000000000004
9 printf("0.3 = %.17f\n", b); // 0.29999999999999999
10
11 if (a == b) {
12 printf("같다\n");
13 } else {
14 printf("다르다!\n"); // 이게 출력됨!
15 }
16
17 // 올바른 비교: 오차 범위(epsilon) 사용
18 double epsilon = 1e-9;
19 if (fabs(a - b) < epsilon) {
20 printf("거의 같다 (올바른 방법)\n"); // 이게 출력됨
21 }
22
23 return 0;
24}
실행 결과
0.1 + 0.2 = 0.30000000000000004 0.3 = 0.29999999999999999 다르다! 거의 같다 (올바른 방법)
[주의] 실수는 == 비교 금지!
0.1 + 0.2 ≠ 0.3 인 이유: 컴퓨터는 실수를 2진수로 저장하는데, 0.1은 2진수로 무한소수!
마치 1/3 = 0.333... 을 소수점 이하 16자리에서 자르는 것과 같아요.

해결법: fabs(a - b) < epsilon 으로 '거의 같은지' 비교
epsilon = 허용 오차 (보통 1e-9 ~ 1e-6)

[심화] 문자열 비교 — strcmp

문자열은 ==로 비교하면 안 돼요! C에서 문자열은 포인터(주소)이기 때문에. strcmp() 함수를 사용해야 해요.

strcmp_demo.c
1#include <stdio.h>
2#include <string.h>
3
4int main(void) {
5 char a[] = "hello";
6 char b[] = "hello";
7
8 // 잘못된 비교 (주소 비교!)
9 if (a == b) {
10 printf("같다\n");
11 } else {
12 printf("다르다 (주소가 다름!)\n"); // 이게 출력됨

[핵심] strcmp 반환값
strcmp(a, b) 반환값:
- 0: a와 b가 같음
- 음수: a가 사전순으로 앞 (a < b)
- 양수: a가 사전순으로 뒤 (a > b)

사전순 = ASCII 코드 값 기준 (대문자 < 소문자)
'A'(65) < 'Z'(90) < 'a'(97) < 'z'(122)

[심화] 비교 연산과 정렬 알고리즘

정렬은 비교를 반복해서 데이터를 순서대로 나열하는 거예요. 가장 간단한 버블 정렬을 볼까요?

bubble_sort.c
1#include <stdio.h>
2
3void bubble_sort(int arr[], int n) {
4 for (int i = 0; i < n - 1; i++) {
5 for (int j = 0; j < n - 1 - i; j++) {
6 if (arr[j] > arr[j+1]) { // 비교!
7 // 교환
8 int temp = arr[j];
9 arr[j] = arr[j+1];
10 arr[j+1] = temp;
11 }
12 }
13 }
14}
15
16int main(void) {
17 int arr[] = {5, 2, 8, 1, 9};
18 int n = 5;
19
20 bubble_sort(arr, n);
21
22 for (int i = 0; i < n; i++)
23 printf("%d ", arr[i]);
24 printf("\n"); // 1 2 5 8 9
25
26 return 0;
27}

[심화 내용] CPU는 비교를 어떻게 할까?

CPU에는 비교 전용 명령어가 있어요. 사실 비교는 뺄셈이에요! a - b를 해보고 결과가 양수면 a > b, 0이면 a == b, 음수면 a < b.

cmp_asm.asm
1; x86 어셈블리: if (a > b)
2 mov eax, [a] ; eax = a
3 cmp eax, [b] ; eax - b를 계산 (결과는 버림)
4 ; 플래그 레지스터만 업데이트!
5 jg greater ; ZF=0, SF=OF이면 점프 (a > b)
6 ; ... a <= b일 때 코드 ...
7greater:
8 ; ... a > b일 때 코드 ...
[심화 내용] 플래그 레지스터 (FLAGS)
CMP 명령은 뺄셈 결과를 버리고 플래그만 설정해요:

- ZF (Zero Flag): 결과가 0이면 1 → a == b
- SF (Sign Flag): 결과가 음수면 1
- CF (Carry Flag): unsigned 오버플로우 시 1
- OF (Overflow Flag): signed 오버플로우 시 1

조건 점프 명령어가 이 플래그를 보고 분기:
JE/JZ (같으면), JNE/JNZ (다르면), JG (크면), JL (작으면)...

[심화 내용] 비교기 회로 (Magnitude Comparator)

디지털 회로에서 두 n비트 수를 비교하는 회로를 비교기(Comparator)라고 해요.

1비트 비교기
입력: A, B
출력: A>B, A=B, A
A>B = A AND NOT B
A=B = A XNOR B = NOT(A XOR B)
A = NOT A AND B

n비트 비교기: 최상위 비트부터 비교, 같으면 다음 비트로 내려감 (cascading)

[심화 내용] 3-way 비교와 spaceship 연산자

비교의 결과를 양수/0/음수 세 가지로 반환하는 것을 3-way 비교라고 해요. C의 strcmp()가 대표적이에요.

[심화 내용] 어셈블리로 보는 비교 연산

CPU가 비교를 어떻게 수행하는지 어셈블리 레벨에서 살펴봐요.

[심화 내용] CMP 명령과 조건 분기
CMP a, b 명령 = 내부적으로 a - b를 계산하고 결과를 버림
대신 플래그 레지스터를 설정:

- ZF (Zero Flag): a-b=0이면 1 (a==b)
- SF (Sign Flag): a-b<0이면 1 (a- CF (Carry Flag): unsigned에서 빌림 발생하면 1
- OF (Overflow Flag): signed 오버플로우 시 1

조건 분기 명령어:
JE (Jump if Equal): ZF=1이면 점프 (==)
JNE: ZF=0이면 점프 (!=)
JG (Jump if Greater): ZF=0 AND SF=OF이면 점프 (>, signed)
JL (Jump if Less): SF!=OF이면 점프 (<, signed)
JGE: SF=OF이면 점프 (>=)
JLE: ZF=1 OR SF!=OF이면 점프 (<=)
JA (Jump if Above): CF=0 AND ZF=0 (>, unsigned)
JB (Jump if Below): CF=1 (<, unsigned)

[심화 내용] IEEE 754 비교의 수학적 분석

부동소수점 비교가 왜 위험한지 더 깊이 파봐요.

[심화 내용] 부동소수점 비교 전략 4가지
1. 절대 오차 (Absolute Epsilon)
fabs(a-b) < epsilon
단점: 큰 수에서는 epsilon이 너무 작음

2. 상대 오차 (Relative Epsilon)
fabs(a-b) / max(fabs(a), fabs(b)) < epsilon
단점: a나 b가 0에 가까우면 분모 문제

3. ULP 비교 (Units in the Last Place)
두 수의 비트 표현 차이가 N ULP 이내인지 확인
가장 정밀하지만 구현 복잡

4. 조합 방식
0 근처에서는 절대 오차, 큰 수에서는 상대 오차 사용
Google Test의 EXPECT_NEAR, EXPECT_FLOAT_EQ가 이 방식
[C 코드] 부동소수점 비교 함수 구현
1#include <stdio.h>
2#include <math.h>
3#include <float.h>
4
5// 상대+절대 오차 조합 비교
6int nearly_equal(double a, double b, double epsilon) {
7 double diff = fabs(a - b);
8
9 // 완전히 같은 경우 (무한대 포함)
10 if (a == b) return 1;
11
12 // 0에 가까운 경우: 절대 오차 사용
13 if (a == 0 || b == 0 || diff < DBL_MIN) {
14 return diff < epsilon * DBL_MIN;
15 }
16
17 // 일반적인 경우: 상대 오차 사용
18 return diff / fmin(fabs(a) + fabs(b), DBL_MAX) < epsilon;
19}
20
21int main() {
22 // 위험한 비교
23 double a = 0.1 + 0.2;
24 printf("0.1 + 0.2 == 0.3? %d\n", a == 0.3); // 0 (거짓!)
25 printf("nearly_equal? %d\n", nearly_equal(a, 0.3, 1e-10)); // 1
26
27 // 큰 수 비교
28 double big1 = 1e15 + 1.0;
29 double big2 = 1e15 + 2.0;
30 printf("big nearly_equal? %d\n", nearly_equal(big1, big2, 1e-10));
31
32 return 0;
33}
실행 결과
0.1 + 0.2 == 0.3? 0 nearly_equal? 1 big nearly_equal? 0

[심화 내용] 비교 기반 정렬의 하한

비교만으로 정렬할 때, 최소 몇 번의 비교가 필요할까요?

[심화 내용] 비교 정렬의 하한 정리
정리: n개 원소를 비교 기반으로 정렬하려면
최소 ceil(log2(n!))번의 비교가 필요하다.

증명 스케치:
1. n개 원소의 가능한 순열: n!가지
2. 비교 1번 = 2갈래 (참/거짓) → k번 비교로 2^k가지 구분 가능
3. 모든 순열을 구분하려면: 2^k >= n!
4. 따라서: k >= log2(n!) >= n*log2(n) - n*log2(e)
5. O(n log n)이 비교 정렬의 이론적 하한!

Merge Sort, Heap Sort: O(n log n) → 최적!
Bubble Sort: O(n^2) → 비최적
Counting Sort: O(n+k) → 비교를 안 하므로 하한 해당 없음

[심화 내용] 3-way Comparison

C++20에서 도입된 우주선 연산자(<=>) 와 3-way comparison 개념이에요.

[심화 내용] 3-way Comparison 개념
기존 비교: 2개의 결과 (참/거짓)
3-way 비교: 3개의 결과 (음수/0/양수)

C의 strcmp: 이미 3-way! (음수, 0, 양수 반환)
C의 qsort comparator: 3-way 비교 함수 필요

int compare(const void *a, const void *b) {
return *(int*)a - *(int*)b; // 음수/0/양수
}

C++20 <=> (우주선 연산자):
auto result = a <=> b;
if (result < 0) ... // a < b
if (result == 0) ... // a == b
if (result > 0) ... // a > b

장점:
- 비교 1번으로 3가지 관계 모두 판단
- 6개의 비교 연산자를 자동 생성
- 정렬 알고리즘에서 비교 횟수 절감 가능

[심화 내용] 비교 연산의 브랜치리스 최적화

현대 CPU에서 조건 분기(if)는 파이프라인 스톨을 유발해요. 비교를 분기 없이 수행하는 기법이 있어요.

[심화 내용] 브랜치리스 비교
문제: if (a > b) x = a; else x = b;
→ 분기 예측 실패 시 ~20 사이클 손실

해결 1: CMOV (Conditional Move)
x86의 cmovg 명령: 분기 없이 조건부 이동
컴파일러가 자동으로 변환 (gcc -O2)

해결 2: 비트 트릭
max(a,b) = a ^ ((a ^ b) & -(a < b))
→ 분기 없이 max 계산!

해결 3: 삼항 연산자
x = (a > b) ? a : b;
→ 컴파일러가 CMOV로 최적화 가능

실전: 게임 엔진, 실시간 시스템, SIMD 코드에서 중요
일반 코드에서는 가독성 우선!
[C 코드] 브랜치리스 min/max
1#include <stdio.h>
2
3// 브랜치리스 max (분기 없음!)
4int branchless_max(int a, int b) {
5 int diff = a - b; // a > b이면 양수
6 int mask = diff >> 31; // 양수면 0x00000000, 음수면 0xFFFFFFFF
7 return a - (diff & mask); // a > b면 a, 아니면 b
8}
9
10// 브랜치리스 min
11int branchless_min(int a, int b) {
12 int diff = a - b;
13 int mask = diff >> 31;
14 return b + (diff & mask);
15}
16
17int main() {
18 printf("max(7, 3) = %d\n", branchless_max(7, 3));
19 printf("max(2, 9) = %d\n", branchless_max(2, 9));
20 printf("min(7, 3) = %d\n", branchless_min(7, 3));
21 printf("min(2, 9) = %d\n", branchless_min(2, 9));
22 return 0;
23}
실행 결과
max(7, 3) = 7 max(2, 9) = 9 min(7, 3) = 3 min(2, 9) = 2

주의: 위의 branchless_max는 오버플로우에 취약!

a - b가 int 범위를 넘으면 잘못된 결과!
