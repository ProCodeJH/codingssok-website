나눗셈의 두 가지 결과

사탕 7개를 3명에게 똑같이 나눠주면? 한 명에 2개씩, 그리고 1개가 남아! 이 '남는 것'이 바로 나머지야.

7 ÷ 3을 계산하면 몫은 2이고 나머지는 1이에요. 왜냐하면 3 × 2 = 6이고, 7 - 6 = 1이니까요.

7 ÷ 3 = 몫 2, 나머지 1 — 사탕을 나눠 주는 것과 같아요!
[핵심] 나눗셈 공식
나누어지는 수 = 나누는 수 × 몫 + 나머지

7 = 3 × 2 + 1 ✓
10 = 3 × 3 + 1 ✓
15 = 4 × 3 + 3 ✓
20 = 6 × 3 + 2 ✓
나눗셈 연습
식몫 (/)나머지 (%)검증
7 ÷ 3213×2+1=7 ✓
10 ÷ 4224×2+2=10 ✓
15 ÷ 5305×3+0=15 ✓
23 ÷ 7327×3+2=23 ✓
100 ÷ 33313×33+1=100 ✓

C언어의 / 와 %

C언어에서는 나눗셈의 몫과 나머지를 따로따로 구할 수 있어요.

div_mod.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 7, b = 3;
5
6 printf("%d / %d = %d (몫)\n", a, b, a / b); // 2
7 printf("%d %% %d = %d (나머지)\n", a, b, a % b); // 1
8
9 // 검증: 나누는수 × 몫 + 나머지 = 원래 수
10 printf("검증: %d × %d + %d = %d\n", b, a/b, a%b, b*(a/b) + a%b);
11
12 return 0;
13}
실행 결과
7 / 3 = 2 (몫) 7 % 3 = 1 (나머지) 검증: 3 × 2 + 1 = 7
[주의] 정수 나눗셈은 소수점을 버린다!
수학에서 7 ÷ 3 = 2.333...이지만
C언어에서 7 / 3 = 2 (소수점 아래를 버려요!)

이건 '버림(truncation)'이에요. 반올림이 아니에요!
9 / 5 = 1 (1.8이 아님!)
99 / 100 = 0 (0.99가 아님!)

이걸 모르면 KOI에서 틀릴 수 있어요!
99 / 100 = 0 이라고?! 맞아. C에서 정수끼리 나누면 소수점을 완전히 잘라버려. 이게 초보자가 제일 많이 당하는 함정이야!

나머지로 할 수 있는 마법

나머지 연산(%)은 프로그래밍에서 정말 많이 써! 짝수인지 홀수인지, 3의 배수인지... 전부 나머지로 알 수 있거든!
[핵심] 나머지의 마법
짝수/홀수 판별: x % 2 == 0이면 짝수, x % 2 == 1이면 홀수

N의 배수 판별: x % N == 0이면 x는 N의 배수
• x % 3 == 0 → 3의 배수
• x % 5 == 0 → 5의 배수
• x % 15 == 0 → 15의 배수 (3과 5의 공배수)
x % 2의 결과로 짝수와 홀수를 판별해요!

FizzBuzz — 나머지의 대표 문제

fizzbuzz.c
1#include <stdio.h>
2
3int main(void) {
4 for (int i = 1; i <= 15; i++) {
5 if (i % 15 == 0) printf("FizzBuzz");
6 else if (i % 3 == 0) printf("Fizz");
7 else if (i % 5 == 0) printf("Buzz");
8 else printf("%d", i);
9 printf(" ");
10 }
11 printf("\n");
12 return 0;
13}
실행 결과
1 2 Fizz 4 Buzz Fizz 7 8 Fizz Buzz 11 Fizz 13 14 FizzBuzz
[팁] FizzBuzz의 비밀
왜 i%15를 먼저 검사할까?

i%3과 i%5를 먼저 검사하면 15의 배수일 때 'Fizz'만 출력돼요!
15 = 3×5이므로 3의 배수이기도 하니까요.

순서가 중요해요! 더 까다로운 조건을 먼저 검사해야 해요.

시계 산술 — 나머지의 일상

시계를 보세요. 12시 다음은 1시이지, 13시가 아니에요 (12시간제). 이것도 나머지예요!

시계는 12로 나눈 나머지로 돌아가요! 13시 → 13 % 12 = 1시
[예제] 시계로 이해하는 나머지
지금 10시. 5시간 후는 몇 시?
(10 + 5) % 12 = 15 % 12 = 3시

지금 월요일(1). 10일 후 무슨 요일?
(1 + 10) % 7 = 11 % 7 = 4(목요일)

이런 '돌아가는 계산'을 모듈러 산술이라고 해요.

자릿수 분리

나머지와 몫을 조합하면 숫자의 각 자릿수를 분리할 수 있어요!

digits.c
1#include <stdio.h>
2
3int main(void) {
4 int num = 1234;
5
6 int ones = num % 10; // 4 (일의 자리)
7 int tens = (num / 10) % 10; // 3 (십의 자리)
8 int hundreds = (num / 100) % 10; // 2 (백의 자리)
9 int thousands = num / 1000; // 1 (천의 자리)
10
11 printf("%d의 자릿수: %d %d %d %d\n", num,
12 thousands, hundreds, tens, ones);
13 printf("자릿수 합: %d\n", thousands + hundreds + tens + ones);
14
15 return 0;
16}
실행 결과
1234의 자릿수: 1 2 3 4 자릿수 합: 10
1234 자릿수 분리 과정
연산결과의미
1234 % 104일의 자리
1234 / 10 = 123, 123 % 103십의 자리
1234 / 100 = 12, 12 % 102백의 자리
1234 / 10001천의 자리
패턴이 보이지? % 10은 맨 끝 자리를 꺼내고, / 10은 맨 끝 자리를 잘라내! 이 두 가지를 반복하면 모든 자릿수를 분리할 수 있어!

[심화] 윤년 판별 — 나머지의 실전

leap_year.c
1#include <stdio.h>
2
3int is_leap(int year) {
4 // 4로 나누어지고, 100으로 안 나누어지거나, 400으로 나누어지면 윤년
5 return (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);
6}
7
8int main(void) {
9 int years[] = {2000, 1900, 2024, 2025, 2100};
10 for (int i = 0; i < 5; i++) {
11 printf("%d년: %s\n", years[i], is_leap(years[i]) ? "윤년" : "평년");
12 }
13 return 0;
14}
실행 결과
2000년: 윤년 1900년: 평년 2024년: 윤년 2025년: 평년 2100년: 평년
[핵심] 윤년 규칙 (나머지 3단계)
1. year % 4 == 0 → 윤년 후보
2. year % 100 == 0 → 아니야, 평년! (예: 1900)
3. year % 400 == 0 → 아냐, 역시 윤년! (예: 2000)

[심화] 음수의 나머지

양수끼리의 나머지는 직관적이지만, 음수가 들어가면 복잡해져요.

negative_mod.c
1#include <stdio.h>
2
3int main(void) {
4 printf("-7 / 3 = %d\n", -7 / 3); // -2 (0방향 버림)
5 printf("-7 %% 3 = %d\n", -7 % 3); // -1 (음수!)
6 printf("7 / -3 = %d\n", 7 / -3); // -2
7 printf("7 %% -3 = %d\n", 7 % -3); // 1
8
9 // 검증: a == (a/b)*b + a%b
10 // -7 == (-7/3)*3 + (-7%3)
11 // -7 == (-2)*3 + (-1) = -6 + (-1) = -7 ✓
12
13 return 0;
14}
실행 결과
-7 / 3 = -2 -7 % 3 = -1 7 / -3 = -2 7 % -3 = 1
✕ 잘못된 코드

✓ 올바른 코드

[주의] 항상 양수 나머지가 필요하면
C에서 음수 나머지가 나올 수 있으니, 항상 양수를 원하면:
int mod = ((a % b) + b) % b;

예: ((-7 % 3) + 3) % 3 = (-1 + 3) % 3 = 2 % 3 = 2

[심화] 실수 나눗셈 vs 정수 나눗셈

int_vs_float_div.c
1#include <stdio.h>
2
3int main(void) {
4 // 정수 나눗셈 (소수점 버림)
5 printf("7 / 3 = %d\n", 7 / 3); // 2
6
7 // 실수 나눗셈 (소수점 유지)
8 printf("7.0 / 3 = %f\n", 7.0 / 3); // 2.333333
9 printf("7 / 3.0 = %f\n", 7 / 3.0); // 2.333333
10
11 // 캐스팅으로 실수 나눗셈
12 int a = 7, b = 3;
13 printf("(double)a / b = %f\n", (double)a / b); // 2.333333
14
15 // 주의! 이건 정수 나눗셈 후 변환
16 double wrong = a / b; // 2.0 (이미 2로 잘린 후 변환)
17 double right = (double)a / b; // 2.333...
18 printf("wrong: %f, right: %f\n", wrong, right);
19
20 return 0;
21}
실행 결과
7 / 3 = 2 7.0 / 3 = 2.333333 7 / 3.0 = 2.333333 (double)a / b = 2.333333 wrong: 2.000000, right: 2.333333
[핵심] 실수 나눗셈 받으려면
피연산자 중 하나라도 실수이면 실수 나눗셈!

• 7 / 3 = 2 (정수÷정수 = 정수)
• 7.0 / 3 = 2.333... (실수÷정수 = 실수)
• (double)7 / 3 = 2.333... (캐스팅)

⚠️ double x = 7/3; → x = 2.0 (이미 잘린 후!)

[심화 내용] 나눗셈과 비트 연산

[심화 내용] 2의 거듭제곱 나눗셈 = 비트 시프트
2로 나누기 = 오른쪽으로 1비트 시프트!

x / 2 = x >> 1
x / 4 = x >> 2
x / 8 = x >> 3
x / 2^n = x >> n

x % 2 = x & 1 (마지막 비트만!)
x % 4 = x & 3
x % 8 = x & 7
x % 2^n = x & (2^n - 1)

비트 연산은 나눗셈보다 훨씬 빨라요!

나눗셈은 CPU에서 가장 느린 산술 연산이에요.

[심화 내용] 모듈러 산술과 암호학

나머지 연산(모듈러)은 암호학의 핵심이에요.

[심화 내용] RSA 암호의 핵심 = 모듈러 거듭제곱
RSA 공개키 암호:
암호화: C = M^e mod n
복호화: M = C^d mod n

큰 수의 모듈러 거듭제곱은 빠르게 계산할 수 있지만,
n을 소인수분해하는 건 어려워요 → 이게 RSA의 안전성!

또한 해시 함수도 모듈러 사용:
hash(key) = key % table_size
[팁] 모듈러 산술의 다양한 응용
• 해시 테이블: index = key % size
• 체크섬: ISBN, 주민번호 검증
• 순환 버퍼: next = (current + 1) % size
• 색상: HSV 색상 각도 (0~359) → angle % 360
• 게임: 캐릭터가 맵 끝에서 반대편으로 나타남

[심화 내용] 모듈러 산술 — 나머지의 수학

나머지 연산을 체계적으로 다루는 것이 모듈러 산술(Modular Arithmetic)이에요. 현대 암호학의 핵심이죠!

[심화 내용] 모듈러 산술의 성질
a mod n에서 n을 법(modulus)이라 해요.

합동(Congruence):
a ≡ b (mod n) ⟺ n이 (a-b)를 나눔
예: 17 ≡ 5 (mod 12) (시계!)

연산 법칙:
(a + b) mod n = ((a mod n) + (b mod n)) mod n
(a * b) mod n = ((a mod n) * (b mod n)) mod n
(a - b) mod n = ((a mod n) - (b mod n) + n) mod n

주의: 나눗셈은 성립 안 함!
(a / b) mod n ≠ ((a mod n) / (b mod n)) mod n
대신 모듈러 역원을 사용
[심화 내용] 큰 수의 거듭제곱 모듈러
문제: 2^100 mod 13 = ?

직접 계산하면 2^100은 31자리 수... 불가능!

빠른 거듭제곱(Fast Exponentiation):
2^1 mod 13 = 2
2^2 mod 13 = 4
2^4 mod 13 = 16 mod 13 = 3
2^8 mod 13 = 3^2 mod 13 = 9
2^16 mod 13 = 9^2 mod 13 = 81 mod 13 = 3
2^32 mod 13 = 3^2 mod 13 = 9
2^64 mod 13 = 9^2 mod 13 = 3

100 = 64 + 32 + 4
2^100 = 2^64 * 2^32 * 2^4 mod 13
= 3 * 9 * 3 mod 13 = 81 mod 13 = 3
[C 코드] 빠른 거듭제곱 모듈러
1#include <stdio.h>
2
3// a^b mod m을 O(log b)에 계산
4long long power_mod(long long a, long long b, long long m) {
5 long long result = 1;
6 a %= m;
7 while (b > 0) {
8 if (b & 1) // b가 홀수면
9 result = result * a % m;
10 b >>= 1; // b를 반으로
11 a = a * a % m; // a를 제곱
12 }
13 return result;
14}
15
16int main() {
17 printf("2^100 mod 13 = %lld\n", power_mod(2, 100, 13));
18 printf("3^1000 mod 7 = %lld\n", power_mod(3, 1000, 7));
19 printf("7^256 mod 11 = %lld\n", power_mod(7, 256, 11));
20 return 0;
21}
실행 결과
2^100 mod 13 = 3 3^1000 mod 7 = 4 7^256 mod 11 = 3

[심화 내용] RSA 암호와 나머지

[심화 내용] RSA 암호의 원리
인터넷 보안의 핵심 RSA 암호는 모듈러 산술로 작동해요!

키 생성:
1. 큰 소수 p, q 선택 (각각 수백 자리)
2. n = p * q
3. e (공개키), d (비밀키) 계산
e * d ≡ 1 (mod (p-1)(q-1))

암호화: C = M^e mod n (공개키로)
복호화: M = C^d mod n (비밀키로)

안전한 이유:
n = p * q를 알아도 p, q를 구하기 극도로 어려움
(소인수분해 문제 = NP)

나머지 연산 없이는 현대 인터넷 보안이 불가능!

[심화 내용] 해시 함수와 나머지

해시 테이블의 핵심인 해시 함수도 나머지 연산을 사용해요.

[심화 내용] 해시 함수 = 나머지
가장 간단한 해시 함수:
hash(key) = key % table_size

예: 테이블 크기 10
hash(42) = 42 % 10 = 2 (인덱스 2에 저장)
hash(17) = 17 % 10 = 7 (인덱스 7에 저장)
hash(32) = 32 % 10 = 2 (충돌! 인덱스 2)

좋은 테이블 크기: 소수가 좋아요
table_size = 7이면 key % 7은 0~6이 균등 분포
table_size = 10이면 끝자리만 보므로 편향 발생

Java HashMap: 2의 거듭제곱 크기 + 비트 조작
Python dict: 오픈 어드레싱 + 소수 기반

[심화 내용] 나눗셈의 하드웨어 구현

[심화 내용] CPU의 나눗셈
덧셈/뺄셈: 1 사이클 (가장 빠름)
곱셈: 3~5 사이클
나눗셈: 20~90 사이클 (가장 느림!)

Restoring Division 알고리즘:
1. 피제수(나눠지는 수)에서 제수(나누는 수)를 뺌
2. 결과가 양수면 -> 몫 비트 = 1
3. 결과가 음수면 -> 몫 비트 = 0, 원래값 복원(restore)
4. 시프트하고 반복 (32비트면 32번)

컴파일러 최적화:
2의 거듭제곱 나눗셈 -> 시프트로 대체!
x / 8 -> x >> 3 (1 사이클!)
x % 8 -> x & 7 (비트 AND, 1 사이클!)

상수 나눗셈 -> 곱셈+시프트로 변환
x / 7 -> x * 0x24924925 >> 35 (마법의 수!)
