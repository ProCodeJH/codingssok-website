1차원: 수직선과 배열

수직선 기억해? 0, 1, 2, 3... 숫자가 줄지어 있는 선! 이거 코딩에서 배열이랑 똑같아!

수직선은 숫자를 한 줄로 나열한 것이에요. 코딩에서도 값을 한 줄로 나열하는 배열이 있어요. 배열에서 각 칸의 번호를 인덱스(index)라고 해요.

[핵심] 수직선 = 배열
수직선: ... -2, -1, 0, 1, 2, 3, 4, ...
배열: arr[0], arr[1], arr[2], arr[3], arr[4], ...

둘 다 '위치(번호)로 값을 찾는' 구조예요!
[주의] 인덱스는 0부터!
수학에서는 1번째, 2번째, 3번째... 라고 세지만,
C언어 배열은 0번째, 1번째, 2번째... 로 세요!

첫 번째 칸 = arr[0] (0번 인덱스!)
이걸 헷갈리면 off-by-one 오류라는 유명한 버그가 나요.
array_demo.c
1#include <stdio.h>
2
3int main(void) {
4 int arr[5] = {10, 20, 30, 40, 50};
5
6 // 인덱스: 0 1 2 3 4
7 // 값: 10 20 30 40 50
8
9 printf("첫 번째 원소: arr[0] = %d\n", arr[0]); // 10
10 printf("세 번째 원소: arr[2] = %d\n", arr[2]); // 30
11 printf("마지막 원소: arr[4] = %d\n", arr[4]); // 50
12 // arr[5]는 범위 초과! 접근하면 안 됨!
13
14 return 0;
15}

첫 번째 원소: arr[0] = 10 세 번째 원소: arr[2] = 30 마지막 원소: arr[4] = 50

2차원: (x, y) 좌표

수직선은 1차원이에요. 왼쪽-오른쪽만 있죠. 하지만 현실 세계는 2차원이에요. 왼쪽-오른쪽 + 위-아래! 이걸 표현하려면 숫자 2개가 필요해요: (x, y) 좌표.

[예제] 좌표로 위치 표현하기
교실의 좌석을 좌표로 나타내면:
• 창가 앞자리: (1, 1)
• 창가 뒷자리: (1, 5)
• 복도 앞자리: (6, 1)

체스판에서도 좌표를 써요: a1, b2, c3...
이것을 숫자로 바꾸면 (1,1), (2,2), (3,3)!

수학 좌표 vs 컴퓨터 좌표

여기가 핵심이야! 수학 좌표와 컴퓨터 좌표는 y축 방향이 반대야! 수학은 위로 갈수록 y가 커지는데, 컴퓨터는 아래로 갈수록 y가 커져!
수학에서 y는 위로 증가하지만, 컴퓨터에서 y는 아래로 증가해요!

2차원 배열 = 격자

2차원 좌표를 코딩에서는 2차원 배열로 표현해요. 바둑판이나 체스판처럼 행(row)과 열(column)로 이루어진 격자예요.

2차원 배열 arr[행][열] — 바둑판이나 엑셀 시트와 같아요!

2

[주의] [y][x] 순서!
수학에서는 (x, y)라고 쓰지만,
C언어 2차원 배열에서는 arr[y][x]로 써요!

(x=3, y=2) 위치 → arr[2][3]

x와 y 순서를 바꾸면 엉뚱한 위치를 가리키게 돼요!

거리 구하기

두 점 사이의 거리를 구하는 방법이 2가지 있어! 맨해튼 거리와 유클리드 거리야. 코딩 문제에서 둘 다 나와!
맨해튼 거리는 격자를 따라가고, 유클리드 거리는 직선으로 가요!
[핵심] 두 가지 거리
점 A(x₁, y₁)와 점 B(x₂, y₂) 사이:

맨해튼 거리 = |x₁-x₂| + |y₁-y₂|
→ 격자를 따라 가는 거리 (도시 블록)
→ (1,2)↔(4,6): |1-4|+|2-6| = 3+4 = 7

유클리드 거리 = √((x₁-x₂)²+(y₁-y₂)²)
→ 직선 거리 (새가 날아가듯)
→ (1,2)↔(4,6): √(9+16) = √25 = 5
distance.c
1#include <stdio.h>
2#include <math.h> // sqrt(), abs()
3#include <stdlib.h> // abs()
4
5int main(void) {
6 int x1=1, y1=2, x2=4, y2=6;
7
8 // 맨해튼 거리
9 int manhattan = abs(x1-x2) + abs(y1-y2);
10 printf("맨해튼: %d\n", manhattan); // 7
11
12 // 유클리드 거리
13 double euclidean = sqrt(pow(x1-x2, 2) + pow(y1-y2, 2));
14 printf("유클리드: %.2f\n", euclidean); // 5.00
15
16 return 0;
17}
실행 결과
맨해튼: 7 유클리드: 5.00

[심화] 4방향 / 8방향 이동

격자에서 이동할 때 방향 배열을 사용하면 깔끔하게 코드를 짤 수 있어요.

22}

4방향 이동은 KOI에서 거의 매 시험 나와요!

int dx[] = {0, 0, -1, 1};
int dy[] = {-1, 1, 0, 0};

경계 체크: if (nx >= 0 && nx < W && ny >= 0 && ny < H)

이 패턴을 외워두면 격자 탐색 문제의 절반은 풀린 거예요!

[심화] 체스 나이트의 이동

체스 나이트는 'L자'로 움직여요. 8방향 이동의 변형!

knight.c
1#include <stdio.h>
2
3int main(void) {
4 // 나이트 8가지 이동
5 int dx[] = {-2, -2, -1, -1, 1, 1, 2, 2};
6 int dy[] = {-1, 1, -2, 2, -2, 2, -1, 1};
7
8 int x = 4, y = 4; // 현재 위치
9 printf("나이트 (%d,%d)의 가능한 이동:\n", x, y);
10
11 for (int i = 0; i < 8; i++) {
12 int nx = x + dx[i];
13 int ny = y + dy[i];
14 if (nx >= 0 && nx < 8 && ny >= 0 && ny < 8)
15 printf(" → (%d, %d)\n", nx, ny);
16 }
17
18 return 0;
19}
[KOI] 나이트 이동 = BFS 문제
"나이트가 (a,b)에서 (c,d)까지 최소 몇 번에 갈 수 있나?"
→ BFS(너비 우선 탐색)로 풀어요!

1. 시작점을 큐에 넣고 이동 횟수 0
2. 큐에서 꺼내고 8방향으로 이동
3. 방문 안 한 곳이면 큐에 넣고 횟수+1
4. 도착하면 그때의 횟수가 최소!

[심화] 미로 탈출 — BFS 맛보기

maze_bfs.c
1#include <stdio.h>
2#include <string.h>
3
4#define N 5
5
6int maze[N][N] = {
7 {0, 1, 0, 0, 0},
8 {0, 1, 0, 1, 0},
9 {0, 0, 0, 1, 0},
10 {0, 1, 0, 0, 0},
11 {0, 0, 0, 1, 0}
12}; // 0=길, 1=벽
13
14int dist[N][N]; // 거리 저장
15int dx[] = {0, 0, -1, 1};
16int dy[] = {-1, 1, 0, 0};
17
18int main(void) {
19 // BFS 큐 (간단한 배열 큐)
20 int qx[N*N], qy[N*N];
21 int front = 0, rear = 0;
22
23 memset(dist, -1, sizeof(dist));
24 dist[0][0] = 0;
25 qx[rear] = 0; qy[rear] = 0; rear++;
26
27 while (front < rear) {
28 int x = qx[front], y = qy[front]; front++;
29 for (int i = 0; i < 4; i++) {
30 int nx = x + dx[i], ny = y + dy[i];
31 if (nx >= 0 && nx < N && ny >= 0 && ny < N
32 && maze[ny][nx] == 0 && dist[ny][nx] == -1) {
33 dist[ny][nx] = dist[y][x] + 1;
34 qx[rear] = nx; qy[rear] = ny; rear++;
35 }
36 }
37 }
38
39 printf("(0,0)→(4,4) 최소 거리: %d\n", dist[4][4]);
40 return 0;
41}
실행 결과
(0,0)→(4,4) 최소 거리: 8

[심화] 3차원 좌표

현실 세계는 3차원! (x, y, z)로 위치를 표현해요.

[심화] 3차원 좌표
2차원: (x, y) — 평면
3차원: (x, y, z) — 공간

맨해튼 거리: |x₁-x₂| + |y₁-y₂| + |z₁-z₂|
유클리드 거리: √((x₁-x₂)² + (y₁-y₂)² + (z₁-z₂)²)

활용: 3D 게임, VR, 로보틱스, 지도(위도/경도/고도)

[심화 내용] 극좌표와 좌표 변환

(x, y) 대신 각도(θ)와 거리(r)로 위치를 표현할 수 있어요. 이것이 극좌표(polar coordinates)예요.

[심화 내용] 직교 ↔ 극좌표 변환
직교 → 극좌표:
r = √(x² + y²)
θ = atan2(y, x)

극좌표 → 직교:
x = r × cos(θ)
y = r × sin(θ)

활용: 레이더, 나선 그리기, 회전 애니메이션, 물리 시뮬레이션

[심화 내용] 동차 좌표와 변환 행렬

[심화 내용] 동차 좌표 (Homogeneous Coordinates)
2D 점 (x, y)를 3개 숫자 (x, y, 1)로 표현하면
이동, 회전, 확대를 모두 행렬 곱으로 처리 가능!

이동(tx, ty): [1 0 tx] [x] [x+tx]
[0 1 ty] [y] = [y+ty]
[0 0 1] [1] [ 1 ]

3D 그래픽(OpenGL, DirectX)의 핵심!
모든 변환을 하나의 4×4 행렬로 합칠 수 있어요.

[심화 내용] 다양한 좌표계

[심화 내용] 좌표계의 종류
1. 데카르트 좌표 (Cartesian)
(x, y) — 가장 기본. 직교 좌표계

2. 극좌표 (Polar)
(r, theta) — 원점에서의 거리 r, 각도 theta
변환: x = r*cos(theta), y = r*sin(theta)
원, 나선 표현에 유리!

3. 3D 좌표
(x, y, z) — 깊이 추가
게임, 3D 모델링에서 필수

4. 동차 좌표 (Homogeneous)
(x, y, w) — 2D 점을 3D로 확장
이동(Translation)을 행렬 곱으로 표현 가능!
3D 그래픽: (x, y, z, w) — 4x4 행렬로 모든 변환 통합

[심화 내용] 행렬 변환

좌표의 이동, 회전, 확대/축소를 행렬 곱셈으로 표현할 수 있어요. 게임 엔진의 핵심이죠!

[심화 내용] 2D 변환 행렬
이동 (Translation):
[1 0 tx] [x] [x+tx]
[0 1 ty] [y] = [y+ty]
[0 0 1] [1] [ 1 ]

회전 (Rotation, theta):
[cos -sin 0] [x] [x*cos - y*sin]
[sin cos 0] [y] = [x*sin + y*cos]
[ 0 0 1] [1] [ 1 ]

스케일 (Scale):
[sx 0 0] [x] [x*sx]
[ 0 sy 0] [y] = [y*sy]
[ 0 0 1] [1] [ 1 ]

여러 변환 = 행렬 곱의 연쇄!
GPU가 초당 수십억 번의 행렬 곱을 수행!
[C 코드] 2D 회전 구현
1#include <stdio.h>
2#include <math.h>
3
4typedef struct { double x, y; } Point;
5
6// 원점 기준 회전
7Point rotate(Point p, double degrees) {
8 double rad = degrees * 3.14159265 / 180.0;
9 Point r;
10 r.x = p.x * cos(rad) - p.y * sin(rad);
11 r.y = p.x * sin(rad) + p.y * cos(rad);
12 return r;
13}
14
15int main() {
16 Point p = {1.0, 0.0};
17 printf("원점: (%.2f, %.2f)\n", p.x, p.y);
18
19 for (int deg = 0; deg <= 360; deg += 90) {
20 Point r = rotate(p, deg);
21 printf("%3d도 회전: (%.2f, %.2f)\n", deg, r.x, r.y);
22 }
23 return 0;
24}
실행 결과
원점: (1.00, 0.00) 0도 회전: (1.00, 0.00) 90도 회전: (0.00, 1.00) 180도 회전: (-1.00, 0.00) 270도 회전: (0.00, -1.00) 360도 회전: (1.00, 0.00)

[심화 내용] 거리 함수의 종류

[심화 내용] 다양한 거리 함수
1. 유클리드 거리 (L2)
d = sqrt((x2-x1)^2 + (y2-y1)^2)
직선 거리. 가장 일반적

2. 맨해튼 거리 (L1)
d = |x2-x1| + |y2-y1|
격자 위의 이동 거리. 도시 블록
BFS/그래프에서 자주 사용

3. 체비셰프 거리 (L-infinity)
d = max(|x2-x1|, |y2-y1|)
체스 킹의 이동 거리. 대각선 1칸

4. 민코프스키 거리 (Lp)
d = (|x2-x1|^p + |y2-y1|^p)^(1/p)
p=1이면 맨해튼, p=2이면 유클리드, p=inf이면 체비셰프

KOI/대회에서 "두 점의 거리"라 하면 보통 맨해튼이에요!
