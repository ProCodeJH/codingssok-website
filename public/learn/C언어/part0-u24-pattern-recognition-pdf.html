패턴이 뭘까?

타일 바닥을 본 적 있지? 같은 무늬가 반복되잖아! 이렇게 반복되는 규칙을 패턴이라고 해. 패턴을 찾으면 엄청 효율적으로 일을 할 수 있어!

패턴 인식이란 데이터나 문제 속에서 반복되는 규칙을 찾는 것이에요. 패턴을 발견하면 같은 코드를 여러 번 쓰는 대신 반복문 하나로 끝낼 수 있어요!

[핵심] 패턴 인식이란?
패턴 인식(Pattern Recognition) = 데이터나 문제에서 반복되는 규칙을 찾는 것

패턴을 찾으면?
1. 다음에 뭐가 올지 예측할 수 있어요
2. 반복 코드를 반복문으로 줄일 수 있어요
3. 규칙을 공식으로 만들 수 있어요
4. 비슷한 문제를 같은 방법으로 풀 수 있어요
패턴은 어디에나 있어요 — 타일, 숫자, 시간표에서!

일상에서 패턴 찾기

신호등, 요일, 체육수업, 노래 후렴, 구구단 — 모두 패턴!
일상 속 패턴 분석
패턴규칙주기C로 표현
신호등초록→노랑→빨강3단계 반복while + switch
요일월→화→...→일7일 주기day % 7
구구단N×1, N×2, ..., N×99번 반복for(i=1;i<=9;i++)
시계1→2→...→12→112시간 주기hour % 12
계절봄→여름→가을→겨울4계절 반복season % 4

코드에서 패턴 찾기 → 반복문!

같은 구조가 반복되는 코드를 발견하면? 반복문(for, while)으로 바꿀 수 있어요!

같은 구조 반복? → for문 하나로 끝!
[핵심] 패턴 → 반복문 변환 3단계
1단계: 뭐가 반복되는지 찾기 (구조)
2단계: 뭐가 변하는지 찾기 (변수)
3단계: for(시작; 조건; 변화)로 표현하기

예: printf("1번"), printf("2번"), ..., printf("30번")
→ 반복 구조: printf("%d번")
→ 변하는 것: 숫자 1→30
→ for(int i=1; i<=30; i++) printf("%d번", i);

구구단 — 2중 패턴

gugudan.c
1#include <stdio.h>
2
3int main(void) {
4 // 바깥 패턴: 단(2~9)
5 for (int dan = 2; dan <= 9; dan++) {
6 printf("--- %d단 ---\n", dan);
7 // 안쪽 패턴: 곱(1~9)
8 for (int i = 1; i <= 9; i++)
9 printf("%d x %d = %d\n", dan, i, dan * i);
10 printf("\n");
11 }
12 return 0;
13}
[팁] 중첩 반복문 = 패턴 안의 패턴
구구단은 패턴 안에 패턴이 있어요:
• 바깥: 단이 2→3→...→9 (8번 반복)
• 안쪽: 곱이 1→2→...→9 (각 단마다 9번 반복)
• 총: 8 × 9 = 72번 출력

2중 for문 = 격자(2차원) 탐색의 기본!

데이터에서 패턴 찾기 → 공식!

데이터 관찰 → 차이 분석 → 공식 발견!
[핵심] 차이 분석법
수열: 3, 5, 7, 9, 11, ...
1차 차이: 2, 2, 2, 2 → 일정! → 등차수열
공식: aₙ = 3 + (n-1)×2 = 2n+1

수열: 1, 4, 9, 16, 25, ...
1차 차이: 3, 5, 7, 9 → 일정하지 않음
2차 차이: 2, 2, 2 → 일정! → 2차식
공식: aₙ = n²

[심화] DRY 원칙 — 반복하지 마!

[심화] DRY = Don't Repeat Yourself
DRY 원칙: 같은 코드를 두 번 쓰지 마라!

반복이 보이면 → 반복문 또는 함수로 추출

❌ WET (Write Everything Twice): 복붙 코드
✅ DRY: 패턴을 찾아 추상화한 코드

DRY를 지키면:
• 코드 양 감소
• 수정할 때 한 곳만 고침
• 버그 가능성 감소
dry_example.c
1// ❌ WET — 같은 패턴 3번 반복
2int sum1 = 0;
3for (int i = 0; i < n1; i++) sum1 += arr1[i];
4int sum2 = 0;
5for (int i = 0; i < n2; i++) sum2 += arr2[i];
6int sum3 = 0;
7for (int i = 0; i < n3; i++) sum3 += arr3[i];
8
9// ✅ DRY — 패턴을 함수로 추출
10int arraySum(int arr[], int n) {
11 int sum = 0;
12 for (int i = 0; i < n; i++) sum += arr[i];
13 return sum;
14}
15int sum1 = arraySum(arr1, n1);
16int sum2 = arraySum(arr2, n2);
17int sum3 = arraySum(arr3, n3);

[심화] 패턴 매칭 — 문자열에서 찾기

숫자뿐 아니라 문자열에서도 패턴을 찾을 수 있어요.

string_pattern.c
1#include <stdio.h>
2#include <string.h>
3
4// 문자열에서 특정 문자 세기 (패턴!)
5int countChar(const char *str, char c) {
6 int count = 0;
7 for (int i = 0; str[i] != '\0'; i++)
8 if (str[i] == c) count++;
9 return count;
10}
11
12int main(void) {
13 char text[] = "hello world";
14 printf("'l' 개수: %d\n", countChar(text, 'l')); // 3
15 printf("'o' 개수: %d\n", countChar(text, 'o')); // 2
16 return 0;
17}

[심화] 코드 냄새 — 나쁜 패턴 찾기

[심화] 코드 냄새 (Code Smell)
코드 냄새 = 코드에서 문제가 될 수 있는 나쁜 패턴

대표적인 코드 냄새:
• 중복 코드: 같은 코드가 여러 곳에 (DRY 위반)
• 긴 함수: 100줄 넘는 함수 (분해 필요)
• 매직 넘버: 의미 불명의 숫자 (if x > 365 → MAX_DAYS)
• 죽은 코드: 실행되지 않는 코드
• 깊은 중첩: if 안 if 안 if...
magic_number.c
1// ❌ 매직 넘버 — 365가 뭐야?
2if (days > 365) {
3 years++;
4 days -= 365;
5}
6
7// ✅ 상수로 의미 부여
8#define DAYS_PER_YEAR 365
9if (days > DAYS_PER_YEAR) {
10 years++;
11 days -= DAYS_PER_YEAR;
12}

[심화 내용] 정규표현식 — 문자열 패턴의 끝판왕

[심화 내용] 정규표현식 (Regular Expression)
문자열에서 복잡한 패턴을 찾는 도구!

기본 문법:
• . — 아무 문자 1개
• * — 앞 문자 0번 이상 반복
• + — 앞 문자 1번 이상 반복
• [abc] — a, b, c 중 하나
• [0-9] — 숫자 하나
• \d — 숫자 (=[0-9])

예: 전화번호 패턴 → \d{3}-\d{4}-\d{4}
이메일 패턴 → [a-zA-Z]+@[a-z]+\.[a-z]+

[심화 내용] 디자인 패턴 — 코드 설계의 패턴

[심화 내용] 디자인 패턴이란?
프로그래밍에서 자주 나오는 설계 문제의 표준 해결법

GoF(Gang of Four) 23개 패턴 중 유명한 것들:
• Singleton: 인스턴스 1개만 만들기
• Observer: 변화를 자동 알림 (이벤트 시스템)
• Factory: 객체 생성을 대신하는 함수
• Iterator: 컬렉션 순회 패턴
• Strategy: 알고리즘을 교체 가능하게

C에서도 함수 포인터로 일부 패턴 구현 가능!

KOI 연결: 수열 패턴 찾기

[KOI] 패턴 찾기 문제 공략
KOI 1차 필기에서 수열 문제가 자주 나와요!

풀이 순서:
1. 연속된 수의 차이를 구한다
2. 차이가 일정 → 등차수열 (1차 공식)
3. 차이의 차이가 일정 → 2차 공식
4. 비율이 일정 → 등비수열
5. 앞 두 수의 합 → 피보나치형

[심화] 정규 표현식 — 텍스트 패턴 매칭

[심화] 정규 표현식 기초
정규 표현식(Regex) = 텍스트 패턴을 기술하는 언어

기본 문법:
. = 아무 문자 1개
* = 앞 문자 0회 이상 반복
+ = 앞 문자 1회 이상 반복
[abc] = a, b, c 중 하나
[0-9] = 숫자 하나
\d = 숫자 ([0-9]와 같음)
^ = 시작, $ = 끝

예:
\d{3}-\d{4} = 전화번호 패턴 (010-1234)
[a-zA-Z]+ = 영어 단어
.*@.*\.com = 이메일 패턴

grep, sed, Python re, C의 regex.h에서 사용!

[심화 내용] 코드 패턴과 복잡도
코드의 구조적 패턴을 보면 복잡도를 바로 알 수 있어요!

O(1): 반복문 없음, 인덱스 접근
O(log n): 매번 절반으로 줄어듦 (이진탐색)
O(n): for 1개 (전체 순회)
O(n log n): 분할정복 + 합치기 (머지소트)
O(n^2): 중첩 for 2개 (버블소트)
O(n^3): 중첩 for 3개 (행렬곱)
O(2^n): 재귀가 2갈래로 분기 (피보나치 재귀)
O(n!): 모든 순열 생성

패턴을 보면 바로 복잡도를 "인식"할 수 있어요!

[심화] 코드 패턴 인식 요령
자릿수 분리 패턴:
n % 10 → 마지막 자릿수, n /= 10 → 제거
이 패턴이 보이면 = 자릿수 관련 작업!

누적 패턴:
sum += something → 합계 계산
count++ → 개수 세기
max = (x > max) ? x : max → 최댓값

쌍 비교 패턴:
arr[i]와 arr[i+1] 비교 → 인접 원소 관계

투 포인터 패턴:
left++, right-- → 양쪽에서 좁혀가기

패턴을 많이 외우면 코드를 빠르게 읽을 수 있어요!

[심화 내용] 오토마타 이론과 패턴

[심화 내용] 유한 오토마타 (FSM)
유한 상태 기계(Finite State Machine)
= 패턴 인식의 수학적 모델

구성요소:
1. 상태(States): 유한 개의 상태 집합
2. 입력(Alphabet): 받아들이는 문자 집합
3. 전이(Transition): 입력에 따른 상태 변화
4. 시작 상태: 처음 시작하는 상태
5. 받아들이는 상태: 패턴 매칭 성공

정규 표현식 = 유한 오토마타 = 같은 것!
(Kleene의 정리, 1956)

컴파일러의 렉서(Lexer)가 이것으로 토큰을 인식해요.
int, if, while 같은 키워드도 FSM으로 구분!
[심화 내용] 촘스키 계층
형식 언어의 4단계 계층 (Chomsky Hierarchy):

Type 3: 정규 언어 (정규식, FSM)
예: a*b+ (aab, abbb, ...)
한계: 괄호 짝 맞추기 불가!

Type 2: 문맥 자유 언어 (CFG, PDA)
예: 올바른 괄호 (()), 프로그래밍 언어 문법
컴파일러의 파서가 처리

Type 1: 문맥 의존 언어 (CSG, LBA)
예: a^n b^n c^n

Type 0: 재귀 열거 가능 언어 (튜링 머신)
모든 계산 가능한 것

Type 3 ⊂ Type 2 ⊂ Type 1 ⊂ Type 0
패턴 인식은 프로그래밍뿐 아니라 수학, 과학, 예술, 심지어 일상생활에서도 핵심이야! 반복을 보면 규칙을 찾고, 규칙을 찾으면 예측할 수 있어!
4단계
촘스키 언어 계층: 정규 -> 문맥자유 -> 문맥의존 -> 재귀열거. 컴퓨터가 인식할 수 있는 패턴의 한계!

DRY

코드 냄새

매직 넘버
코드에 의미 없이 쓰인 숫자. 상수(#define)로 이름을 붙여야.
중첩 반복문
