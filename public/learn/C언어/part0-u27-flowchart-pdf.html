순서도 (Flowchart)

순서도는 알고리즘을 그림으로 표현한 것이에요. 순서도 기호를 배우고, 순서도를 C 코드로 변환해 봐요!

  • 순서도의 기본 기호(시작/끝, 처리, 판단, 입출력)를 알아요.
  • 간단한 알고리즘을 순서도로 그릴 수 있어요.
  • 순서도를 읽고 실행 결과를 추적할 수 있어요.
  • [심화] 반복(루프)을 순서도로 표현할 수 있어요.
  • [심화] 순서도를 C 코드로 변환할 수 있어요.
  • [심화 내용] UML 활동 다이어그램과 순서도의 관계를 알아요.
  • [심화 내용] 상태 다이어그램(FSM)을 이해해요.
  • [심화 내용] BPMN과 비즈니스 프로세스 모델링을 알아요.
  • [심화 내용] 순서도의 한계와 구조적 프로그래밍의 탄생을 이해해요.

순서도가 뭘까?

알고리즘을 말로 쓰면 길어지잖아? 그걸 그림으로 그리면 한눈에 보여! 이게 순서도(Flowchart)야. KOI 1차 필기에 매번 나오니까 확실히 외워!

순서도는 알고리즘의 흐름을 그림 기호로 표현한 것이에요. 기호마다 역할이 정해져 있어서, 기호만 알면 누구나 같은 의미로 읽을 수 있어요.

[핵심] 순서도란?
순서도(Flowchart) = 알고리즘을 그림 기호와 화살표로 표현한 것

왜 순서도를 쓸까?
1. 알고리즘의 흐름을 한눈에 볼 수 있어요
2. 코드로 바꾸기 쉬워요
3. 다른 사람과 소통할 때 편리해요
4. KOI 1차 필기에 매번 출제돼요!

순서도 기호 5가지

이 5가지만 알면 모든 순서도를 읽고 그릴 수 있어요!
[KOI 필수 암기] 순서도 기호
타원 = 시작/끝 → C: main() 시작과 return
사각형 = 처리 → C: 대입, 계산 (a = b + c;)
마름모 = 판단(조건) → C: if, while 조건
평행사변형 = 입출력 → C: scanf, printf
화살표 = 실행 흐름 방향

KOI 1차에서 기호의 이름이나 역할을 묻는 문제가 나와요!
기호 ↔ C코드 대응표
기호이름역할C 코드 대응
⬭타원시작/끝int main() { ... return 0; }
▭사각형처리/계산x = a + b;
◇마름모판단/조건if (x > 0) / while (i < n)
▱평행사변형입력/출력scanf() / printf()
→화살표흐름 방향코드 실행 순서

3가지 기본 구조

모든 프로그램은 딱 3가지 구조의 조합으로 만들 수 있어요.

① 순차: 위→아래 / ② 선택: Y자 분기 / ③ 반복: 화살표가 위로
[핵심] 3가지 기본 구조
① 순차 구조 — 위에서 아래로 순서대로 실행
C: 문장1; 문장2; 문장3;

② 선택 구조 — 조건에 따라 다른 길로
C: if (조건) { ... } else { ... }

③ 반복 구조 — 조건 만족하면 되돌아가서 반복
C: while (조건) { ... } 또는 for (...) { ... }
1966
1966년 뵘(Böhm)과 야코피니(Jacopini)가 증명: 순차+선택+반복 3가지만 있으면 어떤 알고리즘이든 표현 가능! 이것이 구조적 프로그래밍 정리예요.

순서도 → C 코드 변환

순서도를 보고 코드를 쓸 수 있고, 코드를 보고 순서도를 그릴 수 있으면 진짜 실력자야!
기호 하나가 C 코드 한 줄에 대응!
[변환 공식] 순서도 → C 코드
1. 타원(시작) → int main() {
2. 평행사변형(입력) → scanf(...);
3. 사각형(처리) → 대입문 (변수 = 값;)
4. 마름모(판단) → if (...) 또는 while (...)
5. 마름모에서 되돌아가는 화살표 → while/for 반복
6. 평행사변형(출력) → printf(...);
7. 타원(끝) → return 0; }
flowchart_even_odd.c
1// 순서도: 시작→n입력→r=n%2→[r==0?]→예:짝수출력/아니오:홀수출력→끝
2#include <stdio.h>
3
4int main(void) {
5 int n; // 평행사변형: 입력
6 scanf("%d", &n);
7 int r = n % 2; // 사각형: 처리
8 if (r == 0) // 마름모: 판단
9 printf("짝수\n"); // 평행사변형: 출력
10 else
11 printf("홀수\n");
12 return 0; // 타원: 끝
13}

반복 순서도 → C 코드

마름모에서 '예'면 처리 후 돌아오고 '아니오'면 빠져나가요!
flowchart_sum.c
1// 순서도: 시작→s=0,i=1→[i≤10?]→예:s=s+i,i=i+1→되돌아감/아니오:s출력→끝
2#include <stdio.h>
3
4int main(void) {
5 int s = 0, i = 1; // 사각형: 초기화
6 while (i <= 10) { // 마름모: 조건 판단
7 s = s + i; // 사각형: 처리
8 i = i + 1; // 사각형: 처리
9 }
10 printf("%d\n", s); // 평행사변형: 출력 (55)
11 return 0;
12}
실행 결과
55

순서도를 따라가며 변수 값 변화를 표로 기록하면 정확한 답을 구할 수 있어요.

1~5 합 순서도 추적
단계isi≤5?동작

2회23예s=1+2=3, i→3
3회36예s=3+3=6, i→4
4회410예s=6+4=10, i→5
5회515예s=10+5=15, i→6
끝615아니오s 출력 → 15
[팁] 순서도 추적 = 디버깅의 기초
순서도 추적은 코드 디버깅과 같아요!

1. 변수 초기값 기록
2. 한 단계씩 따라가며 값 변경
3. 조건 판단 결과(예/아니오) 기록
4. 최종 출력값 확인

이 습관이 코딩할 때 버그를 빨리 찾게 해줘요!

1// 성적 판별 순서도 (중첩 마름모)
2// [score≥90?]→예:A / 아니오→[score≥80?]→예:B / 아니오→[score≥70?]→예:C / 아니오:F

[심화] 중첩 조건의 순서도 패턴
else if 체인 = 마름모가 계단식으로 이어짐

마름모1 → 아니오 → 마름모2 → 아니오 → 마름모3 → 아니오 → 기본값

각 '예' 경로에서 처리 후 모두 같은 합류점으로!

[심화] 중첩 반복 순서도

flowchart_nested_loop.c
1// 구구단(2~3단) 순서도: 바깥 반복(단) 안에 안쪽 반복(곱)
2#include <stdio.h>
3
4int main(void) {
5 for (int dan = 2; dan <= 3; dan++) { // 바깥 마름모
6 for (int i = 1; i <= 9; i++) { // 안쪽 마름모
7 printf("%d x %d = %d\n", dan, i, dan * i);
8 }
9 }
10 return 0;
11}
[주의] 중첩 순서도 그리기
중첩 반복 순서도를 그릴 때:

1. 바깥 반복의 마름모가 먼저
2. '예' 안에 안쪽 반복의 마름모
3. 안쪽 반복이 끝나면 바깥 반복의 증가로
4. 화살표가 꼬이지 않게 주의!

[심화 내용] 구조적 프로그래밍 정리

[심화 내용] Böhm-Jacopini 정리 (1966)
"모든 계산 가능한 함수는 순차, 선택, 반복 세 가지 제어 구조만으로 표현할 수 있다."

의미: goto 없이도 모든 프로그램을 쓸 수 있다!

이 정리가 goto 논쟁의 이론적 근거:
• Dijkstra(1968): "Go To Statement Considered Harmful"
• goto를 안 써도 되니까 쓰지 말자!
• C에는 goto가 있지만 '거의 안 쓴다'가 관례

[심화 내용] UML Activity Diagram

UML(Unified Modeling Language)의 활동 다이어그램은
순서도의 현대적 확장이에요.

추가된 기능:
1. 포크/조인: 병렬 처리 표현

[심화 내용] 순서도 → 구조적 프로그래밍
순서도의 문제:
1. 화살표가 어디로든 갈 수 있음 (스파게티 코드!)
2. GOTO와 직접 대응 → 읽기 어려운 코드
3. 큰 프로그램에서 순서도가 복잡해짐

뵘-야코피니 정리 (1966):
모든 알고리즘은 3가지 구조만으로 표현 가능!
1. 순차 (Sequence): 위에서 아래로
2. 선택 (Selection): if-else
3. 반복 (Iteration): while/for

GOTO가 필요 없다! → 구조적 프로그래밍의 시작

다익스트라 (1968):
"GOTO Statement Considered Harmful"
→ GOTO 대신 구조적 제어문 사용 운동
[C 코드] 순서도 → C 코드 변환 예제
1// 순서도: 1~N까지 합 구하기
2// [시작] → [N 입력] → [sum=0, i=1]
3// → {i<=N?} --예--> [sum+=i, i++] → {i<=N?}
4// --아니오--> [sum 출력] → [끝]
5
6#include <stdio.h>
7
8int main() {
9 int N;
10 printf("N 입력: ");
11 scanf("%d", &N);
12
13 int sum = 0; // 처리 블록
14 int i = 1; // 처리 블록
15 while (i <= N) { // 판단 마름모
16 sum += i; // 처리 블록
17 i++; // 처리 블록
18 }
19 printf("1~%d 합 = %d\n", N, sum); // 출력 블록
20
21 // 공식으로 검증
22 printf("검증: N*(N+1)/2 = %d\n", N*(N+1)/2);
23 return 0;
24}
실행 결과
N 입력: 100 1~100 합 = 5050 검증: N*(N+1)/2 = 5050

[심화 내용] 페트리 넷 — 병렬 시스템 모델링

[심화 내용] 페트리 넷 (Petri Net)
순서도가 순차 실행을 모델링한다면,
페트리 넷은 병렬/동시 실행을 모델링해요.

구성요소:
- 장소(Place): 조건/상태 (원으로 표현)
- 전이(Transition): 이벤트/동작 (사각형)
- 토큰(Token): 현재 상태를 나타내는 점
- 아크: 장소↔전이 연결

활용:
- 멀티스레드 프로그램 검증
- 데드락(교착상태) 탐지
- 워크플로우 시스템
- 네트워크 프로토콜 검증
