탐욕법이 뭘까?

뷔페에서 접시에 음식을 담을 때, 매번 가장 맛있어 보이는 것부터 담지 않아? 이게 탐욕법이야! 매 순간 지금 가장 좋은 것을 고르는 전략!
[핵심] 탐욕법이란?
탐욕법(Greedy Algorithm) = 매 순간 가장 좋아 보이는 선택을 하는 전략

특징:
1. 한 번 선택하면 되돌리지 않음
2. 현재 상황에서 최선을 선택
3. 빠르다! 보통 O(n) 또는 O(n log n)

주의: 항상 전체 최적이 되는 건 아니에요!

예시 1: 거스름돈 문제

greedy_coin.c
1#include <stdio.h>
2
3int main(void) {
4 int coins[] = {500, 100, 50, 10};
5 int n = 4;
6 int amount = 1260;
7 int count = 0;
8
9 printf("%d원 거슬러주기:\n", amount);
10 for (int i = 0; i < n; i++) {
11 int use = amount / coins[i]; // 가장 큰 동전부터!
12 if (use > 0) {
13 printf(" %d원 × %d개\n", coins[i], use);
14 count += use;
15 amount -= coins[i] * use;
16 }
17 }
18 printf("총 동전 %d개\n", count);
19 return 0;
20}
실행 결과
1260원 거슬러주기: 500원 × 2개 100원 × 2개 50원 × 1개 10원 × 1개 총 동전 6개
큰 동전부터 쓰면 동전 수가 최소! 이게 바로 그리디야. 근데 항상 이게 정답일까?

그리디가 실패하는 경우!

[주의] 그리디는 항상 최적이 아니야!
동전이 {500, 400, 100, 10}일 때:

800원 거스름돈:
• 그리디: 500 + 100×3 = 4개
• 최적: 400 + 400 = 2개!

그리디가 실패한 이유:
큰 동전을 쓴 결과가 나중에 더 안 좋은 결과를 만듦

예시 2: 회의실 배정 (활동 선택)

여러 회의가 있을 때, 최대한 많은 회의를 하려면?

greedy_meeting.c
1#include <stdio.h>
2
3int main(void) {
4 // 회의: {시작, 끝}
5 int start[] = {1, 3, 0, 5, 3, 5, 6, 8};
6 int end[] = {4, 5, 6, 7, 8, 9, 10, 11};
7 int n = 8;
8
9 // 그리디: 끝나는 시간이 빠른 순으로 선택!
10 // (이미 끝 시간 기준 정렬되어 있다고 가정)
11 int count = 0;
12 int lastEnd = 0; // 마지막 회의 끝난 시간
13
14 for (int i = 0; i < n; i++) {
15 if (start[i] >= lastEnd) { // 안 겹치면 선택!
16 printf("회의 %d: %d~%d 선택\n", i+1, start[i], end[i]);
17 lastEnd = end[i];
18 count++;
19 }
20 }
21 printf("최대 %d개 회의 가능\n", count);
22 return 0;
23}
실행 결과
회의 1: 1~4 선택 회의 2: 5~7 선택 회의 3: 8~11 선택 최대 3개 회의 가능
[핵심] 활동 선택의 그리디 전략
핵심: 끝나는 시간이 빠른 것부터 선택!

왜 이게 최적인가?
→ 빨리 끝나는 활동을 고르면 남은 시간이 더 많아져서
→ 더 많은 활동을 넣을 수 있으니까!

이 경우 그리디가 증명 가능하게 최적이에요.

예시 3: 분할 가능 배낭 문제

greedy_knapsack.c
1#include <stdio.h>
2
3int main(void) {
4 // 물건: {무게, 가치}
5 float weight[] = {10, 20, 30};
6 float value[] = {60, 100, 120};
7 int n = 3;
8 float capacity = 50;
9
10 // 가치/무게 비율: 6, 5, 4
11 // 그리디: 비율 높은 것부터!
12 // (이미 비율 순 정렬 가정)
13
14 float totalValue = 0;
15 for (int i = 0; i < n; i++) {
16 if (capacity >= weight[i]) {
17 // 통째로 넣기
18 capacity -= weight[i];
19 totalValue += value[i];
20 printf("물건%d: 전부 넣기 (%.0f)\n", i+1, value[i]);
21 } else {
22 // 잘라서 넣기 (분할 가능!)
23 float fraction = capacity / weight[i];
24 totalValue += value[i] * fraction;
25 printf("물건%d: %.0f%% 넣기 (%.0f)\n",
26 i+1, fraction*100, value[i]*fraction);
27 capacity = 0;
28 break;
29 }
30 }
31 printf("최대 가치: %.0f\n", totalValue);
32 return 0;
33}
실행 결과
물건1: 전부 넣기 (60) 물건2: 전부 넣기 (100) 물건3: 67% 넣기 (80) 최대 가치: 240

[심화] 그리디가 최적인 2가지 조건

[심화] 그리디 정당성의 2가지 조건
그리디가 최적 해를 보장하려면 두 조건이 필요:

1. 탐욕적 선택 속성 (Greedy Choice Property)
→ 매 순간 최선의 선택이 전체 최선에 포함됨

2. 최적 부분 구조 (Optimal Substructure)
→ 하위 문제의 최적 해가 전체 문제의 최적 해에 포함됨

두 조건 모두 성립하면 그리디 = 최적!
그리디 문제 유형 정리
문제전략최적?시간
거스름돈(한국)큰 동전 우선O (배수 조건)O(종류)
거스름돈(임의)큰 동전 우선X (반례 존재)—
활동 선택빨리 끝나는 것 우선O (증명 가능)O(n log n)
분할 배낭가치/무게 비율 우선OO(n log n)
0-1 배낭그리디 불가XDP 필요
허프만 코딩빈도 낮은 것 합치기OO(n log n)

[심화] 그리디 실전: 최소 동전 (DP와 비교)

[심화] 0-1 배낭 vs 분할 배낭
분할 배낭: 물건을 잘라서 넣을 수 있음 → 그리디 OK
0-1 배낭: 물건을 통째로만 넣을 수 있음 → 그리디 X, DP 필요

비슷해 보이지만 '분할 가능 여부'에 따라 전략이 완전히 달라!

[심화 내용] 그리디 정당성 증명

[심화 내용] 교환 논증 (Exchange Argument)
그리디 해가 최적임을 증명하는 대표적 기법:

1. 최적 해 OPT가 있다고 가정
2. OPT가 그리디와 다른 부분이 있으면
3. 그 부분을 그리디의 선택으로 교환해도 더 나빠지지 않음을 보임
4. 따라서 그리디 해 = 최적 해

예: 활동 선택 문제 — 빨리 끝나는 것을 안 고른 최적 해가 있다면,
교환해도 회의 수가 줄지 않으므로 그리디가 최적.

[심화 내용] 허프만 코딩 — 그리디의 명작

[심화 내용] 허프만 코딩
허프만 코딩 = 빈도가 높은 문자에 짧은 코드, 낮은 문자에 긴 코드

그리디 전략: 빈도가 가장 낮은 두 노드를 합치기

예: A(45%), B(13%), C(12%), D(16%), E(9%), F(5%)
→ 합치기: (E5+F5)=14 → (B13+C12)=25 → ...
→ 최소 비트로 데이터 압축 가능!

용어 정리

[심화 내용] 탐욕법이 최적인 조건

[심화 내용] 탐욕적 선택 속성과 최적 부분 구조
탐욕법이 최적해를 보장하려면 두 조건이 필요:

1. 탐욕적 선택 속성 (Greedy Choice Property)
매 단계의 최선의 선택이 전체 최적해에 포함됨

2. 최적 부분 구조 (Optimal Substructure)
부분 문제의 최적해로 전체 최적해를 구성할 수 있음

최적인 경우:
- 거스름돈 (동전이 배수 관계일 때만!)
- 활동 선택 (끝나는 시간 기준)
- Huffman 코딩
- Kruskal/Prim MST
- Dijkstra 최단 경로 (음수 간선 없을 때)

최적이 아닌 경우:
- 거스름돈 {1, 3, 4}로 6원: 그리디=4+1+1(3개), 최적=3+3(2개)!
- 0/1 배낭 문제
- TSP (외판원)
[C 코드] 활동 선택 문제 (그리디)
1#include <stdio.h>
2
3// 끝나는 시간 기준 정렬 후 탐욕 선택
4int main() {
5 // {시작, 끝} 이미 끝시간 순 정렬
6 int start[] = {1, 3, 0, 5, 8, 5};
7 int end[] = {2, 4, 6, 7, 9, 9};
8 int n = 6;
9
10 printf("선택된 활동:\n");
11 int last_end = 0, count = 0;
12 for (int i = 0; i < n; i++) {
13 if (start[i] >= last_end) { // 겹치지 않으면 선택!
14 printf(" 활동 %d: [%d, %d)\n", i, start[i], end[i]);
15 last_end = end[i];
16 count++;
17 }
18 }
19 printf("최대 %d개 활동 가능\n", count);
20 return 0;
21}
실행 결과
선택된 활동: 활동 0: [1, 2) 활동 1: [3, 4) 활동 3: [5, 7) 활동 4: [8, 9) 최대 4개 활동 가능

[심화 내용] Huffman 코딩

[심화 내용] 허프만 코딩 = 탐욕법의 걸작
목표: 빈도 높은 문자는 짧은 코드, 낮은 문자는 긴 코드

문자 빈도: A=45, B=13, C=12, D=16, E=9, F=5

탐욕 전략: 가장 빈도 낮은 두 개를 합치기!

1. E(9)+F(5) = EF(14)
2. C(12)+B(13) = CB(25)
3. EF(14)+D(16) = EFD(30)
4. CB(25)+EFD(30) = CBEFD(55)
5. A(45)+CBEFD(55) = root(100)

4int greedy_coins(int coins[], int n, int amount) {

6 for (int i = n-1; i >= 0; i--)
7 while (amount >= coins[i]) { amount -= coins[i]; count++; }

그리디가 맞는지 확인하는 방법:

1. 반례 찾기
작은 입력에서 완전 탐색과 비교

2. 수학적 증명
탐욕적 선택 속성 + 최적 부분 구조
교환 논법(Exchange Argument)으로 증명

3. 직관 체크

[심화 내용] 유명한 그리디 알고리즘 모음
1. Kruskal MST
간선을 가중치 순 정렬 → 사이클 안 만들면 추가
Union-Find 사용, O(E log E)

2. Prim MST
현재 트리에서 가장 가까운 정점 추가
우선순위 큐 사용, O(E log V)

3. Dijkstra 최단 경로
현재 가장 가까운 미방문 정점 처리
O(V^2) 또는 O(E log V) (힙 사용)

4. 분할 가능 배낭
단가(가치/무게) 순 정렬 → 높은 것부터
물건을 쪼갤 수 있으면 그리디 OK!

5. 작업 스케줄링
마감이 빠른 순 / 소요시간 짧은 순
탐욕법의 매력은 "단순함"이야! 복잡하게 모든 경우를 볼 필요 없이, 매 순간 최선만 고르면 돼. 단, 그게 진짜 최적인지 증명은 해야 해!

[심화] 그리디 정당성 증명 — 교환 논법

[심화] 교환 논법 (Exchange Argument)
그리디가 최적임을 증명하는 가장 일반적인 방법:

단계:
1. 최적해 OPT가 있다고 가정
2. OPT와 그리디 해 G가 다른 첫 번째 지점 찾기
3. OPT에서 해당 선택을 G의 선택으로 교환해도
해가 나빠지지 않음을 보임
4. 반복하면 OPT → G로 변환 가능
5. 따라서 G도 최적!

예: 활동 선택
OPT에서 첫 활동이 a1, G에서 첫 활동이 g1(가장 빨리 끝남)
g1이 a1보다 일찍 끝나므로 a1을 g1로 교환해도
나머지 활동에 더 많은 시간이 남음 → 최적 유지!
O(n log n)
대부분의 그리디 알고리즘 시간 복잡도: 정렬 + 순회. DP의 O(n^2)보다 빠른 것이 장점!
그리디의 핵심: "지금 당장 가장 좋은 걸 골라!" 단순하지만 강력해. 다만 진짜 최적인지 증명은 꼭 해야 해!
[활동] 그리디 판별 연습
다음 문제에 그리디를 써도 되는지 판단해 보세요:

1. 회의실 배정: 최대한 많은 회의 잡기
→ O! 끝시간 순 정렬 + 그리디

2. 동전 {1, 5, 12}로 16원
→ X! 그리디: 12+1+1+1+1=5개, 최적: 5+5+5+1=4개

3. 짐을 박스에 나누기 (bin packing)
→ X! NP-hard 문제. 그리디는 근사만 가능

4. 체육복 빌려주기
→ O! 번호 순서대로 빌려주면 최적

핵심: "지금 선택이 미래 선택에 영향?"
영향 없으면 그리디, 있으면 DP!
