시뮬레이션이란?

시뮬레이션은 문제에서 시키는 대로 정확히 따라하는 거야! 특별한 알고리즘이 필요 없어. 규칙을 그대로 코드로 옮기면 돼!
[핵심] 시뮬레이션
시뮬레이션 = 문제에 주어진 규칙/명령을 한 단계씩 정확히 실행하는 것

🤖 로봇 이동: 명령대로 좌표 변경
🃏 카드 섞기: 교환 명령대로 배열 변경
🎮 게임 규칙: 턴마다 규칙 적용

핵심: 정확한 구현이 전부!

로봇 이동 시뮬레이션

5×5 격자에서 로봇이 (0,4)에서 출발해요. 명령: 위↑, 위↑, 오른쪽→, 오른쪽→, 아래↓, 오른쪽→.

명령을 하나씩 실행하며 로봇 위치를 추적!

카드 섞기 시뮬레이션

교환 명령을 하나씩 실행!
[팁] 교환(swap)의 원리
두 값을 교환할 때는 임시 변수가 필요!

잘못: A=B; B=A; (A 사라짐!)
올바른: temp=A; A=B; B=temp;

C 코드: int temp = a; a = b; b = temp;

시뮬레이션 문제 풀이 4단계

1.규칙 파악 → 2.초기 상태 → 3.단계별 실행 → 4.결과 출력
[주의] 자주 하는 실수
1. 순서 실수: 교환 순서 바꾸면 결과 달라짐!
2. 범위 초과: 격자 밖으로 나가면? → 예외 처리
3. 인덱스 혼동: 0부터? 1부터? → 문제 잘 읽기!
4. 상태 누락: 중간 단계 건너뛰면 틀림!

robot_sim.c
1#include <stdio.h>
2#include <string.h>
3
4int main(void) {
5 int x = 0, y = 4; // 시작 위치
6 int W = 5, H = 5; // 격자 크기
7
8 char cmds[] = "UURRDR"; // U=위, D=아래, L=왼, R=오른
9 int dx[] = {0, 0, -1, 1}; // U D L R
10 int dy[] = {-1, 1, 0, 0};
11 char dir[] = "UDLR";
12
13 for (int i = 0; i < (int)strlen(cmds); i++) {
14 int d = -1;
15 for (int j = 0; j < 4; j++)
16 if (cmds[i] == dir[j]) d = j;
17
18 int nx = x + dx[d], ny = y + dy[d];
19 // 범위 체크!
20 if (nx >= 0 && nx < W && ny >= 0 && ny < H) {
21 x = nx; y = ny;
22 }
23 printf("명령 %c → (%d, %d)\n", cmds[i], x, y);
24 }
25 printf("최종 위치: (%d, %d)\n", x, y);
26 return 0;
27}

[심화] 달팽이 배열 시뮬레이션

snail.c
1#include <stdio.h>
2
3#define N 5
4
5int main(void) {
6 int grid[N][N] = {0};
7 // 방향: 오른쪽→아래→왼쪽→위
8 int dx[] = {1, 0, -1, 0};
9 int dy[] = {0, 1, 0, -1};
10 int dir = 0, x = 0, y = 0;
11
12 for (int num = 1; num <= N * N; num++) {
13 grid[y][x] = num;
14 int nx = x + dx[dir], ny = y + dy[dir];
15 // 범위 밖이거나 이미 채워진 칸이면 방향 전환
16 if (nx < 0 || nx >= N || ny < 0 || ny >= N || grid[ny][nx] != 0)
17 dir = (dir + 1) % 4;
18 x += dx[dir]; y += dy[dir];
19 }
20
21 for (int i = 0; i < N; i++) {
22 for (int j = 0; j < N; j++)
23 printf("%3d", grid[i][j]);
24 printf("\n");
25 }
26 return 0;
27}

[심화] 윷놀이 시뮬레이션
말의 위치를 추적하며 규칙 적용:

1. 도(1칸), 개(2칸), 걸(3칸), 윷(4칸), 모(5칸)

3. 상대 말과 같은 칸이면 잡기 (시작점으로)
4. 자기 말과 같은 칸이면 업기 (함께 이동)

[심화 내용] 몬테카를로 시뮬레이션

[심화 내용] 몬테카를로 방법
랜덤을 이용해 근사값을 구하는 시뮬레이션!

예: 원주율(π) 구하기
1. 1×1 정사각형 안에 랜덤 점 N개 찍기
2. 원 안에 들어간 점 개수 = M
3. π ≈ 4 × M / N

N이 클수록 정확한 값에 가까워져요!

활용: 주식 예측, 물리 시뮬레이션, AI 게임(알파고)
monte_carlo_pi.c
1#include <stdio.h>
2#include <stdlib.h>
3#include <time.h>
4
5int main(void) {
6 srand(time(NULL));
7 int N = 1000000; // 100만 번 시도
8 int inside = 0;
9
10 for (int i = 0; i < N; i++) {
11 double x = (double)rand() / RAND_MAX; // 0~1
12 double y = (double)rand() / RAND_MAX;
13 if (x*x + y*y <= 1.0) // 원 안?
14 inside++;
15 }
16
17 double pi = 4.0 * inside / N;
18 printf("π ≈ %.6f\n", pi); // 약 3.141592
19 return 0;
20}
실행 결과
π ≈ 3.141728

[심화 내용] 시뮬레이션의 한계
시뮬레이션의 장점:
- 복잡한 시스템을 수학 없이 분석 가능
- 실험 불가능한 상황도 모의 가능
- 확률적 현상을 직관적으로 이해

시뮬레이션의 한계:
1. 정확도: 샘플 수에 의존 (많을수록 정확)
2. 모델 오류: 현실을 완벽히 반영 불가
3. 계산 비용: 복잡한 시뮬레이션은 매우 느림
4. 난수 품질: 의사 난수의 주기/패턴 문제
