집합이 뭘까?

'우리 반 남학생 모임', '10보다 작은 소수 모임'... 이렇게 조건을 만족하는 것들을 모아 놓은 것이 집합이야!
[핵심] 집합이란?
집합(Set) = 조건을 만족하는 것들의 모임

표기: A = {1, 3, 5, 7, 9} (중괄호 사용)

규칙:
1. 순서 없음: {1,2,3} = {3,1,2}
2. 중복 없음: {1,1,2} = {1,2}
3. 원소: 집합에 속한 각 것. 1 ∈ A (1은 A의 원소)

집합을 표현하는 방법은 2가지가 있어요.

집합 표현 방법
방법예시설명
원소나열법{2, 4, 6, 8, 10}원소를 직접 나열
조건제시법{x | x는 10 이하 짝수}조건으로 표현

원소와 포함 관계

[핵심] 원소 기호
A = {1, 3, 5, 7}

3 ∈ A → '3은 A의 원소' (포함됨)
4 ∉ A → '4는 A의 원소가 아님' (포함 안 됨)

공집합: ∅ = {} (원소가 하나도 없는 집합)

부분집합

[핵심] 부분집합
A의 모든 원소가 B에도 있으면: A ⊂ B (A는 B의 부분집합)

예: {1, 3} ⊂ {1, 2, 3, 4, 5}

부분집합의 개수: 원소 N개 → 부분집합 2ⁿ개
• {1, 2}: 부분집합 = {}, {1}, {2}, {1,2} → 2² = 4개
• {1, 2, 3}: 2³ = 8개
부분집합 개수가 2ⁿ인 이유? 각 원소마다 '포함(1)/미포함(0)' 2가지 선택! N개 원소면 2×2×...×2 = 2ⁿ! 이거 비트마스크랑 같아!

합집합, 교집합, 차집합

[핵심] 집합 연산 3가지
A = {1, 2, 3, 4}, B = {3, 4, 5, 6}

합집합 A ∪ B = {1, 2, 3, 4, 5, 6}
→ 둘 중 하나라도 속하면 포함 (OR)

교집합 A ∩ B = {3, 4}
→ 둘 다 속해야 포함 (AND)

차집합 A - B = {1, 2}
→ A에는 있지만 B에는 없는 것
✕ 잘못된 코드

벤 다이어그램

집합의 관계를 그림으로 나타낸 것이 벤 다이어그램이에요.

두 원이 겹치는 부분 = 교집합!
[팁] 벤 다이어그램 읽기
• 왼쪽 원만 = A - B (차집합)
• 겹치는 부분 = A ∩ B (교집합)
• 오른쪽 원만 = B - A
• 두 원 전체 = A ∪ B (합집합)
• 두 원 밖 = (A ∪ B)ᶜ (여집합)

1#include <stdio.h>
2
3// 합집합: A ∪ B
4int setUnion(int A[], int nA, int B[], int nB, int result[]) {
5 int n = 0;
6 // A의 모든 원소 추가
7 for (int i = 0; i < nA; i++) result[n++] = A[i];
8 // B에서 A에 없는 것만 추가
9 for (int i = 0; i < nB; i++) {
10 int found = 0;
11 for (int j = 0; j < nA; j++)
12 if (B[i] == A[j]) { found = 1; break; }
13 if (!found) result[n++] = B[i];
14 }
15 return n;
16}
17
18// 교집합: A ∩ B
19int setIntersect(int A[], int nA, int B[], int nB, int result[]) {
20 int n = 0;
21 for (int i = 0; i < nA; i++)
22 for (int j = 0; j < nB; j++)
23 if (A[i] == B[j]) { result[n++] = A[i]; break; }
24 return n;
25}

[심화] 비트마스크로 집합 연산

[심화] 비트마스크 집합
전체 집합 = {0, 1, 2, 3, 4}일 때:

A = {1, 3, 4} → 비트: 11010 (= 26)
B = {0, 1, 2} → 비트: 00111 (= 7)

합집합: A | B = 11111 = {0,1,2,3,4}
교집합: A & B = 00010 = {1}
차집합: A & ~B = 11000 = {3,4}
여집합: ~A & 11111 = 00101 = {0,2}
set_bitmask.c
1#include <stdio.h>
2
3void printSet(int mask, int n) {
4 printf("{");
5 int first = 1;
6 for (int i = 0; i < n; i++)
7 if (mask & (1 << i)) {
8 if (!first) printf(",");
9 printf("%d", i);
10 first = 0;
11 }
12 printf("}");
13}
14
15int main(void) {
16 int n = 5; // 전체 = {0,1,2,3,4}
17 int A = (1<<1) | (1<<3) | (1<<4); // {1,3,4}
18 int B = (1<<0) | (1<<1) | (1<<2); // {0,1,2}
19
20 printf("A = "); printSet(A, n); printf("\n");
21 printf("B = "); printSet(B, n); printf("\n");
22 printf("A∪B = "); printSet(A | B, n); printf("\n");
23 printf("A∩B = "); printSet(A & B, n); printf("\n");
24 printf("A-B = "); printSet(A & ~B, n); printf("\n");
25 return 0;
26}

[KOI 필수] 비트마스크 집합 공식
원소 i 추가: set |= (1 << i)
원소 i 제거: set &= ~(1 << i)
원소 i 포함?: set & (1 << i)
원소 i 토글: set ^= (1 << i)
합집합: A | B
교집합: A & B
차집합: A & ~B
원소 수: __builtin_popcount(set)

[심화] 포함-배제 원리

[심화] 포함-배제 원리
|A ∪ B| = |A| + |B| - |A ∩ B|

왜? A와 B를 더하면 겹치는 부분이 2번 세어지니까 1번 빼주는 것!

3개: |A ∪ B ∪ C| = |A|+|B|+|C| - |A∩B|-|A∩C|-|B∩C| + |A∩B∩C|

KOI 필기에서 '~이거나 ~인 학생 수' 문제로 자주 출제!

[심화 내용] 멱집합 (Power Set)

[심화 내용] 멱집합
멱집합(Power Set) = 어떤 집합의 모든 부분집합의 집합

P({1,2}) = {∅, {1}, {2}, {1,2}}

|P(A)| = 2^|A| (원소 수의 2의 거듭제곱)

집합론에서 P(A)의 크기는 항상 A보다 크다 → 칸토어의 정리

[심화 내용] 카르테시안 곱

[심화 내용] 카르테시안 곱 (Cartesian Product)
A × B = {(a, b) | a ∈ A, b ∈ B}

A = {1, 2}, B = {x, y, z}
A × B = {(1,x), (1,y), (1,z), (2,x), (2,y), (2,z)}

|A × B| = |A| × |B| = 2 × 3 = 6

활용: 데이터베이스 JOIN, 좌표계(ℝ² = ℝ × ℝ)

26}

[심화 내용] 집합론의 기초

[심화 내용] 집합론 — 수학의 기초
칸토어 (Georg Cantor, 1874):
집합론의 창시자

핵심 개념:
- 기수(Cardinality): 집합의 크기
|{1,2,3}| = 3
- 멱집합(Power Set): 모든 부분집합의 집합
P({1,2}) = {{}, {1}, {2}, {1,2}}
|P(S)| = 2^|S|
- 가산 무한: 자연수와 1:1 대응 가능
정수, 유리수 = 가산 무한 (놀랍게도!)
- 비가산 무한: 실수 = 비가산 무한
대각선 논법으로 증명 (칸토어)

프로그래밍 연결:
- Set 자료구조 (HashSet, TreeSet)
- SQL의 UNION, INTERSECT, EXCEPT
- 비트마스크 = 유한 집합의 효율적 표현

[심화 내용] 집합과 데이터베이스

[심화 내용] SQL = 집합 연산
관계형 데이터베이스의 기초 = 집합론!

테이블 = 튜플의 집합
SELECT = 원소 선택 (조건으로 부분집합)

집합 연산:
UNION = 합집합
INTERSECT = 교집합
EXCEPT = 차집합
JOIN = 카르테시안 곱의 부분집합

예:
SELECT * FROM A UNION SELECT * FROM B
→ A ∪ B

SELECT * FROM A INTERSECT SELECT * FROM B
→ A ∩ B

관계 대수(Relational Algebra) = 집합 연산의 확장
에드거 코드(E.F. Codd)가 1970년 제안

[심화 내용] 드 모르간 법칙
드 모르간 법칙 (De Morgan's Laws):

(A ∪ B)ᶜ = Aᶜ ∩ Bᶜ
(A ∩ B)ᶜ = Aᶜ ∪ Bᶜ

"합집합의 여집합 = 여집합들의 교집합"
"교집합의 여집합 = 여집합들의 합집합"

프로그래밍에서:
!(a || b) == (!a && !b)
!(a && !b) == (!a || !b)

O(V+E)
그래프에서 집합 연산(인접 정점 교집합 등)의 시간 복잡도. 해시셋으로 O(min(|A|,|B|))까지 가능!
집합 연산은 비트 연산과 1:1 대응! OR=합집합, AND=교집합, AND NOT=차집합. 비트를 알면 집합이 보이고, 집합을 알면 수학이 보여!
집합 (Set)
조건을 만족하는 것들의 모임. 순서 없고 중복 없음.
원소 (Element)
집합에 속한 각각의 것. ∈ 기호 사용.
부분집합
한 집합의 원소가 모두 다른 집합에 포함될 때. A ⊂ B.
합집합
A ∪ B. 둘 중 하나라도 속하는 원소의 집합. OR.
교집합
A ∩ B. 둘 다 속하는 원소의 집합. AND.
차집합
A - B. A에는 있고 B에는 없는 원소의 집합.
공집합
∅. 원소가 하나도 없는 집합.
벤 다이어그램
집합의 관계를 원으로 표현한 그림.
포함-배제
|A∪B| = |A|+|B|-|A∩B|. 겹치는 부분 보정.
멱집합
모든 부분집합의 집합. 크기 = 2ⁿ.
카르테시안 곱
A×B. 모든 순서쌍 (a,b)의 집합.
비트마스크 집합
정수의 각 비트를 원소 존재 여부로 사용하는 집합 표현. O(1) 연산.
멱집합
어떤 집합의 모든 부분집합을 모은 집합. 크기 = 2^n.
가산 무한
자연수와 1:1 대응 가능한 무한. 정수, 유리수가 해당.
비가산 무한
자연수와 1:1 대응 불가능한 무한. 실수가 대표적.
HashSet
해시 테이블 기반 집합. 평균 O(1) 연산. 순서 없음.
TreeSet
균형 트리 기반 집합. O(log n) 연산. 정렬 순서 유지.
