뽑는 방법이 몇 가지?

3명 중에 반장, 부반장을 뽑는 것과, 3명 중에 대표 2명을 뽑는 건 달라! 순서가 중요한지 아닌지에 따라 경우의 수가 달라지거든!

순열 (Permutation)

[핵심] 순열 = 순서 O
순열(Permutation) = n개에서 r개를 순서 있게 뽑기

nPr = n! / (n-r)!

예: {A, B, C}에서 2개를 순서 있게 뽑기
3P2 = 3! / 1! = 6가지
AB, AC, BA, BC, CA, CB

반장-부반장처럼 자리(역할)가 다를 때 사용!
[핵심] 팩토리얼
n! = n × (n-1) × (n-2) × ... × 2 × 1

0! = 1 (약속!)
1! = 1
3! = 3 × 2 × 1 = 6
5! = 5 × 4 × 3 × 2 × 1 = 120
10! = 3,628,800

조합 (Combination)

[핵심] 조합 = 순서 X
조합(Combination) = n개에서 r개를 순서 없이 뽑기

nCr = n! / (r! × (n-r)!)

예: {A, B, C}에서 2개를 순서 없이 뽑기
3C2 = 3! / (2! × 1!) = 3가지
{A,B}, {A,C}, {B,C}

대표 2명처럼 역할 구분 없이 뽑을 때 사용!

[심화] 순열 생성의 핵심 패턴
1. 첫 번째 자리에 올 수 있는 원소를 하나씩 고정
2. 나머지 원소로 재귀 호출
3. 재귀가 끝나면 원복(swap back)!

원복을 잊으면 배열이 망가져요!

[심화 내용] 카탈란 수

[심화 내용] 카탈란 수 (Catalan Number)
Cₙ = (2n)! / ((n+1)! × n!)

C₀=1, C₁=1, C₂=2, C₃=5, C₄=14, C₅=42

등장하는 곳:
• n쌍 괄호의 올바른 배열 수
• n+1개 잎의 이진 트리 구조 수
• 계단 격자 경로 수 (대각선 아래 안 가는)

KOI 연결: 순열·조합 문제

[KOI] 경우의 수 문제 키워드
• '몇 가지 방법' → 순열 또는 조합
• '순서가 다르면 다른 경우' → 순열
• '순서 상관없이' → 조합
• '중복 가능' → 중복 순열/조합
• N ≤ 20이면 → 완전 탐색으로도 가능
• N ≤ 10이면 → 순열 완전 탐색 가능 (N!)
[KOI 실전] nCr 계산 시 오버플로 방지
큰 수의 nCr 계산 시 팩토리얼이 오버플로!

방법 1: 파스칼 삼각형 (DP)
C[i][j] = C[i-1][j-1] + C[i-1][j]
→ 곱셈 없이 덧셈만! 오버플로 적음

방법 2: 교대 곱·나눗셈
result = 1
for i=0 to r-1: result = result*(n-i)/(i+1)
→ 매 단계 나눠서 수가 안 커짐

방법 3: 모듈러 역원 (MOD 문제)
페르마 소정리로 나눗셈을 곱셈으로 변환
52!
카드 한 벌(52장)을 섞는 경우의 수 = 52! ≈ 8×10⁶⁷. 우주의 원자 수(약 10⁸⁰)보다는 적지만, 지구가 생긴 이래 한 번도 같은 순서로 섞인 적 없다고 봐요!

[심화 내용] next_permutation 알고리즘

[심화 내용] 사전순 다음 순열 알고리즘
C++ STL의 next_permutation 내부 알고리즘:

1. 뒤에서부터 arr[i] < arr[i+1]인 i를 찾음
2. 뒤에서부터 arr[i] < arr[j]인 j를 찾음
3. arr[i]와 arr[j]를 swap
4. i+1부터 끝까지 reverse

예: {1,3,5,4,2}
1. i=1 (3 < 5)
2. j=3 (3 < 4)
3. swap: {1,4,5,3,2}
4. reverse i+1~end: {1,4,2,3,5}

시간: O(n), 공간: O(1)
모든 n! 순열을 사전순으로 생성 가능!

[심화 내용] 조합론의 항등식

[심화 내용] 유용한 조합 항등식
파스칼 규칙: C(n,r) = C(n-1,r-1) + C(n-1,r)
반더몬드: C(m+n,r) = sum C(m,k)*C(n,r-k)
하키 스틱: C(r,r)+C(r+1,r)+...+C(n,r) = C(n+1,r+1)
이항 정리: (1+x)^n = sum C(n,k)*x^k
대칭: C(n,r) = C(n,n-r)
합: C(n,0)+C(n,1)+...+C(n,n) = 2^n

대회에서 조합 문제를 풀 때 이 항등식들이 핵심!
O(n)
next_permutation의 시간 복잡도. 상수 공간으로 사전순 다음 순열을 구할 수 있어요!

[심화] 중복 조합과 별 찍기 문제

[심화] 중복 조합 = 별과 막대
중복 조합 H(n,r) = C(n+r-1, r)

예: 3종류 과일에서 5개 고르기
H(3,5) = C(7,5) = 21가지

별과 막대 (Stars and Bars):
★★|★★★| → 사과2, 바나나3, 포도0
★|★|★★★ → 사과1, 바나나1, 포도3

5개의 별(★)과 2개의 막대(|)를 배열
= 7자리 중 2자리에 막대 배치
= C(7,2) = 21

응용:
x1+x2+x3=5의 음이 아닌 정수 해 = H(3,5) = 21
동전 나누기, 칸 배분 등 다양한 문제에 활용!

[활동] 순열/조합 판별 연습
다음 문제가 순열(P)인지 조합(C)인지 판별하세요:

1. 5명 중 반장+부반장 선출 → P (순서 있음)
2. 10명 중 3명 대표 선발 → C (순서 없음)
3. 숫자 1~9로 3자리 수 만들기 → P
4. 52장에서 포커 5장 → C
5. 4명이 릴레이 주자 순서 → P
6. 반에서 청소 당번 4명 → C
7. 비밀번호 4자리 → P (중복 순열)
8. 뷔페에서 3가지 반찬 고르기 → C

핵심: "순서가 중요하면 순열, 아니면 조합"
순열과 조합의 차이: "대통령+부통령 뽑기"는 순열 (AB≠BA), "위원 2명 뽑기"는 조합 (AB=BA). 순서가 의미 있느냐가 핵심!
[심화 내용] 카탈란 수
카탈란 수(Catalan Number):
C_n = C(2n,n) / (n+1)

C_0=1, C_1=1, C_2=2, C_3=5, C_4=14, C_5=42

카탈란 수가 나타나는 곳:
1. n쌍 괄호의 올바른 배열 수
2. n+1개 잎을 가진 이진 트리 수
3. n*n 격자에서 대각선 위로 안 가는 경로 수
4. 볼록 n+2각형의 삼각분할 수
5. 스택 순열의 수

조합론에서 가장 자주 등장하는 수열 중 하나!
42
C_5 = 42. 카탈란 수 5번째. "인생, 우주, 그리고 모든 것의 답"과도 같은 수!
순열 조합을 코드로 생성할 때는 백트래킹이 왕도야! next_permutation은 C++의 꿀기능이고, C에서는 직접 구현해야 해. 좋은 연습이 될 거야!
n!
n개 원소의 순열 수. 팩토리얼은 매우 빠르게 증가! 10!=3,628,800, 20!=약 2.4*10^18
순열 (Permutation)
n개에서 r개를 순서 있게 뽑는 것. nPr = n!/(n-r)!.
조합 (Combination)
n개에서 r개를 순서 없이 뽑는 것. nCr = n!/(r!(n-r)!).
팩토리얼 (!)
n! = n×(n-1)×...×1. 0!=1.
중복 순열
같은 원소를 여러 번 쓸 수 있는 순열. nʳ.
중복 조합
같은 원소를 여러 번 뽑을 수 있는 조합. H(n,r)=n+r-1Cr.
파스칼 삼각형
이항 계수를 삼각형으로 배열. nCr = n-1Cr-1 + n-1Cr.
이항 정리
(a+b)ⁿ = Σ nCk × aⁿ⁻ᵏ × bᵏ.
카탈란 수
괄호 배열, 이진 트리 등에 등장하는 수열. Cₙ=(2n)!/((n+1)!n!).
next_permutation
사전순으로 다음 순열을 구하는 알고리즘. O(n) 시간, O(1) 공간.
파스칼 규칙
C(n,r) = C(n-1,r-1) + C(n-1,r). 파스칼 삼각형의 재귀적 성질.
반더몬드 항등식
두 집합에서 각각 뽑아 합치는 조합 수. C(m+n,r) = sum C(m,k)*C(n,r-k).
하키 스틱 항등식
파스칼 삼각형의 대각선 합. C(r,r)+...+C(n,r) = C(n+1,r+1).
