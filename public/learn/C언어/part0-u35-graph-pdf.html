그래프가 뭘까?

친구 관계를 생각해 봐! 나↔영수, 나↔민지, 영수↔철수... 사람은 점(정점), 관계는 선(간선)으로 그리면? 이게 바로 그래프야!
[핵심] 그래프란?
그래프(Graph) = 정점(Vertex)과 간선(Edge)으로 이루어진 자료구조

G = (V, E)
V = 정점(노드)의 집합
E = 간선(연결)의 집합

예: 도시(정점) + 도로(간선) = 도로 지도!

인접 행렬로 표현

[핵심] 인접 행렬
N×N 2차원 배열로 그래프 표현

adj[i][j] = 1이면 i→j 간선 있음
adj[i][j] = 0이면 없음

무방향: adj[i][j] = adj[j][i] (대칭!)

장점: 간선 확인 O(1)
단점: 메모리 O(N²)
adj_matrix.c
1#include <stdio.h>
2#define N 5
3
4int adj[N][N] = {0};
5
6void addEdge(int u, int v) {
7 adj[u][v] = 1;
8 adj[v][u] = 1; // 무방향이면 양쪽!
9}
10
11int main(void) {
12 addEdge(0, 1);
13 addEdge(0, 2);
14 addEdge(1, 3);
15 addEdge(2, 3);
16 addEdge(3, 4);
17
18 printf("인접 행렬:\n");
19 for (int i = 0; i < N; i++) {
20 for (int j = 0; j < N; j++)
21 printf("%d ", adj[i][j]);
22 printf("\n");
23 }
24 return 0;
25}

인접 리스트로 표현

[핵심] 인접 리스트
각 정점마다 연결된 정점 목록을 저장

0: [1, 2]
1: [0, 3]
2: [0, 3]
3: [1, 2, 4]
4: [3]

장점: 메모리 효율적 O(V+E)
단점: 간선 확인 O(degree)
adj_list.c
1#include <stdio.h>
2#define N 5
3#define MAX_EDGE 10
4
5int adj[N][MAX_EDGE]; // 각 정점의 인접 리스트
6int deg[N] = {0}; // 각 정점의 차수
7
8void addEdge(int u, int v) {
9 adj[u][deg[u]++] = v;
10 adj[v][deg[v]++] = u;
11}
12
13int main(void) {
14 addEdge(0,1); addEdge(0,2);
15 addEdge(1,3); addEdge(2,3); addEdge(3,4);
16
17 for (int i = 0; i < N; i++) {
18 printf("%d: [", i);
19 for (int j = 0; j < deg[i]; j++)
20 printf("%d%s", adj[i][j], j<deg[i]-1?",":"");
21 printf("]\n");
22 }
23 return 0;
24}
실행 결과
0: [1,2] 1: [0,3] 2: [0,3] 3: [1,2,4] 4: [3]

[심화] BFS — 너비 우선 탐색

[심화] BFS (Breadth-First Search)
가까운 것부터 탐색! 큐(Queue) 사용

1. 시작점을 큐에 넣고 방문 표시
2. 큐에서 꺼내기
3. 인접한 미방문 정점을 큐에 넣기
4. 큐가 빌 때까지 반복

특징: 최단 거리 보장! (간선 가중치 동일 시)
bfs.c
1#include <stdio.h>
2#define N 5
3
4int adj[N][N], visited[N];
5
6void bfs(int start) {
7 int queue[N], front = 0, rear = 0;
8 visited[start] = 1;
9 queue[rear++] = start;
10
11 while (front < rear) {
12 int v = queue[front++];
13 printf("%d ", v);
14 for (int i = 0; i < N; i++) {
15 if (adj[v][i] && !visited[i]) {
16 visited[i] = 1;
17 queue[rear++] = i;
18 }
19 }
20 }
21}
22
23int main(void) {
24 adj[0][1]=adj[1][0]=1; adj[0][2]=adj[2][0]=1;
25 adj[1][3]=adj[3][1]=1; adj[2][3]=adj[3][2]=1;
26 adj[3][4]=adj[4][3]=1;
27 printf("BFS(0): "); bfs(0); printf("\n");
28 return 0;
29}
실행 결과
BFS(0): 0 1 2 3 4

[심화] DFS — 깊이 우선 탐색

[심화] DFS (Depth-First Search)
깊이 끝까지 가본 후 돌아오기! 스택/재귀 사용

1. 시작점 방문 표시
2. 인접한 미방문 정점으로 재귀 호출
3. 더 이상 갈 곳 없으면 돌아옴(백트래킹)

특징: 경로 탐색, 사이클 감지에 유리
dfs.c
1#include <stdio.h>
2#define N 5
3
4int adj[N][N], visited[N];
5
6void dfs(int v) {
7 visited[v] = 1;
8 printf("%d ", v);
9 for (int i = 0; i < N; i++)
10 if (adj[v][i] && !visited[i])
11 dfs(i); // 재귀!
12}
13
14int main(void) {
15 adj[0][1]=adj[1][0]=1; adj[0][2]=adj[2][0]=1;
16 adj[1][3]=adj[3][1]=1; adj[2][3]=adj[3][2]=1;
17 adj[3][4]=adj[4][3]=1;
18 printf("DFS(0): "); dfs(0); printf("\n");
19 return 0;
20}
실행 결과
DFS(0): 0 1 3 2 4

[심화 내용] 위상 정렬 (Topological Sort)

2#include <string.h>
3
4#define MAX 100
5int adj[MAX][MAX]; // 인접 행렬
6int n; // 정점 수
7
8// BFS
9void bfs(int start) {
10 int visited[MAX] = {0};
11 int queue[MAX], front = 0, rear = 0;
12 queue[rear++] = start;
13 visited[start] = 1;
14
15 printf("BFS from %d: ", start);
16 while (front < rear) {
17 int v = queue[front++];
18 printf("%d ", v);
19 for (int u = 0; u < n; u++) {
20 if (adj[v][u] && !visited[u]) {
21 visited[u] = 1;
22 queue[rear++] = u;
23 }
24 }
25 }
26 printf("\n");
27}
28
29// DFS
30int vis[MAX];
31void dfs(int v) {
32 vis[v] = 1;
33 printf("%d ", v);
34 for (int u = 0; u < n; u++)
35 if (adj[v][u] && !vis[u]) dfs(u);
36}

[심화 내용] 그래프 알고리즘 모음

[심화 내용] 핵심 그래프 알고리즘
탐색:
- BFS: 최단 경로(무가중치) O(V+E)
- DFS: 연결 요소, 위상 정렬, 사이클 검출

최단 경로:
- Dijkstra: 양수 가중치 O(E log V)
- Bellman-Ford: 음수 간선 OK O(VE)
- Floyd-Warshall: 모든 쌍 O(V^3)

최소 신장 트리:
- Kruskal: 간선 정렬 + Union-Find O(E log E)
- Prim: 우선순위 큐 O(E log V)

기타:
- 위상 정렬: DAG 순서 결정
- 강결합 요소: Tarjan/Kosaraju
- 네트워크 플로우: 최대 유량

그래프는 CS에서 가장 많이 쓰이는 자료구조!

증명: 각 땅(정점)에 연결된 다리(간선) 수가
모두 짝수여야 한 붓 그리기 가능.
쾨니히스베르크에는 홀수 차수 정점이 4개 → 불가능

오일러 경로: 모든 간선을 한 번씩 (홀수 차수 0 또는 2개)
오일러 회로: 모든 간선을 한 번씩 + 시작점 복귀 (홀수 차수 0개)

이것이 그래프 이론의 탄생! (1736년, 오일러)
수학 역사상 최초의 위상수학적 문제
그래프는 "관계"를 표현하는 만능 도구야! SNS의 친구, 지도의 도로, 인터넷의 링크, 심지어 뇌의 뉴런까지 전부 그래프로 모델링할 수 있어!

[심화] 그래프 문제 판별법
1. 연결 요소 찾기
"몇 개의 그룹?" → BFS/DFS
예: 섬의 개수, 네트워크 그룹

2. 최단 경로
"A에서 B까지 최소 거리?" → BFS/다익스트라
예: 미로 탈출, 네비게이션

3. 사이클 검출
"순환이 있는가?" → DFS (back edge)
예: 교착 상태, 선수과목 체크

4. 위상 정렬
"순서대로 나열" → DFS/BFS(Kahn)
예: 빌드 순서, 수강 순서

5. 이분 그래프
"2색으로 칠할 수 있나?" → BFS/DFS
예: 매칭 문제, 팀 나누기

문제를 그래프로 모델링하는 것이 핵심!
[C 코드] 연결 요소 개수 세기
1#include <stdio.h>
2#include <string.h>
3
4#define MAX 100
5int adj[MAX][MAX], visited[MAX], n;
6
7void dfs(int v) {
8 visited[v] = 1;
9 for (int u = 0; u < n; u++)
10 if (adj[v][u] && !visited[u]) dfs(u);
11}
12
13int count_components() {
14 memset(visited, 0, sizeof(visited));
15 int count = 0;
16 for (int v = 0; v < n; v++) {
17 if (!visited[v]) {
18 dfs(v);
19 count++;
20 }
21 }
22 return count;
23}
24
25int main() {
26 n = 7;
27 // 그룹1: 0-1-2, 그룹2: 3-4, 그룹3: 5-6
28 adj[0][1]=adj[1][0]=1;
29 adj[1][2]=adj[2][1]=1;
30 adj[3][4]=adj[4][3]=1;
31 adj[5][6]=adj[6][5]=1;
32
33 printf("정점 7개, 연결 요소: %d개\n", count_components());
34 return 0;
35}

[심화 내용] 면접 그래프 문제
Q1. BFS vs DFS 차이?
BFS: 큐, 최단경로, 레벨순
DFS: 스택/재귀, 연결성, 사이클검출

Q2. 인접 행렬 vs 인접 리스트 언제 사용?
행렬: 밀집(E≈V^2), O(1) 간선확인
리스트: 희소(E≪V^2), 메모리 절약

Q3. 이분 그래프 판별 방법?
BFS로 2색 칠하기. 인접한 같은 색 → 이분 아님

Q4. 음수 가중치가 있으면?
다익스트라 불가 → 벨만포드 사용
음수 사이클이 있으면 최단경로 정의 불가!

그래프
정점(V)과 간선(E)으로 이루어진 자료구조. G=(V,E).
정점
Vertex/Node. 데이터가 담긴 점.
간선
Edge. 정점 사이의 연결.
차수
Degree. 한 정점에 연결된 간선 수.
인접 행렬
N×N 배열로 그래프 표현. adj[i][j]=1이면 연결.
인접 리스트
각 정점의 연결 목록으로 표현. 메모리 효율적.
BFS
너비 우선 탐색. 큐 사용. 최단 거리 보장.
DFS
깊이 우선 탐색. 스택/재귀. 경로, 사이클 탐지.
DAG
Directed Acyclic Graph. 방향 비순환 그래프.
다익스트라
가중치 그래프의 최단 경로. 음수 불가.
위상 정렬
DAG에서 의존 순서 결정. 선수과목 등.
인접 행렬
V*V 2차원 배열로 그래프 표현. adj[i][j]=1이면 i→j 간선 존재. 공간 O(V^2).
인접 리스트
각 정점의 이웃 목록으로 그래프 표현. 공간 O(V+E). 희소 그래프에 유리.
BFS
Breadth-First Search. 너비 우선 탐색. 큐 사용. 최단 경로(무가중치).
DFS
Depth-First Search. 깊이 우선 탐색. 스택/재귀 사용.
Dijkstra
양수 가중치 그래프의 최단 경로 알고리즘. O(E log V).
PageRank
웹 그래프에서 페이지 중요도를 계산하는 알고리즘. Google 검색의 핵심.
위상 정렬
DAG(비순환 방향 그래프)에서 선후 관계를 만족하는 순서 결정.
