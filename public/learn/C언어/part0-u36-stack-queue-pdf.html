스택 — 접시 쌓기!

식당에서 접시를 쌓아 본 적 있어? 맨 위에 올린 접시를 제일 먼저 가져가지? 이게 스택이야!

스택(Stack)은 데이터를 쌓아 올리는 구조예요. 마지막에 넣은 것을 가장 먼저 꺼내요. 이걸 LIFO(Last In, First Out)라고 해요.

[핵심] 스택의 두 가지 연산
push(값) = 위에 쌓기 (넣기)
접시 위에 새 접시를 올려요.

pop() = 위에서 꺼내기 (빼기)
맨 위의 접시를 가져가요.

LIFO = Last In, First Out
마지막에 넣은 것이 먼저 나온다!

스택에서는 중간이나 바닥의 데이터를 바로 꺼낼 수 없어요!
접시를 쌓고(push) 꺼내는(pop) 과정 — 마지막 접시가 먼저!

일상 속 스택

Ctrl+Z로 되돌리기 할 때, 가장 마지막에 한 작업부터 취소되지? 이것도 스택이야! 웹 브라우저의 뒤로가기도 스택이에요!
[실생활] 스택의 예시
️ 접시 쌓기 — 위에서부터 꺼낸다
️ Ctrl+Z — 마지막 작업부터 되돌린다
뒤로가기 — 가장 최근 페이지로 돌아간다
택배 상자 쌓기 — 위의 상자부터 내린다
앱 전환 — 최근 앱이 맨 위에!

큐 — 줄 서기!

큐(Queue)는 줄 서기와 같은 구조예요. 먼저 온 사람이 먼저 나가요. 이걸 FIFO(First In, First Out)라고 해요.

[핵심] 큐의 두 가지 연산
enqueue(값) = 뒤에 서기 (넣기)
줄 맨 뒤에 새 사람이 서요.

dequeue() = 앞에서 나가기 (빼기)
맨 앞의 사람이 나가요.

FIFO = First In, First Out
먼저 넣은 것이 먼저 나온다!

새치기는 안 돼요! 항상 뒤에 서고 앞에서 나가요.
줄의 뒤에 서고(enqueue) 앞에서 나가요(dequeue)
프린터에 문서 3개를 보내면? 먼저 보낸 문서가 먼저 인쇄돼요! 이것도 큐예요!

LIFO v FIFO 비교

같은 순서로 넣어도 꺼내는 순서가 반대!

C 연결 — 배열로 스택 구현

[C 코드] 배열로 스택 구현
int stack[100]; // 스택 배열
int top = -1; // 비어있으면 -1

// push: 위에 쌓기
void push(int val) {
top++;
stack[top] = val;
}

// pop: 위에서 꺼내기
int pop() {
int val = stack[top];
top--;
return val;
}

push(10); push(20); push(30);
printf("%d\n", pop()); // 30 (마지막!)

C 연결 — 배열로 큐 구현

[C 코드] 배열로 큐 구현
int queue[100];
int front = 0, rear = 0;

// enqueue: 뒤에 넣기
void enqueue(int val) {
queue[rear] = val;
rear++;
}

// dequeue: 앞에서 꺼내기
int dequeue() {
int val = queue[front];
front++;
return val;
}

enqueue(10); enqueue(20); enqueue(30);
printf("%d\n", dequeue()); // 10 (처음!)
[주의] 빈 스택/큐에서 꺼내기
스택이 비어있는데 pop()을 하면? → 에러!
큐가 비어있는데 dequeue()를 하면? → 에러!

실제 코드에서는 반드시 비어있는지 확인한 후 꺼내야 해요.
if (top >= 0) pop(); // 스택이 비어있지 않을 때만
if (front < rear) dequeue(); // 큐가 비어있지 않을 때만
[활동] 카드로 스택/큐 시뮬레이션
카드 5장을 준비하고 1~5 숫자를 쓰세요.

스택 실험:
1. 카드를 1→2→3→4→5 순서로 쌓아요
2. 하나씩 꺼내면 5→4→3→2→1 (LIFO!)

큐 실험:
1. 카드를 1→2→3→4→5 순서로 줄 세워요
2. 앞에서 꺼내면 1→2→3→4→5 (FIFO!)
KOI에서 BFS(너비 우선 탐색)에는 큐를, DFS(깊이 우선 탐색)에는 스택(또는 재귀)을 써요! 자료구조가 알고리즘의 열쇠예요!

[심화] 큐의 활용

[C 코드] 원형 큐 구현
1#include <stdio.h>
2#define SIZE 5
3
4int queue[SIZE];
5int front = 0, rear = 0, count = 0;
6
7int enqueue(int x) {
8 if (count == SIZE) { printf("Full!\n"); return 0; }
9 queue[rear] = x;
10 rear = (rear + 1) % SIZE;
11 count++;
12 return 1;
13}
14
15int dequeue() {
16 if (count == 0) { printf("Empty!\n"); return -1; }
17 int x = queue[front];
18 front = (front + 1) % SIZE;
19 count--;
20 return x;
21}
22
23int main() {

25 printf("dequeue: %d\n", dequeue());
26 printf("dequeue: %d\n", dequeue());
27 enqueue(40); enqueue(50); enqueue(60);
28 printf("큐 크기: %d\n", count);
29 while (count > 0) printf("%d ", dequeue());
30 printf("\n");
31 return 0;
32}

[심화 내용] 스택/큐의 고급 변형

[심화 내용] 스택/큐 변형 자료구조
1. 덱 (Deque)
양쪽 끝에서 삽입/삭제 가능
= 스택 + 큐의 합체!
슬라이딩 윈도우 최대/최소에 활용

2. 우선순위 큐 (Priority Queue)
우선순위가 높은 것이 먼저 나옴
힙(Heap)으로 구현: O(log n) 삽입/삭제
다익스트라, 허프만 코딩에 필수

3. 모노톤 스택 (Monotonic Stack)
스택에 단조증가/감소 순서 유지
Next Greater Element 문제: O(n)!

4. 두 스택으로 큐 만들기
inStack → outStack 이동으로 FIFO 구현
amortized O(1)! 면접 단골 문제

[심화 내용] 스택 머신과 역폴란드 표기법

[심화 내용] 후위 표기법 (Reverse Polish Notation)
중위 표기법: 3 + 4 * 2 (우리가 쓰는 방식)
후위 표기법: 3 4 2 * + (스택으로 계산!)

스택 계산 과정:
3 → push 3 [3]
4 → push 4 [3, 4]
2 → push 2 [3, 4, 2]
* → pop 4,2 → 4*2=8 → push [3, 8]
+ → pop 3,8 → 3+8=11 → push [11]
결과: 11

장점:
- 괄호 불필요!
- 연산자 우선순위 처리 불필요
- 스택 하나로 계산 가능

컴파일러가 수식을 중위→후위 변환 후 계산!
[C 코드] 후위 표기식 계산기
1#include <stdio.h>
2#include <stdlib.h>
3#include <string.h>
4#include <ctype.h>
5
6int stack[100], top = -1;
7void push(int x) { stack[++top] = x; }
8int pop() { return stack[top--]; }
9
10int eval_rpn(const char *expr) {
11 char buf[200];
12 strcpy(buf, expr);
13 char *tok = strtok(buf, " ");
14 while (tok) {
15 if (isdigit(tok[0]) || (tok[0]=='-' && tok[1])) {
16 push(atoi(tok));
17 } else {
18 int b = pop(), a = pop();
19 switch(tok[0]) {
20 case '+': push(a+b); break;
21 case '-': push(a-b); break;
22 case '*': push(a*b); break;
23 case '/': push(a/b); break;
24 }
25 }
26 tok = strtok(NULL, " ");
27 }
28 return pop();
29}
30

2#include <string.h>
3
4#define N 5
5int maze[N][N] = {
6 {0,0,1,0,0},
7 {0,0,0,0,1},
8 {1,0,1,0,0},
9 {0,0,0,1,0},
10 {0,1,0,0,0}
11};
12int dist[N][N];
13int dx[]={-1,1,0,0}, dy[]={0,0,-1,1};
14
15int main() {
16 memset(dist, -1, sizeof(dist));
17 int qx[100], qy[100], front=0, rear=0;
18 qx[rear]=0; qy[rear]=0; rear++;
19 dist[0][0] = 0;
20
21 while (front < rear) {
22 int x=qx[front], y=qy[front]; front++;
23 for (int d=0; d<4; d++) {

25 if (nx>=0&&nx<N&&ny>=0&&ny<N&&maze[nx][ny]==0&&dist[nx][ny]==-1) {
26 dist[nx][ny] = dist[x][y]+1;
27 qx[rear]=nx; qy[rear]=ny; rear++;
28 }
29 }
30 }

37 }

39}

[심화] 스택으로 DFS, 큐로 BFS

[심화] 탐색 알고리즘과 자료구조 매칭
DFS (깊이 우선 탐색):
- 스택 사용 (또는 재귀 = 콜스택)
- 한 방향으로 깊이 파고듦
- 미로에서: 막히면 되돌아가기
- 활용: 연결 요소, 위상정렬, 사이클 검출

BFS (너비 우선 탐색):
- 큐 사용
- 가까운 것부터 탐색 (원 모양 확산)
- 미로에서: 최단 거리 보장!
- 활용: 최단경로, 레벨순 탐색

요약:
스택 → DFS → 깊이 → 되돌아가기
큐 → BFS → 너비 → 최단거리

자료구조만 바꾸면 DFS↔BFS 전환!
[심화 내용] 면접 문제
Q1. 두 스택으로 큐 구현
A: inStack에 push, dequeue 시 outStack이 비면 inStack→outStack으로 이동 후 pop. amortized O(1)!

Q2. 히스토그램 최대 직사각형
A: 모노톤 스택으로 O(n)! 높이가 줄어들면 스택에서 pop하며 넓이 계산.

Q3. 스택 min() O(1)로 구현
A: 메인 스택 + 최소값 스택 병행. push/pop 시 min 스택도 갱신.

Q4. 큐 max() O(1)로 구현
A: 덱(모노톤 큐) 사용. 새 원소보다 작은 건 뒤에서 제거.
스택과 큐는 프로그래밍의 양대 기초 자료구조야! "나중에 넣은 거 먼저"(스택)와 "먼저 넣은 거 먼저"(큐)만 기억하면 돼!

[심화] 스택 응용: 중위→후위 변환 (Shunting-yard)

[심화] 다익스트라의 Shunting-yard 알고리즘
중위 표기 → 후위 표기 변환 알고리즘

규칙:
1. 숫자 → 출력
2. ( → 스택에 push
3. ) → ( 나올 때까지 pop하여 출력
4. 연산자:
스택 top의 우선순위 >= 현재 → pop하여 출력
현재 연산자 push
5. 끝나면 스택 전부 pop

예: 3 + 4 * 2
3 → 출력: 3
+ → push + [+]
4 → 출력: 3 4
* → * > + 이므로 push [+, *]
2 → 출력: 3 4 2
끝 → pop: 3 4 2 * +

컴파일러가 수식을 처리하는 핵심 알고리즘!

9void infix_to_postfix(const char *s) {
10 for (int i = 0; s[i]; i++) {
11 if (isdigit(s[i])) printf("%c ", s[i]);
12 else if (s[i] == '(') opush('(');
13 else if (s[i] == ')') {
14 while (ot >= 0 && ops[ot] != '(') printf("%c ", opop());
15 if (ot >= 0) opop(); // discard (
16 } else if (s[i]=='+'||s[i]=='-'||s[i]=='*'||s[i]=='/') {
17 while (ot >= 0 && prec(ops[ot]) >= prec(s[i])) printf("%c ", opop());
18 opush(s[i]);
19 }
20 }
21 while (ot >= 0) printf("%c ", opop());
22 printf("\n");
23}

27 printf("(3+4)*2 -> "); infix_to_postfix("(3+4)*2");

3void dec_to_bin(int n) {

5 if (n == 0) { printf("0"); return; }
6 while (n > 0) {
7 stack[++top] = n % 2;
8 n /= 2;
9 }
10 printf("이진수: ");
11 while (top >= 0) printf("%d", stack[top--]);
12}

메시지 큐 (Message Queue):
분산 시스템에서 서비스 간 통신 도구

생산자 → [메시지 큐] → 소비자

1. 비동기 처리: 생산자가 기다릴 필요 없음

3. 장애 격리: 한쪽이 죽어도 메시지 보존

- Apache Kafka: 대용량 스트리밍
- AWS SQS: 클라우드 관리형
- Redis: 인메모리 큐

스택 (Stack)
데이터를 쌓아 올리는 구조. 마지막에 넣은 것이 먼저 나옴 (LIFO).
큐 (Queue)

LIFO
Last In, First Out. 마지막에 넣은 것이 가장 먼저 나오는 방식.
FIFO
First In, First Out. 먼저 넣은 것이 가장 먼저 나오는 방식.
push / pop
스택에 데이터를 넣는 것(push)과 꺼내는 것(pop).
enqueue / dequeue
큐에 데이터를 넣는 것(enqueue)과 꺼내는 것(dequeue).
덱(Deque)
