C언어의 탄생

C언어는 1972년에 미국 벨 연구소의 데니스 리치라는 분이 만들었어요. 올해로 50살이 넘었지만 아직도 엄청나게 많이 쓰여요!

데니스 리치는 유닉스(UNIX)라는 운영체제를 만들기 위해 C언어를 개발했어요. 원래 어셈블리어로 운영체제를 만들고 있었는데, 너무 복잡하고 힘들어서 좀 더 편한 언어가 필요했거든요. 그래서 B언어를 개선해서 C언어를 만들었어요. (A → B → C 순서랍니다!)

C언어의 역사 — 1972년 탄생부터 지금까지

C언어는 어디에 쓰일까?

C언어는 지금도 엄청나게 많은 곳에서 쓰여요. 여러분이 매일 쓰는 것들 중에도 C로 만든 게 아주 많아요.

[핵심] C언어가 쓰이는 곳
1. 운영체제 — Linux, Windows 핵심, macOS 기반이 전부 C로 만들어져요.
2. 게임 엔진 — 언리얼 엔진, 유니티 핵심이 C/C++로 만들어져요.
3. 임베디드 시스템 — 엘리베이터, 자동차, 세탁기 속 칩이 C로 동작해요.
4. 프로그래밍 언어 — Python, Java, Ruby도 C로 만들어졌어요!
Python도 C로 만들어졌다니 놀랍죠? 그래서 C를 '모든 언어의 아버지'라고 불러요!

모든 언어의 아버지

C언어를 배우면 다른 언어가 훨씬 쉬워져요. 왜냐하면 C++, Java, C#, JavaScript, Python 등 현대 언어 대부분이 C의 문법을 기반으로 만들어졌거든요. 중괄호 { }, 세미콜론 ;, if-else, for 같은 문법이 전부 C에서 시작됐어요.

C에서 갈라져 나온 프로그래밍 언어들 — C는 모든 언어의 아버지!

C는 왜 빠를까?

C언어는 고급 언어 중에서 가장 빠른 언어 중 하나예요. Python은 코드를 한 줄씩 번역하면서 실행하지만, C는 전체 코드를 미리 번역(컴파일)한 다음 실행해요. 미리 번역해놓으니까 실행할 때 훨씬 빠르죠!

같은 계산을 시켰을 때 C가 Python보다 10~100배 빨라요!
[핵심] C언어를 배우는 3가지 이유
1. 컴퓨터 이해 — 메모리, CPU 동작을 직접 체감할 수 있어요.
2. 다른 언어 정복 — C 문법을 알면 C++, Java, JS가 쉬워져요.
3. 빠른 속도 — 게임, 운영체제 등 성능이 중요한 곳에서 필수예요.
C언어를 배우는 건 자전거 타기를 배우는 것과 같아요. 한번 배우면 전동 킥보드도, 오토바이도 금방 탈 수 있죠!

[역사] 데니스 리치와 켄 톰프슨
데니스 리치는 동료 켄 톰프슨과 함께 UNIX 운영체제를 만들었어요. 이 두 사람 덕분에 오늘날 스마트폰(안드로이드=리눅스 기반), 서버, 슈퍼컴퓨터가 동작해요. 데니스 리치는 2011년에 세상을 떠났지만, 그가 만든 C언어는 영원히 남아있어요.

[심화 내용] C와 하드웨어의 관계

[심화 내용] C가 하드웨어에 가까운 이유
1. 포인터 = 메모리 주소
int *p = (int*)0x1000; → 물리 주소 직접 접근
임베디드에서 하드웨어 레지스터 제어

2. sizeof = 실제 바이트 수
sizeof(int) = 4 → CPU 레지스터 크기와 대응
메모리 레이아웃을 정확히 제어

3. 비트 연산 = CPU 명령어 1:1
a & b → AND 명령어, a >> 3 → SHR 명령어
하드웨어 레지스터 비트 조작에 필수

4. 인라인 어셈블리
__asm__("mov eax, 1"); → C 안에 어셈블리!
극한 최적화, 특수 명령어 사용 가능

5. volatile 키워드
컴파일러 최적화 방지
하드웨어 레지스터가 외부에서 변할 수 있음을 표시

[심화] C 관련 면접 문제

[심화] C 면접 필수 질문
Q1. C와 C++의 차이?
A: C++는 C에 클래스, 템플릿, 예외, STL 추가. C는 절차적, C++는 멀티패러다임.

Q2. C에서 메모리 관리 방법?
A: malloc/calloc/realloc으로 할당, free로 해제. 스택(자동), 힙(수동), 데이터(전역/정적).

Q3. volatile 키워드의 용도?
A: 컴파일러 최적화 방지. 하드웨어 레지스터, 멀티스레드 공유 변수에 사용.

Q4. C가 "이식성 있는 어셈블리어"라고 불리는 이유?
A: 하드웨어에 가까우면서도 다양한 CPU에서 컴파일 가능. 어셈블리의 추상화.
C는 "이식성 있는 어셈블리어"라고 불려. 기계에 가까우면서도 어디서든 돌아가니까! 50년 넘게 살아남은 비결이야.

컴퓨터 동작 원리를 이해하는 최고의 도구
