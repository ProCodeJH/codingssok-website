코딩 도구가 필요해요!

요리를 하려면 주방이 필요하듯, 코딩을 하려면 개발 환경이 필요해요! 오늘은 우리의 코딩 주방을 꾸며볼 거예요.

C언어로 프로그래밍을 하려면 두 가지 도구가 필요해요.
1. 코드 편집기 — 코드를 쓰는 노트 (VS Code)
2. 컴파일러 — 코드를 실행 파일로 바꿔주는 번역기 (gcc)

개발 환경 설치 3단계: VS Code → MinGW(gcc) → 환경 변수 설정

1단계: VS C 설치

[설치] VS Code 다운로드
1. 인터넷 브라우저를 열어요.
2. 주소창에 code.visualstudio.com을 입력해요.
3. 파란색 'Download for Windows' 버튼을 클릭해요.
4. 다운로드된 파일(VSCodeSetup-...)을 더블클릭해요.
5. '다음'을 계속 누르고, '바탕화면에 아이콘 만들기'를 체크해요.
6. '설치' 버튼을 누르고 기다려요. 끝!
VS Code는 무료예요! 전 세계에서 가장 많이 쓰는 코드 편집기이기도 해요. 마이크로소프트가 만들었어요.
VS Code 화면 구성 — 왼쪽은 파일, 가운데는 코드, 아래는 터미널

2단계: MGW() 설치

VS Code는 코드를 쓰는 도구예요. 하지만 코드를 실행 파일로 바꿔주는 도구(컴파일러)는 따로 설치해야 해요. C언어의 컴파일러는 gcc이고, Windows에서는 MinGW를 통해 설치해요.

[설치] MinGW(gcc) 다운로드
1. 인터넷에서 MSYS2를 검색하거나, msys2.org에 접속해요.
2. 설치 파일을 다운로드하고 실행해요.
3. 설치 완료 후 MSYS2 터미널을 열어요.
4. 다음 명령어를 입력해요:
pacman -S mingw-w64-x86_64-gcc
5. 'Y'를 눌러 설치를 진행해요.

3단계: 환경 변수 설정

gcc를 설치했지만, 아직 어디서든 사용할 수는 없어요. 컴퓨터에게 'gcc가 여기 있어!'라고 알려줘야 해요. 이걸 환경 변수 설정이라고 해요.

[설정] 환경 변수(PATH) 등록하기
1. Windows 검색에서 '환경 변수'를 검색해요.
2. '시스템 환경 변수 편집'을 클릭해요.
3. '환경 변수' 버튼을 누르세요.
4. 'Path' 항목을 선택하고 '편집'을 누르세요.
5. '새로 만들기'를 누르고 MinGW의 bin 폴더 경로를 추가해요.
예: C:\msys64\mingw64\bin
6. 확인을 누르고, 새 터미널을 열어 확인:
gcc --version
환경 변수는 컴퓨터에게 '도구가 어디 있는지' 알려주는 주소록이에요. 주소록에 등록하면 어디서든 그 도구를 부를 수 있죠!

4단계: 첫 폴더 만들기

이제 코드를 저장할 폴더를 만들어봐요. 정리된 작업 공간이 좋은 코드의 시작이에요!

[활동] 코딩 폴더 만들기
1. 바탕화면이나 문서 폴더에 c-programming 폴더를 만드세요.
2. VS Code를 열고 파일 → 폴더 열기에서 방금 만든 폴더를 선택하세요.
3. VS Code 왼쪽에 폴더 이름이 나타나면 성공!
4. 이 폴더 안에서 앞으로 모든 코드를 작성할 거예요.
앞으로 이런 구조로 코드 파일을 정리할 거예요
[주의] 설치가 안 되면?
• 백신 프로그램이 설치를 막을 수 있어요 → 잠시 꺼보세요.
• 관리자 권한이 필요할 수 있어요 → 오른쪽 클릭 → '관리자 권한으로 실행'.

[심화] Makefile 예제
프로젝트 구조:
main.c, utils.c, utils.h

Makefile:
CC = gcc
CFLAGS = -Wall -g

main: main.o utils.o
$(CC) -o main main.o utils.o

main.o: main.c utils.h
$(CC) $(CFLAGS) -c main.c

utils.o: utils.c utils.h
$(CC) $(CFLAGS) -c utils.c

clean:
rm -f *.o main

사용:
make → 컴파일
make clean → 정리

변경된 파일만 재컴파일 → 대규모 프로젝트에서 필수!

[심화 내용] GDB — C 디버거
GDB (GNU Debugger):
C/C++ 프로그램을 한 줄씩 실행하며 디버깅

기본 명령어:

gdb ./main

(gdb) break main ← main 함수에 브레이크포인트
(gdb) break 15 ← 15번째 줄에 중단점
(gdb) run ← 실행
(gdb) next ← 다음 줄 (함수 넘어감)
(gdb) step ← 다음 줄 (함수 안으로)
(gdb) print x ← 변수 x 값 출력
(gdb) backtrace ← 콜 스택 보기
(gdb) continue ← 다음 브레이크포인트까지
(gdb) quit ← 종료

[심화] 컴파일 옵션 가이드

[심화] GCC 주요 옵션
기본 컴파일:
gcc main.c -o main

경고 옵션:
-Wall: 주요 경고 모두 켜기 (필수!)
-Wextra: 추가 경고
-Werror: 경고를 에러로 취급
-pedantic: 표준 엄격 준수

최적화:
-O0: 최적화 없음 (디버깅용)
-O1: 기본 최적화
-O2: 권장 최적화 (성능+크기 균형)
-O3: 최대 최적화 (가끔 부작용)
-Os: 크기 최적화 (임베디드)

디버깅:
-g: 디버그 정보 포함 (GDB용)
-fsanitize=address: 메모리 오류 검출
-fsanitize=undefined: 정의되지 않은 동작 검출

추천 조합:
개발: gcc -Wall -Wextra -g -fsanitize=address
릴리스: gcc -Wall -O2

[심화 내용] 컴파일 과정 상세

[심화 내용] C 컴파일의 4단계
1. 전처리 (Preprocessing)
gcc -E main.c -o main.i
#include → 헤더 파일 삽입
#define → 매크로 치환
#ifdef → 조건부 컴파일

2. 컴파일 (Compilation)
gcc -S main.i -o main.s
C 코드 → 어셈블리어
구문 분석, 의미 분석, 최적화

3. 어셈블 (Assembly)
gcc -c main.s -o main.o
어셈블리어 → 기계어 (오브젝트 파일)
아직 실행 불가 (주소 미결정)

4. 링킹 (Linking)
gcc main.o -o main
오브젝트 파일들 + 라이브러리 연결
printf의 실제 코드를 libc에서 가져옴
실행 파일 완성!
4단계
C 컴파일 과정: 전처리→컴파일→어셈블→링킹. 각 단계를 이해하면 에러 디버깅이 쉬워져요!
-Wall
GCC의 "모든 경고 켜기" 옵션. Wall = Warning ALL. C 개발할 때 항상 켜두세요!
개발 환경 설정은 한 번 하면 끝! VS Code + GCC + -Wall이면 충분해. 나머지는 필요할 때 하나씩 배워가면 돼!

12int min(int a, int b) { return (a < b) ? a : b; }
13void swap(int *a, int *b) { int t = *a; *a = *b; *b = t; }
14
15// === main.c ===
16#include <stdio.h>
17#include "utils.h"
18
19int main() {

22 printf("min(%d,%d) = %d\n", x, y, min(x, y));
23
24 printf("swap 전: x=%d, y=%d\n", x, y);
25 swap(&x, &y);
26 printf("swap 후: x=%d, y=%d\n", x, y);
27 return 0;
28}

[심화] 초보자 흔한 설정 실수
1. PATH 미설정
gcc를 설치했는데 "명령을 찾을 수 없습니다"
→ 환경 변수 PATH에 gcc 경로 추가!

2. 확장자 실수
main.c.txt로 저장 → 컴파일 안 됨
→ "파일 확장명 표시" 켜기

3. 한글 경로
C:\사용자\바탕화면\ → 일부 도구에서 에러
→ 영문 경로 사용 권장

4. 32비트 vs 64비트
컴파일러와 OS 비트 불일치
→ 64비트 OS면 64비트 컴파일러!

5. 헤더 파일 경로
#include "myheader.h" 찾기 실패
→ -I 옵션으로 경로 지정
설정에서 막히면 에러 메시지를 그대로 검색해 봐! Stack Overflow에 답이 있을 확률 99%. 모든 개발자가 겪는 통과의례야!
[심화 내용] CI/CD와 자동화

name: C Build

runs-on: ubuntu-latest

MinGW
Minimalist GNU for Windows. Windows에서 gcc를 사용할 수 있게 해주는 도구 모음.
환경 변수 (PATH)
컴퓨터가 프로그램을 찾을 경로 목록. PATH에 등록하면 어디서든 해당 프로그램을 실행할 수 있어요.
터미널
텍스트로 컴퓨터에 명령을 내리는 창. VS Code 아래쪽에서 열 수 있어요.
IDE
Integrated Development Environment. 편집+컴파일+디버깅 통합 도구.
GCC
GNU Compiler Collection. 무료 오픈소스 C/C++ 컴파일러. Linux 표준.
Clang
LLVM 기반 C/C++ 컴파일러. macOS 기본. 에러 메시지가 친절!
MinGW
Minimalist GNU for Windows. Windows에서 GCC 사용 가능하게 하는 도구.
WSL
Windows Subsystem for Linux. Windows에서 Linux 환경을 실행.
Make
빌드 자동화 도구. Makefile에 규칙 정의. 1976년부터 사용.
CMake
크로스 플랫폼 빌드 시스템 생성기. 현대 C/C++ 프로젝트 표준.
GDB
GNU Debugger. C/C++ 프로그램을 한 줄씩 실행하며 디버깅하는 도구.
