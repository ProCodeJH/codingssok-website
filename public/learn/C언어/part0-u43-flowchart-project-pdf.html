주제 1: 자판기

편의점 앞 자판기를 생각해 봐! 돈을 넣고, 음료를 고르고, 음료가 나오고, 거스름돈을 받지. 이걸 순서도로 그려보자!
[분석] 자판기 문제 분해
입력: 돈, 음료 번호
처리: 금액 비교, 재고 확인, 거스름돈 계산
출력: 음료 배출, 거스름돈 반환, 메시지 출력
조건: 돈이 충분한가? 재고가 있는가?

자판기를 단계별로 분해하면 이렇게 돼요:
1. 돈을 넣는다
2. 넣은 금액이 음료 가격 이상인지 확인한다
3. 부족하면 → '돈이 부족해요' 메시지 → 돈 더 넣기
4. 충분하면 → 음료를 선택한다
5. 재고가 있는지 확인한다
6. 없으면 → '다른 음료를 선택하세요' → 다시 선택
7. 있으면 → 음료를 배출한다
8. 거스름돈 = 넣은 돈 - 가격 → 거스름돈 반환
9. '감사합니다' 출력

자판기 순서도 — 조건 분기가 2개 있어요!
[의사코드] 자판기
프로그램 자판기
  반복:
    입력: 돈
    만약 돈 < 가격:
      출력: '돈이 부족해요'
      처음으로 돌아가기
    입력: 음료번호
    만약 재고[음료번호] == 0:
      출력: '다른 음료를 선택하세요'
      음료 선택으로 돌아가기
    음료 배출하기
    거스름돈 = 돈 - 가격
    거스름돈 반환하기
    출력: '감사합니다!'
끝

주제 2: 신호등

신호등은 반복(루프)의 완벽한 예시예요. 초록 → 노랑 → 빨강 → 초록... 끝없이 반복하죠.

[분석] 신호등 문제 분해
입력: 없음 (자동으로 작동)
처리: 각 색 유지 시간 카운트
출력: 현재 신호 색
패턴: 초록(30초) → 노랑(5초) → 빨강(20초) → 초록... (무한 반복)
신호등 순서도 — '반복(loop)'의 실생활 예시!
[의사코드] 신호등
프로그램 신호등
  신호 = 초록
  무한반복:
    만약 신호 == 초록:
      초록 불 켜기
      30초 기다리기
      신호 = 노랑
    만약 신호 == 노랑:
      노랑 불 켜기
      5초 기다리기
      신호 = 빨강
    만약 신호 == 빨강:
      빨강 불 켜기
      20초 기다리기
      신호 = 초록
끝

가위바위보 순서도 — 조건 분기 + 반복이 합쳐진 복합 구조!
[의사코드] 가위바위보
프로그램 가위바위보
  반복:

3// 순서도: [시작]→[금액입력]→{충분?}→[음료선택]→[거스름돈]→[끝]

[C 코드] 가위바위보 (순서도 프로젝트)
1#include <stdio.h>
2#include <stdlib.h>
3#include <time.h>
4
5int main() {
6 srand(time(NULL));
7 const char *hands[] = {"가위", "바위", "보"};
8 int wins = 0, losses = 0, draws = 0;
9
10 printf("=== 가위바위보 (5판 3선승) ===\n\n");
11
12 int round = 0;
13 while (wins < 3 && losses < 3) {
14 round++;
15 printf("[%d라운드] 가위(0) 바위(1) 보(2): ", round);
16 int user;
17 scanf("%d", &user);
18 if (user < 0 || user > 2) { printf("잘못된 입력!\n"); continue; }
19
20 int cpu = rand() % 3;
21 printf(" 나: %s vs 컴퓨터: %s → ", hands[user], hands[cpu]);
22
23 if (user == cpu) { printf("무승부!\n"); draws++; }
24 else if ((user+1)%3 == cpu) { printf("패배...\n"); losses++; }
25 else { printf("승리!\n"); wins++; }
26
27 printf(" (현재: %d승 %d패 %d무)\n\n", wins, losses, draws);
28 }
29
30 printf("=== 최종: %s! ===\n", wins>=3 ? "승리" : "패배");
31 return 0;

[심화 내용] TDD란?
Test-Driven Development:
테스트를 먼저 작성하고, 그 다음 코드를 구현!

3단계 사이클:
1. 🔴 Red: 실패하는 테스트 작성
2. 🟢 Green: 테스트를 통과하는 최소 코드 작성
3. 🔵 Refactor: 코드 정리 (기능 유지)

C에서 TDD:
assert(add(3, 5) == 8); // 먼저 테스트
// 그 다음 add() 함수 구현!

장점:
- 버그 예방 (테스트가 안전망)
- 설계 개선 (테스트하기 좋은 코드 = 좋은 설계)
- 자신감 (리팩토링 후 테스트 통과 = 안심)

모든 순서도 프로젝트에 테스트를 추가해 보세요!

순서도 (Flowchart)
알고리즘의 흐름을 도형과 화살표로 시각적으로 표현한 그림.
의사코드 (Pseudocode)
프로그래밍 언어 문법을 따르지 않고, 사람이 이해할 수 있는 자연어로 알고리즘을 작성한 것.
조건 분기
조건에 따라 서로 다른 경로로 나뉘는 것. 순서도에서는 마름모(◇)로 표현.
반복 (Loop)
특정 조건이 만족될 때까지 같은 과정을 되풀이하는 것. 신호등의 색 변경이 대표적.
검증 (Verification)
순서도나 알고리즘이 올바르게 동작하는지 여러 입력값을 넣어 확인하는 과정.
문제 분해 (Decomposition)
복잡한 문제를 작은 하위 문제들로 나누어 해결하는 컴퓨팅사고력의 핵심 기법.
추상화 (Abstraction)
불필요한 세부사항을 제거하고 핵심 요소만 남기는 것. 자판기에서 '음료 종류'는 추상화.
상태 기계 패턴
프로그램을 상태(state)와 전이(transition)로 설계하는 패턴. switch-case로 구현.
가드 절
함수 시작부에서 예외 조건을 먼저 처리하고 반환하는 패턴. 중첩 감소.
