컴퓨터는 우리 말을 못 알아들어요. 대신 프로그래밍 언어라는 특별한 말로 이야기해야 해요. 오늘은 컴퓨터에게 '화면에 글자를 찍어!'라고 시켜볼 거예요!

[하드웨어] 컴퓨터 안에서 무슨 일이?
여러분이 코드를 작성하면, 그 코드는 RAM(책상 위)에 올라가요. 컴파일러가 코드를 기계어로 번역하면, CPU(두뇌)가 명령을 실행해요. printf로 출력하면 결과가 모니터(눈)에 나타나요.

[코딩 습관] 들여쓰기와 중괄호
코드 안쪽 줄은 스페이스 4칸만큼 들여 써요. 이걸 들여쓰기(indent)라고 해요. 중괄호 { }는 항상 짝이에요 — 여는 괄호가 있으면 닫는 괄호가 반드시 있어야 해요. 들여쓰기를 하면 어디가 main 함수 안쪽인지 한눈에 보여요.
1줄#include <stdio.h> — '나 printf 쓸 거야'라고 미리 알려주는 줄이에요. 이 줄이 없으면 printf를 못 써요. stdio = Standard Input Output(표준 입출력)의 줄임말이에요.
3줄int main(void) — 프로그램의 시작점이에요. 모든 C 프로그램은 main에서 시작해요. 이건 약속이에요.
4줄printf("Hello, World!\n"); — 큰따옴표 안의 글자를 화면에 찍어요. \n은 '여기서 줄 바꿔'라는 뜻이에요. 마지막 세미콜론(;)은 '이 문장 끝!'이라는 표시예요.
5줄return 0; — '프로그램 끝! 문제없이 잘 끝났어'라는 뜻이에요.

[심화 내용] printf의 내부 동작

[심화 내용] printf가 화면에 글자를 출력하는 과정
printf("Hello") 한 줄이 실행되면:

1. 가변 인자 파싱
포맷 문자열 "%d", "%s" 등 해석
va_list로 인자를 하나씩 꺼냄

2. 버퍼링
stdout 버퍼에 문자열 저장 (즉시 출력 아님!)
\n을 만나거나 버퍼가 차면 flush

3. 시스템 콜
write(1, "Hello\n", 6) 시스템 콜 호출
1 = stdout 파일 디스크립터

4. 커널
터미널 드라이버로 문자 전달

5. 터미널
문자를 픽셀로 렌더링하여 화면에 표시

한 줄의 printf에 이렇게 많은 일이!

[심화 내용] printf 포맷 지정자 총정리
기본 타입:
%d, %i — 정수 (int)

%f — 실수 (float/double)

%s — 문자열 (char*)
%p — 포인터 주소
%x — 16진수 (소문자)
%X — 16진수 (대문자)
%o — 8진수
%% — % 문자 자체

폭과 정밀도:
%10d — 최소 10칸, 우측 정렬
%-10d — 최소 10칸, 좌측 정렬
%05d — 5칸, 0으로 채움 (00042)
%.3f — 소수점 3자리
%10.2f — 10칸, 소수점 2자리

크기 수정자:
%ld — long int
%lld — long long int
%zu — size_t

[심화 내용] stdin, stdout, stderr

[심화 내용] 3개의 표준 스트림
C 프로그램은 3개의 표준 입출력 스트림을 가짐:

stdin (0) — 표준 입력
키보드에서 읽기. scanf, getchar, fgets

stdout (1) — 표준 출력
화면에 쓰기. printf, putchar, puts
라인 버퍼링: \n에서 flush

stderr (2) — 표준 에러
에러 메시지 출력. fprintf(stderr, ...)
버퍼링 없음: 즉시 출력!

리다이렉션:
./main > output.txt (stdout만 파일로)
./main 2> error.txt (stderr만 파일로)
./main > out.txt 2>&1 (둘 다 파일로)
./main < input.txt (stdin을 파일에서)

에러 메시지는 반드시 stderr로!
[심화 내용] 면접 문제
Q1. printf와 puts의 차이?
A: puts는 자동으로 \n 추가, 포맷 없음. printf는 포맷 지정자 지원, \n 수동.

Q2. printf의 반환값?
A: 출력된 문자 수. 에러 시 음수.
int n = printf("Hi"); // n = 2

Q3. fprintf(stderr, ...) vs printf(...)의 차이?
A: stderr는 버퍼링 없이 즉시 출력. 리다이렉션 시 분리됨.

Q4. printf에서 %를 출력하려면?
A: %% 사용. printf("100%%"); → 100%

[심화 내용] main 함수 이전에 일어나는 일
프로그램이 시작되면 main()이 호출되기 전에:

1. _start (엔트리 포인트)
OS 로더가 호출하는 진짜 시작점
C 런타임(CRT) 초기화 코드

2. 전역 변수 초기화
.data 섹션: 초기화된 전역변수 설정
.bss 섹션: 0으로 초기화

3. 표준 라이브러리 초기화
stdin/stdout/stderr 스트림 설정
힙(heap) 메모리 관리자 초기화

4. main(argc, argv, envp) 호출
argc: 명령줄 인자 개수
argv: 인자 문자열 배열
envp: 환경 변수 배열 (비표준)

5. main 반환 후
atexit 등록 함수 호출
열린 파일 닫기
exit(main_return_value);
