프로그래밍 언어가 뭘까?

안녕! 오늘은 프로그래밍 언어가 뭔지, 그리고 왜 우리가 C언어를 배우는지 알아볼 거야!

사람끼리 한국어, 영어, 일본어 같은 언어로 대화하듯, 컴퓨터에게도 특별한 말이 필요해요. 이걸 프로그래밍 언어라고 해요. 프로그래밍 언어는 하나가 아니라 수백 가지나 돼요!

[핵심] 프로그래밍 언어의 종류
C — 운영체제, 게임 엔진, 로봇 (빠르고 강력)
Python — AI, 데이터 분석 (쉽고 편리)
Java — 안드로이드 앱, 서버 (어디서든 실행)
JavaScript — 웹사이트, 인터넷 (브라우저에서 실행)
Scratch — 블록 코딩 (초등학생 입문용)
Scratch로 코딩해 본 적 있나요? 블록을 끌어다 놓는 방식이죠? C언어는 글자를 직접 타이핑하는 텍스트 코딩이에요. 더 어렵지만, 할 수 있는 게 훨씬 많아요!

C언어의 구조와 역사

1972
데니스 리치가 벨 연구소에서 C언어를 만들었어요. 50년이 넘은 언어인데 아직도 전 세계에서 가장 많이 쓰이는 언어 중 하나예요!

C언어는 컴퓨터의 뼈대를 만드는 언어예요. 여러분이 쓰는 Windows, 스마트폰의 Android, 게임기 안의 프로그램 — 이런 것들의 핵심이 C언어로 만들어져 있어요.

[하드웨어] C언어와 컴퓨터의 관계
C언어는 다른 언어보다 하드웨어(CPU, 메모리)에 가까운 언어예요. 그래서 컴퓨터가 어떻게 돌아가는지 이해하기 좋아요. 로봇, 드론, 자동차 같은 기계를 제어할 때도 C를 써요.

C언어를 배우면 다른 언어도 쉽게 배울 수 있어요! C는 프로그래밍 언어의 기본기거든요. 축구로 치면 패스 연습 같은 거예요!

C언어의 실수하기 쉬운 점

C언어는 규칙이 엄격해요. 세미콜론 하나만 빠져도 에러가 나요. 하지만 그 덕분에 정확한 코딩 습관이 생겨요!
[주의] C언어가 엄격한 이유
C언어는 세미콜론(;), 중괄호({ }), 대소문자를 틀리면 바로 에러가 나요. 귀찮아 보이지만, 이 규칙 덕분에 나중에 어떤 언어를 배워도 깔끔한 코드를 쓸 수 있어요.

[심화 내용] C 표준 연대기
K&R C (1978) — 최초의 사실상 표준
함수 정의 방식이 현재와 다름

ANSI C / C89 (1989) — 첫 공식 표준
함수 프로토타입, void, const, enum 추가

C99 (1999) — 현대적 기능 추가

stdbool.h, stdint.h, 가변 길이 배열(VLA)

C11 (2011) — 동시성 지원
_Atomic, threads.h, _Generic, 익명 구조체

C17 (2018) — 버그 수정 릴리스
새 기능 없이 C11의 결함 수정만

C23 (2024) — 최신 표준
nullptr, constexpr, typeof, #embed
속성(attributes), 개선된 열거형

[심화 내용] C 언어의 설계 철학

[심화 내용] C의 설계 원칙
1. 프로그래머를 믿는다
배열 범위 검사 안 함 → 빠르지만 위험
"총을 주면 발을 쏠 자유도 준다"

2. 작고 간결하다
키워드 32개 (C89). Python은 35+, Java는 50+
언어 자체는 작고, 라이브러리로 확장

3. 하드웨어에 가깝다
포인터 = 메모리 주소 직접 접근
비트 연산 = CPU 명령어와 1:1 대응

4. 이식성
같은 C 코드를 x86, ARM, RISC-V에서 컴파일
"이식성 있는 어셈블리어"

5. 효율적이다
런타임 오버헤드 최소 (GC 없음, VM 없음)
기계어에 가장 가까운 고급 언어

[심화 내용] C 언어의 위치

[심화 내용] 프로그래밍 언어 계보
C의 조상:
ALGOL (1958) → CPL (1963) → BCPL (1967)
→ B (1970, Thompson) → C (1972, Ritchie)

C의 후손:
C++ (1979): C + 객체지향 + 템플릿
Objective-C (1984): C + 메시지 패싱
Java (1995): C 문법 + GC + VM
C# (2000): Microsoft의 Java 대응
JavaScript (1995): C 문법 + 동적 타입
Go (2009): C의 단순함 + 동시성
Rust (2010): C의 성능 + 메모리 안전
Swift (2014): Objective-C 대체

[심화 내용] C와 UNIX의 공생 관계

[심화 내용] C와 UNIX는 하나
"C 없이 UNIX 없고, UNIX 없이 C 없다"

1. UNIX를 C로 다시 작성 (1973)
→ C가 실전에서 검증됨

2. UNIX가 대학에 무료 배포
→ C를 배우는 학생 폭증

3. BSD(Berkeley) UNIX 확산
→ TCP/IP, 소켓, vi 모두 C로 작성

4. POSIX 표준
→ C API가 표준 OS 인터페이스

5. Linux (1991, 토발즈)
→ C로 작성된 오픈소스 UNIX 클론
→ 서버, Android, 임베디드 지배

인터넷의 기반 = C + UNIX
현대 클라우드의 90%+ = Linux(C)

90%+

[심화 내용] C의 정의되지 않은 동작 (UB) 모음
C에서 절대 하면 안 되는 것들:

1. 배열 범위 초과
int a[5]; a[10] = 42; // UB!

2. 초기화 안 된 변수 사용
int x; printf("%d", x); // UB!

3. 0으로 나누기
int y = 10/0; // UB!

4. NULL 포인터 역참조
int *p = NULL; *p = 5; // UB!

5. 부호 있는 정수 오버플로우
INT_MAX + 1; // UB! (unsigned는 OK)

6. 해제된 메모리 접근

[C 코드] C의 강력함 체험 — 비트 조작
1#include <stdio.h>
2
3int main() {
4 // C의 강점: 비트 단위 조작
5 unsigned char flags = 0; // 8개의 플래그를 1바이트에!
6
7 #define FLAG_READ(1 << 0) // 0b00000001
8 #define FLAG_WRITE(1 << 1) // 0b00000010
9 #define FLAG_EXECUTE(1 << 2) // 0b00000100
10
11 // 플래그 설정
12 flags |= FLAG_READ | FLAG_WRITE; // 읽기+쓰기 ON
13 printf("flags = 0x%02X\n", flags); // 0x03
14
15 // 플래그 확인
16 if (flags & FLAG_READ) printf("읽기 가능\n");
17 if (flags & FLAG_EXECUTE) printf("실행 가능\n");
18 else printf("실행 불가\n");
19
20 // 플래그 토글
21 flags ^= FLAG_EXECUTE; // 실행 ON
22 printf("토글 후: 0x%02X\n", flags); // 0x07
23
24 // 플래그 해제
25 flags &= ~FLAG_WRITE; // 쓰기 OFF
26 printf("해제 후: 0x%02X\n", flags); // 0x05
27
28 return 0;
29}
실행 결과
flags = 0x03 읽기 가능 실행 불가 토글 후: 0x07 해제 후: 0x05
C는 비트 하나까지 제어할 수 있어! 이런 저수준 조작은 Python이나 Java에서는 불가능하거나 비효율적이야. C의 진짜 힘!

[디버깅] C언어 초보가 자주 하는 실수

✕ 잘못된 코드
#include int Main(void) { // ← Main (대문자!) printf("Hello\n"); return 0; }
Main ≠ main! C는 대소문자를 구분!
✓ 올바른 코드
#include int main(void) { // ← main (소문자!) printf("Hello\n"); return 0; }
항상 소문자 main

C언어는 대소문자를 구분해요. Main, MAIN, main은 전부 다른 이름이에요. 프로그램 시작점은 반드시 main(소문자)이어야 해요.
