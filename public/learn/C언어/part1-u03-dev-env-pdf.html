코딩 환경이 뭘까?

오늘은 직접 코딩할 수 있는 환경을 만들어 볼 거야! 요리사에게 주방이 필요하듯, 프로그래머에게는 코딩 환경이 필요해!

코딩 환경은 크게 두 가지가 필요해요. 코드를 쓸 에디터(메모장 같은 것)와, 코드를 기계어로 번역할 컴파일러예요.

1
에디터 (Editor)
코드를 타이핑하는 프로그램이에요. 메모장도 되지만, VS Code가 훨씬 편해요.
2
컴파일러 (Compiler)
우리가 쓴 C 코드를 컴퓨터가 이해하는 기계어로 번역해 줘요. gcc가 대표적이에요.
3
터미널 (Terminal)
컴파일하고 실행하는 명령을 입력하는 곳이에요. 검은 화면에 글자를 치는 방식이에요.

방법 1: 온라인 IDE (설치 없이 바로 시작)

설치가 어려우면 온라인으로 바로 시작할 수 있어요! 인터넷만 되면 OK!
[추천] 온라인 C 코딩 사이트
OnlineGDB — onlinegdb.com (가장 추천! 한글도 잘 나와요)
Replit — replit.com (무료 계정 필요)
Programiz — programiz.com/c-programming/online-compiler

사이트에 접속 → C 선택 → 코드 입력 → Run 버튼 클릭!

방법 2: VS Code + gcc 설치 (추천)

자기 컴퓨터에 설치하면 인터넷 없이도 코딩할 수 있어요. 처음엔 좀 복잡하지만, 한 번만 하면 돼요!

[심화 내용] Makefile 심화

[심화 내용] Makefile 패턴
자동 변수:
$@ = 타겟, $< = 첫 의존성, $^ = 모든 의존성

패턴 규칙:
%.o: %.c
$(CC) $(CFLAGS) -c $< -o $@

변수:
CC = gcc
CFLAGS = -Wall -Wextra -g
SRCS = $(wildcard *.c)
OBJS = $(SRCS:.c=.o)

완전한 Makefile:
CC = gcc
CFLAGS = -Wall -g
TARGET = main
SRCS = main.c utils.c math.c
OBJS = $(SRCS:.c=.o)

$(TARGET): $(OBJS)
$(CC) -o $@ $^

%.o: %.c
$(CC) $(CFLAGS) -c $<

clean:
rm -f $(OBJS) $(TARGET)

.PHONY: clean
[C 코드] 여러 파일 프로젝트
1// === math_utils.h ===
2#ifndef MATH_UTILS_H
3#define MATH_UTILS_H
4int gcd(int a, int b);
5int lcm(int a, int b);
6int is_prime(int n);
7int factorial(int n);
8#endif
9
10// === math_utils.c ===
11#include "math_utils.h"
12int gcd(int a, int b) { return b ? gcd(b, a%b) : a; }
13int lcm(int a, int b) { return a / gcd(a,b) * b; }
14int is_prime(int n) {
15 if (n<2) return 0;
16 for (int i=2; i*i<=n; i++) if (n%i==0) return 0;
17 return 1;
18}
19int factorial(int n) { return n<=1 ? 1 : n*factorial(n-1); }
20
21// === main.c ===
22#include <stdio.h>

[심화 내용] 면접 문제
Q1. gcc와 g++의 차이?
A: gcc=C 컴파일러, g++=C++ 컴파일러. gcc로 .cpp 파일은 C로 처리!

Q2. -Wall -Wextra -Werror의 의미?
A: Wall=주요 경고 켜기, Wextra=추가 경고, Werror=경고를 에러로. 엄격한 컴파일!

Q3. make와 cmake의 관계?
A: cmake는 Makefile을 생성하는 도구. cmake→Makefile→make→빌드.

Q4. 정적 분석 vs 동적 분석?
A: 정적=실행 없이 검사(cppcheck), 동적=실행 중 검사(Valgrind). 둘 다 해야 완벽!

[심화] GCC vs Clang 선택 가이드
GCC를 선택할 때:
- Linux 네이티브 개발
- 최대 최적화 성능 필요
- Fortran, Ada 등 다중 언어
- 오래된 코드 호환성

Clang을 선택할 때:
- macOS/iOS 개발
- 친절한 에러 메시지 필요
- IDE 통합 (코드 완성, 리팩토링)
- 정적 분석 도구 활용
- 컴파일 속도 중시

결론:
Linux → GCC, macOS → Clang
Windows → MinGW(GCC) 또는 WSL(GCC)
어떤 것이든 C 표준 코드는 동일하게 동작!

- run: gcc -Wall -Wextra -Werror -o main main.c

- run: valgrind --error-exitcode=1 ./main

[심화] GCC 옵션 총정리

[심화] GCC 자주 쓰는 옵션 20개
기본 컴파일:
gcc main.c → a.out (기본 출력)
gcc main.c -o main → main으로 출력
gcc -c main.c → main.o (오브젝트만)

경고 옵션:
-Wall: 주요 경고 활성화
-Wextra: 추가 경고
-Werror: 경고→에러 변환
-pedantic: 표준 엄격 준수

표준 선택:
-std=c99: C99 표준
-std=c11: C11 표준
-std=c17: C17 표준

최적화:
-O0: 없음 (디버깅용, 기본)
-O1: 가벼운 최적화
-O2: 권장 최적화 (릴리즈)
-O3: 최대 최적화 (공격적)
-Os: 크기 최적화

디버깅:
-g: 디버깅 정보 포함 (GDB용)
-fsanitize=address: ASan 활성화
-pg: 프로파일링 (gprof)

라이브러리:
-lm: math.h 링크 (Linux 필수!)
-lpthread: 스레드 라이브러리
