모든 C 프로그램의 구조

C 프로그램은 항상 같은 뼈대로 시작해요. 오늘은 그 뼈대를 완벽히 이해하고, 외워 버릴 거예요!
skeleton.c (C 프로그램의 뼈대)
1#include <stdio.h>
2
3int main(void) {
4 // 여기에 코드를 써요
5 return 0;
6}

모든 C 프로그램은 이 뼈대에서 시작해요. 4번 줄 자리에 우리가 원하는 코드를 넣으면 돼요. 이 뼈대를 외우세요 — 앞으로 수백 번 쓸 거예요!

#include 뜯어보기

#include <stdio.h>는 '나 이 기능 쓸 거야'라고 미리 알려주는 줄이에요. stdio.h는 Standard Input Output(표준 입출력)의 줄임말이에요.

[핵심] #include는 도구 상자를 여는 것
printf를 쓰려면 #include <stdio.h>가 필요해요. 마치 요리하기 전에 냉장고에서 재료를 꺼내는 것과 같아요. 재료(stdio.h)를 안 꺼내면 요리(printf)를 못 해요!
나중에 수학 함수를 쓸 때는 #include <math.h>, 문자열 함수를 쓸 때는 #include <string.h>를 추가해요. 필요한 도구 상자를 그때그때 열면 돼요!

main 함수 뜯어보기

int main(void)는 프로그램의 시작점이에요. 컴퓨터는 프로그램을 실행할 때 항상 main부터 시작해요. 이건 모든 C 프로그램의 약속이에요.

0줄int — main이 끝나면 숫자를 돌려준다는 뜻이에요. (int = integer = 정수)
0줄main — '주요한, 중심이 되는'이라는 뜻의 영어 단어. 프로그램의 중심이에요.
0줄(void) — '받을 게 없다'는 뜻이에요. 나중에 여기에 값을 받을 수도 있어요.
0줄{ } — 중괄호 안이 main 함수의 몸통이에요. 코드는 이 안에 써요.
[하드웨어] main과 CPU의 관계
프로그램을 실행하면, 운영체제가 CPU에게 'main부터 시작해!'라고 알려줘요. CPU는 main의 첫 줄부터 마지막 줄까지 한 줄씩 실행해요.

return 0 뜯어보기

return 0;은 '프로그램 끝! 문제없이 잘 끝났어!'라는 뜻이에요. 0은 '성공'을 의미해요.

✕ 잘못된 코드
return 1;
1은 '에러가 있었어'라는 뜻이에요
✓ 올바른 코드
return 0;
0은 '문제없이 성공!'이라는 뜻이에요
심부름을 마치고 돌아와서 '다 했어요!'라고 보고하는 것과 같아요. return 0은 '심부름 성공!' 보고예요!

[심화 내용] 컴파일 단위와 링킹

[심화 내용] 번역 단위 (Translation Unit)
번역 단위 = .c 파일 + 포함된 모든 헤더

각 .c 파일은 독립적으로 컴파일:
main.c → main.o
utils.c → utils.o
math.c → math.o

링커가 .o 파일들을 합쳐서 실행 파일 생성:
main.o + utils.o + math.o + libc → main.exe

파일 간 통신:
- extern: "다른 파일에 있어요" 선언
- static: "이 파일 전용" 선언 (정보 은닉!)
- 헤더 파일: 공유 인터페이스 (프로토타입, 타입)

원칙:
- 헤더(.h): 선언만 (함수 프로토타입, 타입)
- 소스(.c): 정의 (실제 구현)
[심화 내용] 헤더 가드의 필요성
문제: 같은 헤더가 2번 포함되면?
a.h에서 #include "common.h"
b.h에서 #include "common.h"
main.c에서 둘 다 include → common.h 2번!
→ "중복 정의" 에러!

해결: 헤더 가드
#ifndef COMMON_H
#define COMMON_H
// 내용
#endif

또는 (비표준이지만 대부분 지원):
#pragma once

규칙:
모든 .h 파일에 반드시 헤더 가드!
매크로명 = 파일명 대문자 + _H
7
잘 구성된 C 프로그램의 구성요소 수. 전처리기→매크로→타입→프로토타입→전역→main→함수구현.
C 코드의 구조는 건물의 설계도와 같아. 기초(전처리기)→뼈대(타입)→방(함수)→입구(main). 구조가 탄탄해야 건물이 안 무너져!
프로토타입을 먼저 선언하는 이유? 컴파일러는 위에서 아래로 읽으니까! main 아래에 있는 함수도 호출할 수 있게 해줘!

[심화 내용] 소프트웨어 설계 원칙

[심화 내용] SOLID 원칙 (C 관점)
S — Single Responsibility (단일 책임)
하나의 함수/파일 = 하나의 역할
calc.c = 계산만, io.c = 입출력만

O — Open/Closed (개방/폐쇄)
확장에 열려있고, 수정에 닫혀있게
함수 포인터로 동작 교체 가능!

L — Liskov Substitution
C에서는 함수 포인터의 시그니처 일치
같은 프로토타입 = 교체 가능

I — Interface Segregation
헤더 파일을 잘게 분리
math.h, string.h, io.h 각각 독립

D — Dependency Inversion
구체적 구현 대신 함수 포인터(인터페이스)에 의존
qsort의 비교 함수가 대표적 예시!

[심화 내용] 코드 스멜(Code Smell) 감지
"코드 스멜" = 리팩토링이 필요한 신호

1. 긴 함수 (>50줄) → 분리
2. 긴 매개변수 목록 (>4개) → 구조체로
3. 중복 코드 → 함수 추출
4. 매직 넘버 → #define 상수로
5. 깊은 중첩 (3단계+) → 가드 절
6. 주석이 많은 코드 → 코드 자체를 명확하게
7. 전역 변수 → 매개변수로
8. 죽은 코드 → 삭제 (Git이 기억해줌!)

스멜을 느끼면 바로 리팩토링!

[심화 내용] C의 스코프 규칙
1. 블록 스코프
{ } 안에서 선언된 변수는 { } 안에서만 유효
for (int i=0; ...) → i는 for 안에서만!

2. 파일 스코프
함수 밖에서 선언 → 파일 전체에서 유효
static int count; → 이 파일에서만!
int count; → 다른 파일에서도 (extern)

3. 함수 스코프
goto 레이블만 해당 (거의 안 씀)

4. 함수 프로토타입 스코프
void f(int n, int arr[n]); → n은 선언 내에서만

이름 가리기 (Shadowing):
int x = 10;
{ int x = 20; // 바깥 x를 가림! }
→ -Wshadow 옵션으로 경고 받을 수 있음
#pragma once
헤더 가드의 간편 버전. 표준은 아니지만 GCC/Clang/MSVC 모두 지원. 가장 쉬운 중복 방지!
[활동] 코드 구조 체크리스트
□ 모든 함수가 20줄 이하인가?
□ 전역 변수가 0~2개 이하인가?
□ 헤더 파일에 헤더 가드가 있는가?
□ 함수 프로토타입을 먼저 선언했는가?
□ #include 순서가 일관적인가?
□ 매직 넘버가 #define으로 정의되었는가?
□ 가드 절로 중첩을 줄였는가?

6개 이상 ✅이면 좋은 코드 구조!
extern
"이 변수/함수는 다른 파일에 있어요" 선언. 링커가 연결해줌. 반대는 static(이 파일 전용).
함수 하나는 한 화면(20~30줄)에 들어와야 해! 스크롤해야 읽히는 함수는 분리할 시간이야!
static
C에서 "이 파일 전용" 선언. 다른 파일에서 접근 불가. 정보 은닉(캡슐화)의 C 방식!
Translation Unit
번역 단위 = .c 파일 + 모든 #include. 각각 독립 컴파일 후 링커가 합침.
좋은 코드 구조의 비밀: "변경이 한 곳에서만 일어나게" 만드는 거야. 함수/파일 분리의 목적이 바로 이거!
SOLID
객체지향 5대 원칙. C에서도 함수 분리, 인터페이스 분리, 의존성 역전(함수 포인터) 적용 가능!
헤더(.h)는 메뉴판, 소스(.c)는 주방! 손님(다른 파일)은 메뉴판만 보고 주문하고, 주방에서 요리한다!
#include 순서
권장: 1)시스템 헤더() 2)서드파티 3)프로젝트("my.h"). Google Style에서도 동일 규칙.
프로토타입 → main → 함수구현. 이 순서로 쓰면 main이 "목차" 역할을 해서 전체 구조가 한눈에!
#include
헤더 파일을 불러오는 전처리 지시문. '이 도구 상자를 열어줘'라는 뜻.
stdio.h
Standard Input Output의 줄임말. printf, scanf 등 입출력 함수가 들어있는 헤더 파일.
main
프로그램의 시작점(진입점). 모든 C 프로그램은 main에서 시작.
return
함수가 끝나면서 값을 돌려주는 키워드. return 0은 '성공'.
헤더 파일
.h 확장자를 가진 파일. 함수들의 설명서(선언)가 들어있어요.

중괄호 {} 누락
✓ 올바른 코드

✕ 잘못된 코드

int 빠짐! 경고 발생
✓ 올바른 코드
int main(void) { printf("Hello"); return 0; }

[실생활] 프로그램 구조의 비유

일상의 순서 = 프로그램의 순서
요리: 재료 준비(#include) → 불 켜기(main) → 볶기(printf) → 불 끄기(return)
등교: 알람(#include) → 일어나기(main) → 학교가기(printf) → 집오기(return)
게임: 로딩(#include) → 시작(main) → 플레이(printf) → 종료(return)

모든 것에 시작과 끝이 있어요!
undefined
#include 의 stdio = Standard Input Output. 표준 입출력. 화면에 출력하고 키보드 입력받는 도구!

main = "주요한". 프로그램의 시작점! 영어 뜻 그대로 "메인 함수"예요.
undefined
void = "비어있는". int main(void)에서 void는 "입력 없음"이라는 뜻!
#include = 도서관에서 필요한 책을 빌리는 것!
