세미콜론(;) — 문장의 마침표

한국어에서 문장이 끝나면 마침표(.)를 찍잖아요? C언어에서는 세미콜론(;)이 마침표 역할을 해요!

C언어에서 명령 한 줄이 끝날 때마다 세미콜론을 붙여요. 이게 없으면 컴퓨터는 '아직 문장이 안 끝났나?'라고 혼란스러워해요.

semicolons.c
1#include <stdio.h>
2
3int main(void) {
4 printf("첫 번째 줄\n"); // 세미콜론!
5 printf("두 번째 줄\n"); // 세미콜론!
6 return 0; // 세미콜론!
7}
[핵심] 세미콜론을 붙이는 곳 vs 안 붙이는 곳
붙이는 곳: printf, return, 변수 선언, 대입문 등 실행하는 문장
안 붙이는 곳: #include, int main(void) {, 중괄호 } 뒤

쉽게 외우기: '컴퓨터야, 이거 해!'라고 시키는 줄에만 세미콜론을 붙여요.
세미콜론을 빼먹으면 에러 메시지에 expected ';'이 나와요. 이 메시지가 보이면 바로 위 줄을 확인하세요!

중괄호({ }) — 코드의 울타리

중괄호는 코드를 묶는 울타리예요. main 함수의 시작과 끝을 표시해요. 나중에 배울 if, for 등에서도 중괄호로 코드를 묶어요.

braces.c
1#include <stdio.h>
2
3int main(void) { // 여기서 시작!
4 printf("안녕!\n");
5 printf("반가워!\n");
6 return 0;
7} // 여기서 끝!
[코딩 습관] 중괄호 짝 맞추기 비법
규칙: 여는 중괄호 {를 쓰면, 바로 닫는 중괄호 }를 쓰고, 그 다음 안쪽을 채우세요!

1. int main(void) { 쓰기
2. 바로 다음 줄에 } 쓰기
3. 그 사이에 코드 채우기

이렇게 하면 짝을 절대 안 잃어버려요!

들여쓰기와 중괄호의 관계

중괄호 안의 코드는 스페이스 4칸 들여 써요. 이렇게 하면 어디가 main 안쪽인지 한눈에 보여요.

[심화] 세미콜론의 깊은 이해

[심화] C에서 세미콜론의 역할
세미콜론(;) = 문장의 끝

C에서 세미콜론이 필요한 곳:
1. 변수 선언: int x = 5;
2. 함수 호출: printf("hi");
3. 대입: x = x + 1;
4. return: return 0;
5. break/continue: break;

세미콜론이 불필요한 곳:
1. #include → 전처리기 지시문 (줄 끝이 종료)
2. #define → 전처리기
3. 함수 정의 { } 뒤
4. if/for/while 블록 { } 뒤
5. 구조체 정의 { } 뒤에는 필요!
struct Point { int x,y; }; ← 이건 필요!

빈 문장:
; ← 이것만으로도 유효한 문장!
while(getchar()!='\n'); ← 이런 식으로 사용

[심화 내용] Apple goto fail 버그 교훈
2014년 Apple SSL/TLS 보안 버그:

중괄호를 생략한 if 문에서
들여쓰기된 goto fail;이 2줄 연속 있었음
→ 두 번째 goto fail은 항상 실행!
→ SSL 인증이 항상 실패 처리!
→ 중간자 공격(MITM) 가능!

교훈:
1. if 뒤에 항상 { } 사용
2. clang-format으로 자동 포맷
3. 코드 리뷰 (2번째 goto를 잡았을 것!)
4. -Wunreachable-code 옵션 활용

한 줄 코드라도 중괄호를 쓰는 것이 안전!
goto fail
2014년 Apple SSL 버그. 중괄호 생략 + 중복 goto → 보안 인증 우회. 중괄호는 항상 쓰자!
세미콜론과 중괄호는 C의 "문법 부호"야. 작아 보이지만 하나 빠지면 프로그램이 완전히 달라져! Apple 버그처럼!
"항상 중괄호 쓰기" 규칙 하나만 지켜도 수많은 버그를 예방할 수 있어! 한 줄이라도 { }!

[심화 내용] Dangling Else 문제

[심화 내용] Dangling Else
문제: else는 어떤 if에 속할까?

if (a > 0)
if (b > 0)
printf("둘 다 양수");
else
printf("이건 뭐에 속하지?");

C의 규칙: else는 가장 가까운 if에!
→ else는 안쪽 if(b>0)에 속함
→ a>0이고 b<=0일 때 출력
→ a<=0이면? 아무것도 안 출력!

해결: 중괄호로 명확하게!
if (a > 0) {
if (b > 0) {
printf("둘 다 양수");
}
} else {
printf("a가 0 이하");
}

중괄호가 있으면 모호함이 사라진다!

[심화] 매크로와 세미콜론 주의점
#define 끝에 세미콜론 넣지 마세요!

// 나쁜 예
#define PI 3.14; ← 세미콜론 포함!
double area = PI * r * r;
→ 3.14; * r * r → 문법 에러!

// 좋은 예
#define PI 3.14

매크로 함수와 do-while 트릭:
#define SWAP(a,b) do { int t=a; a=b; b=t; } while(0)

왜 do-while(0)? → if 뒤에서 안전하게 사용하려고!
if (cond)
SWAP(x, y); // 세미콜론이 자연스럽게 동작

[심화 내용] 문(Statement) vs 식(Expression)
식(Expression) = 값을 반환
3 + 5 → 8
x > 0 ? 1 : -1 → 1 또는 -1
a = 5 → 5 (대입도 식!)
printf("hi") → 2 (출력 글자 수)

문(Statement) = 행동을 수행
int x = 5; → 변수 선언문
if (x > 0) {} → 조건문
for (;;) {} → 반복문
return 0; → 반환문
break; → 제어문

세미콜론의 역할:
식 + ; = 식 문장 (expression statement)
x + 5; → 합법이지만 결과를 안 쓰니 무의미
gcc -Wall이 "statement with no effect" 경고!

식이 될 수 있는 것들:
대입(=), 함수 호출, 증감(++ --), 콤마(,), 삼항(?:)

[심화 내용] C의 문법 BNF 맛보기
C 문법의 형식 정의 (축약):

statement → expression-stmt | compound-stmt | selection-stmt | iteration-stmt | jump-stmt

expression-stmt → expression? ";"
compound-stmt → "{" declaration* statement* "}"
selection-stmt → "if" "(" expression ")" statement ("else" statement)?
iteration-stmt → "while" "(" expression ")" statement
jump-stmt → "return" expression? ";" | "break" ";" | "continue" ";"

핵심:
- 세미콜론은 문장의 끝 표시
- 중괄호는 여러 문장을 하나로 묶음
- 이 규칙을 이해하면 C 문법 에러의 90%를 잡을 수 있음!
C의 세미콜론 규칙은 간단해: "실행되는 문장 끝에 ;". 전처리기(#)는 실행이 아니라 텍스트 치환이니까 ; 불필요!

[CT] 문법 = 규칙

한국어에도 문법이 있듯이, C언어에도 문법이 있어요. 세미콜론과 중괄호는 C 문법의 가장 기본적인 규칙!

한국어 vs C언어 문법
한국어: 문장 끝에 마침표(.) → C언어: 문장 끝에 세미콜론(;)
한국어: 문단은 들여쓰기로 구분 → C언어: 코드 블록은 중괄호 {}로 구분
한국어: 맞춤법 틀리면 의미 불분명 → C언어: 문법 틀리면 컴파일 에러!
한국어는 맞춤법 틀려도 대충 알아듣지만, C는 세미콜론 하나 빠져도 에러! 컴퓨터는 엄격해요.
