주석이 뭘까?

교과서에 연필로 메모하는 것처럼, 코드에도 메모를 남길 수 있어요. 이걸 주석(comment)이라고 해요!

주석은 컴퓨터가 무시하는 글이에요. 사람이 읽기 위해 쓰는 메모예요. 코드가 뭘 하는지 설명하거나, 나중에 기억하려고 적어 놓는 거예요.

한 줄 주석 // 사용법

comment1.c
1#include <stdio.h>
2
3int main(void) {
4 // 화면에 인사를 출력하는 프로그램
5 printf("안녕하세요!\n"); // 인사 출력
6 printf("반갑습니다!\n"); // 인사 출력 2
7 return 0; // 프로그램 종료
8}

// 뒤에 쓴 글은 그 줄 끝까지 전부 주석이에요. 컴퓨터는 // 뒤를 완전히 무시해요. 코드 옆에 쓸 수도 있고, 한 줄을 통째로 주석으로 만들 수도 있어요.

여러 줄 주석 /* */ 사용법

comment2.c
1#include <stdio.h>
2
3/*
4 작성자: 코딩이
5 날짜: 2026년 2월
6 설명: 인사를 출력하는 프로그램
7*/
8int main(void) {
9 printf("안녕!\n");
10 return 0;
11}

/*로 시작하고 */로 끝나면, 그 사이 전부가 주석이에요. 여러 줄에 걸쳐 긴 설명을 적을 때 편해요.

프로 개발자들은 코드 시작 부분에 작성자, 날짜, 설명을 적어요. 우리도 이 습관을 들여 봐요!

주석은 왜 필요할까?

1
미래의 나를 위해
1주일 뒤에 내가 쓴 코드를 다시 보면 '이게 뭐였지?' 할 수 있어요
2
다른 사람을 위해
팀으로 일할 때, 주석이 없으면 다른 사람이 내 코드를 이해 못 해요
3
코드를 잠시 끄기
테스트할 때 코드를 지우지 않고 주석으로 막아놓을 수 있어요
[코딩 습관] 좋은 주석 vs 나쁜 주석
좋은 주석: // 사용자 이름을 입력받아 인사를 출력
나쁜 주석: // printf를 실행한다 (코드 보면 아는 내용)

규칙: 코드가 '왜' 이렇게 쓰였는지 적어요. '무엇을' 하는지는 코드 자체가 말해줘요.

[심화 내용] 주석 없이 읽히는 코드 만들기
자기 문서화 코드(Self-Documenting Code):
코드 자체가 의도를 설명하는 코드

원칙 1: 의미 있는 이름
int d; → int elapsed_days;
int tp; → int total_price;

원칙 2: 설명적 함수명
int f(int x) → bool is_leap_year(int year)
void p(int* a) → void sort_ascending(int* arr)

원칙 3: 상수에 이름 부여
if (age >= 18) → if (age >= LEGAL_ADULT_AGE)
sleep(86400) → sleep(SECONDS_PER_DAY)

원칙 4: 불리언 함수
if (check(x)) → if (is_valid_input(x))
if (proc(s)) → if (has_permission(user))

이렇게 하면 주석 없이도 코드가 읽힌다!

[C 코드] 주석 활용 실전 예제
1#include <stdio.h>
2#include <string.h>
3#include <ctype.h>
4
5/*
6 * ROT13 암호화/복호화
7 *
8 * ROT13은 알파벳을 13자리 회전시키는 암호.
9 * 암호화와 복호화가 동일한 연산 (13+13=26).
10 * 보안 목적이 아닌, 스포일러 방지 등에 사용.
11 *
12 * 예: "Hello" → "Uryyb"
13 */
14void rot13(char *text) {
15 for (int i = 0; text[i]; i++) {
16 if (isalpha(text[i])) {
17 // 대소문자의 기준점을 결정
18 char base = isupper(text[i]) ? 'A' : 'a';
19
20 // (현재위치 + 13) % 26으로 회전
21 text[i] = (text[i] - base + 13) % 26 + base;
22 }
23 // 알파벳이 아닌 문자는 그대로 유지
24 }
25}
26
27int main() {
28 char msg[] = "Hello, World!";
29 printf("원문: %s\n", msg);
30
31 rot13(msg);
32 printf("암호: %s\n", msg);
33
34 rot13(msg); // 다시 적용하면 원문 복원!
35 printf("복원: %s\n", msg);
36
37 return 0;
38}

왜 이런 일이 생기나:
1. 코드를 수정했는데 주석은 안 고침
2. 복붙하면서 주석도 같이 복붙
3. 급하게 수정하면서 주석 무시

교훈:
- 코드 수정 시 주석도 반드시 업데이트!
- 주석보다 함수명이 더 중요
- 거짓 주석은 주석 없는 것보다 나쁘다
주석의 황금률: "코드는 HOW(어떻게)를, 주석은 WHY(왜)를 설명한다." 이것만 기억하면 돼!

[C 코드] 주석으로 코드 구조 잡기
1#include <stdio.h>

4// ============================================================
5// 문자열 유틸리티 함수들
6// ============================================================
7
8/**
9 * 문자열을 뒤집는다 (in-place).
10 * "hello" → "olleh"
11 */
12void str_reverse(char *s) {
13 int len = strlen(s);
14 for (int i = 0; i < len/2; i++) {
15 char tmp = s[i];
16 s[i] = s[len-1-i];
17 s[len-1-i] = tmp;
18 }
19}
20
21/**
22 * 문자열에서 특정 문자의 개수를 센다.
23 * 예: count_char("hello", 'l') → 2
24 */
25int count_char(const char *s, char c) {
26 int count = 0;
27 for (int i = 0; s[i]; i++) {
28 if (s[i] == c) count++;
29 }
30 return count;
31}
32
33/**
34 * 문자열이 회문(palindrome)인지 확인한다.
35 * "racecar" → 1, "hello" → 0
36 */
37int is_palindrome(const char *s) {
38 int len = strlen(s);
39 for (int i = 0; i < len/2; i++) {
40 if (s[i] != s[len-1-i]) return 0;
41 }
42 return 1;
43}
44
45int main() {
46 char s[] = "hello";
47 printf("원본: %s\n", s);
48 str_reverse(s);
49 printf("뒤집기: %s\n", s);
50 printf("l 개수: %d\n", count_char("hello", 'l'));
51 printf("racecar 회문? %s\n", is_palindrome("racecar") ? "예" : "아니오");
52 return 0;
53}

- 코드 번역 주석: ROI 음수! (유지비만 들고 가치 없음)
