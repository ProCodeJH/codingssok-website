여러 줄에 걸쳐 출력하기

printf를 여러 번 써서 다양한 모양의 출력을 만들어요.

📚 이번에 배울 것

  • printf를 여러 번 써서 여러 줄을 출력할 수 있어요.
  • \n의 위치에 따라 출력이 어떻게 달라지는지 알 수 있어요.
  • 한 줄에 여러 printf의 결과를 이어서 출력할 수 있어요.

✅ 시작 전 체크

  • printf로 Hello, World!를 출력할 수 있다
  • 세미콜론과 중괄호를 쓸 줄 안다
Part 1 진행률: 16%

한 줄만 출력하면 심심하잖아요!

지금까지 Hello, World! 한 줄만 출력했잖아요. 이제 여러 줄을 출력하는 방법을 배워볼게요!

여러 줄을 출력하는 방법은 의외로 간단해요. printf를 여러 번 쓰면 돼요!

printf를 3번 썼고, 각각 뒤에 \n을 넣었어요. 그래서 한 줄씩 출력되는 거예요.

\n을 안 쓰면 어떻게 될까?

줄바꿈 문자 \n을 빼면 어떻게 되는지 실험해 볼게요.

3개의 printf가 한 줄에 이어서 출력됐어요! \n이 없으면 줄바꿈이 안 되니까 다 붙어서 나와요.

중요한 발견! printf는 자동으로 줄바꿈을 안 해요. \n을 써야만 줄이 바뀌어요!

\n의 위치가 중요해요

\n을 어디에 넣느냐에 따라 결과가 완전히 달라져요.

✕ 잘못된 코드
printf("가나다"); printf("라마바");
가나다라마바 (한 줄)
✓ 올바른 코드
printf("가나다\n"); printf("라마바\n");
가나다 (줄바꿈) 라마바

\n이 문자열 끝에 있으면 그 다음 출력이 새 줄에서 시작해요. \n이 문자열 가운데에 있으면 그 지점에서 줄이 바뀌어요.

한 printf 안에서 여러 줄 출력

꼭 printf를 여러 번 쓰지 않아도 돼요. 한 printf 안에 \n을 여러 개 넣으면 여러 줄이 출력돼요.

1#include <stdio.h>
2
3int main() {
4 printf("이름: 코딩쏙\n나이: 1살\n직업: 코딩 선생님\n");
5 return 0;
6}
실행 결과
이름: 코딩쏙 나이: 1살 직업: 코딩 선생님

하나의 printf로 3줄을 출력했어요! \n이 줄바꿈 역할을 해주니까요.

어떤 방법이 더 좋을까?
printf를 여러 번 쓰는 것과 하나에 \n을 여러 개 넣는 것은 결과가 같아요. 하지만 코드가 읽기 쉬운 쪽을 골라요. 보통 짧은 출력은 하나의 printf에, 긴 출력은 여러 printf로 나눠요.

빈 줄 넣기

줄과 줄 사이에 빈 줄을 넣고 싶을 때는 \n을 두 번 쓰면 돼요.

[심화] C에서 여러 줄 출력하는 5가지 방법
1. printf 여러 번

2. printf에 \n 여러 개
printf("줄 1\n줄 2\n줄 3\n");

3. 문자열 연결 (인접 리터럴)

puts("줄 1");

→ \n 안 써도 자동!

5. 매크로로 반복
#define BANNER(s) printf("=== %s ===\n", s)
BANNER("시작");

[심화 내용] C 이스케이프 시퀀스 전체
기본 이스케이프:
\n: 줄바꿈 (LF, 0x0A)
\r: 캐리지 리턴 (CR, 0x0D)
\t: 탭 (0x09)
\\: 역슬래시
\": 큰따옴표
\': 작은따옴표
\0: 널 문자 (0x00)
\a: 벨 (삑 소리!)
\b: 백스페이스
\f: 폼피드 (페이지 넘김)
\v: 수직 탭

8진수/16진수 이스케이프:
\101: A (8진수 101 = 65)
\x41: A (16진수 41 = 65)

유니코드 (C11+):
\u0041: A (4자리 유니코드)
\U0001F600: 😀 (8자리 유니코드)

줄바꿈 차이:
Unix/Mac: \n (LF)
Windows: \r\n (CR+LF)
구 Mac: \r (CR)

[심화 내용] 버퍼링과 출력 제어

[심화 내용] C 출력 버퍼링 모드
3가지 버퍼링 모드:

1. 풀 버퍼링 (Full)
파일 출력의 기본. 버퍼가 차면 flush
→ 4KB~8KB 단위로 디스크에 쓰기

2. 라인 버퍼링 (Line)
stdout의 기본 (터미널 연결 시)
→ \n이 나올 때 flush
→ 진행률 바에서 fflush 필요한 이유!

3. 버퍼링 없음 (Unbuffered)
stderr의 기본. 즉시 출력!
→ 에러 메시지가 항상 보이는 이유

버퍼링 변경:
setvbuf(stdout, NULL, _IONBF, 0); // 버퍼링 끄기
setbuf(stdout, NULL); // 위와 동일

fflush(stdout); // 수동으로 버퍼 비우기
라인 버퍼링
stdout의 기본 모드. \n에서 flush. \n 없이 즉시 출력하려면 fflush(stdout) 필요!
printf가 즉시 안 보이는 건 버퍼링 때문이야! fflush(stdout)하거나 \n을 넣으면 해결!

[심화] sprintf vs snprintf
sprintf (위험!):
char buf[10];
sprintf(buf, "Hello %s!", name);
→ name이 길면 버퍼 오버플로우!

snprintf (안전!):
char buf[10];
snprintf(buf, sizeof(buf), "Hello %s!", name);
→ 최대 9바이트만 쓰고 \0 추가
→ 반환값 = 잘리지 않았다면의 길이

반환값 활용:
int needed = snprintf(NULL, 0, "x=%d", 42);
→ 필요한 길이만 계산 (출력 없음)!
char *buf = malloc(needed + 1);
snprintf(buf, needed + 1, "x=%d", 42);

snprintf는 현대 C의 필수 함수!
snprintf
안전한 sprintf. 버퍼 크기를 지정하여 오버플로우 방지. 현대 C에서 sprintf 대신 필수 사용!
wchar_t
와이드 문자 타입. 유니코드 문자를 정확히 1개로 취급. wcslen()은 문자 수 반환!
printf 마스터의 비결: %d, %f, %s, %c — 이 4개만 완벽히 알면 80%는 해결! 나머지는 필요할 때 찾아보면 돼!
setvbuf
출력 버퍼링 모드 변경 함수. _IONBF(없음), _IOLBF(라인), _IOFBF(풀) 중 선택.
여러 줄 출력의 핵심: printf 포맷 + 이스케이프 시퀀스 + ANSI 코드. 이 세 가지를 조합하면 터미널 UI도 만들 수 있어!
sprintf
문자열에 포맷 출력. 버퍼 오버플로우 위험! snprintf(buf, sizeof(buf), ...)로 대체 필수!
ANSI
American National Standards Institute. 터미널 이스케이프 시퀀스 표준. \033[ 으로 시작.
인접 리터럴
C에서 "A" "B"는 "AB"로 자동 연결. 긴 문자열을 여러 줄로 깔끔하게 작성.
출력은 프로그램의 얼굴이야! 깔끔한 표, 컬러 텍스트, 진행률 바로 사용자 경험을 높이자!
