📚 이번에 배울 것

  • \t(탭)을 사용해서 깔끔한 표를 만들 수 있어요.
  • \\, \"처럼 역슬래시 조합의 의미를 알 수 있어요.
  • 이스케이프 시퀀스라는 개념을 이해할 수 있어요.

✅ 시작 전 체크

  • printf로 여러 줄을 출력할 수 있다
  • \n이 줄바꿈이라는 것을 안다
Part 1 진행률: 18%

\n 말고 또 뭐가 있을까?

지난 유닛에서 \n으로 줄바꿈을 했죠? 사실 이런 특수 문자가 여러 개 있어요. 이걸 이스케이프 시퀀스라고 해요!

이스케이프 시퀀스(escape sequence)는 역슬래시 \로 시작하는 특별한 문자 조합이에요. 화면에 직접 보이지 않지만, 출력 방식을 바꿔주는 역할을 해요.

\t — 탭 문자

\t는 탭(Tab)이에요. 키보드의 Tab 키를 누른 것과 같아요. 일정한 간격을 띄워서 글자를 정렬할 때 쓰여요.

1#include <stdio.h>
2
3int main() {
4 printf("이름\t나이\t취미\n");
5 printf("철수\t12\t축구\n");
6 printf("영희\t11\t피아노\n");
7 printf("민수\t13\t코딩\n");
8 return 0;
9}
실행 결과
이름 나이 취미 철수 12 축구 영희 11 피아노 민수 13 코딩

\t 덕분에 표처럼 깔끔하게 정렬됐어요! 스페이스를 여러 번 눌러서 맞추는 것보다 훨씬 편하죠?

탭은 보통 4칸 또는 8칸 간격이에요. 터미널마다 조금씩 다를 수 있어요.

\\ — 역슬래시 출력

역슬래시 \는 이스케이프 시퀀스의 시작이잖아요? 그러면 역슬래시 자체를 출력하고 싶으면 어떻게 해야 할까요?

✕ 잘못된 코드
printf("\");
에러! \가 이스케이프 시작으로 인식됨
✓ 올바른 코드
printf("\\");
화면에 \ 출력!

역슬래시를 두 번 써야 해요. \\이라고 쓰면 화면에 \ 하나가 나와요. 파일 경로를 출력할 때 자주 쓰여요.

1#include <stdio.h>
2
3int main() {
4 printf("파일 경로: C:\\Users\\student\\hello.c\n");
5 return 0;
6}
실행 결과
파일 경로: C:\Users\student\hello.c

\" — 큰따옴표 출력

문자열은 큰따옴표 "로 감싸죠? 그러면 큰따옴표 자체를 출력하고 싶으면?

1#include <stdio.h>
2
3int main() {
4 printf("그가 말했다. \"안녕!\"\n");
5 return 0;
6}
실행 결과
그가 말했다. "안녕!"

\"라고 쓰면 문자열 안에서 큰따옴표를 출력할 수 있어요.

이스케이프 시퀀스 모음

1
\n
줄바꿈 (newline) — 다음 줄로 이동
2
\t
탭 (tab) — 일정 간격 띄우기
3
\\
역슬래시 — \ 한 개 출력
4
\"

5
\0
널 문자 — 문자열의 끝 표시 (나중에 배워요)

%%로 퍼센트 출력하기

나중에 배울 %d, %f 같은 서식 지정자 때문에 % 기호도 특별해요. 퍼센트 기호를 출력하려면 %%라고 두 번 써야 해요.

1#include <stdio.h>
2
3int main() {
4 printf("할인율: 50%%\n");
5 printf("배터리: 100%%\n");
6 return 0;
7}
실행 결과
할인율: 50% 배터리: 100%

[심화 내용] 문자 인코딩의 역사

[심화 내용] ASCII에서 유니코드까지
ASCII (1963):
7비트, 128문자 (영문+숫자+기호)
0~31: 제어 문자 (\n, \t, \0 등)
32~126: 출력 가능 문자

확장 ASCII (1980s):
8비트, 256문자
문제: 같은 코드가 나라마다 다른 문자!

EUC-KR / CP949 (한국):
2바이트 한글. 11172자 지원

유니코드 (1991~):
전 세계 문자를 하나의 체계로!
143,859+ 문자 (2023 기준)

UTF-8 인코딩:
영문 1바이트, 한글 3바이트
UTF-8이 웹의 98%+ 차지!
ASCII와 하위 호환!

[심화 내용] 널 문자 vs 널 포인터
널 문자 (\0):
값: 0 (정수)
타입: char
용도: 문자열의 끝 표시
char s[] = "Hi"; → s[2] == \0

널 포인터 (NULL):
값: 구현에 따라 다름 (보통 0)
타입: void* (포인터)
용도: "아무것도 가리키지 않음"
int *p = NULL;

혼동 주의!
\0은 문자, NULL은 포인터
if (c == \0) → 문자열 끝 체크
if (p == NULL) → 포인터 유효성 체크
ASCII
American Standard Code for Information Interchange. 1963년 표준화. 7비트 128문자.
UTF-8
유니코드 인코딩. 영어 1바이트, 한글 3바이트. 웹의 98%+. ASCII와 하위 호환.
\0 vs NULL
\0은 널 문자(char, 문자열 끝), NULL은 널 포인터(void*, 아무것도 가리키지 않음). 다른 것!
이스케이프 문자는 보이지 않는 명령어야! \n은 줄바꿈, \t는 탭, \0은 끝 표시. 보이진 않지만 프로그램을 제어해!
UTF-8에서 한글은 3바이트! strlen("한글")=6이지만 실제 문자 수는 2. 이 차이를 알아야 진짜 프로!

[심화] 텍스트 모드 vs 바이너리 모드
텍스트 모드 (기본):
fopen("file.txt", "r");
Windows: \r\n → \n (읽을 때 자동 변환)
Windows: \n → \r\n (쓸 때 자동 변환)
Unix: 변환 없음

바이너리 모드:
fopen("file.bin", "rb");
변환 없음! 바이트 그대로 읽고 쓰기
이미지, 실행 파일, 구조체 저장에 사용

주의:
Windows에서 바이너리를 텍스트 모드로 읽으면
\r\n → \n 변환 때문에 데이터 깨짐!
항상 바이너리 파일은 "rb"/"wb" 사용!

이식성:
텍스트 파일: "r"/"w" (OS가 알아서)
바이너리 파일: "rb"/"wb" (항상 명시)

4// 문자열의 이스케이프 문자를 시각적으로 표시
5void show_escapes(const char *s) {
6 printf("원본 길이: %zu\n", strlen(s));
7 printf("내용: [");
8 for (int i = 0; s[i]; i++) {
9 switch (s[i]) {
10 case '\n': printf("\\n"); break;
11 case '\t': printf("\\t"); break;
12 case '\r': printf("\\r"); break;
13 case '\0': printf("\\0"); break;
14 case '\\': printf("\\\\"); break;
15 case '\"': printf("\\\""); break;
16 default:
17 if (s[i] >= 32 && s[i] < 127)
18 putchar(s[i]);
19 else
20 printf("\\x%02X", (unsigned char)s[i]);
21 }
22 }
23 printf("]\n\n");
24}

[심화 내용] 제어 문자의 역사
제어 문자 = ASCII 0~31, 127

텔레타이프(TTY) 시대의 유산!
기계적 프린터를 제어하던 코드들:

BEL (07): 벨을 울림! (실제 종소리)
BS (08): 인쇄 헤드를 한 칸 뒤로
HT (09): 다음 탭 위치로 이동
LF (0A): 줄을 한 칸 내림
CR (0D): 인쇄 헤드를 줄 처음으로
ESC (1B): 이스케이프 시퀀스 시작

Unix: LF만으로 줄바꿈
Windows: CR+LF로 줄바꿈
→ 프린터 시대: CR로 처음 + LF로 다음 줄
→ Unix: LF 하나로 충분 (소프트웨어니까)

이 차이가 지금도 \r\n vs \n 문제의 원인!

5// CSV 필드 이스케이프 (쉼표/따옴표 처리)
6void csv_escape(const char *field, char *out, int out_sz) {
7 int need_quote = 0;
8 for (const char *p = field; *p; p++) {
9 if (*p == ',' || *p == '\"' || *p == '\n') {
10 need_quote = 1;
11 break;
12 }
13 }
14
15 int pos = 0;
16 if (need_quote && pos < out_sz-1) out[pos++] = '\"';
17 for (const char *p = field; *p && pos < out_sz-2; p++) {
18 if (*p == '\"' && pos < out_sz-3) {
19 out[pos++] = '\"'; // 따옴표 2번!
20 }
21 out[pos++] = *p;
22 }
23 if (need_quote && pos < out_sz-1) out[pos++] = '\"';
24 out[pos] = 0;
25}

[심화 내용] trigraph와 digraph
Trigraph (C89, C++17에서 제거):
??= → # (sharp)
??( → [ (bracket)
??) → ] (bracket)
??< → { (brace)
??> → } (brace)
??/ → \\ (backslash)

옛날 키보드에 #, [, ] 등이 없어서 만든 대체!
현대에서는 사용 금지. -trigraphs로 활성화.

Digraph (C95+):
<% → { %> → }
<: → [ :> → ]
%: → #

trigraph보다 덜 혼란스럽지만 역시 안 씀.

역사적 호기심! 시험에 가끔 나올 수 있음.

[심화] 이스케이프 면접 문제
Q1. \n과 \r의 차이?
A: \n=새줄(LF), \r=줄처음으로(CR). Unix는 \n, Windows는 \r\n

Q2. strlen("Hello\0World")?
A: 5. \0에서 문자열 끝!

Q3. sizeof("Hello")?
A: 6. \0 포함!

Q4. \\의 의미?
A: 역슬래시 1개 출력

Q5. 텍스트 모드에서 Windows가 하는 변환?
A: 읽기: \r\n→\n, 쓰기: \n→\r\n
CR+LF
Windows의 줄바꿈. 0x0D 0x0A. 타자기 시대: 캐리지 리턴(줄 처음)+라인 피드(다음 줄).
sizeof("Hi")=3, strlen("Hi")=2. 이 차이를 기억해! sizeof는 \0 포함, strlen은 \0 앞까지!
