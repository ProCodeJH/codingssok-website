[심화 내용] 좋은 함수 설계 원칙
단일 책임 원칙 (SRP):
한 함수는 한 가지 일만!
line() → 선 그리기만
center() → 중앙 정렬만

함수 크기:
화면 한 페이지에 들어오게 (20~30줄)
넘으면 분리!

명명 규칙:
동사 + 명사: print_line, get_grade
일관성: draw_rect, draw_circle (draw_ 접두어)

매개변수:
3개 이하 권장. 많으면 구조체로!

반환값:
성공/실패: 0=성공, -1=실패 (C 관례)
SRP
Single Responsibility Principle. 한 함수/모듈은 하나의 책임만. SOLID 원칙 중 첫 번째.
명함 프로젝트는 Part 1의 총정리야! printf, 이스케이프, 패턴, 함수 분리 — 배운 걸 전부 활용해봐!

[심화 내용] 리팩토링 원칙
리팩토링 = 동작은 같고 구조 개선

1. 매직 넘버 제거:
40 → #define CARD_WIDTH 40

2. 중복 제거 (DRY):
같은 코드가 2번 이상 → 함수로!
line(), center() 함수 분리

3. 의미 있는 이름:
w → card_width
pad → left_padding

4. 구조체 활용:
매개변수 5개 → Card 구조체 1개

5. 상수 분리:
색상 코드 → #define 또는 enum

Before: 함수 1개에 모든 로직
After: 역할별 함수 분리 + 구조체
DRY
Don't Repeat Yourself. 중복 코드를 제거하고 함수/상수로 추출. 유지보수의 핵심 원칙.
명함 v1 → v2로 발전시키는 과정이 "리팩토링"이야! 기능 추가 전에 구조부터 정리하는 습관을 들이자!

[심화] 프로젝트 흔한 실수
1. 매직 넘버:
printf("%40s") → #define WIDTH 40

2. 하드코딩:
"Kim" 직접 입력 → 변수/파일로

3. 함수 미분리:
main()에 100줄 → 역할별 함수

4. 에러 무시:
snprintf 반환값 체크 안 함

5. 메모리:
고정 배열 크기 → sizeof 활용

이 실수들을 피하면 코드 품질 2배!
매직 넘버
코드에 직접 쓴 의미 불명 상수 (40, 100 등). #define이나 const로 이름을 부여해야 함.
프로젝트는 완벽할 필요 없어! 먼저 동작하게 만들고, 리팩토링하고, 기능 추가 — 이 사이클이 핵심!
[C 코드] 명함 출력 함수 테스트
1#include <stdio.h>
2#include <assert.h>
3#include <string.h>
4
5// 간단한 단위 테스트 예시
6void test_string_length() {
7 // 명함 필드 길이 체크
8 const char *name = "Kim Jahyeon";
9 assert(strlen(name) <= 50); // 50자 이내
10
11 const char *phone = "010-1234-5678";
12 assert(strlen(phone) <= 20); // 20자 이내
13
14 printf("All tests passed!\n");
15}
16
17// snprintf 안전성 테스트
18void test_snprintf_safety() {
19 char buf[10];
20 int ret = snprintf(buf, sizeof(buf), "Hello World!");
21
22 // 반환값 = 잘리지 않았다면의 길이
23 assert(ret == 12); // "Hello World!" = 12자
24 assert(strlen(buf) == 9); // 버퍼 10 - 1(\0) = 9
25 assert(buf[9] == '\0'); // 널 종료 보장
26
27 printf("snprintf safety test passed!\n");
28}
29
30int main() {
31 test_string_length();
32 test_snprintf_safety();
33 return 0;
34}

assert
매크로. 조건이 거짓이면 프로그램 중단 + 에러 메시지. 디버깅/테스트에 필수!
명함 프로젝트 완성! 이제 Part 2에서 변수, 자료형, 연산자를 배우면 훨씬 강력한 프로그램을 만들 수 있어!
snprintf 반환값
버퍼 충분했다면 쓰였을 길이. 잘림 감지: if (ret >= sizeof(buf)) → 잘림! 안전한 코딩의 핵심.
리팩토링
기능 변경 없이 코드 구조 개선. 가독성, 유지보수성 향상. "동작하는 코드를 더 좋은 코드로."
Makefile
빌드 자동화 도구. gcc 명령을 자동화. make 한 번으로 컴파일+링크. 프로젝트 필수!
Part 1 완주! Hello World에서 명함 프로젝트까지. 이제 C의 기초 체력이 완성됐어. Part 2에서 진짜 실력을 키우자!
iteration
반복 개선. v1→v2→v3 점진적 발전. 소프트웨어 개발의 핵심 방법론. 완벽보다 완성이 먼저!
CLI
Command Line Interface. 명령줄 인터페이스. 명함 프로그램도 CLI 앱! GUI 없이 텍스트로 상호작용.
MVP
Minimum Viable Product. 최소 기능 제품. 먼저 동작하게 → 나중에 개선. 명함 v1이 MVP!
incremental
점진적 개발. 작은 기능씩 추가하며 테스트. 한 번에 완성하려 하면 실패 확률 급증!
separation of concerns
관심사 분리. 출력/데이터/로직을 별도 함수로. 변경 시 영향 최소화. 설계의 핵심 원칙.
프로젝트
여러 기술을 조합해서 하나의 완성품을 만드는 것

설계
코드를 쓰기 전에 무엇을 만들지 계획하는 것
