변수가 뭘까?

Part 2에 오신 걸 환영해요! 이제부터는 숫자를 저장하고 계산하는 법을 배울 거예요!

Part 1에서는 printf로 글자를 찍었어요. 그런데 프로그래밍에서는 값을 기억해 두는 것이 정말 중요해요. 점수, 나이, 가격 같은 숫자를 저장해야 하거든요.

변수는 값을 넣어두는 상자예요. 상자에 이름을 붙이고, 안에 숫자를 넣어두면 나중에 꺼내 쓸 수 있어요.

변수 = 이름이 붙은 상자
age ← 12
상자 이름 ←← 상자 안의 값

변수 선언과 대입

[코딩 습관] 변수 이름 짓기
변수 이름은 영어 소문자로 시작해요. a, x 같은 한 글자보다 age, score처럼 의미 있는 이름을 쓰는 게 좋아요. 무슨 값인지 바로 알 수 있거든요.
?

아래 코드를 실행하면 화면에 뭐가 나올까요?

variable1.c
1#include <stdio.h>
2
3int main(void) {
4 int age; // 변수 선언: 상자를 만든다
5 age = 12; // 대입: 상자에 값을 넣는다
6 printf("나이: %d살\n", age);
7 return 0;
8}
실행 결과
나이: 12살
int는 정수(integer)의 줄임말이에요. '이 상자에는 정수를 넣겠다'라는 뜻이에요.

한 줄씩 뜯어보기

4줄int age; — 선언이에요. '정수를 넣을 상자를 하나 만들고, 이름은 age로 하겠다'라는 뜻이에요.
5줄age = 12; — 대입이에요. age 상자에 12를 넣어요. =는 '같다'가 아니라 '넣어라'라는 뜻이에요!
6줄printf("%d", age); — %d 자리에 age 상자 안의 값(12)이 들어가서 출력돼요.
[주의] 수학의 = 와 코딩의 = 는 달라요!
수학에서 x = 5는 'x와 5가 같다'라는 뜻이에요.
코딩에서 x = 5는 'x에 5를 넣어라'라는 뜻이에요.

그래서 코딩에서는 x = x + 1도 가능해요! 'x에 들어있는 값에 1을 더해서, 다시 x에 넣어라'라는 뜻이거든요.

선언과 동시에 값 넣기

상자를 만들면서 바로 값을 넣을 수도 있어요. 이걸 초기화라고 해요.

1#include <stdio.h>
2
3int main(void) {

6
7 printf("점수: %d\n", score);
8 printf("보너스: %d\n", bonus);

실행 결과
점수: 100 보너스: 20 합계: 120

score + bonus처럼 변수끼리 더할 수도 있어요. printf 안에서 바로 계산이 가능해요!

값을 바꿔 넣기

상자니까 당연히 안에 있던 값을 꺼내고 새 값을 넣을 수 있어요.

[심화] 변수의 메모리 모델

[심화 내용] 변수와 메모리 구조
변수 = 메모리 주소 + 크기 + 이름

int x = 42;
→ 메모리 4바이트 확보
→ 주소: 0x7FFF1234 (예시)
→ 값: 42 (이진: 0x0000002A)

메모리 레이아웃:
주소 값
0x1234: 2A 00 00 00 (리틀 엔디안)
0x1238: (다음 변수)

리틀 엔디안 vs 빅 엔디안:
int x = 0x12345678;
리틀 엔디안: 78 56 34 12 (LSB 먼저)
빅 엔디안: 12 34 56 78 (MSB 먼저)
x86/ARM: 리틀 엔디안
네트워크: 빅 엔디안 (Network Byte Order)
[C 코드] 변수의 메모리 확인

7

11 printf("a int %zu %p %d\n", sizeof(a), (void*)&a, a);
12 printf("b double %zu %p %f\n", sizeof(b), (void*)&b, b);

[심화 내용] 변수의 수명과 저장 클래스

[심화 내용] C의 4가지 저장 클래스
1. auto (기본):
지역 변수의 기본. 블록 진입 시 생성, 탈출 시 소멸.
int x = 0; // auto int x = 0; 과 동일

2. static:
함수 끝나도 값 유지! 프로그램 전체 수명.
static int count = 0; // 호출 사이에 유지

3. extern:
다른 파일에서 선언된 변수 참조.
extern int global_var; // 정의는 다른 파일

4. register:
레지스터에 저장 요청 (힌트, 강제 아님).
register int i; // &i 불가!
→ 현대 컴파일러가 알아서 최적화하므로 거의 안 씀

요약:
auto: 스택 | 블록 수명
static: 데이터 영역 | 프로그램 수명
extern: 데이터 영역 | 프로그램 수명
register: 레지스터 | 블록 수명

[심화] 초기화의 중요성
초기화 안 한 변수의 값:

지역 변수 (auto): 쓰레기값!
→ 이전에 스택에 있던 데이터
→ 매번 다를 수 있음 (UB!)

전역 변수: 0으로 자동 초기화
static 변수: 0으로 자동 초기화

규칙:
모든 지역 변수는 선언 시 초기화!
int x = 0; // O
int x; // X (쓰레기값!)

GCC 경고:
-Wuninitialized → 미초기화 경고
-Werror → 경고를 에러로
리틀 엔디안
최하위 바이트(LSB)를 낮은 주소에 저장. x86, ARM(기본). 0x1234 → 34 12 순서로 저장.
static 변수
함수 내 static 변수는 호출 사이에 값이 유지됨. 카운터, ID 생성기에 활용. 프로그램 수명.
변수는 "이름표 붙은 메모리 상자"야! int x=42는 4바이트 상자에 42를 넣고 x라는 이름표를 붙이는 거!
초기화 안 한 변수는 "중고 사물함"이야! 이전 사람이 뭘 넣었는지 모르잖아? 항상 초기화하자!

[심화] 변수 선언 위치의 역사
C89:
변수 선언은 블록 시작에서만!
int main() {
int x = 1; // OK
printf("...");
int y = 2; // ERROR! (C89)
}

C99+:
어디서든 선언 가능!
for (int i = 0; ...) // OK!

현대 스타일:
변수는 사용 직전에 선언 + 초기화
→ 스코프 최소화 = 버그 최소화
[심화 내용] 정수 오버플로우의 위험
signed int 오버플로우 = UB!
int x = INT_MAX; x++; // 정의되지 않은 동작

unsigned int 오버플로우 = 정의됨!
unsigned x = UINT_MAX; x++; // 0 (순환!)

실제 사고:
Ariane 5 로켓 (1996):
64비트 float → 16비트 int 변환
→ 오버플로우 → 로켓 폭발! ($370M)

Boeing 787 (2015):
전원 카운터 248일 후 오버플로우
→ 전원 차단! 248일마다 재부팅 필요

오버플로우는 생명을 앗아갈 수 있다!
INT_MAX
int 최대값 = 2,147,483,647 (약 21억). 에 정의. 32비트 signed integer.
Ariane 5 로켓이 정수 오버플로우로 폭발했어! 370억원짜리 버그. 자료형 선택이 얼마나 중요한지 보여주는 사건!

[심화 내용] 메모리 세그먼트

[심화 내용] 프로그램 메모리 구조
높은 주소 ↑
┌─────────────┐
│ Stack │ ← 지역 변수, 함수 호출
│ ↓ │ (아래로 성장)
│ │
│ ↑ │
│ Heap │ ← malloc/free (위로 성장)
├─────────────┤
│ BSS │ ← 초기화 안 된 전역/static (0)
├─────────────┤
│ Data │ ← 초기화된 전역/static
├─────────────┤
│ Text │ ← 기계어 코드 (읽기 전용)
└─────────────┘
낮은 주소 ↓

int g = 10; → Data 세그먼트
int h; → BSS 세그먼트
void f() {
int x = 1; → Stack
int *p = malloc(4); → Heap
}
[C 코드] 각 세그먼트의 변수 확인
1#include <stdio.h>
2#include <stdlib.h>
3
4int global_init = 42; // Data
5int global_uninit; // BSS
6static int static_var = 10; // Data
7
8int main() {
9 int local = 100; // Stack
10 static int static_local = 5; // Data
11 int *heap = malloc(sizeof(int)); // Heap
12 *heap = 200;
13
14 printf("=== 메모리 세그먼트 ===\n");
15 printf("Text (코드) : %p (main 함수)\n", (void*)main);
16 printf("Data (초기화) : %p (global_init=%d)\n", (void*)&global_init, global_init);
17 printf("Data (static) : %p (static_var=%d)\n", (void*)&static_var, static_var);
18 printf("BSS (미초기화): %p (global_uninit=%d)\n", (void*)&global_uninit, global_uninit);
19 printf("Stack (지역) : %p (local=%d)\n", (void*)&local, local);
20 printf("Heap (동적) : %p (*heap=%d)\n", (void*)heap, *heap);
21
22 free(heap);
23 return 0;
24}
BSS
Block Started by Symbol. 초기화 안 된 전역/static 변수. 0으로 자동 초기화. 실행 파일 크기에 포함 안 됨!
Stack vs Heap
Stack: 자동 관리, 빠름, 작음(1~8MB). Heap: 수동 관리(malloc/free), 느림, 큼(GB급).
Stack은 자동 정리되는 메모지, Heap은 직접 관리하는 서랍장이야! Heap 쓰면 반드시 free!
[활동] 변수 마스터 체크리스트
변수의 3요소(이름, 타입, 값)를 안다
int, char, double, float 크기를 안다
signed/unsigned 차이를 설명한다
메모리 레이아웃(Stack/Heap/Data/BSS)을 안다
리틀 엔디안/빅 엔디안을 안다
auto/static/extern/register를 안다
초기화 안 한 지역 변수가 쓰레기값임을 안다
정수 오버플로우의 위험성을 안다
sizeof 연산자를 활용할 수 있다

7개 이상이면 변수 마스터!
sizeof
타입/변수의 바이트 크기 반환. 컴파일 타임 연산자 (함수 아님!). sizeof(int)=4, sizeof(char)=1.
변수 = 메모리의 이름표! int x=42는 4바이트 공간에 42를 넣고 x라고 부르는 거야. 이게 프로그래밍의 시작!
[C 코드] 변수 교환과 임시 변수
1#include <stdio.h>
2
3void swap_temp(int *a, int *b) {
4 int temp = *a;
5 *a = *b;
6 *b = temp;
7}
8
9void swap_xor(int *a, int *b) {
10 if (a != b) { // 같은 주소면 0이 됨!
11 *a ^= *b;
12 *b ^= *a;
13 *a ^= *b;
14 }
15}
16
17void swap_arithmetic(int *a, int *b) {
18 *a = *a + *b; // 오버플로우 가능!
19 *b = *a - *b;
20 *a = *a - *b;
21}
22
23int main() {
24 int x = 10, y = 20;
25 printf("Before: x=%d, y=%d\n", x, y);
26 swap_temp(&x, &y);
27 printf("After: x=%d, y=%d\n\n", x, y);
28
29 int a = 30, b = 40;
30 printf("Before: a=%d, b=%d\n", a, b);
31 swap_xor(&a, &b);
32 printf("After: a=%d, b=%d\n", a, b);
33 return 0;
34}
실행 결과
Before: x=10, y=20 After: x=20, y=10 Before: a=30, b=40 After: a=40, b=30
XOR swap
임시 변수 없이 교환. a^=b; b^=a; a^=b; 단, 같은 변수면 0이 됨! 실무에서는 temp 방식 권장.
변수 교환 3가지: temp(정석), XOR(해커), 덧셈(위험). 면접에서 자주 나와! temp가 가장 안전해!
[심화 내용] volatile과 restrict
volatile:
volatile int sensor_value;
→ 컴파일러 최적화 방지!
→ 매 접근마다 메모리에서 직접 읽기
→ 하드웨어 레지스터, 시그널 핸들러에 사용

restrict (C99):
void f(int *restrict a, int *restrict b)
→ "a와 b는 같은 메모리를 가리키지 않음" 보장
→ 컴파일러가 더 공격적 최적화 가능
→ memcpy vs memmove의 차이!

_Atomic (C11):
_Atomic int counter;
→ 멀티스레드에서 안전한 접근
→ 경쟁 조건(Race Condition) 방지
volatile
컴파일러 최적화 방지 한정자. 매번 메모리에서 읽기 강제. 하드웨어 레지스터, 인터럽트에 필수.
restrict
C99. 포인터 앨리어싱 없음을 보장. 컴파일러 최적화 허용. memcpy의 src/dst가 겹치지 않는 것과 같은 원리.
volatile은 "매번 진짜로 읽어!"라는 표시야. 없으면 컴파일러가 값을 캐싱해서 하드웨어 변경을 놓칠 수 있어!
Ariane 5
1996년 정수 오버플로우로 폭발한 ESA 로켓. 64비트→16비트 변환 실패. 역사상 가장 비싼 소프트웨어 버그.
alignment
메모리 정렬. CPU가 효율적으로 접근하려면 주소가 타입 크기의 배수여야. struct에서 패딩 발생 원인.
변수는 프로그래밍의 벽돌이야! 타입, 크기, 수명, 스코프 — 이 네 가지를 알면 변수 마스터!
padding
구조체 멤버 사이의 빈 공간. CPU 정렬 요구사항 때문. sizeof(struct)가 멤버 합보다 클 수 있음!
UINT_MAX
unsigned int 최대값 = 4,294,967,295 (약 42억). 정의. 32비트.
C의 변수 시스템은 하드웨어와 가장 가까워! 메모리 주소, 바이트 크기, 엔디안 — 다른 언어에서는 숨기는 것들을 C는 보여줘!
_Atomic
C11 원자적 타입. 멀티스레드에서 안전한 접근. _Atomic int counter; → 경쟁 조건 방지.
memmove
메모리 복사 (겹침 허용). memcpy는 겹치면 UB! 안전한 복사는 memmove 사용.
register
레지스터 저장 요청(힌트). 현대 컴파일러가 자동 최적화하므로 거의 사용 안 함. &연산자 불가.
변수(Variable)
값을 저장하는 이름이 붙은 메모리 공간. '상자'에 비유할 수 있어요.
선언(Declaration)
변수를 만드는 것. int age;처럼 타입과 이름을 적어요.
대입(Assignment)
변수에 값을 넣는 것. = 기호를 사용해요.
초기화
변수를 선언하면서 동시에 값을 넣는 것. int age = 12;
%d
printf에서 정수(int) 값을 출력할 때 쓰는 서식 지정자.
int
정수(integer)를 저장하는 자료형. 소수점 없는 숫자만 넣을 수 있어요.
