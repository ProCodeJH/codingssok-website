scanf로 입력받기

사용자가 키보드로 값을 입력하면, 변수에 저장하는 방법을 배워요.

📚 이번에 배울 것

  • scanf 함수로 키보드 입력을 받을 수 있어요.
  • & 기호의 역할을 이해해요.
  • 입력받은 값으로 계산하고 출력할 수 있어요.

✅ 시작 전 체크

  • 변수를 선언하고 printf로 출력할 수 있다
  • %d 서식 지정자를 안다
Part 1 진행률: 13%

사용자한테 물어보기

지금까지는 숫자를 우리가 직접 코드에 적었는데, 이제는 사용자가 직접 입력할 수 있게 만들어 볼 거예요!

printf가 '화면에 보여주는' 함수라면, scanf는 '키보드에서 받아오는' 함수예요. 둘 다 stdio.h에 들어 있어요.

printf는 출력, scanf는 입력
키보드 → scanf → 변수(RAM)
변수(RAM) → printf → 모니터

scanf 기본 사용법

[코딩 습관] 저장 → 컴파일 → 실행
코드를 쓰고 나면 반드시 Ctrl+S로 저장한 뒤 컴파일하세요! 저장 안 하고 컴파일하면 옛날 코드가 실행돼요.

한 줄씩 뜯어보기

4줄int age; — 값을 받아올 상자를 먼저 만들어요. 아직 안이 비어 있어요.
6줄printf("나이를 입력하세요: "); — 사용자에게 뭘 입력할지 알려주는 안내 메시지예요.
7줄scanf("%d", &age); — 키보드에서 정수를 입력받아 age에 넣어요. &를 빼먹으면 안 돼요!
[주의] &를 절대 빼먹지 마세요!
scanf("%d", age); — & 없음 → 실행 에러! 프로그램이 깨져요.
scanf("%d", &age); — & 있음 → 정상 동작

&는 '이 변수의 주소를 알려줘'라는 뜻이에요. scanf가 값을 넣으려면 상자의 위치를 알아야 하거든요. 지금은 'scanf에서는 & 꼭 붙이기'만 기억하세요!
&의 정확한 이유는 나중에 '포인터'를 배울 때 알게 돼요. 지금은 그냥 scanf에서는 변수 앞에 & 붙이기! 이것만 외우세요.

[심화] scanf 완전 가이드

[심화] scanf 포맷과 주의사항
기본 사용:
int n; scanf("%d", &n);
→ & 필수! (변수의 주소 전달)

포맷 지시자:
%d: int, %ld: long
%f: float, %lf: double
%c: 한 문자 (공백 포함!)
%s: 문자열 (공백까지)
%[^\n]: 한 줄 전체
%*d: 읽되 저장 안 함

반환값:
성공적으로 읽은 항목 수!
int ret = scanf("%d %d", &a, &b);
if (ret != 2) { /* 입력 에러! */ }

주의사항:
1. &를 빼먹으면 크래시/UB!
2. %s는 버퍼 오버플로우 위험!
3. %c는 이전 \n을 읽을 수 있음!
4. scanf 후 버퍼에 \n 남음!

[심화 내용] 입력 검증과 보안

[심화 내용] scanf의 보안 문제
1. 버퍼 오버플로우:
char buf[10];
scanf("%s", buf); // 10자 넘으면 크래시!
→ scanf("%9s", buf); // 최대 9자로 제한
→ 또는 fgets() 사용 (강력 추천!)

2. 포맷 문자열 공격:
scanf(user_input); // 절대 금지!

3. 무한 루프:
while (scanf("%d", &n) != 1); // 문자 입력 시 무한!
→ 실패 시 버퍼 비우기 필수

4. 정수 오버플로우:
scanf("%d", &n); // 99999999999 입력 시?
→ strtol()로 범위 검사하며 변환

안전한 입력 순서:
fgets → sscanf 또는 strtol/strtod
[C 코드] strtol로 안전한 정수 변환
1#include <stdio.h>
2#include <stdlib.h>
3#include <errno.h>
4#include <limits.h>
5
6int safe_atoi(const char *str, int *result) {
7 char *endptr;
8 errno = 0;
9 long val = strtol(str, &endptr, 10);
10
11 // 에러 체크
12 if (errno == ERANGE) return -1; // 범위 초과
13 if (endptr == str) return -2; // 숫자 없음
14 if (*endptr != '\0' && *endptr != '\n') return -3; // 뒤에 쓰레기
15 if (val > INT_MAX || val < INT_MIN) return -4; // int 범위 초과
16
17 *result = (int)val;
18 return 0;
19}
20

22 const char *tests[] = {"42", "-7", "abc", "99999999999", "12xyz", ""};

24

26 int value;
27 int err = safe_atoi(tests[i], &value);

29 printf("\"%s\" -> %d\n", tests[i], value);

fgets
안전한 줄 입력. fgets(buf, size, stdin). 버퍼 크기 제한으로 오버플로우 방지. scanf 대체 권장!
strtol
문자열→long 변환. 에러 감지 가능(endptr, errno). atoi()보다 안전! 실무에서 필수.
scanf는 편하지만 위험해! 실무에서는 fgets+sscanf 또는 fgets+strtol 조합을 써. 안전 제일!

[심화 내용] 입력 파싱 고급 기법
sscanf — 문자열에서 파싱:
char line[] = "Kim 20 95.5";
char name[30]; int age; double score;
sscanf(line, "%s %d %lf", name, &age, &score);

strtok — 토큰 분리:
char csv[] = "Kim,20,95.5";
char *tok = strtok(csv, ",");
while (tok) {
printf("[%s]\n", tok);
tok = strtok(NULL, ",");
}

getline (POSIX) — 동적 줄 입력:
char *line = NULL; size_t len = 0;
getline(&line, &len, stdin);
→ 자동으로 메모리 할당!
→ free(line) 필수
sscanf
문자열에서 포맷 파싱. fgets로 읽고 sscanf로 분석하는 것이 가장 안전한 입력 패턴!
입력 처리의 정석: fgets로 한 줄 읽기 → sscanf 또는 strtol로 파싱 → 에러 체크. 이 3단계를 기억해!

11int parse_csv_line(const char *line, Student *s) {

29 return 0;

5int is_valid_email(const char *email) {
6 const char *at = strchr(email, '@');
7 if (!at || at == email) return 0; // @가 없거나 맨 앞
8 const char *dot = strrchr(at, '.');
9 if (!dot || dot == at+1) return 0; // . 없거나 @바로 뒤
10 if (strlen(dot) < 3) return 0; // .xx 최소 2글자
11 return 1;
12}

14int is_valid_phone(const char *phone) {

20 if (phone[i] != '-') return 0;

22 if (!isdigit(phone[i])) return 0;

strchr
문자열에서 문자 검색. 첫 번째 위치의 포인터 반환. 못 찾으면 NULL. strrchr은 마지막 위치.
입력 검증은 "믿지 마, 확인해" 정신이야! 이메일, 전화번호, 숫자 — 형식이 맞는지 항상 체크!
[심화 내용] 입력 처리 설계 패턴
1. 읽기-파싱-검증 3단계:
fgets(line) → sscanf/strtol → 범위 체크

2. 재시도 루프:
while (!valid) { prompt → read → validate }

3. 기본값 패턴:
값 없으면 기본값 사용
if (strlen(input)==0) use_default();

4. 에코백 패턴:
입력받은 값을 다시 보여주고 확인
"Kim, 20세 맞습니까? (Y/N)"

5. 명령어 파싱:
fgets → 첫 단어 추출 → switch/if
"add Kim 20" → cmd="add", name="Kim", age=20
getchar
stdin에서 한 문자 읽기. 버퍼 비우기에 사용: while(getchar()!="\n"). EOF(-1) 체크 필수!
atoi
문자열→int 변환. 에러 감지 불가! atoi("abc")=0, atoi("12xyz")=12. strtol 사용 권장.
사용자 입력 3원칙: 1)크기 제한 2)타입 검증 3)범위 확인. 이 세 가지만 지키면 입력 관련 버그 90% 예방!
isdigit
. 문자가 숫자(0-9)인지 검사. isalpha(문자), isspace(공백), toupper(대문자 변환).
EOF
End Of File. 보통 -1. getchar()이 EOF 반환하면 입력 끝. Ctrl+D(Unix)/Ctrl+Z(Win)로 발생.
atoi는 에러를 무시해! atoi("hello")=0, atoi("12ab")=12. strtol은 endptr로 에러 감지 가능. 프로는 strtol!
scanf %[
scanf의 스캔셋. %[^\n]은 줄바꿈 전까지 읽기. %[a-z]는 소문자만. 정규식 비슷한 입력 패턴.
getline
POSIX 함수. 한 줄을 동적 할당으로 읽기. 길이 제한 없음! 반환 후 free() 필수.
ungetc
읽은 문자를 다시 스트림에 넣기. 미리보기(peek) 구현에 활용. 파서 작성의 핵심 기법.
scanf의 & 빼먹기 = C 입문자 실수 1위! scanf("%d", &n)의 &는 "주소를 전달한다"는 의미야!
stdin
표준 입력 스트림. 키보드 또는 파이프(|). freopen("input.txt","r",stdin)로 파일 리다이렉션 가능.
feof
파일 끝 검사. while(!feof(f))는 안티패턴! 대신 while(fgets(buf,sz,f))를 사용.
freopen
스트림을 다른 파일로 연결. freopen("in.txt","r",stdin) → scanf가 파일에서 읽기. 테스트에 유용!
입력 테스트 팁: freopen("test.txt","r",stdin)하면 scanf가 파일에서 읽어! 매번 타이핑 안 해도 돼!
scanf 공백
scanf 포맷의 공백은 "0개 이상의 공백 문자"를 의미. scanf(" %c", &c)에서 앞의 공백이 \n을 건너뜀!
%*d
scanf에서 *는 "읽되 저장하지 않음". scanf("%*d %d", &b) → 첫 숫자 무시, 두 번째만 저장.
sscanf
문자열에서 포맷 파싱. sscanf("42 hello", "%d %s", &n, str). fgets+sscanf = 안전한 입력 패턴!
버퍼 비우기
scanf 후 버퍼에 남은 \n 제거. while(getchar()!="\n"); 또는 scanf(" %c", &c)에서 공백으로 건너뛰기.
fgets vs gets
gets()는 버퍼 크기 제한 없어 위험! C11에서 제거됨. fgets(buf, size, stdin)만 사용!
clearerr
스트림 에러/EOF 플래그 초기화. feof() 후 다시 읽으려면 clearerr(stdin) 필요.
C의 입력은 "스트림" 기반이야! stdin은 바이트 흐름. scanf는 그 흐름에서 패턴을 읽는 것!
scanf
키보드에서 값을 입력받아 변수에 저장하는 함수. scan + formatted의 줄임말.
& (앰퍼샌드)
변수의 메모리 주소를 뜻하는 기호. scanf에서 변수 앞에 꼭 붙여야 해요.
입력(Input)
사용자가 키보드로 프로그램에 데이터를 전달하는 것.
출력(Output)
프로그램이 모니터에 결과를 보여주는 것.
