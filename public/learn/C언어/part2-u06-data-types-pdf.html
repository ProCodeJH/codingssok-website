소수점이 있는 숫자

지난 유닛에서 7 / 2 = 3이었죠? 오늘은 3.5를 얻는 방법을 배울 거예요!

int는 정수(소수점 없는 수)만 저장해요. 소수점이 있는 수를 저장하려면 float(단정밀도)이나 double(배정밀도)을 써야 해요.

1
int
정수 전용. 예: 1, -5, 100, 0
2
float
실수(소수점 있는 수). 예: 3.14, -0.5, 2.0
3
double
더 정밀한 실수. float보다 2배 정확해요. 보통 이걸 써요.

[핵심] 언제 int, 언제 double?
int: 나이, 점수, 개수, 번호 — 소수점이 필요 없는 것
double: 평균, 키, 몸무게, 온도, 확률 — 소수점이 필요한 것

잘 모르겠으면 double을 쓰세요. 정수도 담을 수 있거든요.

[심화] 자료형 심화 — 비트와 바이트

[심화 내용] 정수의 이진 표현
부호 없는 정수 (unsigned):
8비트: 0~255 (2^8 - 1)
16비트: 0~65535
32비트: 0~4,294,967,295

부호 있는 정수 (signed) — 2의 보수:
8비트: -128~127
양수: 그대로 (0xxxxxxx)
음수: 비트 반전 + 1
예: -1 = 11111111 (0xFF)
예: -128 = 10000000 (0x80)

왜 2의 보수?
1. 0이 하나! (1의 보수는 +0, -0 두 개)
2. 덧셈 회로가 동일! (부호 무시하고 더해도 OK)
3. 뺄셈 = 보수 덧셈

오버플로우:
127 + 1 = -128 (signed, UB!)
255 + 1 = 0 (unsigned, 정의됨)

[심화 내용] IEEE 754 부동소수점

[심화 내용] float/double의 내부 구조
IEEE 754 float (32비트):
부호(1) | 지수(8) | 가수(23)

값 = (-1)^S × 1.M × 2^(E-127)

예: 3.14
3.14 = 1.1001... × 2^1
S=0, E=128(=1+127), M=10010...

특수값:
0 00000000 000...0 = +0
0 11111111 000...0 = +Infinity
1 11111111 000...0 = -Infinity
x 11111111 ≠0 = NaN

정밀도 한계:
float: ~7자리 유효숫자
double: ~15자리 유효숫자

0.1 + 0.2 ≠ 0.3! (이진수로 무한소수)
[C 코드] 부동소수점 비트 확인

15}

2의 보수
음수 표현법. 비트 반전+1. -1=0xFF. 장점: 0이 하나, 덧셈 회로 공유. 현대 CPU의 표준.
IEEE 754
부동소수점 표준. float(32비트), double(64비트). 0.1+0.2≠0.3인 이유. 모든 현대 CPU 지원.
0.1 + 0.2 ≠ 0.3 — 이건 버그가 아니라 이진수의 한계야! 실수 비교는 항상 epsilon 범위로!
float는 7자리, double은 15자리 정밀도. 돈 계산에는 절대 float 쓰지 마! 정수(원 단위)로 계산해!

[심화 내용] 자료형 함정
1. int의 크기는 플랫폼마다 다름!
16비트 시스템: int = 2바이트
32비트 시스템: int = 4바이트
→ 크기 중요하면 int32_t 사용!

2. char의 부호도 다름!
x86: signed가 기본
ARM: unsigned가 기본
→ 명시적으로 signed char / unsigned char

3. long의 크기:
Windows 64비트: long = 4바이트!
Linux 64비트: long = 8바이트!
→ long long (최소 8바이트) 사용

4. 암묵적 변환:
unsigned + signed → unsigned로 변환!
-1 > 0u → TRUE! (UB 급 위험)
stdint.h
C99 고정 폭 정수 헤더. int8_t~int64_t, uint8_t~uint64_t. 크기를 보장하여 이식성 확보.
size_t
부호 없는 크기 타입. sizeof의 반환 타입. 32비트: 4바이트, 64비트: 8바이트. 배열 인덱스에 권장.
int는 "최소 16비트"만 보장해! 크기가 중요하면 int32_t, uint64_t 등 고정 폭 타입을 써!

[활동] 자료형 마스터 체크리스트
정수형 5종(char~long long)의 크기를 안다
signed/unsigned 차이와 범위를 안다
2의 보수로 음수 표현법을 안다
IEEE 754 부동소수점 구조를 안다
0.1+0.2≠0.3인 이유를 설명한다
실수 비교에 epsilon을 사용한다
stdint.h 고정 폭 타입을 활용한다
구조체 패딩과 정렬을 안다

6개 이상이면 자료형 마스터!
NaN
Not a Number. 0.0/0.0, sqrt(-1) 등의 결과. NaN ≠ NaN (자기 자신과도 같지 않음!). isnan()으로 검사.
구조체 필드 순서만 바꿔도 메모리가 줄어! 큰 타입→작은 타입 순으로 배치하면 패딩 최소화!
[심화] 비트 연산으로 자료형 이해
비트 연산 기초:
& (AND): 두 비트 모두 1이면 1
| (OR): 하나라도 1이면 1
^ (XOR): 다르면 1
~ (NOT): 비트 반전
<< (좌시프트): 2배
>> (우시프트): 1/2

활용:
플래그 설정: flags |= FLAG_A;
플래그 해제: flags &= ~FLAG_A;
플래그 체크: if (flags & FLAG_A)
플래그 토글: flags ^= FLAG_A;

비트 마스크:
#define READ (1 << 0) // 001
#define WRITE (1 << 1) // 010
#define EXEC (1 << 2) // 100
int perm = READ | WRITE; // 011
[C 코드] 비트 플래그 시스템
1#include <stdio.h>

[심화 내용] 타입 관련 면접 문제
Q1. sizeof(char)은 항상 1인가?
A: YES! C 표준이 보장. 하지만 1바이트가 8비트인지는 보장 안 됨(CHAR_BIT).

Q2. -1 > 0u 의 결과는?
A: TRUE! -1이 unsigned로 변환되어 UINT_MAX가 됨.

Q3. float f = 0.1; if(f == 0.1) 결과?
A: FALSE일 수 있음! 0.1은 double, f는 float. 정밀도 차이.

Q4. union의 크기는?
A: 가장 큰 멤버의 크기. struct는 합계+패딩.

Q5. void 포인터의 크기는?
A: 플랫폼 포인터 크기. 32비트:4, 64비트:8.
CHAR_BIT
1바이트의 비트 수. 대부분 8이지만 일부 DSP에서는 16이나 32. 에 정의.
union
모든 멤버가 같은 메모리 공유. 크기 = 가장 큰 멤버. 메모리 절약, 타입 변환에 활용.
-1 > 0u가 TRUE인 건 C의 가장 유명한 함정! signed와 unsigned를 섞지 마. -Wsign-compare로 경고!

stdbool.h
C99. bool, true(1), false(0) 정의. C11에서는 _Bool이 내장 타입. C23에서 헤더 없이 사용 가능.
enum + DAY_COUNT 트릭! 마지막에 COUNT를 넣으면 자동으로 총 개수가 돼. 배열 크기에 바로 사용!
_Bool
C99 내장 불리언. 0 또는 1만 저장. 의 bool은 _Bool의 별칭.
void
타입 없음. void 함수: 반환 없음. void*: 범용 포인터. sizeof(void)는 정의되지 않음(GCC는 1).
float 리터럴
3.14f는 float, 3.14는 double, 3.14L은 long double. 접미사로 타입 명시!
C의 자료형은 하드웨어를 직접 반영해! 비트, 바이트, 엔디안, 정렬 — 이해하면 메모리의 주인이 돼!
pragma pack
구조체 패딩 제어. #pragma pack(push,1)로 패딩 제거. 네트워크 패킷, 파일 포맷에 사용.
offsetof
매크로. 구조체 멤버의 오프셋. offsetof(struct S, member). 패딩 확인에 유용.
자료형의 핵심: 크기(sizeof), 범위(limits.h), 정밀도(float.h), 정렬(alignment). 이 네 가지!
_Generic
C11 제네릭 선택. 타입에 따라 다른 동작. _Generic(x, int: "int", double: "double"). C의 함수 오버로딩!
implicit conversion
암묵적 형변환. int+double → double. char+int → int. signed+unsigned → unsigned (위험!).
integer promotion
char, short는 연산 시 int로 승격. char a=100,b=100; a*b는 int로 계산(10000). 오버플로우 방지!
C의 자료형 = 하드웨어의 언어. int는 레지스터 크기, char는 최소 주소 단위. 기계를 이해하면 C가 보여!
long double
확장 정밀도 실수. x86: 80비트(10바이트), 일부 플랫폼: double과 동일. %Lf로 출력.
DBL_EPSILON
1.0과 구분 가능한 최소 차이. double: ~2.2e-16. 실수 비교에서 적절한 epsilon 기준값.
돈 계산에 float/double 쓰지 마! 0.1+0.2≠0.3 문제. 원 단위 int로 계산하거나 고정소수점 사용!
int
정수를 저장하는 자료형. 소수점 없는 수만 담을 수 있어요.
double
실수를 저장하는 자료형. 소수점이 있는 수를 정밀하게 담아요.
float
실수를 저장하지만, double보다 정밀도가 낮아요. 보통 double을 더 많이 써요.
%f
실수(float/double)를 출력하는 서식 지정자. 기본 소수점 6자리.
%.2f
실수를 소수점 아래 2자리까지 출력하라는 서식.
이번 유닛 핵심 정리
1
int = 정수, double = 실수 (소수점 있는 수)
2
실수 출력은 %f, 자릿수 조절은 %.2f (소수점 2자리)
3
실수 나눗셈을 하려면 한쪽이 double이어야 해요 (7.0/2 = 3.5)
