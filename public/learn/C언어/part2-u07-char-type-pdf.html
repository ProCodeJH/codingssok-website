문자를 저장하는 상자

int는 정수, double은 실수 — 그럼 문자는? char가 있어요!

char는 문자(character) 하나를 저장하는 타입이에요. 문자는 작은따옴표('')로 감싸요.

char1.c
1#include <stdio.h>
2
3int main(void) {
4 char grade = 'A';
5 char initial = 'K';
6
7 printf("학점: %c\n", grade);
8 printf("이니셜: %c\n", initial);
9 return 0;
10}
실행 결과
학점: A 이니셜: K
4줄char grade = 'A'; — 문자 하나를 저장해요. 작은따옴표를 써요! (큰따옴표는 문자열)
7줄%c — 문자(char)를 출력하는 서식 지정자예요.

비밀: 문자도 숫자예요

컴퓨터 안에서 문자는 전부 숫자로 저장돼요. 'A'는 65, 'B'는 66, 'a'는 97로 저장돼요. 이 약속을 ASCII 코드라고 해요.

ascii.c
1#include <stdio.h>
2
3int main(void) {
4 char ch = 'A';
5
6 printf("문자: %c\n", ch); // A
7 printf("숫자: %d\n", ch); // 65
8 return 0;
9}
실행 결과
문자: A 숫자: 65
같은 변수를 %c로 출력하면 문자가, %d로 출력하면 숫자가 나와요! 안에는 같은 값(65)이 들어 있어요.

ASCII 코드 규칙

자주 쓰는 ASCII 코드
문자ASCII 값설명

'A'~'Z'65~90대문자. 'A'=65, 'Z'=90

[핵심] 대문자 → 소문자 = +32
'A'(65)와 'a'(97)의 차이는 32예요!
대문자에 32를 더하면 → 소문자
소문자에서 32를 빼면 → 대문자

'A' + 32 = 'a'
'a' - 32 = 'A'

12void caesar_encrypt(char *text, int shift) {
13 for (int i = 0; text[i]; i++) {
14 if (text[i] >= 'A' && text[i] <= 'Z')
15 text[i] = (text[i] - 'A' + shift) % 26 + 'A';
16 else if (text[i] >= 'a' && text[i] <= 'z')
17 text[i] = (text[i] - 'a' + shift) % 26 + 'a';

20 op = *expr++;

25 b = b * 10 + (*expr - '0');
26 expr++;
27 }

35 }

[심화] 아스키 기반 트릭
대소문자 비트 트릭:
'A' = 0100 0001
'a' = 0110 0001
차이 = 비트 5 (0x20)

대→소: c | 0x20
소→대: c & ~0x20 (= c & 0xDF)
토글: c ^ 0x20

숫자 체크 비트 트릭:
isdigit: (unsigned)(c - '0') <= 9
→ 부호 없는 비교로 한 번에!

문자열→정수 (직접 구현):
int n = 0;
for(; isdigit(*s); s++)
n = n * 10 + (*s - '0');
비트5 트릭
ASCII에서 대소문자는 비트 5(0x20)만 다름. A(0x41)↔a(0x61). XOR 0x20으로 토글!
ROT13
알파벳 13칸 이동 암호. A→N, N→A. 두 번 적용하면 원래로! 시저 암호의 특수 경우.
대소문자 변환에 ±32 대신 비트 연산(^0x20)을 쓰면 분기 없이 빠르게 처리! 임베디드에서 유용해!

문자 빈도 분석으로 시저 암호를 깰 수 있어! 영어에서 e가 가장 많으니까, 가장 많은 글자가 e인 시프트를 찾으면 돼!
wchar_t
와이드 문자. Windows: 2바이트(UTF-16), Linux: 4바이트(UTF-32). L'가'로 한글 1글자 표현.
memcmp
메모리 비교. strcmp는 \0까지, memcmp는 정확히 n바이트. 바이너리 데이터 비교에 사용.
strstr
문자열 내 부분 문자열 검색. strstr("Hello World", "World") → "World" 포인터. 못 찾으면 NULL.
char 마스터의 핵심: char는 정수다! 'A'+1='B', '5'-'0'=5, 'a'|0x20='a'. 이 세 가지를 기억해!
toupper 로케일
toupper/tolower는 로케일에 의존. 터키어에서 i→I가 아닌 다른 문자! 주의 필요.

n바이트까지만 비교. strncmp("Hello","Help",3)=0. 접두어 체크에 유용.
문자열 함수 선택: 비교=strcmp/strncmp, 복사=strncpy/strlcpy, 검색=strchr/strstr, 길이=strlen!
sprintf 위험
버퍼 크기 미지정. sprintf(buf,"%s",s)는 오버플로우 가능! snprintf(buf,sizeof(buf),"%s",s)로 대체!
문자열 리터럴
"Hello"는 읽기 전용 메모리에 저장. char *s="Hi"; s[0]='X'는 UB! char s[]="Hi"로 복사해야 수정 가능.
memmove
겹치는 메모리 안전 복사. memcpy는 겹치면 UB. memmove(dst,src,n)은 항상 안전!
char 배열 vs char 포인터: 배열은 값을 가지고, 포인터는 주소만 가져. 수정 가능 여부가 달라!

C11. 16비트 문자 타입. u"문자열" 리터럴. UTF-16 인코딩에 사용. 에 정의.
mbstowcs
멀티바이트→와이드 문자열 변환. UTF-8 char[] → wchar_t[]. 로케일 설정 필수!

strcspn
문자열에서 특정 문자 집합이 나오기 전까지의 길이. fgets 후 줄바꿈 제거: buf[strcspn(buf,"\n")]=0;
memset
메모리를 바이트 값으로 채우기. memset(buf, 0, sizeof(buf)). 배열 초기화에 활용. 주의: int 배열은 0만 안전!

strtok_r
