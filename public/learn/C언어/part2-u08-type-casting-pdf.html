자료형 변환

자동 변환과 강제 변환(캐스팅)을 배우고, 타입이 섞일 때의 규칙을 이해해요.

📚 이번에 배울 것

  • 자동 형변환이 언제 일어나는지 알아요.
  • (int), (double) 같은 강제 캐스팅을 할 수 있어요.
  • 자료형이 섞일 때 결과 타입을 예측할 수 있어요.

✅ 시작 전 체크

  • int, double, char의 차이를 안다
  • 정수 나눗셈과 실수 나눗셈의 차이를 안다

타입이 섞이면?

int와 double을 같이 계산하면 어떻게 될까요? 오늘은 타입이 자동으로 바뀌는 규칙을 배워요!

C언어에서는 서로 다른 타입이 만나면, 작은 타입이 큰 타입으로 자동 변환돼요. 이걸 자동 형변환(암묵적 변환)이라고 해요.

자료형 크기 순서
char → int → float → double
작은 타입 ─────────────────→ 큰 타입

자동 형변환 예시

auto_cast.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 7;
5 double b = 2.0;
6 double result = a / b; // int가 double로 자동 변환!
7
8 printf("7 / 2.0 = %f\n", result);
9
10 // char → int 자동 변환
11 char ch = 'A';
12 int num = ch; // char가 int로 자동 변환
13 printf("'A' = %d\n", num);
14 return 0;
15}
실행 결과
7 / 2.0 = 3.500000 'A' = 65
6줄a / b — int(7)와 double(2.0)이 만나면, int가 double(7.0)로 바뀌어서 계산돼요. 결과: 3.5
12줄int num = ch; — char('A')가 int(65)로 자동 변환돼요.

강제 형변환 (캐스팅)

직접 타입을 지정해서 바꾸는 것을 강제 형변환(캐스팅)이라고 해요. 타입 이름을 괄호() 안에 적어요.

cast.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 7, b = 2;
5
6 // 강제 캐스팅 없이
7 printf("정수 나눗셈: %d\n", a / b); // 3
8
9 // 강제 캐스팅
10 printf("실수 나눗셈: %f\n", (double)a / b); // 3.5
11
12 // 실수 → 정수 캐스팅
13 double pi = 3.14;
14 int pi_int = (int)pi; // 소수점 버림!
15 printf("pi 정수화: %d\n", pi_int); // 3
16 return 0;
17}
실행 결과
정수 나눗셈: 3 실수 나눗셈: 3.500000 pi 정수화: 3
(double)a는 'a를 잠깐 double로 취급해!'라는 뜻이에요. a 자체가 바뀌는 건 아니에요.
[핵심] 캐스팅은 임시 변환!
(double)a라고 해도 변수 a의 타입은 그대로 int예요. 계산할 때만 잠깐 double로 취급하는 거예요.

주의: 작은 타입에 큰 값 넣기

[주의] 데이터 손실!

용어 정리

[심화] 형변환 심화

[심화] C의 형변환 종류
1. 암묵적 변환 (Implicit):
int + double → double (자동 승격)
char + int → int (정수 승격)
signed + unsigned → unsigned (위험!)

2. 명시적 캐스팅 (Explicit):
(int)3.14 → 3 (소수점 버림)
(double)5/2 → 2.5
(char)65 → 'A'

3. 정수 승격 규칙 순서:
long double > double > float >
unsigned long long > long long >
unsigned long > long >
unsigned int > int

4. 포인터 캐스팅:
(void*) → 범용 포인터
(int*)(char*) → 타입 재해석
→ strict aliasing 규칙 주의!
[C 코드] 형변환 함정들
1#include <stdio.h>
2#include <limits.h>
3
4int main() {
5 // 1. 정수 나눗셈 vs 실수 나눗셈
6 printf("5/2 = %d\n", 5/2); // 2 (정수!)
7 printf("5.0/2 = %f\n", 5.0/2); // 2.5
8 printf("(double)5/2 = %f\n", (double)5/2); // 2.5
9
10 // 2. signed + unsigned 함정
11 int a = -1;
12 unsigned int b = 1;
13 printf("-1 > 1u? %s\n", (a > b) ? "YES" : "NO"); // YES!
14 // -1이 UINT_MAX(4294967295)로 변환!
15
16 // 3. 정밀도 손실
17 int big = 123456789;
18 float f = big; // float은 7자리 정밀도!
19 printf("int: %d, float: %.0f\n", big, f);
20 // 123456789 → 123456792 (변환!)
21
22 // 4. 부호 확장
23 char c = -1; // 0xFF
24 unsigned char uc = c; // 0xFF = 255
25 int i1 = c; // 부호 확장: 0xFFFFFFFF = -1
26 int i2 = uc; // 제로 확장: 0x000000FF = 255
27 printf("char -1 → int: %d, uchar → int: %d\n", i1, i2);
28
29 return 0;
30}

[심화 내용] strict aliasing과 type punning

[심화 내용] Strict Aliasing Rule
규칙: 다른 타입의 포인터로 같은 메모리에 접근하면 UB!

int x = 42;
float *fp = (float*)&x; // UB!
printf("%f", *fp); // 정의되지 않은 동작

예외:
1. char*/unsigned char* → 모든 타입 접근 OK
2. signed/unsigned 변환 OK
3. union을 통한 접근 (C에서는 OK, C++은 UB)

안전한 type punning:
float int_to_float(int i) {
float f;
memcpy(&f, &i, sizeof(f)); // 안전!
return f;
}

왜 중요한가:
컴파일러가 aliasing 없다고 가정하고 최적화!
-fno-strict-aliasing으로 끌 수 있지만 성능 감소
부호 확장
signed char -1(0xFF) → int -1(0xFFFFFFFF). 최상위 비트가 부호로 확장. unsigned는 0으로 채움.
strict aliasing
다른 타입 포인터로 같은 메모리 접근 시 UB. memcpy로 안전한 type punning. -fstrict-aliasing(기본 켜짐).
5/2=2인 이유: 둘 다 int니까 정수 나눗셈! 5.0/2 또는 (double)5/2로 해야 2.5가 돼!
-1 > 1u가 TRUE인 건 C의 가장 위험한 함정! signed와 unsigned를 섞으면 -1이 42억이 돼!
[C 코드] 안전한 형변환 패턴
1#include <stdio.h>
2#include <limits.h>
3#include <math.h>
4
5// 안전한 double→int 변환
6int safe_double_to_int(double d, int *result) {
7 if (d > INT_MAX || d < INT_MIN || isnan(d) || isinf(d))
8 return -1; // 범위 초과 또는 특수값
9 *result = (int)d;
10 return 0;
11}
12
13// 안전한 int 덧셈 (오버플로우 체크)
14int safe_add(int a, int b, int *result) {
15 if ((b > 0 && a > INT_MAX - b) ||
16 (b < 0 && a < INT_MIN - b))
17 return -1;
18 *result = a + b;
19 return 0;
20}
21
22// 안전한 int 곱셈
23int safe_mul(int a, int b, int *result) {
24 if (a > 0 && b > 0 && a > INT_MAX / b) return -1;
25 if (a > 0 && b < 0 && b < INT_MIN / a) return -1;
26 if (a < 0 && b > 0 && a < INT_MIN / b) return -1;
27 if (a < 0 && b < 0 && a < INT_MAX / b) return -1;
28 *result = a * b;
29 return 0;
30}
31
32int main() {
33 int r;
34
35 if (safe_double_to_int(3.14, &r) == 0)
36 printf("3.14 → %d\n", r);
37 if (safe_double_to_int(1e18, &r) != 0)
38 printf("1e18 → 범위 초과!\n");
39
40 if (safe_add(INT_MAX, 1, &r) != 0)
41 printf("INT_MAX + 1 → 오버플로우!\n");
42 if (safe_add(100, 200, &r) == 0)
43 printf("100 + 200 = %d\n", r);
44
45 if (safe_mul(100000, 100000, &r) != 0)
46 printf("100000 * 100000 → 오버플로우!\n");
47 return 0;
48}
실행 결과
3.14 → 3 1e18 → 범위 초과! INT_MAX + 1 → 오버플로우! 100 + 200 = 300 100000 * 100000 → 오버플로우!
[심화 내용] 형변환 관련 보안 취약점
1. Integer Overflow Attack:
size = count * sizeof(item);
→ count가 크면 오버플로우 → 작은 malloc!
→ 이후 버퍼 오버플로우!

2. Signedness Bug:
int len = get_length(); // 음수 가능!
memcpy(dst, src, len); // len이 size_t로 변환 → 거대!

3. Truncation:
int big = 0x12345678;
short small = big; // 0x5678만 남음!

방어:
1. 연산 전 오버플로우 체크
2. signed→unsigned 변환 시 음수 체크
3. 넓은→좁은 타입 변환 시 범위 체크
4. size_t에 signed 값 넣기 전 체크
truncation
넓은 타입→좁은 타입 변환 시 상위 비트 손실. int→short: 상위 16비트 제거. 데이터 손실 위험!
형변환의 3대 함정: 1)정수 나눗셈 2)signed↔unsigned 3)정밀도 손실. 이 세 가지를 항상 경계해!
[C 코드] 실용적 형변환 예제

2

[심화] 다른 언어의 형변환 비교
C: (int)3.14 → 명시적, 위험

Java: (int)3.14 → checked

C의 캐스팅은 가장 위험!
→ 아무 타입이나 변환 가능
→ 컴파일러가 체크 안 함
→ 프로그래머 책임!
narrowing conversion
넓은→좁은 타입 변환. double→int, long→short. 데이터 손실 가능. C++11에서는 {}초기화에서 금지!
(double)5/2=2.5이지만 (double)(5/2)=2.0! 캐스팅 위치가 중요해. 나눗셈 전에 캐스팅해야 해!
[활동] 형변환 마스터 체크리스트
암묵적/명시적 변환의 차이를 안다
정수 승격 규칙을 설명할 수 있다
5/2=2인 이유를 설명한다
signed+unsigned의 위험을 안다
float 정밀도 손실을 이해한다
부호 확장 vs 제로 확장을 안다
안전한 오버플로우 체크를 구현한다
strict aliasing 규칙을 안다

6개 이상이면 형변환 마스터!
(void*)
범용 포인터. 모든 포인터 타입과 변환 가능. malloc 반환 타입. C에서는 캐스트 불필요, C++에서는 필수.
형변환의 황금 규칙: 좁은→넓은은 안전, 넓은→좁은은 위험! double→int, long→short 할 때 범위 체크!
[C 코드] union으로 비트 레벨 접근
1#include <stdio.h>
2#include <string.h>
3
4// union으로 float의 비트 보기 (C에서는 OK)
5union FloatBits {
6 float f;
7 unsigned int i;
8 unsigned char bytes[4];
9};
10
11void inspect_float(float f) {
12 union FloatBits fb;
13 fb.f = f;
14
15 printf("%.6f = 0x%08X = ", f, fb.i);
16 for (int i = 3; i >= 0; i--)
17 printf("%02X ", fb.bytes[i]);
18
19 // 부호, 지수, 가수 추출
20 int sign = (fb.i >> 31) & 1;
21 int exp = (fb.i >> 23) & 0xFF;
22 int mantissa = fb.i & 0x7FFFFF;
23 printf("| S=%d E=%d M=0x%06X\n", sign, exp-127, mantissa);
24}
25
26// 네트워크 바이트 오더 변환 (빅↔리틀 엔디안)
27unsigned int swap_endian(unsigned int x) {
28 return ((x >> 24) & 0xFF) |
29 ((x >> 8) & 0xFF00) |
30 ((x << 8) & 0xFF0000) |
31 ((x << 24) & 0xFF000000);
32}
33

43}

implicit conversion rank
변환 우선순위. long double > double > float > unsigned long long > long long > ... > int.
integer overflow
signed 오버플로우 = UB! unsigned 오버플로우 = 모듈러 연산(순환). UINT_MAX+1=0.
C의 캐스팅은 "만능 열쇠"야. 편하지만 위험해! C++의 4종 캐스트처럼 의도를 명확히 하는 습관을 들이자!
size_t 주의
size_t는 unsigned. int와 비교 시 -1 > sizeof(...) 같은 버그 발생! 항상 같은 타입으로 비교.

char literal
C에서 'a'는 int 타입! sizeof('a')=4. C++에서는 char 타입으로 sizeof('a')=1. C와 C++의 차이!
sizeof('a')가 C에서 4인 건 유명한 퀴즈! C에서 문자 리터럴은 int, C++에서는 char. 면접에서 자주 나와!
round
반올림. round(3.5)=4, round(-3.5)=-4. (int)(x+0.5)는 음수에서 틀림! round() 사용.
trunc
버림 (0 방향). trunc(3.9)=3, trunc(-3.9)=-3. (int)와 동일하지만 double 반환.
floor/ceil
floor: 내림(-방향). floor(-3.1)=-4. ceil: 올림(+방향). ceil(3.1)=4. 정수 나눗셈 올림: (a+b-1)/b.
반올림 방법: round(사사오입), floor(내림), ceil(올림), trunc(버림). 목적에 맞는 걸 골라!
__builtin_add_overflow
GCC/Clang 내장. 오버플로우 감지 덧셈. if(__builtin_add_overflow(a,b,&r)) 에러처리. 가장 안전한 방법!
intmax_t
. 시스템 최대 정수 타입. 보통 long long(64비트). printf는 %jd로 출력.
uintptr_t
포인터를 정수로 저장하는 타입. (uintptr_t)ptr로 변환. 포인터 산술, 해시에 활용.
형변환 안전 순서: 체크→변환→검증. 변환 전 범위 체크, 변환 후 결과 확인. 무조건 캐스팅은 금지!
lround
. 반올림 후 long 반환. lround(3.7)=4L. 범위 초과 시 정의되지 않음. lrint도 유사.
FLT_DIG
. float 유효 십진 자릿수. 보통 6. DBL_DIG는 보통 15. 정밀도 한계 확인에 유용.
형변환 마스터의 비밀: "데이터가 손실되는가?"를 항상 자문해! 손실 없으면 안전, 있으면 체크!
fpclassify
. 부동소수점 분류. FP_NORMAL, FP_ZERO, FP_INFINITE, FP_NAN, FP_SUBNORMAL 중 하나 반환.
isnan/isinf
. NaN/무한대 검사. isnan(0.0/0.0)=1, isinf(1.0/0.0)=1. 실수 연산 결과 검증에 필수!
형변환을 마스터하면 C의 타입 시스템이 보여! 암묵적 변환의 함정을 피하면 버그가 확 줄어!
fma
. Fused Multiply-Add. fma(a,b,c) = a*b+c를 한 번에 계산. 정밀도 향상 + 성능!
modf
. 실수를 정수부+소수부로 분리. double frac = modf(3.14, &ipart); → ipart=3, frac=0.14.
conversion rank
C 표준 정수 변환 순위. _Bool < char < short < int < long < long long. 산술 변환 시 높은 쪽으로 승격.
sign extension
부호 확장. 좁은→넓은 signed 변환 시 최상위 비트(부호)로 채움. char(-1)=0xFF → int(-1)=0xFFFFFFFF.
copysign
. 부호 복사. copysign(3.0,-1.0)=-3.0. 절대값에 원하는 부호 적용. IEEE 754 -0 처리에도 활용.
형변환
한 자료형을 다른 자료형으로 바꾸는 것.
자동 형변환
서로 다른 타입이 만날 때 컴파일러가 알아서 변환. 작은 타입 → 큰 타입.
캐스팅
강제 형변환. (타입)값 형태로 직접 변환을 지정.
데이터 손실
큰 타입에서 작은 타입으로 변환할 때 정보가 사라지는 것. 예: (int)3.9 → 3
