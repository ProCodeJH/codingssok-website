Lesson 5-4

바뀌면 안 되는 값 — const

const와 #define으로 상수를 만들고, 매직넘버를 없애는 습관을 배워요.

📚 이번에 배울 것

  • const로 상수를 선언할 수 있어요.
  • #define으로 매크로 상수를 만들 수 있어요.
  • 매직넘버가 왜 나쁜지 이해해요.

✅ 시작 전 체크

  • 변수 선언과 초기화를 할 수 있다
  • int, double, char 자료형을 안다
Part 1 진행률: 30%

변하면 안 되는 값이 있어요

원주율은 3.14이고, 1시간은 60분이에요. 이런 값은 절대 바뀌면 안 되죠! 오늘은 값을 고정하는 방법을 배워요.

변수는 값을 바꿀 수 있어요. 하지만 원주율, 세율, 중력 가속도처럼 절대 변하면 안 되는 값도 있어요. 실수로 바꾸면 큰 문제가 생기죠.

const — 변수를 잠금

const1.c
1#include <stdio.h>
2
3int main(void) {
4 const double PI = 3.141592;
5 const int MAX_SCORE = 100;
6
7 printf("원주율: %f\n", PI);
8 printf("만점: %d\n", MAX_SCORE);
9
10 // PI = 3.14; // 에러! const는 바꿀 수 없어요!
11 return 0;
12}
실행 결과
원주율: 3.141592 만점: 100
4줄const double PI = 3.141592; — const를 붙이면 값을 바꿀 수 없어요. 잠긴 상자예요!
10줄값을 바꾸려고 하면 컴파일 에러가 나요. 실수로 바꾸는 걸 미리 막아주니까 안전해요!
const 변수 이름은 보통 대문자로 써요. PI, MAX_SCORE처럼요. 한눈에 '이건 상수구나' 알 수 있거든요.

#define — 매크로 상수

const 말고 #define으로도 상수를 만들 수 있어요. #include처럼 코드 맨 위에 적어요.

[핵심] const vs #define
const: 진짜 변수. 타입이 있음. 디버깅 쉬움. 이쪽을 권장!
#define: 전처리기 치환. 타입 없음. 코드에서 이름이 숫자로 바뀜.

둘 다 값을 바꿀 수 없다는 점은 같아요. 초보자는 const를 먼저 익히세요.

매직넘버가 뭘까?

코드에 의미 없는 숫자가 갑자기 등장하면, 나중에 읽을 때 뭔지 모르겠어요. 이런 숫자를 매직넘버라고 해요.

✕ 잘못된 코드
total = price * 1.1; if (score >= 60) ...
1.1이 뭐야? 60이 뭐야? 매직넘버!
✓ 올바른 코드
const double TAX = 1.1; const int PASS = 60; total = price * TAX; if (score >= PASS) ...
TAX, PASS — 의미가 분명해요!
[코딩 습관] 매직넘버 금지!
코드에 숫자가 나오면 '이 숫자의 의미가 뭐지?' 물어보세요. 의미가 있으면 const로 이름을 붙이세요. 0, 1 같은 간단한 수를 제외하고는 대부분 이름을 붙이는 게 좋아요.

용어 정리

[심화] const 완전 가이드

[심화] const의 다양한 위치
변수:
const int x = 42; // 값 변경 불가
int const x = 42; // 위와 동일

포인터 + const (4가지!):
const int *p; // 값 수정 불가, 포인터 변경 가능
int const *p; // 위와 동일
int *const p; // 값 수정 가능, 포인터 변경 불가
const int *const p; // 둘 다 불가

읽는 법:
오른쪽에서 왼쪽으로!
const int *p → "p는 포인터, int const를 가리킴"
int *const p → "p는 const 포인터, int를 가리킴"

함수 매개변수:
void print(const char *s); // s의 내용 수정 안 함
→ 함수가 데이터를 안 바꾼다는 계약!

[심화 내용] const vs #define vs enum

[심화 내용] 상수 정의 방법 비교
#define MAX 100
전처리기 텍스트 치환
타입 없음, 스코프 없음
디버거에서 이름 안 보임

const int MAX = 100;
타입 있음, 스코프 있음
디버거에서 이름 보임
주소 있음 (&MAX 가능)
배열 크기로 사용 불가 (C, VLA 제외)

enum { MAX = 100 };
정수 상수만 가능
타입: int
배열 크기로 사용 가능!
디버거에서 이름 보임

추천:
정수 상수 → enum
실수/문자열 상수 → const
조건부 컴파일 → #define
const correctness
const를 올바르게 사용하는 관행. 수정하지 않는 매개변수에 const 추가. 버그 예방 + 의도 표현.
const 포인터 읽기
오른쪽→왼쪽으로! const int *p: p가 가리키는 int는 const. int *const p: p 자체가 const.
const는 "약속"이야! const char *s는 "이 함수는 s를 안 바꿔"라는 약속. 컴파일러가 지켜줘!
const int *p vs int *const p — 면접 단골! 별(*)을 기준으로 왼쪽 const=값 보호, 오른쪽 const=포인터 보호!

[심화 내용] constexpr과 C23
C23의 constexpr:
constexpr int SIZE = 100;
→ 진정한 컴파일 타임 상수!
→ 배열 크기로 사용 가능!

C에서의 진화:
C89: #define SIZE 100 (전처리기)
C99: const int SIZE = 100 (런타임 상수)
C11: _Static_assert (컴파일 타임 검사)
C23: constexpr (컴파일 타임 상수)

_Static_assert 활용:
_Static_assert(sizeof(int)==4, "int must be 4 bytes");
→ 컴파일 시 조건 검사!
constexpr
C23. 진정한 컴파일 타임 상수. 배열 크기로 사용 가능. C++11에서 먼저 도입되어 C23에 추가.
const의 핵심 가치: 1)실수 방지 2)의도 표현 3)최적화 힌트. 수정 안 할 거면 무조건 const 붙여!
[심화] const의 한계
1. C에서 const는 진짜 상수가 아님!
const int n = 10;
int arr[n]; // VLA! C99에서는 OK, C89에서는 ERROR
→ #define N 10 또는 enum { N = 10 } 사용

2. const를 우회할 수 있음:
const int x = 42;
int *p = (int*)&x;
*p = 99; // 컴파일 됨! 하지만 UB!

3. const 포인터 매개변수의 한계:
void f(const int *p) {
// *p = 42; // 컴파일 에러 (OK)
// 하지만 p가 가리키는 원본은 다른 경로로 수정 가능
}

결론:
const는 "의도 표현"과 "실수 방지"
보안 메커니즘은 아님!
_Static_assert
C11 컴파일 타임 검사. _Static_assert(sizeof(int)==4, "need 32-bit int"). 이식성 검증에 유용.
const int n=10; int arr[n]; 이 C89에서 안 되는 이유: const는 "변수"이고 배열 크기는 "상수"여야 해!

compound literal
C99. (Type){초기화}로 임시 객체 생성. (int[]){1,2,3}처럼 배열도 가능. 함수 인자에 유용.
designated initializer
C99. .field = value로 구조체 필드 지정 초기화. 순서 무관, 가독성 향상. .app_name = "Hi" 등.
const 구조체 + designated initializer = 불변 설정 객체! 생성 후 수정 불가. 안전한 설계 패턴!
[심화 내용] const in embedded/kernel
임베디드에서 const:
const int config[] = {1,2,3};
→ ROM/Flash에 저장! RAM 절약!
→ 임베디드에서 const = 메모리 절약

리눅스 커널:
const struct file_operations fops = {
.read = my_read,
.write = my_write,
};
→ 읽기 전용 섹션에 배치
→ 보안: 함수 포인터 변조 방지

volatile const:
volatile const int *hw_reg = (void*)0x40000000;
→ 하드웨어 레지스터 (읽기 전용, 값 변할 수 있음)
→ const: 쓰기 방지, volatile: 캐싱 방지
volatile const
읽기 전용이지만 외부에서 변할 수 있는 값. 하드웨어 상태 레지스터에 사용. const+volatile 동시 가능!
ROM 배치
임베디드에서 const 데이터는 Flash/ROM에 저장. RAM 절약! const가 없으면 RAM에 복사됨.
임베디드에서 const는 성능이 아니라 생존이야! RAM이 2KB뿐인데 const 없으면 데이터가 RAM을 다 먹어!

함수 포인터 테이블
함수 포인터 배열로 switch 대체. const로 변조 방지. 플러그인 시스템, 상태 머신에 활용.
const 함수 포인터 테이블 = 안전한 전략 패턴! switch 대신 배열로 연산 선택. 깔끔하고 확장 쉬워!
const propagation
컴파일러 최적화. const 값을 상수로 치환. const int x=5; y=x*2; → y=10으로 최적화!
restrict const
const char *restrict s: s가 가리키는 메모리는 이 포인터만 접근+읽기만. 최대 최적화 힌트!
immutable pattern
불변 패턴. 생성 후 변경 불가. const 구조체로 구현. 멀티스레드에서 안전. 함수형 프로그래밍의 핵심.
const 습관: 1)매개변수에 const 2)반환값에 const 3)지역변수도 가능하면 const. 실수를 컴파일러가 잡아줘!
const 함수
C++에는 const 멤버 함수가 있지만 C에는 없음. C에서는 const 포인터 매개변수로 대체.
string literal
"Hello"는 const char[] 타입(C++). C에서는 char[]이지만 수정하면 UB. 항상 const char*로 받기!
extern const
다른 파일의 const 변수 참조. 헤더: extern const int MAX; 소스: const int MAX = 100;
const가 없는 C 코드는 "잠금장치 없는 집"이야! const를 적극적으로 써서 실수를 컴파일 타임에 잡자!
const 포이즌
const가 전파되는 현상. const 구조체의 멤버도 const. const char *를 받는 함수에 char* 전달 OK (역은 불가).
restrict+const
const char *restrict s: 최대 최적화 힌트. memcpy(dst, src, n)에서 src가 restrict const.
Part 2 const: 변수 보호, API 계약, 최적화 힌트, ROM 배치. const 하나가 4가지 역할을 해!
const 배열 크기
C89: const int N=10; int a[N]; 불가! C99: VLA로 가능하지만 권장 안 함. enum이나 #define 사용.
const in header
헤더에 const 정의 시 static const 사용. 안 하면 다중 정의 에러! static으로 파일별 복사.
enum vs const
배열 크기: enum OK, const NO (C89). 디버거 이름: 둘 다 OK. 타입: enum=int만, const=모든 타입.
const 정리: 정수 상수→enum, 실수/문자열→const, 조건부 컴파일→#define. 상황에 맞는 선택!
const 최적화
컴파일러가 const 변수를 레지스터에 캐싱하거나 상수로 인라인. volatile과 반대 효과.
CONST_CAST 위험
const를 캐스팅으로 제거 후 수정하면 UB. 원래 const 객체는 읽기 전용 메모리에 있을 수 있음!
const correctness는 습관이야! 처음부터 const를 붙이는 습관을 들이면 나중에 제거하기 쉽지만, 나중에 추가하기는 어려워!
const qualify
함수 인자에 const 추가 = API 문서화. const char *s는 "이 함수는 s를 읽기만 한다"는 약속.
mutable (C++)
C++에서 const 객체 내 변경 가능한 멤버. C에는 없지만 포인터 우회로 유사 구현 가능 (비권장).
static const 헤더
헤더에 상수 정의: static const int X = 42; 각 TU에 복사본 생성. extern const는 한 곳에서만 정의.
const volatile
const: 코드가 수정 안 함. volatile: 외부에서 변할 수 있음. 하드웨어 상태 레지스터의 정확한 표현!
inline const
C99 inline 함수와 const 결합. static inline const int get_max(void){return 100;} 최적화+안전.
typedef const
typedef const char* LPCSTR; Windows API 스타일. LP=Long Pointer, C=Const, STR=String.
const를 마스터했어! 변수 보호, API 계약, ROM 배치, 컴파일러 최적화 — const 하나로 4마리 토끼!
const 전파
const T* 매개변수에 T* 전달 OK. 역방향(T*에 const T*) 불가 → 경고! const는 한 방향으로만 전파.
const
변수 앞에 붙여서 값을 바꿀 수 없게 만드는 키워드.
#define
전처리기 지시문. 이름을 값으로 치환해요. 타입이 없어요.
상수(Constant)
프로그램 실행 중에 절대 바뀌지 않는 값.
매직넘버
코드에 의미 설명 없이 등장하는 숫자. 이름을 붙여야 해요.
