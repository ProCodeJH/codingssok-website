비교 결과의 자료형

비교 연산자의 결과는 int형이에요. 참이면 1, 거짓이면 0이에요. C99부터는 stdbool.h에 bool 타입도 있지만, 비교 연산자 자체의 결과는 여전히 int예요.

compare_type.c
1#include <stdio.h>
2
3int main(void) {
4 int result = (10 > 5); // result = 1
5 printf("result = %d\n", result);
6 printf("sizeof = %zu\n", sizeof(10 > 5)); // int의 크기 = 4
7 return 0;
8}
실행 결과
result = 1 sizeof = 4

[주의] = 와 == 의 차이

[주의] 가장 흔한 실수!
= — 대입! 값을 넣어라.
== — 비교! 같은지 물어봐라.

이 둘을 헷갈리면 프로그램이 완전히 다르게 동작해요!
if (a = 5) → a에 5를 넣음 (항상 참이 됨!)
if (a == 5) → a가 5인지 비교 (의도한 동작)
✕ 잘못된 코드
if (score = 100)
= 는 대입! score에 100을 넣어버려요
✓ 올바른 코드
if (score == 100)
== 비교! score가 100인지 검사
[팁] 요다 조건(Yoda Condition)
상수를 왼쪽에 놓으면 실수를 방지할 수 있어요!
if (100 == score) — 실수로 = 하나만 쓰면 컴파일 에러!
if (100 = score) — 상수에 대입? 컴파일러가 잡아줘요!

리눅스 커널 코딩 스타일에서도 쓰는 방법이에요.

비교 연산자와 다양한 자료형

지금까지는 정수끼리 비교했지만, 비교 연산자는 실수, 문자 등 다양한 자료형에도 쓸 수 있어요.

문자(char) 비교

문자는 내부적으로 ASCII 코드(숫자)로 저장되므로, 문자끼리 비교하면 ASCII 값을 비교하는 거예요.

char_compare.c
1#include <stdio.h>
2
3int main(void) {
4 char ch = 'B';
5
6 printf("'A' == 'A' : %d\n", 'A' == 'A'); // 1
7 printf("'A' < 'Z' : %d\n", 'A' < 'Z'); // 1 (65 < 90)
8 printf("'a' > 'A' : %d\n", 'a' > 'A'); // 1 (97 > 65)
9 printf("ch == 'B' : %d\n", ch == 'B'); // 1
10
11 // 대문자 판별
12 printf("대문자? %d\n", ch >= 'A' && ch <= 'Z'); // 1
13 return 0;
14}
실행 결과
'A' == 'A' : 1 'A' < 'Z' : 1 'a' > 'A' : 1 ch == 'B' : 1 대문자? 1
[핵심] 문자 비교 = 숫자 비교
'A'는 65, 'Z'는 90, 'a'는 97, 'z'는 122.
문자를 비교하면 이 숫자들을 비교하는 거예요.
ch >= 'A' && ch <= 'Z' → 대문자인지 검사하는 패턴!

비교 연산자의 재미있는 활용

비교 결과가 0 또는 1이라는 점을 이용하면 if문 없이도 재미있는 일을 할 수 있어요!

compare_trick.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 10, b = 20, c = 10;
5
6 // 비교 결과를 숫자처럼 사용
7 int count = (a == c) + (b > a) + (a != b); // 1 + 1 + 1 = 3
8 printf("참인 조건 개수: %d\n", count);
9
10 // 두 수 중 큰 수 구하기 (if 없이!)
11 int max = a * (a >= b) + b * (b > a);
12 printf("큰 수: %d\n", max); // 20
13 return 0;
14}
실행 결과
참인 조건 개수: 3 큰 수: 20
비교 결과를 산술 연산에 활용할 수 있어요! 이런 기법은 KOI 문제에서도 유용해요.

비교 체이닝의 함정

수학에서 1 < x < 10이라고 쓰면 'x는 1 초과 10 미만'이라는 뜻이죠? 하지만 C언어에서 이렇게 쓰면 완전히 다른 의미가 돼요!

chaining_bug.c
1#include <stdio.h>
2
3int main(void) {
4 int x = 15; // 1 < 15 < 10 은 수학적으로 거짓
5
6 // 하지만 C에서는...
7 printf("%d\n", 1 < x < 10); // 1! (왜?)
8
9 // C의 해석: (1 < x) < 10
10 // 1단계: 1 < 15 → 1 (참)
11 // 2단계: 1 < 10 → 1 (참)
12 // 결과: 1 (예상과 다름!)
13 return 0;
14}
실행 결과
1
[경고] 비교 체이닝은 C에서 안 돼요!
a < b < c는 C에서 (a < b) < c로 해석돼요!
(a < b)의 결과는 0 또는 1이므로, 그 결과를 c와 비교하는 거예요.

올바른 방법: a < b && b < c

참고: Python에서는 1 < x < 10이 수학처럼 동작해요! 언어마다 달라요.

[중급] 부동소수점 비교의 함정

실수(float, double)를 비교할 때는 조심해야 해요! 컴퓨터는 소수를 정확하게 저장하지 못하는 경우가 있거든요.
float_compare.c
1#include <stdio.h>
2
3int main(void) {
4 double a = 0.1 + 0.2;
5 double b = 0.3;
6
7 printf("a = %.20f\n", a); // 0.30000000000000004441
8 printf("b = %.20f\n", b); // 0.29999999999999998890
9
10 printf("a == b ? %d\n", a == b); // 0 (거짓!)
11 // 0.1 + 0.2는 0.3이 아니다?!
12 return 0;
13}
실행 결과
a = 0.30000000000000004441 b = 0.29999999999999998890 a == b ? 0
[경고] 0.1 + 0.2 != 0.3
컴퓨터는 소수를 2진수로 저장해요. 0.1, 0.2, 0.3은 2진수로 무한 소수예요!
그래서 아주 작은 오차가 생기고, ==로 비교하면 '다르다'고 나와요.

이건 C언어뿐 아니라 JavaScript, Python 등 거의 모든 언어에서 동일해요.

[중급] epsilon 비교 — 올바른 실수 비교법

실수를 비교할 때는 '정확히 같은지'가 아니라 '차이가 아주 작은지'를 확인해야 해요. 이 아주 작은 값을 epsilon(엡실론)이라고 불러요.

epsilon_compare.c
1#include <stdio.h>
2#include <math.h> // fabs() 함수
3
4#define EPSILON 1e-9 // 0.000000001
5
6int main(void) {
7 double a = 0.1 + 0.2;
8 double b = 0.3;
9
10 // 나쁜 방법: 직접 비교
11 printf("== 비교: %d\n", a == b); // 0 (거짓)
12
13 // 좋은 방법: epsilon 비교
14 printf("epsilon 비교: %d\n", fabs(a - b) < EPSILON); // 1 (참)
15 // |a - b| < 0.000000001 이면 '같다'고 판단
16 return 0;
17}
실행 결과
== 비교: 0 epsilon 비교: 1
[핵심] 실수 비교 공식
fabs(a - b) < EPSILON

fabs()는 절댓값 함수예요.
두 실수의 차이가 EPSILON보다 작으면 '같다'고 판단해요.
EPSILON은 상황에 따라 1e-6, 1e-9 등으로 설정해요.

[중급] 상대 오차 비교

아주 큰 수를 비교할 때는 절대 오차(EPSILON) 대신 상대 오차를 써야 더 정확해요.

relative_epsilon.c
1#include <stdio.h>
2#include <math.h>
3
4// 상대 오차 비교 함수
5int nearly_equal(double a, double b) {
6 double diff = fabs(a - b);
7 double largest = fabs(a) > fabs(b) ? fabs(a) : fabs(b);
8
9 if (largest == 0.0) return diff < 1e-9; // 둘 다 0에 가까울 때
10 return (diff / largest) < 1e-9; // 상대 오차 비교
11}
12
13int main(void) {
14 double x = 1e15 + 1.0;
15 double y = 1e15;
16 printf("절대 비교: %d\n", fabs(x - y) < 1e-9); // 0 (다르다)
17 printf("상대 비교: %d\n", nearly_equal(x, y)); // 1에 가까움
18 return 0;
19}

[중급] unsigned vs signed 비교의 함정

부호 있는 정수(signed)와 부호 없는 정수(unsigned)를 비교하면 놀라운 결과가 나올 수 있어요!
signed_unsigned.c
1#include <stdio.h>
2
3int main(void) {
4 int a = -1; // signed: -1
5 unsigned int b = 0; // unsigned: 0
6
7 printf("-1 > 0u ? %d\n", a > b); // 1! (-1이 0보다 크다?!)
8
9 // 왜? a가 unsigned로 변환되면서
10 // -1 → 4294967295 (0xFFFFFFFF)가 돼요!
11 printf("-1을 unsigned로: %u\n", (unsigned int)a); // 4294967295
12 return 0;
13}
실행 결과
-1 > 0u ? 1 -1을 unsigned로: 4294967295
[경고] signed와 unsigned를 섞지 마세요!
signed와 unsigned를 비교하면 signed가 unsigned로 자동 변환돼요.
-1은 unsigned로 바뀌면 4294967295(최댓값!)가 돼요.
그래서 -1 > 0u가 참이 되는 거예요!

컴파일러 경고를 반드시 켜세요: gcc -Wall

[중급] 문자열 비교

C언어에서 문자열(char 배열)은 ==로 비교할 수 없어요! ==는 주소를 비교하기 때문이에요.

string_compare.c
1#include <stdio.h>
2#include <string.h>
3
4int main(void) {
5 char a[] = "hello";
6 char b[] = "hello";
7
8 // == 로 비교하면? → 주소(포인터) 비교!
9 printf("a == b ? %d\n", a == b); // 0! (다른 메모리 주소)
10
11 // strcmp()로 비교해야 내용 비교!
12 printf("strcmp: %d\n", strcmp(a, b)); // 0이면 같다!
13
14 // strcmp 반환값:
15 // 0 → 같다
16 // 음수 → a가 사전순으로 앞
17 // 양수 → a가 사전순으로 뒤
18 printf("apple vs banana: %d\n", strcmp("apple", "banana")); // 음수
19 return 0;
20}
실행 결과
a == b ? 0 strcmp: 0 apple vs banana: -1
✕ 잘못된 코드
if (name == "hello")
주소를 비교! 내용이 같아도 0이 나올 수 있어요
✓ 올바른 코드
if (strcmp(name, "hello") == 0)
strcmp()로 내용 비교! 0이면 같다
[핵심] 문자열 비교 패턴
strcmp(a, b) == 0 → 같다
strcmp(a, b) < 0 → a가 사전순으로 앞
strcmp(a, b) > 0 → a가 사전순으로 뒤

strncmp(a, b, n) → 처음 n글자만 비교 (더 안전)

[심화] 구조체 비교

구조체(struct)도 ==로 비교할 수 없어요. memcmp()를 쓸 수 있지만, 패딩(padding) 때문에 주의해야 해요.

struct_compare.c
1#include <stdio.h>
2#include <string.h>
3
4struct Point {
5 char type; // 1바이트
6 // 여기에 3바이트 패딩이 들어갈 수 있음!
7 int x; // 4바이트
8 int y; // 4바이트
9};
10
11int main(void) {
12 struct Point p1 = {'A', 10, 20};
13 struct Point p2 = {'A', 10, 20};
14
15 // memcmp: 패딩 영역까지 비교 → 불안정!
16 // 패딩에 쓰레기 값이 있으면 '다르다'고 나올 수 있음
17 printf("memcmp: %d\n", memcmp(&p1, &p2, sizeof(struct Point)));
18
19 // 안전한 방법: 멤버별 비교
20 int equal = (p1.type == p2.type) &&
21 (p1.x == p2.x) &&
22 (p1.y == p2.y);
23 printf("멤버별 비교: %d\n", equal); // 1 (확실!)
24 return 0;
25}
[경고] memcmp()의 패딩 문제
구조체에는 메모리 정렬을 위한 패딩(빈 공간)이 들어갈 수 있어요.
패딩 영역에는 쓰레기 값이 있을 수 있어서, memcmp()가 '다르다'고 판단할 수 있어요.

구조체 비교는 멤버별 비교가 가장 안전해요!

[심화] NaN 비교 — IEEE 754

NaN(Not a Number)은 부동소수점에서 '정의되지 않은 결과'를 나타내는 특별한 값이에요. NaN의 가장 놀라운 특징은 자기 자신과도 같지 않다는 거예요!

nan_compare.c
1#include <stdio.h>
2#include <math.h> // NAN, isnan()
3
4int main(void) {
5 double x = 0.0 / 0.0; // NaN 생성
6 double y = NAN; // NaN 매크로
7
8 printf("x = %f\n", x); // nan
9 printf("x == x ? %d\n", x == x); // 0! (NaN은 자기 자신과도 다르다!)
10 printf("x != x ? %d\n", x != x); // 1!
11 printf("x > 0 ? %d\n", x > 0); // 0
12 printf("x < 0 ? %d\n", x < 0); // 0
13 printf("x == 0 ? %d\n", x == 0); // 0
14
15 // NaN 판별 방법
16 printf("isnan(x) ? %d\n", isnan(x)); // 1
17 printf("x != x ? %d\n", x != x); // 1 (이 트릭도 사용 가능)
18 return 0;
19}
실행 결과
x = nan x == x ? 0 x != x ? 1 x > 0 ? 0 x < 0 ? 0 x == 0 ? 0 isnan(x) ? 1 x != x ? 1
[핵심] NaN의 비교 규칙 (IEEE 754)
NaN은 모든 비교에서 거짓을 반환해요 (자기 자신과의 비교 포함!):
• NaN == NaN → 0 (거짓)
• NaN != NaN → 1 (참) ← 유일한 참!
• NaN > x → 0
• NaN < x → 0

이건 IEEE 754 표준의 설계 의도예요. NaN이 전파되면서 '이 값은 의미없다'는 걸 알려주기 위함이에요.
[팁] NaN 판별 트릭
x != x가 참이면 x는 NaN이에요!
왜냐면 NaN만이 자기 자신과 같지 않은 유일한 값이거든요.
isnan(x) 함수를 쓰는 게 더 명확해요. (math.h 필요)

[심화] 비교의 CPU 레벨 — CMP 명령어와 플래그

CPU 안에서 비교 연산이 어떻게 동작하는지 살펴볼까요? 이건 심화 내용의 내용이에요!

CPU에서 비교는 뺄셈을 수행하되 결과를 버리고, 플래그만 설정하는 방식으로 동작해요. x86 CPU의 CMP 명령어가 이 일을 해요.

[역사] 왜 CMP는 뺄셈을 할까?
초기 CPU는 트랜지스터가 비쌌기 때문에, 비교 전용 회로를 만들지 않고 이미 있는 ALU(산술논리장치)의 뺄셈 기능을 재활용했어요.

A - B의 결과가 0이면 같고, 음수면 A < B, 양수면 A > B.
결과 값은 버리고 플래그만 보는 거예요. 매우 효율적인 설계!

[심화] 3-way 비교

일반 비교 연산자는 '참/거짓'만 알려주지만, 3-way 비교는 '작다/같다/크다'를 한 번에 알려줘요.

three_way.c
1#include <stdio.h>
2#include <string.h>
3#include <stdlib.h>
4
5// qsort용 비교 함수 — 3-way 비교의 전형적인 예
6int compare_int(const void *a, const void *b) {
7 int ia = *(const int *)a;
8 int ib = *(const int *)b;
9 // 음수: a < b, 0: a == b, 양수: a > b
10 if (ia < ib) return -1;
11 if (ia > ib) return 1;
12 return 0;
13}
14
15int main(void) {
16 int arr[] = {5, 2, 8, 1, 9, 3};
17 int n = sizeof(arr) / sizeof(arr[0]);
18
19 qsort(arr, n, sizeof(int), compare_int);
20
21 for (int i = 0; i < n; i++)
22 printf("%d ", arr[i]);
23 printf("\n");
24 return 0;
25}
실행 결과
1 2 3 5 8 9
[경고] qsort 비교 함수의 흔한 실수
return a - b;로 쓰는 건 위험해요!
큰 수의 차이가 int 범위를 넘으면 오버플로우가 발생해요.

안전한 방법:
if (a < b) return -1;
if (a > b) return 1;
return 0;

C++20에서는 <=> (우주선 연산자, spaceship operator)가 도입되어 3-way 비교를 언어 차원에서 지원해요. C에서는 아직 없지만, 비교 함수를 직접 만들어 쓰면 돼요.

[심화] 비교 함수 직접 만들기

1#include <stdio.h>
2
3// 3-way 비교 함수 패턴
4int cmp_int(int a, int b) {
5 return (a > b) - (a < b); // -1, 0, 1 반환
6}
7
8int main(void) {
9 printf("3 vs 5: %d\n", cmp_int(3, 5)); // -1

실행 결과

[팁] (a > b) - (a < b) 트릭
이 한 줄로 3-way 비교가 완성돼요!
• a < b → (0) - (1) = -1
• a == b → (0) - (0) = 0
• a > b → (1) - (0) = 1

오버플로우 걱정 없이 안전해요.

[심화 내용] 관계 대수 — 수학에서의 비교

수학에서 관계(relation)란 두 원소 사이의 연결을 뜻해요. 비교 연산자는 수학적 관계의 프로그래밍 표현이에요.

[수학] 동치 관계(Equivalence Relation)
== 연산은 수학에서 동치 관계를 구현해요. 동치 관계는 세 가지 성질을 만족해야 해요:

1. 반사성: a == a는 항상 참 (단, NaN 제외!)
2. 대칭성: a == b이면 b == a
3. 추이성: a == b이고 b == c이면 a == c

부동소수점의 ==는 NaN 때문에 반사성을 만족하지 않아요!
[수학] 전순서(Total Order)와 부분 순서(Partial Order)
전순서: 모든 두 원소를 비교할 수 있는 관계
• 정수의 < 관계는 전순서 (임의의 두 정수 비교 가능)

부분 순서: 일부 원소는 비교할 수 없는 관계
• 부동소수점의 < 관계는 부분 순서 (NaN은 비교 불가)
• 집합의 ⊂ 관계도 부분 순서 ({1,2}와 {3,4}는 비교 불가)

이런 수학적 성질은 정렬 알고리즘의 정확성에 직접 영향을 줘요. qsort에 전달하는 비교 함수가 전순서를 만족하지 않으면 정렬이 올바르게 동작하지 않을 수 있어요.

[심화 내용] IEEE 754 비교의 완전한 이해

IEEE 754 부동소수점 표준은 특수한 값들(±0, ±∞, NaN)의 비교 규칙을 상세히 정의해요.

[심화 내용] 브랜치리스(Branchless) 비교

조건 분기(branch)는 CPU 파이프라인에서 분기 예측 실패 시 성능 저하를 일으켜요. 브랜치리스 코드는 이를 피하는 고급 최적화 기법이에요.

branchless.c
1#include <stdio.h>
2
3// 브랜치리스 min/max
4int branchless_min(int a, int b) {
5 // 비교 결과(0 또는 1)를 산술에 활용
6 return b + (a - b) * (a < b);
7}
8
9int branchless_max(int a, int b) {
10 return a + (b - a) * (b > a);
11}
12
13// 절댓값 (브랜치리스)
14int branchless_abs(int x) {
15 int mask = x >> 31; // 부호 비트 확장
16 return (x ^ mask) - mask;
17}
18
19int main(void) {
20 printf("min(3,7) = %d\n", branchless_min(3, 7)); // 3
21 printf("max(3,7) = %d\n", branchless_max(3, 7)); // 7
22 printf("abs(-5) = %d\n", branchless_abs(-5)); // 5
23 return 0;
24}

• 대량 데이터 처리에서 분기 예측 실패율이 50%에 가까울 때
• 보통은 컴파일러가 알아서 최적화해주니, 프로파일링 후에 적용하세요

[심화 내용] 비교 연산의 타입 승격(Promotion) 규칙

서로 다른 자료형을 비교하면 C 컴파일러는 공통 타입으로 변환 후 비교해요. 이를 usual arithmetic conversions라고 해요.

1
1단계: 정수 승격
char, short → int로 승격
2
2단계: 같은 부호?
둘 다 signed 또는 unsigned면 더 큰 타입으로
3
3단계: unsigned가 있으면
signed가 unsigned로 변환 (위험!)
4
4단계: 실수가 있으면
정수가 float/double로 변환

부동소수점 비교 시 아주 작은 오차(epsilon)를 허용하는 비교 방법.
NaN
Not a Number. IEEE 754에서 정의되지 않은 결과를 나타내는 특수 값.
strcmp()
문자열 내용을 비교하는 함수. 0이면 같다.
3-way 비교
두 값의 관계를 음수/0/양수로 반환하는 비교 방식.
CMP 명령어
CPU에서 두 값을 빼서 플래그만 설정하고 결과를 버리는 비교 명령어.
패딩(padding)
메모리 정렬을 위해 구조체 멤버 사이에 삽입되는 빈 공간.

[역사] 부동소수점의 역사
IEEE 754 표준은 1985년에 처음 발표되었어요. 그 전에는 컴퓨터마다 실수 표현이 달라서 같은 프로그램이 다른 결과를 내는 경우가 많았어요. 윌리엄 카한(William Kahan) 교수가 주도해서 만든 이 표준 덕분에 전 세계 컴퓨터가 같은 방식으로 실수를 처리하게 되었어요.
