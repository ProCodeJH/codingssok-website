논리 연산자

&&(그리고), ||(또는), !(아니다) — 여러 조건을 조합하는 방법을 배워요.

📚 이번에 배울 것

  • &&, ||, ! 논리 연산자를 사용할 수 있어요.
  • 참/거짓의 조합 결과(진리표)를 이해해요.
  • 여러 조건을 조합한 복합 조건을 만들 수 있어요.
  • 단축 평가(short-circuit evaluation)를 이해해요.
  • 드모르간 법칙을 코드 리팩토링에 활용할 수 있어요.
  • [심화] 불 대수, 카르노 맵, 논리 게이트를 이해해요.

✅ 시작 전 체크

  • 비교 연산자(==, <, >= 등)를 안다
  • 비교 결과가 1(참) 또는 0(거짓)인 걸 안다

조건을 합치기

점수가 90 이상이고 출석률이 80% 이상이면 A등급 — 이렇게 두 조건을 동시에 확인하고 싶을 때가 있어요!

논리 연산자는 참/거짓 값을 조합해요. 비교 연산자와 짝꿍이에요!

논리 연산자 3종

1
&& (AND, 그리고)
두 조건이 모두 참일 때만 참
2
|| (OR, 또는)
두 조건 중 하나라도 참이면 참
3
! (NOT, 아니다)
참을 거짓으로, 거짓을 참으로 뒤집기
undefined
&&는 키보드에서 Shift+7을 두 번, ||는 Shift+\를 두 번 눌러요. !는 Shift+1이에요.

진리표

진리표(truth table)는 모든 입력 조합에 대한 결과를 정리한 표예요. 논리 연산을 이해하는 가장 확실한 방법이에요!

&& (AND) 진리표
ABA && B
1 (참)1 (참)1 (참)
1 (참)0 (거짓)0 (거짓)
0 (거짓)1 (참)0 (거짓)
0 (거짓)0 (거짓)0 (거짓)
AND는 엄격한 선생님이에요. 두 조건 다 만족해야 '합격'! 하나라도 빠지면 '불합격'!
|| (OR) 진리표
ABA || B
111
101
011
000
OR은 너그러운 선생님이에요. 하나만 만족해도 '합격'! 둘 다 안 되면 '불합격'!
! (NOT) 진리표
A!A
1 (참)0 (거짓)
0 (거짓)1 (참)
logic.c
1#include <stdio.h>
2
3int main(void) {
4 int score = 85;
5 int attend = 90; // 출석률 %
6
7 // AND: 둘 다 참이어야 참
8 printf("A등급? %d\n", score >= 90 && attend >= 80);
9
10 // OR: 하나만 참이어도 참
11 printf("장학금? %d\n", score >= 95 || attend >= 95);
12
13 // NOT: 뒤집기
14 printf("불합격? %d\n", !(score >= 60));
15 return 0;
16}
실행 결과
A등급? 0 장학금? 0 불합격? 0
8줄score >= 90 && attend >= 80 — 85 >= 90은 거짓(0). AND에서 하나라도 거짓이면 결과는 0.
11줄score >= 95 || attend >= 95 — 둘 다 거짓이므로 OR도 거짓(0).
14줄!(score >= 60) — 85 >= 60은 참(1). NOT으로 뒤집으면 거짓(0).

범위 조건 만들기

수학에서 '60 ≤ score ≤ 100'이라고 쓰지만, C언어에서는 이렇게 써야 해요.

✕ 잘못된 코드
60 <= score <= 100
C에서 이렇게 쓰면 항상 참이 돼요! (수학 표기)
✓ 올바른 코드
score >= 60 && score <= 100
&&로 두 조건을 연결해야 해요
C언어에서 범위를 표현하려면 반드시 &&로 두 비교를 연결하세요!

실전 예제: 윤년 판별

윤년 조건은 논리 연산자의 좋은 실전 예제예요. '4의 배수이고, 100의 배수가 아니거나, 400의 배수'일 때 윤년이에요.

2

C에서의 참/거짓 (Truthiness)

C언어에서는 0만 거짓이고, 0이 아닌 모든 값은 참이에요. 이걸 'truthiness'라고 불러요.

truthiness.c
1#include <stdio.h>
2
3int main(void) {
4 // 0이 아닌 모든 값은 참!
5 printf("!0 = %d\n", !0); // 1 (0은 거짓 → 뒤집으면 참)
6 printf("!1 = %d\n", !1); // 0 (1은 참 → 뒤집으면 거짓)
7 printf("!42 = %d\n", !42); // 0 (42도 참!)
8 printf("!(-1) = %d\n", !(-1)); // 0 (-1도 참!)
9
10 // 논리 연산에서의 truthiness
11 printf("5 && 3 = %d\n", 5 && 3); // 1 (둘 다 0이 아님 = 둘 다 참)
12 printf("5 || 0 = %d\n", 5 || 0); // 1 (5가 0이 아님 = 참)
13 printf("0 && 100 = %d\n", 0 && 100); // 0 (0은 거짓)
14 return 0;
15}
실행 결과
!0 = 1 !1 = 0 !42 = 0 !(-1) = 0 5 && 3 = 1 5 || 0 = 1 0 && 100 = 0
[핵심] C의 참/거짓 규칙
거짓: 0, 0.0, '\0', NULL
참: 그 외 모든 값 (-1, 42, 3.14, 'A', ...)

논리 연산자의 결과는 항상 0 또는 1이에요.
(입력은 아무 정수나 가능하지만, 출력은 0 아니면 1)

NULL 체크 패턴

포인터가 NULL인지 검사할 때 이 truthiness를 활용해요.

null_check.c
1#include <stdio.h>
2#include <stdlib.h>
3
4int main(void) {
5 int *ptr = malloc(sizeof(int));
6
7 // 긴 버전
8 if (ptr != NULL) {
9 printf("할당 성공!\n");
10 }
11
12 // 짧은 버전 (같은 의미)
13 if (ptr) { // ptr이 NULL(0)이 아니면 참
14 printf("할당 성공!\n");
15 }
16
17 // NOT으로 NULL 체크
18 if (!ptr) { // ptr이 NULL(0)이면 참
19 printf("할당 실패!\n");
20 }
21
22 free(ptr);
23 return 0;
24}

단축 평가 (Short-circuit Evaluation)

논리 연산자의 가장 중요한 특성이에요! &&와 ||는 왼쪽부터 평가하고, 결과가 이미 결정되면 오른쪽은 아예 실행하지 않아요!

단축 평가(short-circuit evaluation)란, 논리 연산의 결과가 왼쪽 피연산자만으로 결정되면 오른쪽 피연산자를 평가하지 않는 것을 말해요.

단축 평가 규칙
연산자왼쪽 값오른쪽 평가?이유
&&거짓(0)안 함 ❌하나가 거짓이면 결과는 무조건 거짓
&&참(≠0)함 ✅오른쪽도 봐야 결과 알 수 있음
||참(≠0)안 함 ❌하나가 참이면 결과는 무조건 참
||거짓(0)함 ✅오른쪽도 봐야 결과 알 수 있음
short_circuit.c
1#include <stdio.h>
2
3int check(int x) {
4 printf("check(%d) 호출!\n", x);
5 return x;
6}
7
8int main(void) {
9 printf("--- AND 단축 평가 ---\n");
10 int r1 = check(0) && check(1); // check(0)이 거짓 → check(1) 안 부름!
11 printf("결과: %d\n\n", r1);
12
13 printf("--- OR 단축 평가 ---\n");
14 int r2 = check(1) || check(0); // check(1)이 참 → check(0) 안 부름!
15 printf("결과: %d\n", r2);
16 return 0;
17}
실행 결과
--- AND 단축 평가 --- check(0) 호출! 결과: 0 --- OR 단축 평가 --- check(1) 호출! 결과: 1
[핵심] 단축 평가의 실전 활용
단축 평가는 안전한 코드를 작성하는 데 핵심이에요!

1. ptr != NULL && ptr->value > 0
→ ptr이 NULL이면 뒤의 ptr->value는 실행 안 됨 (크래시 방지!)

2. n != 0 && total / n > 10
→ n이 0이면 나눗셈 안 함 (0으로 나누기 방지!)

조건부 실행 패턴

단축 평가를 이용한 고급 패턴들이에요. 실무에서 자주 볼 수 있어요.

conditional_patterns.c
1#include <stdio.h>
2#include <stdlib.h>
3
4int main(void) {
5 // 패턴 1: 포인터 안전 접근
6 int *ptr = NULL;
7 int val = (ptr != NULL) && (*ptr > 10); // ptr이 NULL이면 *ptr 실행 안 됨
8 printf("val = %d\n", val); // 0 (안전하게 0 반환)
9
10 // 패턴 2: 기본값 패턴 (|| 활용)
11 int user_input = 0; // 0이면 기본값 사용
12 int value = user_input || 42; // user_input이 0이면 42
13 printf("value = %d\n", value); // 1 (C에서는 0||42 = 1)
14
15 // 주의: C에서 ||의 결과는 항상 0 또는 1이에요!
16 // Python의 'a or b'처럼 값을 반환하지 않아요.
17 return 0;
18}
[경고] 단축 평가와 부작용(Side Effect)
단축 평가 때문에 오른쪽 표현식이 실행되지 않을 수 있어요!

int x = 0;
0 && (x = 5); ← x = 5는 실행 안 됨! x는 여전히 0
1 || (x = 5); ← x = 5는 실행 안 됨! x는 여전히 0

논리 연산자 안에서 대입이나 함수 호출 같은 부작용을 쓸 때 주의하세요!

드모르간 법칙 (De Morgan's Law)

복잡한 조건식을 깔끔하게 변환하는 마법 같은 법칙이에요!

드모르간 법칙은 NOT이 AND/OR을 통과할 때 AND와 OR이 서로 바뀌는 법칙이에요.

[핵심] 드모르간 법칙
법칙 1: !(A && B) == (!A || !B)
법칙 2: !(A || B) == (!A && !B)

쉽게 외우기: NOT이 괄호 안으로 들어가면 AND↔OR이 바뀐다!
드모르간 법칙 검증 — !(A && B) == (!A || !B)
ABA && B!(A && B)!A!B!A || !B
1110000
1001011
0101101
0001111

!(A && B)와 (!A || !B)의 결과가 완전히 같아요! 이것이 드모르간 법칙이에요.

드모르간 법칙의 코드 리팩토링 활용

demorgan_refactor.c
1#include <stdio.h>
2
3int main(void) {
4 int age = 25;
5 int has_ticket = 1;
6
7 // 복잡한 조건: 입장 불가 조건
8 // "성인이 아니거나 티켓이 없으면 입장 불가"
9 // 원본 (이해하기 어려움)
10 if (!(age >= 18 && has_ticket)) {
11 printf("입장 불가\n");
12 }
13
14 // 드모르간 법칙 적용 (더 읽기 쉬움!)
15 if (age < 18 || !has_ticket) {
16 printf("입장 불가\n");
17 }
18 // !(age >= 18) → age < 18
19 // !has_ticket → !has_ticket
20 // && → || (드모르간!)
21 return 0;
22}
[팁] 드모르간 법칙 실전 활용
코드를 읽기 쉽게 만들 때 자주 사용해요:

!(a < 10 && b > 5)
→ a >= 10 || b <= 5

NOT을 안으로 넣으면 조건이 뒤집히고, &&↔|| 교환!
비교 연산자도 뒤집어야 해요: < ↔ >=, > ↔ <=, == ↔ !=

[중급] 논리 연산 vs 비트 연산

C에는 논리 연산자(&&, ||, !)와 비트 연산자(&, |, ~)가 있어요. 비슷해 보이지만 완전히 다른 연산이에요!

논리 연산 vs 비트 연산 비교
구분논리 연산비트 연산
AND&&&
OR|||
NOT!~
피연산자참/거짓(정수)비트 단위
결과0 또는 1비트별 연산 결과
단축 평가있음 ✅없음 ❌
용도조건 판단비트 조작
logic_vs_bitwise.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 5; // 0101 (2진수)
5 int b = 3; // 0011 (2진수)
6
7 // 논리 연산: 값 전체를 참/거짓으로 판단
8 printf("5 && 3 = %d\n", a && b); // 1 (둘 다 0이 아님 = 참)
9 printf("5 || 3 = %d\n", a || b); // 1
10 printf("!5 = %d\n", !a); // 0
11
12 // 비트 연산: 각 비트별로 연산
13 printf("5 & 3 = %d\n", a & b); // 1 (0101 & 0011 = 0001)
14 printf("5 | 3 = %d\n", a | b); // 7 (0101 | 0011 = 0111)
15 printf("~5 = %d\n", ~a); // -6 (비트 전체 반전)
16 return 0;
17}
실행 결과
5 && 3 = 1 5 || 3 = 1 !5 = 0 5 & 3 = 1 5 | 3 = 7 ~5 = -6
[경고] &&와 &를 헷갈리면 큰 버그!
if (a & b)는 논리 연산이 아니라 비트 AND예요!
a = 2 (10), b = 4 (100)이면:
• a && b = 1 (둘 다 0이 아님 → 참)
• a & b = 0 (10 & 100 = 000 → 거짓!)

조건 판단에는 반드시 &&, ||를 쓰세요!

[중급] 불 대수 (Boolean Algebra)

논리 연산에도 수학의 규칙이 있어요. 이걸 불 대수라고 해요. 조지 불(George Boole)이라는 수학자가 만들었어요!
불 대수의 주요 법칙
법칙AND 형태OR 형태
항등법칙A && 1 = AA || 0 = A
영원법칙A && 0 = 0A || 1 = 1
멱등법칙A && A = AA || A = A
보수법칙A && !A = 0A || !A = 1
교환법칙A && B = B && AA || B = B || A
결합법칙(A && B) && C = A && (B && C)(A || B) || C = A || (B || C)
분배법칙A && (B || C) = (A&&B) || (A&&C)A || (B && C) = (A||B) && (A||C)
흡수법칙A && (A || B) = AA || (A && B) = A
드모르간!(A && B) = !A || !B!(A || B) = !A && !B

이 법칙들을 사용하면 복잡한 조건식을 간단하게 줄일 수 있어요!

boolean_simplify.c
1#include <stdio.h>
2
3int main(void) {
4 int A = 1, B = 0;
5
6 // 흡수법칙 예시: A || (A && B) = A
7 printf("%d == %d\n", A || (A && B), A); // 1 == 1 ✓
8
9 // 보수법칙 예시: A && !A = 0
10 printf("%d\n", A && !A); // 0
11
12 // 영원법칙 예시: A || 1 = 1
13 printf("%d\n", A || 1); // 1
14
15 // 코드 간소화 예시
16 // 원본: (x > 0 && y > 0) || (x > 0 && y <= 0)
17 // 분배법칙 역: x > 0 && (y > 0 || y <= 0)
18 // 보수법칙: x > 0 && 1
19 // 항등법칙: x > 0
20 int x = 5, y = -3;
21 printf("원본: %d\n", (x > 0 && y > 0) || (x > 0 && y <= 0));
22 printf("간소화: %d\n", x > 0); // 같은 결과!
23 return 0;
24}

[심화] 카르노 맵 (Karnaugh Map)

카르노 맵(K-map)은 불 대수 식을 시각적으로 간소화하는 도구예요. 진리표를 2차원 격자에 배치하면, 인접한 1들을 묶어서 간단한 식을 만들 수 있어요.

[핵심] 2변수 카르노 맵
함수 F(A,B) = !A&&!B || A&&!B || A&&B를 간소화해 봐요.

B=0 B=1
A=0 | 1 | 0 |
A=1 | 1 | 1 |

1단계: A=1인 행 전체가 1 → A
2단계: B=0인 열 전체가 1 → !B
결과: F = A || !B

3개의 항이 2개로 줄었어요!

[심화] 3변수 카르노 맵

변수가 3개일 때도 같은 원리예요. 다만 열이 그레이 코드 순서(00, 01, 11, 10)로 배치돼요.

[핵심] 3변수 카르노 맵 예시
F(A,B,C) = Σm(0,1,2,3,7) 간소화:

BC=00 BC=01 BC=11 BC=10
A=0 | 1 | 1 | 1 | 1 |
A=1 | 0 | 0 | 1 | 0 |

묶기 1: A=0 행 전체 (4개 묶음) → !A
묶기 2: BC=11 열 전체 (2개 묶음) → B&&C
결과: F = !A || (B && C)

[심화] 논리 게이트 (Logic Gates)

컴퓨터의 모든 연산은 논리 게이트라는 아주 작은 전자 회로로 만들어져요! 논리 연산자가 물리적으로 구현된 거예요!

논리 게이트는 하나 이상의 입력 신호를 받아서 하나의 출력 신호를 만드는 전자 회로예요. 트랜지스터로 만들어져요.

기본 논리 게이트 정리

[역사] NAND 게이트의 마법
NAND 게이트 하나만으로 모든 논리 회로를 만들 수 있어요! 이걸 '기능적 완전성(functional completeness)'이라고 해요.

NOT A = A NAND A
A AND B = (A NAND B) NAND (A NAND B)
A OR B = (A NAND A) NAND (B NAND B)

실제 CPU 칩에서도 NAND 게이트를 기본 단위로 많이 사용해요.

2
3int main(void) {

5 int a = 1, b = 0;

17 int x = 10, y = 20;

19 y ^= x; // y = y ^ (x ^ y) = x
20 x ^= y; // x = (x ^ y) ^ x = y
21 printf("x=%d, y=%d\n", x, y); // x=20, y=10

23}

[심화] 반가산기와 전가산기

논리 게이트를 조합하면 덧셈을 하는 회로를 만들 수 있어요! 이게 CPU에서 실제로 사용되는 가산기(adder)예요.

[핵심] 반가산기 (Half Adder)
1비트 두 개를 더하는 가장 간단한 회로:

입력: A, B
출력: Sum(합) = A XOR B, Carry(올림) = A AND B

예: 1 + 1 = 10 (2진수)
Sum = 1 XOR 1 = 0
Carry = 1 AND 1 = 1
결과: 10 (올림 1, 합 0) ✓
반가산기 진리표
ABSum (A⊕B)Carry (A·B)
0000
0110
1010
1101
[핵심] 전가산기 (Full Adder)
이전 자리의 올림(Cin)까지 고려하는 가산기:

입력: A, B, Cin(이전 올림)
출력:
• Sum = A ⊕ B ⊕ Cin
• Cout = (A · B) + (Cin · (A ⊕ B))

전가산기를 n개 연결하면 n비트 덧셈기가 돼요!

[심화 내용] 논리 연산의 결합법칙과 연산자 오버로딩

C++에서는 &&와 ||를 클래스에 대해 오버로딩할 수 있지만, 이때 단축 평가가 깨져요! 이것은 C++의 유명한 함정이에요.

[C++ 경고] 연산자 오버로딩과 단축 평가
C++에서 &&, ||를 오버로딩하면:
• 단축 평가가 동작하지 않음
• 양쪽 피연산자가 모두 평가됨
• 평가 순서도 보장되지 않음

이 때문에 C++ 코딩 가이드라인에서는 &&, ||의 오버로딩을 금지하는 경우가 많아요.

[심화 내용] 술어 논리와 한정사

불 대수를 확장한 술어 논리(Predicate Logic)에서는 '모든(∀)'과 '존재(∃)' 한정사가 추가돼요.

predicate_logic.c
1#include <stdio.h>
2
3int main(void) {
4 int arr[] = {2, 4, 6, 8, 10};
5 int n = 5;
6
7 // ∀x ∈ arr: x는 짝수 (모든 원소가 짝수인가?)
8 int all_even = 1;
9 for (int i = 0; i < n; i++) {
10 all_even = all_even && (arr[i] % 2 == 0);
11 }
12 printf("모두 짝수? %d\n", all_even); // 1
13
14 // ∃x ∈ arr: x > 7 (7보다 큰 원소가 존재하는가?)
15 int exists_gt7 = 0;
16 for (int i = 0; i < n; i++) {
17 exists_gt7 = exists_gt7 || (arr[i] > 7);
18 }
19 printf("7 초과 존재? %d\n", exists_gt7); // 1
20 return 0;
21}
[수학↔코드] 한정사와 논리 연산
∀(모든) → 전부 AND로 연결: a[0]이 짝수 && a[1]이 짝수 && ...
∃(존재) → 전부 OR로 연결: a[0]>7 || a[1]>7 || ...

드모르간 법칙의 확장:
¬(∀x P(x)) = ∃x ¬P(x)
'모두 참이 아니다' = '거짓인 것이 존재한다'

¬(∃x P(x)) = ∀x ¬P(x)
'존재하지 않는다' = '모두 거짓이다'

용어 정리

&& (AND)
논리곱. 두 조건이 모두 참일 때만 참.
|| (OR)
논리합. 두 조건 중 하나라도 참이면 참.
! (NOT)
논리 부정. 참을 거짓으로, 거짓을 참으로.
진리표
논리 연산의 모든 입력 조합과 결과를 정리한 표.
복합 조건
여러 비교를 논리 연산자로 연결한 조건식.
단축 평가
논리 연산에서 결과가 이미 결정되면 나머지를 평가하지 않는 것.
드모르간 법칙
NOT이 AND/OR을 통과할 때 서로 바뀌는 법칙. !(A&&B)=!A||!B
불 대수
참/거짓(0/1)에 대한 수학적 연산 체계. 조지 불이 고안.
카르노 맵
불 대수 식을 시각적으로 간소화하는 도구.
논리 게이트
논리 연산을 수행하는 전자 회로 (AND, OR, NOT, NAND 등).
XOR
배타적 OR. 두 입력이 다르면 참, 같으면 거짓.
반가산기
1비트 두 개를 더하는 회로. Sum=XOR, Carry=AND.
전가산기
올림(carry)까지 포함하여 1비트를 더하는 회로.
Truthiness
C에서 0이 아닌 모든 값을 참으로 취급하는 성질.
