코드도 판단을 할 수 있을까?

Part 3에 오신 걸 환영해요! 이제부터는 컴퓨터가 스스로 판단하는 법을 배울 거예요. 제어문의 세계에 들어가 봐요!

지금까지 만든 프로그램은 위에서 아래로 무조건 다 실행됐어요. 한 줄도 빠짐없이 순서대로요.

그런데 진짜 프로그램은 상황에 따라 다르게 행동해야 해요. 점수가 90점 이상이면 "합격"을 출력하고, 아니면 아무것도 안 하는 프로그램 — 이럴 때 if문을 써요.

if문의 흐름
조건이 참이면 → 코드 실행!
조건이 거짓이면 → 건너뜀
[코딩 습관] if 뒤에 중괄호 습관
if문 안에 코드가 한 줄이어도 항상 중괄호 { }를 써요. 나중에 코드를 추가할 때 실수를 막아줘요. 프로 개발자들도 지키는 규칙이에요.

if문의 생김새

if_shape.c
1if (조건) {
2 // 조건이 참일 때 실행되는 코드
3}
괄호 ( ) 안에 조건을 넣고, 중괄호 { } 안에 실행할 코드를 넣어요. 조건이 참이면 실행, 거짓이면 통째로 건너뛰어요.

자주 하는 실수 TOP 3

if문을 처음 쓸 때 자주 하는 실수 3가지! 미리 알아두면 에러를 금방 고칠 수 있어요.

[사고력] 순차에서 조건 분기로

이것이 알고리즘 설계의 핵심 전환점이에요!

순서도(Flowchart)의 마름모:
마름모 = 판단 = if문
마름모에서 '예/아니오' 두 길이 갈라져요.

if문을 쓴다는 건 '컴퓨터에게 판단 능력을 주는 것'이에요.
이 능력이 없으면 계산기일 뿐, 프로그램이 아니에요.

컴퓨팅 사고력 4요소와 if문의 관계를 알아봐요.

[사고력] 4요소 연결
1. 분해(Decomposition):
'합격인지 판별하라'를 → '점수를 받는다 / 90 이상인지 비교한다 / 결과를 출력한다'로 나누기

2. 패턴 인식(Pattern Recognition):
온도 판별, 나이 판별, 점수 판별 — 전부 '값을 기준과 비교하는' 같은 패턴!
if (값 비교연산자 기준) { 처리 } ← 이 패턴이 반복돼요.

3. 추상화(Abstraction):
'19세 이상이면 성인'에서 핵심은 '나이 >= 기준'이에요.
이름, 키, 성별 같은 세부사항은 버리고 핵심 조건만 추출하는 게 추상화!

4. 알고리즘 설계(Algorithm Design):
순서도의 마름모 → if문. 입력→판단→출력 흐름을 설계하는 것.
if문을 쓸 때마다 4요소를 연습하는 거예요! 문제를 나누고(분해), 패턴을 찾고(인식), 핵심만 뽑고(추상화), 절차를 만들고(설계)!

문제 (백준 9498):
시험 점수를 입력받아 성적을 출력하라.
90~100: A / 80~89: B / 70~79: C / 60~69: D / 나머지: F

입력: 85
출력: B

풀이 전략:

[디버깅] if문 버그 시나리오

흔한 버그 시나리오 1
버그: 90점인데 합격이 출력 안 됨
if (score > 90) printf("합격");

원인: > 를 쓰면 90은 포함 안 됨! 90 > 90은 거짓
해결: >= 를 써야 90도 포함
if (score >= 90) printf("합격");

교훈: 경계값(90점 딱)으로 항상 테스트하라!
흔한 버그 시나리오 2
버그: 조건에 상관없이 항상 "합격" 출력됨
if (score = 90)

원인: = 는 대입! score에 90을 넣고, 90은 0이 아니라 참!
해결: ==를 써야 비교
if (score == 90)

교훈: '비교하려고 했는데 대입이 됐다' — C 초보의 TOP 1 버그!
방어: if (90 == score) (Yoda 조건)
또는: -Wall 컴파일 옵션
흔한 버그 시나리오 3
버그: if 안에 2줄 넣었는데 두 번째 줄이 항상 실행됨
if (x > 0)
  printf("양수");
  printf("확실합니다");

원인: 중괄호 없으면 if는 바로 다음 한 줄만 묶음!
'확실합니다'는 if 바깥으로 인식됨
해결: 중괄호로 감싸기
if (x > 0) {
  printf("양수");
  printf("확실합니다");
}

교훈: 무조건 중괄호! Apple의 goto fail 버그(2014)도 이 문제!
Apple의 SSL 보안 버그(goto fail)도 중괄호를 안 써서 생긴 거예요! if 뒤에 중괄호는 생명이에요!

용어 정리

[심화] if문의 CPU 내부 동작

[심화 내용] CPU가 if를 처리하는 방법
CPU 수준에서 if는:
1. 조건 평가 → 비교 명령어 (CMP)
2. 플래그 레지스터 설정 (Zero, Sign, Carry)
3. 조건 점프 (JZ, JNZ, JG, JL...)

어셈블리로 보면:
if (x >= 90) →
  CMP eax, 90     ; x와 90 비교
  JL skip_block   ; x < 90이면 건너뜀
  ; printf 호출    ; 참인 경우 실행
skip_block:

분기 예측 (Branch Prediction):
CPU가 if 결과를 '예측'해서 미리 실행!
맞으면 → 파이프라인 유지 (빠름)
틀리면 → 파이프라인 flush (~15 사이클 손실)
현대 CPU 예측 정확도: 90~97%
[C 코드] 조건에 따른 어셈블리 확인
1#include <stdio.h>
2
3// gcc -S -O0 -masm=intel 로 어셈블리 확인 가능
4int check_pass(int score) {
5 if (score >= 90) {
6 return 1;
7 }
8 return 0;
9}
10
11int main() {
12 int scores[] = {95, 70, 90, 45, 100};
13 int n = sizeof(scores) / sizeof(scores[0]);
14
15 for (int i = 0; i < n; i++) {
16 printf("점수 %3d → %s\n",
17 scores[i],
18 check_pass(scores[i]) ? "합격" : "불합격");
19 }
20 return 0;
21}

점수 95 → 합격 점수 70 → 불합격 점수 90 → 합격 점수 45 → 불합격 점수 100 → 합격
분기 예측
CPU가 if 결과를 미리 예측. 현대 CPU 정확도 90~97%. 틀리면 파이프라인 flush로 ~15 사이클 손실.
CMP 명령어
x86 비교 명령. CMP eax, 90은 eax-90을 계산하되 결과 버리고 플래그만 설정. JGE/JL이 플래그를 읽음.
CPU는 if를 만나면 '아마 참이겠지' 하고 미리 실행해! 이게 분기 예측. 틀리면 되돌려야 해서 느려져!

[심화] 가드 절과 입력 검증 패턴

[심화 내용] 가드 절 (Guard Clause)
가드 절: 함수 시작에서 잘못된 입력을 먼저 걸러내는 패턴

Linux 커널 코딩 스타일도 가드 절 권장

[심화] 상태 머신과 조건문

[심화 내용] 유한 상태 머신 (FSM)
상태 머신 = 상태 + 입력 → 다음 상태

자판기: 대기 →[동전]→ 투입됨 →[버튼]→ 배출 중 →[완료]→ 대기
게임 캐릭터: IDLE →[공격키]→ ATTACK →[시간]→ IDLE

if문으로 상태 전이를 구현!
게임 AI, 네트워크 프로토콜, UI 흐름 전부 이 구조
[C 코드] if로 만드는 상태 머신
1#include <stdio.h>
2
3typedef enum { WAITING, COIN_IN, DISPENSING } State;
4
5const char* state_name(State s) {
6 if (s == WAITING) return "대기";
7 if (s == COIN_IN) return "동전 투입됨";
8 if (s == DISPENSING) return "배출 중";
9 return "알 수 없음";
10}
11
12State next_state(State current, char input) {
13 if (current == WAITING && input == 'c') return COIN_IN;
14 if (current == COIN_IN && input == 'b') return DISPENSING;
15 if (current == COIN_IN && input == 'r') return WAITING;
16 if (current == DISPENSING) return WAITING;
17 return current;
18}
19
20int main() {
21 State s = WAITING;
22 const char *inputs = "cbrccb";
23
24 printf("시작: [%s]\n", state_name(s));
25 for (int i = 0; inputs[i]; i++) {
26 s = next_state(s, inputs[i]);
27 printf("입력 '%c' → [%s]\n", inputs[i], state_name(s));
28 }
29 return 0;
30}
실행 결과
시작: [대기] 입력 'c' → [동전 투입됨] 입력 'b' → [배출 중] 입력 'r' → [대기] 입력 'c' → [동전 투입됨] 입력 'c' → [동전 투입됨] 입력 'b' → [배출 중]
상태 머신
FSM (Finite State Machine). 상태+입력→다음 상태. 게임 AI, 프로토콜, UI, 파서에 핵심.

[심화] 분기 없는 조건 처리

[심화 내용] Branchless Programming
분기(if)를 산술로 대체해 파이프라인 유지

절대값: int m = x >> 31; x = (x + m) ^ m;
최솟값: min = (a < b) ? a : b; → CMOVcc 변환
클램프: x = x < 0 ? 0 : (x > 255 ? 255 : x);

게임 물리, 그래픽, 시그널 처리에서 활용
[C 코드] 삼항 연산자와 branchless
1#include <stdio.h>
2
3int abs_branchless(int x) {
4 int mask = x >> 31;
5 return (x + mask) ^ mask;
6}
7
8int main() {
9 printf("abs(-42) = %d\n", abs_branchless(-42));
10
11 int a = 10, b = 20;
12 int min = (a < b) ? a : b; // CMOVcc
13 int max = (a > b) ? a : b;
14 printf("min=%d, max=%d\n", min, max);
15
16 // 클램프
17 int val = 300;
18 int clamped = val < 0 ? 0 : (val > 255 ? 255 : val);
19 printf("clamp(300, 0~255) = %d\n", clamped);
20 return 0;
21}
실행 결과
abs(-42) = 42 min=10, max=20 clamp(300, 0~255) = 255
cmov
x86 조건부 이동 명령. if 없이 조건 대입. 삼항 연산자를 cmov로 변환. 파이프라인 유지.

[심화] if문의 함정과 방어

[심화 내용] if문의 5대 함정
1. = vs ==: if(x=5) 항상 참! -Wall로 경고
2. dangling else: if(a) if(b) x; else y; → else는 안쪽 if에 연결
3. 세미콜론: if(x>0); ← 빈 문장!
4. 실수 비교: if(f==0.1) 오차! fabs(f-0.1)<1e-9
5. truthy/falsy: C에서 0=false, 나머지=true
[C 코드] if 함정 방어 패턴
1#include <stdio.h>
2#include <math.h>
3
4int main() {
5 // Yoda 조건 — 대입 실수 방지
6 int x = 5;
7 if (5 == x) printf("Yoda: x는 5\n");
8
9 // 실수 비교 — epsilon
10 double f = 0.1 + 0.2;
11 if (fabs(f - 0.3) < 1e-9) printf("같음 (epsilon)\n");
12
13 // truthy/falsy
14 if (42) printf("42는 참\n");
15 if (!0) printf("0은 거짓\n");
16 return 0;
17}
실행 결과
Yoda: x는 5 같음 (epsilon) 42는 참 0은 거짓
Yoda 조건
if(5==x) 스타일. 상수를 왼쪽에 두면 =실수 시 컴파일 에러 발생. Star Wars 요다처럼 거꾸로.
dangling else
if 중첩 시 else 연결 모호. C에서는 가장 가까운 if에 연결. 중괄호로 명확히!
epsilon 비교
부동소수점은 == 비교 불가! 0.1+0.2!=0.3. fabs(a-b)<1e-9로 비교. IEEE 754 오차.
goto fail
Apple의 2014년 SSL 보안 버그. 중괄호 없는 if에 줄이 하나 더 붙어서 보안 검증 우회됨!
truthy/falsy
C에서 0=false, 나머지=true. NULL=false. if(ptr)는 if(ptr!=NULL)과 동일.
룩업 테이블
배열로 if 체인 대체. O(n)→O(1). 값이 연속 정수면 배열 인덱스로!
McCabe 복잡도
순환 복잡도. if/for/while마다 +1. 10 이하 권장. 20 초과면 리팩토링 필요!
-Wall
GCC 경고 옵션. =vs== 실수, 미초기화 변수 등 감지. -Wall -Wextra가 안전한 C 코딩의 기본.
short-circuit
&&: 왼쪽 false면 오른쪽 평가 안 함. ||: 왼쪽 true면 오른쪽 평가 안 함.
dead code
절대 실행 안 되는 코드. if(0){...} 같은 패턴. -Wunreachable-code로 감지.
0.1 + 0.2 != 0.3! 부동소수점의 함정. 실수 비교는 반드시 epsilon을 써야 해!
[활동] if문 마스터 체크리스트
if 기본 문법을 안다
비교 연산자 6종을 안다
= vs ==의 차이를 안다
경계값 테스트를 할 수 있다
가드 절 패턴을 안다
분기 예측의 개념을 안다
상태 머신을 if로 구현할 수 있다
KOI 기초 문제(두 수 비교, 윤년)를 풀 수 있다

6개 이상이면 if문 마스터!
if문은 프로그래밍의 심장! 조건 분기가 없으면 계산기일 뿐이야. 여기서부터 진짜 프로그래밍이 시작돼!
if문
조건이 참일 때만 코드를 실행하는 제어문. if (조건) { 코드 }
조건(Condition)
참 또는 거짓으로 판단되는 식. 비교 연산자로 만들어요.
제어문(Control Statement)
코드 실행 순서를 바꾸는 문장. if, for, while 등이 있어요.
블록(Block)
중괄호 { }로 묶인 코드 덩어리.
비교 연산자
두 값을 비교. ==, !=, <, >, <=, >= 6가지.
분기(Branch)
프로그램 흐름이 갈라지는 것. if문은 대표적인 분기 구조.
