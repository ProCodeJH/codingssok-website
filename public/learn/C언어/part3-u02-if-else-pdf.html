거짓일 때도 뭔가 하고 싶다면?

지난 유닛에서 if문을 배웠죠? if는 참일 때만 실행하고 거짓이면 그냥 넘어갔어요. 오늘은 거짓일 때도 다른 코드를 실행하는 방법을 배워요!

"점수가 90 이상이면 합격, 아니면 불합격" — 이렇게 두 가지 경우를 모두 처리하고 싶을 때 if-else를 써요.

if-else의 흐름
조건이 참이면 → if 블록 실행
조건이 거짓이면 → else 블록 실행
둘 중 하나만 실행됨!

if-else의 생김새

if_else_shape.c
1if (조건) {
2 // 조건이 참일 때 실행
3} else {
4 // 조건이 거짓일 때 실행
5}
else는 '그 외에', '아니면'이라는 뜻이에요. if의 조건이 거짓이면 else 블록이 실행돼요. 둘 중 하나만 실행된다는 게 핵심!
[코딩 습관] else도 중괄호로
else 뒤에도 항상 중괄호 { }를 써요. 코드가 한 줄이어도 중괄호를 쓰면 나중에 줄을 추가할 때 안전해요.

if 두 개 vs if-else — 차이가 뭘까?

지난 유닛에서 if 두 개로 성인/미성년을 판단했어요. if-else로 바꾸면 어떤 점이 좋을까요?

✕ 잘못된 코드
if (age >= 19) { printf("성인\n"); } if (age < 19) { printf("미성년\n"); }
if 두 개 — 조건을 두 번 쓰고 두 번 검사!
✓ 올바른 코드
if (age >= 19) { printf("성인\n"); } else { printf("미성년\n"); }
if-else — 조건 한 번, 깔끔하고 안전!
[핵심] 왜 if-else가 더 좋은가?
1. 조건 일관성: if 두 개는 하나만 고치면 논리 깨짐. if-else는 조건 하나로 통제.
2. 성능: if 두 개는 조건을 2번 검사. if-else는 1번만 검사.
3. 상호 배타 보장: if-else는 반드시 하나만 실행. if 두 개는 둘 다 실행될 수도!

예외: 조건이 서로 독립적이면 if 두 개가 맞아요 (예: 온도와 습도를 각각 체크)

실전 예제: 짝수/홀수 판별

even_odd.c
1#include <stdio.h>
2
3int main(void) {
4 int num;
5 printf("정수를 입력하세요: ");
6 scanf("%d", &num);
7
8 if (num % 2 == 0) {
9 printf("%d은(는) 짝수예요.\n", num);
10 } else {
11 printf("%d은(는) 홀수예요.\n", num);
12 }
13 return 0;
14}
실행 결과
정수를 입력하세요: 7 7은(는) 홀수예요.
num % 2는 num을 2로 나눈 나머지. 나머지가 0이면 짝수, 아니면(else) 홀수! 완벽한 이진 분기예요.

음수이면 부호를 뒤집고, 양수이면 그대로 출력 — if-else의 교과서적 활용이에요.

컴퓨팅 사고력 4요소가 if-else에서 어떻게 나타나는지 봐요.

[사고력] 4요소 심화 연결
1. 분해(Decomposition):
'짝수/홀수 판별'을 → (1) 숫자 입력 (2) 2로 나머지 계산 (3) 나머지가 0이면 짝수, 아니면 홀수 (4) 결과 출력 — 4단계로 분해
각 단계가 코드 1~2줄에 대응!

2. 패턴 인식(Pattern Recognition):
짝수/홀수, 합격/불합격, 성인/미성년 — 전부 '이진 분류' 패턴!
공통 구조: if (조건) { A } else { B }
조건과 A, B만 바꾸면 어떤 이진 판별이든 가능!

3. 추상화(Abstraction):
'짝수 판별'의 핵심은 n % 2 == 0 이 한 줄.
입력 방법(키보드? 파일?), 출력 형식(한글? 영어?) 같은 세부사항은 핵심이 아니에요.

4. 알고리즘 설계(Algorithm Design):
순서도에서 마름모(판단)를 거치면 두 갈래로 나뉘어요.
이 구조가 반복되면 '의사결정 트리'가 되고, 이건 머신러닝의 기초이기도 해요.
if-else를 쓸 때마다 사고력 연습을 하는 거예요! 문제를 나누고(분해), 같은 패턴을 찾고(인식), 핵심만 뽑고(추상화), 절차를 설계(알고리즘)!

풀이 전략:

KOI 실전 — 백준 2884 알람 시계
문제: H시 M분에서 45분을 빼면 몇 시 몇 분?
입력: 0 30 → 출력: 23 45

풀이:
if (m >= 45) {
  printf("%d %d", h, m - 45);
} else {
  if (h == 0) h = 24;
  printf("%d %d", h - 1, m + 60 - 45);
}

핵심: 분이 45 미만이면 시간을 빌려와야 해요. 0시→23시 처리가 함정!
사고력: 경계값(0시 0분, 0시 30분)을 반드시 테스트!
대회 문제 풀이 순서: (1) 입력 확인 (2) 어떤 조건으로 나누지? (3) 경계값은? (4) 출력 형식 확인. 이 4단계를 습관으로!

[디버깅] if-else 버그 시나리오

흔한 버그 — else가 엉뚱한 if에 연결됨
버그: else가 바깥 if에 연결되길 원했는데 안쪽 if에 연결됨
if (a > 0)
  if (b > 0)
    printf("둘 다 양수");
else
  printf("a가 음수"); // 의도와 다름!

원인: dangling else — else는 가장 가까운 if에 연결!
실제로는 b<=0일 때 실행됨 (a가 음수일 때가 아님!)

해결: 중괄호로 명확하게
if (a > 0) {
  if (b > 0) printf("둘 다 양수");
} else {
  printf("a가 음수");
}

교훈: 중첩 if-else에서는 중괄호가 생명!
흔한 버그 — 중괄호 없이 여러 줄
버그: if 안에 2줄 넣었는데 두 번째 줄이 항상 실행됨
if (score >= 90)
  printf("합격!");
  printf("축하합니다!");

원인: 중괄호가 없으면 if는 바로 다음 한 줄만 제어!
'축하합니다!'는 if 바깥으로 인식 → 항상 실행됨

해결: 중괄호로 두 줄을 묶기
if (score >= 90) {
  printf("합격!");
  printf("축하합니다!");
}

교훈: Apple goto fail 버그(2014)도 이 문제! 중괄호 습관은 보안까지 지켜요.
흔한 버그 — 알람 시계 경계값
버그: 알람 시계 문제에서 0시 30분 입력 시 -1시 45분 출력
printf("%d %d", h-1, m+60-45);

원인: h가 0일 때 h-1은 -1! 0시→23시 변환 누락

해결: h==0이면 24로 바꾼 후 1 빼기
if (h == 0) h = 24;
printf("%d %d", h-1, m+60-45);

교훈: 시간 문제에서 0시, 12시, 자정 넘김은 반드시 경계값 테스트!

[컴퓨팅사고력 — 패턴 인식] 짝수/홀수, 합격/불합격, 양수/음수 — 전부 '이진 분류' 패턴이에요. if (조건) { A } else { B } 구조가 반복돼요. 이 패턴을 인식하면 새로운 이진 분류 문제를 만나도 즉시 if-else로 풀 수 있어요.

용어 정리

[심화] 삼항 연산자 — if-else를 한 줄로

[심화 내용] 삼항 연산자(Ternary Operator)
문법: 조건 ? 참값 : 거짓값

if-else 5줄을 1줄로:
if-else: if(x>=0) abs=x; else abs=-x;
삼항: abs = (x>=0) ? x : -x;

활용:
min = (amax = (a>b) ? a : b;
printf("%s\n", (age>=19) ? "성인" : "미성년");

주의:
중첩 삼항은 가독성 최악!
a ? b ? c : d : e ← 읽기 불가능!
→ 중첩이 필요하면 if-else를 쓰세요.

[심화] 이진 탐색 — if-else의 극한 활용

[심화 내용] 이진 탐색(Binary Search)
정렬된 배열에서 값 찾기 — O(log n)

100만 개에서 순차 탐색: 최대 100만 번
100만 개에서 이진 탐색: 최대 20번!

원리: 매번 if-else로 절반을 버림!
배열의 중간값과 비교
→ 크면 오른쪽 절반만 탐색
→ 작으면 왼쪽 절반만 탐색
→ 같으면 찾음!

KOI 필수 알고리즘 — 초등부부터 출제
[C 코드] 이진 탐색 구현
1#include <stdio.h>
2
3int binary_search(int arr[], int n, int target) {
4 int lo = 0, hi = n - 1;
5 while (lo <= hi) {
6 int mid = lo + (hi - lo) / 2;
7 if (arr[mid] == target) {
8 return mid;
9 } else {
10 if (arr[mid] < target) lo = mid + 1;
11 else hi = mid - 1;
12 }
13 }
14 return -1;
15}
16
17int main() {
18 int arr[] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};
19 int n = 10;
20 int targets[] = {23, 50, 2, 91};
21 for (int i = 0; i < 4; i++) {
22 int idx = binary_search(arr, n, targets[i]);
23 if (idx >= 0) printf("%d → arr[%d]\n", targets[i], idx);
24 else printf("%d → 없음\n", targets[i]);
25 }
26 return 0;
27}
실행 결과
23 → arr[5] 50 → 없음 2 → arr[0] 91 → arr[9]
이진 탐색
정렬 배열에서 O(log n) 탐색. 매번 if-else로 절반 제거. 100만 개도 20번이면 찾음. KOI 필수.
lo+(hi-lo)/2
(lo+hi)/2는 오버플로우 위험! Java의 Arrays.binarySearch에도 이 버그 있었음(2006).
이진 탐색 = if-else의 반복! 매번 절반을 버리니까 100만 개에서도 20번이면 찾아. KOI에서 가장 중요한 알고리즘!

[심화] 에러 처리와 반환 패턴

[심화 내용] C의 에러 처리 = if-else + 반환값
C에는 try-catch가 없어요!
대신 반환값으로 성공/실패를 구분해요.

성공: 0 또는 양수
실패: -1 또는 음수 또는 NULL

패턴:
int result = do_something();
if (result < 0) { 에러 처리 }
else { 정상 처리 }

FILE *fp = fopen("data.txt", "r");
if (fp == NULL) { perror("에러"); return 1; }
// 정상 사용...
fclose(fp);

이 패턴을 모르면 실전 C 코딩이 불가능해요!
[C 코드] 안전한 나눗셈 — 에러 처리 패턴
1#include <stdio.h>
2
3int safe_divide(int a, int b, int *result) {
4 if (b == 0) {
5 return -1; // 에러
6 } else {
7 *result = a / b;
8 return 0; // 성공
9 }
10}
11
12int main() {
13 int result;
14 if (safe_divide(10, 3, &result) == 0) {
15 printf("10/3 = %d\n", result);
16 } else {
17 printf("에러: 0으로 나눌 수 없음\n");
18 }
19
20 if (safe_divide(10, 0, &result) == 0) {
21 printf("10/0 = %d\n", result);
22 } else {
23 printf("에러: 0으로 나눌 수 없음\n");
24 }
25 return 0;
26}

[심화] 단축 평가(Short-circuit)

[심화 내용] 단축 평가 활용
&&: 왼쪽 거짓이면 오른쪽 평가 안 함!
if (ptr != NULL && *ptr > 0) → ptr이 NULL이면 *ptr 안 읽음! 크래시 방지!

||: 왼쪽 참이면 오른쪽 평가 안 함!
if (n == 0 || total/n > 100) → n이 0이면 나누기 안 함! 0 나누기 방지!

함정:
if (x++ > 0 || y++ > 0) → x가 참이면 y++는 실행 안 됨! 부수효과 주의!
단축 평가
Short-circuit. &&: 왼쪽 false→오른쪽 무시. ||: 왼쪽 true→오른쪽 무시. 안전한 포인터 역참조에 필수!
Segfault
NULL 포인터 역참조 시 발생. if(ptr && *ptr>0) 단축 평가로 방지. C 크래시 원인 1위.
if(ptr && *ptr>0) — 단축 평가의 핵심! ptr이 NULL이면 *ptr를 절대 안 읽어서 크래시 방지!
CMOVcc
x86 조건부 이동 명령. if-else 없이 조건 대입. 삼항→CMOVcc 최적화. 파이프라인 유지.
유클리드 호제법
GCD 알고리즘. gcd(a,b)=gcd(b,a%b). 2300년 전! if-else 기반 반복. KOI 필수.
의사결정 트리
Decision Tree. if-else 연속으로 분류. 머신러닝 핵심 알고리즘! 20 Questions도 같은 원리.
defensive programming
방어적 프로그래밍. 모든 입력을 의심하고 if-else로 검증. '신뢰하지 말고 검증하라'.
goto fail
Apple의 2014년 SSL 보안 버그. 중괄호 없는 if에 줄이 추가돼 보안 검증 우회됨. $수백만 피해.

else
if의 조건이 거짓일 때 실행되는 블록. '아니면'이라는 뜻.
if-else문
참이면 if 블록, 거짓이면 else 블록 실행. 둘 중 하나만 실행.
이진 분기(Binary Branch)
두 갈래로 나뉘는 구조. 모든 복잡한 판단은 이진 분기의 연속.
삼항 연산자
조건 ? 참값 : 거짓값. if-else 한 줄 축약형.
나머지 연산자(%)
나눗셈의 나머지. 7%2=1. 짝수/홀수 판별에 필수.
단축 평가(Short-circuit)
&&/||에서 결과 확정 시 나머지 조건 평가 생략.
