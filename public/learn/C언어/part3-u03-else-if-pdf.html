Lesson 3-3

세 갈래 이상 — else if

조건을 여러 개 연결해서 세 갈래 이상으로 나누는 법을 배워요.

📚 이번에 배울 것

  • else if로 여러 조건을 순서대로 검사할 수 있어요.
  • 조건 순서가 왜 중요한지 알아요.
  • if 여러 개와 else if 체인의 차이를 알아요.

✅ 시작 전 체크

  • if-else문을 사용할 수 있다
  • 비교 연산자를 안다
Part 1 진행률: 11%

두 갈래 말고 세 갈래 이상!

if-else는 두 갈래였죠? 오늘은 세 갈래, 네 갈래, 다섯 갈래까지! 성적을 A/B/C/D/F로 나누려면 else if가 필요해요!

시험 점수를 A(90~100), B(80~89), C(70~79), D(60~69), F(0~59)로 나누려면 두 갈래 if-else로는 부족해요. else if로 조건을 여러 개 연결하면 돼요.

else if의 흐름
조건1 참? → 예 → 블록1 실행
조건1 참? ↓ 아니오
조건2 참? → 예 → 블록2 실행
조건2 참? ↓ 아니오
else → 나머지 처리
위에서 아래로, 처음 참인 곳에서 실행 후 끝!

else if의 생김새

else_if_shape.c
1if (조건1) {
2 // 조건1이 참
3} else if (조건2) {
4 // 조건1 거짓, 조건2 참
5} else if (조건3) {
6 // 조건1,2 거짓, 조건3 참
7} else {
8 // 전부 거짓
9}
위에서 아래로 순서대로 검사해요. 처음으로 참인 조건의 블록만 실행하고 나머지는 전부 건너뛰어요! 이게 if 여러 개와의 결정적 차이!
[핵심] else if는 '그 외에 만약~'
else if = else + if의 축약

사실 이건 else 안에 if를 넣은 거예요:
} else {
  if (조건2) { ... }
}
이걸 한 줄로 줄이면:
} else if (조건2) { ... }

그래서 '이전 조건이 거짓일 때만' 다음 조건을 검사해요.

조건 순서가 매우 중요해요!

else if에서 조건 순서를 바꾸면 결과가 완전히 달라져요.

✕ 잘못된 코드
if (score >= 60) { printf("D등급"); } else if (score >= 70) { printf("C등급"); } else if (score >= 80) { printf("B등급"); } else if (score >= 90) { printf("A등급"); }
95점인데 D등급! 60 이상을 먼저 확인하니까 전부 D로 가요
✓ 올바른 코드
if (score >= 90) { printf("A등급"); } else if (score >= 80) { printf("B등급"); } else if (score >= 70) { printf("C등급"); } else if (score >= 60) { printf("D등급"); }
큰 수부터! 90→80→70→60 순서로 좁혀가요
[주의] 범위 검사는 큰 값부터!
숫자 범위를 else if로 나눌 때는 큰 값부터 확인해야 해요.
작은 값부터 확인하면 넓은 범위가 먼저 잡아버려서 나머지가 죽은 코드가 돼요.

규칙: if (>=90) → else if (>=80) → else if (>=70) → ...
이 순서를 '내림차순 조건 체인'이라고 불러요.

if 여러 개 vs else if — 결정적 차이

✕ 잘못된 코드
if (score >= 90) printf("A"); if (score >= 80) printf("B"); if (score >= 70) printf("C");
95점이면 A, B, C 전부 출력! 각 if가 독립적이라서
✓ 올바른 코드
if (score >= 90) printf("A"); else if (score >= 80) printf("B"); else if (score >= 70) printf("C");
95점이면 A만 출력! 참을 만나면 나머지 건너뜀
if 여러 개는 각각 독립적. else if 체인은 상호 배타적 — 하나만 실행! 이 차이를 꼭 기억하세요.

[컴퓨팅 사고력] 다중 분기와 분류 체계

[사고력] else if = 분류 체계의 코드화
분류(Classification)는 인간 사고의 기본!

도서관 — 책을 10진 분류법으로 나눔 (000 총류, 100 철학, 200 종교...)
병원 — 환자를 중증도로 분류 (응급/준응급/비응급/대기)
학교 — 학생을 점수로 등급 분류 (A/B/C/D/F)

이 모든 분류가 else if 체인과 같은 구조예요!
기준을 정하고 → 순서대로 확인하고 → 처음 해당하는 곳에 배치
[사고력] 4요소 심화 연결
1. 분해(Decomposition):
'성적 등급 판정'을 → (1) 점수 입력 (2) 범위별 비교 (3) 등급 출력으로 분해
각 범위(90~, 80~, 70~, 60~, ~59)가 독립적인 하위 문제!

2. 패턴 인식(Pattern Recognition):
성적 판정, 계절 판별, BMI 분류, 교통수단 추천 — 전부가
'값을 범위별로 나누는' 같은 패턴!
if (값 >= 기준1) ... else if (값 >= 기준2) ...

3. 추상화(Abstraction):
성적 판정의 '핵심'은 기준점(90, 80, 70, 60)뿐이에요.
학생 이름, 과목명, 학교 이름은 판정에 불필요한 세부사항.

4. 알고리즘 설계(Algorithm Design):
'범위가 겹치지 않게 큰 값부터 확인'하는 내림차순 체인은
빠지는 구간 없이 전체를 커버하는 알고리즘 설계의 좋은 예시!
else if를 쓸 때마다 '분류 체계를 설계'하는 거예요. 기준을 정하고, 순서를 정하고, 빠지는 구간이 없는지 확인!

[KOI/백준] else if 실전 대회 문제

KOI 실전 — 백준 9498 시험 성적
문제: 시험 점수를 입력받아 등급 출력
90~100: A / 80~89: B / 70~79: C / 60~69: D / 나머지: F
입력: 85 → 출력: B

코드:
if (s>=90) printf("A");
else if (s>=80) printf("B");
else if (s>=70) printf("C");
else if (s>=60) printf("D");
else printf("F");

핵심: 큰 값부터! 순서 바꾸면 오답.
KOI 실전 — 백준 2525 오븐 시계
문제: 현재 H시 M분에서 C분 후의 시각을 출력
입력: 14 30 / 200 → 출력: 17 50

풀이:
int h, m, c;
scanf("%d %d", &h, &m);
scanf("%d", &c);
m += c;
h += m / 60;
m %= 60;
h %= 24;
printf("%d %d", h, m);

핵심: 분→시 변환(60), 24시 순환(24). if 대신 나머지 연산으로도 풀 수 있지만, else if로 '분이 60 이상이면 시간 올림' 방식도 가능.
KOI 실전 — 백준 2480 주사위 세개
문제: 주사위 3개를 던져서:
같은 눈 3개 → 10000 + 같은눈*1000
같은 눈 2개 → 1000 + 같은눈*100
모두 다름 → 가장 큰 눈*100
입력: 3 3 6 → 출력: 1300

풀이:
if (a==b && b==c) printf("%d", 10000+a*1000);
else if (a==b) printf("%d", 1000+a*100);
else if (a==c) printf("%d", 1000+a*100);
else if (b==c) printf("%d", 1000+b*100);
else {
  int mx = a; if(b>mx) mx=b; if(c>mx) mx=c;
  printf("%d", mx*100);
}

핵심: 조건 순서! 3개 같은 것을 먼저 확인, 그 다음 2개, 마지막에 모두 다른 경우.
KOI 실전 — 백준 2563 색종이 (사고력 확장)
문제: 100x100 도화지에 10x10 색종이 N장을 붙였을 때 검은 영역의 넓이
사고력 연결:
이 문제는 else if로 직접 풀지는 않지만, 조건문 사고를 확장해요.
각 칸이 '색종이에 덮였는가?'를 if로 판별 → 2차원 배열에 마킹

패턴: 2차원 공간의 '포함 여부'를 if로 판별하는 패턴은
게임의 충돌 감지, 지도 범위 확인에도 쓰여요.
주사위 문제 핵심: '3개 같음 → 2개 같음 → 모두 다름' 순서! 좁은 조건(3개 일치)부터 넓은 조건(모두 다름)으로!

[디버깅] else if 버그 시나리오

흔한 버그 — 조건 순서 뒤집힘
버그: 95점인데 D등급 출력!
if (score >= 60) printf("D");
else if (score >= 70) printf("C");
else if (score >= 80) printf("B");
else if (score >= 90) printf("A");

원인: 95 >= 60이 참이라 첫 번째에서 걸림! 나머지는 검사 안 함.
해결: 큰 값부터: 90→80→70→60 순서
교훈: else if 범위 검사는 무조건 큰 값부터!
흔한 버그 — else 누락
버그: 55점인데 아무것도 출력 안 됨
if (s>=90) printf("A");
else if (s>=80) printf("B");
else if (s>=70) printf("C");
else if (s>=60) printf("D");

원인: 55는 어디에도 안 걸림! 마지막 else가 없으니까.
해결: else printf("F"); 추가
교훈: else if 체인의 마지막은 항상 else로 나머지를 잡아요!
흔한 버그 — 범위 겹침/빠짐
버그: 80점이 B도 아니고 C도 아님
if (s > 90) printf("A");
else if (s > 80) printf("B");
else if (s > 70) printf("C");

원인: > 는 초과! 80은 >80이 거짓, >70이 참이라 C가 됨.
90점도 A가 아니라 B!
해결: >= (이상)을 쓸지 > (초과)를 쓸지 명확하게!
교훈: 경계값(정확히 90, 80, 70)으로 반드시 테스트!

10축 심화 연결
[컴퓨팅사고력 — 패턴 인식] 성적 판정, 계절 판별, BMI 분류, 교통수단 추천 — 전부 '값을 기준점으로 범위 분류'하는 같은 패턴이에요. 기준점만 바꾸면 어떤 범위 분류든 즉시 else if로 구현할 수 있어요.

용어 정리

[심화] else if의 내부 구조와 최적화

[심화 내용] 컴파일러가 else if를 처리하는 방법
방법 1: 점프 체인
CMP → JL(skip to next) → 코드 → JMP(end)
CMP → JL(skip) → 코드 → JMP(end)
...
조건이 많으면 점프가 많아져 느려질 수 있음

방법 2: 점프 테이블 (switch와 비슷)
값이 연속 정수면 컴파일러가 배열 인덱스로 변환!
O(n) → O(1)로 최적화

방법 3: 이진 분할
조건이 정렬되어 있으면 이진 탐색처럼 중간부터!
5개 조건 → 최대 3번 비교 (log2(5) ≈ 2.3)
[C 코드] else if vs 룩업 테이블 비교
1#include <stdio.h>
2
3// else if 체인 (O(n))
4const char* grade_if(int s) {
5 if (s >= 90) return "A";
6 else if (s >= 80) return "B";
7 else if (s >= 70) return "C";
8 else if (s >= 60) return "D";
9 else return "F";
10}
11
12// 룩업 테이블 (O(1))
13const char* grade_table(int s) {
14 static const char *grades[] = {
15 "F","F","F","F","F","F","D","C","B","A","A"
16 };
17 if (s < 0 || s > 100) return "?";
18 return grades[s / 10];
19}
20
21int main() {
22 int tests[] = {95, 85, 75, 65, 55, 100, 0};
23 for (int i = 0; i < 7; i++) {
24 printf("%3d: %s (if) %s (table)\n",
25 tests[i], grade_if(tests[i]), grade_table(tests[i]));
26 }
27 return 0;
28}
실행 결과
95: A (if) A (table) 85: B (if) B (table) 75: C (if) C (table) 65: D (if) D (table) 55: F (if) F (table) 100: A (if) A (table) 0: F (if) F (table)
점프 테이블
연속 정수 조건이면 컴파일러가 배열로 변환. O(n)→O(1). switch문에서 주로 사용되는 최적화.
룩업 테이블
배열로 if 체인 대체. s/10을 인덱스로! 메모리 약간, 속도 대폭 향상. 범위 체크 필수.

[심화] 다중 반환값과 함수 분리

[심화 내용] else if 체인을 함수로 분리
긴 else if 체인은 함수로 분리!

char get_grade(int score) {
  if (score >= 90) return 'A';
  if (score >= 80) return 'B';
  if (score >= 70) return 'C';
  if (score >= 60) return 'D';
  return 'F';
}

함수로 분리하면:
1. 재사용 가능
2. 테스트 쉬움 (assert 활용)
3. main이 깔끔해짐

가드 절 패턴(지난 유닛)과 조합하면 더 깔끔!
[C 코드] 함수 분리 + 테스트
1#include <stdio.h>
2#include <assert.h>
3
4char get_grade(int s) {
5 if (s >= 90) return 'A';
6 if (s >= 80) return 'B';
7 if (s >= 70) return 'C';
8 if (s >= 60) return 'D';
9 return 'F';
10}
11
12int main() {
13 // 자동 테스트 — 경계값 체크!
14 assert(get_grade(100) == 'A');
15 assert(get_grade(90) == 'A');
16 assert(get_grade(89) == 'B');
17 assert(get_grade(80) == 'B');
18 assert(get_grade(79) == 'C');
19 assert(get_grade(70) == 'C');
20 assert(get_grade(69) == 'D');
21 assert(get_grade(60) == 'D');
22 assert(get_grade(59) == 'F');
23 assert(get_grade(0) == 'F');
24
25 printf("모든 테스트 통과!\n");
26
27 int score;
28 printf("점수: ");
29 scanf("%d", &score);
30 printf("%c등급\n", get_grade(score));
31 return 0;
32}
실행 결과
모든 테스트 통과! 점수: 85 B등급
assert
조건이 거짓이면 프로그램 중단! . 디버깅용 자동 테스트. 경계값 테스트에 활용.
단위 테스트
함수 하나를 독립적으로 테스트. assert로 경계값(90,89,80,79...) 검증. 버그 조기 발견!
assert로 경계값을 자동 테스트! 90/89/80/79... 전부 넣어서 확인. 이게 프로 개발자의 습관이에요!

[심화] 조건 커버리지와 테스트 설계

[심화 내용] 조건 커버리지(Condition Coverage)
else if 체인을 테스트할 때 모든 분기를 커버해야!

성적 판정 테스트 케이스:
100 → A (최대값)
90 → A (경계값)
89 → B (경계값 바로 아래)
80 → B
79 → C
70 → C
69 → D
60 → D
59 → F
0 → F (최소값)
-1 → ? (유효 범위 밖!)
101 → ? (유효 범위 밖!)

규칙: 각 분기 + 경계값 + 유효 범위 밖 = 완전한 테스트
조건 커버리지
모든 if/else if/else 분기를 최소 1번씩 실행하는 테스트. 100% 달성이 목표. 경계값 필수!
dead code
절대 실행 안 되는 분기. 조건 순서가 잘못되면 발생. if(s>=60) 다음 if(s>=90)은 dead code!
10진 분류법
도서관 분류 체계. 000~999를 10개 주류로. else if 체인과 같은 범위 분류 구조!
triage
의료 분류. 응급/준응급/비응급/대기. else if 체인과 같은 우선순위 분류!
McCabe 복잡도
if/else if마다 +1. 성적 판정은 5. 10 이하 권장, 20 초과면 리팩토링!
decision table
결정 테이블. 모든 조건 조합과 결과를 표로. else if 체인 설계 전 작성하면 빠짐 방지!
else if 체인을 만들었으면 decision table로 빠진 케이스가 없는지 확인! 이게 프로의 설계 습관!
else if는 분류 체계의 코드화! 기준 정하고, 순서 정하고, 빠짐 없는지 확인하면 완벽!
[활동] else if 마스터 체크리스트
else if 기본 구조를 안다
조건 순서(큰 값부터)의 중요성을 안다
if 여러 개 vs else if 차이를 안다
마지막 else로 나머지를 잡는다
경계값 테스트를 할 수 있다
assert로 자동 테스트를 할 수 있다
룩업 테이블로 대체할 수 있다
백준 9498(성적), 2480(주사위)를 풀 수 있다

6개 이상이면 else if 마스터!
else if
이전 조건이 거짓일 때 다음 조건을 검사하는 구조. 여러 개 연결 가능.
다중 분기
세 갈래 이상으로 나뉘는 구조. else if 체인이나 switch로 구현해요.
내림차순 조건 체인
큰 값부터 확인하는 else if 순서. 범위 검사의 올바른 패턴.
경계값(Boundary Value)
범위의 끝 값. 90, 80, 70처럼 분류 기준이 되는 값. 버그가 가장 많이 생기는 곳!
조건 커버리지
테스트가 모든 분기를 한 번씩 실행하는 것.
assert
조건이 거짓이면 프로그램을 중단시키는 매크로. 자동 테스트에 사용.
