조건을 합치자 — &&와 ||

여러 조건을 하나로 합치는 논리 연산자 &&(그리고)와 ||(또는)를 배워요.

📚 이번에 배울 것

  • &&(AND)와 ||(OR)의 뜻과 사용법을 알아요.
  • !(NOT)으로 조건을 뒤집을 수 있어요.
  • 단축 평가의 원리와 활용을 이해해요.

✅ 시작 전 체크

  • if-else문을 사용할 수 있다
  • 중첩 if를 안다
Part 1 진행률: 22%

조건을 합치자!

지난 유닛에서 중첩 if를 배웠죠? '나이 19세 이상이면서 면허 있으면' — 이걸 한 줄로 쓸 수 있어요!

&&는 '그리고(AND)', ||는 '또는(OR)'이에요. 여러 조건을 하나의 if에 합칠 수 있어요.

&&와 ||의 핵심
&& → 둘 다 참이어야 참
|| → 하나만 참이어도 참
! → 참↔거짓 뒤집기

&& — 둘 다 참이어야

and_basic.c
1#include <stdio.h>
2
3int main(void) {
4 int age = 20;
5 int license = 1;
6
7 if (age >= 19 && license == 1) {
8 printf("운전 가능!\n");
9 } else {
10 printf("운전 불가!\n");
11 }
12 return 0;
13}
실행 결과
운전 가능!
20>=19 참 && 1==1 참 → 둘 다 참이니까 전체 참! 하나라도 거짓이면 전체가 거짓이에요.
[핵심] && 진리표
ABA && B
참참참
참거짓거짓
거짓참거짓
거짓거짓거짓

둘 다 참일 때만 참! 하나라도 거짓이면 거짓.

|| — 하나만 참이어도

or_basic.c
1#include <stdio.h>
2
3int main(void) {
4 int day = 6; // 토요일
5
6 if (day == 6 || day == 0) {
7 printf("주말!\n");
8 } else {
9 printf("평일...\n");
10 }
11 return 0;
12}
실행 결과
주말!
6==6 참 || 6==0 거짓 → 하나가 참이니까 전체 참! 둘 다 거짓일 때만 거짓.
[핵심] || 진리표
ABA || B
참참참
참거짓참
거짓참참
거짓거짓거짓

둘 다 거짓일 때만 거짓. 하나만 참이어도 참!

! — 뒤집기(NOT)

not_basic.c
1#include <stdio.h>
2
3int main(void) {
4 int logged_in = 0;
5
6 if (!logged_in) {
7 printf("로그인이 필요합니다.\n");
8 }
9 return 0;

실행 결과
로그인이 필요합니다.
!0은 1(참), !1은 0(거짓). !logged_in은 '로그인 안 했으면'이라는 뜻!

점수가 0~100 사이인지, 또는 특정 범위 안에 있는지 확인할 때 &&를 많이 써요.

중첩 if를 &&로 바꾸기

✕ 잘못된 코드
if (age >= 19) { if (license == 1) { if (!drunk) { printf("운전 가능"); } } }
3단 중첩! 읽기 힘들어요
✓ 올바른 코드
if (age >= 19 && license == 1 && !drunk) { printf("운전 가능"); }
&&로 한 줄! 깔끔!
[주의] 줄이면 안 되는 경우!
각 조건 실패 시 다른 메시지를 보여야 하면 &&로 합칠 수 없어요!

나이 부족 → '미성년자'
면허 없음 → '면허 필요'
음주 → '음주 상태'

이 경우 중첩 if 또는 가드 절을 써야 해요.

실전 예제: 윤년 판별 (&&와 || 조합)

leap_year.c
1#include <stdio.h>
2
3int main(void) {
4 int y;
5 printf("연도: ");
6 scanf("%d", &y);
7
8 // 윤년: 4의 배수이면서 100의 배수가 아니거나, 400의 배수
9 if ((y % 4 == 0 && y % 100 != 0) || y % 400 == 0) {
10 printf("%d년은 윤년!\n", y);
11 } else {
12 printf("%d년은 평년.\n", y);
13 }
14 return 0;
15}
실행 결과
연도: 2024 2024년은 윤년!
&&와 ||를 조합! (4의 배수 AND 100의 배수 아님) OR 400의 배수. 괄호로 우선순위 명확하게!
[핵심] &&가 ||보다 우선순위 높아요!
수학에서 곱셈이 덧셈보다 먼저이듯:
&& (AND)가 || (OR)보다 먼저 계산돼요.

a || b && c → a || (b && c)

헷갈리면 괄호를 쓰세요!
(a || b) && c ← 이렇게 명확하게!

자주 하는 실수

논리 연산자 실수가 초보자 TOP 3에 들어요!
✕ 잘못된 코드
if (score >= 0 & score <= 100)
& 하나는 비트 AND! 논리 AND는 &&
✓ 올바른 코드
if (score >= 0 && score <= 100)
&& 두 개가 논리 AND예요
✕ 잘못된 코드
if (x == 1 || 2 || 3)
2와 3은 항상 참! 의도대로 안 돼요
✓ 올바른 코드
if (x == 1 || x == 2 || x == 3)
각각 비교해야 해요!

[컴퓨팅 사고력] 논리 연산과 불 대수

[사고력] 불 대수(Boolean Algebra) — 컴퓨터의 사고 체계
조르주 불(George Boole, 1815~1864):
'인간의 논리적 사고를 수학으로!' → 불 대수 발명

100년 후 클로드 섀넌(1937)이
'전기 회로로 불 대수를 구현할 수 있다!'고 증명
→ 디지털 컴퓨터의 탄생!

CPU의 모든 연산 = AND, OR, NOT의 조합
if (a && b)를 쓸 때마다 불 대수를 실행하는 거예요!
[사고력] 4요소 심화 연결
1. 분해(Decomposition):
'윤년 판별'을 → (1) 4의 배수? (2) 100의 배수? (3) 400의 배수? 3개 질문으로 분해
각 질문의 답을 &&와 ||로 조합!

2. 패턴 인식(Pattern Recognition):
범위 확인: a<=x && x<=b ← 패턴!
다중 값 확인: x==a || x==b || x==c ← 패턴!
음수/NULL 검사: if(!valid) return; ← 패턴!

3. 추상화(Abstraction):
복잡한 조건을 bool 변수로 이름 붙이기:
bool is_adult = (age >= 19);
bool has_license = (license == 1);
if (is_adult && has_license) ... ← 읽기 쉬움!

4. 알고리즘 설계(Algorithm Design):
드 모르간 법칙으로 조건 변환:
!(A && B) = !A || !B
!(A || B) = !A && !B
복잡한 조건을 단순하게 바꾸는 설계 기법!
드 모르간 법칙! !(a && b)를 !a || !b로 바꿀 수 있어요. 복잡한 부정 조건을 단순하게!

[디버깅] 논리 연산 버그 시나리오

흔한 버그 — & vs &&
버그: 비트 AND(&)를 논리 AND(&&) 대신 사용
if (x > 0 & y > 0)

차이:
& — 비트 연산. 두 피연산자를 비트 단위로 AND. 단축 평가 없음!
&& — 논리 연산. 참/거짓만 판단. 단축 평가 있음!

위험: &는 단축 평가가 없어서 x가 NULL이어도 y를 평가! 크래시 가능!
해결: 논리 판단에는 항상 && 사용
교훈: -Wall 옵션으로 경고 감지!
흔한 버그 — || 다중 비교 실수
버그: x가 1 또는 2 또는 3인지 확인
if (x == 1 || 2 || 3)

원인: C에서 2와 3은 0이 아니므로 항상 참!
실제로는 if(x==1 || true || true)와 같음 → 항상 참!

해결: 각각 비교
if (x==1 || x==2 || x==3)

교훈: ||는 각 항이 독립적인 조건식이어야!
흔한 버그 — 드 모르간 실수
버그: '0~100 범위 밖'을 확인하려다가
if (!(score >= 0 && score <= 100))
→ 실수로 이렇게 변환:
if (score < 0 && score > 100) // 불가능한 조건!

원인: 드 모르간 법칙 적용 잘못
!(A && B) = !A || !B (&&가 ||로 바뀜!)

해결:
if (score < 0 || score > 100)

교훈: 부정할 때 &&↔|| 바뀜! 드 모르간 법칙 외우기!

10축 심화 연결
[컴퓨팅사고력 — 추상화/패턴] bool 변수로 조건에 이름을 붙이면(is_adult, has_license) 코드가 자연어처럼 읽혀요. 이것이 추상화! 범위 확인(a<=x && x<=b), 다중 값(x==a||x==b) 패턴은 어떤 프로그램에서든 반복돼요.

[디버깅 — 단축 평가] if(ptr && *ptr>0)에서 ptr이 NULL이면 *ptr를 평가하지 않아요. 단축 평가가 없는 &를 쓰면 크래시! 이 차이는 C 프로그래밍의 생명선이에요.

용어 정리

[심화] 단축 평가(Short-circuit) 심층 분석

[심화 내용] 단축 평가의 원리와 활용
&&: 왼쪽이 거짓이면 오른쪽 평가 안 함!
거짓 && (뭐든) → 결과 확정(거짓). 오른쪽 불필요!

||: 왼쪽이 참이면 오른쪽 평가 안 함!
참 || (뭐든) → 결과 확정(참). 오른쪽 불필요!

실전 활용 3가지:
1. 안전한 포인터: if(ptr && *ptr>0) — NULL이면 역참조 안 함
2. 0 나누기 방지: if(n!=0 && total/n>100)
3. 배열 범위: if(i>=0 && i0)
[C 코드] 단축 평가 동작 확인
1#include <stdio.h>
2
3int check(int n, const char* label) {
4 printf(" [%s 평가됨] n=%d\n", label, n);
5 return n;
6}
7
8int main() {
9 printf("--- && 단축 ---\n");
10 printf("거짓 && ?:\n");
11 if (check(0,"왼쪽") && check(1,"오른쪽"))
12 printf("통과\n");
13 // 오른쪽 평가 안 됨!
14
15 printf("\n참 && ?:\n");
16 if (check(1,"왼쪽") && check(1,"오른쪽"))
17 printf("통과\n");
18
19 printf("\n--- || 단축 ---\n");
20 printf("참 || ?:\n");
21 if (check(1,"왼쪽") || check(1,"오른쪽"))
22 printf("통과\n");
23 // 오른쪽 평가 안 됨!
24
25 return 0;
26}
실행 결과
--- && 단축 --- 거짓 && ?: [왼쪽 평가됨] n=0 참 && ?: [왼쪽 평가됨] n=1 [오른쪽 평가됨] n=1 통과 --- || 단축 --- 참 || ?: [왼쪽 평가됨] n=1 통과
단축 평가
Short-circuit. &&: 왼쪽 false→오른쪽 무시. ||: 왼쪽 true→오른쪽 무시. 안전한 코드의 핵심!
부수효과
Side effect. if(x++>0 || y++>0)에서 단축 평가로 y++가 실행 안 될 수 있음! 조건에 ++/-- 금지!

[심화] 드 모르간 법칙과 조건 변환

[심화 내용] 드 모르간 법칙(De Morgan's Laws)
법칙 1: !(A && B) = !A || !B
법칙 2: !(A || B) = !A && !B

부정하면 &&↔|| 바뀜!

예시:
'성인이고 면허 있음'의 반대 = '미성년이거나 면허 없음'
!(age>=19 && license) = age<19 || !license

활용: 복잡한 부정 조건을 단순화
if(!(a>0 && b>0 && c>0))
→ if(a<=0 || b<=0 || c<=0)

6개 이상이면 논리 연산 마스터!
&&(AND)
논리 곱. 둘 다 참이어야 참. 단축 평가: 왼쪽 거짓이면 오른쪽 무시.
||(OR)
논리 합. 하나만 참이어도 참. 단축 평가: 왼쪽 참이면 오른쪽 무시.
!(NOT)
논리 부정. 참↔거짓 뒤집기. !0은 1, !1은 0.
단축 평가
결과가 확정되면 나머지 조건을 평가하지 않는 것.
드 모르간 법칙
!(A&&B)=!A||!B, !(A||B)=!A&&!B. 부정 시 &&↔|| 바뀜.
불 대수
AND/OR/NOT으로 모든 논리를 수학적으로 표현하는 체계.
이번 유닛 핵심 정리
1

2
&&가 ||보다 우선순위 높아요. 헷갈리면 괄호!
3
단축 평가: 왼쪽에서 결과 확정되면 오른쪽 평가 안 해요.
