메뉴를 고르는 상황

음식점 키오스크를 생각해 보세요. 1번 짜장면, 2번 짬뽕, 3번 탕수육 — 번호를 누르면 해당 음식이 나오죠? 이런 걸 switch로 만들 수 있어요!

if-else if 체인으로도 할 수 있지만, 선택지가 정해진 값(1, 2, 3...)일 때는 switch가 더 깔끔해요.

switch 순서도

switch는 값을 case와 비교! if처럼 조건식이 아니라 정확히 일치하는 case로 점프해요!

switch는 if-else if 체인과 비슷하지만, 정수 또는 문자 값만 비교할 수 있어요. 실수(float)나 문자열(char[])은 사용할 수 없습니다.

switch의 생김새

switch_shape.c
1switch (변수) {
2 case 값1:
3 // 변수가 값1일 때 실행
4 break;
5 case 값2:
6 // 변수가 값2일 때 실행
7 break;
8 default:
9 // 어디에도 안 맞을 때 실행
10 break;
11}
switch는 변수의 값을 보고, case와 하나씩 비교해요. 맞는 case를 찾으면 그 아래 코드를 실행하고, break를 만나면 switch를 빠져나가요.
[핵심] switch 구조 한눈에
switch (검사할 변수) — 어떤 변수를 검사할지
case 값: — 이 값과 같으면 여기서 실행 시작
break; — switch 탈출! 없으면 다음 case로 '떨어짐'
default: — 아무 case에도 안 맞을 때 (else와 비슷)

break가 없으면? — 떨어짐(Fall-through)

switch에서 가장 헷갈리는 부분이에요! break가 없으면 다음 case로 '떨어져서' 계속 실행돼요.

fall_through.c
1#include <stdio.h>
2
3int main(void) {
4 int n = 2;
5
6 switch (n) {
7 case 1:
8 printf("하나\n");
9 case 2:
10 printf("둘\n");
11 case 3:
12 printf("셋\n");
13 default:
14 printf("기타\n");
15 }
16
17 return 0;
18}
실행 결과
둘 셋 기타
[주의] 떨어짐(Fall-through)을 꼭 이해하세요!
n=2이면 case 2에서 시작하지만, break가 없으니까
case 3도, default도 전부 실행돼요!

비유: 엘리베이터에서 2층 버튼을 눌렀는데
멈추는 버튼(break)이 없으면 3층, 4층... 다 지나가요!

규칙:
1. 각 case 끝에 반드시 break; 쓰기
2. 일부러 떨어뜨릴 때만 break 생략 (주석으로 표시!)
break 까먹는 건 switch에서 가장 흔한 버그! case마다 break를 쓰는 습관을 들이세요!

break를 빼먹으면 의도치 않게 다음 case의 코드까지 실행돼요. 이것이 fall-through입니다. C에서 가장 흔한 버그 중 하나!

일부러 break를 빼서 여러 case가 같은 코드를 실행하게 할 수도 있어요.

[핵심] 언제 switch, 언제 if-else if?
switch를 쓸 때:
- 변수가 정확한 값(1, 2, 3... 또는 'a', 'b', 'c'...)과 비교
- 선택지가 3개 이상일 때 깔끔

if-else if를 쓸 때:
- 범위 비교가 필요할 때 (>=90, >=80...)
- 복합 조건이 필요할 때 (a>0 && b>0)
- 실수(float/double) 비교일 때

switch로 안 되는 것:
- 범위: case >= 90: ← 불가!
- 실수: case 3.14: ← 불가!
- 문자열: case "hello": ← 불가!

[컴퓨팅 사고력] 매핑과 디스패치

실생활 속의 switch

switch는 우리 주변 어디에나 있어요!

[실생활] switch는 어디에?
엘리베이터: 버튼(1~20층) → switch(버튼) → 해당 층으로 이동
자판기: 버튼(A1~D4) → switch(버튼) → 해당 음료 배출
리모컨: 버튼(1~9) → switch(채널) → 해당 채널로 전환
게임: 키보드(WASD) → switch(키) → 캐릭터 이동 방향

모든 "선택 → 동작" 구조는 switch와 같아요!
자판기 버튼 누르면 음료가 나오죠? 그게 switch(버튼) { case A1: 콜라; break; ... }!
[사고력] switch = 값→행동 매핑
매핑(Mapping): 입력값을 특정 행동에 연결하는 것

키보드 → 'W'키 → 위로 이동, 'S'키 → 아래 이동
리모컨 → 1번 → KBS, 2번 → MBC, 3번 → SBS
자판기 → A1 → 콜라, A2 → 사이다

이 '값→행동' 매핑이 switch의 본질이에요!

디스패치(Dispatch):
'값을 보고 어떤 함수/코드를 실행할지 결정'하는 것
게임 이벤트, 네트워크 패킷, UI 이벤트 처리가 전부 디스패치!
[사고력] 4요소 심화 연결
1. 분해(Decomposition):
'메뉴 프로그램'을 → (1) 메뉴 표시 (2) 입력 받기 (3) 선택에 따라 처리 로 분해
(3)이 switch의 역할!

2. 패턴 인식(Pattern Recognition):
메뉴 선택, 요일 판별, 등급 판정, 계산기 연산 — 전부
'정해진 값 → 해당 처리' 같은 매핑 패턴!

3. 추상화(Abstraction):
switch의 핵심은 '값'과 '행동'의 쌍이에요.
메뉴 번호가 1이든 100이든 구조는 동일!

4. 알고리즘 설계(Algorithm Design):
switch는 컴파일러가 점프 테이블로 최적화 가능!
case가 연속 정수면 O(1)에 해당 코드로 점프!
게임에서 키 입력 처리가 전부 switch! W→위, A→왼쪽, S→아래, D→오른쪽. 값→행동 매핑이에요!

[디버깅] switch 버그 시나리오

흔한 버그 — break 누락 (TOP 1!)
버그: case 1을 선택했는데 case 2, 3도 실행됨
case 1: printf("하나");
case 2: printf("둘");
case 3: printf("셋");
→ 출력: '하나둘셋'

원인: break가 없어서 fall-through!
해결: 모든 case 끝에 break 추가
교훈: switch 쓸 때 case 먼저 다 쓰고 break를 채워 넣는 습관!

비유: 엘리베이터에 멈춤 버튼이 없는 것과 같아요!
흔한 버그 — case에 변수 사용
버그: case에 변수를 넣음
int target = 5;
switch (x) {
  case target: printf("OK"); break;
}

원인: C에서 case에는 상수(리터럴)만 가능!
변수, 함수 호출, 계산식 전부 불가!
해결: case 5: 또는 #define TARGET 5
교훈: case 뒤에는 항상 숫자 또는 문자 리터럴!
흔한 버그 — default 누락
버그: 1~3만 처리했는데 사용자가 5를 입력하면 아무 반응 없음
switch (menu) {
  case 1: ... break;
  case 2: ... break;
  case 3: ... break;
}

원인: default가 없어서 예상 외 값을 처리하지 않음
해결: default: printf("잘못된 입력"); break;
교훈: switch에는 항상 default를 넣어요! else처럼!

[컴퓨팅사고력 — 매핑/추상화] switch는 '값→행동' 매핑의 코드화예요. 키보드 입력, 메뉴 선택, 요일 판별이 모두 같은 매핑 패턴. 패턴을 인식하면 어떤 매핑 문제든 switch로 풀 수 있어요.

[디버깅 — fall-through] break 누락은 switch의 최대 함정! C의 switch는 '떨어지는' 것이 기본이고 break로 '멈추는' 거예요. Java/Python의 switch는 자동 break지만 C는 아닙니다!

[KOI 연결] 계산기, 요일 출력, 메뉴 프로그램이 switch 빈출 유형. 동시에 'switch vs if-else 선택'이라는 판단 자체가 알고리즘 설계 능력.

용어 정리

[심화] 컴파일러의 switch 최적화

[심화 내용] 점프 테이블(Jump Table)
if-else if: 조건을 위에서 아래로 순서대로 비교. 10개면 최대 10번!

switch (점프 테이블): 값을 인덱스로 써서 배열에서 주소를 가져옴!

case 0 → table[0] → 코드0 주소
case 1 → table[1] → 코드1 주소
case 2 → table[2] → 코드2 주소
...

100개의 case가 있어도 딱 1번에 해당 코드로 점프!
O(n) → O(1)!

조건: case 값이 연속 정수이고 범위가 좁을 때만 적용
[C 코드] switch vs if-else 성능 비교
1#include <stdio.h>
2#include <time.h>
3
4int with_switch(int n) {
5 switch (n) {
6 case 0: return 10;
7 case 1: return 20;
8 case 2: return 30;
9 case 3: return 40;
10 case 4: return 50;
11 default: return 0;
12 }
13}
14
15int with_if(int n) {
16 if (n == 0) return 10;
17 else if (n == 1) return 20;
18 else if (n == 2) return 30;
19 else if (n == 3) return 40;
20 else if (n == 4) return 50;
21 else return 0;
22}
23
24int main() {
25 long sum1 = 0, sum2 = 0;
26 clock_t t1 = clock();
27 for (long i = 0; i < 100000000; i++)
28 sum1 += with_switch(i % 5);
29 clock_t t2 = clock();
30 for (long i = 0; i < 100000000; i++)
31 sum2 += with_if(i % 5);
32 clock_t t3 = clock();
33
34 printf("switch: %.3f초\n", (double)(t2-t1)/CLOCKS_PER_SEC);
35 printf("if-else: %.3f초\n", (double)(t3-t2)/CLOCKS_PER_SEC);
36 return 0;
37}
점프 테이블
연속 정수 case를 배열 인덱스로! O(n)→O(1). 컴파일러가 자동 생성. -O2 옵션 필요.
fall-through
switch에서 break 없이 다음 case로 '떨어지는' 것. C의 기본 동작. 의도적이면 주석 표시!

[심화] switch와 열거형(enum)

default
switch에서 어떤 case에도 해당 안 되면 실행. if의 else와 같은 역할. 항상 넣는 습관!
점프 테이블
컴파일러가 switch를 배열로 변환! menu=3이면 table[3]번 주소로 직접 점프. O(1)!
enum
열거형. 숫자에 이름 부여. enum Color {RED, GREEN, BLUE}; switch(color) 더 읽기 쉽게!
[심화 내용] enum + switch = 타입 안전 분기
매직 넘버(1, 2, 3) 대신 이름을 붙이면 읽기 쉬워요!

typedef enum { COLA, CIDER, JUICE } Drink;

switch (drink) {
  case COLA: printf("콜라"); break;
  case CIDER: printf("사이다"); break;
  case JUICE: printf("주스"); break;
}

enum은 컴파일러가 0, 1, 2로 바꿔주지만
코드에서는 COLA, CIDER, JUICE라고 읽혀요!

enum
열거형. 관련 상수에 이름 붙이기. switch와 찰떡궁합! 매직 넘버 제거.
매직 넘버
코드에 직접 쓴 의미 불명의 숫자. case 1: 대신 case MENU_PIZZA:가 좋아요!

[심화] switch 대신 함수 포인터 테이블

switch의 case가 100개가 넘으면? 함수 포인터 배열로 대체할 수 있어요!

func_table.c
1#include <stdio.h>
2
3void cmd_new(void) { printf("새 파일\n"); }
4void cmd_open(void) { printf("열기\n"); }
5void cmd_save(void) { printf("저장\n"); }
6void cmd_quit(void) { printf("종료\n"); }
7
8typedef void (*CmdFunc)(void);
9
10int main(void) {
11 CmdFunc table[] = { cmd_new, cmd_open, cmd_save, cmd_quit };
12 int n = sizeof(table)/sizeof(table[0]);
13
14 int choice;
15 printf("명령(0~%d): ", n-1);
16 scanf("%d", &choice);
17
18 if (choice >= 0 && choice < n)
19 table[choice](); // 함수 포인터 호출!
20 else
21 printf("잘못된 명령\n");
22 return 0;
23}
실행 결과
명령(0~3): 2 저장
table[choice]()로 switch 없이 직접 함수 호출! 게임 엔진, OS 커널에서 이 패턴을 써요.

switch
변수의 값에 따라 여러 case로 분기하는 제어문.
case
switch에서 비교할 값. 상수만 가능. case 1: 형태.
break
switch나 반복문을 즉시 탈출하는 키워드.
default
switch에서 아무 case에도 안 맞을 때 실행. else와 비슷.
fall-through
break 없이 다음 case로 '떨어지는' switch의 기본 동작.
enum(열거형)
관련 상수에 이름을 붙이는 타입. switch와 함께 사용하면 가독성 향상.
