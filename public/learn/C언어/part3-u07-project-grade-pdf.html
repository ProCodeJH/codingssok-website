Lesson 3-7

미니 프로젝트: 성적 판정기

지금까지 배운 조건문(if, else if, switch, &&, ||)을 종합하여 완성된 프로그램을 만들어요.

📚 이번에 배울 것

  • 조건문을 종합 활용해 완성된 프로그램을 만들어요.
  • 입력 검증, 등급 판정, 결과 요약을 모두 구현해요.
  • 프로그래밍의 '설계→구현→테스트' 흐름을 경험해요.

✅ 시작 전 체크

  • if, else if, switch를 사용할 수 있다
  • &&와 ||로 조건을 합칠 수 있다

Step 1 — 설계하기

코드를 바로 치기 전에 무엇을 만들지 설계하는 것이 프로의 습관이에요.

[설계] 성적 판정기 요구사항
기능:
1. 과목명과 점수를 입력받는다
2. 점수가 0~100 범위인지 검증한다
3. 등급을 판정한다 (A/B/C/D/F)
4. 합격/불합격을 판정한다 (60점 이상 합격)
5. 격려 메시지를 등급에 따라 다르게 출력한다

사용할 문법:
- scanf/printf (입력/출력)
- if-else (범위 검증)
- else if 체인 (등급 판정)
- && (범위 확인)
- switch (격려 메시지)
설계를 먼저! 어떤 입력을 받고, 어떤 처리를 하고, 어떤 출력을 하는지를 정한 다음에 코드를 써요.
1
1단계: 입력
과목명(문자열)과 점수(정수)를 scanf로 받기
2
2단계: 검증
점수가 0~100인지 if+&&로 확인
3
3단계: 판정
else if 체인으로 A/B/C/D/F 등급 부여
4
4단계: 합격 판정
60점 이상이면 합격, 아니면 불합격
5
5단계: 출력
과목명, 점수, 등급, 합격여부 출력
6
6단계: 격려 메시지
switch로 등급별 코멘트 출력
6단계! 각 단계가 Part 3에서 배운 문법 하나씩과 연결돼요!

Step 3 — 테스트하기

확장: 다중 과목 성적 판정기

한 과목만이 아니라 여러 과목을 입력받아 모두 판정하는 버전입니다. for 반복문을 미리 사용해 볼게요!

2
3int main(void) {
4 int n;
5 printf("과목 수: ");

7
8 int total = 0;

14

21

Step 4 — 확장하기

기본 프로그램이 완성됐으니 기능을 추가해 볼까요?

✕ 잘못된 코드
// +/- 없는 버전 if (score >= 90) grade = 'A'; else if (score >= 80) grade = 'B'; // ... 5단계만
기본 버전: A/B/C/D/F 5단계
✓ 올바른 코드
// +/- 있는 버전 if (score >= 90) { grade = 'A'; if (score >= 95) mod = '+'; } else if (score >= 80) { grade = 'B'; if (score >= 85) mod = '+'; else mod = '-'; }
확장 버전: A+/A/B+/B-/... 세분화!

기본 5단계에서 +/- 세분화로 확장! 중첩 if가 여기서 쓰여요. 바깥 if로 큰 등급, 안쪽 if로 +/- 판정.

기본에서 확장으로! 이것이 프로그래밍의 반복적 개선(Iterative Refinement)이에요!

[컴퓨팅 사고력] 프로젝트 사고 과정

[사고력] 4요소가 프로젝트에서 어떻게 작동하는가
1. 분해(Decomposition):
'성적 판정기'를 → 입력/검증/판정/출력/메시지 5단계로 분해
각 단계가 코드의 한 블록에 대응!

2. 패턴 인식(Pattern Recognition):
범위 검증 = if(xmax) 패턴
등급 판정 = else if 내림차순 패턴
메시지 선택 = switch 매핑 패턴
이미 배운 패턴들의 조합!

3. 추상화(Abstraction):
점수→등급 변환을 함수로 분리하면 재사용 가능!
char get_grade(int score) — 이 함수는 과목과 무관하게 동작.

4. 알고리즘 설계(Algorithm Design):
설계→구현→테스트 3단계 개발 흐름.
경계값 테스트 8개로 모든 분기를 검증.
이것이 소프트웨어 개발의 기본 사이클!

이 프로젝트에서 분해는 가장 중요한 사고력입니다. "성적 판정기"라는 큰 문제를 입력/검증/판정/출력 4개의 작은 문제로 나눈 것이 핵심이에요.

✕ 잘못된 코드
printf("점수: "); scanf("%d", &score); // 바로 판정 시작 if (score >= 90)...
검증 없이 바로 처리
✓ 올바른 코드
printf("점수: "); scanf("%d", &score); // 먼저 검증! if (score < 0 || score > 100) { printf("오류!"); return 1; } if (score >= 90)...
입력 검증 후 처리 = 안전!
[사고력] 프로젝트 = 분해의 실전 연습
작은 프로그램: if 하나로 끝
중간 프로그램: 입력→처리→출력 3단계
큰 프로그램: 입력→검증→처리→포매팅→출력→로깅 6단계

단계가 늘어날수록 분해의 중요성이 커져요!
실제 프로그램은 수십~수백 단계! 분해 없이는 만들 수 없어요.
프로젝트를 할 때마다 이 4요소가 자동으로 작동해요! 분해하고, 패턴 찾고, 추상화하고, 설계하고!

[디버깅] 프로젝트 흔한 버그

프로젝트 버그 — 검증 순서 실수
버그: 등급 판정 후에 검증 → -5점도 F등급으로 처리됨!

원인: 검증(if(score<0))을 등급 판정(else if) 뒤에 넣음
해결: 검증을 맨 먼저! 잘못된 입력은 조기 퇴장(early return)
교훈: 가드 절 → 처리 → 출력 순서!
✕ 잘못된 코드
scanf("%s", subject); scanf("%d", &score); // 입력 검증 없음! if (score >= 90) ...
검증 없이 바로 판정 → score=999도 A!
✓ 올바른 코드
scanf("%s", subject); scanf("%d", &score); if (score < 0 || score > 100) { printf("오류!"); return 1; } if (score >= 90) ...
