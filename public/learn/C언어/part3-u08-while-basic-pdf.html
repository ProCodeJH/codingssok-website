무한 루프 — 끝나지 않는 반복

[주의] i++를 빼먹으면?
i++가 없으면 i는 영원히 1이고, 1 <= 5는 영원히 참이에요. 프로그램이 멈추지 않아요!

[심화] while 반복문 심화

[심화] while의 내부 동작과 최적화
CPU 레벨:
while (cond) { body }
→ L1: CMP cond / JZ L2 / body / JMP L1 / L2:

컴파일러 최적화:
1. 루프 인버전: do-while로 변환 (첫 비교 제거)
2. 루프 언롤링: 반복 본문을 N배 복사
3. SIMD 벡터화: 4~8개 동시 처리
4. 루프 불변 코드 이동: 변하지 않는 계산을 밖으로

루프 불변 코드 이동:
// Before:
while (i < n) { x = strlen(s); arr[i++] = x; }
// After (컴파일러가 자동):
int len = strlen(s);
while (i < n) { arr[i++] = len; }

Loop Invariant. 루프 중 변하지 않는 조건. while의 올바름 증명에 사용. 디버깅의 핵심 사고 도구!
루프 언롤링
Loop Unrolling. 루프 본문을 N번 복사해 반복 횟수 줄임. 분기 오버헤드 감소. -O2에서 자동.

[심화] while 무한 루프와 탈출
무한 루프 패턴:
while (1) { ... } // 가장 일반적
for (;;) { ... } // 동일

안전한 탈출:
1. break: 루프 즉시 탈출
2. return: 함수 자체 탈출
3. 조건 변수: while (running) { if(done) running=0; }
4. goto: 중첩 루프 탈출 (유일한 방법!)

위험한 패턴:
while (scanf("%d", &n) != EOF) → Ctrl+D/Ctrl+Z 필요
while (1) { if (cond) break; } → break 누락 시 행

watchdog 패턴:
int timeout = 1000;
while (!done && timeout-- > 0) { ... }
if (timeout <= 0) printf("Timeout!");

[심화 내용] while과 루프 증명
루프 불변 (Loop Invariant):
루프 실행 중 항상 참인 조건

예: GCD
while (b != 0) { t=b; b=a%b; a=t; }
불변: gcd(a,b) = gcd(원래a, 원래b)

증명 3단계:
1. 초기: 루프 진입 전 불변 성립
2. 유지: 한 반복 후에도 성립
3. 종료: 루프 끝나면 결론 도출

종료 증명:
감소하는 양(variant)이 존재
GCD: b는 매 반복마다 감소
b ≥ 0이므로 유한 반복 보장

왜 중요한가:
프로그램의 올바름을 수학적으로 증명!
[C 코드] while 고급 — 뉴턴 법 (제곱근)
1#include <stdio.h>
2#include <math.h>
3
4// 뉴턴 법으로 제곱근 근사
5double my_sqrt(double n) {
6 if (n < 0) return -1; // 에러
7 if (n == 0) return 0;
8
9 double guess = n / 2.0;
10 double epsilon = 1e-10;
11 int iterations = 0;
12
13 while (fabs(guess * guess - n) > epsilon) {
14 guess = (guess + n / guess) / 2.0;
15 iterations++;
16 if (iterations > 100) break; // 안전장치
17 }
18
19 printf(" sqrt(%.0f): %d iterations\n", n, iterations);
20 return guess;
21}
22
23int main() {

25 for (int i = 0; i < 5; i++) {
26 double result = my_sqrt(tests[i]);
27 printf(" my_sqrt = %.10f, math.h = %.10f\n\n",
28 result, sqrt(tests[i]));
29 }
30 return 0;
31}

뉴턴 법
Newton's Method. f(x)=0의 근 탐색. x_{n+1} = x_n - f(x_n)/f'(x_n). 제곱근: x = (x+n/x)/2. 수렴 빠름!

1#include <stdio.h>
2
3int binary_search(int *arr, int n, int target) {
4 int lo = 0, hi = n - 1;
5
6 while (lo <= hi) {
7 int mid = lo + (hi - lo) / 2; // 오버플로우 방지!
8
9 if (arr[mid] == target) return mid;
10 else if (arr[mid] < target) lo = mid + 1;
11 else hi = mid - 1;
12 }
13 return -1; // 못 찾음
14}
15
16// lower_bound: target 이상인 첫 위치
17int lower_bound(int *arr, int n, int target) {
18 int lo = 0, hi = n;
19 while (lo < hi) {
20 int mid = lo + (hi - lo) / 2;
21 if (arr[mid] < target) lo = mid + 1;
22 else hi = mid;
23 }
24 return lo;
25}

30

32 for (int i = 0; i < 5; i++) {
33 int idx = binary_search(arr, n, targets[i]);
34 if (idx >= 0)
35 printf("Found %d at index %d\n", targets[i], idx);
36 else
37 printf("%d not found (insert at %d)\n",

이진 탐색
정렬된 배열에서 O(log n) 검색. 매번 절반 제거. 10억 개에서 30번 비교로 찾음!
mid 오버플로우
(lo+hi)/2는 오버플로우 위험! lo+(hi-lo)/2가 안전. Java의 Arrays.binarySearch 버그(2006)로 유명.
이진 탐색의 mid=(lo+hi)/2는 오버플로우 버그! lo+(hi-lo)/2로 써야 해. 20년간 숨어있던 버그!
do-while 차이
while: 조건 먼저 → 0회 가능. do-while: 실행 먼저 → 최소 1회. 입력 검증에 do-while 적합!
루프 퓨전
Loop Fusion. 같은 범위의 루프 2개를 1개로 합치기. 캐시 효율 향상. 컴파일러 자동 최적화.
tail call
꼬리 호출. while을 재귀로 변환 시 마지막 호출. 컴파일러가 루프로 최적화. -O2에서 자동!
while과 재귀는 동치! while(b){t=b;b=a%b;a=t;} = gcd(a,b){return b?gcd(b,a%b):a;} 같은 동작!
event loop
이벤트 루프. while(1){event=get_event(); handle(event);} GUI, 서버, 게임의 핵심 구조!
spin lock
스핀 락. while(lock); 다른 스레드가 풀 때까지 대기. 짧은 대기에 유용. 긴 대기엔 mutex!
convergence
수렴. 뉴턴 법, 이분법 등에서 반복할수록 정답에 가까워짐. while(|error|>epsilon) 패턴!
while은 "조건이 참인 동안 반복"! 단순하지만 GCD, 이진탐색, 뉴턴법 — 모든 알고리즘의 기반!
게임 루프
while(!quit){processInput();update();render();} 게임의 메인 루프. 60FPS = 초당 60회 반복!
polling
폴링. while 루프로 상태 주기적 확인. I/O, 네트워크에서 사용. 인터럽트 대비 CPU 사용 높음.
amortized
분할 상환 분석. 최악은 느리지만 평균은 빠른 연산. 동적 배열 확장: 개별 O(n), 평균 O(1).
while 완전 정복! 기본 반복→알고리즘→수치해석→이벤트 루프. 다음은 for — 카운팅의 왕!
비결정적 루프
종료 조건이 입력에 의존. while(scanf(...)!=EOF). 몇 번 반복할지 미리 알 수 없음!
루프 피딩
while 본문에서 다음 반복의 조건 준비. while(p){...p=p->next;} 링크드 리스트 순회의 핵심!
while 마스터 졸업! for문은 "횟수가 정해진 반복", while은 "조건이 정해진 반복". 구분해서 쓰자!
setitimer
POSIX 타이머. 일정 간격으로 SIGALRM 발생. while+timer로 주기적 작업. 게임 틱 구현!
while 마스터 졸업! 다음은 for — 카운터 기반 반복의 왕. 배열 순회, 패턴 출력에 최적화!
for vs while
for: 카운터 명확 (초기화;조건;증감). while: 조건만. 횟수 정해짐→for, 불확정→while.
while과 for는 상호 변환 가능! while(c){...} = for(;c;){...}. 하지만 의도에 맞게 선택!

종료 조건이 없어서 끝나지 않는 반복. Ctrl+C로 강제 종료해요.

보초값(Sentinel)
반복을 멈추기 위한 특별한 값. (예: 0, -1)
