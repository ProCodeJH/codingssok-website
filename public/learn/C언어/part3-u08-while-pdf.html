반복의 시작 — while

같은 코드를 조건이 참인 동안 반복 실행하는 while문을 배워요.

📚 이번에 배울 것

  • while문의 구조와 실행 흐름을 이해해요.
  • 반복 횟수를 제어하는 카운터 변수를 사용할 수 있어요.
  • 무한 루프를 알고 빠져나올 수 있어요.

✅ 시작 전 체크

  • if-else문을 사용할 수 있다
  • 비교 연산자를 안다
Part 1 진행률: 39%

같은 일을 계속 하고 싶다면?

'야호!'를 10번 출력하려면 printf를 10번 써야 할까요? 아니에요! while을 쓰면 한 줄로 10번, 100번, 1000번도 가능!

컴퓨터가 강력한 이유는 반복을 지치지 않고 해내기 때문이에요. 사람은 100번 반복하면 지치지만, 컴퓨터는 100만 번도 1초 만에!

while의 흐름
조건 확인 → 참 → 코드 실행 → 다시 조건 확인으로 ↩
조건 확인 ↓ 거짓
while 끝! 다음 코드로
조건이 거짓이 될 때까지 계속 반복!

while의 생김새

while_shape.c
1while (조건) {
2 // 조건이 참인 동안 반복 실행
3 // 언젠가는 조건이 거짓이 되어야 해요!
4}
while = '~하는 동안'. 조건이 참이면 코드를 실행하고, 다시 조건을 확인하고, 또 실행하고... 거짓이 되면 멈춰요!
[핵심] while과 if의 차이
if: 조건이 참이면 1번 실행하고 끝
while: 조건이 참인 동안 계속 반복!

if는 '문 한 번 열기', while은 '회전문'이에요.
조건이 ❌ 변하지 않으면 while은 영원히 반복!(무한 루프)

첫 번째 while — 1부터 5까지

?

아래 코드에서 화면에 뭐가 나올까요? 몇 줄이 출력될까요?

while_count.c
1#include <stdio.h>
2
3int main(void) {
4 int i = 1; // 카운터 변수: 시작값
5
6 while (i <= 5) { // 조건: i가 5 이하인 동안
7 printf("%d번째 반복!\n", i);
8 i++; // i를 1 증가 (이게 없으면 무한 루프!)
9 }
10
11 printf("반복 끝! i는 %d\n", i);
12 return 0;
13}
실행 결과
1번째 반복! 2번째 반복! 3번째 반복! 4번째 반복! 5번째 반복! 반복 끝! i는 6
i가 1→2→3→4→5→6이 되면 6<=5가 거짓! while 끝! 총 5번 반복했어요.

한 줄씩 뜯어보기 — while의 3요소

4줄int i = 1; — ①초기화: 카운터 변수를 시작값으로 설정.
6줄while (i <= 5) — ②조건: 매 반복 전에 확인. 거짓이면 멈춤.
8줄i++; — ③갱신: 카운터를 변화시켜 언젠가 조건이 거짓이 되게!
[핵심 중의 핵심] while의 3요소를 꼭 기억하세요!
①초기화 — 카운터 변수를 시작값으로 (while 전에)
②조건 — 언제까지 반복할지 (while 괄호 안에)
③갱신 — 카운터를 변화시키기 (while 중괄호 안에)

비유: 줄넘기를 생각하세요!
① 줄넘기 준비 (시작 자세)
② 100번까지만 하기로 (목표)
③ 한 번 뛸 때마다 +1 세기 (세는 것)

하나라도 빠지면?
①빠지면: 시작값 쓰레기, 엉뚱한 결과
②빠지면: 무한 루프 or 한 번도 안 돌아감
③빠지면: 무한 루프! (가장 흔한 실수!)

실전 예제: 1~N까지의 합

sum_n.c
1#include <stdio.h>
2
3int main(void) {
4 int n;
5 printf("N: ");
6 scanf("%d", &n);
7
8 int sum = 0; // 합계를 저장할 변수
9 int i = 1; // ①초기화
10
11 while (i <= n) { // ②조건
12 sum += i; // sum = sum + i
13 i++; // ③갱신
14 }
15
16 printf("1부터 %d까지의 합: %d\n", n, sum);
17 return 0;
18}
실행 결과
N: 100 1부터 100까지의 합: 5050
가우스가 어린 시절 풀었던 1+2+...+100 = 5050! while로 1초 만에 계산! 공식: n*(n+1)/2 = 100*101/2 = 5050.

조건이 항상 참이면 영원히 반복해요. 이걸 무한 루프라고 해요.

의도적 무한 루프:
게임 루프, 서버, 센서 모니터링에서는 일부러 사용!

break로 탈출 조건을 만들면 안전한 무한 루프!

break — 반복 중간에 탈출!

while_break.c
1#include <stdio.h>
2
3int main(void) {
4 int num;
5
6 while (1) { // 무한 루프

9
10 if (num == 0) {
11 printf("종료합니다.\n");

13 }
14
15 printf("%d 입력됨\n", num);
16 }
17
18 printf("프로그램 끝\n");

실행 결과
숫자 입력 (0이면 종료): 5 5 입력됨 숫자 입력 (0이면 종료): 3 3 입력됨 숫자 입력 (0이면 종료): 0 종료합니다. 프로그램 끝
while(1) + break 패턴! '일단 반복하고, 특정 조건이면 탈출'. 게임 메인 루프, 메뉴 프로그램에서 자주 써요.

continue — 이번만 건너뛰기

while_continue.c
1#include <stdio.h>
2
3int main(void) {
4 int i = 0;
5
6 while (i < 10) {
7 i++;
8 if (i % 3 == 0) {
9 continue; // 아래 printf를 건너뛰고 다시 조건으로!
10 }
11 printf("%d ", i);
12 }
13 printf("\n끝!\n");
14 return 0;
15}
실행 결과
1 2 4 5 7 8 10 끝!
3, 6, 9가 빠졌어요! continue를 만나면 아래 코드를 건너뛰고 다시 while 조건 확인으로 돌아가요.
[핵심] break vs continue
break: 반복문 완전 탈출! 더 이상 반복 안 함.
continue: 이번 회차만 건너뛰기! 다시 조건 확인으로 돌아감.

비유: 줄넘기에서
break = '에이 그만두자!' (포기)
continue = '앗 걸렸다! 다음 번!' (건너뛰고 다시)

✕ 잘못된 코드

1~N까지의 곱(팩토리얼): sum 대신 product, += 대신 *=
홀수만의 합: if (i%2==1) sum+=i;
구구단: 특정 단의 곱셈표 출력

[컴퓨팅 사고력] 반복과 자동화

[사고력] while = 자동화의 시작!
사람 vs 컴퓨터:
사람: 1+2+3+...+100을 손으로 → 수 분
컴퓨터: while로 자동 → 0.000001초

사람: 파일 1000개의 이름 바꾸기 → 수 시간
컴퓨터: while로 반복 → 1초

반복 = 자동화의 핵심!
코딩의 진짜 가치는 '반복 작업을 컴퓨터에게 맡기는 것'이에요.
[사고력] 4요소 심화 연결
1. 분해(Decomposition):
'1~N까지의 합'을 → (1) 합계 변수 준비 (2) i=1부터 N까지 반복 (3) 각 반복에서 sum+=i로 분해
반복의 '한 회차'가 분해의 최소 단위!

2. 패턴 인식(Pattern Recognition):
1~N 합, 1~N 곱, 1~N 홀수합 — 전부 '카운터 반복' 패턴!
i=시작; while(i<=끝) { 처리; i++; }
이 패턴을 외우면 어떤 반복이든 가능!

3. 추상화(Abstraction):
반복의 핵심은 '몇 번?'과 '매번 뭐 하지?'뿐!
시작/끝/처리 세 가지를 정하면 while 완성!

4. 알고리즘 설계(Algorithm Design):
반복은 모든 알고리즘의 기초! 정렬, 탐색, 시뮬레이션 — 전부 반복!
반복 횟수 = 시간 복잡도의 핵심 (O(n), O(n²)...)
while문을 배우면서 '시간 복잡도'의 기초를 쌓는 거예요! N번 반복 = O(n). 이게 알고리즘의 핵심!

[디버깅] while 버그 시나리오

TOP 1 버그 — 갱신 누락 → 무한 루프
버그: 프로그램이 멈추지 않아요!
int i = 1;
while (i <= 5) {
  printf("%d ", i);
  // i++; 빠짐!
}
→ 1 1 1 1 1 1 1... 영원히!

원인: i가 항상 1이라 조건이 영원히 참
해결: i++; 추가
탈출: Ctrl+C로 강제 종료
교훈: while을 쓸 때 3요소(초기화/조건/갱신) 체크리스트!
흔한 버그 — while 뒤 세미콜론
버그: 아무것도 출력 안 되거나 무한 루프
while (i <= 5);
{
  printf("%d", i); i++;
}

원인: while (i<=5);에서 세미콜론이 '빈 반복문'!
조건이 참이면 아무것도 안 하고 무한 반복!
해결: while 뒤에 세미콜론 절대 금지
교훈: if와 같은 실수! while() 뒤에 ;는 거의 항상 버그!
흔한 버그 — Off-by-one (하나 차이)
버그: 5번 반복해야 하는데 4번 또는 6번 반복
int i = 0;
while (i <= 5) { ... i++; } // 6번! (0,1,2,3,4,5)
int i = 1;
while (i < 5) { ... i++; } // 4번! (1,2,3,4)

해결: 시작값과 조건을 함께 보기!
i=0, i<5 → 5번 (0,1,2,3,4)
i=1, i<=5 → 5번 (1,2,3,4,5)
i=0, i<=5 → 6번! 주의!

교훈: 첫값과 끝값을 종이에 써서 직접 세어 보세요!

10축 심화 연결

printf("합:%d 개수:%d 평균:%.1f\n",sum,cnt,(double)sum/cnt);

[심화] 루프 불변식과 증명

[심화 내용] 루프 불변식(Loop Invariant)
반복이 올바른지 어떻게 증명할까?

1~N 합 while의 불변식:
'i번째 반복 직전에, sum은 1~(i-1)까지의 합'

i=1 직전: sum=0 (1~0의 합 = 0) ✓
i=2 직전: sum=1 (1~1의 합 = 1) ✓
i=3 직전: sum=3 (1~2의 합 = 3) ✓
...
i=n+1 직전: sum=n*(n+1)/2 (1~n의 합) ✓

불변식이 유지되면 '이 while은 올바르다'고 증명!
카운터 변수
반복 횟수를 세는 변수. 보통 i, j, k 사용. 초기화→조건→갱신의 핵심.
무한 루프
조건이 영원히 참인 반복. while(1), for(;;). 서버/게임 메인 루프에 의도적 사용.
off-by-one
하나 더 또는 하나 덜 반복하는 버그. 프로그래머 최대 적! i
센티널 값
반복 종료를 알리는 특수 값. 예: 0 0, -1, EOF. while(1)+break에서 사용.
Ctrl+C
터미널에서 프로그램 강제 종료. SIGINT 시그널 전송. 무한 루프 탈출!
O(n)
N번 반복. 시간 복잡도의 기본. N이 커지면 시간도 비례 증가.
루프 불변식
반복 전/후에 항상 참인 조건. 알고리즘 정확성 증명에 사용. 대회 고급 기법.
가우스
1+2+...+100=5050을 즉시 계산한 천재 수학자. 공식: n(n+1)/2. while O(n)→공식 O(1).
while 반복 = O(n). 공식으로 바꾸면 O(1). 같은 문제도 더 빠르게 풀 수 있다는 걸 기억!
while을 배웠어요! 반복의 시작! 다음에는 더 깔끔하게 반복하는 for문을 배워요!
[활동] while 마스터 체크리스트
while의 3요소를 안다
카운터 변수로 반복 횟수를 제어할 수 있다
무한 루프와 Ctrl+C를 안다
while(1)+break 패턴을 쓸 수 있다
continue로 건너뛰기를 할 수 있다
off-by-one 에러를 주의한다
백준 2739(구구단), 10952(A+B-5)를 풀 수 있다

while
조건이 참인 동안 코드를 반복 실행하는 반복문.
카운터 변수
반복 횟수를 세는 변수. 보통 i 사용. 초기화→조건→갱신 필수.
무한 루프
조건이 영원히 참인 반복. while(1). 의도적 사용 시 break로 탈출.
break
반복문을 즉시 탈출하는 키워드.
continue
현재 회차를 건너뛰고 다음 반복으로 넘어가는 키워드.
off-by-one
반복 횟수가 1회 많거나 적은 버그. i
