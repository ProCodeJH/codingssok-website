while은 좀 번거로워요

while문은 초기화, 조건, 증감이 따로 흩어져 있어요. for문은 이 세 가지를 한 줄에 모아놨어요!

[심화 내용] 루프 불변량 (Loop Invariant)
루프 불변량이란?
루프 실행 중 항상 참인 조건

예: 선형 검색
int search(int *a, int n, int key) {
for (int i = 0; i < n; i++)
if (a[i] == key) return i;
return -1;
}

불변량: "a[0..i-1]에는 key가 없다"
- 초기: i=0, a[0..-1]은 비어있으므로 참
- 유지: a[i]≠key이므로 i++, 불변량 유지
- 종료: i==n이면 전체에 key 없음 증명!

왜 중요한가:
루프의 정확성을 수학적으로 증명!
Dijkstra가 제안. 형식 검증의 기초.
[C 코드] for문 알고리즘 — 이진 탐색
1#include <stdio.h>
2
3// 이진 탐색 (정렬된 배열)
4int binary_search(const int *arr, int n, int key) {
5 int lo = 0, hi = n - 1;
6 // 불변량: key가 있다면 arr[lo..hi]에 있다
7 while (lo <= hi) {
8 int mid = lo + (hi - lo) / 2; // 오버플로우 방지!
9 if (arr[mid] == key) return mid;
10 if (arr[mid] < key) lo = mid + 1;
11 else hi = mid - 1;
12 }
13 return -1;
14}
15
16// lower_bound: key 이상인 첫 위치
17int lower_bound(const int *arr, int n, int key) {
18 int lo = 0, hi = n;
19 while (lo < hi) {
20 int mid = lo + (hi - lo) / 2;
21 if (arr[mid] < key) lo = mid + 1;
22 else hi = mid;
23 }
24 return lo;
25}
26
27int main() {
28 int arr[] = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};
29 int n = 10;
30
31 printf("38 위치: %d\n", binary_search(arr, n, 38));
32 printf("50 위치: %d (없음)\n", binary_search(arr, n, 50));
33 printf("10 이상 첫 위치: arr[%d] = %d\n",
34 lower_bound(arr, n, 10), arr[lower_bound(arr, n, 10)]);
35 return 0;
36}
실행 결과
38 위치: 6 50 위치: -1 (없음) 10 이상 첫 위치: arr[3] = 12
이진 탐색
정렬된 배열에서 O(log n) 검색. 매번 범위 절반. 10억 개에서도 30번이면 찾음! lo+(hi-lo)/2로 오버플로우 방지.
loop invariant
루프 불변량. 루프 실행 중 항상 참. 정확성 증명의 핵심. Dijkstra가 제안. 알고리즘 분석의 기초.
(lo+hi)/2는 오버플로우 위험! lo+(hi-lo)/2가 안전! Java의 Arrays.binarySearch에서도 이 버그가 있었어!

1#include <stdio.h>
2
3void bubble_sort(int *a, int n) {
4 for (int i = 0; i < n-1; i++) {
5 int swapped = 0;
6 for (int j = 0; j < n-1-i; j++) {
7 if (a[j] > a[j+1]) {
8 int t = a[j]; a[j] = a[j+1]; a[j+1] = t;
9 swapped = 1;
10 }
11 }
12 if (!swapped) break; // 이미 정렬됨!
13 }
14}
15
16void selection_sort(int *a, int n) {
17 for (int i = 0; i < n-1; i++) {
18 int min_idx = i;
19 for (int j = i+1; j < n; j++)
20 if (a[j] < a[min_idx]) min_idx = j;
21 if (min_idx != i) {
22 int t = a[i]; a[i] = a[min_idx]; a[min_idx] = t;
23 }
24 }
25}
26
27void insertion_sort(int *a, int n) {
28 for (int i = 1; i < n; i++) {
29 int key = a[i], j = i - 1;
30 while (j >= 0 && a[j] > key) {
31 a[j+1] = a[j];
32 j--;
33 }
34 a[j+1] = key;
35 }
36}

39 printf("%-12s: ", label);
40 for (int i = 0; i < n; i++) printf("%d ", a[i]);
41 printf("\n");
42}

[심화 내용] 정렬 알고리즘 비교
O(n²) 정렬:
버블: 인접 교환. 안정. 가장 느림.
선택: 최솟값 찾기. 불안정. 교환 횟수 최소.
삽입: 카드 정렬. 안정. 거의 정렬된 데이터에 최적.

O(n log n) 정렬:
병합: 분할 정복. 안정. 메모리 O(n).
퀵: 피벗 분할. 불안정. 실전 최강. 최악 O(n²).
힙: 힙 자료구조. 불안정. 최악도 O(n log n).

실무 선택:
qsort() — C 표준 (보통 퀵소트 기반)
Tim Sort — Python/Java (병합+삽입 하이브리드)
버블 정렬
인접 원소 교환 반복. O(n²). 가장 느리지만 이해하기 쉬움. 거의 정렬된 데이터에서 O(n).
안정 정렬
같은 값의 상대 순서 유지. 버블/삽입/병합=안정. 선택/퀵/힙=불안정. 다중 키 정렬 시 중요!
정렬 3대장: 버블(교환), 선택(최솟값), 삽입(카드). 전부 이중 for! 이해하면 qsort도 이해돼!
[활동] for문 마스터 체크리스트
for 세 부분(초기화/조건/증감)을 안다
두 변수 동시 이동(i++, j--)을 쓸 수 있다
포인터 순회(for(p=arr;*p;p++))를 안다
이진 탐색을 구현할 수 있다
버블/선택/삽입 정렬을 구현한다
에라토스테네스의 체를 안다
루프 불변량의 개념을 안다

5개 이상이면 for문 마스터!
off-by-one
1 차이 에러. for에서 가장 흔한 버그! <=vs<, n vs n-1. "펜스 포스트 문제"라고도 함.
sentinel loop
특수값으로 종료. for(;;){x=scanf(...);if(x==EOF)break;} EOF/-1/NULL로 루프 종료.
Collatz
콜라츠 추측. 짝수→n/2, 홀수→3n+1. 항상 1에 도달? 미해결 문제! while 루프로 시뮬레이션.

5void mat_mul(int a[N][N], int b[N][N], int c[N][N]) {

7 for (int j = 0; j < N; j++) {
8 c[i][j] = 0;
9 for (int k = 0; k < N; k++)

12}

41}

행렬 곱셈
O(n³) 삼중 for. c[i][j]=Σa[i][k]*b[k][j]. Strassen: O(n^2.81). 선형 대수의 핵심 연산!
전치 행렬
행과 열을 교환. a[i][j]↔a[j][i]. j=i+1부터 시작해서 대각선 위만 교환. O(n²).
행렬 곱셈 = 삼중 for의 대표! O(n³)이지만 선형 대수, 그래픽스, ML의 핵심 연산!
cache locality
캐시 지역성. 행 우선 순회(a[i][j])가 열 우선(a[j][i])보다 빠름! 메모리 연속 접근이 핵심.
qsort
. C 표준 정렬. qsort(arr, n, sizeof(int), cmp). 비교 함수를 인자로 전달!
for vs while
for: 횟수가 정해진 반복. while: 조건 기반. for는 초기화/조건/증감이 한 줄에!
행렬 순회에서 a[i][j]가 a[j][i]보다 빠른 이유: 메모리가 행 순서로 저장! 캐시 히트율 차이!

29double calc_pi(int terms) {
30 double sum = 0;
31 for (int i = 0; i < terms; i++) {
32 double term = 1.0 / (2*i + 1);
33 sum += (i % 2 == 0) ? term : -term;
34 }
35 return sum * 4;
36}

조화 급수
H(n) = 1 + 1/2 + 1/3 + ... + 1/n. 발산하지만 매우 느리게. H(10^100) ≈ 230. ln(n)+γ에 근사.
라이프니츠 공식
π/4 = 1 - 1/3 + 1/5 - 1/7 + ... 수렴이 느려서 실용적이진 않지만 아름다운 수학!
for문으로 π를 계산! 1-1/3+1/5-1/7+... 100만 항이면 소수점 5자리까지 정확!
SIMD
Single Instruction Multiple Data. 하나의 명령으로 여러 데이터 병렬 처리. for 루프를 벡터화!
OpenMP
#pragma omp parallel for. for 루프를 멀티코어 병렬화! 컴파일러 지시문만으로 병렬 실행.
restrict 포인터
C99. 포인터 aliasing 없음 보장. for 루프 벡터화의 전제 조건. void f(int *restrict a, int *restrict b).
for문 최적화: 1)캐시 지역성 2)루프 불변 코드 이동 3)SIMD 벡터화 4)OpenMP 병렬화!
stride
배열 접근 간격. stride 1이 캐시에 최적. stride가 크면 캐시 미스 증가. 행렬 전치 시 주의!
loop tiling
루프 타일링. 큰 루프를 작은 블록으로 분할. 캐시에 맞는 크기로 데이터 재사용 극대화!
Duff's Device
switch+while 루프 언롤링. case 8: do{ *to++=*from++; case 7:... C의 가장 기괴한 최적화!
for문 마스터 졸업! 기초→정렬→이진탐색→행렬→수열→최적화. 이제 do-while과 break/continue로!
volatile loop
volatile int i; for 루프에서 최적화 방지. 벤치마크/딜레이에 사용. 안 쓰면 컴파일러가 루프 제거!
prefetch
__builtin_prefetch(addr). 데이터를 캐시에 미리 적재. for 루프에서 다음 반복 데이터를 미리 요청!
for문의 완성! 다음은 do-while(최소 1회 실행)과 break/continue(루프 제어). 반복문 3총사!
strength reduction
강도 감소. 곱셈→덧셈으로. i*4 대신 i+=4. 컴파일러가 자동으로 수행. 루프 최적화의 고전!
loop fusion
루프 융합. 같은 범위의 두 for를 하나로. 메모리 접근 횟수 감소. 캐시 효율 증가!
iterator
C++/Java의 반복자. C에서는 포인터+증감. for(p=begin;p!=end;p++) 패턴. 컨테이너 추상화의 기본!
for문 완전 정복! 기초→정렬→탐색→행렬→수열→최적화→병렬. C의 핵심 반복문 마스터!
