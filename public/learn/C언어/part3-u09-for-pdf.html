while의 3요소를 한 줄에!

while은 초기화, 조건, 갱신이 떨어져 있어서 빼먹기 쉬웠죠? for는 이 3개를 한 줄에 모아요!

for의 실행 순서
①초기화 → ②조건 확인 → 참 → 코드 실행 → ③갱신 → ②로 ↩
①초기화 ②조건 확인 ↓ 거짓
①초기화 for 끝!
①은 딱 한 번! ②→코드→③→②→코드→③→... 반복!
①초기화는 딱 한 번만 실행! 그 다음 ②조건→코드→③갱신→②조건→... 를 반복. ②가 거짓이면 끝!
[핵심] for = while의 정리된 버전

for 버전:

  코드;
}

while vs for — 언제 뭘 쓸까?

[핵심] for vs while 판단 기준
for를 쓸 때:
- 반복 횟수가 정해져 있을 때 (N번, 9번...)
- 카운터 변수가 규칙적으로 변할 때 (i++, i+=2...)

while을 쓸 때:
- 반복 횟수를 모를 때 (사용자가 0 입력할 때까지)
- 조건이 복잡할 때 (파일 끝까지, 에러 날 때까지)

비유:
for = '100미터 달리기' (거리 정해짐)
while = '지칠 때까지 달리기' (언제 멈출지 모름)

자주 하는 실수

✕ 잘못된 코드
for (int i = 0; i < 5; i++) printf("A"); printf("B");
B는 for 바깥! 1번만 출력! 중괄호 없으면 한 줄만 반복!
✓ 올바른 코드
for (int i = 0; i < 5; i++) { printf("A"); printf("B"); }
중괄호로 두 줄 모두 반복!
✕ 잘못된 코드
for (int i = 0; i < 5; i++); { printf("%d", i); }
세미콜론! 빈 for + i 미정의 에러!
✓ 올바른 코드
for (int i = 0; i < 5; i++) { printf("%d", i); }
for 뒤에 세미콜론 NO!

[사고력] 4요소 심화 연결
1. 분해(Decomposition):
for(i=0; i세 요소를 정하면 어떤 반복이든 설계 가능!

2. 패턴 인식(Pattern Recognition):
0~N-1: for(i=0; i1~N: for(i=1; i<=n; i++) — 합, 곱 계산
N~1: for(i=n; i>=1; i--) — 역순
짝수: for(i=2; i<=n; i+=2) — 건너뛰기
이 4개 패턴으로 대부분의 반복 커버!

3. 추상화(Abstraction):
for는 while의 추상화! 같은 동작을 더 간결하게.
'반복 횟수'라는 개념을 한 줄로 표현하는 것이 추상화!

4. 알고리즘 설계(Algorithm Design):
for의 증가 패턴이 시간 복잡도를 결정!
i++ → O(n)
i*=2 → O(log n)
이 차이를 이해하면 효율적인 알고리즘 설계 가능!
i++은 O(n), i*=2는 O(log n)! for의 갱신 방식이 알고리즘 효율을 결정해요!
