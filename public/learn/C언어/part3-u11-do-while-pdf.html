while과 반대!

while은 조건을 먼저 확인하고 코드를 실행했죠? do-while은 코드를 먼저 실행하고 조건을 확인해요! 최소 1번은 반드시 실행!
do-while의 흐름
코드 먼저 실행! → 조건 확인 → 참 → 다시 코드로 ↩
코드 먼저 실행! 조건 확인 ↓ 거짓
코드 먼저 do-while 끝!
while과 달리 무조건 1번 실행 → 그다음부터 조건 확인!

while은 "확인 후 실행", do-while은 "실행 후 확인"! 최소 1번은 무조건 실행돼요.

do-while은 최소 1회 실행이 보장되는 유일한 반복문입니다. 입력 검증, 메뉴 시스템에서 핵심!

1#include <stdio.h>
2
3int main(void) {
4 int x = 10;
5
6 // while: 조건 먼저 → 조건이 거짓이면 0번!
7 printf("while: ");

9 printf("%d ", x);
10 x++;
11 }
12 printf("(실행 안 됨!)\n");
13
14 // do-while: 먼저 실행 → 최소 1번!
15 x = 10;
16 printf("do-while: ");
17 do {
18 printf("%d ", x);
19 x++;
20 } while (x < 5);
21 printf("(1번 실행됨!)\n");
22
23 return 0;
24}
실행 결과
while: (실행 안 됨!) do-while: 10 (1번 실행됨!)
x=10인데 x<5가 거짓! while은 0번 실행, do-while은 1번 실행! 이것이 핵심 차이!
[핵심] while vs do-while 차이
while: 조건 → 실행. 조건이 처음부터 거짓이면 0번
do-while: 실행 → 조건. 무조건 최소 1번

비유:
while = '비가 오지 않으면 놀러 가자' (먼저 확인, 비오면 안 감)
do-while = '일단 나가보고, 비오면 돌아와' (먼저 나감!)

do-while로 자릿수 세기

count_digits.c
1#include <stdio.h>
2
3int main(void) {
4 int num;
5 printf("정수 입력: ");
6 scanf("%d", &num);
7
8 int temp = num;
9 if (temp < 0) temp = -temp; // 음수 처리
10
11 int digits = 0;
12 do {
13 digits++;
14 temp /= 10;
15 } while (temp > 0);
16
17 printf("%d는 %d자리 수\n", num, digits);
18 return 0;
19}
실행 결과
정수 입력: 12345 12345는 5자리 수
12줄do: num=0이어도 최소 1자리!
13줄자릿수 카운터 증가
14줄10으로 나누어 마지막 자릿수 제거
15줄0이 될 때까지 반복
0은 1자리
while(temp>0)이면 0일 때 실행 안 됨! do-while이면 최소 1회 → digits=1. 정확!
실행 결과
1~100 숫자 맞히기! 추측: 50 더 큰 수! 추측: 75 더 작은 수! 추측: 63 3번 만에 맞혔어요!
이진 탐색
이 게임의 최적 전략! 항상 중간값을 추측하면 최대 7번(log₂100≈7)에 맞출 수 있어요.

[컴퓨팅 사고력] 선실행 후확인 패턴

[사고력] 4요소 심화 연결
1. 분해(Decomposition):
메뉴 프로그램을 → (1) 메뉴 표시 (2) 입력 (3) 처리 (4) 종료 조건 확인으로 분해
(1)~(3)이 do 블록, (4)가 while 조건!

2. 패턴 인식(Pattern Recognition):
do-while이 적합한 3대 패턴:
① 입력 검증 (올바를 때까지 반복)
② 메뉴 프로그램 (최소 1번 표시)
③ 게임 루프 (최소 1번 플레이)

3. 추상화(Abstraction):
while vs do-while 선택 기준:
'실행 전에 조건을 알 수 있나?' → while
'실행해봐야 조건을 알 수 있나?' → do-while

4. 알고리즘 설계(Algorithm Design):
숫자 맞히기 게임의 최적 전략 = 이진 탐색!
항상 중간값 → log₂(n)번에 확정. O(log n)!

[디버깅] do-while 버그 시나리오

TOP 1 — 세미콜론 빠뜨림
버그: 컴파일 에러
do { ... } while (cond) // 세미콜론 없음!

원인: do-while은 } while (조건); 세미콜론 필수!
해결: 항상 세미콜론 붙이기
교훈: while문(세미콜론 X) vs do-while(세미콜론 O) 구분!
✕ 잘못된 코드
// while로 입력 검증 int n = -1; while (n < 1 || n > 100) { printf("숫자: "); scanf("%d", &n); } // 초기값 -1 필요 (안 자연스러움)
while: 초기값을 인위적으로 설정해야 함
✓ 올바른 코드
// do-while로 입력 검증 int n; do { printf("숫자(1~100): "); scanf("%d", &n); } while (n < 1 || n > 100); // 초기값 불필요! 자연스러움
do-while: 초기값 없이 자연스러운 흐름!
흔한 버그 — 무한 입력 루프
버그: 숫자 대신 문자를 입력하면 무한 반복!
do { scanf("%d",&n); } while (n<0);
→ 'abc' 입력 시 scanf가 계속 실패, n 변경 안 됨!

원인: scanf가 변환 실패 시 입력 버퍼에 잘못된 문자가 남음
해결: scanf 반환값 확인 + getchar() 버퍼 비우기
while(getchar()!='\n');
교훈: 사용자 입력은 항상 예상 밖의 값이 들어올 수 있어요!
흔한 버그 — 조건 반전 실수
버그: 유효한 입력에서도 반복이 안 멈춤
do { scanf("%d",&n); } while (n>=0 && n<=100);
→ 유효한 입력(0~100)에서 계속 반복!

원인: while 조건은 '계속 반복할 조건'이에요!
유효할 때 멈추려면: while (n<0 || n>100)
해결: '잘못된 입력이면 반복' = while (n<0 || n>100)
교훈: do-while 조건은 '반복을 계속하는 조건'!

linear_probe.c
1#include <stdio.h>
2
3#define SIZE 10
4int table[SIZE] = {0};
5
6void insert(int key) {
7 int idx = key % SIZE;
8 do {
9 if (table[idx] == 0) {
10 table[idx] = key;
11 printf("%d → slot %d\n", key, idx);
12 return;
13 }
14 idx = (idx + 1) % SIZE; // 다음 칸
15 } while (idx != key % SIZE); // 한 바퀴 돌면 가득
16 printf("테이블 가득!\n");
17}

do-while
먼저 실행하고 나중에 조건을 확인하는 반복문. 최소 1번 실행 보장.
입력 검증
사용자 입력이 올바른 범위인지 확인하고, 잘못되면 재입력 요구.
세미콜론(;)
do-while의 } while (조건); 뒤에 반드시 필요한 문장 종결자.
난수(random)

이진 탐색
정렬된 범위에서 중간값으로 좁혀가는 탐색. O(log n).

while vs do-while 정리

언제 뿔 쓸까?
while: 0번 실행될 수 있음. 조건 먼저 확인. 예: 파일 읽기(파일이 비었으면 0번)
do-while: 최소 1번 실행. 실행 후 조건 확인. 예: 입력 검증, 메뉴, 게임 루프

암기법: "일단 해 보고 맞는지 확인" = do-while
while(1)
무한 루프의 또 다른 표현. for(;;)와 동일. break로 탈출!
EOF
End Of File. 파일 끝 표시. while((c=getchar())!=EOF) 패턴에서 사용!
종료 조건
do-while에서 가장 중요한 것! 잘못 설정하면 무한 루프. 항상 조건이 false가 될 수 있는지 확인!
do-while의 세미콜론을 잊지 마세요! } while(조건); ← 빼먹으면 에러!

[심화] do-while 매크로 활용

C언어 매크로에서 do-while(0)은 유명한 관용구입니다.

do { ... } while(0) 매크로 패턴
#define SWAP(a,b) do { int t=a; a=b; b=t; } while(0)

왜 do-while(0)?
1. 블록을 하나의 문장으로 만들기 위해
2. if-else에서 안전하게 사용 가능
3. 세미콜론 문제 방지
do-while(0)
매크로 안전 패턴. 여러 줄을 하나의 문장으로 감싸기. 리눅스 커널에서 필수!
do-while(0)은 아무것도 반복 안 해요! 매크로를 안전하게 만드는 트릭이에요.
✕ 잘못된 코드
do { printf("입력: "); scanf("%d", &n); } while (n >= 1 && n <= 100); // 조건이 반대!
조건 반대 = 올바른 값 넣으면 무한 반복!
✓ 올바른 코드
do { printf("입력: "); scanf("%d", &n); } while (n < 1 || n > 100); // 잘못된 입력일 때 반복!
조건: "언제 다시?" = 잘못됐을 때!

do-while 조건은 "언제 반복하나?"를 묻습니다. 입력 검증에서는 잘못된 입력일 때 반복!

센티널 값
특별한 종료 값. 0, -1, EOF 등. do-while에서 센티널을 만나면 종료!
