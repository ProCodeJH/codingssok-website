break — 복습 + 심화

break vs continue
break

continue

이번만 건너뛰고 다음 반복으로!
break= "그만!" / continue= "다음!" 간단하죠?

break는 반복문을 완전히 빠져나가고, continue는 나머지 코드를 건너뛰고 다음 반복으로 갑니다.

[핵심] break는 가장 가까운 반복문 1개만 탈출!
중첩 for에서 안쪽 break는 안쪽 for만 탈출해요!
바깥 for는 계속 돌아요!

for (i=0; i<3; i++) {
  for (j=0; j<3; j++) {
    if (j==1) break; // 안쪽 for만 탈출!
    printf("(%d,%d) ", i, j);
  }
  // break 후 여기로 옴.
}
→ (0,0) (1,0) (2,0)

j=1이면 안쪽 break → 바깥 i 계속!

실생활 속의 break/continue

[실생활] break/continue는 어디에?
break = 멈춤: 음악 정지 버튼, 게임 오버, 검색 완료, 비상 정지
continue = 건너뛴: 노래 스킵, 광고 스킵, 불량품 통과, 조건부 실행

음악 플레이어: 정지(break) vs 다음곡(continue)!
음악 앱에서 ▶ 누르면 반복 재생, ⏹ 누르면 break, ⏭ 누르면 continue!
[핵심] continue = 가드 절의 반복문 버전
함수에서 가드 절(early return)로 중첩을 줄였듯이
반복문에서 continue로 중첩을 줄일 수 있어요!

중첩 깊은 코드:
if (조건1) {
  if (조건2) {
    처리;
  }
}

continue로 평평하게:
if (!조건1) continue;
if (!조건2) continue;
처리;

[컴퓨팅 사고력] 흐름 제어와 알고리즘

[사고력] 4요소 심화 연결
1. 분해(Decomposition):
소수 판별을 → (1) n 유효성 (2) 약수 탐색 (3) 약수 발견 시 조기 종료로 분해
각 단계에 if+return 할당!

2. 패턴 인식(Pattern Recognition):
break 패턴: 센티널 종료, 검색 완료 후 즉시 탈출
continue 패턴: 원하지 않는 값 필터링, 예외 건너뛰기
이중 탈출 패턴: 플래그 변수 또는 goto

3. 추상화(Abstraction):
break = '찾았다/끝내야 한다'
continue = '이건 해당 없다'
코드의 의도를 한 단어로 표현하는 추상화!

4. 알고리즘 설계(Algorithm Design):
조기 종료로 불필요한 반복 제거!
소수 판별: i*i<=n까지만 → √n번. O(√n)!

[디버깅] break/continue 버그 시나리오

흔한 버그 — break가 바깥 for를 탈출 못 함
버그: 이중 for에서 break 했는데 바깥 for가 계속 돌아감

원인: break는 가장 가까운 반복문 1개만 탈출!
해결: 플래그 변수 + 바깥 for 조건에 !flag 추가
또는 goto 사용
교훈: C에는 '이중 break'가 없다!
흔한 버그 — while에서 continue 후 갱신 누락
버그: while에서 continue 후 무한 루프!
int i=0;
while(i<10) {
  if(i==5) continue; // i++ 건너뜀!
  i++;
}
→ i=5에서 영원히 continue!

해결: continue 전에 i++ 하거나 for문 사용
교훈: while + continue 조합은 갱신 위치 주의!
가드 절
함수 시작에서 잘못된 걸 먼저 걸러내기. return, break, continue 활용!
Early Exit
조기 탈출 패턴. 찾으면 break! 불필요한 반복 방지 = 성능 향상!
goto
C의 무조건 점프. 중첩 break 대신 사용 가능하지만 스파게티 코드의 원인!
흔한 버그 — switch 안의 break 혼동
버그: switch 안에서 break를 했는데 반복문이 안 멈춤
while(1) {
  switch(ch) {
    case 3: break; // switch만 탈출!
  }
}

원인: switch의 break는 switch만 탈출! while은 그대로!
해결: 플래그 + while 조건으로 이중 탈출
교훈: switch의 break ≠ 반복문의 break!

10축 심화 연결
[컴퓨팅사고력 — 조기 종료] 소수 판별에서 약수를 찾는 즉시 return. O(n)→O(√n). 불필요한 반복을 줄이는 것이 알고리즘 최적화의 시작!

continue 심화: for vs while 차이

continue는 for와 while에서 동작이 다릅니다! for에서는 갱신식(i++)을 실행하지만, while에서는 조건으로 바로 감니다.

✕ 잘못된 코드
int i = 0; while (i < 10) { if (i == 5) continue; printf("%d ", i); i++; } // i=5에서 i++ 건너뛴 = 무한루프!
while + continue: i++ 건너뛰면 무한!
✓ 올바른 코드
for (int i = 0; i < 10; i++) { if (i == 5) continue; printf("%d ", i); } // for의 i++는 continue해도 실행!
for + continue: i++ 안전하게 실행!
while에서 continue 쓸 때는 갱신식을 continue 위에 두세요! 아니면 무한루프!

break 실전: 무한 루프 + break 패턴

while(1) + break는 종료 조건이 복잡할 때 유용합니다.

infinite_break.c
1#include <stdio.h>
2
3int main(void) {
4 int sum = 0;
5
6 while (1) {
7 int n;
8 printf("숫자(-1=종료): ");
9 scanf("%d", &n);
10
11 if (n == -1) break; // 종료 조건
12 if (n < 0) {
13 printf("음수는 건너뛰\n");
14 continue;
15 }
16
17 sum += n;
18 printf("누적: %d\n", sum);
19 }
20 printf("최종 합계: %d\n", sum);
21 return 0;
22}
실행 결과
숫자(-1=종료): 10 누적: 10 숫자(-1=종료): -3 음수는 건너뛰 숫자(-1=종료): 20 누적: 30 숫자(-1=종료): -1 최종 합계: 30
-1은 break, 음수는 continue, 양수만 합산! break와 continue를 함께 쓰는 패턴!
센티널 값
특별한 종료 표시. -1, 0, EOF 등. while(1)+break에서 종료 신호!
O(n) 검색
선형 검색. 배열을 처음부터 끝까지. break로 조기 탈출하면 평균 O(n/2)!
labeled break
C에는 없지만 Java에는 있음. C에서는 goto나 플래그 변수로 대체!
