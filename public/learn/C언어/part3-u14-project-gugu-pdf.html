[디버깅] 프로젝트 버그

버그 — switch 안 변수 선언
버그: case 안에서 int n; 선언이 에러
원인: C에서 case 안 변수 선언은 중괄호 {} 필요!
해결: case 1: { int n; ... break; }
교훈: switch의 case에서 변수 선언 시 반드시 블록 사용!
버그 — 범위 검증 논리 오류
버그: s>e인데 통과됨
원인: while 조건에 s>e 체크 누락
해결: while(s<2 || e>9 || s>e)
교훈: 검증 조건은 '모든 잘못된 경우'를 ||로 나열!

[심화] 함수 포인터로 구구단

각 모드를 함수로 분리하고, 함수 포인터 배열로 switch를 대체할 수 있습니다.

gugu_func_ptr.c
1#include <stdio.h>
2
3void mode_single(void) {
4 int n;
5 do { printf("단(2~9): "); scanf("%d",&n); } while(n<2||n>9);
6 for (int i=1;i<=9;i++) printf("%dx%d=%2d\n",n,i,n*i);
7}
8void mode_table(void) {
9 for (int i=1;i<=9;i++) {
10 for (int j=2;j<=9;j++) printf("%dx%d=%2d ",j,i,j*i);
11 printf("\n");
12 }
13}
14void mode_reverse(void) {
15 int n; printf("단: "); scanf("%d",&n);
16 for (int i=9;i>=1;i--) printf("%dx%d=%2d\n",n,i,n*i);
17}
18
19typedef void (*ModeFunc)(void);
20
21int main(void) {
22 ModeFunc funcs[] = {mode_single, mode_table, mode_reverse};
23 int mode;
24 do {
25 printf("\n1.특정 2.전체 3.역순 4.종료: ");
26 scanf("%d", &mode);
27 if (mode>=1 && mode<=3) funcs[mode-1]();
28 } while (mode != 4);
29 return 0;
30}
함수 포인터 배열로 switch 없이! funcs[mode-1]()로 직접 호출!
