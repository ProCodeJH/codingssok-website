나머지 연산의 수학. a≡b(mod n). 구구단의 패턴(3단: 3,6,9 반복)은 mod 9의 순환!

5typedef struct { int dan; int num; int answer; int user; int correct; } QuizQ;
6
7void run_quiz(int count) {
8 srand(time(NULL));
9 QuizQ quiz[20];
10 int score = 0;
11
12 printf("=== 구구단 퀴즈 (%d문제) ===\n\n", count);
13 for (int i = 0; i < count && i < 20; i++) {
14 quiz[i].dan = 2 + rand() % 8;
15 quiz[i].num = 1 + rand() % 9;
16 quiz[i].answer = quiz[i].dan * quiz[i].num;
17
18 printf("Q%d. %d × %d = ? ", i+1, quiz[i].dan, quiz[i].num);
19 // 시뮬레이션
20 quiz[i].user = quiz[i].answer;
21 quiz[i].correct = (quiz[i].user == quiz[i].answer);
22 printf("(정답: %d) %s\n", quiz[i].answer, quiz[i].correct ? "✅" : "❌");
23 if (quiz[i].correct) score++;
24 }
25
26 printf("\n결과: %d/%d (%.0f%%)\n", score, count, 100.0*score/count);
27 if (score == count) printf("🏆 만점! 구구단 마스터!\n");
28 else if (score >= count*0.8) printf("👍 잘했어요!\n");
29 else printf("💪 더 연습하세요!\n");
30}

33 run_quiz(10);
34 return 0;
35}

[심화 내용] 곱셈과 컴퓨터 아키텍처
하드웨어 곱셈기:
배열 곱셈기(Array Multiplier): 조합 회로
Booth 곱셈기: 부호 있는 곱셈 최적화
Wallace Tree: 병렬 부분곱 축약

곱셈 지연 시간:
8비트: ~1ns (조합 회로)
32비트: ~3ns (파이프라인)
64비트: ~5ns
부동소수점: ~7ns (정규화 포함)

소프트웨어 곱셈 알고리즘:
학교 방식: O(n²) — n=자릿수
Karatsuba: O(n^1.585)
Toom-Cook: O(n^1.465)
Schönhage-Strassen: O(n log n log log n)
Harvey-van der Hoeven (2019): O(n log n) 이론적 최적!
Wallace Tree
병렬 곱셈 가속기. 부분곱을 트리 형태로 동시 축약. CPU 곱셈기의 핵심 구조!
곱셈의 하한
n자리 곱셈 하한: Ω(n log n) 추측. 2019년 Harvey-van der Hoeven이 O(n log n) 달성!
곱셈 알고리즘의 역사: 학교 방식(4000년)→Karatsuba(1960)→FFT(1971)→O(n log n)(2019)! 수학의 발전!
Russian peasant
러시안 농부 곱셈. 반으로 나누고 두 배로 곱하기. 이진법 기반! 고대 이집트에서도 사용.
multiply-accumulate
MAC 연산. a=a+b*c. DSP/AI 칩의 핵심 연산. 초당 수조 회! TOPS 단위.
overflow detection
곱셈 오버플로우 검출. if(a>INT_MAX/b) overflow! 나눗셈으로 사전 검사. 안전한 곱셈!
구구단 프로젝트 완전 졸업! 4000년 역사의 곱셈을 코드로. 기본에서 CPU 아키텍처까지!
lookup table
룩업 테이블. 계산 결과를 미리 저장. 구구단 표 = 원조 LUT! 삼각함수, CRC에 활용.
memoization
메모이제이션. 계산 결과 캐싱. 구구단 표를 메모리에 → 반복 곱셈 O(1)! DP의 핵심 기법.
Napier rods
네이피어 막대. 1617년 발명. 곱셈을 덧셈으로 변환하는 기계식 계산 도구. 구구단의 물리적 구현!
schoolbook vs Karatsuba
n=32자리: 학교 방식 1024번 곱셈, Karatsuba 약 340번. n이 클수록 차이 극대화!
