조건문과 반복문을 모두 사용해서 완성도 높은 메뉴 프로그램을 만들어요.

📚 이번에 배울 것

  • if/else if/switch로 다중 분기를 만들 수 있다
  • while/for/do-while로 반복할 수 있다
  • break, continue를 활용할 수 있다
  • 누적합, 최댓값 알고리즘을 사용할 수 있다

메뉴 출력

} while (choice != 0)

[심화 내용] 메뉴 시스템과 MVC
MVC 패턴:
Model: 데이터 (MenuItem[])
View: 화면 출력 (printf)
Controller: 입력 처리 (scanf→dispatch)

메뉴 시스템이 MVC인 이유:
데이터(메뉴 항목)와 로직(액션)이 분리
표시 방법(View)은 독립적으로 변경 가능

이벤트 기반 확장:
while(1) {
Event e = get_event();
dispatch(e);
if (e.type == QUIT) break;
}
→ GUI 프레임워크의 기본 구조!

메뉴 시스템은 MVC의 원형! 데이터(모델)→출력(뷰)→입력(컨트롤러). GUI의 기초가 여기 있어!
plugin system
함수 포인터 배열에 동적으로 추가. 런타임 플러그인! dlopen/dlsym으로 .so/.dll 로드.
state machine menu
메뉴 상태를 FSM으로. 현재화면+입력→다음화면. 복잡한 UI 네비게이션 관리!
undo/redo
명령 스택. 실행→push, 취소→pop+reverse. 메뉴 시스템의 고급 기능. Command 패턴!
accessibility
접근성. 키보드만으로 메뉴 탐색. 스크린 리더 호환. UI 설계의 필수 고려사항!

[C 코드] 메뉴 — 히스토리 + 되돌리기
1#include <stdio.h>
2#include <string.h>
3
4#define MAX_HIST 20
5
6typedef struct {
7 char action[50];
8 int timestamp;
9} HistoryEntry;
10
11HistoryEntry history[MAX_HIST];
12int hist_count = 0;
13int time_counter = 0;
14
15void add_history(const char *action) {
16 if (hist_count < MAX_HIST) {
17 strncpy(history[hist_count].action, action, 49);
18 history[hist_count].timestamp = ++time_counter;
19 hist_count++;
20 }
21}
22
23void show_history() {
24 printf("\n=== 작업 히스토리 ===\n");
25 for (int i = 0; i < hist_count; i++)
26 printf("[T%d] %s\n", history[i].timestamp, history[i].action);
27 if (hist_count == 0) printf("(비어 있음)\n");
28}
29
30void undo_last() {
31 if (hist_count > 0) {
32 hist_count--;
33 printf("되돌림: %s\n", history[hist_count].action);
34 } else {
35 printf("되돌릴 작업이 없습니다\n");
36 }
37}
38
39int main() {
40 add_history("파일 열기: data.txt");
41 add_history("텍스트 입력: Hello");
42 add_history("텍스트 입력: World");
43 show_history();
44
45 undo_last();
46 show_history();
47
48 undo_last();
49 show_history();
50 return 0;
51}

undo stack
되돌리기 스택. 실행→push, Ctrl+Z→pop. 모든 에디터의 핵심 기능. Command 패턴으로 구현!

[C 코드] 메뉴 최종 — 설정 관리
1#include <stdio.h>
2#include <string.h>
3
4typedef struct {
5 char key[20];
6 char value[50];
7} Config;
8
9Config configs[10];
10int config_count = 0;
11
12void config_set(const char *key, const char *val) {
13 for (int i = 0; i < config_count; i++) {
14 if (strcmp(configs[i].key, key) == 0) {
15 strncpy(configs[i].value, val, 49);
16 printf("Updated: %s = %s\n", key, val);
17 return;
18 }
19 }
20 if (config_count < 10) {
21 strncpy(configs[config_count].key, key, 19);
22 strncpy(configs[config_count].value, val, 49);
23 config_count++;
24 printf("Added: %s = %s\n", key, val);
25 }
26}
27
28const char* config_get(const char *key) {
29 for (int i = 0; i < config_count; i++)
30 if (strcmp(configs[i].key, key) == 0)
31 return configs[i].value;
32 return "(not found)";
33}
34
35void config_list() {
36 printf("\n=== 설정 목록 ===\n");
37 for (int i = 0; i < config_count; i++)
38 printf(" %s = %s\n", configs[i].key, configs[i].value);
39 if (config_count == 0) printf(" (비어 있음)\n");
40}
41
42int main() {
43 config_set("language", "ko");
44 config_set("theme", "dark");
45 config_set("font_size", "14");
46 config_list();
47
48 printf("\ntheme = %s\n", config_get("theme"));
49
50 config_set("theme", "light");
51 config_list();
52 return 0;
53}

Key-Value 저장소. 가장 간단한 데이터베이스. Redis, 환경변수, 레지스트리가 모두 KV!
설정 관리 = 미니 Key-Value 저장소! get/set/list. 이게 Redis의 원형이야!
CLI framework
CLI 프레임워크. 메뉴→명령줄 인자 파싱. getopt, argparse, commander. 전문 도구의 인터페이스!
readline
GNU Readline. 입력 편집+히스토리+자동완성. bash, Python REPL이 사용. 메뉴 입력의 고급 버전!
메뉴 시스템의 진화: printf→ncurses→GUI→웹. 하지만 핵심은 항상 같아: 입력→처리→출력 루프!
dispatch table
디스패치 테이블. 키→함수 매핑. switch 대체. O(1) 조회. 인터프리터, OS 시스템 콜 핸들러!
Visitor pattern
방문자 패턴. 메뉴 항목을 순회하며 동작 적용. 메뉴 출력/검색/통계를 분리!
