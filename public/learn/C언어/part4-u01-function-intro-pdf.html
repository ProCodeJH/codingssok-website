함수란 무엇인가

이름이 있는 코드 묶음, 함수를 배워요. 왜 필요한지, main()도 함수라는 사실을 이해해요.

📚 이번에 배울 것

  • 함수가 무엇인지 이해해요.
  • 함수를 사용하면 왜 좋은지 설명할 수 있어요.
  • main()도 함수라는 사실을 알아요.
  • 간단한 함수를 직접 만들어 호출할 수 있어요.

✅ 시작 전 체크

  • if문과 반복문을 사용할 수 있다
  • printf와 scanf로 입출력을 할 수 있다
Part 1 진행률: 4%

같은 코드를 또 쓰고 있다면?

프로그램이 길어지면 같은 코드를 여기저기 복사해서 붙이게 돼. 그런데 거기서 버그가 나면? 복사한 곳을 전부 찾아서 고쳐야 해. 끔찍하지?

프로그래밍에서 함수(function)란, 특정 작업을 수행하는 코드 묶음에 이름을 붙인 것이에요. 한 번 만들어 두면, 이름만 불러서 몇 번이든 다시 쓸 수 있어요.

같은 코드 3번 vs 함수 한 번 정의 후 3번 호출

함수가 필요한 세 가지 이유

함수의 3대 장점
함수의 3가지 장점
1. 코드 중복 제거 — 같은 코드를 여러 번 쓰지 않아도 돼요.
2. 가독성 향상 — 함수 이름만 보면 무슨 일을 하는지 바로 알 수 있어요.
3. 유지보수 편리 — 고칠 곳이 함수 하나뿐이에요.

main()도 함수다!

사실 여러분은 이미 함수를 쓰고 있었어요! main()도 함수거든요. C 프로그램은 항상 main 함수에서 시작해요.
C 프로그램 = main() + 우리가 만든 함수들 + 라이브러리 함수

printf()도 함수예요. C 표준 라이브러리에 이미 만들어져 있어서 #include <stdio.h>만 쓰면 바로 사용할 수 있는 거예요. 이번 Part에서는 우리가 직접 함수를 만들어 볼 거예요.

함수의 기본 형태

함수의 구조
반환형 함수이름(매개변수) {
    실행할 코드;
}

반환형: 함수가 돌려줄 값의 자료형 (없으면 void)
함수이름: 우리가 짓는 이름 (동사형 추천: printHello, calculateSum)
매개변수: 외부에서 받을 값 (없으면 void)
중괄호: 함수가 할 일

순수 함수
Pure Function. 같은 입력→항상 같은 출력. 부수 효과 없음. 테스트 쉬움. 함수형 프로그래밍의 핵심!
스택 프레임
함수 호출 시 생성되는 메모리 블록. 지역변수+매개변수+반환주소. LIFO 구조. 스택 오버플로우의 원인!
DRY
Don't Repeat Yourself. 반복 코드를 함수로 추출. 유지보수성 극대화. 함수의 존재 이유!
함수 = 코드의 블록! 이름을 붙이고, 재사용하고, 조합한다. 프로그래밍의 핵심 추상화!
순수 함수 = 수학 함수! f(x)=x². 같은 x면 항상 같은 결과. 버그가 없는 코드의 비밀!

[심화] 함수 호출 규약 (Calling Convention)
cdecl (C 기본):
인자: 오른쪽→왼쪽으로 스택 push
스택 정리: 호출자(caller)가 담당
가변 인자(printf) 지원 가능!

stdcall (Windows API):
스택 정리: 피호출자(callee)가 담당
코드 크기 약간 작음

fastcall:
처음 2개 인자 → 레지스터(ECX, EDX)
나머지 → 스택. 빠름!

System V AMD64 (Linux/Mac 64bit):
처음 6개 정수 인자 → RDI,RSI,RDX,RCX,R8,R9
처음 8개 실수 → XMM0~XMM7
스택 거의 안 씀 → 매우 빠름!

스택 프레임
함수 호출마다 스택에 프레임 생성. 지역변수+리턴주소+인자 저장. 함수 종료 시 프레임 해제!
cdecl
C Declaration. C 기본 호출 규약. 호출자가 스택 정리. printf 같은 가변 인자 함수 지원!
함수 호출 = 스택에 프레임 쌓기! main→func_a→func_b→func_c. 주소가 점점 감소해!
선언이 먼저, 정의는 나중! 프로토타입 덕분에 main을 위에 쓸 수 있어. 가독성 UP!
inline
인라인 함수. 호출 대신 코드 삽입. 오버헤드 제거. C99 inline, 컴파일러가 판단!
variadic
가변 인자. printf(fmt,...). va_list/va_start/va_arg/va_end. 타입 안전하지 않음!
forward declaration
전방 선언. 함수 프로토타입. 정의 전에 호출 가능. 헤더 파일(.h)의 존재 이유!
함수 = 코드의 레고 블록! 작게 만들어서 조립. 한 함수 = 한 가지 일. SRP 원칙!

37int is_armstrong(int n) {
38 int digits = digit_count(n);
39 int sum = 0, orig = n;
40 while (n > 0) {
41 int d = n % 10;
42 int pw = 1; for(int i=0;i<digits;i++) pw *= d;
43 sum += pw;
44 n /= 10;
45 }
46 return sum == orig;

함수 포인터 캐스팅. void(*)(void)로 저장, 호출 시 원래 타입으로 복원. dlsym의 패턴!

중첩 함수. GCC 확장. 바깥 함수의 변수 접근 가능! ISO C에는 없음. 클로저와 유사.

tail call
꼬리 호출. 함수 마지막에 다른 함수 호출. 스택 프레임 재사용 가능! GCC -O2에서 최적화.

TU
Translation Unit. .c + #include 확장 후 하나의 컴파일 단위. .o 파일 1개 생성. 링커가 합침!

tail call
꼬리 호출. 함수의 마지막 동작이 함수 호출. 스택 재사용 가능! -O2에서 최적화. Scheme은 필수 보장.

상호 재귀를 루프로 변환. 함수가 "다음 호출"을 반환 → 루프에서 실행. 스택 오버플로 방지!

코루틴. 중단/재개 가능한 함수. setjmp/longjmp로 C에서 시뮬레이션. C++20 co_await, Python yield.
