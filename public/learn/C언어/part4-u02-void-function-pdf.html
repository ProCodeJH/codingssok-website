void 함수란?

void 함수는 일만 하고 끝나는 함수야. 계산 결과를 돌려주지 않아. 화면에 무언가를 출력하는 함수가 대표적이야.
void_hello.c
1#include <stdio.h>
2
3void printHello(void) {
4 printf("Hello!\n");
5}
6
7int main(void) {
8 printHello();
9 return 0;
10}
실행 결과
Hello!
3줄void printHello(void) — 앞의 void는 '반환값 없음', 괄호 안의 void는 '매개변수 없음'. 이 함수는 받을 것도, 돌려줄 것도 없어요.
4줄함수가 하는 일: Hello!를 화면에 출력. 이것이 이 함수의 전부예요.
8줄printHello(); — main()에서 함수를 호출. 호출하면 함수의 코드가 실행되고, 끝나면 main()으로 돌아와요.

함수 호출 순서

main()에서 호출 -> 함수 실행 -> main()으로 복귀

프로그램은 위에서 아래로 실행돼요. 함수를 호출하면 그 함수의 코드로 이동했다가, 함수가 끝나면 호출한 곳 다음 줄로 돌아와요.

C 컴파일러는 코드를 위에서 아래로 읽어. 그래서 함수를 main() 위에 정의하거나, 프로토타입을 먼저 써줘야 해.

함수 프로토타입

함수를 main() 아래에 정의하고 싶을 때가 있어요. 그러면 컴파일러가 main()을 읽을 때 '이 함수가 뭔데?'하고 에러를 내요. 이때 프로토타입을 써줘요.

선언(프로토타입) -> 호출 -> 정의: void 함수의 3단계
prototype.c
1#include <stdio.h>
2
3// 프로토타입 (선언)
4void printGreeting(void);
5
6int main(void) {
7 printGreeting(); // 호출
8 return 0;
9}
10
11// 정의 (구현)
12void printGreeting(void) {
13 printf("=================\n");
14 printf(" Welcome!\n");
15 printf("=================\n");
16}
실행 결과
================= Welcome! =================
4줄void printGreeting(void); — 프로토타입. '이런 함수가 아래에 있을 거야'라고 컴파일러에게 미리 알려주는 것. 세미콜론으로 끝남!
7줄main()에서 호출. 프로토타입 덕분에 컴파일러가 이 함수를 알고 있어요.
12줄함수 정의는 main() 아래에. 프로토타입이 있으면 정의 위치는 자유로워요.
프로토타입은 세미콜론으로 끝나고 중괄호가 없다!
프로토타입은 '약속'이고, 정의는 '실제 코드'야. 프로토타입에는 중괄호가 없고, 정의에는 중괄호가 있어.

[심화 내용] 순수 함수 vs 부수 효과
순수 함수 (Pure Function):
같은 입력 → 항상 같은 출력
외부 상태 변경 없음
예: int add(int a, int b) { return a+b; }

부수 효과 (Side Effect):
전역 변수 수정, I/O, 파일 쓰기
예: printf, scanf, 배열 수정

왜 중요한가:
순수 함수 → 테스트 쉬움, 병렬화 가능
부수 효과 → 디버깅 어려움, 순서 의존적

함수형 프로그래밍:
부수 효과 최소화 → 버그 감소
Haskell: IO 모나드로 부수 효과 격리
void
void = "없음". 반환값 없는 함수, 타입 없는 포인터(void*). C에서 "범용"의 의미!
side effect
함수 외부 상태 변경. printf=화면 출력, scanf=변수 수정. 순수 함수는 부수 효과 없음!
void 함수 = "일만 하고 돌려줄 건 없어". 출력, 배열 수정, 파일 쓰기 등 부수 효과!
순수 함수 = 테스트의 왕! 같은 입력이면 항상 같은 출력. 버그 추적이 100배 쉬워!
assertion
assert(조건). 조건 거짓이면 프로그램 중단! 디버깅용. NDEBUG 정의 시 무시. 방어적 프로그래밍!

프로시저 = void 함수. Pascal에서 함수와 구분. C에서는 반환타입 void로 통합!
unit test

void 함수는 "일꾼"! 출력, 수정, 초기화 — 결과를 돌려주는 대신 세상을 바꿔!

[심화 내용] 함수 포인터와 OOP
C에서 OOP 시뮬레이션:
struct Animal {
const char *name;
void (*speak)(struct Animal*);
};

void dog_speak(Animal *a) { printf("%s: 멍멍!\n", a->name); }
void cat_speak(Animal *a) { printf("%s: 야옹!\n", a->name); }

다형성:
Animal dog = {"Rex", dog_speak};
Animal cat = {"Nabi", cat_speak};
dog.speak(&dog); // "Rex: 멍멍!"
cat.speak(&cat); // "Nabi: 야옹!"

이것이 C++의 virtual 함수 원형!

싱크 함수. 데이터를 받아서 소비만. printf, write, send. 반환값 없이 외부로 출력!

초기화 함수. void init(). 전역 상태 설정. main 시작 전 __attribute__((constructor))로 자동 호출!
void = 돌려줄 건 없지만 세상을 바꾼다! 출력, 수정, 초기화. 부수 효과의 세계!

초기화 함수. void init_system(). 전역 상태 설정, 하드웨어 초기화. 프로그램 시작의 첫 호출!
void = "말없는 일꾼"! 결과를 돌려주는 대신 세상을 바꿔. 출력, 수정, 초기화 — 행동의 함수!

미들웨어. void 함수 체인. 요청→핸들러1→핸들러2→응답. Express.js, Django의 핵심 구조!

훅. 특정 시점에 호출되는 콜백. before/after 패턴. Git hooks, React hooks의 원형!
void 함수 = 세상을 바꾸는 함수! 출력, 수정, 알림, 로깅. 반환값 없이도 강력!
void
'없음'이라는 뜻. 반환값이 없는 함수 앞에 붙이거나, 매개변수가 없을 때 괄호 안에 써요.
함수 프로토타입
함수의 반환형, 이름, 매개변수를 미리 선언하는 것. 컴파일러에게 '이런 함수가 있을 거야'라고 알려줌.
함수 선언(declaration)
프로토타입과 같은 말. 함수의 존재를 알리는 것.
함수 정의(definition)
함수가 실제로 하는 일을 코드로 작성하는 것. 중괄호 안에 코드가 있음.
implicit declaration
함수를 선언하지 않고 사용했을 때 나오는 컴파일러 경고 메시지.
