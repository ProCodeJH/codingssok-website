함수 안에서 만든 변수를 다른 함수에서 쓰려고 하면 에러가 나. 왜 그런지 알아보자!

main()의 x와 test()의 x는 이름만 같을 뿐, 완전히 다른 변수야! 각자 자기 함수 안에서만 살아있어.

지역 변수 (Local Variable)

중괄호 {}가 변수의 생존 범위를 결정한다
지역 변수의 특징
1. 함수(또는 블록) 안에서 선언된 변수
2. 해당 함수(또는 블록)가 끝나면 사라짐
3. 다른 함수에서는 접근할 수 없음
4. 함수가 호출될 때마다 새로 만들어짐

같은 이름, 다른 변수

이름이 같아도 다른 함수의 지역 변수는 서로 모른다!

전역 변수 (Global Variable)

모든 함수 바깥에서 선언한 변수를 전역 변수라고 해요. 어떤 함수에서든 접근할 수 있어요. 하지만 위험해요!

global_var.c
1#include <stdio.h>
2
3int count = 0; // 전역 변수
4
5void increment(void) {
6 count++;
7 printf("increment: count = %d\n", count);
8}
9
10int main(void) {
11 printf("main: count = %d\n", count);
12 increment();
13 increment();
14 printf("main: count = %d\n", count);
15 return 0;
16}
실행 결과
main: count = 0 increment: count = 1 increment: count = 2 main: count = 2
전역 변수는 편하지만 위험! 지역 변수를 우선 사용하자
전역 변수는 편해 보이지만, 프로그램이 커지면 '누가 이 값을 바꿨지?'라는 버그를 찾기 정말 어려워져. 가능하면 지역 변수를 쓰자!
전역 변수를 피해야 하는 이유
1. 어떤 함수가 값을 바꿨는지 추적이 어려움
2. 함수 간 이름 충돌 위험
3. 프로그램이 커질수록 버그 발생 확률 증가
4. KOI 등 대회에서도 전역 변수 남용은 감점 요인이 될 수 있어요

디버깅: '변수가 없다' 에러

스코프를 모르면 자주 만나는 에러가 있어. 'undeclared identifier' -- 이건 컴파일러가 '그 변수 여기서 못 찾겠어!'라고 하는 거야.
✕ 잘못된 코드
void setScore(void) { int score = 95; } int main(void) { setScore(); printf("%d", score); }
에러! score는 setScore 안의 지역 변수라 main에서 못 써요
✓ 올바른 코드
int getScore(void) { return 95; } int main(void) { int score = getScore(); printf("%d", score); }
return으로 값을 받아오면 OK!

용어 정리

[심화] 변수 스코프와 수명 심화

[심화] C의 4가지 저장 기간
1. 자동(auto):
지역 변수 기본. 블록 진입 시 생성, 탈출 시 소멸
스택에 할당. 초기화 안 하면 쓰레기 값!

2. 정적(static):
프로그램 시작 시 생성, 종료 시 소멸
데이터 영역. 초기화: 0 (명시 안 해도)
static int count = 0; → 호출 간 값 유지

3. 스레드(thread_local, C11):
스레드별 독립 복사본
_Thread_local int x; (C11)

4. 동적(malloc):
프로그래머가 직접 관리
힙에 할당. malloc/free

링크(Linkage):
external: 다른 파일에서 접근 가능 (기본)
internal: static 전역 → 현재 파일만
none: 지역 변수 → 해당 블록만

shadowing
이름 가리기. 내부 블록의 변수가 외부를 숨김. -Wshadow 옵션으로 경고. 버그의 원인!
BSS segment
초기화되지 않은 전역/static 변수 영역. 0으로 초기화 보장. 실행 파일 크기에 포함 안 됨!
static 지역변수 = 호출 간 기억! 카운터, 캐시에 유용. 하지만 멀티스레드에서는 위험!
변수의 수명 4가지: auto(블록)→static(프로그램)→thread_local(스레드)→malloc(수동)!

[심화 내용] 저장 기간과 링크
4가지 저장 기간:
auto (기본): 블록 진입 시 생성, 탈출 시 소멸
static: 프로그램 시작~종료. 한 번만 초기화
register: CPU 레지스터 배치 힌트 (현대엔 무시)
_Thread_local (C11): 스레드별 독립 복사본

3가지 링크:
외부(external): 다른 파일에서 접근 가능 (기본 전역)
내부(internal): static 전역 → 파일 내부만
없음(none): 지역 변수

메모리 배치:
전역/static → .data/.bss 섹션
지역 → 스택
동적 → 힙 (malloc)
shadowing
내부 스코프의 변수가 외부를 숨김. -Wshadow 경고로 실수 방지. Rust는 의도적 shadowing 허용!
_Thread_local
C11 스레드 로컬 저장소. 각 스레드가 독립 복사본 보유. errno가 대표적 TLS 변수!
static 지역변수 = "기억하는 함수"! 호출 횟수, 캐시, 한 번만 초기화에 활용!
static 전역 = "비공개 전역"! 다른 .c에서 접근 불가. 캡슐화의 시작!
namespace
이름 공간. C에는 없음! static으로 파일 스코프 제한. C++의 namespace, Java의 package가 대안.
BSS segment
초기화되지 않은 전역/static 변수. 0으로 자동 초기화. 파일에 공간 차지 안 함!
lifetime
변수의 생명주기. auto=블록, static=프로그램 전체, dynamic=free까지. 메모리 관리의 핵심!
스코프 = 변수의 영역! 좁을수록 안전. 전역은 편하지만 위험. 지역 변수 우선!
[C 코드] 스코프 — 캡슐화 패턴
1#include <stdio.h>
2
3// 모듈 패턴: static으로 내부 상태 숨기기

5
6void counter_increment() { counter_value++; }
7void counter_decrement() { counter_value--; }
8int counter_get() { return counter_value; }
9void counter_reset() { counter_value = 0; }
10
11// 싱글턴 패턴 (static 지역변수)
12typedef struct { int id; char name[20]; } Config;
13
14Config* get_config() {
15 static Config instance = {0, "default"}; // 한 번만 생성!
16 static int initialized = 0;
17 if (!initialized) {
18 instance.id = 1;
19 snprintf(instance.name, 20, "production");
20 initialized = 1;
21 }
22 return &instance;
23}
24

27 counter_increment();

31 printf("Counter: %d\n", counter_get());
32
33 Config *cfg = get_config();
34 printf("Config: id=%d, name=%s\n", cfg->id, cfg->name);

singleton
싱글턴. 인스턴스가 하나만 존재. static 지역변수로 C에서 구현! 설정, 로거에 활용.
static = C의 캡슐화! 파일 스코프로 내부 상태 숨기고 함수로만 접근. OOP 없이 정보 은닉!
[C 코드] 스코프 — 메모리 영역 실험
1#include <stdio.h>
2#include <stdlib.h>
3
4int global_var = 42; // .data
5int global_zero; // .bss (0으로 초기화)
6static int file_static = 100; // .data (파일 스코프)
7const int CONSTANT = 999; // .rodata
8
9void show_memory_layout() {
10 int stack_var = 10;
11 static int func_static = 20;
12 int *heap_var = (int*)malloc(sizeof(int));
13 *heap_var = 30;
14
15 printf("=== 메모리 영역 ===\n");
16 printf("코드(.text): %p (show_memory_layout)\n", (void*)show_memory_layout);
17 printf("상수(.rodata): %p (CONSTANT=%d)\n", (void*)&CONSTANT, CONSTANT);
18 printf("전역(.data): %p (global_var=%d)\n", (void*)&global_var, global_var);
19 printf("BSS(.bss): %p (global_zero=%d)\n", (void*)&global_zero, global_zero);
20 printf("static(.data): %p (file_static=%d)\n", (void*)&file_static, file_static);
21 printf("func static: %p (func_static=%d)\n", (void*)&func_static, func_static);
22 printf("힙(heap): %p (heap_var=%d)\n", (void*)heap_var, *heap_var);
23 printf("스택(stack): %p (stack_var=%d)\n", (void*)&stack_var, stack_var);
24
25 printf("\n주소 순서: text < rodata < data/bss < heap ... stack\n");
26 free(heap_var);
27}
28
29int main() {
30 show_memory_layout();
31 return 0;
32}

싱글턴 패턴을 구현한다

translation unit
번역 단위. #include 후 하나의 .c 파일. 컴파일의 기본 단위. static = 번역 단위 내부!
linkage
링크. external(전역), internal(static), none(지역). 심볼의 가시성 제어!
스코프 완전 졸업! 블록→함수→파일→프로그램. 변수의 영역과 생명주기를 완벽히 이해!
[C 코드] 스코프 — 간이 메모리 풀
1#include <stdio.h>
2#include <string.h>
3
4#define POOL_SIZE 1024
5
6static char pool[POOL_SIZE]; // 파일 스코프!
7static int pool_offset = 0;
8
9void* pool_alloc(int size) {
10 size = (size + 7) & ~7; // 8바이트 정렬
11 if (pool_offset + size > POOL_SIZE) return NULL;
12 void *ptr = &pool[pool_offset];
13 pool_offset += size;
14 return ptr;
15}
16
17void pool_reset() { pool_offset = 0; }
18int pool_used() { return pool_offset; }
19
20int main() {
21 int *a = (int*)pool_alloc(sizeof(int)*5);
22 for (int i = 0; i < 5; i++) a[i] = (i+1)*10;
23
24 char *s = (char*)pool_alloc(20);
25 strcpy(s, "Hello Pool!");
26
27 printf("배열: "); for(int i=0;i<5;i++) printf("%d ",a[i]);
28 printf("\n문자열: %s\n", s);
29 printf("사용: %d/%d bytes\n", pool_used(), POOL_SIZE);
30
31 pool_reset();
32 printf("리셋 후: %d/%d bytes\n", pool_used(), POOL_SIZE);
33 return 0;
34}
실행 결과
배열: 10 20 30 40 50 문자열: Hello Pool! 사용: 64/1024 bytes 리셋 후: 0/1024 bytes
memory pool
메모리 풀. 큰 블록을 미리 할당→작은 조각 분배. malloc 오버헤드 제거! 게임 엔진, 임베디드!
ASLR
Address Space Layout Randomization. 메모리 영역 주소를 랜덤화. 보안! 실행마다 주소가 다름.
stack canary
스택 카나리. 버퍼 오버플로 탐지. 리턴 주소 앞에 랜덤 값 배치. 변경되면 abort!
arena allocator
아레나 할당. 큰 블록에서 순차 할당, 한번에 해제. 메모리 풀의 일종. 게임, 컴파일러에 활용!
스코프 완전 졸업! 블록→파일→캡슐화→싱글턴→메모리풀→메모리맵. 메모리의 달인!
[심화 내용] 동적 메모리 관리
malloc/free의 내부:
brk/mmap으로 OS에서 메모리 요청
Free list로 빈 블록 관리
Best-fit/First-fit/Buddy 할당

가비지 컬렉션:
C에는 없음! Boehm GC로 추가 가능
Java/Go/Python은 자동 관리
brk/sbrk
힙 끝을 이동하는 시스템 콜. malloc의 내부. 큰 할당은 mmap 사용! strace로 확인 가능.
TU-local
번역 단위 로컬. static 함수/변수. 심볼이 .o 파일 밖으로 노출 안 됨! nm에서 t/d(소문자).
common symbol
공통 심볼. 초기화 없는 전역 int x; 여러 파일에 있으면 하나로 합침. -fno-common으로 방지!
__thread
GCC 스레드 로컬. __thread int x; 각 스레드별 복사본. C11 _Thread_local과 동일!
global state
전역 상태. 프로그램 전체에서 접근. 편하지만 위험! 테스트 어려움, 스레드 안전 문제.
encapsulation
캡슐화. static으로 내부 숨기기. 함수만 공개. C의 정보 은닉! .h에 선언만 노출.
스코프 규칙: 좁을수록 안전! 전역→파일→함수→블록. 변수의 영향 범위를 최소화!
scope resolution
스코프 해석. 이름 충돌 시 가장 가까운 스코프 선택. C++: ::로 명시. C: 파일 분리가 유일한 수단!
information hiding
정보 은닉. 구현 세부사항을 숨김. .h에 인터페이스만 공개. OOP의 핵심 원칙!
compilation firewall
컴파일 방화벽. Pimpl 패턴. 구조체 정의를 .c에 숨기고 .h에 포인터만. 빌드 의존성 감소!
Pimpl = C의 캡슐화 끝판왕! .h에 struct Foo;만 선언, .c에 정의 숨기기!
ODR
One Definition Rule. 정의는 프로그램에 하나만! 링크 에러의 흔한 원인. static으로 방지!
TLS
Thread Local Storage. 스레드별 독립 변수. _Thread_local int x; 또는 __thread. errno가 대표!
objcopy
바이너리 섹션 조작. .data, .text 추출/수정. 펌웨어, 리소스 임베딩에 활용!
linker script
링커 스크립트. 섹션 배치 제어. 임베디드에서 ROM/RAM 매핑. SECTIONS { .text : { *(.text) } }
스코프 = 변수의 세계관! 블록 안에서 태어나고 블록 밖에서 사라져. 좁을수록 안전!
closure simulation
C 클로저 시뮬레이션. 함수 포인터 + void* context 구조체. 환경을 명시적으로 전달!
data segment
데이터 세그먼트. .data=초기화된 전역, .bss=0초기화 전역. 프로그램 시작 시 로드!
stack frame layout
스택 프레임: [인자][리턴주소][저장 레지스터][지역변수]. 아래로 자라남. RBP로 프레임 추적!
스코프 = 변수의 세계관! 좁을수록 안전, 넓을수록 편리. 적절한 균형이 핵심!
스코프(scope)
변수가 접근 가능한 범위. 중괄호 {}로 결정된다.
지역 변수(local variable)
함수(또는 블록) 안에서 선언된 변수. 해당 범위가 끝나면 사라짐.
전역 변수(global variable)
모든 함수 바깥에서 선언된 변수. 프로그램 어디서든 접근 가능.
섀도잉(shadowing)
지역 변수가 같은 이름의 전역 변수를 가리는 현상. 해당 블록 안에서는 지역 변수가 우선.
undeclared identifier
선언되지 않은 변수를 사용했을 때 나오는 컴파일 에러 메시지.
이번 유닛 핵심 정리
1
지역 변수는 함수 안에서만 살아있고, 함수 끝나면 사라진다
2
전역 변수는 어디서든 접근 가능하지만 남용 금지
3
같은 이름의 변수도 다른 함수에 있으면 별개의 변수
4
중괄호 {}가 변수의 생존 범위(스코프)를 결정한다
