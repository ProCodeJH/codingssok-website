왜 리팩토링할까?

Part 2에서 만든 계산기 기억나? 그때는 main()에 모든 코드를 넣었지. 이제 함수를 알았으니, 각 연산을 함수로 분리해 보자!
리팩토링: 같은 기능인데 구조가 훨씬 깔끔해진다
리팩토링이란?
리팩토링(Refactoring)이란 프로그램의 동작은 그대로 두고, 코드의 구조를 개선하는 것이에요. 더 읽기 쉽고, 고치기 쉽고, 확장하기 쉬운 코드로 바꾸는 거예요.

5// 이항 연산

함수 포인터 테이블
연산을 데이터로 관리. 새 연산 추가 = 배열에 한 줄. switch/if-else 수정 불필요!
NaN
Not a Number. 0/0, sqrt(-1)의 결과. NaN≠NaN (자기 자신과도 같지 않음!). IEEE 754 규정.
함수 포인터 계산기 = 확장의 예술! 새 연산 추가는 함수 1개 + 배열 1줄. 코드 수정 제로!
단항+이항 연산을 테이블로! sqrt, sin, cos도 함수 포인터 배열. 과학 계산기 완성!

[심화 내용] 함수 포인터 디스패치 테이블
디스패치 테이블:
연산자 문자열 → 함수 매핑
switch 대신 배열 검색 → O(n)
해시 테이블 사용 → O(1)

장점:
1. 새 연산 추가 = 배열에 1줄
2. switch/if-else 수정 불필요
3. 런타임 동적 추가 가능!

인터프리터: opcode → handler

함수 포인터
typedef double (*Op)(double,double); 함수를 변수처럼! C에서 다형성, 콜백, 이벤트의 핵심!
qsort
C 표준 정렬. 비교 함수를 함수 포인터로 전달! qsort(arr,n,size,compare); 다형성의 원형!
함수 포인터 = C의 다형성! 같은 인터페이스, 다른 구현. OOP 없이도 유연한 설계!
계산기를 함수 포인터로! 새 연산 추가 = 함수 1개 + 배열 1줄. switch 지옥 탈출!
Shunting Yard
Dijkstra의 변환 알고리즘. 중위→후위 표기. 스택 사용. 계산기 구현의 핵심!
RPN
Reverse Polish Notation. 후위 표기. 3 4 + = 7. 괄호 불필요! HP 계산기, Forth 언어.
lexer
렉서/토크나이저. 문자열→토큰 변환. "3+4" → [3, +, 4]. 컴파일러 프론트엔드 첫 단계!
함수형 계산기 = 함수 포인터의 진수! 연산자→함수 매핑으로 확장 가능한 설계!

[C 코드] 계산기 — 후위 표기법 계산
1#include <stdio.h>
2#include <ctype.h>
3#include <stdlib.h>
4
5double stack[100];
6int top = -1;
7
8void push(double val) { stack[++top] = val; }
9double pop() { return stack[top--]; }
10
11// 후위 표기법(RPN) 계산기
12double eval_rpn(const char *expr) {
13 top = -1;
14 char buf[256];
15 int bi = 0;
16
17 for (int i = 0; expr[i]; i++) {
18 if (isdigit(expr[i]) || expr[i] == '.') {
19 buf[bi++] = expr[i];
20 } else if (expr[i] == ' ' && bi > 0) {
21 buf[bi] = 0; bi = 0;
22 push(atof(buf));
23 } else if (expr[i] == '+' || expr[i] == '-' ||
24 expr[i] == '*' || expr[i] == '/') {
25 if (bi > 0) { buf[bi]=0; bi=0; push(atof(buf)); }
26 double b = pop(), a = pop();
27 switch(expr[i]) {
28 case '+': push(a+b); break;
29 case '-': push(a-b); break;
30 case '*': push(a*b); break;
31 case '/': push(b!=0?a/b:0); break;
32 }
33 }
34 }
35 if (bi > 0) { buf[bi]=0; push(atof(buf)); }
36 return pop();
37}
38
39int main() {
40 printf("3 4 + = %.1f\n", eval_rpn("3 4 +"));
41 printf("5 1 2 + 4 * + 3 - = %.1f\n", eval_rpn("5 1 2 + 4 * + 3 -"));
42 printf("2 3 * 4 5 * + = %.1f\n", eval_rpn("2 3 * 4 5 * +"));
43 return 0;
44}

AST
Abstract Syntax Tree. 수식을 트리로. 1+2*3 → +(1, *(2,3)). 컴파일러의 핵심 자료구조!
precedence climbing
연산자 우선순위 파싱. 재귀 하강의 변형. Shunting Yard보다 직관적!
계산기 완전 졸업! 기본→함수포인터→히스토리→RPN. 실전 프로젝트의 완성!
[C 코드] 계산기 — 단위 변환기
1#include <stdio.h>
2
3typedef struct { const char *from; const char *to; double factor; } Conversion;
4
5Conversion conversions[] = {
6 {"km","mile",0.621371}, {"mile","km",1.60934},
7 {"kg","lb",2.20462}, {"lb","kg",0.453592},
8 {"celsius","fahrenheit",0}, {"fahrenheit","celsius",0},
9 {"cm","inch",0.393701}, {"inch","cm",2.54},
10 {"liter","gallon",0.264172}, {"gallon","liter",3.78541}
11};
12int nconv = 10;
13
14#include <string.h>
15double convert(double val, const char *from, const char *to) {
16 if (strcmp(from,"celsius")==0 && strcmp(to,"fahrenheit")==0)
17 return val * 9.0/5.0 + 32;
18 if (strcmp(from,"fahrenheit")==0 && strcmp(to,"celsius")==0)
19 return (val - 32) * 5.0/9.0;
20 for (int i = 0; i < nconv; i++)
21 if (strcmp(conversions[i].from,from)==0 && strcmp(conversions[i].to,to)==0)
22 return val * conversions[i].factor;
23 return -1;
24}
25
26int main() {
27 printf("10 km = %.2f mile\n", convert(10,"km","mile"));
28 printf("100 lb = %.2f kg\n", convert(100,"lb","kg"));
29 printf("37 °C = %.1f °F\n", convert(37,"celsius","fahrenheit"));
30 printf("212 °F = %.1f °C\n", convert(212,"fahrenheit","celsius"));
31 printf("30 cm = %.2f inch\n", convert(30,"cm","inch"));
32 return 0;
33}

IEEE 754
부동소수점 표준. float=32비트, double=64비트. NaN, Inf, -0 표현. 정밀도 한계!
PEMDAS
연산 우선순위. Parentheses, Exponents, Multiply/Divide, Add/Subtract. 계산기의 핵심!
bc
Unix 계산기. 임의 정밀도. echo "2^100" | bc. 정수 오버플로 없음!
계산기 완전 졸업! 기본→함수포인터→RPN→단위변환→히스토리. 실전 프로젝트의 달인!
[심화 내용] 계산기와 컴파일러
계산기 → 컴파일러 파이프라인:
1. 렉서: "3+4*2" → [3, +, 4, *, 2]
2. 파서: 토큰 → AST (추상 구문 트리)
3. 평가: AST 순회 → 결과

AST 예시 (3+4*2):
+
/ \
3 *
/ \
4 2

이것이 프로그래밍 언어의 시작!
산술 → 변수 → 조건 → 함수 → 완전한 언어
parser combinator
파서 조합자. 작은 파서를 조합해서 복잡한 파서 생성. 함수 합성의 실전! Haskell Parsec.
operator overloading
연산자 오버로딩. C++: a+b를 커스텀. C에서는 함수 포인터로 시뮬레이션! add(a,b)
arbitrary precision
임의 정밀도. GMP 라이브러리. 수백 자리 수 계산! RSA, 과학 계산에 필수.
postfix evaluation
후위 평가. 스택 하나로 충분! 숫자→push, 연산자→pop 2개→계산→push. O(n) 시간.
tokenizer
토크나이저. 문자열→토큰 변환. "3+4"→[3,+,4]. strtok이 간단한 토크나이저!
eval
동적 코드 실행. JavaScript eval(), Python eval(). C에는 없음! 인터프리터 직접 구현.
계산기 = 미니 프로그래밍 언어! 렉서→파서→평가기. 컴파일러의 축소판!
REPL
Read-Eval-Print Loop. 입력→평가→출력 반복. Python 대화형, Node.js REPL. 계산기가 바로 REPL!
infix to postfix
중위→후위 변환. Shunting Yard. 연산자 스택+출력 큐. 괄호/우선순위 처리! Dijkstra 발명.
symbolic computation
기호 계산. 수식을 숫자가 아닌 심볼로 처리. 미분, 적분, 단순화. Mathematica, SymPy!
계산기 REPL = 프로그래밍 언어의 원형! 여기에 변수와 조건을 추가하면 인터프리터!
BODMAS
영국식 연산 우선순위. Brackets,Orders,Division,Multiplication,Addition,Subtraction. PEMDAS와 동일!
expression tree
수식 트리. 내부 노드=연산자, 잎=피연산자. 후위 순회=RPN. 중위 순회=일반 수식!
arbitrary precision
임의 정밀도. 자릿수 제한 없는 연산. GMP 라이브러리. Python int가 기본 지원!
calculator TDD
테스트 주도 개발. assert(calc("2+3")==5); 테스트 먼저 → 구현. 버그 없는 계산기!
계산기 = 미니 인터프리터! 렉서→파서→평가. 이게 프로그래밍 언어 만들기의 첫걸음!
REPL
Read-Eval-Print Loop. 대화형 실행 환경. Python, Node.js 대화형 모드. 계산기도 REPL!
operator overloading
연산자 오버로딩. C++에서 +,-,* 재정의. C에서는 함수로 대체: complex_add(a,b).
arbitrary precision
임의 정밀도 연산. 자릿수 제한 없음! GMP 라이브러리. Python은 기본 지원. 암호학에 필수!
계산기 프로젝트 = 함수의 종합예술! 디스패치+히스토리+RPN+단위변환. 실전의 맛!
리팩토링(Refactoring)
프로그램의 동작은 유지하면서 코드의 구조를 개선하는 작업.
모듈화
프로그램을 독립적인 기능 단위(함수, 모듈)로 나누는 것.
형변환(type casting)
자료형을 강제로 바꾸는 것. (double)a는 정수 a를 실수로 바꿈.
0으로 나누기 방어
나눗셈 전에 나누는 수가 0인지 확인하는 안전 장치.
