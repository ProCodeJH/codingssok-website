함수가 자기 자신을 부른다고?

두 장의 거울 사이에 서면 내 모습이 끝없이 반복되지? 재귀 함수도 비슷해. 함수가 자기 자신을 호출하는 거야!

재귀 함수(recursive function)란, 함수 안에서 자기 자신을 다시 호출하는 함수예요. 복잡한 문제를 같은 구조의 작은 문제로 쪼갤 때 유용해요.

countdown.c
1#include <stdio.h>
2
3void countdown(int n) {
4 if (n <= 0) {
5 printf("Go!\n");
6 return; // 종료 조건 (base case)
7 }
8 printf("%d...\n", n);
9 countdown(n - 1); // 자기 자신 호출
10}
11
12int main(void) {
13 countdown(3);
14 return 0;
15}
실행 결과
3... 2... 1... Go!
4줄if (n <= 0) — 종료 조건(base case). 이 조건이 없으면 함수가 영원히 자기 자신을 호출해요!
9줄countdown(n - 1); — 자기 자신을 호출하되, n을 1 줄여서 호출해요. 문제가 점점 작아져요.

팩토리얼 (n!)

팩토리얼의 재귀 정의
n! = n x (n-1) x (n-2) x ... x 2 x 1

재귀로 표현하면:
- factorial(1) = 1 (종료 조건)
- factorial(n) = n x factorial(n-1) (재귀 호출)
factorial.c
1#include <stdio.h>
2
3int factorial(int n) {
4 if (n <= 1) {
5 return 1; // base case
6 }
7 return n * factorial(n - 1); // recursive case
8}
9
10int main(void) {
11 printf("3! = %d\n", factorial(3));
12 printf("5! = %d\n", factorial(5));
13 printf("10! = %d\n", factorial(10));
14 return 0;
15}
실행 결과
3! = 6 5! = 120 10! = 3628800
재귀 호출: 내려가면서 호출하고, 올라오면서 계산한다
재귀는 두 단계야. 내려가기: 종료 조건까지 호출. 올라오기: 결과를 하나씩 계산해서 돌려줌. factorial(4) = 4 x 3 x 2 x 1 = 24!

종료 조건이 없으면?

종료 조건 없는 재귀 = 무한 호출 = 스택 오버플로우!
infinite_recursion.c
1#include <stdio.h>
2
3// 절대 이렇게 하지 마세요!
4void bad(int n) {
5 printf("%d\n", n);
6 bad(n + 1); // 종료 조건 없음!
7}
8
9int main(void) {
10 bad(1); // 스택 오버플로우 발생!
11 return 0;
12}
함수를 호출할 때마다 스택 메모리에 정보가 쌓여. 종료 조건이 없으면 스택이 꽉 차서 프로그램이 죽어 버려. 이게 스택 오버플로우(stack overflow)야.
재귀 함수의 필수 조건
1. 종료 조건(base case): 더 이상 재귀 호출을 하지 않는 조건. 반드시 있어야 함!
2. 수렴: 매번 호출할 때마다 종료 조건에 가까워져야 함. (n이 줄어드는 등)

피보나치 수열

피보나치 수열: 1, 1, 2, 3, 5, 8, 13, 21, ...
규칙: 앞의 두 수를 더하면 다음 수가 나온다. fib(n) = fib(n-1) + fib(n-2)

fibonacci.c
1#include <stdio.h>
2
3int fib(int n) {
4 if (n <= 1) return n; // fib(0)=0, fib(1)=1
5 return fib(n - 1) + fib(n - 2);
6}
7
8int main(void) {
9 for (int i = 1; i <= 10; i++) {
10 printf("fib(%d) = %d\n", i, fib(i));
11 }
12 return 0;
13}
실행 결과
fib(1) = 1 fib(2) = 1 fib(3) = 2 fib(4) = 3 fib(5) = 5 fib(6) = 8 fib(7) = 13 fib(8) = 21 fib(9) = 34 fib(10) = 55
피보나치 재귀는 같은 계산을 여러 번 반복해서 비효율적!
피보나치 재귀는 이해하기 쉽지만 느려. fib(30)만 해도 한참 걸려. 나중에 배우는 동적 프로그래밍(DP)으로 해결할 수 있어!

반복문 vs 재귀

factorial_loop.c
1#include <stdio.h>
2
3// 재귀로
4int factorialRec(int n) {
5 if (n <= 1) return 1;
6 return n * factorialRec(n - 1);
7}
8
9// 반복문으로
10int factorialLoop(int n) {
11 int result = 1;
12 for (int i = 2; i <= n; i++) {
13 result *= i;
14 }
15 return result;
16}
17
18int main(void) {
19 printf("Rec: %d\n", factorialRec(5));
20 printf("Loop: %d\n", factorialLoop(5));
21 return 0;
22}
실행 결과
Rec: 120 Loop: 120
반복문 vs 재귀, 언제 뭘 쓸까?
반복문이 좋을 때: 단순 반복, 성능이 중요할 때
재귀가 좋을 때: 트리 구조, 분할 정복, 문제 자체가 재귀적일 때
KOI에서는: 재귀가 핵심! DFS, 백트래킹, 분할 정복 등이 전부 재귀 기반

[심화 내용] 꼬리 재귀 최적화 (TCO)
일반 재귀:
int fact(int n) { return n * fact(n-1); }
→ n개 스택 프레임 쌓임

꼬리 재귀:
int fact_tail(int n, int acc) {
if(n<=1) return acc;
return fact_tail(n-1, n*acc); // 마지막이 재귀 호출
}
→ 컴파일러가 반복문으로 변환 가능!

GCC: -O2에서 TCO 적용
스택 오버플로우 방지
성능: 반복문과 동일

주의:
C 표준은 TCO를 보장하지 않음
Scheme: TCO 필수 보장!

재귀의 적: 중복 계산! fib(5)가 fib(3)을 2번 호출. 메모이제이션으로 O(2^n)→O(n)!
꼬리 재귀 = 반복문의 변장! 마지막에 자기를 호출하면 컴파일러가 goto로 변환!
divide and conquer
분할 정복. 문제를 작게 나누고 재귀. 합병정렬, 퀵소트, FFT. T(n)=aT(n/b)+f(n).
stack overflow
스택 오버플로우. 재귀가 너무 깊으면 스택 메모리 초과! 기본 스택: 1~8MB. ulimit -s로 확인.
backtracking
백트래킹. 재귀로 모든 가능성 탐색. 8퀸, 스도쿠, 미로. 가지치기로 최적화!
재귀의 3원칙: 1)기저 조건 2)작아지는 문제 3)자기 호출. 하나라도 빠지면 무한 재귀!
[C 코드] 재귀 — 8퀸 문제
1#include <stdio.h>
2#include <math.h>
3
4#define N 8
5int board[N];
6int solutions = 0;
7
8int is_safe(int row, int col) {
9 for (int i = 0; i < row; i++) {
10 if (board[i] == col) return 0; // 같은 열
11 if (abs(board[i] - col) == abs(i - row)) return 0; // 대각선
12 }
13 return 1;
14}
15
16void solve(int row) {
17 if (row == N) {
18 solutions++;
19 if (solutions <= 3) { // 처음 3개만 출력
20 printf("Solution #%d: ", solutions);
21 for (int i = 0; i < N; i++) printf("%d ", board[i]+1);
22 printf("\n");
23 }
24 return;
25 }
26 for (int col = 0; col < N; col++) {
27 if (is_safe(row, col)) {
28 board[row] = col;
29 solve(row + 1);
30 }
31 }
32}

24// 조합 (Combination)
25int comb_count = 0;
26void combine(int n, int k, int *subset, int idx, int start) {
27 if (idx == k) {
28 comb_count++;
29 printf("C%d: ", comb_count);
30 for (int i = 0; i < k; i++) printf("%d ", subset[i]);
31 printf("\n");
32 return;
33 }
34 for (int i = start; i <= n; i++) {
35 subset[idx] = i;
36 combine(n, k, subset, idx+1, i+1);
37 }
38}

43 permute(arr, 0, 3);
44 printf("... 총 %d개\n", perm_count);
45
46 printf("\n=== 조합 C(5,3) ===\n");
47 int subset[3];
48 combine(5, 3, subset, 0, 1);

50 return 0;
51}

3// 분할 정복으로 최대값
4int max_dc(int *arr, int lo, int hi) {
5 if (lo == hi) return arr[lo];
6 int mid = (lo + hi) / 2;
7 int left_max = max_dc(arr, lo, mid);
8 int right_max = max_dc(arr, mid+1, hi);
9 return left_max > right_max ? left_max : right_max;
10}
11
12// 분할 정복으로 거듭제곱 O(log n)
13long long fast_pow(long long base, int exp) {
14 if (exp == 0) return 1;
15 long long half = fast_pow(base, exp/2);
16 if (exp % 2 == 0) return half * half;
17 return half * half * base;
18}

Master Theorem:
T(n) = aT(n/b) + f(n)
Case 1: f(n) = O(n^(log_b(a)-ε)) → T(n) = Θ(n^log_b(a))
Case 2: f(n) = Θ(n^log_b(a)) → T(n) = Θ(n^log_b(a)·log n)
Case 3: f(n) = Ω(n^(log_b(a)+ε)) → T(n) = Θ(f(n))

Merge Sort: a=2, b=2, f(n)=n → Case 2 → O(n log n)
Master Theorem
재귀 시간복잡도 해석. T(n)=aT(n/b)+f(n). 3가지 케이스로 즉시 판단! 분할정복 분석에 필수.
