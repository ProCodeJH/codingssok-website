배열을 함수에 넘기려면?

지금까지 함수에 int나 double 값을 넘겨봤지? 그런데 배열은 좀 달라. 배열 전체를 복사해서 넘기는 게 아니라, 시작 주소만 넘겨!

변수를 함수에 전달하면 값이 복사돼요. 하지만 배열은 원소가 수백, 수천 개일 수 있어서, 전부 복사하면 너무 느려요. 그래서 C언어는 배열의 시작 주소만 전달해요.

배열 이름 arr은 첫 번째 원소의 주소예요

왜 크기를 같이 넘겨야 할까?

함수 안에서 sizeof(arr)를 쓰면 배열 전체 크기가 나올 것 같지? 아니야! 포인터의 크기(보통 4바이트나 8바이트)가 나와. 그래서 크기를 따로 알려줘야 해.
배열 매개변수의 비밀
void func(int arr[])는 사실 void func(int *arr)와 같아요. 배열을 함수에 넘기면, C언어가 자동으로 포인터로 변환해요. 그래서 함수 안에서 sizeof를 쓰면 배열 크기가 아니라 포인터 크기가 나와요.

배열 수정이 원본에 반영되는 이유

함수에 일반 변수를 넘기면 값이 복사되니까, 함수 안에서 바꿔도 원본은 그대로예요. 하지만 배열은 주소가 전달되니까, 함수 안에서 수정하면 원본 배열이 직접 바뀌어요!

주소를 공유하니까, 함수 안의 변경이 원본에 반영돼요

[심화 내용] 정렬 알고리즘 비교
O(n²) 정렬:
Bubble: 안정, 최악 O(n²), 거의 정렬되면 O(n)
Selection: 불안정, 항상 O(n²), 교환 최소
Insertion: 안정, 최선 O(n), 소규모에서 최고

O(n log n) 정렬:
Merge Sort: 안정, 항상 O(n log n), 추가 메모리 O(n)
Quick Sort: 불안정, 평균 O(n log n), 최악 O(n²)
Heap Sort: 불안정, 항상 O(n log n), 추가 메모리 O(1)

실전:
C qsort: 보통 Quick Sort + Insertion Sort 하이브리드
Python: Timsort (Merge + Insertion)
비교 기반 정렬의 하한: Ω(n log n)
binary search
이진 탐색. O(log n). 정렬된 배열 필수! 10억 개에서 30번이면 찾음. lo+(hi-lo)/2 오버플로 방지!
Timsort
Tim Peters (2002). Merge+Insertion 하이브리드. Python, Java, Android 기본 정렬. 안정, 적응적!
정렬 3총사: Bubble(교환), Selection(선택), Insertion(삽입). 전부 O(n²)지만 특성이 다르다!
stability
안정 정렬. 같은 값의 원래 순서 유지. Merge/Insertion=안정, Quick/Heap=불안정!
counting sort
카운팅 소트. O(n+k). 비교 기반 아님! 0~k 범위 정수에만 적용. 기수 정렬의 서브루틴.
sentinel search
센티넬 검색. 배열 끝에 검색값 추가 → 루프 조건 단순화. 비교 횟수 절반!
배열을 함수에 넘기면 자동으로 포인터! sizeof가 달라지니 항상 크기를 같이 전달!

1#include <stdio.h>
2
3void merge(int *arr, int l, int m, int r) {
4 int n1=m-l+1, n2=r-m;
5 int L[50], R[50];
6 for(int i=0;i<n1;i++) L[i]=arr[l+i];
7 for(int i=0;i<n2;i++) R[i]=arr[m+1+i];
8 int i=0,j=0,k=l;
9 while(i<n1&&j<n2) arr[k++]=(L[i]<=R[j])?L[i++]:R[j++];
10 while(i<n1) arr[k++]=L[i++];
11 while(j<n2) arr[k++]=R[j++];
12}

15 if(l<r) {
16 int m=l+(r-l)/2;
17 merge_sort(arr,l,m);
18 merge_sort(arr,m+1,r);
19 merge(arr,l,m,r);
20 }
21}

31}

3void heapify(int *arr, int n, int i) {
4 int largest=i, l=2*i+1, r=2*i+2;
5 if(l<n&&arr[l]>arr[largest]) largest=l;
6 if(r<n&&arr[r]>arr[largest]) largest=r;
7 if(largest!=i) {
8 int t=arr[i];arr[i]=arr[largest];arr[largest]=t;
9 heapify(arr,n,largest);
10 }
11}

13void heap_sort(int *arr, int n) {
14 for(int i=n/2-1;i>=0;i--) heapify(arr,n,i);
15 for(int i=n-1;i>0;i--) {
16 int t=arr[0];arr[0]=arr[i];arr[i]=t;
17 heapify(arr,i,0);
18 }
19}

[심화 내용] 정렬의 하한
비교 기반 정렬의 하한: Ω(n log n)
n! 가지 순서 → log₂(n!) 비교 필요
Stirling: log₂(n!) ≈ n log₂n

비교 기반이 아닌 정렬:
Counting Sort: O(n+k), k=값 범위
Radix Sort: O(d·(n+k)), d=자릿수
Bucket Sort: 평균 O(n), 균등 분포 필요

실전 선택:
n<20: Insertion Sort (오버헤드 최소)
일반: Quick Sort (평균 최고)
안정 필요: Merge Sort
메모리 제한: Heap Sort
comparison lower bound
비교 기반 정렬은 Ω(n log n)이 증명된 하한. 결정 트리 높이 = log₂(n!). 더 빠를 수 없다!
