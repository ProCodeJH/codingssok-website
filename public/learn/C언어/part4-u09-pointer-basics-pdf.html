변수는 메모리 어디에 있을까?

변수를 만들면 컴퓨터 메모리(RAM)에 공간이 생겨. 그 공간에는 주소가 있어. 아파트의 호수처럼!

int a = 42;라고 선언하면, 메모리 어딘가에 4바이트 공간이 잡히고, 거기에 42가 저장돼요. 이 공간의 위치(주소)를 알고 싶으면 &a를 쓰면 돼요.

변수 a는 메모리 주소 0x1000에 살고 있어요

& 연산자: 주소를 알려줘!

address.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 42;
5 printf("a의 값: %d\n", a);
6 printf("a의 주소: %p\n", (void *)&a);
7 return 0;
8}
실행 결과
a의 값: 42 a의 주소: 0x7ffd5e8a3b2c

%p는 주소를 출력하는 서식 지정자예요. 주소는 16진수로 표시되는데, 실행할 때마다 달라질 수 있어요. 중요한 건 주소의 존재를 아는 거예요.

포인터 변수: 주소를 저장하는 변수

주소도 숫자야. 그러면 주소를 저장하는 변수도 있을 수 있겠지? 그게 바로 포인터야!
pointer_basic.c
1#include <stdio.h>
2
3int main(void) {
4 int a = 42;
5 int *p = &a; // p는 a의 주소를 저장하는 포인터
6
7 printf("a의 값: %d\n", a);
8 printf("a의 주소: %p\n", (void *)&a);
9 printf("p의 값(a의 주소): %p\n", (void *)p);
10 printf("p가 가리키는 값: %d\n", *p); // 역참조
11
12 return 0;
13}
실행 결과
a의 값: 42 a의 주소: 0x7ffd5e8a3b2c p의 값(a의 주소): 0x7ffd5e8a3b2c p가 가리키는 값: 42
5줄int *p — int형 변수의 주소를 저장하는 포인터 변수 p를 선언해요. 별표(*)가 '포인터'라는 표시예요.
5줄&a — a의 주소를 구해서 p에 저장해요. 이제 p는 a를 '가리키고' 있어요.
10줄*p — p가 가리키는 곳의 값을 읽어요. 이걸 역참조(dereference)라고 해요.
포인터 p는 변수 a의 주소를 저장하고, *p로 a의 값에 접근해요
& 와 * 정리
&변수 → 변수의 주소를 구해요 (주소 연산자)
*포인터 → 포인터가 가리키는 값을 구해요 (역참조 연산자)
이 둘은 서로 반대 관계예요!

scanf에서 &를 쓰는 이유

지금까지 scanf("%d", &x);에서 왜 &를 붙이는지 궁금했죠? 이제 이유를 알 수 있어요!

scanf와 포인터
scanf는 키보드에서 입력받은 값을 변수에 저장해야 해요. 값을 저장하려면 변수가 메모리 어디에 있는지 알아야 하죠. 그래서 변수의 주소(&x)를 넘겨주는 거예요. scanf는 내부적으로 포인터를 사용해서 그 주소에 값을 써넣어요.
✕ 잘못된 코드
int x; scanf("%d", x); // &가 없음!
주소가 아닌 값을 넘겨서 비정상 동작
✓ 올바른 코드
int x; scanf("%d", &x); // x의 주소를 넘김
x의 주소를 넘겨서 정상 입력

세그멘테이션 폴트: 잘못된 주소 접근

포인터를 잘못 쓰면 세그멘테이션 폴트(Segmentation fault)라는 무서운 에러가 나. 프로그램이 바로 죽어버려!
segfault.c
1#include <stdio.h>
2
3int main(void) {
4 int *p = NULL; // 아무것도 가리키지 않는 포인터
5 *p = 42; // 세그멘테이션 폴트! 접근 불가!
6 return 0;
7}
세그멘테이션 폴트 예방법
1. 포인터를 선언하면 바로 초기화하세요 (NULL 또는 유효한 주소)
2. NULL 포인터를 역참조하지 마세요
3. 사용 전에 if (p != NULL)로 확인하세요
NULL을 가리키는 포인터를 역참조하면 프로그램이 죽어요

[심화] 포인터 심화

[심화] 포인터의 본질
포인터 = 메모리 주소:
int x = 42; → 주소 0x7fff1234에 42 저장
int *p = &x; → p에는 0x7fff1234 저장
*p = 100; → 0x7fff1234의 값을 100으로

포인터 크기:
32비트 OS: 4바이트 (2³² = 4GB 주소 공간)
64비트 OS: 8바이트 (2⁶⁴ = 16EB 주소 공간)
타입과 무관하게 같은 크기!

포인터 연산:
p + 1 → 다음 원소 (sizeof(type)만큼 이동)
int*: +1 = +4바이트
char*: +1 = +1바이트
double*: +1 = +8바이트

다중 포인터:
int **pp = &p; → 포인터의 포인터
**pp = 42; → 원본 값 접근

불투명 포인터. 구조체 정의를 숨기고 포인터만 공개. 정보 은닉! FILE*이 대표적!
포인터의 핵심: & = "주소 알려줘", * = "그 주소의 값". 이 두 연산자가 전부!
[C 코드] 포인터 — 연결 리스트
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node {
5 int data;
6 struct Node *next;
7} Node;
8
9Node* create_node(int data) {
10 Node *n = (Node*)malloc(sizeof(Node));
11 n->data = data;
12 n->next = NULL;
13 return n;
14}
15
16void push_front(Node **head, int data) {
17 Node *n = create_node(data);
18 n->next = *head;
19 *head = n;
20}
21
22void push_back(Node **head, int data) {
23 Node *n = create_node(data);
24 if (!*head) { *head = n; return; }
25 Node *cur = *head;
26 while (cur->next) cur = cur->next;
27 cur->next = n;
28}
29
30void print_list(Node *head) {
31 printf("List: ");
32 while (head) {
33 printf("%d → ", head->data);
34 head = head->next;
35 }
36 printf("NULL\n");
37}
38
39void free_list(Node *head) {
40 while (head) {
41 Node *temp = head;
42 head = head->next;
43 free(temp);
44 }
45}
46
47int list_length(Node *head) {
48 int len = 0;
49 while (head) { len++; head = head->next; }
50 return len;
51}
52
53int main() {
54 Node *list = NULL;
55 push_back(&list, 10);
56 push_back(&list, 20);
57 push_back(&list, 30);
58 push_front(&list, 5);
59 print_list(list);
60 printf("Length: %d\n", list_length(list));
61 free_list(list);
62 return 0;
63}

linked list

[C 코드] 포인터 — 스택/큐 구현
1#include <stdio.h>
2#include <stdlib.h>
3
4// 스택 (연결 리스트)
5typedef struct SNode { int data; struct SNode *next; } SNode;
6
7typedef struct { SNode *top; int size; } Stack;
8
9void stack_push(Stack *s, int val) {
10 SNode *n = (SNode*)malloc(sizeof(SNode));
11 n->data = val; n->next = s->top;
12 s->top = n; s->size++;
13}
14
15int stack_pop(Stack *s) {
16 if (!s->top) return -1;
17 SNode *n = s->top;
18 int val = n->data;
19 s->top = n->next;
20 free(n); s->size--;
21 return val;
22}
23
24// 큐 (연결 리스트)
25typedef struct QNode { int data; struct QNode *next; } QNode;
26typedef struct { QNode *front, *rear; int size; } Queue;
27
28void queue_enq(Queue *q, int val) {
29 QNode *n = (QNode*)malloc(sizeof(QNode));
30 n->data = val; n->next = NULL;
31 if (q->rear) q->rear->next = n;
32 else q->front = n;
33 q->rear = n; q->size++;
34}

36int queue_deq(Queue *q) {
