값에 의한 전달의 한계

지금까지 함수에 변수를 넘기면 값이 복사됐지? 그래서 함수 안에서 바꿔도 원본은 안 바뀌었어. 이게 왜 문제가 되는지 볼까?

함수 안에서는 a와 b가 바뀌었지만, main의 x와 y는 그대로예요! a와 b는 x, y의 복사본이니까요.

복사본을 바꿔봤자 원본은 그대로예요

성공하는 swap 함수

해결법은 간단해! 값 대신 주소를 넘기면 돼. 주소를 알면 원본에 직접 접근할 수 있으니까!
swap_success.c
1#include <stdio.h>
2
3// 성공하는 swap: 주소에 의한 전달
4void swap(int *a, int *b) {
5 int temp = *a;
6 *a = *b;
7 *b = temp;
8}
9
10int main(void) {
11 int x = 10, y = 20;
12 printf("호출 전: x=%d, y=%d\n", x, y);
13 swap(&x, &y); // 주소를 넘김!
14 printf("호출 후: x=%d, y=%d\n", x, y);
15 return 0;
16}
실행 결과
호출 전: x=10, y=20 호출 후: x=20, y=10
4줄int *a, int *b — 포인터 매개변수. x, y의 주소를 받아요.
5줄*a — a가 가리키는 곳(즉, x)의 값을 읽어요.
6줄*a = *b — b가 가리키는 값(y의 값)을 a가 가리키는 곳(x)에 저장해요.
13줄swap(&x, &y) — x와 y의 주소를 넘겨요.
주소를 넘기면 원본에 직접 접근해서 값을 교환할 수 있어요

값 전달 vs 주소 전달 비교

값 전달은 복사본, 주소 전달은 원본에 접근
값 전달 vs 주소 전달
값 전달(call by value): 값이 복사됨 → 원본 변경 불가
주소 전달(call by reference): 주소가 전달됨 → 원본 변경 가능

C언어는 기본적으로 값 전달이에요. 주소 전달을 하려면 포인터를 사용해야 해요.

자주 하는 실수

✕ 잘못된 코드
swap(x, y); // 값을 넘김
주소가 아닌 값을 넘겨서 원본이 안 바뀜
✓ 올바른 코드
swap(&x, &y); // 주소를 넘김
주소를 넘겨서 원본이 정상적으로 교환됨

4typedef enum { EV_START, EV_PAUSE, EV_RESUME, EV_STOP, EV_COUNT } Event;

6typedef State(*Handler)(State, Event);
7
8State on_idle(State s, Event e) {

10 return s;
11}
12State on_running(State s, Event e) {
13 if(e==EV_PAUSE){printf("Pausing...\n");return ST_PAUSED;}
14 if(e==EV_STOP){printf("Stopping...\n");return ST_DONE;}
15 return s;
16}
17State on_paused(State s, Event e) {
18 if(e==EV_RESUME){printf("Resuming...\n");return ST_RUNNING;}
19 if(e==EV_STOP){printf("Stopping...\n");return ST_DONE;}
20 return s;
21}
22State on_done(State s, Event e) { return s; }
23
24Handler handlers[] = {on_idle, on_running, on_paused, on_done};

26
27int main() {

29 Event events[] = {EV_START, EV_PAUSE, EV_RESUME, EV_STOP};
30 for(int i=0;i<4;i++) {
31 printf("[%s] → ", state_names[s]);
32 s = handlers[s](s, events[i]);
33 printf(" → [%s]\n", state_names[s]);
34 }

36}
