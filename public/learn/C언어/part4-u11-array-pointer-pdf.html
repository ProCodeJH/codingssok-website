배열 이름 = 첫 번째 원소의 주소

배열 이름 arr는 사실 arr[0]의 주소야! 배열을 함수에 넘기면 주소가 전달되는 이유가 바로 이거야.

arr[i] == *(arr + i)

배열의 인덱스 접근 arr[i]는 사실 *(arr + i)의 줄임 표현이에요. 컴파일러가 내부적으로 변환하는 거예요.

포인터 산술 연산

포인터 산술의 핵심
포인터에 1을 더하면 자료형 크기만큼 주소가 증가해요.
int *p일 때 p + 1은 주소가 4 증가 (int = 4바이트)
char *p일 때 p + 1은 주소가 1 증가 (char = 1바이트)
double *p일 때 p + 1은 주소가 8 증가 (double = 8바이트)
p + 1은 다음 원소, p + 2는 그 다음 원소를 가리켜요

포인터로 배열 순회하기

pointer_traverse.c
1#include <stdio.h>
2
3int main(void) {
4 int arr[5] = {10, 20, 30, 40, 50};

6

9 printf("arr[%d] = %d\n", i, arr[i]);
10 }
11
12 printf("\n포인터 방식:\n");

14 printf("*(p+%d) = %d\n", i, *(p + i));

16

18 p = arr; // 다시 처음으로

26}
실행 결과
인덱스 방식: arr[0] = 10 arr[1] = 20 arr[2] = 30 arr[3] = 40 arr[4] = 50 포인터 방식: *(p+0) = 10 *(p+1) = 20 *(p+2) = 30 *(p+3) = 40 *(p+4) = 50 포인터 이동 방식: 10 20 30 40 50
세 가지 방식 모두 같은 결과야! 포인터 이동 방식(p++)은 나중에 문자열 처리할 때 아주 유용하게 쓰여.

배열 이름과 포인터의 차이

주의: 배열 이름은 상수 포인터
배열 이름 arr은 첫 번째 원소의 주소를 담고 있지만, 변경할 수 없어요. 즉 arr++는 에러예요!
반면 포인터 변수 p는 p++로 다른 곳을 가리키게 할 수 있어요.
✕ 잘못된 코드
int arr[5] = {1,2,3,4,5}; arr++; // 에러! 배열 이름은 못 바꿈
배열 이름은 상수라서 변경 불가
✓ 올바른 코드
int arr[5] = {1,2,3,4,5}; int *p = arr; p++; // p가 arr[1]을 가리킴
포인터 변수는 자유롭게 이동 가능

용어 정리

[심화] 배열과 포인터의 관계

[심화] 배열 ≠ 포인터 (하지만 거의 같다)
같은 점:
arr[i] == *(arr+i) — 인덱싱 = 포인터 연산
함수 인수에서 int arr[] == int *arr
배열 이름 = 첫 원소 주소 (대부분의 경우)

다른 점:
sizeof(arr) → 전체 크기 / sizeof(ptr) → 포인터 크기
&arr → 전체 배열의 주소 (타입: int(*)[N])
arr++ 불가 (배열은 lvalue가 아님)

2D 배열과 포인터:
int a[3][4]; → 연속 12개 int
a[i][j] == *(*(a+i)+j) == *((int*)a + i*4 + j)
int (*p)[4] = a; → 행 포인터
int *q = &a[0][0]; → 원소 포인터

[심화 내용] 배열 vs 포인터의 차이
int arr[5] vs int *p:
arr: 컴파일 시 크기 결정, sizeof(arr)=20
p: 런타임, sizeof(p)=8 (64비트)

arr은 포인터가 아니다!
arr = 배열의 첫 번째 원소의 주소로 "decay"
&arr = 배열 전체의 주소 (타입이 다름!)
&arr[0] = arr (값은 같지만 타입이 다름)

2D 배열 메모리:
int a[3][4]: 연속 12개 int (48바이트)
int **p: 3개 포인터 + 3×4개 int (비연속!)

캐시 성능:
연속 메모리(flat) > 포인터 배열(2D malloc)
수치 계산에서는 flat 1D 선호!
array decay
배열이 포인터로 변환되는 것. int arr[5]→int*. sizeof와 &에서는 decay 안 됨!
VLA
Variable Length Array. C99. int arr[n]; 스택에 동적 크기. C11에서 선택사항. 스택 오버플로 주의!
2D 배열의 두 가지: int**은 유연하지만 느리고, flat 1D는 빠르지만 인덱스 계산 필요!
flexible array member
유연 배열 멤버. struct { int n; int data[]; }; 구조체 끝에 가변 크기 배열. C99!
restrict pointer
restrict. 이 포인터만 해당 메모리에 접근함을 보장. 컴파일러 최적화 힌트! memcpy에 사용.
pointer aliasing
포인터 앨리어싱. 두 포인터가 같은 메모리를 가리킴. 최적화 방해! strict aliasing rule.
배열=연속 메모리, 포인터=주소. arr[i]=*(arr+i). 이 등가 관계가 C의 DNA!

4int** spiral_matrix(int n) {
5 int **mat = (int**)malloc(n * sizeof(int*));
6 for (int i = 0; i < n; i++)
7 mat[i] = (int*)calloc(n, sizeof(int));
8
9 int top=0, bottom=n-1, left=0, right=n-1;
10 int num = 1;
11 while (top <= bottom && left <= right) {
12 for (int c = left; c <= right; c++) mat[top][c] = num++;
13 top++;
14 for (int r = top; r <= bottom; r++) mat[r][right] = num++;
15 right--;
16 for (int c = right; c >= left; c--) mat[bottom][c] = num++;
17 bottom--;
18 for (int r = bottom; r >= top; r--) mat[r][left] = num++;
19 left++;
20 }
21 return mat;
22}

2
3#define BUF_SIZE 5
4
5typedef struct {
6 int data[BUF_SIZE];
7 int head, tail, count;
8} RingBuffer;
9
10void rb_init(RingBuffer *rb) { rb->head=rb->tail=rb->count=0; }
11
12int rb_push(RingBuffer *rb, int val) {
13 if(rb->count==BUF_SIZE) return -1;
14 rb->data[rb->tail]=val;
15 rb->tail=(rb->tail+1)%BUF_SIZE;
16 rb->count++; return 0;
17}
18
19int rb_pop(RingBuffer *rb, int *val) {
20 if(rb->count==0) return -1;
21 *val=rb->data[rb->head];
22 rb->head=(rb->head+1)%BUF_SIZE;
23 rb->count--; return 0;
24}
25
26void rb_print(RingBuffer *rb) {
27 printf("Ring[%d/%d]: ",rb->count,BUF_SIZE);
28 int idx=rb->head;
29 for(int i=0;i<rb->count;i++) {
30 printf("%d ",rb->data[idx]);
31 idx=(idx+1)%BUF_SIZE;
32 }
33 printf("\n");
34}

ring buffer
원형 버퍼. head/tail이 배열을 순환. O(1) enqueue/dequeue. 네트워크, 오디오, 로그에 필수!
modulo trick
idx = (idx+1) % SIZE. 배열 끝에서 0으로 순환! 원형 자료구조의 핵심 트릭.
zero-copy
제로 카피. 데이터 복사 없이 포인터만 전달. 고성능! sendfile(), mmap. 네트워크/파일 I/O.
memory-mapped IO
mmap. 파일을 메모리에 매핑. 포인터로 파일 접근! 대용량 파일 처리에 최적.
배열 포인터 완전 졸업! 정적→동적→나선→링 버퍼→문자열 배열. 메모리 구조의 달인!
[심화 내용] 메모리 정렬

스트라이드. 원소 간 바이트 간격. 연속 배열: stride=sizeof(T). 비연속 접근: stride≠sizeof(T).
