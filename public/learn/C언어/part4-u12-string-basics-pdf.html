C언어의 문자열은 특별해

다른 언어에는 string 타입이 있는데, C언어에는 없어! C에서 문자열은 문자 배열이야. 끝에 특별한 표시가 붙어 있지.

C언어에서 문자열은 char 배열에 저장돼요. 문자열의 끝을 표시하기 위해, 마지막에 '\0'(널 문자, ASCII 코드 0)이 자동으로 붙어요.

string_basic.c
1#include <stdio.h>
2
3int main(void) {
4 char name[10] = "Hello";
5
6 // 문자 하나씩 확인
7 for (int i = 0; i < 10; i++) {
8 printf("name[%d] = '%c' (ASCII: %d)\n", i, name[i], name[i]);
9 }
10
11 return 0;
12}
실행 결과
name[0] = 'H' (ASCII: 72) name[1] = 'e' (ASCII: 101) name[2] = 'l' (ASCII: 108) name[3] = 'l' (ASCII: 108) name[4] = 'o' (ASCII: 111) name[5] = '' (ASCII: 0) name[6] = '' (ASCII: 0) name[7] = '' (ASCII: 0) name[8] = '' (ASCII: 0) name[9] = '' (ASCII: 0)
'Hello'는 H, e, l, l, o, \0 — 총 6칸을 차지해요
널 종료 문자 '\0'
C언어의 문자열은 반드시 '\0'으로 끝나요. 이걸 널 종료 문자라고 해요.
"Hello"는 글자 5개이지만, 배열에는 최소 6칸이 필요해요 (\0 포함).
printf의 %s, strlen 같은 함수들은 \0을 만나면 '여기가 끝'이라고 판단해요.

[심화] 문자열 보안 취약점
버퍼 오버플로우:
char buf[10]; strcpy(buf, "very long string");
→ 스택 파괴! 해커의 코드 실행 가능

안전한 대안:
strcpy → strncpy / strlcpy
sprintf → snprintf
gets → fgets (gets는 C11에서 삭제!)
strcat → strncat

Format String Attack:
printf(user_input); → %x로 스택 읽기!
→ 항상 printf("%s", user_input);

Off-by-one:
NUL 종료 문자 공간 빠뜨리기
char buf[5]; strcpy(buf, "hello"); → 6바이트 필요!
NUL terminator
\0. C 문자열의 끝 표시. 없으면 strlen이 메모리를 넘어 읽음! 모든 문자열 버그의 시작.
KMP
Knuth-Morris-Pratt. O(n+m) 문자열 검색. 실패 함수로 불필요한 비교 건너뜀. strstr의 최적 버전!
C 문자열 = char 배열 + \0! strlen, strcpy, strcmp를 직접 구현하면 원리가 확실히 보여!
gets()는 삭제된 함수! 버퍼 크기 체크 없이 입력받아서 해킹 당함. 항상 fgets 사용!

[심화 내용] 문자열의 메모리
C 문자열 = char 배열 + \0
"Hello" → {'H','e','l','l','o','\0'} = 6바이트

문자열 리터럴:
char *s = "Hello"; → .rodata 섹션 (읽기 전용!)
char s[] = "Hello"; → 스택에 복사 (수정 가능)

보안 취약점:
gets() → 버퍼 오버플로! (C11에서 삭제)
strcpy() → 길이 검사 없음 → 오버플로
strncpy() → \0 보장 안 함!
snprintf() → 안전한 대안

유니코드:
ASCII: 1바이트/문자 (0~127)
UTF-8: 1~4바이트/문자 (가변 길이)
strlen("한글") → 6 (UTF-8에서 한글 = 3바이트)
null terminator
\0 = 문자열의 끝 표시. ASCII 0. C 문자열의 핵심! 없으면 strlen이 메모리를 넘어 읽음!
buffer overflow
버퍼 오버플로. strcpy에 긴 문자열 → 스택 덮어쓰기 → 코드 실행! Morris Worm (1988) 원인.
C 문자열의 핵심: \0으로 끝나는 char 배열! strlen은 \0을 찾을 때까지 세는 거야.
SSO
Small String Optimization. 짧은 문자열을 힙 대신 스택에 저장. std::string(C++), 15바이트까지!
rope
로프 자료구조. 긴 문자열을 트리로 관리. 삽입/삭제 O(log n)! 텍스트 에디터에 사용.
UTF-8
가변 길이 유니코드 인코딩. ASCII 호환! 한글=3바이트, 이모지=4바이트. 웹 표준!
C 문자열 5대 함수: strlen, strcpy, strcmp, strcat, strstr. 이걸 직접 구현하면 포인터 마스터!

1#include <stdio.h>

4
5// URL 파서
6typedef struct {
7 char protocol[10];
8 char host[50];
9 int port;
10 char path[100];
11} URL;
12
13URL parse_url(const char *url_str) {
14 URL url = {"http", "", 80, "/"};
15 const char *p = url_str;
16
17 // 프로토콜
18 const char *colon = strstr(p, "://");
19 if (colon) {
20 int len = colon - p;
21 strncpy(url.protocol, p, len < 9 ? len : 9);
22 url.protocol[len < 9 ? len : 9] = 0;
23 p = colon + 3;
24 }
25
26 // 호스트:포트
27 const char *slash = strchr(p, '/');
28 const char *port_sep = strchr(p, ':');
29 if (port_sep && (!slash || port_sep < slash)) {
30 int hlen = port_sep - p;
31 strncpy(url.host, p, hlen < 49 ? hlen : 49);
32 url.port = atoi(port_sep + 1);
33 } else {
34 int hlen = slash ? (slash - p) : strlen(p);
35 strncpy(url.host, p, hlen < 49 ? hlen : 49);
36 }
37
38 // 경로
39 if (slash) strncpy(url.path, slash, 99);
40
41 return url;
42}
