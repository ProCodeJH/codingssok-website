구조체란 무엇인가

여러 데이터를 하나로 묶는 구조체를 배워요. struct 키워드, 멤버 접근, 초기화를 익혀요.

📚 이번에 배울 것

  • 구조체가 무엇인지 이해해요.
  • struct 키워드로 구조체를 정의할 수 있어요.
  • 점 연산자(.)로 멤버에 접근할 수 있어요.
  • 구조체 변수를 선언하고 초기화할 수 있어요.

✅ 시작 전 체크

  • 변수와 자료형(int, char, float)을 알고 있다
  • 배열의 기본 개념을 이해하고 있다
Part 1 진행률: 1%

학생 한 명의 정보를 저장하려면?

학생 한 명의 정보를 저장하려면 이름, 점수, 학번이 필요해. 변수를 따로따로 만들면 관리가 너무 힘들어져. 이걸 하나로 묶는 방법이 바로 구조체야!

프로그래밍에서 구조체(struct)란, 서로 다른 자료형의 데이터를 하나로 묶어서 새로운 자료형을 만드는 것이에요. 학생 = 이름 + 점수 + 학번처럼, 관련 있는 데이터를 한 덩어리로 다룰 수 있어요.

변수를 따로 만드는 것 vs 구조체로 묶는 것

구조체의 정의와 선언

구조체를 사용하려면 두 단계가 필요해요. 먼저 설계도(정의)를 만들고, 그 설계도로 실체(변수)를 만들어요.

구조체 정의 -> 변수 선언 -> 값 저장 -> 값 사용
?

아래 코드를 실행하면 화면에 뭐가 나올까요? 먼저 예측해 보세요.

struct_student.c
1#include <stdio.h>
2
3struct Student {
4 char name[20];
5 int score;
6 int id;
7};
8
9int main(void) {
10 struct Student s1;
11
12 s1.id = 1001;
13 s1.score = 95;
14
15 printf("학번: %d\n", s1.id);
16 printf("점수: %d\n", s1.score);
17 return 0;

실행 결과
학번: 1001 점수: 95
3줄struct Student -- 구조체를 정의해요. Student라는 이름의 새로운 자료형 설계도를 만드는 거예요.
4줄char name[20] -- 구조체의 멤버(member)예요. 이름을 저장하는 문자 배열이에요.
10줄struct Student s1; -- 설계도로 실제 변수 s1을 만들어요. 이때 메모리가 할당돼요.
12줄s1.id = 1001; -- 점 연산자(.)로 멤버에 접근해요. 구조체변수.멤버이름 형태예요.
구조체 정의는 설계도일 뿐이야. 메모리가 잡히려면 반드시 변수 선언을 해야 해!

구조체 메모리 레이아웃

struct Student의 멤버가 메모리에 어떻게 배치되는지
구조체 멤버는 메모리에 선언 순서대로 배치돼. 각 멤버의 크기만큼 공간을 차지하지.
구조체 핵심 정리
1. 정의 -- struct 키워드로 설계도 작성
2. 선언 -- struct 이름 변수명; 으로 변수 생성
3. 접근 -- 점 연산자(.)로 멤버에 접근
4. 초기화 -- 선언 시 { } 로 한번에 값 설정
[KOI] 구조체와 올림피아드
KOI 문제에서 좌표(x, y), 선분(x1, y1, x2, y2), 학생 정보 등을 다룰 때 구조체가 유용해요. 관련 데이터를 하나로 묶으면 정렬이나 탐색이 훨씬 편해져요.

8Complex complex_mul(Complex a, Complex b) {

dot product
내적. a·b = |a||b|cosθ. 두 벡터의 유사도. 0이면 직교! 물리, 그래픽, ML에서 핵심.
cross product 2D

1#include <stdio.h>
2#include <math.h>
3
4typedef struct { double m[3][3]; } Mat3;
5
6Mat3 mat3_identity() {
7 Mat3 r = {{{1,0,0},{0,1,0},{0,0,1}}};
8 return r;
9}
10
11Mat3 mat3_mul(Mat3 a, Mat3 b) {
12 Mat3 r = {{{0}}};
13 for (int i = 0; i < 3; i++)
14 for (int j = 0; j < 3; j++)
15 for (int k = 0; k < 3; k++)
16 r.m[i][j] += a.m[i][k] * b.m[k][j];
17 return r;
18}
19
20Mat3 mat3_transpose(Mat3 a) {
21 Mat3 r;
22 for (int i = 0; i < 3; i++)
23 for (int j = 0; j < 3; j++)
24 r.m[i][j] = a.m[j][i];
25 return r;
26}
27
28double mat3_det(Mat3 a) {
29 return a.m[0][0]*(a.m[1][1]*a.m[2][2]-a.m[1][2]*a.m[2][1])
30 -a.m[0][1]*(a.m[1][0]*a.m[2][2]-a.m[1][2]*a.m[2][0])
31 +a.m[0][2]*(a.m[1][0]*a.m[2][1]-a.m[1][1]*a.m[2][0]);
32}
33
34Mat3 mat3_rotate(double angle_deg) {
35 double r = angle_deg * 3.14159265 / 180.0;
36 Mat3 m = {{{cos(r),-sin(r),0},{sin(r),cos(r),0},{0,0,1}}};
37 return m;
38}
39
40Mat3 mat3_scale(double sx, double sy) {
41 Mat3 m = {{{sx,0,0},{0,sy,0},{0,0,1}}};
42 return m;
43}
44
45void mat3_print(const char *name, Mat3 m) {
46 printf("%s:\n", name);
47 for (int i = 0; i < 3; i++)
48 printf(" [%7.3f %7.3f %7.3f]\n", m.m[i][0], m.m[i][1], m.m[i][2]);
49}

51int main() {

53 Mat3 R = mat3_rotate(45);
54 Mat3 S = mat3_scale(2, 3);

[심화 내용] 아핀 변환과 동차 좌표
2D 아핀 변환 (Affine Transform):
이동+회전+확대를 하나의 3×3 행렬로!
동차 좌표: (x,y) → (x,y,1)

변환 합성:
T = R × S × Tr (회전 → 확대 → 이동)
행렬 곱 순서 = 변환 역순!

3D로 확장:
4×4 행렬 (동차 좌표 w=1)
OpenGL, DirectX, Vulkan 전부 이 구조!

행렬식 (Determinant):
det=0 → 역행렬 없음 (퇴화)
|det|=1 → 면적 보존 (회전)
det<0 → 반전 (거울)

[C 코드] 구조체 — 링 버퍼
1#include <stdio.h>
2
3#define RING_SIZE 8
4typedef struct {
5 int data[RING_SIZE];
6 int head, tail, count;
7} RingBuffer;
8
9void ring_init(RingBuffer *r) { r->head = r->tail = r->count = 0; }
10int ring_push(RingBuffer *r, int val) {
11 if (r->count >= RING_SIZE) return -1;
12 r->data[r->tail] = val;
13 r->tail = (r->tail + 1) % RING_SIZE;
14 r->count++;
15 return 0;
16}
17int ring_pop(RingBuffer *r, int *val) {
18 if (r->count <= 0) return -1;
19 *val = r->data[r->head];
20 r->head = (r->head + 1) % RING_SIZE;
21 r->count--;
22 return 0;
23}
24void ring_print(RingBuffer *r) {
25 printf("[%d/%d] head=%d tail=%d: ", r->count, RING_SIZE, r->head, r->tail);
26 for (int i = 0; i < r->count; i++)
27 printf("%d ", r->data[(r->head + i) % RING_SIZE]);
28 printf("\n");
29}
30
31int main() {
32 RingBuffer rb;
33 ring_init(&rb);
34 for (int i = 1; i <= 6; i++) ring_push(&rb, i*10);
35 ring_print(&rb);
36 int v;
37 ring_pop(&rb, &v); printf("pop: %d\n", v);
38 ring_pop(&rb, &v); printf("pop: %d\n", v);
39 ring_push(&rb, 70); ring_push(&rb, 80);
40 ring_print(&rb);
41 return 0;
42}
실행 결과
[6/8] head=0 tail=6: 10 20 30 40 50 60 pop: 10 pop: 20 [6/8] head=2 tail=0: 30 40 50 60 70 80
[심화 내용] 링 버퍼와 운영체제
링 버퍼 = 원형 큐:
head == tail: 비어 있음
(tail+1) % N == head: 가득 참
O(1) 삽입/삭제, 고정 메모리!

사용 사례:
• 커널 로그 (dmesg)
• 오디오 버퍼 (ALSA, PortAudio)
• 네트워크 패킷 큐 (DPDK)
• UART/시리얼 수신 버퍼

Lock-free Ring Buffer:
atomic으로 head/tail 업데이트
→ 락 없이 producer-consumer
→ 고성능 멀티스레드 프로그래밍!
circular queue
원형 큐. 배열의 끝→처음 순환. %연산으로 인덱스 래핑. 고정 메모리로 FIFO! 임베디드 필수.
링 버퍼 = 배열의 끝이 처음과 연결! %연산 하나로 원형 구조. 오디오, 네트워크에 필수!

C99. (Point){.x=3, .y=4}. 멤버 이름으로 초기화! 순서 무관. 가독성↑ 실수↓.
compound literal
C99. (Point){3,4}. 임시 구조체를 표현식으로. 함수 인자에 직접 전달 가능!
struct copy
구조체 대입 = 멤버별 복사(shallow copy). 포인터 멤버는 주소만 복사! 깊은 복사는 직접 구현.
C99 지정 초기화! (Point){.x=3, .y=4}. 어떤 멤버를 설정하는지 명확!
구조체 = C 프로그래밍의 기둥! 배열, 함수, 포인터, 파일 모든 것과 결합!
nested struct
중첩 구조체. struct A { struct B inner; }; 구조체 안에 구조체. 복잡한 데이터 계층 표현!
struct padding rule
패딩 규칙. 각 멤버는 자기 크기의 배수 주소에 정렬. 구조체 전체는 가장 큰 멤버의 배수!
anonymous struct
C11 익명 구조체. struct { struct { int x,y; }; }; 내부 멤버를 바로 접근! s.x (s.inner.x 아닌)
구조체 마스터! 기본→복소수→벡터→행렬→링버퍼. 데이터 모델링의 A to Z!
C의 구조체 = 클래스의 시조! 데이터 묶기→함수 결합→다형성까지. OOP의 기원!
C11 static_assert
컴파일 타임 검증. _Static_assert(sizeof(int)==4,"int must be 4B"); 조건 실패→컴파일 에러!
struct assignment
구조체 대입. Point a = b; 멤버별 복사. memcpy와 동일 효과. 배열 멤버도 복사됨!
tagged union pattern
태그+유니언 패턴. type필드로 현재 값 종류 구분. switch로 안전 접근. 다형성의 C 구현!
구조체 = C의 모든 것의 기반! 배열+포인터+함수+파일과 결합하면 무한한 가능성!
구조체(struct)
서로 다른 자료형의 변수들을 하나로 묶어 만든 사용자 정의 자료형.
멤버(member)
구조체 안에 포함된 개별 변수. 구조체의 구성 요소.
점 연산자(.)
구조체 변수에서 특정 멤버에 접근할 때 사용하는 연산자.
구조체 초기화
선언과 동시에 중괄호 { }로 멤버 값을 설정하는 것.
사용자 정의 자료형
int, char 같은 기본 자료형과 달리, 프로그래머가 직접 만든 자료형.
