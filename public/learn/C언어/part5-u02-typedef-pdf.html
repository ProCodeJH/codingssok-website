typedef와 구조체

typedef를 사용해 구조체를 더 간결하게 쓰는 방법을 배워요. 익명 구조체도 알아봐요.

📚 이번에 배울 것

  • typedef의 의미와 역할을 이해해요.
  • typedef struct로 구조체를 간결하게 사용할 수 있어요.
  • 익명 구조체가 무엇인지 알아요.
  • 기본 자료형에도 typedef를 쓸 수 있다는 것을 알아요.

✅ 시작 전 체크

  • struct로 구조체를 정의하고 사용할 수 있다
  • 점 연산자(.)로 멤버에 접근할 수 있다
Part 1 진행률: 2%

struct를 매번 쓰는 게 귀찮다면

이전 유닛에서 struct Student s1; 이렇게 쓴 거 기억나지? 코드가 길어지면 struct라는 단어를 계속 반복해야 해서 불편해. typedef로 이 문제를 해결할 수 있어!

typedef는 'type definition'의 줄임말로, 기존 자료형에 새 이름(별명)을 붙이는 키워드예요. 구조체와 함께 쓰면 struct 키워드를 생략할 수 있어요.

typedef를 쓰면 struct 키워드 없이 깔끔하게 사용 가능

typedef는 구조체만을 위한 게 아니다

typedef는 기본 자료형에도 쓸 수 있어. 예를 들어 typedef unsigned int uint; 라고 하면 uint라는 짧은 이름으로 쓸 수 있지.
기본 자료형 -> 구조체 -> typedef로 이어지는 타입 계층
typedef를 쓰면 코드가 얼마나 깔끔해지는지 비교
typedef 핵심 정리
typedef = 기존 자료형에 새 이름(별명)을 부여하는 키워드
typedef struct { ... } 이름; = 가장 흔한 사용 패턴
익명 구조체 = struct 뒤에 이름이 없는 형태, typedef와 함께 자주 사용
[KOI] typedef 활용
올림피아드에서는 typedef로 long long을 ll로 줄여 쓰는 경우가 많아요. typedef long long ll; 한 줄이면 코딩 속도가 확 올라요.

[심화 내용] typedef vs #define
typedef:
컴파일러가 처리, 타입 검사 가능
typedef int* IntPtr; IntPtr a,b; // 둘 다 int*

#define:
전처리기가 텍스트 치환
#define IntPtr int*; IntPtr a,b; // a만 int*!

typedef가 우월한 이유:
1. 스코프 존중 (블록 내부 가능)
2. 타입 검사 적용
3. 포인터 선언 안전
4. 디버거에서 이름 표시

C11 _Generic과 typedef:
typedef로 타입 이름을 단순화한 후
_Generic으로 타입별 분기 → 의사 오버로딩
opaque type
불투명 타입. typedef struct Foo Foo; 정의 숨김. 사용자는 Foo*로만 접근. API 안정성!
strong typedef
강한 타입정의. C에서는 typedef double Meters와 Seconds가 호환됨(약함). Rust의 newtype이 강한 typedef!
function pointer typedef
typedef int(*Cmp)(const void*,const void*); 없으면 int(*)(const void*,...) 지옥!
typedef = 타입에 이름 붙이기! 코드 가독성 100% 향상. 함수 포인터에는 필수!
typedef double Meters; 물리 단위를 타입으로! 컴파일러가 못 막지만 사람은 읽기 쉬워!

1#include <stdio.h>

4// 상태 머신 타입 정의

14 /* CONNECT SUCCESS FAIL RESET */
15 /* IDLE */ {ST_CONNECTING, ST_IDLE, ST_IDLE, ST_IDLE},
16 /* CONN */ {ST_CONNECTING, ST_CONNECTED, ST_ERROR, ST_IDLE},
17 /* CONND */ {ST_CONNECTED, ST_CONNECTED, ST_ERROR, ST_IDLE},
18 /* ERROR */ {ST_ERROR, ST_ERROR, ST_ERROR, ST_IDLE}
19};
20
21typedef struct {
22 State current;
23 int transitions;
24 void (*on_enter)(State);
25 void (*on_exit)(State);
26} StateMachine;
27
28void default_enter(State s) { printf(" → Enter %s\n", state_names[s]); }
29void default_exit(State s) { printf(" ← Exit %s\n", state_names[s]); }
30
31StateMachine sm_create() {
32 return (StateMachine){ST_IDLE, 0, default_enter, default_exit};
33}
34
35void sm_process(StateMachine *sm, Event ev) {
36 State next = transition_table[sm->current][ev];
37 printf("[%d] %s + %s:", sm->transitions, state_names[sm->current], event_names[ev]);
38 if (next != sm->current) {
39 sm->on_exit(sm->current);
40 sm->on_enter(next);
41 sm->current = next;
42 } else printf(" (no change)\n");
43 sm->transitions++;
44}

46int main() {

48 Event sequence[] = {EV_CONNECT, EV_SUCCESS, EV_FAIL, EV_RESET, EV_CONNECT, EV_FAIL};
49 for (int i = 0; i < 6; i++) sm_process(&sm, sequence[i]);

플러그인 = 동적 확장:
핵심 코드 수정 없이 기능 추가
typedef로 인터페이스 정의 (init/cleanup/process)
→ 모든 플러그인이 같은 인터페이스 구현

동적 로딩:
Linux: dlopen/dlsym/dlclose
Windows: LoadLibrary/GetProcAddress
→ 런타임에 .so/.dll을 로드!

파이프라인 패턴:
입력 → [플러그인1] → [플러그인2] → ... → 출력
Unix 파이프의 원리 (ls | grep | sort)
데이터 처리, 미들웨어에서 핵심

실제 사례:
• 웹 서버 미들웨어 (Express.js app.use)
• 이미지 편집기 필터
• 오디오 DSP 플러그인 (VST)
• IDE 확장 (VS Code extensions)
dlopen
동적 라이브러리 로딩. dlopen("plugin.so", RTLD_LAZY). dlsym()으로 함수 주소 획득!
middleware
미들웨어. 요청→[미들1]→[미들2]→응답. Express.js app.use(). 파이프라인의 웹 버전!
플러그인 = typedef의 종합 예술! 같은 인터페이스, 다른 구현. 파이프라인으로 합성!
💡 typedef 활용 체크리스트
✅ 함수 포인터에는 반드시 typedef 사용
✅ 플랫폼 의존 타입은 typedef로 추상화 (size_t, int32_t)
✅ typedef struct 패턴으로 struct 키워드 생략
✅ 불투명 타입으로 구현 숨김
✅ 매크로 상수보다 enum 선호
✅ #define 타입보다 typedef 선호 (타입 안전)
stdint.h
. int8_t, int16_t, int32_t, int64_t. 크기 보장 타입! 이식성 필수. C99 표준.
size_t
부호 없는 크기 타입. sizeof 반환값. 배열 인덱스, 메모리 크기에 사용! printf: %zu.
ptrdiff_t
포인터 차이 타입. 두 포인터의 거리. 부호 있음! . 배열 범위 계산에 안전.
int32_t = 어디서든 정확히 32비트! int는 플랫폼마다 다를 수 있지만 int32_t는 보장!
typedef는 리팩토링의 친구! 타입 변경 시 typedef 한 줄만 수정. 전체 코드 자동 반영!
typedef scope
typedef는 스코프 존중! 블록 안에서 정의하면 블록에서만 유효. #define은 파일 끝까지!
bool (stdbool.h)
C99 bool. #include . true=1, false=0. typedef _Bool bool;의 편의 헤더.
type alias vs newtype
C typedef=별칭(호환). Rust newtype=새 타입(비호환). struct Miles(f64)≠struct Km(f64)!
restrict pointer
C99. int * restrict p; 이 포인터만 이 메모리 접근 보장. 컴파일러 최적화 힌트!
typedef 완전 정복! 타입 별칭+함수 포인터+제네릭 매크로+불투명 타입+플러그인!
restrict = 최적화 키워드! memcpy(restrict dest, restrict src, n). 겹치지 않음을 보장!
typedef array
배열 typedef. typedef int Matrix[3][3]; Matrix m; 2D 배열을 깔끔하게! 함수 인자에도 사용.
typedef pointer
포인터 typedef 주의! typedef int* IntPtr; IntPtr a,b; 둘 다 int*. #define IntPtr int*는 b가 int!
opaque pointer
불투명 포인터. typedef struct Foo *FooPtr; 구조체 정의 숨김. API 안정성의 핵심!
typedef = 코드 가독성의 핵심! 복잡한 타입에 이름을 붙이면 코드가 문서가 됨!
typedef 팁
복잡한 타입일수록 typedef가 빛난다. int(*(*fp)(int))[10]? typedef 없으면 읽기 불가능!
typedef 체이닝
typedef int Num; typedef Num* NumPtr; typedef NumPtr* NumPtrPtr; 단계별로 쌓기!
typedef는 "별명 붙이기"야! unsigned long long → u64. 타이핑도 줄고 읽기도 쉬워!
size_t
부호 없는 정수 타입. sizeof 반환값. 배열 인덱스, 메모리 크기에 사용. 64비트에서 8바이트!
typedef unsigned long long u64; 이거 하나면 printf("%llu", val) 대신 깔끔하게!
typedef
기존 자료형에 새로운 이름(별명)을 부여하는 C 키워드.
익명 구조체
struct 뒤에 이름 없이 정의된 구조체. typedef와 함께 자주 사용된다.
타입 별명
typedef로 만든 새 이름. 원래 자료형과 완전히 동일하게 동작한다.
이번 유닛 핵심 정리
1
typedef = 기존 자료형에 별명을 붙이는 키워드
2
typedef struct { ... } Student; 패턴이 가장 흔하다
3
익명 구조체 = struct 뒤에 이름이 없는 형태
4
typedef를 쓰면 struct 키워드를 매번 쓰지 않아도 된다
5
기본 자료형에도 typedef를 사용할 수 있다

undefined
