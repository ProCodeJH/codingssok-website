구조체를 함수에 전달하기

구조체도 int나 float처럼 함수에 전달할 수 있어. 그런데 구조체는 크기가 크니까 전달 방식에 따라 성능 차이가 커질 수 있어!

5 int score;
6} Student;
7
8void printStudent(Student s) {
9 printf("%s: %d점\n", s.name, s.score);
10}
11
12int main(void) {
13 Student s1 = {"Kim", 95};
14 printStudent(s1);
15 return 0;
16}
실행 결과
Kim: 95점
8줄void printStudent(Student s) -- 구조체를 값으로 전달해요. s1의 복사본이 만들어져요.
9줄함수 안에서 s는 복사본이므로, 여기서 값을 바꿔도 원본 s1에는 영향이 없어요.

값 전달 vs 포인터 전달

값 전달은 복사본을 만들고, 포인터 전달은 원본에 직접 접근한다
구조체가 100바이트라면 값 전달 시 100바이트를 통째로 복사해야 해. 포인터를 쓰면 주소값 4~8바이트만 전달하면 되니까 훨씬 효율적이지!

화살표 연산자(->)

구조체 포인터로 멤버에 접근할 때는 점(.) 대신 화살표 연산자(->)를 사용해요. p->score는 (*p).score와 같은 뜻이에요.

변수에는 점(.), 포인터에는 화살표(->)
arrow_operator.c
1#include <stdio.h>
2
3typedef struct {
4 char name[20];
5 int score;
6} Student;
7
8void addBonus(Student *p, int bonus) {
9 p->score += bonus;
10}
11
12int main(void) {
13 Student s1 = {"Kim", 90};
14 printf("보너스 전: %d\n", s1.score);
15
16 addBonus(&s1, 5);
17 printf("보너스 후: %d\n", s1.score);
18 return 0;
19}
실행 결과
보너스 전: 90 보너스 후: 95
8줄Student *p -- 구조체 포인터를 매개변수로 받아요.
9줄p->score += bonus; -- 화살표 연산자로 원본의 score에 직접 접근해서 수정해요.
16줄addBonus(&s1, 5); -- s1의 주소를 전달해요. &를 잊으면 안 돼요!
포인터를 쓰면 원본이 바뀌니까, 수정할 때는 포인터 전달! 읽기만 할 때는 값 전달도 괜찮아.

구조체와 함수 핵심 정리
값 전달: 복사본이 만들어짐. 원본 안전하지만 복사 비용 있음
포인터 전달: 원본에 직접 접근. 효율적이지만 원본이 변경될 수 있음
화살표 연산자(->): 포인터로 멤버 접근. p->x == (*p).x
구조체 반환: 함수에서 구조체를 return으로 돌려줄 수 있음

11 void (*print)(const Shape*);
12 union {
13 struct { double radius; } circle;
14 struct { double width, height; } rect;
15 struct { double a, b, c; } triangle;
16 };
17};
18
19// Circle
20double circle_area(const Shape *s) { return 3.14159*s->circle.radius*s->circle.radius; }
21double circle_peri(const Shape *s) { return 2*3.14159*s->circle.radius; }
22void circle_print(const Shape *s) { printf("Circle(r=%.1f)", s->circle.radius); }
23
24Shape new_circle(double r) {
25 Shape s = {"Circle", circle_area, circle_peri, circle_print};
26 s.circle.radius = r;
27 return s;
28}
29
30// Rectangle
31double rect_area(const Shape *s) { return s->rect.width * s->rect.height; }
32double rect_peri(const Shape *s) { return 2*(s->rect.width + s->rect.height); }
33void rect_print(const Shape *s) { printf("Rect(%.1fx%.1f)", s->rect.width, s->rect.height); }

41// 다형성!
42void describe(const Shape *s) {

45}

50 };
51

53 for (int i = 0; i < 4; i++) {

56 }

1#include <stdio.h>

3
4typedef struct { float x, y; } Vec2;
5typedef struct { Vec2 pos; Vec2 vel; float radius; int active; const char *name; } Entity;
6
7void entity_update(Entity *e, float dt) {
8 e->pos.x += e->vel.x * dt;
9 e->pos.y += e->vel.y * dt;
10 // 경계 반사
11 if (e->pos.x < 0 || e->pos.x > 800) e->vel.x *= -1;
12 if (e->pos.y < 0 || e->pos.y > 600) e->vel.y *= -1;
13}
14
15int entity_collides(Entity *a, Entity *b) {
16 float dx = a->pos.x - b->pos.x;
17 float dy = a->pos.y - b->pos.y;
18 float dist = sqrt(dx*dx + dy*dy);
19 return dist < (a->radius + b->radius);
20}
21
22void entity_print(Entity *e) {
23 printf("%-8s pos=(%.1f,%.1f) vel=(%.1f,%.1f) r=%.0f %s\n",
24 e->name, e->pos.x, e->pos.y, e->vel.x, e->vel.y,
25 e->radius, e->active?"●":"○");
26}
27
28int main() {
29 Entity entities[] = {
30 {{100,100},{50,30},20,1,"Player"},
31 {{400,300},{-30,20},15,1,"Enemy1"},
32 {{200,400},{20,-40},10,1,"Bullet"}
33 };
34 int n = 3;
35 float dt = 0.1;
36
37 printf("=== Frame 0 ===\n");
38 for (int i = 0; i < n; i++) entity_print(&entities[i]);
39
40 for (int frame = 0; frame < 5; frame++)
41 for (int i = 0; i < n; i++) entity_update(&entities[i], dt);
42

44 for (int i = 0; i < n; i++) entity_print(&entities[i]);
45
46 // 충돌 체크
47 printf("\n=== Collisions ===\n");
48 for (int i = 0; i < n; i++)
49 for (int j = i+1; j < n; j++)
50 if (entity_collides(&entities[i], &entities[j]))

52 printf("(no collisions)\n");
53 return 0;
54}

[C 코드] 구조체 함수 — 메모리 풀 할당자
1#include <stdio.h>
2#include <string.h>
3
4#define POOL_SIZE 1024
5#define BLOCK_SIZE 32
6#define NUM_BLOCKS(POOL_SIZE/BLOCK_SIZE)
7
8typedef struct {
9 char memory[POOL_SIZE];
10 int used[NUM_BLOCKS];
11 int alloc_count;
12 int free_count;
13} MemPool;
14
15void pool_init(MemPool *p) {
16 memset(p->memory, 0, POOL_SIZE);
17 memset(p->used, 0, sizeof(p->used));
18 p->alloc_count = p->free_count = 0;
19}
20
21void* pool_alloc(MemPool *p) {
22 for (int i = 0; i < NUM_BLOCKS; i++) {
23 if (!p->used[i]) {
24 p->used[i] = 1;
25 p->alloc_count++;
26 return &p->memory[i * BLOCK_SIZE];
27 }
28 }
29 return NULL; // 풀 가득 참
30}
31
32void pool_free(MemPool *p, void *ptr) {
33 int offset = (char*)ptr - p->memory;
34 int idx = offset / BLOCK_SIZE;
35 if (idx >= 0 && idx < NUM_BLOCKS && p->used[idx]) {
36 p->used[idx] = 0;
37 p->free_count++;
38 }
39}
40
41void pool_stats(MemPool *p) {
42 int used = 0;
43 for (int i = 0; i < NUM_BLOCKS; i++) if (p->used[i]) used++;
44 printf("Pool: %d/%d blocks used, alloc=%d free=%d\n",
45 used, NUM_BLOCKS, p->alloc_count, p->free_count);
46}
47
48int main() {
49 MemPool pool;
50 pool_init(&pool);
51
52 void *ptrs[5];
53 for (int i = 0; i < 5; i++) {
54 ptrs[i] = pool_alloc(&pool);
55 sprintf((char*)ptrs[i], "Block %d data", i);
56 }
57 pool_stats(&pool);
58
59 pool_free(&pool, ptrs[1]);
60 pool_free(&pool, ptrs[3]);
61 pool_stats(&pool);
62
63 void *reuse = pool_alloc(&pool);
64 printf("Reused block: offset=%d\n", (int)((char*)reuse - pool.memory));
65 pool_stats(&pool);
66 return 0;
67}
실행 결과

[심화 내용] 메모리 할당 전략
malloc의 내부:
sbrk/mmap으로 OS에서 메모리 획득

6// 배열 이터레이터
7ArrayIter aiter_new(int *arr, int n) { return (ArrayIter){arr, n, 0}; }
8int aiter_has_next(ArrayIter *it) { return it->pos < it->size; }
9int aiter_next(ArrayIter *it) { return it->data[it->pos++]; }
10void aiter_reset(ArrayIter *it) { it->pos = 0; }
11
12// 범위 이터레이터
13RangeIter range(int start, int end, int step) {
14 return (RangeIter){start, end, step, start};
15}
16int range_has_next(RangeIter *r) {
17 return r->step > 0 ? r->current < r->end : r->current > r->end;
18}
19int range_next(RangeIter *r) { int v = r->current; r->current += r->step; return v; }
20
21// 제네릭 이터레이터 (함수 포인터)
22typedef struct {
23 void *state;
24 int (*has_next)(void*);
25 int (*next)(void*);
26} Iterator;
27
28int aiter_has_next_v(void *s) { return aiter_has_next((ArrayIter*)s); }
29int aiter_next_v(void *s) { return aiter_next((ArrayIter*)s); }
30int range_has_next_v(void *s) { return range_has_next((RangeIter*)s); }
31int range_next_v(void *s) { return range_next((RangeIter*)s); }
32
33// 제네릭 합계 (어떤 이터레이터든!)
34int iter_sum(Iterator it) {
35 int sum = 0;
36 while (it.has_next(it.state)) sum += it.next(it.state);
37 return sum;
38}
39
40int main() {
41 int arr[] = {10, 20, 30, 40, 50};
42 ArrayIter ai = aiter_new(arr, 5);
43 printf("Array: ");
44 while (aiter_has_next(&ai)) printf("%d ", aiter_next(&ai));
45 printf("\n");
46
47 RangeIter ri = range(1, 11, 2);
48 printf("Range(1,11,2): ");
49 while (range_has_next(&ri)) printf("%d ", range_next(&ri));
50 printf("\n");
51
52 // 제네릭
53 aiter_reset(&ai);
54 Iterator it1 = {&ai, aiter_has_next_v, aiter_next_v};
55 printf("Sum(array) = %d\n", iter_sum(it1));
56
57 RangeIter ri2 = range(1, 101, 1);
58 Iterator it2 = {&ri2, range_has_next_v, range_next_v};
59 printf("Sum(1..100) = %d\n", iter_sum(it2));
60 return 0;
61}

💡 구조체 함수 설계 원칙
✅ 읽기 전용이면 const 포인터: void f(const Point *p)
✅ 수정이면 포인터: void f(Point *p)
✅ 작은 구조체(≤16B)는 값 반환: Point f()
✅ 큰 구조체는 포인터로 결과 전달
✅ 생성자 패턴: Point point_new(x, y)
✅ 메서드 패턴: 첫 인자가 self (Point *self)
✅ 에러 반환: int 반환값 + 결과는 포인터 파라미터
self parameter
자기 참조. C 관례: 첫 인자가 구조체 포인터. stack_push(Stack *self, int val). C++의 this!
method chaining
메서드 체이닝. 자기 자신을 반환. builder_set(b,"name","Naba")->set(b,"age","1"). 유연한 API!
callback
콜백. 함수를 인자로 전달. qsort의 비교 함수. 이벤트 핸들러. 비동기 프로그래밍의 기본!
const Point *p = 읽기 전용! 함수가 구조체를 수정하지 않겠다는 약속. API 문서화!
생성자 패턴! Point point_new(3,4). 초기화를 함수로 캡슐화하면 실수 방지!
pass by value
값 전달. 구조체 전체 복사. 작은 struct OK. 큰 struct는 포인터로! 복사 비용 O(sizeof).
designated init
C99. (Vec2){.x=3, .y=4}. 멤버 이름 지정. 0 자동 초기화! 누락 멤버는 0.
builder pattern
빌더 패턴. 단계별 객체 구성. C: 함수 체인으로 시뮬레이션. 복잡한 초기화에 유용!
destructor pattern
소멸자 패턴. C: xxx_destroy(obj) 수동 호출. C++: ~Class() 자동. Rust: Drop 자동.
구조체+함수 마스터! OOP·게임엔진·메모리풀·이터레이터. 함수형 프로그래밍까지!
function overloading C
C에는 오버로딩 없음! C11 _Generic으로 시뮬레이션. print_any(x)→타입별 함수 분기.
struct return
구조체 반환. Point f(){return (Point){3,4};} 컴파일러가 RVO(반환값 최적화) 적용!
RVO
Return Value Optimization. 반환 구조체를 호출자 메모리에 직접 구성. 복사 제거! 컴파일러 최적화.
RVO = 구조체 반환이 무료! 컴파일러가 복사를 제거. Point p = make_point(3,4);
화살표 연산자(->)
구조체 포인터에서 멤버에 접근하는 연산자. p->x는 (*p).x와 같다.
값 전달(pass by value)
함수 호출 시 인자의 복사본을 만들어 전달하는 방식.
포인터 전달(pass by pointer)
함수 호출 시 인자의 주소를 전달하여 원본에 접근하는 방식.
구조체 반환
함수에서 구조체를 return으로 돌려주는 것. 구조체 전체가 복사된다.

1
구조체를 함수에 값으로 전달하면 복사본이 만들어진다
2
포인터로 전달하면 원본에 직접 접근할 수 있다
3
구조체 포인터에는 화살표 연산자(->)를 사용한다
4
p->score 는 (*p).score 와 같다
5
구조체를 return으로 반환할 수도 있다
