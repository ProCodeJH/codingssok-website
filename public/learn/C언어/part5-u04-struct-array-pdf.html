학생 30명을 관리하려면?

배열과 반복문은 늘 같이 다녀. 구조체 배열도 마찬가지야. for문 안에서 조건 검사하는 패턴을 익혀두면 검색, 정렬 등 다양한 작업이 가능해.
구조체 배열 핵심 정리
선언: Student arr[30]; -- 구조체 30개짜리 배열
초기화: { {..}, {..}, {..} } -- 각 요소별 초기값
접근: arr[i].멤버이름 -- 인덱스 + 점 연산자
반복문: for (int i = 0; i < n; i++) arr[i].멤버이름
[실생활] 구조체 배열의 활용
성적 관리 시스템, 도서관 책 목록, 주소록, 게임 캐릭터 목록 등 현실 세계의 데이터 관리 대부분이 '구조체 + 배열' 조합으로 구현돼요.

11int db_add(const char *name, int age, double salary) {
12 if (db_count >= MAX_EMP) return -1;

15 db[db_count].age = age;

19
20Employee* db_find_by_name(const char *name) {

26void db_sort_by_salary() {
27 for (int i = 0; i < db_count-1; i++)
28 for (int j = 0; j < db_count-1-i; j++)

30 Employee t = db[j]; db[j] = db[j+1]; db[j+1] = t;

41void db_stats() {

[심화 내용] 구조체 배열과 데이터베이스
CRUD 연산:
Create: db_add() → INSERT
Read: db_find() → SELECT WHERE
Update: 직접 수정 → UPDATE SET
Delete: 마지막 원소로 덮어쓰기 → DELETE

인덱싱:
선형 탐색: O(n) — 현재 방식
정렬+이진탐색: O(log n)
해시 테이블: O(1) 평균
B-트리: O(log n) 디스크 최적화

메모리 vs 디스크:
현재: 배열 = 인메모리 DB
파일: fwrite/fread = 영구 저장
SQL: SQLite = 파일 기반 관계형 DB
CRUD
Create-Read-Update-Delete. 데이터 관리의 4대 연산. REST API: POST/GET/PUT/DELETE 매핑!
index
인덱스. 검색 속도 향상. 정렬된 별도 구조. 이진 탐색 O(log n). B-트리가 DB 표준!
구조체 배열 = 인메모리 데이터베이스! CRUD+정렬+필터+통계. SQL 없이도 강력!
구조체 배열 정렬: 구조체 전체를 교환! 포인터 배열이면 포인터만 교환→더 빠름!
[C 코드] 구조체 배열 — 해시 테이블
1#include <stdio.h>
2#include <string.h>
3
4#define TABLE_SIZE 16
5
6typedef struct { char key[30]; int value; int used; } Entry;
7
8Entry table[TABLE_SIZE];
9
10unsigned int hash(const char *key) {
11 unsigned int h = 5381;
12 while (*key) h = ((h << 5) + h) + *key++;
13 return h % TABLE_SIZE;
14}
15
16int ht_set(const char *key, int value) {
17 unsigned int idx = hash(key);
18 for (int i = 0; i < TABLE_SIZE; i++) {
19 int probe = (idx + i) % TABLE_SIZE;
20 if (!table[probe].used || strcmp(table[probe].key, key)==0) {
21 strncpy(table[probe].key, key, 29);
22 table[probe].value = value;
23 table[probe].used = 1;
24 return probe;
25 }
26 }
27 return -1;
28}
29
30int ht_get(const char *key, int *value) {
31 unsigned int idx = hash(key);
32 for (int i = 0; i < TABLE_SIZE; i++) {
33 int probe = (idx + i) % TABLE_SIZE;
34 if (!table[probe].used) return 0;
35 if (strcmp(table[probe].key, key)==0) { *value = table[probe].value; return 1; }
36 }
37 return 0;
38}
39
40int main() {
41 ht_set("apple", 3); ht_set("banana", 5); ht_set("cherry", 2);
42 ht_set("date", 8); ht_set("elderberry", 1);
43
44 const char *keys[] = {"apple","banana","cherry","grape"};
45 for (int i = 0; i < 4; i++) {
46 int val;
47 if (ht_get(keys[i], &val)) printf("%s = %d\n", keys[i], val);
48 else printf("%s = (not found)\n", keys[i]);
49 }
50 return 0;
51}

[C 코드] 구조체 배열 — 우선순위 큐
1#include <stdio.h>
2#include <string.h>
3
4typedef struct { char task[50]; int priority; } PQItem;
5
6#define PQ_MAX 20
7PQItem pq[PQ_MAX];
8int pq_size = 0;
9
10void pq_swap(int i, int j) { PQItem t = pq[i]; pq[i] = pq[j]; pq[j] = t; }
11
12void pq_push(const char *task, int pri) {
13 if (pq_size >= PQ_MAX) return;
14 strncpy(pq[pq_size].task, task, 49);
15 pq[pq_size].priority = pri;
16 // sift up
17 int i = pq_size++;
18 while (i > 0) {
19 int parent = (i-1)/2;
20 if (pq[i].priority > pq[parent].priority) { pq_swap(i, parent); i = parent; }
21 else break;
22 }
23}
24
25PQItem pq_pop() {
26 PQItem top = pq[0];
27 pq[0] = pq[--pq_size];
28 // sift down
29 int i = 0;
30 while (2*i+1 < pq_size) {
31 int child = 2*i+1;
32 if (child+1 < pq_size && pq[child+1].priority > pq[child].priority) child++;
33 if (pq[child].priority > pq[i].priority) { pq_swap(i, child); i = child; }
34 else break;
35 }
36 return top;
37}
38

40 pq_push("버그 수정", 10);
41 pq_push("문서 작성", 3);
42 pq_push("서버 장애", 9);
43 pq_push("코드 리뷰", 5);
44 pq_push("배포 준비", 8);
45 pq_push("미팅", 2);
46
47 printf("=== Priority Queue (heap) ===\n");
48 while (pq_size > 0) {
49 PQItem item = pq_pop();

51 }

53}

[심화 내용] 힙과 우선순위 큐
이진 힙 (Binary Heap):
완전 이진 트리를 배열로 표현!
부모: (i-1)/2, 왼쪽: 2i+1, 오른쪽: 2i+2
Max-heap: 부모 ≥ 자식
Min-heap: 부모 ≤ 자식

시간 복잡도:
push: O(log n) — sift up
pop: O(log n) — sift down
peek: O(1) — 루트
heapify: O(n) — 바닥부터!

응용:
Dijkstra 최단 경로, Prim MST
A* 게임 길찾기, 작업 스케줄링
Huffman 압축, 이벤트 시뮬레이션
heap
힙. 완전 이진 트리+순서 속성. 배열로 구현! 부모=(i-1)/2. push/pop O(log n). 우선순위 큐의 표준!
Dijkstra
다익스트라. 최단 경로 알고리즘. 우선순위 큐로 가장 가까운 노드부터 탐색! O(E log V).
힙 = 배열로 트리! 부모=(i-1)/2, 자식=2i+1. 구조체 배열이 곧 트리 자료구조!
우선순위 큐 = 항상 가장 중요한 작업 먼저! 버그 수정(P10)이 미팅(P2)보다 먼저!
[C 코드] 구조체 배열 — LRU 캐시
1#include <stdio.h>
2#include <string.h>
3
4#define CACHE_SIZE 4
5
6typedef struct { char key[20]; int value; int timestamp; } CacheEntry;
7
8CacheEntry cache[CACHE_SIZE];
9int cache_count = 0;
10int global_time = 0;
11
12int cache_get(const char *key, int *value) {
13 for (int i = 0; i < cache_count; i++) {
14 if (strcmp(cache[i].key, key) == 0) {
15 cache[i].timestamp = ++global_time;
16 *value = cache[i].value;
17 return 1; // HIT
18 }
19 }
20 return 0; // MISS
21}
22
23void cache_put(const char *key, int value) {
24 // 이미 있으면 업데이트
25 for (int i = 0; i < cache_count; i++) {
26 if (strcmp(cache[i].key, key) == 0) {
27 cache[i].value = value;
28 cache[i].timestamp = ++global_time;
29 return;
30 }
31 }
32 // 가득 차면 LRU 퇴출
33 if (cache_count >= CACHE_SIZE) {
34 int lru = 0;
35 for (int i = 1; i < CACHE_SIZE; i++)
36 if (cache[i].timestamp < cache[lru].timestamp) lru = i;
37 printf(" evict: %s\n", cache[lru].key);
38 strncpy(cache[lru].key, key, 19);
39 cache[lru].value = value;
40 cache[lru].timestamp = ++global_time;
41 } else {
42 strncpy(cache[cache_count].key, key, 19);
43 cache[cache_count].value = value;
44 cache[cache_count].timestamp = ++global_time;
45 cache_count++;
46 }
47}

49int main() {

51 for (int i = 0; i < 8; i++) {
52 int v;
53 if (cache_get(ops[i], &v)) printf("GET %s → HIT (%d)\n", ops[i], v);
54 else { printf("GET %s → MISS, PUT\n", ops[i]); cache_put(ops[i], (i+1)*10); }
55 }

LRU
Least Recently Used. 가장 오래 사용 안 된 항목 퇴출. CPU 캐시, 페이지 교체, Redis에 활용!
cache hit rate
캐시 적중률. 히트/(히트+미스). 90%↑이면 좋은 캐시. 지역성(locality)이 적중률 결정!
LRU 캐시 = 구조체 배열의 실전! timestamp로 최근 사용 추적. 가장 오래된 걸 퇴출!
sentinel value
센티넬 값. 배열 끝 표시. 문자열의 \0. NULL 포인터. -1 반환. 종료 조건으로 활용!
stable sort
안정 정렬. 같은 값의 원래 순서 유지. 삽입 정렬, 병합 정렬은 안정. 퀵소트는 불안정!
locality
지역성. 시간적: 최근 접근한 것 재접근. 공간적: 근처 메모리 접근. 배열이 리스트보다 빠른 이유!
구조체 배열 마스터! DB·이벤트·해시테이블·우선순위큐·LRU캐시. 자료구조의 기본!
indirect sort
간접 정렬. 인덱스 배열을 정렬! 원본 유지. int idx[]={0,1,2,...}; 비교: arr[idx[i]]
counting sort
계수 정렬. O(n+k). 범위가 작은 정수에 최적! 각 값의 개수를 세고 누적합으로 위치 결정.
radix sort
기수 정렬. O(d·n). 자릿수별 정렬. 문자열/IP주소에 최적! 안정 정렬 필수(counting sort).
간접 정렬 = 포인터 정렬! 큰 구조체를 직접 교환 대신 인덱스만 교환. 2배 빠름!
sentinel value
감시값. 배열 끝에 특별한 값(-1, NULL). 크기 전달 없이 끝 감지! C 문자열의 \0이 대표.
stable sort
안정 정렬. 같은 값의 순서 보존. 삽입정렬=안정, 퀵소트=불안정. 다중 키 정렬에 중요!
구조체 배열 = 엑셀 시트! 각 행이 구조체, 각 열이 멤버. 정렬/필터/검색 전부 가능!
cache locality
캐시 지역성. 연속 메모리가 빠름! 구조체 배열은 연속→캐시 친화적. 연결리스트는 분산→느림.
포인터 배열로 정렬하면 구조체 복사 없이 포인터만 교환! 큰 구조체일수록 효과적!
구조체 배열
같은 구조체 타입의 변수 여러 개를 연속된 메모리에 나란히 배치한 것.
인덱스 접근
배열이름[인덱스].멤버이름 형태로 특정 요소의 특정 멤버에 접근하는 것.
순회(traversal)
배열의 모든 요소를 반복문으로 하나씩 방문하는 것.
