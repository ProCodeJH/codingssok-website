숫자에 이름을 붙이자

코드에서 if (status == 3) 이라고 쓰면, 3이 뭘 뜻하는지 알 수 없어. DONE이라고 쓰면 바로 이해가 되지? 이게 enum의 핵심이야!

열거형(enum)은 관련된 상수들에 이름을 붙여서 정의하는 자료형이에요. 내부적으로는 정수(int)로 저장되지만, 코드에서는 의미 있는 이름으로 사용할 수 있어요.

enum의 각 항목은 0부터 자동으로 번호가 매겨진다

enum 정의와 사용

3줄enum Season -- 계절을 나타내는 열거형을 정의해요. 각 항목은 0부터 순서대로 번호가 매겨져요.
6줄enum Season today = FALL; -- today 변수에 FALL(=2)을 저장해요.
enum 항목은 이름일 뿐, 실제로는 정수야. printf로 %d를 쓰면 숫자가 나와. 하지만 코드에서는 FALL이라고 쓰니까 의미가 명확하지!

매직 넘버를 없애자

매직 넘버 대신 enum을 쓰면 코드 가독성이 크게 향상된다
코드에 의미 없는 숫자가 나오면 '매직 넘버'라고 불러. enum이나 #define으로 이름을 붙여주는 것이 좋은 코딩 습관이야.

[C 코드] enum — 토큰 타입 + 렉서
1#include <stdio.h>
2#include <ctype.h>
3#include <string.h>
4
5typedef enum {
6 TOK_NUMBER, TOK_PLUS, TOK_MINUS, TOK_STAR, TOK_SLASH,
7 TOK_LPAREN, TOK_RPAREN, TOK_EOF, TOK_ERROR
8} TokenType;
9
10const char *tok_names[] = {
11 "NUMBER","PLUS","MINUS","STAR","SLASH","LPAREN","RPAREN","EOF","ERROR"
12};
13
14typedef struct { TokenType type; double value; } Token;
15
16typedef struct { const char *input; int pos; } Lexer;
17
18Token lexer_next(Lexer *l) {
19 while (l->input[l->pos] && isspace(l->input[l->pos])) l->pos++;
20 char c = l->input[l->pos];
21 if (!c) return (Token){TOK_EOF, 0};
22 if (isdigit(c) || c=='.') {
23 double val = 0; int dec = 0; double frac = 1;
24 while (isdigit(l->input[l->pos])) val = val*10+(l->input[l->pos++]-'0');
25 if (l->input[l->pos]=='.') { l->pos++;
26 while(isdigit(l->input[l->pos])){frac/=10;val+=(l->input[l->pos++]-'0')*frac;}}
27 return (Token){TOK_NUMBER, val};
28 }
29 l->pos++;
30 switch(c) {
31 case '+': return (Token){TOK_PLUS,0};
32 case '-': return (Token){TOK_MINUS,0};
33 case '*': return (Token){TOK_STAR,0};
34 case '/': return (Token){TOK_SLASH,0};
35 case '(': return (Token){TOK_LPAREN,0};
36 case ')': return (Token){TOK_RPAREN,0};
37 default: return (Token){TOK_ERROR,0};
38 }
39}
40
41int main() {
42 Lexer l = {"3.14 + (42 - 7) * 2.5", 0};
43 printf("Input: \"%s\"\n\nTokens:\n", l.input);
44 Token t;
45 do {
46 t = lexer_next(&l);
47 if (t.type == TOK_NUMBER) printf(" %-8s %.2f\n", tok_names[t.type], t.value);
48 else printf(" %-8s\n", tok_names[t.type]);
49 } while (t.type != TOK_EOF);

[C 코드] enum — 명령 패턴 + 실행 취소
1#include <stdio.h>
2#include <string.h>
3
4typedef enum { CMD_SET, CMD_ADD, CMD_MUL, CMD_CLEAR } CmdType;
5const char *cmd_names[] = {"SET","ADD","MUL","CLEAR"};
6
7typedef struct {
8 CmdType type;
9 double operand;
10 double prev_value; // undo용
11} Command;
12
13#define HISTORY_SIZE 20
14Command history[HISTORY_SIZE];
15int history_top = -1;
16double value = 0;
17
18void execute(CmdType type, double operand) {
19 Command cmd = {type, operand, value};
20 switch(type) {
21 case CMD_SET: value = operand; break;
22 case CMD_ADD: value += operand; break;
23 case CMD_MUL: value *= operand; break;
24 case CMD_CLEAR: value = 0; break;
25 }
26 if (history_top < HISTORY_SIZE-1)
27 history[++history_top] = cmd;
28 printf(" %s %.1f → value = %.1f\n", cmd_names[type], operand, value);
29}
30
31int undo() {
32 if (history_top < 0) return 0;
33 Command cmd = history[history_top--];
34 printf(" UNDO %s → value = %.1f\n", cmd_names[cmd.type], cmd.prev_value);
35 value = cmd.prev_value;
36 return 1;
37}
38
39int main() {
40 printf("=== Commands ===\n");
41 execute(CMD_SET, 10);
42 execute(CMD_ADD, 5);
43 execute(CMD_MUL, 3);
44 execute(CMD_ADD, -2);
45
46 printf("\n=== Undo ===\n");
47 undo();
48 undo();
49 printf("\nFinal value: %.1f\n", value);
50 return 0;
51}

41 inv->slots[i].count += item.count;
42 return i;
43 }
44 }
45 if (inv->used >= inv->capacity) return -1;
46 inv->slots[inv->used] = item;
47 return inv->used++;
48}

magic number
매직 넘버. 의미 없는 숫자 상수. if(status==3) → 뭐가 3? enum으로 제거! 코드 가독성 핵심.
enum vs define
enum은 타입 정보 있음. #define은 텍스트 치환. enum은 디버거에서 이름 표시! enum 선호.
enum COUNT 트릭! enum Color { RED, GREEN, BLUE, COLOR_COUNT }; 배열 크기에 COLOR_COUNT!
FSM in practice
FSM 실전. 문자열 파서, 정규표현식, 네트워크 프로토콜, UI 상태, 게임 AI. 어디에나!
enum underlying type
C enum의 내부 타입 = int. sizeof(enum) = sizeof(int) = 4. C++11: enum class : uint8_t도 가능!
exhaustive switch
모든 enum 값 처리. -Wswitch-enum 경고. 새 값 추가 시 누락 즉시 발견! default 안 쓰기.
enum 마스터! 상태머신·에러코드·비트플래그·렉서·명령패턴·게임아이템. 만능!
double dispatch
이중 디스패치. 두 타입 조합에 따라 함수 결정. Visitor 패턴. C에서는 함수 포인터 2D 배열!
flyweight enum
플라이웨이트. 공유 가능한 가벼운 객체. enum 값 자체가 인스턴스! 메모리 절약.
enum = 정수에 의미 부여! 0이 뭔지 모르지만 STATE_IDLE은 바로 이해. 코드가 문서!
enum 크기
C에서 enum은 int 크기(보통 4바이트). gcc -fshort-enums로 최소화 가능. 임베디드에서 중요!
anonymous enum
익명 enum. enum { MAX=100, BUF=256 }; 이름 없이 상수만 정의! #define 대체.
enum = 숫자에 이름표! 0,1,2 대신 RED,GREEN,BLUE. 코드 읽을 때 의미가 바로 보여!
dispatch table
디스패치 테이블. enum을 인덱스로 함수 포인터 배열 접근. switch보다 빠름! O(1) 분기.
enum + 함수 포인터 배열 = O(1) 디스패치! switch의 if-else 체인보다 빠르고 깔끔!
열거형(enum)
관련된 정수 상수들에 이름을 붙여 정의하는 사용자 정의 자료형.
열거 상수
enum 안에 정의된 각각의 이름 붙은 정수값.
매직 넘버(magic number)
코드에 직접 쓰인 의미 불명의 숫자. 가독성을 해치므로 이름을 붙여야 한다.
가독성(readability)
코드를 읽고 이해하기 쉬운 정도. enum, 함수명, 주석 등으로 높일 수 있다.
