Part 1 진행률: 8%

struct는 '각자 방', union은 '같은 방'

struct는 멤버마다 자기 방이 있어서 동시에 살 수 있어. 하지만 union은 멤버들이 같은 방 하나를 돌아가며 써. 한 번에 한 명만 쓸 수 있는 거야!

공용체(union)는 여러 멤버가 같은 메모리 공간을 공유하는 자료형이에요. struct처럼 멤버를 선언하지만, 모든 멤버가 같은 시작 주소를 사용해요.

union: 모든 멤버가 같은 메모리를 공유한다

union vs struct 비교

struct vs union: 메모리 배치 차이
struct vs union 핵심 차이
struct: 모든 멤버가 각자 메모리를 가짐. 크기 = 멤버 크기의 합(+패딩)
union: 모든 멤버가 같은 메모리를 공유. 크기 = 가장 큰 멤버의 크기

union은 한 번에 하나의 멤버만 유효하게 사용할 수 있어요.

실행 결과
union 크기: 20바이트 d.i = 42 d.f = 3.14 d.i = 1078523331 (덮어씌워짐!)
3줄union Data — union 키워드로 공용체를 정의해요. 문법은 struct와 동일해요.
11줄sizeof 결과는 가장 큰 멤버인 char str[20]의 20바이트예요.
16줄d.f = 3.14를 쓰면 같은 메모리가 덮어써져서 d.i 값이 엉망이 돼요.
union의 크기가 20바이트인 이유는 가장 큰 멤버 char str[20] 때문이야. int는 4바이트, float도 4바이트지만, 20바이트 안에 모두 들어갈 수 있으니까!

바이트 오버랩 시각화

int, float, char[20]이 같은 시작 주소에서 겹치는 모습

위 그림처럼 모든 멤버는 같은 주소 0x00에서 시작해요. int와 float는 처음 4바이트를 공유하고, str은 20바이트 전체를 차지해요.

실무 활용: 타입 태그 패턴

실무에서는 union과 struct를 함께 써서 여러 타입의 값을 하나의 변수로 다루는 패턴이 많아요. 어떤 멤버가 유효한지 알려주는 태그(tag)를 함께 저장해요.

tagged_union.c
1#include <stdio.h>
2
3typedef struct {
4 int type; /* 0: int, 1: float, 2: string */
5 union {
6 int i;
7 float f;
8 char s[32];
9 } value;
10} Variant;
11
12void printVariant(Variant v) {
13 switch (v.type) {
14 case 0: printf("int: %d\n", v.value.i); break;
15 case 1: printf("float: %.2f\n", v.value.f); break;
16 case 2: printf("string: %s\n", v.value.s); break;
17 }
18}
19
20int main(void) {
21 Variant a = {0, .value.i = 100};
22 Variant b = {1, .value.f = 3.14};
23
24 printVariant(a);
25 printVariant(b);
26 return 0;
27}
실행 결과
int: 100 float: 3.14
이런 패턴을 tagged union 또는 discriminated union이라고 불러. 네트워크 프로토콜 파싱이나 JSON 파서 같은 곳에서 정말 많이 쓰여.

[심화 내용] union의 메모리와 UB
union 크기 = 가장 큰 멤버:
union { int i; double d; char s[50]; } → 56바이트
(50 + 패딩 6)

Type Punning:
union { float f; int i; } u;
u.f = 3.14f; printf("%d", u.i);
→ float의 비트 패턴을 int로 읽기
→ C99에서는 정의됨! C++에서는 UB!

Rust의 안전한 대안:
enum Value { Int(i32), Float(f64), Str(String) }
패턴 매칭 필수 → 잘못된 접근 컴파일 에러!
tagged union
태그+유니언. type 필드로 현재 타입 표시. JSON, 인터프리터 값, 이벤트 시스템에 핵심!
type punning
타입 변장. union으로 같은 메모리를 다른 타입으로 읽기. float→int 비트 변환! C99 허용.
Tagged Union = C의 다형적 값! int도 되고 string도 되는 만능 타입. JSON 파서의 기본!

8struct JsonNode {
9 JsonType type;
10 char *key; // object의 키
11 union {

13 double number;
14 char *string;

16 struct { JsonNode **pairs; int count; } object;
17 };
18};
19
20JsonNode* json_null() { JsonNode *n=calloc(1,sizeof(JsonNode)); n->type=J_NULL; return n; }

24
25void json_print(JsonNode *n, int indent) {
26 for(int i=0;i<indent;i++) printf(" ");

29 case J_NULL: printf("null"); break;

32 case J_STR: printf("\"%s\"", n->string); break;

[심화 내용] 대수적 데이터 타입
Tagged Union = Sum Type:
JsonNode = Null | Bool | Num | Str | Array | Object
값이 여러 타입 중 하나! 'OR' 타입

Product Type (곱 타입):
struct = AND 타입. (name AND age AND ...)
모든 멤버가 동시 존재

Algebraic Data Types (ADT):
Sum + Product = 대수적 타입
Haskell: data Json = JNull | JNum Double | ...
Rust: enum Json { Null, Num(f64), Str(String), ... }
패턴 매칭으로 모든 경우 처리 보장!

재귀적 타입:
JsonNode가 JsonNode 포인터를 포함
→ 트리 구조 자연 표현!

동적 타입 언어의 내부:
모든 값 = Tagged Union (tag + data)
Python PyObject: type pointer + ref count + data
JavaScript V8: NaN boxing 또는 pointer tagging

타입 강제 변환 (Coercion):
int + float → float (promotion)
string + int → string (concatenation)
JavaScript: "5" + 3 = "53" (문자열 우선)
Python: "5" + 3 = TypeError!

Truthiness:
falsy: nil, false, 0, 0.0, ""
truthy: 나머지 전부
Python/JavaScript/Ruby 전부 이 규칙!
coercion
타입 강제 변환. int→float 자동 승격. "5"+3="53" (JS). 암시적 변환은 버그의 원천! 명시적이 안전.
미니 인터프리터! int+float→float, str+int→str. Python/JS의 동적 타입이 이렇게 동작!

anonymous union

discriminated union

variant

union 마스터! tagged union·JSON트리·패킷파서·인터프리터·ADT. 타입의 유연성!

부동소수점 표준. sign(1)+exponent(8)+mantissa(23)=32bit. union으로 비트 분석 가능!

tagged union
