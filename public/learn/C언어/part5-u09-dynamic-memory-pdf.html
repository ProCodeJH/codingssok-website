동적 메모리 할당

malloc, calloc, realloc, free를 배워요. 실행 시간에 필요한 만큼 메모리를 할당하고 해제하는 방법을 익혀요.

📚 이번에 배울 것

  • 동적 메모리 할당이 필요한 이유를 설명할 수 있어요.
  • malloc, calloc, realloc, free를 올바르게 사용할 수 있어요.
  • 메모리 누수와 dangling pointer 문제를 이해해요.
  • 동적 배열을 만들어 활용할 수 있어요.

✅ 시작 전 체크

  • 포인터의 개념과 사용법을 안다
  • 배열과 sizeof를 사용할 수 있다
Part 1 진행률: 9%

배열 크기를 미리 정해야 할까?

학생 수가 30명이면 int scores[30]으로 충분해. 그런데 학생 수를 사용자가 입력한다면? 배열 크기를 1000으로 잡으면 낭비, 10으로 잡으면 부족해. 이런 문제를 해결해 줄게!

지금까지 배열을 선언할 때 크기를 컴파일 시점에 정해야 했어요. 동적 메모리 할당은 프로그램이 실행되는 동안 필요한 만큼 메모리를 요청하는 방법이에요.

스택(자동 변수)과 힙(동적 할당)의 메모리 영역
스택 vs 힙
스택(Stack): 지역 변수가 자동으로 할당/해제되는 영역. 크기가 고정적.
힙(Heap): 프로그래머가 직접 할당/해제하는 영역. 크기가 유동적.

동적 메모리는 힙에 할당돼요.

malloc으로 메모리 할당하기

malloc은 Memory ALLOCation의 줄임말이야. 원하는 바이트 수를 알려주면, 그만큼 메모리를 힙에서 빌려줘!
malloc_basic.c
1#include <stdio.h>
2#include <stdlib.h>
3
4int main(void) {
5 int n;
6 printf("학생 수 입력: ");
7 scanf("%d", &n);
8
9 int *scores = (int *)malloc(n * sizeof(int));
10 if (scores == NULL) {
11 printf("메모리 할당 실패!\n");
12 return 1;
13 }
14
15 for (int i = 0; i < n; i++) {
16 printf("학생 %d 점수: ", i + 1);
17 scanf("%d", &scores[i]);
18 }
19
20 int sum = 0;
21 for (int i = 0; i < n; i++) {
22 sum += scores[i];
23 }
24 printf("평균: %.1f\n", (double)sum / n);
25
26 free(scores);
27 return 0;
28}
2줄#include <stdlib.h> — malloc, free 등 동적 메모리 함수가 들어있어요.
9줄malloc(n * sizeof(int)) — n개의 int 크기만큼 메모리를 힙에서 할당해요.
10줄malloc이 실패하면 NULL을 반환해요. 반드시 확인해야 해요!
26줄free(scores) — 다 쓴 메모리를 반드시 해제해요. 안 하면 메모리 누수!
malloc/free 흐름: 요청 -> 할당 -> 사용 -> 해제

calloc과 realloc

calloc은 malloc과 비슷하지만 할당된 메모리를 0으로 초기화해요. realloc은 이미 할당된 메모리의 크기를 변경해요.

calloc_realloc.c
1#include <stdio.h>
2#include <stdlib.h>
3
4int main(void) {
5 /* calloc: 5개의 int를 0으로 초기화하여 할당 */
6 int *arr = (int *)calloc(5, sizeof(int));
7 if (arr == NULL) return 1;
8
9 printf("calloc 초기값: ");
10 for (int i = 0; i < 5; i++) {
11 printf("%d ", arr[i]); /* 모두 0 */
12 }
13 printf("\n");
14
15 /* realloc: 크기를 10개로 확장 */
16 int *temp = (int *)realloc(arr, 10 * sizeof(int));
17 if (temp == NULL) {
18 free(arr);
19 return 1;
20 }
21 arr = temp;
22
23 for (int i = 5; i < 10; i++) {
24 arr[i] = i * 10;
25 }
26
27 printf("확장 후: ");
28 for (int i = 0; i < 10; i++) {
29 printf("%d ", arr[i]);
30 }
31 printf("\n");
32
33 free(arr);
34 return 0;
35}
실행 결과
calloc 초기값: 0 0 0 0 0 확장 후: 0 0 0 0 0 50 60 70 80 90
realloc을 쓸 때 주의! 실패하면 NULL을 반환하지만 원래 메모리는 그대로야. 그래서 temp 변수로 받은 다음 확인하는 거야. 바로 arr = realloc(arr, ...)하면 실패 시 원래 주소를 잃어버려!

메모리 누수와 dangling pointer

메모리 누수: free를 안 하면 힙 메모리가 계속 쌓인다
동적 메모리의 두 가지 위험
1. 메모리 누수(memory leak): malloc으로 할당한 메모리를 free하지 않으면 프로그램이 끝날 때까지 메모리가 낭비된다.
2. dangling pointer: free한 메모리를 가리키는 포인터. 이 포인터로 접근하면 미정의 동작이 발생한다.

해결: free 후 포인터를 NULL로 설정하는 습관을 들이자.
dangling_fix.c
1#include <stdio.h>
2#include <stdlib.h>
3
4int main(void) {
5 int *p = (int *)malloc(sizeof(int));
6 *p = 42;
7 printf("%d\n", *p);
8
9 free(p);
10 p = NULL; /* dangling pointer 방지! */
11
12 /* if (p != NULL) 체크로 안전하게 사용 */
13 if (p != NULL) {
14 printf("%d\n", *p);
15 } else {
16 printf("메모리가 해제되었습니다.\n");
17 }
18
19 return 0;
20}
segfault(세그멘테이션 오류)가 나면 동적 메모리 문제를 의심해 봐. free한 메모리에 접근하거나, 같은 메모리를 두 번 free(double free)하면 프로그램이 죽어!
[디버깅] 동적 메모리 실수 TOP 3
1. free 안 함 — 메모리 누수. 장시간 실행 프로그램에서 치명적.
2. free 후 접근 — dangling pointer. 세그멘테이션 오류 발생.
3. double free — 이미 해제한 메모리를 또 해제. 프로그램 크래시.

4
5typedef struct {
6 int *data;
7 int size;

9} Vector;
10
11Vector vec_new() {
12 Vector v = {(int*)malloc(4*sizeof(int)), 0, 4};
13 return v;
14}
15
16void vec_push(Vector *v, int val) {
17 if (v->size == v->capacity) {
18 v->capacity *= 2;
19 v->data = (int*)realloc(v->data, v->capacity * sizeof(int));
20 }
21 v->data[v->size++] = val;
22}
23
24int vec_pop(Vector *v) {
25 return v->size > 0 ? v->data[--v->size] : -1;
26}
27
28void vec_insert(Vector *v, int idx, int val) {
29 vec_push(v, 0); // 공간 확보
30 memmove(&v->data[idx+1], &v->data[idx], (v->size-idx-1)*sizeof(int));
31 v->data[idx] = val;
32}
33
34void vec_remove(Vector *v, int idx) {
35 memmove(&v->data[idx], &v->data[idx+1], (v->size-idx-1)*sizeof(int));
36 v->size--;
37}
38
39void vec_print(Vector *v) {

41 for (int i = 0; i < v->size; i++) printf("%d ", v->data[i]);

43}

[심화 내용] 동적 배열의 분할 상환 분석
용량 2배 전략:
push n개: 재할당 log₂n 번
복사 총합: 1+2+4+...+n = 2n-1
평균: O(1) per push! (분할 상환)

1.5배 전략:
메모리 낭비 감소, 재할당 약간 더 자주
C++ vector, Java ArrayList: 보통 1.5~2배

축소 전략:
size < capacity/4 → 절반으로 축소
hysteresis: 확장 2배, 축소 1/4에서 → 요동 방지
amortized O(1)
분할 상환. 가끔 O(n)이지만 평균 O(1). 동적 배열 push, 해시 테이블 확장에 적용!
realloc
메모리 재할당. 기존 데이터 유지+크기 변경. 같은 위치 확장 or 새 위치 복사. NULL 반환 체크 필수!
Vector = 동적 배열의 정석! push O(1) 분할상환, insert/remove O(n). C++의 std::vector!
[C 코드] 동적 메모리 — 해시맵 구현
1#include <stdio.h>
2#include <stdlib.h>
3#include <string.h>
4
5typedef struct Entry { char *key; int value; struct Entry *next; } Entry;
6typedef struct { Entry **buckets; int size; int count; } HashMap;
7
8unsigned int hash_str(const char *s, int size) {
9 unsigned int h = 0;
10 while (*s) h = h * 31 + *s++;
11 return h % size;
12}
13
14HashMap* hm_create(int size) {
15 HashMap *m = (HashMap*)malloc(sizeof(HashMap));
16 m->size = size; m->count = 0;
17 m->buckets = (Entry**)calloc(size, sizeof(Entry*));
18 return m;
19}
20
21void hm_set(HashMap *m, const char *key, int value) {
22 unsigned int idx = hash_str(key, m->size);
23 Entry *e = m->buckets[idx];
24 while (e) { if (strcmp(e->key,key)==0){e->value=value;return;} e=e->next; }
25 Entry *n = (Entry*)malloc(sizeof(Entry));
26 n->key = strdup(key); n->value = value;
27 n->next = m->buckets[idx]; m->buckets[idx] = n; m->count++;
28}
29
30int hm_get(HashMap *m, const char *key, int *value) {
31 Entry *e = m->buckets[hash_str(key,m->size)];
32 while (e) { if(strcmp(e->key,key)==0){*value=e->value;return 1;} e=e->next; }
33 return 0;
34}
35
36void hm_free(HashMap *m) {
37 for (int i = 0; i < m->size; i++) {
38 Entry *e = m->buckets[i];
39 while(e){Entry *n=e->next;free(e->key);free(e);e=n;}
40 }
41 free(m->buckets); free(m);
42}
43

[C 코드] 동적 메모리 — 연결 리스트
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node { int data; struct Node *next; } Node;
5typedef struct { Node *head; int count; } List;
6
7List list_new() { return (List){NULL, 0}; }
8
9void list_push_front(List *l, int val) {
10 Node *n = (Node*)malloc(sizeof(Node));
11 n->data = val; n->next = l->head;
12 l->head = n; l->count++;
13}
14
15void list_push_back(List *l, int val) {
16 Node *n = (Node*)malloc(sizeof(Node));
17 n->data = val; n->next = NULL;
18 if (!l->head) { l->head = n; }
19 else { Node *p = l->head; while(p->next) p = p->next; p->next = n; }
20 l->count++;
21}
22
23int list_pop_front(List *l) {
24 if (!l->head) return -1;
25 Node *old = l->head;
26 int val = old->data;
27 l->head = old->next;
28 free(old); l->count--;
29 return val;
30}
31
32void list_insert_sorted(List *l, int val) {
33 Node *n = (Node*)malloc(sizeof(Node));
34 n->data = val;
35 if (!l->head || val < l->head->data) {
36 n->next = l->head; l->head = n;
37 } else {
38 Node *p = l->head;
39 while (p->next && p->next->data < val) p = p->next;
40 n->next = p->next; p->next = n;
41 }
42 l->count++;
43}
44
45void list_reverse(List *l) {
46 Node *prev = NULL, *curr = l->head, *next;
47 while (curr) { next = curr->next; curr->next = prev; prev = curr; curr = next; }
48 l->head = prev;
49}
50
51void list_print(List *l) {
52 printf("[%d]: ", l->count);
53 for (Node *n = l->head; n; n = n->next) printf("%d→", n->data);
54 printf("NULL\n");
55}
56
57void list_free(List *l) {
58 while(l->head) { Node *n = l->head; l->head = n->next; free(n); }
59 l->count = 0;
60}
61
62int main() {
63 List l = list_new();
64 list_push_back(&l, 10); list_push_back(&l, 30); list_push_back(&l, 50);
65 list_push_front(&l, 5);
66 printf("After push: "); list_print(&l);
67
68 list_insert_sorted(&l, 25);
69 list_insert_sorted(&l, 35);
70 printf("After insert_sorted: "); list_print(&l);
71
72 list_reverse(&l);
73 printf("After reverse: "); list_print(&l);
74
75 printf("pop_front: %d\n", list_pop_front(&l));
76 list_print(&l);
77 list_free(&l);
78 return 0;
79}

[심화 내용] 연결 리스트 vs 배열
시간 복잡도 비교:
|연산|배열|연결리스트|
|접근|O(1)|O(n)|
|앞 삽입|O(n)|O(1)|
|중간 삽입|O(n)|O(1)*|
|검색|O(n)|O(n)|
* 위치를 알고 있을 때

캐시 성능:
배열: 연속 메모리 → 캐시 히트↑
연결리스트: 분산 메모리 → 캐시 미스↑
→ 대부분 배열이 빠름! (Bjarne Stroustrup)

[C 코드] 동적 메모리 — 메모리 누수 탐지기
1#include <stdio.h>
2#include <stdlib.h>
3#include <string.h>
4
5// 메모리 추적 시스템
6typedef struct {
7 void *ptr;
8 int size;
9 const char *file;
10 int line;
11 int freed;
12} MemRecord;
13
14#define MAX_RECORDS 100
15static MemRecord records[MAX_RECORDS];
16static int nrecords = 0;
17static int total_alloc = 0, total_free = 0;
18static int bytes_alloc = 0, bytes_free = 0;
19
20void* tracked_malloc(int size, const char *file, int line) {
21 void *p = malloc(size);
22 if (p && nrecords < MAX_RECORDS) {
23 records[nrecords++] = (MemRecord){p, size, file, line, 0};
24 total_alloc++;
25 bytes_alloc += size;
26 }
27 return p;
28}
29
30void tracked_free(void *ptr, const char *file, int line) {
31 for (int i = 0; i < nrecords; i++) {
32 if (records[i].ptr == ptr && !records[i].freed) {
33 records[i].freed = 1;
34 total_free++;
35 bytes_free += records[i].size;
36 free(ptr);
37 return;
38 }
39 }
40 printf("⚠️ DOUBLE FREE at %s:%d ptr=%p\n", file, line, ptr);
41}
42
43void mem_report() {
44 printf("\n=== Memory Report ===\n");
45 printf(" alloc: %d calls, %d bytes\n", total_alloc, bytes_alloc);
46 printf(" free: %d calls, %d bytes\n", total_free, bytes_free);
47 int leaks = 0;
48 for (int i = 0; i < nrecords; i++) {
49 if (!records[i].freed) {
50 printf(" ⚠️ LEAK: %d bytes at %s:%d\n",
51 records[i].size, records[i].file, records[i].line);
52 leaks++;
53 }
54 }
55 if (leaks == 0) printf(" ✅ No leaks!\n");
56 else printf(" ❌ %d leak(s), %d bytes\n", leaks, bytes_alloc - bytes_free);
57}
58
59#define MALLOC(sz) tracked_malloc(sz, __FILE__, __LINE__)
60#define FREE(p) tracked_free(p, __FILE__, __LINE__)
61

63 int *a = (int*)MALLOC(40); // 10 ints
64 char *s = (char*)MALLOC(100);
65 int *b = (int*)MALLOC(200); // 의도적 누수!
66 double *c = (double*)MALLOC(80);
67
68 strcpy(s, "Hello tracked memory!");
69 printf("%s\n", s);
70
71 FREE(a);
72 FREE(s);
73 FREE(c);
74 // b는 free하지 않음! → 누수
75
76 mem_report();

78 return 0;
79}

[심화 내용] 메모리 안전성
메모리 버그 종류:
• 누수 (leak): free 안 함
• 이중 해제 (double free): 같은 포인터 2번 free
• Use-after-free: 해제 후 접근
• 버퍼 오버플로: 할당 범위 밖 쓰기
• Wild pointer: 초기화 안 된 포인터

도구:
Valgrind: 런타임 메모리 검사 (느리지만 정확)
ASan (Address Sanitizer): 컴파일 타임 계측
→ gcc -fsanitize=address
Rust borrow checker: 컴파일 타임 안전 보장!

스마트 포인터 패턴:
참조 카운트: rc_retain/rc_release
소유권: 한 곳만 free 책임
→ C++: unique_ptr, shared_ptr
Valgrind
메모리 검사 도구. 누수, double free, use-after-free 탐지. 프로그램 10~30배 느려지지만 정확!
ASan
Address Sanitizer. gcc -fsanitize=address. 2배 느림으로 메모리 오류 탐지! Valgrind보다 빠름.
메모리 누수 탐지기를 직접 만들기! malloc/free 래핑+파일/줄 기록. Valgrind의 원리!
💡 동적 메모리 안전 규칙
✅ malloc 반환값 항상 NULL 체크
✅ free 후 포인터를 NULL로 설정
✅ realloc 결과를 임시 변수에 받기 (NULL이면 원본 유지)
✅ 소유권 명확히: 누가 free하는가?
✅ sizeof(*ptr) 패턴: int *p = malloc(n * sizeof(*p))
✅ 배열 크기 오버플로 체크: n * sizeof > SIZE_MAX?
✅ valgrind --leak-check=full 로 검증
dangling pointer
댕글링 포인터. free 후에도 주소를 가진 포인터. 접근하면 UB! free 후 NULL 대입으로 방지.
ownership
소유권. 메모리를 할당한 쪽이 해제 책임. 전달 시 소유권 이전 명시! Rust는 컴파일러가 추적.
free(p); p=NULL; 필수 패턴! 댕글링 포인터 방지. 두 줄로 메모리 버그 절반 제거!
use after free
해제 후 사용. 이미 free된 메모리 접근. 심각한 보안 취약점! ASan으로 탐지. CVE의 단골.
buffer overflow
버퍼 오버플로. 배열 범위 밖 쓰기. 스택 스매싱 → 코드 실행! 가장 유명한 보안 취약점.
동적 메모리 마스터! Vector·LinkedList·HashMap·메모리풀·누수탐지기. 완전 정복!
garbage collection
가비지 컬렉션. 자동 메모리 관리. Mark-Sweep, 참조 카운팅. Java/Python/Go. C는 수동!
reference counting
참조 카운팅. retain/release. 카운트 0이면 해제. 순환 참조 문제! weak reference로 해결.
malloc→사용→free. 단순하지만 어려운 3단계! Rust는 컴파일러가 대신 관리!
동적 메모리 할당
프로그램 실행 중에 힙 영역에서 필요한 만큼 메모리를 할당하는 것.
malloc
지정한 바이트 수만큼 메모리를 할당하는 함수. 초기화하지 않는다.
calloc
지정한 개수와 크기만큼 메모리를 할당하고 0으로 초기화하는 함수.
realloc
이미 할당된 메모리 블록의 크기를 변경하는 함수.
free
동적으로 할당한 메모리를 해제하는 함수.
메모리 누수(memory leak)
할당한 메모리를 해제하지 않아 사용할 수 없는 메모리가 쌓이는 현상.
dangling pointer
해제된 메모리를 여전히 가리키고 있는 포인터.

2
malloc은 할당, calloc은 할당+초기화, realloc은 크기 변경
3
free로 반드시 해제해야 메모리 누수를 막는다
4
free 후 포인터를 NULL로 설정하여 dangling pointer를 방지한다
5
NULL 체크, 범위 초과, double free에 주의한다
