컴파일 전에 일어나는 일

C 코드가 실행 파일이 되려면 여러 단계를 거쳐. 그중 첫 번째가 전처리(preprocessing)야. #으로 시작하는 줄은 컴파일러가 아니라 전처리기가 먼저 처리해!
전처리기는 컴파일 전에 #지시문을 처리한다

전처리기(preprocessor)는 컴파일 전에 소스 코드를 가공하는 프로그램이에요. #으로 시작하는 모든 줄은 전처리기 지시문이에요.

#include는 복사 붙여넣기

#include <stdio.h>는 stdio.h 파일의 내용 전체를 그 자리에 복사해 넣는 거예요. 마법이 아니라 단순한 텍스트 치환이에요.

#include의 두 가지 형태
#include <stdio.h> — 시스템 헤더 파일. 컴파일러가 정한 경로에서 찾는다.
#include "myheader.h" — 사용자 헤더 파일. 현재 디렉토리에서 먼저 찾는다.

#define으로 상수와 매크로 만들기

#define은 이름에 값을 붙여놓으면, 전처리기가 코드 전체에서 그 이름을 값으로 바꿔줘. 찾아서 바꾸기(Find & Replace)와 같아!
define_basic.c
1#include <stdio.h>
2
3#define PI 3.14159
4#define MAX_STUDENTS 100
5#define SQUARE(x) ((x) * (x))
6
7int main(void) {
8 double radius = 5.0;
9 double area = PI * SQUARE(radius);
10
11 printf("반지름: %.1f\n", radius);
12 printf("넓이: %.2f\n", area);
13 printf("최대 학생 수: %d\n", MAX_STUDENTS);
14
15 return 0;
16}
실행 결과
반지름: 5.0 넓이: 78.54 최대 학생 수: 100
#define 매크로가 코드에서 치환되는 과정
3줄#define PI 3.14159 — 코드에서 PI가 나오면 3.14159로 바꿔요. 세미콜론 없어요!
5줄#define SQUARE(x) ((x) * (x)) — 매크로 함수. 괄호를 꼼꼼히 써야 안전해요.
9줄전처리 후: 3.14159 * ((radius) * (radius))로 치환돼요.
매크로 함수에서 괄호를 빼먹으면 위험해! SQUARE(x) x*x로 정의하면, SQUARE(1+2)가 1+2*1+2 = 5가 되어 버려. 올바른 결과는 9인데!
[디버깅] 매크로의 괄호 함정
매크로 함수를 정의할 때는 반드시 인자와 전체 식을 괄호로 감싸야 해요.
나쁜 예: #define SQUARE(x) x*x
좋은 예: #define SQUARE(x) ((x) * (x))

조건부 컴파일

전처리기로 특정 조건에 따라 코드를 포함하거나 제외할 수 있어요. 디버그 모드, 운영체제별 코드 등에 활용돼요.

conditional_compile.c
1#include <stdio.h>
2
3#define DEBUG
4
5int main(void) {
6 int result = 42;
7
8#ifdef DEBUG
9 printf("[DEBUG] result = %d\n", result);
10#endif
11
12 printf("최종 결과: %d\n", result);
13 return 0;
14}
실행 결과
[DEBUG] result = 42 최종 결과: 42
#ifdef DEBUG가 정의되면 디버그 코드 포함, 아니면 제외
조건부 컴파일 지시문
#ifdef NAME — NAME이 정의되어 있으면 아래 코드 포함
#ifndef NAME — NAME이 정의되어 있지 않으면 아래 코드 포함
#else — 조건이 거짓일 때
#endif — 조건부 블록 끝

헤더 가드

같은 헤더 파일이 여러 번 #include되면 중복 정의 오류가 발생해요. 헤더 가드는 이를 방지하는 패턴이에요.

myheader.h
1#ifndef MYHEADER_H
2#define MYHEADER_H
3
4/* 헤더 파일 내용 */
5void greet(void);
6int add(int a, int b);
7
8#endif
헤더 가드 없이 같은 헤더를 두 번 include하면 '함수가 이미 선언되었다'는 오류가 나. 모든 헤더 파일에 헤더 가드를 쓰는 습관을 들이자!

[심화 내용] 전처리기의 힘과 위험
전처리기 = 텍스트 변환기:
#define → 치환, #include → 삽입
#if/#ifdef → 조건부 컴파일
#pragma → 컴파일러 지시

위험한 매크로:
#define SQUARE(x) x*x → SQUARE(1+2)=1+2*1+2=5!
→ 괄호 필수: ((x)*(x))
→ 그래도 부수효과: SQUARE(x++) → x++*x++ (UB!)

안전한 대안:
C11 _Generic: 타입별 분기
static inline: 매크로 함수 대체
enum: 매크로 상수 대체
const: #define 상수 대체
#pragma once
헤더 중복 방지. #ifndef 가드의 간편 버전. 비표준이지만 대부분 컴파일러 지원!
_Generic
C11 제네릭 선택. _Generic(x, int:f_int, double:f_double)(x). 타입별 함수 분기!
container_of
Linux 커널 핵심 매크로. 멤버 포인터→부모 구조체 포인터 역추적! 침투적 자료구조에 필수.

6void Name(T *arr, int n) { \
7 for (int i = 1; i < n; i++) { \
8 T key = arr[i]; \
9 int j = i - 1; \
10 while (j >= 0 && CMP(arr[j], key) > 0) { \
11 arr[j+1] = arr[j]; j--; \
12 } \
13 arr[j+1] = key; \
14 } \
15}
16
17#define CMP_INT(a,b) ((a)-(b))
18#define CMP_DBL(a,b) ((a)>(b)?1:(a)<(b)?-1:0)
19
20DEFINE_SORT(int, sort_int, CMP_INT)
21DEFINE_SORT(double, sort_double, CMP_DBL)
22
23// 제네릭 이진 탐색
24#define DEFINE_BSEARCH(T, Name, CMP) \
25int Name(T *arr, int n, T key) { \
26 int lo = 0, hi = n - 1; \
27 while (lo <= hi) { \
28 int mid = (lo + hi) / 2; \
29 int c = CMP(arr[mid], key); \
30 if (c == 0) return mid; \
31 else if (c < 0) lo = mid + 1; \
32 else hi = mid - 1; \
33 } \
34 return -1; \
35}
36
37DEFINE_BSEARCH(int, bsearch_int, CMP_INT)
38DEFINE_BSEARCH(double, bsearch_double, CMP_DBL)
39
40// 제네릭 출력
41#define PRINT_ARR(arr, n, fmt) do { \
42 for (int _i = 0; _i < (n); _i++) printf(fmt " ", (arr)[_i]); \
43 printf("\n"); \
44} while(0)
45
46int main() {
47 int ints[] = {42, 17, 93, 5, 28, 61, 34};
48 int ni = 7;
49 printf("Before: "); PRINT_ARR(ints, ni, "%d");
50 sort_int(ints, ni);
51 printf("After: "); PRINT_ARR(ints, ni, "%d");

[심화 내용] C의 제네릭 프로그래밍
매크로 제네릭 (C89~):
DEFINE_SORT(int, sort_int, CMP)
→ 타입별 함수 코드 생성
→ 인라인 → 빠르지만 코드 크기 증가

void* 제네릭 (C89~):
qsort(arr, n, sizeof(int), cmp_func)
→ 하나의 함수로 모든 타입
→ 간접 호출 → 약간 느림

_Generic (C11):
_Generic(x, int: f_int, double: f_double)(x)
→ 컴파일 타임 타입 분기!
→ 오버로딩 시뮬레이션

C++ template:
template void sort(T* arr, int n)
→ 타입 안전 + 코드 생성
→ 매크로의 상위 호환
monomorphization
단형화. 제네릭 함수를 타입별로 복제. C매크로, C++template, Rust제네릭. 빠르지만 코드 증가!
C11 _Generic
C11 타입 분기. _Generic(x, int:1, double:2, default:0). printf 대체, 타입별 함수 선택!
DEFINE_SORT 매크로 = C의 template! int용, double용 정렬을 한 번에 생성!
제네릭 이진 탐색까지! 정렬+탐색 매크로를 만들면 어떤 타입이든 즉시 사용!
💡 전처리기 안전 가이드
✅ 매크로 인자에 괄호: #define SQ(x) ((x)*(x))
✅ do { ... } while(0) 패턴으로 세미콜론 안전
✅ 타입 안전이 필요하면 static inline 함수 사용
✅ #include guard 또는 #pragma once 필수
✅ 매크로 이름은 전부 대문자 (관례)
✅ ##, # 연산자는 꼭 필요할 때만
✅ 가변 인자 매크로: ##__VA_ARGS__ (GNU)
inline function
인라인 함수. static inline int max(int a,int b){...} 매크로 대안. 타입 안전+디버깅 가능!
include guard
#ifndef FOO_H #define FOO_H ... #endif. 헤더 중복 포함 방지. 모든 헤더에 필수!
매크로 vs inline! 간단한 상수→enum, 간단한 함수→static inline, 코드 생성→매크로!
전처리기 마스터! #define+#ifdef+#pragma+_Generic. C의 메타프로그래밍 도구 총출동!
stringize
문자열화. #x → "x". ASSERT(a==b) → "a==b" 출력. 디버깅 매크로에 필수!
variadic macro
가변 인자 매크로. #define LOG(fmt,...) printf(fmt,__VA_ARGS__). C99. 로그 시스템에 핵심!
predefined macros
미리 정의된 매크로. __FILE__ __LINE__ __func__ __DATE__. 디버깅+로깅에 자동 활용!
전처리기 마스터! 디버그·테스트·제네릭·로그·색상·타이머. 50줄 안에 프레임워크!
pragma
#pragma. 컴파일러 지시. pack(패딩), once(헤더가드), warning(경고 제어). 비표준이지만 필수!
token pasting
##연산자. a##b→ab. 식별자 합성. DEFINE_SORT(int, sort_int). sort_##int→sort_int!
__func__
C99. 현재 함수 이름. 로그에 함수명 자동 포함! printf("%s: error", __func__);
__FILE__+__LINE__+__func__ = 자동 디버그 정보! 어디서 에러났는지 즉시 알 수 있음!
include path
헤더 검색 경로. ""는 현재 디렉토리 먼저, <>는 시스템 경로. gcc -I로 경로 추가!
predefined macros
미리 정의된 매크로. __FILE__, __LINE__, __DATE__, __TIME__, __STDC_VERSION__. 디버깅에 필수!
전처리기 = 컴파일 전 텍스트 변환기! #include는 파일 붙여넣기, #define은 찾아바꾸기!
stringification
문자열화. #x → "x". ASSERT(a>b)에서 #(a>b) → "a>b". 디버그 메시지에 변수명 포함!
전처리기(preprocessor)
컴파일 전에 #지시문을 처리하여 소스 코드를 가공하는 프로그램.
#include
다른 파일의 내용을 현재 소스 코드에 삽입하는 전처리 지시문.
#define
매크로를 정의하는 전처리 지시문. 텍스트 치환에 사용된다.
매크로 함수
인자를 받는 매크로. 함수처럼 보이지만 실제로는 텍스트 치환이다.
헤더 가드
#ifndef/#define/#endif 패턴으로 헤더 파일의 중복 포함을 방지하는 기법.
조건부 컴파일
#ifdef, #ifndef 등으로 특정 조건에 따라 코드를 포함하거나 제외하는 것.
