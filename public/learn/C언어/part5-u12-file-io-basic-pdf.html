파일 입출력 기초

fopen, fclose, fprintf, fscanf, fgets, fputs로 텍스트 파일을 읽고 쓰는 방법을 배워요.

📚 이번에 배울 것

  • fopen과 fclose로 파일을 열고 닫을 수 있어요.
  • fprintf, fscanf로 파일에 쓰고 읽을 수 있어요.
  • fgets, fputs로 한 줄씩 읽고 쓸 수 있어요.
  • 파일 열기 모드(r, w, a)를 구분할 수 있어요.

✅ 시작 전 체크

  • printf, scanf로 화면 입출력을 할 수 있다
  • 포인터의 기본 개념을 안다

데이터를 영구적으로 저장하려면?

지금까지 만든 프로그램은 종료하면 데이터가 사라졌어. 학생 점수를 입력했는데 프로그램을 끄면 다 날아가잖아. 파일에 저장하면 다음에 다시 불러올 수 있어!
프로그램은 스트림(stream)을 통해 파일과 데이터를 주고받는다

C에서 파일 입출력은 파일 포인터(FILE *)를 통해 이루어져요. 파일을 열면 스트림이 만들어지고, 이 스트림을 통해 데이터를 읽거나 써요.

fopen과 fclose

fopen으로 파일을 열고, 다 쓰면 반드시 fclose로 닫아야 해. 동적 메모리의 malloc/free 관계와 비슷해!
파일 열기 모드에 따라 동작이 달라진다
fopen 모드 정리
"r" — 읽기. 파일이 없으면 실패(NULL 반환).
"w" — 쓰기. 파일이 없으면 생성, 있으면 내용 삭제 후 새로 쓰기.
"a" — 추가. 파일이 없으면 생성, 있으면 끝에 이어쓰기.
"r+" — 읽기+쓰기. 파일이 있어야 함.
"w+" — 읽기+쓰기. 파일 내용 삭제 후 시작.

파일에 쓰기: fprintf, fputs

4줄fopen("scores.txt", "w") — scores.txt 파일을 쓰기 모드로 열어요.
5줄fopen 실패 시 NULL 반환. 반드시 확인해야 해요.
10줄fprintf(fp, ...) — printf와 같지만, 화면 대신 파일에 출력해요.
14줄fputs — 문자열을 파일에 써요. 서식 지정 없이 그대로 출력.
16줄fclose(fp) — 파일을 닫아요. 닫지 않으면 데이터가 유실될 수 있어요.

텍스트 파일은 문자열의 연속이고 줄 바꿈으로 구분된다

[C 코드] 파일 I/O — 텍스트 에디터 코어
1#include <stdio.h>
2#include <string.h>
3#include <stdlib.h>
4
5#define MAX_LINES 500
6#define LINE_LEN 256
7
8typedef struct {
9 char lines[MAX_LINES][LINE_LEN];
10 int count;
11 int modified;
12 char filename[100];
13} TextBuffer;
14
15void buf_init(TextBuffer *b) { b->count = 0; b->modified = 0; b->filename[0] = 0; }
16
17int buf_load(TextBuffer *b, const char *content) {
18 b->count = 0;
19 const char *p = content;
20 while (*p && b->count < MAX_LINES) {
21 int i = 0;
22 while (*p && *p != '\n' && i < LINE_LEN-1) b->lines[b->count][i++] = *p++;
23 b->lines[b->count][i] = 0;
24 if (*p == '\n') p++;
25 b->count++;
26 }
27 b->modified = 0;
28 return b->count;
29}
30
31void buf_insert_line(TextBuffer *b, int pos, const char *text) {
32 if (b->count >= MAX_LINES || pos > b->count) return;
33 for (int i = b->count; i > pos; i--)
34 strcpy(b->lines[i], b->lines[i-1]);
35 strncpy(b->lines[pos], text, LINE_LEN-1);
36 b->count++; b->modified = 1;
37}
38
39void buf_delete_line(TextBuffer *b, int pos) {
40 if (pos >= b->count) return;
41 for (int i = pos; i < b->count-1; i++)
42 strcpy(b->lines[i], b->lines[i+1]);
43 b->count--; b->modified = 1;
44}
45
46void buf_replace(TextBuffer *b, const char *old, const char *new_s) {
47 for (int i = 0; i < b->count; i++) {
48 char *pos = strstr(b->lines[i], old);
49 if (pos) {
50 char tmp[LINE_LEN];
51 int idx = pos - b->lines[i];
52 strncpy(tmp, b->lines[i], idx);
53 tmp[idx] = 0;
54 strcat(tmp, new_s);
55 strcat(tmp, pos + strlen(old));
56 strcpy(b->lines[i], tmp);
57 b->modified = 1;
58 }
59 }
60}
61
62void buf_print(TextBuffer *b, int from, int to) {
63 if (to > b->count) to = b->count;
64 for (int i = from; i < to; i++)
65 printf("%3d| %s\n", i+1, b->lines[i]);
66}

22// 간단한 diff (줄 단위 비교)
23void simple_diff(TextFile *a, TextFile *b) {
24 int ia = 0, ib = 0;
25 while (ia < a->count || ib < b->count) {
26 if (ia < a->count && ib < b->count) {
27 if (strcmp(a->lines[ia], b->lines[ib]) == 0) {
28 printf(" %s\n", a->lines[ia]);
29 ia++; ib++;
30 } else {
31 // 삭제된 줄
32 printf("- %s\n", a->lines[ia]);
33 ia++;
34 // 추가된 줄
35 if (ib < b->count) {
36 printf("+ %s\n", b->lines[ib]);
37 ib++;
38 }
39 }
40 } else if (ia < a->count) {
41 printf("- %s\n", a->lines[ia++]);
42 } else {
43 printf("+ %s\n", b->lines[ib++]);
44 }
45 }

48// 통계
49void diff_stats(TextFile *a, TextFile *b) {
50 int same = 0, added = 0, removed = 0;
51 int ia = 0, ib = 0;
52 while (ia < a->count && ib < b->count) {
53 if (strcmp(a->lines[ia], b->lines[ib]) == 0) { same++; ia++; ib++; }
54 else { removed++; added++; ia++; ib++; }
55 }
56 removed += a->count - ia;

[심화 내용] Diff 알고리즘
LCS (Longest Common Subsequence):
두 시퀀스의 최장 공통 부분수열
DP로 O(mn) 시간/공간
diff = 전체 - LCS (추가/삭제된 부분)

Myers Diff Algorithm:
Git이 사용하는 알고리즘!
최소 편집 거리를 O(nd) 시간에 계산
d = 편집 거리 (보통 작음) → 실전에서 빠름

Patience Diff:
Git의 대안 diff
유일한 공통 줄을 앵커로 → 더 읽기 좋은 결과
코드 리팩토링 시 특히 우수
LCS
최장 공통 부분수열. DP로 O(mn). diff, DNA 서열 비교, 표절 검사에 핵심! Git diff의 기반.
Myers diff
Myers 차분 알고리즘. 최소 편집 거리. O(nd). Git의 기본 diff. d=차이 수가 작으면 매우 빠름!
diff 도구 = 두 파일 비교! + 추가, - 삭제. Git diff의 원리를 직접 구현!
LCS = diff의 핵심! 공통 줄을 찾고, 나머지가 추가/삭제. DP로 O(mn)!
💡 파일 I/O 안전 규칙
✅ fopen 반환값 NULL 체크 필수
✅ fclose 항상 호출 (goto cleanup 패턴)
✅ fgets > gets (버퍼 오버플로 방지)
✅ snprintf > sprintf (크기 제한)
✅ 바이너리 모드 "rb"/"wb" (Windows 줄바꿈 차이)
✅ fflush로 버퍼 강제 출력 (로그 등)
✅ 임시 파일로 쓰고 rename (원자적 교체)
✅ 에러 시 errno/perror로 원인 확인
errno
에러 번호. 전역 변수. fopen 실패 시 errno=ENOENT(파일 없음). perror("msg")로 출력!
atomic rename
원자적 이름 변경. rename()은 원자적! 임시 파일에 쓰고 rename→중간에 죽어도 데이터 안전.
text vs binary
텍스트 vs 바이너리 모드. Windows: \n↔\r\n 자동 변환(텍스트). 바이너리는 그대로! "rb" 사용.
fgets(buf,SIZE,fp)! gets()는 절대 쓰지 마! 버퍼 크기 제한 없어서 해킹당함!
파일 I/O 마스터! 텍스트 처리+로그+에디터+diff. Unix 도구의 원리를 C로 직접 구현!
stdin/stdout/stderr
표준 스트림. stdin=입력, stdout=출력, stderr=에러. fprintf(stderr,"error!"). 리다이렉션 분리!
pipe
파이프. ls | grep | sort. 한 프로그램의 stdout→다른 프로그램의 stdin. Unix 철학의 핵심!
setvbuf
버퍼 모드 설정. _IONBF(버퍼 없음), _IOLBF(줄 단위), _IOFBF(블록 단위). 성능 vs 즉시성!
파일 I/O 마스터! wc·nl·grep·로그·에디터·diff. Unix 도구를 C로 직접 만들기!
file descriptor
파일 디스크립터. 정수 ID. 0=stdin, 1=stdout, 2=stderr. open()이 반환. 모든 I/O의 기본!
inode
아이노드. Unix 파일 메타데이터. 크기, 권한, 타임스탬프, 데이터 블록 위치. ls -i로 확인!
unbuffered IO
비버퍼 I/O. read()/write() 시스템 콜 직접. stdio보다 빠를 수 있지만 호출 오버헤드!
fd 0,1,2 = stdin/stdout/stderr! 파일, 소켓, 파이프 전부 파일 디스크립터. Unix 철학!
stdin/stdout/stderr
표준 입출력. stdin=키보드, stdout=화면, stderr=에러 화면. 리다이렉션으로 파일 연결!
text vs binary mode
텍스트 모드: \n↔\r\n 자동 변환(Windows). 바이너리 모드: 변환 없음. 이미지는 반드시 rb/wb!
파일 = 영구 저장소! printf는 화면, fprintf는 파일. FILE*만 바꾸면 출력 대상 변경!
EOF
End Of File. fgetc() 반환. int(-1). char로 받으면 255와 구분 불가! 반드시 int로 받기.
fgets가 scanf보다 안전! 버퍼 크기 지정 가능. scanf("%s")는 버퍼 오버플로 위험!
파일 포인터(FILE *)
열린 파일을 나타내는 구조체 포인터. 파일 입출력 함수의 핵심.
fopen
파일을 지정한 모드로 열어 파일 포인터를 반환하는 함수.
fclose
열린 파일을 닫고 자원을 해제하는 함수.
fprintf
서식 지정하여 파일에 출력하는 함수. printf의 파일 버전.
fgets
파일에서 한 줄을 읽는 함수. 줄 바꿈 문자를 포함한다.
스트림(stream)
프로그램과 파일 사이에서 데이터가 흐르는 통로.
