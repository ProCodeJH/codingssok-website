왜 정렬해야 할까?

사전에서 단어를 찾을 때, 만약 단어가 아무 순서 없이 나열되어 있다면? 찾으려면 처음부터 끝까지 다 봐야 해. 하지만 가나다순으로 정렬되어 있으니 빠르게 찾을 수 있지!

정렬(sorting)이란 데이터를 일정한 기준에 따라 순서대로 나열하는 것이에요. 컴퓨터가 다루는 거의 모든 작업에서 정렬은 기본 중의 기본이에요.

정렬이 필요한 이유: 탐색을 빠르게!
실생활에서 정렬이 쓰이는 곳
1. 전화번호부 -- 이름 가나다순
2. 성적표 -- 점수 높은 순서
3. 검색 엔진 -- 관련도 높은 순서
4. 음악 앱 -- 제목, 가수, 최신순 정렬
5. 게임 랭킹 -- 점수 내림차순

오름차순과 내림차순

오름차순 vs 내림차순

오름차순(ascending): 작은 값부터 큰 값 순서로 나열해요. 1, 3, 5, 7
내림차순(descending): 큰 값부터 작은 값 순서로 나열해요. 7, 5, 3, 1

C 코드에서는 비교 조건의 부등호 방향만 바꾸면 오름차순과 내림차순을 전환할 수 있어요.

KOI 문제에서 '오름차순으로 정렬하시오'라고 나오면 작은 수가 앞에! '내림차순'이면 큰 수가 앞에! 문제를 잘 읽어야 해.

[심화 내용] 정렬의 하한선
비교 기반 정렬의 하한:
n개 원소의 순열 = n!가지
비교 1회 → 2가지 분기
k번 비교 → 최대 2^k 구분
2^k ≥ n! → k ≥ log₂(n!)
스털링 근사: log₂(n!) ≈ n·log₂n
→ Ω(n log n)이 비교 정렬의 하한!

하한을 깨는 방법:
비교를 안 하면 됨!
Counting Sort: O(n+k) — 정수, 범위 제한
Radix Sort: O(d·n) — 자릿수 기반
Bucket Sort: O(n) 평균 — 균등 분포
⚠️ 정렬이 왜 중요할까?
정렬된 데이터의 힘:
• 이진 탐색: O(n) → O(log n)
• 중복 제거: 인접 비교만으로 가능
• 합집합/교집합: 두 정렬 배열 병합
• 데이터베이스: INDEX = 정렬된 B-트리
• 스케줄링: 마감일 순 정렬 → 최적 순서
⚡ 연습문제: 정렬 기초~심화

5// Counting Sort: O(n+k), 비비교 정렬!
6void counting_sort(int *a, int n, int max_val) {
7 int *count = (int*)calloc(max_val+1, sizeof(int));
8 int *output = (int*)malloc(n * sizeof(int));
9 for (int i = 0; i < n; i++) count[a[i]]++;
10 for (int i = 1; i <= max_val; i++) count[i] += count[i-1];
11 for (int i = n-1; i >= 0; i--) { output[count[a[i]]-1] = a[i]; count[a[i]]--; }
12 memcpy(a, output, n*sizeof(int));
13 free(count); free(output);
14}
15
16// Merge Sort: O(n log n), 안정!
17void merge(int *a, int l, int m, int r) {
18 int n1=m-l+1, n2=r-m;
19 int *L=(int*)malloc(n1*sizeof(int)), *R=(int*)malloc(n2*sizeof(int));
20 for(int i=0;i<n1;i++) L[i]=a[l+i];
21 for(int i=0;i<n2;i++) R[i]=a[m+1+i];
22 int i=0,j=0,k=l;
23 while(i<n1&&j<n2) a[k++]=(L[i]<=R[j])?L[i++]:R[j++];
24 while(i<n1) a[k++]=L[i++];
25 while(j<n2) a[k++]=R[j++];
26 free(L); free(R);
27}
28void merge_sort(int *a, int l, int r) {
29 if(l<r){int m=(l+r)/2; merge_sort(a,l,m); merge_sort(a,m+1,r); merge(a,l,m,r);}
30}

[심화 내용] 비비교 정렬 총정리
비비교 정렬 = O(n) 가능!

Counting Sort: 값의 빈도 세기. O(n+k)
→ 범위 작은 정수에 최적

Radix Sort: 자릿수별 Counting. O(d·n)
→ 큰 정수도 OK (d=자릿수)

Bucket Sort: 구간별 분류. O(n) 평균
→ 균등 분포에 최적

비교 정렬과의 차이:
비교: 범용, 하한 Ω(n log n)
비비교: 제한적, 하한 깨뜨림!
radix sort
기수 정렬. 자릿수별 정렬. LSD(낮은 자리부터) vs MSD(높은 자리부터). O(d·n) 선형!
bucket sort
버킷 정렬. 값을 구간별 버킷에 분배→각 버킷 정렬→합치기. 균등 분포에서 O(n)!
Radix Sort: 1의 자리→10의 자리→100의 자리 순서로 정렬. 비비교라 O(n)!

9 Node *n = (Node*)malloc(sizeof(Node));
10 n->val = val;
11 // 정렬 삽입
12 if (!*bucket || val < (*bucket)->val) { n->next = *bucket; *bucket = n; return; }
13 Node *p = *bucket;
14 while (p->next && p->next->val < val) p = p->next;
15 n->next = p->next; p->next = n;
16}

18void bucket_sort(double *a, int n) {
19 Node *buckets[NBUCKETS] = {NULL};
20 for (int i = 0; i < n; i++) {
21 int idx = (int)(a[i] * NBUCKETS);
22 if (idx >= NBUCKETS) idx = NBUCKETS-1;
23 bucket_insert(&buckets[idx], a[i]);
24 }
25 int k = 0;
26 for (int i = 0; i < NBUCKETS; i++) {
27 Node *p = buckets[i];
28 while (p) { a[k++] = p->val; Node *t = p; p = p->next; free(t); }
29 }
30}

4void merge(int *a, int l, int m, int r) {
5 int n1=m-l+1, n2=r-m;
6 int *L=(int*)malloc(n1*sizeof(int)), *R=(int*)malloc(n2*sizeof(int));
7 for(int i=0;i<n1;i++) L[i]=a[l+i];
8 for(int i=0;i<n2;i++) R[i]=a[m+1+i];
9 int i=0,j=0,k=l;
10 while(i<n1&&j<n2) a[k++]=(L[i]<=R[j])?L[i++]:R[j++];
11 while(i<n1) a[k++]=L[i++];
12 while(j<n2) a[k++]=R[j++];
13 free(L); free(R);
14}

16void merge_sort(int *a, int l, int r) {
17 if(l<r) {
18 int m=l+(r-l)/2;
19 merge_sort(a, l, m);
20 merge_sort(a, m+1, r);
21 merge(a, l, m, r);
22 }
23}

25void quick_sort(int *a, int lo, int hi) {
26 if(lo>=hi) return;
27 int pivot=a[hi], i=lo-1;
28 for(int j=lo;j<hi;j++)
29 if(a[j]<=pivot){i++;int t=a[i];a[i]=a[j];a[j]=t;}
30 int t=a[i+1];a[i+1]=a[hi];a[hi]=t;
31 int pi=i+1;
32 quick_sort(a, lo, pi-1);
33 quick_sort(a, pi+1, hi);
34}

counting sort
계수 정렬. O(n+k). 비교 안 함! 정수 범위 제한 시 최적. 기수 정렬의 기반.
radix sort
기수 정렬. O(d·n). 자릿수별 정렬. LSD(최하위)→MSD(최상위). 문자열 정렬에도!
비교 안 하는 정렬! 범위가 작으면 Counting, 자릿수가 작으면 Radix. O(n) 가능!
external sort
외부 정렬. 메모리에 안 들어오는 데이터. 청크 정렬→병합. 데이터베이스, 대용량 로그 처리!
정렬 알고리즘 총정리: 버블<선택<삽입<셸<퀵≈병합<힙. 상황에 맞는 걸 골라!
in-place merge
제자리 병합. 추가 메모리 없이 병합! 복잡하지만 O(1) 공간. 실전에서는 거의 안 쓰임.
정렬 선택 가이드: n<50→삽입, 범용→퀵소트, 안정 필요→병합, 정수→계수/기수!

counting sort
계수 정렬. 값 범위 배열에 빈도 저장. O(n+k). 비교 없음! 정수, 범위 제한에 최적.
radix sort
기수 정렬. 자릿수별 안정 정렬 반복. O(d·n). d=자릿수. 정수, 문자열에 초고속!
비교 안 하고 정렬? 가능! Counting Sort는 세기만. Radix는 자릿수별로. O(n)!
정렬(sorting)
데이터를 일정한 기준에 따라 순서대로 나열하는 것.
오름차순(ascending)
작은 값에서 큰 값 순서로 나열. 예: 1, 2, 3, 4, 5
내림차순(descending)
큰 값에서 작은 값 순서로 나열. 예: 5, 4, 3, 2, 1
비교(compare)
두 값의 크기를 비교하는 연산. 정렬의 기본 동작.
교환(swap)
두 원소의 위치를 서로 바꾸는 연산.
