버블 정렬

이웃한 두 원소를 비교하고 교환하는 버블 정렬을 배워요. 이중 반복문 구조와 O(n^2) 시간 복잡도를 이해해요.

Part 1 진행률: 4%

거품처럼 떠오르는 정렬

물속에서 거품이 올라오는 걸 본 적 있지? 버블 정렬도 비슷해. 큰 값이 거품처럼 배열의 뒤쪽으로 점점 떠올라!

버블 정렬(Bubble Sort)은 인접한 두 원소를 비교해서, 순서가 맞지 않으면 교환하는 알고리즘이에요. 한 번 배열을 쭉 훑으면(pass) 가장 큰 값이 맨 뒤로 이동해요. 이걸 반복하면 전체가 정렬돼요.

단계별 시각화

배열 [5,3,8,1,2]의 버블 정렬 과정
매 Pass가 끝날 때마다 가장 큰 값이 제자리를 찾아! 그래서 다음 Pass에서는 비교 범위가 하나씩 줄어들어.

이중 반복문 구조

이중 for문으로 구현하는 버블 정렬
bubble_sort.c
1#include <stdio.h>
2
3void bubbleSort(int arr[], int n) {
4 for (int i = 0; i < n - 1; i++) {
5 for (int j = 0; j < n - 1 - i; j++) {
6 if (arr[j] > arr[j + 1]) {
7 int temp = arr[j];
8 arr[j] = arr[j + 1];
9 arr[j + 1] = temp;
10 }
11 }
12 }
13}
14
15int main(void) {
16 int arr[] = {5, 3, 8, 1, 2};
17 int n = 5;
18
19 printf("정렬 전: ");
20 for (int i = 0; i < n; i++) printf("%d ", arr[i]);
21 printf("\n");
22
23 bubbleSort(arr, n);
24
25 printf("정렬 후: ");
26 for (int i = 0; i < n; i++) printf("%d ", arr[i]);
27 printf("\n");
28 return 0;
29}
실행 결과
정렬 전: 5 3 8 1 2 정렬 후: 1 2 3 5 8
4줄바깥 루프: i는 0부터 n-2까지. 총 n-1번의 pass를 수행해요.
5줄안쪽 루프: j는 0부터 n-2-i까지. 이미 정렬된 뒤쪽은 건너뛰어요.
6줄arr[j] > arr[j+1]이면 두 원소를 교환해요. 이 비교 조건을 <로 바꾸면 내림차순이 돼요.
7줄temp 변수를 이용한 교환(swap). 세 줄의 교환 패턴은 꼭 외워두세요!

시간 복잡도 O(n^2)

데이터가 많아지면 O(n^2)는 급격히 느려진다
버블 정렬의 시간 복잡도
n개 원소를 정렬할 때:
비교 횟수 = (n-1) + (n-2) + ... + 1 = n(n-1)/2
n=10이면 45번, n=100이면 4,950번, n=1000이면 499,500번!

이것을 O(n^2)이라고 표기해요. 데이터가 2배 되면 시간은 4배!

조기 종료 최적화

만약 한 번의 pass에서 교환이 한 번도 일어나지 않았다면? 이미 정렬이 끝난 거야! 더 반복할 필요 없어.
bubble_sort_opt.c
1#include <stdio.h>
2
3void bubbleSortOpt(int arr[], int n) {
4 for (int i = 0; i < n - 1; i++) {
5 int swapped = 0;
6 for (int j = 0; j < n - 1 - i; j++) {
7 if (arr[j] > arr[j + 1]) {
8 int temp = arr[j];
9 arr[j] = arr[j + 1];
10 arr[j + 1] = temp;
11 swapped = 1;
12 }
13 }
14 if (!swapped) break;
15 }
16}
17
18int main(void) {
19 int arr[] = {1, 2, 3, 5, 4};
20 int n = 5;
21
22 bubbleSortOpt(arr, n);
23
24 for (int i = 0; i < n; i++) printf("%d ", arr[i]);
25 printf("\n");
26 return 0;
27}
5줄swapped 변수를 0으로 초기화. 교환이 일어나면 1로 설정해요.
14줄한 번의 pass에서 교환이 없으면(!swapped) 이미 정렬 완료이므로 break해요.
[디버깅] 버블 정렬에서 자주 하는 실수
1. 배열 범위 초과 -- j < n-1-i에서 i를 빼지 않으면 이미 정렬된 부분까지 비교
2. 교환 코드 실수 -- temp 없이 직접 대입하면 값이 사라짐
3. 루프 범위 -- i < n이 아니라 i < n-1이어야 해요
[KOI] 버블 정렬 응용
KOI에서는 '교환 횟수 세기' 문제가 나와요. 버블 정렬의 교환 횟수는 배열의 역쌍(inversion) 개수와 같아요. 이 개념을 알면 관련 문제를 풀 수 있어요.

1#include <stdio.h>
2
3// v1: 기본 버블
4void bubble_v1(int *a, int n, int *comps, int *swaps) {
5 *comps = *swaps = 0;
6 for (int i = 0; i < n-1; i++)
7 for (int j = 0; j < n-1-i; j++) {
8 (*comps)++;
9 if (a[j] > a[j+1]) { int t=a[j]; a[j]=a[j+1]; a[j+1]=t; (*swaps)++; }
10 }
11}
12
13// v2: 조기 종료 (교환 없으면 정렬 완료)
14void bubble_v2(int *a, int n, int *comps, int *swaps) {
15 *comps = *swaps = 0;
16 for (int i = 0; i < n-1; i++) {
17 int swapped = 0;
18 for (int j = 0; j < n-1-i; j++) {
19 (*comps)++;
20 if (a[j] > a[j+1]) { int t=a[j]; a[j]=a[j+1]; a[j+1]=t; (*swaps)++; swapped=1; }
21 }
22 if (!swapped) break; // 이미 정렬됨!
23 }
24}
25
26// v3: 마지막 교환 위치 기억
27void bubble_v3(int *a, int n, int *comps, int *swaps) {
28 *comps = *swaps = 0;
29 int end = n - 1;
30 while (end > 0) {
31 int last_swap = 0;
32 for (int j = 0; j < end; j++) {
33 (*comps)++;
34 if (a[j] > a[j+1]) {
35 int t=a[j]; a[j]=a[j+1]; a[j+1]=t;
36 (*swaps)++;
37 last_swap = j;
38 }
39 }
40 end = last_swap; // 여기 이후는 이미 정렬!
41 }
42}

[C 코드] 버블 정렬 — 범용 (void* + 비교 함수)
1#include <stdio.h>
2#include <string.h>
3
4// void*로 범용 버블 정렬!
5void generic_bubble(void *base, int n, int size, int(*cmp)(const void*,const void*)) {
6 char *arr = (char*)base;
7 char *temp = (char*)__builtin_alloca(size); // 스택에 임시 공간
8 for (int i = 0; i < n-1; i++) {
9 int swapped = 0;
10 for (int j = 0; j < n-1-i; j++) {
11 if (cmp(arr+j*size, arr+(j+1)*size) > 0) {
12 memcpy(temp, arr+j*size, size);
13 memcpy(arr+j*size, arr+(j+1)*size, size);
14 memcpy(arr+(j+1)*size, temp, size);
15 swapped = 1;
16 }
17 }
18 if (!swapped) break;
19 }
20}
21
22int cmp_int(const void *a, const void *b) { return *(int*)a - *(int*)b; }
23int cmp_str(const void *a, const void *b) { return strcmp(*(char**)a, *(char**)b); }
24
25int main() {
26 int nums[] = {42, 17, 93, 8, 55};
27 generic_bubble(nums, 5, sizeof(int), cmp_int);
28 printf("Ints: "); for(int i=0;i<5;i++) printf("%d ",nums[i]); printf("\n");
29
30 char *words[] = {"cherry","apple","banana","date"};
31 generic_bubble(words, 4, sizeof(char*), cmp_str);
32 printf("Strs: "); for(int i=0;i<4;i++) printf("%s ",words[i]); printf("\n");
33 return 0;
34}

쉽게 이해하기: void*와 함수 포인터
void* = "타입 모르는 주소"

택배 상자를 생각해봐:
상자 안에 뭐가 들었는지 몰라도
주소만 알면 배달할 수 있잖아?

void* = 상자 주소
size = 상자 크기
cmp = "상자 2개를 비교하는 방법"

왜 char*로 캐스팅?
char*는 1바이트 단위 포인터
arr + j*size → j번째 원소 주소로 이동!
int*면 j*4씩 이동, double*면 j*8씩
→ char*+size로 아무 타입이나 가능!
generic function
범용 함수. void*+size+cmp로 아무 타입 정렬! qsort가 이 방식. C의 제네릭 프로그래밍.
void*+memcpy = C의 제네릭! 타입을 몰라도 바이트 단위로 복사하면 교환 가능!
[C 코드] 버블 정렬 — 안정성 실험
1#include <stdio.h>
2#include <string.h>
3
4typedef struct { char name[10]; int score; int original; } Student;
5
6void stable_bubble(Student *a, int n) {
7 for (int i = 0; i < n-1; i++)
8 for (int j = 0; j < n-1-i; j++)
9 if (a[j].score > a[j+1].score) {
10 Student t = a[j]; a[j] = a[j+1]; a[j+1] = t;
11 }
12}
13
14void unstable_selection(Student *a, int n) {
15 for (int i = 0; i < n-1; i++) {
16 int min = i;
17 for (int j = i+1; j < n; j++) if (a[j].score < a[min].score) min = j;
18 Student t = a[i]; a[i] = a[min]; a[min] = t;
19 }
20}
21
22int main() {
23 Student data[] = {
24 {"Kim",80,1}, {"Lee",90,2}, {"Park",80,3}, {"Choi",70,4}, {"Jung",80,5}
25 };
26 Student copy[5];
27
28 memcpy(copy, data, sizeof(data));
29 stable_bubble(copy, 5);
30 printf("Bubble (stable):\n");
31 for (int i = 0; i < 5; i++)
32 printf(" %s score=%d orig=#%d\n", copy[i].name, copy[i].score, copy[i].original);
33
34 memcpy(copy, data, sizeof(data));
35 unstable_selection(copy, 5);
36 printf("\nSelection (unstable):\n");
37 for (int i = 0; i < 5; i++)
38 printf(" %s score=%d orig=#%d\n", copy[i].name, copy[i].score, copy[i].original);
39 return 0;
40}

쉽게 이해하기: 안정 정렬이 왜 중요한가
점수 80점인 3명의 순서:
원래: Kim(#1) → Park(#3) → Jung(#5)

버블(안정): Kim → Park → Jung ✅ 순서 유지!
선택(불안정): Jung → Park → Kim ❌ 순서 바뀜!

왜 중요한가?
1차 정렬: 이름순
2차 정렬: 점수순 (안정 정렬 사용)
→ 같은 점수 내에서 이름순 유지!

불안정 정렬이면 1차 정렬이 깨짐!
stability proof
버블이 안정인 이유: a[j]>a[j+1]일 때만 교환. 같으면 교환 안 함→원래 순서 보존!
안정 정렬 = 같은 값의 원래 순서 유지! 다중 키 정렬의 핵심. 엑셀도 안정 정렬 사용!
[C 코드] 버블 정렬 — 연결 리스트 정렬
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node { int data; struct Node *next; } Node;
5
6Node* push(Node *h, int d) { Node *n=malloc(sizeof(Node)); n->data=d; n->next=h; return n; }
7
8void bubble_list(Node *head) {
9 if (!head) return;
10 int swapped;
11 do {
12 swapped = 0;
13 Node *p = head;
14 while (p->next) {
15 if (p->data > p->next->data) {
16 int t = p->data; p->data = p->next->data; p->next->data = t;
17 swapped = 1;
18 }
19 p = p->next;
20 }
21 } while (swapped);
22}
23
24void print_list(Node *h) { for(;h;h=h->next) printf("%d→",h->data); printf("NULL\n"); }
25
26int main() {
27 Node *h = NULL;
28 int vals[] = {34, 8, 64, 51, 32, 21};
29 for (int i = 5; i >= 0; i--) h = push(h, vals[i]);
30 printf("Before: "); print_list(h);
31 bubble_list(h);
32 printf("After: "); print_list(h);
33 return 0;
34}

Before: 34→8→64→51→32→21→NULL After: 8→21→32→34→51→64→NULL

void* = "아무 타입이나 가리키는 포인터"

문제: bubble_sort(int*), bubble_sort(Student*)
→ 타입마다 함수를 새로 만들어야?

해결: void*로 받으면 하나로 통일!
대신 "크기"를 알려줘야 함 (size 인자)

memcpy로 교환:
타입을 모르니까 대입(=) 불가
→ memcpy(tmp, a, size)로 바이트 복사!

비교 함수:
타입을 모르니까 비교(< >) 불가
→ 비교 함수를 인자로 받아서 위임!
→ qsort가 정확히 이 방식!
memcpy swap
타입 모를 때 교환: memcpy(tmp,a,size); memcpy(a,b,size); memcpy(b,tmp,size); 바이트 복사!
void* = 타입 안전 포기, 범용성 획득! 비교 함수만 바꾸면 int든 구조체든 정렬!
[C 코드] 버블 정렬 — 칵테일 정렬 구현
1#include <stdio.h>
2
3void cocktail_sort(int *a, int n) {
4 int start = 0, end = n-1;
5 int swapped = 1;
6 int rounds = 0;
7 while (swapped) {
8 swapped = 0;
9 // 좌→우
10 for (int i = start; i < end; i++)
11 if (a[i] > a[i+1]) { int t=a[i];a[i]=a[i+1];a[i+1]=t; swapped=1; }
12 if (!swapped) break;
13 end--;
14 swapped = 0;
15 // 우→좌
16 for (int i = end-1; i >= start; i--)
17 if (a[i] > a[i+1]) { int t=a[i];a[i]=a[i+1];a[i+1]=t; swapped=1; }
18 start++;
19 rounds++;
20 }
21 printf("Cocktail: %d rounds\n", rounds);
22}
23
24int main() {
25 int a[] = {5,1,4,2,8,0,2};
26 int n = 7;
27 cocktail_sort(a, n);
28 for(int i=0;i<n;i++) printf("%d ",a[i]);
29 printf("\n");
30 return 0;
31}

turtle problem
거북이 문제. 작은 값이 배열 끝에 있으면 한 칸씩만 이동. 칵테일 정렬이 양방향으로 해결!
칵테일 = 양방향 버블! 좌→우로 큰 값 밀고, 우→좌로 작은 값 당기기. 3라운드면 끝!
odd-even sort
홀짝 정렬. 홀수/짝수 인덱스 교대 비교. 병렬화 가능! GPU 정렬의 기초.

Bogosort
보고소트. 랜덤 섞기→정렬됐나 확인→반복. 평균 O(n·n!). 절대 쓰지 마! 최악의 정렬.
버블 O(n²) vs 병합 O(n log n). n=10000이면 5천만 vs 13만. 알고리즘 선택이 곧 성능!

void* = "타입 모르는 주소"
int도 될 수 있고, 구조체도 될 수 있고
어떤 타입이든 주소는 주소!

memcpy로 교환하는 이유:
void*는 * 연산자로 값을 못 읽음
→ 바이트 단위로 복사해야 함!
memcpy(dest, src, size) = size바이트 복사

비교 함수가 필요한 이유:
void*니까 대소 비교를 모름
→ 사용자가 비교 방법을 함수로 전달!
이게 qsort의 원리!
memcpy swap
void* 교환. memcpy(temp,a,size); memcpy(a,b,size); memcpy(b,temp,size); 타입 무관!
void*+함수포인터 = C의 제네릭! 같은 정렬 코드로 int도 struct도 정렬. qsort의 원리!
버블 정렬(Bubble Sort)
인접한 두 원소를 비교해서 교환하며 정렬하는 알고리즘.
pass
배열을 한 번 처음부터 끝까지 훑는 과정.
swap(교환)
두 변수의 값을 서로 바꾸는 것. temp 변수를 이용.
O(n^2)
데이터 n개에 대해 약 n^2번의 연산이 필요한 시간 복잡도.
조기 종료(early termination)
한 pass에서 교환이 없으면 정렬 완료로 판단하고 멈추는 최적화.
이번 유닛 핵심 정리
1
버블 정렬 = 이웃한 두 원소를 비교/교환하는 정렬
2
바깥 루프(pass) x 안쪽 루프(비교) = 이중 반복문
3
한 pass가 끝나면 가장 큰 값이 맨 뒤로 확정
4
시간 복잡도 O(n^2), 조기 종료로 최적화 가능
