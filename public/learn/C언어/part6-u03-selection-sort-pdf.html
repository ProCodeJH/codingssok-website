가장 작은 걸 골라서 앞에!

체육 시간에 키 순서대로 줄 서기를 할 때, 가장 작은 친구를 먼저 찾아 맨 앞에 세우고, 그 다음 작은 친구를 찾아 두 번째에 세우고... 이게 바로 선택 정렬이야!

선택 정렬(Selection Sort)은 정렬되지 않은 부분에서 가장 작은 값을 찾아 맨 앞 원소와 교환하는 알고리즘이에요. 매 단계마다 하나의 원소가 제자리를 찾아요.

선택 정렬의 핵심 흐름
버블 정렬은 비교할 때마다 교환하지만, 선택 정렬은 최솟값을 찾은 다음 딱 한 번만 교환해! 그래서 교환 횟수가 적어.

선택 정렬 C 코드

selection_sort.c
1#include <stdio.h>
2
3void selectionSort(int arr[], int n) {
4 for (int i = 0; i < n - 1; i++) {
5 int minIdx = i;
6 for (int j = i + 1; j < n; j++) {
7 if (arr[j] < arr[minIdx]) {
8 minIdx = j;
9 }
10 }
11 if (minIdx != i) {
12 int temp = arr[i];
13 arr[i] = arr[minIdx];
14 arr[minIdx] = temp;
15 }
16 }
17}
18
19int main(void) {
20 int arr[] = {4, 2, 5, 1, 3};
21 int n = 5;
22
23 printf("정렬 전: ");
24 for (int i = 0; i < n; i++) printf("%d ", arr[i]);
25 printf("\n");
26
27 selectionSort(arr, n);
28
29 printf("정렬 후: ");
30 for (int i = 0; i < n; i++) printf("%d ", arr[i]);
31 printf("\n");
32 return 0;
33}
실행 결과
정렬 전: 4 2 5 1 3 정렬 후: 1 2 3 4 5
5줄minIdx = i -- 현재 위치를 최솟값 후보로 시작해요.
6줄안쪽 루프에서 i+1부터 끝까지 훑으며 더 작은 값이 있으면 minIdx를 갱신해요.
11줄minIdx != i일 때만 교환해요. 이미 최솟값이 제자리면 교환 불필요!

버블 정렬 vs 선택 정렬

같은 O(n^2)이지만 특성이 다르다!
핵심 비교
공통점: 둘 다 시간 복잡도 O(n^2), 이중 반복문 사용
차이점:
- 버블: 비교할 때마다 교환 가능 -> 교환 많음
- 선택: 한 pass에 교환 최대 1번 -> 교환 적음
- 버블: 조기 종료 가능 / 선택: 항상 같은 횟수 비교
교환 연산이 비싼 상황(예: 큰 구조체를 교환해야 할 때)에서는 선택 정렬이 버블 정렬보다 유리해!
[디버깅] 선택 정렬에서 자주 하는 실수
1. minIdx 초기화 위치 -- 바깥 루프 안에서 매번 초기화해야 해요
2. 안쪽 루프 시작점 -- j = i+1이어야 해요. j = 0이면 이미 정렬된 부분도 다시 비교
3. 교환 대상 -- arr[i]와 arr[minIdx]를 교환. 인덱스를 헷갈리지 마세요
[KOI] 선택 정렬 활용
KOI에서 'k번째로 작은 수 찾기' 문제가 나오면, 선택 정렬을 k번만 수행하면 돼요. 전체 정렬이 필요 없으니 효율적이에요!

27// 이중 선택 정렬 (양쪽에서 동시에)
28void double_selection_sort(int *a, int n) {
29 int left = 0, right = n-1;
30 int rounds = 0;
31 while (left < right) {
32 int min_idx = left, max_idx = left;
33 for (int i = left; i <= right; i++) {
34 if (a[i] < a[min_idx]) min_idx = i;
35 if (a[i] > a[max_idx]) max_idx = i;
36 }
37 // 최소값을 왼쪽에
38 int t = a[left]; a[left] = a[min_idx]; a[min_idx] = t;
39 // max_idx가 left였으면 위치가 바뀜
40 if (max_idx == left) max_idx = min_idx;
41 // 최대값을 오른쪽에
42 t = a[right]; a[right] = a[max_idx]; a[max_idx] = t;
43 left++; right--; rounds++;
44 }
45 printf(" Double selection: %d rounds (vs %d normal)\n", rounds, n-1);
46}

[C 코드] 선택 정렬 — Heap Sort 맛보기
1#include <stdio.h>
2
3void heapify(int *a, int n, int i) {
4 int largest = i, left = 2*i+1, right = 2*i+2;
5 if (left < n && a[left] > a[largest]) largest = left;
6 if (right < n && a[right] > a[largest]) largest = right;
7 if (largest != i) {
8 int t = a[i]; a[i] = a[largest]; a[largest] = t;
9 heapify(a, n, largest);
10 }
11}
12
13void heap_sort(int *a, int n) {
14 // 힙 구성
15 for (int i = n/2-1; i >= 0; i--) heapify(a, n, i);
16 // 하나씩 추출
17 for (int i = n-1; i > 0; i--) {
18 int t = a[0]; a[0] = a[i]; a[i] = t; // 최대값을 뒤로
19 heapify(a, i, 0); // 나머지 다시 힙 구성
20 }
21}
22

[심화 내용] 힙 정렬 = 선택 정렬의 진화
선택 정렬의 문제:
최소값 찾기 = O(n) → 전체 O(n²)

힙 정렬의 해결:
최대값 찾기 = O(1) (루트!)
힙 복구 = O(log n)
→ 전체 O(n log n)!

힙 = 완전 이진 트리:
부모 ≥ 자식 (최대 힙)
배열로 표현: children[i] = 2i+1, 2i+2
parent[i] = (i-1)/2

in-place + O(n log n) + 불안정
priority queue
우선순위 큐. 힙으로 구현. 삽입/삭제 O(log n). 다익스트라, 작업 스케줄링에 핵심!
힙 정렬 = 선택 정렬 + 힙! 최대값을 O(1)에 찾고 O(log n)에 복구. n²→n log n!
[C 코드] 선택 정렬 — 부분 정렬 (Top-K)
1#include <stdio.h>
2
3// 상위 K개만 정렬 (전체 정렬 불필요!)
4void partial_selection(int *a, int n, int k) {
5 for (int i = 0; i < k && i < n-1; i++) {
6 int min = i;
7 for (int j = i+1; j < n; j++) if (a[j] < a[min]) min = j;
8 int t = a[i]; a[i] = a[min]; a[min] = t;
9 }
10}
11
12// k번째 작은 수 찾기 (Quick Select)
13int quick_select(int *a, int lo, int hi, int k) {
14 if (lo == hi) return a[lo];
15 int pivot = a[hi], i = lo;
16 for (int j = lo; j < hi; j++)
17 if (a[j] <= pivot) { int t=a[i];a[i]=a[j];a[j]=t; i++; }
18 int t=a[i]; a[i]=a[hi]; a[hi]=t;
19
20 if (i == k) return a[i];
21 else if (k < i) return quick_select(a, lo, i-1, k);
22 else return quick_select(a, i+1, hi, k);
23}

[심화 내용] Quick Select
k번째 원소 찾기:
정렬: O(n log n) → 과도함!
Quick Select: O(n) 평균!

원리:
partition 후 피벗 위치 = p
k == p → 찾았다!
k < p → 왼쪽만 재귀
k > p → 오른쪽만 재귀
→ 한쪽만 재귀! n+n/2+n/4+... = 2n = O(n)

Median of Medians:
최악 O(n) 보장! 5개씩 묶어 중앙값의 중앙값을 피벗으로
Quick Select
빠른 선택. k번째 원소를 O(n)에 찾기. 파티션 후 한쪽만 재귀! C++ nth_element가 이것.
Top-K problem
상위 K개 문제. 전체 정렬 불필요! 힙 O(n log k) or Quick Select O(n). 검색 엔진 핵심.
Quick Select: 정렬 안 해도 k번째 찾기! partition 한 번에 절반 버림. O(n)!

1#include <stdio.h>
2
3void heapify(int *a, int n, int i) {
4 int largest = i;
5 int left = 2*i+1, right = 2*i+2;
6 if (left < n && a[left] > a[largest]) largest = left;
7 if (right < n && a[right] > a[largest]) largest = right;
8 if (largest != i) {
9 int t = a[i]; a[i] = a[largest]; a[largest] = t;
10 heapify(a, n, largest);
11 }
12}
13
14void heap_sort(int *a, int n) {
15 // Build max heap
16 for (int i = n/2-1; i >= 0; i--) heapify(a, n, i);
17 // Extract elements
18 for (int i = n-1; i > 0; i--) {
19 int t = a[0]; a[0] = a[i]; a[i] = t;
20 heapify(a, i, 0);
21 }
22}

힙 = 완전 이진 트리:
Max heap: 부모 ≥ 자식
Min heap: 부모 ≤ 자식
배열로 구현! children: 2i+1, 2i+2

Heap Sort = 선택 정렬의 O(n log n) 버전:
선택: 최대값 O(n)으로 찾기 → O(n²)
힙: 최대값 O(log n)으로 추출 → O(n log n)!

우선순위 큐:
삽입: O(log n)
최대/최소 추출: O(log n)
→ 다익스트라, 허프만, 이벤트 드리븐에 필수
heap property

Heap Sort = 선택 정렬 + 힙! 최대값 찾기를 O(n)→O(log n)으로. 항상 O(n log n)!

[C 코드] 선택 정렬 — Heap Sort (선택 정렬의 진화!)
1#include <stdio.h>
2
3void swap(int *a, int *b) { int t=*a; *a=*b; *b=t; }
4
5void heapify(int *a, int n, int i) {
6 int largest = i;
7 int left = 2*i+1, right = 2*i+2;
8 if (left < n && a[left] > a[largest]) largest = left;
9 if (right < n && a[right] > a[largest]) largest = right;
10 if (largest != i) {
11 swap(&a[i], &a[largest]);
12 heapify(a, n, largest);
13 }
14}
15
16void heap_sort(int *a, int n) {
17 // 최대 힙 구성
18 for (int i = n/2-1; i >= 0; i--) heapify(a, n, i);
19 // 하나씩 추출
20 for (int i = n-1; i > 0; i--) {
21 swap(&a[0], &a[i]); // 최대값을 끝으로
22 heapify(a, i, 0); // 나머지 힙 복원
23 }
24}
25
26void print_heap(int *a, int n) {
27 printf("Heap: ");
28 for (int i = 0; i < n; i++) printf("%d ", a[i]);
29 printf("\n Tree: ");
30 int level = 1, count = 0;
31 for (int i = 0; i < n; i++) {
32 printf("%d ", a[i]);
33 count++;
34 if (count == level) { printf("| "); level *= 2; count = 0; }
35 }
36 printf("\n");
37}

[심화 내용] 힙과 우선순위 큐
힙 = 완전 이진 트리 + 힙 속성:
최대 힙: 부모 ≥ 자식
최소 힙: 부모 ≤ 자식

배열로 표현:
부모: (i-1)/2
왼쪽 자식: 2*i+1
오른쪽 자식: 2*i+2

Heap Sort = 선택 정렬의 업그레이드!
선택 정렬: 최대값 찾기 O(n) → 전체 O(n²)
힙 정렬: 최대값 찾기 O(log n) → 전체 O(n log n)

우선순위 큐:
삽입: O(log n), 최대 추출: O(log n)
→ 다익스트라, 허프만, 작업 스케줄링!
complete binary tree
완전 이진 트리. 마지막 레벨 빼고 꽉 참. 배열로 효율적 표현! 힙의 기본 구조.
priority queue
우선순위 큐. 최대/최소를 O(log n)에 추출. 힙으로 구현! 다익스트라, A*의 핵심.
힙 정렬 = 선택 정렬의 O(n log n) 버전! 최대값 찾기를 힙으로 가속. 항상 O(n log n)!
선택 정렬(Selection Sort)
남은 원소 중 최솟값을 선택해 앞에 배치하는 정렬 알고리즘.
최솟값 인덱스(minIdx)
현재까지 찾은 가장 작은 값의 배열 인덱스.
교환 횟수
정렬 과정에서 두 원소의 위치를 바꾼 총 횟수.
비교 횟수
정렬 과정에서 두 원소의 크기를 비교한 총 횟수.
제자리 정렬(in-place sort)
추가 배열 없이 원래 배열 안에서 정렬하는 방식. 버블/선택/삽입 모두 해당.
