카드 정리하듯 정렬하기

카드 게임할 때 새 카드를 받으면 어떻게 해? 이미 정리된 손패에서 올바른 위치를 찾아 끼워 넣지! 삽입 정렬이 바로 그 방식이야.

삽입 정렬(Insertion Sort)은 배열의 두 번째 원소부터 시작해서, 각 원소를 이미 정렬된 앞부분의 올바른 위치에 삽입하는 알고리즘이에요.

단계별 시각화

배열 [3,1,4,5,2]의 삽입 정렬 과정

코드 구조 이해하기

삽입 정렬의 핵심: key 저장 -> 밀기 -> 삽입
삽입 정렬은 교환(swap)이 아니라 이동(shift)을 해! key 값을 따로 저장해두고, 큰 원소들을 한 칸씩 뒤로 밀어서 빈 자리를 만들어.

삽입 정렬 C 코드

insertion_sort.c
1#include <stdio.h>
2
3void insertionSort(int arr[], int n) {
4 for (int i = 1; i < n; i++) {
5 int key = arr[i];
6 int j = i - 1;
7 while (j >= 0 && arr[j] > key) {
8 arr[j + 1] = arr[j];
9 j--;
10 }
11 arr[j + 1] = key;
12 }
13}
14
15int main(void) {
16 int arr[] = {3, 1, 4, 5, 2};
17 int n = 5;
18
19 printf("정렬 전: ");
20 for (int i = 0; i < n; i++) printf("%d ", arr[i]);
21 printf("\n");
22
23 insertionSort(arr, n);
24
25 printf("정렬 후: ");
26 for (int i = 0; i < n; i++) printf("%d ", arr[i]);
27 printf("\n");
28 return 0;
29}
실행 결과
정렬 전: 3 1 4 5 2 정렬 후: 1 2 3 4 5
4줄i = 1부터 시작. 첫 번째 원소는 혼자이므로 이미 '정렬된 부분'이에요.
5줄key = arr[i] -- 삽입할 값을 따로 저장해요. 이동 과정에서 덮어쓰이니까!
7줄arr[j] > key인 동안, 원소를 한 칸씩 오른쪽으로 이동해요.
11줄arr[j+1] = key -- while 루프가 끝난 뒤, 빈 자리에 key를 삽입해요.

거의 정렬된 데이터에서 빠르다!

데이터 상태에 따른 삽입 정렬 성능 차이
삽입 정렬의 시간 복잡도
최선(Best): O(n) -- 이미 정렬된 경우. while 루프가 거의 실행 안 됨.
평균/최악(Worst): O(n^2) -- 역순인 경우. 매번 모든 원소를 밀어야 함.

거의 정렬된 데이터에서는 버블/선택보다 훨씬 빠르다!
실전에서 데이터가 '거의 정렬'되어 있는 경우가 많아! 예를 들어 이미 정렬된 리스트에 몇 개만 추가된 경우. 이럴 때 삽입 정렬이 최고야.

세 가지 정렬 비교

버블 vs 선택 vs 삽입
버블 정렬: 이웃 비교/교환, 교환 많음, 조기종료 가능
선택 정렬: 최솟값 찾기, 교환 적음(n-1), 조기종료 불가
삽입 정렬: 끼워넣기, 이동 사용, 거의 정렬시 O(n)

모두 O(n^2)이지만, 데이터 특성에 따라 성능이 달라요!
[디버깅] 삽입 정렬에서 자주 하는 실수
1. key 저장 누락 -- key를 저장하지 않으면 이동 과정에서 원래 값이 사라져요
2. while 조건 순서 -- j >= 0을 arr[j] > key보다 먼저 체크해야 해요 (단축 평가)
3. 삽입 위치 -- arr[j+1]이에요. j가 아니라 j+1!
[KOI] 삽입 정렬의 활용
KOI에서 '거의 정렬된 배열'이 주어지는 문제가 있어요. 이때 삽입 정렬을 쓰면 O(n)에 가깝게 풀 수 있어요. 또한 이동 횟수를 세는 문제도 출제돼요.

12// 이진 삽입 정렬 (삽입 위치를 이진 탐색)
13void binary_insertion_sort(int *a, int n) {
14 int comps = 0;
15 for (int i = 1; i < n; i++) {
16 int key = a[i];
17 int lo = 0, hi = i;
18 while (lo < hi) { comps++; int mid = (lo+hi)/2; if (a[mid] <= key) lo=mid+1; else hi=mid; }
19 for (int j = i; j > lo; j--) a[j] = a[j-1];
20 a[lo] = key;
21 }
22 printf(" Binary insertion: %d comparisons\n", comps);
23}
24
25// Shell Sort (삽입 정렬의 일반화)
26void shell_sort(int *a, int n) {
27 int gaps[] = {701, 301, 132, 57, 23, 10, 4, 1}; // Ciura gap
28 for (int g = 0; g < 8; g++) {
29 int gap = gaps[g];
30 if (gap >= n) continue;
31 for (int i = gap; i < n; i++) {
32 int key = a[i], j = i;
33 while (j >= gap && a[j-gap] > key) { a[j] = a[j-gap]; j -= gap; }
34 a[j] = key;
35 }
36 }
37}

[C 코드] 삽입 정렬 — Timsort 핵심
1#include <stdio.h>

4// Timsort의 핵심: 작은 배열은 삽입 정렬!
5#define MIN_RUN 4
6
7void tim_insertion(int *a, int l, int r) {
8 for (int i = l+1; i <= r; i++) {
9 int key = a[i], j = i-1;
10 while (j >= l && a[j] > key) { a[j+1] = a[j]; j--; }
11 a[j+1] = key;
12 }
13}
14
15void tim_merge(int *a, int l, int m, int r) {
16 int n1=m-l+1, n2=r-m;
17 int L[50], R[50];
18 for(int i=0;i<n1;i++) L[i]=a[l+i];
19 for(int i=0;i<n2;i++) R[i]=a[m+1+i];
20 int i=0,j=0,k=l;
21 while(i<n1&&j<n2) a[k++]=(L[i]<=R[j])?L[i++]:R[j++];
22 while(i<n1) a[k++]=L[i++];
23 while(j<n2) a[k++]=R[j++];
24}
25
26void mini_timsort(int *a, int n) {
27 // Step 1: 작은 블록(run)을 삽입 정렬
28 for (int i = 0; i < n; i += MIN_RUN) {
29 int end = i + MIN_RUN - 1;
30 if (end >= n) end = n - 1;
31 tim_insertion(a, i, end);
32 }
33 // Step 2: 블록을 병합
34 for (int size = MIN_RUN; size < n; size *= 2) {
35 for (int l = 0; l < n; l += 2*size) {
36 int m = l + size - 1;
37 int r = l + 2*size - 1;
38 if (m >= n) continue;
39 if (r >= n) r = n - 1;
40 tim_merge(a, l, m, r);
41 }
42 }
43}

minrun
Timsort의 최소 블록 크기. 보통 32~64. 삽입 정렬이 작은 배열에서 빠르니까!
Timsort = 삽입+병합! 작은 블록은 삽입(빠름), 큰 블록은 병합(안정). Python의 선택!
[심화 내용] 삽입 정렬의 수학적 분석
비교 횟수:
최선(정렬됨): n-1 = O(n)
최악(역순): n(n-1)/2 = O(n²)
평균: n(n-1)/4 = O(n²)

이동 횟수 = 역쌍(inversion) 수:
평균 역쌍 = n(n-1)/4
거의 정렬된 데이터: 역쌍 적음 → 빠름!

삽입 정렬이 퀵소트보다 나은 경우:
1. n < 16 (오버헤드 적음)
2. 거의 정렬됨 (적응적)
3. 안정성 필요
→ 실전 정렬은 삽입+퀵/병합 하이브리드!
sentinel
감시값. a[0]에 최소값 배치. while 루프에서 j>=0 검사 제거! 삽입 정렬 미세 최적화.
삽입 정렬은 거의 정렬된 데이터에 최강! 역쌍이 적으면 O(n). 실전에서 중요!

[C 코드] 삽입 정렬 — 연결 리스트 정렬
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node { int data; struct Node *next; } Node;
5
6Node* create(int d) { Node *n=malloc(sizeof(Node)); n->data=d; n->next=NULL; return n; }
7
8// 연결 리스트 삽입 정렬
9Node* insertion_sort_list(Node *head) {
10 Node *sorted = NULL;
11 Node *cur = head;
12 while (cur) {
13 Node *next = cur->next;
14 // sorted 리스트의 올바른 위치에 삽입
15 if (!sorted || cur->data <= sorted->data) {
16 cur->next = sorted;
17 sorted = cur;

3#define MIN_RUN 4
4
5void insertion_range(int *a, int lo, int hi) {
6 for (int i = lo+1; i <= hi; i++) {
7 int key = a[i], j = i-1;
8 while (j >= lo && a[j] > key) { a[j+1] = a[j]; j--; }
9 a[j+1] = key;
10 }
11}

13void merge_range(int *a, int l, int m, int r) {
14 int buf[200];
15 int i=l, j=m+1, k=0;
16 while(i<=m && j<=r) buf[k++]=(a[i]<=a[j])?a[i++]:a[j++];
17 while(i<=m) buf[k++]=a[i++];
18 while(j<=r) buf[k++]=a[j++];
19 for(int x=0;x<k;x++) a[l+x]=buf[x];
20}

22void mini_timsort(int *a, int n) {
23 // 단계1: MIN_RUN 크기 run을 삽입 정렬
24 for (int i = 0; i < n; i += MIN_RUN) {
25 int end = i + MIN_RUN - 1;
26 if (end >= n) end = n - 1;
27 insertion_range(a, i, end);
28 }
29 // 단계2: run들을 병합
30 for (int size = MIN_RUN; size < n; size *= 2) {
31 for (int l = 0; l < n; l += 2*size) {
32 int m = l + size - 1;
33 int r = l + 2*size - 1;
34 if (m >= n) continue;
35 if (r >= n) r = n - 1;
36 merge_range(a, l, m, r);
37 }
38 }
39}

[C 코드] 삽입 정렬 — 연결 리스트 + Merge Sort
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node { int data; struct Node *next; } Node;
5
6Node* create(int d) { Node *n=malloc(sizeof(Node)); n->data=d; n->next=NULL; return n; }
7
8// 연결 리스트 삽입 정렬
9Node* insertion_sort_list(Node *head) {
10 Node *sorted = NULL;
11 while (head) {
12 Node *cur = head;
13 head = head->next;
14 if (!sorted || cur->data < sorted->data) {
15 cur->next = sorted; sorted = cur;
16 } else {
17 Node *p = sorted;
18 while (p->next && p->next->data < cur->data) p = p->next;
19 cur->next = p->next; p->next = cur;
20 }
21 }
22 return sorted;
23}
24
25// Merge Sort for linked list
26Node* merge(Node *a, Node *b) {
27 if(!a) return b; if(!b) return a;
28 if(a->data<=b->data){a->next=merge(a->next,b);return a;}
29 else{b->next=merge(a,b->next);return b;}
30}
31
32void split(Node *head, Node **a, Node **b) {
33 Node *slow=head, *fast=head->next;
34 while(fast&&fast->next){slow=slow->next;fast=fast->next->next;}
35 *a=head; *b=slow->next; slow->next=NULL;
36}
37
38Node* merge_sort(Node *head) {
39 if(!head||!head->next) return head;
40 Node *a, *b;
41 split(head, &a, &b);
42 return merge(merge_sort(a), merge_sort(b));
43}

[심화 내용] 병합 정렬의 원리
분할 정복:
1. 분할: 리스트를 반으로 나눔
2. 정복: 각각 재귀 정렬
3. 합병: 두 정렬 리스트를 병합

시간: O(n log n) 항상!
최선=평균=최악 전부 O(n log n)
퀵 정렬과 달리 최악이 없음!

연결 리스트에서 특히 유리:
배열: 추가 O(n) 공간 필요
연결 리스트: 추가 공간 O(1)!
→ 연결 리스트 정렬 = 병합 정렬이 정답
merge sort
병합 정렬. 분할+정복+합병. O(n log n) 항상! 안정 정렬. 연결 리스트에 최적. Java의 기반!
slow-fast pointer
느린/빠른 포인터. slow=1칸, fast=2칸. fast 끝나면 slow=중간! 리스트 분할, 순환 검출.
병합 정렬 = 반으로 쪼개고 합치기! 쪼갤 때 O(1), 합칠 때 O(n). 총 O(n log n)!
삽입 정렬(Insertion Sort)
새 원소를 이미 정렬된 부분의 올바른 위치에 삽입하는 정렬 알고리즘.
key
현재 삽입할 원소의 값을 임시 저장하는 변수.
이동(shift)
원소를 한 칸 오른쪽으로 복사하는 연산. 교환과 다름.
적응적 정렬(adaptive sort)
데이터가 거의 정렬되어 있을 때 더 빠르게 동작하는 정렬. 삽입 정렬이 대표적.
단축 평가(short-circuit evaluation)
&&에서 첫 조건이 거짓이면 두 번째 조건을 평가하지 않는 것. j>=0을 먼저 체크하는 이유.
