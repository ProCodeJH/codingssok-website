Lesson 14-6

이진 탐색

정렬된 배열에서 반씩 나누며 빠르게 찾는 이진 탐색을 배워요. 선형 탐색과 비교하고, KOI에서의 활용법을 익혀요.

📚 이번에 배울 것

  • 선형 탐색과 이진 탐색의 차이를 설명할 수 있어요.
  • 이진 탐색을 C 코드로 구현할 수 있어요.
  • O(n)과 O(log n)의 차이를 이해해요.
  • 이진 탐색의 전제 조건(정렬)을 알아요.

✅ 시작 전 체크

  • 배열과 반복문을 사용할 수 있다
  • 정렬의 개념을 안다 (Unit 01~05)
Part 1 진행률: 4%

찾기의 두 가지 방법

사전에서 '하마'를 찾을 때, 1페이지부터 한 장씩 넘기는 사람은 없지? 대충 뒷부분을 펴서 'ㅎ' 근처를 찾잖아. 그게 이진 탐색의 아이디어야!

선형 탐색(Linear Search)은 배열의 처음부터 끝까지 하나씩 확인하는 방법이에요. 간단하지만 느려요.
이진 탐색(Binary Search)은 정렬된 배열에서 가운데를 확인하고, 찾는 값이 어느 쪽에 있는지 판단해서 반씩 줄여나가는 방법이에요.

선형 탐색 vs 이진 탐색

같은 배열에서 11 찾기: 6번 vs 2번!

선형 탐색 코드

linear_search.c
1#include <stdio.h>
2
3int linearSearch(int arr[], int n, int target) {
4 for (int i = 0; i < n; i++) {
5 if (arr[i] == target) {
6 return i;
7 }
8 }
9 return -1;
10}
11
12int main(void) {

16

18 if (result != -1)
19 printf("%d은(는) 인덱스 %d에 있습니다.\n", target, result);

실행 결과
9은(는) 인덱스 3에 있습니다.
선형 탐색은 배열이 정렬되어 있지 않아도 사용할 수 있어. 하지만 최악의 경우 배열 전체를 다 봐야 하니까 O(n)이야.

이진 탐색 알고리즘

이진 탐색의 핵심 흐름
binary_search.c
1#include <stdio.h>
2
3int binarySearch(int arr[], int n, int target) {
4 int low = 0, high = n - 1;
5
6 while (low <= high) {
7 int mid = (low + high) / 2;
8
9 if (arr[mid] == target) {
10 return mid;
11 } else if (arr[mid] < target) {
12 low = mid + 1;
13 } else {
14 high = mid - 1;
15 }
16 }
17 return -1;
18}
19
20int main(void) {
21 int arr[] = {1, 3, 5, 7, 9, 11, 13, 15};
22 int n = 8;
23 int target = 11;
24
25 int result = binarySearch(arr, n, target);
26 if (result != -1)
27 printf("%d은(는) 인덱스 %d에 있습니다.\n", target, result);
28 else
29 printf("%d을(를) 찾지 못했습니다.\n", target);
30 return 0;
31}
실행 결과
11은(는) 인덱스 5에 있습니다.
4줄low = 0, high = n-1 -- 탐색 범위의 시작과 끝을 설정해요.
6줄low <= high -- 탐색 범위가 유효한 동안 반복. 이 조건이 핵심!
7줄mid = (low + high) / 2 -- 가운데 인덱스를 계산해요.
11줄찾는 값이 mid보다 크면 오른쪽 반만 탐색 (low = mid + 1)
13줄찾는 값이 mid보다 작으면 왼쪽 반만 탐색 (high = mid - 1)

O(n) vs O(log n) 비교

데이터가 많을수록 이진 탐색의 위력이 커진다!
10억 개의 데이터에서도 이진 탐색은 30번이면 찾아! 선형 탐색은 10억 번 비교해야 하는데. 이게 O(log n)의 마법이야.
이진 탐색의 전제 조건
이진 탐색은 반드시 정렬된 배열에서만 동작해요!
정렬되지 않은 배열에 이진 탐색을 적용하면 엉뚱한 결과가 나와요.

전략: qsort()로 먼저 정렬 -> 이진 탐색 적용
[디버깅] 이진 탐색에서 자주 하는 실수
1. 정렬 안 한 배열에 적용 -- 반드시 정렬 후 사용!
2. while 조건 -- low < high가 아니라 low <= high이어야 해요
3. mid 갱신 -- low = mid가 아니라 low = mid + 1이어야 무한루프 방지
4. 오버플로 -- (low + high)가 int 범위 초과 가능. low + (high - low) / 2 사용 권장
[KOI] 이진 탐색 활용
KOI에서 '특정 값 찾기', '경계값 찾기' 문제에 이진 탐색은 필수에요.
응용: Lower Bound(값 이상인 첫 위치), Upper Bound(값 초과인 첫 위치)를 이진 탐색으로 구현하면 범위 내 개수도 빠르게 구할 수 있어요.

[C 코드] 이진 탐색 — lower/upper bound + 응용
1#include <stdio.h>
2
3// 기본 이진 탐색
4int binary_search(int *a, int n, int target) {
5 int lo = 0, hi = n - 1;
6 while (lo <= hi) {
7 int mid = lo + (hi - lo) / 2; // 오버플로 방지!
8 if (a[mid] == target) return mid;
9 else if (a[mid] < target) lo = mid + 1;
10 else hi = mid - 1;
11 }
12 return -1;
13}
14
15// lower_bound: target 이상인 첫 위치
16int lower_bound(int *a, int n, int target) {
17 int lo = 0, hi = n;
18 while (lo < hi) {
19 int mid = lo + (hi - lo) / 2;
20 if (a[mid] < target) lo = mid + 1;
21 else hi = mid;
22 }
23 return lo;
24}
25
26// upper_bound: target 초과인 첫 위치
27int upper_bound(int *a, int n, int target) {
28 int lo = 0, hi = n;
29 while (lo < hi) {
30 int mid = lo + (hi - lo) / 2;
31 if (a[mid] <= target) lo = mid + 1;
32 else hi = mid;
33 }
34 return lo;
35}
36
37// 응용: 제곱근 구하기
38double sqrt_bisect(double x) {
39 double lo = 0, hi = x > 1 ? x : 1;
40 for (int i = 0; i < 100; i++) { // 충분한 정밀도
41 double mid = (lo + hi) / 2;
42 if (mid * mid < x) lo = mid;
43 else hi = mid;
44 }
45 return (lo + hi) / 2;
46}

[C 코드] 이진 탐색 — 회전 배열 + bsearch
1#include <stdio.h>
2#include <stdlib.h>
3
4// 회전된 정렬 배열에서 탐색
5int search_rotated(int *a, int n, int target) {
6 int lo = 0, hi = n-1;
7 while (lo <= hi) {
8 int mid = lo + (hi-lo)/2;
9 if (a[mid] == target) return mid;
10 if (a[lo] <= a[mid]) { // 왼쪽이 정렬됨
11 if (a[lo] <= target && target < a[mid]) hi = mid-1;
12 else lo = mid+1;
13 } else { // 오른쪽이 정렬됨
14 if (a[mid] < target && target <= a[hi]) lo = mid+1;
15 else hi = mid-1;
16 }
17 }
18 return -1;
19}

[C 코드] 이진 탐색 — Parametric Search
1#include <stdio.h>
2
3// 나무 자르기: 높이 H로 잘라서 M미터 이상 얻을 수 있는 최대 H
4long long cut_wood(int *trees, int n, int H) {
5 long long total = 0;
6 for (int i = 0; i < n; i++)
7 if (trees[i] > H) total += trees[i] - H;
8 return total;
9}
10
11int max_cut_height(int *trees, int n, int M) {
12 int lo = 0, hi = 0;
13 for (int i = 0; i < n; i++) if (trees[i] > hi) hi = trees[i];
14 int answer = 0;
15 while (lo <= hi) {
16 int mid = (lo + hi) / 2;
17 if (cut_wood(trees, n, mid) >= M) {
18 answer = mid; // 가능! 더 높이 시도
19 lo = mid + 1;
20 } else {
21 hi = mid - 1;
22 }
23 }
24 return answer;
25}
26
27int main() {
28 int trees[] = {20, 15, 10, 17};
29 int M = 7;
30 int H = max_cut_height(trees, 4, M);
31 printf("Trees: 20 15 10 17\n");
32 printf("Need: %d meters\n", M);
33 printf("Max height H = %d\n", H);
34 printf("Wood obtained: %lld meters\n", cut_wood(trees, 4, H));
35 return 0;
36}

2

22// 피크 원소 찾기 (양쪽보다 큰 원소)
23int find_peak(int *a, int n) {
24 int lo = 0, hi = n-1;
25 while (lo < hi) {
26 int mid = lo + (hi-lo)/2;
27 if (a[mid] < a[mid+1]) lo = mid+1;
28 else hi = mid;
29 }
30 return lo;
31}

33// 이진 탐색으로 정답 범위 좁히기
34// 문제: n개 케이크를 m명에게 나눠줄 때 최대 크기
35int max_cake(int *cakes, int n, int m) {
36 int lo = 1, hi = 0;
37 for (int i = 0; i < n; i++) if (cakes[i] > hi) hi = cakes[i];
38 int ans = 0;
39 while (lo <= hi) {
40 int mid = lo + (hi-lo)/2;
41 int pieces = 0;
42 for (int i = 0; i < n; i++) pieces += cakes[i] / mid;
43 if (pieces >= m) { ans = mid; lo = mid+1; }
44 else hi = mid-1;
45 }
46 return ans;
47}

4int interpolation_search(int *a, int n, int target) {
5 int lo = 0, hi = n - 1;
6 int steps = 0;
7 while (lo <= hi && target >= a[lo] && target <= a[hi]) {
8 steps++;
9 if (lo == hi) { return a[lo]==target ? lo : -1; }
10 // 보간 공식: 값의 비율로 위치 추정
11 int pos = lo + (int)((double)(target-a[lo]) / (a[hi]-a[lo]) * (hi-lo));
12 if (a[pos] == target) { printf(" Found in %d steps\n", steps); return pos; }
13 if (a[pos] < target) lo = pos + 1;
14 else hi = pos - 1;
15 }
16 return -1;
17}

1#include <stdio.h>
2
3// 나무 자르기 (Parametric Search)
4// 높이 H로 자르면 H 위 부분만 가져감
5// 최소 M미터 필요할 때 최대 H?
6int max_cut_height(int *trees, int n, int M) {
7 int lo = 0, hi = 0;
8 for (int i = 0; i < n; i++) if (trees[i] > hi) hi = trees[i];
9 int ans = 0;
10 while (lo <= hi) {
11 int mid = lo + (hi-lo)/2;
12 long long total = 0;
13 for (int i = 0; i < n; i++)
14 if (trees[i] > mid) total += trees[i] - mid;
15 if (total >= M) { ans = mid; lo = mid+1; }
16 else hi = mid-1;
17 }
18 return ans;
19}

27 // H=15: (20-15)+(17-15) = 5+2 = 7 ✓
28 return 0;
29}

[C 코드] 이진 탐색 — 회전 배열 + Parametric Search
1#include <stdio.h>
2
3// 회전된 정렬 배열에서 탐색
4// [4,5,6,7,0,1,2] — 7에서 회전됨
5int search_rotated(int *a, int n, int target) {
6 int lo = 0, hi = n-1;
7 while (lo <= hi) {
8 int mid = lo + (hi-lo)/2;
9 if (a[mid] == target) return mid;
10
11 // 왼쪽 절반이 정렬됨
12 if (a[lo] <= a[mid]) {
13 if (target >= a[lo] && target < a[mid]) hi = mid-1;
14 else lo = mid+1;
15 }
16 // 오른쪽 절반이 정렬됨
17 else {
18 if (target > a[mid] && target <= a[hi]) lo = mid+1;
19 else hi = mid-1;
20 }
21 }
22 return -1;
23}
24
25// Parametric Search: 최소 운반 용량 구하기
26// n개 짐을 days일 안에 운반할 최소 용량은?
27int can_deliver(int *weights, int n, int cap, int days) {
28 int cur = 0, d = 1;
29 for (int i = 0; i < n; i++) {
30 if (weights[i] > cap) return 0; // 짐 하나가 용량 초과
31 if (cur + weights[i] > cap) { d++; cur = weights[i]; }
32 else cur += weights[i];
33 }
34 return d <= days;
35}
36
37int min_capacity(int *weights, int n, int days) {
38 int lo = 0, hi = 0;
39 for (int i = 0; i < n; i++) {
40 if (weights[i] > lo) lo = weights[i];
41 hi += weights[i];
42 }
43 while (lo < hi) {
44 int mid = lo + (hi-lo)/2;
45 if (can_deliver(weights, n, mid, days)) hi = mid;
46 else lo = mid+1;
47 }
48 return lo;
49}

[심화 내용] 이진 탐색의 변형들
회전 배열 탐색:
정렬 배열을 중간에서 회전
→ 한쪽은 항상 정렬됨!
→ 정렬된 쪽에서 target 범위 확인

Parametric Search:
"최소 X를 찾아라" → "X로 가능한가?" 이진탐색
결정 문제로 변환! (yes/no)
lo=최소가능, hi=최대가능
→ 코딩 테스트 단골!

Exponential Search:
경계를 2배씩 확장 → 이진 탐색
무한 배열이나 크기 모를 때 유용!
rotated array
회전 배열. [4,5,6,7,0,1,2]. 한쪽이 항상 정렬! O(log n) 탐색. 코딩 면접 단골.
Parametric Search = "답이 X 이상?" 이진 탐색. 최적화→결정 문제 변환! 코테의 핵심 기법!
선형 탐색(Linear Search)
배열의 처음부터 끝까지 하나씩 확인하며 찾는 방법. O(n).
이진 탐색(Binary Search)
정렬된 배열에서 가운데를 기준으로 반씩 줄여가며 찾는 방법. O(log n).
O(log n)
데이터가 2배 되면 탐색 횟수가 1번만 늘어나는 시간 복잡도.
low, high, mid
이진 탐색에서 탐색 범위의 시작, 끝, 가운데 인덱스.
Lower Bound
찾는 값 이상인 첫 번째 위치. 이진 탐색의 응용.
