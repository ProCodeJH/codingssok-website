접시 쌓기와 스택

식당에서 접시를 쌓아 본 적 있어? 새 접시는 항상 맨 위에 올리고, 쓸 때도 맨 위에서 꺼내지. 이게 바로 스택이야!

스택(Stack)은 데이터를 쌓아 올리는 자료구조예요. 가장 나중에 넣은 데이터가 가장 먼저 나오는 LIFO(Last In, First Out) 구조입니다.

스택: 마지막에 넣은 것이 먼저 나온다 (LIFO)

스택의 세 가지 연산

push: 넣기 / pop: 빼기(제거) / peek: 엿보기(유지)
스택의 3대 연산
push(x) -- 스택 맨 위에 데이터 x를 넣는다.
pop() -- 스택 맨 위 데이터를 꺼내서 반환한다 (제거됨).
peek() -- 스택 맨 위 데이터를 확인만 한다 (제거 안 됨).
pop과 peek의 차이가 중요해! pop은 꺼내서 없어지는 거고, peek은 엿보기만 하는 거야. 시험에 자주 나와!

1#include <stdio.h>
2#define MAX 100
3
4int stack[MAX];
5int top = -1;
6
7void push(int x) {
8 if (top >= MAX - 1) {
9 printf("Stack Overflow!\n");
10 return;
11 }
12 stack[++top] = x;
13}
14
15int pop(void) {
16 if (top < 0) {
17 printf("Stack Underflow!\n");
18 return -1;
19 }
20 return stack[top--];
21}
22
23int peek(void) {
24 if (top < 0) return -1;
25 return stack[top];
26}
27
28int main(void) {
29 push(10);
30 push(20);
31 push(30);
32 printf("peek: %d\n", peek());
33 printf("pop: %d\n", pop());
34 printf("peek: %d\n", peek());
35 return 0;
36}
실행 결과

5줄top = -1은 스택이 비어 있다는 뜻이에요. top은 가장 위 데이터의 인덱스를 가리켜요.
11줄stack[++top] = x -- top을 1 증가시킨 뒤 그 자리에 데이터를 넣어요.
18줄return stack[top--] -- 현재 top 위치의 값을 반환한 뒤 top을 1 감소시켜요.
Overflow와 Underflow
Stack Overflow: 스택이 꽉 찬 상태에서 push하려고 할 때 발생.
Stack Underflow: 스택이 빈 상태에서 pop하려고 할 때 발생.
반드시 경계 조건을 검사해야 해요!

실생활 속 스택
웹 브라우저 뒤로 가기 -- 방문한 페이지를 스택에 push, 뒤로 가기를 누르면 pop.
Ctrl+Z (되돌리기) -- 수행한 작업을 스택에 저장, 되돌릴 때 pop.
함수 호출 스택 -- 함수가 호출될 때마다 스택에 쌓이고, 끝나면 pop.
함수 호출도 스택이야! main()이 a()를 부르고, a()가 b()를 부르면 스택에 main -> a -> b 순서로 쌓여. b()가 끝나면 pop되어 a()로 돌아가지.

22int check_brackets(const char *expr) {
23 CharStack s; cs_init(&s);
24 for (int i = 0; expr[i]; i++) {
25 char c = expr[i];
26 if (c=='('||c=='['||c=='{') cs_push(&s, c);
27 else if (c==')'||c==']'||c=='}') {
28 if (cs_empty(&s)) return 0;
29 char top = cs_pop(&s);
30 if ((c==')'&&top!='(') || (c==']'&&top!='[') || (c=='}'&&top!='{'))
31 return 0;
32 }
33 }
34 return cs_empty(&s);

37// 후위 표기법 계산기
38int eval_postfix(const char *expr) {
39 IntStack s; is_init(&s);
40 for (int i = 0; expr[i]; i++) {
41 if (isdigit(expr[i])) { is_push(&s, expr[i]-'0'); }
42 else if (expr[i]==' ') continue;
43 else {
44 int b = is_pop(&s), a = is_pop(&s);
45 switch(expr[i]) {
46 case '+': is_push(&s, a+b); break;
47 case '-': is_push(&s, a-b); break;
48 case '*': is_push(&s, a*b); break;
49 case '/': is_push(&s, a/b); break;
50 }
51 }
52 }
53 return is_pop(&s);
54}

13 if(op=='*'||op=='/') return 2;

17void infix_to_postfix(const char *expr, char *out) {
18 CharStack ops; cs_init(&ops);
19 int k = 0;
20 for (int i = 0; expr[i]; i++) {
21 char c = expr[i];
22 if (c == ' ') continue;
23 if (isdigit(c) || isalpha(c)) { out[k++] = c; out[k++] = ' '; }
24 else if (c == '(') cs_push(&ops, c);
25 else if (c == ')') {
26 while (!cs_empty(&ops) && cs_peek(&ops) != '(') {
27 out[k++] = cs_pop(&ops); out[k++] = ' ';
28 }
29 cs_pop(&ops); // '(' 제거
30 } else {
31 while (!cs_empty(&ops) && precedence(cs_peek(&ops)) >= precedence(c)) {
32 out[k++] = cs_pop(&ops); out[k++] = ' ';
33 }
34 cs_push(&ops, c);
35 }
36 }
37 while (!cs_empty(&ops)) { out[k++] = cs_pop(&ops); out[k++] = ' '; }
38 out[k] = 0;
39}

[C 코드] 스택 — 미니 가상 머신
1#include <stdio.h>
2
3typedef enum { OP_PUSH, OP_POP, OP_ADD, OP_SUB, OP_MUL, OP_DUP, OP_PRINT, OP_HALT } OpCode;
4
5typedef struct { OpCode op; int operand; } Instruction;
6
7int stack[100], sp = -1;
8void push(int v) { stack[++sp] = v; }
9int pop() { return stack[sp--]; }
10
11void run(Instruction *program) {
12 int pc = 0;
13 while (1) {
14 Instruction inst = program[pc++];
15 switch (inst.op) {
16 case OP_PUSH: push(inst.operand); break;
17 case OP_POP: pop(); break;
18 case OP_ADD: { int b=pop(),a=pop(); push(a+b); } break;
19 case OP_SUB: { int b=pop(),a=pop(); push(a-b); } break;
20 case OP_MUL: { int b=pop(),a=pop(); push(a*b); } break;
21 case OP_DUP: push(stack[sp]); break;
22 case OP_PRINT: printf("%d\n", stack[sp]); break;
23 case OP_HALT: return;
24 }
25 }
26}

[심화 내용] Shunting Yard 알고리즘
Dijkstra의 정거장 알고리즘:
중위(infix) → 후위(postfix) 변환

규칙:
• 숫자/변수 → 바로 출력
• ( → 스택에 push
• ) → ( 만날 때까지 pop+출력
• 연산자 → 우선순위 비교
스택 top ≥ 현재 → pop+출력
현재를 push
• 끝 → 스택 전부 pop+출력

왜 후위 표기법?
괄호 불필요! 우선순위 내장!
스택 하나로 O(n) 계산!
컴파일러, 계산기의 핵심!
postfix notation
후위 표기법. 3 4 + = 7. 괄호 불필요! 스택으로 O(n) 계산. RPN 계산기. 컴파일러 IR.
a+b*c → a b c * + ! 곱셈이 먼저니까 *가 +보다 앞에. 우선순위가 순서에 반영!
stack frame
스택 프레임. 함수 호출마다 생성. 지역변수+매개변수+복귀주소. 재귀 깊으면 스택오버플로!
stack machine
스택 머신. 모든 연산이 스택 기반. JVM, Python VM, WebAssembly. push/pop+연산!
JVM = 스택 머신! iadd = pop 2개 → 더하기 → push. 후위 표기법 그 자체!
스택은 어디에나! 함수 호출, 괄호 검사, 계산기, Undo, DFS. 만능 자료구조!
monotone stack
단조 스택. 항상 오름/내림차순 유지. next greater element 문제 O(n)! 히스토그램 최대 직사각형.
단조 스택: 스택에 들어올 때 조건 불만족 원소를 pop! O(n)에 모든 NGE(Next Greater Element)!

17void infix_to_postfix(const char *expr, char *output) {
18 CharStack ops; cs_init(&ops);
19 int j = 0;
20 for (int i = 0; expr[i]; i++) {
21 char c = expr[i];
22 if (c == ' ') continue;
23 if (isdigit(c)) {
24 output[j++] = c;
25 output[j++] = ' ';
26 }

28 else if (c == ')') {
29 while (!cs_empty(&ops) && cs_peek(&ops) != '(') {
30 output[j++] = cs_pop(&ops); output[j++] = ' ';
31 }
32 cs_pop(&ops); // '(' 제거
33 }

38 cs_push(&ops, c);
39 }
40 }
41 while (!cs_empty(&ops)) { output[j++] = cs_pop(&ops); output[j++] = ' '; }
42 output[j] = 0;
43}

스택 머신. 연산=스택에서 pop+계산+push. JVM, Python VM. 후위 표기법과 1:1 대응!
Shunting Yard = 스택의 종합예술! 우선순위 비교+괄호 처리+출력. 컴파일러의 핵심!
스택(Stack)
LIFO 구조의 자료구조. 데이터를 쌓아 올리고, 맨 위에서만 넣고 뺀다.
LIFO
Last In, First Out. 마지막에 넣은 것이 가장 먼저 나온다.
push
스택에 데이터를 넣는 연산.
pop
스택에서 데이터를 꺼내는 연산. 꺼낸 데이터는 스택에서 사라진다.
peek
스택의 맨 위 데이터를 확인하는 연산. 데이터를 제거하지 않는다.
top
스택의 맨 위 위치를 가리키는 변수.
Stack Overflow
스택이 가득 찬 상태에서 push를 시도할 때 발생하는 오류.
Stack Underflow
스택이 빈 상태에서 pop을 시도할 때 발생하는 오류.
