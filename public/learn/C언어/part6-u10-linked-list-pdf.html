배열의 한계

배열은 편리하지만, 크기를 미리 정해야 하고 중간에 끼워 넣으려면 뒤의 데이터를 전부 밀어야 해. 데이터 수가 계속 바뀌는 상황에서는 불편하지!

연결 리스트(Linked List)는 배열의 한계를 해결하는 자료구조예요. 각 데이터를 노드(Node)에 담고, 노드끼리 포인터로 연결하는 방식이에요.

노드의 구조

노드 = 데이터(data) + 다음 노드를 가리키는 포인터(next)
node_struct.c
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node {
5 int data;
6 struct Node* next;
7} Node;
8
9Node* createNode(int val) {
10 Node* newNode = (Node*)malloc(sizeof(Node));
11 newNode->data = val;
12 newNode->next = NULL;
13 return newNode;
14}
15
16int main(void) {
17 Node* head = createNode(10);
18 head->next = createNode(20);
19 head->next->next = createNode(30);
20
21 Node* cur = head;
22 while (cur != NULL) {
23 printf("%d -> ", cur->data);
24 cur = cur->next;
25 }
26 printf("NULL\n");
27 return 0;
28}
실행 결과
10 -> 20 -> 30 -> NULL
4줄struct Node -- 노드 구조체. data(값)와 next(다음 노드 포인터)를 가져요.
10줄malloc(sizeof(Node)) -- 노드 크기만큼 메모리를 동적으로 할당해요.
22줄cur = cur->next -- 현재 노드에서 다음 노드로 이동하는 핵심 코드예요.
-> 연산자가 낯설다면 (*cur).data와 같다고 생각하면 돼. 포인터로 구조체 멤버에 접근할 때 쓰는 편의 문법이야.

배열 vs 연결 리스트

삽입/삭제가 잦으면 연결 리스트, 검색이 잦으면 배열

메모리 관리 주의
malloc으로 할당한 메모리는 반드시 free로 해제해야 해요.
해제하지 않으면 메모리 누수(Memory Leak)가 발생해요.
삭제할 노드를 temp에 먼저 저장한 후 free(temp)를 호출하세요.

[심화] 연결 리스트 심화

[C 코드] 연결 리스트 — 포인터 쉽게 이해하기
1#include <stdio.h>
2#include <stdlib.h>
3
4/*
5 * 포인터 = 집 주소!
6 *
7 * int a = 10; ← 집에 10이 들어있음
8 * int *p = &a; ← p에 집 주소를 적어둠
9 * printf("%d", *p); ← 주소 따라가서 값 읽기 → 10
10 *
11 * Node *head; ← 첫 번째 집 주소
12 * head->data ← 그 집에 들어있는 데이터
13 * head->next ← 다음 집 주소가 적힌 메모
14 *
15 * [data|next] → [data|next] → [data|NULL]
16 * ↑ head
17 */
18
19typedef struct Node {
20 int data;
21 struct Node *next; // "다음 노드의 주소"
22} Node;
23
24// 새 노드 생성
25Node* create_node(int data) {
26 Node *n = (Node*)malloc(sizeof(Node));
27 n->data = data;
28 n->next = NULL; // 아직 다음 없음
29 return n;
30}
31
32// 맨 앞에 추가
33Node* push_front(Node *head, int data) {
34 Node *n = create_node(data);
35 n->next = head; // 새 노드가 기존 head를 가리킴
36 return n; // 새 노드가 새로운 head!
37}
38
39// 맨 뒤에 추가
40Node* push_back(Node *head, int data) {
41 Node *n = create_node(data);
42 if (!head) return n; // 빈 리스트
43 Node *p = head;
44 while (p->next) p = p->next; // 끝까지 이동
45 p->next = n; // 마지막 노드의 next = 새 노드
46 return head;
47}
48
49// 삭제
50Node* delete_node(Node *head, int data) {
51 if (!head) return NULL;
52 if (head->data == data) { // 첫 노드 삭제
53 Node *next = head->next;
54 free(head);
55 return next;
56 }
57 Node *p = head;
58 while (p->next && p->next->data != data) p = p->next;
59 if (p->next) {
60 Node *del = p->next;
61 p->next = del->next; // 건너뛰기!
62 free(del);
63 }
64 return head;
65}
66
67void print_list(Node *head) {
68 for (Node *p = head; p; p = p->next) printf("[%d]→", p->data);
69 printf("NULL\n");
70}
71
72void free_list(Node *head) {
73 while (head) { Node *n = head->next; free(head); head = n; }
74}
75
76int main() {
77 Node *list = NULL;
78 list = push_back(list, 10);
79 list = push_back(list, 20);
80 list = push_back(list, 30);
81 list = push_front(list, 5);
82 printf("After push: "); print_list(list);
83
84 list = delete_node(list, 20);
85 printf("After delete 20: "); print_list(list);
86
87 list = delete_node(list, 5);
88 printf("After delete 5: "); print_list(list);
89
90 free_list(list);
91 return 0;
92}
실행 결과
After push: [5]→[10]→[20]→[30]→NULL After delete 20: [5]→[10]→[30]→NULL After delete 5: [10]→[30]→NULL
쉽게 이해하기: 포인터와 연결 리스트
포인터 = 집 주소 메모지!

int *p 는 "어떤 int가 있는 집 주소"
Node *head 는 "어떤 Node가 있는 집 주소"

배열 vs 연결 리스트:
배열: 아파트. 101호, 102호, 103호... 연속!
리스트: 각자 다른 동네 집. 주소 메모로 연결!

→ (화살표)가 뜻하는 것:
[10|●]→[20|●]→[30|NULL]
● = 다음 집 주소가 적힌 메모
NULL = "다음 집 없음" 표시

삭제가 쉬운 이유:
배열: 중간 삭제 → 뒤의 모든 원소 이동
리스트: 메모지만 고치면 끝! (포인터 변경)

[C 코드] 연결 리스트 — 병합 정렬
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node { int data; struct Node *next; } Node;
5
6Node* create(int d) { Node *n=malloc(sizeof(Node)); n->data=d; n->next=NULL; return n; }
7
8// 리스트를 반으로 분할 (slow/fast pointer)
9void split(Node *head, Node **left, Node **right) {
10 Node *slow = head, *fast = head->next;
11 while (fast && fast->next) { slow = slow->next; fast = fast->next->next; }
12 *left = head;
13 *right = slow->next;
14 slow->next = NULL;
15}
16
17// 두 정렬 리스트 병합
18Node* merge(Node *a, Node *b) {
19 if (!a) return b; if (!b) return a;
20 if (a->data <= b->data) { a->next = merge(a->next, b); return a; }
21 else { b->next = merge(a, b->next); return b; }
22}
23
24// 연결 리스트 병합 정렬
25Node* merge_sort(Node *head) {
26 if (!head || !head->next) return head;
27 Node *left, *right;
28 split(head, &left, &right);
29 left = merge_sort(left);
30 right = merge_sort(right);
31 return merge(left, right);
32}
33
34void print_list(Node *h) { for(;h;h=h->next) printf("%d→",h->data); printf("NULL\n"); }
35
36int main() {
37 Node *head = NULL;
38 int vals[] = {38, 27, 43, 3, 9, 82, 10};
39 for (int i = 6; i >= 0; i--) { Node *n = create(vals[i]); n->next = head; head = n; }
40 printf("Before: "); print_list(head);
41 head = merge_sort(head);
42 printf("After: "); print_list(head);
43 return 0;
44}
실행 결과
Before: 38→27→43→3→9→82→10→NULL After: 3→9→10→27→38→43→82→NULL
slow/fast pointer
느린/빠른 포인터. slow 1칸, fast 2칸. fast가 끝이면 slow는 중간! 순환 검출에도 사용.
리스트 병합 정렬: split→재귀→merge. 배열과 달리 추가 메모리 불필요! O(n log n).
[심화 내용] 연결 리스트와 현대 프로그래밍
배열이 거의 항상 이긴다:
캐시 지역성: 배열 >> 리스트
Bjarne Stroustrup: "벡터를 써라!"

그래도 리스트가 필요한 경우:
1. O(1) 삽입/삭제가 빈번 (LRU 캐시)
2. 메모리 할당/해제가 빈번
3. 커널 자료구조 (Linux list_head)
4. 함수형 프로그래밍 (불변 리스트)

현대 대안:
Gap Buffer: 에디터
Unrolled Linked List: 노드에 여러 원소
→ 캐시 + 리스트 장점 결합
unrolled list
언롤드 리스트. 노드에 원소 여러 개. 캐시 지역성↑+포인터 오버헤드↓. 배열+리스트 하이브리드!
현실: 배열(vector)이 거의 항상 빠름! 캐시 때문. 리스트는 특수한 경우에만 선택!
[C 코드] 연결 리스트 — 순환 검출
1#include <stdio.h>
2#include <stdlib.h>
3
4typedef struct Node { int data; struct Node *next; } Node;
5
6Node* make_list(int *vals, int n) {
7 Node *head = NULL, *tail = NULL;
8 for (int i = 0; i < n; i++) {
9 Node *node = (Node*)malloc(sizeof(Node));
10 node->data = vals[i]; node->next = NULL;
11 if (!head) head = tail = node;
12 else { tail->next = node; tail = node; }
13 }
14 return head;
15}
16
17// Floyd 순환 검출
18Node* detect_cycle(Node *head) {
19 Node *slow = head, *fast = head;
20 while (fast && fast->next) {
21 slow = slow->next;
22 fast = fast->next->next;
23 if (slow == fast) {
24 // 순환 시작점 찾기
25 slow = head;
26 while (slow != fast) { slow = slow->next; fast = fast->next; }
27 return slow;
28 }
29 }
30 return NULL;
31}
32
33int main() {
34 int vals[] = {1,2,3,4,5,6,7,8};
35 Node *h = make_list(vals, 8);
36
37 // 순환 만들기: 8→4
38 Node *p = h, *target = NULL;
39 while (p->next) { if (p->data == 4) target = p; p = p->next; }
40 p->next = target; // 순환!
41
42 Node *cycle = detect_cycle(h);
43 printf("Cycle detected at node %d\n", cycle ? cycle->data : -1);
44 return 0;
45}
실행 결과
Cycle detected at node 4
Floyd algorithm
Floyd 알고리즘. 토끼와 거북이. slow 1칸, fast 2칸. 만나면 순환! 시작점도 찾기 가능. O(1) 공간.
Floyd: 느린 거북이+빠른 토끼. 원형 트랙이면 반드시 만남! 순환 시작점도 수학으로 찾기!

1#include <stdio.h>

3
4typedef struct Node { int data; struct Node *next; } Node;
5
6// 메모리 맵 출력

쉽게 이해하기: 더블 포인터 (Node **)
왜 Node **가 필요할까?

Node *head = ...; 에서
head 자체를 바꾸려면?

함수에 head를 넘기면 복사본!
→ 함수 안에서 head = new_node 해도
→ 호출자의 head는 안 바뀜!

해결: head의 주소를 넘기기
void insert(Node **head, int data)
→ *head = new_node; // 원본 변경!

비유:
포인터 = 집 주소가 적힌 메모
더블 포인터 = 그 메모가 어디 있는지의 주소
→ 메모의 내용을 바꿀 수 있음!
double pointer
이중 포인터. Node **pp. 포인터 자체를 수정! head를 함수에서 변경할 때 필수. Linus가 추천하는 패턴!
Node ** = Linus Torvalds가 추천! if(head==NULL) 특수 처리 없이 깔끔하게 삽입!
sentinel node
센티넬 노드. 더미 head 노드. NULL 체크 불필요! head 삽입/삭제 특수 처리 제거.
memory fragmentation
메모리 단편화. malloc/free 반복→작은 구멍들. 연결 리스트의 단점. 메모리 풀로 해결!
센티넬 노드: 빈 리스트에도 노드 1개! head==NULL 체크 불필요. 코드가 깔끔해져!
포인터 그림을 직접 그려봐! 화살표로 연결 관계를 그리면 삽입/삭제가 눈에 보여!
memory pool for list
리스트용 메모리 풀. malloc 대신 풀에서 노드 할당. 단편화 방지+캐시 지역성 향상!
포인터 디버깅: 리스트가 이상하면 메모리 맵을 출력! 주소와 next를 보면 문제가 보여!
container_of
Linux 커널 매크로. 멤버 주소→구조체 주소 역산. 침투적 리스트의 핵심! offsetof 활용.

7 * ═══════════════════════════════════════════
8 *
9 * int *p; → p는 int를 가리키는 포인터
10 * int **pp; → pp는 "int 포인터"를 가리키는 포인터
11 * int (*fp)(int) → fp는 "int를 받아 int를 반환하는 함수"의 포인터
12 * int *a[5] → a는 "int 포인터" 5개의 배열
13 * int (*a)[5] → a는 "int 5개 배열"을 가리키는 포인터
14 *
15 * ═══════════════════════════════════════════
16 * 이중 포인터로 head 수정하기
17 * ═══════════════════════════════════════════
18 *
19 * void add(Node *head, int val) ← head 복사본! 수정 안 됨
20 * void add(Node **head, int val) ← head의 주소! 원본 수정 가능
21 *
22 * main: Node *head = NULL;
23 * add(&head, 10); ← head의 주소를 전달
24 * add: *head = new_node; ← 주소 따라가서 원본 수정!
25 */

29// 이중 포인터 버전 (head 수정 가능!)
30void push_front(Node **head, int data) {
31 Node *n = (Node*)malloc(sizeof(Node));
32 n->data = data;
33 n->next = *head; // *head = 기존 head 값
34 *head = n; // head 자체를 수정!
35}
36
37void push_back(Node **head, int data) {
38 Node *n = (Node*)malloc(sizeof(Node));
39 n->data = data; n->next = NULL;
40 if (!*head) { *head = n; return; }
41 Node *p = *head;
42 while (p->next) p = p->next;
43 p->next = n;
44}
45
46// 포인터의 포인터로 삭제 (우아한 방법!)
47void delete_val(Node **head, int val) {
48 Node **pp = head; // pp = "next 포인터의 주소"
49 while (*pp) {
50 if ((*pp)->data == val) {
51 Node *del = *pp;
52 *pp = del->next; // 건너뛰기
53 free(del);
54 return;
55 }
56 pp = &((*pp)->next); // 다음 next 포인터의 주소로
57 }
58}

60void print_list(Node *h) {
61 for (; h; h = h->next) printf("[%d]→", h->data);
62 printf("NULL\n");
63}

71 push_back(&list, 40);
72 printf("After push: "); print_list(list);

74 delete_val(&list, 20);
75 printf("Delete 20: "); print_list(list);
76

쉽게 이해하기: 이중 포인터 (Node **)
왜 Node **head가 필요한가?

Node *head = NULL; ← main의 head

add(head, 10); ← head의 "복사본" 전달!
→ 함수 안에서 수정해도 main의 head는 그대로!

add(&head, 10); ← head의 "주소" 전달!
→ *head = new_node; 로 원본 수정 가능!

비유:
head = "집 주소가 적힌 메모지"
&head = "그 메모지가 있는 서랍 위치"
서랍 위치를 알면 메모지 내용을 바꿀 수 있음!

Linus Torvalds의 우아한 삭제:
Node **pp = &head;
pp가 "next 포인터의 주소"를 따라감
→ 특별 케이스(첫 노드) 처리 불필요!
double pointer
이중 포인터. Node **head. head 자체를 수정! 함수에서 포인터 변수를 바꾸려면 그 포인터의 주소 필요.
Linus deletion
Linus 스타일 삭제. Node **pp로 next 포인터 주소 추적. 첫 노드 특별 처리 불필요! 우아!
Node *p = "노드의 주소". Node **pp = "그 주소가 적힌 변수의 주소". 한 단계 더!
Linus 삭제법: pp = &head부터 시작, pp = &(*pp)->next로 이동. *pp = del->next로 끝!
연결 리스트(Linked List)
노드들이 포인터로 연결된 자료구조. 배열과 달리 크기가 유동적이다.
노드(Node)
연결 리스트의 기본 단위. 데이터와 다음 노드의 주소를 저장한다.
head
연결 리스트의 첫 번째 노드를 가리키는 포인터.
NULL
아무것도 가리키지 않는 포인터. 연결 리스트의 끝을 표시한다.
malloc
메모리를 동적으로 할당하는 C 표준 라이브러리 함수.
free
malloc으로 할당한 메모리를 해제하는 함수.
메모리 누수
할당한 메모리를 해제하지 않아 사용하지 않는 메모리가 쌓이는 현상.

5
포인터 연결 순서를 틀리면 노드가 사라진다
