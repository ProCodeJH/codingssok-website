문제 1: 정렬 유형

문제: 성적 순위
N명의 학생의 점수가 주어진다. 점수를 내림차순으로 정렬하여 출력하시오.
입력: 첫 줄에 N(1 <= N <= 1000), 둘째 줄에 N개의 점수
출력: 내림차순으로 정렬된 점수
koi_sort.c
1#include <stdio.h>
2
3int main(void) {
4 int n, arr[1000];
5 int i, j, temp;
6
7 scanf("%d", &n);
8 for (i = 0; i < n; i++)
9 scanf("%d", &arr[i]);
10
11 /* 선택 정렬 (내림차순) */
12 for (i = 0; i < n - 1; i++) {
13 int maxIdx = i;
14 for (j = i + 1; j < n; j++)
15 if (arr[j] > arr[maxIdx])
16 maxIdx = j;
17 temp = arr[i];
18 arr[i] = arr[maxIdx];
19 arr[maxIdx] = temp;
20 }
21
22 for (i = 0; i < n; i++)
23 printf("%d ", arr[i]);
24 printf("\n");
25 return 0;
26}
풀이 해설
N이 최대 1000이므로 O(n^2) 정렬이 충분해요. 선택 정렬로 가장 큰 값을 앞으로 보내는 방식이에요. arr[j] > arr[maxIdx]로 내림차순을 구현했어요.

문제 2: 탐색 유형

문제: 숫자 찾기
N개의 정수가 오름차순으로 정렬되어 있다. 정수 M이 이 중에 있으면 위치(1부터)를, 없으면 -1을 출력하시오.
입력: 첫 줄에 N과 M, 둘째 줄에 N개의 정수
출력: M의 위치 또는 -1
koi_search.c
1#include <stdio.h>
2
3int main(void) {
4 int n, m, arr[100000];
5 int i, low, high, mid, result = -1;
6
7 scanf("%d %d", &n, &m);
8 for (i = 0; i < n; i++)
9 scanf("%d", &arr[i]);
10
11 /* 이진 탐색 */
12 low = 0;
13 high = n - 1;
14 while (low <= high) {
15 mid = (low + high) / 2;
16 if (arr[mid] == m) {
17 result = mid + 1; /* 1부터 시작하는 위치 */
18 break;
19 } else if (arr[mid] < m) {
20 low = mid + 1;
21 } else {
22 high = mid - 1;
23 }
24 }
25
26 printf("%d\n", result);
27 return 0;
28}
풀이 해설
N이 최대 100,000이므로 O(n) 순차 탐색은 가능하지만, 이진 탐색(O(log n))이 훨씬 빨라요. 이미 정렬되어 있으므로 바로 적용 가능해요.

문제 3: 시뮬레이션 유형

문제: 로봇 이동
로봇이 (0,0)에서 시작한다. 명령어 'U'(위), 'D'(아래), 'L'(왼쪽), 'R'(오른쪽)이 주어질 때 최종 위치를 출력하시오.
입력: 첫 줄에 명령어 수 N, 둘째 줄에 N개의 명령어
출력: 최종 x y 좌표
koi_simulation.c
1#include <stdio.h>
2
3int main(void) {

6

12 else if (cmd == 'R') x++;
13 else if (cmd == 'L') x--;

15

18}
풀이 해설
시뮬레이션 문제는 문제의 규칙을 그대로 코드로 옮기는 것이 핵심이에요. 복잡한 알고리즘보다 정확한 조건 분기가 중요해요. scanf(" %%c", ...)에서 공백을 넣어 줄바꿈을 무시하는 것에 주의하세요.
시뮬레이션 문제는 알고리즘을 몰라도 풀 수 있어! 문제가 시키는 대로 코드를 짜면 돼. 그래서 KOI에서 제일 먼저 풀기 좋은 유형이야.
KOI 실전에서는 반례를 항상 생각해! 입력이 0이면? 최대값이면? 음수면? 이런 경계 케이스에서 많이 틀려.

30int max_subarray(int *a, int n) {
31 int max_sum = a[0], cur_sum = a[0];
32 int start = 0, end = 0, temp_start = 0;
33 for (int i = 1; i < n; i++) {
34 if (cur_sum + a[i] < a[i]) { cur_sum = a[i]; temp_start = i; }

DP
Dynamic Programming. 중복 부분 문제를 캐시. 피보나치: 재귀 O(2^n)→DP O(n). 테이블 채우기!
greedy
그리디. 매 순간 최선 선택. 동전 거스름돈, 허프만 코딩. 최적 부분 구조+탐욕 선택 속성 필요.
Kadane
카데인 알고리즘. 최대 부분합 O(n). cur_sum<0이면 리셋. DP의 특수 형태!
그리디 vs DP: 그리디는 "지금 최선", DP는 "전체 최적". 동전은 그리디 OK, 배낭은 DP!
계단 오르기 = 피보나치! dp[n] = dp[n-1] + dp[n-2]. 1칸 or 2칸씩 올라가니까!
[C 코드] KOI — 0-1 배낭 문제 (DP)
1#include <stdio.h>
2
3#define MAX_W 50
4#define MAX_N 10
5
6int dp[MAX_N+1][MAX_W+1];
7
8int knapsack(int weights[], int values[], int n, int W) {
9 for (int i = 0; i <= n; i++) dp[i][0] = 0;
10 for (int w = 0; w <= W; w++) dp[0][w] = 0;
11
12 for (int i = 1; i <= n; i++) {
13 for (int w = 1; w <= W; w++) {
14 dp[i][w] = dp[i-1][w]; // 안 넣기
15 if (weights[i-1] <= w) {
16 int with = dp[i-1][w-weights[i-1]] + values[i-1];
17 if (with > dp[i][w]) dp[i][w] = with; // 넣기
18 }
19 }
20 }
21 return dp[n][W];
22}
23
24void trace_items(int weights[], int n, int W) {
25 printf("Selected items: ");
26 int w = W;
27 for (int i = n; i > 0; i--) {
28 if (dp[i][w] != dp[i-1][w]) {
29 printf("#%d(w=%d,v=%d) ", i, weights[i-1], dp[i][w]-dp[i-1][w-weights[i-1]]);
30 w -= weights[i-1];
31 }
32 }
33 printf("\n");
34}

knapsack
배낭 문제. 무게 제한 내 최대 가치. NP-Hard! DP로 의사 다항시간. 자원 배분의 기본 모델.
배낭 DP: dp[i][w] = i번째까지 물건 중 무게 w에서 최대 가치. 테이블 채우기!
[심화 내용] 알고리즘 문제 해결 전략
문제 분석 순서:
1. 입력 크기 확인 → n이 뭔지
2. 시간 제한 역산: 1초 ≈ 10⁸ 연산
n ≤ 10: O(n!), O(2ⁿ) 가능
n ≤ 1000: O(n²) 가능
n ≤ 10⁵: O(n log n) 필요
n ≤ 10⁷: O(n) 필요
3. 패턴 매칭: DP? 그리디? 그래프?

문제 유형별 접근:
최적화 → DP or 그리디
탐색/순회 → BFS/DFS
정렬 관련 → 정렬+이진탐색
문자열 → 해싱, KMP, 트라이
그래프 → BFS, 다익스트라, MST
competitive programming
경쟁 프로그래밍. ICPC, IOI, Codeforces. 시간 내 정확한 풀이. 알고리즘+자료구조+구현력!
n≤1000이면 O(n²) OK! n≤10⁵이면 O(n log n) 필요. 시간 제한으로 알고리즘 선택!
LIS
최장 증가 부분수열. DP: O(n²). 이진 탐색+DP: O(n log n). 면접 단골 문제!
LCS
최장 공통 부분수열. DP 테이블 O(nm). diff, git merge의 기반 알고리즘!
edit distance
편집 거리. 삽입/삭제/교체 최소 횟수. DP O(nm). 스펠 체크, DNA 비교에 활용!
DP 테이블 읽기: dp[i][j]가 뭘 의미하는지 정의부터! 정의가 명확하면 점화식은 자동!
[C 코드] KOI — 배낭 문제 (DP)
1#include <stdio.h>
2
3// 0-1 배낭 문제
4int knapsack(int *weights, int *values, int n, int capacity) {
5 int dp[101] = {0}; // dp[w] = 무게 w일 때 최대 가치
6 for (int i = 0; i < n; i++)
7 for (int w = capacity; w >= weights[i]; w--)
8 if (dp[w-weights[i]] + values[i] > dp[w])
9 dp[w] = dp[w-weights[i]] + values[i];
10 return dp[capacity];
11}

14int lis(int *a, int n) {
15 int dp[100];
16 int max_len = 0;
17 for (int i = 0; i < n; i++) {
18 dp[i] = 1;
19 for (int j = 0; j < i; j++)
20 if (a[j] < a[i] && dp[j]+1 > dp[i])
21 dp[i] = dp[j] + 1;
22 if (dp[i] > max_len) max_len = dp[i];
23 }
24 return max_len;

27// 편집 거리
28int edit_distance(const char *a, const char *b) {
29 int m=0,n=0;
30 while(a[m]) m++; while(b[n]) n++;
31 int dp[51][51];
32 for(int i=0;i<=m;i++) dp[i][0]=i;
33 for(int j=0;j<=n;j++) dp[0][j]=j;
34 for(int i=1;i<=m;i++)
35 for(int j=1;j<=n;j++) {
36 if(a[i-1]==b[j-1]) dp[i][j]=dp[i-1][j-1];
37 else {
38 int ins=dp[i][j-1], del=dp[i-1][j], rep=dp[i-1][j-1];
39 dp[i][j]=1+(ins<del?(ins<rep?ins:rep):(del<rep?del:rep));
40 }
41 }
42 return dp[m][n];
43}

knapsack
배낭 문제. 무게 제한 내 최대 가치. DP: dp[w]=max(dp[w], dp[w-wi]+vi). NP-완전의 대표!
LIS
최장 증가 부분수열. O(n²) DP, O(n log n) 이진탐색. 코딩테스트 빈출! patience sorting과 동치.
edit distance
편집 거리. 삽입/삭제/치환 최소 횟수. 스펠체크, DNA 비교, 문서 diff에 활용!
배낭 문제 = DP의 정석! dp[w]에 무게 w일 때 최대 가치 저장. 역순 순회가 핵심!
memoization
메모이제이션. 계산 결과 캐싱. fib(5) 한번 계산→저장→재사용. DP의 Top-down 방식!
backtracking
백트래킹. 가능한 경로 탐색+가지치기. N-Queen, 스도쿠. DFS+조건 불만족 시 되돌아감.
DP 핵심: "이 문제를 작은 문제로 쪼갤 수 있는가?" 가능하면 DP, 불가능하면 그리디/백트래킹!
편집 거리 = 스펠체크의 핵심! "kitten"→"sitting" 3번 변환. 유사도 측정에 필수!
[C 코드] KOI — N-Queen 백트래킹
1#include <stdio.h>
2#include <math.h>
3
4int board[20];
5int count = 0;
6
7int is_safe(int row, int col) {
8 for (int i = 0; i < row; i++)
9 if (board[i]==col || abs(board[i]-col)==abs(i-row)) return 0;
10 return 1;
11}
12
13void solve(int row, int n) {
14 if (row == n) { count++; return; }
15 for (int col = 0; col < n; col++)
16 if (is_safe(row, col)) {
17 board[row] = col;
18 solve(row+1, n);
19 }
20}
21
22int main() {
23 for (int n = 4; n <= 10; n++) {
24 count = 0;
25 solve(0, n);
26 printf("%d-Queen: %d solutions\n", n, count);
27 }
28 return 0;
29}

[C 코드] KOI — 0-1 배낭 문제 (DP)
1#include <stdio.h>
2
3#define MAX_W 100
4
5typedef struct { int weight, value; const char *name; } Item;
6
7int max(int a, int b) { return a > b ? a : b; }
8
9int knapsack(Item *items, int n, int capacity) {
10 int dp[MAX_W+1] = {0};
11
12 for (int i = 0; i < n; i++) {
13 // 뒤에서부터! (각 아이템 1번만 사용)
14 for (int w = capacity; w >= items[i].weight; w--) {
15 dp[w] = max(dp[w], dp[w - items[i].weight] + items[i].value);
16 }
17 }
18 return dp[capacity];
19}

32 }

[심화 내용] DP의 핵심 원리
DP의 조건:
1. 최적 부분 구조: 큰 문제의 최적해가 작은 문제의 최적해로 구성
2. 중복 부분 문제: 같은 작은 문제가 반복 등장

Top-down vs Bottom-up:
Top-down: 재귀+메모이제이션
Bottom-up: 반복문+테이블 채우기
→ 같은 결과, 다른 구현!

배낭 문제의 DP 점화식:
dp[w] = max(dp[w], dp[w-weight[i]] + value[i])
"이 물건을 넣을까 말까?"
넣으면: 남은 용량에서의 최적 + 이 물건 가치
안 넣으면: 기존 최적 유지
knapsack
배낭 문제. 무게 제한 내 최대 가치. DP로 O(nW). NP-hard지만 의사다항시간 풀이 존재!
LIS
최장 증가 부분수열. DP O(n²), 이진탐색 O(n log n). 패턴 매칭, 주가 분석에 활용!
DP = "큰 문제 = 작은 문제의 조합!" 배낭: dp[w]=넣을까 말까의 최적. 점화식이 핵심!
시뮬레이션
문제에서 주어진 규칙을 그대로 코드로 구현하여 결과를 구하는 유형.
이진 탐색
정렬된 데이터에서 중간값과 비교하며 범위를 반씩 줄여 나가는 탐색 방법.
선택 정렬
매 단계마다 최소(또는 최대) 값을 찾아 앞으로 보내는 정렬 알고리즘. O(n^2).
반례
프로그램이 틀리는 특수한 입력. 경계값(0, 최대값 등)에서 자주 발생한다.
격자
2차원 배열로 표현되는 바둑판 모양의 공간. 시뮬레이션 문제에서 자주 등장.
