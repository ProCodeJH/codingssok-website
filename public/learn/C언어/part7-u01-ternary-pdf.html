삼항 연산자

if-else를 한 줄로 줄이는 삼항 연산자를 배워요. 조건 ? 참 : 거짓 형태를 익히고, 실전에서 활용해요.

📚 이번에 배울 것

  • 삼항 연산자의 문법과 동작을 이해해요.
  • if-else 문을 삼항 연산자로 바꿀 수 있어요.
  • 중첩 삼항 연산자를 읽고 쓸 수 있어요.
  • 삼항 연산자로 최솟값, 최댓값, 절대값을 구할 수 있어요.

✅ 시작 전 체크

  • if-else 조건문을 사용할 수 있다
  • 변수와 대입 연산자를 이해하고 있다
Part 1 진행률: 1%

if-else를 한 줄로 줄일 수 없을까?

if-else로 조건에 따라 다른 값을 넣는 코드, 많이 써 봤지? 그런데 이렇게 간단한 경우에도 4줄이나 쓰는 건 좀 아깝지 않아? 삼항 연산자를 쓰면 딱 한 줄이면 돼!

삼항 연산자(ternary operator)는 C언어에서 유일하게 피연산자가 3개인 연산자예요. 조건을 검사해서 참이면 하나의 값을, 거짓이면 다른 값을 돌려줘요.

undefined

삼항 연산자 기본 사용법

삼항 연산자의 형태는 조건식 ? 참일_때_값 : 거짓일_때_값이에요. if-else와 완전히 같은 동작을 하지만, 값을 바로 돌려준다는 점이 달라요.

ternary_basic.c
1#include <stdio.h>
2
3int main(void) {
4 int score = 85;
5
6 /* if-else 방식 */
7 char *result1;
8 if (score >= 60) {
9 result1 = "합격";
10 } else {
11 result1 = "불합격";
12 }
13
14 /* 삼항 연산자 방식 */
15 char *result2 = (score >= 60) ? "합격" : "불합격";
16
17 printf("if-else: %s\n", result1);
18 printf("삼항: %s\n", result2);
19 return 0;
20}
실행 결과
if-else: 합격 삼항: 합격
15줄(score >= 60) ? "합격" : "불합격" -- score가 60 이상이면 "합격"을, 아니면 "불합격"을 돌려줘요.
15줄삼항 연산자의 결과를 바로 변수에 대입할 수 있어요. if-else 5줄이 1줄로 줄었어요!
삼항 연산자는 값을 선택하는 연산자야. if-else는 문장이고, 삼항 연산자는 식(expression)이라서 printf 안에서도 바로 쓸 수 있어!

삼항 연산자 실전 활용

삼항 연산자는 절대값 구하기, 최솟값/최댓값 구하기 같은 간단한 조건 분기에 딱 맞아요.

ternary_practical.c
1#include <stdio.h>
2
3int main(void) {
4 int a = -7, b = 3;
5
6 /* 절대값 */
7 int abs_a = (a >= 0) ? a : -a;
8 printf("|%d| = %d\n", a, abs_a);
9
10 /* 최솟값 */
11 int min = (a < b) ? a : b;
12 printf("min(%d, %d) = %d\n", a, b, min);
13
14 /* 최댓값 */
15 int max = (a > b) ? a : b;
16 printf("max(%d, %d) = %d\n", a, b, max);
17
18 /* 홀짝 판별 */
19 printf("%d는 %s\n", b, (b % 2 == 0) ? "짝수" : "홀수");
20
21 return 0;
22}
실행 결과
|-7| = 7 min(-7, 3) = -7 max(-7, 3) = 3 3는 홀수

중첩 삼항 연산자

삼항 연산자 안에 또 삼항 연산자를 넣을 수 있어요. 하지만 중첩이 많아지면 읽기 어려워지니까 주의해야 해요.

ternary_nested.c
1#include <stdio.h>
2
3int main(void) {
4 int score = 75;
5
6 /* 중첩 삼항: 등급 판정 */
7 char grade = (score >= 90) ? 'A'
8 : (score >= 80) ? 'B'
9 : (score >= 70) ? 'C'
10 : 'F';
11
12 printf("점수: %d -> 등급: %c\n", score, grade);
13 return 0;
14}
실행 결과
점수: 75 -> 등급: C
undefined
중첩 삼항은 2단계까지만 쓰는 게 좋아. 3단계 이상은 if-else가 훨씬 읽기 쉬워! KOI 시험에서도 가독성이 중요해.
[KOI] 삼항 연산자로 코드 줄이기
KOI에서는 제한 시간 안에 빠르게 코드를 작성해야 해요. 간단한 최솟값, 최댓값을 구할 때 삼항 연산자를 쓰면 코드가 짧아져서 실수할 확률도 줄어요.
삼항 연산자 핵심 정리
1. 문법 -- 조건 ? 참값 : 거짓값
2. 특징 -- 식(expression)이라서 값을 바로 돌려줌
3. 활용 -- 절대값, 최솟값, 최댓값, 홀짝 판별
4. 주의 -- 중첩은 2단계까지만. 복잡하면 if-else 사용
🔧 바꿔보기

ternary_practical.c에서 세 수 a, b, c 중 가장 큰 값을 삼항 연산자로 구해 보세요.

삼항 연산자를 중첩하면 돼요: (a > b) ? ((a > c) ? a : c) : ((b > c) ? b : c)

연습문제

OX 퀴즈

삼항 연산자의 결과를 변수에 대입할 수 있다.

정답: ⭕ O
삼항 연산자는 식(expression)이므로 결과를 변수에 대입하거나 printf에 바로 넣을 수 있어요.
객관식

다음 코드의 출력 결과는? int x = 10; printf("%d", (x > 5) ? x * 2 : x + 1);

  1. 10
  2. 20
  3. 11
  4. 5
정답: 2
x가 10이고 10 > 5는 참이므로 x * 2 = 20이 출력돼요.
단답형

삼항 연산자에서 물음표(?) 앞에 오는 것은 무엇인가요?

정답: 조건식
삼항 연산자는 '조건식 ? 참값 : 거짓값' 구조예요. 물음표 앞에 조건식이 와요.
객관식

다음 중 삼항 연산자를 올바르게 사용한 것은?

  1. int r = (a > b) ? a; b;
  2. int r = (a > b) : a ? b;
  3. int r = (a > b) ? a : b;
  4. int r = a > b ? : a b;
정답: 3
삼항 연산자는 조건 ? 참값 : 거짓값 형태예요. 3번이 올바른 문법이에요.
OX 퀴즈

삼항 연산자는 if-else문보다 항상 실행 속도가 빠르다.

정답: ❌ X
삼항 연산자와 if-else문은 컴파일러 최적화 후 동일한 성능을 보입니다. 가독성과 상황에 맞게 선택하는 것이 중요합니다.

쉽게 이해하기: 삼항 연산자
삼항 = "미니 if-else"

if (a > b) result = a;
else result = b;

↓ 한 줄로!

result = a > b ? a : b;

읽는 법:
"a가 b보다 크면(?) a, 아니면(:) b"

⚠️ 주의:
• 중첩은 2단계까지만! 그 이상은 if-else가 낫다
• 부수 효과 있는 코드에 쓰지 말기 (x++ 등)

cmov
조건부 이동 명령. 삼항→cmov로 컴파일. 분기 없이 조건 실행! 분기 예측 실패 방지.
branch prediction
분기 예측. CPU가 if 결과를 미리 예측. 틀리면 파이프라인 플러시! 삼항=cmov로 회피 가능.
삼항 = if-else의 축약! a>b?a:b 한 줄. 간단한 조건에 딱! 복잡하면 if-else 쓰자.
clamp(x,lo,hi) = xhi?hi:x. 삼항 중첩이지만 패턴이 명확하면 OK!

[심화 내용] 삼항과 컴파일러 최적화
삼항 → cmov (조건부 이동):
if-else는 분기 명령(jmp) 생성
삼항은 cmov 명령으로 컴파일 가능!

왜 cmov가 빠른가:
CPU 파이프라인에서 분기 예측 실패 → 20 사이클 손실
cmov는 분기 없음 → 항상 일정한 속도

언제 cmov가 안 되나:
• 부수 효과가 있는 표현식
• 복잡한 연산 (양쪽 다 실행됨)
• 컴파일러가 판단: -O2에서 자동 선택

매크로 주의:
MAX(a++,b) → a가 2번 증가할 수 있음!
→ ({int _a=a,_b=b; _a>_b?_a:_b;}) GCC 확장

cmov
Conditional Move. x86 명령어. 분기 없이 조건부 대입. 파이프라인 스톨 방지! 삼항→cmov.
branch prediction
분기 예측. CPU가 if 결과 예측. 맞으면 0 페널티, 틀리면 15~20 사이클 낭비!
lookup table
룩업 테이블. 배열[인덱스]로 O(1) 변환. switch/if 체인 대체! 성능+가독성 동시 향상.
매크로 MAX(a++,b) 함정! a가 2번 증가! 인자가 2번 평가되니까. GCC 확장 or inline 함수로!

5void sort3(int *a, int *b, int *c) {
6 int t;
7 t = *a > *b ? (*a^=*b, *b^=*a, *a^=*b, 0) : 0; (void)t;
8 t = *b > *c ? (*b^=*c, *c^=*b, *b^=*c, 0) : 0; (void)t;
9 t = *a > *b ? (*a^=*b, *b^=*a, *a^=*b, 0) : 0; (void)t;
10}

[심화] 삼항 실전 정리
1. 룩업 테이블 대체:
code==200?"OK":code==404?"Not Found":...
→ switch보다 짧지만, 5개 이상이면 배열이 나음

2. 단위 변환:
bytes>=1GB ? "GB" : bytes>=1MB ? "MB" : ...
→ 큰 단위부터 체크!

3. 보간(Lerp):
result = a + (b-a)*t (t=0~1)
clamp: t<0?0:t>1?1:t
→ 색상, 위치, 크기 보간에 필수

4. 방향 매트릭스:
dx,dy 조합으로 8방향+정지
중첩 삼항이지만 패턴이 규칙적!
sorting network
정렬 네트워크. 고정 비교 횟수로 정렬. 3개=3비교, 4개=5비교. 하드웨어 정렬에 사용!
lerp
선형 보간. a+(b-a)*t. t=0→a, t=1→b. 색상, 위치, 애니메이션의 기본! 게임 엔진 필수.
format_bytes: GB→MB→KB→B 순서로 체크! 삼항 중첩이지만 패턴이 명확하면 가독성 OK!
8방향 판별: dx,dy 조합 9가지. 삼항 중첩 3×3으로 깔끔! 게임에서 이동 방향 표시에 활용!
comma operator
쉼표 연산자. (a,b)에서 a 실행 후 b 반환. 삼항과 결합: (check(),val). for문 초기화에 활용!
conditional expression
조건 표현식. ISO C에서 삼항의 공식 명칭. 유일한 3항(ternary) 연산자. 좌값 불가(C에서).
null coalescing
널 병합. C에는 없지만 삼항으로 구현: ptr?ptr:default. C#/??, JS/??와 같은 패턴!
Elvis operator
엘비스 연산자. a?:b (Groovy/Kotlin). a가 참이면 a, 아니면 b. 삼항의 축약! C에서는 GCC 확장.
삼항의 결과는 좌값(lvalue)이 아님! (a>b?a:b)=10 불가. 대입 대상이 될 수 없어!
ternary in Python
파이썬 삼항: a if cond else b. C와 순서 다름! C: cond?a:b. 같은 개념, 다른 문법.
nullish coalescing
널 병합. JS: a??b. C에는 없지만 삼항으로: ptr?ptr:def. 모던 언어의 표준 기능!
삼항은 표현식! 값을 반환한다. if-else는 문장(statement). 이 차이가 핵심!
branchless sorting
분기 없는 정렬. 삼항+swap으로 소팅 네트워크. 3개 정렬=3비교. GPU/SIMD에서 핵심!
ternary type rules
삼항 타입 규칙. int와 double 혼합 시 double로 승격. 1?1:2.0의 결과는 double! ISO C 6.5.15.
GCC statement expr
GCC 문장 표현식. ({int a=f(); a*a;}). 블록을 표현식으로! 삼항 대안. 표준은 아님!
삼항에서 1?1:2.0은 double이 됨! 양쪽 타입이 다르면 더 큰 타입으로 승격!
short circuit ternary
삼항도 단축 평가! a?b:c에서 a가 참이면 c 실행 안 함. 부수 효과 있을 때 중요!
nested ternary limit
중첩 삼항 한계. 2~3단 OK, 4단 이상은 if-else. 리눅스 코딩 스타일: 중첩 자제 권장!
삼항 연산자 vs switch
삼항 vs switch 비교:
3개 이하 분기 → 삼항이 깔끔
4개 이상 분기 → switch가 가독성 좋음
10개 이상 → 배열 룩업 테이블!

성능:
삼항: cmov 명령 가능 (분기 없음)
switch: 점프 테이블 (O(1))
배열: 인덱싱 (O(1), 캐시 친화)
ternary in macros
매크로에서 삼항 필수. #define MAX(a,b)((a)>(b)?(a):(b)). 괄호 빠뜨리면 우선순위 버그!
nested ternary limit
중첩 삼항 2단계까지! 3단계 이상은 if-else. Linux 커널 코딩 스타일 가이드 권장.
ternary in assembly
삼항→어셈블리. gcc -O2: cmov 생성. 분기 없이 조건부 이동! 파이프라인 스톨 0. 성능 핵심.
nested ternary limit
삼항 중첩 제한. 2~3단계까지만! 그 이상은 switch/if-else 또는 배열 룩업이 가독성 우수.
ternary type rules
삼항 타입 규칙. 참/거짓 표현식의 타입이 다르면 암묵적 변환! int?char→int. 주의!
삼항의 타입: 3>2?"yes":0 → char*과 int 혼합! 컴파일러 경고. 같은 타입으로 맞추자!
GCC statement expression
GCC 문장 표현식. ({int t=a;a=b;b=t;t;}). 블록을 표현식으로! 안전한 MAX 매크로 구현.
({int _a=(a),_b=(b);_a>_b?_a:_b;}) GCC 확장! 인자 2번 평가 방지. 안전한 MAX!
삼항 연산자(ternary operator)
피연산자 3개를 사용하는 연산자. 조건 ? 참값 : 거짓값 형태로, 조건에 따라 두 값 중 하나를 선택한다.
식(expression)
값을 만들어 내는 코드 조각. 삼항 연산자는 식이므로 대입이나 함수 인자로 쓸 수 있다.
문(statement)
실행되는 코드 단위. if-else는 문이라서 값을 직접 돌려주지 않는다.
중첩 삼항
삼항 연산자의 참값이나 거짓값 자리에 또 다른 삼항 연산자를 넣는 것.
