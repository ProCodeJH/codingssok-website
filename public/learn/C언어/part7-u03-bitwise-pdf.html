비트(bit)란 무엇인가?

컴퓨터는 모든 것을 0과 1로 저장해. 이 0과 1 하나하나를 비트(bit)라고 해. 비트 연산자는 이 0과 1을 직접 조작하는 도구야!

정수 13을 2진수로 쓰면 00001101이에요. 비트 연산자는 이 2진수의 각 자릿수에 대해 연산을 수행해요. 논리 연산자(&&, ||)가 전체 값의 참/거짓을 판단한다면, 비트 연산자(&, |)는 각 비트를 개별적으로 처리해요.

AND, OR, XOR, NOT

bitwise_basic.c
1#include <stdio.h>
2
3int main(void) {
4 unsigned char a = 13; /* 00001101 */
5 unsigned char b = 10; /* 00001010 */
6
7 printf("a & b = %d\n", a & b); /* AND: 00001000 = 8 */
8 printf("a | b = %d\n", a | b); /* OR: 00001111 = 15 */
9 printf("a ^ b = %d\n", a ^ b); /* XOR: 00000111 = 7 */
10 printf("~a = %d\n", (unsigned char)~a); /* NOT: 11110010 = 242 */
11 return 0;
12}
실행 결과
a & b = 8 a | b = 15 a ^ b = 7 ~a = 242
7줄a & b -- AND 연산. 두 비트가 모두 1일 때만 결과가 1이에요.
8줄a | b -- OR 연산. 두 비트 중 하나라도 1이면 결과가 1이에요.
9줄a ^ b -- XOR 연산. 두 비트가 서로 다르면 1, 같으면 0이에요.
10줄~a -- NOT 연산. 모든 비트를 뒤집어요. 0은 1로, 1은 0으로.

시프트 연산자

시프트 연산자는 비트를 왼쪽이나 오른쪽으로 밀어요. 왼쪽 시프트(<<)는 곱하기 2, 오른쪽 시프트(>>)는 나누기 2와 같은 효과가 있어요.

bitwise_shift.c
1#include <stdio.h>
2
3int main(void) {
4 int x = 5; /* 00000101 */
5
6 printf("%d << 1 = %d\n", x, x << 1); /* 00001010 = 10 */
7 printf("%d << 2 = %d\n", x, x << 2); /* 00010100 = 20 */
8 printf("%d >> 1 = %d\n", x, x >> 1); /* 00000010 = 2 */
9
10 /* 2의 거듭제곱 만들기 */
11 printf("1 << 0 = %d\n", 1 << 0); /* 1 */
12 printf("1 << 3 = %d\n", 1 << 3); /* 8 */
13 printf("1 << 10 = %d\n", 1 << 10); /* 1024 */
14 return 0;
15}
실행 결과
5 << 1 = 10 5 << 2 = 20 5 >> 1 = 2 1 << 0 = 1 1 << 3 = 8 1 << 10 = 1024
x << n은 x에 2의 n제곱을 곱하는 것과 같아! x >> n은 2의 n제곱으로 나누는 것과 같고. 컴퓨터에서 곱셈보다 시프트가 훨씬 빨라!

비트 연산에서 &와 &&를 헷갈리면 큰일나! &는 비트 AND이고, &&는 논리 AND야. 비트 연산은 각 비트를, 논리 연산은 전체 값의 참/거짓을 다뤄.
[하드웨어] 비트 연산과 레지스터
CPU의 레지스터는 비트의 모음이에요. 하드웨어를 직접 제어할 때는 특정 비트를 켜거나 끄는 것이 핵심이에요. 비트 연산은 하드웨어 프로그래밍의 기본이에요.
비트 연산자 핵심 정리
1. & (AND) -- 둘 다 1이면 1
2. | (OR) -- 하나라도 1이면 1
3. ^ (XOR) -- 서로 다르면 1
4. ~ (NOT) -- 비트 뒤집기
5. << -- 왼쪽 시프트 (x2)
6. >> -- 오른쪽 시프트 (/2)

연습문제

OX 퀴즈

n & 1의 결과가 1이면 n은 홀수이다.

정답: ⭕ O
2진수에서 마지막 비트가 1이면 홀수예요. n & 1은 마지막 비트만 확인하는 연산이에요.
객관식

5 << 2의 결과는?

  1. 10
  2. 20
  3. 7
  4. 2
정답: 2
왼쪽 시프트 2는 곱하기 4와 같아요. 5 * 4 = 20이에요.
단답형

비트 OR 연산자의 기호는 무엇인가요?

정답: |
비트 OR 연산자는 |, 논리 OR 연산자는 ||이에요.
객관식

12 & 10의 결과는? (12=1100, 10=1010)

  1. 14
  2. 8
  3. 6
  4. 2
정답: 2
1100 & 1010 = 1000 = 8이에요. 두 비트가 모두 1인 자리만 1이 돼요.
객관식

0xFF & 0x0F의 결과는?

  1. 0xFF
  2. 0x0F
  3. 0xF0
  4. 0x00
정답: 1
0xFF(11111111)와 0x0F(00001111)를 AND 연산하면 0x0F(00001111)가 됩니다.

쉽게 이해하기: 비트 연산
비트 = 0 또는 1, 컴퓨터의 최소 단위

&(AND): 둘 다 1이면 1
1010 & 1100 = 1000

|(OR): 하나라도 1이면 1
1010 | 1100 = 1110

^(XOR): 다르면 1
1010 ^ 1100 = 0110

~(NOT): 뒤집기
~1010 = 0101

<<(왼쪽 시프트): ×2
>>(오른쪽 시프트): ÷2

n & (n-1) == 0 → 2의 거듭제곱!
8 = 1000, 7 = 0111, 1000 & 0111 = 0000 ✅

[심화] 비트 연산 실전 패턴 정리
1. 플래그 관리:
켜기: flags |= FLAG
끄기: flags &= ~FLAG
토글: flags ^= FLAG
확인: (flags & FLAG) != 0

2. 대소문자 변환:
'A'=0x41=01000001
'a'=0x61=01100001
→ 비트 5(32)만 다름!
소문자: c | 32
대문자: c & ~32
토글: c ^ 32

3. IP/색상 패킹:
4바이트를 int 하나에: (a<<24)|(b<<16)|(c<<8)|d
추출: (val>>shift) & 0xFF
subnet mask
서브넷 마스크. IP & mask = 네트워크 주소. /24 = 255.255.255.0. 같은 네트워크인지 판별!
bit 5 trick
비트5(32) 트릭. A(65)와 a(97)의 차이=32=비트5. XOR 32로 대소문자 토글! ASCII의 숨은 설계.
chmod 755 = rwxr-xr-x. 7=111(rwx), 5=101(r-x). 비트 연산이 파일 권한의 핵심!

CRC
순환 중복 검사. 다항식 나눗셈 나머지. 이더넷, USB, ZIP에 사용! 전송 오류 검출의 표준.
Morton code
Z-order curve. 2D→1D 비트 인터리빙. 공간 지역성 보존! 쿼드트리, 공간 DB에 활용.
FNV-1a: h^=byte; h*=prime; 2줄로 좋은 해시! XOR로 섞고 곱셈으로 퍼뜨리기!
Morton: (3,5)→0b00110101. x,y 비트를 번갈아 배치! 2D를 1D로 공간 보존 변환!
endianness

bit manipulation tricks

SIMD
Single Instruction Multiple Data. 비트 병렬 처리. SSE/AVX. 4개 float를 한 번에 계산! 이미지/비디오 필수.
bitmask iteration
비트마스크 순회. for(m=s;m;m=(m-1)&s) 부분집합 열거. 경쟁 프로그래밍의 비밀 무기!

[심화] 비트 연산 성능 비교
비트 vs 일반 연산 속도: n * 2 vs n << 1 → 컴파일러가 자동 변환! n / 4 vs n >> 2 → unsigned만 자동 변환 n % 8 vs n & 7 → 2의 거듭제곱만 가능 현대 컴파일러(-O2): 대부분 자동 최적화 → 가독성 우선! 수동 비트 트릭은 컴파일러가 못하는 경우만 비트가 유리한 경우: 1. 플래그/권한 관리 (비트 필드) 2. 집합 연산 (비트 집합 O(1)) 3. 그래픽/색상 (패킹/언패킹) 4. 네트워크 (IP/프로토콜 헤더) 5. 암호학 (XOR, 시프트) 6. 임베디드 (레지스터 제어) 가독성 규칙: 비트 연산은 의도를 주석으로 명시! // Extract red channel: (color >> 16) & 0xFF
compiler auto-optimize
컴파일러 자동 최적화. n*2→n<<1, n/4→n>>2. -O2면 자동! 수동 비트 트릭보다 가독성 우선.
bit trick collection
비트 트릭 모음. swap: a^=b^=a^=b. abs: (x^(x>>31))-(x>>31). sign: 1|(x>>31). 면접 단골!
byte order mark
BOM. UTF-8: EF BB BF. 엔디안 표시. UTF-16: FF FE(리틀)/FE FF(빅). 파일 인코딩 감지!
x&7 = x%8! 2의 거듭제곱 나머지는 비트 AND로! x&(n-1) = x%n (n이 2의 거듭제곱일 때)
bit scan
비트 스캔. 최하위/최상위 1비트 위치 찾기. __builtin_ctz/clz. 하드웨어 명령 BSF/BSR!
bitwise NOT trap
비트 NOT 함정. ~0 = -1 (signed). ~0u = UINT_MAX. 부호에 따라 결과 다름! 마스크에 주의.
rotate bits
비트 회전. 좌회전: (x<>(32-n)). 시프트와 다르게 비트 손실 없음! 암호학 필수.
arithmetic shift
산술 시프트. >>는 부호 비트 유지(산술) vs 0 채움(논리). signed에서 -8>>1=-4. 구현 정의!
비트 회전 vs 시프트: 시프트는 밀려나면 소멸. 회전은 반대편으로! 암호학의 기본 연산!
bitfield portability
비트 필드 이식성. 비트 순서, 패딩이 컴파일러마다 다름! 네트워크 프로토콜에 쓰지 마!
XOR cipher
XOR 암호. data^key=cipher. cipher^key=data. 간단하지만 키 재사용 취약! OTP(일회용 패드)는 완벽!
XOR 암호: 같은 키로 두 번 XOR하면 원문 복원! 대칭 암호의 기본 원리. OTP=이론적 완벽!
비트 시프트 주의사항
시프트 주의:
음수 시프트: 정의되지 않은 동작!
비트폭 이상 시프트: UB!
int x = 1 << 32; // UB! (int=32비트)
int y = 1 << -1; // UB!

arithmetic shift
산술 시프트. >>가 부호 비트를 유지. -8>>1=-4. 구현 정의! unsigned 우시프트는 항상 논리(0채움).
bit field portability
비트 필드 이식성. 비트 순서, 패딩이 컴파일러마다 다름! 네트워크 프로토콜에선 비트 시프트 사용.
power of 2 alignment
2의 거듭제곱 정렬. (addr+align-1)&~(align-1). 메모리 정렬에 필수! malloc 내부에서 사용.
arithmetic vs logical shift
산술 vs 논리 시프트. >>는 signed에서 부호 확장(MSB 복사). unsigned는 0 채움. 차이 중요!
메모리 정렬: (addr+7)&~7로 8바이트 정렬! CPU가 정렬된 주소를 더 빠르게 접근!
비트(bit)
0 또는 1의 값을 가지는 데이터의 최소 단위. Binary digit의 줄임말이다.
비트 연산자
정수의 각 비트에 대해 연산을 수행하는 연산자. &, |, ^, ~, <<, >> 등이 있다.
시프트 연산
비트를 왼쪽(<<)이나 오른쪽(>>)으로 이동시키는 연산. 곱하기 2, 나누기 2 효과가 있다.
비트 플래그(bit flag)
하나의 정수에서 각 비트를 독립적인 참/거짓 값으로 사용하는 기법.
