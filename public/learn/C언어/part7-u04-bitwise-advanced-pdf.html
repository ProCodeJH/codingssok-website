비트마스크로 집합 표현하기

정수 하나로 집합을 표현할 수 있다면 믿을 수 있겠어? 예를 들어 {0, 2, 3}이라는 집합은 2진수 1101, 즉 정수 13으로 표현할 수 있어!

비트마스크(bitmask)란 정수의 각 비트를 원소의 포함 여부로 사용하는 기법이에요. i번째 비트가 1이면 원소 i가 집합에 포함된 것이에요.

undefined
bitmask_set.c
1#include <stdio.h>
2
3void printSet(int mask, int n) {
4 printf("{ ");
5 for (int i = 0; i < n; i++) {
6 if (mask & (1 << i)) {
7 printf("%d ", i);
8 }
9 }
10 printf("}\n");
11}

13int main(void) {
14 int A = (1 << 0) | (1 << 2) | (1 << 3); /* {0, 2, 3} = 1101 = 13 */
15 int B = (1 << 1) | (1 << 2); /* {1, 2} = 0110 = 6 */
16
17 printf("A = "); printSet(A, 4);
18 printf("B = "); printSet(B, 4);
19
20 /* 합집합: OR */
21 printf("A | B = "); printSet(A | B, 4);
22
23 /* 교집합: AND */
24 printf("A & B = "); printSet(A & B, 4);
25
26 /* 차집합: A & ~B */
27 printf("A - B = "); printSet(A & ~B, 4);
28
29 /* 원소 추가: OR */
30 A |= (1 << 1); /* 원소 1 추가 */
31 printf("A에 1 추가 = "); printSet(A, 4);
32
33 /* 원소 제거: AND NOT */
34 A &= ~(1 << 0); /* 원소 0 제거 */
35 printf("A에서 0 제거 = "); printSet(A, 4);
36
37 return 0;
38}

A = { 0 2 3 } B = { 1 2 } A | B = { 0 1 2 3 } A & B = { 2 } A - B = { 0 3 } A에 1 추가 = { 0 1 2 3 } A에서 0 제거 = { 1 2 3 }

XOR의 신기한 성질

XOR에는 재미있는 성질이 있어요. 같은 값을 두 번 XOR하면 원래 값으로 돌아가요. 이 성질을 이용하면 임시 변수 없이 두 수를 교환하거나, 간단한 암호화를 할 수 있어요.

xor_tricks.c
1#include <stdio.h>
2
3int main(void) {
4 /* XOR 스왑 (임시 변수 없이 교환) */
5 int a = 5, b = 3;
6 printf("교환 전: a=%d, b=%d\n", a, b);
7
8 a ^= b; /* a = 5^3 = 6 */
9 b ^= a; /* b = 3^6 = 5 */
10 a ^= b; /* a = 6^5 = 3 */
11 printf("교환 후: a=%d, b=%d\n", a, b);
12
13 /* XOR 암호화/복호화 */
14 char msg[] = "Hello";
15 char key = 'K';
16
17 /* 암호화 */
18 for (int i = 0; msg[i] != '\0'; i++) {
19 msg[i] ^= key;
20 }
21 printf("암호화: %s\n", msg);
22
23 /* 복호화 (같은 키로 다시 XOR) */
24 for (int i = 0; msg[i] != '\0'; i++) {
25 msg[i] ^= key;
26 }
27 printf("복호화: %s\n", msg);
28
29 return 0;
30}
실행 결과
교환 전: a=5, b=3 교환 후: a=3, b=5 암호화: (암호문) 복호화: Hello
XOR 스왑은 신기하지만, 실전에서는 임시 변수를 쓰는 게 더 안전해. a와 b가 같은 변수를 가리키면 값이 0이 되어 버리거든!

2의 보수와 음수 표현

컴퓨터는 음수를 2의 보수(two's complement)로 표현해요. 양수의 비트를 모두 뒤집고(NOT) 1을 더하면 음수가 돼요.

[KOI] 비트마스크 DP 맛보기
KOI 고난도 문제에서 비트마스크 DP가 등장해요. N명 중 일부를 선택하는 모든 경우를 정수로 표현하면, 배열 dp[1 << N]으로 모든 부분집합을 다룰 수 있어요. N이 20 이하일 때 사용해요.
비트마스크 연산 정리
원소 추가 -- mask |= (1 << i)
원소 제거 -- mask &= ~(1 << i)
원소 확인 -- mask & (1 << i)
원소 토글 -- mask ^= (1 << i)
합집합 -- A | B
교집합 -- A & B
차집합 -- A & ~B

연습문제

OX 퀴즈

XOR 연산을 같은 값에 두 번 적용하면 원래 값으로 돌아온다.

정답: ⭕ O
a ^ b ^ b = a 이에요. XOR은 자기 자신을 상쇄하는 성질이 있어요.
객관식

집합 {1, 3}을 비트마스크로 표현하면?

  1. 3
  2. 5
  3. 10
  4. 13
정답: 3
비트 1과 비트 3을 켜면 0000_1010 = 10이에요. (1<<1) | (1<<3) = 2 + 8 = 10.
단답형

양수 n을 음수 -n으로 바꾸는 2의 보수 공식을 쓰세요. (비트 연산으로)

정답: ~n + 1
NOT 연산(~n)으로 비트를 뒤집고 1을 더하면 2의 보수, 즉 -n이 돼요.
객관식

비트마스크에서 원소를 제거하는 연산은?

  1. mask |= (1 << i)
  2. mask &= ~(1 << i)
  3. mask ^= (1 << i)
  4. mask >>= i
정답: 2
~(1 << i)로 i번째 비트만 0인 마스크를 만들고, AND로 해당 비트를 지워요.
객관식

비트 필드에서 unsigned int flags : 3;이 저장할 수 있는 최대값은?

  1. 3
  2. 7
  3. 8
  4. 15
정답: 1
3비트로 표현 가능한 최대 unsigned 값은 2^3 - 1 = 7입니다.

[심화 내용] 비트 조작과 최적화
비트 집합 = 32개 bool을 int 하나에!
합집합=OR, 교집합=AND, 차집합=AND NOT
→ O(1)로 집합 연산! (배열은 O(n))

XOR의 마법:
A ^ A = 0 (자기 자신과 XOR = 0)
A ^ 0 = A (0과 XOR = 자기 자신)
→ 암호화: data ^ key → 암호문
→ 복호화: 암호문 ^ key → 원본!

비트 트릭 성능:
분기 없는 abs: (n^(n>>31))-(n>>31)
분기 없는 max: b&((a-b)>>31)|a&~((a-b)>>31)
→ SIMD, 게임 엔진에서 성능 핵심!

1#include <stdio.h>
2#include <stdint.h>

4typedef uint64_t Bitboard;
5
6// 체스판 좌표 → 비트 인덱스
7#define SQ(r,c) ((r)*8+(c))
8#define BIT(sq) (1ULL << (sq))
9
10void print_board(Bitboard b) {
11 printf(" a b c d e f g h\n");
12 for (int r = 7; r >= 0; r--) {
13 printf("%d ", r+1);
14 for (int c = 0; c < 8; c++)
15 printf("%s ", (b & BIT(SQ(r,c))) ? "♟" : ".");
16 printf("\n");
17 }
18}
19
20// 나이트 이동 (비트 시프트!)
21Bitboard knight_moves(int sq) {
22 Bitboard b = BIT(sq);
23 Bitboard moves = 0;
24 Bitboard notA = ~0x0101010101010101ULL;
25 Bitboard notH = ~0x8080808080808080ULL;
26 Bitboard notAB = notA & ~0x0202020202020202ULL;
27 Bitboard notGH = notH & ~0x4040404040404040ULL;
28 moves |= (b << 17) & notA; // 위2 오른1
29 moves |= (b << 15) & notH; // 위2 왼1
30 moves |= (b << 10) & notAB; // 위1 오른2
31 moves |= (b << 6) & notGH; // 위1 왼2
32 moves |= (b >> 17) & notH;
33 moves |= (b >> 15) & notA;
34 moves |= (b >> 10) & notGH;
35 moves |= (b >> 6) & notAB;
36 return moves;
37}

51 Bitboard rook = 0;

53 for(int i=0;i<8;i++) { rook |= BIT(SQ(r,i)); rook |= BIT(SQ(i,c)); }

[C 코드] 비트 고급 — Bloom Filter + 비트 스트림
1#include <stdio.h>
2#include <string.h>
3#include <stdint.h>
4
5// === Bloom Filter ===
6#define BF_SIZE 64
7unsigned long long bloom = 0;
8
9unsigned int hash1(const char *s) { unsigned int h=0; while(*s) h=h*31+*s++; return h%BF_SIZE; }
10unsigned int hash2(const char *s) { unsigned int h=0; while(*s) h=h*37+*s++; return h%BF_SIZE; }
11unsigned int hash3(const char *s) { unsigned int h=5381; while(*s) h=h*33+*s++; return h%BF_SIZE; }
12
13void bf_add(const char *s) {
14 bloom |= (1ULL << hash1(s));
15 bloom |= (1ULL << hash2(s));
16 bloom |= (1ULL << hash3(s));
17}
18
19int bf_maybe_contains(const char *s) {
20 return (bloom & (1ULL << hash1(s))) &&
21 (bloom & (1ULL << hash2(s))) &&
22 (bloom & (1ULL << hash3(s)));
23}
24
25// === 비트 스트림 (패킹) ===
26typedef struct { unsigned char buf[100]; int bit_pos; } BitStream;
27
28void bs_init(BitStream *bs) { memset(bs, 0, sizeof(BitStream)); }
29
30void bs_write_bits(BitStream *bs, unsigned int val, int nbits) {
31 for (int i = nbits-1; i >= 0; i--) {
32 int byte_idx = bs->bit_pos / 8;
33 int bit_idx = 7 - (bs->bit_pos % 8);
34 if (val & (1u << i)) bs->buf[byte_idx] |= (1u << bit_idx);
35 bs->bit_pos++;
36 }
37}
38
39unsigned int bs_read_bits(BitStream *bs, int start, int nbits) {
40 unsigned int val = 0;
41 for (int i = 0; i < nbits; i++) {
42 int byte_idx = (start+i) / 8;
43 int bit_idx = 7 - ((start+i) % 8);
44 val = (val << 1) | ((bs->buf[byte_idx] >> bit_idx) & 1);
45 }
46 return val;
47}

49// === 비트 조작 퍼즐 ===
50// 2의 보수에서 부호 반전
51int negate(int x) { return ~x + 1; }
52
53// 절대값 (분기 없이!)
54int abs_branchless(int x) {
55 int mask = x >> 31; // 음수면 0xFFFFFFFF, 양수면 0
56 return (x ^ mask) - mask;
57}
58
59// 두 수 중 큰 값 (분기 없이!)
60int max_branchless(int a, int b) {
61 int diff = a - b;
62 int sign = (diff >> 31) & 1;
63 return a - diff * sign;
64}

[심화 내용] Bloom Filter와 비트 스트림
Bloom Filter:
확률적 집합. "확실히 없음" or "아마 있음"
k개 해시 함수로 비트 배열 설정
→ 거짓 음성(false negative) 불가능!
→ 거짓 양성(false positive) 가능
→ 캐시, DB, 네트워크에서 빠른 사전 필터!

비트 스트림:
데이터를 비트 단위로 읽고/쓰기
→ 압축 (Huffman, LZ77)
→ 프로토콜 (TCP 헤더의 플래그 비트)
→ 이미지/비디오 코덱 (JPEG, H.264)

Branchless 연산:
abs: (x^mask)-mask (mask=x>>31)
max: a-diff*sign
→ SIMD, GPU에서 분기 없이 계산!
Bloom filter detail
블룸 필터. m비트+k해시. 오탐률≈(1-e^(-kn/m))^k. Redis, Chrome(악성URL), 카산드라에서 사용!
branchless programming
분기 없는 프로그래밍. 비트 마스크로 if 대체. GPU/SIMD에서 필수! 파이프라인 스톨 방지.
Bloom Filter: 없으면 확실히 없음! 있으면 아마 있음. 빠른 사전 필터로 DB 조회 줄이기!
9비트를 2바이트에 패킹! 5(3b)+13(4b)+3(2b). 압축 알고리즘의 기초가 비트 스트림!
perfect hash
완전 해시. 충돌 0! gperf로 키워드→인덱스. 컴파일러 키워드 테이블에 사용!
count trailing zeros
CTZ. 마지막 1비트의 위치. __builtin_ctz(). x&(-x)로 최저 비트 추출. 비트보드에 필수!
carry-less multiplication
캐리 없는 곱셈. XOR로 더함. CRC, AES-GCM에 사용. PCLMULQDQ 명령!
popcount hardware
하드웨어 popcount. POPCNT 명령(SSE4.2). __builtin_popcount(). 소프트웨어보다 10배 빠름!
CTZ(Count Trailing Zeros): 비트보드에서 말 위치 찾기! __builtin_ctzll(board)로 O(1)!
[심화] 비트 연산 면접 문제
Q1: 배열에서 1개만 다른 수 찾기 모든 수를 XOR! 같은 수는 상쇄→홀수 번 나타난 수만 남음 a^a=0, a^0=a 성질 활용 Q2: 임시 변수 없이 swap a^=b; b^=a; a^=b; 주의: a와 b가 같은 주소면 0이 됨! Q3: 2의 거듭제곱 판별 n > 0 && (n & (n-1)) == 0 8=1000, 7=0111, AND=0000 ✅ Q4: 비트 반전 (32비트) 5번의 시프트+마스크로 O(log n) 또는 루프로 한 비트씩 Q5: 덧셈을 비트 연산만으로 sum = a^b (캐리 없는 합) carry = (a&b)<<1 반복: sum=a^b, carry=(a&b)<<1, a=sum, b=carry
XOR single number
XOR로 유일한 수 찾기. 배열 전체 XOR. 쌍은 상쇄(a^a=0), 혼자인 수만 남음! O(n) O(1).
bit permutation
비트 순열. 비트 위치를 재배열. 암호학(DES P-box), 셔플링에 사용. PDEP/PEXT 명령!
parallel bit extract
PEXT/PDEP. BMI2 명령. 비트 추출/배치를 병렬로! __builtin_ia32_pext. 체스 엔진 최적화!
면접 필살기: XOR로 배열에서 홀수 번 나오는 수 찾기! O(n) O(1). a^a=0 성질 활용!
De Bruijn sequence
드 브루인 수열. 비트 위치→인덱스 O(1) 변환. 해시 기반. __builtin_ctz 대안!
intrinsics vs builtin
인트린식 vs __builtin. MSVC: _BitScanForward. GCC: __builtin_ctz. 같은 하드웨어 명령!
bit manipulation set
비트 집합 순회. for(s=mask;s;s=(s-1)&mask) 모든 부분집합. 비트 DP의 핵심! O(3^n).
popcount application
popcount 응용. Hamming distance=XOR후 popcount. DNA 서열 비교, 에러 검출에 활용!
Hamming distance = XOR + popcount! "abc"와 "abd"의 비트 차이 개수. 에러 검출의 기본!
gray code
그레이 코드. 인접 값이 1비트만 다름. i^(i>>1)로 변환. 로터리 인코더, 에러 최소화에 사용!
bit reverse
비트 반전. 32비트: 5단계 swap. 또는 룩업 테이블. FFT(고속 푸리에 변환)에서 필수! O(log n).
그레이 코드: 0→1→3→2→6→7→5→4. 인접 값이 1비트만 다름! 기계식 인코더의 오류 방지!
비트 해킹 실전 목록
필수 비트 트릭:
n & (n-1): 최저 1비트 제거
n | (n+1): 최저 0비트 설정
n & (-n): 최저 1비트 추출
~n & (n+1): 최저 0비트 추출
n ^ (n>>1): 그레이 코드 변환

활용처:
Fenwick Tree: n&(-n)으로 구간 탐색
비트보드: CTZ로 말 위치 추출
네트워크: 서브넷 마스크 계산
Fenwick tree
펜윅 트리(BIT). i&(-i)로 구간 합 O(log n). 비트 트릭의 정수! 경쟁 프로그래밍 필수 자료구조.
de Bruijn sequence
드 브뤼인 시퀀스. 비트 위치 찾기 O(1)! 곱셈+시프트+룩업. CTZ/CLZ의 소프트웨어 구현.
Hamming code
해밍 코드. 에러 정정 코드. 패리티 비트로 1비트 에러 수정! ECC 메모리, QR코드에 사용.
bit reverse FFT
비트 리버스 FFT. 비트 반전 순서로 입력 재배치. 쿨리-튜키 FFT 알고리즘의 전처리 단계!
해밍 코드: 4비트 데이터+3비트 패리티=7비트. 1비트 에러 자동 수정! ECC RAM의 원리!
비트마스크(bitmask)
정수의 각 비트를 독립적인 플래그로 사용하여 집합이나 상태를 표현하는 기법.
2의 보수(two's complement)
음수를 표현하는 방법. 양수의 비트를 모두 뒤집고 1을 더하면 대응하는 음수가 된다.
XOR 스왑
임시 변수 없이 두 변수의 값을 교환하는 기법. a ^= b; b ^= a; a ^= b; 3단계로 수행한다.
