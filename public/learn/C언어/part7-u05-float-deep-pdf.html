왜 오차가 생기나?

컴퓨터는 모든 수를 2진수로 저장해요. 10진수 0.1은 2진수로 바꾸면 0.00011001100110011...처럼 끝없이 반복돼요. 유한한 비트에 저장해야 하니까 어딘가에서 잘라야 하고, 이때 오차가 생겨요.

float는 32비트(약 7자리 정밀도), double은 64비트(약 15자리 정밀도)예요. double이 훨씬 정확하지만, 여전히 오차는 있어요.

실수 비교: epsilon 사용하기

실수를 == 로 직접 비교하면 안 돼요. 대신 두 수의 차이가 아주 작은 값(epsilon) 이내인지 확인해요.

float_compare.c
1#include <stdio.h>
2#include <math.h>
3
4#define EPSILON 1e-9
5
6int isEqual(double a, double b) {
7 return fabs(a - b) < EPSILON;
8}
9
10int main(void) {
11 double a = 0.1 + 0.2;
12 double b = 0.3;
13
14 /* 잘못된 비교 */
15 if (a == b) {
16 printf("== : 같다\n");
17 } else {
18 printf("== : 다르다\n");
19 }
20
21 /* 올바른 비교 */
22 if (isEqual(a, b)) {
23 printf("epsilon : 같다\n");
24 } else {
25 printf("epsilon : 다르다\n");
26 }
27
28 return 0;
29}

실수 비교에서 ==를 쓰는 건 가장 흔한 버그 중 하나야! KOI 시험에서도 실수형을 다룰 때 반드시 epsilon 비교를 써야 해.
[실생활] 돈 계산에 실수형을 쓰면 안 되는 이유
은행이나 쇼핑몰에서 1000.00원을 실수형으로 다루면 오차가 누적돼서 잔액이 맞지 않게 돼요. 실제로는 '원' 단위 정수로 계산하거나, 전용 라이브러리를 사용해요.
실수 자료형 핵심 정리
1. float -- 32비트, 약 7자리 정밀도
2. double -- 64비트, 약 15자리 정밀도 (기본 추천)
3. 부동소수점 오차 -- 2진수로 정확히 표현 못 하는 수가 존재
4. 비교 -- == 대신 fabs(a-b) < epsilon 사용
5. 돈 계산 -- 실수형 사용 금지, 정수로 처리

연습문제

OX 퀴즈

float보다 double이 더 정확하다.

정답: ⭕ O
float는 약 7자리, double은 약 15자리 정밀도예요. double이 2배 더 많은 비트를 사용해요.
객관식

실수를 비교할 때 올바른 방법은?

  1. if (a == b)
  2. if (a > b)
  3. if (fabs(a - b) < epsilon)
  4. if ((int)a == (int)b)
정답: 3
부동소수점 오차 때문에 == 비교는 위험해요. 두 수의 차이가 epsilon보다 작은지 확인해야 해요.
단답형

double의 크기는 몇 바이트인가요?

정답: 8
double은 64비트, 즉 8바이트예요. float는 4바이트예요.
객관식

0.1 + 0.2 == 0.3의 결과는?

  1. 참(true)
  2. 거짓(false)
  3. 컴파일 오류
  4. 항상 참
정답: 2
부동소수점 오차 때문에 0.1 + 0.2는 정확히 0.3이 아니에요. 그래서 == 비교 결과는 거짓이에요.
OX 퀴즈

float 타입은 소수점 이하 약 7자리까지 정밀도를 보장한다.

정답: ❌ X
IEEE 754 단정밀도(float)는 유효숫자 약 7자리(23비트 가수부)의 정밀도를 가집니다.

5// 1. 큰 수 + 작은 수 = 작은 수 사라짐!
6void catastrophic_cancel() {
7 float big = 1e8f;
8 float small_val = 1.0f;
9 float result = (big + small_val) - big;
10 printf("(1e8 + 1) - 1e8 = %.1f (기대: 1.0)\n", result);
11
12 // 해결: 보상 합산 (Kahan Summation)
13 float sum = 0.0f, c = 0.0f;
14 for (int i = 0; i < 10000000; i++) {
15 float y = 0.1f - c;
16 float t = sum + y;
17 c = (t - sum) - y; // 손실된 부분 보상
18 sum = t;
19 }
20 printf("0.1 × 10^7 = %.2f (Kahan, 기대: 1000000.0)\n", sum);

22 // 단순 합산 vs Kahan
23 float naive_sum = 0.0f;
24 for (int i = 0; i < 10000000; i++) naive_sum += 0.1f;

[심화 내용] 수치 안정성
Catastrophic Cancellation:
비슷한 두 수의 뺄셈 → 유효 자릿수 사라짐!
(1e8+1)-1e8=0 (float에서!)

Kahan Summation:
손실된 하위 비트를 보상 변수(c)에 저장
다음 합산 시 보상 적용
→ O(1) 추가 비용으로 정밀도 대폭 향상!

비교 방법 3가지:
1. 절대: |a-b| < epsilon (작은 수에만 적합)
2. 상대: |a-b| < max(|a|,|b|) × epsilon
3. ULP: 마지막 비트 자릿수 차이

Subnormal:
지수=0인 특수 숫자. 0 근처 점진적 언더플로

5// === 뉴턴 법 (제곱근) ===
6double my_sqrt(double x) {
7 if (x < 0) return -1; // NaN
8 if (x == 0) return 0;
9 double guess = x / 2;
10 for (int i = 0; i < 50; i++) {
11 double next = (guess + x/guess) / 2;
12 if (fabs(next - guess) < 1e-15) break;
13 guess = next;
14 }
15 return guess;
16}
17
18// === 이분법 (방정식 풀기) ===
19// f(x) = x³ - 2x - 5 (근 ≈ 2.0946)
20double f(double x) { return x*x*x - 2*x - 5; }
21
22double bisection(double a, double b, double eps) {
23 int iter = 0;
24 while (b - a > eps) {
25 double mid = (a + b) / 2;
26 if (f(mid) * f(a) < 0) b = mid;
27 else a = mid;
28 iter++;
29 }
30 printf(" 이분법: %d 반복\n", iter);
31 return (a + b) / 2;
32}

34// === 수치 미분 ===
35double numerical_derivative(double (*fn)(double), double x, double h) {
36 return (fn(x+h) - fn(x-h)) / (2*h); // 중앙 차분
37}
38
39// === 수치 적분 (심프슨) ===
40double simpson(double (*fn)(double), double a, double b, int n) {
41 double h = (b - a) / n;
42 double sum = fn(a) + fn(b);
43 for (int i = 1; i < n; i++)
44 sum += fn(a + i*h) * (i%2 ? 4 : 2);
45 return sum * h / 3;
46}

[심화 내용] 수치 해석
Newton법 (뉴턴-랩슨):
x_{n+1} = x_n - f(x_n)/f'(x_n)
제곱근: x_{n+1} = (x_n + a/x_n) / 2
수렴 속도: 2차 (자릿수가 매 반복 2배!)

이분법:
f(a)*f(b)<0 → 사이에 근 존재
중점에서 부호 확인 → 반으로 줄이기
수렴: 선형 (매 반복 1비트 확정)

수치 미분:
전방: (f(x+h)-f(x))/h → O(h) 오차
중앙: (f(x+h)-f(x-h))/(2h) → O(h²) 오차!
→ h가 너무 작으면 정밀도 오차!

심프슨 적분:
포물선 근사. O(h⁴) 정확도!
사다리꼴보다 훨씬 정밀
Newton method
뉴턴법. x=x-f(x)/f'(x). 2차 수렴! 매 반복 유효 자릿수 2배. 제곱근, 방정식 풀이의 표준.
ULP
Unit in the Last Place. 부동소수점의 최소 차이. 1ULP=비트 1개 차이. 정밀 비교의 기준!
뉴턴법 제곱근: (guess + x/guess)/2. 5번이면 15자리 정밀도! CPU의 fsqrt도 이 원리!
심프슨 적분: 사다리꼴이 아니라 포물선 근사! 같은 분할 수에서 훨씬 정확!
round-to-even
짝수 반올림. IEEE 754 기본 모드. 0.5→0, 1.5→2. 통계적 편향 제거! 은행가 반올림.

FMA. a*b+c를 한 번에 계산. 중간 반올림 없음→정밀도 향상! GPU, BLAS의 핵심.
double-double

interval arithmetic
구간 산술. [a_lo, a_hi]. 연산마다 오차 범위 추적. 수치 검증에 사용!

10진 부동소수점. IEEE 754-2008. _Decimal64. 금융에서 0.1이 정확! DB에서 DECIMAL 타입.

26 double mean = 0;

32// === 수치 안정 분산 (Welford) ===
33void welford_stats(const double *arr, int n, double *mean, double *var) {
34 double m = 0, s = 0;
35 for (int i = 0; i < n; i++) {
36 double delta = arr[i] - m;
37 m += delta / (i + 1);
38 s += delta * (arr[i] - m);
39 }
40 *mean = m;
41 *var = (n > 1) ? s / (n - 1) : 0;
42}
