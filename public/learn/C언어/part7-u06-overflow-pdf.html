Lesson 16-6

오버플로우와 언더플로우

정수가 표현할 수 있는 범위를 넘으면 어떤 일이 생기는지 배워요. limits.h와 안전한 코딩 방법을 익혀요.

📚 이번에 배울 것

  • 정수 오버플로우가 왜 발생하는지 이해해요.
  • 부호 있는 정수와 없는 정수의 오버플로우 차이를 알아요.
  • limits.h의 INT_MAX, INT_MIN, UINT_MAX를 사용할 수 있어요.
  • KOI에서 long long을 써야 하는 상황을 판단할 수 있어요.

✅ 시작 전 체크

  • 정수 자료형(int, long)을 사용할 수 있다
  • 2의 보수의 개념을 알고 있다
Part 1 진행률: 6%

KOI에서 long long이 필요한 경우

KOI 문제에서 'N이 최대 100,000이고 각 값이 최대 100,000'이라면? 두 수를 곱하면 최대 10,000,000,000(100억)인데, int의 최대는 약 21억이야. long long을 써야 해!

2

[KOI] 오버플로우 판단법
문제에서 주어진 값의 범위를 확인하세요. 두 수를 곱하거나 N개를 더할 때 결과가 21억(약 2x10^9)을 넘을 수 있다면 long long을 사용하세요. 출력 형식도 %lld로 바꿔야 해요.
오버플로우 핵심 정리
1. 오버플로우 -- 최댓값을 초과하면 최솟값으로 돌아감
2. 언더플로우 -- 최솟값 아래로 내려가면 최댓값으로 돌아감
3. limits.h -- INT_MAX, INT_MIN, LLONG_MAX 등 제공
4. int -- 약 -21억 ~ +21억
5. long long -- 약 -9.2x10^18 ~ +9.2x10^18

연습문제

OX 퀴즈

int 변수에 INT_MAX + 1을 넣으면 프로그램이 에러 메시지를 출력한다.

정답: ❌ X
오버플로우는 에러 메시지 없이 조용히 발생해요. 잘못된 값이 저장될 뿐이에요.
객관식

int의 최댓값(약 21억)을 넘는 계산이 필요할 때 사용할 자료형은?

  1. short
  2. unsigned char
  3. long long
  4. float
정답: 3
long long은 약 9.2x10^18까지 표현할 수 있어 큰 정수 계산에 적합해요.
단답형

INT_MAX의 값을 정의하고 있는 헤더 파일의 이름은?

정답: limits.h
limits.h에 정수 자료형의 최솟값, 최댓값 상수가 정의되어 있어요.
객관식

unsigned char 변수에 255 + 1을 넣으면?

  1. 256
  2. 0
  3. -1
  4. 컴파일 오류
정답: 2
unsigned char의 범위는 0~255예요. 255 + 1은 오버플로우되어 0이 돼요.
객관식

unsigned char x = 255; x++;의 결과는?

  1. 256
  2. 0
  3. -1
  4. 컴파일 에러
정답: 1
unsigned char의 범위는 0~255입니다. 255에서 1을 더하면 오버플로우가 발생하여 0으로 돌아갑니다.

비유: 자동차 주행거리계!

정수도 마찬가지:
INT_MAX(2147483647) + 1 → INT_MIN(-2147483648)
unsigned: UINT_MAX + 1 → 0

왜 위험한가:
• 게임: HP가 음수→무적 버그
• 보안: 버퍼 크기 계산 오류→해킹
• 우주: Ariane 5 로켓 폭발(64→16비트 변환)

방지법:
1. 연산 전 범위 체크 (safe_add)
2. 더 큰 타입 사용 (int→long long)
3. __builtin_add_overflow (GCC)
wraparound
래핑. unsigned 오버플로=정의된 동작(모듈러). signed 오버플로=UB! 컴파일러가 최적화에 활용.
UB
Undefined Behavior. 정의되지 않은 동작. signed 오버플로, null 역참조. 컴파일러가 무엇이든 할 수 있음!
INT_MAX+1 = INT_MIN! 자동차 계기판처럼 돌아감. 보안 취약점의 원인!
safe_add: 더하기 전에 범위 체크! a > INT_MAX - b이면 오버플로. 간단하고 안전!
