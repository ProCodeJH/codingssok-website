함수도 주소가 있다

변수에 주소가 있듯이, 함수도 메모리 어딘가에 저장되어 있어. 그 주소를 변수에 담으면? 함수를 데이터처럼 다룰 수 있게 돼!

콜백 함수

콜백(callback)이란 '나중에 불러줘'라는 뜻이에요. 함수 포인터를 매개변수로 받으면, 어떤 함수를 실행할지 호출하는 쪽이 결정할 수 있어요.

callback_demo.c
1#include <stdio.h>
2
3int add(int a, int b) { return a + b; }
4int multiply(int a, int b) { return a * b; }
5
6/* 콜백 함수를 받는 함수 */
7void applyAndPrint(int x, int y, int (*operation)(int, int), const char *name) {
8 int result = operation(x, y);
9 printf("%s(%d, %d) = %d\n", name, x, y, result);
10}
11
12int main(void) {
13 applyAndPrint(10, 3, add, "add");
14 applyAndPrint(10, 3, multiply, "multiply");
15 return 0;
16}
실행 결과
add(10, 3) = 13 multiply(10, 3) = 30
applyAndPrint 함수는 '어떤 연산을 할지' 몰라도 돼. 연산 함수를 매개변수로 받으니까, 호출하는 쪽이 원하는 연산을 골라서 넘기면 돼. 이게 추상화의 힘이야!

실전: qsort와 비교 함수

C 표준 라이브러리의 qsort 함수는 비교 함수를 함수 포인터로 받아요. 오름차순, 내림차순 등 정렬 기준을 비교 함수로 자유롭게 바꿀 수 있어요.

qsort_demo.c
1#include <stdio.h>
2#include <stdlib.h>
3
4/* 오름차순 비교 함수 */
5int compareAsc(const void *a, const void *b) {
6 return *(int *)a - *(int *)b;
7}
8
9/* 내림차순 비교 함수 */
10int compareDesc(const void *a, const void *b) {
11 return *(int *)b - *(int *)a;
12}
13
14void printArray(int arr[], int n) {
15 for (int i = 0; i < n; i++) {
16 printf("%d ", arr[i]);
17 }
18 printf("\n");
19}
20
21int main(void) {
22 int arr[] = {42, 17, 93, 5, 68};
23 int n = sizeof(arr) / sizeof(arr[0]);
24
25 printf("원본: ");
26 printArray(arr, n);
27
28 /* 오름차순 정렬 */
29 qsort(arr, n, sizeof(int), compareAsc);
30 printf("오름차순: ");
31 printArray(arr, n);
32
33 /* 내림차순 정렬 */
34 qsort(arr, n, sizeof(int), compareDesc);
35 printf("내림차순: ");
36 printArray(arr, n);
37
38 return 0;
39}
실행 결과
원본: 42 17 93 5 68 오름차순: 5 17 42 68 93 내림차순: 93 68 42 17 5
undefined
qsort의 비교 함수는 KOI에서 정말 자주 나와! 구조체 배열을 정렬할 때도 비교 함수만 바꾸면 돼. 이 패턴을 꼭 외워 두자.
[KOI] qsort 비교 함수 패턴
qsort 비교 함수의 규칙: a가 b보다 앞이면 음수, 같으면 0, 뒤면 양수를 반환해요. 오름차순은 *(int*)a - *(int*)b, 내림차순은 *(int*)b - *(int*)a예요.
함수 포인터 핵심 정리
1. 선언 -- 반환형 (*이름)(매개변수형들)
2. 대입 -- fp = 함수이름 (함수 이름이 곧 주소)
3. 호출 -- fp(인자들) 또는 (*fp)(인자들)
4. 콜백 -- 함수 포인터를 매개변수로 전달
5. qsort -- 비교 함수를 함수 포인터로 전달하는 대표 예시
🔧 바꿔보기

절대값이 작은 순서로 정렬하는 비교 함수 compareAbs를 만들어 보세요.

abs(*(int*)a) - abs(*(int*)b)를 반환하면 돼요. stdlib.h의 abs 함수를 사용하세요.

연습문제

OX 퀴즈

함수 이름 자체가 함수의 주소를 나타낸다.

정답: ⭕ O
배열 이름이 배열의 시작 주소인 것처럼, 함수 이름도 함수의 시작 주소예요.
객관식

int형 2개를 받아 int를 반환하는 함수 포인터 선언은?

  1. int *fp(int, int);
  2. int (*fp)(int, int);
  3. int *(fp)(int, int);
  4. (*int)(fp)(int, int);
정답: 2
함수 포인터는 반환형 (*이름)(매개변수형들) 형태로 선언해요. 괄호 위치가 중요해요.
단답형

함수 포인터를 매개변수로 전달하여 나중에 호출되는 함수를 무엇이라 하나요?

정답: 콜백 함수
콜백(callback) 함수는 '나중에 불러줘'라는 뜻으로, 호출되는 시점을 호출하는 쪽이 결정해요.
객관식

qsort의 비교 함수가 양수를 반환하면?

  1. a가 b보다 앞에 온다
  2. a와 b가 같다
  3. a가 b보다 뒤에 온다
  4. 정렬하지 않는다
정답: 3
양수를 반환하면 a가 b보다 뒤에 와야 한다는 뜻이에요. 음수면 앞, 0이면 같아요.
객관식

함수 포인터 선언 int (*fp)(int, int);에서 fp가 가리킬 수 있는 함수의 반환 타입은?

  1. void
  2. int
  3. int*
  4. double
정답: 1
선언의 맨 앞 int가 반환 타입을 나타냅니다. fp는 int를 반환하고 int 2개를 매개변수로 받는 함수를 가리킵니다.

[C 코드] 함수 포인터 — 단계별 이해
1#include <stdio.h>
2
3/*
4 * 🎯 함수 포인터 = "함수의 주소를 저장하는 변수"
5 *
6 * 변수의 주소: int *p = &x; → p에 x의 주소 저장
7 * 함수의 주소: int (*fp)(int) = &add; → fp에 add의 주소 저장
8 *
9 * 변수 사용: *p → 주소 따라가서 값 읽기
10 * 함수 사용: fp(10) → 주소 따라가서 함수 실행!
11 */
12
13// 1단계: 보통 함수
14int add(int a, int b) { return a + b; }
15int sub(int a, int b) { return a - b; }
16int mul(int a, int b) { return a * b; }
17
18int main() {
19 // 2단계: 함수 포인터 선언
20 // int (*fp)(int, int);
21 // ↑반환형 ↑이름 ↑매개변수
22 // "int 2개를 받아 int를 반환하는 함수의 포인터"
23
24 int (*fp)(int, int);
25
26 // 3단계: 함수 주소 대입
27 fp = add; // &add도 OK. 함수 이름 = 함수 주소!

29
30 fp = sub;
31 printf("fp = sub: fp(3,4) = %d\n", fp(3, 4)); // -1
32
33 fp = mul;
34 printf("fp = mul: fp(3,4) = %d\n", fp(3, 4)); // 12
35
36 // 4단계: 함수 포인터 배열!
37 int (*ops[3])(int, int) = {add, sub, mul};
38 const char *names[] = {"+", "-", "×"};
39
40 printf("\n=== 함수 포인터 배열 ===\n");
41 for (int i = 0; i < 3; i++)
42 printf(" 10 %s 3 = %d\n", names[i], ops[i](10, 3));
43
44 // 5단계: typedef로 깔끔하게!

46 BinOp calculator[] = {add, sub, mul};
47
48 printf("\n=== typedef 버전 ===\n");

53}

쉽게 이해하기: 함수 포인터 선언 읽기
함수 포인터 선언이 어려운 이유:
int (*fp)(int, int) ← 뭐가 뭔지 헷갈림!

읽는 법 (안에서 밖으로):
1. fp → 이름
2. (*fp) → fp는 포인터
3. (*fp)(int,int) → int 2개를 받는 함수의 포인터
4. int (*fp)(int,int) → 반환형은 int

함수 포인터 vs 배열 포인터:
int (*fp)(int) → 함수 포인터
int (*ap)[5] → 배열 포인터
int *fp(int) → 포인터를 반환하는 함수 (다른 것!)

typedef가 구원자!
typedef int (*BinOp)(int, int);
BinOp fp = add; // ← 깔끔!
⚠️ 함수 포인터 흔한 실수
실수 1: 괄호 빼먹기
int *fp(int) ← 이건 "int*를 반환하는 함수 선언"!
int (*fp)(int) ← 이것이 함수 포인터!

실수 2: 시그니처 불일치
int add(int,int)에 void(*fp)(int)를 대입
→ 컴파일 경고/런타임 크래시!

실수 3: NULL 호출
fp = NULL; fp(3,4); → 크래시!
→ 호출 전 NULL 체크!

규칙: typedef를 쓰면 실수가 줄어든다!

[C 코드] 함수 포인터 — 다형성 구현
1#include <stdio.h>
2#include <string.h>
3#include <math.h>
4
5/*
6 * 🎯 함수 포인터로 다형성!
7 *
8 * 비유: 리모컨
9 * - "전원" 버튼 → TV든 에어컨이든 꺼짐
10 * - 같은 버튼, 다른 동작 = 다형성!
11 * - 함수 포인터 = 리모컨의 버튼
12 */
13
14// === 도형 다형성 ===
15typedef struct Shape Shape;
16struct Shape {
17 const char *name;
18 double (*area)(const Shape*);
19 double (*perimeter)(const Shape*);
20 void (*draw)(const Shape*);
21 union { struct{double w,h;} rect; struct{double r;} circle; struct{double a,b,c;} tri; } data;
22};
23
24// 사각형
25double rect_area(const Shape *s) { return s->data.rect.w * s->data.rect.h; }
26double rect_peri(const Shape *s) { return 2*(s->data.rect.w+s->data.rect.h); }
27void rect_draw(const Shape *s) {
28 int w=(int)s->data.rect.w, h=(int)s->data.rect.h;
29 for(int r=0;r<h;r++){for(int c=0;c<w;c++) printf("█");printf("\n");}
30}
31
32// 원
33double circ_area(const Shape *s) { return M_PI * s->data.circle.r * s->data.circle.r; }
34double circ_peri(const Shape *s) { return 2 * M_PI * s->data.circle.r; }
35void circ_draw(const Shape *s) { printf(" ⬤ (r=%.1f)\n", s->data.circle.r); }
36
37// 삼각형
38double tri_area(const Shape *s) {
39 double a=s->data.tri.a,b=s->data.tri.b,c=s->data.tri.c;
40 double p=(a+b+c)/2;
41 return sqrt(p*(p-a)*(p-b)*(p-c));
42}
43double tri_peri(const Shape *s) { return s->data.tri.a+s->data.tri.b+s->data.tri.c; }
44void tri_draw(const Shape *s) { printf(" △ (%.0f,%.0f,%.0f)\n",s->data.tri.a,s->data.tri.b,s->data.tri.c); }
45
46// 생성자
47Shape make_rect(double w, double h) {
48 Shape s = {"Rectangle", rect_area, rect_peri, rect_draw};
49 s.data.rect.w=w; s.data.rect.h=h; return s;
50}
51Shape make_circle(double r) {
52 Shape s = {"Circle", circ_area, circ_peri, circ_draw};
53 s.data.circle.r=r; return s;
54}
55Shape make_tri(double a, double b, double c) {
56 Shape s = {"Triangle", tri_area, tri_peri, tri_draw};
57 s.data.tri.a=a; s.data.tri.b=b; s.data.tri.c=c; return s;
58}
59
60// 다형적 함수! Shape 타입 몰라도 동작!
61void print_shape_info(const Shape *s) {
62 printf(" %-10s area=%-8.2f peri=%.2f\n",
63 s->name, s->area(s), s->perimeter(s));
64}
65
66int main() {
67 Shape shapes[] = { make_rect(5,3), make_circle(4), make_tri(3,4,5) };
68
69 printf("=== 도형 다형성 (함수 포인터) ===\n");
70 for (int i = 0; i < 3; i++)
71 print_shape_info(&shapes[i]);
72
73 printf("\n=== 그리기 ===\n");
74 for (int i = 0; i < 3; i++) {
75 printf("%s:\n", shapes[i].name);
76 shapes[i].draw(&shapes[i]);
77 }

79}

쉽게 이해하기: 함수 포인터 = C의 다형성
C++의 virtual 함수 = C의 함수 포인터!

구조체에 함수 포인터를 넣으면:
1. 같은 인터페이스 (area, draw)
2. 다른 구현 (rect_area, circ_area)
3. 호출하는 쪽은 타입 몰라도 OK!

이것이 바로 다형성!
s->area(s) → 사각형이면 w×h, 원이면 πr²
→ 같은 코드, 다른 동작!

3// === 미니 가상 머신 (VM) ===
4// 명령어 = 함수 포인터 배열!
5typedef struct { int stack[100]; int sp; int regs[4]; } VM;
6
7typedef void (*Instruction)(VM*);
8
9void op_push_0(VM *v) { v->stack[v->sp++] = v->regs[0]; }
10void op_push_1(VM *v) { v->stack[v->sp++] = v->regs[1]; }
11void op_pop_0(VM *v) { v->regs[0] = v->stack[--v->sp]; }
12void op_add(VM *v) { int b=v->stack[--v->sp]; int a=v->stack[--v->sp]; v->stack[v->sp++]=a+b; }
13void op_sub(VM *v) { int b=v->stack[--v->sp]; int a=v->stack[--v->sp]; v->stack[v->sp++]=a-b; }
14void op_mul(VM *v) { int b=v->stack[--v->sp]; int a=v->stack[--v->sp]; v->stack[v->sp++]=a*b; }
15void op_dup(VM *v) { v->stack[v->sp] = v->stack[v->sp-1]; v->sp++; }
16void op_print(VM *v) { printf(" [VM] top = %d\n", v->stack[v->sp-1]); }
17
18void vm_run(VM *v, Instruction *prog, int len) {
19 for (int i = 0; i < len; i++) prog[i](v);
20}
21
22// === 명령 디스패처 (셸 시뮬레이션) ===
23typedef struct { const char *name; const char *desc; void (*fn)(void); } Command;
24
25void cmd_help(void);
26void cmd_status(void) { printf(" System OK, uptime: 42h\n"); }
27void cmd_version(void) { printf(" v1.0.0 (2026-03-05)\n"); }
28void cmd_clear(void) { printf(" (screen cleared)\n"); }
29void cmd_exit(void) { printf(" Goodbye!\n"); }

32 {"help", "Show commands", cmd_help},
33 {"status", "System status", cmd_status},
34 {"version", "Show version", cmd_version},
35 {"clear", "Clear screen", cmd_clear},
36 {"exit", "Exit shell", cmd_exit}
37};
38int ncmds = 5;
39
40void cmd_help(void) {
41 printf(" Available commands:\n");
42 for (int i = 0; i < ncmds; i++)
43 printf(" %-10s %s\n", commands[i].name, commands[i].desc);
44}

46void dispatch(const char *input) {
47 for (int i = 0; i < ncmds; i++)
48 if (strcmp(input, commands[i].name) == 0) { commands[i].fn(); return; }
49 printf(" Unknown command: %s\n", input);
50}

63 op_mul, // 13*7=91
64 op_print
65 };
66 printf(" Program: (R0+R1) * (R0-R1) where R0=10, R1=3\n");
67 vm_run(&v, prog, 8);
68
69 // 디스패처
70 printf("\n=== 명령 디스패처 ===\n");
71 const char *inputs[] = {"help", "status", "version", "unknown", "exit"};
72 for (int i = 0; i < 5; i++) {
73 printf("> %s\n", inputs[i]);
74 dispatch(inputs[i]);
75 }
76 return 0;
77}

push 10, push 3, add → 13
JVM, Python, WebAssembly 모두 스택 VM!

add r0, r1, r2 → r0 = r1 + r2
Lua VM, Dalvik (Android) 사용

명령 디스패처:
문자열 → 함수 매핑
쉘, CLI, 게임 명령어에 활용
switch/if-else 대신 테이블!
stack VM
스택 가상 머신. push/pop/op. JVM, CPython, .NET CLR. 구현 간단! 함수포인터 배열로 디스패치.
dispatch table
디스패치 테이블. 인덱스→함수 O(1) 매핑. switch보다 빠름! VM, 이벤트 핸들러, 파서에 사용.
VM = 함수 포인터 배열 + 스택! prog[i](vm)으로 명령 실행. JVM의 원리가 이것!
명령 디스패처: {name, desc, fn} 배열. 이름으로 검색→fn() 호출. 쉘/CLI의 기본 구조!
function pointer syntax
함수 포인터 문법. int(*fp)(int)=포인터. int*fp(int)=함수! 괄호가 핵심. typedef로 해결!
signal handler
시그널 핸들러. signal(SIGINT, handler). 함수 포인터 전달! Ctrl+C 처리. 비동기 이벤트!
atexit callback
atexit 콜백. atexit(cleanup_func). 프로그램 종료 시 자동 호출. 자원 정리에 활용!
signal(SIGINT, handler): Ctrl+C 누르면 handler 함수 호출! 함수 포인터의 OS 활용!
qsort comparator
qsort 비교 함수. int cmp(const void*,const void*). 음수/0/양수 반환. 함수 포인터의 대표 예!
dlsym runtime loading
dlsym. 런타임에 함수 로딩. dlopen("lib.so")+dlsym("func"). 플러그인의 핵심! 함수 포인터로 호출.
dlsym: 실행 중에 함수 로딩! 플러그인 시스템의 핵심. 함수 포인터로 호출!
bsearch callback
bsearch. 이진 탐색 표준 함수. qsort와 같은 비교 콜백! 정렬된 배열에서 O(log n) 검색.
function pointer cast
함수 포인터 캐스팅. (void(*)(void))fp. 시그니처 불일치→UB! 항상 같은 시그니처로 호출!
stdqsort comparison
qsort vs 직접 정렬. qsort: 범용(함수포인터 오버헤드). 직접: 빠름(인라인 가능). 성능 민감하면 직접!
함수 포인터 캐스팅 위험! int(*)(int)를 void(*)(void)로 바꾸면 UB. 시그니처 일치 필수!
trampoline function
트램펄린 함수. 중간 함수로 시그니처 변환. void* context→구체 타입 캐스팅. 콜백 어댑터!
closure vs fp
클로저 vs 함수 포인터. 클로저=함수+환경. C 함수 포인터는 환경 없음! void* context로 시뮬레이션.
C에 클로저가 없어서 void* context 패턴! 함수 포인터+데이터 포인터=의사 클로저!
함수 포인터 선언 읽기 연습
연습 문제:
int (*fp)(int) → int를 받아 int 반환하는 함수의 포인터
int (*arr[5])(int) → 함수 포인터 5개짜리 배열
int (*(*fp)(int))(double) → int를 받아 "double→int 함수포인터"를 반환

cdecl 도구:
"declare fp as pointer to function(int) returning int"
→ int (*fp)(int)

function pointer cast
함수 포인터 캐스트. 다른 시그니처로 캐스트 후 호출=UB! 항상 정확한 타입으로. dlsym은 void* 반환.
qsort comparator rules
qsort 비교 규칙. 음수=ab. 일관성 필수! aa 보장해야.
trampoline function
트램폴린 함수. 함수 포인터로 간접 호출. 동적 디스패치. Go/Rust의 trait/interface 내부 구현!
qsort의 비교 함수: return a-b 하면 오버플로 위험! ab?1:0이 안전!
함수 포인터(function pointer)
함수의 시작 주소를 저장하는 포인터 변수. 이를 통해 함수를 간접적으로 호출할 수 있다.
콜백(callback)
다른 함수에 인자로 전달되어 특정 시점에 호출되는 함수. 함수 포인터로 구현한다.
qsort
C 표준 라이브러리의 범용 정렬 함수. 비교 함수를 함수 포인터로 받아 정렬 기준을 결정한다.
비교 함수
두 원소의 순서를 결정하는 함수. 음수(앞), 0(같음), 양수(뒤)를 반환한다.
