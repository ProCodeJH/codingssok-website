함수 포인터 활용

함수 포인터 배열, 구조체 멤버로 함수 포인터 사용, typedef로 깔끔하게 정리하는 방법을 배워요.

📚 이번에 배울 것

  • 함수 포인터 배열을 선언하고 사용할 수 있어요.
  • 구조체에 함수 포인터를 멤버로 넣어 가짜 객체지향을 구현해요.
  • typedef로 함수 포인터 타입을 깔끔하게 정의해요.
  • 메뉴 시스템을 함수 포인터 배열로 만들 수 있어요.

✅ 시작 전 체크

  • 함수 포인터의 기본 개념을 알고 있다 (U7-09)
  • 구조체의 기본 사용법을 알고 있다
  • 배열의 기본 개념을 알고 있다
Part 1 진행률: 10%

함수 포인터 배열: 인덱스로 함수를 고른다

계산기를 만들 때 if-else를 4번 쓰는 대신, 함수들을 배열에 넣고 번호로 골라서 호출하면 어떨까? 이게 바로 함수 포인터 배열이야!

함수 포인터 배열이란, 함수 포인터를 여러 개 모아서 배열로 만든 것이에요. 인덱스(번호)로 원하는 함수를 골라서 호출할 수 있어요. if-else나 switch를 줄여주는 강력한 기법이에요.

1#include <stdio.h>
2
3int add(int a, int b) { return a + b; }
4int sub(int a, int b) { return a - b; }
5int mul(int a, int b) { return a * b; }
6int div_f(int a, int b) { return b != 0 ? a / b : 0; }
7
8int main(void) {
9 // 함수 포인터 배열 선언
10 int (*func[4])(int, int) = { add, sub, mul, div_f };
11 char *names[] = { "더하기", "빼기", "곱하기", "나누기" };
12
13 int a = 10, b = 3;
14 for (int i = 0; i < 4; i++) {
15 printf("%s: %d\n", names[i], func[i](a, b));
16 }
17 return 0;
18}
실행 결과
더하기: 13 빼기: 7 곱하기: 30 나누기: 3
10줄int (*func[4])(int, int) -- 'int를 2개 받아 int를 반환하는 함수 포인터'를 4개 담는 배열이에요.
10줄= { add, sub, mul, div_f } -- 함수 이름 자체가 함수의 주소예요. 배열에 주소를 넣는 거죠.
15줄func[i](a, b) -- i번째 함수 포인터를 통해 함수를 호출해요. func[0](10,3)은 add(10,3)과 같아요.

typedef로 함수 포인터를 깔끔하게

int (*func[4])(int, int)는 읽기 어렵죠? typedef를 사용하면 함수 포인터 타입에 이름을 붙여서 훨씬 깔끔하게 만들 수 있어요.

1#include <stdio.h>
2
3// typedef로 함수 포인터 타입 정의
4typedef int (*CalcFunc)(int, int);
5
6int add(int a, int b) { return a + b; }
7int sub(int a, int b) { return a - b; }
8
9int main(void) {
10 // 깔끔한 선언!
11 CalcFunc func[2] = { add, sub };
12
13 printf("add: %d\n", func[0](5, 3));
14 printf("sub: %d\n", func[1](5, 3));
15 return 0;
16}
typedef를 쓰면 CalcFunc라는 이름으로 함수 포인터를 마치 보통 자료형처럼 쓸 수 있어! 코드가 훨씬 읽기 쉬워져.

구조체 + 함수 포인터 = 가짜 OOP

C에는 객체지향(OOP)이 없지만, 구조체에 함수 포인터를 넣으면 비슷한 효과를 낼 수 있어요. 같은 구조체 타입인데 함수 포인터가 달라서 다른 행동을 하게 만들 수 있습니다. 이것이 다형성(polymorphism)의 기본 원리예요.

1#include <stdio.h>
2#include <string.h>
3
4typedef struct Animal {
5 char name[20];
6 int age;
7 void (*speak)(void);
8} Animal;
9
10void dog_speak(void) { printf("Woof! Woof!\n"); }
11void cat_speak(void) { printf("Meow~\n"); }
12
13Animal create_dog(const char *name, int age) {
14 Animal a;
15 strcpy(a.name, name);
16 a.age = age;
17 a.speak = dog_speak;
18 return a;
19}
20
21Animal create_cat(const char *name, int age) {
22 Animal a;
23 strcpy(a.name, name);
24 a.age = age;
25 a.speak = cat_speak;
26 return a;
27}
28
29int main(void) {
30 Animal dog = create_dog("Buddy", 3);
31 Animal cat = create_cat("Nabi", 2);
32
33 printf("%s says: ", dog.name);
34 dog.speak(); // Woof! Woof!
35
36 printf("%s says: ", cat.name);
37 cat.speak(); // Meow~
38
39 return 0;
40}

실전: 메뉴 시스템을 함수 포인터 배열로

프로그램의 메뉴를 만들 때 switch 대신 함수 포인터 배열을 쓰면 코드가 훨씬 깔끔해져요. 메뉴 항목이 늘어나도 배열에 함수를 추가하기만 하면 돼요.

1#include <stdio.h>
2
3void menu_hello(void) { printf("Hello!\n"); }
4void menu_time(void) { printf("It is study time!\n"); }
5void menu_bye(void) { printf("Goodbye!\n"); }
6
7int main(void) {
8 void (*menu[])(void) = { menu_hello, menu_time, menu_bye };
9 char *labels[] = { "1. 인사하기", "2. 시간 알려주기", "3. 종료" };
10 int n = 3;
11
12 while (1) {
13 printf("\n--- 메뉴 ---\n");
14 for (int i = 0; i < n; i++) {
15 printf("%s\n", labels[i]);
16 }
17 printf("선택: ");
18
19 int choice;
20 scanf("%d", &choice);
21 if (choice < 1 || choice > n) {
22 printf("잘못된 입력입니다.\n");
23 continue;
24 }
25 if (choice == n) {
26 menu[choice - 1]();
27 break;
28 }
29 menu[choice - 1](); // 인덱스로 함수 호출!
30 }
31 return 0;
32}
switch문으로 만들면 case가 점점 늘어나지만, 함수 포인터 배열을 쓰면 배열에 함수 하나만 추가하면 끝이야. 유지보수가 쉬워지지!
[KOI] 함수 포인터와 qsort
C 표준 라이브러리의 qsort 함수는 비교 함수를 함수 포인터로 받아요. 정렬 기준을 바꾸고 싶을 때 비교 함수만 바꿔 넣으면 되니까 매우 유연해요. KOI 같은 대회에서 정렬 문제를 풀 때 자주 활용돼요.

연습문제

OX 퀴즈

함수 포인터 배열을 사용하면 if-else나 switch를 줄일 수 있다.

정답: ⭕ O
인덱스로 함수를 선택해서 호출하므로 조건 분기 코드가 필요 없어져요.
객관식

다음 중 함수 포인터 배열을 올바르게 선언한 것은?

  1. int *func[3](int);
  2. int (*func[3])(int);
  3. int func[3](int);
  4. (*int func[3])(int);
정답: 2
int (*func[3])(int)은 'int를 받아 int를 반환하는 함수 포인터 3개짜리 배열'이에요. 괄호 위치가 핵심이에요.
단답형

typedef를 사용해 'int를 2개 받아 int를 반환하는 함수 포인터' 타입을 MathFunc라는 이름으로 정의하세요.

정답: typedef int (*MathFunc)(int, int);
typedef와 함수 포인터 선언을 결합해요. (*MathFunc) 부분이 타입 이름이 됩니다.
OX 퀴즈

구조체 멤버로 함수 포인터를 넣을 수 없다.

정답: ❌ X
구조체 멤버로 함수 포인터를 넣을 수 있어요. 이를 통해 C에서도 객체지향과 비슷한 패턴을 구현할 수 있습니다.
객관식

아래 코드에서 func[2](4, 5)의 결과는? int add(int a, int b) { return a+b; } int sub(int a, int b) { return a-b; } int mul(int a, int b) { return a*b; } int (*func[])(int,int) = {add, sub, mul};

  1. 9
  2. -1
  3. 20
  4. 컴파일 오류
정답: 3
func[2]는 mul이므로 mul(4, 5) = 20이 됩니다.

[C 코드] 함수 포인터 응용 — 콜백 + 이벤트
1#include <stdio.h>
2#include <string.h>
3
4/*
5 * 콜백 = "이거 끝나면 이 함수 불러줘"
6 *
7 * 비유: 배달 주문
8 * 1. 치킨 주문 (작업 시작)
9 * 2. "도착하면 전화해주세요" (콜백 등록)
10 * 3. 치킨 도착 → 전화 옴! (콜백 실행)
11 */
12
13// === 콜백 패턴 1: 정렬 비교 ===
14typedef int (*Comparator)(int, int);
15
16int asc(int a, int b) { return a - b; }
17int desc(int a, int b) { return b - a; }
18
19void sort_with(int *arr, int n, Comparator cmp) {
20 for (int i = 0; i < n-1; i++)
21 for (int j = 0; j < n-1-i; j++)
22 if (cmp(arr[j], arr[j+1]) > 0) {
23 int t = arr[j]; arr[j] = arr[j+1]; arr[j+1] = t;
24 }
25}
26
27// === 콜백 패턴 2: 이벤트 리스너 ===
28typedef void (*EventHandler)(const char* msg);
29
30typedef struct {
31 const char *name;
32 EventHandler on_success;
33 EventHandler on_error;
34} Task;
35
36void log_handler(const char *msg) { printf(" [LOG] %s\n", msg); }
37void alert_handler(const char *msg) { printf(" ⚠️ %s\n", msg); }
38void email_handler(const char *msg) { printf(" 📧 Sending: %s\n", msg); }
39
40void run_task(Task *t, int success) {
41 printf("Running \"%s\": ", t->name);
42 if (success) {
43 printf("SUCCESS\n");
44 if (t->on_success) t->on_success(t->name);
45 } else {
46 printf("FAILED\n");
47 if (t->on_error) t->on_error(t->name);
48 }
49}
50
51// === 콜백 패턴 3: map/filter ===
52typedef int (*Transform)(int);
53typedef int (*Predicate)(int);
54
55void array_map(int *arr, int n, Transform fn) {
56 for (int i = 0; i < n; i++) arr[i] = fn(arr[i]);
57}
58
59int array_filter(int *src, int *dst, int n, Predicate fn) {
60 int count = 0;
61 for (int i = 0; i < n; i++)
62 if (fn(src[i])) dst[count++] = src[i];
63 return count;
64}
65
66int double_it(int x) { return x * 2; }
67int square(int x) { return x * x; }
68int is_even(int x) { return x % 2 == 0; }
69int is_positive(int x) { return x > 0; }
70
71int main() {
72 // 정렬 콜백
73 int nums[] = {5, 3, 8, 1, 9};
74 sort_with(nums, 5, asc);
75 printf("Ascending: "); for(int i=0;i<5;i++) printf("%d ",nums[i]); printf("\n");
76 sort_with(nums, 5, desc);
77 printf("Descending: "); for(int i=0;i<5;i++) printf("%d ",nums[i]); printf("\n");
78
79 // 이벤트
80 printf("\n");
81 Task t1 = {"Download", log_handler, alert_handler};
82 Task t2 = {"Deploy", email_handler, alert_handler};
83 run_task(&t1, 1);
84 run_task(&t2, 0);
85
86 // map/filter
87 printf("\n");
88 int arr[] = {1,2,3,4,5,6,7,8};
89 printf("Original: "); for(int i=0;i<8;i++) printf("%d ",arr[i]); printf("\n");
90
91 array_map(arr, 8, double_it);
92 printf("map(×2): "); for(int i=0;i<8;i++) printf("%d ",arr[i]); printf("\n");
93
94 int filtered[8];
95 int n = array_filter(arr, filtered, 8, is_even);
96 printf("filter(even): "); for(int i=0;i<n;i++) printf("%d ",filtered[i]); printf("\n");
97 return 0;
98}

쉽게 이해하기: 콜백의 3가지 패턴
패턴 1: 전략 (Strategy)
같은 정렬인데 비교 방법만 다르게!
sort(arr, asc) vs sort(arr, desc)
→ 정렬 코드는 1개, 비교만 갈아끼움

패턴 2: 이벤트 (Observer)
"성공하면 log, 실패하면 alert"
on_success = log_handler;
on_error = alert_handler;
→ GUI 버튼 클릭, 네트워크 응답에 활용

패턴 3: 변환 (Higher-order)
map: 모든 원소에 함수 적용 [1,2,3]→[2,4,6]
filter: 조건 만족하는 것만 [1,2,3,4]→[2,4]
→ JavaScript의 arr.map/filter와 동일!
higher-order function
고차 함수. 함수를 인자로 받거나 반환. map/filter/reduce. 함수형 프로그래밍의 핵심!
strategy pattern
전략 패턴. 알고리즘을 캡슐화하여 교환. sort+comparator. 런타임에 전략 변경!
콜백 = "나중에 불러줘!" sort_with(arr, asc)에서 asc가 콜백. qsort도 같은 원리!
map+filter = C에서도 함수형! double_it, is_even을 넘기면 배열 변환. JS와 같은 개념!

32// === 미들웨어 체인 (함수 포인터 배열) ===
33typedef void (*Middleware)(char *data);
34
35void trim_whitespace(char *d) {
36 int len = strlen(d);
37 while (len > 0 && d[len-1] == ' ') d[--len] = 0;
38 char *s = d; while (*s == ' ') s++;
39 if (s != d) memmove(d, s, strlen(s)+1);
40 printf(" [trim] \"%s\"\n", d);
41}
42
43void to_uppercase(char *d) {
44 for (int i = 0; d[i]; i++)
45 if (d[i] >= 'a' && d[i] <= 'z') d[i] -= 32;
46 printf(" [upper] \"%s\"\n", d);
47}
48
49void add_timestamp(char *d) {
50 char buf[200];
51 sprintf(buf, "[2026-03-05] %s", d);
52 strcpy(d, buf);
53 printf(" [stamp] \"%s\"\n", d);
54}
55
56void pipeline(char *data, Middleware *mw, int n) {
57 printf(" Pipeline: \"%s\"\n", data);
58 for (int i = 0; i < n; i++) mw[i](data);
59 printf(" Final: \"%s\"\n\n", data);
60}
61
62// === 이벤트 시스템 ===
63typedef void (*EventCb)(const char *event, void *ctx);
64typedef struct { const char *event; EventCb cb; void *ctx; } Listener;
65
66Listener listeners[20];
67int nlisteners = 0;
68
69void on(const char *event, EventCb cb, void *ctx) {
70 listeners[nlisteners++] = (Listener){event, cb, ctx};
71}
72
73void emit(const char *event) {
74 for (int i = 0; i < nlisteners; i++)
75 if (strcmp(listeners[i].event, event) == 0)
76 listeners[i].cb(event, listeners[i].ctx);
77}
78
79void log_cb(const char *ev, void *ctx) { printf(" [LOG] Event: %s\n", ev); }
80void save_cb(const char *ev, void *ctx) { printf(" [SAVE] Saving on %s\n", ev); }
81void notify_cb(const char *ev, void *ctx) { printf(" [NOTIFY] Alert: %s → %s\n", ev, (char*)ctx); }
82

[심화 내용] 함수 포인터 디자인 패턴
1. 플러그인 패턴:
구조체에 init/process/cleanup 함수 포인터
새 포맷 추가 = 구조체 하나 추가! 기존 코드 수정 없음

2. 미들웨어/파이프라인:
함수 포인터 배열을 순서대로 실행
trim→upper→stamp 체인
Express.js의 middleware와 같은 원리!

3. 이벤트 시스템 (Observer):
on("event", callback) → 리스너 등록
emit("event") → 등록된 콜백 전부 실행
→ GUI, 게임 엔진의 핵심 패턴!

핵심:
함수 포인터 = "나중에 뭘 할지 미리 등록"
확장 가능하고 유연한 코드의 기반
plugin pattern
플러그인 패턴. init/process/cleanup 함수 포인터. 새 기능=구조체 추가. 기존 코드 수정 불필요!
middleware
미들웨어. 함수 포인터 체인. A→B→C 순서 실행. Express.js, Django, Linux 네트워크 스택에 활용!
observer pattern
옵저버 패턴. on(event,callback)+emit(event). 이벤트 발생→등록된 콜백 실행. GUI의 핵심!
플러그인 = 구조체에 함수 포인터 3개(init/process/cleanup). 새 포맷 추가가 이렇게 쉬워!
[C 코드] 함수 포인터 — 제네릭 컨테이너
1#include <stdio.h>
2#include <stdlib.h>
3#include <string.h>

5// === 제네릭 동적 배열 (void* + 함수 포인터) ===
6typedef int (*Comparator)(const void*, const void*);
7typedef void (*Printer)(const void*);
8
9typedef struct {
10 void *data;
11 int size, cap, elem_size;
12 Comparator cmp;
13 Printer print;
14} GenericArray;
15
16GenericArray ga_new(int elem_size, Comparator cmp, Printer print) {
17 GenericArray a;
18 a.elem_size = elem_size;
19 a.size = 0; a.cap = 4;
20 a.data = malloc(a.cap * elem_size);
21 a.cmp = cmp; a.print = print;
22 return a;
23}
24
25void* ga_at(GenericArray *a, int i) {
26 return (char*)a->data + i * a->elem_size;
27}
28
29void ga_push(GenericArray *a, const void *elem) {
30 if (a->size >= a->cap) {
31 a->cap *= 2;
32 a->data = realloc(a->data, a->cap * a->elem_size);
33 }
34 memcpy(ga_at(a, a->size++), elem, a->elem_size);
35}
36
37void ga_sort(GenericArray *a) {
38 qsort(a->data, a->size, a->elem_size, a->cmp);
39}
40
41void ga_print_all(GenericArray *a) {
42 printf(" [");
43 for (int i = 0; i < a->size; i++) {
44 if (i > 0) printf(", ");
45 a->print(ga_at(a, i));
46 }
47 printf("]\n");
48}
49
50int ga_find(GenericArray *a, const void *key) {
51 for (int i = 0; i < a->size; i++)
52 if (a->cmp(ga_at(a, i), key) == 0) return i;
53 return -1;
54}
55
56// int 타입
57int cmp_int(const void *a, const void *b) { return *(int*)a - *(int*)b; }
58void print_int(const void *p) { printf("%d", *(int*)p); }
59
60// string 타입
61typedef struct { char str[50]; } Str;
62int cmp_str(const void *a, const void *b) { return strcmp(((Str*)a)->str, ((Str*)b)->str); }
63void print_str(const void *p) { printf("\"%s\"", ((Str*)p)->str); }
64
65// Student 타입
66typedef struct { char name[20]; int score; } Student;
67int cmp_student_score(const void *a, const void *b) { return ((Student*)b)->score - ((Student*)a)->score; }
68void print_student(const void *p) { Student *s=(Student*)p; printf("%s(%d)",s->name,s->score); }
69

71 // int 배열
72 printf("=== Generic Int Array ===\n");
73 GenericArray ints = ga_new(sizeof(int), cmp_int, print_int);
74 int vals[] = {42, 7, 99, 3, 55};
75 for (int i = 0; i < 5; i++) ga_push(&ints, &vals[i]);
76 printf(" Before: "); ga_print_all(&ints);
77 ga_sort(&ints);
78 printf(" After: "); ga_print_all(&ints);
79 int key = 99;
80 printf(" find(99) = index %d\n", ga_find(&ints, &key));
81
82 // Student 배열
83 printf("\n=== Generic Student Array ===\n");
84 GenericArray students = ga_new(sizeof(Student), cmp_student_score, print_student);
85 Student ss[] = {{"Kim",85},{"Lee",92},{"Park",78},{"Choi",95}};
86 for (int i = 0; i < 4; i++) ga_push(&students, &ss[i]);
87 printf(" Before: "); ga_print_all(&students);
88 ga_sort(&students);
89 printf(" Sorted: "); ga_print_all(&students);
90

92 return 0;
93}

[심화 내용] C의 제네릭 프로그래밍
void* = C의 제네릭!
타입을 모르지만 크기(elem_size)는 안다
→ memcpy로 복사, 함수 포인터로 비교/출력

qsort의 원리:
void qsort(void *base, size_t n, size_t size, cmp)
→ 어떤 타입이든 정렬 가능!
→ 비교 함수만 바꾸면 됨

C++ 템플릿 vs C void*:
C++: vector → 컴파일 시 타입 확정, 빠름
C: void* + size → 런타임 유연성, 타입 안전 없음

실전:
qsort, bsearch, pthread_create 전부 void*
→ 함수 포인터 + void* = C의 다형성 기반
void* generic
void* = 범용 포인터. 모든 타입 가리킴. qsort/bsearch/pthread의 기반. 타입 안전은 프로그래머 책임!
제네릭 배열: void*로 데이터, cmp/print 함수 포인터로 동작. int든 Student든 같은 코드!
dependency injection
의존성 주입. 함수 포인터로 동작 주입. 테스트 시 mock 함수 전달! 유연한 설계의 핵심.
factory pattern
팩토리 패턴. 함수 포인터로 생성 함수 선택. create_shape(type)→사각형/원. 다형적 생성!
closure simulation
클로저 시뮬레이션. 함수포인터+void* context. JavaScript 클로저를 C에서! qsort_r의 원리.
qsort_r: 비교 함수에 context(void*) 전달! 전역 변수 없이 상태 전달. 클로저의 C 버전!
visitor pattern
비지터 패턴. 함수 포인터로 자료 구조 순회+처리 분리. AST 처리, 직렬화에 활용!
command pattern
커맨드 패턴. 동작을 구조체에 캡슐화. {execute, undo} 함수 포인터. 실행 취소의 핵심!
opaque pointer pattern
불투명 포인터. typedef struct Impl *Handle. 내부 구조 숨김! 공개 API에 함수 포인터만 노출.
vtable implementation
vtable 구현. 구조체 첫 필드=함수 포인터 배열 포인터. C++의 virtual이 내부적으로 이렇게!
file operations pattern
file_operations 패턴. Linux 커널. open/read/write/close 함수 포인터. 디바이스 드라이버의 핵심!
function table
함수 테이블. 배열 인덱스→함수 호출 O(1). switch 대체! VM opcode, 이벤트 핸들러에 사용.
visitor pattern
비지터 패턴. 데이터 구조 순회+연산 분리. accept(visitor). C에서는 함수 포인터로 구현!
opaque pointer
불투명 포인터. 구현 숨김. typedef struct Impl *Handle;. 인터페이스만 공개! C의 캡슐화.
function table versioning
함수 테이블 버전. struct ops { int version; void(*fn1)(); void(*fn2)(); }. ABI 호환성!
함수 포인터 배열
같은 시그니처의 함수 포인터 여러 개를 하나의 배열에 저장한 것. 인덱스로 함수를 선택해 호출할 수 있다.
다형성(polymorphism)
같은 인터페이스(함수 시그니처)를 통해 서로 다른 동작을 수행하는 것. C에서는 함수 포인터로 구현한다.
콜백 함수(callback)
다른 함수에 인자로 전달되어 나중에 호출되는 함수. 함수 포인터를 통해 전달한다.
typedef (함수 포인터)
복잡한 함수 포인터 타입에 이름을 붙여 읽기 쉽게 만드는 방법.
