Lesson 17-11

가변 인자 함수

printf처럼 인자 개수가 자유로운 함수를 직접 만들어 봐요. stdarg.h의 va_list, va_start, va_arg, va_end를 배워요.

📚 이번에 배울 것

  • 가변 인자 함수가 무엇인지 이해해요.
  • stdarg.h의 매크로(va_list, va_start, va_arg, va_end)를 사용할 수 있어요.
  • 나만의 가변 인자 함수를 직접 만들 수 있어요.
  • printf가 어떻게 인자 개수를 처리하는지 이해해요.

✅ 시작 전 체크

  • 함수의 선언과 호출을 알고 있다
  • 포인터의 기본 개념을 알고 있다
  • 함수 포인터를 알고 있다 (U7-09, U7-10)
Part 1 진행률: 11%

printf는 어떻게 인자를 여러 개 받을까?

printf("Hello")는 인자가 1개, printf("%d %d", a, b)는 인자가 3개야. 같은 함수인데 인자 개수가 다르지? 이런 함수를 가변 인자 함수라고 해!

C에서는 함수를 선언할 때 매개변수 개수를 미리 정해야 해요. 하지만 가변 인자 함수를 사용하면 호출할 때마다 인자 개수를 다르게 할 수 있어요. printf, scanf 같은 함수가 대표적인 예입니다.

stdarg.h 매크로 4총사

가변 인자 함수를 만들려면 stdarg.h 헤더 파일이 필요해요. 이 안에 있는 4가지 매크로를 순서대로 사용합니다.

1#include <stdio.h>
2#include <stdarg.h>
3
4// count개의 정수를 모두 더하는 함수
5int my_sum(int count, ...) {
6 va_list ap; // 1. 가변 인자 목록 선언
7 va_start(ap, count); // 2. count 다음부터 읽기 시작
8
9 int total = 0;
10 for (int i = 0; i < count; i++) {
11 total += va_arg(ap, int); // 3. 하나씩 꺼내기
12 }
13
14 va_end(ap); // 4. 정리
15 return total;
16}
17
18int main(void) {
19 printf("%d\n", my_sum(3, 10, 20, 30)); // 60
20 printf("%d\n", my_sum(5, 1, 2, 3, 4, 5)); // 15
21 return 0;
22}
실행 결과
60 15
5줄int my_sum(int count, ...) -- ...은 '여기에 인자가 더 올 수 있다'는 뜻이에요. count는 고정 인자예요.
6줄va_list ap -- 가변 인자들을 가리키는 포인터 변수를 선언해요.
7줄va_start(ap, count) -- count 뒤에 오는 인자들부터 읽겠다고 초기화해요.
11줄va_arg(ap, int) -- 다음 인자를 int 타입으로 하나 꺼내요. 호출할 때마다 다음 인자로 이동해요.
14줄va_end(ap) -- 사용이 끝나면 반드시 정리해야 해요. 짝을 맞추는 것이 중요해요.

vprintf는 va_list를 직접 받는 printf야. 가변 인자를 다른 가변 인자 함수에 전달할 때 유용해!

가변 인자 함수의 주의점

주의사항
1. 고정 인자가 최소 1개는 있어야 해요. (...)만으로는 안 돼요.
2. va_arg에서 잘못된 타입을 지정하면 쓰레기 값을 읽어요.
3. 인자 개수를 잘못 전달하면 위험해요 (컴파일러가 체크 못함).
4. va_start와 va_end는 반드시 짝을 맞춰야 해요.
가변 인자 함수는 컴파일러가 인자 타입이나 개수를 검사하지 못해. 그래서 실수하면 런타임에 버그가 생기니까 조심해야 해!

실전: 최대값 구하기

1#include <stdio.h>
2#include <stdarg.h>
3
4int my_max(int count, ...) {
5 va_list ap;
6 va_start(ap, count);
7
8 int max_val = va_arg(ap, int); // 첫 번째 값
9 for (int i = 1; i < count; i++) {
10 int val = va_arg(ap, int);
11 if (val > max_val) {
12 max_val = val;
13 }
14 }
15
16 va_end(ap);
17 return max_val;
18}
19
20int main(void) {
21 printf("max: %d\n", my_max(4, 30, 10, 50, 20));
22 printf("max: %d\n", my_max(2, 100, 200));
23 return 0;
24}

max: 50 max: 200

연습문제

OX 퀴즈

가변 인자 함수에서 ...만 있고 고정 인자가 없어도 된다.

정답: ❌ X
고정 인자가 최소 1개는 있어야 해요. va_start의 두 번째 인자로 마지막 고정 인자를 전달해야 하기 때문이에요.
객관식

stdarg.h에서 제공하는 매크로가 아닌 것은?

  1. va_list
  2. va_start
  3. va_count
  4. va_end
정답: 3
va_count라는 매크로는 없어요. stdarg.h는 va_list, va_start, va_arg, va_end를 제공해요. 인자 개수는 프로그래머가 직접 관리해야 해요.
단답형

va_arg(ap, int)는 어떤 역할을 하나요?

정답: 가변 인자 목록에서 다음 인자를 int 타입으로 하나 꺼낸다.
va_arg는 현재 위치의 인자를 지정된 타입으로 읽고, 포인터를 다음 인자로 이동시킵니다.

  1. 100
  2. 300
  3. 600
  4. 컴파일 오류
정답: 3
n=3이고 100+200+300=600이에요.
OX 퀴즈

vprintf는 va_list를 직접 받는 printf 버전이다.

정답: ⭕ O
vprintf, vfprintf, vsprintf 등은 va_list를 인자로 받아서 가변 인자를 다른 출력 함수에 전달할 때 사용해요.

15// 미니 printf
16void my_printf(const char *fmt, ...) {
17 va_list args;
18 va_start(args, fmt);
19 while (*fmt) {
20 if (*fmt == '%' && *(fmt+1)) {
21 fmt++;
22 switch(*fmt) {
23 case 'd': printf("%d", va_arg(args, int)); break;
24 case 'f': printf("%f", va_arg(args, double)); break;
25 case 's': printf("%s", va_arg(args, char*)); break;
26 case 'c': printf("%c", va_arg(args, int)); break;
27 case '%': printf("%%"); break;
28 }
29 } else putchar(*fmt);
30 fmt++;
31 }
32 va_end(args);
33}

← 인자 개수가 매번 다름! 어떻게?

가변 인자 4단계:
1. va_list args; → 인자 목록 변수 선언
2. va_start(args, last_fixed); → 시작
3. va_arg(args, int); → 하나씩 꺼내기
4. va_end(args); → 정리

⚠️ 타입 안전 없음!
va_arg(args, int)인데 실제로 double을 넘기면?
→ 쓰레기 값! 컴파일러가 못 잡음!
→ printf의 %d/%f가 타입을 알려주는 이유
va_list
가변 인자 목록. va_start로 시작, va_arg로 꺼내기, va_end로 정리. printf의 핵심 메커니즘!
vprintf
va_list를 받는 printf. 가변 인자 래퍼 함수에서 사용. logmsg(fmt,...) 안에서 vprintf(fmt,args)!
가변 인자 = printf의 비밀! va_start→va_arg→va_end. 인자를 하나씩 꺼내는 마법!
vprintf가 핵심! 자체 로그 함수에서 fmt+args를 그대로 printf에 전달. 래퍼의 비밀!

va_list 전달 패턴:
void wrapper(const char *fmt, ...) {
va_list args;
va_start(args, fmt);
vprintf(fmt, args); // v-접두사 함수에 전달!
va_end(args);
}

v-접두사 함수들:
vprintf, vsprintf, vsnprintf, vfprintf
→ va_list를 직접 받는 버전!

⚠️ 타입 안전:
GCC: __attribute__((format(printf,1,2)))
→ 포맷 문자열과 인자 타입 체크!

C11 _Generic 대안:
#define print(x) _Generic((x),
int: print_int, double: print_double)(x)
→ 타입별 자동 디스패치!
_Generic
C11 타입 제네릭. 컴파일 타임에 타입별 함수 선택. C의 제한적 오버로딩! 타입 안전 매크로.
format attribute
GCC format 속성. 포맷 문자열 인자 체크. printf %d에 문자열 넘기면 경고! 타입 안전.
vprintf = printf의 va_list 버전! 래퍼 함수 만들 때 필수. v 접두사 기억!
assert_eq(actual,expected,msg,...) 테스트 프레임워크가 가변 인자! 실전에서 진짜 쓰는 패턴!
