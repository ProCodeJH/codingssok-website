Lesson 18-12

문자열 자르기 (strtok)

strtok 함수로 문자열을 구분자 기준으로 잘라내는 방법을 배워요. CSV 파싱과 strtok의 함정도 알아봐요.

📚 이번에 배울 것

  • strtok 함수의 동작 원리를 이해해요.
  • 구분자 기준으로 문자열을 분리할 수 있어요.
  • CSV 데이터를 파싱할 수 있어요.
  • strtok의 함정(원본 변경, 중첩 불가)을 알고 피할 수 있어요.

✅ 시작 전 체크

  • 문자열과 char 배열의 기본 사용법을 알고 있다
  • NULL 포인터의 개념을 알고 있다
  • while 반복문을 사용할 수 있다
Part 1 진행률: 12%

쉼표로 구분된 데이터를 나누려면?

"Kim,25,Seoul" 같은 데이터에서 이름, 나이, 도시를 따로 뽑아내고 싶어. 이때 사용하는 함수가 strtok이야!

strtok(string tokenize)은 문자열을 구분자(delimiter) 기준으로 잘라서 하나씩 반환하는 함수예요. string.h에 들어 있어요.

1#include <stdio.h>
2#include <string.h>
3
4int main(void) {
5 char str[] = "Kim,25,Seoul";
6
7 // 첫 번째 호출: 문자열과 구분자 전달
8 char *token = strtok(str, ",");
9
10 while (token != NULL) {
11 printf("[%s]\n", token);
12 // 두 번째 호출부터: NULL 전달
13 token = strtok(NULL, ",");
14 }
15
16 return 0;
17}
실행 결과
[Kim] [25] [Seoul]
5줄char str[] -- 반드시 배열이어야 해요. 문자열 리터럴(char *)은 수정할 수 없어서 오류가 나요.
8줄strtok(str, ",") -- 첫 호출 시 원본 문자열과 구분자를 전달해요. 첫 번째 토큰을 반환해요.
13줄strtok(NULL, ",") -- 두 번째 호출부터는 첫 인자에 NULL을 넣어요. 내부에서 이전 위치를 기억하고 있어요.
10줄더 이상 토큰이 없으면 NULL을 반환해요. 이것으로 반복을 종료해요.
undefined

strtok의 위험한 함정 3가지

undefined
strtok은 원본 문자열을 직접 수정해! 구분자 위치에 \0을 넣어버려. 원본을 보존하려면 strcpy로 복사본을 만든 뒤 그걸 사용해야 해.

실전: CSV 파싱

CSV(Comma-Separated Values)는 엑셀 등에서 사용하는 데이터 형식이에요. 각 항목이 쉼표로 구분되어 있어요. strtok으로 쉽게 파싱할 수 있어요.

1#include <stdio.h>
2#include <string.h>
3#include <stdlib.h>
4
5int main(void) {
6 char data[] = "Kim,95,Seoul\nLee,88,Busan\nPark,72,Daegu";
7 char *line = strtok(data, "\n");
8
9 printf("%-10s %-6s %s\n", "Name", "Score", "City");
10 printf("-----------------------------\n");
11
12 while (line != NULL) {
13 // 복사본을 만들어서 사용
14 char copy[100];
15 strcpy(copy, line);
16
17 char *name = strtok(copy, ",");
18 char *score = strtok(NULL, ",");
19 char *city = strtok(NULL, ",");
20
21 if (name && score && city) {
22 printf("%-10s %-6d %s\n", name, atoi(score), city);
23 }
24
25 line = strtok(NULL, "\n");
26 }
27
28 return 0;
29}
위 코드에서 주의할 점: strtok을 중첩해서 쓰고 있어. 바깥 strtok(줄 분리)과 안쪽 strtok(쉼표 분리)이 충돌할 수 있으니 복사본을 꼭 만들어야 해!
[KOI] 입력 파싱에서 strtok 활용
KOI 같은 프로그래밍 대회에서 '공백으로 구분된 여러 숫자'를 입력받을 때 fgets로 한 줄 읽은 뒤 strtok으로 분리하는 방법이 유용해요. scanf만 쓰는 것보다 안전하고 유연해요.

연습문제

OX 퀴즈

strtok은 원본 문자열을 변경하지 않고 토큰을 반환한다.

정답: ❌ X
strtok은 구분자를 \0으로 바꿔서 원본 문자열을 직접 수정해요.
객관식

strtok의 두 번째 호출부터 첫 번째 인자로 전달해야 하는 것은?

  1. 원본 문자열
  2. 빈 문자열 ""
  3. NULL
  4. 구분자 문자열
정답: 3
두 번째 호출부터는 NULL을 전달해요. strtok이 내부 static 포인터로 이전 위치를 기억하고 있기 때문이에요.
단답형

strtok을 문자열 리터럴(char *s = "hello,world")에 직접 사용하면 왜 위험한가요?

정답: 문자열 리터럴은 읽기 전용 메모리에 저장되어 있어서, strtok이 구분자를 \0으로 바꾸려고 하면 오류가 발생한다.
char *s = "..."는 읽기 전용이고, char s[] = "..."는 배열(수정 가능)이에요.
객관식

다음 코드 실행 후 str 배열의 내용은? char str[] = "A-B-C"; strtok(str, "-");

  1. "A-B-C" (변화 없음)
  2. "A\0B-C"
  3. "A"
  4. "A\0B\0C"
정답: 2
첫 번째 호출에서 첫 번째 구분자(-)만 \0으로 바꿔요. 나머지는 아직 그대로에요.
OX 퀴즈

strtok을 두 개의 다른 문자열에 동시에 사용할 수 있다.

정답: ❌ X
strtok은 내부 static 포인터를 1개만 사용하므로 중첩 사용이 불가능해요. 한 문자열의 작업이 끝난 후에 다음 문자열을 처리해야 해요.

56// === 수식 토크나이저 ===
57typedef enum { T_NUM, T_OP, T_END } TokenType;
58typedef struct { TokenType type; double num; char op; } Token;
59
60Token next_token(const char **s) {
61 while (**s == ' ') (*s)++;
62 if (!**s) return (Token){T_END, 0, 0};
63 if (**s >= '0' && **s <= '9') {
64 double val = 0;
65 while (**s >= '0' && **s <= '9') val = val*10 + (*(*s)++ - '0');
66 return (Token){T_NUM, val, 0};
67 }
68 char c = *(*s)++;
69 return (Token){T_OP, 0, c};
70}

[심화 내용] 파서 vs 토크나이저
토크나이저 (Lexer):
문자열 → 토큰 배열
"3+42*7" → [3][+][42][*][7]
→ strtok의 고급 버전!

파서 (Parser):
토큰 → 구조 (트리)
[3][+][42*7] → AST (추상 구문 트리)

실전 파서 종류:
• 재귀 하강 (Recursive Descent)
• Pratt Parser (연산자 우선순위)
• LALR (yacc/bison)

strtok의 한계:
1. 원본 수정
2. 연속 구분자 무시
3. 정규식 불가
→ 직접 파서 작성이 필요한 경우 많음
tokenizer
토크나이저. 문자열→토큰. 컴파일러의 첫 단계(Lexer). strtok의 확장. 정규식 or 수동 구현.
AST
Abstract Syntax Tree. 코드의 구조를 트리로 표현. 컴파일러, 인터프리터의 핵심! 파서가 생성.
INI 파서 = strtok(\n으로 행분리) + strchr(=으로 key/value). 실전 파싱의 기본!

1#include <stdio.h>

3#include <ctype.h>
4#include <stdlib.h>

6// === 미니 JSON 토크나이저 ===
7typedef enum { J_STR, J_NUM, J_LBRACE, J_RBRACE, J_LBRACK, J_RBRACK, J_COLON, J_COMMA, J_END } JType;
8typedef struct { JType type; char str[100]; double num; } JTok;
9
10const char *jsrc;
11
12JTok jnext() {
13 while (isspace(*jsrc)) jsrc++;
14 if (!*jsrc) return (JTok){J_END,"",0};
15 char c = *jsrc++;
16 if (c == 123) return (JTok){J_LBRACE,"{",0}; // {
17 if (c == 125) return (JTok){J_RBRACE,"}",0}; // }
18 if (c == 91) return (JTok){J_LBRACK,"[",0}; // [
19 if (c == 93) return (JTok){J_RBRACK,"]",0}; // ]
20 if (c == 58) return (JTok){J_COLON,":",0}; // :
21 if (c == 44) return (JTok){J_COMMA,",",0}; // ,
22 if (c == 34) { // double quote
23 JTok t = {J_STR,"",0};
24 int i = 0;
25 while (*jsrc && *jsrc != 34) t.str[i++] = *jsrc++;
26 t.str[i] = 0;
27 if (*jsrc == 34) jsrc++;
28 return t;
29 }
30 if (isdigit(c) || c == 45) { // digit or minus
31 JTok t = {J_NUM,"",0};
32 char buf[50]; int i = 0;
33 buf[i++] = c;
34 while (isdigit(*jsrc) || *jsrc == 46) buf[i++] = *jsrc++;
35 buf[i] = 0;
36 t.num = atof(buf);
37 return t;
38 }
39 return (JTok){J_END,"",0};
40}

42void jtokenize(const char *json) {
43 jsrc = json;
44 JTok t;
45 const char *names[] = {"STR","NUM","{","}","[","]",":","COMMA","END"};
46 int indent = 0;
47 while ((t = jnext()).type != J_END) {
48 if (t.type == J_RBRACE || t.type == J_RBRACK) indent--;
49 for (int i = 0; i < indent; i++) printf(" ");
50 printf(" %s", names[t.type]);
51 if (t.type == J_STR) printf(" = \"%s\"", t.str);
52 if (t.type == J_NUM) printf(" = %g", t.num);
53 printf("\n");
54 if (t.type == J_LBRACE || t.type == J_LBRACK) indent++;
55 }
56}

7int parse_http_headers(const char *raw, Header *hdrs, int max) {
8 char buf[2000]; strncpy(buf, raw, 1999);
9 int count = 0;
10 char *line = strtok(buf, "\n");
11 while (line && count < max) {
12 while (*line == 32 || *line == 9) line++;
13 if (*line == 0 || *line == 13) { line = strtok(NULL, "\n"); continue; }
14 char *colon = strchr(line, 58);
15 if (colon) {
16 *colon = 0;
17 char *val = colon + 1;
18 while (*val == 32) val++;
19 // trim CR
20 int vl = strlen(val);
21 while (vl > 0 && (val[vl-1]==13||val[vl-1]==10)) val[--vl]=0;
22 strncpy(hdrs[count].name, line, 49);
23 strncpy(hdrs[count].value, val, 199);
24 count++;
25 }
26 line = strtok(NULL, "\n");
27 }
28 return count;
29}

3// 미니 정규식: . * 지원
4// . = 아무 문자 1개, * = 앞 문자 0개 이상
5int regex_match(const char *pattern, const char *text) {
6 if (*pattern == 0) return *text == 0;
7 // *가 다음에 있으면
8 if (*(pattern+1) == 42) { // *
9 // 0개 매치 시도
10 if (regex_match(pattern+2, text)) return 1;
11 // 1개 이상 매치 시도
12 while (*text && (*pattern == 46 || *pattern == *text)) {
13 text++;
14 if (regex_match(pattern+2, text)) return 1;
15 }
16 return 0;
17 }
18 // . 매치
19 if (*text && (*pattern == 46 || *pattern == *text))
20 return regex_match(pattern+1, text+1);
21 return 0;
22}

24// 와일드카드 매칭: ? = 아무 문자 1개, * = 아무 문자열
25int wildcard_match(const char *pattern, const char *text) {
26 if (*pattern == 0) return *text == 0;
27 if (*pattern == 42) { // *
28 return wildcard_match(pattern+1, text) || // * = 빈 문자열
29 (*text && wildcard_match(pattern, text+1)); // * = 1개 더 소비
30 }
31 if (*text && (*pattern == 63 || *pattern == *text)) // ? or exact
32 return wildcard_match(pattern+1, text+1);
33 return 0;
34}

2

3

4
char * 리터럴에 사용하면 안 된다 (char [] 배열 사용)
5
내부 static 포인터 때문에 중첩 사용이 불가능하다
