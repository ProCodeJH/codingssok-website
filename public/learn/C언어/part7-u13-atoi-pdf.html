Lesson 18-13

문자열-숫자 변환

atoi, atof로 문자열을 숫자로, sprintf, snprintf로 숫자를 문자열로 바꾸는 방법을 배워요. strtol로 안전하게 변환하는 법도 알아봐요.

📚 이번에 배울 것

  • atoi, atof로 문자열을 숫자로 변환할 수 있어요.
  • strtol, strtod로 안전하게 변환하고 에러를 체크할 수 있어요.
  • sprintf, snprintf로 숫자를 문자열로 바꿀 수 있어요.
  • 사용자 입력을 검증하는 방법을 이해해요.

✅ 시작 전 체크

  • 문자열(char 배열)의 기본 사용법을 알고 있다
  • ASCII 코드의 기본 개념을 알고 있다
  • 포인터의 기본 개념을 알고 있다
Part 1 진행률: 13%

문자열 "123"을 숫자 123으로 바꾸려면?

사용자가 키보드로 입력한 것은 항상 문자열이야. "123"은 문자 '1', '2', '3'이지 숫자 123이 아니야. 계산하려면 숫자로 바꿔야 해!

간단한 변환: atoi, atof

1#include <stdio.h>
2#include <stdlib.h> // atoi, atof가 여기에!
3
4int main(void) {
5 char s1[] = "123";
6 char s2[] = "3.14";
7 char s3[] = "99abc";
8 char s4[] = "hello";
9
10 printf("atoi(\"%s\") = %d\n", s1, atoi(s1)); // 123
11 printf("atof(\"%s\") = %.2f\n", s2, atof(s2)); // 3.14
12 printf("atoi(\"%s\") = %d\n", s3, atoi(s3)); // 99 (숫자 부분만)
13 printf("atoi(\"%s\") = %d\n", s4, atoi(s4)); // 0 (변환 실패)
14
15 return 0;
16}
실행 결과
atoi("123") = 123 atof("3.14") = 3.14 atoi("99abc") = 99 atoi("hello") = 0
atoi의 문제점: "hello"를 넣으면 0을 반환하는데, "0"을 넣어도 0을 반환해. 변환이 실패한 건지 진짜 0인지 구분할 수 없어!

안전한 변환: strtol

strtol(string to long)은 atoi보다 안전해요. 변환이 끝난 위치를 알려주고, 진법도 지정할 수 있어요.

1#include <stdio.h>
2#include <stdlib.h>
3
4int main(void) {
5 char s1[] = "123abc";
6 char *endptr;
7
8 long val = strtol(s1, &endptr, 10); // 10진법
9 printf("값: %ld\n", val); // 123
10 printf("나머지: \"%s\"\n", endptr); // "abc"
11
12 // 변환 성공 여부 확인
13 if (*endptr == '\0') {
14 printf("완전히 변환됨\n");
15 } else {
16 printf("일부만 변환됨 (남은 부분: %s)\n", endptr);
17 }
18
19 // 16진법 변환
20 printf("0xFF = %ld\n", strtol("FF", NULL, 16)); // 255
21 // 2진법 변환
22 printf("0b1010 = %ld\n", strtol("1010", NULL, 2)); // 10
23
24 return 0;
25}
실행 결과
값: 123 나머지: "abc" 일부만 변환됨 (남은 부분: abc) 0xFF = 255 0b1010 = 10

연습문제

OX 퀴즈

atoi("hello")의 반환값은 0이다.

정답: ⭕ O
atoi는 숫자로 시작하지 않는 문자열을 받으면 0을 반환해요. 하지만 이것이 변환 실패인지 진짜 0인지 구분할 수 없어요.
객관식

strtol("FF", NULL, 16)의 반환값은?

  1. 0
  2. 15
  3. 255
  4. 컴파일 오류
정답: 3
16진법으로 FF = 15*16 + 15 = 255에요.
단답형

sprintf와 snprintf의 차이점을 한 문장으로 설명하세요.

정답: snprintf는 버퍼 크기를 지정하여 버퍼 오버플로를 방지하지만, sprintf는 크기 제한이 없어 위험하다.
snprintf(buf, size, fmt, ...)에서 size가 최대 기록 바이트 수를 제한해요.
객관식

다음 중 가장 안전한 문자열-숫자 변환 방법은?

  1. atoi를 사용한다
  2. atof를 사용한다
  3. strtol을 사용하고 endptr을 확인한다
  4. sscanf를 사용한다
정답: 3
strtol은 endptr로 변환 실패 여부를 확인할 수 있고, errno로 범위 초과도 체크할 수 있어서 가장 안전해요.
OX 퀴즈

atoi("99abc")는 99를 반환한다.

정답: ⭕ O
atoi는 문자열 앞부분의 숫자만 변환하고 나머지는 무시해요. 99까지만 읽고 abc는 버려요.

쉽게 이해하기: 문자 → 숫자 변환
핵심 원리: '0'~'9'의 ASCII 값이 연속!
'0'=48, '1'=49, ..., '9'=57

'5' - '0' = 53 - 48 = 5 ← 숫자로 변환!

"123"을 123으로:
result = 0
result = 0×10 + 1 = 1
result = 1×10 + 2 = 12
result = 12×10 + 3 = 123

atoi vs strtol:
atoi: 간단하지만 에러 감지 불가
strtol: 에러 감지, 진법 지정 가능
→ 실전에서는 strtol 추천!
strtol
문자열→long. strtol("42",&end,10). 에러 감지 가능! 진법 지정(2~36). atoi보다 안전!
ASCII trick
'5'-'0'=5. ASCII 연속 배치 덕분! 'A'-'A'=0, 'a'-'a'=0. 문자→숫자 변환의 기본.
atoi의 핵심: result = result*10 + (c-'0'). 한 자리씩 10배 곱하고 더하기!
오버플로 체크! "9999999999"를 atoi하면? INT_MAX 반환. 안전한 구현의 핵심!

7long parse_number(const char *s, int base) {
8 while (isspace(*s)) s++;
9 int sign = 1;
10 if (*s == '-') { sign = -1; s++; }
11 else if (*s == '+') s++;
12
13 // 접두사 자동 감지

16 else if (s[0]=='0' && (s[1]=='b'||s[1]=='B')) { base=2; s+=2; }
17 else if (s[0]=='0') { base=8; s++; }
18 else base = 10;
19 }
20
21 long result = 0;
22 while (*s) {
23 int digit;
24 if (*s >= '0' && *s <= '9') digit = *s - '0';

26 else if (*s >= 'A' && *s <= 'F') digit = *s - 'A' + 10;

36void int_to_str(long n, int base, char *buf) {

44 if (neg) tmp[i++] = '-';
45 int j = 0;
46 while (i > 0) buf[j++] = tmp[--i];
47 buf[j] = 0;
48}

53 const char *p = s;

55 while (isspace(*p)) p++;
56 if (*p=='K'||*p=='k') return (long)(val * 1024);
57 if (*p=='M'||*p=='m') return (long)(val * 1024 * 1024);
58 if (*p=='G'||*p=='g') return (long)(val * 1024 * 1024 * 1024);

[심화 내용] 문자열 파싱 심화
strtol이 atoi보다 좋은 이유:
long val = strtol("42abc", &end, 10);
→ val=42, *end='a' (어디서 멈췄는지 알 수 있음!)
→ errno로 오버플로 감지 가능!

진법 접두사 관례:
0x → 16진 (C/Java/Python/JS)
0b → 2진 (GCC 확장, C++14)
0 → 8진 (C/Java, Python은 0o)

파싱 패턴:
1. 공백 건너뛰기 (isspace)
2. 부호 처리 (+/-)
3. 접두사 감지 (0x/0b/0)
4. 자릿수 변환 (digit = c - '0')
5. 누적 (result = result * base + digit)
strtol
문자열→long. 진법 지정(2~36). 끝 포인터로 파싱 위치 확인. errno로 오버플로 감지! atoi 상위 호환.
base 36
36진법. 0-9 + A-Z = 36자. strtol(s,NULL,36). 짧은 URL, 해시 인코딩에 활용!
parse_number: 0x→16진, 0b→2진, 0→8진, 기본→10진. 접두사 하나로 진법 자동 감지!

11 int vi = 0;

13 if (isdigit(*s)) {
14 vals[vi] = vals[vi]*10 + (*s - 48);
15 n++;
16 } else {
17 if (n > 0) { vi++; n = 0; }
18 }
19 s++;
20 }
21 if (n > 0) vi++;
22 dt.y=vals[0]; dt.m=vals[1]; dt.d=vals[2];

24 return dt;

27// duration 파서: "2h30m15s" -> seconds
28int parse_duration(const char *s) {
29 int total=0, cur=0;
30 while (*s) {
31 if (isdigit(*s)) cur = cur*10 + (*s-48);
32 else if (*s==104) { total+=cur*3600; cur=0; } // h
33 else if (*s==109) { total+=cur*60; cur=0; } // m
34 else if (*s==115) { total+=cur; cur=0; } // s
35 else if (*s==100) { total+=cur*86400; cur=0; } // d
36 s++;
37 }
38 return total+cur;
39}
