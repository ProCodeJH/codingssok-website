파일을 이용해 학생 정보를 추가, 조회, 수정, 삭제하는 CRUD 프로그램을 만들어요.

핵심 포인트 정리

CRUD 핵심 기법
Create: fopen("ab")로 파일 끝에 추가, fwrite로 구조체 저장
Read: fopen("rb")로 읽기, fread로 구조체 단위로 읽기
Update: fopen("r+b")로 읽기+쓰기, fseek으로 위치 이동 후 fwrite
Delete: 논리적 삭제 (active=0으로 표시), 물리적으로는 남아있음

연습문제

데이터가 많아지면 배열 대신 파일이나 데이터베이스를 사용해야 해. 하지만 기본 원리는 같아 -- 저장, 검색, 수정, 삭제!
OX 퀴즈

fopen("ab")의 'a'는 append(추가) 모드를 의미한다.

정답: ⭕ O
'a'는 파일 끝에 데이터를 추가하는 모드예요. 'b'는 바이너리 모드를 의미해요.
객관식

fseek(fp, -sizeof(Student), SEEK_CUR)의 의미는?

  1. 파일의 시작으로 이동한다
  2. 현재 위치에서 Student 크기만큼 앞으로 이동한다
  3. 현재 위치에서 Student 크기만큼 뒤로 이동한다
  4. 파일의 끝으로 이동한다
정답: 3
음수 오프셋은 현재 위치에서 뒤로(파일 시작 방향) 이동하는 것이에요. 방금 읽은 레코드 위치로 돌아가서 덮어쓰기 위해 사용해요.
단답형

논리적 삭제란 무엇인가요?

정답: 데이터를 실제로 파일에서 제거하지 않고, 삭제 표시(active=0)만 해두는 방식이다.
물리적으로 파일을 재구성하는 것보다 간단하고 빠르지만, 파일 크기는 줄어들지 않아요.
객관식

strstr(s.name, keyword)는 어떤 역할을 하나?

  1. 두 문자열이 같은지 비교한다
  2. s.name에서 keyword가 포함되어 있는지 찾는다
  3. s.name을 keyword로 복사한다
  4. 두 문자열을 이어붙인다
정답: 2
strstr은 문자열 안에서 부분 문자열을 찾는 함수예요. 찾으면 포인터를, 못 찾으면 NULL을 반환해요.
객관식

구조체 배열에서 특정 필드로 검색할 때 가장 기본적인 알고리즘은?

  1. 이진 탐색
  2. 해시 탐색
  3. 선형 탐색
  4. 트리 탐색
정답: 2
정렬되지 않은 구조체 배열에서 특정 필드 값을 찾을 때는 처음부터 끝까지 순서대로 비교하는 선형 탐색이 가장 기본적입니다.

5// === 간단한 SQL 파서 ===
6typedef enum { CMD_SELECT, CMD_INSERT, CMD_DELETE, CMD_UPDATE, CMD_UNKNOWN } CmdType;
7
8typedef struct {
9 CmdType type;
10 char table[50];
11 char columns[5][20];
12 int ncols;
13 char where_col[20];
14 char where_val[50];
15 int has_where;
16} SqlCmd;
17
18CmdType parse_cmd_type(const char *sql) {
19 if (strncmp(sql,"SELECT",6)==0||strncmp(sql,"select",6)==0) return CMD_SELECT;
20 if (strncmp(sql,"INSERT",6)==0||strncmp(sql,"insert",6)==0) return CMD_INSERT;
21 if (strncmp(sql,"DELETE",6)==0||strncmp(sql,"delete",6)==0) return CMD_DELETE;
22 if (strncmp(sql,"UPDATE",6)==0||strncmp(sql,"update",6)==0) return CMD_UPDATE;
23 return CMD_UNKNOWN;
24}
25
26void parse_sql(const char *sql, SqlCmd *cmd) {
27 memset(cmd, 0, sizeof(SqlCmd));
28 cmd->type = parse_cmd_type(sql);
29
30 // 간단한 SELECT 파싱
31 if (cmd->type == CMD_SELECT) {
32 const char *from = strstr(sql, "FROM");
33 if (!from) from = strstr(sql, "from");
34 if (from) {
35 sscanf(from + 5, "%s", cmd->table);
36 }
37 const char *where = strstr(sql, "WHERE");
38 if (!where) where = strstr(sql, "where");
39 if (where) {
40 cmd->has_where = 1;
41 sscanf(where + 6, "%s = %s", cmd->where_col, cmd->where_val);
42 }
43 }
44}

46void print_cmd(SqlCmd *cmd) {

49 printf(" Table: %s\n", cmd->table);
50 if (cmd->has_where)
51 printf(" Where: %s = %s\n", cmd->where_col, cmd->where_val);

54// === 해시 인덱스 ===
55#define HASH_SIZE 16
56
57typedef struct HashNode {
58 char key[50];
59 int row_id;
60 struct HashNode *next;
61} HashNode;
62
63HashNode *hash_table[HASH_SIZE];
64
65unsigned int hash(const char *s) {
66 unsigned int h = 0;
67 while (*s) h = h * 31 + *s++;
68 return h % HASH_SIZE;
69}
70
71void index_insert(const char *key, int row_id) {
72 unsigned int h = hash(key);
73 HashNode *node = (HashNode*)malloc(sizeof(HashNode));
74 strncpy(node->key, key, 49);
75 node->row_id = row_id;
76 node->next = hash_table[h];
77 hash_table[h] = node;
78}
79
80int index_find(const char *key) {
81 unsigned int h = hash(key);
82 for (HashNode *n = hash_table[h]; n; n = n->next)
83 if (strcmp(n->key, key) == 0) return n->row_id;
84 return -1;
85}

6typedef struct { int id; char name[20]; int val; int deleted; } Record;
7
8// === 테이블 ===
9Record table[MAX];
10int nrows = 0;
11
12// === WAL (Write-Ahead Log) ===
13typedef enum { WAL_INSERT, WAL_UPDATE, WAL_DELETE } WalType;
14typedef struct { WalType type; int row_id; Record old_val; Record new_val; } WalEntry;
15WalEntry wal[MAX];
16int wal_size = 0;
17int committed = 0;
18
19void wal_log(WalType type, int row_id, Record old_v, Record new_v) {
20 wal[wal_size++] = (WalEntry){type, row_id, old_v, new_v};
21}
22
23// === CRUD ===
24int db_insert(const char *name, int val) {
25 int id = nrows;
26 Record r = {id, "", val, 0};
27 strncpy(r.name, name, 19);
28 Record empty = {0,"",0,0};
29 wal_log(WAL_INSERT, id, empty, r);
30 table[nrows++] = r;
31 return id;
32}
33
34void db_update(int id, int new_val) {
35 if (id < 0 || id >= nrows) return;
36 Record old = table[id];
37 table[id].val = new_val;
38 wal_log(WAL_UPDATE, id, old, table[id]);
39}
40
41void db_delete(int id) {
42 if (id < 0 || id >= nrows) return;
43 Record old = table[id];
44 table[id].deleted = 1;
45 wal_log(WAL_DELETE, id, old, table[id]);
46}
47
48void db_select_all() {
49 printf(" ID Name Val\n");
50 printf(" ----------------------\n");
51 for (int i = 0; i < nrows; i++)
52 if (!table[i].deleted)
53 printf(" %-4d %-12s %d\n", table[i].id, table[i].name, table[i].val);
54}
55
56// === COMMIT / ROLLBACK ===
57void db_commit() {
58 committed = wal_size;
59 printf(" COMMIT (%d operations)\n", committed);
60}
61
62void db_rollback() {
63 printf(" ROLLBACK: undoing %d operations\n", wal_size - committed);
64 for (int i = wal_size-1; i >= committed; i--) {
65 WalEntry *e = &wal[i];
66 if (e->type == WAL_INSERT) {
67 table[e->row_id].deleted = 1;
68 nrows--;
69 } else {
70 table[e->row_id] = e->old_val;
71 }
72 }
73 wal_size = committed;
74}
75
76void print_wal() {
77 const char *types[] = {"INSERT","UPDATE","DELETE"};
78 printf("\n === WAL ===\n");
79 for (int i = 0; i < wal_size; i++)
80 printf(" [%d] %s row=%d\n", i, types[wal[i].type], wal[i].row_id);
81}
82
83int main() {
84 printf("=== Mini DB with Transactions ===\n\n");
85
86 db_insert("Alice", 100);
87 db_insert("Bob", 200);
88 db_insert("Charlie", 300);
89 printf("After INSERT x3:\n"); db_select_all();
90 db_commit();
91
92 // 트랜잭션 시작
93 printf("\n--- New transaction ---\n");
94 db_update(1, 999);
95 db_delete(2);
96 printf("After UPDATE+DELETE:\n"); db_select_all();
97
98 // 롤백!
99 db_rollback();
100 printf("\nAfter ROLLBACK:\n"); db_select_all();
101
102 print_wal();
103 return 0;
104}

[심화 내용] 트랜잭션과 WAL
트랜잭션 = 원자적 작업 단위 COMMIT: 변경 확정 ROLLBACK: 변경 취소 WAL 원리: 1. 변경 전 원본값(old) 기록 2. 변경 실행 3. COMMIT → WAL 비움 4. ROLLBACK → WAL 역순 재생! UNDO vs REDO: UNDO 로그: 원본값 저장 → ROLLBACK용 REDO 로그: 새 값 저장 → 크래시 복구용 실전 DB는 둘 다 사용! 실전 적용: SQLite: journal 파일 or WAL 모드 PostgreSQL: WAL (pg_wal/) MySQL: redo log + undo log
transaction
트랜잭션. 원자적 작업 단위. COMMIT=확정, ROLLBACK=취소. ACID 보장! 은행 이체의 핵심.
ROLLBACK: WAL을 역순으로! old값 복원. UPDATE→원래값, DELETE→부활, INSERT→삭제!
query optimizer
쿼리 옵티마이저. SQL→실행 계획 최적화. 인덱스 선택, 조인 순서. 비용 기반 최적화!
MVCC
Multi-Version Concurrency Control. 읽기/쓰기 동시. 스냅샷 격리. PostgreSQL, InnoDB의 핵심!
page-based storage
페이지 기반 저장. 4KB/8KB 페이지 단위. 디스크 I/O 최적화. B-트리의 노드=페이지!
MVCC: 읽기가 쓰기를 차단하지 않음! 스냅샷 격리. PostgreSQL이 빠른 이유!
normalization
정규화. 데이터 중복 제거. 1NF/2NF/3NF. 테이블 분리+외래키. 데이터 무결성의 핵심!
NoSQL comparison
NoSQL vs SQL. MongoDB(문서)/Redis(키값)/Neo4j(그래프). 유연하지만 ACID 약함. 용도에 맞게!
SQL vs NoSQL: 정형 데이터=SQL(MySQL), 유연한 데이터=NoSQL(MongoDB). 용도에 맞게!
prepared statement
prepared statement. SQL 주입 방지! 파라미터 바인딩. "SELECT * WHERE id=?" + bind(42). 보안 필수!
connection pool
커넥션 풀. DB 연결 재사용. 연결 비용 절약! 최대 N개 유지. 웹 서버 성능의 핵심!
sharding
샤딩. 데이터를 여러 DB에 분산. user_id%4로 4개 서버. 수평 확장! 대규모 서비스 필수.
SQL 주입 방지: 절대 문자열 연결 금지! prepared statement로 파라미터 바인딩. 보안의 기본!
database lock
DB 락. 행 수준/테이블 수준/DB 수준. 비관적(선 잠금) vs 낙관적(충돌 감지). 동시성의 핵심!
비관적 락: 먼저 잠그고 작업. 낙관적 락: 작업 후 충돌 체크. 읽기 많으면 낙관적이 유리!
LSM tree
LSM 트리. 쓰기 최적화 자료구조. 메모리→디스크 병합. LevelDB, RocksDB, Cassandra의 핵심!
prepared statement
프리페어드 스테이트먼트. SQL 인젝션 방지! 쿼리 컴파일 1번, 실행 N번. 성능+보안 동시 향상.
cursor
커서. 결과 집합을 한 행씩 순회. 메모리 절약! 대량 데이터 처리. DECLARE CURSOR FOR SELECT...
prepared statement
프리페어드 스테이트먼트. SQL 인젝션 방지! 쿼리 템플릿+파라미터 바인딩. 보안의 기본!
database normalization
정규화. 1NF~3NF. 중복 제거, 이상(anomaly) 방지. 하지만 JOIN 증가→비정규화도 실전에서!
SQL 인젝션: WHERE name="Kim"; DROP TABLE--;. 프리페어드 스테이트먼트로 방지!
CRUD
Create(생성), Read(조회), Update(수정), Delete(삭제). 데이터 관리의 4가지 기본 동작.
논리적 삭제
데이터를 실제로 지우지 않고 삭제 표시만 하는 방식. 복구가 가능하다.
레코드(record)
데이터베이스에서 하나의 데이터 항목. 여기서는 Student 구조체 하나가 한 레코드.
바이너리 모드 ("rb", "wb")
파일을 바이트 단위로 정확하게 읽고 쓰는 모드. 구조체를 파일에 저장할 때 사용한다.
