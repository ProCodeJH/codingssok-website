동적 프로그래밍이란

동적 프로그래밍(Dynamic Programming, DP)은 큰 문제를 작은 부분 문제로 나누고, 부분 문제의 답을 저장하여 중복 계산을 피하는 알고리즘 기법입니다. 실버에서 배운 메모이제이션이 DP의 핵심입니다.

DP가 가능하려면 두 가지 조건이 필요합니다.

조건의미예시
최적 부분 구조큰 문제의 최적해가 작은 문제의 최적해로 구성피보나치: f(n) = f(n-1) + f(n-2)
중복 부분 문제같은 작은 문제가 여러 번 등장f(5)를 구할 때 f(3)이 2번 계산

DP의 두 가지 접근법

그래프란 무엇인가

그래프(graph)는 정점(vertex, 노드)과 간선(edge)으로 이루어진 자료구조입니다. 지하철 노선도, SNS 친구 관계, 도로 네트워크가 모두 그래프입니다. 실버에서 2차원 격자를 DFS로 탐색한 것도 그래프 탐색의 일종이었습니다.

용어의미예시
정점(vertex)그래프의 점지하철역, 사람, 도시
간선(edge)정점 사이의 연결노선, 친구관계, 도로
무방향 그래프간선에 방향 없음A—B (양방향)
방향 그래프간선에 방향 있음A→B (일방향)
가중치 그래프간선에 비용/거리 있음A→B (거리 5)

DFS — 깊이 우선 탐색

DFS는 한 방향으로 끝까지 갔다가 돌아오는 탐색입니다. 재귀 또는 스택으로 구현합니다.

Pythondfs_graph.py
1def dfs(graph, start):
2 visited = [False] * len(graph)
3 result = []
4
5 def _dfs(v):
6 visited[v] = True
7 result.append(v)
8 for next_v in sorted(graph[v]):
9 if not visited[next_v]:
10 _dfs(next_v)
11
12 _dfs(start)
13 return result
14
15# graph = 위에서 만든 인접 리스트
16print(dfs(graph, 1)) # [1, 2, 4, 3, 5]
호출방문인접 정점다음 호출
_dfs(1)1 방문[2, 3]_dfs(2)
_dfs(2)2 방문[1, 4] → 1방문됨_dfs(4)
_dfs(4)4 방문[2, 3] → 2방문됨_dfs(3)
_dfs(3)3 방문[1, 4, 5] → 1,4방문됨_dfs(5)
_dfs(5)5 방문[3] → 3방문됨돌아감

BFS — 너비 우선 탐색

BFS는 가까운 정점부터 차례로 탐색합니다. 큐(deque)로 구현합니다. 가중치 없는 그래프에서 최단 경로를 구할 때 BFS를 사용합니다.

Pythonbfs_graph.py
1from collections import deque
2
3def bfs(graph, start):
4 visited = [False] * len(graph)
5 visited[start] = True
6 queue = deque([start])
7 result = []
8
9 while queue:
10 v = queue.popleft()
11 result.append(v)
12 for next_v in sorted(graph[v]):
13 if not visited[next_v]:
14 visited[next_v] = True
15 queue.append(next_v)
16
17 return result
18
19print(bfs(graph, 1)) # [1, 2, 3, 4, 5]

BFS 최단 경로 — 미로 탈출

N×M 격자에서 (0,0)부터 (N-1,M-1)까지의 최단 거리를 구합니다. 1은 이동 가능, 0은 벽입니다.

Pythonbfs_maze.py
1from collections import deque
2
3def bfs_maze(grid, n, m):
4 visited = [[False]*m for _ in range(n)]
5 dist = [[0]*m for _ in range(n)]
6 visited[0][0] = True
7 dist[0][0] = 1
8 queue = deque([(0, 0)])
9 dx = [-1, 1, 0, 0]
10 dy = [0, 0, -1, 1]
11
12 while queue:
13 x, y = queue.popleft()
14 for d in range(4):
15 nx, ny = x + dx[d], y + dy[d]
16 if 0 <= nx < n and 0 <= ny < m:
17 if not visited[nx][ny] and grid[nx][ny] == 1:
18 visited[nx][ny] = True
19 dist[nx][ny] = dist[x][y] + 1
20 queue.append((nx, ny))
21
22 return dist[n-1][m-1]
23
24grid = [
25 [1, 0, 1, 1, 1],
26 [1, 0, 1, 0, 1],
27 [1, 1, 1, 0, 1],
28 [0, 0, 0, 0, 1]
29]
30print(bfs_maze(grid, 4, 5)) # 12
4방향 이동 패턴 dx = [-1, 1, 0, 0]
dy = [0, 0, -1, 1]

상(x-1), 하(x+1), 좌(y-1), 우(y+1)

이 패턴은 격자 BFS/DFS 문제에서 거의 반드시 사용됩니다.
범위 검사: 0 <= nx < n and 0 <= ny < m

연결 요소 세기

그래프에서 서로 연결된 정점들의 그룹을 연결 요소(connected component)라 합니다. 모든 정점을 순회하면서 방문하지 않은 정점에서 DFS/BFS를 시작하면 됩니다.

Pythonconnected.py
1def count_components(n, graph):
2 visited = [False] * (n + 1)
3 count = 0
4 for v in range(1, n + 1):
5 if not visited[v]:
6 # BFS로 연결된 정점 모두 방문
7 queue = deque([v])
8 visited[v] = True
9 while queue:
10 cur = queue.popleft()
11 for nxt in graph[cur]:
12 if not visited[nxt]:
13 visited[nxt] = True
14 queue.append(nxt)
15 count += 1
16 return count

이분 탐색 — O(log n) 검색

이분 탐색(binary search)은 정렬된 데이터에서 원하는 값을 O(log n)에 찾는 알고리즘입니다. 매번 탐색 범위를 절반으로 줄입니다.

Pythonbinary_search.py
1def binary_search(arr, target):
2 left, right = 0, len(arr) - 1
3 while left <= right:
4 mid = (left + right) // 2
5 if arr[mid] == target:
6 return mid
7 elif arr[mid] < target:
8 left = mid + 1
9 else:
10 right = mid - 1
11 return -1 # 못 찾음
12
13nums = [1, 3, 5, 7, 9, 11, 13, 15]
14print(binary_search(nums, 7)) # 3 (인덱스)
15print(binary_search(nums, 6)) # -1

107377 == 7 → 찾음!

target=6을 찾는 경우:

회차leftrightmidarr[mid]비교

202133 < 6 → left=2
322255 < 6 → left=3
432——left > right → 못 찾음
이분 탐색의 위력 순차 탐색: O(n) — 100만 개에서 100만 번
이분 탐색: O(log n) — 100만 개에서 20번

전제 조건: 데이터가 정렬되어 있어야 합니다.
정렬 O(n log n) + 이분 탐색 O(log n) = O(n log n)

파라메트릭 서치 — 이분 탐색의 응용

파라메트릭 서치(parametric search)는 '최적화 문제를 결정 문제로 바꾸어' 이분 탐색으로 푸는 기법입니다. "최대/최소 값을 구하라" → "이 값이 가능한가?"로 바꿉니다.

예제: 나무 M미터를 가져가려 합니다. 절단기 높이 H를 설정하면 H보다 높은 부분만 잘립니다. M미터를 가져갈 수 있는 최대 H는?

그리디 알고리즘 — 탐욕적 선택

그리디(greedy) 알고리즘은 매 순간 가장 좋아 보이는 선택을 하는 기법입니다. 항상 최적해를 보장하지는 않지만, 특정 조건에서는 최적해를 줍니다.

그리디 예제 1 — 동전 교환

그리디가 안 되는 경우 동전이 [500, 400, 100, 50, 10]이고 800원이라면?
그리디: 500+100+100+100 = 4개
최적: 400+400 = 2개

큰 동전이 작은 동전의 배수가 아니면 그리디가 실패합니다.
이런 경우 DP를 사용해야 합니다.

DP 심화 — 0/1 배낭 문제

배낭(knapsack) 문제는 DP의 대표 문제입니다. N개의 물건에 무게와 가치가 있고, 배낭 용량 K가 주어질 때, 배낭에 넣을 수 있는 물건의 최대 가치를 구합니다.

단계내용
1. 정의dp[i][w] = i번째 물건까지 고려, 용량 w일 때 최대 가치
2. 점화식넣지 않음: dp[i-1][w] 넣음 (w≥wi): dp[i-1][w-wi] + vi dp[i][w] = max(두 경우)
3. 초기값dp[0][w] = 0 (물건 0개면 가치 0)
Pythonknapsack.py
1def knapsack(n, k, items):
2 dp = [[0] * (k + 1) for _ in range(n + 1)]
3 for i in range(1, n + 1):
4 w, v = items[i - 1]
5 for j in range(k + 1):
6 dp[i][j] = dp[i-1][j] # 넣지 않음
7 if j >= w:
8 dp[i][j] = max(dp[i][j], dp[i-1][j-w] + v) # 넣음
9 return dp[n][k]
10
11items = [(6, 13), (4, 8), (3, 6), (5, 12)]
12print(knapsack(4, 7, items)) # 14
물건무게가치용량 7에서 최대 가치
1번 (6,13)61313
2번 (4,8)4813
3번 (3,6)3614 (2번+3번: 8+6)
4번 (5,12)51214

DP 심화 — 최장 증가 부분수열(LIS)

LIS(Longest Increasing Subsequence)는 수열에서 순서를 유지하면서 값이 증가하는 가장 긴 부분수열의 길이를 구하는 문제입니다.

Pythonlis_dp.py
1# O(n²) DP 풀이
2def lis(arr):
3 n = len(arr)
4 dp = [1] * n # dp[i] = arr[i]로 끝나는 LIS 길이
5 for i in range(1, n):
6 for j in range(i):
7 if arr[j] < arr[i]:
8 dp[i] = max(dp[i], dp[j] + 1)
9 return max(dp)
10
11arr = [10, 20, 10, 30, 20, 50]
12print(lis(arr)) # 4 (10, 20, 30, 50)

16. 디버깅

아래 이분 탐색 코드가 무한루프에 빠진다. 원인을 찾고 수정하시오.

[버그 코드]
def search(arr, target):
left, right = 0, len(arr) - 1
while left <= right:
mid = (left + right) // 2
if arr[mid] < target:
left = mid
elif arr[mid] > target:
right = mid
else:
return mid
return -1
