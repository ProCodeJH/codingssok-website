key 매개변수 — 정렬 기준 지정

key에 함수를 전달하면 그 함수의 반환값을 기준으로 정렬합니다. 이것이 실버 정렬의 핵심입니다.

Pythonsort_key.py
1# 문자열 길이 기준 정렬

3print(sorted(words, key=len))
4# ['date', 'apple', 'banana', 'cherry']
5
6# 절대값 기준 정렬
7nums = [-5, 3, -1, 4, -2]
8print(sorted(nums, key=abs))
9# [-1, -2, 3, 4, -5]
줄코드역할
3key=len각 요소에 len()을 적용한 결과(문자열 길이)를 기준으로 정렬합니다.
8key=abs각 요소의 절대값을 기준으로 정렬합니다. 원래 값은 유지됩니다.
요소key=len 적용정렬 순서
"banana"63번째
"apple"52번째
"cherry"64번째 (같으면 원래 순서)
"date"41번째
안정 정렬(stable sort) 파이썬의 정렬은 안정 정렬입니다. key 값이 같으면 원래 순서를 유지합니다.
"banana"와 "cherry"는 둘 다 길이 6이므로 원래 순서(banana → cherry)가 유지됩니다.
이 성질은 다중 기준 정렬에서 매우 중요합니다.

다중 기준 정렬

key에서 튜플을 반환하면 첫 번째 기준으로 정렬하고, 같으면 두 번째 기준으로 정렬합니다.

Pythonmulti_sort.py
1students = [
2 ("코딩쏙", 90), ("파이썬", 85),
3 ("알고", 90), ("자바", 85)
4]
5
6# 1순위: 점수 내림차순, 2순위: 이름 오름차순
7result = sorted(students, key=lambda x: (-x[1], x[0]))
8print(result)
9# [('알고', 90), ('코딩쏙', 90), ('자바', 85), ('파이썬', 85)]
줄코드역할
7-x[1]점수에 음수를 붙여 내림차순 효과. 90 → -90, 85 → -85이므로 -90 < -85, 즉 90이 먼저.
7x[0]점수가 같을 때 이름을 오름차순(사전순)으로. '알고' < '코딩쏙'.
음수 트릭 숫자를 내림차순으로 정렬하려면 -를 붙입니다.
key=lambda x: (-x[1], x[0]) → 점수 내림 + 이름 오름

문자열은 -를 붙일 수 없으므로 reverse=True와 조합하거나 두 번 정렬합니다.

정렬 알고리즘 원리 — 선택 정렬

sorted()가 내부에서 어떻게 동작하는지 이해하기 위해 가장 간단한 정렬 알고리즘인 선택 정렬을 직접 구현합니다.

Pythonselection_sort.py
1def selection_sort(arr):
2 n = len(arr)
3 for i in range(n - 1):
4 min_idx = i
5 for j in range(i + 1, n):
6 if arr[j] < arr[min_idx]:
7 min_idx = j
8 arr[i], arr[min_idx] = arr[min_idx], arr[i]
9 return arr
10
11nums = [64, 25, 12, 22, 11]
12print(selection_sort(nums))
i탐색 범위최소값교환배열 상태
0[64,25,12,22,11]11(idx=4)64↔11[11, 25, 12, 22, 64]
1[25,12,22,64]12(idx=2)25↔12[11, 12, 25, 22, 64]
2[25,22,64]22(idx=3)25↔22[11, 12, 22, 25, 64]
3[25,64]25(idx=3)교환 없음[11, 12, 22, 25, 64]
실행 결과
[11, 12, 22, 25, 64]
시간 복잡도 선택 정렬: O(n²) — 느림, 학습용
파이썬 sorted(): O(n log n) — 팀소트(Timsort), 매우 빠름

대회에서는 sorted()를 사용하되, 원리를 이해해야 응용 문제를 풀 수 있습니다.

재귀의 구조

모든 재귀 함수에는 두 가지가 반드시 있어야 합니다.

구성 요소역할없으면?
기저 조건 (base case)재귀를 멈추는 조건무한 재귀 → RecursionError
재귀 호출 (recursive call)자기 자신을 더 작은 입력으로 호출재귀가 아닌 일반 함수
Pythonfactorial.py
1def factorial(n):
2 if n <= 1: # 기저 조건
3 return 1
4 return n * factorial(n - 1) # 재귀 호출
5
6print(factorial(5)) # 120
호출nn <= 1?반환값
factorial(5)5False5 × factorial(4) = 5 × 24 = 120
factorial(4)4False4 × factorial(3) = 4 × 6 = 24
factorial(3)3False3 × factorial(2) = 3 × 2 = 6
factorial(2)2False2 × factorial(1) = 2 × 1 = 2
factorial(1)1True1 (기저 조건)
줄코드역할
2if n <= 1: return 1기저 조건. n이 1 이하이면 더 이상 호출하지 않고 1을 반환합니다.
4n * factorial(n-1)재귀 호출. 문제 크기를 1씩 줄여 나갑니다.

피보나치 수열 — 재귀의 함정

Pythonfibonacci.py
1# 순수 재귀 — 느림!
2def fib(n):
3 if n <= 1:
4 return n
5 return fib(n - 1) + fib(n - 2)
6
7print(fib(10)) # 55
8# fib(30)부터 매우 느려짐

fib(5)를 호출하면 fib(3)이 2번, fib(2)가 3번 중복 계산됩니다. 이를 해결하는 방법이 메모이제이션입니다.

Pythonfib_memo.py
1# 메모이제이션 — 빠름!
2memo = {}
3def fib(n):
4 if n in memo:
5 return memo[n]
6 if n <= 1:
7 memo[n] = n
8 else:
9 memo[n] = fib(n - 1) + fib(n - 2)
10 return memo[n]
11
12print(fib(50)) # 12586269025 (즉시 계산)
메모이제이션 이미 계산한 결과를 딕셔너리에 저장하여 중복 계산을 방지합니다.
시간 복잡도: O(2ⁿ) → O(n)으로 극적으로 개선.
이것이 골드 단계에서 배울 동적 프로그래밍(DP)의 핵심입니다.

집합(set) — 중복 없는 모음

집합(set)은 중복을 허용하지 않고 순서가 없는 자료구조입니다. 중복 제거, 포함 여부 검사, 집합 연산(교집합, 합집합)에 사용합니다.

Pythonset_basic.py
1# 생성
2a = {1, 2, 3, 2, 1}
3print(a) # {1, 2, 3} — 중복 제거
4
5# 리스트 → 집합 (중복 제거)
6nums = [1, 3, 2, 3, 1, 4]
7unique = set(nums)
8print(unique) # {1, 2, 3, 4}
9
10# 집합 → 리스트
11print(sorted(unique)) # [1, 2, 3, 4]

집합 연산

Pythonset_ops.py
1a = {1, 2, 3, 4, 5}
2b = {3, 4, 5, 6, 7}
3
4print(a & b) # 교집합: {3, 4, 5}
5print(a | b) # 합집합: {1, 2, 3, 4, 5, 6, 7}
6print(a - b) # 차집합: {1, 2}
7print(a ^ b) # 대칭차: {1, 2, 6, 7}
연산기호메서드의미
교집합a & ba.intersection(b)둘 다 포함
합집합a | ba.union(b)하나라도 포함
차집합a - ba.difference(b)a에만 포함
대칭차a ^ ba.symmetric_difference(b)한쪽에만 포함
in 검사 속도 리스트: x in list → O(n) (느림)
집합: x in set → O(1) (빠름)

포함 여부를 많이 검사해야 할 때 리스트 대신 집합을 사용하면 속도가 크게 개선됩니다.
이것은 대회에서 시간 초과를 해결하는 핵심 기법입니다.

큐(queue)는 먼저 넣은 것을 먼저 꺼내는 자료구조입니다. 줄 서기와 같습니다. 리스트의 pop(0)으로 구현할 수 있지만 느립니다(O(n)). collections.deque를 사용합니다.

스택 vs 큐 스택(LIFO): append() + pop() → 마지막 것 먼저
큐(FIFO): append() + popleft() → 첫 번째 것 먼저

BFS(너비 우선 탐색) → 큐(deque)

deque 추가 기능

Pythondeque_extra.py
1from collections import deque
2
3d = deque([1, 2, 3])
4
5d.appendleft(0) # 앞에 추가: deque([0, 1, 2, 3])
6d.pop() # 뒤에서 꺼냄: 3, deque([0, 1, 2])

큐 활용 — 요세푸스 문제

N명이 원형으로 앉아 있습니다. K번째 사람을 반복적으로 제거합니다. 마지막에 남는 사람은 누구입니까? 이것이 유명한 요세푸스 문제입니다.

Pythonjosephus.py
1from collections import deque
2
3def josephus(n, k):
4 queue = deque(range(1, n + 1))
5 result = []
6 while queue:
7 for _ in range(k - 1):
8 queue.append(queue.popleft()) # k-1명 뒤로
9 result.append(queue.popleft()) # k번째 제거
10 return result
11
12print(josephus(7, 3))
13# [3, 6, 2, 7, 5, 1, 4]

종합 복습 — 실버 전체 정리

실버 단계에서 배운 모든 내용을 한눈에 정리합니다.

1. 정렬 sorted(list) → 새 리스트, list.sort() → 원본 수정
key=함수 → 정렬 기준 지정
lambda x: x[1] → 이름 없는 함수
다중 기준: key=lambda x: (-x[1], x[0])
안정 정렬: key 값이 같으면 원래 순서 유지
선택 정렬 O(n²), 파이썬 sorted() O(n log n)
2. 2차원 리스트 grid[행][열]로 접근
생성: [[0]*m for _ in range(n)] (독립 행)
순회: 중첩 for문 또는 for row in grid:
입력: [list(map(int, input().split())) for _ in range(n)]
주의: [[0]*m]*n은 공유 참조 — 사용 금지
3. 리스트 컴프리헨션 [식 for 변수 in 반복대상]
[식 for 변수 in 반복대상 if 조건]
{키: 값 for ...} → 딕셔너리 컴프리헨션
{식 for ...} → 집합 컴프리헨션
가독성이 떨어지면 일반 for문 사용
4. 재귀 기저 조건(base case) + 재귀 호출(recursive call)
기저 조건 없으면 RecursionError
메모이제이션: 중복 계산 방지 → O(2ⁿ) → O(n)
활용: 팩토리얼, 피보나치, DFS, 분할 정복
5. 스택과 큐 스택(LIFO): append() + pop() → DFS, 괄호 검사, 후위 표기법
큐(FIFO): deque의 append() + popleft() → BFS, 시뮬레이션
deque: O(1) 양쪽 추가/제거, rotate()
주의: list의 pop(0)은 O(n) — deque 사용 필수
6. 집합(set) 중복 불허, 순서 없음
연산: & 교집합, | 합집합, - 차집합, ^ 대칭차
in 검사: O(1) (리스트는 O(n))
생성: set(), {1, 2, 3}
주의: {} → dict, set() → 빈 집합

자주 틀리는 함정 TOP 7
