UNIT S-02 불 대수

왜 논리식을 줄여야 할까?

CPU 안에는 게이트가 수십억 개 들어 있어요. 게이트 하나를 줄이면 별것 아닌 것 같지만, 그게 수백만 번 반복되면 엄청난 차이가 나요. 불 대수는 논리식을 수학적으로 줄이는 방법이에요.

[예시] 간소화의 효과 원래 논리식: F = (A AND B) OR (A AND NOT B)
→ AND 게이트 2개 + OR 게이트 1개 + NOT 게이트 1개 = 4개

간소화 후: F = A
→ 게이트 0개 (그냥 A를 연결하면 끝!)

왜 같을까? A가 1이면 B가 뭐든 결과가 1이고, A가 0이면 B가 뭐든 결과가 0이니까요.
[핵심] 불 대수란? 불 대수(Boolean Algebra) = 0과 1만 사용하는 수학 체계

일반 수학에서 +, -, x 로 숫자를 계산하듯이,
불 대수에서는 AND, OR, NOT으로 참/거짓을 계산해요.

표기법:
• A AND B = A · B = AB (곱셈처럼)
• A OR B = A + B (덧셈처럼)
• NOT A = A' = A̅ (따옴표 또는 윗줄)

기본 법칙 1 — 항등법칙

항등법칙은 '아무것도 바꾸지 않는 값'에 대한 법칙이에요. 숫자에 0을 더하거나 1을 곱해도 변하지 않는 것과 같아요.

[법칙 1] 항등법칙 (Identity Law) OR 항등: A + 0 = A   (0을 OR해도 변하지 않아요)
AND 항등: A · 1 = A   (1을 AND해도 변하지 않아요)

0은 OR의 항등원, 1은 AND의 항등원이에요.

비유: '아무 일도 안 일어남 OR 비가 옴' = '비가 옴'
비유: '항상 참 AND 비가 옴' = '비가 옴'

기본 법칙 2 — 지배법칙

지배법칙은 '결과를 무조건 하나로 만드는 값'에 대한 법칙이에요.

[법칙 2] 지배법칙 (Domination Law) OR 지배: A + 1 = 1   (1을 OR하면 무조건 1)
AND 지배: A · 0 = 0   (0을 AND하면 무조건 0)

1은 OR의 지배원, 0은 AND의 지배원이에요.

비유: '뭐든지 OR 항상 참' = '항상 참'
비유: '뭐든지 AND 절대 거짓' = '절대 거짓'

기본 법칙 3 — 멱등법칙

[법칙 3] 멱등법칙 (Idempotent Law) OR: A + A = A   (같은 값을 OR해도 변하지 않아요)
AND: A · A = A   (같은 값을 AND해도 변하지 않아요)

'비가 옴 OR 비가 옴' = '비가 옴'
'비가 옴 AND 비가 옴' = '비가 옴'

중복은 의미가 없어요. 하나로 줄일 수 있어요.

기본 법칙 4 — 보원법칙

[법칙 4] 보원법칙 (Complement Law) OR: A + A' = 1   (반대끼리 OR하면 무조건 1)
AND: A · A' = 0   (반대끼리 AND하면 무조건 0)

A가 1이면 A'는 0 → 1 OR 0 = 1, 1 AND 0 = 0
A가 0이면 A'는 1 → 0 OR 1 = 1, 0 AND 1 = 0

어떤 경우든 성립해요! 이 법칙은 간소화에서 자주 쓰여요.

기본 법칙 5 — 이중 부정

[법칙 5] 이중 부정 (Double Negation) (A')' = A

NOT을 두 번 하면 원래대로 돌아와요.

'비가 안 오는 게 아니다' = '비가 온다'
NOT(NOT(1)) = NOT(0) = 1 ← 원래대로!

기본 법칙 6 — 교환법칙, 결합법칙, 분배법칙

일반 수학의 덧셈, 곱셈과 비슷한 법칙들이에요. 순서를 바꾸거나, 묶거나, 펼칠 수 있어요.

[법칙 6] 교환·결합·분배법칙 교환법칙 (Commutative):
A + B = B + A   /   A · B = B · A
순서를 바꿔도 결과가 같아요.

결합법칙 (Associative):
(A + B) + C = A + (B + C)
(A · B) · C = A · (B · C)
괄호를 옮겨도 결과가 같아요.

분배법칙 (Distributive):
A · (B + C) = A·B + A·C   ← 일반 수학과 같음
A + (B · C) = (A+B) · (A+C)   ← 일반 수학에는 없는 법칙!
[주의] 불 대수만의 특별한 분배법칙 일반 수학: 2 + (3 × 4) = 2 + 12 = 14
(2+3) × (2+4) = 5 × 6 = 30   ← 다르죠?

불 대수: A + (B · C) = (A+B) · (A+C)   ← 이게 성립해요!

확인: A=1, B=0, C=1
왼쪽: 1 + (0·1) = 1 + 0 = 1
오른쪽: (1+0)·(1+1) = 1·1 = 1 ← 같아요!

기본 법칙 7 — 흡수법칙

[법칙 7] 흡수법칙 (Absorption Law) 흡수 1: A + A·B = A
흡수 2: A · (A+B) = A

증명 (흡수 1):
A + A·B
= A·1 + A·B   (항등법칙: A = A·1)
= A·(1 + B)   (분배법칙으로 A를 묶기)
= A·1         (지배법칙: 1+B = 1)
= A           (항등법칙)

'비가 옴 OR (비가 옴 AND 우산 없음)' = '비가 옴'
→ 비가 오면 우산이 있든 없든 결과는 '비가 옴'이에요.

기본 법칙 8 — 드모르간 법칙

S-01에서 이미 배운 법칙이에요. 불 대수에서 가장 중요한 법칙 중 하나예요.

[법칙 8] 드모르간 법칙 (De Morgan's Law) 법칙 1: (A · B)' = A' + B'

간소화 실전 연습

[실전 1] AB + AB' = ? AB + AB'
= A(B + B')   ← 분배법칙 (A를 묶음)
= A · 1       ← 보원법칙 (B + B' = 1)
= A          ← 항등법칙

게이트: AND 2개 + OR 1개 + NOT 1개 → 0개로 줄어듦!
[실전 2] A + A'B = ? A + A'B
= (A + A')(A + B)   ← 분배법칙 (OR 버전)
= 1 · (A + B)       ← 보원법칙
= A + B           ← 항등법칙

또는 다른 방법:
A + A'B
= A·1 + A'B       ← 항등법칙
= A(1+B) + A'B   ← 지배법칙으로 1+B=1이므로 이 방향은 복잡

첫 번째 방법이 더 깔끔해요. 여러 방법 중 짧은 걸 고르면 돼요.
[실전 3] AB + A'C + BC = ? 합의 정리 (Consensus Theorem)

AB + A'C + BC
= AB + A'C + BC(A + A')   ← 보원법칙: A+A'=1 → BC·1=BC
= AB + A'C + ABC + A'BC   ← 분배법칙
= AB(1+C) + A'C(1+B)   ← 재배열 후 묶기
= AB + A'C              ← 지배법칙: 1+C=1

BC 항이 통째로 사라졌어요! 이걸 합의 정리라고 해요.
게이트 1개 + NOT 1개를 줄인 거예요.

카르노 맵 — 눈으로 간소화하기

카르노 맵(K-map)은 진리표를 특별한 형태의 표로 바꿔서, 인접한 1들을 묶으면 간소화된 식이 바로 나오는 방법이에요.

[핵심] 2변수 카르노 맵 변수 A, B에 대한 카르노 맵:

        B=0   B=1
A=0  |  m0  |  m1  |
A=1  |  m2  |  m3  |

m0 = A'B', m1 = A'B, m2 = AB', m3 = AB

규칙:
1. 진리표에서 출력이 1인 칸에 1을 써요
2. 인접한 1들을 2의 거듭제곱(1,2,4) 크기로 묶어요
3. 묶인 영역에서 변하지 않는 변수만 남겨요
위치AB민텀 번호민텀 표현
왼쪽 위00m0A'B'
오른쪽 위01m1A'B
왼쪽 아래10m2AB'
오른쪽 아래11m3AB
[예제] F = A'B + AB를 카르노 맵으로 진리표: F(0,0)=0, F(0,1)=1, F(1,0)=0, F(1,1)=1

        B=0   B=1
A=0  |   0   |   1   |
A=1  |   0   |   1   |

1이 세로로 나란히 → B열 전체가 1
→ A는 변하고(0→1), B는 변하지 않음(항상 1)
→ 결과: F = B

검증: A'B + AB = B(A'+A) = B·1 = B ← 맞아요!
[예제] F = A'B' + A'B + AB를 카르노 맵으로 진리표: F(0,0)=1, F(0,1)=1, F(1,0)=0, F(1,1)=1

        B=0   B=1
A=0  |   1   |   1   |
A=1  |   0   |   1   |

묶기 1: 윗줄 전체 (A=0) → A'
묶기 2: 오른쪽 열 전체 (B=1) → B

결과: F = A' + B

검증: 원래식 3항 → 간소화 후 2항. 게이트 줄었어요!

[심화] 3변수 카르노 맵

변수가 3개면 카르노 맵이 2x4 표가 돼요. 중요한 점은 열 순서가 00, 01, 11, 10 (그레이 코드)이라는 거예요. 인접한 칸끼리 1비트만 달라야 하거든요.

[심화] 3변수 카르노 맵 구조           BC=00   BC=01   BC=11   BC=10
A=0   |   m0   |   m1   |   m3   |   m2   |
A=1   |   m4   |   m5   |   m7   |   m6   |

주의: 열 순서가 00→01→11→10 (10이 아니라 11 먼저!)
이유: 인접한 칸이 1비트만 다르게 하기 위해서

묶기 규칙: 1, 2, 4, 8개씩 직사각형으로 묶기
양쪽 끝도 인접해요 (맵이 원통처럼 연결됨)
[심화 예제] F(A,B,C) = Σm(0,2,4,6) m0=A'B'C', m2=A'BC', m4=AB'C', m6=ABC'

          BC=00   BC=01   BC=11   BC=10
A=0   |   1    |   0    |   0    |   1    |

C언어 조건문 간소화

C조건문 간소화 전
1#include <stdio.h>
2
3int main(void) {
4 int score = 85;
5 int attendance = 1; // 1 = 출석 양호
6
7 // 복잡한 조건

9 (score >= 60 && !attendance && score >= 90)) {
10 printf("합격!\n");
11 }
12

이 조건을 불 대수로 정리해 볼게요. A = (score >= 60), B = attendance로 놓으면:

불 대수로 조건문 간소화 원래: (A AND B) OR (A AND NOT B AND C)
여기서 C = (score >= 90)이고, C가 참이면 A도 반드시 참

정리:
AB + AB'C
= A(B + B'C)   ← A를 묶음
= A((B+B')(B+C))   ← 분배법칙
= A(1·(B+C))   ← 보원법칙
= A(B + C)   ← 항등법칙

즉: score >= 60 AND (출석 양호 OR score >= 90)

[주의] 조건 뒤집기 규칙 NOT(A AND B) = (NOT A) OR (NOT B) ← AND가 OR로!
NOT(A OR B) = (NOT A) AND (NOT B) ← OR이 AND로!

NOT(>=) → <   NOT(>) → <=
NOT(==) → !=   NOT(<=) → >

예: NOT(age >= 12 AND hasTicket)
= (age < 12) OR (!hasTicket)

역사 — 불 대수의 탄생

[역사] 조지 불 — '사고의 법칙' 조지 불(1815~1864)은 14살에 그리스어를 독학할 만큼 천재였어요. 1854년에 '사고의 법칙에 대한 연구'라는 책을 써서, 인간의 논리적 사고를 수학 공식으로 표현할 수 있다고 주장했어요. 당시 사람들은 '이게 뭐에 쓸모가 있지?' 했지만, 80년 뒤 섀넌이 이걸로 컴퓨터를 만들었어요. 불 대수가 없었으면 지금의 컴퓨터, 스마트폰, 인터넷도 없었을 거예요.
[역사] 모리스 카르노 — 눈으로 푸는 간소화 모리스 카르노(Maurice Karnaugh, 1924~)는 벨 연구소의 엔지니어였어요. 1953년에 카르노 맵을 발명했어요. 복잡한 대수 계산 없이 표에 1을 채우고 묶기만 하면 최적의 논리식이 나오는 방법이에요. 전화 교환기 회로를 설계할 때 사용하려고 만들었다고 해요.

[KOI] 불 대수 실전 기출 유형

[KOI 실전 문제 1] 논리식 간소화 문제: F = ABC + ABC' + A'BC를 간소화하시오.

풀이:
ABC + ABC' + A'BC
= AB(C + C') + A'BC   ← 분배법칙 (처음 두 항에서 AB 묶기)
= AB·1 + A'BC        ← 보원법칙
= AB + A'BC          ← 항등법칙
= B(A + A'C)         ← B를 묶기
= B(A + C)           ← A + A'C = A + C
= AB + BC
[KOI 실전 문제 2] 카르노 맵 문제: F(A,B) = Σm(0,1,3)을 카르노 맵으로 간소화하시오.

        B=0   B=1
A=0  |   1   |   1   |
A=1  |   0   |   1   |

묶기 1: 윗줄 전체 → A'
묶기 2: 오른쪽 열 → B

F = A' + B
[KOI 실전 문제 3] 게이트 수 최소화 문제: F = A'B'C + A'BC + AB'C + ABC를 최소 게이트로 구현하시오.

카르노 맵:
          BC=00   BC=01   BC=11   BC=10
A=0   |   0    |   1    |   1    |   0    |
A=1   |   0    |   1    |   1    |   0    |

C=1인 열에 모두 1 → F = C
원래 4개 항 → 변수 1개로!
필요한 게이트: 0개 (C를 직접 출력에 연결)

용어 정리

용어설명
불 대수 (Boolean Algebra)0과 1만 사용하는 수학 체계. AND, OR, NOT 연산으로 논리식을 계산하고 간소화해요.
항등법칙A+0=A, A·1=A. 결과를 바꾸지 않는 값(0, 1)과의 연산.
보원법칙A+A'=1, A·A'=0. 반대 값끼리 연산하면 항상 일정한 결과가 나와요.
흡수법칙A+AB=A. 큰 항이 작은 항을 '흡수'해서 사라지게 해요.
드모르간 법칙(AB)'=A'+B', (A+B)'=A'B'. NOT이 안으로 들어가면 AND↔OR이 바뀌어요.
카르노 맵 (K-map)진리표를 특별한 표로 바꿔서, 인접한 1들을 묶어 논리식을 간소화하는 방법.
간소화 (Simplification)같은 결과를 내면서 논리식을 더 짧게 줄이는 것. 게이트 수가 줄어들어요.
합의 정리 (Consensus Theorem)AB+A'C+BC = AB+A'C. 세 번째 항이 불필요한 경우를 찾아 제거해요.

3. 간소화 = 같은 결과, 더 적은 게이트 → 더 빠르고 저렴한 회로
4. 카르노 맵: 인접한 1을 묶으면 눈으로 간소화할 수 있어요
