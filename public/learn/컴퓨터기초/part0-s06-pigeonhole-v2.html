비둘기와 집

너무 당연한 이야기 같죠? 하지만 이 단순한 사실이 수학과 컴퓨터 과학에서 놀라울 정도로 강력한 도구가 돼요.

[핵심] 비둘기집 원리 (Pigeonhole Principle) n+1개의 물건을 n개의 상자에 넣으면,
적어도 한 상자에는 2개 이상의 물건이 들어간다.

더 일반적으로:
n개의 물건을 k개의 상자에 넣으면,
적어도 한 상자에는 ⌈n/k⌉개 이상 들어간다.

(⌈ ⌉는 올림. 예: ⌈7/3⌉ = ⌈2.33⌉ = 3)
[팁] 왜 '비둘기집'일까? 독일 수학자 디리클레가 1834년에 이 원리를 처음 공식화했어요.
독일어로 'Schubfachprinzip' (서랍 원리)라고 했는데,
영어로 번역하면서 'Pigeonhole Principle'이 되었어요.

비둘기장(pigeonhole)은 비둘기가 한 마리씩 들어가는 칸이에요.
비둘기가 칸보다 많으면, 어떤 칸에는 2마리 이상 들어갈 수밖에 없어요!
비둘기 수집 수비둘기 > 집?결론

55X (5 = 5)모든 집에 1마리씩 가능

[예시] 양말 문제 서랍에 빨간 양말, 파란 양말, 검정 양말이 섞여 있어요.
어둠 속에서 양말을 꺼내는데, 같은 색 한 쌍을 확실히 얻으려면 몇 개를 꺼내야 할까요?

상자(색깔) = 3개
같은 상자에 2개 이상 → 4개를 꺼내면 확실해요!

3개만 꺼내면 운이 나쁘면 빨/파/검 하나씩일 수 있어요.
4개째를 꺼내면 반드시 어떤 색이 2개가 돼요.
[예시] 생일 문제 1년은 365일이에요.
366명이 모이면, 반드시 생일이 같은 쌍이 존재해요!

비둘기(사람) = 366마리
집(날짜) = 365개
→ 어떤 날짜에 반드시 2명 이상

S-05에서는 '확률이 높다'고 했지만,
비둘기집 원리로는 '반드시(100%)'라고 말할 수 있어요!
[예시] 머리카락 문제 사람의 머리카락은 최대 약 20만 개예요.
서울 인구는 약 950만 명이에요.

비둘기(사람) = 950만
집(머리카락 수) = 20만+1가지 (0개~20만 개)

950만 ÷ 20만 ≈ 47.5
→ 서울에는 머리카락 개수가 정확히 같은 사람이 최소 48명 있어요!

누군지는 모르지만, 반드시 존재한다는 걸 증명할 수 있어요.

나머지와 비둘기집 원리

비둘기집 원리의 '상자'로 나머지를 사용하면, 수학에서 놀라운 결과를 얻을 수 있어요.

정수5로 나눈 몫5로 나눈 나머지배정된 상자
303상자 3
712상자 2
1121상자 1
1530상자 0
2040상자 0 (충돌!)
2550상자 0 (충돌!)
[예시] 같은 나머지 찾기 6개의 정수가 있어요: 3, 7, 11, 15, 20, 25

5로 나눈 나머지를 구하면:
3→3, 7→2, 11→1, 15→0, 20→0, 25→0

나머지의 종류(상자): 0, 1, 2, 3, 4 → 5개
숫자(비둘기): 6개

6 > 5이므로, 반드시 나머지가 같은 쌍이 존재해요!
(15, 20, 25가 모두 나머지 0)
[핵심] 나머지 + 비둘기집 원리 임의의 n+1개의 정수 중에서,
차이가 n의 배수인 쌍이 반드시 존재해요!

증명:
n으로 나눈 나머지는 0, 1, 2, ..., n-1 → n가지(상자)
정수는 n+1개(비둘기)
→ 나머지가 같은 쌍이 반드시 존재
→ 두 수의 차이는 n의 배수!
[예시] 증명: 6개 중 차이가 5의 배수인 쌍 존재 아무 정수 6개를 골라요: 17, 8, 33, 41, 52, 29

5로 나눈 나머지:
17→2, 8→3, 33→3, 41→1, 52→2, 29→4

나머지가 같은 쌍: (17, 52)→나머지 2, (8, 33)→나머지 3
• 52 - 17 = 35 = 5 × 7 ✓ (5의 배수!)
• 33 - 8 = 25 = 5 × 5 ✓ (5의 배수!)

어떤 6개를 골라도, 반드시 이런 쌍이 있어요.

연속 합과 비둘기집 원리

더 놀라운 응용을 볼게요. 연속된 수의 합에도 비둘기집 원리가 숨어 있어요.

부분합값5로 나눈 나머지나머지 0?같은 나머지 존재?
S0 (빈 합)00O -> 바로 해당 없음 (빈 합)-

[증명] n개 정수 중 연속 부분합이 n의 배수 양의 정수 a₁, a₂, ..., aₙ이 있을 때,
연속 부분합 중 n의 배수인 것이 반드시 존재해요!

증명:
부분합: S₁ = a₁, S₂ = a₁+a₂, ..., Sₙ = a₁+...+aₙ
n으로 나눈 나머지를 보면:

경우 1: 어떤 Sᵢ의 나머지가 0 → 끝! (S₁~Sᵢ가 n의 배수)

경우 2: 나머지가 모두 1~n-1 (n-1가지 상자에 n개 비둘기)
→ 비둘기집 원리로 나머지가 같은 Sᵢ, Sⱼ 존재 (i < j)
→ Sⱼ - Sᵢ = aᵢ₊₁ + ... + aⱼ 의 나머지가 0
→ 이 연속 부분합이 n의 배수! □

무손실 압축의 한계

[핵심] 모든 파일을 줄일 수는 없다 무손실 압축 프로그램이 모든 파일을 더 작게 만들 수 있을까요?

불가능해요! 비둘기집 원리로 증명할 수 있어요.

n비트 파일의 가짓수: 2ⁿ개
(n-1)비트 이하 파일의 가짓수: 2⁰ + 2¹ + ... + 2ⁿ⁻¹ = 2ⁿ - 1개

2ⁿ개(비둘기) > 2ⁿ-1개(집)
→ 반드시 서로 다른 두 파일이 같은 압축 결과를 가져요
→ 압축을 풀면 원본을 복구할 수 없어요 → 모순!

따라서 어떤 압축 알고리즘이든, 압축하면 오히려 커지는 파일이 반드시 존재해요.

배열에서 중복 찾기

비둘기집 원리의 가장 유명한 프로그래밍 응용이에요.

[예시] 중복 숫자 찾기 문제 1~n 범위의 정수가 n+1개 들어있는 배열이 있어요.
반드시 중복이 있다는 걸 비둘기집 원리로 알 수 있어요!

비둘기(숫자) = n+1개
집(가능한 값) = n개 (1~n)

예: [3, 1, 4, 1, 5] → 값 범위 1~4, 5개 숫자
→ 비둘기집 원리에 의해 중복 존재 → 1이 중복!

C 코드 — 비둘기집으로 중복 찾기

C1~n에서 중복 찾기 (카운팅)
1#include <stdio.h>
2#include <string.h>
3
4int find_duplicate(int arr[], int size) {
5 // 값 범위가 1~(size-1)이므로 size 크기 배열 필요
6 int count[100001];
7 memset(count, 0, sizeof(count));
8
9 for (int i = 0; i < size; i++) {
10 count[arr[i]]++;
11 if (count[arr[i]] > 1) {
12 return arr[i]; // 중복 발견!
13 }
14 }
15 return -1; // 비둘기집 원리에 의해 여기 올 수 없음
16}
17
18int main(void) {
19 int arr[] = {3, 1, 4, 1, 5, 9, 2, 6, 5};
20 int size = 9; // 값 범위 1~8, 9개 원소
21
22 int dup = find_duplicate(arr, size);
23 printf("중복 숫자: %d\n", dup);
24
25 return 0;
26}

C 코드 — 배열 합 문제 (부분합 + 비둘기집)

Cn의 배수인 연속 부분합 찾기
1#include <stdio.h>
2
3int main(void) {
4 int arr[] = {3, 7, 2, 5, 8};
5 int n = 5;
6
7 // 부분합의 나머지를 계산
8 int prefix_mod[6]; // prefix_mod[0] = 0
9 prefix_mod[0] = 0;
10
11 printf("부분합 나머지:\n");
12 int sum = 0;
13 for (int i = 0; i < n; i++) {
14 sum += arr[i];
15 prefix_mod[i + 1] = sum % n;
16 printf(" S[%d] = %d, 나머지 = %d\n",
17 i + 1, sum, prefix_mod[i + 1]);
18 }
19
20 // 비둘기집: n+1개 나머지 중 같은 것 찾기
21 for (int i = 0; i <= n; i++) {
22 for (int j = i + 1; j <= n; j++) {
23 if (prefix_mod[i] == prefix_mod[j]) {
24 printf("\na[%d]~a[%d]의 합이 %d의 배수!\n",
25 i + 1, j, n);
26 int subsum = 0;
27 for (int k = i + 1; k <= j; k++) {
28 subsum += arr[k - 1];
29 }
30 printf("합 = %d = %d × %d\n",
31 subsum, n, subsum / n);
32 return 0;
33 }
34 }
35 }
36
37 return 0;
38}
실행 결과
부분합 나머지: S[1] = 3, 나머지 = 3 S[2] = 10, 나머지 = 0 a[1]~a[2]의 합이 5의 배수! 합 = 10 = 5 × 2
[코드 해설] • prefix_mod[0] = 0: 빈 부분합의 나머지를 포함해야 해요
• 나머지가 같은 두 지점 i, j를 찾으면 → a[i+1]~a[j]의 합이 n의 배수
• 비둘기집 원리에 의해 반드시 찾을 수 있어요!

수학적 퍼즐 — 체스판 위의 기사

[예시] 체스판 위의 기사 (나이트) 8×8 체스판에 9개의 나이트를 놓으면,
서로 공격할 수 있는 쌍이 반드시 존재할까요?

체스판을 2×2 블록 16개로 나눠요.
하지만 이것만으로는 부족해요 (16 > 9).

대신 체스판을 4개 영역으로 나누면:
비둘기(나이트) = 9개
집(영역) = 4개 → ⌈9/4⌉ = 3
→ 한 영역에 3개 이상의 나이트
→ 같은 영역의 나이트끼리 공격 가능한 위치가 반드시 존재!

친구 문제 — 파티에서의 악수

[예시] 6명 파티의 친구/낯선 사람 문제: 6명이 파티에 모였어요. 아는 사이이거나 모르는 사이예요.
반드시 서로 아는 3명 또는 서로 모르는 3명이 존재할까요?

증명 (비둘기집 원리):
한 사람 A를 고르면, 나머지 5명과의 관계는 '안다' 또는 '모른다'
상자 2개에 5개(비둘기) → 한 상자에 최소 3명

경우 1: A가 아는 사람이 3명 이상 (B, C, D라 하자)
• B, C, D 중 서로 아는 쌍이 있으면 → 서로 아는 3명 완성
• 없으면 → B, C, D는 서로 모르는 3명!

경우 2: A가 모르는 사람이 3명 이상 → 같은 논리

어느 쪽이든 반드시 서로 아는 3명 또는 모르는 3명이 존재해요!
R(s,t)값 또는 범위의미상태
R(3,3)66명이면 서로 아는 3명 또는 모르는 3명 보장정확히 알려짐
R(3,4)99명이면 아는 3명 또는 모르는 4명 보장정확히 알려짐
R(3,5)1414명이면 아는 3명 또는 모르는 5명 보장정확히 알려짐
R(4,4)1818명이면 아는 4명 또는 모르는 4명 보장정확히 알려짐
R(4,5)2525명이면 아는 4명 또는 모르는 5명 보장정확히 알려짐
R(5,5)43~48아직 정확한 값을 아무도 모른다!미해결
R(6,6)102~165범위만 알려져 있다미해결
[팁] 램지 이론 이 문제는 램지 이론(Ramsey Theory)의 시작이에요.

'충분히 큰 구조에서는 반드시 특정 패턴이 나타난다'는 이론이에요.

R(3,3) = 6: 6명이면 '서로 아는 3명' 또는 '서로 모르는 3명' 보장
R(4,4) = 18: 18명이면 '서로 아는 4명' 또는 '서로 모르는 4명' 보장
R(5,5) = ? : 아직 아무도 모릅니다!

에르되시: '만약 외계인이 R(5,5)를 알려달라고 하면, 우리는 모든 힘을 합쳐 계산해야 할 것이다. 하지만 R(6,6)을 요구하면, 차라리 외계인과 싸우는 게 나을 것이다.'

• IPv6: 128비트 → 3.4×10³⁸개 주소 (사실상 무한)

IPv4의 IP 부족은 비둘기집 원리의 직접적인 결과예요.

[KOI 유형] 문제 2 — 양말 최소 개수 문제: 서랍에 N가지 색 양말이 있다. 같은 색 양말을 K쌍 확실히 얻으려면 최소 몇 개를 꺼내야 하는지 출력하시오.

입력: 3 2 (3색, 2쌍)
출력: 7

풀이:
1쌍 보장: N + 1 = 4개 (최악 시 모두 다른 색 + 1개)
하지만 2쌍은 단순히 2배가 아니에요.
최악: 각 색 1개씩 + 3개 더 → 3 + 2×(2-1) + 2 = ...
정확히는: N × (K-1) + K + (N-K) = N + K × (대응 계산)
답: N × 1 + K × 1 ... → 일반화: N(K-1) + 1개

3색, 2쌍: 3×(2-1)+1 = 4... 아니에요.
같은 색 2개 = 1쌍. 2쌍 = 같은 색 2개씩 2종류.
최악: 각 색 1개씩(3개) + 다시 각 색 1개씩(3개) + 1개 = 7개
[KOI 유형] 문제 3 — 나머지 부분합 문제: N개의 양의 정수가 주어진다. 연속 부분합 중 N의 배수가 되는 것이 있는지 확인하고, 있으면 시작과 끝 인덱스를 출력하시오.

무손실 압축이 모든 n비트 파일을 (n-1)비트 이하로 줄일 수 없음을 비둘기집 원리로 증명하세요. 증명 과정을 단계별로 서술하세요.

정답과 풀이

용어 정리

용어설명
비둘기집 원리 (Pigeonhole Principle)n+1개를 n개 상자에 넣으면 한 상자에 2개 이상 들어가는 원리
올림 (Ceiling, ⌈⌉)소수점 이하를 올려서 정수로 만드는 연산. ⌈2.3⌉=3
존재 증명 (Existence Proof)어떤 것이 '존재한다'는 것만 보여주는 증명 방법
비구성적 증명존재는 보이지만, 구체적으로 어떤 것인지는 알려주지 않는 증명
해시 충돌 (Hash Collision)서로 다른 입력이 같은 해시 값을 가지는 현상
램지 이론 (Ramsey Theory)충분히 큰 구조에서 특정 패턴이 반드시 나타난다는 이론
조합적 폭발원소 수가 선형으로 늘면 조합(쌍)의 수가 제곱으로 폭증하는 현상
모듈러 연산 (Modular Arithmetic)나머지를 이용한 연산. a mod n = a를 n으로 나눈 나머지
