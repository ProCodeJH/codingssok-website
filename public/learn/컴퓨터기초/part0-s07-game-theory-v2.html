UNIT S-07 게임 이론

게임에도 수학이 있어요

세상에는 두 종류의 게임이 있어요. 운이 필요한 게임과, 실력만으로 결정되는 게임이에요. 실력만으로 결정되는 게임 중에는, 수학적으로 '누가 이기는지' 미리 알 수 있는 것들이 있어요.

[핵심] 조합 게임이란? 조합 게임(Combinatorial Game)의 조건:

1. 두 사람이 번갈아 둔다
2. 모든 정보가 공개되어 있다 (숨김 없음)
3. 운(랜덤) 요소가 없다
4. 언젠가 반드시 끝난다
5. 마지막에 둘 수 없는 사람이 진다

예: 체스, 바둑, 오목, 돌 가져가기 게임
반례: 가위바위보(동시), 포커(숨김), 주사위 게임(운)

가장 간단한 게임 — 돌 1개씩 가져가기

[예시] 돌 5개, 1개씩 가져가기 규칙: 돌 5개가 있고, 번갈아 1개씩 가져가요. 마지막 돌을 가져가면 이겨요.

A: 1개 가져감 → 남은 돌: 4
B: 1개 가져감 → 남은 돌: 3
A: 1개 가져감 → 남은 돌: 2
B: 1개 가져감 → 남은 돌: 1
A: 1개 가져감 → 남은 돌: 0 → A 승리!

돌이 홀수 개면 먼저 하는 사람이 이기고,
짝수 개면 나중에 하는 사람이 이겨요.

돌 가져가기 — 1~3개 버전

이번에는 한 번에 1개, 2개, 또는 3개를 가져갈 수 있어요. 이러면 전략이 훨씬 재미있어져요!

[예시] 돌 가져가기 (1~3개) 돌 10개, 한 번에 1~3개, 마지막 돌을 가져가면 승리

거꾸로 생각해 봐요:
• 돌 0개: 내 차례면 패배 (가져갈 게 없어!)
• 돌 1개: 1개 가져감 → 승리
• 돌 2개: 2개 가져감 → 승리
• 돌 3개: 3개 가져감 → 승리
• 돌 4개: 1~3개 가져가면 상대에게 1~3개 남김 → 상대 승리 → 패배!
• 돌 5개: 1개 가져가면 상대에게 4개 → 상대 패배 → 승리
• ...
• 돌 8개: 뭘 해도 상대에게 5~7개 → 상대 승리 가능 → 패배!

[핵심] 1~k개 가져가기 필승 전략 한 번에 1~k개를 가져갈 수 있을 때:

• 돌이 (k+1)의 배수이면 → 내 차례에 패배 (어떻게 해도 짐)
• 그 외이면 → 승리 가능

전략: 상대에게 항상 (k+1)의 배수를 넘겨요!

k=3일 때: 상대에게 4의 배수를 넘기면 됨
내가 1개 → 상대에게 3개 내기 → 합 4
내가 2개 → 상대에게 2개 내기 → 합 4
내가 3개 → 상대에게 1개 내기 → 합 4

한 라운드에 둘이 합쳐서 항상 4개를 가져가니까,
4의 배수에서 시작한 사람이 결국 0개를 받아요!

Nim 게임 — 여러 더미의 돌

Nim은 가장 유명한 조합 게임이에요. 여러 더미의 돌에서 번갈아 가져가는 게임이에요.

[핵심] Nim 게임 규칙 • 돌 더미가 여러 개 있어요 (예: 3, 5, 7개)
• 자기 차례에 한 더미에서 1개 이상 원하는 만큼 가져가요
• 마지막 돌을 가져가는 사람이 이겨요
• 더 이상 가져갈 돌이 없으면 져요

한 더미를 통째로 가져갈 수도 있어요!
[예시] Nim 게임 진행 더미: [3, 5, 7]

A: 7에서 4개 가져감 → [3, 5, 3]
B: 5에서 2개 가져감 → [3, 3, 3]
A: 3에서 3개 가져감 → [0, 3, 3]
B: 3에서 3개 가져감 → [0, 0, 3]
A: 3에서 3개 가져감 → [0, 0, 0] → A 승리!

Nim의 비밀 — 님-합 (XOR)

Nim 게임의 필승 전략은 S-01에서 배운 XOR 연산에 숨어 있어요!

[핵심] 님-합 (Nim-sum) 님-합 = 모든 더미 크기를 XOR한 값

더미 [3, 5, 7]의 님-합:
3 = 011
5 = 101
7 = 111
────── XOR
= 001 = 1

스프라그-그런디 정리:
• 님-합 ≠ 0 → 현재 차례 플레이어가 이길 수 있어요!
• 님-합 = 0 → 현재 차례 플레이어가 반드시 져요!
[왜 XOR?] 직관적 이해 XOR의 핵심 성질:
• a ⊕ a = 0 (같은 것끼리 상쇄)
• a ⊕ 0 = a

님-합 = 0이면 '균형 상태'예요.
내가 뭘 해도 균형이 깨지고 (님-합 ≠ 0),
상대가 다시 균형을 맞출 수 있어요.

님-합 ≠ 0이면 '불균형 상태'예요.
내가 한 수로 님-합 = 0을 만들 수 있어요!
그러면 상대가 뭘 해도 다시 ≠ 0이 되고...
결국 상대가 [0, 0, ..., 0]에서 차례를 받아요.

[예시] 님-합으로 이기는 수 찾기 더미: [3, 5, 7] → 님-합: 3⊕5⊕7 = 011⊕101⊕111 = 001 = 1 (≠0 → 유리!)

님-합을 0으로 만들어야 해요. 어떻게?

방법: 님-합의 최상위 비트가 1인 더미를 찾아서 조정

님-합 = 001, 최상위 비트 = 위치 0 (값 1)
• 더미 3(011): 비트 0이 1 → 후보!
• 새 값: 3 ⊕ 1 = 2 → 더미를 3→2로 (1개 가져감)

확인: [2, 5, 7] → 010⊕101⊕111 = 000 = 0 ✓

3에서 1개를 가져가면 님-합이 0이 돼요!

Nim 전략 — 더 자세한 예시

왜 님-합 = 0이면 지는 걸까?

[핵심] 님-합 = 0이 패배인 이유 핵심 관찰 3가지:

1. 게임 끝 상태 [0, 0, ..., 0]의 님-합 = 0
→ 이 상태에서 차례인 사람이 짐

2. 님-합 = 0에서 어떤 수를 두어도 님-합 ≠ 0이 됨
(한 더미만 바꾸면 XOR 결과가 반드시 변함)

3. 님-합 ≠ 0에서 항상 님-합 = 0으로 만드는 수가 존재함

따라서:
• 님-합 = 0인 사람은 뭘 해도 상대에게 ≠ 0을 줌
• 상대는 다시 = 0으로 만듦
• 이 과정이 반복되어 결국 [0,0,...,0]에서 차례를 받음 → 패배!

C 코드 — Nim 게임 구현

CNim 게임 — 컴퓨터 vs 사람
1#include <stdio.h>
2
3// 님-합 계산
4int nim_sum(int piles[], int n) {
5 int result = 0;
6 for (int i = 0; i < n; i++) {
7 result ^= piles[i];
8 }
9 return result;
10}
11
12// 컴퓨터의 최적 수 찾기
13void computer_move(int piles[], int n) {
14 int ns = nim_sum(piles, n);
15
16 if (ns == 0) {
17 // 불리한 상황: 아무 더미에서 1개만 가져감
18 for (int i = 0; i < n; i++) {
19 if (piles[i] > 0) {
20 piles[i]--;
21 printf("컴퓨터: 더미 %d에서 1개 가져감\n", i + 1);
22 return;
23 }
24 }
25 }
26
27 // 유리한 상황: 님-합을 0으로 만드는 수
28 for (int i = 0; i < n; i++) {
29 int target = piles[i] ^ ns;
30 if (target < piles[i]) {
31 int take = piles[i] - target;
32 printf("컴퓨터: 더미 %d에서 %d개 가져감\n",
33 i + 1, take);
34 piles[i] = target;
35 return;
36 }
37 }
38}

44

61 else
62 printf("당신이 승리!\n");

64 }

72 take > 0 && take <= piles[pile]) {

74 } else {
75 printf("잘못된 입력!\n");

77 }
78 } else {

80 }

83 }

[코드 해설] • nim_sum: 모든 더미를 XOR해서 님-합을 구해요
• target = piles[i] ^ ns: 더미 i를 이 값으로 바꾸면 님-합 = 0
• target < piles[i]: 줄일 수만 있으니까 (늘리면 안 됨)
• 님-합 = 0이면 어쩔 수 없이 1개만 가져가요 (상대 실수를 기대)

반대로 마지막 돌을 가져가면 지는 게임은 어떨까요? 이것을 미제르(Misère) Nim이라고 해요.

[핵심] 미제르 Nim 전략 미제르 Nim: 마지막 돌을 가져가면 지는 게임

전략이 일반 Nim과 거의 같지만, 끝 처리만 달라요:

• 모든 더미가 1 이하일 때 → 홀수 개 더미가 남으면 불리
• 그 외에는 일반 Nim과 동일하게 님-합 사용!

구체적으로:

스프라그-그런디 정리

[핵심] 스프라그-그런디 정리 모든 조합 게임은 Nim 게임과 동치예요!

어떤 게임 상태든 하나의 그런디 수(Grundy number)를 가져요.
여러 독립적인 게임을 합치면, 그런디 수를 XOR해요.

그런디 수 구하기:
• 이동할 수 없는 상태: G = 0 (패배 상태)
• 이동할 수 있는 상태: G = mex({이동 가능한 상태들의 G값})

mex = '최소 비포함값' (minimum excludant)
mex({0, 1, 2}) = 3
mex({0, 2, 3}) = 1
mex({1, 3}) = 0
[예시] 그런디 수 계산 — 1~3개 가져가기 돌 0개: 이동 불가 → G(0) = 0
돌 1개: → {G(0)} = {0} → G(1) = mex({0}) = 1
돌 2개: → {G(1), G(0)} = {1, 0} → G(2) = mex({0, 1}) = 2
돌 3개: → {G(2), G(1), G(0)} = {2, 1, 0} → G(3) = mex({0, 1, 2}) = 3
돌 4개: → {G(3), G(2), G(1)} = {3, 2, 1} → G(4) = mex({1, 2, 3}) = 0
돌 5개: → {G(4), G(3), G(2)} = {0, 3, 2} → G(5) = mex({0, 2, 3}) = 1

패턴: 0, 1, 2, 3, 0, 1, 2, 3, ... → G(n) = n mod 4

디버깅 -- 게임 전략의 흔한 실수

[디버깅] 실수 1: XOR 우선순위 잘못된 코드:
if (a ^ b ^ c == 0)  // == 가 ^보다 우선순위 높음!
// 실제로는: a ^ b ^ (c == 0) 으로 계산됨

올바른 코드:
if ((a ^ b ^ c) == 0)  // 괄호 필수!

C에서 ==는 ^보다 우선순위가 높아요. 반드시 괄호를 써야 해요!
[디버깅] 실수 2: 님-합 = 0일 때 필승이라고 착각 잘못: '님-합이 0이면 내가 이긴다!'

올바름: 님-합 = 0이면 현재 차례인 사람이 불리해요!
님-합 ≠ 0이면 현재 차례인 사람이 유리해요.

0 = 균형 = 움직이는 쪽이 불리
≠ 0 = 불균형 = 움직이는 쪽이 유리
[디버깅] 실수 3: 탐욕적 전략의 함정 잘못: '가장 큰 더미에서 많이 가져가면 유리하지 않을까?'

이것은 탐욕적(greedy) 전략이고, Nim에서는 틀려요!

예: [1, 2, 3] → 님-합 = 0 (불리)
3에서 3개 가져가기 → [1, 2, 0] → 님-합 = 3 (상대 유리!)

님-합을 계산하지 않고 '감'으로 두면 최적이 아니에요.

용어 정리

용어설명
조합 게임 (Combinatorial Game)두 플레이어, 완전 정보, 운 없음, 유한 길이의 게임
Nim 게임여러 더미에서 번갈아 돌을 가져가는 대표적 조합 게임
님-합 (Nim-sum)모든 더미 크기를 XOR한 값. 0이면 불리, ≠0이면 유리
그런디 수 (Grundy Number)게임 상태를 Nim의 더미 크기로 변환한 값
mex (Minimum Excludant)집합에 포함되지 않은 최소 음이 아닌 정수
미니맥스 (Minimax)상대가 최선을 다한다고 가정하고 최적 수를 찾는 알고리즘
동적 프로그래밍 (DP)작은 문제의 답을 저장해서 큰 문제를 효율적으로 푸는 기법
전략 탈취 논증후수에게 유리한 전략이 있으면 선수가 훔칠 수 있다는 증명 기법
