테이프와 헤드만으로 모든 계산을 할 수 있다고? 현대 컴퓨터의 이론적 원조, 튜링 머신을 만나 봐요!

이 유닛에서 배우는 것

  • 튜링 머신이 현대 컴퓨터의 이론적 모델임을 알아요.
  • 결정 문제와 '계산 가능성'의 개념을 이해해요.
  • C언어로 간단한 튜링 머신을 시뮬레이션할 수 있어요.
  • [심화] 정지 문제가 왜 풀 수 없는지 직관적으로 이해해요.

UNIT S-08 튜링 머신 맛보기

컴퓨터가 없던 시절의 '컴퓨터'

지금 우리가 쓰는 컴퓨터, 스마트폰, 슈퍼컴퓨터는 모두 1936년에 앨런 튜링이 상상한 기계와 본질적으로 같아요. 이 상상의 기계를 튜링 머신(Turing Machine)이라고 해요.

[배경] 튜링이 풀려던 문제 1928년, 수학자 힐베르트가 물었어요:
'모든 수학 문제를 기계적으로 풀 수 있는 방법이 있을까?'

튜링은 이 질문에 답하려고 '기계적으로 푼다'는 게 정확히 뭔지 정의해야 했어요.
그래서 가장 단순하면서도 모든 계산을 할 수 있는 기계를 설계했어요.

결론: '풀 수 없는 문제가 존재한다!'
이것은 수학과 컴퓨터 과학의 역사를 바꾼 발견이에요.

튜링 머신의 구조

튜링 머신은 놀랍도록 단순해요. 부품이 딱 4개뿐이에요.

[핵심] 튜링 머신의 4가지 구성 요소 1. 테이프(Tape): 양쪽으로 무한히 긴 종이 테이프. 칸마다 기호 하나가 적혀 있어요.

2. 헤드(Head): 테이프 위를 움직이며 기호를 읽고 쓰는 장치. 한 번에 한 칸만 봐요.

3. 상태(State): 기계의 현재 '기분'. 유한 개의 상태가 있어요.

4. 규칙표(Transition Table): '현재 상태 + 읽은 기호 → 쓸 기호 + 이동 방향 + 다음 상태'

이게 전부예요! 이 4가지만으로 컴퓨터가 할 수 있는 모든 것을 할 수 있어요.
[비유] 튜링 머신 = 사람 + 종이 + 연필 튜링 머신을 사람에 비유하면:

• 테이프 = 무한히 긴 줄노트
• 헤드 = 연필을 든 손 (한 칸만 봄)
• 상태 = 지금 뭘 하고 있는지 (읽는 중? 쓰는 중? 찾는 중?)
• 규칙표 = '이럴 때는 이렇게 해'라고 적힌 매뉴얼

사람이 매뉴얼을 보며 기계적으로 작업하는 것 = 튜링 머신의 동작!

첫 번째 튜링 머신 — 1을 모두 0으로 바꾸기

가장 간단한 튜링 머신을 만들어 볼게요. 테이프에 적힌 1을 모두 0으로 바꾸는 기계예요.

[규칙표] 1→0 변환기 상태: q0(작동 중), qH(정지)
기호: 0, 1, _(빈칸)

현재상태 | 읽은기호 → 쓸기호 | 이동 | 다음상태
---------+----------+---------+------+---------
   q0    |    1    →    0    |  →   |   q0
   q0    |    0    →    0    |  →   |   q0
   q0    |    _    →    _    |  멈춤 |   qH

의미: 1이든 0이든 오른쪽으로 가면서, 1을 만나면 0으로 바꿔요.
빈칸(_)을 만나면 정지!
단계상태테이프헤드 위치행동
0q0[1] 0 1 1 _01 읽음 → 0 쓰기, 오른쪽
1q0 0 [0] 1 1 _10 읽음 → 0 유지, 오른쪽
2q0 0 0 [1] 1 _21 읽음 → 0 쓰기, 오른쪽
3q0 0 0 0 [1] _31 읽음 → 0 쓰기, 오른쪽
4q0 0 0 0 0 [_]4_ 읽음 → 정지!
끝qH 0 0 0 0 _-모든 1이 0으로!
단계상태테이프헤드 위치행동
0q0[0] 1 1 0 _00 읽음 -> 1 쓰기, 오른쪽
1q0 1 [1] 1 0 _11 읽음 -> 0 쓰기, 오른쪽
2q0 1 0 [1] 0 _21 읽음 -> 0 쓰기, 오른쪽
3q0 1 0 0 [0] _30 읽음 -> 1 쓰기, 오른쪽
4q0 1 0 0 1 [_]4_ 읽음 -> 정지!
끝qH 1 0 0 1 _-0110 -> 1001 (모든 비트 반전!)

두 번째 튜링 머신 — 1의 개수 세기 (짝수/홀수)

좀 더 재미있는 걸 만들어 볼게요. 테이프에 있는 1의 개수가 짝수인지 홀수인지 판별하는 기계예요.

[규칙표] 1의 개수 짝홀 판별기 상태: qE(짝수 개 봤음), qO(홀수 개 봤음), qH(정지)
기호: 0, 1, _(빈칸)

현재상태 | 읽은기호 → 쓸기호 | 이동 | 다음상태
---------+----------+---------+------+---------
   qE    |    1    →    1    |  →   |   qO     ← 짝→홀
   qE    |    0    →    0    |  →   |   qE
   qE    |    _    →    E    |  멈춤 |   qH     ← 짝수!
   qO    |    1    →    1    |  →   |   qE     ← 홀→짝
   qO    |    0    →    0    |  →   |   qO
   qO    |    _    →    O    |  멈춤 |   qH     ← 홀수!

1을 만날 때마다 상태가 짝↔홀 전환돼요.
끝에 도달하면 현재 상태에 따라 E(짝수) 또는 O(홀수)를 써요.
단계상태헤드 읽기행동1의 개수
0qE1짝→홀, 오른쪽1개(홀)
1qO0유지, 오른쪽1개(홀)
2qO1홀→짝, 오른쪽2개(짝)
3qE1짝→홀, 오른쪽3개(홀)
4qO_O 쓰기, 정지!3개(홀)
단계상태헤드 읽기행동1의 개수
0qE1짝->홀, 오른쪽1개(홀)
1qO1홀->짝, 오른쪽2개(짝)
2qE0유지, 오른쪽2개(짝)
3qE0유지, 오른쪽2개(짝)
4qE_E 쓰기, 정지!2개(짝)
[연결] 패리티와 튜링 머신 이 튜링 머신은 S-04에서 배운 짝수 패리티 검사와 완전히 같아요!

• 상태 qE/qO = 1의 개수가 짝수/홀수
• 1을 만날 때마다 상태 전환 = XOR 연산
• 최종 상태 = 패리티 비트

패리티 검사를 튜링 머신으로 구현한 거예요.

세 번째 튜링 머신 — 이진수 +1

이번에는 진짜 계산을 하는 튜링 머신이에요. 이진수에 1을 더해요!

[규칙표] 이진수 +1 계산기 아이디어: 오른쪽 끝에서 시작해서 왼쪽으로 가며 올림 처리

상태: qR(오른쪽 이동), qC(올림 중), qD(완료), qH(정지)

현재상태 | 읽은기호 → 쓸기호 | 이동 | 다음상태
---------+----------+---------+------+---------
   qR    |  0 또는 1 → 그대로 |  →   |   qR     ← 끝까지 이동
   qR    |    _     →    _    |  ←   |   qC     ← 끝 도달, 올림 시작
   qC    |    0     →    1    |  ←   |   qD     ← 0→1, 올림 끝
   qC    |    1     →    0    |  ←   |   qC     ← 1→0, 올림 계속
   qC    |    _     →    1    |  멈춤 |   qH     ← 맨 앞 올림
   qD    |  0 또는 1 → 그대로 |  ←   |   qD     ← 왼쪽 끝까지
   qD    |    _     →    _    |  멈춤 |   qH     ← 완료!
단계상태테이프행동
0qR[1] 0 1 1 _오른쪽 이동...
1-3qR 1 0 1 [1] _끝까지 이동
4qR 1 0 1 1 [_]빈칸 → 올림 시작(←)
5qC 1 0 1 [1]1→0, 올림 계속(←)
6qC 1 0 [1] 01→0, 올림 계속(←)
7qC 1 [0] 0 00→1, 올림 끝!(←)
8qD[1] 1 0 0왼쪽 이동...
9qD[_] 1 1 0 0빈칸 → 정지!
결과qH1 1 0 01011 + 1 = 1100 ✓
단계상태테이프행동
0-2qR 1 1 [1] _오른쪽 끝까지 이동
3qR 1 1 1 [_]빈칸 -> 올림 시작(왼쪽)
4qC 1 1 [1]1->0, 올림 계속(왼쪽)
5qC 1 [1] 01->0, 올림 계속(왼쪽)
6qC[1] 0 01->0, 올림 계속(왼쪽)
7qC[_] 0 0 0빈칸 -> 1 쓰기, 정지!
결과qH1 0 0 0111(=7) + 1 = 1000(=8)
단계상태테이프행동
0-3qR 1 0 0 [0] _오른쪽 끝까지 이동
4qR 1 0 0 0 [_]빈칸 -> 올림 시작(왼쪽)
5qC 1 0 0 [0]0->1, 올림 끝!(왼쪽)
6qD 1 0 [0] 1왼쪽 이동...
7qD 1 [0] 0 1왼쪽 이동...
8qD[1] 0 0 1왼쪽 이동...
9qD[_] 1 0 0 1빈칸 -> 정지!
결과qH1 0 0 11000(=8) + 1 = 1001(=9)

튜링 머신 = 컴퓨터

이렇게 단순한 기계가 정말 컴퓨터와 같을까요?

[핵심] 처치-튜링 논제 처치-튜링 논제(Church-Turing Thesis):

'사람이 기계적 절차로 계산할 수 있는 것은
튜링 머신으로도 계산할 수 있다.'

즉, 튜링 머신은 모든 알고리즘을 실행할 수 있어요!

여러분의 노트북, 스마트폰, 슈퍼컴퓨터는
튜링 머신보다 빠를 뿐, 계산 능력 자체는 같아요.
튜링 머신이 못 하는 건 어떤 컴퓨터도 못 해요.

만능 튜링 머신 — 프로그램을 실행하는 프로그램

튜링의 가장 놀라운 아이디어가 하나 더 있어요.

[핵심] 만능 튜링 머신 (Universal Turing Machine) 보통 튜링 머신은 규칙표가 고정되어 있어요. (계산기처럼)

하지만 만능 튜링 머신은:
테이프에 다른 튜링 머신의 규칙표 + 입력을 적어주면,
그 튜링 머신을 흉내 내서 실행해요!

이것은 현대 컴퓨터의 핵심 아이디어예요:
• 규칙표를 테이프에 적음 = 프로그램을 메모리에 저장
• 만능 튜링 머신 = 프로그램을 실행하는 컴퓨터(CPU)

폰 노이만은 이 아이디어를 실제 컴퓨터로 만들었어요.
이것이 저장 프로그램 방식(stored-program)이에요.
[비유] 만능 튜링 머신 이해하기 일반 튜링 머신 = 계산기 (덧셈만 하는 기계)
만능 튜링 머신 = 컴퓨터 (프로그램에 따라 뭐든 하는 기계)

컴퓨터에 게임을 설치하면 게임기,
워드를 설치하면 타자기,
코딩 도구를 설치하면 개발 환경이 되잖아요?

하나의 기계가 프로그램에 따라 다르게 동작하는 것 —
이 아이디어의 원조가 바로 만능 튜링 머신이에요!

결정 문제 — 컴퓨터도 못 푸는 문제?

튜링이 튜링 머신을 만든 진짜 이유는 '풀 수 없는 문제'가 있다는 걸 증명하기 위해서였어요.

[핵심] 정지 문제 (Halting Problem) 문제: 프로그램 P와 입력 X가 주어졌을 때,
P(X)가 언젠가 끝나는지(정지), 영원히 돌아가는지(무한루프) 판별하시오.

결론: 이 문제를 풀 수 있는 프로그램은 존재하지 않아요!

즉, '이 프로그램이 무한루프에 빠질까?'를
항상 정확히 답하는 프로그램을 만드는 건 불가능해요.
[예시] 정지 문제가 왜 중요할까? 만약 정지 문제를 풀 수 있다면:

• 모든 버그를 자동으로 찾을 수 있어요
• 프로그램이 올바른지 완벽히 검증할 수 있어요
• 수학의 모든 추측을 자동으로 증명/반증할 수 있어요

하지만 불가능하기 때문에:
• 디버깅은 영원히 사람의 몫이에요
• 프로그램 검증에는 한계가 있어요
• 수학에는 기계적으로 풀 수 없는 문제가 있어요

정지 문제 — 왜 풀 수 없을까?

정지 문제를 풀 수 없다는 걸 귀류법(모순 증명)으로 보여 줄게요.

[증명] 정지 문제의 불가능성 (간단 버전) 가정: 정지 문제를 풀 수 있는 프로그램 H가 있다고 해요.
H(P, X) = '정지' 또는 '무한루프'를 정확히 답해요.

악마의 프로그램 D를 만들어요:
D(P) {
    if (H(P, P) == '정지') {
        while(1);  // 무한루프에 빠지기
    } else {
        return;    // 정지하기
    }
}

질문: D(D)는 정지할까, 무한루프일까?

경우 1: H가 'D(D)는 정지'라고 답하면 → D는 무한루프에 빠짐 → 모순!
경우 2: H가 'D(D)는 무한루프'라고 답하면 → D는 정지함 → 모순!

어느 쪽이든 모순 → H는 존재할 수 없어요! □
[비유] 이발사의 역설 정지 문제의 증명은 이발사의 역설과 같은 구조예요:

'스스로 면도하지 않는 사람만 면도해 주는 이발사'
→ 이발사 자신은? 면도하면 모순, 안 하면 모순!

정지 문제에서 D(D)도 마찬가지:
정지하면 무한루프, 무한루프면 정지 → 모순!

이것은 S-06에서 배운 비둘기집 원리처럼
'존재할 수 없음'을 증명하는 거예요.
시나리오H(D,D)의 답D의 행동실제 결과H의 판정
경우 1정지if 참 -> while(1)무한루프틀림! (모순)
경우 2무한루프else -> return정지틀림! (모순)

계산 가능 vs 계산 불가능

[핵심] 계산 가능성의 분류 계산 가능(Computable): 알고리즘으로 풀 수 있는 문제
예: 정렬, 최단경로, 소수 판별, 체스 최적수

계산 불가능(Uncomputable): 어떤 알고리즘으로도 풀 수 없는 문제
예: 정지 문제, '이 프로그램에 버그가 있는가?'

중요: '아직 못 풀었다'가 아니라 '절대 못 푼다'예요!
아무리 빠른 컴퓨터, 아무리 똑똑한 AI도 풀 수 없어요.

C 코드 — 범용 튜링 머신 시뮬레이터

C규칙표 기반 범용 튜링 머신
1#include <stdio.h>
2#include <string.h>
3
4#define TAPE_SIZE 200
5#define BLANK '_'
6#define MAX_RULES 20
7
8typedef struct {
9 int state; // 현재 상태
10 char read; // 읽은 기호
11 char write; // 쓸 기호
12 int move; // -1=왼쪽, 0=멈춤, 1=오른쪽
13 int next_state; // 다음 상태
14} Rule;
15
16int main(void) {
17 char tape[TAPE_SIZE];
18 memset(tape, BLANK, sizeof(tape));
19
20 // 짝홀 판별기 규칙 (qE=0, qO=1, qH=2)
21 Rule rules[] = {
22 {0, '1', '1', 1, 1}, // qE + 1 → 오른쪽, qO
23 {0, '0', '0', 1, 0}, // qE + 0 → 오른쪽, qE
24 {0, BLANK, 'E', 0, 2}, // qE + _ → E쓰기, 정지
25 {1, '1', '1', 1, 0}, // qO + 1 → 오른쪽, qE
26 {1, '0', '0', 1, 1}, // qO + 0 → 오른쪽, qO
27 {1, BLANK, 'O', 0, 2}, // qO + _ → O쓰기, 정지
28 };
29 int num_rules = 6;
30
31 // 입력: 10110
32 char input[] = "10110";
33 int start = 50;
34 strncpy(&tape[start], input, strlen(input));
35
36 int head = start;
37 int state = 0; // 시작: qE
38 int steps = 0;
39 char *state_names[] = {"qE", "qO", "qH"};
40
41 printf("=== 짝홀 판별 튜링 머신 ===\n");
42 printf("입력: %s\n\n", input);
43
44 while (state != 2 && steps < 100) {
45 char read = tape[head];
46
47 // 규칙 찾기
48 int found = 0;
49 for (int r = 0; r < num_rules; r++) {
50 if (rules[r].state == state && rules[r].read == read) {
51 printf("%2d: [%s] '%c' → '%c' %s → [%s]\n",
52 steps, state_names[state], read,
53 rules[r].write,
54 rules[r].move == 1 ? "→" :
55 rules[r].move == -1 ? "←" : "■",
56 state_names[rules[r].next_state]);
57
58 tape[head] = rules[r].write;
59 head += rules[r].move;
60 state = rules[r].next_state;
61 found = 1;
62 break;
63 }
64 }
65
66 if (!found) {
67 printf("규칙 없음! 비정상 정지.\n");
68 break;
69 }
70 steps++;
71 }
72
73 // 결과 출력
74 printf("\n결과: 1의 개수는 ");
75 if (tape[head] == 'E') printf("짝수\n");
76 else if (tape[head] == 'O') printf("홀수\n");
77 printf("(총 %d단계)\n", steps);
78
79 return 0;
80}
실행 결과
=== 짝홀 판별 튜링 머신 === 입력: 10110 0: [qE] '1' → '1' → → [qO] 1: [qO] '0' → '0' → → [qO] 2: [qO] '1' → '1' → → [qE] 3: [qE] '1' → '1' → → [qO] 4: [qO] '0' → '0' → → [qO] 5: [qO] '_' → 'O' ■ → [qH] 결과: 1의 개수는 홀수 (총 6단계)
[코드 해설] • Rule 구조체: 튜링 머신의 규칙 하나를 표현해요
• 규칙 배열: 규칙표 전체를 데이터로 저장
• 규칙 찾기: 현재 상태 + 읽은 기호로 맞는 규칙을 검색
• 규칙 적용: 쓰기 → 이동 → 상태 변경

규칙만 바꾸면 다른 튜링 머신이 돼요! 이것이 범용의 의미예요.

C 코드 — 무한루프 탐지 (제한적)

정지 문제는 일반적으로 풀 수 없지만, 제한된 범위에서는 탐지할 수 있어요.

C간단한 무한루프 탐지 (타임아웃 방식)
1#include <stdio.h>
2
3// 간단한 프로그램 시뮬레이션
4// program[i] = 다음에 실행할 줄 번호 (0이면 종료)
5int simulate(int program[], int size, int max_steps) {
6 int line = 1; // 현재 줄
7 int steps = 0;
8
9 while (line != 0 && steps < max_steps) {
10 if (line < 1 || line > size) return -1; // 오류
11 line = program[line]; // 다음 줄로
12 steps++;
13 }
14
15 if (line == 0) return steps; // 정상 종료
16 return -1; // 아마 무한루프 (확실하지 않음)
17}
18
19int main(void) {
20 // 프로그램 1: 1→2→3→0 (정상 종료)
21 int prog1[] = {0, 2, 3, 0};
22 int r1 = simulate(prog1, 3, 1000);
23 printf("프로그램 1: %s (%d단계)\n",
24 r1 >= 0 ? "정지" : "무한루프?", r1);
25
26 // 프로그램 2: 1→2→3→1 (무한루프)
27 int prog2[] = {0, 2, 3, 1};
28 int r2 = simulate(prog2, 3, 1000);
29 printf("프로그램 2: %s\n",
30 r2 >= 0 ? "정지" : "무한루프?");
31
32 // 프로그램 3: 1→2→3→4→0 (정상)
33 int prog3[] = {0, 2, 3, 4, 0};
34 int r3 = simulate(prog3, 4, 1000);
35 printf("프로그램 3: %s (%d단계)\n",
36 r3 >= 0 ? "정지" : "무한루프?", r3);
37
38 return 0;
39}
실행 결과
프로그램 1: 정지 (3단계) 프로그램 2: 무한루프? 프로그램 3: 정지 (4단계)
[주의] 타임아웃의 한계 이 방법은 완벽하지 않아요:

• 1000단계 안에 안 끝나면 '무한루프?'라고 판단하지만
• 실제로는 1001단계에 끝날 수도 있어요!

이것이 정지 문제의 본질이에요:
• 끝나는 프로그램: 기다리면 알 수 있어요
• 안 끝나는 프로그램: 아무리 오래 기다려도 '아직 안 끝남'과 '영원히 안 끝남'을 구별할 수 없어요

튜링 완전 — 이것도 컴퓨터?

튜링 머신과 같은 능력을 가진 시스템을 튜링 완전(Turing Complete)하다고 해요.

[핵심] 튜링 완전이란? 어떤 시스템이 튜링 머신을 흉내 낼 수 있으면 튜링 완전이에요.

튜링 완전한 것들:
• 모든 일반 프로그래밍 언어 (C, Python, Java, ...)
• 엑셀 스프레드시트 (놀랍죠?)
• 마인크래프트의 레드스톤
• 생명 게임 (Conway's Game of Life)
• HTML + CSS (이론적으로!)
• 매직 더 개더링 카드 게임

튜링 완전하지 않은 것:
• 계산기 (루프가 없어서)
• HTML만 (프로그래밍은 아니에요)
• 정규 표현식 (일부 언어에서)
[재미] 엑셀이 튜링 완전? 엑셀에서 셀 참조와 IF 함수만으로 튜링 머신을 만들 수 있어요!

• 셀 = 테이프의 칸
• IF/VLOOKUP = 규칙표
• 행 = 시간 단계

2021년에 마이크로소프트가 공식적으로 엑셀에 LAMBDA 함수를 추가하며
'엑셀은 이제 튜링 완전한 프로그래밍 언어'라고 발표했어요!

디버깅 — 튜링 머신 시뮬레이터의 흔한 실수

[디버깅] 실수 1: 테이프 범위 초과 잘못: 헤드가 배열 범위를 벗어남

    printf("테이프 범위 초과!\n");
    break;
}
[디버깅] 실수 2: 무한루프 보호 없음 잘못: 규칙에 실수가 있으면 영원히 돌아감

    // ...
}

올바름: 최대 실행 횟수 제한
int max_steps = 10000;

[디버깅] 실수 3: 규칙 누락 잘못: 모든 (상태, 기호) 조합에 대한 규칙이 없음
