UNIT S-09 증명 기법 기초

증명이란 뭘까?

수학에서 증명(proof)이란, 어떤 주장이 반드시 참이라는 것을 논리적으로 보여주는 거예요. '아마 그럴 거야'가 아니라 '절대 틀릴 수 없다'를 보이는 거예요.

[핵심] 증명 vs 확인 확인(테스트): 몇 가지 예시를 넣어서 맞는지 봐요
• 1+2 = 3 ✓, 1+2+3 = 6 ✓, 1+2+...+100 = 5050 ✓
• 하지만 101번째도 맞다는 보장은 없어요!

증명: 모든 경우에 대해 참임을 논리적으로 보여요
• 어떤 n이든 1+2+...+n = n(n+1)/2
• 예외가 없다는 걸 100% 확신할 수 있어요!

프로그래밍에서도 마찬가지예요:
• 테스트: 몇 가지 입력으로 확인 → 버그를 발견할 수 있지만 없다는 보장은 못 해요
• 증명: 알고리즘이 모든 입력에 대해 올바르다는 걸 보여요
[팁] 프로그래머에게 증명이 왜 중요할까? • 반복문이 반드시 끝나는지 확인할 때
• 알고리즘의 시간 복잡도를 분석할 때
• 정렬 알고리즘이 올바른지 증명할 때
• 재귀 함수가 맞는 답을 주는지 확인할 때
• KOI/정보올림피아드에서 풀이의 정확성을 논할 때

코딩 대회에서 '이 방법이 왜 맞는지' 설명하는 것이 증명이에요.

증명 기법 1 — 직접 증명

가장 기본적인 증명 방법이에요. P에서 출발해서 Q까지 한 걸음씩 논리적으로 걸어가요.

[핵심] 직접 증명 (Direct Proof) 목표: 'P이면 Q이다'를 증명

방법:
1. P가 참이라고 가정해요
2. 논리적 단계를 거쳐 Q가 참이 됨을 보여요

P → ... → ... → Q

가장 자연스럽고 직관적인 증명 방법이에요!
[증명 1] 짝수의 합은 짝수 명제: 두 짝수의 합은 짝수이다.

증명:
1. a, b가 짝수라고 가정 (P)
2. 짝수의 정의: a = 2m, b = 2n (m, n은 정수)
3. a + b = 2m + 2n = 2(m + n)
4. m + n은 정수이므로, 2(m+n)은 짝수 (Q)

따라서 두 짝수의 합은 짝수이다. □
[증명 2] n이 짝수이면 n²도 짝수 명제: n이 짝수이면 n²도 짝수이다.

증명:
1. n이 짝수라고 가정 → n = 2k (k는 정수)
2. n² = (2k)² = 4k² = 2(2k²)
3. 2k²은 정수이므로, n² = 2×(정수) = 짝수

따라서 n이 짝수이면 n²도 짝수이다. □

□ 기호는 '증명 끝'이라는 뜻이에요. QED(증명 완료)라고도 써요.
단계수식근거결과

정의 적용a=2m, b=2n짝수 정의m, n은 정수

인수 분해2m+2n = 2(m+n)분배 법칙2(m+n)
정수 확인m+n은 정수정수의 합은 정수정수
결론a+b = 2(정수)짝수 정의짝수 -- QED

가정n은 짝수전제n = 2k (k는 정수)
제곱n^2 = (2k)^2대입(2k)^2
전개(2k)^2 = 4k^2거듭제곱 계산4k^2
인수 분해4k^2 = 2(2k^2)2를 묶음2(2k^2)
정수 확인2k^2은 정수정수의 곱은 정수정수
결론n^2 = 2 x (정수)짝수 정의짝수 -- QED
단계수식근거결과

정수 확인m+n+1은 정수정수의 합은 정수정수
결론a+b = 2 x (정수)짝수 정의짝수 -- QED

직접 증명 — 프로그래밍 응용

[증명 3] 가우스의 공식 명제: 1 + 2 + 3 + ... + n = n(n+1)/2

직접 증명 (가우스의 방법):
S = 1 + 2 + 3 + ... + n
S = n + (n-1) + (n-2) + ... + 1 (역순으로 한 번 더)

두 줄을 더하면:
2S = (1+n) + (2+n-1) + (3+n-2) + ... + (n+1)
2S = (n+1) + (n+1) + (n+1) + ... + (n+1) ← n개
2S = n(n+1)
S = n(n+1)/2 □

가우스가 10살 때 1+2+...+100을 이 방법으로 계산했대요!

증명 기법 2 — 귀류법 (모순 증명)

[핵심] 귀류법 (Proof by Contradiction) 목표: 명제 Q를 증명

방법:
1. Q가 거짓이라고 가정해요 (¬Q 가정)
2. 논리를 전개해요
3. 모순이 발생해요!
4. 따라서 가정이 틀림 → Q는 참!

핵심 논리: 거짓에서 출발하면 반드시 모순에 도달
→ 출발점(가정)이 틀렸다 → 원래 명제가 참!
[증명 4] √2는 무리수다 명제: √2는 무리수이다 (분수로 나타낼 수 없다).

귀류법 증명:
1. 가정: √2가 유리수라고 하자 → √2 = a/b (기약분수)
2. 양변을 제곱: 2 = a²/b² → a² = 2b²
3. a²이 짝수 → a도 짝수 → a = 2k
4. (2k)² = 2b² → 4k² = 2b² → b² = 2k²
5. b²이 짝수 → b도 짝수
6. a, b 둘 다 짝수 → 2로 더 약분 가능
7. 하지만 a/b는 기약분수라고 했음 → 모순!

가정이 틀림 → √2는 유리수가 아님 → 무리수! □
단계내용결과상태
1sqrt(2)가 유리수라고 가정sqrt(2) = a/b (기약분수)가정 설정
2양변 제곱2 = a^2/b^2, 즉 a^2 = 2b^2변환
3a^2이 짝수a도 짝수 (짝수^2만 짝수)추론
4a = 2k로 치환4k^2 = 2b^2, 즉 b^2 = 2k^2대입
5b^2이 짝수b도 짝수추론
6a, b 모두 짝수2로 약분 가능모순 발견!
7기약분수 가정과 충돌가정이 틀림QED
[증명 5] 소수는 무한히 많다 명제: 소수의 개수는 무한하다.

귀류법 증명 (유클리드, 기원전 300년!):
1. 가정: 소수가 유한하다고 하자 → p₁, p₂, ..., pₙ이 전부
2. 새로운 수 만들기: N = p₁ × p₂ × ... × pₙ + 1
3. N을 p₁으로 나누면 나머지 1 (p₁의 배수 아님)
4. N을 p₂로 나누면 나머지 1, ..., pₙ으로도 나머지 1
5. N은 어떤 소수로도 안 나눠짐
6. 그러면 N 자체가 소수이거나, 목록에 없는 새 소수로 나눠짐
7. 어느 쪽이든 '전체 소수 목록'에 없는 소수가 존재 → 모순!

가정이 틀림 → 소수는 무한히 많다! □

증명 기법 3 — 반례 (반증)

명제가 거짓임을 보이는 건 훨씬 쉬워요. 반례 하나만 찾으면 돼요!

[핵심] 반례 (Counterexample) '모든 X에 대해 P이다'라는 명제를 반증하려면,
P가 성립하지 않는 X 하나만 찾으면 돼요!

예:
• '모든 소수는 홀수다' → 반례: 2 (짝수인 소수)
• '모든 정수의 제곱은 양수다' → 반례: 0 (0²=0은 양수 아님)
• 'n²+n+41은 항상 소수다' → 반례: n=41 (41²+41+41 = 41×43)
[예시] 유명한 반례 이야기 오일러의 추측 (1769):
'a⁴ + b⁴ + c⁴ = d⁴를 만족하는 양의 정수는 없다'

200년 동안 아무도 반증하지 못했는데...
1988년: 컴퓨터로 반례를 찾았어요!

증명 기법 4 — 수학적 귀납법

[핵심] 수학적 귀납법 (Mathematical Induction) 목표: '모든 자연수 n에 대해 P(n)이 성립'을 증명

2단계로 증명해요:

1단계 (기초): P(1)이 참임을 보여요
→ 첫 번째 도미노를 넘어뜨리는 것

2단계 (귀납): P(k)가 참이면 P(k+1)도 참임을 보여요
→ 도미노가 넘어지면 다음도 넘어진다는 것

이 두 가지를 보이면:
P(1) ✓ → P(2) ✓ → P(3) ✓ → ... → P(n) ✓ (전부!)
[증명 6] 귀납법: 1+2+...+n = n(n+1)/2 P(n): 1+2+...+n = n(n+1)/2

1단계 (기초): P(1) 확인
왼쪽: 1, 오른쪽: 1×2/2 = 1 → 같으므로 P(1) ✓

2단계 (귀납): P(k)가 참이면 P(k+1)도 참?

P(k) 가정: 1+2+...+k = k(k+1)/2

P(k+1) 보이기: 1+2+...+k+(k+1)
= [1+2+...+k] + (k+1)

증명 3단계 (귀납법과 같은 구조!):

1. 초기화: 루프 시작 전에 불변량이 참인가? (기초)

경우 3: a<0, b≥0

경우 4: a<0, b<0

[디버깅] 실수 2: 증명해야 할 것을 가정하기 (순환 논증) 잘못: 증명하려는 결론을 가정 속에 이미 사용

[디버깅] 실수 3: 몇 개 확인하고 '증명됨' 잘못: n=1,2,3,4,5에서 맞으니까 '항상 맞다'

이것은 증명이 아니라 테스트예요!

n=40에서 41² = 1681 → 소수 아님!

테스트 = 반례를 못 찾은 것
증명 = 반례가 존재할 수 없음을 보인 것
