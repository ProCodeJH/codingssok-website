UNIT S-12 입출력과 버스

컴퓨터 부품들은 어떻게 대화할까?

S-10에서 CPU를, S-11에서 메모리를 배웠어요. CPU는 계산하고, 메모리는 데이터를 저장해요. 그런데 컴퓨터가 혼자서만 계산하면 우리에게 아무 쓸모가 없어요. 사람이 데이터를 넣어 주고, 결과를 보여 줘야 비로소 쓸모가 생기거든요.

이때 사람이 컴퓨터에게 데이터를 전달하는 장치를 입력 장치, 컴퓨터가 사람에게 결과를 보여주는 장치를 출력 장치라고 해요. 그리고 이 모든 부품들 사이에서 데이터가 오가는 통로를 버스(Bus)라고 불러요.

[핵심] 입출력과 버스란?
입력(Input): 사람이나 외부 환경에서 컴퓨터로 데이터를 보내는 것
출력(Output): 컴퓨터가 사람이나 외부 환경에 결과를 보내는 것
버스(Bus): 컴퓨터 내부에서 데이터가 이동하는 전선 다발(통로)

줄여서 I/O(Input/Output)라고 불러요. C 언어에서 stdio.h의 'io'가 바로 이 I/O예요!
[비유] 학교에 비유한 I/O와 버스
학교를 컴퓨터라고 생각해 보세요.

- 입력 장치 = 교실 문 (학생들이 들어오는 곳)
- 출력 장치 = 칠판, 스피커 (선생님이 결과를 보여주는 곳)
- CPU = 선생님 (생각하고 판단하는 사람)
- 메모리 = 선생님 책상 (자료를 올려놓는 곳)
- 버스 = 복도 (사람과 물건이 교실 사이를 이동하는 통로)

복도가 좁으면 사람들이 막히고, 넓으면 빨리 지나가요. 버스도 똑같아요!

입력 장치 -- 컴퓨터에게 말을 거는 도구들

입력 장치는 사람의 동작이나 외부 정보를 컴퓨터가 이해할 수 있는 전기 신호(0과 1)로 바꿔 주는 장치예요. 우리가 매일 사용하는 것들이 대부분 입력 장치예요.

[핵심] 대표적인 입력 장치
1. 키보드: 글자와 숫자를 입력하는 가장 기본적인 장치. 키를 누르면 해당 키의 코드(스캔코드)가 CPU로 전달돼요.

2. 마우스: 화면 위의 위치를 지정하는 장치. 움직임과 클릭 정보를 보내요.

3. 마이크: 소리를 전기 신호로 바꿔요. 아날로그 소리를 디지털 데이터로 변환(ADC)해요.

4. 카메라(웹캠): 빛을 전기 신호로 바꿔서 영상 데이터를 만들어요.

5. 센서: 온도, 습도, 압력, 거리 등 물리적 환경 정보를 측정해요. IoT 시대에 아주 중요해요.

6. 스캐너: 종이 문서나 사진을 디지털 이미지로 바꿔요.

입력 장치의 공통점이 보이나요? 전부 '사람이 하는 행동'이나 '바깥 세상의 정보'를 0과 1의 디지털 신호로 변환하는 거예요. 키보드는 키 번호를, 마우스는 좌표를, 마이크는 소리 파형을 숫자로 바꿔서 컴퓨터에 전달해요.

[비유] 입력 장치는 통역사
입력 장치는 '사람의 말을 컴퓨터 언어로 통역'하는 통역사예요.

- 키보드: '가' 키를 누르면 -> 유니코드 0xAC00 (숫자로 변환)
- 마이크: 목소리 -> 초당 44,100개의 숫자 샘플로 변환
- 카메라: 빛 -> 각 픽셀의 RGB 값(숫자 3개)으로 변환

컴퓨터는 숫자밖에 모르니까, 모든 걸 숫자로 바꿔 줘야 해요!

출력 장치 -- 컴퓨터가 우리에게 답하는 도구들

출력 장치는 컴퓨터 안의 디지털 데이터(0과 1)를 사람이 느낄 수 있는 형태(그림, 소리, 글자)로 바꿔 주는 장치예요.

[핵심] 대표적인 출력 장치
1. 모니터: 가장 많이 사용하는 출력 장치. 픽셀이라는 작은 점들에 색을 입혀서 화면을 보여줘요. Full HD 모니터는 1920x1080 = 약 207만 개의 픽셀이 있어요.

2. 스피커: 디지털 데이터를 아날로그 소리로 바꿔서 들려줘요. DAC(디지털-아날로그 변환기)가 이 일을 해요.

3. 프린터: 디지털 문서를 종이에 인쇄해요. 잉크젯, 레이저 등 여러 방식이 있어요.

4. LED: 간단한 신호를 빛으로 보여줘요. 아두이노 같은 보드에서 많이 사용해요. 전원 표시등도 LED 출력이에요.

출력 장치는 입력 장치의 반대 방향이에요. 입력이 '아날로그 -> 디지털' 변환이었다면, 출력은 '디지털 -> 아날로그' 변환이에요. 마이크가 소리를 숫자로 바꿨다면, 스피커는 숫자를 소리로 되돌리는 거예요.

입출력 겸용 장치 -- 양쪽 다 할 수 있는 장치

어떤 장치는 입력도 하고 출력도 해요. 이런 장치를 입출력 겸용 장치라고 불러요.

[핵심] 입출력 겸용 장치
1. 터치스크린: 화면을 보여 주는 출력 + 손가락 터치를 감지하는 입력을 동시에 해요. 스마트폰이 대표적이에요.

2. SSD/HDD (저장 장치): 데이터를 저장(쓰기 = 출력)하고, 저장된 데이터를 읽어오기(읽기 = 입력)도 해요. CPU 관점에서 보면, 데이터를 보내는 것이 출력이고, 받는 것이 입력이에요.

3. 네트워크 카드 (NIC): 인터넷으로 데이터를 보내기(출력)도 하고, 받기(입력)도 해요. Wi-Fi 모듈, 이더넷 카드가 여기에 해당해요.

버스(Bus) -- 데이터가 이동하는 도로

이제 부품들은 알았어요. 그런데 CPU, 메모리, 입력 장치, 출력 장치가 어떻게 서로 데이터를 주고받을까요? 바로 버스(Bus)를 통해서예요.

[핵심] 버스란?
버스(Bus): 컴퓨터 내부에서 데이터, 주소, 제어 신호가 이동하는 공유 전선 다발이에요.

버스라는 이름은 '여러 사람이 함께 타는 버스(공공 교통수단)'에서 왔어요. 도로를 여러 차가 함께 쓰듯이, 버스는 여러 부품이 공유하는 통로예요.
[비유] 버스는 고속도로
컴퓨터를 도시라고 생각해 보세요.

- CPU = 시청 (모든 결정을 내리는 곳)
- 메모리 = 창고 (물건을 보관하는 곳)
- 입력 장치 = 물류 센터 (외부에서 물건이 들어오는 곳)
- 출력 장치 = 배송 센터 (물건을 외부로 보내는 곳)
- 버스 = 이 건물들을 연결하는 고속도로

고속도로가 없으면 물건을 주고받을 수 없어요. 마찬가지로 버스가 없으면 CPU와 메모리, 장치들이 데이터를 주고받을 수 없어요.

버스는 물리적으로는 메인보드(마더보드) 위에 있는 아주 가느다란 구리 선들이에요. 메인보드를 자세히 보면 초록색 판 위에 가느다란 금색 선들이 그려져 있는데, 그게 바로 버스예요. 눈에 보이지 않을 만큼 가는 선이 수십 개에서 수백 개가 묶여서 데이터를 나르고 있어요.

버스의 세 종류

버스는 실어 나르는 것에 따라 세 가지로 나뉘어요. 각각의 역할이 명확히 다르답니다.

[핵심] 버스의 세 종류
1. 데이터 버스 (Data Bus)
실제 데이터(숫자, 명령어)를 실어 나르는 버스. '짐을 실은 트럭'과 같아요.
양방향이에요: CPU -> 메모리(쓰기)도 하고, 메모리 -> CPU(읽기)도 해요.

2. 주소 버스 (Address Bus)
'어디로 갈 건지' 목적지 주소를 전달하는 버스. '내비게이션'과 같아요.
단방향이에요: CPU가 메모리에게 '몇 번 주소의 데이터를 줘!'라고 알려줘요.

3. 제어 버스 (Control Bus)
'무엇을 할 건지' 명령 신호를 전달하는 버스. '신호등'과 같아요.
읽기 신호, 쓰기 신호, 인터럽트 신호 등을 보내요.
[비유] 택배 시스템으로 이해하는 세 가지 버스
택배를 보내는 상황을 생각해 보세요.

- 데이터 버스 = 택배 트럭 (실제 물건을 실어 나름)
- 주소 버스 = 운송장에 적힌 주소 (어디로 보낼지 알려줌)
- 제어 버스 = 배달 지시서 ('배송해', '반품해', '보관해' 같은 명령)

택배를 보내려면 세 가지가 다 필요하듯이, 컴퓨터도 데이터 + 주소 + 제어 신호가 모두 있어야 데이터를 주고받을 수 있어요.

예를 들어, CPU가 메모리의 100번 주소에서 데이터를 읽고 싶다면 이런 과정이 벌어져요.

버스 폭과 성능 -- 차선이 넓을수록 빨라요

버스 폭(Bus Width)은 한 번에 몇 비트의 데이터를 동시에 보낼 수 있는지를 나타내요. 도로의 차선 수와 같은 개념이에요.

[핵심] 버스 폭이란?
버스 폭(Bus Width): 버스가 한 번에 전송할 수 있는 비트 수

- 8비트 버스: 한 번에 1바이트(8비트)를 보냄 -- 1차선 도로
- 16비트 버스: 한 번에 2바이트를 보냄 -- 2차선 도로
- 32비트 버스: 한 번에 4바이트를 보냄 -- 4차선 도로
- 64비트 버스: 한 번에 8바이트를 보냄 -- 8차선 고속도로

차선이 많을수록 동시에 많은 차가 지나가듯이, 버스 폭이 넓을수록 한 번에 더 많은 데이터를 보낼 수 있어요.
버스 폭한 번에 전송비유대표 시대
8비트1바이트1차선 시골길1980년대 초기 PC
16비트2바이트2차선 도로1990년대 초반
32비트4바이트4차선 도로2000년대
64비트8바이트8차선 고속도로현재 대부분의 PC

주소 버스의 폭도 중요해요. 주소 버스가 32비트라면 2^32 = 약 42억 개의 주소를 지정할 수 있어요. 이건 최대 4GB 메모리를 사용할 수 있다는 뜻이에요. 64비트 주소 버스라면? 2^64 = 약 1844경 개의 주소를 쓸 수 있어서, 사실상 무한한 메모리를 연결할 수 있어요.

[비유] 주소 버스 폭 = 우편번호 자릿수
우편번호가 2자리라면? 00~99, 즉 100개의 지역만 구분할 수 있어요.
우편번호가 5자리라면? 00000~99999, 즉 10만 개의 지역을 구분할 수 있어요.

주소 버스도 마찬가지예요. 비트 수(자릿수)가 많을수록 더 많은 메모리 주소를 지정할 수 있어요.

32비트 = 약 42억 개 주소 (4GB)
64비트 = 약 1844경 개 주소 (16EB, 사실상 무제한)

시스템 버스 vs 확장 버스

컴퓨터 안에는 한 종류의 버스만 있는 게 아니에요. 속도와 역할에 따라 여러 종류의 버스가 계층적으로 연결되어 있어요.

[핵심] 시스템 버스와 확장 버스
시스템 버스 (System Bus)
CPU, 메모리, 핵심 장치를 연결하는 내부 고속 버스예요. '도시 안 고속도로'와 같아요.
가장 빠른 속도로 동작하고, 컴퓨터의 핵심 통로예요.

확장 버스 (Expansion Bus)
외부 장치(그래픽 카드, 사운드 카드, USB 장치 등)를 연결하는 버스예요. '도시 밖으로 나가는 국도'와 같아요.
시스템 버스보다는 느리지만, 다양한 장치를 연결할 수 있어요.

시스템 버스와 확장 버스 사이에는 브릿지(Bridge)라는 중간 다리가 있어서, 속도 차이를 조절해 줘요.

포트와 인터페이스 -- 컴퓨터의 출입문

외부 장치를 컴퓨터에 연결하려면 포트(Port)가 필요해요. 포트는 컴퓨터 외부에 있는 연결 단자(구멍)이고, 인터페이스(Interface)는 그 연결 방식의 규칙(약속)이에요.

[핵심] 주요 포트와 인터페이스
1. USB (Universal Serial Bus)
가장 널리 쓰이는 연결 방식. 키보드, 마우스, 외장 하드, 충전기 등 거의 모든 것을 연결해요.

2. HDMI (High-Definition Multimedia Interface)
모니터, TV에 영상과 소리를 동시에 보내는 케이블이에요.

3. 이더넷 (Ethernet)
유선 인터넷을 연결하는 네모난 포트(RJ-45)예요.

4. 블루투스 (Bluetooth)
무선으로 가까운 거리의 장치를 연결해요. 무선 이어폰, 마우스 등에 쓰여요.

5. Wi-Fi
무선으로 인터넷에 연결하는 기술이에요. 공유기와 컴퓨터 사이를 전파로 연결해요.
[비유] 포트는 집의 문과 창문
집에도 여러 종류의 출입구가 있어요.

- USB = 현관문 (가장 많이 사용, 여러 종류의 방문자가 드나듦)
- HDMI = 거실 대형 창문 (큰 영상과 소리가 지나감)
- 이더넷 = 수도관 (안정적으로 데이터가 흐름)
- 블루투스 = 인터폰 (선 없이 가까운 거리에서 소통)
- Wi-Fi = 무전기 (먼 거리까지 무선으로 소통)

각각 용도가 다르고, 보내는 데이터의 양과 속도가 달라요.

- 모뎀: 직렬 포트 (COM, 작은 사다리꼴)

인터럽트 -- 수업 중 손 들기

CPU는 프로그램을 실행하느라 바빠요. 그런데 키보드를 누르거나, 마우스를 클릭하면 CPU가 즉시 반응해야 해요. 어떻게 하는 걸까요?

방법은 두 가지가 있어요. 하나는 폴링(Polling)이고, 다른 하나는 인터럽트(Interrupt)예요.

[핵심] 폴링 vs 인터럽트
폴링 (Polling) -- '계속 물어보기' 방식
CPU가 주기적으로 모든 장치에게 '할 말 있어? 할 말 있어?'라고 물어보는 방식이에요.
단점: CPU가 쓸데없이 바빠요. 대부분의 경우 '없어요'라는 대답만 돌아오니까요.

인터럽트 (Interrupt) -- '손 들기' 방식
장치가 할 말이 있을 때만 CPU에게 '잠깐요!'라고 신호를 보내는 방식이에요.
장점: CPU는 자기 일에 집중하다가, 신호가 오면 그때만 처리하면 돼요.
[비유] 선생님과 학생
폴링 방식의 선생님:
'1번, 질문 있어?' '없어요.' '2번, 질문 있어?' '없어요.' '3번, 질문 있어?' '없어요.' ...
30명을 매번 다 물어봐요. 아무도 질문 없어도 계속 물어봐야 해서 선생님이 지쳐요.

인터럽트 방식의 선생님:
선생님이 수업을 하다가, 학생이 손을 들면 그때 멈추고 질문을 받아요.
'선생님, 질문이요!' -> '그래, 뭐니?' -> 대답 -> 다시 수업 계속

인터럽트 방식이 훨씬 효율적이에요! 현대 컴퓨터는 거의 다 인터럽트 방식을 써요.

인터럽트 처리 과정

인터럽트가 발생하면 CPU는 정해진 순서대로 처리해요. 이 과정을 인터럽트 처리 과정이라고 해요.

[핵심] ISR이란?
ISR (Interrupt Service Routine, 인터럽트 서비스 루틴)

특정 인터럽트가 발생했을 때 실행되는 미리 작성된 프로그램이에요.

예시:
- 키보드 인터럽트 ISR: 어떤 키가 눌렸는지 읽어서 버퍼에 저장
- 마우스 인터럽트 ISR: 마우스 좌표 변화를 읽어서 커서를 이동
- 타이머 인터럽트 ISR: 일정 시간마다 시계를 업데이트

운영체제(OS)가 이런 ISR들을 미리 등록해 놓아요.

DMA -- 택배 직배송

인터럽트 방식에서도 문제가 있어요. 대량의 데이터를 전송할 때(예: 파일 복사, 동영상 재생), CPU가 데이터를 한 바이트씩 일일이 옮겨야 한다면 CPU가 다른 일을 거의 못 해요.

[핵심] DMA란?
DMA (Direct Memory Access, 직접 메모리 접근)

CPU를 거치지 않고, 장치가 메모리와 직접 데이터를 주고받는 방식이에요.

DMA 컨트롤러라는 전용 하드웨어가 데이터 전송을 대신 처리해요. CPU는 '이 데이터를 저기로 옮겨!'라고 지시만 하고, 실제 전송은 DMA 컨트롤러가 알아서 해요. 전송이 끝나면 DMA 컨트롤러가 인터럽트로 '다 됐어요!'라고 알려줘요.
[비유] 택배 직배송 vs 직접 전달
CPU가 직접 옮기는 방식 (프로그램 I/O):
사장님이 직접 창고에 가서 물건을 하나씩 가져다 배송 트럭에 실어요.
물건을 옮기느라 사장님이 다른 일을 전혀 못 해요.

DMA 방식 (직접 메모리 접근):
사장님이 '창고 3번에서 물건 100개를 배송 센터로 옮겨!'라고 직원(DMA 컨트롤러)에게 지시해요.
직원이 알아서 옮기는 동안 사장님은 다른 중요한 일을 해요.
옮기기가 끝나면 직원이 '다 끝났습니다!'라고 보고해요 (인터럽트).

대량의 데이터를 옮길 때 DMA가 훨씬 효율적이에요!

C 언어와 입출력 -- printf와 scanf의 비밀

C 언어에서 우리가 매일 사용하는 printf와 scanf도 결국은 하드웨어의 입출력을 사용하는 거예요. 이 함수들이 어떤 과정을 거치는지 알아볼게요.

[핵심] C 언어의 I/O 함수와 하드웨어 연결
printf (출력)
printf("Hello") -> C 라이브러리 -> 운영체제(OS) -> 그래픽 드라이버 -> GPU/모니터
우리가 printf를 호출하면, 글자 데이터가 버스를 타고 모니터까지 전달되는 거예요.

scanf (입력)
키보드 키 누름 -> 키보드 컨트롤러 -> 인터럽트 -> OS -> C 라이브러리 -> scanf 반환
키를 누르면 인터럽트가 발생하고, OS가 그 키 값을 버퍼에 저장했다가 scanf가 읽어가요.

파일 I/O (fopen, fprintf, fscanf)
프로그램 -> OS -> 디스크 컨트롤러 -> DMA -> SSD/HDD
파일을 읽고 쓸 때는 대량 데이터를 옮기므로 보통 DMA 방식을 써요.

C 언어에서 #include 의 stdio는 Standard Input/Output(표준 입출력)의 줄임말이에요. 이 헤더 파일 안에 printf, scanf, fopen 같은 I/O 함수들이 선언되어 있어요.

버퍼(Buffer) -- 속도 차이를 해결하는 대기실

CPU는 아주 빠르고, 입출력 장치(키보드, 디스크 등)는 상대적으로 느려요. 이 속도 차이를 어떻게 해결할까요? 바로 버퍼(Buffer)를 사용해요.

[핵심] 버퍼란?
버퍼(Buffer): 데이터를 임시로 저장해 두는 메모리 공간이에요.

빠른 쪽(CPU)과 느린 쪽(장치) 사이에서 데이터를 모았다가 한꺼번에 전달하는 역할을 해요.

예: 키보드로 'Hello'를 치면, 각 글자가 하나씩 버퍼에 쌓이다가 Enter를 누르면 한꺼번에 프로그램에 전달돼요.
[비유] 버퍼는 대기실
병원의 대기실을 생각해 보세요.

환자(데이터)가 한 명씩 도착하는데, 의사(CPU)는 한 명을 진찰하는 데 시간이 걸려요.
대기실(버퍼)이 없으면 환자가 올 때마다 의사를 부르고, 의사가 하던 일을 멈춰야 해요.
대기실이 있으면 환자가 먼저 대기실에 앉아 있다가, 의사가 준비되면 순서대로 진찰해요.

또 다른 비유: 유튜브 영상 버퍼링!
인터넷이 느릴 때 영상이 멈추면서 '로딩 중...'이 뜨죠? 그건 영상 데이터를 버퍼에 미리 채우는 중이에요. 버퍼에 충분히 쌓이면 영상이 끊기지 않고 재생돼요.

C 언어에서 printf를 호출하면 데이터가 바로 화면에 나타나지 않을 수도 있어요. printf는 데이터를 출력 버퍼에 먼저 쌓아놓고, 버퍼가 가득 차거나 줄바꿈(\n)을 만나면 그때 한꺼번에 화면에 보내거든요. 이것을 버퍼링(Buffering)이라고 해요.

[C 코드 1] printf와 scanf의 동작 살펴보기

printf와 scanf가 실제로 어떻게 동작하는지 코드로 확인해 볼게요.

Cprintf와 scanf의 동작 확인
1#include <stdio.h>
2
3int main(void) {
4 int age;
5 char name[50];
6
7 /* printf: 데이터가 출력 버퍼에 저장됨 */
8 printf("이름을 입력하세요: ");
9 /* 줄바꿈이 없으므로 버퍼에 남아있을 수 있어요 */
10 fflush(stdout); /* 버퍼를 강제로 비워서 화면에 출력! */
11
12 /* scanf: 키보드 입력을 입력 버퍼에서 읽어옴 */
13 scanf("%s", name);
14
15 printf("나이를 입력하세요: ");
16 fflush(stdout);
17 scanf("%d", &age);
18
19 /* \n이 있으면 자동으로 버퍼가 비워져요 */
20 printf("안녕하세요, %s! %d살이시군요!\n", name, age);
21
22 return 0;
23}
실행 결과
이름을 입력하세요: 민수 나이를 입력하세요: 13 안녕하세요, 민수! 13살이시군요!
[설명] fflush(stdout)이 필요한 이유
printf에 \n(줄바꿈)이 없으면, 데이터가 출력 버퍼에 남아서 바로 화면에 안 나올 수 있어요.

fflush(stdout)는 '출력 버퍼를 지금 당장 비워!'라는 명령이에요.
stdout은 '표준 출력'(보통 화면)을 뜻해요.

이렇게 하면 \n이 없어도 즉시 화면에 글자가 나타나요.

[C 코드 2] 파일 I/O -- 데이터를 파일에 읽고 쓰기

printf/scanf가 화면과 키보드를 사용한다면, 파일 I/O는 SSD나 HDD에 데이터를 저장하고 읽어오는 거예요. fopen, fprintf, fscanf, fclose 함수를 사용해요.

C파일에 데이터 쓰고 읽기
1#include <stdio.h>
2
3int main(void) {
4 FILE *fp; /* 파일 포인터: 파일을 가리키는 변수 */
5 char name[50];
6 int score;
7
8 /* === 파일에 쓰기 (출력) === */
9 fp = fopen("student.txt", "w"); /* "w" = 쓰기 모드로 열기 */
10 if (fp == NULL) {
11 printf("파일을 열 수 없어요!\n");
12 return 1;
13 }
14
15 fprintf(fp, "민수 95\n"); /* 파일에 데이터 쓰기 */
16 fprintf(fp, "지은 88\n");
17 fprintf(fp, "현우 92\n");
18 fclose(fp); /* 반드시 닫아야 해요! */
19 printf("파일 쓰기 완료!\n");
20
21 /* === 파일에서 읽기 (입력) === */
22 fp = fopen("student.txt", "r"); /* "r" = 읽기 모드로 열기 */
23 if (fp == NULL) {
24 printf("파일을 열 수 없어요!\n");
25 return 1;
26 }
27
28 printf("--- 파일 내용 ---\n");
29 while (fscanf(fp, "%s %d", name, &score) == 2) {
30 printf("%s: %d점\n", name, score);
31 }
32 fclose(fp);
33
34 return 0;
35}
실행 결과
파일 쓰기 완료! --- 파일 내용 --- 민수: 95점 지은: 88점 현우: 92점
[핵심] 파일 I/O의 핵심 함수들
fopen(파일명, 모드): 파일을 열어요. "r"=읽기, "w"=쓰기, "a"=추가
fprintf(fp, ...): 파일에 데이터를 써요 (printf의 파일 버전)
fscanf(fp, ...): 파일에서 데이터를 읽어요 (scanf의 파일 버전)
fclose(fp): 파일을 닫아요. 반드시 닫아야 데이터가 안전하게 저장돼요!

fclose를 빼먹으면 버퍼에 남은 데이터가 파일에 기록되지 않을 수 있어요.

[C 코드 3] 버퍼 동작 확인 프로그램

버퍼가 정말 존재하는지, 어떻게 동작하는지 눈으로 확인해 볼게요.

C버퍼 동작 확인하기
1#include <stdio.h>
2
3int main(void) {
4 FILE *fp;
5
6 fp = fopen("buffer_test.txt", "w");
7 if (fp == NULL) {
8 printf("파일을 열 수 없어요!\n");
9 return 1;
10 }
11
12 /* 데이터를 쓰지만, 아직 버퍼에만 있어요 */
13 fprintf(fp, "첫 번째 줄\n");
14 fprintf(fp, "두 번째 줄\n");
15 printf("아직 파일에는 안 써졌을 수도 있어요 (버퍼에 있음)\n");
16
17 /* fflush로 버퍼를 강제로 비우기 */
18 fflush(fp);
19 printf("fflush 후: 이제 파일에 확실히 써졌어요!\n");
20
21 fprintf(fp, "세 번째 줄\n");
22
23 /* fclose는 자동으로 fflush를 해 줘요 */
24 fclose(fp);
25 printf("fclose 후: 남은 데이터도 모두 파일에 써졌어요!\n");
26
27 return 0;
28}
실행 결과
아직 파일에는 안 써졌을 수도 있어요 (버퍼에 있음) fflush 후: 이제 파일에 확실히 써졌어요! fclose 후: 남은 데이터도 모두 파일에 써졌어요!
[설명] 왜 버퍼를 사용할까?
디스크(SSD/HDD)에 데이터를 한 글자씩 쓰면 매번 디스크를 작동시켜야 해서 아주 느려요.

버퍼에 모아놓고 한꺼번에 쓰면 디스크를 한 번만 작동시키면 되니까 훨씬 빨라요!

[C 코드 4] 버퍼 있는 입력 vs 없는 입력

키보드 입력에서도 버퍼의 차이를 느낄 수 있어요. 일반적인 scanf는 Enter를 누를 때까지 버퍼에 모아놓고, 한꺼번에 전달해요.

[설명] 줄 단위 버퍼링 (Line Buffering)
키보드 입력은 보통 줄 단위 버퍼링(Line Buffering)을 사용해요.

Enter를 누르기 전까지는 입력한 글자가 버퍼에만 쌓여 있어요.
Enter를 누르면 버퍼에 있는 글자들이 한꺼번에 프로그램에 전달돼요.

그래서 scanf로 입력받을 때 Enter를 꼭 눌러야 하는 거예요!
Enter 이전에는 글자를 수정(Backspace)할 수도 있어요. 이것도 버퍼 덕분이에요.

디버깅 -- 입출력에서 자주 만나는 버그 3가지

입출력 코드에서 자주 발생하는 버그를 살펴보고 고쳐 볼게요.

[디버깅 1] scanf 버퍼에 남은 개행 문자
버그 코드:
int num;
char ch;
scanf("%d", &num); /* 숫자 입력 후 Enter */
scanf("%c", &ch); /* 문자를 입력하고 싶은데... */

문제: 두 번째 scanf가 건너뛰어져요!
원인: 첫 번째 scanf에서 숫자를 읽고 남은 Enter(\n)가 버퍼에 남아있어서, 두 번째 scanf가 그 \n을 읽어 버려요.

해결:
scanf("%d", &num);
scanf(" %c", &ch); /* %c 앞에 공백을 넣으면 공백 문자를 건너뜀 */

또는 while(getchar() != '\n');으로 버퍼를 비워요.
[디버깅 2] 파일 닫기(fclose) 누락
버그 코드:
FILE *fp = fopen("data.txt", "w");
fprintf(fp, "중요한 데이터");
/* fclose(fp)를 빼먹음! */
return 0;

문제: 파일에 데이터가 저장되지 않거나 일부만 저장돼요!
원인: fprintf는 데이터를 버퍼에 쓰는데, fclose를 하지 않으면 버퍼에 남은 데이터가 파일에 기록되지 않아요. 또한 열린 파일이 쌓이면 시스템 자원이 부족해져요.

해결:
fclose(fp); /* 반드시 fclose로 파일을 닫으세요! */

규칙: fopen을 했으면 반드시 fclose를 해야 해요!
[디버깅 3] 실시간 출력이 안 되는 문제 (fflush 미사용)
버그 코드:
printf("진행 중... ");
/* 시간이 오래 걸리는 작업 */
printf("완료!");

문제: '진행 중...'이 바로 안 나오고, '완료!' 출력 때 한꺼번에 나와요.
원인: printf에 \n이 없으면 데이터가 출력 버퍼에 남아있다가, 버퍼가 가득 차거나 프로그램이 끝날 때 한꺼번에 출력돼요.

해결:
printf("진행 중... ");
fflush(stdout); /* 버퍼를 즉시 비움 -> 바로 화면에 나타남 */
/* 시간이 오래 걸리는 작업 */
printf("완료!\n");

역사 이야기 -- 입출력 장치의 발전

[역사] 천공카드 -- 최초의 데이터 입력 장치 (1890)
컴퓨터 이전 시대에는 천공카드(Punched Card)가 데이터 입력 장치였어요.

1890년, 미국 인구조사에서 허먼 홀러리스(Herman Hollerith)가 천공카드 시스템을 발명했어요. 카드에 구멍을 뚫어서 데이터를 표현하는 방식이었어요. 구멍이 있으면 1, 없으면 0이에요.

이 시스템 덕분에 이전에 8년 걸리던 인구조사를 단 1년 만에 끝낼 수 있었어요!
홀러리스가 세운 회사는 나중에 IBM이 되었답니다.
[역사] 키보드의 발전 -- 타자기에서 컴퓨터로
키보드의 역사는 1868년 크리스토퍼 숄스(Christopher Sholes)의 타자기에서 시작돼요.

지금 우리가 쓰는 QWERTY 자판 배열도 그때 만들어진 거예요! 자주 쓰는 글자를 일부러 떨어뜨려 놓았다는 설이 있어요. (타자기 활자가 엉키는 것을 방지하기 위해)

1964년, MIT에서 최초의 컴퓨터 전자 키보드가 등장했고, 이후 PC의 보급과 함께 오늘날의 키보드가 만들어졌어요.
[역사] USB의 탄생 (1996)과 블루투스의 이름
USB의 탄생:
1990년대 초, 컴퓨터 뒤에는 각기 다른 포트가 7~8개나 있었어요. 인텔, 마이크로소프트, IBM 등 7개 회사가 모여서 '하나의 포트로 통일하자!'고 만든 것이 USB예요. 1996년 USB 1.0이 출시되었고, 이후 세계에서 가장 많이 사용되는 연결 표준이 되었어요.

블루투스의 이름 유래:
10세기 덴마크 왕 하랄 블루투스(Harald Bluetooth)에서 이름을 따왔어요. 이 왕이 스칸디나비아의 부족들을 통합했듯이, 블루투스 기술도 다양한 장치를 하나로 통합한다는 의미예요. 블루투스 로고도 하랄 왕의 이름 H와 B를 룬 문자로 합친 거예요!

KOI 도전 문제

입출력과 버스 개념이 KOI(한국 정보올림피아드)에서 어떻게 나올 수 있는지 연습해 볼게요.

[KOI 1] 데이터 전송 시간 계산
버스 폭이 32비트이고, 버스 클럭이 초당 1억 회(100MHz)인 시스템이 있다.
이 시스템에서 1MB(= 1,048,576 바이트)의 데이터를 전송하는 데 걸리는 최소 시간을 구하시오.

(힌트: 한 번의 클럭에 32비트 = 4바이트를 전송할 수 있어요)
[KOI 2] 주소 버스와 메모리 용량
어떤 컴퓨터의 주소 버스가 24비트이다. 이 컴퓨터가 접근할 수 있는 최대 메모리 용량은 몇 MB인가?

(힌트: 2^24개의 주소를 지정할 수 있고, 각 주소는 1바이트를 가리켜요)
[KOI 3] 파일 I/O 프로그램 분석
다음 C 코드의 실행 후 output.txt 파일에 저장되는 내용을 쓰시오.

FILE *fp = fopen("output.txt", "w");
int i;
for (i = 1; i <= 5; i++) {
if (i % 2 == 1) {
fprintf(fp, "%d ", i * i);
}
}
fclose(fp);
[KOI 풀이]
KOI 1 풀이:
한 클럭당 전송량 = 32비트 = 4바이트
초당 전송량 = 4바이트 x 100,000,000 = 400,000,000 바이트/초 = 약 400MB/s
1MB 전송 시간 = 1,048,576 / 400,000,000 = 약 0.00262초 = 약 2.62ms

KOI 2 풀이:
24비트 주소 -> 2^24 = 16,777,216 바이트 = 16,384 KB = 16 MB

KOI 3 풀이:
i=1: 1%2==1 참 -> 1*1=1 출력 -> "1 "
i=2: 2%2==1 거짓 -> 건너뜀
i=3: 3%2==1 참 -> 3*3=9 출력 -> "9 "
i=4: 4%2==1 거짓 -> 건너뜀
i=5: 5%2==1 참 -> 5*5=25 출력 -> "25 "
결과: 1 9 25

연습문제 정답

용어 정리

용어설명
버스 (Bus)컴퓨터 내부에서 데이터, 주소, 제어 신호가 이동하는 공유 전선 다발. 데이터 버스, 주소 버스, 제어 버스 세 종류가 있음.
인터럽트 (Interrupt)입출력 장치가 CPU에 처리할 일이 있을 때 보내는 신호. CPU는 하던 일을 멈추고 해당 요청을 먼저 처리함.
DMA (Direct Memory Access)CPU를 거치지 않고 장치가 메모리와 직접 데이터를 주고받는 방식. 대량 데이터 전송에 효율적.
버퍼 (Buffer)속도가 다른 장치 사이에서 데이터를 임시로 저장하는 메모리 공간. 모아서 한꺼번에 전달해 효율을 높임.
포트 (Port)컴퓨터 외부에 있는 장치 연결 단자. USB, HDMI, 이더넷 등 다양한 종류가 있음.
ISR (Interrupt Service Routine)인터럽트가 발생했을 때 실행되는 미리 작성된 처리 프로그램. 키보드, 마우스 등 각 장치마다 전용 ISR이 있음.
폴링 (Polling)CPU가 주기적으로 모든 장치의 상태를 확인하는 방식. 비효율적이지만 구현이 간단함.
USB (Universal Serial Bus)다양한 외부 장치를 하나의 통일된 방식으로 연결하는 범용 직렬 버스. 1996년 등장 이후 가장 많이 쓰이는 연결 표준.

핵심 요약

입출력과 버스 핵심 요약
1. 입력 장치는 사람의 행동을 디지털 신호로, 출력 장치는 디지털 신호를 사람이 느낄 수 있는 형태로 변환
2. 버스는 컴퓨터 부품 사이의 공유 통로: 데이터 버스(짐), 주소 버스(목적지), 제어 버스(신호등)
3. 버스 폭이 넓을수록 한 번에 더 많은 데이터를 전송 (8비트 -> 64비트, 차선 수 비유)
4. 시스템 버스(내부 고속도로)와 확장 버스(외부 국도)가 브릿지로 연결됨
5. 포트와 인터페이스: USB(범용), HDMI(영상), 이더넷(유선 인터넷), 블루투스/Wi-Fi(무선)
6. 폴링(계속 물어보기)보다 인터럽트(손 들기)가 효율적, DMA(직원에게 맡기기)가 가장 효율적
7. 인터럽트 처리: 요청 -> 현재 작업 저장 -> ISR 실행 -> 복귀
8. 버퍼는 속도 차이를 해결하는 임시 저장 공간 (대기실 비유)
9. C 언어: printf=출력, scanf=입력, fopen/fclose=파일 I/O, fflush=버퍼 비우기
10. 디버깅 핵심: scanf 후 개행 문자 처리, fclose 잊지 않기, fflush로 즉시 출력
10축 매핑
