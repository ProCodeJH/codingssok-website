UNIT S-14 실수 표현

정수만으로는 부족해!

여러분의 키가 정확히 150cm인가요? 아마 150.3cm, 148.7cm처럼 소수점이 붙을 거예요. 체온은 36.5도, 원주율은 3.14159..., 시험 평균 점수는 82.4점. 이런 수들은 정수로는 절대 표현할 수 없어요.

[생각해 보기] 소수점이 필요한 순간들
1. 체온: 36.5도 -- 정수로 쓰면 36도? 37도? 정확하지 않아요.
2. 키: 162.3cm -- 162cm와 163cm 사이 어딘가!
3. 시험 평균: 82.4점 -- 과목 합계를 과목 수로 나누면 딱 안 떨어져요.
4. 원주율: 3.141592... -- 끝없이 이어지는 소수!
5. 나눗셈: 3 / 2 = 1.5 -- 정수끼리 나눠도 실수가 나와요.
6. 환율: 1달러 = 1,327.50원 -- 경제에서 소수점은 필수!

이런 수들을 0과 1만으로 어떻게 표현할 수 있을까요?

정수는 1, 2, 3처럼 띄엄띄엄 있는 수예요. 수직선에서 점으로 찍히죠. 반면 실수(real number)는 수직선 위의 모든 점이에요. 1과 2 사이에도 1.1, 1.01, 1.001... 무한히 많은 수가 있어요.

[핵심] 정수 vs 실수
정수(Integer): ..., -2, -1, 0, 1, 2, 3, ... (소수점 없이 딱 떨어지는 수)
실수(Real Number): 수직선 위의 모든 수. 정수 + 소수를 모두 포함해요.

컴퓨터 용어로 실수를 부동소수점수(Floating-Point Number)라고 불러요. 왜 이런 이름이 붙었는지는 곧 배울 거예요!

컴퓨터가 정수를 저장하는 방법은 S-13에서 배웠어요. 2의 보수를 사용하면 깔끔하게 양수와 음수를 다룰 수 있었죠. 하지만 실수는 사정이 달라요. 소수점 아래 자릿수가 있으니까 정수 방식을 그대로 쓸 수 없거든요.

고정소수점 -- 소수점 위치를 약속하자

소수를 2진수로 표현할 수 있다는 걸 알았으니, 이제 컴퓨터에 저장하는 방법을 생각해 봐요. 가장 단순한 아이디어는 소수점의 위치를 미리 고정해 두는 거예요.

[핵심] 고정소수점 (Fixed Point)
비트를 두 구간으로 나눠서, 왼쪽은 정수부, 오른쪽은 소수부로 고정해 두는 방식이에요.

예: 8비트를 4비트.4비트로 나누면
- 왼쪽 4비트: 정수부 (0 ~ 15)
- 오른쪽 4비트: 소수부 (0, 0.0625, 0.125, ... 0.9375)
- 소수점 위치가 절대 변하지 않아요!

[주의] 고정소수점의 치명적 단점

- 최대 15.9375까지밖에 표현 못 해요. 100은? 안 돼요!
- 최소 정밀도가 0.0625이에요. 0.01 같은 작은 소수도 정확히 못 표현해요!

과학자들은 아주 큰 수(태양까지의 거리: 149,600,000km)와 아주 작은 수(원자의 크기: 0.0000000001m)를 동시에 다뤄야 해요. 고정소수점으로는 이게 불가능해요. 그래서 더 영리한 방법이 필요했어요.

부동소수점 -- 소수점이 떠다닌다!

부동소수점의 핵심 아이디어는 과학적 표기법(Scientific Notation)에서 왔어요. 여러분도 과학 시간에 본 적 있을 거예요.

[예제] 과학적 표기법 (10진수)
아주 큰 수:
- 태양까지 거리: 149,600,000 km = 1.496 x 10^8 km
- 빛의 속도: 300,000,000 m/s = 3.0 x 10^8 m/s

아주 작은 수:
- 원자 크기: 0.0000000001 m = 1.0 x 10^(-10) m
- 바이러스 크기: 0.0000001 m = 1.0 x 10^(-7) m

형태: (부호) x 가수 x 10^지수
- 가수(Mantissa): 1 이상 10 미만의 수 (1.496, 3.0 등)
- 지수(Exponent): 소수점을 얼마나 이동할지

핵심을 눈치챘나요? 과학적 표기법에서는 소수점의 위치가 지수에 따라 바뀌어요. 지수가 8이면 소수점이 오른쪽으로 8칸, 지수가 -10이면 왼쪽으로 10칸 이동하는 거예요. 소수점이 '떠다니는' 거죠! 그래서 부동(floating)소수점이에요.

2진수에서도 똑같이 할 수 있어요! 10 대신 2를 밑(base)으로 쓰면 돼요.

[예제] 2진 과학적 표기법
10진수 6.75 = 2진수 110.11

110.11(2)을 과학적 표기법으로 바꾸면:
110.11 = 1.1011 x 2^2

소수점을 왼쪽으로 2칸 이동하고, 지수를 2로 놓았어요!

다른 예:
- 0.375(10) = 0.011(2) = 1.1 x 2^(-2)
- 5.0(10) = 101.0(2) = 1.01 x 2^2
- 0.5(10) = 0.1(2) = 1.0 x 2^(-1)

정규화 -- 1.xxx 형태로 맞추기

같은 수를 여러 가지로 쓸 수 있으면 혼란스럽겠죠? 그래서 정규화(Normalization)라는 규칙을 정했어요.

[핵심] 정규화 규칙
2진 부동소수점에서 정규화란, 소수점 왼쪽에 정확히 1 하나만 오도록 소수점을 이동하는 거예요.

형태: 1.xxxxx x 2^n

맨 앞이 항상 1이니까, 이 1은 저장하지 않아도 알 수 있어요!
이걸 숨겨진 비트(Hidden Bit) 또는 암묵적 1(Implicit 1)이라고 해요.
덕분에 비트 하나를 아낄 수 있어요!

IEEE 754 표준 -- 전 세계가 약속한 규칙

1970~80년대에는 컴퓨터 회사마다 실수를 저장하는 방식이 달랐어요. A 회사 컴퓨터로 저장한 실수를 B 회사 컴퓨터에서 읽으면 엉뚱한 값이 나왔죠! 그래서 1985년, 전 세계가 하나의 표준을 만들었어요.

[핵심] IEEE 754 표준
IEEE 754는 부동소수점 수를 저장하는 국제 표준이에요.

모든 부동소수점 수는 세 부분으로 구성돼요:

1. 부호(Sign) -- 양수인지 음수인지 (1비트)
2. 지수(Exponent) -- 소수점을 얼마나 이동할지
3. 가수(Mantissa/Fraction) -- 유효한 숫자들

바이어스(Bias) -- 지수에 127을 더하는 이유

지수는 양수일 수도 있고 음수일 수도 있어요. 2^3도 필요하고 2^(-5)도 필요하니까요. 그런데 지수부에 2의 보수를 쓰면 크기 비교가 복잡해져요. 그래서 IEEE 754는 다른 방법을 선택했어요.

[핵심] 바이어스 (Bias) 방식
실제 지수에 바이어스라는 고정된 수를 더해서 항상 양수로 저장해요.

32비트 float: 바이어스 = 127
- 저장되는 값 = 실제 지수 + 127
- 실제 지수 = 저장된 값 - 127

64비트 double: 바이어스 = 1023
- 저장되는 값 = 실제 지수 + 1023
- 실제 지수 = 저장된 값 - 1023

비유: 엘리베이터에서 지하 2층을 '-2층' 대신 '(127-2)=125층'이라고 부르는 것과 비슷해요!

특수한 값들 -- 0, 무한대, NaN

IEEE 754에는 일반적인 수 외에도 몇 가지 특수한 값이 정의되어 있어요. 지수부가 모두 0이거나 모두 1인 경우에 해당해요.

[핵심] NaN (Not a Number)
NaN은 '수가 아닌 것'이에요. 수학적으로 정의되지 않는 연산의 결과예요.

- sqrt(-1.0) (음수의 제곱근)
- INF - INF (무한대 빼기 무한대)

NaN의 특이한 성질: NaN != NaN이 참이에요!
자기 자신과 같지 않은 유일한 값이에요.

정밀도의 한계 -- 왜 0.1 + 0.2 != 0.3 인가?

컴퓨터가 바보라서가 아니에요. 문제는 10진수 0.1을 2진수로 정확히 표현할 수 없다는 점이에요.

C 언어에서 실수를 다루는 두 가지 주요 자료형이 있어요. float와 double이에요. double은 float보다 2배 크기(64비트)라서 '배정밀도(double precision)'라는 이름이 붙었어요.

실수 비교의 위험 -- == 대신 epsilon!

부동소수점 오차 때문에, 실수를 ==으로 비교하면 안 돼요! 이건 초보 프로그래머가 가장 자주 범하는 실수 중 하나예요.

[주의] 절대 하면 안 되는 실수 비교
if (a == b)    /* 위험! 거의 같아도 false가 될 수 있어요! */

0.1 + 0.2는 0.30000000000000004이므로 0.3과 ==로 비교하면 거짓이 돼요!

대신 이렇게 비교하세요:
if (fabs(a - b) < 0.000001)  /* 두 수의 차이가 아주 작으면 같다고 판단 */

여기서 0.000001 같은 아주 작은 값을 epsilon이라고 불러요.

[역사] 부동소수점의 탄생 -- 콘라트 추제와 Z3 (1941)
세계 최초의 프로그래밍 가능한 컴퓨터 Z3(1941, 독일)을 만든 콘라트 추제(Konrad Zuse)는 부동소수점 연산을 지원하는 최초의 컴퓨터를 설계했어요. 당시 다른 컴퓨터들은 고정소수점만 지원했는데, 추제는 과학 계산에 부동소수점이 필수적이라는 것을 일찍 깨달았어요. Z3은 22비트 부동소수점(부호 1비트 + 지수 7비트 + 가수 14비트)을 사용했어요.
[역사] IEEE 754 표준의 아버지 -- 윌리엄 카한 (1985)
1970년대까지 컴퓨터마다 부동소수점 형식이 달랐어요. UC 버클리의 수학자 윌리엄 카한(William Kahan) 교수가 이 혼란을 정리하기 위해 표준화 작업을 이끌었고, 1985년에 IEEE 754 표준이 탄생했어요. 카한 교수는 이 공로로 1989년에 컴퓨터 과학의 노벨상이라 불리는 튜링상을 수상했어요. 오늘날 전 세계 모든 컴퓨터가 이 표준을 따르고 있어요.
[역사] 부동소수점 오류로 인한 사건들
1991 패트리어트 미사일: 0.1초의 고정소수점 오차가 100시간 누적되어 0.34초 오차. 28명 사망.

1996 아리안 5 로켓: 64비트 부동소수점을 16비트 정수로 변환할 때 오버플로우 발생. 발사 37초 만에 폭발. 5억 달러 손실.

2010 밴쿠버 증권거래소: 주가 지수 계산에서 부동소수점 반올림 오차가 22개월간 누적. 지수가 실제보다 약 50포인트 낮게 표시됨.

이 사례들은 부동소수점을 정확히 이해하는 것이 얼마나 중요한지 보여 줘요.

KOI 도전 문제

[KOI 도전 1] 원의 넓이 비교
문제: 반지름이 각각 r1, r2인 두 원이 주어져요. 두 원의 넓이가 같은지 판단하는 프로그램을 작성하세요. (넓이 = 3.141592653589793 x r x r)

입력: 두 줄에 실수 r1, r2가 주어져요. (0.0 < r1, r2 <= 1000.0)
출력: 넓이가 같으면 "EQUAL", 다르면 "NOT EQUAL"

핵심: 실수 비교 시 == 대신 epsilon 비교를 해야 해요!

풀이 방향:
double area1 = PI * r1 * r1;
double area2 = PI * r2 * r2;
if (fabs(area1 - area2) < 1e-9)
    printf("EQUAL\n");
else
    printf("NOT EQUAL\n");

입력: 실수 N (|N| <= 10000.0)과 정수 K (0 <= K <= 10)
출력: 반올림된 값

핵심: printf의 서식 지정자를 동적으로 만들어야 해요!

풀이 방향:
printf("%.*f\n", K, N);
/* %.*f 에서 * 자리에 K가 들어가요! */
[KOI 도전 3] 정밀도 함정
문제: 양의 실수 X가 주어져요. X에 0.001을 1000번 더한 값과, X에 1.0을 더한 값이 같은지 판단하세요.

입력: 실수 X (0.0 < X <= 1000000.0)
출력: 같으면 "YES", 다르면 "NO"

핵심: X가 커질수록 0.001의 누적 오차가 심해져요. X가 작을 때는 같을 수 있지만, X가 매우 클 때는 float 정밀도에서 차이가 날 수 있어요.

풀이 방향: double로 계산하고, epsilon 비교를 적용하세요. 루프의 누적 오차와 한 번의 덧셈의 차이를 이해하는 것이 핵심이에요.

용어 정리

용어설명
부동소수점 (Floating Point)소수점의 위치가 고정되지 않고, 지수에 따라 이동하는 실수 표현 방식. 넓은 범위의 수를 표현할 수 있어요.
고정소수점 (Fixed Point)소수점의 위치를 미리 고정해 두는 실수 표현 방식. 구현이 간단하지만 표현 범위가 좁아요.
IEEE 754전 세계 컴퓨터가 따르는 부동소수점 표현 국제 표준. 1985년에 제정되었고, 부호+지수+가수 구조를 정의해요.
가수 (Mantissa / Fraction)부동소수점에서 유효 숫자를 담는 부분. 1.xxxxx에서 소수점 이하 부분을 저장해요.
지수 (Exponent)부동소수점에서 소수점의 위치(크기의 스케일)를 나타내는 부분. 바이어스를 더해서 저장해요.
바이어스 (Bias)지수를 항상 양수로 저장하기 위해 더하는 고정 값. float는 127, double은 1023이에요.
NaN (Not a Number)수가 아닌 특수한 값. 0/0이나 sqrt(-1)의 결과예요. 자기 자신과도 같지 않은 특이한 성질이 있어요.
epsilon (엡실론)실수 비교 시 '같다'고 판단하는 기준이 되는 아주 작은 값. fabs(a-b) < epsilon이면 두 수를 같다고 봐요.

이 단원의 핵심 정리

실수 표현 -- 한눈에 보기
1. 실수(소수점이 있는 수)는 정수 방식으로는 저장할 수 없어요. 별도의 표현 방법이 필요해요.
2. 고정소수점: 소수점 위치를 고정. 간단하지만 범위가 좁아요.
3. 부동소수점: 소수점이 이동! 과학적 표기법(1.xxx x 2^n)과 같은 원리예요.
4. 정규화: 항상 1.xxx 형태로 만들어요. 맨 앞의 1은 저장하지 않아도 돼요 (숨겨진 비트).
5. IEEE 754: 부호(1비트) + 지수(8/11비트) + 가수(23/52비트) 구조.
6. 바이어스: 지수를 양수로 만들기 위해 127(float) 또는 1023(double)을 더해요.
7. 0.1은 2진수에서 무한소수! 그래서 0.1 + 0.2 != 0.3이에요.
8. float(32비트, 유효 7자리) vs double(64비트, 유효 15자리). 기본으로 double을 쓰세요!
9. 실수 비교: == 대신 fabs(a-b) < epsilon을 사용하세요!

[10축 매핑] 이 단원에서 기른 능력
1. 논리적 사고력: 고정소수점과 부동소수점의 장단점을 논리적으로 비교하고 분석했어요.
2. 수학적 사고력: 2진 소수 변환, 정규화, 바이어스 계산 등 수학 연산을 수행했어요.
3. 추상화: '소수'라는 추상적 개념을 부호+지수+가수라는 비트 패턴으로 표현하는 과정을 이해했어요.
4. 패턴 인식: 0.1의 2진 변환에서 0011 반복 패턴을 발견하고, 무한소수가 되는 원리를 파악했어요.

7. 컴퓨터 구조 이해: 부동소수점 하드웨어(FPU)의 필요성과 IEEE 754 표준의 역할을 이해했어요.
8. 데이터 표현: 같은 수를 고정소수점, 부동소수점, float, double 등 다양한 방식으로 표현하는 법을 배웠어요.
9. C 언어 활용: float/double 사용, printf 서식 지정자, union을 통한 비트 패턴 확인, epsilon 비교 함수를 작성했어요.
10. 역사적 맥락: Z3 컴퓨터, IEEE 754 제정, 패트리어트 미사일 사건 등 부동소수점의 역사와 실제 영향을 배웠어요.
