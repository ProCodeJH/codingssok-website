UNIT S-17 네트워크 기초

인터넷은 어떻게 작동할까?

여러분은 매일 인터넷을 사용해요. 스마트폰으로 친구에게 메시지를 보내고, 컴퓨터로 웹사이트를 열고, 태블릿으로 영상을 봐요. 하지만 이런 일이 어떻게 가능한 걸까요? 그 비밀은 바로 네트워크(Network)에 있어요.

이번 단원에서는 컴퓨터가 서로 대화하는 방법, 즉 네트워크의 기초를 배울 거예요. 전화를 걸 때 전화번호가 필요하듯이, 컴퓨터끼리 통신할 때도 주소와 규칙이 필요하거든요.

[생각해 보기] 네트워크가 없다면?
네트워크가 없다면 이런 일이 벌어져요:

1. 카카오톡, 인스타그램 등 메신저를 사용할 수 없어요.
2. 웹사이트를 열 수 없어요 -- 구글 검색도 불가능해요.
3. 온라인 게임을 할 수 없어요 -- 혼자서만 플레이해야 해요.
4. 파일을 USB에 담아서 직접 들고 가야 해요.
5. 클라우드 저장소(구글 드라이브 등)를 사용할 수 없어요.

네트워크 덕분에 우리는 전 세계 어디와도 순식간에 연결될 수 있어요!

네트워크란 무엇일까?

네트워크(Network)란 두 대 이상의 컴퓨터가 서로 연결되어 데이터를 주고받을 수 있는 통신망이에요. 컴퓨터뿐만 아니라 스마트폰, 프린터, 서버 등 다양한 장치가 네트워크에 연결될 수 있어요.

[비유] 전화망
네트워크를 이해하는 가장 쉬운 비유는 전화망이에요.

- 전화기 = 컴퓨터 (데이터를 보내고 받는 장치)
- 전화번호 = IP 주소 (상대방을 찾기 위한 주소)
- 전화선/기지국 = 케이블/라우터 (데이터가 지나가는 길)
- 통화 규칙 = 프로토콜 ("여보세요?" -> 대화 -> "끊을게요")
- 전화번호부 = DNS (이름으로 번호를 찾아주는 서비스)

전화를 걸려면 번호를 알아야 하고, 서로 같은 언어로 말해야 하듯이, 컴퓨터 통신도 주소와 규칙이 필요해요!

LAN vs WAN

네트워크는 크기에 따라 여러 종류로 나뉘어요. 가장 중요한 두 가지는 LAN(Local Area Network)과 WAN(Wide Area Network)이에요.

LAN(근거리 통신망)은 학교, 사무실, 집처럼 좁은 범위에서 컴퓨터를 연결하는 네트워크예요. 우리 집 와이파이가 대표적인 LAN이에요. WAN(광역 통신망)은 도시와 도시, 나라와 나라를 연결하는 넓은 범위의 네트워크예요. 인터넷이 가장 큰 WAN이에요.

네트워크 토폴로지

토폴로지(Topology)란 네트워크에서 컴퓨터들이 어떤 형태로 연결되어 있는지를 나타내는 구조예요. 도시의 도로망이 다양한 형태로 설계되듯이, 네트워크도 여러 가지 형태로 구성할 수 있어요.

토폴로지모양장점단점비유

프로토콜이란?

프로토콜(Protocol)이란 컴퓨터끼리 통신할 때 지켜야 하는 규칙이에요. 사람이 대화할 때 같은 언어를 사용해야 하고, 인사 -> 대화 -> 작별 순서를 지키듯이, 컴퓨터도 정해진 규칙에 따라 데이터를 주고받아야 해요.

[비유] 편지 보내기 예절
편지를 보낼 때도 규칙이 있어요:

1. 봉투에 주소를 적어야 해요 -- 받는 사람을 알아야 하니까요. (= IP 주소)
2. 우표를 붙여야 해요 -- 비용을 지불해야 배달해 줘요. (= 패킷 형식)
3. 우체통에 넣어야 해요 -- 정해진 방법으로 보내야 해요. (= 네트워크 장비)
4. 우편번호를 적으면 더 빨라요 -- 라우팅이 효율적이에요. (= 서브넷)

이런 규칙들이 바로 프로토콜이에요!

OSI 7계층 모델

네트워크 통신은 매우 복잡해요. 이 복잡한 과정을 이해하기 쉽게 7개의 계층으로 나눈 것이 OSI 7계층 모델이에요. ISO(국제표준화기구)에서 만든 표준 모델이에요.

[비유] 편지 보내기로 이해하는 OSI 7계층
친구에게 편지를 보내는 과정에 비유하면:

7계층 (응용): 편지 내용을 써요 -- "내일 놀자!"
6계층 (표현): 상대방이 읽을 수 있는 언어로 바꿔요 (한국어 -> 한국어 OK!)
5계층 (세션): 편지를 시작하고 끝내요 -- "안녕? ... 잘 자!"
4계층 (전송): 긴 편지를 여러 장으로 나누고 번호를 매겨요
3계층 (네트워크): 봉투에 주소를 적어요 -- 서울시 OO구...
2계층 (데이터 링크): 우체국에서 같은 지역 배달 경로를 정해요
1계층 (물리): 실제로 우편 트럭이 편지를 운반해요
계층이름역할데이터 단위대표 장비/프로토콜
7응용사용자 인터페이스 제공데이터HTTP, FTP, SMTP, DNS
6표현데이터 형식 변환/암호화데이터SSL/TLS, JPEG, ASCII
5세션연결 설정/유지/해제데이터NetBIOS, RPC
4전송신뢰성 있는 전송, 흐름 제어세그먼트TCP, UDP
3네트워크경로 설정 (라우팅)패킷IP, ICMP, 라우터
2데이터 링크물리 주소(MAC) 기반 전송프레임이더넷, 스위치
1물리전기/광 신호로 비트 전송비트케이블, 허브, 리피터

TCP/IP 4계층 모델

OSI 7계층은 이론적인 모델이고, 실제 인터넷에서 사용하는 것은 TCP/IP 4계층 모델이에요. OSI 7계층을 더 실용적으로 4개로 합친 거예요.

IP 주소

집마다 주소가 있듯이, 네트워크에 연결된 모든 장치에는 IP 주소(Internet Protocol Address)가 있어요. IP 주소가 있어야 데이터를 정확한 목적지로 보낼 수 있어요.

IPv4는 현재 가장 많이 사용하는 IP 주소 형식이에요. 32비트(4바이트)로 구성되고, 0부터 255까지의 숫자 4개를 점(.)으로 구분해서 표시해요. 예를 들어 192.168.1.1처럼요.

서브넷 마스크(Subnet Mask)는 IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 데 사용해요. 예를 들어 서브넷 마스크가 255.255.255.0이면, 앞의 3자리(24비트)가 네트워크 주소이고, 마지막 1자리(8비트)가 호스트 주소예요.

항목10진수2진수
IP 주소192.168.1.10011000000.10101000.00000001.01100100
서브넷 마스크255.255.255.011111111.11111111.11111111.00000000
네트워크 주소 (AND 연산)192.168.1.011000000.10101000.00000001.00000000
호스트 부분0.0.0.10000000000.00000000.00000000.01100100
[비유] 서브넷 마스크 = 아파트 주소 체계
서울시 강남구 역삼동 123번지 아파트 502호

- 네트워크 주소 = 서울시 강남구 역삼동 123번지 (같은 아파트인지 확인)
- 호스트 주소 = 502호 (몇 호인지 확인)
- 서브넷 마스크 = 어디까지가 아파트 주소이고 어디부터가 호수인지 알려주는 규칙

같은 아파트(네트워크) 안의 사람들은 직접 찾아가면 되지만, 다른 아파트(네트워크)의 사람에게는 우편으로 보내야 해요!

MAC 주소

MAC 주소(Media Access Control Address)는 네트워크 장치(랜카드, Wi-Fi 칩)에 공장에서 부여하는 고유한 하드웨어 주소예요. IP 주소가 바뀔 수 있는 '논리적' 주소라면, MAC 주소는 변하지 않는 '물리적' 주소예요.

MAC 주소는 48비트(6바이트)로 구성되고, 16진수로 표시해요. 예를 들어 AA:BB:CC:11:22:33처럼 콜론(:)으로 구분해요. 앞 3바이트(AA:BB:CC)는 제조사를 나타내고, 뒤 3바이트(11:22:33)는 장치 고유 번호예요.

DNS -- 인터넷의 전화번호부

우리는 웹사이트에 접속할 때 www.google.com처럼 이름을 입력하지, 142.250.196.110같은 IP 주소를 입력하지 않아요. 이름을 IP 주소로 바꿔주는 시스템이 바로 DNS(Domain Name System)예요.

[비유] DNS = 전화번호부
친구에게 전화하고 싶은데 번호를 모르면? 전화번호부에서 이름으로 검색하죠!

- 이름 = 도메인 이름 (www.google.com)
- 전화번호 = IP 주소 (142.250.196.110)
- 전화번호부 = DNS 서버
- 검색 = DNS 질의 (Query)

DNS가 없다면 모든 웹사이트의 IP 주소를 외워야 해요. 상상만 해도 끔찍하죠?

패킷(Packet) -- 데이터를 쪼개서 보내기

큰 파일을 네트워크로 보낼 때, 파일 전체를 한 번에 보내면 문제가 생겨요. 중간에 오류가 나면 처음부터 다시 보내야 하고, 다른 사람이 네트워크를 사용할 수 없거든요. 그래서 데이터를 작은 조각으로 나누어 보내는데, 이 조각을 패킷(Packet)이라고 해요.

부분내용역할비유
출발지 IP보내는 컴퓨터의 IP 주소누가 보냈는지 알려줌보낸 사람 주소
도착지 IP받는 컴퓨터의 IP 주소어디로 가야 하는지 알려줌받는 사람 주소
순서 번호전체에서 몇 번째 패킷인지도착 후 순서대로 재조립편지 페이지 번호
데이터 길이이 패킷에 담긴 데이터 크기얼마나 많은 데이터가 있는지편지 분량
체크섬오류 검출용 코드전송 중 데이터가 변하지 않았는지 확인봉인 스티커
데이터 (페이로드)실제 전송할 내용진짜 보내고 싶은 정보편지 내용

TCP vs UDP

데이터를 전송하는 방법에는 크게 두 가지가 있어요. TCP(Transmission Control Protocol)와 UDP(User Datagram Protocol)예요. 둘 다 전송 계층(4계층)의 프로토콜이지만, 성격이 매우 달라요.

HTTP/HTTPS -- 웹 통신

HTTP(HyperText Transfer Protocol)는 웹 브라우저와 웹 서버 사이에서 웹 페이지를 주고받는 프로토콜이에요. 우리가 웹사이트를 열 때마다 HTTP가 사용되고 있어요.

HTTPS는 HTTP에 S(Secure)를 더한 것이에요. HTTP로 주고받는 데이터를 암호화해서 중간에 누가 엿볼 수 없게 해요. 로그인, 결제 등 중요한 정보를 다루는 사이트는 반드시 HTTPS를 사용해야 해요.

C 코드 1: IP 주소를 2진수로 변환하기

IP 주소(예: 192.168.1.1)의 각 숫자를 8비트 2진수로 변환해서 출력하는 프로그램을 만들어 보아요.

1#include <stdio.h>
2
3void print_binary(int num) {
4 int i;
5 for (i = 7; i >= 0; i--) {
6 printf("%d", (num >> i) & 1);
7 }
8}
9
10int main(void) {
11 int ip[4] = {192, 168, 1, 1};
12 int i;
13
14 printf("IP 주소: %d.%d.%d.%d\n", ip[0], ip[1], ip[2], ip[3]);
15 printf("2진수: ");
16
17 for (i = 0; i < 4; i++) {
18 print_binary(ip[i]);

26}

print_binary 함수는 숫자를 8비트 2진수로 출력해요. 비트 시프트(>>)와 AND 연산(&)을 사용해서, 각 비트 위치의 값(0 또는 1)을 하나씩 꺼내요. i가 7부터 0까지 줄어들면서 최상위 비트(MSB)부터 출력해요.

C 코드 2: 패킷 구조체 만들기

C 언어의 구조체(struct)를 이용해서 간단한 패킷을 표현하고 출력해 보아요.

1#include <stdio.h>
2#include <string.h>
3
4typedef struct {

11

14 printf("출발지: %s\n", p.src_ip);
15 printf("도착지: %s\n", p.dst_ip);
16 printf("순서: %d번\n", p.seq_num);
17 printf("크기: %d바이트\n", p.data_len);
18 printf("내용: %s\n", p.data);
19 printf("=================\n");
20}
21
22int main(void) {
23 Packet pkt1, pkt2;
24
25 strcpy(pkt1.src_ip, "192.168.1.10");
26 strcpy(pkt1.dst_ip, "10.0.0.1");
27 pkt1.seq_num = 1;
28 strcpy(pkt1.data, "Hello, ");
29 pkt1.data_len = strlen(pkt1.data);
30
31 strcpy(pkt2.src_ip, "192.168.1.10");
32 strcpy(pkt2.dst_ip, "10.0.0.1");
33 pkt2.seq_num = 2;
34 strcpy(pkt2.data, "World!");
35 pkt2.data_len = strlen(pkt2.data);
36
37 print_packet(pkt1);
38 print_packet(pkt2);
39

41

43}

C 코드 4: 서브넷 마스크 계산기

IP 주소와 서브넷 마스크를 AND 연산하여 네트워크 주소를 구하는 프로그램을 만들어 보아요.

C서브넷 마스크 계산기
1#include <stdio.h>
2
3int main(void) {
4 int ip[4] = {192, 168, 1, 100};
5 int mask[4] = {255, 255, 255, 0};
6 int net[4];
7 int host[4];
8 int i;
9
10 /* 네트워크 주소 = IP AND 서브넷 마스크 */
11 for (i = 0; i < 4; i++) {
12 net[i] = ip[i] & mask[i];
13 host[i] = ip[i] & (~mask[i] & 0xFF);
14 }
15
16 printf("IP 주소: %d.%d.%d.%d\n",
17 ip[0], ip[1], ip[2], ip[3]);
18 printf("서브넷 마스크: %d.%d.%d.%d\n",
19 mask[0], mask[1], mask[2], mask[3]);
20 printf("네트워크 주소: %d.%d.%d.%d\n",
21 net[0], net[1], net[2], net[3]);
22 printf("호스트 부분: %d.%d.%d.%d\n",
23 host[0], host[1], host[2], host[3]);
24
25 printf("\n--- 같은 네트워크인지 확인 ---\n");
26
27 int other[4] = {192, 168, 1, 200};
28 int other_net[4];
29 int same = 1;
30
31 for (i = 0; i < 4; i++) {
32 other_net[i] = other[i] & mask[i];
33 if (net[i] != other_net[i]) {
34 same = 0;
35 }
36 }
37
38 printf("%d.%d.%d.%d 와 %d.%d.%d.%d 는 ",
39 ip[0], ip[1], ip[2], ip[3],
40 other[0], other[1], other[2], other[3]);
41
42 if (same) {
43 printf("같은 네트워크입니다!\n");
44 } else {
45 printf("다른 네트워크입니다!\n");
46 }
47
48 return 0;
49}
실행 결과
IP 주소: 192.168.1.100 서브넷 마스크: 255.255.255.0 네트워크 주소: 192.168.1.0 호스트 부분: 0.0.0.100 --- 같은 네트워크인지 확인 --- 192.168.1.100 와 192.168.1.200 는 같은 네트워크입니다!

IP 주소와 서브넷 마스크를 비트 AND(&) 연산하면 네트워크 주소가 나와요. 두 IP의 네트워크 주소가 같으면 같은 네트워크(LAN) 안에 있다는 뜻이에요. ~mask[i] & 0xFF는 마스크를 반전(NOT)시켜서 호스트 부분만 추출하는 거예요.

디버깅 연습

네트워크 관련 C 코드에서 자주 발생하는 버그 3가지를 찾아보아요.

[디버깅 1] IP 주소 범위 오류
다음 코드에서 버그를 찾아보세요:

int ip[4] = {192, 168, 1, 300};
printf("IP: %d.%d.%d.%d\n", ip[0], ip[1], ip[2], ip[3]);

버그: ip[3]의 값이 300이에요! IPv4 주소의 각 옥텟은 0~255 범위여야 하는데 300은 범위를 벗어나요. 또한 C 언어에서 int는 300을 저장할 수 있지만, 이것은 유효하지 않은 IP 주소예요.

수정: 입력값이 0~255 범위인지 검사하는 코드를 추가해야 해요.
if (ip[i] < 0 || ip[i] > 255) {
  printf("오류: %d는 유효하지 않은 값!\n", ip[i]);
}
[디버깅 2] 배열 인덱스 오류
다음 코드에서 버그를 찾아보세요:

char name[5] = "hello";
DnsEntry table[3];
for (int i = 0; i <= 3; i++) {
  printf("%s\n", table[i].domain);
}

버그 1: name[5] = "hello"는 문자열 끝의 '\0'을 포함하면 6바이트가 필요한데, 5바이트만 할당했어요. name[6]으로 바꿔야 해요.

버그 2: 반복문에서 i <= 3이면 i가 0, 1, 2, 3까지 돌아요. table[3]은 배열 범위(0~2)를 벗어나요! i < 3으로 고쳐야 해요.
[디버깅 3] 구조체 문자열 복사 오류
다음 코드에서 버그를 찾아보세요:

typedef struct {
  char ip[10];
  char name[20];
} Host;

Host h;
h.ip = "192.168.1.1";
h.name = "server";

버그 1: C 언어에서 배열에 직접 문자열을 대입(=)할 수 없어요. strcpy(h.ip, "192.168.1.1")을 사용해야 해요.

버그 2: ip 배열 크기가 10인데, "192.168.1.1"은 '\0' 포함 13바이트예요. 배열이 너무 작아요! char ip[16]으로 크기를 늘려야 해요.

네트워크의 역사

[역사] ARPANET -- 인터넷의 탄생 (1969)
인터넷의 시작은 미국 국방부의 ARPANET(Advanced Research Projects Agency Network)이에요. 1969년 10월 29일, UCLA와 스탠포드 연구소 사이에서 처음으로 컴퓨터 간 메시지가 전송되었어요. 전송하려던 메시지는 "LOGIN"이었는데, "LO"까지만 보내고 시스템이 다운되었어요! 하지만 이것이 인터넷 역사의 첫 걸음이었답니다.
[역사] TCP/IP의 탄생 (1983) -- 빈트 서프와 밥 칸
빈트 서프(Vint Cerf)와 밥 칸(Bob Kahn)은 서로 다른 네트워크를 하나로 연결하는 프로토콜인 TCP/IP를 발명했어요. 1983년 1월 1일, ARPANET이 TCP/IP로 전환되었고, 이 날이 사실상 '인터넷의 생일'이에요. 빈트 서프는 '인터넷의 아버지'라고 불려요.
[역사] WWW의 등장 (1991) -- 팀 버너스리
팀 버너스리(Tim Berners-Lee)는 유럽 입자물리연구소(CERN)에서 일하면서 월드 와이드 웹(WWW)을 발명했어요. HTML, HTTP, URL이라는 3가지 핵심 기술을 만들어서 누구나 쉽게 정보를 공유할 수 있게 했어요. 놀랍게도 그는 이 기술의 특허를 내지 않고 무료로 공개했어요!
[역사] Wi-Fi의 등장 (1997)
1997년 IEEE 802.11 표준이 발표되면서 무선 네트워크(Wi-Fi) 시대가 열렸어요. 처음에는 2Mbps 속도였지만, 현재 Wi-Fi 6는 9.6Gbps까지 가능해요. 이제 카페, 학교, 공항 어디에서든 케이블 없이 인터넷에 접속할 수 있게 되었어요.

KOI 도전 문제

한국정보올림피아드(KOI)에서 네트워크 관련 개념이 활용되는 문제를 풀어 보아요.

[KOI 문제 1] IP 주소 유효성 검사
문자열로 주어진 IP 주소가 유효한 IPv4 주소인지 판별하세요.

유효 조건:
- 점(.)으로 구분된 4개의 숫자
- 각 숫자는 0 이상 255 이하
- 앞에 불필요한 0이 없어야 함 (예: 01.02.03.04는 무효)

입력 예시: "192.168.1.1" -> 유효
"256.1.1.1" -> 무효 (256은 255 초과)
"1.2.3" -> 무효 (4개가 아님)

핵심 알고리즘: 문자열 파싱과 범위 검사. sscanf나 strtok으로 점 기준 분리 후 각 값을 검사해요.
[KOI 문제 2] 네트워크 연결 -- 최소 케이블 수
N대의 컴퓨터를 모두 연결하려면 최소 몇 개의 케이블이 필요할까요?

조건: 모든 컴퓨터가 직접 또는 간접적으로 연결되어야 해요. (연결 그래프가 하나로 이어져야 해요.)

예시: N=5일 때, 최소 4개의 케이블 (트리 구조)

핵심 개념: N개의 노드를 연결하는 트리의 간선 수는 항상 N-1이에요. 이것은 '최소 신장 트리(MST)' 문제의 기초예요. 그래프 이론(U0-35)에서 배운 내용이 여기에 적용돼요!
[KOI 문제 3] 패킷 재조립
N개의 패킷이 순서가 뒤섞여서 도착했어요. 각 패킷에는 순서 번호(1~N)가 적혀 있어요. 패킷을 원래 순서대로 재조립하세요.

입력: 5개의 패킷 [3, 1, 4, 2, 5]
출력: [1, 2, 3, 4, 5]

핵심 알고리즘: 정렬 알고리즘을 사용하면 돼요. 하지만 순서 번호가 1~N으로 연속이므로, 크기 N의 배열을 만들어 result[seq_num - 1] = packet으로 O(N)에 해결할 수도 있어요!

용어 사전

용어설명
네트워크 (Network)두 대 이상의 컴퓨터가 연결되어 데이터를 주고받을 수 있는 통신망이에요. LAN, WAN 등의 종류가 있어요.
프로토콜 (Protocol)컴퓨터끼리 통신할 때 지켜야 하는 규칙이에요. HTTP, TCP, IP 등이 대표적이에요.
IP 주소 (IP Address)네트워크에 연결된 장치의 논리적 주소예요. IPv4(32비트)와 IPv6(128비트)가 있어요.
MAC 주소 (MAC Address)네트워크 장치에 공장에서 부여하는 48비트 고유 하드웨어 주소예요. 물리적 주소라고도 해요.
DNS (Domain Name System)도메인 이름(예: google.com)을 IP 주소로 변환해 주는 시스템이에요. 인터넷의 전화번호부와 같아요.
패킷 (Packet)네트워크에서 데이터를 작은 조각으로 나눈 전송 단위예요. 헤더(주소 정보)와 데이터(내용)로 구성돼요.
TCP (Transmission Control Protocol)신뢰성 있는 데이터 전송을 보장하는 프로토콜이에요. 데이터 도착을 확인하고 순서를 보장해요.
서브넷 마스크 (Subnet Mask)IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 데 사용하는 32비트 값이에요.

한눈에 보기

네트워크 기초 -- 한눈에 보기
1. 네트워크는 컴퓨터끼리 데이터를 주고받는 통신망이에요. 전화망과 비슷한 원리예요.
2. LAN은 좁은 범위(집, 학교), WAN은 넓은 범위(도시, 나라 간) 네트워크예요. 인터넷은 가장 큰 WAN이에요.
3. 토폴로지: 버스형(일자), 스타형(중앙 집중), 링형(원형), 메시형(모두 연결). 가정은 스타형, 인터넷은 메시형이에요.
4. 프로토콜은 통신 규칙이에요. OSI 7계층(이론)과 TCP/IP 4계층(실용)으로 체계화되어 있어요.
5. IP 주소(IPv4: 32비트, IPv6: 128비트)는 논리적 주소, MAC 주소(48비트)는 물리적 주소예요.
6. 서브넷 마스크로 IP 주소의 네트워크 부분과 호스트 부분을 구분해요. AND 연산으로 네트워크 주소를 구해요.
7. DNS는 도메인 이름(google.com)을 IP 주소(142.250.196.110)로 바꿔주는 인터넷의 전화번호부예요.
8. 패킷은 데이터를 쪼갠 조각이에요. 각각 독립적으로 전송되고, 도착 후 순서대로 재조립돼요.
9. TCP는 신뢰성 우선(등기우편), UDP는 속도 우선(일반우편). 용도에 따라 선택해요.
10. C 언어는 네트워크 프로그래밍의 기초 언어예요. 소켓, 구조체, 비트 연산 등이 핵심이에요.
