UNIT S-23 컴퓨터 보안 기초

왜 보안이 중요할까?

우리는 매일 스마트폰으로 메시지를 보내고, 인터넷으로 숙제를 검색하고, 게임을 하며 디지털 세상에서 살고 있어요. 그런데 이 디지털 세상에도 도둑과 사기꾼이 있다는 것, 알고 있나요?

컴퓨터 보안(Computer Security)이란 컴퓨터 시스템과 데이터를 허가되지 않은 접근, 변경, 파괴로부터 보호하는 것이에요. 쉽게 말해서 '디지털 세상에서 나와 내 정보를 지키는 방법'이에요.

보안이 왜 중요한지 실제 사례를 볼까요? 2017년 WannaCry 랜섬웨어는 전 세계 150개국, 30만 대 이상의 컴퓨터를 감염시켰어요. 영국 병원들의 컴퓨터가 마비되어 수술이 취소되고, 환자 기록을 볼 수 없게 되었죠. 보안 문제가 사람의 생명까지 위협할 수 있다는 뜻이에요.

이것들이 한순간에 사라지거나 다른 사람에게 공개된다면 어떨까요? 보안을 배우면 이런 일을 막을 수 있어요!

보안의 3대 원칙 -- CIA

보안 전문가들은 보안의 핵심을 3가지로 정리했어요. 영어 이니셜을 따서 CIA 삼각형이라고 불러요. 미국의 정보기관 CIA와는 다른 뜻이에요!

1. 기밀성 (Confidentiality) -- 허가된 사람만 정보를 볼 수 있어야 해요. 일기장을 자물쇠로 잠그는 것과 같아요. 내 비밀번호는 나만 알아야 하고, 내 성적표는 나와 부모님, 선생님만 볼 수 있어야 해요.

2. 무결성 (Integrity) -- 정보가 허가 없이 바뀌면 안 돼요. 시험 성적이 몰래 바뀌면 큰 문제가 되겠죠? 은행 계좌의 잔액이 누군가에 의해 멋대로 바뀌면 안 되는 것과 같아요.

3. 가용성 (Availability) -- 필요할 때 정보를 사용할 수 있어야 해요. 시험 기간에 학교 홈페이지가 다운되어 공지사항을 볼 수 없다면? 필요할 때 쓸 수 있어야 의미가 있어요.

악성 소프트웨어 (맬웨어)

맬웨어(Malware)는 'Malicious Software(악의적인 소프트웨어)'의 줄임말이에요. 컴퓨터에 해를 끼치는 모든 프로그램을 통틀어 말해요. 감기처럼 컴퓨터에 '병'을 일으키는 프로그램이라고 생각하면 돼요.

바이러스 (Virus)는 다른 프로그램에 몰래 붙어서 퍼지는 맬웨어예요. 진짜 바이러스처럼 숙주(다른 프로그램)가 있어야 활동해요. USB에 있는 파일을 열거나, 이메일 첨부 파일을 실행하면 감염될 수 있어요.

웜 (Worm)은 바이러스와 달리 숙주 없이 스스로 복제하고 네트워크를 통해 퍼져요. 마치 벌레가 땅속을 기어다니듯, 네트워크를 타고 컴퓨터에서 컴퓨터로 이동해요. 1988년 모리스 웜이 대표적이에요.

트로이 목마 (Trojan Horse)는 유용한 프로그램인 척 위장해서 사용자가 직접 설치하게 만드는 맬웨어예요. 그리스 신화에서 목마 안에 병사를 숨긴 것처럼, 게임이나 유틸리티 프로그램 안에 악성 코드가 숨어 있어요.

랜섬웨어 (Ransomware)는 컴퓨터의 파일을 암호화한 뒤 '돈을 내면 풀어주겠다'고 협박하는 맬웨어예요. 'Ransom(몸값)'에서 이름이 왔어요. 2017년 WannaCry가 대표적인 랜섬웨어예요.

소셜 엔지니어링 -- 사람을 속이는 공격

컴퓨터를 직접 해킹하는 것보다 사람을 속이는 것이 훨씬 쉬울 때가 많아요. 소셜 엔지니어링(Social Engineering)은 기술이 아닌 사람의 심리를 이용해서 정보를 빼내는 공격 방법이에요.

피싱(Phishing)은 가짜 웹사이트나 이메일로 개인 정보를 빼내는 공격이에요. 'Fishing(낚시)'에서 이름이 왔어요. 진짜 은행 사이트처럼 생긴 가짜 사이트를 만들어서 아이디와 비밀번호를 입력하게 만드는 거예요.

스미싱(Smishing)은 SMS(문자 메시지)를 이용한 피싱이에요. '택배가 도착했습니다. 확인: http://xx.xx' 같은 문자를 보내서 가짜 사이트로 유도하는 방법이에요.

[주의] 피싱을 알아보는 핵심 신호
1. 급한 톤: '지금 당장', '24시간 이내', '계정이 정지됩니다' -- 서두르게 만들어서 판단력을 흐려요.
2. 개인 정보 요청: 정상적인 서비스는 이메일이나 문자로 비밀번호를 물어보지 않아요.
3. URL 확인: 링크 위에 마우스를 올려서(클릭하지 말고!) 실제 주소를 확인하세요.
4. 맞춤법 오류: 가짜 사이트는 종종 맞춤법이 틀리거나 어색한 문장이 있어요.
5. 너무 좋은 조건: '100만원 당첨', '무료 아이폰' 같은 건 거의 100% 사기예요.

비밀번호 보안

비밀번호는 디지털 세상에서 나를 증명하는 첫 번째 열쇠예요. 하지만 많은 사람들이 약한 비밀번호를 사용하거나, 여러 사이트에서 같은 비밀번호를 쓰고 있어요. 매년 가장 많이 사용되는 비밀번호 1위는 '123456'이에요!

이중 인증(2FA, Two-Factor Authentication)은 비밀번호 외에 추가로 한 가지 더 확인하는 방법이에요. 집 현관문에 열쇠(비밀번호)로 열고, 추가로 지문 인식(2FA)까지 해야 들어갈 수 있는 것과 비슷해요.

소유(Something you have)내가 가진 것스마트폰, OTP 토큰, 보안 카드물리적으로 필요분실 위험

네트워크 보안

인터넷은 마치 우체국과 같아요. 편지(데이터)가 여러 우체국(서버)을 거쳐 전달되죠. 그런데 누군가가 중간에서 편지를 몰래 열어볼 수 있다면? 네트워크 보안은 이런 위험에서 데이터를 보호하는 것이에요.

방화벽(Firewall)은 네트워크의 '경비원'이에요. 허가된 데이터만 통과시키고, 수상한 데이터는 차단해요. 아파트 경비실에서 방문자를 확인하고 출입을 통제하는 것과 같아요.

HTTPS는 인터넷에서 데이터를 암호화해서 전송하는 방법이에요. HTTP에 'S(Secure, 보안)'가 추가된 것이에요. 편지를 봉투에 넣고 밀봉하는 것과 같아서, 중간에서 누가 가로채도 내용을 읽을 수 없어요.

VPN(Virtual Private Network, 가상 사설 네트워크)은 인터넷 위에 암호화된 터널을 만드는 기술이에요. 공개된 도로 대신 비밀 지하 통로를 만드는 것과 비슷해요. 카페의 공용 Wi-Fi를 쓸 때 VPN을 켜면 훨씬 안전해요.

소프트웨어 취약점

프로그램을 만들 때 실수로 '구멍'을 남기면, 해커가 그 구멍을 통해 들어올 수 있어요. 이런 구멍을 취약점(Vulnerability)이라고 해요. 특히 C 언어는 메모리를 직접 다루기 때문에 취약점이 생기기 쉬워요.

버퍼 오버플로우(Buffer Overflow)는 배열(버퍼)의 크기를 넘어서 데이터를 쓰는 것이에요. 10명만 앉을 수 있는 교실에 20명이 들어가면 어떻게 될까요? 옆 교실까지 넘쳐서 혼란이 생기겠죠. 컴퓨터에서도 마찬가지예요.

주소정상 상태오버플로우 후설명

SQL 인젝션(SQL Injection)은 웹사이트의 입력 칸에 SQL 명령어를 넣어서 데이터베이스를 조작하는 공격이에요. 예를 들어 로그인 화면에서 아이디 칸에 ' OR '1'='1을 입력하면, SQL 쿼리가 항상 참이 되어 비밀번호 없이 로그인될 수 있어요.

XSS(Cross-Site Scripting)는 웹페이지에 악성 스크립트를 삽입하는 공격이에요. 게시판에 글을 쓸 때 <script>alert('해킹!')</script>같은 코드를 넣으면, 그 글을 읽는 다른 사용자의 브라우저에서 악성 스크립트가 실행될 수 있어요.

[코드 3] 비밀번호 강도 검사기

1#include <stdio.h>
2#include <string.h>
3#include <ctype.h>
4
5int check_strength(const char *pw) {
6 int len = strlen(pw);
7 int has_upper = 0; /* 대문자 포함? */
8 int has_lower = 0; /* 소문자 포함? */
9 int has_digit = 0; /* 숫자 포함? */
10 int has_special = 0; /* 특수문자 포함? */
11 int score = 0;
12 int i;
13
14 for (i = 0; i < len; i++) {
15 if (isupper(pw[i])) has_upper = 1;
16 else if (islower(pw[i])) has_lower = 1;
17 else if (isdigit(pw[i])) has_digit = 1;
18 else has_special = 1;
19 }
20
21 /* 길이 점수 */
22 if (len >= 8) score += 1;
23 if (len >= 12) score += 1;
24 if (len >= 16) score += 1;
25
26 /* 문자 종류 점수 */
27 score += has_upper + has_lower
28 + has_digit + has_special;
29
30 return score; /* 0~7 */
31}

[코드 4] 간단한 로그인 시스템 (해시 비교)
비밀번호를 평문으로 저장하면 위험해요. 간단한 해시 함수를 사용해서 비밀번호를 변환한 값을 비교하는 방법을 배워요.
C간단한 로그인 시스템
1#include <stdio.h>
2#include <string.h>
3
4/* 간단한 해시 함수 (교육용) */
5unsigned int simple_hash(const char *str) {
6 unsigned int hash = 5381;
7 int i;
8 for (i = 0; str[i] != '\0'; i++) {
9 hash = hash * 33 + str[i];
10 }
11 return hash;
12}
13
14int main(void) {
15 /* 저장된 사용자 정보 (해시값으로 저장) */
16 const char *stored_user = "admin";
17 unsigned int stored_hash = 3143856977u;
18 /* simple_hash("s3cuR!ty") 의 결과 */
19
20 char user[50];
21 char pw[50];
22 int attempts = 0;
23 const int MAX_TRY = 3;
24
25 while (attempts < MAX_TRY) {
26 printf("\n아이디: ");
27 fgets(user, sizeof(user), stdin);
28 user[strcspn(user, "\n")] = '\0';
29
30 printf("비밀번호: ");
31 fgets(pw, sizeof(pw), stdin);
32 pw[strcspn(pw, "\n")] = '\0';
33
34 if (strcmp(user, stored_user) == 0
35 && simple_hash(pw) == stored_hash) {
36 printf("로그인 성공!\n");
37 return 0;
38 }
39
40 attempts++;
41 printf("로그인 실패! (시도 %d/%d)\n",
42 attempts, MAX_TRY);
43 }
44
45 printf("시도 횟수 초과! 계정이 잠겼어요.\n");
46 return 1;
47}

[디버깅 1] 버퍼 오버플로우 취약점

void copy_name(char *input) {
  char name[10];
  strcpy(name, input); /* 버그! */
  printf("이름: %s\n", name);
}

문제: strcpy는 input의 길이를 확인하지 않아요. input이 10자보다 길면 name 버퍼를 넘쳐서 버퍼 오버플로우가 발생해요.

수정:
void copy_name(char *input) {
  char name[10];
  strncpy(name, input, sizeof(name) - 1);
  name[sizeof(name) - 1] = '\0';
  printf("이름: %s\n", name);
}

보안의 역사

[역사] Creeper -- 최초의 컴퓨터 바이러스 (1971년)
1971년, BBN 테크놀로지의 밥 토머스(Bob Thomas)가 만든 Creeper는 최초의 자기 복제 프로그램이에요. ARPANET(인터넷의 전신)을 통해 컴퓨터에서 컴퓨터로 이동하면서 'I'm the creeper, catch me if you can!'이라는 메시지를 표시했어요. 악의적인 목적은 아니었고 실험이었지만, 이후 Creeper를 잡기 위한 Reaper라는 프로그램이 만들어졌어요. Reaper가 최초의 백신 프로그램인 셈이에요.
[역사] 모리스 웜 -- 인터넷을 마비시킨 사건 (1988년)
1988년, 코넬대 대학원생 로버트 모리스(Robert Morris)가 만든 모리스 웜은 인터넷의 약 10%에 해당하는 6,000대의 컴퓨터를 감염시켰어요. 당시 인터넷에 연결된 컴퓨터가 약 60,000대였으니 엄청난 피해였죠. 모리스는 호기심에 만들었다고 주장했지만, 미국 최초의 컴퓨터 사기 및 남용법(CFAA) 위반으로 유죄 판결을 받았어요. 이 사건을 계기로 CERT(컴퓨터 비상 대응팀)가 만들어졌어요.
[역사] 에스토니아 사이버 공격 (2007년)
2007년, 유럽의 작은 나라 에스토니아가 대규모 사이버 공격을 받았어요. 정부, 은행, 언론사의 웹사이트가 DDoS(분산 서비스 거부) 공격으로 마비되었어요. 에스토니아는 세계에서 가장 디지털화된 국가 중 하나였기에 피해가 막대했어요. 이 사건은 사이버 공격이 국가 안보 위협이 될 수 있음을 보여주었고, NATO는 사이버 방어 센터를 에스토니아에 설립했어요.
[역사] WannaCry 랜섬웨어 (2017년)
2017년 5월, WannaCry 랜섬웨어가 전 세계를 덮쳤어요. 윈도우의 SMB 취약점을 이용해 150개국, 30만 대 이상의 컴퓨터를 감염시켰어요. 감염된 컴퓨터의 파일을 암호화하고 비트코인으로 300달러를 요구했어요. 영국 NHS(국민건강서비스)의 병원들이 마비되어 수술이 취소되는 심각한 사태가 벌어졌어요. 22살의 영국 보안 연구원 마커스 허친스가 '킬 스위치'를 발견해 확산을 막았어요.

KOI 도전 문제

[KOI 유형 1] 비밀번호 조합 수 계산
비밀번호에 사용할 수 있는 문자가 다음과 같을 때, 가능한 비밀번호 조합의 수를 계산하세요.

- 소문자 영문: 26가지
- 대문자 영문: 26가지
- 숫자: 10가지
- 특수문자: 10가지

(a) 소문자만 사용한 4자리 비밀번호: 26^4 = 456,976가지
(b) 소문자+숫자를 사용한 6자리 비밀번호: 36^6 = 2,176,782,336가지
(c) 전부 사용한 8자리 비밀번호: 72^8 = 722,204,136,308,736가지

사용하는 문자 종류와 길이가 늘어날수록 조합 수가 기하급수적으로 증가해요. 이것이 바로 강한 비밀번호의 원리예요!
[KOI 유형 2] 버퍼 오버플로우 추적
다음 C 코드에서 입력값에 따른 메모리 상태를 추적하세요.

char a[4] = "AB";
char b[4] = "CD";

메모리 배치 (연속 저장 가정):
| 주소 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
| 변수 | a[0] | a[1] | a[2] | a[3] | b[0] | b[1] | b[2] | b[3] |
| 값 | 'A' | 'B' | '\0' | ? | 'C' | 'D' | '\0' | ? |

만약 strcpy(a, "HELLO")를 실행하면?
| 값 | 'H' | 'E' | 'L' | 'L' | 'O' | '\0' | '\0' | ? |

a 배열을 넘어서 b의 영역까지 덮어썼어요! 이제 b를 출력하면 "O"가 나와요. 이것이 버퍼 오버플로우예요.
[KOI 유형 3] 해시 충돌 문제
다음 간단한 해시 함수가 있어요.

int hash(char *s) {
  int h = 0, i;
  for (i = 0; s[i]; i++)
    h = (h + s[i]) % 100;
  return h;
}

(a) hash("AB")의 결과는?
'A'=65, 'B'=66 -- h = (0+65)%100 = 65, h = (65+66)%100 = 31

(b) hash("BA")의 결과는?
'B'=66, 'A'=65 -- h = (0+66)%100 = 66, h = (66+65)%100 = 31

(c) "AB"와 "BA"의 해시값이 같아요! 이것을 해시 충돌이라고 해요. 이 해시 함수는 문자의 순서를 구별하지 못하는 약점이 있어요. 실제 보안용 해시 함수(SHA-256)는 이런 충돌이 사실상 불가능하도록 설계되어 있어요.

CIA 삼각형 (CIA Triad)보안의 3대 원칙인 기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability)을 묶어 부르는 말이에요.

해시 함수 (Hash Function)임의의 데이터를 고정 길이의 값으로 변환하는 함수예요. 비밀번호 저장에 사용돼요.
방화벽 (Firewall)네트워크에서 허가되지 않은 접근을 차단하는 보안 장치예요.
HTTPS (HyperText Transfer Protocol Secure)HTTP에 암호화(SSL/TLS)를 추가한 프로토콜로, 데이터를 안전하게 전송해요.
