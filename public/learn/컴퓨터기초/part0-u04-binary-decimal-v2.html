2진수 ↔ 10진수 변환

10진수를 2진수로, 2진수를 10진수로 바꾸는 방법을 배워요. 음수 표현, 소수점, 실무 활용까지!

이 유닛에서 배우는 것

  • 10진수 자릿값과 2진수 자릿값의 차이를 알 수 있어요.
  • 10진수를 2진수로 변환할 수 있어요 (나누기 2 반복).
  • 2진수를 10진수로 변환할 수 있어요 (자릿값 더하기).
  • 음수를 2진수로 표현하는 방법(부호 비트, 1의 보수, 2의 보수)을 알아요.
  • 2진수 소수(고정소수점)를 이해하고 변환할 수 있어요.
  • BCD, IEEE 754, 오버플로우 등 심화 개념을 맛볼 수 있어요.

UNIT 04 2진수 ↔ 10진수 변환

10진수의 자릿값

우리가 쓰는 숫자는 10진수예요. 0~9까지 10개의 숫자를 사용하고, 자리가 하나 올라갈 때마다 값이 10배씩 커져요.

[핵심] 10진수 자릿값 숫자 3725를 뜯어보면:
3 x 1000(천의 자리) + 7 x 100(백의 자리) + 2 x 10(십의 자리) + 5 x 1(일의 자리)
= 3000 + 700 + 20 + 5 = 3725

자릿값: ... → 1000 → 100 → 10 → 1 (10배씩 커짐)

2진수의 자릿값

2진수는 0과 1만 사용해요. 자리가 올라갈 때마다 값이 2배씩 커져요.

[핵심] 2진수 자릿값 2진수의 자릿값: ... → 128 → 64 → 32 → 16 → 8 → 4 → 2 → 1

10진수: 자릿값이 10배씩 커짐 (1, 10, 100, 1000...)
2진수: 자릿값이 2배씩 커짐 (1, 2, 4, 8, 16, 32...)

2진수 → 10진수 변환

[핵심] 2진수 → 10진수 방법 각 자리 숫자 x 자릿값을 전부 더하면 돼요.

예: 1101(2) → 10진수

🏛️ 2진수의 역사

[역사] 라이프니츠와 2진수의 탄생 2진수를 처음 체계적으로 연구한 사람은 독일의 수학자 고트프리트 라이프니츠(1646~1716)예요. 그는 1703년에 2진수 체계를 논문으로 발표했어요. 라이프니츠는 중국의 고대 점술서 '주역(I Ching)'의 음양 기호에서 영감을 받았다고 해요. 음(⚋)은 0, 양(⚊)은 1과 같은 원리거든요!
[역사] 조지 불의 논리 대수 조지 불(1815~1864)은 영국의 수학자예요. 그는 '참(1)과 거짓(0)'만으로 논리를 수학적으로 표현하는 불 대수(Boolean Algebra)를 만들었어요. AND, OR, NOT 같은 논리 연산이 바로 불 대수에서 나왔어요. 이것이 나중에 컴퓨터의 핵심 원리가 됩니다.
[역사] 클로드 섀넌 — 디지털 시대의 아버지 클로드 섀넌(1916~2001)은 MIT 석사 논문(1937)에서 전기 스위치로 불 대수를 구현할 수 있다는 걸 증명했어요. 스위치 ON=1, OFF=0 → 이것이 바로 현대 디지털 컴퓨터의 시작이에요! 그의 논문은 '20세기 가장 중요한 석사 논문'이라 불려요.

음수 표현: 부호 비트

컴퓨터에는 '-' 기호가 없어요. 오직 0과 1뿐이에요. 그래서 음수를 표현하기 위해 맨 앞 비트를 특별하게 사용해요. 이것을 부호 비트(sign bit)라고 해요.

[핵심] 부호-크기 표현 (Sign-Magnitude) 가장 단순한 방법: 맨 앞 비트로 부호를 나타내요.

• 맨 앞 비트 0 → 양수 (+)
• 맨 앞 비트 1 → 음수 (-)

8비트 예시:
0 1101001 = +105
1 1101001 = -105

⚠️ 문제점: +0(00000000)과 -0(10000000) 두 가지가 생겨요!

음수 표현: 1의 보수

1의 보수(One's Complement)는 양수의 모든 비트를 뒤집어서 음수를 만드는 방법이에요.

[핵심] 1의 보수 만들기 방법: 모든 비트를 뒤집기 (0→1, 1→0)

예: +13 = 00001101
비트 뒤집기 → 11110010 = -13 (1의 보수)

예: +7 = 00000111
비트 뒤집기 → 11111000 = -7 (1의 보수)

⚠️ 여전히 +0(00000000)과 -0(11111111) 두 가지가 있어요.

음수 표현: 2의 보수 ⭐

2의 보수(Two's Complement)는 현대 컴퓨터에서 음수를 표현하는 표준 방법이에요. 1의 보수에서 1을 더하면 돼요.

[핵심] 2의 보수 만들기 방법 1: 비트 뒤집고 + 1
1단계: 모든 비트 뒤집기 (1의 보수)
2단계: 1 더하기

예: +13 = 00001101
1단계(뒤집기): 11110010
2단계(+1): 11110011 = -13 (2의 보수)

방법 2: 오른쪽부터 첫 번째 1까지 그대로, 나머지 뒤집기
+13 = 00001101
→ 오른쪽 '1'까지 유지: ...01
→ 나머지 뒤집기: 11110011 ✓
[팁] 2의 보수의 장점 • 0이 하나뿐이에요 (00000000)
• 덧셈 회로 하나로 뺄셈도 가능해요 (A - B = A + (-B))
• 하드웨어 구현이 간단해요

8비트 2의 보수 범위: -128 ~ +127
n비트 2의 보수 범위: -2^(n-1) ~ +2^(n-1) - 1

[심화] 2의 보수 덧셈과 뺄셈

2의 보수의 최대 장점은 덧셈 회로만으로 뺄셈을 할 수 있다는 거예요. A - B를 하고 싶으면 A + (-B)로 바꾸면 돼요. -B는 2의 보수로 구하면 되고요.

[심화] 2의 보수 연산 예시 예1: 7 + (-3) = 4

예2: -5 + (-3) = -8
11111011 (-5)
+ 11111101 (-3)
─────────
1 11111000 → 캐리 무시 → 11111000 = -8 ✓

[심화] 오버플로우와 언더플로우

오버플로우(overflow)는 계산 결과가 표현 가능한 범위를 넘어갈 때 발생해요. 마치 자동차 계기판이 999999에서 000000으로 돌아가는 것과 같아요.

[주의] 오버플로우 발생 조건 양수 오버플로우: 양수 + 양수 = 음수?! 🤯
8비트: 01111111(+127) + 00000001(+1) = 10000000(-128)

음수 오버플로우(언더플로우): 음수 + 음수 = 양수?!
8비트: 10000000(-128) + 11111111(-1) = 01111111(+127)

오버플로우 감지법:
• 두 양수를 더했는데 결과가 음수 → 오버플로우!
• 두 음수를 더했는데 결과가 양수 → 오버플로우!
• 양수 + 음수는 절대 오버플로우 안 남
[실화] 오버플로우가 만든 사고 1996년 아리안 5호 로켓이 발사 37초 만에 폭발했어요. 원인은? 64비트 숫자를 16비트로 변환할 때 오버플로우가 발생한 것! 소프트웨어 한 줄의 오류가 5억 달러짜리 로켓을 날려버렸어요. 오버플로우는 정말 무서운 버그예요.

BCD (Binary Coded Decimal)

BCD는 10진수의 각 자리를 4비트 2진수로 따로따로 표현하는 방식이에요. 일반적인 2진수 변환과는 달라요.

[핵심] BCD 인코딩 10진수 각 자리를 독립적으로 4비트로 변환해요.

예: 10진수 93
• 일반 2진수: 1011101(2) (7비트)
• BCD: 1001 0011 (9→1001, 3→0011) (8비트)

예: 10진수 256
• 일반 2진수: 100000000(2) (9비트)
• BCD: 0010 0101 0110 (2→0010, 5→0101, 6→0110) (12비트)
[팁] BCD는 어디에 쓰이나요? • 디지털 시계: 각 자리를 7세그먼트 디스플레이에 바로 보여줘야 하니까
• 전자 계산기: 10진수 그대로 저장해야 반올림 오류가 없으니까
• 금융 시스템: 돈 계산에서 소수점 오류가 생기면 안 되니까

BCD는 메모리를 더 쓰지만, 10진수 ↔ 표시 변환이 빠르고 정확해요.

고정소수점: 2진수로 소수 표현하기

소수점 아래 자릿값은 10진수에서 0.1, 0.01, 0.001... 이렇게 10분의 1씩 작아지죠? 2진수에서는 2분의 1씩 작아져요.

[핵심] 2진수 소수점 자릿값 소수점 오른쪽 자릿값:
• 첫째 자리: 2⁻¹ = 0.5
• 둘째 자리: 2⁻² = 0.25
• 셋째 자리: 2⁻³ = 0.125
• 넷째 자리: 2⁻⁴ = 0.0625

예: 0.101(2) = 0×0.5 + 1×0.5 + 0×0.25 + 1×0.125
아, 정정! 0.101(2) = 1×0.5 + 0×0.25 + 1×0.125 = 0.5 + 0.125 = 0.625
[주의] 무한 소수가 될 수 있어요! 10진수 0.1을 2진수로 바꾸면?
0.1 × 2 = 0.2 → 0
0.2 × 2 = 0.4 → 0
0.4 × 2 = 0.8 → 0
0.8 × 2 = 1.6 → 1
0.6 × 2 = 1.2 → 1
0.2 × 2 = 0.4 → 0 (반복 시작!)

0.1(10) = 0.000110011001100... (2) — 영원히 반복!
이래서 컴퓨터에서 0.1 + 0.2 ≠ 0.3 같은 오류가 생겨요.

[심화 내용] IEEE 754 부동소수점 맛보기

고정소수점은 소수점 위치가 고정되어 있어요. 하지만 아주 큰 수(은하 사이 거리)와 아주 작은 수(원자 크기)를 동시에 다루려면 소수점이 움직이는 방식이 필요해요. 이것이 부동소수점(floating point)이에요.

[심화 내용] IEEE 754 구조 (32비트 단정밀도) 32비트를 세 부분으로 나눠요:

| 부호 (1비트) | 지수 (8비트) | 가수 (23비트) |

• 부호 비트: 0이면 양수, 1이면 음수
• 지수(exponent): 소수점을 얼마나 이동할지 (편향값 127 사용)
• 가수(mantissa): 실제 숫자의 유효 자릿수

예: -6.5를 IEEE 754로 표현
-6.5 = -110.1(2) = -1.101 × 2²
부호: 1 (음수)
지수: 2 + 127 = 129 = 10000001(2)
가수: 101 (1. 뒤의 부분)
→ 1 10000001 10100000000000000000000
[팁] 부동소수점은 왜 중요할까? C언어의 float가 바로 32비트 IEEE 754예요!
double은 64비트 버전(배정밀도)이고요.
