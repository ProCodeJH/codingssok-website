UNIT 07 파일과 폴더

파일이 뭐야?

파일은 컴퓨터에 저장되는 데이터 덩어리예요. 사진, 음악, 동영상, 문서, 코드 — 컴퓨터에 있는 모든 것이 파일이에요.

모든 파일에는 이름이 있어요. 그리고 이름 뒤에 점(.)과 확장자가 붙어요. 예를 들어 hello.c에서 'hello'가 이름이고, '.c'가 확장자예요.

[핵심] 파일의 구성 요소 파일 = 이름 + 확장자 + 내용(데이터) + 메타데이터
메타데이터 = 파일 크기, 생성 날짜, 수정 날짜, 권한 등
운영체제가 이 정보를 관리해서 우리가 파일을 찾고 열 수 있는 거예요.

확장자: 파일의 이름표

확장자를 보면 그 파일이 어떤 종류인지 바로 알 수 있어요. 마치 이름표처럼요.

확장자종류설명열리는 프로그램
.cC 소스코드사람이 쓴 C 코드VS Code, 메모장
.h헤더 파일함수 선언 모음VS Code
.exe실행 파일컴파일 결과더블클릭으로 실행
.o / .obj오브젝트 파일컴파일 중간 결과링커가 사용
.txt텍스트일반 글자 파일메모장
.pdf문서인쇄용 문서PDF 뷰어
.jpg/.png이미지사진/그림사진 뷰어
.mp3/.wav오디오소리/음악음악 플레이어

[주의] 확장자를 바꾸면? hello.txt의 확장자를 .exe로 바꾼다고 프로그램이 되지 않아요!
확장자는 '이름표'일 뿐, 파일의 실제 내용은 바뀌지 않아요.
사진 파일의 확장자를 .txt로 바꾸면 메모장이 열지만 알 수 없는 글자만 보여요.

매직 넘버: 실제로 파일 종류를 구별하는 건 파일 맨 앞 몇 바이트예요.
- PNG: 89 50 4E 47
- PDF: 25 50 44 46 (%PDF)
- EXE: 4D 5A (MZ)

폴더: 파일을 정리하는 서랍장

파일이 많아지면 정리가 필요해요. 그때 쓰는 게 폴더(folder)예요. 폴더는 파일을 넣어두는 서랍장 같은 거예요. 폴더 안에 또 폴더를 만들 수도 있어요.

학교 사물함을 생각해 보세요. 큰 사물함(C: 드라이브) 안에 칸(Users 폴더)이 있고, 칸 안에 파일 넣는 주머니(Documents 폴더)가 있는 거예요.

[팁] 폴더 = 디렉터리 폴더(folder)와 디렉터리(directory)는 같은 말이에요.
Windows에서는 '폴더', Unix/Linux에서는 '디렉터리'라고 더 많이 불러요.
프로그래밍에서는 주로 디렉터리라고 해요.

경로: 파일의 주소

파일이 어디에 있는지 알려주는 주소를 경로(path)라고 해요. 우편 주소처럼 큰 곳부터 작은 곳까지 순서대로 써요.

[핵심] 경로 읽는 법 C:\Users\내이름\coding\hello.c

C: → C 드라이브 (컴퓨터의 큰 저장소)
Users → 사용자들 폴더
내이름 → 나의 폴더
coding → 코딩용 폴더
hello.c → 파일!

역슬래시(\)가 폴더와 폴더 사이를 구분해줘요.

[심화] 절대 경로 vs 상대 경로

경로에는 두 종류가 있어요. 절대 경로는 맨 처음(루트)부터 쭉 쓰는 거고, 상대 경로는 지금 내가 있는 위치부터 쓰는 거예요.

[심화] 트리 구조와 루트

파일 시스템은 트리(나무) 구조예요. 맨 꼭대기를 루트(root)라고 해요. Windows에서는 C:\, D:\ 같은 드라이브가 루트이고, Linux에서는 / (슬래시 하나)가 루트예요.

용도WindowsLinux/Mac
루트C:\/
사용자 홈C:\Users\이름/home/이름
시스템 파일C:\Windows/usr, /bin, /etc
임시 파일C:\Windows\Temp/tmp
구분자\ (역슬래시)/ (슬래시)
대소문자구분 안 함구분함!
[주의] 대소문자 구분 Windows: Hello.c = hello.c = HELLO.C (같은 파일!)
Linux: Hello.c ≠ hello.c ≠ HELLO.C (전부 다른 파일!)

코딩할 때는 항상 소문자로 통일하는 습관을 들이세요.

[심화] 환경 변수 PATH

터미널에서 gcc라고 치면 컴퓨터가 gcc.exe를 어떻게 찾을까요? 모든 폴더를 다 뒤지는 건 아니에요. PATH라는 환경 변수에 '여기서 찾아봐'라고 적힌 폴더 목록이 있어요.

3
4# Linux/Mac에서 PATH 확인

6
7# gcc가 어디 있는지 찾기
8# Windows: where gcc
9# Linux: which gcc
[팁] PATH 추가하는 법 새 프로그램을 설치했는데 터미널에서 안 되면?
→ 그 프로그램의 폴더를 PATH에 추가해야 해요!

Windows: 시스템 환경 변수 → Path → 편집 → 새로 만들기
Linux: export PATH=$PATH:/new/path (또는 ~/.bashrc에 추가)

코딩할 때 파일 정리 규칙

[주의] 파일 이름 짓기 규칙 1. 영어 소문자로 쓰세요 (한글 이름은 에러의 원인!)
2. 공백(띄어쓰기) 금지 — 대신 하이픈(-)이나 밑줄(_) 사용

[심화] 텍스트 파일 vs 바이너리 파일

파일은 크게 두 종류로 나눠요. 텍스트 파일은 사람이 읽을 수 있는 글자로 이루어져 있고, 바이너리 파일은 컴퓨터만 이해하는 0과 1의 데이터예요.

[심화] 파일 시스템이란?

하드디스크나 SSD에 파일을 저장하려면 규칙이 필요해요. '어디에 저장할지, 이름은 어떻게 관리할지' — 이 규칙을 파일 시스템(File System)이라고 해요.

파일 시스템OS최대 파일 크기최대 볼륨특징
FAT32모든 OS4GB2TBUSB 메모리에 많이 사용, 호환성 최고
exFAT모든 OS16EB128PBFAT32 후속, 대용량 USB/SD카드
NTFSWindows16TB256TBWindows 기본, 보안/저널링/압축
ext4Linux16TB1EBLinux 기본, 저널링, 안정적
APFSmacOS/iOS8EB8EBApple 최신, SSD 최적화, 암호화
HFS+macOS(구)8EB8EBAPFS 이전 Mac 표준
[팁] USB에 4GB 이상 파일이 안 들어가요! FAT32는 파일 하나의 최대 크기가 4GB예요. 4GB 이상의 동영상을 USB에 넣으려면 USB를 exFAT으로 포맷해야 해요. (주의: 포맷하면 기존 데이터 삭제!)

[심화 내용] inode — 파일의 진짜 정체

Linux에서 파일 이름은 사실 '라벨'일 뿐이에요. 파일의 진짜 정보는 inode라는 자료구조에 들어 있어요.

[핵심] inode에 저장되는 정보 - 파일 크기
- 소유자/그룹 (UID/GID)
- 권한 (rwxrwxrwx)
- 타임스탬프 (생성/수정/접근 시간)
- 데이터 블록의 위치 (디스크 어디에 저장됐는지)
- 링크 카운트

❌ 파일 이름은 inode에 없어요! 이름은 디렉터리 엔트리에 있어요.
디렉터리 = (파일이름, inode번호) 쌍의 목록

[심화 내용] 파일 디스크립터 (File Descriptor)

C에서 파일을 열면 운영체제가 파일 디스크립터(fd)라는 정수 번호를 줘요. 프로그램은 이 번호로 파일을 읽고 써요.

번호이름C 표준 이름용도
0stdinstdin표준 입력 (키보드)
1stdoutstdout표준 출력 (화면)
2stderrstderr표준 에러 (에러 메시지)
3+-fopen() 반환사용자가 연 파일
Cfd_demo.c
1#include <stdio.h>
2#include <fcntl.h> // open()
3#include <unistd.h> // read(), write(), close()
4
5int main(void) {
6 // 저수준 I/O (시스템 콜)
7 int fd = open("test.txt", O_RDONLY); // fd = 3 (보통)
8 if (fd < 0) {
9 perror("open");
10 return 1;
11 }
12
13 char buf[100];
14 int n = read(fd, buf, sizeof(buf) - 1);
15 buf[n] = '\0';
16
17 write(1, buf, n); // fd=1 = stdout에 출력
18 close(fd);
19
20 return 0;
21}

[심화 내용] 파일 권한 (Unix)

Linux/Mac에서 ls -l을 치면 -rwxr-xr-- 같은 문자가 보여요. 이게 파일 권한이에요.

문자의미비트8진수
r읽기 (read)1004
w쓰기 (write)0102
x실행 (execute)0011
-권한 없음0000
Cchmod_example.sh
1# 권한 변경
2chmod 755 hello.c # rwxr-xr-x(소유자 전부, 나머지 읽기+실행)
3chmod 644 hello.c # rw-r--r-- (소유자 읽기쓰기, 나머지 읽기만)
4chmod +x script.sh # 실행 권한 추가
5chmod u+w,g-w file # 소유자 쓰기 추가, 그룹 쓰기 제거

[심화 내용] 저널링 파일 시스템

파일을 쓰는 도중에 정전이 되면? 파일이 반만 저장되어 손상될 수 있어요. 저널링(Journaling)은 이걸 방지하는 기술이에요.

[핵심] 저널링 원리 1. 변경 내용을 먼저 저널(로그)에 기록
2. 실제 파일에 변경 적용
3. 완료되면 저널에서 삭제

정전 시: 재부팅 후 저널을 보고 미완성 작업을 복구하거나 되돌림!
마치 은행 거래 기록처럼 안전하게 데이터를 보호해요.
NTFS, ext4, APFS 모두 저널링을 지원해요.

용어 정리

용어설명
파일 (file)컴퓨터에 저장되는 데이터 덩어리. 이름과 확장자로 구분해요.
폴더 / 디렉터리파일을 정리하는 묶음. Windows에서는 폴더, Linux에서는 디렉터리.
경로 (path)파일이 어디에 있는지 알려주는 주소. 절대 경로와 상대 경로가 있어요.
확장자 (extension)파일 이름 뒤의 .c, .exe 등. 파일의 종류를 나타내요.
절대 경로루트(C:\ 또는 /)부터 시작하는 전체 경로.
상대 경로현재 위치 기준의 경로. .은 현재, ..은 부모 디렉터리.
파일 시스템디스크에 파일을 저장하는 규칙 체계. FAT32, NTFS, ext4 등.
inodeUnix/Linux에서 파일의 메타데이터를 저장하는 자료구조.
파일 디스크립터 (fd)열린 파일을 가리키는 정수 번호. 0=stdin, 1=stdout, 2=stderr.
저널링파일 변경을 로그에 먼저 기록하여 정전 시 데이터를 보호하는 기술.
심볼릭 링크다른 파일의 경로를 가리키는 특수 파일. 바로가기와 비슷.
매직 넘버파일 맨 앞의 바이트로, 실제 파일 형식을 식별하는 값.
이번 유닛 핵심 정리
1. 파일 = 데이터 덩어리, 폴더 = 파일을 정리하는 서랍장
2. 확장자(.c, .exe, .txt)를 보면 파일 종류를 알 수 있어요
3. 경로는 파일의 주소 — Windows \, Linux /
4. [심화] 절대 경로(루트부터) vs 상대 경로(현재부터)
5. [심화] FAT32(4GB 제한), NTFS(Windows), ext4(Linux)
6. [심화 내용] inode = 파일의 진짜 정보, 이름은 디렉터리에
7. [심화 내용] fd 0/1/2 = stdin/stdout/stderr
이 유닛에서 연결된 축 [C언어] 소스파일(.c), 실행파일(.exe), 헤더파일(.h), fopen/fclose
[컴퓨팅 사고력] 분류와 정리 — 데이터를 구조화하는 첫 걸음
[코딩 기초 습관] 파일명 규칙 (영어 소문자, 공백 금지, 확장자 필수)
[실생활] 컴퓨터 파일 정리 = 방 정리와 같은 원리
[실생활] 파일 정리의 중요성 프로 개발자도 파일 정리를 잘 못하면 고생해요. 프로젝트가 커지면 파일이 수백, 수천 개가 되거든요. 처음부터 폴더를 깔끔하게 나누는 습관을 들이면 나중에 엄청 편해요. 정리 안 한 바탕화면에 파일 100개... 상상만 해도 무섭죠?

파일 확장자 심화

파일 확장자는 프로그램에게 "이 파일을 어떻게 열어야 하는지" 알려주는 역할을 해요.

확장자용도여는 프로그램
.cC 소스코드VS Code, gcc
.exe실행 파일Windows
.txt텍스트메모장
.jpg사진그림판
.mp3음악미디어 플레이어

경로 심화: 절대 vs 상대

절대 경로: C:\Users\me\hello.c — 처음부터 전체 주소
상대 경로: .\hello.c — 현재 위치 기준

파일 크기 단위

파일 크기 1 Byte = 8비트
1 KB = 1,024 Byte (약 1천)
1 MB = 1,024 KB (약 백만)
1 GB = 1,024 MB (약 10억)
1 TB = 1,024 GB (약 1조)

사진 1장 ≈ 3MB, 영화 1편 ≈ 4GB, 게임 1개 ≈ 50GB
