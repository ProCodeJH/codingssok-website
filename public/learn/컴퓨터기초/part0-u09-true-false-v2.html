UNIT 09 참과 거짓

참일까? 거짓일까?

우리가 매일 쓰는 말 중에는 맞다/틀리다를 확실히 말할 수 있는 것이 있어요. 예를 들어볼까요?

[예시] 맞다/틀리다를 말할 수 있는 문장 "서울은 한국의 수도다" → 참 (맞아요!)
"1 + 1 = 3" → 거짓 (틀려요! 1+1=2니까)
"물은 100°C에서 끓는다" → 참
"고양이는 새다" → 거짓

이렇게 확실하게 참인지 거짓인지 판단할 수 있는 문장을 명제라고 해요.

[핵심] 명제(Proposition)의 정의 명제란 참(True)인지 거짓(False)인지 확실하게 판단할 수 있는 문장이에요.

중요한 조건: 누가 판단해도 같은 결과가 나와야 해요!

명제가 아닌 것

그런데 모든 문장이 명제인 건 아니에요. 사람마다 다르게 생각할 수 있는 문장은 명제가 아니에요.

[주의] 이런 건 명제가 아니에요! "오늘 날씨 좋다" → 사람마다 다르게 느껴요
"피자가 치킨보다 맛있다" → 취향이에요
"수학은 재미있다" → 사람마다 달라요
"빨리 와!" → 명령이지 판단할 수 있는 문장이 아니에요
"지금 몇 시야?" → 질문이에요

컴퓨터는 참/거짓으로 생각해요

컴퓨터는 0과 1만 이해한다고 했죠? (U0-03에서 배웠어요!) 참과 거짓도 마찬가지예요. 컴퓨터에서는 참 = 1, 거짓 = 0이에요.

[핵심] 컴퓨터의 참/거짓 참 (True) = 1 = 전기가 흐른다 = 스위치 ON
거짓 (False) = 0 = 전기가 안 흐른다 = 스위치 OFF

컴퓨터는 모든 판단을 이 두 가지로 해요!

참/거짓은 왜 중요할까?

참과 거짓은 프로그래밍의 가장 기본적인 뼈대예요. 컴퓨터가 하는 거의 모든 일은 '이 조건이 참이면 이걸 해라, 거짓이면 저걸 해라'의 반복이에요.

줄코드역할
1로그인 화면비밀번호가 맞으면(참) → 로그인 성공, 틀리면(거짓) → '비밀번호가 틀립니다' 표시
2게임체력이 0 이하면(참) → 게임 오버, 아니면(거짓) → 계속 진행
3알람 앱현재 시각 = 설정 시각이면(참) → 알람 울림, 아니면(거짓) → 대기
4자동문센서에 사람이 감지되면(참) → 문 열림, 아니면(거짓) → 문 닫힘

[심화] 변수가 있는 문장 — 열린 문장

"x + 3 = 7"은 명제일까요? x가 4이면 참이고, x가 5이면 거짓이에요. 이렇게 변수의 값에 따라 참/거짓이 달라지는 문장을 열린 문장(open sentence)이라고 해요.

[심화] 명제 vs 열린 문장 명제: 참/거짓이 확정됨 — "3 + 4 = 7" (참)
열린 문장: 변수 값에 따라 달라짐 — "x + 4 = 7"

열린 문장은 변수에 값을 대입하면 명제가 돼요!
x = 3 → "3 + 4 = 7" → 참인 명제

[심화] 역사 — 불리언의 아버지, 조지 불

[역사] 조지 불 (George Boole, 1815-1864) 영국의 수학자 조지 불은 1854년에 《사고의 법칙》이라는 책을 썼어요. 이 책에서 그는 참/거짓으로 수학적 계산을 하는 체계를 만들었어요.

이것이 바로 불리언 대수(Boolean Algebra)예요. 당시에는 '쓸모없는 이론'이라는 평가를 받았지만, 약 80년 후 클로드 섀넌이 전기 회로 설계에 응용하면서 컴퓨터 시대의 기반이 되었어요!

조지 불의 이름에서 Boolean이라는 단어가 나왔어요.
[역사] 클로드 섀넌 — 디지털 시대를 연 사람 1937년, MIT 대학원생 클로드 섀넌(Claude Shannon)은 석사 논문에서 불리언 대수를 전기 스위치 회로에 적용했어요.

스위치 ON = 참(1), OFF = 거짓(0)

이 아이디어가 현대 디지털 컴퓨터의 기초가 되었어요. 이 논문은 '20세기 가장 중요한 석사 논문'으로 불려요!

[심화] 하드웨어 — 트랜지스터와 참/거짓

컴퓨터 내부에서 참/거짓은 전기 신호로 구현돼요. 트랜지스터라는 아주 작은 스위치가 전기를 흘려보내거나(참) 차단하거나(거짓) 해요.

[HW 연결] 트랜지스터 = 참/거짓 스위치 현대 CPU 안에는 트랜지스터가 수십억 개 들어있어요.

각 트랜지스터는 전기가 흐르면 1(참), 안 흐르면 0(거짓).
이 수십억 개의 참/거짓 조합으로 게임, 영상, 인공지능 등 모든 계산이 이루어져요!

예: Apple M2 칩 — 약 200억 개의 트랜지스터
줄코드역할
1논리 1 (참/HIGH)보통 3.3V 또는 5V의 전압. 트랜지스터가 ON 상태.
2논리 0 (거짓/LOW)0V(접지). 트랜지스터가 OFF 상태.
3문턱 전압 (Threshold)이 전압 이상이면 1, 이하이면 0으로 판단해요. 애매한 영역은 '불안정'이라고 해요.

[심화 내용] 진리값과 불리언 대수

불리언 대수에서는 참/거짓에 대해 수학적 연산 규칙을 정의해요. 이것이 디지털 회로 설계와 프로그래밍의 수학적 기반이에요.

[심화 내용] 불리언 대수의 기본 법칙 항등 법칙: A AND 1 = A, A OR 0 = A
영 법칙: A AND 0 = 0, A OR 1 = 1
보수 법칙: A AND NOT A = 0, A OR NOT A = 1
멱등 법칙: A AND A = A, A OR A = A
이중 부정: NOT (NOT A) = A

이 법칙들은 다음 유닛(AND, OR, NOT)에서 더 자세히 배워요!

[심화 내용] CPU 플래그 레지스터 CPU에는 플래그 레지스터(FLAGS)가 있어요. 비교(CMP) 명령 실행 후:

• ZF (Zero Flag): 결과가 0이면 1 → 두 값이 같다
• SF (Sign Flag): 결과가 음수면 1
• CF (Carry Flag): 자리올림 발생 시 1

조건 분기 명령(JE, JNE, JG, JL…)은 이 플래그를 보고 점프 여부를 결정해요.

[면접 대비] 자주 나오는 질문들 Q1: C 언어에서 NULL은 참인가요 거짓인가요?
→ 거짓. NULL은 보통 0으로 정의되어 있어요.

Q2: if (ptr)는 무슨 뜻인가요?
→ ptr이 NULL이 아닌지(유효한 포인터인지) 확인하는 관용적 표현이에요.

Q3: 불리언 값을 정수로 변환하면?
→ true = 1, false = 0. 연산에 활용 가능: count += (x > 0); 처럼 쓸 수 있어요.

연습문제

용어 정리

용어설명
명제 (Proposition)참인지 거짓인지 확실히 판단할 수 있는 문장.
참 (True)맞다는 뜻. 컴퓨터에서는 1로 표현해요.
거짓 (False)틀리다는 뜻. 컴퓨터에서는 0으로 표현해요.
불리언 (Boolean)참과 거짓, 단 2가지 값만 가지는 자료형. 영국 수학자 조지 불(George Boole)의 이름에서 왔어요.
조건 (Condition)참인지 거짓인지 판단할 수 있는 식. if문에서 사용해요.
if문C 언어에서 조건이 참일 때만 코드를 실행하는 명령어.
비교 연산자두 값을 비교해서 참/거짓을 반환하는 연산자. ==, !=, >, <, >=, <=
열린 문장 (Open Sentence)변수가 포함되어 값에 따라 참/거짓이 달라지는 문장. 변수에 값을 대입하면 명제가 돼요.
플래그 (Flag)프로그래밍에서 어떤 상태를 참/거짓으로 기억하는 불리언 변수.
불리언 대수 (Boolean Algebra)참/거짓에 대한 수학적 연산 체계. 조지 불이 창시하고, 클로드 섀넌이 디지털 회로에 응용했어요.
트랜지스터 (Transistor)전기 신호를 켜고 끄는 아주 작은 스위치. CPU 안에 수십억 개가 들어있어요.
명제 논리 (Propositional Logic)문장 단위로 참/거짓을 다루는 논리 체계.
술어 논리 (Predicate Logic)주어와 술어로 분해하고 한정사(∀, ∃)를 사용하는 더 강력한 논리 체계.
falsy프로그래밍에서 거짓으로 취급되는 값들. 언어마다 다르지만 보통 0, 빈 값, null 등.
이번 유닛 핵심 정리
1. 명제 = 참/거짓을 확실히 판단할 수 있는 문장
2. 참 = 1 = ON, 거짓 = 0 = OFF
3. 사람마다 다르게 생각할 수 있는 문장은 명제가 아니에요
4. C 언어에서 0만 거짓, 나머지는 전부 참!
5. 비교 연산자(==, !=, >, <, >=, <=)가 참/거짓을 만들어요
6. = 는 대입, == 는 비교 — 초보자 실수 1위!
7. 불리언(Boolean)은 조지 불의 이름에서 유래
8. 트랜지스터의 ON/OFF가 컴퓨터 참/거짓의 물리적 구현
