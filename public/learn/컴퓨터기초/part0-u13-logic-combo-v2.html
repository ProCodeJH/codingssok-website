논리를 섞어볼까?

일상에서도 우리는 복잡한 조건을 자주 말해요. 예를 들어볼까요?

[예시] 일상 속 복합 논리 "비가 오지 않고, (우산이 있거나 택시가 있으면) 나간다"

이 문장을 논리식으로 바꾸면:
• 비가 온다 = A
• 우산이 있다 = B
• 택시가 있다 = C

→ NOT A AND (B OR C)

비가 안 오고, 우산이나 택시 중 하나라도 있으면 나갈 수 있어요!

이처럼 현실의 판단은 하나의 논리 연산만으로는 부족해요. AND, OR, NOT을 조합해야 정확한 판단을 내릴 수 있어요.

복합 논리식 읽는 법

복합 논리식은 안쪽부터 바깥쪽으로 계산해요. 수학에서 괄호 안을 먼저 푸는 것과 같아요.

[핵심] 복합 논리식 계산 순서 1단계: 괄호 안을 먼저 계산한다
2단계: NOT을 계산한다
3단계: AND를 계산한다
4단계: OR을 계산한다

우선순위: 괄호 > NOT > AND > OR

괄호의 마법

괄호 하나가 결과를 완전히 바꿀 수 있어요. 정말 중요하니까 잘 봐요!

[예시] 괄호가 결과를 바꾸는 경우 A = 참, B = 거짓, C = 거짓:
A OR B AND C = 참 OR (거짓 AND 거짓) = 참 OR 거짓 = 참
(A OR B) AND C = (참 OR 거짓) AND 거짓 = 참 AND 거짓 = 거짓!

같은 값인데 괄호 하나로 결과가 뒤집혔어요!

[심화] 불 대수 기본 법칙들

불 대수(Boolean Algebra)는 참/거짓을 다루는 수학 체계예요. 논리식을 간소화할 때 이 법칙들을 사용해요.

법칙 이름AND 버전OR 버전
항등법칙A AND 1 = AA OR 0 = A
영원법칙A AND 0 = 0A OR 1 = 1
멱등법칙A AND A = AA OR A = A
보수법칙A AND NOT A = 0A OR NOT A = 1
이중부정NOT NOT A = ANOT NOT A = A
교환법칙A AND B = B AND AA OR B = B OR A
결합법칙(A AND B) AND C = A AND (B AND C)(A OR B) OR C = A OR (B OR C)
분배법칙A AND (B OR C) = (A AND B) OR (A AND C)A OR (B AND C) = (A OR B) AND (A OR C)
흡수법칙A AND (A OR B) = AA OR (A AND B) = A
드모르간NOT(A AND B) = NOT A OR NOT BNOT(A OR B) = NOT A AND NOT B
[팁] 간소화 예시 문제: A AND B OR A AND NOT B 를 간소화하세요.

풀이:
= A AND (B OR NOT B) — 분배법칙 역적용
= A AND 1 — 보수법칙 (B OR NOT B = 1)
= A — 항등법칙

복잡한 식이 A 하나로 줄었어요!

[심화] XOR — 배타적 논리합

AND, OR, NOT 외에 XOR(Exclusive OR, 배타적 논리합)도 중요해요. '둘 중 하나만 참일 때 참'이에요.

ABA XOR B설명

011하나만 참 → 참
101하나만 참 → 참

[핵심] XOR = (A OR B) AND NOT (A AND B) XOR은 OR처럼 '하나라도 참이면 참'이지만, 둘 다 참이면 거짓이에요!
C에서 비트 XOR: ^ 연산자
논리 XOR은 C에 없어서: (a || b) && !(a && b) 또는 a != b

활용: 두 값이 '다른지' 비교, 암호화, 오류 검출

[심화] 카르노 맵 (K-map)

변수가 3~4개인 논리식을 간소화하려면 불 대수 법칙을 여러 번 적용해야 해서 복잡해요. 카르노 맵은 시각적으로 간단하게 간소화할 수 있는 도구예요.

[핵심] 2변수 카르노 맵 F = A'B + AB + AB' 를 간소화하려면:

| B=0 | B=1 |
A=0 | 0 | 1 |
A=1 | 1 | 1 |

인접한 1을 묶으면:
- 세로 묶음 (A=1 행): A (B 무관)
- 가로 묶음 (B=1 열): B (A 무관)

결과: F = A OR B

복잡한 3항이 2항으로 줄었어요!
[팁] 카르노 맵 규칙 1. 인접한 1들을 2^n개 (1, 2, 4, 8...)씩 직사각형으로 묶기
2. 묶음은 클수록 좋다 (더 많이 간소화)
3. 가장자리는 서로 연결됨 (맵이 도넛 모양으로 이어짐)
4. 하나의 1은 여러 묶음에 포함될 수 있음

[심화 내용] 다수결 회로 (Majority Circuit)

3개 입력 중 2개 이상이 1이면 출력이 1인 회로. 투표 시스템과 같아요!

다수결 함수 F = AB + BC + AC

- 000→0, 001→0, 010→0, 011→1, 100→0, 101→1, 110→1, 111→1

[심화 내용] 멀티플렉서 (MUX)

멀티플렉서(MUX)는 여러 입력 중 하나를 선택하는 회로예요. 선택 신호(select)에 따라 어떤 입력을 출력할지 결정해요.

2-to-1 MUX 입력: A, B / 선택: S / 출력: Y

Y = (NOT S AND A) OR (S AND B)

S=0이면 A를 출력, S=1이면 B를 출력

MUX 하나로 어떤 논리 함수든 구현할 수 있어요 — 만능 회로!

[심화 내용] NAND와 NOR — 만능 게이트

NAND = NOT AND, NOR = NOT OR. 이 두 게이트는 각각 만능 게이트(Universal Gate)예요 — 하나만으로 AND, OR, NOT 전부 구현 가능!

게이트NAND로 구현설명
NOT AA NAND A입력을 같게 넣으면 NOT
A AND B(A NAND B) NAND (A NAND B)NAND의 NOT
A OR B(A NAND A) NAND (B NAND B)입력 각각 NOT 후 NAND
[역사] 왜 NAND가 중요할까? 실제 반도체 칩에서 NAND 게이트가 가장 적은 트랜지스터로 만들어져요 (4개). AND는 6개, OR도 6개 필요해요. 그래서 현대 CPU는 NAND 게이트를 기본 빌딩 블록으로 사용해요!

SSD의 저장 방식도 NAND 플래시라고 불러요 — NAND 게이트 구조를 사용하기 때문이에요.

[심화] 단락 평가 (Short-circuit Evaluation)

C에서 &&와 ||는 단락 평가를 해요. 결과가 확정되면 나머지는 계산하지 않아요!

[핵심] 단락 평가 규칙 A && B — A가 거짓이면 B를 안 봄 (어차피 거짓)
A || B — A가 참이면 B를 안 봄 (어차피 참)

이걸 이용한 패턴:
if (ptr != NULL && ptr->value > 0)
ptr이 NULL이면 뒤쪽 ptr->value는 실행 안 됨 → 안전!

만약 단락 평가가 없으면 NULL 포인터 접근으로 크래시!

용어 정리

[심화 내용] Quine-McCluskey 알고리즘

카르노 맵은 4~5변수까지만 실용적이에요. 변수가 많아지면 Quine-McCluskey 알고리즘을 사용해요.

[심화 내용] Quine-McCluskey 알고리즘 목적: 임의 개수의 변수에 대해 최소 논리식을 기계적으로 구하는 알고리즘

단계:
1. 민텀을 1의 개수로 그룹화
2. 인접 그룹 간 1비트만 다른 쌍 결합 (- 표시)
3. 더 이상 결합 불가능한 항 = 주항(Prime Implicant)
4. 주항 도표(PI Chart)로 필수 주항 선택
5. 필수 주항만으로 모든 민텀 커버되면 완료

예: F(A,B,C) = m(0,2,4,6)
그룹0(0개의 1): 000
그룹1(1개의 1): 010, 100
그룹2(2개의 1): 110

결합: 000+010=0-0, 000+100=-00, 010+110=-10, 100+110=1-0
재결합: 0-0+-10=--0, -00+1-0=--0
결과: --0 = NOT C (카르노 맵과 동일!)

[심화 내용] 단락 평가의 심화

C의 && 와 || 은 단락 평가(Short-circuit evaluation)를 해요. 이것이 버그도 만들고 테크닉도 만들어요.

C[C 코드] 단락 평가 활용과 위험
1#include <stdio.h>
2#include <stdlib.h>
3
4// 활용 1: 안전한 포인터 접근
5int safe_strlen(const char *s) {
6 // s가 NULL이면 오른쪽은 평가 안 함!
7 if (s != NULL && s[0] != '\0') {
8 int len = 0;
9 while (s[len]) len++;
10 return len;
11 }
12 return 0;
13}
14
15// 위험: 부작용이 있는 함수와 단락 평가
16int count = 0;
17int increment() { return ++count; }
18
19int main() {
20 // 안전한 포인터 접근
21 printf("NULL 문자열 길이: %d\n", safe_strlen(NULL));
22 printf("Hello 길이: %d\n", safe_strlen("Hello"));
23
24 // 위험! increment()가 호출 안 될 수 있음
25 int x = 0;
26 if (x && increment()) { // x=0이라 increment() 호출 안 됨!
27 printf("true\n");
28 }
29 printf("count = %d (0이면 increment 미호출!)\n", count);
30
31 // || 도 마찬가지
32 int y = 1;
33 if (y || increment()) { // y=1이라 increment() 호출 안 됨!
34 printf("true\n");
35 }
36 printf("count = %d\n", count);
37
38 return 0;
39}
실행 결과
NULL 문자열 길이: 0 Hello 길이: 5 count = 0 (0이면 increment 미호출!) true count = 0
[심화 내용] 단락 평가 주의사항 문제가 되는 경우:
- 오른쪽 피연산자에 부작용(side effect)이 있을 때
- 함수 호출, 증감 연산(++, --), 대입(=) 등

안전한 규칙:
1. && 와 || 의 피연산자에서는 부작용을 피하세요
2. 부작용이 필요하면 별도 문장으로 분리
3. 의도적 단락 평가는 주석으로 명시

면접 빈출:
Q: if (a++ && b++) 에서 a=0이면 b++은 실행되나요?
A: 아니요! a++이 0(거짓)이므로 b++은 평가되지 않습니다.
a는 1이 되지만, b는 변하지 않아요.

[심화 내용] 논리식 동치 증명 기법

두 논리식이 같은지(동치인지) 증명하는 방법은 여러 가지예요.

[심화 내용] 동치 증명 3가지 방법 방법 1: 진리표 대조
모든 입력 조합에 대해 두 식의 결과가 같으면 동치
n변수면 2^n행 비교 (변수 많으면 비효율적)

방법 2: 대수적 변환
부울 대수 법칙으로 한 식을 다른 식으로 변환
사용하는 법칙: 교환, 결합, 분배, 흡수, 드모르간, 멱등, 보원 등

방법 3: 정규형 비교
두 식을 같은 정규형(SOP 또는 POS)으로 변환 후 비교
같은 민텀/맥스텀 집합이면 동치

예: A AND (A OR B) = A 증명
= A AND A OR A AND B (분배법칙)
= A OR A AND B (멱등법칙: A AND A = A)
= A (흡수법칙: A OR A AND B = A)
[심화 내용] 주요 부울 대수 법칙 총정리 기본 법칙:
멱등: A AND A = A, A OR A = A
보원: A AND NOT A = 0, A OR NOT A = 1
항등: A AND 1 = A, A OR 0 = A
영원: A AND 0 = 0, A OR 1 = 1
이중부정: NOT NOT A = A

결합 법칙:
교환: A op B = B op A
결합: (A op B) op C = A op (B op C)
분배: A AND (B OR C) = (A AND B) OR (A AND C)
A OR (B AND C) = (A OR B) AND (A OR C) [이중 분배!]

단축 법칙:
흡수: A OR (A AND B) = A
A AND (A OR B) = A
합의: AB OR NOT(A)C OR BC = AB OR NOT(A)C

드모르간:
NOT(A AND B) = NOT A OR NOT B
NOT(A OR B) = NOT A AND NOT B
용어설명
복합 논리식AND, OR, NOT 중 두 가지 이상을 섞어서 만든 논리식.
연산자 우선순위여러 연산자가 있을 때 어떤 것을 먼저 계산할지 정한 규칙. NOT > AND > OR 순서.
진리표 (Truth Table)논리식의 모든 입력 조합에 대한 결과를 정리한 표.
드모르간 법칙NOT(A AND B) = NOT A OR NOT B, NOT(A OR B) = NOT A AND NOT B.
불 대수 (Boolean Algebra)참/거짓을 다루는 수학 체계. 조지 불(George Boole)이 창시.
XOR (배타적 논리합)두 입력이 다를 때만 참인 연산. 같으면 거짓.
카르노 맵 (K-map)논리식을 시각적으로 간소화하는 도구. 인접한 1을 묶어서 최소 논리식 도출.
SOP (Sum of Products)민텀의 합. AND 항들을 OR로 연결한 정규형.
POS (Product of Sums)맥스텀의 곱. OR 항들을 AND로 연결한 정규형.
NAND 게이트NOT AND. 만능 게이트 — 이것만으로 모든 논리 회로 구현 가능.
MUX (멀티플렉서)선택 신호에 따라 여러 입력 중 하나를 출력하는 회로.
단락 평가논리 연산에서 결과가 확정되면 나머지를 평가하지 않는 것.
Quine-McCluskey임의 변수 개수의 논리식을 기계적으로 최소화하는 알고리즘. 카르노 맵의 자동화 버전.
주항(Prime Implicant)더 이상 결합할 수 없는 최소 항. 논리식 간소화의 후보.
부작용(Side Effect)함수나 연산이 값을 반환하는 것 외에 상태를 변경하는 것. 단락 평가 시 주의 필요.
동치(Equivalence)두 논리식이 모든 입력에 대해 같은 결과를 내는 관계.
흡수법칙A OR (A AND B) = A. 불필요한 항을 흡수하여 간소화.
이중 분배법칙A OR (B AND C) = (A OR B) AND (A OR C). 일반 분배법칙의 OR 버전.
이번 유닛 핵심 정리
1. AND, OR, NOT을 섞어서 복합 논리식을 만들 수 있어요
2. 연산자 우선순위: NOT > AND > OR (괄호가 최우선!)
3. 괄호 위치에 따라 결과가 완전히 달라질 수 있어요
4. C 언어에서는 !, &&, ||로 복합 조건문을 만들어요
5. [심화] 드모르간 법칙: NOT을 안으로 넣으면 AND↔OR 교환
6. [심화] XOR = 둘 중 하나만 참일 때 참
7. [심화] 카르노 맵으로 논리식 간소화
