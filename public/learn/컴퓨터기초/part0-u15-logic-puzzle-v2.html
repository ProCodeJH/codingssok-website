UNIT 15 논리 퍼즐

논리 퍼즐이란?

논리 퍼즐은 탐정이 범인을 찾는 것과 비슷해요. 단서(조건)를 하나씩 따져가면서 확실한 결론을 이끌어내요.

거짓말쟁이 문제

거짓말쟁이 문제는 논리 퍼즐의 대표 문제예요. 규칙은 간단해요.

[핵심] 거짓말쟁이 규칙 • 진실쟁이: 항상 참만 말해요. 절대 거짓말 안 해요.
• 거짓말쟁이: 항상 거짓만 말해요. 절대 참말 안 해요.

핵심: 누가 진실쟁이고 누가 거짓말쟁이인지 추론하는 거예요!
[예시] 기본 거짓말쟁이 문제 A와 B가 있어요. 한 명은 진실쟁이, 한 명은 거짓말쟁이예요.

A가 말했어요: "나는 거짓말쟁이야."

풀이:
만약 A가 진실쟁이라면? → 자기가 거짓말쟁이라고 말하면 거짓말이 되니까 모순!
만약 A가 거짓말쟁이라면? → 자기가 거짓말쟁이라고 말하면 참말이 되니까 모순!

이 문제는 사실 풀 수 없어요! 이런 걸 역설(패러독스)이라고 해요.
[예시] 풀 수 있는 거짓말쟁이 문제 A와 B가 있어요. 한 명은 진실쟁이, 한 명은 거짓말쟁이예요.

A가 말했어요: "B는 거짓말쟁이야."
B가 말했어요: "A는 거짓말쟁이야."

풀이:
두 사람이 서로 "상대가 거짓말쟁이"라고 하네요.

• A가 진실쟁이라면 → A의 말이 참 → B는 거짓말쟁이 ✓ (모순 없음)
• A가 거짓말쟁이라면 → A의 말이 거짓 → B는 진실쟁이
→ B의 말이 참 → A는 거짓말쟁이 ✓ (이것도 모순 없음)

두 경우 다 가능해요! 다른 조건이 더 필요해요.
[예시] 3명 거짓말쟁이 (풀 수 있는 문제!) A, B, C 세 명 중 한 명만 거짓말쟁이예요.

A: "B는 진실쟁이야."
B: "C는 거짓말쟁이야."
C: "A는 진실쟁이야."

풀이 — 경우를 나눠보자:

① A가 거짓말쟁이라면:
A의 말 거짓 → B는 진실쟁이가 아님 → B도 거짓말쟁이?
→ 거짓말쟁이가 2명이 되니까 조건 위반!

② B가 거짓말쟁이라면:
A의 말 참 → B는 진실쟁이? → B가 거짓말쟁이라고 했는데 모순!
잠깐, A가 "B는 진실쟁이"라고 했는데 B가 거짓말쟁이면 A의 말은 거짓
→ A도 거짓말쟁이? → 2명 위반

③ C가 거짓말쟁이라면:
A의 말 참 → B는 진실쟁이 ✓
B의 말 참 → C는 거짓말쟁이 ✓
C의 말 거짓 → A는 진실쟁이가 아님? → A도 거짓말쟁이? → 2명?
잠깐, C가 거짓이면 "A는 진실쟁이"가 거짓이니까 A도 거짓말쟁이...
→ 2명 위반

다시 ②를 보자: B가 거짓말쟁이면,
A: "B는 진실쟁이" → 거짓 → A도 거짓말쟁이 → 2명

③ 다시: C가 거짓말쟁이면,
C: "A는 진실쟁이" → 거짓 → A는 거짓말쟁이 → 2명

① 다시: A가 거짓말쟁이면,
A: "B는 진실쟁이" → 거짓 → B는 거짓말쟁이 → 2명

️ 이 문제의 조건으로는 답이 없네요! 조건을 바꿔볼게요...
[예시] 수정된 3명 문제 A, B, C 세 명 중 한 명만 거짓말쟁이예요.

A: "나는 진실쟁이야."
B: "A의 말이 맞아."
C: "B는 거짓말쟁이야."

풀이:
① A가 거짓말쟁이라면:
A: "나는 진실쟁이" → 거짓 (거짓말쟁이니까 OK ✓)
B: "A의 말이 맞아" → B가 진실쟁이면 A의 말이 맞아야 하는데, A는 거짓말 → 모순!
→ B도 거짓말쟁이 → 2명

② B가 거짓말쟁이라면:
B: "A의 말이 맞아" → 거짓 → A의 말은 틀림 → A도 거짓말쟁이? → 2명

③ C가 거짓말쟁이라면:
A: "나는 진실쟁이" → 참 ✓
B: "A의 말이 맞아" → 참 ✓
C: "B는 거짓말쟁이" → 거짓 (C가 거짓말쟁이니까 OK ✓)
→ 거짓말쟁이 1명만! 정답: C가 거짓말쟁이!

조건 추론: "A이면 B이다"

"A이면 B이다"는 논리에서 아주 중요한 형태예요. 이것으로 새로운 결론을 이끌어낼 수 있어요.

[핵심] 조건문과 변환 원래 조건: "비가 오면 우산을 쓴다" (A → B)

역: "우산을 쓰면 비가 온다" (B → A) — 반드시 참은 아님!
이: "비가 오지 않으면 우산을 쓰지 않는다" (NOT A → NOT B) — 반드시 참은 아님!
대우: "우산을 쓰지 않으면 비가 오지 않는다" (NOT B → NOT A) — 항상 참!

원래 조건이 참이면 대우도 반드시 참이에요!
[예시] 대우 활용 조건: "이 동물이 개면, 다리가 4개다" (참)

대우: "다리가 4개가 아니면, 이 동물은 개가 아니다" → 참!
역: "다리가 4개면, 이 동물은 개다" → 반드시 참은 아님! (고양이도 4개)
이: "이 동물이 개가 아니면, 다리가 4개가 아니다" → 반드시 참은 아님!

스도쿠의 논리 원리

스도쿠도 논리 퍼즐이에요! 스도쿠에서 쓰는 논리를 알아볼까요?

[예시] 스도쿠 속 논리 스도쿠 규칙: 한 줄에 1~9가 한 번씩만 나와야 해요.

한 줄에 [1, _, 3, _, 5, 6, 7, 8, 9]가 있으면:
• 빈칸에 올 수 있는 수: 2, 4
• 세로줄과 3×3 칸도 확인해서 하나로 좁혀요

이것은 소거법이에요:
"이 칸에 1이 올 수 없고, 3도 올 수 없고, ... 그러면 2만 가능!"

→ NOT(1) AND NOT(3) AND ... → 2만 남음!

[심화] 귀류법 — 모순으로 증명하기

귀류법(Proof by Contradiction)은 결론의 부정을 가정하고, 모순을 이끌어내서 원래 결론이 참임을 보이는 증명법이에요.

[심화] 귀류법의 원리 증명하고 싶은 것: P가 참이다

귀류법 단계:
1. "P가 거짓이다"라고 가정 (NOT P)
2. 논리적으로 추론해 나감
3. 모순(A AND NOT A)이 발생!
4. 가정(NOT P)이 잘못됨 -> P가 참!

핵심: 거짓말쟁이 문제 풀이도 귀류법!
"A가 진실쟁이라고 가정" -> 모순 -> "A는 거짓말쟁이"
[심화] 귀류법 예제: 소수가 무한함을 증명 증명: 소수는 무한히 많다.

1. (귀류법) 소수가 유한하다고 가정. 모든 소수: p1, p2, ..., pn
2. N = p1 * p2 * ... * pn + 1 을 만들자
3. N을 어떤 소수 pi로 나누면 나머지가 항상 1
4. N은 기존 소수 어느 것으로도 나누어지지 않음!
5. N 자체가 소수이거나, 목록에 없는 새 소수로 나누어짐
6. "모든 소수를 나열했다"에 모순!
7. 따라서 소수는 무한하다. (유클리드, 기원전 300년)

[심화] 논리 퍼즐을 코드로 풀기

C[C 코드] 브루트포스로 거짓말쟁이 문제 풀기

2
3// 3명 중 거짓말쟁이 찾기
4// A: "B가 거짓말쟁이야"
5// B: "C가 거짓말쟁이야"
6// C: "A와 B 둘 다 거짓말쟁이야"
7// 진실쟁이는 항상 참, 거짓말쟁이는 항상 거짓
8
9int main() {
10 // 0 = 진실쟁이, 1 = 거짓말쟁이
11 for (int a = 0; a <= 1; a++) {
12 for (int b = 0; b <= 1; b++) {
13 for (int c = 0; c <= 1; c++) {
14 // A의 발언: "B가 거짓말쟁이"
15 int a_says = (b == 1); // B가 거짓말쟁이면 참
16 int a_ok = (a == 0) ? a_says : !a_says;
17
18 // B의 발언: "C가 거짓말쟁이"
19 int b_says = (c == 1);
20 int b_ok = (b == 0) ? b_says : !b_says;
21
22 // C의 발언: "A와 B 둘 다 거짓말쟁이"
23 int c_says = (a == 1 && b == 1);
24 int c_ok = (c == 0) ? c_says : !c_says;
25
26 if (a_ok && b_ok && c_ok) {
27 printf("A=%s, B=%s, C=%s\n",
28 a ? "거짓말쟁이" : "진실쟁이",
29 b ? "거짓말쟁이" : "진실쟁이",
30 c ? "거짓말쟁이" : "진실쟁이");
31 }
32 }
33 }
34 }
35 return 0;
36}

[심화] 스도쿠를 코드로 — 백트래킹

스도쿠도 논리 퍼즐이에요. 컴퓨터로 풀 때는 백트래킹(Backtracking) 기법을 사용해요.

[심화] 백트래킹 알고리즘 백트래킹 = 시도 + 실패하면 되돌리기

스도쿠 풀이 알고리즘:
1. 빈 칸을 찾는다
2. 1~9를 하나씩 넣어본다
3. 행/열/3x3 박스에서 충돌 검사
4. 충돌 없으면 -> 다음 빈 칸으로 진행 (재귀)
5. 충돌 있으면 -> 다른 숫자 시도
6. 1~9 다 실패 -> 이전 칸으로 되돌아감 (백트래킹!)
7. 모든 칸이 채워지면 완성!

스도쿠의 모든 빈 칸 = 변수
1~9 = 각 변수의 가능한 값
행/열/박스 규칙 = 제약 조건

이것이 CSP(제약 충족 문제)의 전형적 예시!

[심화 내용] 명제 논리 vs 술어 논리

지금까지 배운 건 명제 논리(Propositional Logic)예요. 더 강력한 술어 논리(Predicate Logic)도 있어요.

[심화 내용] 명제 논리 vs 술어 논리 명제 논리 (Propositional Logic):
- 기본 단위: 명제 (참/거짓인 문장)
- 연산: AND, OR, NOT, 조건(->), 쌍조건(<->)
- 예: "비가 온다 AND 우산이 있다"
- 한계: "모든 x에 대해..."를 표현 못 함

술어 논리 (Predicate Logic, 1차 논리):
- 기본 단위: 술어 P(x) (변수에 따라 참/거짓)
- 추가 연산: 전칭 양화사(모든), 존재 양화사(어떤)
- 예: "모든 x에 대해, x가 사람이면 x는 죽는다"
- 기호: (ALL x)(Person(x) -> Mortal(x))

차이:
명제 논리: "소크라테스는 사람이다" (고정 명제)
술어 논리: "모든 사람은 죽는다" (변수 + 양화사)

프로그래밍 연결:
명제 논리 = if문의 조건
술어 논리 = for문 + if문 (배열의 모든 원소에 대해...)
[심화 내용] 술어 논리 예제 "모든 짝수는 2로 나누어진다"
ALL x: Even(x) -> Divisible(x, 2)

"어떤 소수는 짝수이다"
EXISTS x: Prime(x) AND Even(x) (x=2!)

"모든 사람에게 엄마가 있다"
ALL x: Person(x) -> EXISTS y: Mother(y, x)

C 코드로 표현:
// "모든 원소가 0보다 크다"
// ALL x in arr: x > 0
int all_positive = 1;
for (int i = 0; i < n; i++) {
if (arr[i] <= 0) { all_positive = 0; break; }
}

// "어떤 원소가 0이다"
// EXISTS x in arr: x == 0
int has_zero = 0;
for (int i = 0; i < n; i++) {
if (arr[i] == 0) { has_zero = 1; break; }
}

[심화 내용] SAT 문제 — 컴퓨터 과학 최대의 미해결 문제

논리 퍼즐을 일반화하면 SAT(Satisfiability) 문제가 돼요. 이것이 P vs NP 문제와 연결돼요!

[심화 내용] SAT 문제란? SAT (Boolean Satisfiability Problem):
"주어진 논리식을 참으로 만드는 변수 배정이 존재하는가?"

예: (A OR NOT B) AND (NOT A OR C) AND (B OR NOT C)
이 식을 참으로 만드는 A, B, C 값이 있는가?

왜 중요한가:
1. 최초의 NP-완전 문제 (Cook, 1971)
2. 모든 NP 문제를 SAT로 변환 가능
3. SAT를 다항 시간에 풀 수 있으면 P=NP 증명!
4. 풀 수 없으면... 아직 아무도 증명 못 함

실용적 중요성:
- 칩 검증 (Intel, AMD가 실제 사용)
- 소프트웨어 검증
- 스케줄링, 자원 배분
- 암호학

현대 SAT 솔버: 수백만 변수의 문제도 풀 수 있음!
[심화 내용] 거짓말쟁이 문제 = SAT 문제 거짓말쟁이 문제를 SAT로 변환:

변수: a (A가 진실쟁이), b (B가 진실쟁이), c (C가 진실쟁이)

A의 발언 "B는 거짓말쟁이": (a -> NOT b) AND (NOT a -> b)
= a XOR b의 NOT = a XNOR b의 NOT... 아, 복잡하네

더 간단하게:
A가 진실쟁이이고 "B는 거짓말쟁이" 했으면 → b는 거짓
A가 거짓말쟁이이고 "B는 거짓말쟁이" 했으면 → b는 참

즉: (a AND NOT b) OR (NOT a AND b) = a XOR b

이렇게 모든 조건을 논리식으로 변환 후
SAT 솔버에 넣으면 자동으로 답을 찾아줘요!

[심화 내용] 결의 원리 (Resolution)

자동 정리 증명기의 핵심 기법인 결의 원리를 알아봐요.

[심화 내용] 결의 원리 결의 원리: 두 절(clause)에서 상보적 리터럴을 제거하고 합치는 규칙

(A OR B) 와 (NOT A OR C) 에서
A와 NOT A를 제거하면 → (B OR C)

이것만으로 모든 논리 증명 가능! (반박 증명법)

반박 증명 단계:
1. 증명할 것의 부정을 가정
2. 모든 식을 CNF(절들의 AND)로 변환
3. 결의 규칙 반복 적용
4. 빈 절(모순)이 나오면 → 원래 가정의 부정이 모순 → 증명 완료!

Prolog 언어가 이 원리로 동작해요!

[심화 내용] CSP와 AC-3 알고리즘

스도쿠, 지도 색칠, 시간표 작성 등은 모두 CSP(Constraint Satisfaction Problem)예요.

[심화 내용] CSP 개념 CSP의 3요소:
1. 변수(Variables): 값을 정해야 하는 것들
스도쿠: 각 빈 칸
2. 도메인(Domain): 각 변수가 가질 수 있는 값
스도쿠: {1, 2, 3, 4, 5, 6, 7, 8, 9}
3. 제약(Constraints): 변수 간의 관계
스도쿠: 같은 행/열/박스에 같은 수 금지

풀이 기법:
- 백트래킹: 기본 (위에서 배움)
- AC-3 (Arc Consistency): 제약 전파로 도메인 줄이기
- MRV (Minimum Remaining Values): 도메인이 작은 변수부터
- Forward Checking: 값 배정 시 영향받는 변수 즉시 확인

AI의 핵심 기법 중 하나!
[심화 내용] 지도 색칠 문제 문제: 한국 지도를 3색으로 칠하기. 인접한 도는 다른 색!

변수: 서울, 경기, 강원, 충북, 충남, ...
도메인: {빨강, 파랑, 초록}
제약: 인접한 도 != 같은 색

이것이 바로 그래프 색칠 문제(Graph Coloring)!

[심화] 논리 퍼즐 추가 문제

[심화] 추가 퍼즐: 모자 문제 문제:
A, B, C 세 사람이 일렬로 서 있어요 (A가 맨 뒤, C가 맨 앞).
5개의 모자(빨강3, 파랑2)에서 각자 하나를 씌웠어요.
A는 B와 C의 모자가 보이고, B는 C의 모자만 보여요.
C는 아무것도 안 보여요.

"당신 모자 색을 아시나요?"
A: "모르겠어요"
B: "모르겠어요"
C: "알겠어요! 제 모자는 빨강이에요!"

왜 C가 알 수 있을까?

A가 모름 → B와 C가 둘 다 파랑은 아님 (파랑 2개뿐이니까)
B가 모름 → C가 파랑이면 B는 자기가 빨강임을 알아야 함
(A의 정보 + C가 파랑이면 B는 파랑일 수 없으므로)
B가 모른다는 건 C가 파랑이 아니라는 뜻!
따라서 C = 빨강!
용어설명
논리 퍼즐주어진 조건만으로 정답을 추론하는 문제. 수학 공식 없이 논리만 사용해요.
진실쟁이항상 참만 말하는 사람. 절대 거짓말을 안 해요.
거짓말쟁이항상 거짓만 말하는 사람. 절대 참말을 안 해요.
대우"A이면 B이다"의 대우는 "B가 아니면 A가 아니다". 원래 조건이 참이면 대우도 참.
역"A이면 B이다"의 역은 "B이면 A이다". 원래가 참이어도 역은 거짓일 수 있어요.
소거법불가능한 것을 하나씩 지워나가서 답을 찾는 방법. 스도쿠에서 많이 사용해요.
삼단논법A→B이고 B→C이면 A→C. 조건을 연결해서 새로운 결론을 이끌어내는 추론법.
귀류법결론의 부정을 가정하고 모순을 이끌어내어 원래 결론이 참임을 증명하는 방법.
브루트포스모든 가능한 경우를 다 시도하는 방법. 완전 탐색이라고도 해요.
백트래킹시도하다 실패하면 이전 단계로 되돌아가는 알고리즘 기법.
명제 논리참/거짓인 명제를 AND, OR, NOT으로 조합하는 논리 체계.
술어 논리변수와 양화사(모든, 어떤)를 사용하는 더 강력한 논리 체계.
SAT 문제논리식을 참으로 만드는 변수 배정이 존재하는지 묻는 문제. 최초의 NP-완전 문제.
NP-완전다항 시간에 풀 수 있는지 알려지지 않았지만, 답을 검증하는 건 다항 시간에 가능한 문제.
결의 원리두 절에서 상보적 리터럴을 제거하고 합치는 추론 규칙. 자동 정리 증명의 핵심.
