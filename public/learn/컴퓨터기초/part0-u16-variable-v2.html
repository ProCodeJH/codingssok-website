수학 시간의 x

수학에서 x + 3 = 7이라고 쓰면, x가 뭔지 찾아야 해요. x는 4죠? 여기서 x는 '아직 모르는 수', 즉 미지수예요. 등호(=)는 '왼쪽과 오른쪽이 같다'는 뜻이에요.

[핵심] 수학의 변수 수학에서 변수 = 미지수 (모르는 수를 찾는 것)
수학에서 = 기호 = '같다' (왼쪽과 오른쪽이 같은 값)

코딩 세계의 x

코딩에서 x = 7이라고 쓰면, 뜻이 완전히 달라요. 'x라는 이름의 상자에 7을 넣어라'라는 뜻이에요. x라는 이름표가 붙은 상자를 하나 꺼내서, 그 안에 7이라는 숫자를 넣는 거예요.

[핵심] 코딩의 변수 코딩에서 변수 = 이름표가 붙은 저장 상자 (값을 넣어두는 곳)
코딩에서 = 기호 = '넣어라' (오른쪽 값을 왼쪽 상자에 넣어라)

C언어에서 변수 사용하기

C언어에서 변수를 쓰려면 먼저 선언해야 해요. '이런 이름의 상자를 만들겠다'라고 컴퓨터에게 알려주는 거예요.

1#include <stdio.h>
2
3int main(void) {

6 printf("x = %d\n", x); // x = 7
7
8 x = x + 1; // x를 꺼내서 1 더하고 다시 넣어라

10
11 x++; // x = x + 1의 줄임말!
12 printf("x = %d\n", x); // x = 9
13
14 // 선언과 동시에 초기화도 가능

16 printf("y = %d\n", y); // y = 100
17

19}
실행 결과
x = 7 x = 8 x = 9 y = 100

불가능: 숫자로 시작, 공백, 특수문자

[심화] 변수의 정체 — 메모리 주소

변수라는 '상자'는 사실 컴퓨터 메모리(RAM)의 특정 위치예요. 모든 메모리 위치에는 주소(address)가 있어요.

Caddress_demo.c
1#include <stdio.h>
2
3int main(void) {
4 int x = 42;
5
6 printf("x의 값: %d\n", x); // 42
7 printf("x의 주소: %p\n", &x); // 0x7ffd1234abcd (매번 다름)
8 printf("x의 크기: %zu바이트\n", sizeof(x)); // 4
9
10 // & = 주소 연산자 ("어디에 있니?")
11 // sizeof = 크기 연산자 ("얼마나 크니?")
12
13 return 0;
14}
실행 결과
x의 값: 42 x의 주소: 0x7ffd1234abcd x의 크기: 4바이트
[심화] 변수 = 이름 + 주소 + 크기 + 값 변수는 4가지 속성이 있어요:

1. 이름: 프로그래머가 부르는 이름 (예: score)
2. 주소: 메모리에서의 위치 (예: 0x7ffd...)
3. 크기: 차지하는 바이트 수 (int = 4바이트)
4. 값: 현재 저장된 데이터 (예: 42)

컴파일러는 이름을 주소로 바꿔요. 실행 파일에는 변수 이름이 없어요!

[심화] 초기화하지 않은 변수의 위험

1#include <stdio.h>
2
3int main(void) {

[심화] 변수의 스코프 (범위)

변수가 살아있는 범위를 스코프(scope)라고 해요. 중괄호 {} 안에서 선언한 변수는 그 안에서만 사용 가능!

Cscope_demo.c
1#include <stdio.h>
2
3int g = 100; // 전역 변수: 프로그램 전체에서 사용 가능
4
5int main(void) {
6 int a = 10; // 지역 변수: main 함수 안에서만
7
8 {
9 int b = 20; // 블록 변수: 이 중괄호 안에서만
10 printf("a=%d, b=%d, g=%d\n", a, b, g); // OK
11 }
12
13 // printf("%d", b); // ❌ 에러! b는 위 블록에서만 존재
14 printf("a=%d, g=%d\n", a, g); // OK
15
16 return 0;
17}

[심화] const — 변하지 않는 변수

값이 절대 바뀌면 안 되는 변수는 const를 붙여요. 실수로 바꾸려 하면 컴파일러가 에러를 내요.

Cconst_demo.c
1#include <stdio.h>
2
3int main(void) {
4 const int MAX_SCORE = 100; // 상수: 바꿀 수 없음
5 const double PI = 3.14159265358979;
6
7 printf("최고 점수: %d\n", MAX_SCORE);
8 printf("원주율: %.2f\n", PI);
9
10 // MAX_SCORE = 200; // ❌ 컴파일 에러! const는 못 바꿔요
11
12 return 0;
13}
[팁] #define vs const #define MAX 100 — 전처리기 매크로 (타입 없음, 디버깅 어려움)
const int MAX = 100; — 상수 변수 (타입 있음, 디버깅 쉬움)

현대 C에서는 const를 추천해요!

[심화 내용] x86-64 주요 레지스터 범용 레지스터 (64비트): RAX, RBX, RCX, RDX, RSI, RDI, R8~R15
스택 포인터: RSP (스택 꼭대기 주소)

컴파일러가 int x = 42;를 mov eax, 42로 변환해요!

[심화 내용] 어셈블리로 보는 변수

Cvar_asm.asm
1; int x = 42; x = x + 1;
2; x86-64 어셈블리
3
4 mov DWORD PTR [rbp-4], 42 ; x = 42 (스택에 저장)
5 mov eax, DWORD PTR [rbp-4] ; eax = x(메모리 → 레지스터)
6 add eax, 1 ; eax = eax + 1
7 mov DWORD PTR [rbp-4], eax ; x = eax(레지스터 → 메모리)
8
9; 최적화하면:
10 mov DWORD PTR [rbp-4], 43 ; 컴파일러가 미리 계산!
[팁] 컴파일러 최적화 컴파일러는 x = 42; x = x + 1;을 보면 x = 43으로 미리 계산해요!
이걸 상수 접기(constant folding)라고 해요.

활동: 수학 등호 vs 코딩 등호 비교표

[활동] 직접 분류해 보기 아래 식을 '수학 의미'와 '코딩 의미'로 구분해서 적어 보세요.

1. x = 5 → 수학: ( ) / 코딩: ( )
2. x + 2 = 7 → 수학: ( ) / 코딩: ( )
3. y = x + 3 → 수학: ( ) / 코딩: ( )
4. x = x + 1 → 수학: ( ) / 코딩: ( )
5. 5 = x → 수학: ( ) / 코딩: ( )
단계명령abc
1int a = 3;3--
2int b = a + 2;35-
3int c = a + b;358
4a = c - b;358
5a = c - b; (결과)358

4단계에서 a = c - b = 8 - 5 = 3이므로 a는 다시 3이 돼요. 대입문은 오른쪽을 먼저 계산하고, 결과를 왼쪽 변수에 넣는다는 점을 기억하세요.

단계명령abtemp
1int a = 10;10--
2int b = 20;1020-
3int temp = a;102010
4a = b;202010
5b = temp;201010

두 변수의 값을 바꾸려면 임시 변수(temp)가 필요해요. temp 없이 a = b; b = a; 하면 원래 a의 값이 사라져요.

단계명령x의 값설명
1int x = 100;100초기값 설정
2x += 20;120x = x + 20
3x -= 50;70x = x - 50
4x *= 2;140x = x * 2
5x /= 7;20x = x / 7
6x %= 3;2x = x % 3 (나머지)

+=, -=, *=, /=, %= 는 모두 '꺼내서 연산하고 다시 넣어라'의 줄임말이에요. x += 20은 x = x + 20과 같아요.

단계명령ab설명
1int a = 5;5-a 선언
2int b = a;55a의 값이 b에 복사됨
3a = 99;995a를 바꿔도 b는 그대로
4b = b + 1;996b를 바꿔도 a는 그대로

C에서 int b = a;는 a의 값을 복사하는 거예요. 이후 a와 b는 완전히 독립적이에요. 하나를 바꿔도 다른 하나는 영향을 받지 않아요.

단계명령xyz
1int x = 2, y = 3, z = 0;230
2z = x + y;235
3x = z * y;1535
4y = x - z;15105
5z = x / y;15101

5단계에서 z = 15 / 10은 정수 나눗셈이므로 소수점 아래를 버려서 1이 돼요. C의 정수 나눗셈은 항상 소수점을 버린다는 점을 기억하세요.

[심화 내용] 컴파일러가 int x = 42; x = x + 1;을 최적화하면 어셈블리가 어떻게 되나요?

용어 정리

[심화 내용] 변수 바인딩 — 이름과 값의 연결

프로그래밍 언어 이론에서 변수는 이름(Name)과 속성(Attribute)의 바인딩(Binding)이에요.

[심화 내용] 변수 바인딩의 6가지 속성 변수 = 다음 6가지의 바인딩:

1. 이름(Name): 식별자 (x, count, total)
2. 타입(Type): int, float, char 등
3. 주소(Address): 메모리 위치 (l-value)
4. 값(Value): 저장된 데이터 (r-value)
5. 수명(Lifetime): 메모리가 할당/해제되는 시점
6. 스코프(Scope): 이름이 유효한 코드 범위

바인딩 시점:
- 컴파일 타임: C의 int (타입), static 변수 (주소)
- 링크 타임: extern 변수 (주소)
- 로드 타임: 전역 변수 (주소)
- 런타임: 지역 변수 (주소, 값)
[심화 내용] l-value vs r-value l-value = 주소가 있는 것 (대입의 왼쪽에 올 수 있는 것)
r-value = 값 (대입의 오른쪽에 올 수 있는 것)

int x = 5;
x = x + 1;

왼쪽 x = l-value (x의 주소, "어디에 넣을까?")
오른쪽 x = r-value (x의 값 5, "뭘 읽을까?")
x + 1 = r-value (계산 결과 6)

5 = x; // 에러! 5는 l-value가 아님 (주소 없음)
x + 1 = 10; // 에러! 수식은 l-value가 아님

[심화 내용] SSA — 컴파일러의 변수 관리

현대 컴파일러는 내부적으로 변수를 SSA(Static Single Assignment) 형태로 변환해요.

[심화 내용] SSA 형태 SSA 규칙: 모든 변수는 딱 한 번만 대입된다

원래 코드:
x = 1;
x = x + 2;
x = x * 3;

SSA 변환:
x1 = 1;
x2 = x1 + 2; // x1을 읽어서 x2에 저장
x3 = x2 * 3; // x2를 읽어서 x3에 저장

왜 SSA?
- 각 "버전"이 독립적 -> 최적화가 쉬움
- 불필요한 변수 제거, 상수 전파, 루프 최적화
- LLVM, GCC 등 모든 현대 컴파일러가 사용

x = x + 1이 수학적으로 불가능해 보이지만,
SSA에서는 x2 = x1 + 1이라 자연스러워요!

[심화 내용] Rust의 소유권 시스템 Rust는 제3의 방법: 소유권(Ownership)

- 각 값은 딱 하나의 소유자(변수)만 가짐
- 소유자가 스코프를 벗어나면 자동 해제
- GC 없이도 메모리 안전!
- 컴파일 타임에 메모리 오류 검출

C의 성능 + GC의 안전성 = Rust의 소유권

[심화 내용] 변수와 타입 시스템의 역사

[심화 내용] 변수의 진화 1940s — 기계어: 메모리 주소를 직접 사용 (변수 이름 없음)
MOV [0x1000], 42 (주소 0x1000에 42 저장)

1950s — FORTRAN: 최초의 변수 이름 도입
X = 42 (컴파일러가 주소를 자동 배정)

1970s — C: 타입 시스템 + 포인터
int x = 42; (타입 + 이름 + 값)

1990s — Java: 가비지 컬렉션 + 참조 타입
Integer x = 42; (자동 메모리 관리)

2010s — Rust: 소유권 시스템
let x: i32 = 42; (컴파일 타임 메모리 안전)

2020s — AI 코드 생성: 변수 이름도 AI가 추천!
용어설명
변수(Variable)값을 저장할 수 있는 이름 붙은 상자. 수학에서는 미지수, 코딩에서는 저장 공간.
대입(Assignment)변수에 값을 넣는 것. C에서 = 기호가 하는 일.
미지수수학에서 아직 모르는 수. 방정식을 풀어서 값을 찾아요.
선언(Declaration)변수를 처음 만드는 것. int x; 처럼 타입과 이름을 정해요.
초기화(Initialization)변수를 만들면서 동시에 값을 넣는 것. int x = 0;
스코프(Scope)변수가 사용 가능한 범위. 중괄호 {} 안에서 선언하면 그 안에서만 사용 가능.
전역 변수함수 바깥에 선언. 프로그램 전체에서 사용 가능. 자동 0 초기화.
지역 변수함수/블록 안에 선언. 그 안에서만 사용 가능. 초기화 안 하면 쓰레기 값.
const값을 바꿀 수 없는 상수 변수. 실수로 수정하면 컴파일 에러.
레지스터CPU 내부의 초고속 저장소. RAM보다 약 300배 빠름.
메모리 주소RAM에서 변수가 위치한 곳의 번호. & 연산자로 확인.
++ 연산자x = x + 1의 줄임말. x의 값을 1 증가시켜요.
== (비교 연산자)'같은가?'를 확인하는 기호. = (대입)과 헷갈리면 안 돼요!
바인딩이름과 속성(타입, 주소, 값 등)의 연결. 컴파일/런타임에 결정.
l-value주소가 있는 표현식. 대입문의 왼쪽에 올 수 있는 것.
r-value값을 나타내는 표현식. 대입문의 오른쪽에 올 수 있는 것.
SSAStatic Single Assignment. 각 변수가 딱 한 번만 대입되는 중간 표현. 컴파일러 최적화의 핵심.
가비지 컬렉션(GC)사용하지 않는 메모리를 자동으로 해제하는 시스템. Java, Python 등에서 사용.
소유권(Ownership)Rust의 메모리 관리 방식. 각 값에 하나의 소유자만 허용.
메모리 누수할당한 메모리를 해제하지 않아 계속 쌓이는 버그. C에서 free 누락 시 발생.
