1차원: 수직선과 배열

수직선은 숫자를 한 줄로 나열한 것이에요. 코딩에서도 값을 한 줄로 나열하는 배열이 있어요. 배열에서 각 칸의 번호를 인덱스(index)라고 해요.

[핵심] 수직선 = 배열 수직선: ... -2, -1, 0, 1, 2, 3, 4, ...
배열: arr[0], arr[1], arr[2], arr[3], arr[4], ...

둘 다 '위치(번호)로 값을 찾는' 구조예요!
[주의] 인덱스는 0부터! 수학에서는 1번째, 2번째, 3번째... 라고 세지만,
C언어 배열은 0번째, 1번째, 2번째... 로 세요!

첫 번째 칸 = arr[0] (0번 인덱스!)
이걸 헷갈리면 off-by-one 오류라는 유명한 버그가 나요.

2차원: (x, y) 좌표

수직선은 1차원이에요. 왼쪽-오른쪽만 있죠. 하지만 현실 세계는 2차원이에요. 왼쪽-오른쪽 + 위-아래! 이걸 표현하려면 숫자 2개가 필요해요: (x, y) 좌표.

[예제] 좌표로 위치 표현하기 교실의 좌석을 좌표로 나타내면:
• 창가 앞자리: (1, 1)
• 창가 뒷자리: (1, 5)
• 복도 앞자리: (6, 1)

체스판에서도 좌표를 써요: a1, b2, c3...
이것을 숫자로 바꾸면 (1,1), (2,2), (3,3)!

거리 구하기

[핵심] 두 가지 거리 점 A(x₁, y₁)와 점 B(x₂, y₂) 사이:

맨해튼 거리 = |x₁-x₂| + |y₁-y₂|
→ 격자를 따라 가는 거리 (도시 블록)
→ (1,2)↔(4,6): |1-4|+|2-6| = 3+4 = 7

유클리드 거리 = √((x₁-x₂)²+(y₁-y₂)²)
→ 직선 거리 (새가 날아가듯)
→ (1,2)↔(4,6): √(9+16) = √25 = 5
Cdistance.c
1#include <stdio.h>
2#include <math.h> // sqrt(), abs()
3#include <stdlib.h> // abs()
4
5int main(void) {
6 int x1=1, y1=2, x2=4, y2=6;
7
8 // 맨해튼 거리
9 int manhattan = abs(x1-x2) + abs(y1-y2);
10 printf("맨해튼: %d\n", manhattan); // 7
11
12 // 유클리드 거리
13 double euclidean = sqrt(pow(x1-x2, 2) + pow(y1-y2, 2));
14 printf("유클리드: %.2f\n", euclidean); // 5.00
15
16 return 0;
17}

y 차이|2-6| = 4(2-6)² = 16
합산3 + 4 = 79 + 16 = 25
최종 결과7√25 = 5

[심화] 미로 탈출 — BFS 맛보기

Cmaze_bfs.c
1#include <stdio.h>
2#include <string.h>
3
4#define N 5
5
6int maze[N][N] = {
7 {0, 1, 0, 0, 0},
8 {0, 1, 0, 1, 0},
9 {0, 0, 0, 1, 0},
10 {0, 1, 0, 0, 0},
11 {0, 0, 0, 1, 0}
12}; // 0=길, 1=벽
13
14int dist[N][N]; // 거리 저장
15int dx[] = {0, 0, -1, 1};
16int dy[] = {-1, 1, 0, 0};
17
18int main(void) {
19 // BFS 큐 (간단한 배열 큐)
20 int qx[N*N], qy[N*N];
21 int front = 0, rear = 0;
22
23 memset(dist, -1, sizeof(dist));
24 dist[0][0] = 0;
25 qx[rear] = 0; qy[rear] = 0; rear++;
26
27 while (front < rear) {
28 int x = qx[front], y = qy[front]; front++;
29 for (int i = 0; i < 4; i++) {
30 int nx = x + dx[i], ny = y + dy[i];
31 if (nx >= 0 && nx < N && ny >= 0 && ny < N
32 && maze[ny][nx] == 0 && dist[ny][nx] == -1) {
33 dist[ny][nx] = dist[y][x] + 1;
34 qx[rear] = nx; qy[rear] = ny; rear++;
35 }
36 }
37 }
38
39 printf("(0,0)→(4,4) 최소 거리: %d\n", dist[4][4]);
40 return 0;
41}
실행 결과
(0,0)→(4,4) 최소 거리: 8
단계큐에서 꺼낸 점거리탐색한 이웃큐에 추가
1(0, 0)0상: 벽 / 하: (0,1) / 좌: 밖 / 우: 벽(0, 1)
2(0, 1)1상: 방문 / 하: (0,2) / 좌: 밖 / 우: 벽(0, 2)
3(0, 2)2상: 방문 / 하: (0,3) / 좌: 밖 / 우: (1,2)(0,3), (1,2)
4(0, 3)3상: 방문 / 하: (0,4) / 좌: 밖 / 우: 벽(0, 4)
5(1, 2)3상: 벽 / 하: 벽 / 좌: 방문 / 우: (2,2)(2, 2)

[심화 내용] 극좌표와 좌표 변환

(x, y) 대신 각도(θ)와 거리(r)로 위치를 표현할 수 있어요. 이것이 극좌표(polar coordinates)예요.

[심화 내용] 직교 ↔ 극좌표 변환 직교 → 극좌표:
r = √(x² + y²)
θ = atan2(y, x)

극좌표 → 직교:
x = r × cos(θ)
y = r × sin(θ)

활용: 레이더, 나선 그리기, 회전 애니메이션, 물리 시뮬레이션
r (거리)θ (각도)x = r×cos(θ)y = r×sin(θ)직교좌표
50°5×1.0 = 55×0.0 = 0(5, 0)
590°5×0.0 = 05×1.0 = 5(0, 5)
5180°5×(-1.0) = -55×0.0 = 0(-5, 0)
553.13°5×0.6 = 35×0.8 = 4(3, 4)
1030°10×0.87 = 8.710×0.5 = 5(8.7, 5)

[심화 내용] 동차 좌표와 변환 행렬

[심화 내용] 동차 좌표 (Homogeneous Coordinates) 2D 점 (x, y)를 3개 숫자 (x, y, 1)로 표현하면
이동, 회전, 확대를 모두 행렬 곱으로 처리 가능!

이동(tx, ty): [1 0 tx] [x] [x+tx]
[0 1 ty] [y] = [y+ty]
[0 0 1] [1] [ 1 ]

3D 그래픽(OpenGL, DirectX)의 핵심!
모든 변환을 하나의 4×4 행렬로 합칠 수 있어요.

[심화 내용] 좌표계의 종류 1. 데카르트 좌표 (Cartesian)
(x, y) — 가장 기본. 직교 좌표계

2. 극좌표 (Polar)
(r, theta) — 원점에서의 거리 r, 각도 theta
변환: x = r*cos(theta), y = r*sin(theta)
원, 나선 표현에 유리!

3. 3D 좌표
(x, y, z) — 깊이 추가
게임, 3D 모델링에서 필수

4. 동차 좌표 (Homogeneous)
(x, y, w) — 2D 점을 3D로 확장
이동(Translation)을 행렬 곱으로 표현 가능!
3D 그래픽: (x, y, z, w) — 4x4 행렬로 모든 변환 통합

[심화 내용] 행렬 변환

좌표의 이동, 회전, 확대/축소를 행렬 곱셈으로 표현할 수 있어요. 게임 엔진의 핵심이죠!

[심화 내용] 2D 변환 행렬 이동 (Translation):
[1 0 tx] [x] [x+tx]
[0 1 ty] [y] = [y+ty]
[0 0 1] [1] [ 1 ]

회전 (Rotation, theta):
[cos -sin 0] [x] [x*cos - y*sin]
[sin cos 0] [y] = [x*sin + y*cos]
[ 0 0 1] [1] [ 1 ]

스케일 (Scale):
[sx 0 0] [x] [x*sx]
[ 0 sy 0] [y] = [y*sy]
[ 0 0 1] [1] [ 1 ]

여러 변환 = 행렬 곱의 연쇄!
GPU가 초당 수십억 번의 행렬 곱을 수행!
C[C 코드] 2D 회전 구현
1#include <stdio.h>
2#include <math.h>
3
4typedef struct { double x, y; } Point;
5
6// 원점 기준 회전
7Point rotate(Point p, double degrees) {
8 double rad = degrees * 3.14159265 / 180.0;
9 Point r;
10 r.x = p.x * cos(rad) - p.y * sin(rad);
11 r.y = p.x * sin(rad) + p.y * cos(rad);
12 return r;
13}
14
15int main() {
16 Point p = {1.0, 0.0};
17 printf("원점: (%.2f, %.2f)\n", p.x, p.y);
18
19 for (int deg = 0; deg <= 360; deg += 90) {
20 Point r = rotate(p, deg);
21 printf("%3d도 회전: (%.2f, %.2f)\n", deg, r.x, r.y);
22 }
23 return 0;
24}

원점: (1.00, 0.00) 0도 회전: (1.00, 0.00) 90도 회전: (0.00, 1.00) 180도 회전: (-1.00, 0.00) 270도 회전: (0.00, -1.00) 360도 회전: (1.00, 0.00)

[심화 내용] 거리 함수의 종류

[심화 내용] 다양한 거리 함수 1. 유클리드 거리 (L2)
d = sqrt((x2-x1)^2 + (y2-y1)^2)
직선 거리. 가장 일반적

2. 맨해튼 거리 (L1)
d = |x2-x1| + |y2-y1|
격자 위의 이동 거리. 도시 블록
BFS/그래프에서 자주 사용

3. 체비셰프 거리 (L-infinity)
d = max(|x2-x1|, |y2-y1|)
체스 킹의 이동 거리. 대각선 1칸

4. 민코프스키 거리 (Lp)
d = (|x2-x1|^p + |y2-y1|^p)^(1/p)
p=1이면 맨해튼, p=2이면 유클리드, p=inf이면 체비셰프

KOI/대회에서 "두 점의 거리"라 하면 보통 맨해튼이에요!

용어설명
좌표(Coordinate)위치를 숫자로 나타낸 것. 1차원은 1개, 2차원은 (x,y) 2개.
인덱스(Index)배열에서 각 칸의 번호. C에서는 0부터 시작.
2차원 배열행과 열로 이루어진 격자 데이터 구조. arr[행][열] = arr[y][x].
맨해튼 거리격자를 따라 이동하는 거리. |x₁-x₂|+|y₁-y₂|.
유클리드 거리두 점 사이의 직선 거리. √((x₁-x₂)²+(y₁-y₂)²).
행(Row)격자의 가로 줄. 2차원 배열의 첫 번째 인덱스(y).
열(Column)격자의 세로 줄. 2차원 배열의 두 번째 인덱스(x).
off-by-one인덱스가 1 차이 나는 버그. 0 기반 인덱싱에서 자주 발생.
BFS너비 우선 탐색. 큐를 사용해 가까운 곳부터 탐색. 최단 거리 보장.
극좌표거리(r)와 각도(θ)로 위치를 표현하는 좌표계.
동차 좌표(x,y)를 (x,y,1)로 확장. 이동/회전/확대를 행렬 곱으로 처리.
극좌표(r, theta) 형태. 원점에서의 거리와 각도로 위치 표현. 원형 패턴에 유리.
동차 좌표차원을 하나 추가하여 이동 변환을 행렬 곱으로 표현하는 좌표계.
변환 행렬이동/회전/스케일을 행렬 곱셈으로 표현한 것. GPU의 핵심 연산.
맨해튼 거리|x2-x1| + |y2-y1|. 격자 위의 이동 거리. 도시 블록 거리.
체비셰프 거리max(|x2-x1|, |y2-y1|). 체스 킹의 이동 거리.
NDCNormalized Device Coordinates. -1~+1 범위의 정규화된 좌표.
