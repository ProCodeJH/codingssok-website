유클리드 호제법 — 나머지로 GCD 구하기!

유클리드 호제법(互除法)의 원리는 간단해요: GCD(a, b) = GCD(b, a % b). 큰 수를 작은 수로 나눈 나머지를 이용해서 GCD를 구하면, 약수를 일일이 찾지 않아도 돼요!

[핵심] 유클리드 호제법 GCD(a, b) = GCD(b, a % b)

나머지가 0이 되면 그때의 나누는 수가 GCD!

GCD(48, 18)
= GCD(18, 48%18) = GCD(18, 12)
= GCD(12, 18%12) = GCD(12, 6)
= GCD(6, 12%6) = GCD(6, 0)
= 6 ← 나머지가 0! 여기서 끝!

LCM 구하기 — GCD만 알면 바로 계산!

최소공배수(LCM)를 구하는 공식은 놀라울 정도로 간단해요. 두 수의 곱을 GCD로 나누면 끝이에요!

[핵심] LCM 공식 LCM(A, B) = A × B ÷ GCD(A, B)

또는: A × B = GCD(A, B) × LCM(A, B)

예: LCM(12, 18) = 12 × 18 ÷ 6 = 216 ÷ 6 = 36
예: LCM(252, 105) = 252 × 105 ÷ 21 = 26460 ÷ 21 = 1260

    a = temp;
  }
  return a;
}

[심화] C로 GCD/LCM 구현

C[C 코드] GCD/LCM 4가지 방법
1#include <stdio.h>
2
3// 방법 1: 브루트포스 O(min(a,b))
4int gcd_brute(int a, int b) {
5 int g = 1;
6 for (int i = 2; i <= a && i <= b; i++)
7 if (a % i == 0 && b % i == 0) g = i;
8 return g;
9}
10
11// 방법 2: 유클리드 호제법 (반복) O(log(min(a,b)))
12int gcd_iter(int a, int b) {
13 while (b != 0) {
14 int t = b;
15 b = a % b;
16 a = t;
17 }
18 return a;
19}
20
21// 방법 3: 유클리드 호제법 (재귀)
22int gcd_rec(int a, int b) {
23 return b == 0 ? a : gcd_rec(b, a % b);
24}
25
26// LCM = a * b / GCD
27long long lcm(int a, int b) {
28 return (long long)a / gcd_iter(a, b) * b;
29}
30
31int main() {
32 int pairs[][2] = {{12,18},{48,36},{100,75},{17,13},{360,240}};
33 for (int i = 0; i < 5; i++) {
34 int a = pairs[i][0], b = pairs[i][1];
35 printf("GCD(%d,%d) = %d, LCM = %lld\n",
36 a, b, gcd_iter(a,b), lcm(a,b));
37 }
38 return 0;
39}

[심화 내용] 유클리드 호제법의 시간 복잡도

[심화 내용] 유클리드 알고리즘 분석 시간 복잡도: O(log(min(a, b)))

증명 핵심:
a mod b < a/2 (나머지는 항상 피제수의 절반 미만)
따라서 2단계마다 수가 최소 절반으로 줄어듦
최대 단계 수 = 2 * log2(min(a,b))

최악의 경우: 연속 피보나치 수!
GCD(F(n+1), F(n)) — 매 단계 나머지 = 이전 피보나치
GCD(89, 55) -> GCD(55, 34) -> GCD(34, 21) -> ...

라메의 정리 (1844):
유클리드 알고리즘의 단계 수 <= 5 * (십진 자릿수)
10자리 수도 최대 50단계면 끝!

2300년 전 알고리즘이 여전히 최적이라는 게 놀라워요!

[심화 내용] 확장 유클리드 알고리즘

[심화 내용] 확장 유클리드 알고리즘 (Extended GCD) 베주의 항등식:
GCD(a, b) = a*x + b*y를 만족하는 정수 x, y가 존재!

예: GCD(12, 8) = 4
4 = 12*1 + 8*(-1) (x=1, y=-1)

확장 유클리드 알고리즘:
GCD를 구하면서 동시에 x, y도 계산

용도:
1. 모듈러 역원 계산
a*x ≡ 1 (mod m)인 x 찾기
-> GCD(a, m) = 1이면 x 존재!

2. RSA 키 생성
d = e^(-1) mod phi(n)
확장 유클리드로 d를 계산!

3. 디오판토스 방정식
ax + by = c의 정수해 찾기
C[C 코드] 확장 유클리드
1#include <stdio.h>
2
3// ax + by = gcd(a,b)를 만족하는 x, y를 찾는다
4int ext_gcd(int a, int b, int *x, int *y) {
5 if (b == 0) {
6 *x = 1; *y = 0;
7 return a;
8 }
9 int x1, y1;
10 int g = ext_gcd(b, a % b, &x1, &y1);
11 *x = y1;
12 *y = x1 - (a / b) * y1;
13 return g;
14}
15
16// 모듈러 역원: a*x ≡ 1 (mod m)
17int mod_inverse(int a, int m) {
18 int x, y;
19 int g = ext_gcd(a, m, &x, &y);
20 if (g != 1) return -1; // 역원 없음
21 return (x % m + m) % m;
22}
23
24int main() {
25 int x, y;
26 int g = ext_gcd(12, 8, &x, &y);
27 printf("GCD(12,8) = %d = 12*(%d) + 8*(%d)\n", g, x, y);
28
29 // 모듈러 역원
30 printf("3^(-1) mod 7 = %d\n", mod_inverse(3, 7)); // 5
31 printf("검증: 3*5 mod 7 = %d\n", (3*5) % 7); // 1
32 return 0;
33}
실행 결과
GCD(12,8) = 4 = 12*(1) + 8*(-1) 3^(-1) mod 7 = 5 검증: 3*5 mod 7 = 1

[심화 내용] GCD의 성질과 응용

[심화 내용] GCD의 수학적 성질 기본 성질:
GCD(a, 0) = a
GCD(a, a) = a
GCD(a, b) = GCD(b, a) (교환법칙)
GCD(a, b) = GCD(a-b, b) (뺄셈 버전)
GCD(a, b) = GCD(a mod b, b) (유클리드)

분배 성질:
GCD(ka, kb) = k * GCD(a, b)
GCD(a, b) * LCM(a, b) = a * b

서로소:
GCD(a, b) = 1이면 a와 b는 서로소
서로소이면 ax + by = 1인 정수 x, y 존재

3개 이상의 GCD:
GCD(a, b, c) = GCD(GCD(a, b), c)

[심화 내용] GCD와 선형 대수

[심화 내용] GCD의 격자(Lattice) 해석 2차원 격자에서 GCD(a,b)의 기하학적 의미:

원점 (0,0)에서 점 (a,b)까지 직선을 그으면
직선이 지나는 격자점의 수 = GCD(a,b) - 1

예: (12, 8) 직선
GCD(12,8) = 4
격자점: (3,2), (6,4), (9,6) = 3개 = 4-1

이것이 유클리드 알고리즘의 기하학적 해석!

또한 GCD(a,b) = 직사각형 a*b를 정사각형으로 채울 때
가장 큰 정사각형의 변의 길이

12*8 직사각형 -> 8*8 + 4*8 -> 4*4 + 4*4 = GCD=4
[심화 내용] 면접 문제 Q1. GCD(0, 0)은?
A: 정의에 따라 0. (0은 모든 수의 배수이므로)

Q2. GCD를 재귀 없이 구현하세요.
A: while (b) { int t=b; b=a%b; a=t; } return a;

Q3. GCD(2^a - 1, 2^b - 1) = ?
A: 2^GCD(a,b) - 1. 메르센 수의 GCD도 메르센 수!

Q4. LCM을 구할 때 a*b/GCD 대신 a/GCD*b로 쓰는 이유?
A: a*b에서 오버플로우 방지! 먼저 나누면 안전.

Q5. 유클리드 알고리즘이 반드시 종료하는 이유?
A: b가 매 단계 감소(나머지 < 제수). 0 이상 정수이므로 반드시 0에 도달.

[심화 내용] 다항식의 GCD

[심화 내용] 다항식에도 GCD가 있다! 정수뿐 아니라 다항식에도 유클리드 알고리즘 적용 가능!

GCD(x^2-1, x^2-2x+1)
= GCD(x^2-1, x^2-2x+1)

x^2-1 = (x+1)(x-1)
x^2-2x+1 = (x-1)^2

공통 인수: (x-1)
GCD = x-1

이 개념은:
- 심볼릭 수학 소프트웨어 (Mathematica, SymPy)
- 오류 정정 코드 (Reed-Solomon)

[심화 내용] 이진 GCD (Stein 알고리즘)

[심화 내용] 이진 GCD 알고리즘 나머지 연산(%) 없이 비트 연산만으로 GCD를 구하는 방법!

규칙:
1. 둘 다 짝수: GCD(a,b) = 2 * GCD(a/2, b/2)
2. a만 짝수: GCD(a,b) = GCD(a/2, b)
3. b만 짝수: GCD(a,b) = GCD(a, b/2)
4. 둘 다 홀수: GCD(a,b) = GCD(|a-b|/2, min(a,b))

장점:
- 나눗셈(느림) 대신 시프트+뺄셈(빠름) 사용
- 임베디드 시스템, 빅넘 연산에서 유리
- GMP(GNU 다정밀 산술 라이브러리)에서 실제 사용

시간 복잡도: O(log(a*b)) = O(log a + log b)
C[C 코드] 이진 GCD
1#include <stdio.h>
2
3int binary_gcd(int a, int b) {
4 if (a == 0) return b;
5 if (b == 0) return a;
6
7 int shift = 0;
8 while (((a | b) & 1) == 0) { // 둘 다 짝수
9 a >>= 1; b >>= 1; shift++;
10 }
11 while ((a & 1) == 0) a >>= 1; // a를 홀수로
12
13 do {
14 while ((b & 1) == 0) b >>= 1; // b를 홀수로
15 if (a > b) { int t = a; a = b; b = t; }
16 b -= a;
17 } while (b != 0);
18
19 return a << shift;
20}
21
22int main() {
23 printf("binary_gcd(48, 18) = %d\n", binary_gcd(48, 18));
24 printf("binary_gcd(100, 75) = %d\n", binary_gcd(100, 75));
25 printf("binary_gcd(1071, 462) = %d\n", binary_gcd(1071, 462));
26 return 0;
27}
실행 결과
binary_gcd(48, 18) = 6 binary_gcd(100, 75) = 25 binary_gcd(1071, 462) = 21

용어설명
약수(Divisor)어떤 수를 나머지 없이 나눌 수 있는 수. 예: 12의 약수는 1, 2, 3, 4, 6, 12.
공약수(Common Divisor)두 수가 공통으로 가지는 약수. 예: 12와 18의 공약수는 1, 2, 3, 6.
최대공약수(GCD)Greatest Common Divisor. 공약수 중 가장 큰 수.
배수(Multiple)어떤 수에 자연수를 곱한 수. 예: 4의 배수는 4, 8, 12, 16, ...
공배수(Common Multiple)두 수가 공통으로 가지는 배수. 예: 4와 6의 공배수는 12, 24, 36, ...
최소공배수(LCM)Least Common Multiple. 공배수 중 가장 작은 수.
유클리드 호제법GCD(a,b) = GCD(b, a%b)를 반복하여 GCD를 구하는 효율적 알고리즘.
재귀(Recursion)함수가 자기 자신을 호출하는 프로그래밍 기법. 유클리드 호제법이 대표적 예시.
유클리드 호제법GCD를 구하는 알고리즘. a mod b를 반복. 기원전 300년, 가장 오래된 알고리즘.
확장 유클리드GCD와 함께 ax+by=GCD를 만족하는 x,y를 구하는 알고리즘.
베주의 항등식GCD(a,b) = ax + by를 만족하는 정수 x,y가 반드시 존재한다는 정리.
모듈러 역원a*x ≡ 1 (mod m)인 x. 확장 유클리드로 계산. RSA 핵심.
서로소GCD(a,b)=1인 관계. 공통 약수가 1뿐.
라메의 정리유클리드 알고리즘 단계 수 <= 5*(자릿수). 효율성 증명.
디오판토스 방정식정수해만 구하는 방정식. ax+by=c 형태.
이번 유닛 핵심 정리
1. GCD = 두 수의 최대공약수, LCM = 최소공배수
2. GCD * LCM = a * b (항상 성립!)
3. 유클리드 호제법: GCD(a,b) = GCD(b, a%b)
4. [심화] C로 GCD 3줄: b==0 ? a : gcd(b, a%b)
5. [심화] 기약분수 = 분자분모를 GCD로 나누기
6. [심화 내용] 확장 유클리드: ax+by=GCD 의 x,y도 구함
7. [심화 내용] 모듈러 역원: RSA 키 생성의 핵심 도구
8. [심화 내용] 최악 = 피보나치 수, 그래도 O(log n)
