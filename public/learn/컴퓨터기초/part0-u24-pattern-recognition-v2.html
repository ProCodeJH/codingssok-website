[심화] DRY 원칙 — 반복하지 마!

[심화] DRY = Don't Repeat Yourself DRY 원칙: 같은 코드를 두 번 쓰지 마라!

반복이 보이면 → 반복문 또는 함수로 추출

❌ WET (Write Everything Twice): 복붙 코드
✅ DRY: 패턴을 찾아 추상화한 코드

DRY를 지키면:
• 코드 양 감소
• 수정할 때 한 곳만 고침
• 버그 가능성 감소

1// ❌ WET — 같은 패턴 3번 반복
2int sum1 = 0;
3for (int i = 0; i < n1; i++) sum1 += arr1[i];
4int sum2 = 0;
5for (int i = 0; i < n2; i++) sum2 += arr2[i];
6int sum3 = 0;
7for (int i = 0; i < n3; i++) sum3 += arr3[i];
8
9// ✅ DRY — 패턴을 함수로 추출
10int arraySum(int arr[], int n) {
11 int sum = 0;
12 for (int i = 0; i < n; i++) sum += arr[i];
13 return sum;
14}

[심화] 코드 냄새 (Code Smell) 코드 냄새 = 코드에서 문제가 될 수 있는 나쁜 패턴

대표적인 코드 냄새:
• 중복 코드: 같은 코드가 여러 곳에 (DRY 위반)
• 긴 함수: 100줄 넘는 함수 (분해 필요)

• 죽은 코드: 실행되지 않는 코드
• 깊은 중첩: if 안 if 안 if...

[심화 내용] 정규표현식 (Regular Expression) 문자열에서 복잡한 패턴을 찾는 도구!

기본 문법:
• . — 아무 문자 1개
• * — 앞 문자 0번 이상 반복
• + — 앞 문자 1번 이상 반복
• [abc] — a, b, c 중 하나
• [0-9] — 숫자 하나
• \d — 숫자 (=[0-9])

예: 전화번호 패턴 → \d{3}-\d{4}-\d{4}
이메일 패턴 → [a-zA-Z]+@[a-z]+\.[a-z]+

[심화 내용] 디자인 패턴 맛보기

[심화 내용] GoF 디자인 패턴 디자인 패턴 = 소프트웨어 설계에서 반복되는 문제의 해결 템플릿

1995년 Gang of Four (GoF) 책에서 23가지 패턴 정리

생성 패턴:
- Singleton: 인스턴스 1개만 (설정 관리자)
- Factory: 객체 생성을 함수에 위임

구조 패턴:
- Adapter: 호환 안 되는 인터페이스 연결
- Decorator: 기능을 동적으로 추가

행동 패턴:
- Observer: 상태 변화 알림 (이벤트 시스템)
- Strategy: 알고리즘을 교체 가능하게

패턴 인식 능력 = 디자인 패턴을 "알아보는" 능력!

[심화 내용] 알고리즘 복잡도 패턴

[심화 내용] 코드 패턴과 복잡도 코드의 구조적 패턴을 보면 복잡도를 바로 알 수 있어요!

O(1): 반복문 없음, 인덱스 접근
O(log n): 매번 절반으로 줄어듦 (이진탐색)
O(n): for 1개 (전체 순회)
O(n log n): 분할정복 + 합치기 (머지소트)
O(n^2): 중첩 for 2개 (버블소트)
O(n^3): 중첩 for 3개 (행렬곱)
O(2^n): 재귀가 2갈래로 분기 (피보나치 재귀)
O(n!): 모든 순열 생성

패턴을 보면 바로 복잡도를 "인식"할 수 있어요!

[심화 내용] 오토마타 이론과 패턴

[심화 내용] 유한 오토마타 (FSM) 유한 상태 기계(Finite State Machine)
= 패턴 인식의 수학적 모델

구성요소:
1. 상태(States): 유한 개의 상태 집합
2. 입력(Alphabet): 받아들이는 문자 집합
3. 전이(Transition): 입력에 따른 상태 변화
4. 시작 상태: 처음 시작하는 상태
5. 받아들이는 상태: 패턴 매칭 성공

정규 표현식 = 유한 오토마타 = 같은 것!
(Kleene의 정리, 1956)

컴파일러의 렉서(Lexer)가 이것으로 토큰을 인식해요.
int, if, while 같은 키워드도 FSM으로 구분!
[심화 내용] 촘스키 계층 형식 언어의 4단계 계층 (Chomsky Hierarchy):

Type 3: 정규 언어 (정규식, FSM)
예: a*b+ (aab, abbb, ...)
한계: 괄호 짝 맞추기 불가!

Type 2: 문맥 자유 언어 (CFG, PDA)
예: 올바른 괄호 (()), 프로그래밍 언어 문법
컴파일러의 파서가 처리

Type 1: 문맥 의존 언어 (CSG, LBA)
예: a^n b^n c^n

Type 0: 재귀 열거 가능 언어 (튜링 머신)
모든 계산 가능한 것

Type 3 ⊂ Type 2 ⊂ Type 1 ⊂ Type 0
