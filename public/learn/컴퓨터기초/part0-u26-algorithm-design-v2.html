알고리즘의 5가지 조건:
1. 입력이 있어요 (0개 이상)
2. 출력이 있어요 (1개 이상)
3. 유한성: 유한한 단계로 끝나야 해요
4. 명확성: 각 단계가 애매하지 않아야 해요
5. 실행가능성: 각 단계를 실제로 수행할 수 있어야 해요

Cfind_max_linear.c
1#include <stdio.h>
2
3// 방법 3: 순차 탐색 — 가장 효율적! O(n)
4int findMax(int arr[], int n) {
5 int max = arr[0]; // 첫 번째를 일단 최대로
6 for (int i = 1; i < n; i++) {
7 if (arr[i] > max)
8 max = arr[i]; // 더 큰 수 발견!
9 }
10 return max;
11}
12
13int main(void) {
14 int data[] = {5, 2, 8, 1, 9, 3, 7};
15 int n = 7;
16 printf("최댓값: %d\n", findMax(data, n)); // 9
17 return 0;
18}

[심화] 알고리즘 설계 실전: 동전 거슬러주기

Ccoin_greedy.c
1#include <stdio.h>
2
3int main(void) {
4 int coins[] = {500, 100, 50, 10};
5 int n = 4;
6 int amount = 1260;
7 int count = 0;
8
9 printf("%d원 거슬러주기:\n", amount);
10 for (int i = 0; i < n; i++) {
11 int use = amount / coins[i];
12 if (use > 0) {
13 printf(" %d원 × %d개\n", coins[i], use);
14 count += use;
15 amount -= coins[i] * use;
16 }
17 }
18 printf("총 동전 %d개\n", count);
19 return 0;
20}

[심화 내용] 알고리즘 정당성 증명

[심화 내용] 정당성 증명 방법 알고리즘이 정확하다는 것을 어떻게 증명할까?

1. 수학적 귀납법: n=1일 때 성립 → n=k 성립 가정 → n=k+1 증명
2. 루프 불변식(Loop Invariant): 반복문 시작 전·중·후에 항상 성립하는 조건
3. 모순법: 최적이 아니라고 가정 → 모순 발견 → 최적 증명

예: 순차 탐색의 정당성
불변식: 'i번째까지 검사한 원소 중 최대값이 max에 저장되어 있다'

[심화 내용] 정지 문제 — 계산의 한계

[심화 내용] 정지 문제 (Halting Problem) 질문: "주어진 프로그램이 유한 시간에 끝나는가?"
를 판단하는 프로그램을 만들 수 있는가?

답: 불가능! (튜링, 1936)

증명 (대각선 논법):
만약 Halt(P, I) 함수가 존재한다면...
D(P) = if Halt(P,P) then 무한루프 else 멈춤
D(D)는? Halt(D,D)가 참이면 무한루프... 모순!

의미:
1. 완벽한 디버거/바이러스 탐지기는 불가능
2. 모든 프로그램의 정확성을 자동 검증하는 건 불가능
3. 수학에서 증명 불가능한 참인 명제가 존재 (괴델)

[활동] 알고리즘 설계 연습 다음 문제를 어떤 설계 기법으로 풀지 골라보세요:

1. 사전에서 단어 찾기 → 분할 정복 (이진 탐색)
2. 거스름돈 최소 동전 수 → 탐욕법
3. 계단 오르기 경우의 수 → DP
4. 비밀번호 맞추기 → 완전 탐색
5. 미로 탈출 → 백트래킹

문제를 보고 어떤 패러다임이 맞는지
판단하는 능력 = 알고리즘 설계 실력!
용어설명
알고리즘문제를 해결하기 위한 유한한 단계별 절차. 입력→처리→출력.
의사코드프로그래밍 언어는 아니지만 코드처럼 쓴 알고리즘 표현.
정확성모든 입력에 대해 올바른 답을 내는 성질.
효율성적은 시간·공간으로 문제를 해결하는 성질.
Big-O알고리즘의 시간/공간 복잡도를 나타내는 표기법. O(n), O(n²) 등.
시간 복잡도입력 크기에 따른 연산 횟수의 증가 패턴.
그리디매 순간 최선의 선택을 하는 알고리즘 전략.
분할정복문제를 나누고, 각각 풀고, 합치는 전략.
DPDynamic Programming. 중복 계산을 저장하여 재사용하는 전략.
백트래킹유망하지 않은 경로를 가지치기하여 탐색을 줄이는 전략.
NP답의 검증이 다항 시간인 문제 클래스. P=NP? 미해결.
루프 불변식반복문 실행 전·중·후에 항상 성립하는 조건. 정당성 증명에 사용.
Big-O 표기법알고리즘의 최악 시간 복잡도를 나타내는 표기. O(n), O(n log n) 등.
동적 프로그래밍(DP)중복 부분 문제를 메모이제이션으로 한 번만 계산하는 기법.
NP-완전NP 중 가장 어려운 문제 클래스. 하나 풀면 전부 풀림.
정지 문제프로그램이 유한 시간에 끝나는지 판단하는 문제. 불가능함이 증명됨(튜링, 1936).
근사 알고리즘최적해는 못 구하지만 "충분히 좋은" 해를 빠르게 구하는 알고리즘.
메모이제이션계산 결과를 저장해두고 같은 입력이 오면 재사용하는 기법.
