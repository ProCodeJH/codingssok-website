UNIT 29 완전 탐색 (Brute Force)

완전 탐색이 뭘까?

[핵심] 완전 탐색이란? 완전 탐색(Brute Force) = 가능한 모든 경우를 빠짐없이 시도하는 방법

장점: 반드시 정답을 찾는다!
단점: 경우가 많으면 시간이 오래 걸린다

언제 쓸까?
• N이 작을 때 (보통 N ≤ 20)
• 다른 방법이 떠오르지 않을 때
• 정답을 확인하고 싶을 때

예시: 자물쇠 비밀번호 찾기

1#include <stdio.h>
2

5
6 // 완전 탐색: 000~999 전부 시도!

8 if (guess == password) {
9 printf("비밀번호 찾았다! %03d\n", guess);

11 }
12 }
13 return 0;
14}
실행 결과
비밀번호 찾았다! 742
[팁] 완전 탐색의 핵심 패턴 1. 모든 경우 나열: for문으로 전부 돌기
2. 조건 확인: if로 정답인지 체크
3. 정답이면 멈춤: break 또는 return

이 3단계만 기억하면 어떤 완전 탐색이든 구현 가능!

1#include <stdio.h>
2
3int main(void) {
4 char items[] = {'A', 'B', 'C'};
5 int n = 3;
6
7 // 0부터 2^n - 1까지 모든 비트마스크
8 for (int mask = 0; mask < (1 << n); mask++) {
9 printf("{ ");
10 for (int i = 0; i < n; i++) {
11 if (mask & (1 << i)) // i번째 비트가 1이면 포함

13 }

17}

[심화 내용] 백트래킹 (Backtracking) 완전 탐색 + 가지치기(pruning) = 백트래킹

모든 경우를 다 보지 않고, 유망하지 않은 경로를 미리 포기!

예: N-Queen 문제
• 완전 탐색: N! 가지 모두 시도
• 백트래킹: 같은 열/대각선이면 즉시 건너뜀
• 8-Queen: 40,320 → 약 114가지만 탐색!
Cnqueen.c
1#include <stdio.h>
2#include <stdlib.h>
3
4#define N 8
5int col[N]; // col[i] = i행의 퀸 열 번호
6int count = 0;
7
8int promising(int row) {
9 for (int i = 0; i < row; i++) {
10 if (col[i] == col[row]) return 0; // 같은 열
11 if (abs(col[i]-col[row]) == row-i) return 0; // 대각선
12 }
13 return 1;
14}
15
16void solve(int row) {
17 if (row == N) { count++; return; } // 해 발견!
18 for (int c = 0; c < N; c++) {
19 col[row] = c;
20 if (promising(row)) // 가지치기!
21 solve(row + 1);
22 }
23}
24
25int main(void) {
26 solve(0);
27 printf("%d-Queen 해의 수: %d\n", N, count); // 92
28 return 0;
29}

[심화 내용] 완전 탐색의 최적화 기법

[심화 내용] 가지치기 (Pruning) 완전 탐색은 모든 경우를 탐색하지만,
불필요한 경우를 미리 제거하면 훨씬 빨라져요!

1. 조건 가지치기
현재까지의 부분해가 이미 조건을 위반하면 중단
예: 부분합이 목표를 초과하면 더 탐색할 필요 없음

2. 대칭 제거
대칭인 경우를 한 번만 탐색
예: N-Queen에서 첫 행은 절반만 시도

3. 상한/하한 (Bound)
현재까지의 최적해보다 나을 수 없으면 중단
= 분기한정법 (Branch and Bound)

4. 메모이제이션
같은 상태를 다시 방문하면 저장된 결과 재사용
= 완전탐색 → DP로 전환!
C[C 코드] N-Queen (백트래킹 + 가지치기)
1#include <stdio.h>
2#include <stdlib.h>
3
4int N, count = 0;
5int col[20];
6
7int is_safe(int row, int c) {
8 for (int i = 0; i < row; i++) {
9 if (col[i] == c) return 0; // 같은 열
10 if (abs(col[i]-c) == abs(i-row)) return 0; // 대각선
11 }
12 return 1;
13}
14
15void solve(int row) {
16 if (row == N) { count++; return; }
17 for (int c = 0; c < N; c++) {
18 if (is_safe(row, c)) { // 가지치기!
19 col[row] = c;
20 solve(row + 1);
21 }
22 }
23}
24
25int main() {
26 for (N = 1; N <= 12; N++) {
27 count = 0;
28 solve(0);
29 printf("N=%2d: %d solutions\n", N, count);
30 }
31 return 0;
32}

[심화 내용] 완전 탐색을 써야 하는 상황 1. 입력이 작을 때

n <= 10이면 10! = 360만 → 가능

2. 최적해를 보장해야 할 때

작은 입력에서 완전 탐색 = 정답
이것과 비교해서 DP/그리디의 정확성 검증

1#include <stdio.h>
2
3// 부분집합 합 문제: 합이 target인 부분집합이 있는가?
4int subset_sum(int arr[], int n, int target) {
5 // 모든 부분집합을 비트마스크로 순회
6 for (int mask = 0; mask < (1 << n); mask++) {
7 int sum = 0;
8 for (int i = 0; i < n; i++)
9 if (mask & (1 << i)) sum += arr[i];
10 if (sum == target) {
11 printf("합이 %d인 부분집합: {", target);
12 int first = 1;
13 for (int i = 0; i < n; i++) {
14 if (mask & (1 << i)) {
15 if (!first) printf(",");
16 printf("%d", arr[i]);
17 first = 0;
18 }
19 }
20 printf("}\n");
21 return 1;
22 }
23 }
24 return 0;
25}

[심화] 비트마스크 완전 탐색 핵심 n개 원소의 부분집합 = 2^n개

비트마스크 0~(2^n - 1)의 각 비트가
해당 원소를 선택(1)/미선택(0) 표현

mask = 0b10101 → {arr[0], arr[2], arr[4]} 선택

시간: O(2^n * n)
한계: n <= 20~25 정도까지만 실용적
n=20: 2^20 = 약 100만 (OK)
n=30: 2^30 = 약 10억 (느림)
[심화 내용] 대회에서 완전 탐색 판단 기준 n의 범위로 판단:

n <= 10: O(n!) 가능 → 순열 완전 탐색
n <= 20: O(2^n) 가능 → 비트마스크 부분집합
n <= 25: O(2^(n/2)) 가능 → Meet in the Middle
n <= 100: O(n^3) 정도까지
n <= 1000: O(n^2)
n <= 10^5: O(n log n)
n <= 10^7: O(n)

시간 제한 1초 기준:
C/C++: ~10^8 연산/초
Python: ~10^6 연산/초 (100배 느림!)

이 표를 외우면 문제를 보자마자
"완전 탐색 가능/불가능"을 즉시 판단!

[심화 내용] Meet in the Middle 기법

[심화 내용] Meet in the Middle O(2^n)을 O(2^(n/2))로 줄이는 기법!

원리:
1. 입력을 절반으로 나눔 (A 그룹, B 그룹)
2. A의 모든 부분집합 합 계산 (2^(n/2)개)
3. B의 모든 부분집합 합 계산 (2^(n/2)개)
4. 두 그룹에서 합이 target인 조합 찾기 (정렬+이진탐색)

예: n=40일 때
완전 탐색: 2^40 = 약 10^12 → 불가능
MitM: 2^20 * 2 = 약 200만 → 가능!

대회에서 n이 20~40 사이면 MitM 의심!
[활동] 완전 탐색 도전 문제 1. 비밀번호 크래킹
4자리 숫자(0000~9999)를 전부 시도 → 10000번

2. 부분집합 합
{2, 5, 8, 12, 16}에서 합이 25인 부분집합 찾기
힌트: 비트마스크 0~31 순회

3. 순열 정렬
{3,1,2}의 모든 순열 중 정렬된 것 찾기

4. 두 수의 합
배열에서 합이 target인 두 수의 인덱스 찾기
O(n^2) 완전 탐색 → O(n) 해시맵으로 최적화 가능!

용어설명
완전 탐색가능한 모든 경우를 빠짐없이 시도하는 알고리즘. Brute Force.
브루트 포스Brute Force. '무차별 대입'. 완전 탐색의 영어 이름.
비트마스크비트(0/1)로 집합의 포함 여부를 표현하는 기법. 부분집합 탐색에 사용.
부분집합집합의 일부 원소로 이루어진 집합. N개 원소 → 2ⁿ개 부분집합.
순열원소들의 순서 있는 배열. N개 원소 → N!개 순열.
백트래킹완전 탐색 + 가지치기. 유망하지 않은 경로를 미리 포기.
가지치기Pruning. 탐색 트리에서 불필요한 가지를 잘라내는 기법.
N-QueenN×N 체스판에 N개의 퀸을 서로 공격 못하게 배치하는 문제.
경우의 수가능한 결과의 총 개수. 완전 탐색의 시간을 결정.
가지치기(Pruning)탐색 트리에서 불필요한 가지를 미리 잘라내는 최적화 기법.
분기한정법Branch and Bound. 상한/하한을 이용한 가지치기. 최적화 문제에 사용.
N-QueenN*N 체스판에 N개 퀸을 서로 공격하지 않게 놓는 문제. 백트래킹의 대표 예제.
KISS 원칙Keep It Simple, Stupid. 단순한 해결책이 최선이라는 소프트웨어 원칙.
