탐욕법이 뭘까?

[핵심] 탐욕법이란? 탐욕법(Greedy Algorithm) = 매 순간 가장 좋아 보이는 선택을 하는 전략

특징:
1. 한 번 선택하면 되돌리지 않음
2. 현재 상황에서 최선을 선택
3. 빠르다! 보통 O(n) 또는 O(n log n)

주의: 항상 전체 최적이 되는 건 아니에요!

예시 1: 거스름돈 문제

Cgreedy_coin.c
1#include <stdio.h>
2
3int main(void) {
4 int coins[] = {500, 100, 50, 10};
5 int n = 4;
6 int amount = 1260;
7 int count = 0;
8
9 printf("%d원 거슬러주기:\n", amount);
10 for (int i = 0; i < n; i++) {
11 int use = amount / coins[i]; // 가장 큰 동전부터!
12 if (use > 0) {
13 printf(" %d원 × %d개\n", coins[i], use);
14 count += use;
15 amount -= coins[i] * use;
16 }
17 }
18 printf("총 동전 %d개\n", count);
19 return 0;
20}

예시 3: 분할 가능 배낭 문제

Cgreedy_knapsack.c
1#include <stdio.h>
2
3int main(void) {
4 // 물건: {무게, 가치}
5 float weight[] = {10, 20, 30};
6 float value[] = {60, 100, 120};
7 int n = 3;
8 float capacity = 50;
9
10 // 가치/무게 비율: 6, 5, 4
11 // 그리디: 비율 높은 것부터!
12 // (이미 비율 순 정렬 가정)
13
14 float totalValue = 0;
15 for (int i = 0; i < n; i++) {
16 if (capacity >= weight[i]) {
17 // 통째로 넣기
18 capacity -= weight[i];
19 totalValue += value[i];
20 printf("물건%d: 전부 넣기 (%.0f)\n", i+1, value[i]);
21 } else {
22 // 잘라서 넣기 (분할 가능!)
23 float fraction = capacity / weight[i];
24 totalValue += value[i] * fraction;
25 printf("물건%d: %.0f%% 넣기 (%.0f)\n",
26 i+1, fraction*100, value[i]*fraction);
27 capacity = 0;
28 break;
29 }
30 }
31 printf("최대 가치: %.0f\n", totalValue);
32 return 0;
33}

[심화] 그리디가 최적인 2가지 조건

[심화] 그리디 정당성의 2가지 조건 그리디가 최적 해를 보장하려면 두 조건이 필요:

1. 탐욕적 선택 속성 (Greedy Choice Property)
→ 매 순간 최선의 선택이 전체 최선에 포함됨

2. 최적 부분 구조 (Optimal Substructure)
→ 하위 문제의 최적 해가 전체 문제의 최적 해에 포함됨

두 조건 모두 성립하면 그리디 = 최적!
문제전략최적?시간
거스름돈(한국)큰 동전 우선O (배수 조건)O(종류)
거스름돈(임의)큰 동전 우선X (반례 존재)—
활동 선택빨리 끝나는 것 우선O (증명 가능)O(n log n)
분할 배낭가치/무게 비율 우선OO(n log n)
0-1 배낭그리디 불가XDP 필요
허프만 코딩빈도 낮은 것 합치기OO(n log n)

[심화] 그리디 실전: 최소 동전 (DP와 비교)

[심화] 0-1 배낭 vs 분할 배낭 분할 배낭: 물건을 잘라서 넣을 수 있음 → 그리디 OK
0-1 배낭: 물건을 통째로만 넣을 수 있음 → 그리디 X, DP 필요

비슷해 보이지만 '분할 가능 여부'에 따라 전략이 완전히 달라!

[심화 내용] 그리디 정당성 증명

[심화 내용] 교환 논증 (Exchange Argument) 그리디 해가 최적임을 증명하는 대표적 기법:

1. 최적 해 OPT가 있다고 가정
2. OPT가 그리디와 다른 부분이 있으면
3. 그 부분을 그리디의 선택으로 교환해도 더 나빠지지 않음을 보임
4. 따라서 그리디 해 = 최적 해

예: 활동 선택 문제 — 빨리 끝나는 것을 안 고른 최적 해가 있다면,
교환해도 회의 수가 줄지 않으므로 그리디가 최적.

[심화 내용] 허프만 코딩 — 그리디의 명작

[심화 내용] 허프만 코딩 허프만 코딩 = 빈도가 높은 문자에 짧은 코드, 낮은 문자에 긴 코드

[심화 내용] 탐욕법이 최적인 조건

[심화 내용] 탐욕적 선택 속성과 최적 부분 구조 탐욕법이 최적해를 보장하려면 두 조건이 필요:

1. 탐욕적 선택 속성 (Greedy Choice Property)
매 단계의 최선의 선택이 전체 최적해에 포함됨

2. 최적 부분 구조 (Optimal Substructure)
부분 문제의 최적해로 전체 최적해를 구성할 수 있음

최적인 경우:
- 거스름돈 (동전이 배수 관계일 때만!)
- 활동 선택 (끝나는 시간 기준)
- Huffman 코딩
- Kruskal/Prim MST
- Dijkstra 최단 경로 (음수 간선 없을 때)

최적이 아닌 경우:
- 거스름돈 {1, 3, 4}로 6원: 그리디=4+1+1(3개), 최적=3+3(2개)!
- 0/1 배낭 문제
- TSP (외판원)

18 }

1#include <stdio.h>
2
3// 동전: {1, 3, 4}로 6원 만들기
4int greedy_coins(int coins[], int n, int amount) {
5 int count = 0;
6 for (int i = n-1; i >= 0; i--)
7 while (amount >= coins[i]) { amount -= coins[i]; count++; }
8 return count;
9}
10
11int main() {

13 printf("그리디: 6원 = %d개 동전\n", greedy_coins(coins, 3, 6));
14 // 그리디: 4+1+1 = 3개
15 // 최적: 3+3 = 2개!
16 printf("최적: 3+3 = 2개 (그리디 실패!)\n");
17 return 0;
18}

[심화 내용] 탐욕법 함정 주의 그리디가 맞는지 확인하는 방법:

1. 반례 찾기
작은 입력에서 완전 탐색과 비교
하나라도 다르면 그리디 실패!

2. 수학적 증명
탐욕적 선택 속성 + 최적 부분 구조
교환 논법(Exchange Argument)으로 증명

3. 직관 체크
"이 선택이 미래에 후회될 수 있는가?"
후회 가능성이 있으면 그리디가 아닌 DP!
[심화 내용] 유명한 그리디 알고리즘 모음 1. Kruskal MST
간선을 가중치 순 정렬 → 사이클 안 만들면 추가
Union-Find 사용, O(E log E)

2. Prim MST
현재 트리에서 가장 가까운 정점 추가
우선순위 큐 사용, O(E log V)

3. Dijkstra 최단 경로
현재 가장 가까운 미방문 정점 처리
O(V^2) 또는 O(E log V) (힙 사용)

4. 분할 가능 배낭
단가(가치/무게) 순 정렬 → 높은 것부터
물건을 쪼갤 수 있으면 그리디 OK!

5. 작업 스케줄링
마감이 빠른 순 / 소요시간 짧은 순

[심화] 그리디 정당성 증명 — 교환 논법

[심화] 교환 논법 (Exchange Argument) 그리디가 최적임을 증명하는 가장 일반적인 방법:

단계:
1. 최적해 OPT가 있다고 가정
2. OPT와 그리디 해 G가 다른 첫 번째 지점 찾기
3. OPT에서 해당 선택을 G의 선택으로 교환해도
해가 나빠지지 않음을 보임
4. 반복하면 OPT → G로 변환 가능
5. 따라서 G도 최적!

예: 활동 선택
OPT에서 첫 활동이 a1, G에서 첫 활동이 g1(가장 빨리 끝남)
g1이 a1보다 일찍 끝나므로 a1을 g1로 교환해도
나머지 활동에 더 많은 시간이 남음 → 최적 유지!
[활동] 그리디 판별 연습 다음 문제에 그리디를 써도 되는지 판단해 보세요:

1. 회의실 배정: 최대한 많은 회의 잡기
→ O! 끝시간 순 정렬 + 그리디

2. 동전 {1, 5, 12}로 16원
→ X! 그리디: 12+1+1+1+1=5개, 최적: 5+5+5+1=4개

3. 짐을 박스에 나누기 (bin packing)
→ X! NP-hard 문제. 그리디는 근사만 가능

4. 체육복 빌려주기
→ O! 번호 순서대로 빌려주면 최적

핵심: "지금 선택이 미래 선택에 영향?"
영향 없으면 그리디, 있으면 DP!
