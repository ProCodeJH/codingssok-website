집합이 뭘까?

[핵심] 집합이란? 집합(Set) = 조건을 만족하는 것들의 모임

표기: A = {1, 3, 5, 7, 9} (중괄호 사용)

규칙:
1. 순서 없음: {1,2,3} = {3,1,2}
2. 중복 없음: {1,1,2} = {1,2}
3. 원소: 집합에 속한 각 것. 1 ∈ A (1은 A의 원소)

집합을 표현하는 방법은 2가지가 있어요.

[핵심] 부분집합 A의 모든 원소가 B에도 있으면: A ⊂ B (A는 B의 부분집합)

벤 다이어그램

집합의 관계를 그림으로 나타낸 것이 벤 다이어그램이에요.

[팁] 벤 다이어그램 읽기 • 왼쪽 원만 = A - B (차집합)
• 겹치는 부분 = A ∩ B (교집합)
• 오른쪽 원만 = B - A
• 두 원 전체 = A ∪ B (합집합)
• 두 원 밖 = (A ∪ B)ᶜ (여집합)
영역조건원소의미
A만 (왼쪽)A에 있고 B에 없음{1, 2}A - B
겹침 (가운데)A에도 B에도 있음{3, 4}A ∩ B
B만 (오른쪽)B에 있고 A에 없음{5, 6}B - A
바깥A에도 B에도 없음{7, 8}(A ∪ B)ᶜ

3// 합집합: A ∪ B
4int setUnion(int A[], int nA, int B[], int nB, int result[]) {
5 int n = 0;
6 // A의 모든 원소 추가
7 for (int i = 0; i < nA; i++) result[n++] = A[i];
8 // B에서 A에 없는 것만 추가
9 for (int i = 0; i < nB; i++) {
10 int found = 0;
11 for (int j = 0; j < nA; j++)
12 if (B[i] == A[j]) { found = 1; break; }
13 if (!found) result[n++] = B[i];
14 }
15 return n;
16}
17
18// 교집합: A ∩ B
19int setIntersect(int A[], int nA, int B[], int nB, int result[]) {
20 int n = 0;
21 for (int i = 0; i < nA; i++)
22 for (int j = 0; j < nB; j++)
23 if (A[i] == B[j]) { result[n++] = A[i]; break; }
24 return n;
25}

[심화] 포함-배제 원리

[심화] 포함-배제 원리 |A ∪ B| = |A| + |B| - |A ∩ B|

왜? A와 B를 더하면 겹치는 부분이 2번 세어지니까 1번 빼주는 것!

3개: |A ∪ B ∪ C| = |A|+|B|+|C| - |A∩B|-|A∩C|-|B∩C| + |A∩B∩C|

KOI 필기에서 '~이거나 ~인 학생 수' 문제로 자주 출제!

[심화 내용] 멱집합 멱집합(Power Set) = 어떤 집합의 모든 부분집합의 집합

[심화 내용] 카르테시안 곱

[심화 내용] 카르테시안 곱 (Cartesian Product) A × B = {(a, b) | a ∈ A, b ∈ B}

A = {1, 2}, B = {x, y, z}
A × B = {(1,x), (1,y), (1,z), (2,x), (2,y), (2,z)}

|A × B| = |A| × |B| = 2 × 3 = 6

활용: 데이터베이스 JOIN, 좌표계(ℝ² = ℝ × ℝ)

[심화 내용] 집합론의 기초

[심화 내용] 집합론 — 수학의 기초 칸토어 (Georg Cantor, 1874):
집합론의 창시자

핵심 개념:
- 기수(Cardinality): 집합의 크기
|{1,2,3}| = 3
- 멱집합(Power Set): 모든 부분집합의 집합
P({1,2}) = {{}, {1}, {2}, {1,2}}
|P(S)| = 2^|S|
- 가산 무한: 자연수와 1:1 대응 가능
정수, 유리수 = 가산 무한 (놀랍게도!)
- 비가산 무한: 실수 = 비가산 무한
대각선 논법으로 증명 (칸토어)

프로그래밍 연결:
- Set 자료구조 (HashSet, TreeSet)
- SQL의 UNION, INTERSECT, EXCEPT
- 비트마스크 = 유한 집합의 효율적 표현

[심화 내용] SQL = 집합 연산 관계형 데이터베이스의 기초 = 집합론!

집합 연산:
UNION = 합집합

예:

→ A ∪ B

SELECT * FROM A INTERSECT SELECT * FROM B

[심화 내용] 드 모르간 법칙 드 모르간 법칙 (De Morgan's Laws):

(A ∪ B)ᶜ = Aᶜ ∩ Bᶜ
(A ∩ B)ᶜ = Aᶜ ∪ Bᶜ

"합집합의 여집합 = 여집합들의 교집합"
"교집합의 여집합 = 여집합들의 합집합"

프로그래밍에서:
!(a || b) == (!a && !b)

용어설명
집합 (Set)조건을 만족하는 것들의 모임. 순서 없고 중복 없음.
원소 (Element)집합에 속한 각각의 것. ∈ 기호 사용.
부분집합한 집합의 원소가 모두 다른 집합에 포함될 때. A ⊂ B.
합집합A ∪ B. 둘 중 하나라도 속하는 원소의 집합. OR.
교집합A ∩ B. 둘 다 속하는 원소의 집합. AND.
차집합A - B. A에는 있고 B에는 없는 원소의 집합.
공집합∅. 원소가 하나도 없는 집합.
벤 다이어그램집합의 관계를 원으로 표현한 그림.
포함-배제|A∪B| = |A|+|B|-|A∩B|. 겹치는 부분 보정.
멱집합모든 부분집합의 집합. 크기 = 2ⁿ.
카르테시안 곱A×B. 모든 순서쌍 (a,b)의 집합.
비트마스크 집합정수의 각 비트를 원소 존재 여부로 사용하는 집합 표현. O(1) 연산.
멱집합어떤 집합의 모든 부분집합을 모은 집합. 크기 = 2^n.
가산 무한자연수와 1:1 대응 가능한 무한. 정수, 유리수가 해당.
비가산 무한자연수와 1:1 대응 불가능한 무한. 실수가 대표적.

TreeSet균형 트리 기반 집합. O(log n) 연산. 정렬 순서 유지.
