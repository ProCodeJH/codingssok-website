재귀가 뭘까?

[핵심] 재귀란? 재귀(Recursion) = 함수가 자기 자신을 호출하는 것

반드시 필요한 2가지:
1. 기저 조건(Base Case): 멈추는 조건. 없으면 무한 루프!
2. 재귀 호출(Recursive Call): 자기 자신을 호출하되 문제를 줄여서!

재귀 = '큰 문제를 작은 문제로 분해' (U23 분해와 연결!)

팩토리얼 — 재귀의 교과서

[핵심] 팩토리얼의 재귀 정의 5! = 5 × 4!
4! = 4 × 3!
3! = 3 × 2!
2! = 2 × 1!
1! = 1 ← 기저 조건!

일반화: n! = n × (n-1)! (n ≥ 2)
기저: 1! = 1 (또는 0! = 1)
Cfactorial_recursive.c
1#include <stdio.h>
2
3int factorial(int n) {
4 if (n <= 1) return 1; // 기저 조건!
5 return n * factorial(n - 1); // 재귀 호출!
6}
7
8int main(void) {
9 for (int i = 0; i <= 10; i++)
10 printf("%d! = %d\n", i, factorial(i));
11 return 0;
12}

[심화 내용] 꼬리 재귀 최적화

[심화 내용] 꼬리 재귀 최적화

[심화 내용] 꼬리 재귀 (Tail Recursion) 꼬리 재귀 = 재귀 호출이 함수의 마지막 동작

일반 재귀 (꼬리 아님):
int fact(int n) {
if (n <= 1) return 1;
return n * fact(n-1); // 곱셈이 마지막!
}

꼬리 재귀:
int fact_tail(int n, int acc) {
if (n <= 1) return acc;
return fact_tail(n-1, n*acc); // 재귀가 마지막!
}

꼬리 재귀 → 반복문으로 자동 변환!

[심화 내용] 재귀와 수학적 귀납법

[심화 내용] 재귀 = 수학적 귀납법 재귀 함수의 정확성 = 수학적 귀납법으로 증명!

귀납법 구조:
1. 기저 사례(Base Case): n=0 또는 n=1에서 참
2. 귀납 단계: n-1에서 참이면 n에서도 참

팩토리얼 증명:
fact(n) = n! 을 증명
Base: fact(1) = 1 = 1! ✓
귀납: fact(n-1) = (n-1)!이면
fact(n) = n * fact(n-1) = n * (n-1)! = n! ✓

하노이 탑 이동 횟수:
T(n) = 2T(n-1) + 1
Base: T(1) = 1 = 2^1 - 1 ✓
귀납: T(n) = 2(2^(n-1)-1)+1 = 2^n - 1 ✓

재귀적 사고 = 귀납적 사고!

[심화 내용] 재귀와 프랙탈

[심화 내용] 프랙탈 = 재귀적 도형 프랙탈: 부분이 전체와 닮은 도형 (자기 유사성)

코흐 눈송이:
삼각형의 각 변을 1/3로 나누고 가운데에 삼각형 추가
→ 무한 반복하면 눈송이 모양!
둘레 = 무한대, 면적 = 유한

C[C 코드] 메모이제이션 피보나치
1#include <stdio.h>
2
3long long memo[100] = {0};
4int call_count = 0;
5
6// 메모이제이션 없이
7long long fib_slow(int n) {
8 call_count++;
9 if (n <= 1) return n;
10 return fib_slow(n-1) + fib_slow(n-2);
11}
12
13// 메모이제이션 사용
14long long fib_fast(int n) {
15 call_count++;
16 if (n <= 1) return n;
17 if (memo[n]) return memo[n];
18 return memo[n] = fib_fast(n-1) + fib_fast(n-2);
19}
20
21int main() {
22 call_count = 0;
23 printf("fib(30) = %lld\n", fib_slow(30));
24 printf("호출 횟수 (메모이제이션 X): %d\n\n", call_count);
25
26 call_count = 0;
27 printf("fib(30) = %lld\n", fib_fast(30));
28 printf("호출 횟수 (메모이제이션 O): %d\n", call_count);
29 return 0;
30}

DPDynamic Programming. 중복 하위 문제를 저장해 효율적으로 해결.
꼬리 재귀재귀 호출이 함수의 마지막 동작인 재귀. 컴파일러가 반복문으로 최적화 가능.
스택 오버플로우재귀가 너무 깊어 호출 스택이 넘치는 오류. 보통 수만~수십만 단계에서 발생.
