그래프가 뭘까?

[핵심] 그래프란? 그래프(Graph) = 정점(Vertex)과 간선(Edge)으로 이루어진 자료구조

G = (V, E)
V = 정점(노드)의 집합
E = 간선(연결)의 집합

예: 도시(정점) + 도로(간선) = 도로 지도!

인접 행렬로 표현

[핵심] 인접 행렬 N×N 2차원 배열로 그래프 표현

adj[i][j] = 1이면 i→j 간선 있음
adj[i][j] = 0이면 없음

무방향: adj[i][j] = adj[j][i] (대칭!)

장점: 간선 확인 O(1)
단점: 메모리 O(N²)
단계간선 추가adj[u][v]adj[v][u]변경된 셀
1(0,1)adj[0][1]=1adj[1][0]=12개
2(0,2)adj[0][2]=1adj[2][0]=12개
3(1,3)adj[1][3]=1adj[3][1]=12개
4(2,3)adj[2][3]=1adj[3][2]=12개
5(3,4)adj[3][4]=1adj[4][3]=12개

1#include <stdio.h>
2#define N 5
3
4int adj[N][N] = {0};
5
6void addEdge(int u, int v) {
7 adj[u][v] = 1;
8 adj[v][u] = 1; // 무방향이면 양쪽!
9}
10
11int main(void) {
12 addEdge(0, 1);

23 }

25}

[심화] BFS — 너비 우선 탐색

[심화] BFS (Breadth-First Search) 가까운 것부터 탐색! 큐(Queue) 사용

1. 시작점을 큐에 넣고 방문 표시
2. 큐에서 꺼내기
3. 인접한 미방문 정점을 큐에 넣기
4. 큐가 빌 때까지 반복

특징: 최단 거리 보장! (간선 가중치 동일 시)
단계큐에서 꺼낸 정점방문 확인큐에 추가현재 큐 상태
초기—0 방문0[0]
101,2 미방문1, 2[1, 2]
213 미방문3[2, 3]
323 이미 방문없음[3]
434 미방문4[4]
54이웃 없음없음[] (종료)
Cbfs.c
1#include <stdio.h>
2#define N 5
3
4int adj[N][N], visited[N];
5
6void bfs(int start) {
7 int queue[N], front = 0, rear = 0;
8 visited[start] = 1;
9 queue[rear++] = start;
10
11 while (front < rear) {
12 int v = queue[front++];
13 printf("%d ", v);
14 for (int i = 0; i < N; i++) {
15 if (adj[v][i] && !visited[i]) {
16 visited[i] = 1;
17 queue[rear++] = i;
18 }
19 }
20 }
21}
22
23int main(void) {
24 adj[0][1]=adj[1][0]=1; adj[0][2]=adj[2][0]=1;
25 adj[1][3]=adj[3][1]=1; adj[2][3]=adj[3][2]=1;
26 adj[3][4]=adj[4][3]=1;
27 printf("BFS(0): "); bfs(0); printf("\n");
28 return 0;
29}

[심화] DFS — 깊이 우선 탐색

[심화] DFS (Depth-First Search) 깊이 끝까지 가본 후 돌아오기! 스택/재귀 사용

1. 시작점 방문 표시
2. 인접한 미방문 정점으로 재귀 호출
3. 더 이상 갈 곳 없으면 돌아옴(백트래킹)

특징: 경로 탐색, 사이클 감지에 유리
단계현재 정점동작재귀 스택방문 완료 목록
10방문 → 이웃 1로 이동[0][0]
21방문 → 이웃 3으로 이동[0, 1][0, 1]
33방문 → 이웃 2로 이동[0, 1, 3][0, 1, 3]
42방문 → 이웃 모두 방문됨 → 돌아감[0, 1, 3, 2][0, 1, 3, 2]
53이웃 4로 이동[0, 1, 3][0, 1, 3, 2]
64방문 → 이웃 모두 방문됨 → 종료[0, 1, 3, 4][0, 1, 3, 2, 4]
Cdfs.c
1#include <stdio.h>
2#define N 5
3
4int adj[N][N], visited[N];
5
6void dfs(int v) {
7 visited[v] = 1;
8 printf("%d ", v);
9 for (int i = 0; i < N; i++)
10 if (adj[v][i] && !visited[i])
11 dfs(i); // 재귀!
12}
13
14int main(void) {
15 adj[0][1]=adj[1][0]=1; adj[0][2]=adj[2][0]=1;
16 adj[1][3]=adj[3][1]=1; adj[2][3]=adj[3][2]=1;
17 adj[3][4]=adj[4][3]=1;
18 printf("DFS(0): "); dfs(0); printf("\n");
19 return 0;
20}

[심화 내용] 그래프 알고리즘 모음

[심화 내용] 핵심 그래프 알고리즘 탐색:
- BFS: 최단 경로(무가중치) O(V+E)
- DFS: 연결 요소, 위상 정렬, 사이클 검출

최단 경로:
- Dijkstra: 양수 가중치 O(E log V)
- Bellman-Ford: 음수 간선 OK O(VE)
- Floyd-Warshall: 모든 쌍 O(V^3)

최소 신장 트리:
- Kruskal: 간선 정렬 + Union-Find O(E log E)
- Prim: 우선순위 큐 O(E log V)

기타:
- 위상 정렬: DAG 순서 결정
- 강결합 요소: Tarjan/Kosaraju
- 네트워크 플로우: 최대 유량

그래프는 CS에서 가장 많이 쓰이는 자료구조!
알고리즘목적시간 복잡도자료구조제한 조건
BFS최단 경로 (무가중치)O(V+E)큐가중치 동일
DFS경로 탐색, 사이클 검출O(V+E)스택/재귀없음
다익스트라최단 경로 (양수 가중치)O(E log V)우선순위 큐음수 가중치 불가

"몇 개의 그룹?" → BFS/DFS

2. 최단 경로
"A에서 B까지 최소 거리?" → BFS/다익스트라
예: 미로 탈출, 네비게이션

3. 사이클 검출
"순환이 있는가?" → DFS (back edge)
예: 교착 상태, 선수과목 체크

4. 위상 정렬
"순서대로 나열" → DFS/BFS(Kahn)
예: 빌드 순서, 수강 순서

5. 이분 그래프
"2색으로 칠할 수 있나?" → BFS/DFS
예: 매칭 문제, 팀 나누기

[심화 내용] 면접 그래프 문제 Q1. BFS vs DFS 차이?
BFS: 큐, 최단경로, 레벨순
DFS: 스택/재귀, 연결성, 사이클검출

Q2. 인접 행렬 vs 인접 리스트 언제 사용?
행렬: 밀집(E≈V^2), O(1) 간선확인
리스트: 희소(E≪V^2), 메모리 절약

Q3. 이분 그래프 판별 방법?
BFS로 2색 칠하기. 인접한 같은 색 → 이분 아님

Q4. 음수 가중치가 있으면?
다익스트라 불가 → 벨만포드 사용
음수 사이클이 있으면 최단경로 정의 불가!

2. SNS 팔로우
정점=사용자, 간선=팔로우 (방향!)
문제: A가 B에게 도달 가능한가?
→ BFS/DFS!

3. 수강 신청
정점=과목, 간선=선수과목 관계
문제: 어떤 순서로 들을까?
→ 위상 정렬!

용어설명
그래프정점(V)과 간선(E)으로 이루어진 자료구조. G=(V,E).
정점Vertex/Node. 데이터가 담긴 점.
간선Edge. 정점 사이의 연결.
차수Degree. 한 정점에 연결된 간선 수.
인접 행렬N×N 배열로 그래프 표현. adj[i][j]=1이면 연결.
인접 리스트각 정점의 연결 목록으로 표현. 메모리 효율적.
BFS너비 우선 탐색. 큐 사용. 최단 거리 보장.
DFS깊이 우선 탐색. 스택/재귀. 경로, 사이클 탐지.
DAGDirected Acyclic Graph. 방향 비순환 그래프.
다익스트라가중치 그래프의 최단 경로. 음수 불가.
위상 정렬DAG에서 의존 순서 결정. 선수과목 등.
인접 행렬V*V 2차원 배열로 그래프 표현. adj[i][j]=1이면 i→j 간선 존재. 공간 O(V^2).
인접 리스트각 정점의 이웃 목록으로 그래프 표현. 공간 O(V+E). 희소 그래프에 유리.
BFSBreadth-First Search. 너비 우선 탐색. 큐 사용. 최단 경로(무가중치).
DFSDepth-First Search. 깊이 우선 탐색. 스택/재귀 사용.
Dijkstra양수 가중치 그래프의 최단 경로 알고리즘. O(E log V).
PageRank웹 그래프에서 페이지 중요도를 계산하는 알고리즘. Google 검색의 핵심.
위상 정렬DAG(비순환 방향 그래프)에서 선후 관계를 만족하는 순서 결정.
