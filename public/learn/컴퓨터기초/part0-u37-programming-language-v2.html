사람의 말 v 기계의 말

사람은 한국어, 영어, 일본어 같은 자연어를 써요. 컴퓨터는 0과 1로만 이루어진 기계어를 써요. 서로 말이 완전히 다르죠. 외국인과 대화할 때 통역사가 필요한 것처럼, 컴퓨터와 대화하려면 프로그래밍 언어라는 통역사가 필요해요.

[핵심] 프로그래밍 언어란? 프로그래밍 언어 = 사람의 생각을 컴퓨터가 알아들을 수 있게 번역해주는 도구
사람이 쓰기 편한 형태로 명령을 작성하면, 컴퓨터가 이해하는 기계어로 바꿔줘요.

저급 언어 v 고급 언어

프로그래밍 언어에도 등급이 있어요. 컴퓨터에 가까운 언어를 저급 언어, 사람에 가까운 언어를 고급 언어라고 해요. '저급'이 나쁘다는 뜻이 아니에요! 기계 수준(low level)이라는 뜻이에요.

[핵심] 저급 언어 기계어 — 0과 1로만 이루어진 언어. 컴퓨터가 바로 이해하지만 사람은 읽기 거의 불가능.
예: 10110000 01100001

어셈블리어 — 기계어를 사람이 조금이나마 읽을 수 있게 약자로 바꾼 언어.
예: MOV AL, 61h
[핵심] 고급 언어 C, Python, Java, JavaScript 등 — 영어 단어처럼 사람이 읽기 쉬운 언어.
예: printf("Hello!"); ← 이게 뭘 하는지 대충 느낌이 오죠?

고급 언어로 쓴 코드는 컴파일러나 인터프리터가 기계어로 번역해줘요.
하고 싶은 일기계어어셈블리어C (고급 언어)

3세대: 고급 언어 (1960s~)

4세대: 도메인 특화 (1970s~)
SQL (데이터베이스), MATLAB (수학), R (통계)

[심화 내용] 프로그래밍 언어의 패러다임

[심화 내용] 4대 프로그래밍 패러다임 1. 명령형 (Imperative)
"이렇게 해라" — 순서대로 명령
C, Pascal, Assembly
int sum = 0; for(...) sum += i;

2. 객체지향 (Object-Oriented)
"객체가 상호작용" — 데이터+메서드 캡슐화
Java, C++, Python, C#
class Dog { void bark(); }

3. 함수형 (Functional)
"함수의 조합" — 부작용 없는 순수 함수
Haskell, Lisp, Erlang, (부분적) JS/Python
map(lambda x: x*2, [1,2,3])

4. 논리형 (Logic)
"규칙과 사실" — 추론 엔진이 답을 찾음
Prolog
parent(tom, bob). ?- parent(tom, X).
패러다임코드 스타일핵심 아이디어
절차적 (C)for(i=1;i<=10;i++) sum+=i;순서대로 명령 실행
객체지향 (Java)Calculator c = new Calculator(); c.sum(10);객체에게 일을 시킴
함수형 (Haskell)sum [1..10]함수 조합으로 결과 도출
선언적 (SQL)SELECT SUM(n) FROM numbers WHERE n<=10원하는 결과만 선언

[심화 내용] 언어의 타입 시스템

[심화 내용] 타입 시스템 비교 정적 타입 (Static Typing)
컴파일 시 타입 체크. 에러 조기 발견.
C, Java, C++, Rust, Go, TypeScript
int x = 10; // 타입 고정

동적 타입 (Dynamic Typing)
실행 시 타입 결정. 유연하지만 런타임 에러 위험.
Python, JavaScript, Ruby, PHP
x = 10; x = "hello"; // OK

강타입 (Strong Typing)
암묵적 변환 최소. "1" + 2 → 에러!
Python, Rust, Haskell

약타입 (Weak Typing)
암묵적 변환 허용. "1" + 2 → "12" or 3
JavaScript, C, PHP

C는 정적+약타입. 빠르지만 위험할 수 있어요!

[심화 내용] 좋은 언어의 조건 1. 간결성 (Simplicity)
배우기 쉽고 읽기 쉬움
Python: x = [i**2 for i in range(10)]

2. 안전성 (Safety)
버그를 예방하는 타입 시스템
Rust: 컴파일 시 메모리 안전 보장

3. 성능 (Performance)
실행 속도와 메모리 효율
C: 기계에 가장 가까운 고급 언어

4. 표현력 (Expressiveness)
적은 코드로 많은 것을 표현
Haskell: quicksort xs = ... (3줄!)

5. 이식성 (Portability)
다양한 플랫폼에서 실행
Java: JVM 위에서 어디서든 실행

모든 조건을 만족하는 완벽한 언어는 없다!
→ 용도에 맞는 언어를 선택하는 것이 핵심

[심화 내용] 언어 구현 기술

[심화 내용] 언어 실행 방식 1. 순수 컴파일
소스 → 기계어. C, C++, Rust, Go
장점: 최고 성능, 배포 간편
단점: 플랫폼별 컴파일 필요

2. 순수 인터프리트
한 줄씩 읽고 실행. 초기 BASIC, Bash
장점: 즉시 실행, 디버깅 쉬움
단점: 느림

3. 바이트코드 + VM
소스 → 바이트코드 → VM 실행. Java, C#, Python
장점: 크로스플랫폼
단점: VM 필요

4. JIT (Just-In-Time)
실행 중 기계어 변환. V8(JS), HotSpot(Java)
장점: 인터프리터 편의 + 컴파일러 성능
단점: 시작 시 느림 (워밍업)

5. Transpile
소스 → 다른 언어 소스. TypeScript→JS, Kotlin→Java
[심화 내용] 면접 문제 Q1. C와 Python의 가장 큰 차이?
A: 타입(정적 vs 동적), 실행(컴파일 vs 인터프리트), 메모리 관리(수동 vs GC), 성능(C가 ~100배 빠름)

Q2. "garbage collection"이란?
A: 사용하지 않는 메모리를 자동 회수. Java/Python/Go 지원, C/C++는 수동 관리(malloc/free)

Q3. 왜 아직도 C를 배우나?
A: 컴퓨터 동작 원리 이해(메모리, 포인터), OS/드라이버 개발, 임베디드, 성능 크리티컬 분야

Q4. 새 언어를 만든다면 어떤 특징?
A: (열린 질문) 타입 안전성, 간결한 문법, 좋은 에러 메시지, 빠른 컴파일...

[심화 내용] 형식 의미론 프로그래밍 언어의 의미를 수학적으로 정의하는 방법

1. 조작적 의미론 (Operational Semantics)
프로그램 실행 과정을 규칙으로 정의
"이 코드를 실행하면 상태가 이렇게 변한다"

2. 표시적 의미론 (Denotational Semantics)
프로그램을 수학 함수로 대응
"이 코드의 의미는 이 함수이다"

3. 공리적 의미론 (Axiomatic Semantics)
사전/사후 조건으로 정의 (호어 논리)
{P} S {Q}: P가 참이면 S 실행 후 Q가 참

이 이론들이 컴파일러 최적화, 프로그램 검증,

컴파일러고급 언어로 쓴 코드를 기계어로 한꺼번에 번역해주는 프로그램.

정적 타입컴파일 시 타입을 검사하는 방식. C, Java 등. 에러 조기 발견.
동적 타입실행 시 타입이 결정되는 방식. Python, JS 등. 유연하지만 런타임 에러 위험.
JITJust-In-Time 컴파일. 실행 중 자주 쓰는 코드를 기계어로 변환. Java, JS 엔진.
도메인 특화 언어특정 분야에 최적화된 언어. SQL(DB), MATLAB(수학), HTML(웹).
