C언어의 탄생

데니스 리치는 유닉스(UNIX)라는 운영체제를 만들기 위해 C언어를 개발했어요. 원래 어셈블리어로 운영체제를 만들고 있었는데, 너무 복잡하고 힘들어서 좀 더 편한 언어가 필요했거든요. 그래서 B언어를 개선해서 C언어를 만들었어요. (A → B → C 순서랍니다!)

연도사건의미영향
1969UNIX 개발 시작어셈블리어로 작성이식성 없음

1972C 언어 탄생B에 타입 추가UNIX를 C로 재작성

C언어는 어디에 쓰일까?

C언어는 지금도 엄청나게 많은 곳에서 쓰여요. 여러분이 매일 쓰는 것들 중에도 C로 만든 게 아주 많아요.

[핵심] C언어가 쓰이는 곳 1. 운영체제 — Linux, Windows 핵심, macOS 기반이 전부 C로 만들어져요.
2. 게임 엔진 — 언리얼 엔진, 유니티 핵심이 C/C++로 만들어져요.
3. 임베디드 시스템 — 엘리베이터, 자동차, 세탁기 속 칩이 C로 동작해요.
4. 프로그래밍 언어 — Python, Java, Ruby도 C로 만들어졌어요!

모든 언어의 아버지

C언어를 배우면 다른 언어가 훨씬 쉬워져요. 왜냐하면 C++, Java, C#, JavaScript, Python 등 현대 언어 대부분이 C의 문법을 기반으로 만들어졌거든요. 중괄호 { }, 세미콜론 ;, if-else, for 같은 문법이 전부 C에서 시작됐어요.

C는 왜 빠를까?

C언어는 고급 언어 중에서 가장 빠른 언어 중 하나예요. Python은 코드를 한 줄씩 번역하면서 실행하지만, C는 전체 코드를 미리 번역(컴파일)한 다음 실행해요. 미리 번역해놓으니까 실행할 때 훨씬 빠르죠!

[심화 내용] C와 하드웨어의 관계

[심화 내용] C가 하드웨어에 가까운 이유 1. 포인터 = 메모리 주소
int *p = (int*)0x1000; → 물리 주소 직접 접근
임베디드에서 하드웨어 레지스터 제어

2. sizeof = 실제 바이트 수
sizeof(int) = 4 → CPU 레지스터 크기와 대응
메모리 레이아웃을 정확히 제어

3. 비트 연산 = CPU 명령어 1:1
a & b → AND 명령어, a >> 3 → SHR 명령어
하드웨어 레지스터 비트 조작에 필수

4. 인라인 어셈블리
__asm__("mov eax, 1"); → C 안에 어셈블리!
극한 최적화, 특수 명령어 사용 가능

27 port |= (1 << LED_PIN); // 5번 비트 SET
28 printf("LED ON: 0x%02X (0b%08b에서 bit5=1)\n", port, port);

[심화] C 관련 면접 문제

[심화] C 면접 필수 질문 Q1. C와 C++의 차이?
A: C++는 C에 클래스, 템플릿, 예외, STL 추가. C는 절차적, C++는 멀티패러다임.

Q2. C에서 메모리 관리 방법?
A: malloc/calloc/realloc으로 할당, free로 해제. 스택(자동), 힙(수동), 데이터(전역/정적).

Q3. volatile 키워드의 용도?
A: 컴파일러 최적화 방지. 하드웨어 레지스터, 멀티스레드 공유 변수에 사용.

Q4. C가 "이식성 있는 어셈블리어"라고 불리는 이유?
A: 하드웨어에 가까우면서도 다양한 CPU에서 컴파일 가능. 어셈블리의 추상화.
