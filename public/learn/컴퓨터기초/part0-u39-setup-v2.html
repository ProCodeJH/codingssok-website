2단계: MGW() 설치

VS Code는 코드를 쓰는 도구예요. 하지만 코드를 실행 파일로 바꿔주는 도구(컴파일러)는 따로 설치해야 해요. C언어의 컴파일러는 gcc이고, Windows에서는 MinGW를 통해 설치해요.

[설치] MinGW(gcc) 다운로드 1. 인터넷에서 MSYS2를 검색하거나, msys2.org에 접속해요.
2. 설치 파일을 다운로드하고 실행해요.
3. 설치 완료 후 MSYS2 터미널을 열어요.
4. 다음 명령어를 입력해요:
pacman -S mingw-w64-x86_64-gcc
5. 'Y'를 눌러 설치를 진행해요.

3단계: 환경 변수 설정

gcc를 설치했지만, 아직 어디서든 사용할 수는 없어요. 컴퓨터에게 'gcc가 여기 있어!'라고 알려줘야 해요. 이걸 환경 변수 설정이라고 해요.

[설정] 환경 변수(PATH) 등록하기 1. Windows 검색에서 '환경 변수'를 검색해요.
2. '시스템 환경 변수 편집'을 클릭해요.
3. '환경 변수' 버튼을 누르세요.
4. 'Path' 항목을 선택하고 '편집'을 누르세요.
5. '새로 만들기'를 누르고 MinGW의 bin 폴더 경로를 추가해요.
예: C:\msys64\mingw64\bin
6. 확인을 누르고, 새 터미널을 열어 확인:

[심화] Makefile 예제 프로젝트 구조:
main.c, utils.c, utils.h

Makefile:
CC = gcc
CFLAGS = -Wall -g

main: main.o utils.o
$(CC) -o main main.o utils.o

main.o: main.c utils.h
$(CC) $(CFLAGS) -c main.c

utils.o: utils.c utils.h
$(CC) $(CFLAGS) -c utils.c

clean:
rm -f *.o main

사용:
make → 컴파일
make clean → 정리

[심화] 컴파일 옵션 가이드

[심화] GCC 주요 옵션 기본 컴파일:
gcc main.c -o main

경고 옵션:
-Wall: 주요 경고 모두 켜기 (필수!)
-Wextra: 추가 경고
-Werror: 경고를 에러로 취급
-pedantic: 표준 엄격 준수

최적화:
-O0: 최적화 없음 (디버깅용)
-O1: 기본 최적화
-O2: 권장 최적화 (성능+크기 균형)
-O3: 최대 최적화 (가끔 부작용)
-Os: 크기 최적화 (임베디드)

디버깅:
-g: 디버그 정보 포함 (GDB용)
-fsanitize=address: 메모리 오류 검출
-fsanitize=undefined: 정의되지 않은 동작 검출

추천 조합:
개발: gcc -Wall -Wextra -g -fsanitize=address
릴리스: gcc -Wall -O2
옵션분류효과사용 시점
-Wall경고주요 경고를 모두 표시합니다항상 사용합니다
-Wextra경고추가적인 경고를 표시합니다개발 중 사용합니다
-g디버깅디버그 정보를 실행 파일에 포함합니다디버깅할 때 사용합니다
-O0최적화최적화를 하지 않습니다 (기본값)디버깅할 때 사용합니다
-O2최적화성능과 크기의 균형 잡힌 최적화입니다배포할 때 사용합니다
-fsanitize=address디버깅메모리 오류를 실행 중 검출합니다메모리 버그를 찾을 때 사용합니다

[심화 내용] 컴파일 과정 상세

[심화 내용] C 컴파일의 4단계 1. 전처리 (Preprocessing)
gcc -E main.c -o main.i
#include → 헤더 파일 삽입
#define → 매크로 치환
#ifdef → 조건부 컴파일

2. 컴파일 (Compilation)
gcc -S main.i -o main.s
C 코드 → 어셈블리어
구문 분석, 의미 분석, 최적화

3. 어셈블 (Assembly)
gcc -c main.s -o main.o
어셈블리어 → 기계어 (오브젝트 파일)
아직 실행 불가 (주소 미결정)

4. 링킹 (Linking)
gcc main.o -o main
오브젝트 파일들 + 라이브러리 연결
printf의 실제 코드를 libc에서 가져옴
실행 파일 완성!

C[C 코드] 여러 파일 프로젝트 예제
1// === utils.h ===
2#ifndef UTILS_H
3#define UTILS_H
4int max(int a, int b);
5int min(int a, int b);
6void swap(int *a, int *b);
7#endif
8
9// === utils.c ===
10#include "utils.h"
11int max(int a, int b) { return (a > b) ? a : b; }
12int min(int a, int b) { return (a < b) ? a : b; }
13void swap(int *a, int *b) { int t = *a; *a = *b; *b = t; }
14
15// === main.c ===
16#include <stdio.h>
17#include "utils.h"
18
19int main() {
20 int x = 10, y = 20;
21 printf("max(%d,%d) = %d\n", x, y, max(x, y));
22 printf("min(%d,%d) = %d\n", x, y, min(x, y));
23
24 printf("swap 전: x=%d, y=%d\n", x, y);
25 swap(&x, &y);
26 printf("swap 후: x=%d, y=%d\n", x, y);
27 return 0;
28}
29
30// 컴파일: gcc main.c utils.c -o main
31// 또는 Makefile 사용

[심화] 초보자 흔한 설정 실수 1. PATH 미설정
gcc를 설치했는데 "명령을 찾을 수 없습니다"
→ 환경 변수 PATH에 gcc 경로 추가!

2. 확장자 실수
main.c.txt로 저장 → 컴파일 안 됨
→ "파일 확장명 표시" 켜기

3. 한글 경로
C:\사용자\바탕화면\ → 일부 도구에서 에러
→ 영문 경로 사용 권장

4. 32비트 vs 64비트
컴파일러와 OS 비트 불일치
→ 64비트 OS면 64비트 컴파일러!

5. 헤더 파일 경로
#include "myheader.h" 찾기 실패
→ -I 옵션으로 경로 지정
[심화 내용] CI/CD와 자동화 CI/CD (Continuous Integration/Deployment)
코드 변경 → 자동 빌드 → 자동 테스트 → 자동 배포

GitHub Actions 예제 (.github/workflows/c.yml):
name: C Build
on: push
jobs:
build:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v3
- run: gcc -Wall -Werror main.c -o main
- run: ./main

용어설명
VS CodeVisual Studio Code. 마이크로소프트가 만든 무료 코드 편집기. 전 세계에서 가장 많이 쓰여요.
gccGNU Compiler Collection. C 코드를 기계어로 번역하는 컴파일러.
MinGWMinimalist GNU for Windows. Windows에서 gcc를 사용할 수 있게 해주는 도구 모음.
환경 변수 (PATH)컴퓨터가 프로그램을 찾을 경로 목록. PATH에 등록하면 어디서든 해당 프로그램을 실행할 수 있어요.
터미널텍스트로 컴퓨터에 명령을 내리는 창. VS Code 아래쪽에서 열 수 있어요.
IDEIntegrated Development Environment. 편집+컴파일+디버깅 통합 도구.
GCCGNU Compiler Collection. 무료 오픈소스 C/C++ 컴파일러. Linux 표준.
ClangLLVM 기반 C/C++ 컴파일러. macOS 기본. 에러 메시지가 친절!
MinGWMinimalist GNU for Windows. Windows에서 GCC 사용 가능하게 하는 도구.
WSLWindows Subsystem for Linux. Windows에서 Linux 환경을 실행.
Make빌드 자동화 도구. Makefile에 규칙 정의. 1976년부터 사용.
CMake크로스 플랫폼 빌드 시스템 생성기. 현대 C/C++ 프로젝트 표준.
GDBGNU Debugger. C/C++ 프로그램을 한 줄씩 실행하며 디버깅하는 도구.
