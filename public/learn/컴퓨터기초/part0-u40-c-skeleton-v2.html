#lu <.> — 도구 가져오기

#include <stdio.h>는 '도구를 가져와!'라는 뜻이에요. stdio.h는 Standard Input Output의 약자로, 화면에 글자를 출력하거나 키보드 입력을 받는 도구가 들어있어요. 요리를 시작하기 전에 도구함에서 칼, 도마, 냄비를 꺼내오는 것과 같아요.

(v) — 시작점

int main(void) { }는 프로그램의 시작점이에요. 컴퓨터는 프로그램을 실행할 때 항상 main 함수부터 시작해요. 연극에서 '공연 시작!'이라는 신호와 같아요. main이 없으면 컴퓨터는 어디서부터 시작해야 할지 몰라요.

구성 요소코드의미필수 여부

닫는 중괄호}함수 본문 끝필수

u 0; — 프로그램 끝

return 0;은 '프로그램이 잘 끝났어요!'라는 신호예요. 0은 '문제없이 성공했다'는 뜻이에요. 시험을 다 보고 '다 풀었습니다!' 하고 손을 드는 것과 같아요.

-1심각한 오류실패메모리가 부족할 때

세미콜론(;) — 문장 끝 마침표

한국어 문장 끝에 마침표(.)를 찍듯이, C언어 문장 끝에는 세미콜론(;)을 찍어요. 세미콜론을 빠뜨리면 컴퓨터가 '문장이 아직 안 끝났나?'하고 혼란스러워해요.

[심화 내용] 실행 파일의 구조

[심화 내용] ELF/PE 실행 파일 구조 C 코드가 컴파일되면 실행 파일이 생성돼요.

Linux: ELF (Executable and Linkable Format)
Windows: PE (Portable Executable)

주요 섹션:
1. .text — 기계어 코드 (실행할 명령어)
2. .data — 초기화된 전역 변수
3. .bss — 초기화되지 않은 전역 변수
4. .rodata — 읽기 전용 데이터 (문자열 리터럴)
5. 심볼 테이블 — 함수/변수 이름↔주소 매핑

int x = 10; → .data 섹션
int y; → .bss 섹션
"Hello" → .rodata 섹션
main() { ... } → .text 섹션

[심화 내용] 링커와 심볼 해석

[심화 내용] 링커가 하는 일 링커 (Linker): 여러 오브젝트 파일을 하나로 합치는 도구

1. 심볼 해석 (Symbol Resolution)
printf()를 호출하면 → libc에서 printf 주소 찾기
add()를 호출하면 → utils.o에서 add 주소 찾기

2. 재배치 (Relocation)
각 오브젝트 파일의 주소를 최종 위치로 조정
main.o의 코드 뒤에 utils.o 코드 배치

3. 정적 vs 동적 링킹
정적: 라이브러리 코드를 실행 파일에 복사
장점: 독립 실행, 단점: 파일 크기 큼
동적: 실행 시 공유 라이브러리(.so/.dll) 로드
장점: 파일 작음, 단점: 라이브러리 필요

gcc -static main.c → 정적 링킹
gcc main.c → 기본 동적 링킹

[심화] main 함수의 매개변수

[심화] argc와 argv main 함수는 두 가지 형태가 가능:

형태 1: int main(void)
매개변수 없음. 가장 간단.

형태 2: int main(int argc, char *argv[])
argc = 인자 개수 (Argument Count)
argv = 인자 문자열 배열 (Argument Vector)

./main hello world 실행하면:
argc = 3
argv[0] = "./main"
argv[1] = "hello"
argv[2] = "world"

활용:
- 파일 이름을 인자로 받기
- 옵션(-v, --help) 처리
- 배치 처리 자동화
실행 명령어argc 값argv[0]argv[1]argv[2]
./main1"./main"(없음)(없음)
./main hello2"./main""hello"(없음)
./main hello world3"./main""hello""world"
./main --help2"./main""--help"(없음)
./main -n 53"./main""-n""5"

[심화] static과 extern 키워드

[심화] 변수/함수의 가시성 제어 static (정적):
1. 전역에 사용: 이 파일에서만 접근 가능
static int count = 0; // 다른 파일에서 안 보임

2. 함수 안에 사용: 함수 호출 간 값 유지
void f() { static int n=0; n++; printf("%d\n",n); }
f(); f(); f(); → 1, 2, 3

extern (외부):
다른 파일에 정의된 변수/함수를 가져옴
extern int global_count; // "어딘가에 있을 거야"

조합:
- 아무것도 안 붙인 전역 = 공개
- static 전역 = 파일 내부 전용 (캡슐화!)
- extern = 다른 파일에서 가져오기

static을 적극 사용하면 이름 충돌 방지!

[심화 내용] 면접 문제 Q1. main의 반환값 0은 무슨 의미?
A: EXIT_SUCCESS. OS에게 정상 종료를 알림. 셸에서 $?로 확인 가능.

Q2. static 전역 변수 vs 일반 전역 변수?
A: static은 해당 파일(.c)에서만 접근. 일반은 extern으로 다른 파일에서도 접근 가능.

Q3. 함수 프로토타입이 필요한 이유?
A: 컴파일러가 함수 호출 시 매개변수 타입/개수를 검증. 없으면 implicit declaration 경고.

Q4. #include 와 #include "stdio.h"의 차이?
A: <>는 시스템 헤더 경로 검색. ""는 현재 디렉토리 먼저, 없으면 시스템 경로.
