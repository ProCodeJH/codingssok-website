UNIT 41 컴파일이란?

컴파일이 뭐야?

컴파일(compile)은 사람이 쓴 소스 코드(.c 파일)를 컴퓨터가 이해하는 기계어로 한꺼번에 번역하는 과정이에요. 이 번역을 해주는 프로그램이 컴파일러이고, 우리는 gcc라는 컴파일러를 사용해요.

구성 요소역할예시
gcc컴파일러 프로그램 이름gcc
hello.c번역할 소스 코드 파일.c 확장자
-o출력 파일 이름을 지정하는 옵션output의 약자
hello생성될 실행 파일 이름hello.exe (Windows)
[핵심] 컴파일 실습 명령어 터미널에서 다음 명령어를 입력해요:

gcc hello.c -o hello

• gcc — 컴파일러 이름
• hello.c — 번역할 소스 코드 파일
• -o hello — 결과 파일 이름을 hello로 지정

실행하려면:
./hello (Mac/Linux) 또는 hello.exe (Windows)

[핵심] 컴파일 vs 인터프리터 비교 컴파일 방식 (C, C++, Java)
• 코드 전체를 한꺼번에 번역 → 실행 파일 생성
• 실행 속도가 빠름
• 번역 중 에러가 나면 실행 파일이 안 만들어짐

인터프리터 방식 (Python, JavaScript)
• 코드를 한 줄씩 번역하며 실행
• 실행 속도가 느림
• 에러가 난 줄까지는 실행됨

에러의 3가지 종류

코드를 쓰다 보면 반드시 에러를 만나게 돼요. 에러는 3종류가 있어요. 각각 원인과 해결법이 달라요.

[심화] 컴파일 과정 완전 분석

[심화] 컴파일 4단계 상세 소스 코드 → 실행 파일까지:

1단계: 전처리 (Preprocessing)
gcc -E main.c -o main.i
- #include → 헤더 파일 내용 삽입
- #define → 매크로 치환
- 주석 제거
- #ifdef 조건부 컴파일
결과: 순수 C 코드 (수천~수만 줄!)

2단계: 컴파일 (Compilation)
gcc -S main.i -o main.s
- 어휘 분석 (토큰화)
- 구문 분석 (파싱, AST 생성)
- 의미 분석 (타입 체크)
- 중간 코드 생성
- 최적화
- 어셈블리 코드 생성

3단계: 어셈블 (Assembly)
gcc -c main.s -o main.o
- 어셈블리 → 기계어 (바이너리)
- 오브젝트 파일 생성
- 아직 주소 미결정 (재배치 정보 포함)

4단계: 링킹 (Linking)
gcc main.o -o main
- 여러 .o 파일 결합
- 라이브러리 함수 연결 (printf 등)
- 최종 주소 결정
- 실행 파일 생성!
단계명령어입력 파일출력 파일설명
전처리gcc -E main.c -o main.imain.c.i 파일매크로 치환과 헤더 삽입을 처리합니다
컴파일gcc -S main.i -o main.smain.i.s 파일어셈블리 코드로 변환합니다
어셈블gcc -c main.s -o main.omain.s.o 파일기계어 바이너리로 변환합니다
링킹gcc main.o -o mainmain.o실행 파일라이브러리를 연결하여 최종 실행 파일을 생성합니다

[심화 내용] 컴파일러 내부 구조

[심화 내용] 컴파일러의 6단계 Front-end (언어 의존):

1. 어휘 분석 (Lexical Analysis)
소스 코드 → 토큰 스트림
int main() → [INT] [IDENT:main] [LPAREN] [RPAREN]
도구: Lex, Flex

2. 구문 분석 (Syntax Analysis)
토큰 → 추상 구문 트리 (AST)
a + b * c → +(a, *(b, c))
도구: Yacc, Bison

3. 의미 분석 (Semantic Analysis)
타입 체크, 선언 확인
int x = "hello"; → 에러!

Middle-end (최적화):

4. 중간 코드 생성 + 최적화
상수 접기: 3+5 → 8
죽은 코드 제거
루프 최적화

Back-end (기계 의존):

5. 코드 생성
중간 코드 → 타겟 기계어
x86, ARM, RISC-V 등

6. 레지스터 할당
변수를 CPU 레지스터에 매핑
NP-완전 문제! 휴리스틱 사용
소스 코드 조각토큰 종류토큰 값
int키워드INT
main식별자IDENT: main
(구분 기호LPAREN
)구분 기호RPAREN
{구분 기호LBRACE
return키워드RETURN
0정수 리터럴NUM: 0
;구분 기호SEMICOLON
}구분 기호RBRACE

[심화] 컴파일 vs 인터프리트 상세 비교

[심화] 같은 코드, 다른 실행 방식 printf("Hello"); 실행 경로:

C (컴파일):
1. gcc: C → 어셈블리 → 기계어 (.exe)
2. OS가 .exe 로드
3. CPU가 기계어 직접 실행
4. 시스템 콜로 화면에 출력
속도: ★★★★★ (가장 빠름)

Java (바이트코드+JIT):
1. javac: Java → 바이트코드 (.class)
2. JVM 로드
3. 인터프리트 시작, 자주 쓰는 코드는 JIT 컴파일
4. 점점 빨라짐 (워밍업)
속도: ★★★★ (JIT 후 C에 근접)

Python (인터프리트):
1. CPython: 소스 → 바이트코드 (.pyc)
2. PVM(Python VM)이 바이트코드 해석
3. 한 명령씩 실행
속도: ★★ (C의 ~1/50)

[심화 내용] 크로스 컴파일과 타겟 아키텍처

[심화 내용] 크로스 컴파일 크로스 컴파일: 다른 플랫폼용 실행 파일 생성

예: PC(x86)에서 ARM용 코드 컴파일
arm-linux-gnueabihf-gcc main.c -o main_arm

왜 필요한가:
1. 라즈베리파이(ARM)에서 직접 컴파일 → 느림
2. PC에서 크로스 컴파일 → 빠름!
3. 임베디드 장치는 컴파일러 설치 불가

대표 타겟:
- x86_64: PC, 서버
- ARM: 스마트폰, 라즈베리파이
- RISC-V: 차세대 오픈소스 CPU
- MIPS: 라우터, 게임기(PSP)
- AVR: 아두이노

같은 C 코드가 다른 CPU에서 돌아가는 이유!
= C의 이식성 (Portability)
타겟 CPU사용 장치크로스 컴파일러 명령어
x86_64PC, 서버gcc main.c -o main (기본 컴파일)
ARM스마트폰, 라즈베리파이arm-linux-gnueabihf-gcc main.c -o main_arm
RISC-V차세대 오픈소스 CPUriscv64-linux-gnu-gcc main.c -o main_riscv
MIPS라우터, PSP 게임기mips-linux-gnu-gcc main.c -o main_mips
AVR아두이노avr-gcc main.c -o main_avr

[심화] 링커 에러와 해결법

[심화] 흔한 링커 에러 1. undefined reference to '함수명'
함수를 선언했지만 정의가 없음!
해결: .c 파일을 같이 컴파일
gcc main.c utils.c -o main

2. multiple definition of '변수명'
같은 전역 변수가 여러 파일에 정의됨
해결: 하나만 정의, 나머지는 extern 선언

3. undefined reference to 'sqrt'
math.h 함수인데 -lm 안 붙임!
해결: gcc main.c -lm

4. cannot find -l라이브러리
해당 라이브러리 설치 안 됨
해결: apt install lib라이브러리-dev

컴파일 에러 vs 링커 에러:
컴파일 에러: 문법, 타입 문제 (한 파일)
링커 에러: 연결, 정의 문제 (파일 간)

[심화 내용] 컴파일러 만들기 — 미니 프로젝트

[심화 내용] 간단한 수식 계산기 컴파일러 목표: "3+5*2" → 13 계산하는 미니 컴파일러

1단계: 토크나이저
"3+5*2" → [3] [+] [5] [*] [2]

2단계: 파서 (재귀 하강)
expr = term ((+|-) term)*
term = factor ((*|/) factor)*
factor = number | (expr)

AST: +(3, *(5, 2))

3단계: 평가
*(5,2) = 10
+(3, 10) = 13

이게 컴파일러의 핵심 원리!
GCC도 이것의 수만 배 복잡한 버전일 뿐

대학 컴파일러 수업 첫 과제가 바로 이것!
C[C 코드] 미니 수식 파서
1#include <stdio.h>
2#include <ctype.h>
3
4const char *p;
5
6int number() {
7 int n = 0;
8 while (isdigit(*p)) n = n*10 + (*p++ - '0');
9 return n;
10}
11
12int factor() {
13 if (*p == '(') { p++; int r = expr(); p++; return r; }
14 return number();
15}
16
17int term() {
18 int r = factor();
19 while (*p=='*'||*p=='/') {
20 char op = *p++;
21 int f = factor();
22 r = (op=='*') ? r*f : r/f;
23 }
24 return r;
25}
26
27int expr() {
28 int r = term();
29 while (*p=='+'||*p=='-') {
30 char op = *p++;
31 int t = term();
32 r = (op=='+') ? r+t : r-t;
33 }
34 return r;
35}
36

39 for (int i = 0; i < 4; i++) {
40 p = tests[i];
41 printf("%s = %d\n", tests[i], expr());
42 }

44}

[심화 내용] 면접 문제 Q1. 전처리, 컴파일, 어셈블, 링킹의 차이?
A: 전처리=#매크로 처리, 컴파일=C→어셈블리, 어셈블=어셈블리→기계어, 링킹=여러 .o 결합+라이브러리 연결

Q2. -O2와 -O0의 차이?
A: -O0=최적화 없음(디버깅용), -O2=상수접기/인라이닝/루프최적화 등 적용(릴리스용)

Q3. 정적 링킹 vs 동적 링킹?
A: 정적=라이브러리 코드 복사(독립 실행, 큼), 동적=실행 시 .so/.dll 로드(작음, 의존성)

Q4. "undefined reference" 에러 원인?
A: 함수를 선언했지만 정의가 없거나, 해당 .c 파일 / 라이브러리를 링킹하지 않음
용어설명
컴파일소스 코드 전체를 기계어로 한꺼번에 번역하는 과정.
컴파일러컴파일을 해주는 프로그램. C언어는 gcc를 사용.
소스 코드프로그래머가 작성한 원본 코드 파일. C에서는 .c 확장자.
실행 파일컴파일 결과 만들어진 파일. Windows에서는 .exe 확장자.
인터프리터코드를 한 줄씩 번역하며 실행하는 프로그램. Python이 대표적.
컴파일 에러문법 실수로 번역 자체가 실패하는 에러.
런타임 에러번역은 됐지만 실행 중에 발생하는 에러.
논리 에러프로그램은 돌아가지만 결과가 틀린 에러. 가장 찾기 어려움.
이번 유닛 핵심 정리
1. 컴파일 = 소스코드(.c)를 기계어로 번역 → 실행파일(.exe) 생성
2. gcc hello.c -o hello로 컴파일, ./hello로 실행
3. 컴파일(C) = 전체 번역, 인터프리터(Python) = 한 줄씩 번역
4. 에러 3종류: 컴파일 에러(문법) / 런타임 에러(실행 중) / 논리 에러(결과 틀림)
