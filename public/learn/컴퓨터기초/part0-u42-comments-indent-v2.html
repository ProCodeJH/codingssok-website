UNIT 42 주석과 들여쓰기

주석이란?

주석(comment)은 코드 속에 쓰는 메모예요. 컴퓨터는 주석을 무시하고 건너뛰어요. 주석은 오직 사람을 위한 것이에요. 1주일 전에 내가 쓴 코드도 까먹을 수 있으니까, 주석을 남겨두면 나중에 '아, 이게 이런 뜻이었구나!' 하고 바로 이해할 수 있어요.

[핵심] 주석 쓰는 법 1. 한 줄 주석 — // 뒤에 쓰기
// 이것은 한 줄 주석이에요
printf("안녕!"); // 화면에 출력

2. 여러 줄 주석 — /* */ 사이에 쓰기
/* 이것은
여러 줄에 걸친
주석이에요 */

들여쓰기란?

들여쓰기(indentation)는 중괄호 { } 안의 코드를 오른쪽으로 밀어서 쓰는 것이에요. 보통 Tab 키 또는 Space 4칸으로 해요. 들여쓰기를 하면 '이 코드가 어디에 속하는지' 한눈에 보여요.

[주의] 나쁜 코드 예시 #include <stdio.h>
int main(void){
int a=10;int b=20;int c;
c=a+b;printf("%d",c);return 0;}

이 코드는 돌아가긴 하지만, 읽기가 너무 힘들어요! 변수 이름도 a, b, c라서 뭘 하는 코드인지 모르겠어요.
[핵심] 좋은 코드 예시 #include <stdio.h>

int main(void) {
int score1 = 10; // 첫 번째 점수
int score2 = 20; // 두 번째 점수
int total; // 합계

total = score1 + score2;
printf("%d", total);

return 0;
}

들여쓰기도 되어있고, 변수 이름도 의미가 있고, 주석도 있어요!

[심화 내용] 클린 코드 원칙

[심화 내용] Robert C. Martin의 클린 코드 "Clean Code" (2008)의 핵심 원칙:

1. 의미 있는 이름
int d; → int elapsed_days;
int t; → int total_score;
이름만 보고 용도를 알 수 있게!

2. 함수는 한 가지만
하나의 함수 = 하나의 역할
20줄 이하가 이상적

3. 주석 대신 코드로 표현
// 학생이 합격인지 확인
if (score >= 60) → if (is_passing(score))

4. 중복 제거 (DRY)
Don't Repeat Yourself
같은 코드 2번 쓰면 → 함수로!

5. 일관된 형식
들여쓰기, 중괄호, 네이밍 규칙 통일
팀 전체가 같은 스타일

6. 에러 처리
해피 패스만 코딩하지 말 것
예외 상황을 먼저 처리 (가드 절)

용어설명
주석 (comment)코드 속에 쓰는 메모. 컴퓨터는 무시하고 사람만 읽어요.
한 줄 주석// 뒤에 쓰는 주석. 그 줄 끝까지만 주석이에요.
여러 줄 주석/* */로 감싸는 주석. 여러 줄에 걸쳐 쓸 수 있어요.
들여쓰기 (indentation)코드를 오른쪽으로 밀어 쓰는 것. 코드의 구조를 눈으로 보이게 해줘요.
가독성코드를 읽기 쉬운 정도. 주석, 들여쓰기, 좋은 이름이 가독성을 높여요.
K&R 스타일여는 중괄호를 같은 줄에 쓰는 스타일. C의 원조 Kernighan & Ritchie가 사용.
Allman 스타일여는 중괄호를 새 줄에 쓰는 스타일. Visual Studio 기본.
clang-formatC/C++ 코드 자동 포맷팅 도구. .clang-format으로 스타일 설정.
Doxygen소스 코드 주석에서 API 문서를 자동 생성하는 도구.
TODO/FIXME코드 내 작업 표시. TODO=나중에 할 것, FIXME=버그 수정 필요.
EditorConfig다양한 에디터에서 코딩 스타일을 통일하는 설정 파일.
