주제 1: 자판기

[분석] 자판기 문제 분해 입력: 돈, 음료 번호
처리: 금액 비교, 재고 확인, 거스름돈 계산
출력: 음료 배출, 거스름돈 반환, 메시지 출력
조건: 돈이 충분한가? 재고가 있는가?

자판기를 단계별로 분해하면 이렇게 돼요:
1. 돈을 넣는다
2. 넣은 금액이 음료 가격 이상인지 확인한다
3. 부족하면 → '돈이 부족해요' 메시지 → 돈 더 넣기
4. 충분하면 → 음료를 선택한다
5. 재고가 있는지 확인한다
6. 없으면 → '다른 음료를 선택하세요' → 다시 선택
7. 있으면 → 음료를 배출한다
8. 거스름돈 = 넣은 돈 - 가격 → 거스름돈 반환
9. '감사합니다' 출력

[의사코드] 자판기 프로그램 자판기
  반복:
    입력: 돈
    만약 돈 < 가격:
      출력: '돈이 부족해요'

    입력: 음료번호

      출력: '다른 음료를 선택하세요'
      음료 선택으로 돌아가기
    음료 배출하기
    거스름돈 = 돈 - 가격
    거스름돈 반환하기
    출력: '감사합니다!'

주제 2: 신호등

신호등은 반복(루프)의 완벽한 예시예요. 초록 → 노랑 → 빨강 → 초록... 끝없이 반복하죠.

[분석] 신호등 문제 분해 입력: 없음 (자동으로 작동)
처리: 각 색 유지 시간 카운트
출력: 현재 신호 색
패턴: 초록(30초) → 노랑(5초) → 빨강(20초) → 초록... (무한 반복)
[의사코드] 신호등 프로그램 신호등
  신호 = 초록
  무한반복:
    만약 신호 == 초록:
      초록 불 켜기
      30초 기다리기
      신호 = 노랑

      노랑 불 켜기
      5초 기다리기
      신호 = 빨강
    만약 신호 == 빨강:

      신호 = 초록

[심화 내용] TDD란? Test-Driven Development:
테스트를 먼저 작성하고, 그 다음 코드를 구현!

3단계 사이클:
1. 🔴 Red: 실패하는 테스트 작성
2. 🟢 Green: 테스트를 통과하는 최소 코드 작성
3. 🔵 Refactor: 코드 정리 (기능 유지)

C에서 TDD:
assert(add(3, 5) == 8); // 먼저 테스트
// 그 다음 add() 함수 구현!

장점:
- 버그 예방 (테스트가 안전망)
- 설계 개선 (테스트하기 좋은 코드 = 좋은 설계)
- 자신감 (리팩토링 후 테스트 통과 = 안심)
