불(bool)이 뭘까?

안녕! 오늘은 참과 거짓을 배울 거야. 컴퓨터는 모든 판단을 '맞아' 아니면 '아니야'로 해. 이걸 불(bool)이라고 해!

우리가 매일 하는 판단을 생각해 봐요. '오늘 비가 와?' → 네/아니요. '나이가 13살 이상이야?' → 네/아니요. 이렇게 네(True) 아니면 아니요(False)로 대답할 수 있는 것을 프로그래밍에서는 불(bool)이라고 해요.

'불'이라는 이름은 영국 수학자 조지 불(George Boole)에게서 따왔어요. 참과 거짓으로 논리를 계산하는 '불 대수'를 만든 사람이에요.

1854

비교 연산의 결과: True 또는 False
[주의] =와 ==의 차이
= — 대입 (값을 넣어라!)
x = 10 → x에 10을 저장

== — 비교 (같니?)
x == 10 → x가 10과 같으면 True

이 둘을 헷갈리면 큰일 나요! = 하나는 저장, == 두 개는 비교.
✕ 잘못된 코드
if x = 10: # SyntaxError!
= 하나는 대입이에요. 비교가 아니에요!
✓ 올바른 코드
if x == 10: # OK!
== 두 개로 비교해야 해요
C언어에서도 =와 ==를 헷갈리는 게 초보자의 최대 실수였는데, 파이썬도 마찬가지예요. 비교할 때는 반드시 ==이에요!

논리 연산자 — 조건 합치기

비교 연산자로 하나의 조건을 판단할 수 있었어요. 그런데 '나이가 13살 이상이고 19살 이하인가?'처럼 여러 조건을 합쳐야 할 때가 있어요. 이때 쓰는 게 논리 연산자예요.

논리 연산자 — and, or, not 진리표
[핵심] 논리 연산자 3개
A and B — A와 B 둘 다 True여야 True (그리고)
A or B — A나 B 하나만 True여도 True (또는)
not A — A의 반대. True→False, False→True (부정)

C언어: && || !
파이썬: and or not (영어 단어!)

C언어와 뭐가 다를까?

✕ 잘못된 코드
// C언어 int x = 1; // 참 int y = 0; // 거짓 if (x && !y) { ... }
C: 1=참, 0=거짓, &&, ||, !
✓ 올바른 코드
# 파이썬 x = True y = False if x and not y: ...
파이썬: True/False, and, or, not
[핵심] C언어 vs 파이썬 논리 연산
C언어
&& (and), || (or), ! (not)
참=1, 거짓=0, bool 타입 없음(C99부터 _Bool)

파이썬
and, or, not (영어 단어!)
True / False (bool 타입 있음)

파이썬이 더 직관적이고 읽기 쉬워요!

✓ 올바른 코드

오차 범위 안에 있는지 비교!

[주의] 실수(float) 비교의 함정

[심화] Truthy와 Falsy

파이썬에서는 True/False뿐만 아니라, 다른 값들도 참 또는 거짓으로 취급돼요. 이걸 Truthy(참처럼)와 Falsy(거짓처럼)라고 해요.

Truthy vs Falsy 기준표
truthy_falsy.py
1# Falsy 값들 — bool()로 확인
2print(bool(0)) # False
3print(bool("")) # False (빈 문자열)
4print(bool(None)) # False
5print(bool([])) # False (빈 리스트)
6
7# Truthy 값들
8print(bool(1)) # True
9print(bool(-5)) # True (0이 아닌 숫자)
10print(bool("Hello")) # True (비어있지 않은 문자열)
11print(bool([1, 2])) # True (내용이 있는 리스트)

False False False False True True True True
[핵심] Falsy 값 외우기
Falsy(거짓 취급)는 딱 6가지만 외우면 돼요:

False, 0, 0.0, ""(빈 문자열), None, []/()/{}(빈 컬렉션)

나머지는 전부 Truthy(참 취급)!

단축 평가 (Short-circuit Evaluation)

and와 or는 결과가 확정되면 나머지를 계산하지 않아요. 이걸 단축 평가라고 해요.

short_circuit.py
1# and: 첫 번째가 False면 두 번째는 안 봐요
2print(False and print("안 실행!")) # False
3
4# or: 첫 번째가 True면 두 번째는 안 봐요
5print(True or print("안 실행!")) # True
6
7# 실전 활용: 0으로 나누기 방지
8x = 0
9result = x != 0 and 10 / x # x가 0이면 나누기 안 함!
10print(result) # False (10/x 계산 안 함)
실행 결과
False True False
단축 평가는 지금 당장 외울 필요는 없어요. 나중에 조건문을 배우면 자연스럽게 이해될 거예요.

백준 2480 — 주사위 세개
문제: 주사위 3개를 던져서 상금을 계산해요.
- 같은 눈 3개: 10000 + 눈 × 1000
- 같은 눈 2개: 1000 + 눈 × 100
- 모두 다른 눈: 가장 큰 눈 × 100

입력: 세 개의 주사위 눈 (1~6)
출력: 상금
boj_2480.py
1# 백준 2480 — 주사위 세개

3
4# 비교 연산자로 같은 눈 판별
5if a == b == c: # ← 비교 체이닝! 셋 다 같으면
6 print(10000 + a * 1000)
7elif a == b or a == c: # a가 다른 하나와 같으면

9elif b == c: # b와 c가 같으면

12 print(max(a, b, c) * 100)

이 문제의 핵심은 a == b == c(비교 체이닝)와 a == b or a == c(or 논리 연산)예요. 이번 유닛에서 배운 비교+논리 연산이 실전에서 어떻게 쓰이는지 보여주는 대표 문제예요.

용어 정리

bool
참(True)과 거짓(False) 두 가지 값만 가지는 데이터 타입.
비교 연산자
두 값을 비교해서 True/False를 반환하는 연산자. ==, !=, <, >, <=, >=
논리 연산자
여러 조건을 합치는 연산자. and(그리고), or(또는), not(부정)
Truthy
bool()로 변환하면 True가 되는 값. 0이 아닌 숫자, 비어있지 않은 문자열 등.
Falsy
bool()로 변환하면 False가 되는 값. 0, "", None, [], {}, () 등.
단축 평가
and/or에서 결과가 확정되면 나머지를 계산하지 않는 것.
비교 체이닝
1 < x < 10처럼 비교를 연결하는 파이썬만의 문법.
is
두 변수가 같은 객체를 가리키는지 확인. ==와 다름.
